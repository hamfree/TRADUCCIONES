Ejercicios

El número 1 y 2 son calentamientos: clases regulares de Java, no servlets. El problema n° 3 es el objetivo por el que estás trabajando.

  1. Defina una clase llamada StatePair que almacena cadenas representando un nombre del estado (p.e., "California") y una abreviatura de estado (p.e., "CA", pero este bean no tendrá lógica -- simplemente almacena dos cadenas). Note que Eclipse tiene un atajo muy útil para crear constructores y métodos getter/setter en los beans. Empiece a escribir una clase como esta: 
    		package myPackage; 
		
		public class StatePair { 
			private String stateName,stateAbbreviation; 
		}

    Entonces, haga click derecho en la clase, vaya al sub-menu Source, y elija "Generate Constructor Using Fields". Eclipse hará entonces el constructor para ti (pero removerá el falso "super();" en la primera línea). Entonces, haga click derecho en la clase de nuevo, vaya al sub-menu Source, y elija "Generate Getters and Setters". Puede entonces hacer que Eclipse automáticamente cree getStateName, setStateName, getStateAbbreviation, y setStateAbbreviation. Dependiendo de su lógica, podría no querer los dos métodos setter, en cuyo caso puede decirle a Eclipse que haga sólo los métodos getter.

  2. Haga una clase con un método que toma el nombre de un estado de EEUU (p.e., "Maryland") y devolver la correspondiente abreviación de la oficina postal (p.e. "MD"). Podría empezar con un mapeo simple donde sólo hay uno o dos nombres de estado que sean soportados).
  3. Haga una aplicación donde el usuario puede introducir un nombre de estado de EEUU y obtiene la abreviación de la oficina postal. Empiece con un formulario donde el usuario puede introducir el nombre de estado. Envie el dato a un servlet que busque la correspondiente abreviatura. No olvide manejar los datos erróneos. El servlet crearía un bean StatePair, y si los datos son buenos, envíe el bean a una página de resultados para visualizarlos. Si los datos son erróneos, envía el usuario a una página de error. (Podría tener dos errores de páginas: una para los datos no introducidos y una para los nombres de estado desconocidos pero no perdidos). ¡Este es el ejercicio más importante en todo el servlet y el curso de JSP!
  4. Repita el problema 3, pero esta vez, si el mismo usuario vuelve al formulario de entrada en la misma sesión de navegación, el nombre de estado introducido previamente se debe completar automáticamente en el campo de texto.
