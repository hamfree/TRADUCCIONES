Seguimiento de Sesión

Ejercicios

  1. Use rastreo de la sesión al hacer un servlet que diga "Bienvenido a bordo" a los visitantes de primera vez (en una sesión de navegación) y "Bienvenido de vuelta" a visitantes que repitan. Si hizo esta misma tarea anteriormente con el API de Cookies, ¿era este servlet más dificil o más facil que su versión equivalente utilizando cookies explícitamente?
     
  2. Escriba un servlet que muestre los valores del Nombre, Apellidos y Dirección de correo electrónico introducidos como parámetros de petición. Pero, recuerde que los usuarios se lo dijeron en el pasado, y use los viejos valores si se han perdido los valores actuales. Así, si un parámetro se ha perdido y el cliente es un visitante de primera-vez, haga que el servlet muestre "Desconocido" para los valores perdidos. Si un parámetro se ha perdido y el cliente es un visitante que repite, haga que el servlet use los valores previamente introducidos para los valores perdidos. Esto será definitivamente más fácil que la versión que usa cookies explícitamente.
     
  3. Haga un servlet que imprima una lista de las URLs de las páginas que el usuario actual ha utilizado para enlazar a él (dentro de la sesión de navegación actual). Esto es, si un usuario ha seguido enlaces de hipertexto desde tres diferentes páginas al servlet, éste mostrará al usuario las URLs de esas tres páginas. Compruebe su servlet realizando un par de páginas web estáticas que enlacen a éste. Si se siente inspirado, modifique la aproximación básica de forma que almacene entradas repetidas; si el mismo usuario sigue un enlace desde la página 1 al servlet dos veces, el servlet mostrará la URL de la página sólo una vez en la lista. Empiece con mi clase ShowItems, y haga unos pequeños cambios para obtener los datos de una cabecera de la petición (¿cuál?) y cambie de doPost a doGet. Si usted no está familiarizado con las esstructuras de datos relacionadas con listas en Java, vea las notas en la siguiente página referente a la clase ArrayList.
     
  4. [Medio dificil, para el moderadamente inspirado]. El problema anterior rastreaba la página referente e ignoraba las entradas repetidas. En este problema, usted rastreará un parámetro de petición y contará las repeticiones. Haga un servlet que mantenga la pista del número de cada item que está siendo pedido. Por ejemplo, si el usuario pide yate, coche, libro, coche, el servlet mostrará algo como esto: - yate (2) - coche (1) - libro (1). Para simplificar el código, puede utilizar el nombre del item tal cual fue enviado del formulario HTML; no hay necesidad de comprobar cada nombre contra una tabla de nombres legales como en el mucho más complicado ejemplo de Cesta de la Compra en el libro.
