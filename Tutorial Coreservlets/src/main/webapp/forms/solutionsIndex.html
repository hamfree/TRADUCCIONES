Formularios

Índice de Soluciones

  1. Abra el proyecto "forms". Ejecute el servlet ThreeParams directamente vía http://localhost/forms/ three-params. Debería ver null para los tres parámetros. Ejecútelo de nuevo iniciando con threeparams-form.html y submitiendo el formulario. Debería ver ahora las entradas para los campos de texto.
  2. Haga un proyecto nuevo de Eclipse llamado exercises-forms (a algo parecido). Use este proyecto para el resto de los ejercicios. Copie el servlet ThreeParams desde el proyecto "forms" y colóquelo en un nuevo paquete/directorio en su proyecto. Similarmente, haga una copia de three-params-form.html. Pruebe el servlet y el formulario.
  3. Copie/renombre El servlet ThreeParams y su formulario asociado y haga las nuevas versiones usando POST en vez de GET. Note que tendrá que hacer pequeños cambios tanto al servlet como al formulario. Probablemente necesitará reiniciar Tomcat despues de realizar los cambios (Click con el botón derecho sobre Tomcat y seleccione "Reiniciar").
  4. Utilize three-params-form.html para enviar datos al servlet ThreeParams que contenga caracteres específicos de HTML. Verifique que esto puede causar resultados malformados. Haga una variación del servlet que filtre las cadenas antes de mostrarlas. Asegúrese de copiar ServletUtilities del proyecto "forms" o "test-app" primero, de forma que pueda simplemente hacer esto: 
    	 someString = ServletUtilities.filter(someString);
  5. Haga un formulario "registro" que recoja nombre, apellidos y dirección de correo electrónico. Envíe los datos a un servlet que los muestre. Siéntase libre de modificar el formulario HTML ThreeParams y el servlet para llevar a cabo esto. A continuación, modifique el servlet para que utilize un valor por defecto (o de un mensaje de error) si el usuario omite cualquiera de los tres parámetros requeridos.
  6. [Dificil; para el realmente inspirado] Haga una variación del servlet anterior y su formulario acompañante, pero, en este caso, si el usuario falla en dar un valor para cualquiera de los campos en el formulario, debería redibujar el formulario pero con un mensaje de error diciendo que el valor se ha omitido. No haga a los usuarios reintroducir valores que hayan introducido ya. Consejo: Tiene dos opciones dadas las herramientas disponibles hasta ahora para nosotros : tener un gran servlet que genere tanto el formulario HTML y la página de resultados ( el formulario se submite los datos a sí mismo), o dos servlets separados (uno que genera el formulario HTML y otro que genera la página de resultados). Hay un ejemplo de la primera aproximación en el libro. Si quiere intentar la segunda aproximación, querrá saber sobre response.sendRedirect(), el cual le permite redirigir al navegaor a la página especificada. Por ejemplo, aquí tenemos un método doGet que envía al usuario a www.whitehouse.gov: 
    	public void doGet(HttpServletRequest request,HttpServletResponse response)
		throws ServletException, IOException {
		response.sendRedirect("http://www.whitehouse.gov/");
	}
