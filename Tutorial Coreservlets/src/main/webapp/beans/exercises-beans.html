Usando Componentes JavaBeans en Documentos JSP

Ejercicios

Para estos ejercicios, evite cualquier código Java en las páginas JSP.

  1. Defina una clase llamada ColorBean que almacene cadenas representando un color de primer plano y un color de fondo. Compile y pruebelo de una manera independiente (p.e., sin usar un servlet o página JSP). Recuerde que en Eclipse, si su clase tiene un método "main", puede hacer click derecho en cualquier parte del fichero, elegir "Ejecutar Como --> Aplicación Java", y Eclipse ejecutará el método main y colocará la salida en la consola en la parte inferior. Además, tenga en cuenta que Eclipse tiene un atajo sencillo para crear métodos getter y setter en beans. Comience por escribir una clase como esta: 
    package myPackage;
 
public class ColorBean { 
    private String foregroundColor = "black"; 
    private String backgroundColor = "white"; 
}
    Entonces, haga click derecho en la clase, vaya al submenú "Source", y elija “Generate Getters and Setters”. Puede entonces hacer que Eclipse automáticamente crea getForegroundColor, setForegroundColor, getBackgroundColor, y setBackgroundColor.
  2. Haga un formulario de “preferencia de color” que recoja los colores preferidos del usuario para el texto y el fondo . Envíe los datos a una página JSP que visualize algún mensaje usando esos colores. Esta página JSP usará un valor por defecto para cualquier valor del formulario que el usuario no indique (pero no se preocupe por las cadenas vacías). Así, por ejemplo, si el usuario va directamente a la página JSP (eludiendo el formulario), la página JSP todavía debería funcionar bien. Por ahora, no se preocupe si el usuario le manda espacios en blanco; simplemente maneje los valores perdidos.
  3. Rehaga el ejemplo de preferencia de color, pero si el usuario falla en suministrar alguno de los colores, use cualquier valor que dió la última vez. Si no tiene valor previo, use uno por defecto. (Consejo: este problema tiene casi exactamente la misma dificultad que el anterior).
  4. Rehaga el ejemplo de preferencia de color, pero si el usuario falla en suministrar cualquiera de los parámetros, use cualquier color que el usuario más reciente dió la última vez. ¿Porque podría esto dar malos resultados?
  5. Solucione el problema nº 2 de forma que también maneje los espacios en blanco. Hágalo sin añadir ningún código Java explícito a la pagina JSP.
