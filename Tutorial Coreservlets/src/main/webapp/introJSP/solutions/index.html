  1. Haga un proyecto nuevo de Eclipse, tome OrderConfirmation.jsp del proyecto "jsp-intro", y cópielo en la carpeta WebContent de su nuevo proyecto. También copie el fichero JSPStyles.css que OrderConfirmation.jsp usa. Compare el tiempo que tarda el acceder la primera vez con el tiempo que tarda en las peticiones subsiguientes.

    Mire el código fuente de OrderConfirmation.jsp. Vea si puede figurarse cómo hace su salida "Gracias por pedir un yate." en vez de "Gracias por pedir null". Pista: no necesita realizar ningún cambio a la página JSP.

  2. Si usted quisiera un servlet para construir una página web que muestre una imagen, ¿dónde pondría usted la imagen y cómo se referiría a ella desde el servlet? ¿Como difiere esto de un JSP? (Esta cuestión es mayormente fácil, pero aun así ligeramente más complicada de lo que suena al principio).
  3. Haga un página HTML estática muy simple. Si usted quiere, descargue una de la Web o use alguna de un ejercicio previo. Renómbrela para que tenga una extensión .jsp. Tendrá un formulario HTML con nada más que un botón SUBMIT que dirije al usuario a la página estática.

    Ahora, cambie el formulario para que use POST en vez de GET. ¿Porqué esto funciona en ambas ocasiones? (Respuesta: el el código auto-generado no puede estar en el método doGet. ¿Dónde va el código autogenerado, entonces? )
