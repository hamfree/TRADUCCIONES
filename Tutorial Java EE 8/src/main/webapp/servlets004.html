Creando e Inicializando un Servlet

Utilice la anotación @WebServlet para definir un componente de servlet en una aplicación web. Esta anotación se especifica en una clase y contiene metadatos sobre el servlet que se declara. El servlet anotado debe especificar al menos un patrón de URL. Esto se hace usando el atributo urlPatterns o value en la anotación. Todos los demás atributos son opcionales, con configuraciones predeterminadas. Use el atributo value cuando el único atributo en la anotación es el patrón de URL; de lo contrario, utilice el atributo urlPatterns cuando también se utilicen otros atributos.

Las clases anotadas con @WebServlet deben extender la clase javax.servlet.http.HttpServlet. Por ejemplo, el siguiente fragmento de código define un servlet con el patrón de URL /report: 

              
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
@WebServlet("/report")
public class MoodServlet extends HttpServlet {
    ...

El contenedor web inicializa un servlet después de cargar e instanciar la clase de servlet y antes de entregar las solicitudes de los clientes. Para personalizar este proceso para permitir que el servlet lea datos de configuración persistentes, inicialice recursos y realice cualquier otra actividad única, puede anular el método init del Servlet o especifique el atributo initParams de la anotación @WebServlet. El atributo initParams contiene una anotación @WebInitParam. Si no puede completar su proceso de inicialización, un servlet arroja una UnavailableException.

Utilice un parámetro de inicialización para proporcionar los datos que necesita un servlet en particular. Por el contrario, un parámetro de contexto proporciona datos que están disponibles para todos los componentes de una aplicación web.
