Configuración de la Seguridad Mediante Descriptores de Implementación

La forma recomendada de configurar la seguridad en la plataforma Java EE 8 es con anotaciones. Si desea anular la configuración de seguridad en el momento de la implementación, puede usar elementos de seguridad en el descriptor de implementación web.xml para hacerlo. Esta sección describe cómo usar el descriptor de implementación para especificar la autenticación básica y anular la asignación predeterminada de principal a rol.

Aquí se tratan los siguientes temas:

  • Especificación de Seguridad para la Autenticación Básica en el Descriptor de Implementación

  • Especificación de la Asignación no Predeterminada de la Asociación Principal-to-Role en el Descriptor de Implementación

Especificación de Seguridad para la Autenticación Básica en el Descriptor de Implementación

Los elementos del descriptor de implementación que agregan autenticación básica a un ejemplo le indican al servidor o navegador que realice las siguientes tareas.

  • Enviar un cuadro de diálogo de inicio de sesión estándar para recopilar datos de nombre de usuario y contraseña.

  • Verificar que el usuario esté autorizado para acceder a la aplicación.

  • Si está autorizado, muestra el servlet al usuario.

El siguiente código de muestra muestra los elementos de seguridad para un descriptor de implementación que podría usarse en el ejemplo de autenticación básica que se encuentra en el directorio tut-install`/examples/security/hello2_basicauth/`: 

                
    <security-constraint>
        <display-name>SecurityConstraint</display-name>
        <web-resource-collection>
             <web-resource-name>WRCollection</web-resource-name>
            <url-pattern>/greeting</url-pattern>
        </web-resource-collection>
        <auth-constraint>
            <role-name>TutorialUser</role-name>
        </auth-constraint>
        <user-data-constraint>
             <transport-guarantee>CONFIDENTIAL</transport-guarantee>
        </user-data-constraint>
    </security-constraint>
    <login-config>
        <auth-method>BASIC</auth-method>
        <realm-name>file</realm-name>
    </login-config>
    <security-role>
        <role-name>TutorialUser</role-name>
    </security-role>

Este descriptor de implementación especifica que solo los usuarios que hayan ingresado sus nombres de usuario y contraseñas y que hayan sido autorizados para acceder a esta URL porque se verificó que tienen el rol TutorialUsuario. Los datos de usuario y contraseña serán enviados por transporte protegido para evitar su lectura en tránsito.

Especificación de la Asignación no Predeterminada de la Asociación Principal-to-Role en el Descriptor de Implementación

Para asignar un nombre de rol permitido por la aplicación o módulo a principales (usuarios) y grupos definidos en el servidor, use el elemento security-role-mapping en el archivo descriptor de implementación en tiempo de ejecución (glassfish- application.xml, glassfish-web.xml o glassfish-ejb-jar.xml). La entrada debe declarar una asignación entre un rol de seguridad utilizado en la aplicación y uno o más grupos o principales definidos para el dominio aplicable del servidor GlassFish. A continuación se muestra un ejemplo del archivo glassfish-web.xml: 

                
<glassfish-web-app>
    <security-role-mapping>
        <role-name>DIRECTOR</role-name>
        <principal-name>schwartz</principal-name>
    </security-role-mapping>
    <security-role-mapping>
        <role-name>DEPT-ADMIN</role-name>
        <group-name>dept-admins</group-name>
    </security-role-mapping>
</glassfish-web-app>

El nombre del rol se puede asignar a un principal (usuario) específico, a un grupo o a ambos. Los nombres principales o de grupo a los que se hace referencia deben ser principales o grupos válidos en el dominio predeterminado actual del servidor GlassFish. El role-name en este ejemplo debe coincidir exactamente con el role-name en el elemento security-role del web.xml correspondiente o el nombre del rol definido en las anotaciones @DeclareRoles y/o @RolesAllowed.
