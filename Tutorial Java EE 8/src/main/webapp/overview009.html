IPAs de Java EE 8 en la Plataforma Java, Edición Estándar 8

Varias IPA requeridas por la plataforma Java EE 8 se incluyen en Plataforma Java, Standard Edition 8 (Java SE 8) y, por lo tanto, están disponibles para las aplicaciones Java EE.

IPA de la Conectividad a Bases de Datos de Java

La IPA de Java Database Connectivity (JDBC) le permite invocar comandos SQL desde métodos de lenguaje de programación Java. Utilice la IPA de JDBC en un enterprise bean cuando un bean de sesión acceda a la base de datos. También puede utilizar la IPA de JDBC desde un servlet o una página JSP para acceder a la base de datos directamente sin pasar por un enterprise bean.

La IPA de JDBC tiene dos partes:

  • Una interfaz de nivel de aplicación utilizada por los componentes de la aplicación para acceder a una base de datos.

  • Una interfaz de proveedor de servicios para conectar un controlador JDBC a la plataforma Java EE.

La plataforma Java EE 8 requiere JDBC 4.1.

IPA de la Interfaz de Nombrado y Directorio de Java

La IPA de Java Naming and Directory Interface (JNDI) proporciona funciones de nombres y directorios, lo que permite que las aplicaciones accedan a varios servicios de nombres y directorios, como LDAP, DNS y NIS. La IPA JNDI proporciona a las aplicaciones métodos para realizar operaciones de directorio estándar, como asociar atributos con objetos y buscar objetos utilizando sus atributos. Con JNDI, una aplicación Java EE puede almacenar y recuperar cualquier tipo de objeto Java con nombre, lo que permite que las aplicaciones Java EE coexistan con muchas aplicaciones y sistemas heredados.

Los servicios de nombres Java EE proporcionan a los clientes de aplicaciones, beans empresariales y componentes web acceso a un entorno de nombres JNDI. Un entorno de nomenclatura permite personalizar un componente sin necesidad de acceder o cambiar el código fuente del componente. Un contenedor implementa el entorno del componente y lo proporciona al componente como un contexto de nomenclatura JNDI.

El entorno de nombres proporciona cuatro espacios de nombres lógicos: java:comp, java:module, java:app y java:global para objetos disponibles para componentes, módulos o aplicaciones o compartidos por todas las aplicaciones implementadas. Un componente Java EE puede acceder a objetos definidos por el usuario y proporcionados por el sistema con nombre. Los nombres de algunos objetos proporcionados por el sistema, como un objeto JDBC DataSource predeterminado, una fábrica de conexiones JMS predeterminada y un objeto JTA UserTransaction, se almacenan en el espacio de nombres java:comp. La plataforma Java EE permite que un componente nombre objetos definidos por el usuario, como beans de empresa, entradas de entorno, objetos JDBC DataSource y destinos de mensajería.

Un componente Java EE también puede localizar su contexto de denominación de entorno mediante interfaces JNDI. Un componente puede crear un objeto javax.naming.InitialContext y buscar el contexto de nomenclatura del entorno en InitialContext con el nombre java:comp/env. El entorno de denominación de un componente se almacena directamente en el contexto de denominación del entorno o en cualquiera de sus subcontextos directos o indirectos.

Marco de Trabajo de Activación de JavaBeans

El marco de activación de JavaBeans (JAF) es utilizado por LA IPA de JavaMail. JAF proporciona servicios estándar para determinar el tipo de un dato arbitrario, encapsular el acceso al mismo, descubrir las operaciones disponibles en él y crear el componente JavaBean apropiado para realizar esas operaciones.

IPA de Java para el Procesado de XML

La IPA de Java para procesamiento de XML (JAXP), que forma parte de la plataforma Java SE, admite el procesamiento de documentos XML mediante el modelo de objetos de documento (DOM), la IPA simple para XML (SAX) y las transformaciones de lenguaje de hoja de estilo extensible (XSLT). JAXP permite a las aplicaciones analizar y transformar documentos XML independientemente de una implementación de procesamiento XML en particular.

JAXP también proporciona soporte de espacio de nombres, lo que le permite trabajar con esquemas que de otro modo podrían tener conflictos de nombres. Diseñado para ser flexible, JAXP le permite utilizar cualquier analizador compatible con XML o procesador XSL desde su aplicación y es compatible con el esquema del Worldwide Web Consortium (W3C). Puede encontrar información sobre el esquema W3C en http://www.w3.org/XML/Schema .

Arquitectura Java para el Enlazado XML

La arquitectura Java para enlaces XML (JAXB) proporciona una forma conveniente de enlazar un esquema XML a una representación en programas de lenguaje Java. JAXB se puede utilizar de forma independiente o en combinación con JAX-WS, en cuyo caso proporciona un enlace de datos estándar para los mensajes del servicio web. Todos los contenedores de cliente de aplicación Java EE, contenedores web y contenedores EJB admiten la API de JAXB.

La plataforma Java EE 8 requiere JAXB 2.2.

IPA de Java para los Servicios Web XML

La especificación Java API for XML Web Services (JAX-WS) proporciona soporte para servicios web que utilizan la API JAXB para vincular datos XML a objetos Java. La especificación JAX-WS define las IPA de cliente para acceder a servicios web, así como técnicas para implementar puntos finales de servicios web. La especificación Implementación de servicios web empresariales describe la implementación de clientes y servicios basados en JAX-WS. Las especificaciones de EJB y Java Servlet también describen aspectos de dicha implementación. Las aplicaciones basadas en JAX-WS se pueden implementar utilizando cualquiera de estos modelos de implementación.

La especificación JAX-WS describe el soporte para manejadores de mensajes que pueden procesar solicitudes y respuestas de mensajes. En general, estos manejadores de mensajes se ejecutan en el mismo contenedor y con los mismos privilegios y contexto de ejecución que el cliente JAX-WS o el componente de punto final con el que están asociados. Estos manejadores de mensajes tienen acceso al mismo espacio de nombres JNDI que su componente asociado. Los serializadores y deserializadores personalizados, si se admiten, se tratan de la misma forma que los controladores de mensajes.

La plataforma Java EE 8 requiere JAX-WS 2.2.

IPA de SOAP con Adjuntos para Java

La API SOAP con archivos adjuntos para Java (SAAJ) es una API de bajo nivel de la que depende JAX-WS. SAAJ permite la producción y consumo de mensajes que cumplen con las especificaciones SOAP 1.1 y 1.2 y la nota SOAP con adjuntos. La mayoría de los desarrolladores no utilizan la API SAAJ, sino que utilizan la API JAX-WS de nivel superior.

Servicio de Autenticación y Autorización de Java

El Servicio de autenticación y autorización de Java (JAAS) proporciona una forma para que una aplicación Java EE autentique y autorice a un usuario o grupo de usuarios específico a ejecutarla.

JAAS es una versión en lenguaje de programación Java del marco estándar Pluggable Authentication Module (PAM), que amplía la arquitectura de seguridad Plataforma Java para admitir la autorización basada en el usuario.

Anotaciones Comunes para la Plataforma Java

Las anotaciones permiten un estilo declarativo de programación en la plataforma Java.

La plataforma Java EE 8 requiere anotaciones comunes para la Plataforma Java 1.2.
