Envío de Trabajos al Tiempo de Ejecución de Batch

La interfaz JobOperator del paquete javax.batch.operations le permite enviar trabajos al tiempo de ejecución por lotes y obtener información sobre los trabajos existentes. Esta interfaz proporciona la siguiente funcionalidad.

  • Obtenga los nombres de todos los trabajos conocidos.

  • Iniciar, detener, reiniciar y abandonar trabajos.

  • Obtener instancias de trabajos y ejecuciones de trabajos.

La clase BatchRuntime del paquete javax.batch.runtime proporciona el método de fábrica getJobOperator para obtener objetos JobOperator.

Comenzando un Trabajo

El siguiente código de ejemplo muestra cómo obtener un objeto JobOperator y enviar un trabajo por lotes: 

                
JobOperator jobOperator = BatchRuntime.getJobOperator();
Properties props = new Properties();
props.setProperty("parameter1", "value1");
...
long execID = jobOperator.start("simplejob", props);

El primer argumento del método JobOperator.start es el nombre del trabajo tal como se especifica en su archivo de definición de trabajo. El segundo parámetro es un objeto Properties que representa los parámetros para la ejecución de este trabajo. Puede usar parámetros de trabajo para pasar a un trabajo información que solo se conoce en tiempo de ejecución.

Comprobación del Estado de un Trabajo

La interfaz JobExecution del paquete javax.batch.runtime proporciona métodos para obtener información sobre los trabajos enviados. Esta interfaz proporciona la siguiente funcionalidad.

  • Obtener el estado de lote y salida de la ejecución de un trabajo.

  • Obtener la hora en que se inició, actualizó o finalizó la ejecución.

  • Obtener el nombre del trabajo.

  • Obtener el ID de ejecución.

El siguiente código de ejemplo muestra cómo obtener el estado del lote de un trabajo utilizando su ID de ejecución: 

                
JobExecution jobExec = jobOperator.getJobExecution(execID);
String status = jobExec.getBatchStatus().toString();

Invocación del Tiempo de Ejecución Batch en su Aplicación

El componente desde el que invoca el tiempo de ejecución por lotes depende de la arquitectura de su aplicación en particular. Por ejemplo, puede invocar el tiempo de ejecución por lotes desde un bean empresarial, un servlet, un bean gestionado, etc.

Consulte La Aplicación de Ejemplo webserverlog y La Aplicación de Ejemplo phonebilling para obtener más información. sobre cómo invocar el tiempo de ejecución por lotes desde un bean administrado controlado por una interfaz de usuario de JavaServer Faces.
