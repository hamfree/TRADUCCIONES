Sintaxis de Evaluación Inmediata y Diferida

El EL apoya la evaluación inmediata y diferida de expresiones. La evaluación inmediata significa que la expresión se evalúa y el resultado se devuelve tan pronto como la página se representa por primera vez. La evaluación diferida significa que la tecnología que usa el lenguaje de expresión puede usar su propia maquinaria para evaluar la expresión en algún momento posterior durante el ciclo de vida de la página, siempre que sea apropiado hacerlo.

Las expresiones que se evalúan inmediatamente utilizan la sintaxis ${}. Las expresiones cuya evaluación se aplaza utilizan la sintaxis #{}.

Debido a su ciclo de vida multifásico, la tecnología JavaServer Faces utiliza principalmente expresiones de evaluación diferidas. Durante el ciclo de vida, se manejan los eventos de los componentes, se validan los datos y se realizan otras tareas en un orden particular. Por lo tanto, una implementación de JavaServer Faces debe posponer la evaluación de expresiones hasta el punto apropiado del ciclo de vida.

Otras tecnologías que usan EL pueden tener diferentes razones para usar expresiones diferidas.

Evaluación Inmediata

Todas las expresiones que utilizan la sintaxis ${} se evalúan inmediatamente. Estas expresiones pueden aparecer como parte de un texto de plantilla (estático) o como el valor de un atributo de etiqueta que puede aceptar expresiones en tiempo de ejecución.

El siguiente ejemplo muestra una etiqueta cuyo atributo value hace referencia a una expresión de evaluación inmediata que actualiza la cantidad de libros recuperados del bean de respaldo llamado catalog: 

                
<h:outputText value="${catalog.bookQuantity}" />

La implementación de JavaServer Faces evalúa la expresión ${catalog.bookQuantity}, la convierte y pasa el valor devuelto al controlador de etiquetas. El valor se actualiza en la página.

Evaluación Diferida

Las expresiones de evaluación diferidas toman la forma #{expr} y se pueden evaluar en otras fases del ciclo de vida de una página según lo definido por cualquier tecnología que utilice la expresión. En el caso de la tecnología JavaServer Faces, su controlador puede evaluar la expresión en diferentes fases del ciclo de vida, dependiendo de cómo se utilice la expresión en la página.

El siguiente ejemplo muestra una etiqueta h:inputText de JavaServer Faces, que representa un componente de campo en el que un usuario ingresa un valor. El atributo value de la etiqueta h:inputText hace referencia a una expresión de evaluación diferida que apunta a la propiedad name del bean customer: 

                
<h:inputText id="name" value="#{customer.name}" />

Para una solicitud inicial de la página que contiene esta etiqueta, la implementación de JavaServer Faces evalúa la expresión #{customer.name} durante la fase de respuesta de procesamiento del ciclo de vida. Durante esta fase, la expresión simplemente accede al valor de name del bean customer, como se hace en la evaluación inmediata.

Para una solicitud de devolución, la implementación de JavaServer Faces evalúa la expresión en diferentes fases del ciclo de vida, durante las cuales el valor se recupera de la solicitud, se valida y se propaga al bean customer.

Como se muestra en este ejemplo, las expresiones de evaluación diferida pueden ser

  • Expresiones de valor que se pueden usar para leer y escribir datos

  • Expresiones de método
