Introducción a JSON

JSON es un formato de intercambio de datos basado en texto derivado de JavaScript que se utiliza en servicios web y otras aplicaciones conectadas. Las siguientes secciones proporcionan una introducción a la sintaxis JSON, una descripción general de los usos de JSON y una descripción de los enfoques más comunes para generar y analizar JSON.

Aquí se tratan los siguientes temas:

  • Sintaxis JSON

  • Usos de JSON

  • Generando y Analizando Datos JSON

Sintaxis JSON

JSON define solo dos estructuras de datos: objetos y matrices. Un objeto es un conjunto de pares de nombre-valor y una matriz es una lista de valores. JSON define siete tipos de valores: string, number, object, array, true, false y null.

El siguiente ejemplo muestra datos JSON para un objeto de muestra que contiene pares de nombre y valor. El valor del nombre "phoneNumbers" es una matriz cuyos elementos son dos objetos. 


{
   "firstName": "Duke",
   "lastName": "Java",
   "age": 18,
   "streetAddress": "100 Internet Dr",
   "city": "JavaTown",
   "state": "JA",
   "postalCode": "12345",
   "phoneNumbers": [
      { "Mobile": "111-111-1111" },
      { "Home": "222-222-2222" }
   ]
}

JSON tiene la siguiente sintaxis.

  • Los objetos están encerrados entre llaves ({}), sus pares de nombre-valor están separados por una coma (,), y el nombre y el valor de un par están separados por un dos puntos (:). Los nombres de un objeto son cadenas, mientras que los valores pueden ser de cualquiera de los siete tipos de valor, incluido otro objeto o una matriz.

  • Las matrices están encerradas entre corchetes ([]) y sus valores están separados por una coma (,). Cada valor de una matriz puede ser de un tipo diferente, incluida otra matriz o un objeto.

  • Cuando los objetos y las matrices contienen otros objetos o matrices, los datos tienen una estructura en forma de árbol.

Usos de JSON

JSON se usa a menudo como un formato común para serializar y deserializar datos en aplicaciones que se comunican entre sí a través de Internet. Estas aplicaciones se crean utilizando diferentes lenguajes de programación y se ejecutan en entornos muy diferentes. JSON se adapta a este escenario porque es un estándar abierto, es fácil de leer y escribir y es más compacto que otras representaciones.

Los servicios web RESTful utilizan JSON ampliamente como formato para los datos dentro de las solicitudes y respuestas. El encabezado HTTP utilizado para indicar que el contenido de una solicitud o una respuesta son datos JSON es 


Content-Type: application/json

Las representaciones JSON suelen ser más compactas que las representaciones XML porque JSON no tiene etiquetas de cierre. A diferencia de XML, JSON no tiene un esquema ampliamente aceptado para definir y validar la estructura de los datos JSON.

Generando y Analizando Datos JSON

Para generar y analizar datos JSON, existen dos modelos de programación, que son similares a los que se utilizan para los documentos XML.

  • El modelo de objetos crea un árbol que representa los datos JSON en la memoria. A continuación, se puede navegar, analizar o modificar el árbol. Este enfoque es el más flexible y permite un procesamiento que requiere acceso al contenido completo del árbol. Sin embargo, a menudo es más lento que el modelo de transmisión y requiere más memoria. El modelo de objetos genera una salida JSON navegando por todo el árbol a la vez.

  • El modelo de transmisión usa un analizador basado en eventos que lee los datos JSON un elemento a la vez. El analizador genera eventos y se detiene para procesar cuando un objeto o una matriz comienza o termina, cuando encuentra una clave o cuando encuentra un valor. Cada elemento puede ser procesado o descartado por el código de la aplicación, y luego el analizador pasa al siguiente evento. Este enfoque es adecuado para el procesamiento local, en el que el procesamiento de un elemento no requiere información del resto de los datos. El modelo de transmisión genera una salida JSON a una transmisión determinada al realizar una llamada de función con un elemento a la vez.

Hay muchos generadores y analizadores JSON disponibles para diferentes lenguajes y entornos de programación. Procesamiento JSON en la plataforma Java EE describe la funcionalidad proporcionada por IPA de Java para procesamiento JSON (JSR 374).
