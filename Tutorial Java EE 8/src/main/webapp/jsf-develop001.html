Beans Gestionados en la Tecnología JavaServer Faces

Una aplicación típica de JavaServer Faces incluye uno o más beans gestionados, cada uno de los cuales puede asociarse con los componentes utilizados en una página en particular. Esta sección presenta los conceptos básicos de creación, configuración y uso de beans gestionados en una aplicación.

Note:

El Capítulo 10, "Uso de Tecnología JavaServer Faces en páginas web" y el Capítulo 11, "Uso de convertidores, oyentes y validadores" muestran cómo agregar componentes a una página y conectarlos a objetos del lado del servidor mediante etiquetas de componentes y etiquetas principales. Estos capítulos también muestran cómo proporcionar funcionalidad adicional a los componentes a través de convertidores, oyentes y validadores. El desarrollo de una aplicación JavaServer Faces también implica la tarea de programar los objetos del lado del servidor: beans gestionados, convertidores, controladores de eventos y validadores.

Aquí se tratan los siguientes temas:

  • Creando un Bean Gestionado

  • Usando el EL para Referenciar Beans Gestionados

Creando un Bean Gestionado

Un bean gestionado se crea con un constructor sin argumentos, un conjunto de propiedades y un conjunto de métodos que realizan funciones para un componente. Cada una de las propiedades del bean gestionado se puede vincular a una de las siguientes:

  • Un valor de componente

  • Una instancia de componente

  • Una instancia de convertidor

  • Una instancia de oyente

  • Una instancia de validador

Las funciones más comunes que realizan los métodos de los beans gestionados incluyen las siguientes:

  • Validar los datos de un componente

  • Manejo de un evento desencadenado por un componente

  • Realizar el procesamiento para determinar la siguiente página a la que debe navegar la aplicación

Al igual que con todos los componentes de JavaBeans, una propiedad consta de un campo de datos privados y un conjunto de métodos de acceso, como se muestra en este código: 


private Integer userNumber = null;
...
public void setUserNumber(Integer user_number) {
    userNumber = user_number;
}
public Integer getUserNumber() {
    return userNumber;
}

Cuando está vinculada al valor de un componente, una propiedad de bean puede ser cualquiera de los tipos primitivos y numéricos básicos o cualquier tipo de objeto Java para el cual la aplicación tiene acceso a un convertidor apropiado. Por ejemplo, una propiedad puede ser de tipo java.util.Date si la aplicación tiene acceso a un convertidor que puede convertir el tipo Date en una String y viceversa. Consulte Escribiendo Propiedades del Bean para obtener información sobre qué tipos son aceptados por qué etiquetas de componentes.

Cuando una propiedad de bean está vinculada a una instancia de componente, el tipo de propiedad debe ser el mismo que el objeto del componente. Por ejemplo, si un componente javax.faces.component.UISelectBoolean está vinculado a la propiedad, la propiedad debe aceptar y devolver un objeto UISelectBoolean. Del mismo modo, si la propiedad está vinculada a una instancia de conversor, validador o escucha, la propiedad debe ser del tipo de conversor, validador u oyente apropiado.

Para obtener más información sobre cómo escribir beans y sus propiedades, consulte Escribiendo Propiedades del Bean.

Usando el EL para Referenciar Beans Gestionados

Para vincular valores de componentes y objetos a propiedades de bean gestionado o para hacer referencia a métodos de bean gestionado desde etiquetas de componente, los autores de páginas utilizan la sintaxis del lenguaje de expresión. Como se explica en Descripción general del EL, las siguientes son algunas de las características que ofrece el EL:

  • Evaluación diferida de expresiones

  • La capacidad de usar una expresión de valor para leer y escribir datos.

  • Expresiones de método

La evaluación diferida de expresiones es importante porque el ciclo de vida de JavaServer Faces se divide en varias fases en las que el manejo de eventos de componentes, la conversión y validación de datos y la propagación de datos a objetos externos se realizan de manera ordenada. La implementación debe poder retrasar la evaluación de expresiones hasta que se haya alcanzado la fase adecuada del ciclo de vida. Por lo tanto, los atributos de etiqueta de la implementación siempre usan la sintaxis de evaluación diferida, que se distingue por el delimitador #{}.

Para almacenar datos en objetos externos, casi todos los atributos de etiquetas de JavaServer Faces usan expresiones lvalue, que son expresiones que permiten obtener y configurar datos en objetos externos.

Finalmente, algunos atributos de etiqueta de componente aceptan expresiones de método que hacen referencia a métodos que manejan eventos de componentes o validan o convierten datos de componentes.

Para ilustrar una etiqueta JavaServer Faces usando EL, la siguiente etiqueta hace referencia a un método que valida la entrada del usuario: 


<h:inputText id="inputGuess"
    value="#{userNumberBean.userNumber}"
    required="true" size="3"
    disabled="#{userNumberBean.number eq userNumberBean.userNumber ...}"
    validator="#{userNumberBean.validateNumberRange}">
</h:inputText>

Esta etiqueta vincula el valor del componente inputGuess a la propiedad de bean gestionada UserNumberBean.userNumber mediante el uso de una expresión lvalue. La etiqueta usa una expresión de método para referirse al método UserNumberBean.validateNumberRange, que realiza la validación del valor local del componente. El valor local es lo que el usuario escriba en el campo correspondiente a esta etiqueta. Este método se invoca cuando se evalúa la expresión.

Casi todos los atributos de etiqueta de JavaServer Faces aceptan expresiones de valor. Además de hacer referencia a las propiedades del bean, las expresiones de valor pueden hacer referencia a listas, mapas, matrices, objetos implícitos y paquetes de recursos.

Otro uso de las expresiones de valor es vincular una instancia de componente a una propiedad de bean gestionado. El autor de una página hace esto haciendo referencia a la propiedad del atributo binding: 


<h:outputLabel for="fanClub"
               rendered="false"
               binding="#{cashierBean.specialOfferText}">
               value="#{bundle.DukeFanClub}"/>
</h:outputLabel>

Además de utilizar expresiones con las etiquetas de componentes estándar, puede configurar las propiedades de sus componentes personalizados para aceptar expresiones creando instancias de javax.el.ValueExpression o javax.el.MethodExpression para ellos.

Para obtener información sobre EL, consulte el Capítulo 9, "Lenguaje de Expresión".

Para obtener información sobre cómo hacer referencia a métodos de bean gestionados desde etiquetas de componentes, consulte Referenciando un Método de un Bean Gestionado.
