Aplicaciones de Ejemplo para JAX-RS

Esta sección proporciona una introducción a la creación, implementación y ejecución de sus propias aplicaciones JAX-RS. Esta sección demuestra los pasos necesarios para crear, compilar, implementar y probar una aplicación web muy simple que utiliza anotaciones JAX-RS.

Aquí se tratan los siguientes temas:

  • Creando un Servicio Web RESTful Simple

  • La Aplicación de Ejemplo rsvp

  • Ejemplos del Mundo Real

Creando un Servicio Web RESTful Simple

Esta sección explica cómo usar el EID NetBeans para crear un servicio web RESTful usando un arquetipo Maven. El arquetipo genera un esqueleto para la aplicación y simplemente necesita implementar el método apropiado.

Puede encontrar una versión de esta aplicación en tut-install/examples/jaxrs/hello/.

Aquí se tratan los siguientes temas:

  • Para Crear un Servicio Web RESTful Usando el EID NetBeans

Para Crear un Servicio Web RESTful Usando el EID NetBeans

  1. Asegúrese de haber instalado los arquetipos del tutorial como se describe en Instalación de los arquetipos del tutorial.

  2. En el EID NetBeans, cree una aplicación web simple utilizando el arquetipo jaxrs-service-archetype de Maven. Este arquetipo crea una aplicación web "Hello, World" muy simple.

  3. En el menú Archivo, elija Nuevo proyecto.

  4. En Categorías, seleccione Maven. En Proyectos, seleccione Proyecto de arquetipo. Haga clic en Siguiente.

  5. En Buscar, ingrese jaxrs-service, seleccione jaxrs-service-archetype y haga clic en Siguiente.

  6. En Nombre del Proyecto, ingrese HelloWorldApplication, establezca la ubicación del proyecto y establezca el nombre del paquete en javaeetutorial.hello y haga clic en Finalizar.

    Se crea el proyecto.

  7. En HelloWorld.java, busque el método getHtml(). Reemplace el comentario //TODO con el siguiente texto, para que el producto final se asemeje al siguiente método: 

                            
@GET
@Produces("text/html")
public String getHtml() {
    return "<html lang=\"en\"><body><h1>Hello, World!!</body></h1></html>";
}

    Nota:

    Debido a que el tipo MIME producido es HTML, puede usar etiquetas HTML en su declaración de devolución.

  8. Haga clic con el botón derecho en el proyecto HelloWorldApplication en el panel Proyectos y seleccione Ejecutar.

    Esto compilará e implementará la aplicación en el Servidor GlassFish.

  9. En un navegador, abra la siguiente URL:

                            
http://localhost:8080/HelloWorldApplication/HelloWorldApplication

    Se abre una ventana del navegador y muestra el valor de retorno de Hello, World!!

También puede consultar los tutoriales en el sitio de tutoriales del EID NetBeans, como el titulado "Introducción a los servicios web RESTful" en https://netbeans.org/kb/docs/websvc/rest.html . La Aplicación de Ejemplo rsvp

La aplicación de ejemplo rsvp, ubicada en el directorio tut-install /examples/jaxrs/rsvp/, permite a los invitados a un evento indicar si asistirán. Los eventos, las personas invitadas al evento y las respuestas a la invitación se almacenan en Apache Derby utilizando la persistencia IPA de Java. Los recursos JAX-RS en rsvp están expuestos en un bean empresarial de sesión sin estado.

Aquí se tratan los siguientes temas:

  • Componentes de la Aplicación de Ejemplo rsvp

  • Ejecutando la Aplicación de Ejemplo rsvp

Componentes de la Aplicación de Ejemplo rsvp

Los tres beans empresariales en la aplicación de ejemplo rsvp son rsvp.ejb.ConfigBean, rsvp.ejb.StatusBean y rsvp.ejb.ResponseBean.

ConfigBean es un bean de sesión singleton que inicializa los datos en la base de datos.

StatusBean expone un recurso JAX-RS para mostrar el estado actual de todos los invitados a un evento. La plantilla de ruta URI se declara primero en la clase y luego en el método getEvent: 

                  
@Stateless
@Named
@Path("/status")
public class StatusBean {
    ...
    @GET
    @Produces({MediaType.APPLICATION_XML, MediaType.APPLICATION_JSON})
    @Path("{eventId}/")
    public Event getEvent(@PathParam("eventId") Long eventId) {
         ...

La combinación de las dos anotaciones @Path da como resultado la siguiente plantilla de ruta URI: 

                  
@Path("/status/{eventId}/")

La variable de ruta URI eventId es una variable @PathParam en el método getEvent, que responde a solicitudes HTTP GET y se ha anotado con @GET. La variable eventId se usa para buscar todas las respuestas actuales en la base de datos para ese evento en particular.

ResponseBean expone un recurso JAX-RS para configurar la respuesta de un invitado a un evento en particular. La plantilla de ruta de URI para ResponseBean se declara de la siguiente manera: 

                  
@Path("/{eventId}/{inviteId}")

Se declaran dos variables de ruta URI en la plantilla de ruta: eventId y inviteId. Al igual que en StatusBean, eventId es la identificación única para un evento en particular. Cada invitado a ese evento tiene una ID única para la invitación, y ese es el inviteId. Ambas variables de ruta se utilizan en dos métodos JAX-RS en ResponseBean: getResponse y putResponse. El método getResponse responde a solicitudes HTTP GET y muestra la respuesta actual del invitado y un formulario para cambiar la respuesta.

La clase javaeetutorial.rsvp.rest.RsvpApplication define la ruta de la aplicación raíz para los recursos aplicando la anotación javax.ws.rs.ApplicationPath en el nivel de clase. 

                  
@ApplicationPath("/webapi")
public class RsvpApplication extends Application {
}

Un invitado que desea cambiar su respuesta selecciona la nueva respuesta y envía los datos del formulario, que se procesan como una solicitud HTTP POST mediante el método putResponse. La nueva respuesta se extrae de la solicitud HTTP POST y se almacena como la cadena userResponse. El método putResponse utiliza userResponse, eventId y inviteId para actualizar la respuesta del invitado en la base de datos.

Los eventos, personas y respuestas en rsvp están encapsulados en entidades de la IPA de persistencia de Java. Las entidades rsvp.entity.Event, rsvp.entity.Person y rsvp.entity.Response representan respectivamente eventos, invitados y respuestas a un evento.

La clase rsvp.util.ResponseEnum declara un tipo enumerado que representa todos los posibles estados de respuesta que puede tener un invitado.

La aplicación web también incluye dos beans administrados por CDI, StatusManager y EventManager, que usan la IPA de cliente JAX-RS para llamar a los recursos expuestos en StatusBean y ResponseBean. Para obtener información sobre cómo se utiliza la IPA de Cliente en rsvp, consulte "La IPA de Cliente en la Aplicación de Ejemplo rsvp" .

Ejecutando la Aplicación de Ejemplo rsvp

Tanto el EID NetBeans como Maven se pueden usar para implementar y ejecutar la aplicación de ejemplo rsvp.

Aquí se tratan los siguientes temas:

  • Para Ejecutar la Aplicación de Ejemplo rsvp Usando el EID NetBeans

  • Para Ejecutar la Aplicación de Ejemplo rsvp Con Maven

Para Ejecutar la Aplicación de Ejemplo rsvp Usando el EID NetBeans

  1. Si el servidor de la base de datos aún no se está ejecutando, inícielo siguiendo las instrucciones en Arrancando y Parando Apache Derby.

  2. Asegúrese de que el Servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  3. En el menú Archivo, elija Abrir proyecto.

  4. En el cuadro de diálogo Abrir proyecto, vaya a:

                              
tut-install/examples/jaxrs

  5. Seleccione la carpeta rsvp.

  6. Haga clic en Abrir proyecto.

  7. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto rsvp y seleccione Ejecutar.

    El proyecto se compilará, ensamblará e implementará en el Servidor GlassFish. Se abrirá una ventana del navegador web con la siguiente URL: 

                              
http://localhost:8080/rsvp/index.xhtml

  8. En la ventana del navegador web, haga clic en el enlace Estado del evento para el evento de cumpleaños de Duke.

    Verá los invitados actuales y sus respuestas.

  9. Haga clic en la respuesta actual de uno de los invitados en la columna Estado de la tabla, seleccione una nueva respuesta y haga clic en Actualizar su estado.

    El nuevo estado del invitado debería mostrarse ahora en la tabla de invitados y sus estados de respuesta.

Para Ejecutar la Aplicación de Ejemplo rsvp Con Maven

  1. Si el servidor de la base de datos aún no se está ejecutando, inícielo siguiendo las instrucciones en Arrancando y Parando Apache Derby.

  2. Asegúrese de que el Servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  3. En una ventana de terminal, vaya a:

                              
tut-install/examples/jaxrs/rsvp/

  4. Introduzca el siguiente comando:

                              
mvn install

    Este comando compila, ensambla e implementa rsvp en el servidor GlassFish.

  5. Abra una ventana del navegador web en la siguiente URL:

                              
http://localhost:8080/rsvp/

  6. En la ventana del navegador web, haga clic en el enlace Estado del evento para el evento de cumpleaños de Duke.

    Verá los invitados actuales y sus respuestas.

  7. Haga clic en la respuesta actual de uno de los invitados en la columna Estado de la tabla, seleccione una nueva respuesta y haga clic en Actualizar su estado.

    El nuevo estado del invitado debería mostrarse ahora en la tabla de invitados y sus estados de respuesta.

Ejemplos del Mundo Real

La mayoría de los sitios de blogs utilizan servicios web RESTful. Estos sitios implican la descarga de archivos XML, en formato RSS o Atom, que contienen listas de enlaces a otros recursos. Otros sitios web y aplicaciones web que utilizan interfaces de desarrollador similares a REST para datos incluyen Twitter y Amazon S3 (Simple Storage Service). Con Amazon S3, los depósitos y los objetos se pueden crear, enumerar y recuperar mediante una interfaz HTTP de estilo REST o una interfaz SOAP. Los ejemplos que se envían con Jersey incluyen un ejemplo de servicio de almacenamiento con una interfaz RESTful.
