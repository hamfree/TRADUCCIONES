Gestionando Eventos para Componentes Personalizados

Como se explica en Implementando un Oyente de Evento, los eventos se ponen en cola automáticamente en los componentes estándar que activan eventos. Un componente personalizado, por otro lado, debe poner en cola manualmente los eventos de su método decode si dispara eventos.

Realización de Decodificación explica cómo poner en cola un evento en MapComponent usando su método decode. Esta sección explica cómo escribir la clase que representa el evento de hacer clic en el mapa y cómo escribir el método que procesa este evento.

Como se explica en Entendiendo la Página de Facelets, el atributo actionListener de la etiqueta bookstore:map apunta a la clase MapBookChangeListener. El método processAction de la clase de escucha procesa el evento de hacer clic en el mapa de la imagen. Aquí está el método processAction: 


@Override
public void processAction(ActionEvent actionEvent)
        throws AbortProcessingException {
    AreaSelectedEvent event = (AreaSelectedEvent) actionEvent;
    String current = event.getMapComponent().getCurrent();
    FacesContext context = FacesContext.getCurrentInstance();
    String bookId = books.get(current);
    context.getExternalContext().getSessionMap().put("bookId", bookId);
}

Cuando la implementación de JavaServer Faces llama a este método, pasa un objeto ActionEvent que representa el evento generado al hacer clic en el mapa de la imagen. A continuación, lo convierte en un objeto AreaSelectedEvent (ver tut-install`/examples/case-studies/dukes-bookstore/src/java/dukesbookstore/listeners/ AreaSelectedEvent.java`). Entonces este método obtiene el MapComponent asociado con el evento. A continuación, obtiene el valor del atributo current del objeto MapComponent, que indica el área actualmente seleccionada. Luego, el método usa el valor del atributo current para obtener el valor de ID del libro de un objeto HashMap, que se construye en otra parte de la clase MapBookChangeListener. Finalmente, el método coloca el ID obtenido del objeto HashMap en el mapa de sesión de la aplicación.

Además del método que procesa el evento, necesita la propia clase de evento. Esta clase es muy sencilla de escribir; tiene que extender ActionEvent y proporcionar un constructor que toma el componente en el que el evento está en cola y un método que devuelve el componente. Aquí está la clase AreaSelectedEvent usada con el mapa de imagen: 


public class AreaSelectedEvent extends ActionEvent {
    public AreaSelectedEvent(MapComponent map) {
        super(map);
    }
    public MapComponent getMapComponent() {
        return ((MapComponent) getComponent());
    }
}

Como se explica en la sección Creando Clases de Componentes Personalizados, para que MapComponent active eventos en primer lugar, debe implementar ActionSource. Debido a que MapComponent extiende UICommand, también implementa ActionSource.
