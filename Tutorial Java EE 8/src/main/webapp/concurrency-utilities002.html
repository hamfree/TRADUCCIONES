Componentes Principales de las Utilidades de Concurrencia

Los recursos simultáneos son objetos administrados que brindan capacidades de simultaneidad a las aplicaciones Java EE. En el servidor GlassFish, usted configura recursos simultáneos y luego los pone a disposición de los componentes de la aplicación, como servlets y beans empresariales. Se accede a los recursos simultáneos a través de la búsqueda JNDI o la inyección de recursos.

Los componentes principales de las utilidades de concurrencia son los siguientes.

  • ManagedExecutorService: las aplicaciones utilizan un servicio de ejecutor administrado para ejecutar las tareas enviadas de forma asíncrona. Las tareas se ejecutan en subprocesos iniciados y administrados por el contenedor. El contexto del contenedor se propaga al subproceso que ejecuta la tarea.

    Por ejemplo, al usar una llamada ManagedExecutorService.submit(), una tarea, como GenerateReportTask, podría enviarse para ejecutarse en un momento posterior y luego, al usar la devolución de llamada del objeto Future, recuperar el resultado cuando esté disponible.

  • ManagedScheduledExecutorService: las aplicaciones utilizan un servicio ejecutor programado administrado para ejecutar tareas enviadas de forma asincrónica en momentos específicos. Las tareas se ejecutan en subprocesos iniciados y administrados por el contenedor. El contexto del contenedor se propaga al subproceso que ejecuta la tarea. La IPA proporciona la funcionalidad de programación que permite a los usuarios establecer una fecha/hora específica para la ejecución de tareas mediante programación en la aplicación.

  • ContextService: un servicio de contexto se usa para crear objetos proxy dinámicos que capturan el contexto de un contenedor y permiten que las aplicaciones se ejecuten dentro de ese contexto en un momento posterior o se envíen a un Servicio de Ejecutor Administrado. El contexto del contenedor se propaga al subproceso que ejecuta la tarea.

  • ManagedThreadFactory: las aplicaciones utilizan una fábrica de subprocesos administrados para crear subprocesos administrados. Los subprocesos son iniciados y administrados por el contenedor. El contexto del contenedor se propaga al subproceso que ejecuta la tarea. Este objeto también se puede usar para proporcionar fábricas personalizadas para casos de uso específicos (con subprocesos personalizados) y, por ejemplo, establecer propiedades específicas/propietarias para estos objetos.
