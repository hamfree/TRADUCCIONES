Manejando Excepciones

Las excepciones lanzadas por los enterprise beans se dividen en dos categorías: sistema y aplicación.

Una excepción del sistema indica un problema con los servicios que admiten una aplicación. Por ejemplo, no se puede obtener una conexión a un recurso externo o no se puede encontrar un recurso inyectado. Si encuentra un problema a nivel del sistema, su enterprise bean debe generar una javax.ejb.EJBException. Debido a que la EJBException es una subclase de RuntimeException, no es necesario especificarla en la cláusula throws de la declaración del método. Si se lanza una excepción del sistema, el contenedor EJB podría destruir la instancia del bean. Por lo tanto, una excepción del sistema no puede ser manejada por el programa cliente del bean, sino que requiere la intervención de un administrador del sistema.

Una excepción de aplicación señala un error en la lógica de negocio de un enterprise bean. Las excepciones de aplicación suelen ser excepciones que usted mismo ha codificado, como la BookException lanzada por los métodos de negocio del ejemplo CartBean. Cuando un enterprise bean lanza una excepción de aplicación, el contenedor no lo envuelve en otra excepción. El cliente debe poder manejar cualquier excepción de aplicación que reciba.

Si se produce una excepción del sistema dentro de una transacción, el contenedor EJB revierte la transacción. Sin embargo, si se lanza una excepción de aplicación dentro de una transacción, el contenedor no revierte la transacción.
