Server Push

Server push es la capacidad del servidor para anticipar lo que el cliente necesitará antes de la solicitud del cliente. Permite que el servidor rellene previamente la memoria caché del navegador antes de que el navegador solicite el recurso para colocar en la memoria caché.

Server push es la más visible de las mejoras en HTTP/2 que aparece en la IPA de servlet. Todas las nuevas funciones de HTTP/2, incluida la inserción del servidor, tienen como objetivo mejorar el rendimiento de la experiencia de navegación web.

Server push deriva su contribución a la mejora del rendimiento del navegador del hecho de que los servidores saben qué recursos adicionales (como imágenes, hojas de estilo y secuencias de comandos) acompañan a las solicitudes iniciales. Por ejemplo, los servidores pueden saber que cada vez que un navegador solicita index.html, poco después solicitará header.gif, footer.gif y style.css. Los servidores pueden comenzar a enviar de manera preventiva los bytes de estos activos junto con los bytes de index.html.

Para usar server push, obtenga una referencia a un PushBuilder desde un HttpServletRequest, edite el constructor como desee y luego llame a push(). Consulte el javadoc para la clase javax.servlet.http.PushBuilder y el método javax.servlet.http.HttpServletRequest.newPushBuilder().

Para ver el código de muestra de GlassFish para esta función, consulte https://github.com/javaee/glassfish-samples/tree/master/ws/javaee8 .
