Anotaciones de Metadatos

La Arquitectura de los Conectores de JavaEE proporciona un conjunto de anotaciones para minimizar la necesidad de descriptores de implementación.

  • El desarrollador del adaptador de recursos puede utilizar la anotación @Connector para especificar que el componente JavaBeans es un componente JavaBeans del adaptador de recursos. Esta anotación se usa para proporcionar metadatos sobre las capacidades del adaptador de recursos. Opcionalmente, puede proporcionar un componente JavaBeans que implemente la interfaz ResourceAdapter, como en el siguiente ejemplo: 

                        
@Connector(
    displayName = "TrafficResourceAdapter",
    vendorName = "Java EE Tutorial",
    version = "7.0"
)
public class TrafficResourceAdapter implements ResourceAdapter,
                                               Serializable {
    ...
}

  • La anotación @ConnectionDefinition define un conjunto de interfaces de conexión y clases pertenecientes a un tipo de conexión en particular, como en el siguiente ejemplo: 

                        
@ConnectionDefinition(
    connectionFactory = ConnectionFactory.class,
    connectionFactoryImpl = TradeConnectionFactory.class,
    connection = Connection.class,
    connectionImpl = TradeConnection.class
)
public class TradeManagedConnectionFactory ... {
    ...
}

  • La anotación @AdministeredObject designa un componente JavaBeans como un objeto administrado.

  • La anotación @Activation contiene información de configuración relacionada con la conectividad entrante desde una instancia de EIS, como en el siguiente ejemplo: 

                        
@Activation(
        messageListeners = { TrafficListener.class }
)
public class TrafficActivationSpec implements ActivationSpec,
                                              Serializable {
    ...
    @ConfigProperty()
    /* port to listen to requests from the EIS */
    private String port;
    ...
}

  • La anotación @ConfigProperty se puede utilizar en los componentes de JavaBeans para proporcionar información de configuración adicional que puede utilizar el implementador y el proveedor del adaptador de recursos. El código de ejemplo anterior muestra varias anotaciones @ConfigProperty.

  • La anotación @ConnectionFactoryDefinition es una anotación de definición de recursos que se utiliza para definir una fábrica de conexiones de conector y registrarla en JNDI con el nombre especificado en el elemento de anotación obligatorio name. El elemento de anotación obligatorio interfaceName especifica el nombre completo de la clase de interfaz de la fábrica de conexiones. El elemento de anotación transactionSupport especifica el nivel de soporte de transacciones que necesita soportar la fábrica de conexiones. Los elementos de anotación minPoolSize y maxPoolSize especifican el número mínimo o máximo de conexiones que se deben asignar para un grupo de conexiones que respalda este recurso de fábrica de conexiones. Las propiedades adicionales asociadas con la fábrica de conexiones que se está definiendo se pueden especificar a través del elemento properties.

    Dado que no se permiten anotaciones repetidas, la anotación @ConnectionFactoryDefinitions actúa como un contenedor para varias definiciones de fábrica de conexiones de conector. El elemento de anotación value contiene las definiciones de la fábrica de conexiones de varios conectores.

  • La anotación @AdministeredObjectDefinition es una anotación de definición de recursos que se utiliza para definir un objeto administrado y registrarlo en JNDI con el nombre especificado en el elemento de anotación obligatorio name. El nombre completo obligatorio de la clase del objeto administrado debe indicarse mediante el elemento className. Las propiedades adicionales que deben configurarse en el objeto administrado se pueden especificar a través del elemento properties.

    Dado que no se permiten anotaciones repetidas, la anotación @AdministeredObjectDefinitions actúa como un contenedor para múltiples definiciones de objetos administrados. El elemento de anotación value contiene las múltiples definiciones de objetos administrados.

La especificación permite desarrollar un adaptador de recursos en modo mixto, es decir, la capacidad de un desarrollador de adaptadores de recursos de utilizar anotaciones de metadatos y descriptores de implementación en las aplicaciones. Un ensamblador o implementador de aplicaciones puede usar el descriptor de implementación para anular las anotaciones de metadatos especificadas por el desarrollador del adaptador de recursos.

El descriptor de implementación de un adaptador de recursos, si está presente, se llama ra.xml. El atributo metadata-complete define si el descriptor de implementación para el módulo del adaptador de recursos está completo o si los archivos de clase disponibles para el módulo y empaquetados con el adaptador de recursos deben examinarse en busca de anotaciones que especifiquen la información de implementación.

Para obtener la lista completa de anotaciones y componentes JavaBeans proporcionados en la plataforma Java EE, consulte la especificación Arquitectura de los Conectores de JavaEE 1.7.
