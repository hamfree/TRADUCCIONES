Soporte de Servicios Web

Los servicios web son aplicaciones empresariales basadas en web que utilizan protocolos de transporte y estándares abiertos basados en XML para intercambiar datos con los clientes que llaman. La plataforma Java EE proporciona las API y las herramientas XML que necesita para diseñar, desarrollar, probar e implementar rápidamente servicios web y clientes que interoperen completamente con otros servicios web y clientes que se ejecutan en plataformas basadas en Java o no basadas en Java.

Para escribir servicios web y clientes con las API XML de Java EE, todo lo que necesita hacer es pasar datos de parámetros a las llamadas al método y procesar los datos devueltos; para los servicios web orientados a documentos, envía documentos que contienen los datos del servicio de un lado a otro. No se necesita programación de bajo nivel porque las implementaciones de API XML hacen el trabajo de traducir los datos de la aplicación hacia y desde un flujo de datos basado en XML que se envía a través de los protocolos de transporte basados en XML estandarizados. Estos estándares y protocolos basados en XML se presentan en las siguientes secciones.

La traducción de datos a un flujo de datos estandarizado basado en XML es lo que hace que los servicios web y los clientes escritos con las API XML de Java EE sean completamente interoperables. Esto no significa necesariamente que los datos transportados incluyan etiquetas XML, porque los datos transportados pueden ser en sí mismos texto sin formato, datos XML o cualquier tipo de datos binarios, como audio, video, mapas, archivos de programa, diseño asistido por computadora ( CAD) documentos y similares. La siguiente sección presenta XML y explica cómo las partes que hacen negocios pueden usar etiquetas y esquemas XML para intercambiar datos de manera significativa.

XML

Extensible Markup Language (XML) es un estándar multiplataforma, extensible y basado en texto para representar datos. Las partes que intercambian datos XML pueden crear sus propias etiquetas para describir los datos, configurar esquemas para especificar qué etiquetas se pueden usar en un tipo particular de documento XML y usar hojas de estilo XML para administrar la visualización y el manejo de los datos.

Por ejemplo, un servicio web puede usar XML y un esquema para producir listas de precios, y las empresas que reciben las listas de precios y el esquema pueden tener sus propias hojas de estilo para manejar los datos de la manera que mejor se adapte a sus necesidades. A continuación se muestran algunos ejemplos.

  • Una empresa podría poner la información de precios XML a través de un programa para traducir el XML a HTML para que pueda publicar las listas de precios en su intranet.

  • Una empresa asociada puede poner la información de precios XML a través de una herramienta para crear una presentación de marketing.

  • Otra empresa podría leer la información de precios XML en una aplicación para su procesamiento.

Protocolo de Transporte SOAP

Las solicitudes de los clientes y las respuestas de los servicios web se transmiten como mensajes de Protocolo simple de acceso a objetos (SOAP) a través de HTTP para permitir un intercambio completamente interoperable entre los clientes y los servicios web, todos ejecutándose en diferentes plataformas y en varias ubicaciones en Internet. HTTP es un estándar familiar de solicitud y respuesta para enviar mensajes a través de Internet, y SOAP es un protocolo basado en XML que sigue el modelo de solicitud y respuesta HTTP.

La parte SOAP de un mensaje transportado hace lo siguiente:

  • Define un sobre basado en XML para describir lo que hay en el mensaje y explicar cómo procesar el mensaje.

  • Incluye reglas de codificación basadas en XML para expresar instancias de tipos de datos definidos por la aplicación dentro del mensaje

  • Define una convención basada en XML para representar la solicitud al servicio remoto y la respuesta resultante

Formato Estándar WSDL

El lenguaje de descripción de servicios web (WSDL) es un formato XML estandarizado para describir los servicios de red. La descripción incluye el nombre del servicio, la ubicación del servicio y las formas de comunicarse con el servicio. Las descripciones de los servicios WSDL se pueden publicar en la Web. el Servidor GlassFish proporciona una herramienta para generar la especificación WSDL de un servicio web que utiliza llamadas a procedimientos remotos para comunicarse con los clientes.
