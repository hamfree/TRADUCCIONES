Interfaz de Cliente Común

Esta sección explica cómo los componentes usan la IPA de Interfaz de Cliente Común (CCI) de la Arquitectura del Conector y un adaptador de recursos para acceder a los datos de un EIS. La IPA de CCI define un conjunto de interfaces y clases cuyos métodos permiten que un cliente realice operaciones típicas de acceso a datos. Las interfaces y clases de CCI son las siguientes.

  • ConnectionFactory: proporciona un componente de aplicación con una instancia de Connection a un EIS.

  • Connection: representa la conexión con el EIS subyacente.

  • ConnectionSpec: proporciona un medio para que un componente de aplicación pase propiedades específicas de solicitud de conexión a ConnectionFactory al realizar una solicitud de conexión.

  • Interaction: proporciona un medio para que un componente de aplicación ejecute funciones EIS, como procedimientos almacenados en bases de datos.

  • InteractionSpec: contiene propiedades relacionadas con la interacción de un componente de aplicación con un EIS.

  • Record: La superinterfaz para los distintos tipos de instancias de registro. Las instancias de registro pueden ser instancias MappedRecord, IndexedRecord o ResultSet, todas las cuales heredan de la interfaz Record.

  • RecordFactory: proporciona un componente de aplicación con una instancia de Record.

  • IndexedRecord: representa una colección ordenada de instancias de Record basadas en la interfaz java.util.List.

Un cliente o componente de aplicación que utiliza la CCI para interactuar con un EIS subyacente lo hace de una manera prescrita. El componente debe establecer una conexión con el administrador de recursos del EIS y lo hace mediante ConnectionFactory. El objeto Connection representa la conexión con el EIS y se utiliza para interacciones posteriores con el EIS.

El componente realiza sus interacciones con el EIS, como acceder a datos de una tabla específica, utilizando un objeto Interaction. El componente de la aplicación define el objeto Interaction mediante el uso de un objeto InteractionSpec. Cuando lee datos del EIS, como de las tablas de la base de datos, o escribe en esas tablas, el componente de la aplicación lo hace usando un tipo particular de instancia de Record: un MappedRecord, una instancia de IndexedRecord o ResultSet.

Tenga en cuenta, también, que una aplicación cliente que se basa en un adaptador de recursos CCI es muy parecida a cualquier otro cliente Java EE que utilice métodos de enterprise beans.
