Ciclo de Vida de la Petición Ajax

Una solicitud Ajax varía de otras solicitudes típicas de JavaServer Faces, y su procesamiento también es manejado de manera diferente por el ciclo de vida de JavaServer Faces.

Como se describe en Procesamiento Parcial y Renderizado Parcial, cuando se recibe una solicitud Ajax, el estado asociado con esa solicitud es capturado por javax.faces.context.PartialViewContext. Este objeto proporciona acceso a información como, por ejemplo, qué componentes están destinados a procesamiento / renderizado. El método processPartial de PartialViewContext utiliza esta información para realizar el procesamiento y la representación del árbol de componentes parciales.

El atributo execute de la etiqueta f:ajax identifica qué segmentos del árbol de componentes del lado del servidor deben procesarse. Debido a que los componentes se pueden identificar de forma única en el árbol de componentes de JavaServer Faces, es fácil identificar y procesar un solo componente, algunos componentes o un árbol completo. Esto es posible gracias al método visitTree de la clase UIComponent. Los componentes identificados luego se ejecutan a través de las fases del ciclo de vida de la solicitud de JavaServer Faces.

Similar al atributo execute, el atributo render identifica qué segmentos del árbol de componentes de JavaServer Faces necesitan ser renderizados durante la fase de respuesta de renderizado.

Durante la fase de respuesta de renderizado, se examina el atributo render. Los componentes identificados se encuentran y se les pide que se rendericen ellos mismos y sus hijos. Luego, los componentes se empaquetan y se envían al cliente como respuesta.
