Escribiendo Propiedades del Bean

Como se explica en Beans Gestionados en la Tecnología JavaServer Faces, una propiedad de bean gestionada se puede vincular a uno de los siguientes elementos:

  • Un valor de componente

  • Una instancia de componente

  • Una implementación de convertidor

  • Una implementación de oyente

  • Una implementación de validador

Estas propiedades siguen las convenciones de los componentes JavaBeans (también llamados beans). Para obtener más información sobre los componentes de JavaBeans, consulte el Tutorial de JavaBeans en http://docs.oracle.com/javase/tutorial/javabeans/index.html .

La etiqueta del componente vincula el valor del componente a una propiedad de bean gestionado usando su atributo value y vincula la instancia del componente a una propiedad de bean gestionado usando su atributo binding. Del mismo modo, todas las etiquetas de convertidor, oyente y validador usan sus atributos binding para vincular sus implementaciones asociadas a las propiedades de bean administradas. Consulte Enlazando Valores e Instancias de Componente a Propiedades del Bean Gestionado y Enlazando Convertidores, Oyentes, y Validadores a Propiedades de un Bean Gestionado para obtener más información.

Para vincular el valor de un componente a una propiedad de bean gestionado, el tipo de propiedad debe coincidir con el tipo de valor del componente al que está vinculado. Por ejemplo, si una propiedad de bean gestionado está vinculada al valor de un componente UISelectBoolean, la propiedad debe aceptar y devolver un valor boolean o una instancia de Object contenedor booleano.

Para vincular una instancia de componente a una propiedad de bean gestionado, la propiedad debe coincidir con el tipo de componente. Por ejemplo, si una propiedad de bean gestionada está vinculada a una instancia de UISelectBoolean, la propiedad debe aceptar y devolver un valor de UISelectBoolean.

De manera similar, para vincular una implementación de convertidor, oyente o validador a una propiedad de bean gestionado, la propiedad debe aceptar y devolver el mismo tipo de objeto convertidor, oyente o validador. Por ejemplo, si está utilizando la etiqueta convertDateTime para vincular un javax.faces.convert.DateTimeConverter a una propiedad, esa propiedad debe aceptar y devolver una instancia de DateTimeConverter.

El resto de esta sección explica cómo escribir propiedades que se pueden vincular a valores de componentes, a instancias de componentes para los objetos de componentes descritos en Agregando Componentes a Una Página Usando la Librería de Etiquetas de Etiquetas HTML y a implementaciones de convertidor, oyente y validador.

Escribiendo Propiedades Ligadas a Valores del Componente

Para escribir una propiedad de bean gestionado que está vinculada al valor de un componente, debe hacer coincidir el tipo de propiedad con el valor del componente.

La tabla 12-1 enumera las clases javax.faces.component y los tipos aceptables de sus valores.

Tabla 12-1 Tipos Aceptables de Valores de Componentes

Clase Componente

Tipos Aceptables de Valores de Componentes

UIInput, UIOutput, UISelectItem, UISelectOne

Cualquiera de los tipos primitivos y numéricos básicos o cualquier tipo de objeto de lenguaje de programación Java para el que esté disponible una implementación javax.faces.convert.Converter adecuada

UIData

array de beans, List de beans, un único bean, java.sql.ResultSet, javax.servlet.jsp.jstl.sql.Result, javax.sql.RowSet

UISelectBoolean

boolean o Boolean

UISelectItems

java.lang.String, Collection, Array, Map

UISelectMany

array o List, aunque los elementos del array o List pueden ser cualquiera de los tipos estándar

Cuando enlazan componentes a propiedades mediante el uso de los atributos value de las etiquetas de los componentes, los autores de la página deben asegurarse de que las propiedades correspondientes coincidan con los tipos de valores de los componentes.

Las Propiedades UIInput y UIOutput

Las clases de componentes UIInput y UIOutput están representadas por las etiquetas de componentes que comienzan con h:input y h:output, respectivamente (por ejemplo, h:inputText y h:outputText).

En el siguiente ejemplo, una etiqueta h:inputText vincula el componente name a la propiedad name de un bean gestionado llamado CashierBean. 


<h:inputText id="name"
             size="30"
             value="#{cashierBean.name}"
    ...>
</h:inputText>

El siguiente fragmento de código del bean gestionado CashierBean muestra el tipo de propiedad del bean vinculado por la etiqueta del componente anterior: 


protected String name = null;

public void setName(String name) {
    this.name = name;
}
public String getName() {
    return this.name;
}

Como se describe en Usando los Convertidores Estándar, para convertir el valor de un componente de entrada o salida, puede aplicar un convertidor o crear el límite de propiedad del bean. al componente con el tipo correspondiente. Aquí está la etiqueta de ejemplo, de Usando DateTimeConverter, que muestra la fecha en la que se enviarán los artículos. 


<h:outputText value="#{cashierBean.shipDate}">
    <f:convertDateTime type="date" dateStyle="full" />
</h:outputText>

La propiedad del bean representada por esta etiqueta debe tener un tipo de java.util.Date. El siguiente fragmento de código muestra la propiedad shipDate, del bean gestionado CashierBean, que está vinculado por el valor de la etiqueta en el ejemplo anterior: 


private Date shipDate;

public Date getShipDate() {
    return this.shipDate;
}
public void setShipDate(Date shipDate) {
    this.shipDate = shipDate;
}

Las Propiedades UIData

La clase de componente UIData está representada por la etiqueta de componente h:dataTable.

Los componentes UIData deben estar vinculados a uno de los tipos de propiedad de bean gestionados que se enumeran en la Tabla 12-1. Los componentes de datos se describen en Uso de Componentes de Tabla Enlazados a Datos. Aquí está parte de la etiqueta de inicio de dataTable de esa sección: 


<h:dataTable id="items"
    ...
    value="#{cart.items}"
    ...
    var="item">

La expresión de valor apunta a la propiedad items de un bean de carrito de compras llamado cart. El bean cart mantiene un mapa de beans ShoppingCartItem.

El método getItems del bean cart rellena una List con instancias de ShoppingCartItem que se guardan en los elementos del mapa cuando el cliente agrega libros al carrito, como se muestra en el siguiente segmento de código: 


public synchronized List<ShoppingCartItem> getItems() {
    List<ShoppingCartItem> results = new ArrayList<ShoppingCartItem>();
    results.addAll(this.items.values());
    return results;
}

Todos los componentes contenidos en el componente UIData están vinculados a las propiedades del bean cart que está vinculado a todo el componente UIData. Por ejemplo, aquí está la etiqueta h:outputText que muestra el título del libro en la tabla: 

<h:commandLink action="#{showcart.details}">
    <h:outputText value="#{item.item.title}"/>
</h:commandLink>

El título es en realidad un enlace a la página bookdetails.xhtml. La etiqueta h:outputText usa la expresión de valor #{item.item.title} para vincular su componente UIOutput a la propiedad title de la entidad Book. El primer elemento de la expresión es la instancia de ShoppingCartItem a la que hace referencia la etiqueta h:dataTable mientras representa la fila actual. El segundo elemento de la expresión se refiere a la propiedad item de ShoppingCartItem, que devuelve un Object (en este caso, un Book). La parte title de la expresión se refiere a la propiedad title de Book. El valor del componente UIOutput correspondiente a esta etiqueta está vinculado a la propiedad title de la entidad Book: 


private String title;
...
public String getTitle() {
    return title;
}

public void setTitle(String title) {
    this.title = title;
}

El componente UIData (y UIRepeat) admite las interfaces Map y Iterable, así como tipos personalizados.

Para UIData y UIRepeat, los tipos admitidos son:

  • null (se convierte en lista vacía)

  • javax.faces.model.DataMode

  • java.util.List

  • java.lang.Object[]

  • java.sql.ResultSet

  • javax.servlet.jsp.jstl.sql.Result

  • java.util.Collection

  • java.lang.Iterable

  • java.util.Map

  • java.lang.Object (se convierte en ScalarDataModel)

Las Propiedades UISelectBoolean

La clase de componente UISelectBoolean está representada por la etiqueta de componente h:selectBooleanCheckbox.

Las propiedades de bean administradas que contienen los datos de un componente UISelectBoolean deben ser de tipo boolean o Boolean. El ejemplo de etiqueta selectBooleanCheckbox de la sección Visualización de Componentes para Seleccionar un Valor vincula un componente a una propiedad. El siguiente ejemplo muestra una etiqueta que vincula un valor de componente a una propiedad boolean: 


<h:selectBooleanCheckbox title="#{bundle.receiveEmails}"
                         value="#{custFormBean.receiveEmails}">
</h:selectBooleanCheckbox>
<h:outputText value="#{bundle.receiveEmails}">

A continuación, se muestra una propiedad de ejemplo que se puede vincular al componente representado por la etiqueta de ejemplo: 


private boolean receiveEmails = false;
...
public void setReceiveEmails(boolean receiveEmails) {
    this.receiveEmails = receiveEmails;
}
public boolean getReceiveEmails() {
    return receiveEmails;
}

Las Propiedades UISelectMany

La clase de componente UISelectMany está representada por las etiquetas de componente que comienzan con h:selectMany (por ejemplo, h:selectManyCheckbox y h:selectManyListbox).

Debido a que un componente UISelectMany permite a un usuario seleccionar uno o más elementos de una lista de elementos, este componente debe correlacionarse con una propiedad de bean de tipo List o array. Esta propiedad de bean representa el conjunto de elementos seleccionados actualmente de la lista de elementos disponibles.

El siguiente ejemplo de la etiqueta selectManyCheckbox proviene de Visualización de Componentes para Seleccionar Múltiples Valores : 


<h:selectManyCheckbox id="newslettercheckbox"
                      layout="pageDirection"
                      value="#{cashierBean.newsletters}">
    <f:selectItems value="#{cashierBean.newsletterItems}"/>
</h:selectManyCheckbox>

Aquí está la propiedad del bean que se asigna al value de la etiqueta selectManyCheckbox del ejemplo anterior: 


private String[] newsletters;

public void setNewsletters(String[] newsletters) {
    this.newsletters = newsletters;
}
public String[] getNewsletters() {
    return this.newsletters;
}

Los componentes UISelectItem y UISelectItems se utilizan para representar todos los valores en un componente UISelectMany. Consulte Las Propiedades UISelectItem y Propiedades de UISelectItems para obtener información sobre cómo escribir las propiedades del bean para el UISelectItem y componentes UISelectItems.

Las Propiedades UISelectOne

La clase de componente UISelectOne está representada por las etiquetas de componente que comienzan con h:selectOne (por ejemplo, h:selectOneRadio y h:selectOneListbox).

Las propiedades UISelectOne aceptan los mismos tipos que las propiedades UIInput y UIOutput, porque un componente UISelectOne representa el único elemento seleccionado de un conjunto de elementos. Este elemento puede ser cualquiera de los tipos primitivos y cualquier otra cosa para la que pueda aplicar un convertidor.

A continuación, se muestra un ejemplo de la etiqueta h:selectOneMenu de Visualización de un menú mediante la etiqueta h:selectOneMenu : 


<h:selectOneMenu id="shippingOption"
                 required="true"
                 value="#{cashierBean.shippingOption}">
    <f:selectItem itemValue="2"
                  itemLabel="#{bundle.QuickShip}"/>
    <f:selectItem itemValue="5"
                  itemLabel="#{bundle.NormalShip}"/>
    <f:selectItem itemValue="7"
                  itemLabel="#{bundle.SaverShip}"/>
 </h:selectOneMenu>

Aquí está la propiedad del bean correspondiente a esta etiqueta:


private String shippingOption = "2";

public void setShippingOption(String shippingOption) {
    this.shippingOption = shippingOption;
}
public String getShippingOption() {
    return this.shippingOption;
}

Tenga en cuenta que shippingOption representa el elemento seleccionado actualmente de la lista de elementos en el componente UISelectOne.

Los componentes UISelectItem y UISelectItems se utilizan para representar todos los valores en un componente UISelectOne. Esto se explica en Visualizando un Menú Usando la Etiqueta h:selectOneMenu.

Para obtener información sobre cómo escribir las propiedades del bean administrado para los componentes UISelectItem y UISelectItems, consulte Las Propiedades UISelectItem y Propiedades de UISelectItems.

Las Propiedades UISelectItem

Un componente UISelectItem representa un valor único en un conjunto de valores en un componente UISelectMany o UISelectOne. Un componente UISelectItem debe estar vinculado a una propiedad de bean administrado de tipo javax.faces.model.SelectItem. Un objeto SelectItem está compuesto por un Object que representa el valor junto con dos Strings que representan la etiqueta y la descripción del objeto UISelectItem.

El ejemplo de etiqueta selectOneMenu de Las Propiedades UISelectOne contiene etiquetas selectItem que establecen los valores de la lista de elementos en la página. A continuación, se muestra un ejemplo de una propiedad de bean que puede establecer los valores para esta lista en el bean: 


SelectItem itemOne = null;

SelectItem getItemOne(){
    return itemOne;
}
void setItemOne(SelectItem item) {
    itemOne = item;
}

Las Propiedades UISelectItems

Los componentes UISelectItems son hijos de los componentes UISelectMany y UISelectOne. Cada componente UISelectItems se compone de un conjunto de instancias de UISelectItem o de cualquier colección de objetos, como una matriz, una lista o incluso un POJO.

El siguiente fragmento de código de CashierBean muestra cómo escribir las propiedades de las etiquetas selectItems que contienen instancias de SelectItem. 


private String[] newsletters;
private static final SelectItem[] newsletterItems = {
    new SelectItem("Duke's Quarterly"),
    new SelectItem("Innovator's Almanac"),
    new SelectItem("Duke's Diet and Exercise Journal"),
    new SelectItem("Random Ramblings")
};
...
public void setNewsletters(String[] newsletters) {
    this.newsletters = newsletters;
}

public String[] getNewsletters() {
    return this.newsletters;
}

public SelectItem[] getNewsletterItems() {
    return newsletterItems;
}

Aquí, la propiedad newsletters representa el objeto SelectItems, mientras que la propiedad newsletterItems representa una matriz estática de objetos SelectItem. La clase SelectItem tiene varios constructores; en este ejemplo, el primer argumento es un Object que representa el valor del elemento, mientras que el segundo argumento es una String que representa la etiqueta que aparece en el componente UISelectMany en la página.

Escribiendo Propiedades Ligadas a Instancias de Componente

Una propiedad vinculada a una instancia de componente devuelve y acepta una instancia de componente en lugar de un valor de componente. Los siguientes componentes vinculan una instancia de componente a una propiedad de bean administrada: 


<h:selectBooleanCheckbox id="fanClub"
                         rendered="false"
                         binding="#{cashierBean.specialOffer}" />
<h:outputLabel for="fanClub"
               rendered="false"
               binding="#{cashierBean.specialOfferText}"
               value="#{bundle.DukeFanClub}" />
</h:outputLabel>

La etiqueta selectBooleanCheckbox muestra una casilla de verificación y vincula el componente fanClub UISelectBoolean a la propiedad specialOffer de CashierBean. La etiqueta outputLabel vincula el valor del atributo value, que representa la etiqueta de la casilla de verificación, a la propiedad specialOfferText de CashierBean. Si el usuario pide más de $100 en libros y hace clic en el botón Enviar, el método submit de CashierBean establece las propiedades de rendered de ambos componentes en true, lo que hace que la casilla de verificación y la etiqueta se muestren cuando se vuelve a representar la página.

Debido a que los componentes correspondientes a las etiquetas de ejemplo están vinculados a las propiedades del bean administrado, estas propiedades deben coincidir con los tipos de los componentes. Esto significa que la propiedad specialOfferText debe ser de tipo UIOutput, y la propiedad specialOffer debe ser de tipo UISelectBoolean: 


UIOutput specialOfferText = null;
UISelectBoolean specialOffer = null;

public UIOutput getSpecialOfferText() {
    return this.specialOfferText;
}
public void setSpecialOfferText(UIOutput specialOfferText) {
    this.specialOfferText = specialOfferText;
}

public UISelectBoolean getSpecialOffer() {
    return this.specialOffer;
}
public void setSpecialOffer(UISelectBoolean specialOffer) {
    this.specialOffer = specialOffer;
}

Para obtener más información general sobre el enlace de componentes, consulte Beans Gestionados en la Tecnología JavaServer Faces.

Para obtener información sobre cómo hacer referencia a un método de bean gestionado que realiza la navegación cuando se hace clic en un botón, consulte Referenciando un Método Que Realiza la Navegación.

Para obtener más información sobre cómo escribir métodos de bean administrados que manejen la navegación, consulte Escribiendo un Método para Manejar la Navegación.

Escribiendo Propiedades Ligadas a Convertidores, Oyentes, o Validadores

Todas las etiquetas estándar de convertidor, oyente y validador incluidas con la tecnología JavaServer Faces admiten atributos de enlace que le permiten enlazar implementaciones de convertidor, oyente o validador a propiedades de bean gestionadas.

El siguiente ejemplo muestra una etiqueta convertDateTime estándar que utiliza una expresión de valor con su atributo binding para vincular la instancia javax.faces.convert.DateTimeConverter a la propiedad convertDate de LoginBean: 


<h:inputText value="#{loginBean.birthDate}">
    <f:convertDateTime binding="#{loginBean.convertDate}" />
</h:inputText>

Por lo tanto, la propiedad convertDate debe aceptar y devolver un objeto DateTimeConverter, como se muestra aquí: 


private DateTimeConverter convertDate;
public DateTimeConverter getConvertDate() {
       ...
    return convertDate;
}
public void setConvertDate(DateTimeConverter convertDate) {
    convertDate.setPattern("EEEEEEEE, MMM dd, yyyy");
    this.convertDate = convertDate;
}

Debido a que el convertidor está vinculado a una propiedad de bean gestionado, la propiedad de bean gestionado puede modificar los atributos del convertidor o agregarle nuevas funciones. En el caso del ejemplo anterior, la propiedad establece el patrón de fecha que usa el convertidor para analizar la entrada del usuario en un objeto Date.

Las propiedades de bean gestionadas que están vinculadas a implementaciones de validadores u oyentes se escriben de la misma manera y tienen el mismo propósito general.
