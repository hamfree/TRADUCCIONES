La Aplicación de Ejemplo standalone

La aplicación de ejemplo standalone demuestra cómo crear una instancia del contenedor de enterprise bean incrustado en una clase de prueba JUnit y llamar a un método de negocio de bean de sesión.

Aquí se tratan los siguientes temas:

  • Para Ejecutar la Aplicación de Ejemplo standalone Usando el EID NetBeans

  • Para Ejecutar la Aplicación de Ejemplo standalone Usando Maven

Descripción General de la Aplicación de Ejemplo standalone

Probar los métodos de negocio de un enterprise bean en una prueba unitaria permite a los desarrolladores ejercitar la lógica de negocio de una aplicación por separado de las demás capas de la aplicación, como la capa de presentación, y sin tener que implementar la aplicación en un servidor Java EE.

El ejemplo standalone tiene dos componentes principales: StandaloneBean, un bean de sesión sin estado, y StandaloneBeanTest, una clase de prueba JUnit que actúa como un cliente para StandaloneBean utilizando el contenedor incrustado.

StandaloneBean es un bean de sesión simple que expone una vista local sin interfaz con un único método de negocio, returnMessage, que devuelve "¡Saludos!" como una String: 

                
@Stateless
public class StandaloneBean {
    private static final String message = "¡Saludos!";
    public String returnMessage() {
        return message;
    }
}

StandaloneBeanTest llama a StandaloneBean.returnMessage y comprueba que el mensaje devuelto es correcto. Primero, se crea una instancia de contenedor incrustado y un contexto inicial dentro del método setUp, que se anota con org.junit.Before para indicar que el método debe ejecutarse antes de la prueba. métodos: 

                
@Before
public void setUp() {
    ec = EJBContainer.createEJBContainer();
    ctx = ec.getContext();
}

El método testReturnMessage, anotado con org.junit.Test para indicar que el método incluye una prueba unitaria, obtiene una referencia a StandaloneBean a través de una instancia de Context y llama a StandaloneBean.returnMessage. El resultado se compara con el resultado esperado mediante una aserción JUnit, assertEquals. Si la cadena devuelta de StandaloneBean.returnMessage es igual a "¡Saludos!" se pasa la prueba: 

                
@Test
public void testReturnMessage() throws Exception {
    logger.info("Testing standalone.ejb.StandaloneBean.returnMessage()");
    StandaloneBean instance = (StandaloneBean)
            ctx.lookup("java:global/classes/StandaloneBean");
    String expResult = "¡Saludos!";
    String result = instance.returnMessage();
    assertEquals(expResult, result);
}

Finalmente, el método tearDown, anotado con org.junit.After para indicar que el método debe ejecutarse después de que se hayan ejecutado todas las pruebas unitarias, cierra la instancia del contenedor incrustado: 

                
@After
public void tearDown() {
    if (ec != null) {
        ec.close();
    }
}

Para Ejecutar la Aplicación de Ejemplo standalone Usando el EID NetBeans

  1. Asegúrese de que el servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, vaya a:

                          
tut-install/examples/ejb

  4. Seleccione la carpeta independiente y haga clic en Abrir proyecto.

  5. En la pestaña Proyectos, haga clic con el botón derecho en standalone y seleccione Test.

    Esto ejecutará la clase de prueba JUnit StandaloneBeanTest. La pestaña Salida muestra el progreso de la prueba y el registro de salida.

Para Ejecutar la Aplicación de Ejemplo standalone Usando Maven

  1. Asegúrese de que el servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En una ventana de terminal, vaya a:

                          
tut-install/examples/ejb/standalone/

  3. Introduzca el siguiente comando:

                          
mvn install

    Este comando compila y empaqueta la aplicación en un archivo JAR y ejecuta la clase de prueba JUnit StandaloneBeanTest.
