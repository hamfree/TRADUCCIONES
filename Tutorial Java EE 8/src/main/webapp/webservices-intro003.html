Decidir Qué Tipo de Servicio Web Usar

Básicamente, desea utilizar los servicios web RESTful para la integración a través de la web y los grandes servicios web en escenarios de integración de aplicaciones empresariales que tienen requisitos avanzados de calidad de servicio (QoS).

  • JAX-WS: Aborda los requisitos avanzados de QoS que comúnmente ocurren en la informática empresarial. En comparación con JAX-RS, JAX-WS facilita la compatibilidad con el conjunto de protocolos WS-*, que proporciona estándares de seguridad y confiabilidad, entre otras cosas, e interopera con otros clientes y servidores que cumplen con WS-*.

  • JAX-RS: Facilita la escritura de aplicaciones web que aplican algunas o todas las restricciones del estilo REST para inducir propiedades deseables en la aplicación, como el bajo acoplamiento (la evolución del servidor es más fácil sin romper los clientes existentes), escalabilidad (iniciar pequeño y creciente) y simplicidad arquitectónica (utilice componentes listos para usar, como proxies o enrutadores HTTP). Elegiría usar JAX-RS para su aplicación web porque es más fácil para muchos tipos de clientes consumir servicios web RESTful mientras permite que el lado del servidor evolucione y se escale. Los clientes pueden optar por consumir algunos o todos los aspectos del servicio y mezclarlo con otros servicios basados en la web.
