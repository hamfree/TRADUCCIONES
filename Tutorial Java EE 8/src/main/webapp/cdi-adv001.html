Empaquetando Aplicaciones CDI

Cuando implementa una aplicación Java EE, CDI busca beans dentro de los archivos de beans. Un archivo de beans es cualquier módulo que contiene beans que el tiempo de ejecución de CDI puede administrar e inyectar. Hay dos tipos de archivos de beans: archivos de beans explícitos y archivos de beans implícitos.

Un archivo de bean explícito es un archivo que contiene un descriptor de implementación beans.xml, que puede ser un archivo vacío, no contener un número de versión o contener el número de versión 1.1 con el bean-discovery- atributo mode establecido en all. Por ejemplo: 

              
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://xmlns.jcp.org/xml/ns/javaee"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                           http://xmlns.jcp.org/xml/ns/javaee/beans_1_1.xsd"
       version="1.1" bean-discovery-mode="all">
    ...
</beans>

CDI puede administrar e inyectar cualquier bean en un archivo explícito, excepto aquellos anotados con @Vetoed.

Un archivo de beans implícito es un archivo que contiene algunos beans anotados con un tipo de alcance, no contiene un descriptor de implementación beans.xml o contiene un descriptor de implementación beans.xml con el atributo bean-discovery-mode establecido en annotated.

En un archivo implícito, CDI solo puede administrar e inyectar beans anotados con un tipo de alcance.

Para una aplicación web, el descriptor de implementación beans.xml, si está presente, debe estar en el directorio WEB-INF. Para módulos EJB o archivos JAR, el descriptor de implementación beans.xml, si está presente, debe estar en el directorio META-INF.
