Un Ejemplo de Servicio Web: helloservice

Este ejemplo demuestra un servicio web simple que genera una respuesta basada en la información recibida del cliente. HelloServiceBean es un bean de sesión sin estado que implementa un solo método: sayHello. Este método coincide con el método sayHello invocado por el cliente descrito en Una Aplicación Cliente JAX-WS Simple.

  • La Clase de Implementación del Endpoint del Servicio Web

  • La Clase de Implementación del Bean de Sesión sin Estado

La Clase de Implementación del Endpoint del Servicio Web

HelloServiceBean es la clase de implementación del punto final, normalmente el artefacto de programación principal para puntos finales de servicios web de enterprise beans. La clase de implementación del punto final del servicio web tiene los siguientes requisitos.

  • La clase se debe anotar con la anotación javax.jws.WebService o javax.jws.WebServiceProvider.

  • La clase de implementación puede hacer referencia explícita a una SEI a través del elemento endpointInterface de la anotación @WebService, pero no está obligada a hacerlo. Si no se especifica endpointInterface en @WebService, se define implícitamente un SEI para la clase de implementación.

  • Los métodos de negocio de la clase de implementación deben ser públicos y no deben declararse static o final.

  • Los métodos de negocio que se exponen a los clientes de servicios web se deben anotar con javax.jws.WebMethod.

  • Los métodos de negocio que se exponen a los clientes de servicios web deben tener parámetros y tipos de retorno compatibles con JAXB. Consulte la lista de enlaces de tipos de datos predeterminados de JAXB en Tipos Admitidos por JAX-WS.

  • La clase de implementación no debe declararse final y no debe ser abstract.

  • La clase de implementación debe tener un constructor público predeterminado.

  • La clase de punto final se debe anotar @Stateless.

  • La clase de implementación no debe definir el método finalize.

  • La clase de implementación puede usar las anotaciones javax.annotation.PostConstruct o javax.annotation.PreDestroy en sus métodos para las devoluciones de llamadas de eventos del ciclo de vida.

    El contenedor llama al método @PostConstruct antes de que la clase de implementación comience a responder a los clientes del servicio web.

    El contenedor llama al método @PreDestroy antes de que el punto final se elimine de la operación.

La Clase de Implementación del Bean de Sesión sin Estado

La clase HelloServiceBean implementa el método sayHello, que se anota como @WebMethod. El código fuente de la clase HelloServiceBean es el siguiente: 

                
package javaeetutorial.helloservice.ejb;
import javax.ejb.Stateless;
import javax.jws.WebMethod;
import javax.jws.WebService;
@Stateless
@WebService
public class HelloServiceBean {
    private final String message = "Hola, ";
    public void HelloServiceBean() {}
    @WebMethod
    public String sayHello(String name) {
        return message + name + ".";
    }
}

Ejecutando el Ejemplo helloservice

Puede usar el EID NetBeans o Maven para compilar, empaquetar e implementar el ejemplo de helloservice. A continuación, puede utilizar la Consola de administración para probar los métodos de punto final del servicio web.

Aquí se tratan los siguientes temas:

  • Para Construir, Empaquetar, y Desplegar el Ejemplo helloservice Usando el EID NetBeans

  • Para Construir, Empaquetar, y Desplegar el Ejemplo helloservice Usando Maven

  • Para Probar el Servicio sin un Cliente

Para Construir, Empaquetar, y Desplegar el Ejemplo helloservice Usando el EID NetBeans

  1. Asegúrese de que el servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En el menú Archivo, elija Abrir proyecto.

  3. IEn el cuadro de diálogo Abrir proyecto, navegue hasta:

                            
tut-install/examples/ejb

  4. Seleccione la carpeta helloservice.

  5. Haga clic en Abrir proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto helloservice y seleccione Build.

    Esto construye y empaqueta la aplicación en helloservice.ear, ubicado en tut-install/examples/ejb/helloservice/target/, y despliega este archivo EAR al servidor GlassFish.

Para Construir, Empaquetar, y Desplegar el Ejemplo helloservice Usando Maven

  1. Asegúrese de que el servidor GlassFish se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/ejb/helloservice/

  3. Introduzca el siguiente comando:

                            
mvn install

    Esto compila los archivos fuente y empaqueta la aplicación en un archivo JAR EJB ubicado en tut-install/examples/ejb/helloservice/target/helloservice.jar. Entonces el archivo EJB JAR se implementa en el servidor GlassFish.

    Tras la implementación, el servidor GlassFish genera artefactos adicionales necesarios para la invocación del servicio web, incluido el archivo WSDL.

Para Probar el Servicio sin un Cliente

La consola de administración del servidor GlassFish le permite probar los métodos de un punto final de servicio web. Para probar el método sayHello de HelloServiceBean, siga estos pasos.

  1. Abra la Consola de administración abriendo la siguiente URL en un navegador web:

                            
http://localhost:4848/

  2. En el árbol de navegación, seleccione el nodo Aplicaciones.

  3. En la tabla Aplicaciones, haga clic en el vínculo helloservice.

  4. En la tabla Módulos y componentes, haga clic en el enlace Ver punto final.

  5. En la página de información del punto final del servicio web, haga clic en el enlace Tester: 

                            
/HelloServiceBeanService/HelloServiceBean?Tester

  6. En la página Vínculos de prueba del servicio web, haga clic en el vínculo no seguro (el que especifica el puerto 8080).

  7. En la página HelloServiceBeanService Web Service Tester, en Métodos, ingrese un nombre como parámetro para el método sayHello.

  8. Click sayHello.

    Se abre la página de invocación del método sayHello. En Devuelto por el Método, verá la respuesta del extremo.
