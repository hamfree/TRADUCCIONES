Sobre los Beans Gestionados de CDI

Un bean administrado se implementa mediante una clase de Java, que se denomina su clase de bean. Una clase Java de nivel superior es un bean gestionado si cualquier otra especificación de tecnología Java EE, como la especificación de tecnología JavaServer Faces, o si cumple todas las condiciones siguientes lo define como bean gestionado.

  • No es una clase interna no estática.

  • Es una clase concreta o está anotada con @Decorator.

  • No se anota con una anotación que define el componente EJB ni se declara como una clase de bean EJB en ejb-jar.xml.

  • Tiene un constructor apropiado. Es decir, uno de los siguientes es el caso.

    • La clase tiene un constructor sin parámetros.

    • La clase declara un constructor anotado con @Inject.

No se requiere ninguna declaración especial, como una anotación, para definir un bean administrado.
