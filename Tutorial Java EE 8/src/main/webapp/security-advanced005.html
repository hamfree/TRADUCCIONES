Protección de clientes de aplicaciones

Los requisitos de autenticación de Java EE para clientes de aplicaciones son los mismos que para otros componentes de Java EE y se pueden utilizar las mismas técnicas de autenticación que para otros componentes de aplicaciones de Java EE. No es necesaria la autenticación cuando se accede a recursos web no protegidos.

Al acceder a recursos web protegidos, se pueden usar las variedades habituales de autenticación: autenticación básica HTTP, autenticación de formulario de inicio de sesión HTTP o autenticación de cliente SSL. Especificación de un Mecanismo de Autenticación en el Descriptor de Implementación describe cómo especificar la autenticación básica HTTP y la autenticación de formulario de inicio de sesión HTTP. Autenticación del Cliente describe cómo especificar la autenticación de cliente SSL.

Se requiere autenticación para acceder a enterprise beans protegidos. Los mecanismos de autenticación para enterprise beans se analizan en Protección de Enterprise Beans.

Un cliente de aplicación utiliza un servicio de autenticación proporcionado por el contenedor de cliente de aplicación para autenticar a sus usuarios. El servicio del contenedor se puede integrar con el sistema de autenticación de la plataforma nativa para que se utilice una capacidad de inicio de sesión único. El contenedor puede autenticar al usuario cuando se inicia la aplicación o cuando se accede a un recurso protegido.

Un cliente de aplicación puede proporcionar una clase, denominada módulo de inicio de sesión, para recopilar datos de autenticación. Si es así, se debe implementar la interfaz javax.security.auth.callback.CallbackHandler y se debe especificar el nombre de la clase en su descriptor de implementación. El controlador de devolución de llamada de la aplicación debe ser completamente compatible con los objetos Callback especificados en el paquete javax.security.auth.callback.

Uso de Módulos de Inicio de Sesión

Un cliente de aplicación puede utilizar el Servicio de Autenticación y Autorización de Java (JAAS, en inglés) para crear módulos de inicio de sesión para la autenticación. Una aplicación basada en JAAS implementa la interfaz javax.security.auth.callback.CallbackHandler para que pueda interactuar con los usuarios para ingresar datos de autenticación específicos, como nombres de usuario o contraseñas, o para mostrar mensajes de errores y advertencias.

Las aplicaciones implementan la interfaz CallbackHandler y la pasan al contexto de inicio de sesión, que la reenvía directamente a los módulos de inicio de sesión subyacentes. Un módulo de inicio de sesión utiliza el controlador de devolución de llamada tanto para recopilar entradas, como una contraseña o un PIN de tarjeta inteligente, de los usuarios como para proporcionar información, como información de estado, a los usuarios. Debido a que la aplicación especifica el controlador de devolución de llamada, un módulo de inicio de sesión subyacente puede permanecer independiente de las diversas formas en que las aplicaciones interactúan con los usuarios.

Por ejemplo, la implementación de un controlador de devolución de llamada para una aplicación IGU podría mostrar una ventana para solicitar la entrada del usuario, o la implementación de un controlador de devolución de llamada para una herramienta de línea de comandos podría simplemente solicitar al usuario que ingrese directamente desde la línea de comandos.

El módulo de inicio de sesión pasa una serie de devoluciones de llamada apropiadas al método handle del controlador de devolución de llamada, como NameCallback para el nombre de usuario y PasswordCallback para la clave; el controlador de devolución de llamada realiza la interacción del usuario solicitada y establece los valores apropiados en las devoluciones de llamada. Por ejemplo, para procesar un NameCallback, CallbackHandler podría solicitar un nombre, recuperar el valor del usuario y llamar al método setName de el NameCallback para almacenar el nombre.

Para obtener más información sobre el uso de JAAS para la autenticación en los módulos de inicio de sesión, consulte la documentación incluida en Más Información sobre Temas Avanzados de Seguridad .

Uso del Inicio de Sesión Programático

El inicio de sesión programático permite que el código del cliente proporcione las credenciales de usuario. Si está usando un cliente EJB, puede usar la clase com.sun.appserv.security.ProgrammaticLogin con sus convenientes métodos login y logout . El inicio de sesión programático es específico de un servidor.
