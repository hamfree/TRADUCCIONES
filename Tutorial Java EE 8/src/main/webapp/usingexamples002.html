Arrancando y Parando el Servidor GlassFish

Puede iniciar y detener el Servidor GlassFish utilizando el EID de NetBeans o la línea de comandos.

Para Arrancar el Servidor GlassFish Usando el EID NetBeans

  1. Haga clic en la pestaña Servicios.

  2. Expanda Servidores.

  3. Haga clic con el botón derecho en la instancia del Servidor GlassFish y seleccione Iniciar.

Para Parar el Servidor GlassFish Usando el EID NetBeans

Para detener el Servidor GlassFish usando el EID de NetBeans, haga clic con el botón derecho en la instancia del Servidor GlassFish y seleccione Detener.

Para Arrancar el Servidor GlassFish Usando la Línea de Comandos

Para iniciar el Servidor GlassFish desde la línea de comando, abra una ventana de terminal o símbolo del sistema y ejecute lo siguiente: 

                
asadmin start-domain --verbose

Un dominio es un conjunto de una o más instancias del Servidor GlassFish administradas por un servidor de administración. Los siguientes elementos están asociados a un dominio:

  • El número de puerto del servidor GlassFish: el valor predeterminado es 8080.

  • Número de puerto del servidor de administración: el valor predeterminado es 4848.

  • Un nombre de usuario y contraseña de administración: el nombre de usuario predeterminado es admin y, de manera predeterminada, no se requiere contraseña.

Usted especifica estos valores cuando instala el Servidor GlassFish. Los ejemplos de este tutorial asumen que eligió los puertos predeterminados, así como el nombre de usuario predeterminado y la falta de contraseña.

Sin argumentos, el comando start-domain inicia el dominio predeterminado, que es domain1. El indicador --verbose hace que todas las salidas de registro y depuración aparezcan en la ventana del terminal o en el símbolo del sistema. La salida también va al registro del servidor, que se encuentra en domain-dir`/logs/server.log`.

Para Parar el Servidor GlassFish Usando la Línea de Comandos

Para detener el Servidor GlassFish, abra una ventana de terminal o símbolo del sistema y ejecute: 

                
asadmin stop-domain domain1
