Manteniendo el Estado de un Cliente

Debido a que el contenedor crea una instancia de la clase de punto final para cada conexión, puede definir y usar variables de instancia para almacenar información de estado del cliente. Además, el método Session.getUserProperties proporciona un mapa modificable para almacenar propiedades de usuario. Por ejemplo, el siguiente extremo responde a los mensajes de texto entrantes con el contenido del mensaje anterior de cada cliente: 

              
@ServerEndpoint("/delayedecho")
public class DelayedEchoEndpoint {
   @OnOpen
   public void open(Session session) {
      session.getUserProperties().put("previousMsg", " ");
   }
   @OnMessage
   public void message(Session session, String msg) {
      String prev = (String) session.getUserProperties()
                                    .get("previousMsg");
      session.getUserProperties().put("previousMsg", msg);
      try {
         session.getBasicRemote().sendText(prev);
      } catch (IOException e) { ... }
   }
}

Para almacenar información común a todos los clientes conectados, puede utilizar variables de clase (estáticas); sin embargo, usted es responsable de garantizar un acceso seguro para subprocesos a ellos.
