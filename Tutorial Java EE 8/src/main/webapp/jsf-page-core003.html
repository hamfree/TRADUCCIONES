Usando los Validadores Estándar

La tecnología JavaServer Faces proporciona un conjunto de clases estándar y etiquetas asociadas que los autores de páginas y los desarrolladores de aplicaciones pueden utilizar para validar los datos de un componente. La Tabla 11-6 enumera todas las clases de validadores estándar y las etiquetas que le permiten utilizar los validadores de la página.

Tabla 11-6 Las Clases de Validadores

Clase Validadora

Etiqueta

Función

BeanValidator

validateBean

Registra un validador de beans para el componente.

BeanValidator

validateWholeBean

Permite la validación entre campos al habilitar la validación de beans a nivel de clase en beans de respaldo basados en CDI.

DoubleRangeValidator

validateDoubleRange

Comprueba si el valor local de un componente está dentro de un rango determinado. El valor debe ser de coma flotante o convertible a coma flotante.

LengthValidator

validateLength

Comprueba si la longitud del valor local de un componente está dentro de un rango determinado. El valor debe ser un objeto del tipo java.lang.String.

LongRangeValidator

validateLongRange

Comprueba si el valor local de un componente está dentro de un rango determinado. El valor debe ser de cualquier tipo numérico o String que se pueda convertir en un long.

RegexValidator

validateRegex

Comprueba si el valor local de un componente coincide con una expresión regular del paquete java.util.regex.

RequiredValidator

validateRequired

Asegura que el valor local no esté vacío en un componente EditableValueHolder.

Todas estas clases de validadores implementan la interfaz Validator. Los escritores de componentes y los desarrolladores de aplicaciones también pueden implementar esta interfaz para definir su propio conjunto de restricciones para el valor de un componente.

Al igual que los convertidores estándar, cada uno de estos validadores tiene uno o más mensajes de error estándar asociados. Si ha registrado uno de estos validadores en un componente de su página y el validador no puede validar el valor del componente, el mensaje de error del validador se mostrará en la página. Por ejemplo, el mensaje de error que se muestra cuando el valor del componente excede el valor máximo permitido por LongRangeValidator es el siguiente: 


{1}: Error de validación: el valor es mayor que el máximo permitido de "{0}"

En este caso, el parámetro de sustitución {1} se reemplaza por la etiqueta del componente o id, y el parámetro de sustitución {0} se reemplaza por el valor máximo permitido por el validador.

Consulte Visualización de mensajes de error con las etiquetas h:message y h:messages para obtener información sobre cómo mostrar mensajes de error de validación en la página cuando la validación falla.

En lugar de utilizar los validadores estándar, puede utilizar Validación de Beans para validar los datos. Si especifica restricciones de validación de beans en sus propiedades de bean gestionados, las restricciones se colocan automáticamente en los campos correspondientes de las páginas web de sus aplicaciones. Consulte el Capítulo 23, "Introducción a la validación de beans" para obtener más información. No necesita especificar la etiqueta validateBean para usar Validación de Beans, aunque la etiqueta le permite usar características de Validación de Beans más avanzadas. Por ejemplo, puede utilizar el atributo validationGroups de la etiqueta para especificar grupos de restricciones.

También puede crear y registrar validadores personalizados, aunque Bean Validation ha hecho que esta característica sea menos útil. Para obtener más información, consulte Creando y Usando un Validador Personalizado.

Validando un Valor de un Componente

Para validar el valor de un componente usando un validador particular, necesita registrar ese validador en el componente. Puede hacer esto de una de las siguientes formas.

  • Anide la etiqueta correspondiente del validador (que se muestra en la Tabla 11-6) dentro de la etiqueta del componente. Usando Etiquetas de Validador explica cómo usar la etiqueta validateLongRange. Puede utilizar las otras etiquetas estándar de la misma forma.

  • Consulte un método que realiza la validación desde el atributo validator de la etiqueta del componente.

  • Anide una etiqueta de validación dentro de la etiqueta del componente y utilice el atributo validatorId de la etiqueta de validación o su atributo binding para hacer referencia al validador.

Consulte Referenciando un Método Que Realiza la Validación para obtener más información sobre el uso del atributo validator.

El atributo validatorId funciona de manera similar al atributo converterId de la etiqueta converter, como se describe en Convirtiendo el Valor de un Componente.

Tenga en cuenta que la validación solo se puede realizar en componentes que implementan EditableValueHolder, porque estos componentes aceptan valores que se pueden validar.

Usando Etiquetas de Validador

El siguiente ejemplo muestra cómo usar la etiqueta de validación f:validateLongRange en un componente de entrada llamado quantity: 


<h:inputText id="quantity" size="4" value="#{item.quantity}">
    <f:validateLongRange minimum="1"/>
</h:inputText>
<h:message for="quantity"/>

Esta etiqueta requiere que el usuario ingrese un número que sea al menos 1. La etiqueta validateLongRange también tiene un atributo maximum, que establece un valor máximo para la entrada.

Los atributos de todas las etiquetas de validación estándar aceptan expresiones de valor EL. Esto significa que los atributos pueden hacer referencia a propiedades de beans gestionados en lugar de especificar valores literales. Por ejemplo, la etiqueta f:validateLongRange en el ejemplo anterior puede hacer referencia a propiedades de bean gestionados llamadas minimum y maximum para obtener los valores mínimos y máximos aceptables para la implementación del validador, como se muestra en este fragmento del ejemplo guessnumber-jsf: 


<h:inputText id="userNo"
             title="Type a number from 0 to 10:"
             value="#{userNumberBean.userNumber}">
    <f:validateLongRange minimum="#{userNumberBean.minimum}"
                         maximum="#{userNumberBean.maximum}"/>
</h:inputText>

La siguiente etiqueta f:validateRegex muestra cómo puede asegurarse de que una contraseña tenga de 4 a 10 caracteres y contenga al menos un dígito, al menos una letra minúscula y al menos una letra mayúscula: 


<f:validateRegex pattern="((?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{4,10})"
                 for="passwordVal"/>
