Compartiendo Información

Los componentes web, como la mayoría de los objetos, suelen trabajar con otros objetos para realizar sus tareas. Los componentes web pueden hacerlo haciendo lo siguiente.

  • Uso de objetos auxiliares privados (por ejemplo, componentes JavaBeans).

  • Compartir objetos que son atributos de un ámbito público.

  • Usando una base de datos.

  • Invocar otros recursos web. Los mecanismos de la tecnología Java Servlet que permiten que un componente web invoque otros recursos web se describen en Invocando Otros Recursos Web.

Usando Objetos de Ámbito

Los componentes web colaboradores comparten información por medio de objetos que se mantienen como atributos de cuatro objetos de ámbito. Accede a estos atributos usando los métodos getAttribute y setAttribute de la clase que representa el alcance. La Tabla 18-2 enumera los objetos de alcance.

Controlando el Acceso Concurrente a Recursos Compartidos

En un servidor multiproceso, se puede acceder a los recursos compartidos simultáneamente. Además de los atributos de objeto de ámbito, los recursos compartidos incluyen datos en memoria, como variables de clase o de instancia, y objetos externos, como archivos, conexiones de base de datos y conexiones de red.

El acceso concurrente puede surgir en varias situaciones.

  • Múltiples componentes web que acceden a objetos almacenados en el contexto web.

  • Múltiples componentes web que acceden a objetos almacenados en una sesión.

  • Múltiples hilos dentro de un componente web accediendo a variables de instancia. Un contenedor web normalmente creará un hilo para manejar cada solicitud. Para garantizar que una instancia de servlet maneje solo una solicitud a la vez, un servlet puede implementar la interfaz SingleThreadModel. Si un servlet implementa esta interfaz, no se ejecutarán dos subprocesos simultáneamente en el método de servicio del servlet. Un contenedor web puede implementar esta garantía sincronizando el acceso a una sola instancia del servlet o manteniendo un conjunto de instancias de componentes web y enviando cada nueva solicitud a una instancia libre. Esta interfaz no evita los problemas de sincronización que resultan del acceso de los componentes web a recursos compartidos, como variables de clase estáticas u objetos externos.

Cuando se puede acceder a los recursos simultáneamente, se pueden usar de manera inconsistente. Puede evitar esto controlando el acceso utilizando las técnicas de sincronización descritas en la lección Subprocesos en http://docs.oracle.com/javase/tutorial/essential/concurrency/.
