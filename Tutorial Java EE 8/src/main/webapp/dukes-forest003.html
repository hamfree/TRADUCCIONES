Creación y Despliegue del Caso de Estudio de la Aplicación del Bosque de Duke

Puede usar el EID de NetBeans o Maven para compilar e implementar El Bosque de Duke.

Aquí se tratan los siguientes temas:

  • Para Compilar y Desplegar la Aplicación del Bosque de Duke Mediante el EID de NetBeans

  • Para Compilar y Desplegar la Aplicación del Bosque de Duke Mediante Maven

Para Compilar y Desplegar la Aplicación del Bosque de Duke Mediante el EID de NetBeans

  1. Asegúrese de que se haya iniciado el servidor GlassFish (consulte Arrancando y Parando el Servidor GlassFish), así como el servidor de la base de datos (consulte Arrancando y Parando Apache Derby).

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, vaya a:

                          
tut-install/examples/case-studies

  4. Selecciona la carpeta dukes-forest.

  5. Seleccione la casilla de verificación Abrir proyectos necesarios y haga clic en Abrir proyecto.

  6. Haz clic con el botón derecho en la carpeta dukes-forest y selecciona Generar.

    Esta tarea configura el servidor, crea y llena la base de datos, crea todos los subproyectos, los ensambla en archivos JAR y WAR, e implementa las aplicaciones dukes-payment, dukes-store, y dukes-shipment.

    Para configurar el servidor, esta tarea crea un dominio de seguridad JDBC llamado jdbcRealm, habilita la asignación predeterminada de principal a rol y habilita el inicio de sesión único (SSO) para el servicio HTTP.

Para Compilar y Desplegar la Aplicación del Bosque de Duke Mediante Maven

  1. Asegúrese de que se haya iniciado el servidor GlassFish (consulte Arrancando y Parando el Servidor GlassFish), así como el servidor de la base de datos (consulte Arrancando y Parando Apache Derby).

  2. En una ventana de terminal, vaya a:

                          
tut-install/examples/case-studies/dukes-forest/

  3. Ingrese el siguiente comando para configurar el servidor, crear y completar la base de datos, compilar todos los subproyectos, ensamblarlos en archivos JAR y WAR e implementar las aplicaciones dukes-payment, dukes-store, y dukes-shipment: 

                          
mvn install

    Para configurar el servidor, esta tarea crea un dominio de seguridad JDBC llamado jdbcRealm, habilita la asignación predeterminada de principal a rol y habilita el inicio de sesión único (SSO) para el servicio HTTP.
