Cargando JavaScript como un Recurso

El archivo de recursos de JavaScript incluido con la tecnología JavaServer Faces se llama jsf.js y está disponible en la biblioteca javax.faces. Esta biblioteca de recursos admite la funcionalidad Ajax en aplicaciones JavaServer Faces.

Si usa la etiqueta f:ajax en una página, el recurso jsf.js se entrega automáticamente al cliente. No es necesario utilizar la etiqueta h:outputScript para especificar este recurso. Es posible que desee utilizar la etiqueta h:outputScript para especificar otras bibliotecas de JavaScript.

Para usar un recurso de JavaScript directamente con un UIComponent, debe cargar explícitamente el recurso como se describe en cualquiera de los siguientes temas:

  • Usando la IPA de JavaScript en una Aplicación Facelets – Utiliza la etiqueta h:outputScript directamente en una página de Facelets

  • Usando la Anotación @ResourceDependency en una Clase Bean – Utiliza la anotación javax.faces.application.ResourceDependency en una clase Java UIComponent

Usando la IPA de JavaScript en una Aplicación Facelets

Para usar la IPA de recursos de JavaScript directamente en una aplicación web, como una página de Facelets:

  1. Identifique el recurso JavaScript predeterminado para la página con la ayuda de la etiqueta h:outputScript.

    Por ejemplo, considere la siguiente sección de una página de Facelets:

    
<h:form>
    <h:outputScript name="jsf.js" library="javax.faces" target="head"/>
</h:form>

    Especificar el destino como head hace que el recurso de secuencia de comandos se represente dentro del elemento head en la página HTML.

  2. Identifique el componente al que le gustaría adjuntar la funcionalidad Ajax.

  3. Agregue la funcionalidad Ajax al componente utilizando la API de JavaScript. Por ejemplo, considere lo siguiente: 

    
<h:form>
    <h:outputScript name="jsf.js" library="javax.faces" target="head">
    <h:inputText id="inputname" value="#{userBean.name}"/>
    <h:outputText id="outputname" value="#{userBean.name}"/>
    <h:commandButton id="submit" value="Submit"
                     onclick="jsf.ajax.request(this, event,
                              {execute:'inputname',render:'outputname'});
                              return false;" />
</h:form>

    El método jsf.ajax.request toma hasta tres parámetros que especifican la fuente, el evento y las opciones. El parámetro de origen identifica el elemento DOM que desencadenó la solicitud Ajax, generalmente this. El parámetro de evento opcional identifica el evento DOM que desencadenó esta solicitud. El parámetro de opciones opcionales contiene un conjunto de pares de nombre/valor de la tabla 13-5.

Tabla 13-5 Valores Posibles para el Parámetro Options

Nombre

valor

execute

Una lista delimitada por espacios de identificadores de cliente o una de las palabras clave enumeradas en la Tabla 13-2. Los identificadores hacen referencia a los componentes que se procesarán durante la fase de ejecución del ciclo de vida.

render

Una lista delimitada por espacios de identificadores de cliente o una de las palabras clave enumeradas en la Tabla 13-2. Los identificadores hacen referencia a los componentes que se procesarán durante la fase de procesamiento del ciclo de vida.

onevent

Un objeto de tipo String que es el nombre de la función de JavaScript para llamar cuando ocurre un evento.

onerror

Un objeto de tipo String que es el nombre de la función de JavaScript para llamar cuando ocurre un error.

params

Un objeto que puede incluir parámetros adicionales para incluir en la solicitud.

Si no se especifica ningún identificador, la palabra clave asumida por defecto para el atributo execute es @this, y para el atributo render es @none.

También puede colocar el método JavaScript en un archivo e incluirlo como recurso.

Usando la Anotación @ResourceDependency en una Clase Bean

Utilice la anotación javax.faces.application.ResourceDependency para hacer que la clase de bean cargue la biblioteca jsf.js predeterminada.

Para cargar el recurso Ajax desde el lado del servidor:

  1. Utilice el método jsf.ajax.request dentro de la clase de bean.

    Nota:

    Este método se usa generalmente al crear un componente personalizado o un renderizador personalizado para un componente.

    El siguiente ejemplo muestra cómo se carga el recurso en una clase de bean:

    
@ResourceDependency(name="jsf.js" library="javax.faces" target="head")
