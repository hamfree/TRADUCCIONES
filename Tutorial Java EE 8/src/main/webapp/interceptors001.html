Descripción General de los Interceptores

Los interceptores se utilizan junto con las clases administradas de Java EE para permitir que los desarrolladores invoquen métodos de interceptor en una clase de destino asociada, junto con invocaciones de métodos o eventos de ciclo de vida. Los usos comunes de los interceptores son el registro, la auditoría y la creación de perfiles.

Aunque los interceptores son parte de Enterprise JavaBeans 3.2 y Contextos e Inyección de Dependencias para Java EE 1.1, la especificación Interceptores 1.2 se puede descargar como parte de una versión de mantenimiento de JSR 318, Enterprise JavaBeans 3.1, disponible en http://jcp.org/en/jsr/detail?id=318 . Puede utilizar interceptores con beans de sesión, beans controlados por mensajes y beans gestionados por CDI. En todos estos casos, la clase de destino del interceptor es la clase de bean.

Un interceptor se puede definir dentro de una clase de destino como un método de interceptor, o en una clase asociada denominada clase de interceptor. Las clases de interceptor contienen métodos que se invocan junto con los métodos o eventos del ciclo de vida de la clase de destino.

Las clases y los métodos de interceptor se definen mediante anotaciones de metadatos o en el descriptor de implementación de la aplicación que contiene los interceptores y las clases de destino.

Nota:

Las aplicaciones que usan el descriptor de implementación para definir interceptores no son portables entre servidores Java EE.

Los métodos de interceptor dentro de la clase de destino o en una clase de interceptor se anotan con una de las anotaciones de metadatos definidas en la Tabla 57-1.

Tabla 57-1 Anotaciones de Metadatos del Interceptor

Anotación de Metadatos del Interceptor

Descripción

javax.interceptor.AroundConstruct

Designa el método como un método interceptor que recibe una devolución de llamada después de que se construye la clase de destino

javax.interceptor.AroundInvoke

Designa el método como un método interceptor

javax.interceptor.AroundTimeout

Designa el método como un interceptor de tiempo de espera para interponerse en métodos de tiempo de espera para temporizadores de bean empresariales.

javax.annotation.PostConstruct

Designa el método como método interceptor para eventos de ciclo de vida posteriores a la construcción.

javax.annotation.PreDestroy

Designa el método como método interceptor para eventos de ciclo de vida previos a la destrucción

Clases de Interceptores

Las clases de interceptor se pueden designar con la anotación opcional javax.interceptor.Interceptor, pero no es necesario que las clases de interceptor se anoten de esa manera. Una clase de interceptor debe tener un constructor público sin argumentos.

La clase de destino puede tener cualquier número de clases de interceptor asociadas. El orden en que se invocan las clases de interceptor está determinado por el orden en que se definen las clases de interceptor en la anotación javax.interceptor.Interceptors. Sin embargo, este orden se puede anular en el descriptor de implementación.

Las clases de interceptor pueden ser objetivos de inyección de dependencia. La inyección de dependencia se produce cuando se crea la instancia de la clase de interceptor, utilizando el contexto de nomenclatura de la clase de destino asociada, y antes de que se invoquen las devoluciones de llamada de @PostConstruct.

Ciclo de Vida del Interceptor

Las clases de interceptor tienen el mismo ciclo de vida que su clase de destino asociada. Cuando se crea una instancia de clase de destino, también se crea una instancia de clase de interceptor para cada clase de interceptor declarada en la clase de destino. Es decir, si la clase de destino declara varias clases de interceptor, se crea una instancia de cada clase cuando se crea la instancia de la clase de destino. La instancia de la clase de destino y todas las instancias de la clase de interceptor se instancian por completo antes de que se invoquen las devoluciones de llamada @PostConstruct, y las devoluciones de llamada de @PreDestroy se invocan antes de que se invoque la destrucción de las instancias de la clase de destino y la clase de interceptor.

Interceptores y CDI

Contextos e Inyección de Dependencias para Java EE (CDI) se basa en la funcionalidad básica de los interceptores Java EE. Para obtener información sobre los interceptores CDI, incluida una discusión sobre los tipos de vinculación de interceptores, consulte Usando Interceptores en Aplicaciones CDI.
