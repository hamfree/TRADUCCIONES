Las Transacciones en las Aplicaciones Java EE

En una aplicación Java EE, una transacción es una serie de acciones que deben completarse correctamente o, de lo contrario, se revierten todos los cambios en cada acción. Las transacciones terminan en una confirmación (commit, en inglés) o una reversión (rollback en inglés).

La IPA de Java Transaction (JTA) permite que las aplicaciones accedan a transacciones de forma independiente a implementaciones específicas. JTA especifica las interfaces estándar de Java entre un administrador de transacciones y las partes involucradas en un sistema de transacciones distribuidas: la aplicación transaccional, el servidor Java EE y el administrador que controla el acceso a los recursos compartidos afectados por las transacciones.

La JTA define la interfaz UserTransaction que utilizan las aplicaciones para iniciar, confirmar o revertir transacciones. Los componentes de la aplicación obtienen un objeto UserTransaction a través de una búsqueda JNDI usando el nombre java:comp/UserTransaction o solicitando la inyección de un objeto UserTransaction. Un servidor de aplicaciones utiliza varias interfaces definidas por JTA para comunicarse con un administrador de transacciones; un administrador de transacciones usa interfaces definidas por JTA para interactuar con un administrador de recursos.

La especificación JTA 1.2 está disponible en http://jcp.org/en/jsr/detail?id=907.
