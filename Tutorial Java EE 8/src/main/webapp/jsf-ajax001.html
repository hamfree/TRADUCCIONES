Descripción General de Ajax

Las primeras aplicaciones web se crearon principalmente como páginas web estáticas. Cuando un cliente actualiza una página web estática, toda la página debe recargarse para reflejar la actualización. En efecto, cada actualización necesita una recarga de página para reflejar el cambio. Las recargas repetitivas de páginas pueden resultar en un acceso excesivo a la red y pueden afectar el rendimiento de la aplicación. Se crearon tecnologías como Ajax para superar estas deficiencias.

Ajax se refiere a JavaScript y XML, tecnologías que se utilizan ampliamente para crear contenido web dinámico y asincrónico. Si bien Ajax no se limita a las tecnologías JavaScript y XML, la mayoría de las veces las aplicaciones web las usan juntas. El enfoque de este tutorial es el uso de la funcionalidad Ajax basada en JavaScript en aplicaciones web JavaServer Faces.

JavaScript es un lenguaje de secuencias de comandos dinámico para aplicaciones web. Permite a los usuarios agregar funcionalidad mejorada a las interfaces de usuario y permite que las páginas web interactúen con los clientes de forma asincrónica. JavaScript se ejecuta principalmente en el lado del cliente (como en un navegador) y, por lo tanto, reduce el acceso al servidor por parte de los clientes.

Cuando una función de JavaScript envía una solicitud asincrónica del cliente al servidor, el servidor devuelve una respuesta que se utiliza para actualizar el Modelo de objetos de documento (DOM) de la página. Esta respuesta suele tener el formato de un documento XML. El término Ajax se refiere a esta interacción entre el cliente y el servidor.

No es necesario que la respuesta del servidor esté solo en XML; también puede estar en otros formatos, como JSON (ver Introducción a JSON y http://www.json.org/). Este tutorial no se centra en los formatos de respuesta.

Ajax permite la actualización asincrónica y parcial de aplicaciones web. Dicha funcionalidad permite páginas web altamente receptivas que se renderizan casi en tiempo real. Las aplicaciones web basadas en Ajax pueden acceder al servidor y procesar información y también pueden recuperar datos sin interferir con la visualización y representación de la página web actual en un cliente (como un navegador).

Algunas de las ventajas de usar Ajax son las siguientes:

  • Validación de datos de formularios en tiempo real, eliminando la necesidad de enviar el formulario para verificación

  • Funcionalidad mejorada para páginas web, como solicitudes de nombre de usuario y contraseña

  • Actualización parcial del contenido web, evitando recargas completas de la página.
