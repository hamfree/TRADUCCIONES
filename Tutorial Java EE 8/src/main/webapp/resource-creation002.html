Objetos DataSource y Agrupaciones de Connection

Para almacenar, organizar y recuperar datos, la mayoría de las aplicaciones utilizan una base de datos relacional. Los componentes Java EE pueden acceder a bases de datos relacionales a través de la IPA JDBC. Para obtener información sobre esta API, consulte http://docs.oracle.com/javase/8/docs/technotes/guides/jdbc/ .

En la IPA de JDBC, se accede a las bases de datos mediante objetos DataSource. Un DataSource tiene un conjunto de propiedades que identifican y describen la fuente de datos del mundo real que representa. Estas propiedades incluyen información como la ubicación del servidor de la base de datos, el nombre de la base de datos, el protocolo de red que se utilizará para comunicarse con el servidor, etc. En el Servidor GlassFish, una fuente de datos se denomina recurso JDBC.

Las aplicaciones acceden a una fuente de datos mediante una conexión, y un objeto DataSource puede considerarse una fábrica para las conexiones a la fuente de datos particular que representa la instancia de DataSource. En una implementación básica de DataSource, una llamada al método getConnection devuelve un objeto de conexión que es una conexión física a la fuente de datos.

Un objeto DataSource puede registrarse con un servicio de nombres JNDI. Si es así, una aplicación puede usar la IPA JNDI para acceder a ese objeto DataSource, que luego puede usarse para conectarse a la fuente de datos que representa.

Los objetos DataSource que implementan la agrupación de conexiones también producen una conexión a la fuente de datos particular que representa la clase DataSource. El objeto de conexión que devuelve el método getConnection es un identificador para un objeto PooledConnection en lugar de una conexión física. Una aplicación usa el objeto de conexión de la misma manera que usa una conexión. La agrupación de conexiones no tiene ningún efecto en el código de la aplicación, excepto que una conexión agrupada, como todas las conexiones, siempre debe cerrarse explícitamente. Cuando una aplicación cierra una conexión que se agrupa, la conexión se devuelve a un grupo de conexiones reutilizables. La próxima vez que se llame a getConnection, se devolverá un identificador para una de estas conexiones agrupadas, si hay alguna disponible. Debido a que la agrupación de conexiones evita la creación de una nueva conexión física cada vez que se solicita una, las aplicaciones pueden ejecutarse significativamente más rápido.

Un grupo de conexiones JDBC es un grupo de conexiones reutilizables para una base de datos en particular. Dado que la creación de cada nueva conexión física requiere mucho tiempo, el servidor mantiene un conjunto de conexiones disponibles para aumentar el rendimiento. Cuando solicita una conexión, una aplicación obtiene una del grupo. Cuando una aplicación cierra una conexión, la conexión se devuelve al grupo.

Las aplicaciones que usan la IPA de persistencia especifican el objeto DataSource que están usando en el elemento jta-data-source del archivo persistence.xml: 


<jta-data-source>jdbc/MyOrderDB</jta-data-source>

Esta suele ser la única referencia a un objeto JDBC para una unidad de persistencia. El código de la aplicación no hace referencia a ningún objeto JDBC.
