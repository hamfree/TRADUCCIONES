Configurando Reglas de Navegación

La navegación entre diferentes páginas de una aplicación JavaServer Faces, como elegir la siguiente página que se mostrará después de hacer clic en un botón o componente de enlace, se define mediante un conjunto de reglas. Las reglas de navegación pueden ser implícitas o pueden definirse explícitamente en el archivo de recursos de configuración de la aplicación. Para obtener más información sobre las reglas de navegación implícitas, consulte Modelo de Navegación.

Cada regla de navegación especifica cómo navegar de una página a otra página o conjunto de páginas. La implementación de JavaServer Faces elige la regla de navegación adecuada según la página que se muestra actualmente.

Una vez seleccionada la regla de navegación adecuada, la elección de a qué página acceder a continuación desde la página actual depende de dos factores:

  • El método de acción invocado cuando se hizo clic en el componente.

  • El resultado lógico al que hace referencia la etiqueta del componente o devuelto por el método de acción

El resultado puede ser cualquier cosa que elija el desarrollador, pero la Tabla 16-3 enumera algunos resultados que se utilizan comúnmente en aplicaciones web.

Tabla 16-3 Cadenas de resultados comunes

Resultado

Lo Que Significa

success

Todo funcionó. Ir a la página siguiente.

failure

Algo está mal. Vaya a una página de error.

login

El usuario debe iniciar sesión primero. Vaya a la página de inicio de sesión.

no results

La búsqueda no encontró nada. Vuelve a la página de búsqueda.

Por lo general, el método de acción realiza algún procesamiento en los datos del formulario de la página actual. Por ejemplo, el método puede verificar si el nombre de usuario y la contraseña ingresados en el formulario coinciden con el nombre de usuario y la contraseña en el archivo. Si coinciden, el método devuelve el resultado success. De lo contrario, devuelve el resultado failure. Como demuestra este ejemplo, tanto el método utilizado para procesar la acción como el resultado devuelto son necesarios para determinar la página correcta para acceder.

Aquí hay una regla de navegación que podría usarse con el ejemplo que se acaba de describir:


<navigation-rule>
    <from-view-id>/login.xhtml</from-view-id>
    <navigation-case>
        <from-action>#{LoginForm.login}</from-action>
        <from-outcome>success</from-outcome>
        <to-view-id>/storefront.xhtml</to-view-id>
    </navigation-case>
    <navigation-case>
        <from-action>#{LoginForm.logon}</from-action>
        <from-outcome>failure</from-outcome>
        <to-view-id>/logon.xhtml</to-view-id>
    </navigation-case>
</navigation-rule>

Esta regla de navegación define las posibles formas de navegar desde login.xhtml. Cada elemento navigation-case define una posible ruta de navegación desde login.xhtml. El primer navigation-case dice que si LoginForm.login devuelve un resultado de success, entonces storefront.xhtml puede ser accedido. El segundo navigation-case dice que login.xhtml se volverá a representar si LoginForm.login devuelve failure.

La configuración del flujo de página de una aplicación consta de un conjunto de reglas de navegación. Cada regla está definida por el elemento navigation-rule en el archivo faces-config.xml.

Cada elemento navigation-rule corresponde a un identificador de árbol de componentes definido por el elemento opcional from-view-id. Esto significa que cada regla define todas las formas posibles de navegar desde una página en particular en la aplicación. Si no hay un elemento from-view-id, las reglas de navegación definidas en el elemento navigation-rule se aplican a todas las páginas de la aplicación. El elemento from-view-id también permite patrones de coincidencia de comodines. Por ejemplo, este elemento from-view-id dice que la regla de navegación se aplica a todas las páginas del directorio books: 


<from-view-id>/books/*</from-view-id>

Un elemento navigation-rule puede contener cero o más elementos navigation-case. El elemento navigation-case define un conjunto de criterios de coincidencia. Cuando se satisfacen estos criterios, la aplicación navegará a la página definida por el elemento to-view-id contenido en el mismo elemento navigation-case.

Los criterios de navegación se definen mediante elementos opcionales from-result y from-action. El elemento from-result define un resultado lógico, como success. El elemento from-action usa una expresión de método para referirse a un método de acción que devuelve una String, que es el resultado lógico. El método realiza cierta lógica para determinar el resultado y devuelve el resultado.

Los elementos navigation-case se comparan con el resultado y la expresión del método en el siguiente orden.

  1. Casos que especifican un valor de from-result y un valor de from-action. Ambos elementos se pueden usar si el método de acción devuelve resultados diferentes según el resultado del procesamiento que realiza.

  2. Casos que especifican solo un valor from-result. El elemento from-result debe coincidir con el resultado definido por el atributo action del componente javax.faces.component.UICommand o el resultado devuelto por el método al que hace referencia el componente UICommand.

  3. Casos que especifican solo un valor from-action. Este valor debe coincidir con la expresión action especificada por la etiqueta del componente.

Cuando cualquiera de estos casos coincide, el árbol de componentes definido por el elemento to-view-id se seleccionará para renderizar.
