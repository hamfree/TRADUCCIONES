Beans como Objetos Inyectables

El concepto de inyección ha sido parte de la tecnología Java durante algún tiempo. Desde que se introdujo la plataforma Java EE 5, las anotaciones han hecho posible inyectar recursos y otros tipos de objetos en objetos gestionados por contenedores. CDI hace posible inyectar más tipos de objetos e inyectarlos en objetos que no están administrados por contenedores.

Se pueden inyectar los siguientes tipos de objetos:

  • Casi cualquier clase de Java

  • Beans de Sesión

  • Recursos de Java EE: fuentes de datos, temas de Java Message Service, colas, fábricas de conexiones y similares

  • Contextos de persistencia (objetos de la IPA de Java Persistence EntityManager)

  • Campos de Producer

  • Objetos devueltos por métodos de Producer

  • Referencias de servicios web

  • Referencias remotas de beans empresariales

Por ejemplo, suponga que crea una clase Java simple con un método que devuelve una cadena: 

              
package greetings;
public class Greeting {
    public String greet(String name) {
        return "Hola, " + name + ".";
    }
}

Esta clase se convierte en un bean que luego puede inyectar en otra clase. Este bean no está expuesto al EL en esta forma. Dando a los Beas Nombres con EL explica cómo puede hacer que un bean sea accesible para EL.
