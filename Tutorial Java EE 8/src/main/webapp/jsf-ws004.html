Ámbitos de WebSocket y Usuarios

De forma predeterminada, el WebSocket tiene un ámbito de aplicación. Por ejemplo, cualquier vista o sesión en toda la aplicación web que tenga el mismo canal WebSocket abierto recibirá el mismo mensaje push. El mensaje push puede ser enviado por todos los usuarios y la aplicación. Para restringir los mensajes push a todas las vistas solo en la sesión de usuario actual, establezca el atributo de ámbito opcional en session . En este caso, el mensaje push solo puede ser enviado por el usuario y no por la aplicación.

<f:websocket channel="someChannel" scope="session" …​ />

Para restringir los mensajes push solo a la vista actual, puede establecer el atributo de alcance en view. El mensaje push no aparecerá en otras vistas en la misma sesión, incluso si tiene la misma URL. Este mensaje push solo puede ser enviado por el usuario y no por la aplicación.

<f:websocket channel="someChannel" scope="view" …​ />

El atributo de ámbito puede no ser una expresión EL.

Además, puede establecer el atributo opcional user en el identificador único del usuario que inició sesión, normalmente el nombre de inicio de sesión o el ID de usuario. Como tal, el mensaje push puede dirigirse a un usuario específico y también puede ser enviado por otros usuarios y la aplicación. El valor del atributo user debe implementar Serializable y tener una huella de memoria baja, por lo que no se recomienda una entidad de usuario completa.

Por ejemplo, cuando utiliza autenticación administrada por contenedor o un marco o biblioteca relacionada:

<f:websocket channel="someChannel"user="#{request.remoteUser}" …​ />

O bien, cuando tiene una entidad de usuario personalizada accesible a través de EL, como #{someLoggedInUser} que tiene una propiedad id que representa su identificador: 

              
<f:websocket channel="someChannel"
user="#{someLoggedInUser.id}" ... />

Cuando se especifica el atributo user, el alcance predeterminado es session y no se puede establecer en application.

En el lado del servidor, el mensaje push se puede dirigir al usuario especificado en el atributo user usando PushContext.send(Object, Serializable). El mensaje push puede ser enviado por todos los usuarios y la aplicación. 

              
@Inject @Push
private PushContext someChannel;
public void sendMessage(Object message, User recipientUser) { Long recipientUserId = recipientUser.getId(); someChannel.send(message, recipientUserId);
}

Se pueden orientar varios usuarios pasando una Collection que contiene identificadores de usuario a PushContext.send(Object, Collection). 

              
public void sendMessage(Object message, Group recipientGroup) {
Collection<Long> recipientUserIds = recipientGroup.getUserIds();
someChannel.send(message, recipientUserIds);
}
