Ejemplos: Protección de Enterprise Beans

Los siguientes ejemplos muestran cómo proteger los beans empresariales utilizando seguridad declarativa y programática.

Aquí se tratan los siguientes temas:

  • El Ejemplo de cart-secure: Asegurar un Enterprise Bean con Seguridad Declarativa

  • El ejemplo converter-secure: Asegurar un Enterprise Bean con Seguridad Programática

El Ejemplo de cart-secure: Asegurar un Enterprise Bean con Seguridad Declarativa

Esta sección explica cómo configurar un enterprise bean para la autenticación básica de nombre de usuario/contraseña. Cuando se solicita un bean que está restringido de esta manera, el servidor solicita un nombre de usuario y una contraseña del cliente y verifica que el nombre de usuario y la contraseña sean válidos comparándolos con una base de datos de usuarios autorizados en el servidor GlassFish.

Si el tema de la autenticación es nuevo para usted, consulte la Especificación de Mecanismos de Autenticación .

Este ejemplo demuestra la seguridad comenzando con la aplicación de enterprise bean no segura, cart, que se encuentra en el directorio _tut-install_/examples/ejb/cart/ y se analiza en El Ejemplo cart.

En general, los siguientes pasos son necesarios para agregar autenticación de nombre de usuario/contraseña a una aplicación existente que contiene un enterprise bean. En la aplicación de ejemplo que se incluye con este tutorial, estos pasos se completaron y se enumeran aquí simplemente para mostrar lo que se debe hacer si desea crear una aplicación similar.

  1. Cree una aplicación como la que se encuentra en El Ejemplo cart. El ejemplo de este tutorial comienza con este ejemplo y demuestra cómo agregar la autenticación básica del cliente a esta aplicación. La aplicación de ejemplo que se analiza en esta sección se puede encontrar en _tut-install_/examples/security/cart-secure/.

  2. Si aún no lo ha hecho, complete los pasos en Cómo Configurar su Sistema para Ejecutar los Ejemplos de Seguridad para configurar su sistema para ejecutar las aplicaciones del tutorial.

  3. Modifique el código fuente del enterprise bean, CartBean.java, para especificar qué funciones están autorizadas para acceder a qué métodos protegidos. Este paso se analiza en Anotando el Bean.

  4. Compile, empaquete e implemente el enterprise bean; luego construya y ejecute la aplicación cliente siguiendo los pasos en Para Ejecutar el Ejemplo cart-secure usando el EID NetBeans o Para Ejecutar el Ejemplo cart-secure Usando Maven.

Anotando el Bean

El código fuente de la aplicación cart original se modificó como se muestra en el siguiente fragmento de código (modificaciones en negrita). El archivo resultante se puede encontrar en el archivo _tut-install_/examples/security/cart-secure/cart-secure-ejb/src/main/java/ javaeetutorial/cart/ejb/CartBean.java .

El fragmento de código es el siguiente:

                  
package javaeetutorial.cartsecure.ejb;
import java.io.Serializable;
import java.util.ArrayList;
import java.util.List;
import javaeetutorial.cart.util.BookException;
import javaeetutorial.cart.util.IdVerifier;
import javax.ejb.Remove;
import javax.ejb.Stateful;
import javax.annotation.security.DeclareRoles;
import javax.annotation.security.RolesAllowed;
@Stateful
@DeclareRoles("TutorialUser")
public class CartBean implements Cart, Serializable {
    List<String> contents;
    String customerId;
    String customerName;
    @Override
    public void initialize(String person) throws BookException {
        if (person == null) {
            throw new BookException("Null person not allowed.");
        } else {
            customerName = person;
        }
        customerId = "0";
        contents = new ArrayList<>();
    }
    @Override
    public void initialize(String person, String id) throws BookException {
        if (person == null) {
            throw new BookException("Null person not allowed.");
        } else {
            customerName = person;
        }
        IdVerifier idChecker = new IdVerifier();
        if (idChecker.validate(id)) {
            customerId = id;
        } else {
            throw new BookException("Invalid id: " + id);
        }
        contents = new ArrayList<>();
    }
    @Override
    @RolesAllowed("TutorialUser")
    public void addBook(String title) {
        contents.add(title);
    }
    @Override
    @RolesAllowed("TutorialUser")
    public void removeBook(String title) throws BookException {
        boolean result = contents.remove(title);
        if (result == false) {
            throw new BookException("\"" + title + "\" not in cart.");
        }
    }
    @Override
    @RolesAllowed("TutorialUser")
    public List<String> getContents() {
        return contents;
    }
    @Override
    @Remove()
    @RolesAllowed("TutorialUser")
    public void remove() {
        contents = null;
    }
}

La anotación @RolesAllowed se especifica en los métodos para los que desea restringir el acceso. En este ejemplo, solo los usuarios con el rol de TutorialUser podrán agregar y eliminar libros del carrito y enumerar el contenido del carrito. Una anotación @RolesAllowed declara implícitamente un rol al que se hará referencia en la aplicación; por lo tanto, no se requiere ninguna anotación @DeclareRoles. La presencia de la anotación @RolesAllowed también declara implícitamente que se requerirá autenticación para que un usuario acceda a estos métodos. Si no se especifica ningún método de autenticación en el descriptor de implementación, el tipo de autenticación será autenticación de nombre de usuario/contraseña.

Para Ejecutar el Ejemplo cart-secure Usando el EID NetBeans

  1. Siga los pasos en Cómo Configurar su Sistema para Ejecutar los Ejemplos de Seguridad .

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, vaya a:

                            
tut-install/examples/security

  4. Seleccione la carpeta cart-secure.

  5. Seleccione la casilla de verificación Abrir proyectos requeridos.

  6. Haga clic en Abrir proyecto.

  7. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto cart-secure y seleccione Construir.

    Este paso compila y empaqueta la aplicación en cart-secure.ear, ubicado en el directorio cart-secure-ear/target/, e implementa este archivo EAR en su servidor GlassFish. instancia, recupera los esbozos del cliente y ejecuta el cliente.

  8. En el cuadro de diálogo Iniciar sesión para el usuario:, ingrese el nombre de usuario y la contraseña de un usuario del dominio file creado en el servidor GlassFish y asignado al grupo TutorialUser; luego haga clic en Aceptar.

    Si el nombre de usuario y la contraseña que ingresa están autenticados, la salida del cliente de la aplicación aparece en la pestaña Salida: 

                            
...
Retrieving book title from cart: Infinite Jest
Retrieving book title from cart: Bel Canto
Retrieving book title from cart: Kafka on the Shore
Removing "Gravity's Rainbow" from cart.
Caught a BookException: "Gravity's Rainbow" not in cart.
Java Result: 1
...

    Si el nombre de usuario y la contraseña no están autenticados, el cuadro de diálogo vuelve a aparecer hasta que ingrese los valores correctos.

El ejemplo converter-secure: Asegurar un Enterprise Bean con Seguridad Programática

Este ejemplo demuestra cómo utilizar los métodos getCallerPrincipal y isCallerInRole con un enterprise bean. Este ejemplo comienza con una aplicación EJB muy simple, converter, y modifica los métodos de ConverterBean para que la conversión de moneda ocurra solo cuando el solicitante tiene el rol de TutorialUsuario.

Este ejemplo se puede encontrar en el directorio _tut-install_/examples/security/converter-secure. Este ejemplo se basa en la aplicación no segura Enterprise Bean, converter, que se analiza en el Capítulo 36, "Introducción a los Enterprise Beans" y se encuentra en el directorio _tut-install_/examples/ejb/converter/. Esta sección se basa en el ejemplo al agregar los elementos necesarios para proteger la aplicación mediante los métodos getCallerPrincipal y isCallerInRole, que se analizan con más detalle en Acceder al Contexto de Seguridad de un Llamador de Enterprise Bean .

En general, los siguientes pasos son necesarios cuando se utilizan los métodos getCallerPrincipal y isCallerInRole con un enterprise bean. En la aplicación de ejemplo incluida en este tutorial, muchos de estos pasos se han completado y se enumeran aquí simplemente para mostrar lo que se debe hacer si desea crear una aplicación similar.

  1. Cree una aplicación de bean empresarial simple.

  2. Configure un usuario en el servidor GlassFish en el dominio file, en el grupo TutorialUser, y configure el principal predeterminado para la asignación de funciones. Para ello, siga los pasos de Cómo Configurar su Sistema para Ejecutar los Ejemplos de Seguridad .

  3. Modifique el bean para agregar los métodos getCallerPrincipal y isCallerInRole.

  4. Si la aplicación contiene un cliente web que es un servlet, especifique la seguridad para el servlet, como se describe en Especificación de Seguridad para la Autenticación Básica Mediante Anotaciones .

  5. Cree, empaquete, implemente y ejecute la aplicación.

Modificación de ConverterBean

El código fuente de la clase ConverterBean original se modificó para agregar la cláusula if..else que prueba si la persona que llama tiene el rol de TutorialUser. Si el usuario tiene el rol correcto, la conversión de moneda se calcula y se muestra. Si el usuario no tiene el rol correcto, no se realiza el cálculo y la aplicación muestra el resultado como 0. El código de ejemplo se puede encontrar en _tut-install_/examples/security/converter-secure/converter-secure-ejb/ src/main/java/javaeetutorial/converter/ejb/ConverterBean.java .

El fragmento de código (con las modificaciones que se muestran en negrita) es el siguiente: 

                  
package javaeetutorial.convertersecure.ejb;
import java.math.BigDecimal;
import java.security.Principal;
import javax.ejb.Stateless;
import javax.annotation.Resource;
import javax.ejb.SessionContext;
import javax.annotation.security.DeclareRoles;
import javax.annotation.security.RolesAllowed;
@Stateless()
@DeclareRoles("TutorialUser")
public class ConverterBean{
    @Resource SessionContext ctx;
    private final BigDecimal yenRate = new BigDecimal("104.34");
    private final BigDecimal euroRate = new BigDecimal("0.007");
    @RolesAllowed("TutorialUser")
     public BigDecimal dollarToYen(BigDecimal dollars) {
        BigDecimal result = new BigDecimal("0.0");
        Principal callerPrincipal = ctx.getCallerPrincipal();
        if (ctx.isCallerInRole("TutorialUser")) {
            result = dollars.multiply(yenRate);
            return result.setScale(2, BigDecimal.ROUND_UP);
        } else {
            return result.setScale(2, BigDecimal.ROUND_UP);
        }
    }
    @RolesAllowed("TutorialUser")
    public BigDecimal yenToEuro(BigDecimal yen) {
        BigDecimal result = new BigDecimal("0.0");
        Principal callerPrincipal = ctx.getCallerPrincipal();
        if (ctx.isCallerInRole("TutorialUser")) {
             result = yen.multiply(euroRate);
             return result.setScale(2, BigDecimal.ROUND_UP);
        } else {
             return result.setScale(2, BigDecimal.ROUND_UP);
        }
    }
}
