Enlazando Convertidores, Oyentes, y Validadores a Propiedades de un Bean Gestionado

Como se describe en Agregando Componentes a Una Página Usando la Librería de Etiquetas de Etiquetas HTML , un autor de página puede vincular implementaciones de convertidor, oyente y validador a las propiedades de un bean administrado utilizando los atributos binding de las etiquetas que se utilizan para registrar las implementaciones en los componentes.

Esta técnica tiene ventajas similares a las de vincular instancias de componentes a propiedades de beans gestionados, como se describe en Enlazando Valores e Instancias de Componente a Propiedades del Bean Gestionado . En particular, vincular una implementación de convertidor, oyente o validador a una propiedad de bean gestionado produce los siguientes beneficios.

  • El bean gestionado puede crear una instancia de la implementación en lugar de permitir que el autor de la página lo haga.

  • El bean gestionado puede modificar programáticamente los atributos de la implementación. En el caso de una implementación personalizada, la única otra forma de modificar los atributos fuera de la clase de implementación sería crear una etiqueta personalizada para ella y requerir que el autor de la página establezca los valores de los atributos de la página.

Ya sea que esté vinculando un convertidor, oyente o validador a una propiedad de bean gestionado, el proceso es el mismo para cualquiera de las implementaciones.

  • Anide la etiqueta de convertidor, oyente o validador dentro de una etiqueta de componente adecuada.

  • Asegúrese de que el bean gestionado tenga una propiedad que acepte y devuelva la clase de implementación del convertidor, oyente o validador que desea vincular a la propiedad.

  • Haga referencia a la propiedad del bean gestionado utilizando una expresión de valor del atributo binding del convertidor, oyente o etiqueta de validación.

Por ejemplo, digamos que desea vincular el convertidor DateTime estándar a una propiedad de bean gestionado porque desea establecer el patrón de formato de la entrada del usuario en el bean gestionado en lugar de en la página Facelets. Primero, la página registra el convertidor en el componente anidando la etiqueta f:convertDateTime dentro de la etiqueta del componente. Luego, la página hace referencia a la propiedad con el atributo binding de la etiqueta f:convertDateTime: 


<h:inputText value="#{loginBean.birthDate}">
    <f:convertDateTime binding="#{loginBean.convertDate}" />
</h:inputText>

La propiedad convertDate se vería así:


private DateTimeConverter convertDate;
public DateTimeConverter getConvertDate() {
    ...
    return convertDate;
}
public void setConvertDate(DateTimeConverter convertDate) {
    convertDate.setPattern("EEEEEEEE, MMM dd, yyyy");
    this.convertDate = convertDate;
}

Vea Escribiendo Propiedades Ligadas a Convertidores, Oyentes, o Validadores para obtener más información sobre cómo escribir propiedades de bean gestionado para implementaciones de convertidor, oyente y validador.
