Cómo Escribir un Mouse-Motion Listener

Los eventos de movimiento del ratón se notifican cuando el usuario utiliza el ratón (o un dispositivo de entrada similar) para mover el cursor en pantalla. Para obtener información sobre cómo escuchar otros tipos de eventos del ratón, tales como las pulsaciones del ratón, vea Cómo Escribir un Mouse Listener. Para información sobre la escucha de eventos de la rueda del ratón, vea Cómo Escribir un Mouse Wheel Listener.

Si una aplicación requiere la detección de los eventos de ratón y de los eventos de movimientos del ratón, use la clase MouseInputAdapter, la cual implementa MouseInputListener un interfaz conveniente que implementa ambos interfaces, MouseListener y MouseMotionListener.

Alternativamente, use la clase AWT correspondiente MouseAdapter, la cual implementa la interfaz MouseMotionListener, para crear un MouseMotionEvent y sobreescribe los métodos para los eventos específicos.

La demostración siguiente contiene un mouse-motion listener. Esta demostración es exactamente la misma que la demostración descrita en la sección Cómo Escribir un Mouse Listener, excepto que substituye el interfaz MouseMotionListener por el interfaz MouseListener. Adicionalmente, MouseMotionEventDemo implementa los métodos mouseDragged y mouseMoved en vez de los métodos del mouse listener, y visualiza las coordenadas en vez de la cantidad de pulsaciones.

Intente esto: 
  1. Pulse el botón Lanzar para ejecutar MouseMotionEventDemo usando Java™ Web Start ( descargue KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo por sí mismo, consulte el índice de ejemplos.
  2. Mueva el cursor dentro del rectángulo amarillo en lo alto de la ventana.
    Verá uno o más eventos de movimiento del ratón.
  3. Presione y mantenga pulsado el botón del ratón, y después mueva el ratón de forma que el cursor salga del rectángulo amarillo.
    Verá eventos de arrastre del ratón.

Puede encontrar el código de la demostración en MouseMotionEventDemo.java y BlankArea.java. El trozo de código siguiente de MouseMotionEventDemo implementa el manejo del evento de movimiento del ratón: 

public class MouseMotionEventDemo extends JPanel
                                  implements MouseMotionListener {
    //...in initialization code:
        //Register for mouse events on blankArea and panel.
        blankArea.addMouseMotionListener(this);
        addMouseMotionListener(this);
        ...
    }

    public void mouseMoved(MouseEvent e) {
       saySomething("Mouse moved", e);
    }

    public void mouseDragged(MouseEvent e) {
       saySomething("Mouse dragged", e);
    }

    void saySomething(String eventDescription, MouseEvent e) {
        textArea.append(eventDescription
                        + " (" + e.getX() + "," + e.getY() + ")"
                        + " detected on "
                        + e.getComponent().getClass().getName()
                        + newline);
    }
}

El ejemplo SelectionDemo, dibuja un rectángulo que ilusta el arrastre actual del usuario. Para dibujar el rectángulo, la aplicación debe implementar un gestor de eventos para las tres clases de eventos del ratón: pulsaciones del ratón, arrastres del ratón, y liberaciónes del ratón. Para informarse de todos estos eventos, el manejador deber implementar ambos interfaces, MouseListener y MouseMotionListener, y registrarse a la vez como mouse listener y mouse-motion listener. Para evitar el tener que definir métodos vacíos, el manejador no implementa directamente el interfaz del listener. En vez de eso, extiende MouseInputAdapter, como muestra el trozo de código siguiente. 

...//where initialization occurs:
    MyListener myListener = new MyListener();
    addMouseListener(myListener);
    addMouseMotionListener(myListener);
...
private class MyListener extends MouseInputAdapter {
    public void mousePressed(MouseEvent e) {
        int x = e.getX();
        int y = e.getY();
        currentRect = new Rectangle(x, y, 0, 0);
        updateDrawableRect(getWidth(), getHeight());
        repaint();
    }

    public void mouseDragged(MouseEvent e) {
        updateSize(e);
    }

    public void mouseReleased(MouseEvent e) {
        updateSize(e);
    }

    void updateSize(MouseEvent e) {
        int x = e.getX();
        int y = e.getY();
        currentRect.setSize(x - currentRect.x,
                            y - currentRect.y);
        updateDrawableRect(getWidth(), getHeight());
        Rectangle totalRepaint = rectToDraw.union(previouseRectDrawn);
        repaint(totalRepaint.x, totalRepaint.y,
                totalRepaint.width, totalRepaint.height);
    }
}

La IPA de Mouse-Motion Listener

La Interfaz MouseMotionListener

Las clases adaptadoras correspondientes son MouseMotionAdapter y MouseAdapter.

La IPA de MouseMotionListener
Método Propósito
mouseDragged(MouseEvent) Llamado en respuesta a cuando el usuario mueve el ratón mientras mantiene pulsado un botón del ratón. Este evento se dispara por el componente que disparó el evento de ratón presionado más reciente, incluso si el cursor ya no se encuentra encima de ese componente.
mouseMoved(MouseEvent) Llamada en respuesta a cuando el usuario mueve el ratón sin ningún botón del ratón presionado, Este evento se dispara por el componente que está actualmente bajo el cursor.

Cada método del evento de movimiento del ratón tiene un parámetro único ─ ¡y no se llama MouseMotionEvent! En vez de eso, cada método del evento de movimiento del ratón usa un argumento MouseEvent. Vea La IPA de MouseEvent para información sobre el uso de objetos MouseEvent.
