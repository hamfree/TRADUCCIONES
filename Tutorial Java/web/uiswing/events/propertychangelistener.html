Cómo Escribir un Property Change Listener

Los eventos de cambio de propiedad ocurren siempre que el valor de una propiedad vinculada cambie en un bean ─ componente que cumple la especificación JavaBeans™. Puede encontrar más información sobre los beans en el recorrido JavaBeans del Tutorial de Java. Todos los componentes de Swing son también beans.

Una propiedad de JavaBeans es accedida mediante sus métodos get y set. Por ejemplo, JComponent tiene la propiedad font la cual es accesible a través de los métodos getFont y setFont.

Aparte de los métodos get y set, una propiedad vinculada dispara un evento de cambio de propiedad cuando su valor cambia. Para más información, vea Propiedades Vinculadas en la página del recorrido JavaBeans.

Algunos escenarios que requieren comunmente property-change listeners incluyen:

  • Ha implementado un campo de texto formateado y necesita una forma de detectar cuándo el usuario ha introducido un valor nuevo. Puede registrar un property-change listener en el campo de texto formateado para monitorizar los cambios en la propiedad value. Vea FormattedTextFieldDemo en Cómo Usar Campos de Texto Formateado para un ejemplo de esto.
  • Ha implementado un diálogo y necesita saber cuándo un usuario ha pulsado uno de los botones del diálogo o cambia una selección en el diálogo. Vea DialogDemo en Cómo Hacer Diálogos para un ejemplo de registrar un property-change listener en un diálogo para monitorizar los cambios a la propiedad value. Puede también comprobar FileChooserDemo2 en Cómo Usar Selectores de Fichero por un ejemplo de cómo registrar un property-change listener para monitorizar los cambios en las propiedades directoryChanged y selectedFileChanged.
  • Necesita ser notificado cuándo el componente que tiene el foco cambia. Puede registrar un property-change listener en el gestor del foco del teclado para monitorizar los cambios en la propiedad focusOwner. Vea TrackFocusDemo y DragPictureDemo en Cómo Usar el Subsistema de Foco para ejemplos de esto.

Aunque estos son algunos de los usos más comunes para los property-change listeners, puede registrar un property-change listener en la propiedad vinculada de cualquier componente que cumpla la especificación JavaBeans.

Puede registrar un property-change listener de dos formas. La primera usa el método addPropertyChangeListener(PropertyChangeListener). Cuando registra un listener de esta forma, será notificado de cada cambio en cada propiedad vinculada de ese objeto. En el método propertyChange, puede obtener el nombre de la propiedad que ha cambiado usando el método getPropertyName de PropertyChangeEvent, como en el trozo de código siguiente: 

KeyboardFocusManager focusManager =
   KeyboardFocusManager.getCurrentKeyboardFocusManager();
focusManager.addPropertyChangeListener(new FocusManagerListener());
...
public FocusManagerListener implements PropertyChangeListener {
    public void propertyChange(PropertyChangeEvent e) {
        String propertyName = e.getPropertyName();
        if ("focusOwner".equals(propertyName) {
            ...
        } else if ("focusedWindow".equals(propertyName) {
            ...
        }
    }
    ...
}

La segunda forma de registrar un property change listener usa el método addPropertyChangeListener(String, PropertyChangeListener). El argumento String es el nombre de la propiedad. Si usa este método usted sólo recibe una notificación cuando ocurre un cambio a esa propiedad en particular. Así, por ejemplo, si registró un property change listener como este: 

aComponent.addPropertyChangeListener("font",
                                     new FontListener());

FontListener sólo recibe notificaciones cuando el valor de la propiedad font del componente cambia. No recibe notificaciones cuando el valor cambia en transferHandler, opaque, border, o cualquier otra propiedad.

El ejemplo siguiente muestra cómo registrar un property change listener en la propiedad value de un campo de texto formateado usando la versión de dos argumentos de addPropertyChangeListener: 

//...where initialization occurs:
double amount;
JFormattedTextField amountField;
...
amountField.addPropertyChangeListener("value",
                                      new FormattedTextFieldListener());
...
class FormattedTextFieldListener implements PropertyChangeListener {
    public void propertyChanged(PropertyChangeEvent e) {
        Object source = e.getSource();
        if (source == amountField) {
            amount = ((Number)amountField.getValue()).doubleValue();
            ...
        }
        ...//re-compute payment and update field...
    }
}

La IPA de Property Change Listener

Registrar un PropertyChangeListener

La IPA de Property Change Listener
Método Propósito
addPropertyChangeListener(PropertyChangeListener) Agregar un property-change listener a la lista de listener.
addPropertyChangeListener(String, PropertyChangeListener) Agrega un property-change listener para una propiedad específica. Se llama al listener sólo cuando hay un cambio en la propiedad especificada.

El Interfaz PropertyChangeListener

Ya que PropertyChangeListener tiene sólo un métood, no tiene su clase adaptadora correspondiente.

El Interfaz PropertyChangeListener
Método Propósito
propertyChange(PropertyChangeEvent) Llamado cuando el bean monitorizado cambia una propiedad vinculada.

La Clase PropertyChangeEvent

Método Propósito
Object getNewValue()
Object getOldValue() 		Devuelve el valor de la propiedad, nueva, o vieja, respectivamente.
String getPropertyName() Devuelve el nombre de la propiedad que fue cambiada.
void setPropagationId() Obtiene o establece el valor del ID de propagación. Reservada para uso futuro.
