Cómo Escribir un Table Model Listener

Tras la instanciación, a cada objeto JTable se le pasa un objeto de modelo de tabla que gestiona los datos que muestra. Por defecto, un objeto JTable hereda un objeto DefaultTable si no se especificó un objeto TableModel personalizado, pero por defecto, este modelo sólo gestiona cadenas. Para manejar objetos, realizar cálculos, o recuperar datos desde bases de datos u otros programas, debe diseñar su propio objeto TableModel personalizado, el cual implementa el interfaz TableModel. Vea Crear un Modelo de Tabla para los detalles.

Para detectar los cambios a los datos gestionados por un objeto de modelo de tabla, la clase JTable necesita implementar el interfaz TableModelListener, llamar a addTableModelListener() para capturar eventos, y entonces sobreescribir tableChanged() para responder a los eventos del listener. Vea Monitorizar Cambios de Datos para los detalles.

La IPA de Table Model Listener

La Interfaz de TableModelListener

Ya que TableModelListener sólo tiene un método, no tiene clase adaptadora correspondiente.

La Interfaz de TableModelListener
Método Propósito
tableChanged(TableModelEvent) Llamada cuando la estructura o los datos de la tabla ha cambiado.

La IPA de TableModelEvent

Método Propósito
Object getSource()
(en java.util.EventObject) 		Devuelve el objeto que disparó el evento.
int getFirstRow() Devuelve el índice de la primera fila que cambió. TableModelEvent.HEADER_ROW especifica la cabecera de la tabla.
int getLastRow() La última fila que cambió. De nuevo, HEADER_ROW es un valor posible.
int getColumn() Devuelve el índice de la columna que cambió. La constante TableModelEvent.ALL_COLUMNS especifica que todas las columnas habrían cambiado.
int getType() Lo que ocurrió a las celdas cambiadas. El valor devuelto es uno de los siguientes: TableModelEvent.INSERT, TableModelEvent.DELETE, o TableModelEvent.UPDATE.
