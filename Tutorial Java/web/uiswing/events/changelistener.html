Cómo Escribir un Change Listener

Un change listener es similar a un property change listener. Un change listener se registra sobre un objeto — normalmente un componente, pero podría ser otro objeto, como un modelo — y se notifica al listener cuando el objeto ha cambiado. La gran diferencia con un property change listener es que a un change listener no se le notifica lo que ha cambiado, si no que simplemente el objeto fuente ha cambiado. Por lo tanto, un change listener es más útil cuando sólo es necesario saber cuándo un objeto ha cambiado de alguna manera.

Varios componentes Swing (incluyendo JTabbedPane, JViewPort) confían en los eventos de cambio para su funcionalidad básica — controles deslizantes, selectores de color y contadores numéricos. Para aprender cuándo el valor en un contador numérico cambia, necesita registrar un change listener. De igual manera, necesita registrar un change listener en un selector de color para ser informado cuando selecciona un nuevo color. Usted registra un change listener en un contador numérico, para ser notificado cuando el valor de un contador numérico cambia.

Aquí tiene un ejemplo de código de manejo de evento de cambio para un control deslizante:
