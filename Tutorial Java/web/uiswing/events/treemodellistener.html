Cómo Escribir un Tree Model Listener

Al implementar un tree model listener, puede detectar cuándo los datos visualizados por un árbol cambian. Puede usar un tree model listener para detectar cuándo el usuario edita nodos del árbol. Todas las notificaciones describen cambios relativos a un nodo en el árbol. Para los detalles, lea Cambiar Dinámicamente un Árbol.

La IPA de Tree Model Listener

El Interfaz de TreeModelListener

TreeModelListener no tiene clase adaptadora.

El Interfaz de TreeModelListener
Método Propósito
treeNodesChanged(TreeModelEvent) Llamado cuando uno o más nodos hermanos han cambiado de alguna forma.
treeNodesInserted(TreeModelEvent) Llamado después de que se hayan insertado nodos en el árbol.
treeNodesRemoved(TreeModelEvent) Llamado después de que se hayan eliminado nodos del árbol.
treeStructureChanged(TreeModelEvent) Llamado después de que la estructura del árbol haya cambiado drásticamente desde el nodo actual hacia abajo. Este evento aplica a todos los nodos conectados a éste nodo.

La IPA de TreeModelEvent

Método Propósito
Object getSource()
(en java.util.EventObject) 		Devuelve el objeto que disparó el evento.
int[] getChildIndices() Para treeNodesChanged, treeNodesInserted, y treeNodesRemoved, devuelve los índices de los nodos cambiados, insertados, o borrados, respectivamente. No devuelve nada útil para treeStructureChanged.
Object[] getChildren() Devuelve los objetos correspondientes a los índices hijos.
Object[] getPath() Devuelve la ruta al padre de los nodos cambiados, insertados, o borrados. Para treeStructureChanged, devuelve la ruta al nodo debajo del cual la estructura ha cambiado.
TreePath getTreePath() Devuelve lo mismo que getPath, pero como un objeto TreePath.
