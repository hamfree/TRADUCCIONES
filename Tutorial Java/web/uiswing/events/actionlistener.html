Cómo Escribir un Action Listener

Los action listeners son probablemente los manejadores de eventos más fáciles — y más comunes — de implementar. Se implementa un action listener para definir que debería ser hecho cuando un usuario realiza cierta operación.

Un evento de acción ocurre, siempre que una acción es realizada por el usuario. Ejemplos: Cuando el usuario pulsa un botón, elige un elemento de menú, presiona Intro en un campo de texto. El resultado es que un mensaje actionPerformed se envía a todos los action listeners que están registrados en el componente relevante.

Para escribir un Action Listener, siga los pasos dados abajo:

  1. Declare una clase de manejo de eventos y especifique que la clase o implementa un interfaz de ActionListener o extiende una clase que implementa un interfaz de ActionListener. Por ejemplo: 
    public class MyClass implements ActionListener {
  2. Registre una instancia de la clase del manejador de eventos como un listener en uno o más componentes. Por ejemplo: 
    someComponent.addActionListener(instanceOfMyClass);
  3. Incluye código que implemente los métodos del interfaz del listener. Por ejemplo: 
    public void actionPerformed(ActionEvent e) {
    ...//code that reacts to the action...
}

En general, para detectar cuando el usuario pulsa en un botón en pantalla (o realiza su equivalente en el teclado), un progrma debe tener un objeto que implementa el interfaz ActionListener. El programa debe registrar este objeto como un action listener en el botón (la fuente del evento), usando el método addActionListener. Cuando el usuario pulsa el botón en pantalla, el botón dispara un evento de acción. Esto resulta en la invocación del método actionPerformed del action listener (el único método en la interfaz ActionListener). El único argumento al método es un objeto ActionEvent que da información sobre el evento y su origen.

Escribamos un programa simple que muestra cuantas vences el usuario ha pulsado un botón. Primero, aquí está el código que configura el campo de texto, el botón y la variable numClicks: 

public class AL extends Frame implements WindowListener,ActionListener {
TextField text = new TextField(20);
Button b;
private int numClicks = 0;

En el ejemplo de arriba, la clase manejadora de ventos es AL que implementa ActionListener.

Nos gustaría manejar el evento de clic de botón, por lo que agregamos un action listener al botón b como se muestra a continuación: 

b = new Button("Click me");
b.addActionListener(this);

En el código de arriba, Button b es un componente en el que se registra una instancia de la clase manejadora de eventos AL.

Ahora, queremos mostrar el texto de cuantas veces el usuario pulsó el botón. Podemos hacer esto escribiendo el código de debajo:: 

public void actionPerformed(ActionEvent e) {
         numClicks++;
         text.setText("Button Clicked " + numClicks + " times");

Ahora, cuando el usuario pulsa el botón b, el botón dispara un evento de acción el cual invoca el método actionPerfomed del action listener. Cada vez que el usuario presiona el botón, la variable numClicks se agrega y el mensaje es mostrado en el campo de texto.

Aquí tiene el programa completo (AL.java): 


import java.awt.*;
import java.awt.event.*;

public class AL extends Frame implements WindowListener,ActionListener {
        TextField text = new TextField(20);
        Button b;
        private int numClicks = 0;

        public static void main(String[] args) {
                AL myWindow = new AL("My first window");
                myWindow.setSize(350,100);
                myWindow.setVisible(true);
        }

        public AL(String title) {

                super(title);
                setLayout(new FlowLayout());
                addWindowListener(this);
                b = new Button("Click me");
                add(b);
                add(text);
                b.addActionListener(this);
        }

        public void actionPerformed(ActionEvent e) {
                numClicks++;
                text.setText("Button Clicked " + numClicks + " times");
        }

        public void windowClosing(WindowEvent e) {
                dispose();
                System.exit(0);
        }

        public void windowOpened(WindowEvent e) {}
        public void windowActivated(WindowEvent e) {}
        public void windowIconified(WindowEvent e) {}
        public void windowDeiconified(WindowEvent e) {}
        public void windowDeactivated(WindowEvent e) {}
        public void windowClosed(WindowEvent e) {}

}

El otro ejemplo descrito en esa sección, MultiListener.java, tiene dos fuentes de acción y dos action listeners, con un listener monitorizando ambas fuentes y el otro monitorizando sólo una.

La IPA de Action Listener

El Interfaz de ActionListener

Debido a que ActionListener sólo tiene un método, no tiene una clase adaptador correspondiente.

interfaz ActionListener
Método Propósito
actionPerformed(actionEvent) Llamada justo después de que el usuario realize una acción.

La Clase ActionEvent

clase ActionEvent
Método Propósito
String getActionCommand() Devuelve la cadena asociada con esta acción. La mayoría de los objetos pueden disparar eventos de acción soportan un metodo llamado setActionCommand que le permite establecer esta cadena.
int getModifiers() Devuelve un entero que representa las teclas modificadora que el usuario estaba presionando cuando el evento de acción ocurrió. Puede usar las constantes definidas en ActionEvent SHIFT_MASK,CTRL_MASK, META_MASK, y ALT_MASK para determinar qué teclas fueron presionadas. Por ejemplo, si el usurio selecciona con Shift un elemento de menú, entonces la siguiente expresión es distinta de cero: 
actionEvent.getModifiers() & ActionEvent.SHIFT_MASK
Object getSource()
(in java.util.EventObject) 		Devuelve el objeto que disparó el evento.
