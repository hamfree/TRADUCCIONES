Cómo Escribir un Item Listener

Los eventos de elemento son disparados por componentes que implementan la interfaz ItemSelectable. Generalmente, los componentes ItemSelectable mantienen el estado encendido/apagado para uno o más elementos. Los componentes de Swing que disparan eventos de elemento incluyen botones como casillas de verificación, menús, botones de activación etc... y cuadros combinados.

Aquí hay un código de manejo de eventos de elemento tomado de ComponentEventDemo.java: 

//where initialization occurs
checkbox.addItemListener(this);
...
public void itemStateChanged(ItemEvent e) {
    if (e.getStateChange() == ItemEvent.SELECTED) {
        label.setVisible(true);
        ...
    } else {
        label.setVisible(false);
    }
}

La IPA de Item Listener

El Interfaz de ItemListener

Ya que ItemListener tiene sólo un método, no tiene clase adaptadora correspondiente.

El Interfaz ItemListener
Método Propósito
itemStateChanged(ItemEvent) Llamada justo después de un cambio de estado en un componente monitorizado.

La Clase ItemEvent

La Clase ItemEvent
Método Propósito
Object getItem() Devuelve el objeto específico del componente asociado con el elemento cuyo estado cambió. A menudo éste es un objeto String conteniendo el texto del elemento seleccionado.
ItemSelectable getItemSelectable() Devuelve el componente que disparó el evento de elemento. Puede usar esto en vez del método getSource.
int getStateChange() Devuelve el nuevo estado del elemento. La clase ItemEvent define dos estados: SELECTED y DESELECTED.
