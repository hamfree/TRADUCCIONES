Respuestas: Escribiendo Listener Events

Use estas tablas de la lección, la Lección: Usando Componentes Swing e Implementando Listeners para los Eventos Comunmente Manejados para completar estas preguntas y los ejercicios.

Preguntas

Pregunta 1: ¿Qué listener debe implementar para recibir una notificación cuando un componente particular ha aparecido en la pantalla? ¿Qué método le dice esta información?
Respuesta 1: Debe registrar un ComponentListener en el componente. El método componentShown. Se llama a este método cuando se muestra por primera vez la ventana o se desiconifica.

Pregunta 2: ¿Qué listener debe implementar para recibir una notificación cuando el usuario ha terminado de editar un campo de texto al presionar Intro? ¿Qué listener debe implementar para recibir una notificación a medida que se escribe cada caracter en un campo de texto? Dese cuenta que no debe implementar un key listener de propósito general , sino un listener específico para texto.
Respuesta 2: Para recibir una notificación cuando el usuario presiona Intro, debe registrar un ActionListener en el campo de texto; el método actionPerformed se llama cuando el usuario presiona Intro. Note que el carácter Intro no es parte de la cadena resultante. Para recibir una notificación a medida que se escribe cada carácter, debe registrar un DocumentListener en el Document del campo de texto. Se llama al método insertUpdate a medida que se escribe cada carácter. Note que ésta no es la forma correcta de implementar la validación de entrada. Para ese comportamiento compruebe la sección La IPA de Verificación de Entradas en Cómo Usar el Subsistema de Foco.

Pregunta 3: ¿Qué listener debe implementar para recibir una notificación cuando el valor de un contador numérico ha cambiado? ¿Cómo debe obtener el nuevo valor del contador numérico?
Respuesta 3: Para recibir una notificación cuando el valor ha cambiado, debe registrar un ChangeListener en el contador numérico. Obtendrá el nuevo valor a través de la fuente del evento en el método stateChanged. El trozo de código siguiente muestra como podría ser hecho esto: 

public void stateChanged(ChangeEvent e) {
    JSpinner mySpinner = (JSpinner)(e.getSource());
    SpinnerNumberModel model = (SpinnerNumberModel)(mySpinner.getModel());
    Number currentValue = model.getNumber();
    ...
}

Pregunta 4: El comportamiento por defecto para el subistema de foco es consumir las teclas de salto del foco, tales como Tabulador y Mayúsculas-Tabulador. Digamos que quiere prevenir que esto pase en uno de los componentes de su aplicación. ¿Cómo lograría esto?
Respuesta 4: Llame a setFocusTraversalKeysEnabled(false) en ese componente particular. Note que ustede debe manejar el salto de foco manualmente. Vea Cómo Escribir un Key Listener y Cómo Usar el Subsistema de Foco para más información.

Ejercicios

Ejercicio 1. Tome el ejemplo Beeper.java y agregue un campo de texto. Impleméntelo de forma que cuando el usuario ha terminado de introducir datos, el sistema pite.
Respuesta 1: Vea Beeper1.java

Ejercicio 2. Tome el ejemplo Beeper.java y agregue un componente seleccionable que le permita al usuario introducir un número del 1 al 10. Por ejemplo, puede usar un cuadro combinado, un conjunto de botones de opción, o un contador numérico. Impleméntelo de forma que cuando el usuario ha seleccionado el número, el sistema emita un pitido tantas veces.
Respuesta 2: Vea Beeper2.java
