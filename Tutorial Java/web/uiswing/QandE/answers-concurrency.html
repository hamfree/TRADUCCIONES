Respuestas: Concurrencia en Swing

Preguntas

Pregunta 1: Para cada una de las siguientes tareas, especifique en qué hilo debería ser ejecutado y porqué.
Respuesta 1:

  • Iniciar el IGU. El hilo despachador de eventos; la mayoría de las interacciones con el marco de trabajo del IGU deben ocurrir en este hilo.
  • Cargar un gran fichero. Un hilo de trabajo. Ejecutar esta tarea en el hilo despachador de eventos prevendría a los eventos del IGU de ser procesados, "congelando" el IGU hasta que la tarea se acabe. Ejecutar esta tarea en un hilo inicial causaría un retraso en la creación del IGU.
  • Invocar javax.swing.JComponent.setFont para cambiar la fuente de un componente. El hilo despachador de eventos. Como con la mayoría de los métodos de Swing, no es seguro invocar setFont desde cualquier otro hilo.
  • Invocar javax.swing.text.JTextComponent.setText para cambiar el texto de un componente. Este método está documentado como seguro en multihilo, así que puede ser invocado desde cualquier hilo.

Pregunta 2: Un conjunto de hilos no es usado para ninguna de las tareas mencionadas en la pregunta previa. Nombre este hilo y explique porqué sus aplicaciones son muy limitadas.
Respuesta 2: Los hilos iniciales lanzan la primera tarea del IGU en el hilo despachador de eventos. Después de eso, un programa Swing se conduce primariamente por los eventos del IGU, los cuales disparan tareas en el hilo despachador de eventos y en el hilo de trabajo. Usualmente, los hilos iniciales quedan sin nada que hacer.

Pregunta 3: SwingWorker tiene dos parámetros de tipos. Explique cómo estos parámetros de tipos son usados, y porque a menudo no importa cuáles son.
Respuesta 3: Los parámetros tipo especifican el tipo del resultado final (también el tipo de vuelta del método doInBackground) y el tipo de los resultados provisionales (también los argumentos tipo para publish y process). Muchas tareas de fondo no suministran resultados provisionales o finales.

Ejercicios

Pregunta 1: Modifique el ejemplo Flipper de forma que haga una pausa de 5 segundos entre las "tiradas de monedas." Si el usuario pulsa "Cancelar", el bucle de lanzamiento de monedas termina inmediatamente.
Respuesta 1: Vea el código fuente para Flipper2. El programa modificado añade un retraso en el bucle central doInBackground: 

protected Object doInBackground() {
    long heads = 0;
    long total = 0;
    Random random = new Random();
    while (!isCancelled()) {
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
        //¡Cancelado!
        return null;
        }
        total++;
        if (random.nextBoolean()) {
            heads++;
        }
        publish(new FlipPair(heads, total));
    }
    return null;
}

El try ... catch causa que doInBackground vuelva si una interrupción es recibida mientas el hilo está durmiendo. Invocar cancel con un argumento true asegura que una interrupción sea enviada cuando la tarea se cancela.
