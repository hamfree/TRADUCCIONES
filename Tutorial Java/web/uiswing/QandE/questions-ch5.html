Preguntas y Ejercicios: Escribiendo Listener Events

Use estas tablas de la lección, la Lección: Usando Componentes Swing e Implementando Listeners para los Eventos Comunmente Manejados para completar estas preguntas y los ejercicios.

Preguntas

1. ¿Qué listener debe implementar para recibir una notificación cuando un componente particular ha aparecido en la pantalla? ¿Qué método le dice esta información?

2. ¿Qué listener debe implementar para recibir una notificación cuando el usuario ha terminado de editar un campo de texto al presionar Intro? ¿Qué listener debe implementar para recibir una notificación a medida que se escribe cada caracter en un campo de texto? Dese cuenta que no debe implementar un key listener de propósito general , sino un listener específico para texto.

3. ¿Qué listener debe implementar para recibir una notificación cuando el valor de un contador numérico ha cambiado? ¿Cómo debe obtener el nuevo valor del contador numérico?

4. El comportamiento por defecto para el subistema de foco es consumir las teclas de salto del foco, tales como Tabulador y Mayúsculas-Tabulador. Digamos que quiere prevenir que esto pase en uno de los componentes de su aplicación. ¿Cómo lograría esto?

Ejercicios

1. Tome el ejemplo Beeper.java y agregue un campo de texto. Impleméntelo de forma que cuando el usuario ha terminado de introducir datos, el sistema pite.

2. Tome el ejemplo Beeper.java y agregue un componente seleccionable que le permita al usuario introducir un número del 1 al 10. Por ejemplo, puede usar un cuadro combinado, un conjunto de botones de opción, o un contador numérico. Impleméntelo de forma que cuando el usuario ha seleccionado el número, el sistema emita un pitido tantas veces.

Compruebe sus respuestas.
