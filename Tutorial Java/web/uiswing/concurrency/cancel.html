Cancelando Tareas en Segundo Plano

Para cancelar un tarea de segundo plano ejecutándose, invoque SwingWorker.cancel La tarea debe cooperar con su propia cancelación. Hay dos formas de que se pueda hacer esto:

  • Terminando cuando recibe una interrupción. Este procedimiento es descrito en Interrupciones en Concurrencia.
  • Invocando SwingWorker.isCancelled en intervalos pequeños. Este método devuelve true si cancel ha sido invocado desde este SwingWorker.

El método cancel toma un argumento único booleano. Si el argumento es true, cancel envia a la tarea en segundo plano una interrupción. Si el argumento es true o false, invocar cancel cambia el estado de la cancelación del objeto a true. Este es el valor devuelto por isCancelled. Una vez cambiado, el estado de cancelación no se puede cambiar de nuevo.

El ejemplo Flipper de la sección previa usa la expresión idiomática de estado solamente. El bucle principal en doInBackground existe cuando isCancelled devuelve true. Esto ocurrirá cuando el usuario pulse el botón "Cancelar", disparando código que invoca cancel con un argumento de false.

La aproximación de sólo-el-estado tiene sentido para Flipper porque la implementación de SwingWorker.doInBackground no incluye ningún código que podría lanzar InterruptedException. Para responder a una interrupción, la tarea de fondo debería invocar a Thread.isInterrupted em pequeños intervalos. Es simplemente tan fácil como usar SwingWorker.isCancelled para el mismo proposito.

Nota: Si get es invocado sobre un objeto SwingWorker después de que la tarea en segundo plano haya sido cancelada, una java.util.concurrent.CancellationException es lanzada.
