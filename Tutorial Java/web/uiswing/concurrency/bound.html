Propiedades Ligadas y Métodos de Estado

SwingWorker soporta propiedades ligadas, las cuales son útiles para comunicarse con otros hilos. Dos propiedades ligadas están predefinidas: progress y state. Como con todas las propiedades ligadas, progress y state pueden ser usadas para disparar tareas de manejo de evento en el hilo despachador de eventos.

Al implementar un oyente de cambios de propiedad, un programa puede rastrear los cambios en progress, state, y otras propiedades ligadas. Para más información, consulte Cómo Escribir un Property Change Listener en Escribiendo Listener Events.

La Variable Ligada progress

La variable ligada progress es un valor int que puede variar de 0 a 100. Tiene un método setter predefinido (el protegido SwingWorker.setProgress) y un método getter predefinido (el público SwingWorker.getProgress).

El ejemplo ProgressBarDemo usa progress para actualizar un control ProgressBar desde una tarea en segundo plano. Para una descripción detallada de este ejemplo, consulte Cómo Usar Barras de Progreso en Usando Componentes Swing.

La Variable Ligada state

La variable ligada state indica dónde está el objeto SwingWorker en su ciclo de vida. La variable ligada contiene un valor enumerado del tipo SwingWorker.StateValue. Los valores posibles son:

PENDING
El estado durante el periodo desde la construcción del objeto hasta justo antes de que se invoque doInBackground.
STARTED
El estado durante el periodo de poco antes de que doInBackground sea invocado hasta un poco antes de que se invoque done.
DONE
El estado por el resto de la existencia del objeto.

El valor actual de la variable ligada state es devuelto por SwingWorker.getState.

Métodos de Estado

Dos métodos, parte del interfaz Future, también informan sobre el estado de la tarea en segundo plano. Como vimos en Cancelando Tareas en Segundo Plano, isCancelled devuelve true si la tarea ha sido cancelada. Además de esto, isDone devuelve true si la tarea ha finalizado, ya sea normalmente, o siendo cancelada.
