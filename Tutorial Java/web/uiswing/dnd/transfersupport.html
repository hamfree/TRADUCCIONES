Clase TransferSupport

La clase TransferSupport tiene dos funciones. Como el nombre sugiere, su primera función es soportar el proceso de transferencia y para ese propósito ofrece varios métodos de utilidad usados para acceder a los detalles de la transferencia de datos. La lista siguiente muestra los métodos que pueden ser utilizados para obtener información del TransferHandler. Varios de estos métodos están relacionados con acciones de soltado, los cuales serán descritos en Estableciendo el Modo Soltar.

  • Component getComponent()— Este método devuelve el componente destino de la transferencia.
  • int getDropAction() — Este método devuelve la acción elegida (COPY, MOVE o LINK) cuando la transferencia es un gesto de soltado. Si la transferencia no es un gesto de soltado, este método lanza una excepción.
  • int getUserDropAction() — Este método devuelve la acción de soltado elegida por el usuario. Por ejemplo, si el usuario mantiene pulsadas las teclas Control y Mayúsuculas simultáneamente durante el gesto de arrastre, esto indica una acción ACTION_LINK. Para más información sobre las acciones de soltado del usuario, vea la IPA para DropTargetDragEvent. Si la transferencia no es un gesto de soltado, este método lanza una excepción.
  • int getSourceDropActions() — Este método devuelve el conjunto de acciones soportadas por el componente fuente. Si la transferencia no es un gesto de soltado, este método lanza una excepción.
  • DataFlavor[] getDataFlavors() — Este método devuelve todos los sabores de datos soportados por este componente. Por ejemplo, un componente podría soportar ficheros y texto, o texto y color. Si la transferencia no es un gesto de soltado, este método lanza una excepción.
  • boolean isDataFlavorSupported(DataFlavor) — Este método devuelve verdadero si se soporta el DataFlavor especificado. El DataFlavor indica el tipo de un dato representado, tal como una imagen (imageFlavor), una cadena (stringFlavor), una lista de ficheros (javaFileListFlavor), etc.
  • Transferable getTransferable() — Este método devuelve los datos de Transferable para esta transferencia. Es más eficiente utilizar uno de estos métodos para consultar información sobre la transferencia que buscar el transferible y consultarlo, por lo que este método no se recomienda a menos que no pueda obtener la información de otra manera.
  • DropLocation getDropLocation() — Este método devuelve la ubicación de soltado en el componente. Los componentes con soporte de soltado integrado, tales como la lista, la tabla y el árbol, sobreescriben este método para devolver datos más útiles. Por ejemplo, la versión de este método para el componente JList devuelve el índice en la lista donde el gesto de soltado ocurrió. Si la transferencia no es un gesto de soltado, este método lanza una excepción.

Ejemplo de Métodos de Importación

Ahora que está familiarizado con los métodos de utilidad de TransferSupport, veamos los métodos de muestra canImport y importData: 

public boolean canImport(TransferSupport supp) {
    // Verifica el sabor de cadena
    if (!supp.isDataFlavorSupported(stringFlavor)) {
        return false;
    }

    // Obtiene la ubicación de colocación
    DropLocation loc = supp.getDropLocation();

    // Devuelve si aceptamos la ubicación
    return shouldAcceptDropLocation(loc);
}

public boolean importData(TransferSupport supp) {
    if (!canImport(sup)) {
        return false;
    }

    // Obtiene el Transferable y sus datos
    Transferable t = supp.getTransferable();
    String data = t.getTransferData(stringFlavor);

    // Obtiene la ubicación de soltado
    DropLocation loc = supp.getDropLocation();

    // Inserta los datos en esta ubicación
    insertAt(loc, data);

    return true;
}
