Soporte de DnD por Defecto

Hablando técnicamente, el marco de trabajo para arrastrar y soltar soporta todos los componentes de Swing — el mecanismo de transferencia de datos está integrado en cada JComponent. Si usted quisiera, podría implementar el soporte de soltado para una JSlider de forma que podría participar plenamente en una transferencia de datos. Si bien JSlider no soporta el soltado por defecto, los componentes que querría (y esperaría) que soporten la función de arrastrar y soltar proporcionan soporte integrado especializado.

Los componentes siguientes reconocen el gesto de arrastre una vez que el método setDragEnabled(true) se invoca en el componente. Por ejemplo, una vez invoque myColorChooser.setDragEnabled(true) usted puede arrastrar colores desde su selector de color:

  • JColorChooser
  • JEditorPane
  • JFileChooser
  • JFormattedTextField
  • JList
  • JTable
  • JTextArea
  • JTextField
  • JTextPane
  • JTree

Los siguientes componentes son compatibles con el soltado tal cual. Si está usando uno de estos componentes, su trabajo ya está hecho.

  • JEditorPane
  • JFormattedTextField
  • JPasswordField
  • JTextArea
  • JTextField
  • JTextPane
  • JColorChooser

El marco de trabajo para soltar está en su lugar para los componentes siguientes, pero necesita conectar una pequeña cantidad de código para personalizar el soporte según sus necesidades.

  • JList
  • JTable
  • JTree

Para estos componentes críticos, Swing realiza los cálculos de la ubicación del soltado y representación; le permite especificar un modo de soltado; y maneja los detalles específicos del componente, tales como las expansiones del árbol. Su trabajo es bastante mínimo.

Nota: 

Puede también instalar el soporte de soltado en contenedores de nivel superior, tales como JFrame y JDialog. Puede aprender más sobre esto en Soltado Nivel Superior.
