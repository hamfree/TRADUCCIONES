Mostrando la Ubicación de lo Soltado

Generalmente durante una operación de arrastre, un componente devuelve retroalimentación visual cuando puede aceptar los datos. Podría resaltar la ubicación de soltado, o podría mostrar un símbolo de intercalación o una línea horizontal donde se produciría la inserción. Swing representa la ubicación de colocación cuando el método canImport para el componente TransferHandler devuelve verdadero.

Para controlar esto mediante programación, puede usar el método setShowDropLocation. Llamar a este método con true hace que la información visual de la ubicación de colocación se muestre siempre, incluso si la acción de soltar no es aceptada. Llamar a este método con false evita cualquier retroalimentación visual, incluso si la acción de soltar es aceptada. Invoque siempre este método desde canImport.

La página Demo - LocationSensitiveDemo incluye una caja combo que le permite elegir el mostrar siempre la ubicación del área de recepción, o no mostrar nunca el área de recepción, o el comportamiento por defecto. Pero primero hablaremos sobre la acción de soltar sensible a la ubicación.
