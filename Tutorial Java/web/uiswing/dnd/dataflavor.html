Usando y Creando un DataFlavor

La clase DataFlavor le permite especificar el tipo de contenido de sus datos. Necesita especificar un DataFlavor cuando extrae los datos desde el método importData. Varios tipos de sabores están predefinidos para usted:

  • imageFlavor representa datos en el formato de java.awt.Image. Esto se usa cuando se arrastra datos de imagen.
  • stringFlavor representa datos en la forma más básica de texto — java.lang.String. Este es el tipo de sabor más comunmente usado en la mayoría de las aplicaciones.
  • javaFileListFlavor representa objetos java.io.File en un formato java.util.List. Esto es útil para aplicaciones que arrastran ficheros, tales como el ejemplo TopLevelTransferHandler, descrito en la lección Soltado Nivel Superior.

Para la mayoría de las aplicaciones, esto es todo lo que usted necesita saber sobre los sabores de datos. Sin embargo, si requiere un sabor distinto a los tipos predefinidos, puede crear el suyo propio. Si crea un componente personalizado y quiere que participe en una transferencia de datos, necesitará crea un sabor de datos personalizado. El constructor para especificar un sabor de datos es DataFlavor(Class, String). Por ejemplo, para crear un sabor de datos para la clase java.util.ArrayList: 

new DataFlavor(ArrayList.class, "ArrayList");

Para crear un sabor de datos para un vector de enteros: 

new DataFlavor(int[].class, "Integer Array");

Transferir los datos usando este mecanismo usa la serialización de Object, así que la clase que use para transferir los datos debe implementar el interfaz Serializable, al igual que cualquier cosa que esté serializada con él. Si todo no es serializable, verá una NotSerializableException durante la colocación o copia en el portapapeles.

Crear un sabor de datos usando el constructor DataFlavor(Class, String) le permite transferir datos entre aplicaciones, incluyendo aplicaciones nativas. Si quiere crear un sabor de datos que transfiere datos sólo dentro de una aplicación, use javaJVMLocalObjectMimeType y el constructor DataFlavor(String). Por ejemplo, para especificar un sabor de datos que transfiera color desde un JColorChooser sólo dentro de su aplicación, podría usar este código: 

String colorType = DataFlavor.javaJVMLocalObjectMimeType +
                   ";class=java.awt.Color";
DataFlavor colorFlavor = new DataFlavor(colorType);

Para crear un sabor de datos para un ArrayList que trabajase sólo dentro de su aplicación: 

new DataFlavor(DataFlavor.javaJVMLocalObjectMimeType +
               ";class=java.util.ArrayList");

Para crear un sabor de datos para un vector de enteros: 

new DataFlavor(DataFlavor.javaJVMLocalObjectMimeType +
               ";class=\"" + int[].class.getName() + "\"");

Un tipo MIME contiene caracteres especiales, tales como [ o ;, debe tener esos caracteres encerrados entre comillas.

Un Transferable puede ser implementado para soportar sabores múltiples. Por ejemplo, puede usar los sabores locales y de serialización juntos, o puede usar dos formas de los mismos datos, tales como los sabores ArrayList y de vector de enteros, juntos, o puede crear un TransferHandler que acepte diferentes tipos de datos, tales como color y texto.

Cuando crea un vector de DataFlavors para ser devuelto desde el método de Transferable  getTransferDataFlavors, los sabores serán insertados en orden de preferencia, con el más preferido apareciendo en el elemento 0 del vector. Generalmente el orden preferido va desde la más rica, o más compleja, forma de datos hata la más simple — la forma más probable de ser entendida por otros objetos.
