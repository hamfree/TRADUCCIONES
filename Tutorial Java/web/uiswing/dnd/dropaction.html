Eligiendo la Acción al Soltar

Cada fuente de arrastre (basada en Java o de otra forma) avisa del conjunto de acciones que soporta cuando se exportan datos. Si soporta los datos que están siendo copiados, avisa de la acción COPY; si soporta que los datos se muevan desde ella, entonces avisa de la acción MOVE, etcétera. Para los componentes de Swing, las acciones de las fuentes son avisadas a través del método getSourceActions.

Cuando se inicia un arrastre, el usuario tiene algo de control sobre cuales de las acciones de la fuente se elijen para la transferencia a través de los modificadores del teclado usandos en conjunción con el gesto de arrastre — a esto se le llama acción de usuario. Por ejemplo, el valor predeterminado (cuando no se usan modificadores) indican generalmente una acción de mover, mantener pulsada la tecla Control indica una acción de copiar, y mantener presionadas las teclas Mayúsculas y Control indican una acción de enlace. La acción de usuario está disponible a través del método getUserDropAction.

La acción de usuario indica una preferencia, pero al final es el destino el que decide la acción de soltar. Por ejemplo, considere un componente que sólo aceptará datos copiados. Y considere un origen de arrastre que soporta tanto copiar como mover. El TransferHandler para el destino de sólo-copia puede ser codificado para aceptar sólo datos del origen usando el método setDropAction, incluso si el usuario ha indicado una preferencia para una acción de movimiento.

Este trabajo ocurre en el método canImport, donde el TransferHandler del destino decide si acepta los datos entrantes. Una implementación que elige explícitamente la acción COPY, si está soportado por el origen, podría ser como esto: 

public boolean canImport(TransferHandler.TransferSupport support) {
    // para la demostración, sólo soportaremos soltados (no pegados desde el portapapeles)
    if (!support.isDrop()) {
        return false;
    }

    // sólo importamos Cadenas
    if (!support.isDataFlavorSupported(DataFlavor.stringFlavor)) {
        return false;
    }

    // comprobar si las acciones de origen (un OR bit a bit de acciones soportadas)
    // contiene la acción COPY
    boolean copySupported = (COPY & support.getSourceDropActions()) == COPY;
    if (copySupported) {
        support.setDropAction(COPY);
        return true;
    }

    // COPY no se soporta, así que rechazamos la transferencia
    return false;
}

El trozo de código visualizado en negrita muestra dónde se consultan las acciones de soltado admitidas por la fuente. Si la copia se soporta, el método setDropAction se invoca para asegurar que sólo se realize una acción de copiado y el metodo devuelva verdadero.
