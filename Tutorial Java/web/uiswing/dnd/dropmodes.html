Estableciendo el Modo Soltar

Cuando se habilita el soltado en un componente, como una lista, necesita decidir cómo quiere que se interprete la ubicación de soltado. Por ejemplo, ¿quiere restringir que el usuario reemplaze las entradas existentes? ¿Quiere que sólo permita agregar o insertar nuevas entradas? ¿Quiere permitir ambas cosas? Para configurar este comportamiento, la clase JList ofrece el método setDropMode el cual soporta los modos de soltado siguientes.

  • El modo de soltado por defecto para JList es DropMode.USE_SELECTION. Cuando se arrastra en este modo, el item seleccionado en la lista se mueve para hacer eco del punto de soltado potencial. En un soltado, el elemento seleccionado cambia a la ubicación de soltado. Este modo se proporciona para compatibilidad con versiones anteriores, pero por lo demás no se recomienda.
  • En DropMode.ON, el item seleccionado en la lista se mueve para hacer eco del punto de soltado potencial, pero el item seleccionado no está afectado en el soltado. Este modo se puede usar para soltar sobre los elementos de la lista existente.
  • En DropMode.INSERT, se restringe al usuario a seleccionar el espacio entre elementos existentes de la lista, o antes del primer elemento o después del último elemento en la lista. No se permite seleccionar elementos existentes de la lista.
  • DropMode.ON_OR_INSERT es una combinación de los modos ON eINSERT.

La clase JTree ofrece el mismo conjunto de modos de soltado y la clase JTable tiene varios más específicos para agregar filas o columnas.

Para obtener la ublicación del soltado, la clase TransferSupport ofrece el método getDropLocation que devuelve el punto preciso donde el gesto de soltado ha ocurrido. Pero para un componente lista, el índice del soltado es más útil que una ubicación por pixel, así que JList ofrece una subclase especial, llamada JList.DropLocation. Esta clase ofrece los métodos getIndex e isInsert, que manejan las matemáticas por ti.

Los componentes de tabla, árbol, y texto proporcionan una implementación de DropLocation con métodos que tienen más sentido para cada componente. El método JTable.setDropMode tiene más opciones. La tabla siguiente muestra los métodos para las cuatro clases:
