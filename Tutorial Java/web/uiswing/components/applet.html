Cómo Hacer Applets

Esta sección cubre JApplet ─ una clase que habilita a los applets a usar componentes de Swing. JApplet es una subclase de java.applet.Applet, la cual está cubierta en el recorrido Applets de Java. Si usted nunca escribió un applet normal antes, le conminamos a que lea ese recorrido antes de proceder con ésta sección. La información ofrecida en ese recorrido se aplica a los applets de Swing, con unas pocas excepciones que esta sección explica.

Cualquier applet que contenga componentes Swing debe ser implementado con una sublcase de JApplet. Aquí tiene una versión Swing de uno de los applets que ayudaron a Java a hacerse famoso ─ un applet de animación que (en su más conocida configuración) muestra nuestra mascota Duke haciendo piruetas:

Puede encontrar el código fuente principal para este applet en TumbleItem.java.

Características Ofrecidas por JApplet

Debido a que JApplet es un contenedor de nivel superior de Swing, cada applet Swing tiene un panel raíz. Los efectos más noticiables de la presencia del panel raíz son el soporte para añadir una barra de menú y la necesidad de usar un panel de contenido.

Como se describió en Usando Contenedores de Nivel Superior, cada contenedor de nivel superior tal como un JApplet tiene un único panel de contenido. El panel de contenido hace a los applets de Swing diferentes de los applets regulares de la siguientes formas:

  • Añade componentes al panel de contenido del applet de Swing, no directamente al applet. Añadir Componentes al Panel de Contenido le muestra cómo.
  • Establece el gestor de disposición sobre el panel de contenido del applet Swing, no directamente sobre el applet.
  • El gestor de disposición por defecto para el panel de contenido de un applet Swing es BorderLayout. Esto difiere del gestor de disposición por defecto para Applet, el cual es FlowLayout.
  • No debería colocar código de pintado directamente en un objeto JApplet. Vea Realización de Pintado Personalizado para ejemplos de cómo realizar pintado personalizado en applets.

Hilos en los Applets

Los componentes de Swing deben crearse, consultarse, y manipularse en el hilo de envío de eventos, pero los navegadores no invocan los métodos "hito" del applet desde ese hilo. Por esta razón, los métodos hito ─ init, start, stop, y destroy ─ deben usar el método SwingUtilities invokeAndWait (o, si es apropiado , invokeLater) de forma que el código que se refiere a estos componentes Swing es ejecutado en el hilo de envío de eventos. Más información sobre estos métodos y el hilo de envío de eventos está en Concurrencia en Swing.

Aquí hay un ejemplo de un método init: 

public void init() {
    //Ejecuta un trabajo en el hilo de envío de eventos:
    //creando el IGU de este applet.
    try {
        javax.swing.SwingUtilities.invokeAndWait(new Runnable() {
            public void run() {
                createGUI();
            }
        });
    } catch (Exception e) {
        System.err.println("createGUI no se completó exitosamente");
    }
}

private void createGUI() {
    JLabel label = new JLabel(
                       "¡Usted ha ejecutado correctamente un applet Swing!");
    label.setHorizontalAlignment(JLabel.CENTER);
    label.setBorder(BorderFactory.createMatteBorder(1,1,1,1,Color.black));
    getContentPane().add(label, BorderLayout.CENTER);
}

El método invokeLater no es apropiado para esta implementación porque permite a init volver antes de que la inicialización esté completa, lo cual puede causar problemas en el applet que son difíciles de depurar.

El método init en TumbleItem es más complejo, como el código siguiente muestra. Como el primer ejemplo, ésta implementación del método init usa SwingUtilities.invokeAndWait para ejecutar el código de creación del IGU en el hilo del envío de eventos. Este método init configura un temporizador de Swing para disparar los eventos de acción que actualizan la animación. Usar Imágenes en un Applet Swing

La clase Applet ofrece el método getImage para cargar imágenes en un applet. El método getImage crea y devuelve un objeto Image que representa la imagen cargada. Ya que los componentes Swing usan Icons en vez de Images para referirse a fotos, los applets de Swing tienden a no usar getImage. En vez de eso los applets de Swing crean instancias de ImageIcon ─ un icono cargado desde un fichero de imagen. ImageIcon viene con un beneficio de ahorro en código: el maneja el monitoreo de la imagen automáticamente. Refiérase a Cómo Usar Iconos para más información.

La animación de Duke haciendo volteretas requiere 17 dibujos distintos. El applet usa un ImageIcon por dibujo y los carga en su método init. Ya que las imágenes pueden tomarse un tiempo largo en cargarse, los iconos son cargados en un hilo separado implementado por un objeto SwingWorker. Aquí está el código: 

public void init() {
    ...
    imgs = new ImageIcon[nimgs];
    (new SwingWorker<ImageIcon[], Object>() {
        public ImageIcon[] doInBackground() {
            //Las imagenes están numeradas de 1 a nimgs,
            //pero llenan el vector desde 0 a nimgs - 1.
            for (int i = 0; i < nimgs; i++) {
                imgs[i] = loadImage(i+1);
            }
            return imgs;
        }
        ...
    }).execute();

}
...
protected ImageIcon loadImage(int imageNum) {
    String path = dir + "/T" + imageNum + ".gif";
    int MAX_IMAGE_SIZE = 2400;  //Cambie esto al tamaño de
                                 //su imagen más grande, en bytes.
    int count = 0;
    BufferedInputStream imgStream = new BufferedInputStream(
       this.getClass().getResourceAsStream(path));
    if (imgStream != null) {
        byte buf[] = new byte[MAX_IMAGE_SIZE];
        try {
            count = imgStream.read(buf);
            imgStream.close();
        } catch (java.io.IOException ioe) {
            System.err.println("No pude leer el flujo desde el fichero: " + path);
            return null;
        }
        if (count <= 0) {
            System.err.println("Fichero vacio: " + path);
            return null;
        }
        return new ImageIcon(Toolkit.getDefaultToolkit().createImage(buf));
    } else {
        System.err.println("No puede encontrar el fichero: " + path);
        return null;
    }
}

El método loadImage carga la imagen para el cuadro especificado de la animación. Usa el método getResourceAsStream en vez del usual método getResource para obtener las imágenes. El código resultante no es bonito, pero getResourceAsStream es más eficiente que getResource para cargar imágenes desde ficheros JAR en applets que son ejecutados usando el programa Java Plug-in™. Para más detalles, véase Cargar Imágenes En Applets.

Embeber un Applet en una Página HTML

Puede desplegar un applet simple usando la etiqueta applet. O, puede usar el Kit de Herramientas de Implementación. Aquí está el código para el applet de Duke haciendo piruetas: 

<script src="https://www.java.com/js/deployJava.js" >
</script><script >
//<![CDATA[
    var attributes = { archive: 'https://docs.oracle.com/javase/tutorialJWS/samples/uiswing/TumbleItemProject/TumbleItem.jar',
                       codebase: 'https://docs.oracle.com/javase/tutorialJWS/samples/uiswing/TumbleItemProject',
                       code:'components.TumbleItem', width:'600', height:'95' };
    var parameters = { permissions:'sandbox', nimgs:'17', offset:'-57',
                       img: 'images/tumble', maxwidth:'120' };

La IPA de JApplet

La siguiente tabla lista los métodos interesantes que JApplet añade a la IPA de applet. Le dan acceso a características ofrecidas por el panel raíz. Otros métodos que podría usar están definidos por las clases Component y Applet . Vea los Métodos de Component para una lista de los métodos comunmente usados de Component, y Applets de Java para ayuda en usar los métodos de Applet.

Métodos de JApplet
Método Propósito
void setContentPane(Container)
Container getContentPane() 		Establece u obtiene el panel de contenido del applet. El panel de contenido contiene los componentes visibles del IGU del applet y serán opacos.
void setRootPane(JRootPane)
JRootPane getRootPane() 		Crea, establece, u obtiene el panel raíz del applet. El panel raíz gestiona el interior del applet incluyendo el panel de contenido, el panel de cristal, etcétera.
void setJMenuBar(JMenuBar)
JMenuBar getJMenuBar() 		Establece u obtiene la barra de menú del applet para gestionar un conjunto de menús para el applet.
void setGlassPane(Component)
Component getGlassPane() 		Establece u obtiene el panel de cristal del applet. Puede usar el panel de cristal para interceptar eventos del ratón.
void setLayeredPane(JLayeredPane)
JLayeredPane getLayeredPane() 		Establece u obtiene el panel estratificado del applet. Puede usar el panel estratificado del applet para colocar componentes por encima o debajo de otros componentes.
