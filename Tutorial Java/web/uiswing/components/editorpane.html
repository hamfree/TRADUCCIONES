Cómo Usar Paneles de Edición y Paneles de Texto

Dos clases de Swing soportan texto estilizado: JEditorPane y su subclase JTextPane. La clase JEditorPane es la base para los componentes de texto estilizado de Swing y ofrece un mecanismo a través del cual puede añadir soporte para formatos de texto personalizado. Si quiere texto sin estilo, use un área de texto en su lugar.

Puede ver un panel de edición y un panel de texto en uso ejecutando TextSamplerDemo. Aquí tiene una foto del ejemplo TextSamplerDemo.

Pulse el botón Lanza para ejecutar TextSamplerDemo usando Java™ Web Start ( descargue el KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo por usted mismo, consulte el índice de ejemplos.


El ejemplo TextSamplerDemo apenas comienza a demostrar las capacidades de los paneles de edición y los paneles de texto. Sin embargo, el panel de edición de arriba a la derecha ilustra una característica práctica y fácil de usar: visualiza una información de ayuda no editable cargada desde un Localizador Uniforme de Recursos (LUR, más conocido por la sigla URL, del inglés Uniform Resource Locator). El panel de texto más abajo a la derecha demuestra que puede fácilmente embeber imágenes e incluso componentes directamente dentro de los paneles de texto.

Nota: 

Si necesita un sistema de ayuda completo, eche un vistazo al sistema JavaHelp™.

La IPA de texto de Swing es poderosa e inmensa, y podríamos dedicar un libro entero sólo a usar los paneles de edición y los paneles de texto. Esta sección presenta sus capacidades, ofrece consejos de cual podría querer usar, y apunta a otras fuentes de información.

Usar un Panel de Edición para Visualizar Texto Desde una LUR

Una tarea que puede realizar sin saber nada sobre el sistema de texto de Swing es visualizar texto desde una LUR. Aquí tiene el código de TextSamplerDemo.java que crea un panel de edición no editable que visualiza texto formateado con etiquetas HTML. 

JEditorPane editorPane = new JEditorPane();
editorPane.setEditable(false);
java.net.URL helpURL = TextSamplerDemo.class.getResource(
                                "TextSamplerDemoHelp.html");
if (helpURL != null) {
    try {
        editorPane.setPage(helpURL);
    } catch (IOException e) {
        System.err.println("Attempted to read a bad URL: " + helpURL);
    }
} else {
    System.err.println("Couldn't find file: TextSamplerDemoHelp.html");
}

//Coloca el panel de edición en un panel de desplazamiento.
JScrollPane editorScrollPane = new JScrollPane(editorPane);
editorScrollPane.setVerticalScrollBarPolicy(
                JScrollPane.VERTICAL_SCROLLBAR_ALWAYS);
editorScrollPane.setPreferredSize(new Dimension(250, 145));
editorScrollPane.setMinimumSize(new Dimension(10, 10));

El código usa el constructor por defecto para crear el panel de edición, entonces llama a setEditable(false) de forma que el usuario no pueda editar el texto. A continuación, el código crea el objeto URL, y llama al método setPage con él.

El método setPage abre el recurso apuntado por la LUR y calcula el formato de texto (el cual es un HTML en el ejemplo). Si el formato de texto es conocido, el panel de edición se inicializa a sí mismo con el texto encontrado en la LUR. Un panel de edición estándar puede entender texto plano, HTML, y RTF. Note que la página podría cargarse asincrónicamente, lo cual hace que el IGU responda a los cambios pero significa que no puede contar con que los datos sean completamente cargados después de que la llamada a setPage vuelva.

Paneles de Edición vs. Paneles de Texto

Con el fin de usar los paneles de edición y los paneles de texto, necesita entender el sistema de texto, el cual es descrito en Características del Componente de Texto. Varios hechos sobre los paneles de edición y los paneles de texto están esparcidos por toda esa sección. Aquí listamos los hechos de nuevo y ofrecemos un poco más de detalle. La información aquí le ayudaría a entender las diferencias entre los paneles de edición y los paneles de texto, y cuándo usar cada cual.

  • Un panel de edición o un panel de texto pueden ser fácilmente cargados con texto desde una LUR usando el método setPage. La clase JEditorPane también ofrece constructores que le permiten inicializar un panel de edición desde una LUR. La clase JTextPane no tiene tales constructores. Vea Usar un Panel de Edición para Visualizar Texto Desde una URL para un ejemplo que usa esta característica para cargar un panel de edición no editable con texto formateado en HTML.

    Tenga en cuenta que el documento y el kit del editor podrían cambiar cuando use el método setPage. Por ejemplo, si un panel de edición contiene texto plano (por defecto), y usted lo carga con HTML, el documento cambiará a una instancia HTMLDocument y el kit del editor cambiará a una instancia HTMLEditorKit. Si su programa usa el método setPage, asegúrese de que ajusta su código por posibles cambios a las instancias del kit del editor y del documento del panel (re-registre los oyentes del documento al nuevo documento, etc ).

  • Los paneles de edición, por defecto, saben como leer, escribir, y editar texto plano, HTML, y RTF. Los paneles de texto heredan esta capacidada pero imponen ciertas limitaciones. Un panel de texto insiste en que su documento implemente el interfaz StyledDocument. HTMLDocument y RTFDocument son ambos StyledDocuments de forma que el HTML y el RTF funcionan como se espera dentro de un panel de texto. Sin embargo, si usted carga un panel de texto con un texto plano, el documento del panel de texto no es un PlainDocument como podría esperar, sino un DefaultStyledDocument.
  • Para soportar un formato de texto personalizado, implemente un kit de editor que pueda leer, escribir, y editar texto de ese formato. Entonces llame al método registerEditorKitForContentType para registrar su kit con la clase JEditorPane. Al registrar un kit de editor de esta forma, todos los paneles de edición y paneles de texto en su programa serán capaces de leer, escribir, y editar el nuevo formato. Sin embargo, si el el kit de editor nuevo no es un StyledEditorKit, los paneles de texto no soportarán el nuevo formato.
  • Como se mencionó previamente, un panel de texto requiere que su documento implemente el interfaz StyledDocument. El paquete de texto de Swing ofrece una implementación por defecto de esta interfaz, DefaultStyledDocument, el cual es el documento que los paneles de texto usan por defecto. Un panel de texto también requiere que su kit editor sea una instancia de un StyledEditorKit (o una subclase). Tenga en cuenta que los métodos read y write para StyleEditorKit funcionan con texto plano.
  • A través de su documento estilizado y kit de editor estilizado, los paneles de texto ofreden soporte para estilos con nombre y estilos lógicos. La clase JTextPane contiene en sí misma muchos métodos para trabajar con estilos que simplemente llaman métodos en su documento o kit de editor.
  • A través de la IPA ofrecida en la clase JTextPane, puede embeber imágenes y componentes en un panel de texto. Puede embeber imágenes en un panel de edición, también, pero sólo incluyendo las imágenes en un fichero HTML o RTF.

Un Ejemplo de Uso de un Panel de Texto

Aquí está el código del ejemplo TextSamplerDemo que crea e inicializa un panel de texto. 

String[] initString =
        { /* ...  fill array with initial text  ... */ };

String[] initStyles =
        { /* ...  fill array with names of styles  ... */ };

JTextPane textPane = new JTextPane();
StyledDocument doc = textPane.getStyledDocument();
addStylesToDocument(doc);

//Carga el panel de texto con texto estilizado.
try {
    for (int i=0; i < initString.length; i++) {
        doc.insertString(doc.getLength(), initString[i],
                         doc.getStyle(initStyles[i]));
    }
} catch (BadLocationException ble) {
    System.err.println("Couldn't insert initial text into text pane.");
}

En pocas palabras, este código codifica el texto inicial en una matriz y crea y codifica varios estilos ─ objetos que representan diferentes párrafos y formatos de carácter ─ dentro de otra matriz. A continuación, el código recorre las matrices, inserta el texto dentro del panel de texto, y especifica el estilo a usar para el texto insertado.

Aunque esto es un ejemplo interesante que demuestra de forma concisa varias características de JTextPane, los programas del "mundo-real" probablemente no inicializen un panel de texto de esta forma. En su lugar, un programa utilizaría un panel de edición para guardar un documento que luego se utilizaría para inicializar el panel de texto.

La IPA del Panel de Edición y el Panel de Texto

Esta sección lista algunas de las IPA relacionadas con los paneles de edición y de texto. Muchos de los más útiles métodos para JEditorPane y su subclase JTextPane son heredadas de la clase JTextComponent. Puede buscar las tablas de la IPA para JTextComponent en La IPA del Componente de Texto. Vea también La Clase JComponent, la cual describe la IPA heredada de JComponent.

IPA de JEditorPane para Visualizar Texto desde una LUR
Método o Constructor Descripción
JEditorPane(URL)
JEditorPane(String) 		Crea un panel de edición cargado con el texto de la LUR especificada.
setPage(URL)
setPage(String) 		Carga un panel de edición (o un panel de texto) con el texto especificado en la LUR.
URL getPage() Obtiene la LUR para la página actual del panel de edición (o del panel de texto).

JTextPane IPA
Método o Constructor Descripción
JTextPane()
JTextPane(StyledDocument) 		Crea un panel de texto. El argumento opcional especifica el modelo del panel de texto.
StyledDocument getStyledDocument
setStyledDocument(StyledDocument) 		Obtiene o establecer el modelo del panel de texto.
