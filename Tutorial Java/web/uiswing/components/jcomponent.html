La Clase JComponent

Con la excepción de los contenedores de nivel superior, todos los componentes de Swing cuyos nombres empiecen con "J" descienden de la clase JComponent. Por ejemplo, JPanel, JScrollPane, JButton, y JTable todas heredan desde JComponent. Sin embargo, JFrame y JDialog no lo hacen porque implementan contenedores de nivel superior.

La clase JComponent extiende la clase Container, la cual extiende a su vez a Component. La clase Component incluye todo desde ofrecer consejos de disposición a soporte de pintado y eventos. La clase Container tiene soporte para añadir componentes al contenedor y disponerlos en él. La sección Tablas de la IPA es un resumen de los métodos más a menudo usados de Component y Container, como también de JComponent.

Características de JComponent

La clase JComponent ofrece la siguiente funcionalidad a sus descendientes:

  • Sugerencias de herramientas
  • Pintar y bordes
  • Apariencia configurable a nivel de aplicación
  • Propiedades personalizadas
  • Soporte para diseño
  • Soporte para accesibilidad
  • Soporte para arrastrar y soltar
  • Doble almacenamiento en memoria intermedia
  • Atajos de teclado
Sugerencias de herramientas
Al especificar una cadena con el método setToolTipText, puede ofrecer ayuda a los usuarios de un componente. Cuando el cursor se para sobre el componente, la cadena especificada es visualizada en una pequeña ventana que aparece cerca del componente. Vea Cómo Usar Tool Tips para más información.
Pintar y bordes
El método setBorder le permite especificar el borde que un componente muestra alrededor de sus bordes. Para pintar dentro de un componente, sobreescriba el método paintComponent. Vea Cómo Usar Bordes y Realización de Pintado Personalizado para los detalles.
Apariencia configurable a nivel de aplicación
Entre bastidores, cada objeto JComponent tiene su correspondiente objeto ComponentUI que realiza todo el dibujado, el manejo de eventos, la determinación del tamaño, y demás para ese JComponent. Exactamente cual objeto ComponentUI es utilizado depende de la apariencia actual, la cual puede establecer usando el método UIManager.setLookAndFeel. Vea Cómo Establecer la Apariencia para los detalles.
Propiedades personalizadas
Puede asociar una o más propiedades (pares de nombre/objeto) con cualquier JComponent. Por ejemplo, un gestor de disposición podría usar propiedades para asociar un objeto de restricciones con cada JComponent que él maneja. Usted establece y obtiene propiedades usando los métodos putClientProperty y getClientProperty. Para información general sobre las propiedades, vea Propiedades.
Soporte para diseño
Aunque la clase Component ofrece métodos de sugerencia de diseño tales como getPreferredSize y getAlignmentX, no ofrece ninguna manera de establecer estas sugerencias de diseño, salvo la creación de una subclase y la sobreescritura de sus métodos. Para darle otra forma de establecer las sugerencias de diseño, la clase JComponent añade metodos establecedores (setter, en inglés) ─ setMinimumSize, setMaximumSize, setAlignmentX, y setAlignmentY. Vea Disponer Componentes Dentro de un Contenedor para más información.
Soporte para accesibilidad
La clase JComponent ofrece una IPA y funcionalidad básica para ayuda de las tecnologías de asistencia tales como lectores de pantalla para que obtengan información de los componentes Swing. Para más información sobre accesibilidad, vea Cómo Soportar Tecnologías de Asistencia.
Soporte para arrastrar y soltar
La clase JComponent ofrece una IPA para establecer un manejador de transferencias del componente, el cual es la base para el soporte de arrastrar y soltar de Swing. Vea Introducción al Arrastrar y Soltar (Dnd) para los detalles.
Doble almacenamiento en memoria intermedia
El doble almacenamiento suaviza el pintado en pantalla. Para los detalles, vea Realización de Pintado Personalizado.
Atajos de teclado
Esta característica hace que los componentes reaccionen cuando el usuario presiona una tecla en el teclado. Por ejemplo, en muchas apariencias cuando un botón tiene el foco, pulsar la tecla espaciadora es el equivalente a la pulsación del ratón sobre el botón. La apariencia automáticamente configura los atajos entre la pulsación y suelta de la tecla espaciadora y los efectos resultantes en el botón. Para más información sobre los atajos de teclado, vea Cómo Usar Atajos de Teclado.

Establecer y Obtener el Estado del Componente
Método Propósito
void setComponentPopupMenu(JPopupMenu) Establece el JPopupMenu para este JComponent. La IU es responsable para registrar las uniones y añadir los oyentes necesarios tales como que el JPopupMenu sea mostrado en el tiempo apropiado. Cuándo se muestre JPopupMenu depende de la apariencia: algunos pueden mostrarlo en un evento del ratón, algunos pueden habilitar un atajo de teclado.

Si popup es nulo, y getInheritsPopupMenu devuelve true, entonces getComponentPopupMenu será delegado al padre. Esto ofrece una forma para hacer que todos los componentes hijo hereden el popupmenu de la padre.
void setTransferHandler(TransferHandler)
TransferHandler getTransferHandler() 		Establece o elimina la propiedad transferHandler. El TransferHandler soporta el intercambio de datos vía cortar, copiar o pegar a/desde un portapapeles como también el arrastrar y soltar. Vea Introducción al Arrastrar y Soltar (Dnd) para más detalles.
void setToolTipText(String) Establece el texto para visualizar en una tool tip. Vea Cómo Usar Tool Tips para más información.
void setName(String)
String getName() 		Establece u obtiene el nombre del componente. Esto puede ser útil cuando necesite asociar texto con un componente que no visualize texto.
boolean isShowing() Determina si el componente se está mostrando en pantalla. Esto significa que el componente debe estar visible, y debe estar en un contenedor que esté visible y mostrándose.
void setEnabled(boolean)
boolean isEnabled() 		Establece u obtiene si el componente está habilitado. Un componente habilitado puede responder a la entrada del usuario y generar eventos.
void setVisible(boolean)
boolean isVisible() 		Establece u obtiene si el componente es visible. Los componentes son inicialmente visibles, con la excepción de los componentes de nivel superior.
Manejar Eventos
(vea Escribiendo Listener Events para los detalles)
Método Propósito
void addHierarchyListener(hierarchyListener l)
void removeHierarchyListener(hierarchyListener l)
Añade o elimina el oyente (listener, en inglés) de la jerarquía especificada para recibir eventos de jerarquía modificados de este componente cuando la jerarquía a la cual este contenedor pertenece cambia. Si el oyente l es nulo, no se lanza ninguna excepción ni se ejecuta ninguna acción.
void addMouseListener(MouseListener)
void removeMouseListener(MouseListener) 		Añade o elimina un oyente de ratón a o del componente. Los oyentes del ratón son notificados cuando el usuario usa el ratón para interactuar con el componente al que se escucha.
void addMouseMotionListener(MouseMotionListener)
void removeMouseMotionListener(MouseMotionListener) 		Añade o elimina un oyente del movimiento del ratón a o del componente. Los oyentes del movimiento del ratón son notificados cuando el usuario mueve el ratón dentro de los límites del componente al que se escucha.
void addKeyListener(KeyListener)
void removeKeyListener(KeyListener) 		Añade o elimina un oyente de tecla a o del componente. Los oyentes de tecla son notificados cuando el usuario teclea en el teclado y el componente al que se escucha tiene el foco del teclado.
void addComponentListener(ComponentListener)
void removeComponentListener(ComponentListener) 		Añade o elimina un oyente de componente a o del componente. Los oyentes del componente son notificados cuando el componente a escuchar está oculto, es mostrado, es movido, o se redimensiona.
boolean contains(int, int)
boolean contains(Point) 		Determina si el punto especificado está dentro del componente. El argumento deberá ser especificado en términos del sistema de coordenadas del componente. Los dos argumentos int especifican las coordinadas x e y, respectivamente.
Component getComponentAt(int, int)
Component getComponentAt(Point) 		Devuelve el componente que contiene la posición x, y especificada. Se devuelve el componente de más arriba en el caso de que los componentes se solapen. Esto se determina encontrando el componente más cercano al índice 0 que clama que contiene el punto dado vía Component.contains().
Component setComponentZOrder(component comp, int index)
Mueve el componente especificado al índice orden-z especificado en el contenedor.

Si el componente es un hijo de algún otro contenedor, éste es eliminado de ese contenedor antes de ser añadido a este contenedor. La importante diferencia entre este método y java.awt.Container.add(Component, int) es que este método no llama removeNotify sobre el componente mientras se elimina de su contenedor anterior a menos que sea necesario y cuando esté permitido por el sistema de ventanas nativo subyacente. De esta forma, si el componente tiene el foco del teclado, mantiene el foco cuando es movido a la nueva posición.

Nota:  El orden-z determina el orden en el que los componentes son pintados. El componente con el orden-z más alto se pinta primero y el componente con el orden-z más bajo se pinta el último. Donde los componentes se solapan, el componente con el orden-z más bajo se pinta sobre el componete con el orden-z más alto.

Component getComponentZOrder(component comp) 		Devuelve el índice de orden-z del componente dentro del contenedor. Cuanto más alto esté un componente en la jerarquía del orden-z, más bajo será su índice. El componente con el índice de orden-z más bajo es pintado el último, sobre todos los otros componentes hijo.
Pintar Componentes
(vea Realización de Pintado Personalizado para los detalles).
Método Propósito
void repaint()
void repaint(int, int, int, int) 		Pide que todos o parte de los componentes sean redibujados. Los cuatro argumentos int especifican los límites (x, y, anchura, altura, en ese orden) del rectángulo a ser pintado.
void repaint(Rectangle) Pide que el área especificada dentro del componente sea redibujada.
void revalidate() Pide que el componente y sus contenedores afectados se presenten nuevamente. En general, no debe invocar este método a menos que cambie explícitamente las sugerencias de tamaño / alineación de un componente después de que sea visible o cambie una jerarquía de contención después de que esté visible. Invoque siempre repaint después de revalidate.
void paintComponent(Graphics) Pinta el componente. Sobreescriba este método para implementar el pintado de componentes personalizados.

Colocación de componentes
(vea Disponer Componentes Dentro de un Contenedor para más información)
Método Propósito
void setPreferredSize(Dimension)
void setMaximumSize(Dimension)
void setMinimumSize(Dimension) 		Establece el tamaño máximo o mínimo preferido del componente, medido en píxeles. El tamaño preferido indica el mejor tamaño para el componente. El componente no debería ser más grande que su tamaño máximo y no más pequeño que su tamaño mínimo. Tenga en cuenta que esto son sólo sugerencias y podrían ser ignoradas por ciertos gestores de disposición.
Dimension getPreferredSize()
Dimension getMaximumSize()
Dimension getMinimumSize() 		Obtiene el tamaño máximo o mínimo preferido del componente, medido en píxeles. Muchas clases de JComponent tienen métodos establecedores (setter, en inglés) y obtenedores (getter, en inglés). Para aquellas subclases no-JComponent, las cuales no tienen los métodos establecedores correspondientes, puede establecer un tamaño máximo o mínimo preferido creando una subclase y sobreescribiendo estos métodos.
void setAlignmentX(float)
void setAlignmentY(float) 		Establece la alineación a lo largo del eje x- o y-. Estos valores indican cómo debería ser alineado el componente relativamente a otros componentes. El valor será un número entre 0 y 1 donde 0 representa una alineacíón a lo largo del origen, 1 es alineado lo más lejos del origen, y 0.5 es el centrado, etcetera. Tenga en cuenta que estos son sólo sugerencias y podrían ser ignorados por ciertos gestors de disposición.
float getAlignmentX()
float getAlignmentY() 		Obtiene la alineación del componente a lo largo del ejex- o y-. Para las subclases que no derivan de JComponent, las cuales no tienen los métodos establecedores correspondientes, puede establecer una alineación del componente, creando una subclase y sobreescribiendo estos métodos.
void setLayout(LayoutManager)
LayoutManager getLayout() 		Establece u obtiene el gestor de disposición del componente. El gestor de disposición es responsable del posicionamiento y tamaño de los componentes dentro de un contenedor.
void applyComponentOrientation(ComponentOrientation) void setComponentOrientation(ComponentOrientation) Establece la propiedad ComponentOrientation de este contenedor y todos los componentes contenidos dentro de él. Vea Establecer la Orientación del Contenedor para más información.
