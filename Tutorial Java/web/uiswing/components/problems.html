Resolviendo Problemas Comunes con los Componentes

Esta sección describe los problemas que podría encontrarse mientras usa componentes. Si no encuentra su problema en esta sección, consulte las siguientes secciones:

  • Resolviendo Problemas Comunes Usando Otras Características de Swing
  • Resolviendo Problemas Comunes de Diseño
  • Resolviendo Problemas Comunes en el Manejo de Eventos
  • Resolviendo Problemas Comunes de Pintado

Problema: Tengo problemas implementando un modelo (o algún otro código que sea similar a algo que ya está en Java SE Platform, Standard Edition).

  • Mire en el código fuente de Java SE. Es distribuido con el KDJ, y es un gran recurso para encontrar ejemplos de código de implementar modelos, disparar eventos, y similares.

Problema: Cada vez que el texto en mi campo de texto se actualiza, el tamaño del campo de texto cambia.

  • Debe especificar el ancho preferido del campo de texto especificando el número de columnas que tendrá de espacio para mostrar. Para hacer esto, puede usar o un argumento int en el constructor de JTextField o en el método setColumns.

Problema: Ciertas áreas del panel de contenido se ven extrañas cuando se vuelven a pintar.

  • Si configura el panel de contenido, asegúrese de que es opaco. Puede hacer esto invocando setOpaque(true) sobre su panel de contenido. Note que aunque los JPanels son opacos en la mayoría de los aspectos, eso no es verdad en el aspecto GTK+. Vea Añadir Componentes al Panel de Contenido para más detalles.
  • Si uno o más de sus componentes realizan pintado personalizado, asegúrese de que los implementó correctamente. Vea Resolviendo Problemas Comunes de Pintado para más ayuda.
  • Podría tener un problema de seguridad de hilo. Vea la siguiente entrada.

Problema: Mi programa está exhibiendo síntomas extraños que algunas veces parecen estar relacionados con el tiempo.

  • Asegúrese de que su código sea seguro para subprocesos. Vea Concurrencia en Swing para más detalles.

Problema: Mi diálogo modal se pierde detrás de otras ventanas.

  • Si el diálogo tiene un componente padre nulo, intente establecerlo a un marco válido o componente cuando lo crea.
  • Este error fue arreglado en la versión 6.0. Para más información, vea 4255200.

Problema: Las políticas de la barra de desplazamiento no parecen estar trabajando como se anuncia.

  • Algunas versiones de Swing contienen errores en las implementaciones para las políticas VERTICAL_SCROLLBAR_AS_NEEDED y HORIZONTAL_SCROLLBAR_AS_NEEDED. Si es factible para su proyecto, use la versión más reciente de Swing.
  • Si el cliente del panel de desplazamiento puede cambiar su tamaño dinámicamente, el programa deberá establecer el tamaño preferido del cliente y llamar a revalidate sobre el cliente.
  • Asegúrese de que especificó la política que deseaba para la orientación que quería.

Problema: Mi panel de desplazamiento no tiene barras de desplazamiento.

  • Si quiere que una barra de desplazamiento aparezca todo el tiempo, especifique o VERTICAL_SCROLLBAR_ALWAYS o HORIZONTAL_SCROLLBAR_ALWAYS para la política de la barra de desplazamiento según corresponda.
  • Si quiere que las barras de desplazamiento aparezcan como se necesita, y quiere forzar a las barras de desplazamiento para que se necesiten cuando el panel de desplazamiento se crea, tiene dos elecciones: o establece el tamaño preferido del panel de desplazamiento o el de su contenedor, o implementa una clase de navegación inteligente y devuelve un valor más pequeño que el del tamaño preferido estándar del componente desde el método getPreferredScrollableViewportSize. Consulte Dimensionar un Panel de Desplazamiento para más información.

Problema: ¡El divisor en mi panel de división no se mueve!

  • Necesita establecer el tamaño mínimo de al menos uno de los componentes en el panel de división. Consulte Colocación del Divisor y Restricción de su Rango para más información.

Problema: El método setDividerLocation de JSplitPane no funciona.

  • El método setDividerLocation(double) no tiene efecto si el panel de división no tiene tamaño (habitualmente verdad si no está en pantalla aún). Puede o usar setDividerLocation(int) o especificar los tamaños preferidos de los componentes contenidos en el panel de división o el panel de división cambia el tamaño de la propiedad weight en su lugar. Consulte Colocación del Divisor y Restricción de su Rango para más información.

Problema: Los bordes en paneles de división anidados se ven demasiado anchos.

  • Si anida paneles de división, los bordes se acumulan ─ el borde de los paneles divididos internos se muestra junto al borde del panel de división exterior, lo que provoca que los bordes se vean más anchos. El problema es particularmente notable al anidar muchos paneles divididos. La solución es establecer el borde a nulo en cualquier panel de división que se colocó dentro de otro panel de divisón. Para más información, vea el error #4131528 en la Base de datos de Errores de Java.

Problema: Los botones en mi barra de herramientas son demasiado grandes.

  • Intente reducir el margen de los botones. Por ejemplo: 
    button.setMargin(new Insets(0,0,0,0));

Problema: Los componentes en mi panel estratificado no están estratificados correctamente. De hecho, las capas parecen estar invertidas ─ cuanto menor es la profundidad, mayor es el componente.

  • Esto puede ocurrir si usa un int en vez de un Integer cuando añade componentes a un panel estratificado. Para ver lo que ocurre, en la clase LayeredPaneDemo, cambie
    layeredPane.add(label, new Integer(i));
    por
    layeredPane.add(label, i);.

Problema: La llamada al método colorChooser.setPreviewPanel(null) no elimina el panel de vista previa del selector de colores como se esperaba.

  • Un argumento null especifica el panel de vista previa por defecto. Para eliminar un panel de vista previa, especifique un panel estándar sin tamaño, como este: colorChooser.setPreviewPanel(new JPanel());
