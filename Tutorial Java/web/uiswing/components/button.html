Cómo Usar Botones, Casillas de Verificación y Botones de Radio

Para crear un botón, puede instanciar una de las muchas clases que descienden de la clase AbstractButton . La siguiente tabla le muestra las subclases AbstractButton definidas en Swing que podría querer usar:

Subclases AbstractButton definidas por Swing
Clase Sumario Dónde se describe
JButton Un botón común. Cómo Usar la IPA del Botón Común y Cómo Usar las Características de JButton
JCheckBox Un botón de casilla de verificación. Cómo utilizar Casillas de Verificación
JRadioButton Uno de un grupo de botones de opción. Cómo Utilizar Botones de Opción
JMenuItem Un ítem en un menú. Cómo Usar Menús
JCheckBoxMenuItem Un ítem de menú que tiene una casilla de verificación. Cómo Usar Menús y Cómo utilizar Casillas de Verificación
JRadioButtonMenuItem Un ítem de menú que tiene un botón de opción. Cómo Usar Menús y Cómo Utilizar Botones de Opción
JToggleButton Implementa la funcionalidad de conmutación heredada por JCheckBox y JRadioButton. Puede ser instanciada o subclasificada para crear botones de dos estados. Usados en algunos ejemplos
Nota: Si quiere recopilar un grupo de botones en una fila o columna, entonces debería echar un vistazo a Cómo Usar Barras de Herramientas.

Primero, esta sección explica la IPA del botón básico que AbstractButton define ─ y por lo tanto que todos los botones Swing tienen en común. A continuación, describe la pequeña cantida de IPA que JButton añade a AbstractButton. Después de eso, esta sección le muestra cómo usar IPA especializada para implementar casillas de verificación y botones de opción.

Cómo Usar la IPA del Botón Común

A continuación se muestra una imagen de una aplicación que muestra tres botones:

Pruebe esto: 
  1. Pulse el botón Lanzar para ejecutar la Demo Button usando Java™ Web Start (descargue el KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo por usted mismo, consulte el índice de ejemplos.
  2. Pulse el botón izquierdo.
    Deshabilita el botón del medio (y a sí mismo, ya que no es ya útil) y habilita el botón derecho.
  3. Pulse el botón derecho.
    Habilita el botón del medio y el botón de la izquierda, y se deshabilita a sí mismo.

Como el ejemplo ButtonDemo muestra, un botón Swing puede mostrar tanto texto como una imagen. En ButtonDemo, cada botón tiene su texto en un lugar diferente, relativo a su imagen. La letra subrayada en cada texto del botón muestra el mnemotécnico ─ la alternativa de teclado ─ para cada botón. En la mayoría de las apariencias, el usuario puede pulsar un botón presionando la tecla Alt y el mnemotécnico. Por ejemplo, Alt-M pulsaría el botón Centro en ButtonDemo.

Cuando se deshabilita un botón, la apariencia automáticamente genera el aspecto de botón desactivado. Sin embargo, podría proporcionar una imágen para sustituir la imágen normal. Por ejemplo, podría proporcionar versiones grises de las imágenes usadas en los botones de la izquierda y derecha.

Como implemente el manejo de eventos depeden del tipo de botón que use y cómo lo use. Generalmente, usted implementa un oyente de acción, el cual es notificado cada vez que el usuario pulsa el botón. Para casillas de verificación usted usualmente usa un oyente de ítem, el cual es notificado cuando la casilla de verificación es seleccionada o deseleccionada.

Debajo está el código de ButtonDemo.java que crea los botones en el ejemplo previo y reacciona a las pulsaciones en los botones. El código en negrita es el código que permanecería si los botones no tuvieran imágenes. 

//En código de inicialización:
    ImageIcon leftButtonIcon = createImageIcon("images/right.gif");
    ImageIcon middleButtonIcon = createImageIcon("images/middle.gif");
    ImageIcon rightButtonIcon = createImageIcon("images/left.gif");

    b1 = new JButton("Deshabilita botón central", leftButtonIcon);
    b1.setVerticalTextPosition(AbstractButton.CENTER);
    b1.setHorizontalTextPosition(AbstractButton.LEADING); //tcc LEFT, para lenguajes de izquierda-a-derecha
    b1.setMnemonic(KeyEvent.VK_D);
    b1.setActionCommand("disable");

    b2 = new JButton("Botón Central", middleButtonIcon);
    b2.setVerticalTextPosition(AbstractButton.BOTTOM);
    b2.setHorizontalTextPosition(AbstractButton.CENTER);
    b2.setMnemonic(KeyEvent.VK_M);

    b3 = new JButton("Habilita botón central", rightButtonIcon);
    //Usa la posición del texto por defecto de CENTER, TRAILING (RIGHT).
    b3.setMnemonic(KeyEvent.VK_E);
    b3.setActionCommand("enable");
    b3.setEnabled(false);

    //Escucha para las acciones en los botones 1 y 3.
    b1.addActionListener(this);
    b3.addActionListener(this);

    b1.setToolTipText("Pulse este botón para deshabilitar "
                      + "el botón central.");
    b2.setToolTipText("Este botón central no hace nada "
                      + "cuando lo pulsa.");
    b3.setToolTipText("Pulse este botón para habilitar el "
                      + "botón central.");
    ...
}

public void actionPerformed(ActionEvent e) {
    if ("disable".equals(e.getActionCommand())) {
        b2.setEnabled(false);
        b1.setEnabled(false);
        b3.setEnabled(true);
    } else {
        b2.setEnabled(true);
        b1.setEnabled(true);
        b3.setEnabled(false);
    }
}

Cómo Usar las Características de JButton

Los botones ordinarios ─ objetos JButton ─ tienen sólo un poco más de funcionalidad que la que la clase AbstractButton proporciona: Puede hacer que un JButton sea el botón por defecto.

Como máximo un botón en un contenedor de nivel superior puede ser el botón predeterminado. El botón por defecto típicamente tiene una apariencia destacada y es pulsado cada vez que el contenedor de nivel superior tiene el foco del teclado y el usuario presiona la tecla Intro o Retorno. Aquí tiene la foto de un diálogo, implementado en el ejemplo ListDialog , en el cual el botón Set es el botón por defecto:

Establezca el botón por defecto invocando el método setDefaultButton en en el panel raíz del contenedor superior. Aquí está el código que configura el botón por defecto para el ejemplo ListDialog : 

//En el constructor de una sublcase de JDialog:
getRootPane().setDefaultButton(setButton);

La implementación exacta de la característica del botón por defecto dende de la apariencia. Por ejemplo, en la Apariencia Windows, el botón por defecto cambia al botón que tenga el foco, así que presionar Intro pulsa el botón con el foco. Cuando ningún botónn tiene el foco, el botón que usted especificó originalmente como el botón por defecto se convierte en el botón por defecto de nuevo.

Cómo utilizar Casillas de Verificación

La clase JCheckBox proporciona soporta para los botones de casilla de verificación. Puede también colocar casillas de verificación en menus, usando la clase JCheckBoxMenuItem . Ya que JCheckBox y JCheckBoxMenuItem heredan de AbstractButton, las casillas de verificación de Swing tienen todas las características usuales de los botones, como se describió más temprano en esta sección. Por ejemplo, puede especificar imágenes para ser usadas en las casillas de verificación.

Las casillas de verificación son similares a los botones de opción pero su modelo de selección es diferente, por convención. En cualquier número de casillas de verificación en un grupo ─ ninguna, alguna, o todas ─ pueden ser seleccionadas. Un grupo de botones de opción, por el otro lado, puede tener sólo un botón seleccionado.

Pruebe esto: 
  1. Pulse el botón Lanzar para ejecutar la Demo CheckBox usando Java™ Web Start ( descargue KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo por usted mismo, consulte el índice de ejemplos.
  2. Pulse el botón Chin o presione Alt-c.
    La casilla de verificacion Chin se convierte en no seleccionada, y la barbilla desaparece del dibujo. Las otras casillas de verificación permanecen seleccionadas. Esta aplicación tiene un oyente de item que escucha a todas las casillas de verificación. Cada vez que el oyente de item recibe un evento, la aplicación carga un dibujo nuevo que refleja el estado actual de las casillas de verificación.

Una casilla de verificación genera un evento de ítem event y un evento de acción por pulsación. Usualmente, usted escucha sólo los eventos de ítem, ya que ellos le permiten determina si la pulsación selección o deseleccionó la casilla de verificación. Cómo Utilizar Botones de Opción

Los botones de opción son grupos de botones en los cuales, por convención, sólo un botón a la vez puede estar seleccionado. La versión de Swing soporta botones de opción con las clases JRadioButton y ButtonGroup . Para colocar un botón de opción en un menu, use la clase JRadioButtonMenuItem . Otras formas de visualizar una-de-muchas elecciones son las Cuadros Combinados y las listas. Los botones de opcioón son similares a las casillas de verificación, pero, por convención, las casillas de verificación no imponen límites a la cantidad de elementos que se pueden seleccionar a la vez.

Debido a que JRadioButton hereda de AbstractButton, los botones de opción de Swing tienen todas las características usuales de los botones, como se describió más temprano en esta sección. Por ejemplo, puede especificar la imagen que se muestra en un botón de opción.

Aquí tiene una foto de una aplicación que usa cinco botones de opcón para permitirle elegir que clase de mascota se muestra:

Pruebe esto: 
  1. Pulse el botón Lanzar para ejecutar la Demo RadioButton usando Java™ Web Start ( descargue KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo por usted mismo, consulte el índice de ejemplos.
  2. Pulse el botón Dog o presione Alt-d.
    El botón Dog se selecciona, el cual hace que botón Bird se deseleccione. La imagen cambia de un pájaro a un perro. Esta aplicación tiene un oyente de acción que escucha a todos los botones de opción. Cada vez que el oyente de acción recibe un evento, la aplicación visualiza la imagen para el botón de opción que justo fue pulsado.

Cada vez que el usuario pulsa un botón de opción (incluso si este estaba ya seleccionado), el botón dispara un evento de acción. Uno o dos eventos de item pueden también ocurrir ─ uno del botón que ha sido seleccionado ahora, y otro desde el botón que perdió la selección (si hubiera alguno). Usualmente, usted maneja las pulsaciones de los botones de opción usando un oyente de acción.

Debajo está el código de RadioButtonDemo.java que crea los botones de opción y en el ejemplo previo y reacciona a las pulsaciones.

    birdButton.setMnemonic(KeyEvent.VK_B);
    birdButton.setActionCommand(birdString);
    birdButton.setSelected(true);

    JRadioButton catButton = new JRadioButton(catString);
    catButton.setMnemonic(KeyEvent.VK_C);
    catButton.setActionCommand(catString);

    JRadioButton dogButton = new JRadioButton(dogString);
    dogButton.setMnemonic(KeyEvent.VK_D);
    dogButton.setActionCommand(dogString);

    JRadioButton rabbitButton = new JRadioButton(rabbitString);
    rabbitButton.setMnemonic(KeyEvent.VK_R);
    rabbitButton.setActionCommand(rabbitString);

    JRadioButton pigButton = new JRadioButton(pigString);
    pigButton.setMnemonic(KeyEvent.VK_P);
    pigButton.setActionCommand(pigString);

    //Agrupa los botones de opción.
    ButtonGroup group = new ButtonGroup();
    group.add(birdButton);
    group.add(catButton);
    group.add(dogButton);
    group.add(rabbitButton);
    group.add(pigButton);

    //Registra un oyente para los botones de opción.
    birdButton.addActionListener(this);
    catButton.addActionListener(this);
    dogButton.addActionListener(this);
    rabbitButton.addActionListener(this);
    pigButton.addActionListener(this);

Para cada grupo de botones de opción, necesita crear una instancia de ButtonGroup y añadir cada botón de opción a él. El ButtonGroup se cuida de deseleccionar el botón seleccionado previamente cuando el usuario selecciona otro botón en el grupo.

Por lo general, debe inicializar un grupo de botones de radio para que se seleccione uno. Sin embargo, la IPA no fuerza esta regla ─ un grupo de botones de opción puede no tener selección inicial. Una vez el usuario haga una selección, se selecciona exactamente un botón a partir de entonces.

El argumento de cadena especifica el texto, si lo hay, que el botón de conmutación visualizaría. Similarmente, el argumento Icon especifica la imagen que sería usada. Especificar el argumento booleano como true inicializa el botón de conmutación a estar seleccionado. Si el argumento booleano está ausente o es false, entonces el botón de conmutación está inicialmente deseleccionado.
