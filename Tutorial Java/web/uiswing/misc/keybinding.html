Cómo Usar Atajos de Teclado

La clase JComponent soporta atajos de teclado como una forma de responder a teclas individuales pulsadas por un usuario. Aquí tiene algunos ejemplos de cuando los atajos de teclado son apropiados:

  • Está creando un componente personalizado y quiere que soporte el acceso del teclado a él.
    Por ejemplo, querría que el componente reaccione cuando tiene el foco y el usuario presiona el Espaciador.
  • Quiere sobreescribir el comportamiento de un atajo de teclado existente.
    Por ejemplo, si su aplicación normalmente reacciona a las presiones de la tecla F2 de una manera particular, podría querer que ejecute una acción diferente o ignorar la pulsación de la tecla.
  • Quiere suministrar un atajo de teclado nuevo para una acción existente.
    Por ejemplo, es posible que sienta con firmeza que Control-Shift-Insert debería realizar una operación de pegado.

Usted no necesita a menudo usar los atajos de teclado directamente. Estos se usan entre bastidores por mnemotécnicos (compatibles con todos los botones y paneles con pestañas, así como por JLabel ) y aceleradores (apoyados por elementos de menú). Puede encontrar cobertura sobre mnemotécnicos y aceleradores en la sección Habilitación del Funcionamiento del Teclado.

Una alternativa a los atajos de teclado es usar oyentes de teclas. Los oyentes de teclas tienen su lugar como un interfaz de bajo nivel a la entrada del teclado, pero para responder a teclas individuales, las asociaciones de teclas son más apropiadas y tienden a dar como resultado un código de mantenimiento más fácil. Los oyentes de teclado son también difíciles si el atajo de teclado está activo cuando el componente no tiene el foco. Algunas de las ventajas de los atajos de teclado son que de alguna forma se auto documentan, tienen en cuenta la jerarquía de contención, fomentan los fragmentos reutilizables de código (objetos Action), y permiten que las acciones sean facilmente eliminadas, personalizadas, o compartidas. También, hacen fácil cambiar la tecla a la cual la acción está vinculada. Otra ventaja de las Acciones es que tienen un estado habilitado el cual suministra una forma fácil de deshabilitar la acción sin tener que llevar la pista de a qué componente está vinculado.

Cómo Funcionan los Atajos de Teclado

El soporte del atajo de teclado es suministrado por JComponent recae en las clases InputMap y ActionMap. Un mapa de entrada vincula pulsaciones de tecla a nombres de acciones, y un mapa de acción especifica la acción correspondiente a cada nombre de acción. Técnicamente, no necesita usar nombres de acciones en los mapas; puede usar cualquier objeto como la "tecla" dentro de los mapas. Por convención, sin embargo, use una cadena que nombra una acción.

Cada InputMap/ActionMap tienen un padre que habitualmente viene desde la IU. Cada vez que cambia la apariencia, el padre se restablece. De esta forma, cualquier enlace especificado por el desarrollador nunca se pierde en cambios de apariencia.

Cada JComponent tiene un mapa de acción y tres mapas de entrada. Los mapas de entrada corresponden a las siguientes situaciones con el foco:

JComponent.WHEN_FOCUSED
El componente tiene el foco del teclado. El mapa de entrada de WHEN_FOCUSED es habitualmente usado cuando el componente no tiene hijos. Por ejemplo, los botones vinculan la tecla Espacio usando el mapa WHEN_FOCUSED.
Estas vinculaciones son efectivas sólo cuando el componente tiene el foco.
JComponent.WHEN_ANCESTOR_OF_FOCUSED_COMPONENT
El componente contiene (o es) el componente que tiene el foco. Este mapa de entrada es usado comunmente para un componente compuesto ─ un componente cuya implmementación depende de los componentes hijo. Por ejemplo, JTables hace todas sus vinculaciones usando WHEN_ANCESTOR_OF_FOCUSED_COMPONENT de forma que si el usuario está editando, la tecla de flecha-arriba (por ejemplo) todavía cambia la celda seleccionada.
JComponent.WHEN_IN_FOCUSED_WINDOW
La ventana del componente o tiene el foco o contiene el componente que tiene el foco. Este mapa de entrada es usado habitualmente pra mnemotécnicos o aceleradores, los cuales necesitan estar activos independientemente de dónde se encuentre el foco en la ventana.

Cuando el usuario pulsa una tecla, el codigo que procesa el evento de tecla de JComponent busca a través de uno o más mapas de entrada para buscar una vinculación válida para la tecla. Cuando encuentra una vinculación, busca la acción correspondiente en el mapa de acción. Si la acción está habilitada, el vínculo es válido y la acción es ejecutada. Si está deshabilitada, la búsqueda para una vinculación válida continúa.

Si existen más de una vinculación para la tecla, solo se usa la primera válida encontrada. Los mapas de entrada son comprobados en este orden:

  1. El mapa de entrada del componente enfocado WHEN_FOCUSED.
  2. El mapa de entrada del componente enfocado WHEN_ANCESTOR_OF_FOCUSED_COMPONENT.
  3. Los mapas de entrada WHEN_ANCESTOR_OF_FOCUSED_COMPONENT del padre del componente enfocado, y entonces el padre de su padre, y así, continuando hacia arriba en la jerarquía de contención. Nota: Los mapas de entrada de los componentes deshabilitados se saltan.
  4. Los mapas de entrada WHEN_IN_FOCUSED_WINDOW de todos los componentes habilitados en la ventana enfocada son buscados. Ya que el orden de búsqueda de los componentes no es predecible, ¡evite duplicar los vinculos WHEN_IN_FOCUSED_WINDOW!

Consideremos lo que sucede en dos casos típicos de atajos de teclado: un botón reaccionando a la tecla Espacio, y un marco con un botón por defecto reaccionando a la tecla Intro.

En el primer caso, asumimos que el usuario presione la tecla Espacio mientras un JButton tiene el foco del teclado. Primero, los oyentes de tecla del botón son notificados del evento. Asumiendo que ninguno de los oyentes de tecla consumen el evento (al invocar el método consume sobre el KeyEvent) el mapa de entrada WHEN_FOCUSED del botón es consultado. Se encuentra una vinculación porque JButton usa ese mapa de entrada para vincular Espacio con un nombre de acción. El nombre de acción es buscado en el mapa de acciones del botón, y el método actionPerformed de la acción es invocado. El KeyEvent es consumido, y el proceso para.

En el segundo caso, asumimos que la tecla Intro es presionada mientras el foco está en cualquier parte dentro de un marco que tiene un botón por defecto (establecido usando el método setDefaultButton de JRootPane). Cualquiera que sea el componente enfocado, primero se notifica a sus oyentes de tecla. Asumiendo que ninguno de ellos consume el evento de la tecla el mapa de entrada WHEN_FOCUSED del componente enfocado es consultado. Si no tiene vínculo para la tecla o la Acción vinculada a la tecla está deshabilitada, el mapa de entrada WHEN_ANCESTOR_OF_FOCUSED_COMPONENT del componente enfocado es consultado y entonces (si no se encuentra vinculación o la Acción vinculada a la tecla está deshabilitada) los mapas de entrada WHEN_ANCESTOR_OF_FOCUSED_COMPONENT de cada uno de los ascendientes del componente en la jerarquía de componentes es consultado. Eventualmente, el mapa de entrada WHEN_ANCESTOR_OF_FOCUSED_COMPONENT del panel raíz es buscado. Ya que el mapa de entrada tiene una vinculación válida para Intro, la acción es ejecutada, causando que el botón por defecto sea pulsado.

Cómo Hacer y Eliminar Atajos de Teclado

Aquí tiene un ejemplo de cómo especificar que un componente debe reaccionar a la tecla F2: 

component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "hazAlgo");
component.getActionMap().put("hazAlgo",
                             anAction);
//donde anAction es una javax.swing.Action

Como el código precedente muestra, para obtener el mapa de acción de un componente se usa el método getActionMap (heredado de JComponent). Para obtener un mapa de entrada, puede usar el método getInputMap(int), donde el entero es una de las constantes JComponent.WHEN_*FOCUSED* mostradas en la lista precedente. O, en el caso usual donde la constante es JComponent.WHEN_FOCUSED, puede simplemente usar getInputMap sin argumentos.

Para añadir una entra a uno de los mapas, use el método put. Especifique una tecla usando un objeto KeyStroke, el cual puede obtener usando el método KeyStroke.getKeyStroke(String) . Puede encontrar ejemplos de creación de una Action (para colocarla en un mapa de acción) en Cómo Usar Acciones.

Aquí tiene un ejemplo un poquito más complejo que especifica que un componente debería reaccionar a la tecla Espacio como si el usuario pulsara el ratón. 

component.getInputMap().put(KeyStroke.getKeyStroke("SPACE"),
                            "presionado");
component.getInputMap().put(KeyStroke.getKeyStroke("released SPACE"),
                            "soltado");
component.getActionMap().put("presionado",
                             pressedAction);
component.getActionMap().put("soltado",
                             releasedAction);
//donde pressedAction y releasedAction son objetos javax.swing.Action

Para hacer que un componente ignore una tecla a la que normalmente responde, puede usar el nombre de acción especial "none". Por ejemplo, el código siguiente hace que un componente ignore la tecla F2. 

component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "none");
Nota: 

El código precedente no previene los mapas de entrada relevantes WHEN_ANCESTOR_OF_FOCUSED_COMPONENT y WHEN_IN_FOCUSED_WINDOW de ser buscados por una vinculación de la tecla F2. Para prevenir esta búsqueda, debe usar una acción válida en vez de "none". Por ejemplo: 

Action doNothing = new AbstractAction() {
    public void actionPerformed(ActionEvent e) {
        //no hacer nada
    }
};
component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "noHacerNada");
component.getActionMap().put("noHacerNada",
                             doNothing);

La IPA de los Atajos de Teclado

Las tablas siguientes listan la IPA comunmente usada para los atajos de teclado. También ve la tabla de la IPA Creando y Usando una Acción, en la sección Cómo Usar Acciones.

  • Crear y Usar InputMaps
  • Crear y Usar ActionMaps
Crear y Usar InputMaps
Método Propósito
InputMap getInputMap()
InputMap getInputMap(int)
(in JComponent) 		Obtiene uno de los mapas de entrada del componente. Los argumentos pueden ser una de estas constantes de JComponent: WHEN_FOCUSED, WHEN_IN_FOCUSED_WINDOW, o WHEN_ANCESTOR_OF_FOCUSED_COMPONENT. El método sin argumento obtiene el mapa de entrada WHEN_FOCUSED.
void put(KeyStroke, Object)
(en InputMap) 		Establece el nombre de acción asociado con la pulsación de tecla especificada. Si el segundo argumento es null, este método elimina la vinculación para la pulsación de tecla. Para hacer que la pulsación de tecla sea ignorada, use "none" como segundo argumento.
static KeyStroke getKeyStroke(String)
(en KeyStroke) 		Obtiene el objeto especificando una actividad particular del teclado del usuario. Argumentos típicos son "alt shift X", "INSERT", y "typed a". Vea la documentación de la IPA de KeyStroke para los detalles completos y para otras formas del método getKeyStroke.

Crear y Usar ActionMaps
Método Propósito
ActionMap getActionMap()
(en JComponent) 		Obtiene el objeto que asigna los nombres en acciones para el componente.
void put(Object, Action)
(en ActionMap) 		Establece la acción asociada con el nombre especificado. Si el segundo argumento es null, este método elimina la vinculación para el nombre.
