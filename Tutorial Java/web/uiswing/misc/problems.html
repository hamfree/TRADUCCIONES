Resolviendo Problemas Comunes Usando Otras Características de Swing

Problema: Mi aplicación no está mostrando la apariencia que yo he solicitado vía UIManager.setLookAndFeel.

Probablemente o estableció la apariencia a una apariencia no válida o la estableció después de que el gestor IU cargase la apariencia por defecto. Si está segunro de que la apariencia que especificó es válida y establecer la apariencia es la primera cosa que su programa hace (en la parte superio de su método main, por ejemplo), compruebe si tiene un campo estático que hace referencia a una clase de Swing. Esta referencia puede causar que la apariencia por defecto sea cargada si no se especificó ninguna. Para más información, incluyendo cóno establecer una apariencia después de que el IGU ha sido creado, vea la sección Cómo Establecer la Apariencia.

Problema: ¿Porqué mi componente no está obteniendo el foco?

  • ¿Es un componente personalizado (por ejemplo, una subclase directa de JComponent) que creó usted? si es así, puede necesitar dar a su componente un mapa de entrada y un oyente de ratón. Vea Cómo Hacer un Componente Personalizado Enfocable para más información y una demostración.
  • ¿Está el componente dentro de un objeto JWindow? El sistema de foco requiere que el marco que posee los JWindow sea visible para cualesquiera componentes en el objeto JWindow para obtener el foco. Por defecto, si no especifica un marco propietario para un objeto JWindow, un marco propietario invisible se crea para él. La solución es o especificar un marco propietario visible y enfocable cuando crea el objeto JWindow o usar los objetos JDialog o JFrame en su lugar.

Problema: ¿Porqué no recibe mi diálogo los eventos generados cuando el usuario pulsa la tecla Escape?

Si su diálogo contiene un campo de texto, puede estar consumiendo el evento.

  • Si quiere obtener el evento Escape independientemente de si un componente lo consume, debe usar un KeyEventDispatcher .
  • Si quiere obtener el evento Escpae sólo si un componente no lo ha consumido, entonces registre un atajo de teclado en cualquier componente JComponent del objeto JDialog, usando el mapa de entrada WHEN_IN_FOCUSED_WINDOW. Para más información, vea la página Cómo Usar Atajos de Teclado.

Problema: ¿Porqué no puedo aplicar componentes Swing a un icono del sistema? La implementación actual de la clase TrayIcon soporta el componente PopupMenu, pero no su contraparte Swing JPopupMenu. Esta limitación reduce las capacidades para emplear funciones adicionales de Swing, por ejemplo, iconos de menú. Vea el ID de Error 6285881.

  • Una clase nueva JTrayIcon será creada para eliminar este inconveniente. Hasta entonces, use componentes AWT para agregar un elemento de menú, casillas de verificación de elemento de menú, o submenús.

Si no encuentra su problema en esta sección, consulte Resolviendo Problemas Comunes con los Componentes.
