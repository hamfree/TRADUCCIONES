Cómo Usar BorderLayout

Nota: Esta lección cubre la escritura de código de diseño a mano, lo cual puede ser desafiante. Si no está interesado en aprender todos los detalles de la gestión del diseño, preferirá usar el gestor de diseño GroupLayout combinado con una herramienta de construcción para diseñar su IGU. Una de tales herramientas de construcción es el EID NetBeans. De otra manera, si quiere codificar a mano y no quiere usar GroupLayout, entonces se recomienda GridBagLayout como el gestor de diseño más flexible y potente.

Si está interesado en usar JavaFX para crear su IGU, vea Trabajar Con Diseños en JavaFX.

El código completo de esta demo está en el fichero BorderLayoutDemo.java.

Como la imagen precedente muestra, un objeto BorderLayout tiene cinco áreas. Estas áreas son especificadas por las constantes BorderLayout:

  • PAGE_START
  • PAGE_END
  • LINE_START
  • LINE_END
  • CENTER
Nota de versión: 

Antes del KDJ versión 1.4, los nombres preferidos para las diversas áreas eran diferentes, que van desde los puntos cardinales (por ejemplo, BorderLayout.NORTH para el área superior) a versiones con más palabras de las constantes que usamos en nuestros ejemplos. Las constantes que usan nuestros ejemplos son preferidas porque son estándar y permiten que los programas se ajusten a los idiomas que tienen orientaciones diferentes.

Si la ventana es ampliada, el área del centro obtiene la mayor cantidad del espacio disponible en lo posible. Las otras áreas se expanden sólo tanto como es necesario para rellenar todo el espacio disponible. A menudo un contenedor usa sólo una o dos de las áreas del objeto BorderLayout ─ simplemente el centro, o el centro y la parte inferior.

El código siguiente añade componentes al panel de contenido del marco. Porque los paneles de contenido usa la clase BorderLayout por defecto, el código no necesita establecer el gestor de diseño. El programa completo está en el fichero BorderLayoutDemo.java. 

...//Container pane = aFrame.getContentPane()...
JButton button = new JButton("Button 1 (PAGE_START)");
pane.add(button, BorderLayout.PAGE_START);

//Hace el componente central grande, ya que es el
//uso típico de BorderLayout.
button = new JButton("Button 2 (CENTER)");
button.setPreferredSize(new Dimension(200, 100));
pane.add(button, BorderLayout.CENTER);

button = new JButton("Button 3 (LINE_START)");
pane.add(button, BorderLayout.LINE_START);

button = new JButton("Long-Named Button 4 (PAGE_END)");
pane.add(button, BorderLayout.PAGE_END);

button = new JButton("5 (LINE_END)");
pane.add(button, BorderLayout.LINE_END);

Especificar la ubicación de los componentes (por ejemplo, BorderLayout.LINE_END) como uno de los argumentos al método add. Si este componente falta en un contenedor controlado por un objeto BorderLayout, asegúrese que la ubicación del componente fue especificada y que ningún otro componente fuera colocado en la misma ubicación.

Todos los ejemplos del tutorial que usa la clase BorderLayout especifican el componente como el primer argumento al método add. Por ejemplo: 

add(component, BorderLayout.CENTER)  //preferido

Sin embargo, el código en otros programas especifican el componente como segundo argumento. Por ejemplo, aquí hay una forma alternativa de escribir el código precedente: 

add(BorderLayout.CENTER, component)  //válido pero pasado de moda
    or
add("Center", component)             //válido pero propenso a errores

La IPA de BorderLayout

La siguiente tabla lista los constructores y métodos para especificar los huecos (en píxeles).

Especificar huecos
Constructor o Método Propósito
BorderLayout(int horizontalGap, int verticalGap) Define un diseño del borde con los huecos especificados entre componentes.
setHgap(int) Establece el hueco horizontal entre componentes.
setVgap(int) Establece e hueco vertical entre componentes.
