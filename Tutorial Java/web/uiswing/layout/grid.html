Cómo Usar GridLayout

Nota: Esta lección cubre la escritura de código de diseño a mano, lo cual puede ser desafiante. Si no está interesado en aprender todos los detalles de la gestión del diseño, preferirá usar el gestor de diseño GroupLayout combinado con una herramienta de construcción para diseñar su IGU. Una de tales herramientas de construcción es el EID NetBeans. De otra manera, si quiere codificar a mano y no quiere usar GroupLayout, entonces se recomienda GridBagLayout como el gestor de diseño más flexible y potente.

Si está interesado en usar JavaFX para crear su IGU, vea Trabajar Con Diseños en JavaFX.

La figura siguiente representa una captura de una aplicación que usa la clase GridLayout.

Pulse el botón Lanzar para ejecutar GridLayoutDemo usando Java™ Web Start ( descargue KDJ 7 o posterior). Alternativamente, para compilar y ejecutar el ejemplo usted mismo, consulte el índice de ejemplos.


El código completo de esta demostración está en el fichero GridLayoutDemo.java.

Un objeto GridLayout coloca los componentes en una cuadrícula de celdas. Cada componente toma todo el espacio disponible dentro de la celda, y cada celda es exactamente del mismo tamaño. Si la ventana GridLayoutDemo se redimensiona, el objeto GridLayout cambia el tamaño de la celda de forma que las celdas se hacen tan grandes como es posible, dado el espacio disponible en el contenedor.

El trozo de código de abajo crea el objeto GridLayout y los componentes que éste maneja. 


GridLayout experimentLayout = new GridLayout(0,2);

...

        compsToExperiment.setLayout(experimentLayout);

        compsToExperiment.add(new JButton("Button 1"));
        compsToExperiment.add(new JButton("Button 2"));
        compsToExperiment.add(new JButton("Button 3"));
        compsToExperiment.add(new JButton("Long-Named Button 4"));
        compsToExperiment.add(new JButton("5"));

El constructor de la clase GridLayout crea una instancia que tiene dos columnas y tantas filas como sea necesario.

Usa cuadros combinados para configurar cuanto relleno vertical o horizintal es colocado alrededor de los componentes. Entonces pulse el botón Aplicar espacios. El trozo de código siguiente muestra cómo su selección es procesada usando los métodos setVgap and setHgap de la clase GridLayout: 


applyButton.addActionListener(new ActionListener(){
            public void actionPerformed(ActionEvent e){
                //Obtiene el valor horizonal del espacio
                String horGap = (String)horGapComboBox.getSelectedItem();
                //Obtiene el valor vertical del espacio
                String verGap = (String)verGapComboBox.getSelectedItem();
                //Establece el valor del espacio horizontal
                experimentLayout.setHgap(Integer.parseInt(horGap));
                //Establece el valor del espacio vertical
                experimentLayout.setVgap(Integer.parseInt(verGap));
                //Configura el diseño de los botones
                experimentLayout.layoutContainer(compsToExperiment);
            }
        });

La IPA de GridLayout

La tabla siguiente lista los constructores de la clase GridLayout que especifican el número de filas y columnas.

Los constructores de la clase GridLayout
Constructor Propósito
GridLayout(int rows, int cols) Crea un diseño de cuadrícula con el número especificado de filas y columnas. Todos los compoentes en el diseño tienen el mismo tamaño. Una, pero no ambas, de las filas y columnas puede ser cero, lo que significa que cualquier número de objetos pueden ser colocados en una fila o en una columna.
GridLayout(int rows, int cols, int hgap, int vgap) Crea un diseño de cuadricula con el número especificado de filas y columnas. Además, los espacios horizontales y verticales son establecidos a los valores especificados. Los espacios horizontales son lugares entre las columnas. Los espacios verticales son colocados entre las filas.
