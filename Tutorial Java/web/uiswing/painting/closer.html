Una Mirada más Cercana al Mecanismo de Pintado

Por ahora sabe que el método paintComponent es donde se debe colocar todo su código de pintado. Es verdad que este método será invocado cuando es el tiempo de pintar, pero el pintado en realidad empieza mucho más arriba en la jerarquía de clases, con el método paint (definido por java.awt.Component). Este método será ejecutado por el subsistema de pintado siempre que su componente necesite ser representado. Su firma es:

  • public void paint(Graphics g)

javax.swing.JComponent extiende esta clase y divide aun más el método paint en tres métodos separados, que se invocan en el siguiente orden:

  • protected void paintComponent(Graphics g)
  • protected void paintBorder(Graphics g)
  • protected void paintChildren(Graphics g)

La IPA no hace nada para prevenir que su código sobreescriba paintBorder y paintChildren, pero hablando de forma general , no hay razón para que usted haga eso. Para todos los propósitos prácticos, paintComponent será el único método que usted necesitará sobreescribir.

Como se mencionó previamente, la mayoría de los componentes de Swing tienen su apariencia implementada por Delegados de IU separados. Esto significa que la mayoría (o todo) del pintado para los componentes estándar de Swing procede como sigue.

  1. paint() invoca paintComponent().
  2. Si la propiedad ui no es nula, paintComponent() invoca ui.update().
  3. Si la propiedad opaque del componente es verdadera, ui.update() llena el fondo del componente con el color del fondo e invoca ui.paint().
  4. ui.paint() representa el contenido del componente.

Esto es porque nuestro código SwingPaintDemo invoca a super.paintComponent(g). Podríamos agregar un comentario adicional para hacer esto más claro: 

public void paintComponent(Graphics g) {
    // Let UI Delegate paint first, which 
    // includes background filling since 
    // this component is opaque.

    super.paintComponent(g);       
    g.drawString("This is my custom Panel!",10,20);
    redSquare.paintSquare(g);
}

Si ha entendido todo el código de la demostración ofrecido en esta lección, ¡felicitaciones! Tiene suficiente conocimiento práctico para escribir código de pintado eficiente en sus propias aplicaciones. Si sin embargo quiere una mirada más cercana "bajo el bosque", por favor refiérase al articulo del SDN enlazado en la primera página de esta lección.
