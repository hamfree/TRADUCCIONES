Resolviendo Problemas Comunes de Pintado

Problema: No sé donde colocar mi código de pintado.

  • El código de pintado pertenece al método paintComponent de cualquier componente que descienda de JComponent.

Problema: Las cosas que pinto no aparecen.

  • Compruebe si su componente se está mostrando. Resolviendo Problemas Comunes de Componentes le ayudará con esto.
  • Compruebe si repaint se invoca en su componente cada vez que su apariencia necesita ser actualizada.

Problema: Aparece el primer plano de mi componente, pero su fondo es invisible. El resultado es que uno o más componentes directamente detrás de mi componente son inesperadamente visibles.

  • Asegúrese de que su componente es opaco. Los JPanels, por ejemplo, son opacos por defecto en la mayoría de los aspectos pero no en todos. Para hacer que componentes tales como las JLabels y los JPanels de GTK+ sean opacos debe invocar setOpaque(true) en ellos.
  • Si su componente personalizado extiende JPanel o un descendiente más especializado de JComponent, entonces puede pintar el fondo invocando super.paintComponent antes de pintar los contenidos de su componente.
  • Puede pintar el fondo usted mismo usando éste código en lo alto del método paintComponent de un componente personalizado: 
    g.setColor(getBackground());
g.fillRect(0, 0, getWidth(), getHeight());
g.setColor(getForeground());

Problema: He usado setBackground para establecer el color del fondo de mi componente, pero parece que no tuvo efecto.

  • Lo más probable es que su componente no esté pintando su fondo, ya sea porque no es opaco o porque su código de pintura personalizado no pinta el fondo. Si establece el color de fondo para una JLabel, por ejemplo, debe también invocar setOpaque(true) en la etiqueta para hacer que el fondo de la etiqeuta sea pintado.

Problema: Estoy usando el mismo código exacto que el del ejemplo del tutorial, pero no funciona. ¿Porqué?

  • ¿Es el código ejecutado en el mismo método que el ejemplo del tutorial? Por ejemplo, si el ejemplo del tutorial tiene el código en el método paintComponent del ejemplo, entonces este método podría ser el único lugar donde se garantiza que el código funciona.

Problema: ¿Cómo pinta líneas gordas? ¿patrones?

  • La IPA de Java™ 2D ofrece soporte extensivo para implementar anchos de línea y estilos, como también patrones para usar en formas de relleno y trazado. Vea el recorrido Gráficos 2D para más información sobre el uso de la IPA de Java 2D.

Problema: Los bordes de un componente en particular se ven extraños.

  • Debido a que los componente a menudo actualizan sus bordes para reflejar el estado del componente, debería generalmente evitar invocar setBorder excepto en los JPanels y las subclases personalizadas de JComponent.
  • ¿Está el componente pintado por una apariencia tal como GTK+ o Windows XP que usa bordes pintados por IU en vez de objetos Border? Si es así, no invoque setBorder en el componente.
  • ¿Tiene el componente codigo de pintado personalizado? Si es así, ¿El código de pintura tiene en cuenta las inserciones del componente?

Problema: Aparecen artefactos visuales en mi IGU.

  • Si establece el color de fondo de un componente, asegurese de que el color no tenga transpariencia si se supone que el componente es opaco.
  • Use el método setOpaque para establecer la opacidad del componente si es necesario. Por ejemplo, el panel de contenido debe ser opaco, pero los componentes con fondos transparentes no deben ser opacos.
  • Asegúrese de que su componente personalizado rellena completamente su área de pintado si es opaco.

Problema: El rendimiento de mi código de pintado personalizado es pobre.

  • Si puede pintar parte de su componente, use el método getClip o getClipBounds de Graphics para determinar qué área necesita pintar. Cuanto menos pinte, más rápido será.
  • Si sólo parte de sus componentes necesitan ser actualizados, haga peticiones de pintado usando una versión de repaint que especifique la región de pintado.
  • Para ayudar en la elección de técnicas eficientes de pintado, mire por la cadena "performance" en la Página de Inicio de la IPA de Java .

Problema: Las mismas transformaciones aplicadas a objetos Graphics aparentemente idénticos a veces tienen efectos ligeramente diferentes.

  • Debido a que el código de pintado de Swing establece la transformación (utilizando el método translate de Graphics) antes de invocar paintComponent, cualquier transformación que aplique tendrá un efecto acumulativo. Esto no importa cuando se hace una traducción simple, pero un AffineTransform más complejo, por ejemplo, puede tener resultados inesperados.

Si usted no ve su problema en esta lista, vea Resolviendo Problemas Comunes con los Componentes y Resolviendo Problemas Comunes de Diseño .
