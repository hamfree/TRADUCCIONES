Sobre JFC y Swing

JFC es la abreviatura para Java Foundation Classes (Clases de Fundación de Java, en español), las cuales abarcan un grupo de funcionalidades para construir interfaces gráficas de usuario (IGUs o GUIs en inglés) y añaden funcionalidad e interactividad con gráficos complejos a las aplicaciones de Java. Están definidas conteniendo las características mostradas en la tabla siguiente.

Características de las Clases de Fundación de Java
Característica Descripción
Componentes de Swing para el IGU Incluye todo desde botones hasta paneles separados y tablas. Muchos componentes son capaces de ordenamiento, impresión, y soporte de arrastrar y soltar, por nombrar unas pocas de las características soportadas.
Soporte Enchufable de Apariencia La apariencia de las aplicaciones Swing es enchufable, permitiendo la selección de la apariencia. Por ejemplo, el mismo programa puede usar o la apariencia de Java o la de Windows. Adicionalmente, la plataforma Java soporta la apariencia de GTK+, lo cual hace que cientos de apariencias existentes estén disponibles en los programas de Swing. Muchos más paquetes de apariencia están disponibles desde varias fuentes.
IPA de Accesibilidad Habilita las tecnologías de asistencia, tales como los lectores de pantalla y los visualizadores Braille, para obtener información del interfaz de usuario.
IPA de Java 2D Habilita a los desarrolladores para incorporar facilmente gráficos de alta calidad en 2D, texto, e imágenes en aplicaciones y applets. Java 2D incluye extensas IPAs para generar y enviar salidas de alta calidad a los dispositivos de impresión.
Internacionalización Permite a los desarrolladores el construir aplicaciones que puedan interactuar con usuarios de todo el mundo en sus propios lenguajes y convenciones culturales. Con el marco de trabajo de métodos de entrada los desarrolladores pueden construir aplicaciones que acepten texto en lenguajes que usan miles de caracteres diferentes, como el japonés, el chino, o el coreano.

Este recorrido se concentra en los componentes de Swing. Le ayudamos a elegir los componentes apropiados para su IGU, le contamos cómo usarlos, y le damos la información de fondo que necesita para usarlos efectivamente. También describimos otras características y cómo se aplican a los componentes de Swing.

¿Qué paquetes Swing debo usar?

La IPA de Swing es poderosa, flexible — e inmensa. La IPA de Swing tiene 18 paquetes públicos:

javax.accessibility javax.swing.plaf javax.swing.text
javax.swing javax.swing.plaf.basic javax.swing.text.html
javax.swing.border javax.swing.plaf.metal javax.swing.text.html.parser
javax.swing.colorchooser javax.swing.plaf.multi javax.swing.text.rtf
javax.swing.event javax.swing.plaf.synth javax.swing.tree
javax.swing.filechooser javax.swing.table javax.swing.undo

Afortunadamente, la mayoria de los programas usan solo un pequeño subconjunto de la IPA. Este recorrido ordena la IPA para usted, dándole ejemplos de código común y apuntando a los métodos y clases que usted probablemente necesite. La mayoría del código en este recorrido usa uno o dos paquetes Swing:

  • javax.swing
  • javax.swing.event (no requerido siempre)
