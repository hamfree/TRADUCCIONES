Lección: Trabajando con Imágenes

Como usted ha aprendido ya desde la lección de Imágenes, las Images se describen por un ancho y alto, medidos en pixels, y tienen un sistema de coordenadas que es independiente de la superficie de dibujo.

Hay un número de tareas comunes cuando se trabaja con imágenes.

  • Cargar un formato de archivo de imagen externo GIF, PNG o JPEG en una representación interna de imagen utilizada por Java 2D.
  • Crear directamente una imagen de Java 2D y presentarla.
  • Dibujar el contenido de una imagen Java 2D en una superficie de dibujo.
  • Guardar el contenido de una imagen Java 2D en un archivo de imagen GIF, PNG o JPEG externo.

Esta lección le enseña los conceptos básicos para cargar, mostrar y guardar imágenes.

Hay dos clases principales que debe aprender para trabajar con imágenes:

  • La clase java.awt.Image es la superclase que representa las imágenes gráficas como matrices rectangulares de pixeles.
  • La clase java.awt.image.BufferedImage, que extiende la clase Image para permitir que la aplicación opere directamente con los datos de la imagen (por ejemplo, recuperar o configurar el color del pixel). Las aplicaciones pueden construir directamente instancias de esta clase.

La clase BufferedImage es una piedra angular de la IPA de imágenes de modo inmediato de Java 2D. Administra la imagen en la memoria y proporciona métodos para almacenar, interpretar y obtener datos de píxeles. Dado que BufferedImage es una subclase de Image, puede ser renderizada por los métodos Graphics y Graphics2D que aceptan un parámetro de Image.

Un BufferedImage es esencialmente una Image con un búfer de datos accesible. Es por lo tanto más eficiente trabajar directamente con BufferedImage. Un BufferedImage tiene un ColorModel y un Raster de datos de imagen. El ColorModel proporciona una interpretación de color de los datos de los pixel de la imagen.

El Raster realiza las siguientes funciones:

  • Representa las coordenadas rectangulares de la imagen
  • Mantiene los datos de la imagen en memoria
  • Proporciona un mecanismo para crear múltiples subimágenes desde un único búfer de datos de imagen
  • Proporciona métodos para acceder a pixeles específicos dentro de la imagen
