Escribiendo/Salvando una Imagen

Esta lección comenzó con una explicación sobre el uso del paquete javax.imageio, para cargar imágenes desde un formato de imagen externo en el formato interno BufferedImage utilizado por Java 2D. Luego explica cómo usar Graphics.drawImage() para dibujar esa imagen, con filtrado opcional.

La etapa final es guardar un objeto BufferedImage en un formato de imagen externo. Esta puede ser una imagen que fue cargada originalmente por la clase ImageIO desde un formato de imagen externo y quizás modificada usando la IPA 2D de Java, o puede ser una que fue creada por Java 2D.

La clase ImageIO proporciona una forma sencilla de guardar imágenes en una variedad de formatos de imagen en el siguiente ejemplo: 

static boolean ImageIO.write(RenderedImage im, 
                             String formatName,
                             File output)  throws IOException
Nota:   La clase BufferedImage implementa la interfaz RenderedImage.

 

El parámetro formatName selecciona el formato de imagen en el que guardar la BufferedImage. 

try {
    // recuperar imagen
    BufferedImage bi = getMyImage();
    File outputfile = new File("saved.png");
    ImageIO.write(bi, "png", outputfile);
} catch (IOException e) {
    ...
}

El método ImageIO.write llama al código que implementa PNG escribiendo un “ complemento de escritura PNG ”. El término plug-in se utiliza ya que ImageIO es extensible y puede admitir una amplia gama de formatos.

Pero los siguientes complementos de formato de imagen estándar: JPEG, PNG, GIF, BMP y WBMP siempre están presentes.

Cada formato de imagen tiene sus ventajas y desventajas:

Formato Más Menos
GIF Admite animación y píxeles transparentes Admite solo 256 colores y sin translucidez
PNG Mejor alternativa que GIF o JPG para imágenes de alto color sin pérdida, admite translucidez No es compatible con la animación.
JPG Ideal para imágenes fotográficas Pérdida de compresión, no es bueno para texto, capturas de pantalla o cualquier aplicación donde la imagen original deba conservarse exactamente

Para la mayoría de las aplicaciones, es suficiente utilizar uno de estos complementos estándar. Tienen la ventaja de estar fácilmente disponibles. La clase ImageIO proporciona una forma de conectar la compatibilidad con formatos adicionales que se pueden utilizar, y existen muchos de estos complementos. Si está interesado en los formatos de archivo disponibles para cargar o guardar en su sistema, puede usar los métodos getReaderFormatNames y getWriterFormatNames de la clase ImageIO. Estos métodos devuelven una matriz de cadenas que enumeran todos los formatos admitidos en esta EEJ. 

String writerNames[] = ImageIO.getWriterFormatNames();

La matriz de nombres devuelta incluirá cualquier complemento adicional que esté instalado y cualquiera de estos nombres se puede utilizar como nombre de formato para seleccionar un escritor de imágenes. El siguiente ejemplo de código es una versión simple de un programa completo de edición/retoque de imágenes que usa una versión revisada de ImageDrawingApplet.java programa de muestra que se puede utilizar de la siguiente manera:

  • Una imagen se carga primero a través de ImageIO
  • El usuario selecciona un filtro de la lista desplegable y se dibuja una nueva imagen actualizada
  • El usuario selecciona un formato de guardado de la lista desplegable
  • A continuación, aparece un selector de archivos y el usuario selecciona dónde guardar la imagen.
  • La imagen modificada ahora puede ser vista por otras aplicaciones de escritorio.

El código completo de este ejemplo está representado en SaveImage.java.

En esta lección, ha aprendido solo los conceptos básicos de ImageIO, que proporciona un amplio soporte para escribir imágenes, incluido el trabajo directo con un complemento ImageWriter para lograr un control más preciso sobre el proceso de codificación. ImageIO puede escribir varias imágenes, metadatos de imágenes y determinar las compensaciones entre calidad y tamaño. Para más información, ver la Guía de la IPA de Image E/S de Java.
