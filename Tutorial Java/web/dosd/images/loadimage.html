Leyendo/Cargando una Imagen

Cuando piensa en imágenes digitales, probablemente piense en formatos de imagen de muestra, como el formato de imagen JPEG que se usa en la fotografía digital, o imágenes GIF que se usan comúnmente en las páginas web. Todos los programas que pueden usar estas imágenes deben primero convertirlas de ese formato externo a un formato interno.

Java 2D admite la carga de estos formatos de imagen externos en su formato BufferedImage utilizando su IPA de E/S de imagen que se encuentra en el paquete javax.imageio. Image E/S tiene soporte integrado para GIF, PNG, JPEG, BMP y WBMP. Image E/S también es extensible para que los desarrolladores o administradores puedan suministrar "plug-in" para formatos adicionales. Por ejemplo, los complementos para TIFF y JPEG 2000 están disponibles por separado.

Para cargar una imagen desde un archivo específico, use el siguiente código, que es de LoadImageApp.java: 

BufferedImage img = null;
try {
    img = ImageIO.read(new File("strawberry.jpg"));
} catch (IOException e) {
}

Image E/S reconoce el contenido del archivo como una imagen en formato JPEG y lo decodifica en un BufferedImage que puede ser utilizado directamente por Java 2D.

LoadImageApp.java muestra cómo mostrar esta imagen.

Si el código se está ejecutando en un applet, entonces es igual de fácil obtener la imagen del código base del subprograma. El siguiente extracto es de LoadImageApplet.java: 

try {
    URL url = new URL(getCodeBase(), "examples/strawberry.jpg");
    img = ImageIO.read(url);
} catch (IOException e) {
}

El método getCodeBase utilizado en este ejemplo devuelve la URL del directorio que contiene este subprograma cuando el applet se implementa en un servidor web. Si el applet se implementa localmente, getCodeBase devuelve un valor nulo y el subprograma no se ejecutará.

El siguiente ejemplo muestra cómo usar el método getCodeBase para cargar el archivo strawberry.jpg.

Nota:   si no ve el applet ejecutándose, debe instalar al menos la versión 7 del kit de desarrollo de Java SE (KDJ).

LoadImageApplet.java contiene el código completo para este ejemplo y este applet requiere el archivo de imagen strawberry.jpg.

Además de leer archivos o URLs, Image E/S puede leer de otras fuentes, tales como un InputStream. ImageIO.read() es la IPA de conveniencia más sencilla para la mayoría de las aplicaciones, pero la clase javax.imageio.ImageIO proporciona muchos más métodos estáticos para usos más avanzados de la IPA de Image E/S. La colección de métodos de esta clase representa solo un subconjunto del rico conjunto de IPA para descubrir información sobre las imágenes y para controlar el proceso de decodificación (lectura) de imágenes.

Exploraremos algunas de las otras capacidades de Image E/S más tarde en la sección Escribiendo/Salvando una Imagen.
