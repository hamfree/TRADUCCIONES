Texto

La IPA 2D de Java tiene varias capacidades de presentación de texto incluyendo métodos para presentar cadenas y clases enteras para establecer atributos de fuente y realizar diseño de texto.

Si simplemente quiere dibujar una cadena de texto estático, la forma más directa de presentarlo directamente es a través de la clase Graphics usando el método drawString. Para especificar la fuente, use el método setFont de la clase Graphics.

Si quiere implementar sus propias rutinas de edición de texto o necesita más control sobre el diseño del texto que los que proveen los componentes de texto, puede usar las clases de diseño de texto de Java 2D en el paquete java.awt.font.

Fuentes

Las formas que una fuente usa para representar los caracters en una cadena se llaman glifos. Un carácter particular o combinación de caracteres puede ser representada con uno o más glifos. Por ejemplo, á puede ser representada por dos glifos, mientras que la ligadura fi podría estar representada por un solo glifo.

Una fuente se puede considerar como una colección de glifos. Una única fuente puede tener muchos estilos, tales como cursiva y regular. Todas los estilos de una fuente tienen características tipográficas similares y pueden reconocerse como miembros de la misma familia. En otras palabras, una colección de glifos con un estilo particular forma un estilo de fuente. Una colección de estilos de fuentes forman una familia de fuente. La colección de familias de fuentes forman el conjunto de fuentes que están disponibles en el sistema.

Cuando está usando la IPA 2D de Java, puede especificar fuentes usando una instancia deFont. Puede determinar qué fuentes están disponibles llamando al método estático GraphicsEnvironment GraphicsEnvironment.getLocalGraphicsEnvironment y después consultar el GraphicsEnvironment devuelto. El método getAllFonts devuelve una matriz que contiene las instancias Font de todas las fuentes disponibles en el sistema. El método getAvailableFontFamilyNames devuelve una lista de las familias de fuentes disponibles.

Diseño de Texto

Antes de que el texto pueda ser visualizado, debe disponerse de modo que los caracteres estén representados por los glifos apropiados en las posiciones adecuadas. Los siguientes son dos mecanismos de Java 2D para gestionar el diseño de texto:

  • La clase TextLayout gestiona el diseño de texto, el resaltado, y la detección de colisiones. Las facilidades proporcionadas por TextLayout gestionan los casos más comunes, incluyendo cadenas con fuentes mezcladas, idiomas mezclados, y texto bidireccional.
  • Puede cerar los propios objetos GlyphVector usando la clase Font y después representa cada objeto GlyphVector a través de la clase Graphics2D. Así, puede controlar completamente como se forma y posicional el texto.

Sugerencias de Renderizado para texto

La IPA 2D de Java le permite controlar la calidad de las formas y presentación de texto usando las sugerencias de renderizado. Las sugerencias de renderizado están encapsuladas en la clase java.awt.RenderingHints.

Cuando se aplica al texto, esta capacidad se usa para suavizar (que también se conoce como bordes suaves). Por ejemplo, la sugerencia KEY_TEXT_ANTIALIASING le permite controlar la suavización del texto de forma separada de la suavización de otras formas. Para aprender más sobre las sugerencias de renderizado vea la lección Controlando la Calidad del Renderizado.
