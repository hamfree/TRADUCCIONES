Imprimiendo

Todos los gráficos 2D de Java y de Swing, incluyendo los gráficos y las imágenes compuestas, pueden ser presentadas en una impresora usando la IPA de Impresión 2D de Java. Esta IPA también proporciona características de composición de documentos que le permiten realizar operaciones como cambiar el orden en el cual las páginas se imprimen.

Presentar en una impresora es como presentar en una pantalla. El sistema de impresión controla cuándo las páginas se presentan, al igual que sistema de dibujo controla cuándo se pinta un componente en pantalla.

La IPA de Impresión 2D de Java se basa en un modelo de retrollamada (callback, en inglés) en el cual es el sistema de impresión, no la aplicación, la que controla cuando se imprimen las páginas. La aplicación proporciona al sistema de impresión la información sobre el documento a ser impreso, y el sistema de impresión determina cuándo se imprime cada página.

Las dos características siguientes son importantes para admitir la impresión:

  • Control de trabajos – Iniciando y gestionando los trabajos de impresión incluyendo la visualización de los cuadros de diálogo estándar de impresión y configuración
  • Paginación – Renderizando cada página cuando el sistema de impresión lo solicite.

Cuando es necesario crear imágenes de las páginas, el sistema de impresión llama al método print de la aplicación con un contexto Graphics apropiado. Para utilizar las características de Java 2D IPA cuando imprime, convierta el objeto Graphics en una clase Graphics2D, tal como lo hace cuando está renderizando en la pantalla.
