Coordenadas

La IPA 2D de Java mantiene dos espacios de coordenadas:

  • Espacio de usuario – El espacio en el cual se especifican las primitivas gráficas
  • Espacio del dispositivo – El sistema de coordenadas de un dispositivo de salida tal como la pantalla, una ventana, o una impresora

El espacio de usuario es un sistema lógico de coordenadas independiente del dispositivo, el espacio de coordenadas que su programa usa. Todas las geometrías que se pasan a las rutinas de presentación de Java 2D se especifican en coordenadas del espacio de usuario.

Cuando se usa la transformación predeterminada desde el espacio de usuario al espacio de dispositivo, el origen del espacio de usuario es la esquina superior izquierda del área de dibujo del componente. La coordenada x se incrementa a la derecha, y la coordenada y se incrementea hacia abajo, como se muestra en la figura siguiente. La esquina superior izquierda de una ventana es 0,0. Todas las coordenadas se especifican usando enteros, lo cual es habitualmente suficiente. Sin embargo, algunos casos requieren del punto flotante o incluso precisión doble que también está admitido.

El espacio del dispositivo es un sistema de coordenadas dependiente del dispositivo que varía de acuerdo al dispositivo de presentación objetivo. Aunque el sistema de coordenadas para una ventana o pantalla puede ser muy diferente del sistema de coordenadas de una impresora, estas diferencias son invisibles para los programas Java. Las conversiones necesarias entre espacio de usuario y espacio del dispositivo son realizadas automáticamente durante la presentación.
