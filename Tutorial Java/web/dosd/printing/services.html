Trabajando con Servicios de Impresión y Atributos

De las lecciones anteriores, ha aprendido que la IPA de impresión Java 2D admite imágenes de página, muestra cuadros de diálogo de configuración de página e impresión y especifica atributos de impresión. Los servicios de impresión son otro componente clave de cualquier subsistema de impresión.

La IPA de Servicio de Impresión de Java (JPS, en inglés) amplía las funciones de impresión Java 2D actuales para ofrecer las siguientes funciones:

  • La aplicación descubre impresoras que satisfacen sus necesidades consultando dinámicamente las capacidades de la impresora.
  • La aplicación amplía los atributos incluidos con la IPA de JPS.
  • Los terceros pueden conectar sus propios servicios de impresión con la interfaz del proveedor de servicios, que imprime diferentes formatos, incluidos Postscript, PDF y SVG.

La API del Servicio de Impresión de Java consta de cuatro paquetes:

El paquete javax.print proporciona las principales clases e interfaces para la IPA del Servicio de Impresión de Java. Permite que las aplicaciones de cliente y servidor:

  • Descubran y seleccionen servicios de impresión en función de sus capacidades.
  • Especifiquen el formato de los datos de impresión.
  • Envíen trabajos de impresión a servicios que admitan el tipo de documento que se va a imprimir.

Especificación del Tipo de Documento

La clase DocFlavor representa el formato de los datos de impresión, como JPEG o PostScript. El formato DocFlavor consta de dos partes: un tipo MIME y un nombre de clase de representación. Un tipo MIME describe el formato y un nombre de clase de representación del documento indica cómo se envía el documento a la impresora o al flujo de salida. Una aplicación utiliza DocFlavor y un conjunto de atributos para buscar impresoras con las capacidades especificadas por el conjunto de atributos. Este ejemplo de código muestra cómo obtener una matriz de objetos StreamPrintServiceFactory que pueden devolver objetos StreamPrintService capaces de convertir una imagen GIF en PostScript: 

DocFlavor flavor  = DocFlavor.INPUT_STREAM.GIF;
String psMimeType = DocFlavor.BYTE_ARRAY.
                    POSTSCRIPT.getMimeType();
StreamPrintServiceFactory[] psfactories =
              StreamPrintServiceFactory.
              lookupStreamPrintServiceFactories(
              flavor, psMimeType);

Definiciones de Atributos

Los paquetes javax.print.attribute y javax.print.attribute.standard definen atributos de impresión que describen las capacidades de un servicio de impresión, especifican los requisitos de un trabajo de impresión y realizan un seguimiento del progreso del trabajo de impresión.

Por ejemplo, si desea utilizar el formato de papel A4 e imprimir tres copias de su documento, tendrá que crear un conjunto de los siguientes atributos implementando la interfaz PrintRequestAttributeSet: 

PrintRequestAttributeSet attr_set =
    new HashPrintRequestAttributeSet();
attr_set.add(MediaSize.ISO_A4); 
attr_set.add(new Copies(3));

Luego, debe pasar el conjunto de atributos al método print del trabajo de impresión, junto con DocFlavor.

Descubrimiento del Servicio de Impresión

Una aplicación invoca los métodos estáticos de la clase abstracta PrintServiceLookup para localizar servicios de impresión que tienen la capacidad de satisfacer la solicitud de impresión de la aplicación. Por ejemplo, para imprimir dos copias de un documento a doble cara, la aplicación primero necesita encontrar impresoras que tengan capacidad de impresión a doble cara: 

DocFlavor doc_flavor = DocFlavor.INPUT_STREAM.PDF;
PrintRequestAttributeSet attr_set =
    new HashPrintRequestAttributeSet();
attr_set.add(new Copies(2));
attr_set.add(Sides.DUPLEX);
PrintService[] service = PrintServiceLookup.
              lookupPrintServices(doc_flavor,
              attr_set);

Uso común de la IPA

En conclusión, la IPA de Servicio de Impresión de Java realiza los siguientes pasos para procesar una solicitud de impresión:

  1. Elije un DocFlavor.
  2. Crea un conjunto de atributos.
  3. Localiza un servicio de impresión que puede manejar la solicitud de impresión según lo especificado por DocFlavor y el conjunto de atributos.
  4. Crea un objeto Doc que encapsula el DocFlavor y los datos de impresión reales.
  5. Obtiene un trabajo de impresión, representado por DocPrintJob, del servicio de impresión.
  6. Llama al método print del trabajo de impresión.

Para obtener más información sobre el Servicio de Impresión de Java, consulte la Guía del usuario de IPA del Servicio de Impresión 2D de Java.
