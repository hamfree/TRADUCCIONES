Midiendo Texto

Para medir correctamente el texto, debe aprender algunos métodos y evitar algunos errores. Las métricas de fuentes son medidas de texto representado por un objeto Font como la altura de una línea de texto en la fuente. La forma más común de medir el texto es utilizar una instancia de FontMetrics que encapsula esta información de métricas. Por ejemplo: 

// obtiene métricas de los gráficos
FontMetrics metrics = graphics.getFontMetrics(font);
// obtiene la altura de una línea de texto en esta
// fuente y contexto de renderizado
int hgt = metrics.getHeight();
// obtiene el avance de mi texto en esta fuente
// y renderiza el contexto
int adv = metrics.stringWidth(text);
// calcula el tamaño de una caja para contener el
// texto con algo de relleno.
Dimension size = new Dimension(adv+2, hgt+2);

Esta forma es suficiente para que muchas aplicaciones espacien uniformemente líneas de texto o para dimensionar componentes de Swing.

Tenga en cuenta lo siguiente:

  • Las métricas se obtienen del objeto Graphics, porque esta clase encapsula el FontRenderContext, que es necesario para medir el texto con precisión. En las resoluciones de pantalla, las fuentes se ajustan para facilitar la lectura. A medida que aumenta el tamaño del texto, este ajuste no se escala linealmente. Por lo tanto, a 20 puntos, una fuente no mostrará el texto exactamente el doble de largo que a 10 puntos. Además del texto en sí y la fuente, la otra pieza importante de información necesaria para medir el texto es FontRenderContext. Este método incluye la transformación del espacio del usuario a los píxeles del dispositivo que se utiliza para medir el texto.
  • La altura se informa sin referencia a ninguna cadena de texto en particular. Es útil, por ejemplo, en un editor de texto en el que desea el mismo espacio entre líneas entre cada línea de texto.
  • stringWidth() devuelve el ancho de avance del texto. Ancho de avance es la distancia desde el origen del texto hasta la posición de una cadena renderizada posteriormente.

Cuando utilice estos métodos para medir texto, tenga en cuenta que el texto puede extenderse en cualquier dirección fuera de un rectángulo, definido por la altura de la fuente y el avance de la cadena.

Normalmente, la solución más sencilla es asegurarse de que el texto no se recorta, por ejemplo, por componentes que rodean el texto. Agregue relleno en los casos en que el texto podría recortarse.

Si esta solución es insuficiente, otra IPA de medición de texto en el software 2D de Java puede devolver cuadros delimitadores rectangulares. Estos cuadros tienen en cuenta la altura del texto específico que se va a medir y los efectos de pixelización.
