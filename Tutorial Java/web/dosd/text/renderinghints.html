Visualizando Texto Suavizado Usando Consejos de Renderizado

La representación de texto Java 2D puede verse afectada por sugerencias de representación.

Recuerde que el método de dibujo de texto más importante es el siguiente: 

Graphics.drawString(String s, int x, int y);

Por lo general, este método dibuja cada glifo en una cadena de texto con un color sólido y cada píxel que está “ encendido ” en ese glifo se establece en ese color. Este tipo de dibujo produce el texto de mayor contraste, pero a veces con bordes irregulares (alias).

El suavizado de texto es una técnica que se utiliza para suavizar los bordes del texto en una pantalla. La IPA 2D de Java permite a las aplicaciones especificar si se debe utilizar esta técnica y qué algoritmo utilizar mediante la aplicación de una sugerencia de representación de texto al objeto Graphics.

La sugerencia de representación más común combina el color de primer plano (texto) con los píxeles de fondo en pantalla en los bordes del texto. Para solicitar esta sugerencia, una aplicación debe invocar lo siguiente: 

graphics2D.setRenderingHint(
        RenderingHints.KEY_TEXT_ANTIALIASING,
        RenderingHints.VALUE_TEXT_ANTIALIAS_ON);

La siguiente figura ilustra la capacidad de suavizado.

Si se usa de manera inapropiada, este método puede hacer que el texto parezca demasiado borroso. En tales casos, una mejor pista para usar es la siguiente: 

graphics2D.setRenderingHint(
        RenderingHints.KEY_TEXT_ANTIALIASING,
        RenderingHints.VALUE_TEXT_ANTIALIAS_GASP);

Este método utiliza automáticamente la información de la fuente para decidir si utilizar antialiasing o colores sólidos.

Las pantallas LCD tienen una propiedad que la IPA 2D de Java puede usar para producir texto que no es tan borroso como el suavizado típico, pero es más legible en tamaños pequeños. Para solicitar que el texto se dibuje utilizando el modo de texto LCD de subpíxeles para una pantalla LCD típica, una aplicación debe invocar lo siguiente: 

graphics2D.setRenderingHint(
        RenderingHints.KEY_TEXT_ANTIALIASING,
        RenderingHints.VALUE_TEXT_ANTIALIAS_LCD_HRGB);

El ejemplo de código representado a continuación ilustra la capacidad de suavizado en el siguiente orden:

  1. El suavizado está desactivado.
  2. El suavizado está activado.
  3. Suavizado usando la sugerencia TEXT_ANTIALIAS_GASP.
    Nota: Por lo tanto, la tabla GASP especifica que se utilicen solo sugerencias para esos tamaños y no "suavizado". Entonces, en muchos casos, la visualización de texto resultante es equivalente a VALUE_TEXT_ANTIALIAS_OFF.
  4. Suavizado usando la sugerencia TEXT_ANTIALIAS_LCD_HRGB.

Note: Si no ve el subprograma ejecutándose, debe instalar al menos la versión 7 del Kit de Desarrollo Java SE (KDJ).
