Fuentes Físicas y Lógicas

Hay dos tipos de fuentes: fuentes físicas y fuentes lógicas. Las fuentes físicas son las bibliotecas de fuentes reales que constan, por ejemplo, de fuentes TrueType o PostScript Type 1. Las fuentes físicas pueden ser Time, Helvetica, Courier o cualquier otra fuente, incluidas las internacionales. Las fuentes lógicas son las siguientes cinco familias de fuentes: Serif, SansSerif, Monospaced, Dialog y DialogInput. Estas fuentes lógicas no son bibliotecas de fuentes reales. En su lugar, los nombres de fuentes lógicas se asignan a fuentes físicas mediante el entorno de ejecución de Java.

Esta sección le ayuda a determinar qué tipo de fuente utilizar en su aplicación. Cubre los siguientes temas: topics:

  • Fuentes Físicas

    • Combinación de Fuentes Físicas con su Aplicación
  • Fuentes Lógicas

Fuentes Físicas

Las fuentes físicas son las bibliotecas de fuentes reales que contienen datos de glifos y tablas para mapear desde secuencias de caracteres a secuencias de glifos, utilizando una tecnología de fuentes como TrueType o PostScript Type 1. Para obtener los nombres de todas las familias de fuentes disponibles instaladas en su sistema, llame a lo siguiente: 

GraphicsEnvironment ge = GraphicsEnvironment.getLocalGraphicsEnvironment();
String []fontFamilies = ge.getAvailableFontFamilyNames();

El programa de muestra FontSelector (disponible en FontSelector.java) ilustra cómo localizar y seleccionar estas fuentes.

Nota: las aplicaciones no deben asumir que hay una fuente física en particular. Sin embargo, las fuentes lógicas son una opción segura porque siempre están presentes. Ver Fuentes Lógicas para más información.

Nota:  si no ve el applet ejecutándose, debe instalar al menos la versión del Kit de Desarrollo Java SE (KDJ) 7.

Fuentes Lucidia

Los EEJ de Oracle contienen esta familia de fuentes físicas, que también tiene licencia para su uso en otras implementaciones de la plataforma Java. Estas fuentes son fuentes físicas, pero no dependen del sistema operativo anfitrión.

Las aplicaciones que utilizan estas fuentes pueden lograr el mismo aspecto siempre que estén disponibles. Además, estas fuentes cubren una amplia gama de idiomas (especialmente europeos y del Medio Oriente), por lo que puede crear aplicaciones completamente multilingües para los idiomas admitidos. Sin embargo, es posible que estas fuentes no estén disponibles en todos los EEJ. Además, actualmente no cubren el conjunto completo de caracteres Unicode; en particular, no se admiten chino, japonés y coreano.

Combinación de Fuentes Físicas con su Aplicación

A veces, una aplicación no puede depender de la instalación de una fuente en el sistema, generalmente porque la fuente es una fuente personalizada que de otro modo no está disponible. En este caso, debe empaquetar los archivos de fuentes con su aplicación.

Utilice uno de estos métodos para crear un objeto Font a partir de una fuente física existente: 

Font java.awt.Font.createFont(int fontFormat, InputStream in);
Font java.awt.Font.createFont(int fontFormat, File fontFile);

Para crear un objeto Font a partir de una fuente TrueType, el parámetro formal fontFormat debe ser la constante Font.TRUETYPE_FONT. El siguiente ejemplo crea un objeto Font a partir del archivo de fuentes TrueType A.ttf: 

Font font = Font.createFont(Font.TRUETYPE_FONT, new File("A.ttf"));

Acceder a la fuente directamente desde un archivo es más simple y conveniente. Sin embargo, es posible que necesite un objeto InputStream si su código no puede acceder a los recursos del sistema de archivos, o si la fuente está empaquetada en un archivo Java Archive (JAR) junto con el resto de la aplicación o applet.

El método createFont crea un nuevo objeto Font con un tamaño de punto de 1 y estilo PLAIN. Esta fuente base se puede usar con los métodos Font.deriveFont para derivar nuevos objetos Font con diferentes tamaños, estilos, transformaciones y características de fuente. Por ejemplo: 

try {
     //La fuente devuelta es de tamaño 1 pt
     Font font = Font.createFont(Font.TRUETYPE_FONT, new File("A.ttf"));

     //Deriva y devuelve una versión de 12 pt:
     //Necesita el uso de float de otra manera
     //sería interpretado como un estilo

     return font.deriveFont(12f);

} catch (IOException|FontFormatException e) {
     // Maneja la excepción
}

Es importante utilizar el método deriveFont porque las fuentes creadas por una aplicación no forman parte del conjunto de fuentes conocidas por el sistema de fuentes subyacente. Debido a que el método deriveFont funciona a partir de la fuente creada originalmente, no tiene esta limitación.

La solución a este problema es registrar la fuente creada con el entorno gráfico. Por ejemplo: 

try {
     GraphicsEnvironment ge = 
         GraphicsEnvironment.getLocalGraphicsEnvironment();
     ge.registerFont(Font.createFont(Font.TRUETYPE_FONT, new File("A.ttf"));
} catch (IOException|FontFormatException e) {
     //Maneja la excepción
}

Después de registrar la fuente con el entorno de gráficos, la fuente está disponible en llamadas a getAvailableFontFamilyNames() y se puede utilizar en constructores de fuentes.

Fuentes Lógicas

Java SE define las siguientes cinco familias de fuentes lógicas:

  • Dialog
  • DialogInput
  • Monospaced
  • Serif
  • SansSerif

Estas fuentes están disponibles en cualquier plataforma Java y se pueden considerar como alias de alguna fuente subyacente que tiene las propiedades implícitas en su nombre. Una fuente Serif es una fuente similar a Times New Roman, que se usa comúnmente en la impresión. Una fuente Sans Serif es más típica para el uso en pantalla.

Estas fuentes se pueden personalizar para la configuración regional del usuario. Además, estas fuentes admiten la más amplia gama de puntos de código (caracteres Unicode).

Aparte de la familia, las fuentes tienen otros atributos, los más importantes son estilo y tamaño. Los estilos son Negrita y Cursiva.

La fuente predeterminada utilizada por la IPA 2D de Java es Dialog de 12 puntos. Esta fuente tiene un tamaño de puntos típico para leer texto en un dispositivo de visualización normal de 72 – 120 DPI. Una aplicación puede crear una instancia de esta fuente directamente especificando lo siguiente: 

Font font = new Font("Dialog", Font.PLAIN, 12);

Ventajas y Desventajas del Uso de Fuentes Físicas y Lógicas

Las fuentes físicas permiten que una aplicación aproveche al máximo todas las fuentes disponibles, para lograr tanto diferentes apariencias de texto como la máxima cobertura de idioma. Sin embargo, es mucho más difícil crear aplicaciones que utilicen fuentes físicas.

La combinación de fuentes físicas con su aplicación le permite crear aplicaciones que tienen el mismo aspecto en todas partes y tener un control total sobre las aplicaciones que desea admitir. Sin embargo, las fuentes empaquetadas pueden ser bastante grandes, especialmente si desea que sus aplicaciones admitan chino, japonés y coreano. Además, es posible que deba resolver problemas de licencias.

Se garantiza que los nombres de fuentes lógicas funcionan en cualquier lugar y permiten la representación de texto en al menos el idioma para el que está localizado el sistema operativo anfitrión (a menudo, una gama de idiomas mucho más amplia). Sin embargo, las fuentes físicas utilizadas para representar el texto varían entre diferentes implementaciones, sistemas operativos anfitrión y configuraciones regionales, por lo que una aplicación no puede lograr el mismo aspecto en todas partes. Además, los mecanismos de mapeo ocasionalmente limitan el rango de caracteres que se pueden representar. Este último solía ser un gran problema en las versiones EEJ anteriores a la 5.0: por ejemplo, el texto en japonés solo se podía representar en sistemas operativos de anfitrión localizados en japonés, no en otros sistemas localizados, incluso si se habían instalado fuentes japonesas. Para las aplicaciones que usan renderizado de fuentes 2D, este problema es mucho más raro con EEJ versión 5.0 y posterior, porque el mecanismo de mapeo ahora generalmente reconoce y usa fuentes para todos los sistemas de escritura admitidos si están instalados.

Archivos de Configuración de Fuentes

El entorno de ejecución de Java SE utiliza archivos de configuración de fuentes para asignar nombres de fuentes lógicas a fuentes físicas. Hay varios archivos para admitir diferentes asignaciones según la versión del sistema operativo del anfitrión. Los archivos se encuentran en el directorio lib dentro de la instalación del EEJ. Puede editar o crear sus propios archivos de configuración de fuentes para ajustar las asignaciones a la configuración de su sistema en particular. Ver Archivos de Configuración de Fuentes para más información.
