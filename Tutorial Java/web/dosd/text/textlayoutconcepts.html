Conceptos de Diseño del Texto

Antes de que se pueda mostrar un fragmento de texto, debe tener la forma y la posición adecuadas con los glifos y ligaduras adecuados. Este proceso se conoce como diseño de texto. El proceso de diseño de texto implica lo siguiente:

  • Dar forma al texto con los glifos y ligaduras adecuados
  • Ordenar correctamente el texto
  • Medir y posicionar el texto

La información utilizada para diseñar el texto también es necesaria para realizar operaciones de texto como el posicionamiento de los signos de intercalación, la detección de colisiones y el resaltado. Vea Trabajando con Texto Bidireccional para obtener más información sobre estas operaciones de texto.

Para desarrollar software que se pueda implementar en los mercados internacionales, el texto debe presentarse en diferentes idiomas de manera que se ajuste a las reglas del sistema de escritura apropiado.

Esta sección cubre los siguientes temas:

  • Dar forma al texto
  • Ordenar texto
  • Medición y posicionamiento de texto

Dar forma al texto

Un glifo es la representación visual de uno o más caracteres. La forma, el tamaño y la posición de un glifo dependen de su contexto. Se pueden usar muchos glifos diferentes para representar un solo carácter o una combinación de caracteres, según la fuente y el estilo.

Por ejemplo, en el texto cursivo escrito a mano, un carácter en particular puede adoptar diferentes formas dependiendo de cómo esté conectado a los caracteres adyacentes.

En algunos sistemas de escritura, particularmente el árabe, siempre se debe tener en cuenta el contexto de un glifo. A diferencia del inglés, las formas cursivas son obligatorias en árabe; es inaceptable presentar texto sin utilizar formas cursivas.

Ordenar texto

En el lenguaje de programación Java, el texto se codifica mediante la codificación de caracteres Unicode. El texto que usa codificación de caracteres Unicode se almacena en la memoria en orden lógico. El orden lógico es el orden en el que se leen y escriben los caracteres y las palabras. El orden lógico no es necesariamente el mismo que el orden visual, el orden en el que se muestran los glifos correspondientes.

El orden visual de los glifos en un sistema de escritura en particular (secuencia de comandos) se denomina orden de secuencia de comandos. Por ejemplo, el orden de escritura para el texto romano es de izquierda a derecha y el orden de escritura para árabe y hebreo es de derecha a izquierda.

Algunos sistemas de escritura tienen reglas además del orden de la escritura para organizar glifos y palabras en líneas de texto. Por ejemplo, los números árabes y hebreos van de izquierda a derecha, aunque las letras van de derecha a izquierda. Esto significa que el árabe y el hebreo, incluso sin texto en inglés incorporado, son verdaderamente bidireccionales. Ver Trabajando con Texto Bidireccional para más información.

Medición y posicionamiento de texto

A menos que esté trabajando con una fuente monoespaciada, los diferentes caracteres de una fuente tienen diferentes anchos. Esto significa que todo posicionamiento y medición de texto debe tener en cuenta exactamente qué caracteres se utilizan, no solo cuántos. Por ejemplo, para alinear a la derecha una columna de números mostrada en una fuente proporcional, no puede simplemente usar espacios adicionales para colocar el texto. Para alinear correctamente la columna, necesita saber el ancho exacto de cada número para poder ajustar en consecuencia.

El texto a menudo se muestra con varias fuentes y estilos, como negrita o cursiva. En este caso, incluso el mismo carácter puede tener diferentes formas y anchos, dependiendo de su estilo. Para posicionar, medir y representar correctamente el texto, debe realizar un seguimiento de cada carácter individual y el estilo aplicado a ese carácter. Afortunadamente, la clase TextLayout lo hace por usted.

Para mostrar correctamente el texto en idiomas como el hebreo y el árabe, cada carácter individual debe medirse y colocarse dentro del contexto de los caracteres vecinos. Debido a que las formas y posiciones de los caracteres pueden cambiar según el contexto, medir y posicionar dicho texto sin tener en cuenta el contexto produce resultados inaceptables.

Además, Java SE le proporciona la clase FontMetrics, que le permite obtener medidas de texto representado por un objeto Font como la altura de una línea de texto en la fuente. Puede utilizar esta información para colocar con precisión el texto en aplicaciones gráficas Java. Ver Midiendo Texto para más información.
