Respuestas a Preguntas y Ejercicios

Preguntas

  1. Pregunta: Al principio de esta lección, aprendió que el núcleo de interfaces de colección está organizado en dos árboles de herencia distintos. Un interfaz en particular no es considerado una Collection verdadera, y por lo tanto se sienta en la cima de su propio árbol. ¿Cuál es el nombre de esta interfaz?
    Respuesta: Map

  2. Pregunta: Cada interfaz en el marco de trabajo de colecciones se declara con la sintaxis <E>, la cual le indica que es un genérico. Cuando declara una instancia Collection, ¿cuál es la ventaja de especificar el tipo de objetos que lo contendrá?
    Respuesta: Especificar el tipo permite al compilador verificar (en tiempo de compilación) que el tipo de objetos que coloca en la colección es correcto, lo que reduce los errores en tiempo de ejecución.

  3. Pregunta: ¿Qué interfaz representa una colección que no permite elementos duplicados?
    Respuesta: Set

  4. Pregunta: ¿Qué interfaz forma la raíz de la jerarquía de colecciones?
    Respuesta: Collection

  5. Pregunta: ¿Qué interfaz representa una colección ordenada que puede contener elementos duplicados?
    Respuesta: List

  6. Pregunta: ¿Qué interfaz representa una colección que contiene elementos antes del procesamiento?
    Respuesta: Queue

  7. Pregunta: ¿Qué interfaz representa un tipo que asoica claves a valores?
    Respuesta: Map

  8. Pregunta: ¿Qué interfaz representa una cola de doble extremo?
    Respuesta: Deque

  9. Pregunta: Nombra tres formas diferentes para iterar sobre los elementos de una List.
    Respuesta: Puede iterar sobre una List usando flujos, la sentencia mejorada for, o iteradores.

  10. Pregunta: Verdadero o Falso: Las operaciónes de agregación son operaciones mutativas que modifican la colección subyacente.
    Respuesta: Falso. Las operaciones agregadas no mutan la colección subyacente. En realidad, debe ser cuidadoso para nunca modificar una colección mientras invoca sus operaciones de agregación. Si lo hace, podría generar problemas de concurrencia si el flujo se cambia a un flujo paralelo en algún momento del futuro.

Ejercicios

  1. Ejercicio: Escriba un programa que imprima sus argumentos en orden aleatorio. No haga una copia de la matriz de argumentos. Demuestra cómo imprimir los elementos usando ambos flujos y la sentencia for mejorada tradicional.
    Respuesta: 
    
import java.util.*;

public class Ran {

    public static void main(String[] args) {
        
        // Obtiene y mezcla la lista de argumentos
        List<String> argList = Arrays.asList(args);
        Collections.shuffle(argList);

        // Imprime los elementos usando Streams de KDJ 8
        argList.stream()
        .forEach(e->System.out.format("%s ",e));

        // Imprime los elementos usando for-each
        for (String arg: argList) {
            System.out.format("%s ", arg);
        }

        System.out.println();
    }
}
  2. Ejercicio: Toma el ejemplo FindDups y modifíquelo para usar un SortedSet en vez de un Set. Especifique un Comparator por lo que ese caso se ignora cuando se ordena e identifican elementos establecidos.
    Respuesta: 
    import java.util.*;

public class FindDups {
    public static void main(String[] args) {
        Set<String> s = new HashSet<String>();
        for (String a : args)
               s.add(a);
               System.out.println(s.size() + " palabras distintas: " + s);
    }
}
  3. Ejercicio: Escriba un método que toma una List<String> y aplique String.trim a cada elemento.
    Respuesta:
    La sentencia mejorada for no le permite modificar la List. Usar una instancia de la clase Iterator le permite eliminar elementos, pero no reemplzar un elemento existente o añadir uno nuevo. Eso nos deja con ListIterator: 
    import java.util.*;

public class ListTrim {
    static void listTrim(List<String> strings) {
        for (ListIterator<String> lit = strings.listIterator(); lit.hasNext(); ) {
            lit.set(lit.next().trim());
        }
    }

    public static void main(String[] args) {
        List<String> l = Arrays.asList(" red ", " white ", " blue ");
        listTrim(l);
        for (String s : l) {
            System.out.format("\"%s\"%n", s);
        }
    }
}
  4. Ejercicio: Considere las cuatro interfaces principales, Set, List, Queue, y Map. Para cada una de las siguientes cuatro asignaciones, especifique cuál de las cuatro interfaces principales se ajusta mejor, y explique cómo usarla para implementar la asignación.
    Respuestas:
    • Whimsical Toys Inc (WTI) necesita grabar los nombres de todos sus empleados. Cada mes, un empleado será elegido aleatoriamente desde estos registros para recibir un juguete gratis.
      Use una List. Seleccione un empleado aleatorio eligiendo un número entre 0 y size()-1.
    • WTI ha decidido que cada producto nuevo sea nombrado después de que un empleado pero sólo los nombres se usarán, y cada nombre será usado sólo una vez. Prepare una lista de nombre únicos.
      Use un Set. Las colecciones que implementan esta interfaz no permiten al mismo elemento ser introducido más de una vez.
    • WTI decide que sólo quiere usar los nombres más populares para sus juguetes. Cuente el número de empleados que tienen cada nombre.
      Use un Map, dodne las claves son los nombres, y cada valor es una cuenta del número de empleados con ese nombre.
    • WTI adquiere boletos de temporada para el equipo local de lacrosse, para ser compartido por empleados. Cree una lista de espera para este deporte popular.
      Use una Queue. Invoque add() para agregar empleados a una lista de espera, y remove() para eliminarlosto remove them.
