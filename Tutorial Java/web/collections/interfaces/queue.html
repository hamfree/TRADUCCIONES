El Interfaz Queue

Una Queue es una colección para contener elementos antes del procesamiento. Además de las operaciones básicas de Collection, las queues (colas, en español) suministran las operaciones adicionales de inserción, eliminación e inspección. La interfaz Queue se muestra a continuación. 

public interface Queue<E> extends Collection<E> {
    E element();
    boolean offer(E e);
    E peek();
    E poll();
    E remove();
}

Cada método de Queue existe en dos formas: (1) uno lanza una excepción si la operación falla, y (2) el otro devuelve un valor especial si la operación falla (o null o false, dependiendo de la operación). La estructura regular de la interfaz es ilustrada en la tabla siguiente.

Estructura de la Interfaz Queue
Tipo de Operación Lanza excepción Devuelve valor especial
Insertar add(e) offer(e)
Eliminar remove() poll()
Examinar element() peek()

Las colas típicamente, pero no necesariamente, ordenan los elementos de una manera FIFO (first-in-first-out, primero en entrar, primero en salir, en español). Entre las excepciones están las colas de prioridad, las cuales ordenan los elementos de acuerdo a sus valores — vea la sección Ordenación de Objetos para más detalles). Cualquiera que sea el orden utilizado, la cabeza de la cola es el elemento que será eliminado por una llamada a remove o poll. En una cola FIFO, todos los elementos nuevos son insertados al final de la cola. Otras clases de colas pueden usar reglas diferentes de colocación. Cada implementación de Queue debe especificar sus propiedades de ordenación.

Es posible para una implementación de Queue restringir el número de elementos que almacena; tales colas son conocidas como limitadas. Algunas implementaciones de Queue en java.util.concurrent son limitadas, pero las implementaciones en java.util no lo son.

El método add, el cual Queue hereda desde Collection, inserta un elemento a menos que viole las restricciones de capacidad de la cola, en cuyo caso lanza IllegalStateException. El método offer, el cual está destinado únicamente para su uso en colas limitadas, difiere de add solo en que indica fallo al insertar un elemento devolviendo false.

Los métodos remove y poll eliminan y devuelven la cabeza de la cola. Exactamente qué elemento se elimina es una función de la política de ordenación de la cola. Los métodos remove y poll difieren en su comportamiento sólo cuando la cola está vacía. Bajo estas circunstancias, remove lanza NoSuchElementException, mientras poll devuelve null.

Los métodos element y peek devuelven, pero no eliminan, la cabeza de la cola. Se diferencian entre sí precisamente de la misma manera queremove y poll: Si la cola está vacía, element lanza NoSuchElementException, mientras peek devuelve null.

Las implementaciones de Queue generalmente no permiten inserciones de elementos null. La implementación de LinkedList, que fue adaptada para implementar Queue, es una excepción. Por razones históricas, permite elementos null, pero debe abstenerse de tomar ventaja de esto, porque null es usado como un valor de retorno especial por los métodos poll y peek.

Las implementaciones de Queue generalmente no definen versiones basadas en elementos de los métodos equals y hashCode sino que heredan las versiones basadas en identidad de Object.

La interfaz Queue no define los métodos de bloqueo de la cola, los cuales son comunes en la programación concurrente. Estos métodos, que esperan que aparezcan los elementos o que el espacio esté disponible, se definen en la interfaz java.util.concurrent.BlockingQueue , el cual extiende Queue.

En el programa de ejemplo siguiente, una cola es usada para implementar un temporizador de cuenta regresiva. La cola es precargada con todos los valores enteros desde un número especificado en la línea de comandos a cero, en orden descendente. Después, los valores son eliminados desde la cola e impresos en intervalos de un segundo. El programa es artificial en el sentido de que sería más natural hacer lo mismo sin utilizar una cola, pero ilustra el uso de una cola para almacenar elementos antes del procesamiento posterior. 

import java.util.*;

public class Countdown {
    public static void main(String[] args) throws InterruptedException {
        int time = Integer.parseInt(args[0]);
        Queue<Integer> queue = new LinkedList<Integer>();

        for (int i = time; i >= 0; i--)
            queue.add(i);

        while (!queue.isEmpty()) {
            System.out.println(queue.remove());
            Thread.sleep(1000);
        }
    }
}

En el ejemplo siguiente, una cola de prioridad es usada para ordenar una colección de elementos. De nuevo, este programa es artificial porque no hay ninguna razón para usarlo a favor del método sort proporcionado en Collections, pero ilustra el comportamiento de las colas de prioridad. 

static <E> List<E> heapSort(Collection<E> c) {
    Queue<E> queue = new PriorityQueue<E>(c);
    List<E> result = new ArrayList<E>();

    while (!queue.isEmpty())
        result.add(queue.remove());

    return result;
}
