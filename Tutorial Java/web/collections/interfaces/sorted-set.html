El Interfaz SortedSet

Un SortedSet (Conjunto Ordenado, en español, Nota del Traductor.) es un Set que mantiene sus elementos en orden ascendete, ordenados de acuerdo al orden natural de los elementos o de acuerdo a un Comparator suminitrado cuando se crea el SortedSet. Además de las operaciones normales de Set, el interfaz SortedSet suministra operaciones para lo siguiente:

  • Vista de rango — permite operaciones de rango arbitrarias sobre el conjunto ordenado
  • Puntos finales — devuelve el primer o último elemento en el conjunto ordenado
  • Acceso al Comparator — devuelve el Comparator, si existe, usado para ordenar el conjunto

El código para la interfaz SortedSet a continuación. 

public interface SortedSet<E> extends Set<E> {
    // Vista de rango
    SortedSet<E> subSet(E fromElement, E toElement);
    SortedSet<E> headSet(E toElement);
    SortedSet<E> tailSet(E fromElement);

    // Puntos finales
    E first();
    E last();

    // Acceso al comparador
    Comparator<? super E> comparator();
}

Operaciones de Set

Las operaciones que SortedSet hereda de Set se comportan de manera idéntica en conjuntos ordenados y conjuntos normales con dos excepciones:

  • El Iterator devuelto por la operación iterator atraviesa el conjunto ordenado en orden.
  • La matriz devuelta por toArray contiene los elementos del conjunto ordenado en orden.

Aunque la interfaz no lo garantiza, el método toString de las implementaciones de SortedSet de la plataforma de Java devuelven una cadena conteniendo todos los elementos del conjunto ordenado, en orden.

Constructores Estándar

Por convención, todas las implementaciones de propósito general de Collection suministran un constructor estándar de conversión que toman una Collection; las implementaciones de SortedSet no son una excepción. En TreeSet, este constructor crea una instancia que almacena sus elementos de acuerdo a su orden natural. Esto fue probablemente un error. Hubiera sido mejor verificar dinámicamente para ver si la colección especificada era una instancia de SortedSet y, si es así, ordenar el nuevo TreeSet de acuerdo al mismo criterio (comparator u orden natural). Debido a que TreeSet tomó el enfoque que lo hace, también suministra un constructor que toma un SortedSet y devuelve un nuevo TreeSet conteniendo los mismos elementos ordenados de acuerdo al mismo criterio. Dese cuenta de que es el tipo del argumento en tiempo de compilación, no su tipo en tiempo de ejecución, que determina cual de estos dos constructores es invocado (y si el criterio de ordenación es preservado).

Las implementaciones de SortedSet también suministran, por convención, un constructor que toma un Comparator y devuelven un conjunto vacío ordenado de acuerdo al Comparator especificado. Si se pasa null a este constructor, devuelve un conjunto que ordena sus elementos de acuerdo a su orden natural.

Operaciones Vista de Rango

Las operaciones vista de rango son de alguna forma análogas a las provistas por el interfaz List pero hay una gran diferencia. Las vistas de rango de un conjunto ordenado permanecen válidas incluso si el conjunto ordenado subyacente es modificado directamente. Esto es factible porque los puntos finales de una vista de rango de un conjunto ordenado son puntos absolutos en el espacio de elementos en lugar de elementos específicos en la colección de respaldo, como es el caso de las listas. Una vista de rango de un conjunto ordenado es realmente sólo una ventana en cualquier parte del conjunto que se encuentre en la parte designada del espacio del elemento. Los cambios a la vista de rango se escriben de vuelta en el conjunto ordenado de respaldo y viceversa. Por lo tanto, está bien utilizar vistas de rangos en conjuntos ordenados durante largos períodos de tiempo, a diferencia de las vistas de rangos en listas.

Los conjuntos ordenados suministran tres operaciones vista de rango. Los primeros, subSet, toman dos puntos finales, como subList. En lugar de índices, los puntos finales son objetos y deben ser comparables a los elementos del conjunto ordenado, usando el Comparator de Set o el orden natural de sus elementos, lo que el Set usa para ordernarse. Como subList, el rango está medio abierto, incluyendo su punto final bajo pero excluyendo el alto.

Así, la siguiente línea de código le indica cuántas palabras hay entre "doorbell" y "pickle", incluyendo "doorbell" pero excluyendo "pickle", son contenidos en un SortedSet de cadenas llamado dictionary: 

int count = dictionary.subSet("doorbell", "pickle").size();

De la misma manera, la siguiente línea elimina todos los elementos que comienzan con la letra f. 

dictionary.subSet("f", "g").clear();

Un truco similar puede ser usado para imprimir una tabla indicándole cuantas palabras empiezan cada letra. 

for (char ch = 'a'; ch <= 'z'; ) {
    String from = String.valueOf(ch++);
    String to = String.valueOf(ch);
    System.out.println(from + ": " + dictionary.subSet(from, to).size());
}

Suponga que quiere ver un intervalo cerrado, el cual contiene ambos puntos finales, en vez de un intervalo abierto. Si el tipo del elemento permite el cálculo del sucesor de un vaor dado en el espacio de elementos, simplemente solicite el subSet desde lowEndpoint a successor(highEndpoint). Aunque no es enteramente obvio, el sucesor de una cadena s en el orden natural de String es s + "\0" — que es, s con un carácter null añadido.

Así, la siguiente línea le indica cuantas palabras hay entre "doorbell" y "pickle", incluyendo doorbell y pickle, están contenidos en el diccionario. 

count = dictionary.subSet("doorbell", "pickle\0").size();

Una técnica similar puede ser usado para ver un intervalo abierto, que no contiene ningún punto final. La vista de intervalo abierto desde lowEndpoint a highEndpoint es el intervalo medio abierto desde successor(lowEndpoint) a highEndpoint. Use lo siguiente para calcular el número de palabras entre "doorbell" y "pickle", excluyendo ambos. 

count = dictionary.subSet("doorbell\0", "pickle").size();

La interfaz SortedSet contiene dos operaciones más de vista de rango — headSet y tailSet, ambas toman un único argumento Object. El primero devuelve una vista de la porción inicial de la SortedSet subyacente, hasta, pero sin incluir, el objeto especificado. El último devuelve una vista de la porción final de la SortedSet subyacente, empezando con el objeto especificado y continuando hasta el final de la SortedSet subyacente. Así, el código siguiente permite ver el diccionario como dos volúmenes disjuntos (a-m y n-z). 

SortedSet<String> volume1 = dictionary.headSet("n");
SortedSet<String> volume2 = dictionary.tailSet("n");

Operaciones de punto final

El interfaz SortedSet contiene operaciones para devolver el primero y último elemento en el conjunto ordenado, no sorprendentemente llamados first y last. Además de sus usos obvios, last permite una solución a una deficiencia en la interfaz SortedSet. Una cosa que le gustaría hacer con un SortedSet es ir al interior del Set e iterar hacia adelante o hacia atrás. Es bastante fácil ir hacia adelante desde el interior: simplemente obtenga un tailSet e itere sobre él. Desafortunadamente, no hay una forma fácil de ir hacia atrás.

La expresión siguiente obtiene el primer elemento que es menor que un objeto especificado o en el espacio de elementos. 

Object predecessor = ss.headSet(o).last();

Esta es una buena forma de ir un elemento hacia atrás desde un punto en el interior de un conjunto ordenado. Podría ser aplicado repetidamente para iterar hacia atrás, pero esto es muy ineficiente, requiriendo una búsqueda para cada elemento devuelto.

Acceso al Comparador

El interfaz SortedSet contiene un método accesor llamado comparator que devuelve el Comparator usado para ordenar el conjunto, o null si el conjunto es almacenado de acuerdo al orden natural de sus elementos. Este método es suministrado de forma que los conjuntos ordenados puedan ser copiados en nuevos conjuntos ordenados con el mismo orden. Es usado por el constructor de SortedSet descrito previamente.
