Sumario de Interfaces

El núcleo de interfaces de colección son la base del Marco de Trabajo de Colecciones de Java.

La jerarquía del Marco de Trabajo de Colecciones de Java consiste en dos árboles distintos de interfaces:

  • El primer arbol empieza con el interfaz Collection, el cual suministra la funcionalidad básica usada por todas las colecciones, tales como los métodos add y remove. las subinterfaces —Set, List, y Queue — suministran colecciones más especializadas.

  • El interfaz Set no permite elementos duplicados. Esto puede ser útil para almacenar colecciones tales como una baraja de cartas o registros de estudiantes. El interfaz Set tiene una subinterfaz, SortedSet, que suministra la ordenación de elementos en el conjunto.

  • La interfaz List suministra una colección ordenada, para situaciones en las cuales necesita control preciso sobre dónde es insertado cada elemento. Puede recuperar elementos de una List por su posición exacta.

  • La interfaz Queue habilita las operaciones adicionales de inserción, extracción e inspección. Los elementos en una Queue están ordenados típicamente sobre una base FIFO.

  • La interfaz Deque habilita las operaciones de inserción, borrado, e inspección en ambos extremos. Los elementos en un Deque pueden ser usados tanto en ordenación LIFO como FIFO.

  • El segundo árbol empieza con el interfaz Map, el cual asigna claves y valores de forma similar a una Hashtable.

  • La subinterfaz de Map, SortedMap, mantiene sus pares claves-valor en orden ascendente o en un orden especificado por un Comparator.

Estos interfaces permiten a las colecciones ser manipuladas independientemente de los detalles de su representación.
