Lección: Interfaces

El núcleo de interfaces de colección encapsula tipos diferentes de colecciones, los cuales son mostradas en la figura de abajo. Estas interfaces permiten a las colecciones ser manipuladas independientemente de los detalles de su representación. El nucleo de interfaces de colección es la base del Marco de trabajo de Colecciones de Java. Como puede ver en la siguiente figura, el núcleo de interfaces de colección forma una jerarquía.

El núcleo de interfaces de colección.

Un Set es una clase especial de Collection, un SortedSet es una clase especial de Set, etcétera. Dese cuenta de que la jerarquía consiste en dos árboles distintos — un Map no es una verdadera Collection.

Dese cuenta de que todo el núcleo de interfaces de colección son genéricas. Por ejemplo, esta es la declaración de la interfaz Collection. 

public interface Collection<E>...

La sintaxis <E> le indica que la interfaz es genérica. Cuando declara una instancia de Collection usted puede y y debe especificar el tipo de objeto contenido en la colección. Especificar el tipo permite al compilador verificar (en tiempo de compilación) que el tipo de objeto que coloca en la colección es correcto, y así reducir errores en tiempo de ejecución. Para información sobre los tipos genéricos, vea la lección Genéricos (Actualizado).

Cuando entiende cómo usar estas interfaces, conocerá la mayoría de lo que hay que coneocer sobre el Marco de trabajo de Colecciones de Java. Este capítulo describe las guias generales para el uso efectivo de las interfaces, incluyendo cuando usar qué interfaz. Aprenderá los idiomas de programación para cada interfaz para ayudarle a aprovecharlo al máximo.

Para mantener el número de interfaces de colección en el núcleo manejable, la plataforma Jva no suministra interfaces separadas para cada variante de cada tipo de colección. (Tales variantes podrían ser inmutables, de tamaño fijo, y de-sólo-añadir). En vez de eso, las operaciones de modificación en cada interfaz son designadas optional — una implementación dada puede elegir no soportar todas las operaciones. Si una operación no soportada es invocada, una colección lanza una UnsupportedOperationException . Las implementaciones osn responsables de documentar cuales de las operaciones opcionales que soportan. Todas las implementaciones de propósito general de la plataforma Java soportan todas las operaciones opcionales.

La lista siguiente describe el núcleo de interfaces de colección:

  • Collection — la raíz de la jerarquía de colecciones. Una colección representa un grupo de objetos conocidas como sus elementos. La interfaz de Collection es es el mínimo común denominador que todas las colecciones implementan y se usa para pasar colecciones y manipularlas cuando se desea la máxima generalidad. Algunos tipos de colecciones permiten elementos duplicados, y otras no. Algunas son ordenadas y otras son desordenadas. La plataforma de Java no suministra ninguna implementación directa de esta interfaz pero suministra implementaciones de subinterfaces más específicas, tales como Set y List. Vea también la sección La Interfaz Collection.
  • Set — una colección que no puede contener elementos duplicados. Este interfaz modela la abstración matemática del conjunto (set en inglés) y es usada para representar conjuntos, tales como las cartas que componen una mano de poker, los cursos que componen el horario de un estudiante, o los procesos que se ejecutan en una máquina. Vea también la sección El Interfaz Set.
  • List — una colección ordenada (algunas vecs llamada una secuencia). Las Lists pueden contener elementos duplicados. El usuario de una List generalmente tienen un control preciso sobre en qué parte d ela lista se inserta cada elemento y puede acceder a los elementos por su índice entero (posición). Si ha usado Vector, está familiarizado con el sabor general deList. Vea también la sección La Interfaz List.
  • Queue — una colección usada para mantener múltiples elementos antes del procesamiento. Además de las operaciones básicas de Collection, un Queue suministra operaciones de inserciones, extracciones, e inspección adicionales.

    Las Queues típicamente, pero no necesariamente, ordena los elementos de una manera PEPS (primero en entrar, primero en salir, FIFO o first-in first-out en inglés). Entre las excepciones están las colas de prioridades, las cuales ordenan los elementos de acuerdo con un comparador suministrado o el orden natural de los elementos. Cualquiera que sea el orden usado, el encabezado de la cola es el elemento que sería eliminado por una llamada a remove o poll. En una cola FIFO, todos los elementos nuevos son insertados al final de la cola. Otras clases de colas pueden usar reglas diferentes de colocación. Cada implementación de Queue tiene que especificar sus propiedades de ordenación. También vea la sección El Interfaz Queue.

  • Deque — una colección usada para contener múltiples elementos antes de procesarlos. Más allá de las operaciones básicas de Collection, una Deque suministra operaciones adicionales de inserción, extracción, e inspección.

    Las deques pueden usarse tanto en colas PEPS (primero en entrar, primero en salir) y UEUS (última en entrar, última en salir). En una deque todos los elementos nuevos pueden ser insertados, recuperados y eliminados en ambos extremos. Vea también la sección El Interfaz Deque.

  • Map — un objeto que asigna claves a valores. Un Map no puede contener claves duplicadas; cada clave puede asignar como máximo un valor. Si ha usado Hashtable, ya está familiarizado con los fundamentos de Map. Vea también la seccion La Interfaz Map.

Los dos últimos interfaces del núcleo de colecciones son meramente versiones ordenadas de Set y Map:

  • SortedSet — Un Set que mantiene sus elementos en orden ascendente. Se ofrecen varias operaciones adicionales para aprovechar el ordenamiento. Los sets ordenados se usan para los conjuntos ordenados naturalmente, como listas de palabras y listas de miembros. Vea también la sección La Interfaz SortedSet.
  • SortedMap — un Map que mantiene sus mapeos en orden ascendente de la clave. Este es el Map análogo de SortedSet. Los mapas ordenados se usan para las colecciones naturalmente ordenadas de pares clave/valor, tales como diccionarios y directorios de teléfono. Vea también la sección La Interfaz SortedMap.

Para entender como las interfaces ordenadas mantienen el orden de sus elementos, vea la sección Orden de Objetos.
