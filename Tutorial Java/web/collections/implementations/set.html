Implementaciones de Set

Las implementaciones de Set se agrupan en implementaciones de propósito general y de propósito especial.

Implementaciones de Set de Propósito General

Hay tres implementaciones Set de propósito general — HashSet , TreeSet , y LinkedHashSet . Cuál de estos tres usar es generalmente directo. HashSet es mucho más rápido que TreeSet (tiempo constante versus tiempo de registro para la mayoría de las operaciones) pero no ofrece garantías de ordenamiento. Si necesita usar las operaciones en la interfaz SortedSet, o si la iterción ordenad por valor se requiere, use TreeSet; de otra forma, use HashSet. Es una buena apuesta que terminará usando HashSet la mayor parte del tiempo.

LinkedHashSet es en algún sentido lo intermedio entre HashSet y TreeSet. Implementada como una tabla hash con una lista enlazada recorriéndola, suministra iteración ordenada-de-inserción (menos recientemente insertado a más recientemente) y se ejecuta iteration (least recently inserted to most recently) y funciona casi tan rápido como HashSet. La implementación de LinkedHashSet ahorra a sus clientes del ordenamiento no especificado, generalmente caótico proporcionado por HashSet sin incurrir en el mayor costo asociado con TreeSet.

Una cosa que vale la pena tener en cuenta sobre HashSet es que la iteración es lineal en la suma de la cantidad de entradas y el número de almacenamientos (la capacidad). Por lo tanto, elegir una capacidad inicial que sea demasiado alta puede desperdiciar espacio y tiempo. Por el otro lado, elegir una capacidad inicial que sea demasiado baja gastará tiempo en copiar la estructura de datos cada vez que se vea forzado a incrementar su capacidad. si no especifica una capacidad inicial, el valor por defecto es 16. En el pasado, había alguna ventana en elegir un número primo como la capacidad inicial. Esto ya no es verdad. Internamente, la capacidad siempre se redondea a una potencia de dos. La capacidad inicial es especificada usando el constructor int. La siguiente línea de código asigna un HashSet cuya capacidad iniciales es 64. 

Set<String> s = new HashSet<String>(64);

La clase HashSet tiene otro parámetro de ajuste llamado el factor de carga. Si le preocupa mucho el consumo de espacio de su HashSet, lea la documentación de HashSet para más información. De otra forma, acepte el valor por defecto, casi siempre es lo correcto.

Si acepta el valor por defecto del factor de carga pero quiere especificar una capacidad inicial, elija un número que sea sobre dos veces el tamaño que espera que el set crezca. Si falla en su conjetura, puede derrochar un poco de espacio, tiempo, o ambos, pero es poco probable que sea un gran problema.

LinkedHashSet tiene los mismos parámetros de ajuste que HashSet, pero el tiempo de iteración no es afectado por la capacidad. TreeSet no tiene parámetros de ajustes.

Implementaciones de Set de Propósito Especial

Hay dos implementaciones Set de propósito especial — EnumSet y CopyOnWriteArraySet.

EnumSet es una implementación de alto rendimiento de Set para tipos enumerados. Todos los miembros de un set de enumeraciones deben ser del mismo tipo de enumeración. Internamente, es representado por un vector de bits, típicamente un único long. Los sets de Enumeraciones soportan la iteración sobre rangos de tipos de enumeración. Por ejemplo, dado la declaración de enumeración para los días de la semana, usted puede iterar por los días de la semana. La clase EnumSet suministrar una factoría estática que lo hace fácil. 

    for (Day d : EnumSet.range(Day.MONDAY, Day.FRIDAY))
        System.out.println(d);

Los sets de enumeraciones suministran un reemplazo rico y seguro con los tipos de las tradicionales banderas de bits. 

    EnumSet.of(Style.BOLD, Style.ITALIC)

CopyOnWriteArraySet es una implementación de Set soportada por una matriz de tipo copiar-al-escribir. Todas las operaciones mutativas, tales como add, set, y remove, son implementadas haciendo una nueva copia de la matriz; nunca se requiere bloqueo. Incluso la iteración puede proceder de forma segura al mismo tiempo que se insertan y eliminan los elementos. A diferencia de la mayoría de las implementaciones de Set, los métodos add, remove, y contains requieren tiempo proporcional al tamaño del set. Esta implementación es sólo apropiada para sets que se modifican raramente pero que son frecuentemente recorridos por bucles. Es adecuado para mantener listas de controladores de eventos que deben evitar duplicados.
