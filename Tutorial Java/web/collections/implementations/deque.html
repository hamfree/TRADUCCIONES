Implementaciones de Deque

El interfaz Deque, pronunciado como "deck", representa una cola de doble extremo. El interfaz Deque puede ser implementado como varios tipos de Collections. Las implementaciones del interfaz Deque están agrupadas en implementaciones de propósito general y concurrentes.

Implementaciones de Deque de Propósito General

Las implementaciones de propósito general incluyen las clases LinkedList y ArrayDeque. El interfaz Deque soporta la inserción, eliminación y recuperación de los elementos en ambos extremos. La clase ArrayDeque es la implementación con una matriz redimensionable de la interfaz Deque, mientras que la clase LinkedList es la implementación de la lista.

Las operaciones básicas de inserción, eliminación y recuperación en la interfaz Deque son addFirst, addLast, removeFirst, removeLast, getFirst y getLast. El método addFirst agrega un elemento en la cabeza mientras que addLast agrega un elemento al final de la instancia Deque.

La implementación LinkedList es más flexible que la implementación ArrayDeque. LinkedList implementa todas las operaciones de lista opcionales. Los elementos null están permitidos en la implementación de LinkedList pero no en la implementación ArrayDeque.

En términos de eficiencia, ArrayDeque es más eficiente que LinkedList para las operaciones agregar y eliminar en ambos extremos. La mejor operación en una implementación LinkedList es eliminar el elemento actual durante la iteración. Las implementaciones de LinkedList no son estructuras ideales para iterar.

La implementación LinkedList consume más memoria que la implementación ArrayDeque. Para el recorrido de la instancia de ArrayDeque utilice cualquiera de los siguientes:

foreach

El foreach es más rápido y puede ser usado para toda clase de listas. 


ArrayDeque<String> aDeque = new ArrayDeque<String>();

. . .
for (String str : aDeque) {
    System.out.println(str);
}

Iterator

El Iterator puede ser usado para el recorrido hacia adelante de todas las clases de listas para todas las clases de datos. 

ArrayDeque<String> aDeque = new ArrayDeque<String>();
. . .
for (Iterator<String> iter = aDeque.iterator(); iter.hasNext();  ) {
    System.out.println(iter.next());
}

La clase ArrayDeque es usada en este tutorial para implementar el interfaz Deque. El código completo de este ejemplo usado en este tutorial está disponible en ArrayDequeSample . Tanto la clase LinkedList como la clase ArrayDeque no soporta el acceso concurrente por múltiples hilos.

Implementaciones de Deque Concurrentes

La clase LinkedBlockingDeque es la implementación concurrente de la interfaz Deque. Si la deque está vacía entonces los métodos tales como takeFirst y takeLast esperan hasta que el elemento se vuelve disponible, y entonces recupera y elimina el mismo elemento.
