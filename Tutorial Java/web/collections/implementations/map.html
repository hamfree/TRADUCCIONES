Implementaciones de Map

Las implementaciones de Map están agrupadas en implementaciones de propósito general, de propósito especial y concurrentes.

Implementaciones de Map de Propósito General

Las tres implementaciones de propósito general de Map son HashMap , TreeMap y LinkedHashMap . Si necesita operaciones de SortedMap o iteración ordenada por clave de una vista de Collection, use TreeMap; si quiere velocidad máxima y no le importa el orden de iteración, use un HashMap; si quiere un rendimiento cercano al de HashMap y una iteración en orden de inserción, use LinkedHashMap. En este sentido, la situación para Map es análoga a Set. Igualmente, todo lo demás en la sección Implementaciones de Set también se aplica a las implementaciones de Map.

LinkedHashMap proporciona dos capacidades que no son disponibles con LinkedHashSet. Cuando crea un LinkedHashMap, puede ordenarlo basado en el acceso por clave en lugar de la inserción. En otras palabras, simplemente buscar el valor asociado con una clave lleva esa clave al final del mapa. Además, LinkedHashMap ofrece el método removeEldestEntry, el cual puede ser sobreescrito para imponer una política para eliminar mapeos obsoletos automáticamente cuando se agregan nuevas asignaciones al mapa. Esto hace muy fácil implementar una caché personalizada.

Por ejemplo, esta sobreescritura permitirá al mapa crecer hasta 100 entradas y entonces eliminará la entrada más antigua cada vez que una entrada nueva se agregue, manteniendo un estado constante de 100 entradas. 

private static final int MAX_ENTRIES = 100;

protected boolean removeEldestEntry(Map.Entry eldest) {
    return size() > MAX_ENTRIES;
}

Implementaciones de Map de Propósito Especial

Hay tres implementaciones de Map de propósito especial — EnumMap , WeakHashMap y IdentityHashMap . EnumMap, el cual está internamente implementado como una matriz, es una implementación de alto rendimieinto de Map para usar con claves enumeradas. Esta implementación combina la riqueza y seguridad del interfaz Map con una velocidad que se aproxima a la de una matriz. Si quiere asociar una enumeración a un valor, debería usar un EnumMap en vez de una matriz.

WeakHashMap es una implementación del interfaz Map que almacena sólo referencias débiles a sus claves. Almacenar sólo referencias débiles permite a un par clave-valor ser eliminado cuando su clave ya no es referenciada fuera del WeakHashMap. Esta clase proporciona la manera más fácil de aprovechar el poder de las referencias débiles. Esto es útil para implementar estructuras de datos "como registro", donde la utilidad de una entrada desaparece cuando su clave ya no es alcanzable por ningún hilo.

IdentityHashMap es una implementación de Map basada en identidad y fundada sobre una tabla hash. Esta clase es útil para las transformaciones de gráficos de objetos que preservan la topología, tales como la serialización o la copia profunda. Para realizar tales transformaciones, necesita mantener una "tabla de nodos" basada en identidad que siga la pista de qué objetos han sido ya vistos. Los mapas basados en identidad son también usados para mantener asociaciones objeto-a-meta-información en depuradores dinámicos y sistemas similares. Finalmente, los mapas basados en identidad son útiles para frustar los "ataques falsos" que son el resultado de métodos equals intencionalmente perversos porque IdentityHashMap nunca invoca el método equals sobre sus claves. Un beneficio añadido de esta implementación es que es rápido.

Implementaciones de Map Concurrentes

El paquete java.util.concurrent contiene el interfaz ConcurrentMap , el cual extiende a Map con los métodos atómicos putIfAbsent, remove, y replace, y la implementación ConcurrentHashMap de esa interfaz.

ConcurrentHashMap es una implementación altamente concurrente y de alto rendimiento respaldada por una tabla hash. Esta implementación nunca bloquea cuando realiza recuperaciones y permite al cliente seleccionar el nivel de concurrencia para las actualizaciones. Está pensado como un reemplazo directo para Hashtable: además de implementar ConcurrentMap, soporta todos los métodos heredados peculiares de Hashtable. De nuevo, si no necesita las operaciones heredadas, tenga cuidado al manipularla con el interfaz ConcurrentMap.
