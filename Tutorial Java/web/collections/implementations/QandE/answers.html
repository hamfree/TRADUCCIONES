Respuestas a Preguntas y Ejercicios:

Preguntas

  1. Pregunta: Usted planea escribir un programa que usa varias interfaces básicas de colección: Set, List, Queue, y Map. No está seguro de qué implementaciones trabajarán mejor, así que decide usar las implementaciones de uso general hasta que tenga una mejor idea de cómo su programa funcionará en el mundo real. ¿Qué implementaciones son estas? Respuesta:
    Set: HashSet
    List: ArrayList
    Queue: LinkedList
    Map: HashMap
  2. Pregunta: Si necesita una implementación Set que suministre una iteración ordenada-por-valor ¿qué clase usaría?
    Respuesta:
    TreeSet garantiza que el conjunto ordenado esté en orden ascendiente de elementos, ordenados de acuerdo al orden natural de los elementos o suministrado por el Comparator.
  3. Pregunta: ¿Qué clase usa para acceder a las implementaciones de envoltura?
    Respuesta:
    Usted usa la clase Collections, la cual suministra métodos estáticos que operan o devuelven colecciones.

Ejercicios

  1. Ejericicio: Escriba un programa que lea un fichero de texto, especificado por el primer argumento de la línea de comandos, en una List. El programa imprimirá después líneas aleatorias del fichero, el número de líneas impresas estará especificado por el segundo argumento de la línea de comandos. Escriba el programa de forma que se asigne una colección de tamaño correcto, de una vez, en vez de ser gradualmente expandida mientras se lee el fichero. Consejo: Para determinar el número de líneas en el fichero, use java.io.File.length para obtener el tamaño del fichero, y después divida por el tamaño supuesto de una línea promedio.
    Respuesta:
    Ya que estamos accediendo a la List aleatoriamente, usaremos una ArrayList. Estimamos el número de línea tomando el tamaño del fihcero y dividiendo por 50. Luego duplicamos esa cifra, ya que es más eficiente sobreestimar que subestimar. 
    
    import java.util.*;
    import java.io.*;

    public class FileList {
        public static void main(String[] args) {
            final int assumedLineLength = 50;
            File file = new File(args[0]);
            List<String> fileList =
                new ArrayList<String>((int)(file.length() / assumedLineLength) * 2);
            BufferedReader reader = null;
            int lineCount = 0;
            try {
                reader = new BufferedReader(new FileReader(file));
                for (String line = reader.readLine(); line != null;
                        line = reader.readLine()) {
                    fileList.add(line);
                    lineCount++;
                }
            } catch (IOException e) {
                System.err.format("Could not read %s: %s%n", file, e);
                System.exit(1);
            } finally {
                if (reader != null) {
                    try {
                        reader.close();
                    } catch (IOException e) {}
                }
            }
            int repeats = Integer.parseInt(args[1]);
            Random random = new Random();
            for (int i = 0; i < repeats; i++) {
                System.out.format("%d: %s%n", i,
                        fileList.get(random.nextInt(lineCount - 1)));
            }
        }
    }
    Este programa en realidad gasta la mayoría de su tiempo leyenedo en el fichero, así que preasignar el ArrayList tiene poco efecto en su rendimiento. Especificar una capacidad inicial por adelantado es más probable que sea útil cuando su programa repetidamente crea grandes objetos ArrayList sin intervención de E/S.
