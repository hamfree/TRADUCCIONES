Sumario de Implementaciones

Las implementaciones son los objetos de datos usados para almacenar colecciones, los cuales implementan las interfaces descritas en la lección de Interfaces.

El Marco de Trabajo de Colecciones de Java suministra varias implementaciones de propósito general de los interfaces principales:

  • Para el interfaz Set, HashSet es la implementación más comunmente utilizada.
  • Para el interfaz List, ArrayList es la implementación más comunmente utilizada.
  • Para el interfaz Map, HashMap es la implementación más comunmente utilizada.
  • Para el interfaz Queue, LinkedList es la implementación más comunmente utilizada.
  • Para el interfaz Deque, ArrayDeque es la implementación más comunmente utilizada.

Cada una de las implementaciones de propósito general suministra todas las operaciones opcionales contenidas en sus interfaces.

El Marco de Trabajo de Colecciones de Java también suministra varias implementaciones de propósito especial para situaciones que requieren un rendimiento no estándar, el uso de restricciones, u otro comportamiento inusual.

El paquete java.util.concurrent contiene varias implementaciones de colecciones, loas cuales son seguras en multihilo pero no se rigen por un sólo bloqueo de exclusión.

La clase Collections (a diferencia de la interfaz Collection), suministra métodos estáticos que operan sobre o devuelven colecciones, las cuales son conocidas como implementaciones Wrapper (o de Envoltura).

Finalmente, hay varias implementaciones de Convenencia, las cuales pueden ser más eficientes que las implementaciones de propósito general cuando no necesita su poder completo. Las implementaciones de Convenencia se hacen disponibles a través de métodos de factoría estáticos.
