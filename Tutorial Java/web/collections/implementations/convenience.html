Implementaciones de Convenencia

Esta sección describe varias mini-implementaciones que pueden ser más convenientes y más eficientes que las implemetanciones de propósito general cuando no necesita todo su poder. Todas las implementaciones en esta sección están disponibles vía métodos estáticos de factoría en vez de clases public.

Vista de Lista de una Matriz

El método Arrays.asList devuelve una vista de List de su argumento matriz. Los cambios a la List se escriben en la matriz y viceversa. El tamaño de la colección es el de la matriz y no puede ser cambiado. Si se llama a los métodos add o remove en la List, se producirá una UnsupportedOperationException.

El uso normal de esta implementación es como un puente entre las IPAs basadas en colecciones y las basadas en matrices. Le permite pasar una matriz a un método esperando una Collection o una List. Sin embargo, esta implementación tiene también otro uso. Si necesita una List de tamaño fijo, es más eficiente que cualquier implementación de List de propósito general. Esta es la expresión idiomática. 

List<String> list = Arrays.asList(new String[size]);

Tenga en cuenta que no se conserva una referencia a la matriz de respaldo.

Lista de copias múltiples inmutables

Ocasionalmente necesitará una List inmutable que consista de copias múltiples del mismo elemento. El método Collections.nCopies devuelve tal lista. Esta implementación tiene dos usos principales. El primero es inicializar una List recién creada; por ejemplo, suponga que quiere un ArrayList que inicialmente consiste de 1000 elementos null. El siguiente encantamiento realiza el truco. 

List<Type> list = new ArrayList<Type>(Collections.nCopies(1000, (Type)null);

Por supuesto, el valor inicial de cada elemento no necesita ser null. El segundo principal uso es hacer crecer una List existente. Por ejemplo, suponga que quiere añadir 69 copias de la cadena "murciélago de la fruta" al final de una List<String>. No está claro por qué querría hacer tal cosa, pero supongamos que lo hizo. Lo siguiente es cómo lo haría. 

    mascotasAdorables.addAll(Collections.nCopies(69, "murciélago de la fruta"));

Al usar la forma de addAll que toma tanto un índice como una Collection, puede añadir los nuevos elementos en la mitada de una List en vez de al final de ella.

Set Inmutable y Singleton

Algunas veces necesitará un Set inmutable y singleton, el cual consite de un elemento único y especificado. El método Collections.singleton devuelve tal Set. Un uso de esta implementación es eliminar todas ls ocurrencias de un elemento especificado de una Collection. 

c.removeAll(Collections.singleton(e));

Un modismo relacionado elimina todos los elementos que se asignan a un valor especificado de un Map. Por ejemplo, suponga que tiene un Map — job — que asigna personas a su línea de trabajo y supong que quiere eliminar todos los abogados. La siguiente sentencia realizará este trabajo. 

job.values().removeAll(Collections.singleton(LAWYER));

Un uso más de esta implementación es suministrar un valor de entrada único a un método que está escrito para aceptar una colección de valores.

El Set y la List vacíos, y las Constantes de Map

La clase Collections suministra métodos para devolver el Set, List, y el Map vacíos — emptySet, emptyList, y emptyMap. El uso principal de estas constantes es como entrada a métodos que toman una Collection de valores cuando usted no quiere suministrar ningún valor en absoluto, como en este ejemplo. 

tourist.declarePurchases(Collections.emptySet());
