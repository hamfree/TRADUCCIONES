Preguntas y Ejercicios: Operaciones de Agregación

Preguntas

  1. Una secuencia de operaciones agregadas es conocida como un ___ .
  2. Cada tubería contiene cero o más ___ operaciones.
  3. Cada tubería finaliza con una operación ___ .
  4. ¿Qué clase de operación produce otro flujo como su salida?
  5. Describa una forma en la cual la operación agregada forEach difiere de la sentencia mejorada for o los iteradores.
  6. Verdadero o Falso: Un flujo es similar a una colección en que es una estructura de datos que almacena elementos.
  7. Identifique las operaciones intermedias y finales en este código: 
    double average = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .mapToInt(Person::getAge)
    .average()
    .getAsDouble();
  8. ¿El código p -> p.getGender() == Person.Sex.MALE es un ejemplo de qué?
  9. ¿El código Person::getAge es un ejemplo de qué?
  10. ¿Las operaciones terminales que combinan los contenidos de un flujo y devuelven un valor son conocidos como?
  11. Indique una diferencia importante entre el método Stream.reduce y el método Stream.collect.
  12. Si quisiera procesar un flujo de nombres, extraer los nombres masculinos, y almacenarlos en una nueva List, ¿Sería Stream.reduce o Stream.collect la operación más apropiada para usar?
  13. Verdadero o Falso: Las operaciones de agregación hacen posible implementar el paralelismo con colecciones no seguras para multihilo.
  14. Los flujos son siempre en serie a menos que se especifique lo contrario. ¿Como solicita que un flujo sea procesado en paralelo?

Ejercicios

  1. Escriba la siguiente sentencia mejorada for como una tubería con expresiones lambda. Consejo: Use la operación intermedia filter y la operación terminal forEach. 
    for (Person p : roster) {
    if (p.getGender() == Person.Sex.MALE) {
        System.out.println(p.getName());
    }
}
  2. Convierta el código siguiente en una implementación nueva uqe use expresiones lambda y operaciones agregadas en vez de bucles for anidados. Consejo: Haga una tubería que invoque las operaciones filter, sorted, y collect, en ese orden. 
    List<Album> favs = new ArrayList<>();
for (Album a : albums) {
    boolean hasFavorite = false;
    for (Track t : a.tracks) {
        if (t.rating >= 4) {
            hasFavorite = true;
            break;
        }
    }
    if (hasFavorite)
        favs.add(a);
}
Collections.sort(favs, new Comparator<Album>() {
                           public int compare(Album a1, Album a2) {
                               return a1.name.compareTo(a2.name);
                           }});
