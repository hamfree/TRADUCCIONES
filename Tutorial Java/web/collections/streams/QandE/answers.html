Respuestas a Preguntas y Ejercicios: Operaciones de Agregación

Preguntas

  1. P: Una secuencia de operaciones agregadas es conocida como una ___ .
    R: Tubería

  2. P: Cada tubería contiene cero o más operaciones ___ .
    R: Intermedias

  3. P: Cada tubería finaliza con una operación ___ .
    R: Terminal

  4. P: ¿Qué clase de operación produce otro flujo como su salida?
    R: Una operación intermedia

  5. P: Describa una forma en la cual la operación agregada forEach difiere de la sentencia mejorada for o los iteradores.
    R: La operación agregada forEach le permite al sistema decidir "cómo" la iteración toma lugar. Usar operaciones agregadas le permite enfocarse en el "qué" en vez de en el "cómo"

  6. P: Verdadero o Falso: Un flujo es similar a una colección en que es una estructura de datos que almacena elementos.
    R: Falso. A diferencia de una colección, un flujo no es una estructura de datos. En su lugar, transporta los valores de una fuente de datos a través de una tubería.

  7. P: Identifique las operaciones intermedias y finales en este código: 
    double average = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .mapToInt(Person::getAge)
    .average()
    .getAsDouble();

    R: Intermedias: filter, mapToInt
    Terminales: average
    La operación terminal average devuelve un OptionalDouble. El método getAsDouble es entonces invocado sobre ese objeto devuelto. Es siempre una buena idea consultar la Especificación de la IPA para obtener información acerca de si una operación es intermedia o terminal.

  8. P: ¿El código p -> p.getGender() == Person.Sex.MALE es un ejemplo de qué?
    R: Una expresión lambda.

  9. P: ¿El código Person::getAge es un ejemplo de qué?
    R: Una referencia de método.

  10. P: ¿Las operaciones terminales que combinan los contenidos de un flujo y devuelven un valor son conocidos como?
    R: Operaciones de reducción.

  11. P: Indique una diferencia importante entre el método Stream.reduce y el método Stream.collect.
    R: Stream.reduce siempre crea un valor nuevo cuando procesa un elemento. Stream.collect modifica (o muta) el valor existente.

  12. P: Si quisiera procesar un flujo de nombres, extraer los nombres masculinos, y almacenarlos en una nueva List, ¿Sería Stream.reduce o Stream.collect la operación más apropiada para usar?
    R: La operación collect es más apropiada para recoger en un List.

    Ejemplo: 
    List<String> namesOfMaleMembersCollect = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .map(p -> p.getName())
    .collect(Collectors.toList());


  13. P: Verdadero o Falso: Las operaciones de agregación hacen posible implementar el paralelismo con colecciones no seguras para multihilo.
    R: Verdadero, siempre que no modifique (mute) la colección subyacente mientras está operando en ella.

  14. P: Los flujos son siempre en serie a menos que se especifique lo contrario. ¿Como solicita que un flujo sea procesado en paralelo?
    R: Obtenga el flujo paralelo invocando parallelStream() en vez de stream().

Ejercicios

  1. Escriba la siguiente sentencia mejorada for como una tubería con expresiones lambda. Consejo: Use la operación intermedia filter y la operación terminal forEach.
    for (Person p : roster) {
    if (p.getGender() == Person.Sex.MALE) {
        System.out.println(p.getName());
    }
}

    Respuesta: 
    roster
    .stream()
    .filter(e -> e.getGender() == Person.Sex.MALE)
    .forEach(e -> System.out.println(e.getName());
  2. Convierta el código siguiente en una implementación nueva uqe use expresiones lambda y operaciones agregadas en vez de bucles for anidados. Consejo: Haga una tubería que invoque las operaciones filter, sorted, y collect, en ese orden. 
    List<Album> favs = new ArrayList<>();
for (Album a : albums) {
    boolean hasFavorite = false;
    for (Track t : a.tracks) {
        if (t.rating >= 4) {
            hasFavorite = true;
            break;
        }
    }
    if (hasFavorite)
        favs.add(a);
}
Collections.sort(favs, new Comparator<Album>() {
                           public int compare(Album a1, Album a2) {
                               return a1.name.compareTo(a2.name);
                           }});

    Respuesta: 
    List<Album> sortedFavs =
  albums.stream()
        .filter(a -> a.tracks.anyMatch(t -> (t.rating >= 4)))
        .sorted(Comparator.comparing(a -> a.name))
        .collect(Collectors.toList());

    Aquí hemos usado las operaciones de flujo para simplificar cada uno de los tres pasos principales -- Here we have used the stream operations to simplify each of the three major steps -- identificación de si una canción en un álbum tiene una calificación de al menos 4(anyMatch), el ordenado, y la colección de álbumes que coinciden con nuestros criterios en una List. El método Comparator.comparing() toma una función que extrae un clave de ordenación Comparable, y devuelve un Comparator que compara con esa clave.
