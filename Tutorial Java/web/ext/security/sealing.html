Sellando Paquetes en Extensiones

Puede opcionalmente sellar paquetes en los ficheros JAR de extensión como una medida adicional de seguridad. Si un paquete se sella, todas las clases definidas en ese paquete deben estar en un fichero JAR único.

Sin el sellado, un programa "hostil" podría crear una clase y definirla como miembro de uno de sus paquetes de extensión. El software hostil tendría entonces acceso libre a los miembros protegidos del paquete de su paquete de extensión.

El sellado de paquetes en extensiones no es diferente del sellado de cualquier clase empaquetada en JAR. Para sellar su paquetes de extensión, debe agregar la cabecera Sealed al manifiesto del fichero JAR que contiene tu extensión. Puede sellar paquetes individuales asociando una cabecera Sealed con las cabeceras Name del paquete. Un encabezado Sealed no asociado con un paquete individual en el archivo indica que todos los paquetes están sellados. Un encabezado Sealed "global" es anulado por cualquier cabecera Sealed asociada con paquetes individuales. El valor asociado con la cabecera Sealed es true o false.

Ejemplos

Veamos unos pocos ejemplos de ficheros de manifiesto. Para estos ejemplos suponga que el fichero JAR contiene estos paquetes: 

com/myCompany/package_1/
com/myCompany/package_2/
com/myCompany/package_3/
com/myCompany/package_4/

Suponga que quiere sellar todos los paquetes. Podría hacer eso simplemente agregando una cabecera Sealed a nivel de archivo al manifiesto de esta forma: 

Manifest-Version: 1.0
Sealed: true

Todos los paquetes en cualquier fichero JAR que tenga este manifiesto estará sellado.

Si quería sellar sólo com.myCompany.package_3, podría hacerlo con este manifiesto: 

Manifest-Version: 1.0

Name: com/myCompany/package_3/
Sealed: true

En este ejemplo la única cabecera Sealed es aquella asocida con la cabecera Name del paquete com.myCompany.package_3, así que sólo ese paquete es sellado. (La cabecera Sealed se asocia con la cabecera Name porque no hay líneas en blanco entre ellas).

Para un ejemplo final, suponga que quería sellar todos los paquetes excepto el com.myCompany.package_2. Podría logras esto con un manifiesto como este: 

Manifest-Version: 1.0
Sealed: true

Name: com/myCompany/package_2/
Sealed: false

En este ejemplo la cabecera a nivel de archivo Sealed: true indica que todos los paquetes en el fichero JAR van a ser sellados. Sin embargo, el manifiesto también tiene una cabecera Sealed: false asociada con el paquete com.myCompany.package_2, y esa cabecera sobreescribe el sellado a nivel de archivo para ese paquete. Por lo tanto este manifiesto causará que todos los paquetes serán sellados excepto com.myCompany.package_2.
