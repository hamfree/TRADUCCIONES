Lección: Haciendo Extensiones Seguras

Ahora que ha visto cómo usar extensiones, puede estar preguntándose que privilegios de seguridad tienen las extensiones. Si está desarrollando una extensión que hace E/S de ficheros, por ejemplo, necesitará saber cómo se le otorgan a su extensión los permisos apropiados para leer y escribir ficheros. A la inversa, si está pensando sobre usar una extensión desarrollada por alguien más, querrá comprender claramente qué privilegios de seguridad tiene la extensión y cómo cambiar esos privilegios si así lo desea.

Esta lección le muestra cómo la arquitectura de seguridad de la plataforma de Java™ trata las extensiones. Verá como decir qué privilegios son otorgados al software de extensión, y aprenderá cómo modificar los privilegios de la extensión siguiendo algunos pasos simples. Además, aprenderá a sellar paquetes dentro de sus extensiones para restringir el acceso a partes específicas de su código.

This lesson has two sections:

Fijando Privilegios para Extensiones

Esta sección contiene algunos ejemplos que le muestran qué condiciones deben darse para que a las extensiones se les otorgue permisos para realizar operaciones sensibles a la seguridad.

Sellando Paquetes en Extensiones

Puede opcionalmente sellar paquetes en los ficheros JAR de extensión como una medida adicional de seguridad. Si un paquete está sellado, significa que todas las clases definidias en ese paquete deben originarse en un sólo archivo JAR. Esta sección le muestra cómo modificar el manifiesto de una extensión para sellar paquetes de extensión.
