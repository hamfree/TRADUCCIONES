Recorrido: El Mecanismo de Extensión

El mecanismo de extensión proporciona una forma estándar y escalable de hacer IPAs personalizadas disponibles a todas las apliaciones que se ejecutan en la plataforma Java. Las extensiones Java son también referidas como paquetes opcionales. Este recorrido puede usar ambos términos indistintamente.

Las extensiones son grupos de paquetes y clases que aumentan la plataforma Java a través del mecanismo de extensión. El mecanismo de extensión permite que el entorno de ejecución encontrar y cargar clases de extensión sin que las clases de extensión tengan que se nombradas en la ruta de clases. En ese sentido, las clases de extensión son similares a las clases principales de la plataforma Java. De ahí es también de donde las extensiones toman su nombre -- ellas, en efecto, extienden la IPA principal de la plataforma.

Ya que este mecanismo extiende la IPA principal de la plataforma, su uso debe ser aplicado con prudencia. Por lo general, se utiliza para interfaces bien estandarizadas, como las definidas por el Proceso Java de la Comunidad, aunque también puede ser apropiado para interfaces de todo el sitio.

Como el diagrama indica, las extensiones actúan como módulos "agregables" a la plataforma Java. Sus clases e IPAs públicas quedan automáticamente disponibles para las aplicaciones que se ejecutan en la plataforma.

El mecanismo de extensión también proporciona un medio para que las clases de extensión se descarguen desde ubicaciones remotas para que las utilizen los applets.

Las extensiones se empaquetan como ficheros de Archivos Java (JAR), y este recorrido asume que está familiarizado con el formato de ficheros JAR. Si no está al día con las ficheros JAR, es posible que desee revisar parte de la documentación de los archivos JAR antes de continuar con las lecciones de este recorrido:

  • La lección Empaquetando Programas en Ficheros JAR en este tutorial.
  • La Guía de JAR en la documentación del KDJ™.
