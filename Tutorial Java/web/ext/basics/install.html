Extensiones Instaladas

Las extensiones instaladas son ficheros JAR en el directorio lib/ext del software del Entorno de Ejecución de Java (EEJ™). Como su nombre implica, el EEJ es la porción de sistema de ejecución del Kit de Desarrollo de Java que contiene el núcleo de las IPA de la plataforma pero sin las herramientas de desarrollo tales como compiladores y depuradores. El EEJ está disponible por si mismo o como parte del Kit de Desarrollo de Java.

El EEJ es un subconjunto estricto del software del KDJ. Un subconjunto del árbol de directorios del software del KDJ se ve así:

El EEJ consiste en esos directorios dentro del cuadro resaltado en el diagrama. Ya sea que su EEJ sea independiente o sea parte del software KDJ, cualquier archivo JAR en el directorio lib/ext del EEJ es tratado automáticamente por el entorno de ejecución como una extensión.

Dado que las extensiones instaladas amplían la IPA principal de la plataforma, úselas con prudencia. Rara vez son apropiadas para interfaces utilizadas por una única aplicación o un pequeño conjunto de aplicaciones.

Es más, dado que los símbolos definidos por las extensiones instaladas serán visibles en todos los procesos de Java, se debe tener cuidado para asegurar que todos los símbolos visibles siguen las convenciones del apropiado "nombre de dominio invertido" y jerarquía de clases. Por ejemplo, com.mycompany.MyClass.

A partir de Java 6, los archivos JAR de extensión también se pueden colocar en una ubicación que es independiente de cualquier EEJ en particular, así que las extensiones pueden ser compartidas por todos los EEJs instalados en el sistema. Antes de Java 6, el valor de java.ext.dirs se refería a un único directorio, pero a partir de Java 6 es una lista de directorios (como CLASSPATH) que especifica las ubicaciones en las cuales se buscan las extensiones. El primer elemento de ruta es siempre el directorio lib/ext del EEJ. El segundo elemento es un directorio fuera del EEJ. Esta otra ubicación permite que los ficheros JAR de extensión sean instalados una vez y usados por varios EEJs instalados en ese sistema. La ubicación varía dependiendo del sistema operativo:

  • Sistema Operativo Solaris™: /usr/jdk/packages/lib/ext
  • Linux: /usr/java/packages/lib/ext
  • Microsoft Windows: %SystemRoot%\Sun\Java\lib\ext

Dese cuenta de que una extensión instalada colocada en alguno de los directorios de arriba extienden la plataforma de cada EEJ (Java 6 o posterior) en ese sistema.

Un Ejemplo Simple

Creemos una extensión instalada simple. Nuestra extensión consiste de una clase, RectangleArea, que calcula el área de los rectángulos: 

public final class RectangleArea {
    public static int area(java.awt.Rectangle r) {
        return r.width * r.height;
    }
}

Esta clase tiene un único método, area, que toma una instancia de java.awt.Rectangle y devuelve el área del rectángulo.

Suponga que quiere comprobar RectangleArea con una aplicación llamada AreaApp: 

import java.awt.*;

public class AreaApp {
    public static void main(String[] args) {
        int width = 10;
        int height = 5;

        Rectangle r = new Rectangle(width, height);
        System.out.println("The rectangle's area is " 
                           + RectangleArea.area(r));
    }
}

Esta aplicación instancia un rectangulo de 10 x 5, y entonces imprime el área del rectángulo usando el método RectangleArea.area.

Ejecutar AreaApp Sin el Mecanismo de Extensión

Primero revisemos como ejecutaríamos la aplicación AreaApp sin usar el mecanismo de extensión. Asumiremos que la clase RectangleArea está empaqueta en un fichero JAR de nombre area.jar.

La clase RectangleArea no es parte de la plataforma Java, por supuesto, así que necesitaría colocar el fichero area.jar en la ruta de clases para poder ejecutar AreaApp sin obtener una excepción de tiempo de ejecución. Si area.jar estaba en el directorio /home/user, por ejemplo, podría usar este comando: 

java -classpath .:/home/user/area.jar AreaApp

La ruta de clases especificada en este comando contiene tanto el directorio actual, que contiene a AreaApp.class, y la ruta al fichero JAR que contiene el paquete RectangleArea. Obtendría la salida deseada al ejecutar este comando: 

The rectangle's area is 50

Ejecutar AreaApp Usando el Mecanismo de Extensión

Ahora veamos como ejecutaría AreaApp usando la clase RectangleArea como una extensión.

Para convertir la clase RectangleArea en una extensión, coloque el fichero area.jar en el directorio lib/ext del EEJ. Hacer eso automáticamente le da a RectangleArea el estado de ser una extensión instalada.

Con area.jar instalada como una extnesión, pùede ejecutar AreaApp sin necesidad de especificar la ruta de clases: 

java AreaApp

Ya que está usando area.jar como una extensión instalada, el entorno de ejecución será capaza de encontrar y cargar la clase RectangleArea incluso aunque no la haya especificado en la ruta de clases. De forma similar, cualquier applet o aplicación que se ejecute por cualquier usuario en su sistema sería capaz de encontrar y usar la clase RectangleArea.

Si hay EEJs múltiples (Java 6 o posterior) instalados en un sistema y quiere que la clase RectangleArea esté disponible como una extensión para todas ellas, en vez de instalarla en el directorio lib/ext de una EEJ particular, instálelea en una ubicación de todo el sistema. Por ejemplo, en un sistema que ejecuta Linux, instale area.jar en el directorio /usr/java/packages/lib/ext. Entonces AreaApp puede ejecutarse usando diferentes EEJs que están instalados en ese sistema, por ejeplo si diferentes navegadores están configurados para usar diferentes EEJs.
