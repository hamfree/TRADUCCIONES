Utilizando Objetos SQLXML

La interfaz Connection proporciona soporte para la creación de objetos SQLXML usando el método createSQLXML. El objeto que se crea no contiene ningún dato. Los datos pueden ser agregados al objeto llamando al método setString, setBinaryStream, setCharacterStream o setResult de la interfaz SQLXML.

Los siguientes temas están cubiertos:

  • Crear Objetos SQLXML
  • Recuperar Valores SQLXML en ResultSet
  • Acceder a los Datos de un Objeto SQLXML
  • Almacenar Objetos SQLXML
  • Inicializar Objetos SQLXML
  • Liberar Recursos SQLXML
  • Código de Ejemplo

Crear Objetos SQLXML

En el extracto siguiente, se usa el método Connection.createSQLXML para crear un objeto SQLXML. Se usa el método SQLXML.setString para escribir datos al objeto SQLXML que fue creado. 

Connection con = DriverManager.getConnection(url, props);
SQLXML xmlVal = con.createSQLXML();
xmlVal.setString(val);

Recuperar Valores SQLXML en ResultSet

El tipo de datos SQLXML es tratado de forma similiar a los tipos integrados más primitivos. Un valor SQLXML puede ser recuperado llamando al método getSQLXML en la interfaz ResultSet o CallableStatement.

Por ejemplo, el extracto siguiente recupera un valor SQLXML desde la primera columna del ResultSet rs: 

SQLXML xmlVar = rs.getSQLXML(1);

Los objetos SQLXML permanecen validos por al menos la duración de la transacción en la cual fueron creados, a menos que se invoquen sus métodos free.

Acceder a los Datos de un Objeto SQLXML

La interfaz SQLXML proporciona los métodos getString, getBinaryStream, getCharacterStream, y getSource para acceder a su contenido interno. El siguiente extracto recuperar los contenidos de un objeto SQLXML usando el método getString: 

SQLXML xmlVal= rs.getSQLXML(1);
String val = xmlVal.getString();

Se pueden usar los métodos getBinaryStream o getCharacterStream para obtener un objeto InputStream o a Reader que puede ser pasado directamente a un analizador XML. El extracto siguiente obtiene un objeto InputStream desde un objeto SQLXML y después procesa el flujo usando un analizador DOM (Modelo de Objeto de Documento o Document Object Model, en inglés): 

SQLXML sqlxml = rs.getSQLXML(column);
InputStream binaryStream = sqlxml.getBinaryStream();
DocumentBuilder parser = 
    DocumentBuilderFactory.newInstance().newDocumentBuilder();
Document result = parser.parse(binaryStream);

El método getSource devuelve un objeto javax.xml.transform.Source. Los fuentes se usan como entrada a los analizadores XML y los transformadores XSLT.

El siguiente extracto recupera y analiza los datos de un objeto SQLXML usando el objeto SAXSource devuelto al invocar el método getSource: 

SQLXML xmlVal= rs.getSQLXML(1);
SAXSource saxSource = sqlxml.getSource(SAXSource.class);
XMLReader xmlReader = saxSource.getXMLReader();
xmlReader.setContentHandler(myHandler);
xmlReader.parse(saxSource.getInputSource());

Almacenar Objetos SQLXML

Se puede pasar un objeto SQLXML como un parámetro de entrada a un objeto PreparedStatement al igual que otros tipos de datos. El método setSQLXML establece el parámetro designado PreparedStatement con un objeto SQLXML.

En el extracto siguiente, authorData es una instancia de la interfaz java.sql.SQLXML cuyos datos fueron inicializados previamente. 

PreparedStatement pstmt = conn.prepareStatement("INSERT INTO bio " +
                              "(xmlData, authId) VALUES (?, ?)");
pstmt.setSQLXML(1, authorData);
pstmt.setInt(2, authorId);

Se puede usar el método updateSQLXML para actualizar el valor de una columna en un conjunto de resultados actualizable.

Si el objeto java.xml.transform.Result, Writer, o OutputStream para el objeto SQLXML no ha sido cerrado antes de llamar a setSQLXML o updateSQLXML, se lanzará una SQLException.

Inicializar Objetos SQLXML

La interfaz SQLXML proporciona los métodos setString, setBinaryStream, setCharacterStream, o setResult para inicializar el contenidos de un objeto SQLXML que ha sido creado llamando al método Connection.createSQLXML.

El siguiente extracto usa el método setResult para devolver un objeto SAXResult y llenar un objeto SQLXML nuevamente creado: 

SQLXML sqlxml = con.createSQLXML();
SAXResult saxResult = sqlxml.setResult(SAXResult.class);
ContentHandler contentHandler = saxResult.getXMLReader().getContentHandler();
contentHandler.startDocument();
    
// set the XML elements and
// attributes into the result
contentHandler.endDocument();

El siguiente extracto usa el método setCharacterStream para obtener un objeto java.io.Writer con el fin de inicializar un objeto SQLXML: 

SQLXML sqlxml = con.createSQLXML();
Writer out= sqlxml.setCharacterStream();
BufferedReader in = new BufferedReader(new FileReader("xml/foo.xml"));
String line = null;
while((line = in.readLine() != null) {
    out.write(line);
}

Del mismo modo, el método setString de SQLXML puede usarse para inicializar un objeto SQLXML.

Si se intenta llamar a los métodos setString, setBinaryStream, setCharacterStream y setResult en un objeto SQLXML que se ha inicializado previamente, se lanzará una SQLException. Si se produce más de una llamada a los métodos setBinaryStream, setCharacterStream y setResult para el mismo objeto SQLXML, se lanza una SQLException y el objeto javax.xml.transform.Result, Writer o OutputStream devuelto anteriormente no es afectado.

Liberar Recursos SQLXML

Los objetos SQLXML permanecen válidos por al menos la duración de la transacción en la cual se han creado. Esto podría provocar que una aplicación se quede sin recursos durante una transacción de larga ejecución. La aplicación puede liberar recursos SQLXML invocando su método free.

En el extracto siguiente, se llama al método SQLXML.free para liberar los recursos retenidos por un objeto SQLXML creado previamente. 

SQLXML xmlVar = con.createSQLXML();
xmlVar.setString(val);
xmlVar.free();

Código de Ejemplo

MySQL y Java DB y sus respectivos controladores JDBC no soportan completamente el tipo de datos JDBC SQLXML como se describe en esta sección. Sin embargo, el ejemplo RSSFeedsTable demuestra cómo manejar los datos XML con MySQL y Java Db

El propietario de The Coffee Break sigue varias fuentes RSS de varios sitios web que cubren noticias de la industria de restaurantes y bebidas. Una fuente RSS (Nota del Traductor: Distribución realmente simple o resumen de sitio enriquecido o Really Simple Syndication or Rich Site Summary, en inglés) es un documento XML que contiene una serie de artículos y metadatos asociados, tales como la fecha de la publicación y autor para cada artículo. Al propietario le gustaría almacenar estas fuentes RSS en una tabla de la base de datos, incluyendo la fuente RSS de la bitácora de The Coffe Break.

El fichero rss-the-coffee-break-blog.xml es una fuente RSS de ejemplo de la bitácora de The Coffee Break.

Trabajar con Datos XML en MySQL

El ejemplo RSSFeedsTable almacena fuentes RSS en la tabla RSS_FEEDS, la cual se crea con el comando siguiente: 

create table RSS_FEEDS
    (RSS_NAME varchar(32) NOT NULL,
    RSS_FEED_XML longtext NOT NULL,
    PRIMARY KEY (RSS_NAME));

MySQL no soporta el tipo de dato XML. En cambio, este ejemplo almacena los datos XML en una columna de tipo LONGTEXT, que es del tipo de dato SQL CLOB. MySQL tiene cuatro tipos de datos CLOB; el tipo de datos LONGTEXT contiene la mayor cantidad de caracteres entre los cuatro.

El método RSSFeedsTable.addRSSFeed agrega una fuente RSS a la tabla RSS_FEEDS. La primera sentencia de este método convierte la fuente RSS (la cual está representada por un fichero XML en este ejemplo) en un objeto de tipo org.w3c.dom.Document, la cual representa un documento DOM (Modelo de Objeto de Documentos o Document Object Model, en inglés). Esta clase, junto con las clases e interfacs contenidas en el paquete javax.xml, contiene métodos que le permiten manipular el contenido de datos XML. Por ejemplo, la sentencia siguiente usa una expresión XPath para recuperar el título de la fuente RSS desde el objeto Document: 

Node titleElement =
    (Node)xPath.evaluate("/rss/channel/title[1]",
        doc, XPathConstants.NODE);

La expresión XPath /rss/channel/title[1] recupera los contenidos del primer elemento <title>. Para el fichero rss-the-coffee-break-blog.xml, este es la cadena The Coffee Break Blog.

Las siguientes sentencias agregan la fuente RSS a la tabla RSS_FEEDS: 

// For databases that support the SQLXML
// data type, this creates a
// SQLXML object from
// org.w3c.dom.Document.

System.out.println("Adding XML file " + fileName);
String insertRowQuery =
    "insert into RSS_FEEDS " +
    "(RSS_NAME, RSS_FEED_XML) values " +
    "(?, ?)";
insertRow = con.prepareStatement(insertRowQuery);
insertRow.setString(1, titleString);

System.out.println("Creating SQLXML object with MySQL");
rssData = con.createSQLXML();
System.out.println("Creating DOMResult object");
DOMResult dom = (DOMResult)rssData.setResult(DOMResult.class);
dom.setNode(doc);

insertRow.setSQLXML(2, rssData);
System.out.println("Running executeUpdate()");
insertRow.executeUpdate();

El método RSSFeedsTable.viewTable recupera los contenidos de RSS_FEEDS. Para cada fila, el método crea un objeto de tipo org.w3c.dom.Document llamado doc en el cual almacena el contenido de XML en la columna RSS_FEED_XML. El método recupera el contenido XML y lo almacena en un objeto de tipo SQLXML llamado rssFeedXML. Los contenidos de rssFeedXML son analizados y almacenados en el objeto doc.

Trabajar con Datos XML en Java DB

Nota: Vea la sección "Tipos de datos XML y operadores" en la Guía del Desarrollador de Java DB para más información sobre el trabajo con los datos XML en Java DB.

El ejemplo RSSFeedsTable almacena fuentes RSS en la tabla RSS_FEEDS, la cual se crea con el siguiente comando: 

create table RSS_FEEDS
    (RSS_NAME varchar(32) NOT NULL,
    RSS_FEED_XML xml NOT NULL,
    PRIMARY KEY (RSS_NAME));

Java DB soporta el tipo de datos XML, pero no soporta el tipo de datos JDBC SQLXML. Por consiguiente, debe convertir cualquier dato XML a un formato de caracteres, y entonces usar el operador de Java DB XMLPARSE para convertirlo al tipo de datos XML.

El método RSSFeedsTable.addRSSFeed agrega una fuente RSS a la tabla RSS_FEEDS. La primera sentencia de este método convierte la funete RSS (que está representada por un fichero XML en este ejemplo) en un objeto de tipo org.w3c.dom.Document. Esto se describe en la sección Trabajar con Datos XML en MySQL.

El método RSSFeedsTable.addRSSFeed convierte la fuente RSS en un objeto String con el método JDBCTutorialUtilities.convertDocumentToString.

Java DB tiene un operador llamado XMLPARSE que analiza una representación de cadena de caracteres en un valor XML de Java DB, que se demuestra en el siguiente extracto: 

String insertRowQuery =
    "insert into RSS_FEEDS " +
    "(RSS_NAME, RSS_FEED_XML) values " +
    "(?, xmlparse(document cast " +
    "(? as clob) preserve whitespace))";

El operador XMLPARSE requiere que convierta la representación de caracteres del documento XMl en un tipo de datos de cadena que Java DB reconozca. En este ejemplo, lo convierte en un tipo de datos CLOB. Vea Empezando y la documentación de Java DB para más información sobre los requisitos de Apache Xalan y Java DB.

El método RSSFeedsTable.viewTable recupera los contenidos de RSS_FEEDS. Debido a que Java DB no soporta el tipo de datos JDBC SQLXML debe recuperar el contenido XML como una cadena. Java DB tiene un operador llamado XMLSERIALIZE que convierte un tipo XML en un tipo de carácter: 

String query =
    "select RSS_NAME, " +
    "xmlserialize " +
    "(RSS_FEED_XML as clob) " +
    "from RSS_FEEDS";

Como con el operador XMLPARSE, el operador XMLSERIALIZE requiere que Apache Xalan aparezca en la ruta de clases de Java.
