Usando JDBC con una IPA de IGU

El ejemplo CoffeesFrame.java demuestra cómo integrar JDBC con una IPA de IGU, en particular, la IPA de Swing. Se muestra los contenidos de la tabla de base de datos COFFEES en una tabla y contiene campos y botones que le ha habilitan a agregar filas a la tabla. Lo siguiente es una captura de este ejemplo:

El ejemplo contiene cinco campos de texto que se corresponden con cada una de las columnas en la tabla COFFEES. También contiene tres botones:

  • Agregar fila a la tabla: Agrega una fila a la tabla del ejemplo basado en los datos introducidos en los campos de texto.
  • Actualizar base de datos: Actualiza la tabla COFFEES basado en los datos en la tabla de ejemplo.
  • Descartar cambios: Recupera los contenidos de la tabla COFFEES, reemplazando los datos existentes en la tabla del ejemplo.

Este ejemplo (el cual requiere un CoffeesTableModel) demuestra los siguientes pasos generales para integrar JDBC con la IPA de Swing:

  1. Implementar la interfaz TableModel
  2. Implementar la interfaz RowSetListener
  3. Disponer los componentes de Swing
  4. Agregar listeners para los botones en el ejemplo

Implementar javax.swing.event.TableModel

La interfaz TableModel capacita a una aplicación Java Swing a manejar datos en un objeto JTable. El ejemplo, CoffeesTableModel.java, implementa esta interfaz. Especifica cómo un objeto JTable recuperaría los datos desde un objeto RowSet y mostrarlos en una tabla.

Nota: Aunque este ejemplo visualiza los contenidos de la tabla COFFEES en una aplicación Swing, la clase CoffeesTableModel debería funcionar para cualquier tabla SQL siempre que sus datos puedan ser representados con objetos String. (Sin embargo, los campos que permiten a los usuarios a agregar filas a COFFEES, los cuales se especifican en la clase CoffeesFrame, tendrían que ser modificados para otras tablas SQL).

Antes de implementar los métodos de la interfaz TableModel, el constructor de la clase CoffeeTableModel inicializa varias variables miembro requeridas parar estos métodos implementados como sigue: 

public CoffeesTableModel(CachedRowSet rowSetArg)
    throws SQLException {

    this.coffeesRowSet = rowSetArg;
    this.metadata = this.coffeesRowSet.getMetaData();
    numcols = metadata.getColumnCount();

    // Retrieve the number of rows.
    this.coffeesRowSet.beforeFirst();
    this.numrows = 0;
    while (this.coffeesRowSet.next()) {
        this.numrows++;
    }
    this.coffeesRowSet.beforeFirst();
}

Lo siguiente describe las variables miembro inicializadas en este constructor:

  • CachedRowSet coffeesRowSet: Almacena los conteidos de la tabla COFFEES.

    Este ejemplo usa un objeto RowSet, en particular, un objeto CachedRowSet, en vez de un objeto ResultSet por dos razones. Un objeto CachedRowSet habilita al usuario de la aplicacción a hacer cambios a los datos contenidos en él sin estar conectado a la base de datos. Además, ya que un objeto CachedRowSet es un componente JavaBeans, puede notificar a otros componentes cuando le ocurran ciertas cosas. En este ejemplo, cuando se agrega una fila nueva al objeto CachedRowSet, notifica al componente Swing que está representando sus datos en una tabla a refrescarse a sí mismo y visualizar la nueva fila.

  • ResultSetMetaData metadata: Recuperar el número de columnas en la tabla COFFEES como tambien los nombres de cada una de ellas.

  • int numcols, numrows: Almacena el número de columnas y filas, respectivamente, en la tabla COFFEES.

El ejemplo CoffeesTableModel.java implementa los métodos siguientes de la interfaz TableModel:

  • Class<?> getColumnClass(int columnIndex): Devuelve la superclase más específica para todos los valores de celda en la columna.
  • int getColumnCount(): Devuelve el número de columnas en el modelo.
  • String getColumnName(int columnIndex): Devuelve el número de la columna especificada por el parámetro columnIndex.
  • int getRowCount(): Devuelve el número de filas en el modelo.
  • Object getValueAt(int rowIndex, int columnIndex): Devuelve el valor para la celda en la intersección de la columna columnIndex y la fila rowIndex.
  • boolean isCellEditable(int rowIndex, int columnIndex): Devuelve verdadero si la celda en la intersección de la columna rowIndex y la fila columnIndex puede ser editada.

Los métodos siguientes no han sido implementados porque este ejemplo no permite a los usuarios editar directamente los contenidos de la tabla:

  • void addTableModelListener(TableModelListener l): Agrega un listener a la lista que es notificada cada vez que un cambio ocurre en el modelo de datos.
  • void removeTableModelListener(TableModelListener l): Elimina un listener de la lista que es notificada cada vez que un cambio ocurren en el modelo de datos.
  • void setValueAt(Object aValue, int rowIndex, int columnIndex): Establece el valor en la celda en la intersección de la columna columnIndex y la fila rowIndex al objeto aValue.

Implementar getColumnCount y getRowCount

Los métodos getColumnCount y getRowCount devuelven el valor de los miembros variables numcols y numrows, respectivamente: 

public int getColumnCount() {
    return numcols;
}

public int getRowCount() {
    return numrows;
}

Implementar getColumnClass

El método getColumnClass devuelve el tipo de dato de la columna especificada. Para mantener las cosas simples, este método devuelve la clase String, de este modo convirtiendo todos los datos en la tabla a objetos String. La clase JTable usa este método para determinar cómo representar los datos en la aplicación IGU. 

public Class getColumnClass(int column) {
    return String.class;
}

Implementar getColumnName

El método getColumnName devuelve el nombre de la columna especificada. La clase JTable usa este método para etiquetar cada una de sus columnas. 

public String getColumnName(int column) {
    try {
        return this.metadata.getColumnLabel(column + 1);
    } catch (SQLException e) {
        return e.toString();
    }
}

Implementar getColumnAt

El método getColumnAt recupera el valor en la fila y columna especificada en el conjunto de filas coffeesRowSet. La clase JTable usa este método para rellenar su tabla. Note que SQL empieza a numerar sus filas y columnas en 1, pero la interfaz de TableModel empieza en 0; esta es la razón porque los valores rowIndex y columnIndex están incrementados en 1. 

public Object getValueAt(int rowIndex, int columnIndex) {

    try {
        this.coffeesRowSet.absolute(rowIndex + 1);
        Object o = this.coffeesRowSet.getObject(columnIndex + 1);
        if (o == null)
            return null;
        else
            return o.toString();
    } catch (SQLException e) {
        return e.toString();
    }
}

Implementar isCellEditable

Debido a que este ejemplo no permite a los usuarios a editar directamente los contenidos de la tabla (las filas son agregadas por otro control de la ventana), este método devuelve false independientemente de los valores de rowIndex y columnIndex: 

public boolean isCellEditable(int rowIndex, int columnIndex) {
    return false;
}

Implementar javax.sql.RowSetListener

La clase CoffeesFrame implementa sólo un método de la interfaz RowSetListener, rowChanged. Este método se llama cuando un usuario agrega una fila a la tabla. 

public void rowChanged(RowSetEvent event) {

    CachedRowSet currentRowSet =
        this.myCoffeesTableModel.coffeesRowSet;

    try {
        currentRowSet.moveToCurrentRow();
        myCoffeesTableModel = new CoffeesTableModel(
            myCoffeesTableModel.getCoffeesRowSet());
        table.setModel(myCoffeesTableModel);

    } catch (SQLException ex) {

        JDBCTutorialUtilities.printSQLException(ex);

        // Display the error in a dialog box.

        JOptionPane.showMessageDialog(
            CoffeesFrame.this,
            new String[] {
                // Display a 2-line message
                ex.getClass().getName() + ": ",
                ex.getMessage()
            }
        );
    }
}

Este método actualiza la tabla en la aplicación IGU.

Disponer los componentes de Swing

El constructor de la clase CoffeesFrame inicializa y distribuye los componentes. La sentencia siguiente recupera los contenidos de la tabla COFFEES, almacena los contenidos en el objeto CachedRowSet myCachedRowSet, e inicializa el componente de Swing JTable: 

CachedRowSet myCachedRowSet = getContentsOfCoffeesTable();
myCoffeesTableModel = new CoffeesTableModel(myCachedRowSet);
myCoffeesTableModel.addEventHandlersToRowSet(this);

// Displays the table   
table = new JTable(); 
table.setModel(myCoffeesTableModel);

Como se mencionó previamente, en vez de un objeto ResultSet para representar los contenidos de la tabla COFFEES, este ejemplo usa un objeto RowSet, notablemente un objeto CachedRowSet.

El método CoffeesFrame.getContentsOfCoffeesTable recupera los contenidos de la tabla COFFEES.

El método CoffeesTableModel.addEventHandlersToRowSet agrega el controlador de eventos definido en la clase CoffeesFrame, el cual es el método rowChanged, a la variable miembro del conjunto de filas CoffeesTableModel.coffeesRowSet. Esto habilita a la clase CoffeesFrame a notificar al conjunto de fila coffeesRowSet de cualquier evento, en particular, cuando un usuario pulsa el botón Agegar fila a la tabla, Actualizar la base de datos, o Descartar Cambios. Cuando se le notifica al conjunto de filas coffeesRowSet uno de estos cambios, se llama al método CoffeesFrame.rowChanged.

La sentencia table.setModel(myCoffeesTableModel) especifica que se usa el objeto CoffeesTableModel myCoffeesTableModel para rellenar el componente Swing JTable table.

Las sentencias siguientes especifican que la clase CoffeesFrame usa el diseño GridBagLayout para disponer sus componentes Swing: 

Container contentPane = getContentPane();
contentPane.setComponentOrientation(
    ComponentOrientation.LEFT_TO_RIGHT);
contentPane.setLayout(new GridBagLayout());
GridBagConstraints c = new GridBagConstraints();

Vea Cómo Usar GridBagLayout en Creando una IGU (Interfaz Gráfica de Usuario) con JFC/Swing para más información sobre el uso del diseño GridBagLayout.

Vea el código fuente de CoffeesFrame.java para ver cómo los componentes Swing de este ejemplo son agregados al diseño GridBagLayout.

Agregar listeners para los botones en el ejemplo

La siguiente sentencia agrega un listener al botón Agregar fila a la tabla: 

button_ADD_ROW.addActionListener(
    new ActionListener() {
      
    public void actionPerformed(ActionEvent e) {

        JOptionPane.showMessageDialog(
            CoffeesFrame.this, new String[] {
                "Adding the following row:",
                "Coffee name: [" +
                textField_COF_NAME.getText() +
                "]",
                "Supplier ID: [" +
                textField_SUP_ID.getText() + "]",
                "Price: [" +
                textField_PRICE.getText() + "]",
                "Sales: [" +
                textField_SALES.getText() + "]",
                "Total: [" +
                textField_TOTAL.getText() + "]"
            }
        );

        try {
            myCoffeesTableModel.insertRow(
                textField_COF_NAME.getText(),
                Integer.parseInt(textField_SUP_ID.getText().trim()),
                Float.parseFloat(textField_PRICE.getText().trim()),
                Integer.parseInt(textField_SALES.getText().trim()),
                Integer.parseInt(textField_TOTAL.getText().trim())
            );
        } catch (SQLException sqle) {
            displaySQLExceptionDialog(sqle);
        }
    }
});

Cuando un usuario pulsa este botón, se realiza lo siguiente:

  • Crea un cuadro de diálogo de mensaje que muestra la fila que se agregará a la tabla.
  • Llama al método CoffeesTableModel.insertRow, el cual agrega la fila a la variable miembro CoffeesTableModel.coffeesRowSet.

Si se lanza una SQLException, después el método CoffeesFrame.displaySQLExceptionDialog crea un cuadro de diálogo de mensaje que visualiza el contenido de la SQLException.

La sentencia siguiente agrega un listener al botón Actualizar base de datos: 

button_UPDATE_DATABASE.addActionListener(
    new ActionListener() {
        public void actionPerformed(ActionEvent e) {
            try {
                myCoffeesTableModel.coffeesRowSet.acceptChanges();
                msgline.setText("Updated database");
            } catch (SQLException sqle) {
                displaySQLExceptionDialog(sqle);
                // Now revert back changes
                try {
                    createNewTableModel();
                    msgline.setText("Discarded changes");
                } catch (SQLException sqle2) {
                    displaySQLExceptionDialog(sqle2);
                }
            }
        }
    }
);

Cuando un usuario pulse este botón, la tabla COFFEES es actualizada con los contenidos del conjunto de fila myCoffeesTableModel.coffeesRowSet.

La sentencia siguiente agrega un listener al botón Descartar cambios: 

button_DISCARD_CHANGES.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        try {
            createNewTableModel();
        } catch (SQLException sqle) {
            displaySQLExceptionDialog(sqle);
        }
    }
});

Cuando un usuario pulsa este botón, se llama al método CoffeesFrame.createNewTableModel, el cual rellena el componente JTable con los contenidos de la tabla COFFEES.
