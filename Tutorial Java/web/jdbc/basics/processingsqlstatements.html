Procesando Sentencias SQL con JDBC

En general, para procesar cualquier sentencia SQL con JDBC, sigue estos pasos:

  1. Establece una conexión.
  2. Crea una sentencia (statement, en inglés).
  3. Ejecuta la consulta.
  4. Procesa el objeto ResultSet.
  5. Cierra la conexión.

Esta página usa el método siguiente, CoffeesTables.viewTable, del ejemplo del tutorial para demostrar estos pasos. Este método genera el contenido de la tabla COFFEES. Este método será descrito en más detalle más tarde en este tutorial: 

public static void viewTable(Connection con, String dbName)
    throws SQLException {

    Statement stmt = null;
    String query = "select COF_NAME, SUP_ID, PRICE, " +
                   "SALES, TOTAL " +
                   "from " + dbName + ".COFFEES";
    try {
        stmt = con.createStatement();
        ResultSet rs = stmt.executeQuery(query);
        while (rs.next()) {
            String coffeeName = rs.getString("COF_NAME");
            int supplierID = rs.getInt("SUP_ID");
            float price = rs.getFloat("PRICE");
            int sales = rs.getInt("SALES");
            int total = rs.getInt("TOTAL");
            System.out.println(coffeeName + "\t" + supplierID +
                               "\t" + price + "\t" + sales +
                               "\t" + total);
        }
    } catch (SQLException e ) {
        JDBCTutorialUtilities.printSQLException(e);
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}

Establecer conexiones

Primero, establezca una conexión con la fuente de datos que quiera usar. Una fuente de datos puede ser un DBMS, un sistema de ficheros heredado, o alguna otra fuente de datos con su correspondiente controlador JDBC. Esta conexión se representa con un objeto Connection. Vea Estableciendo una Conexión para más información.

Crear Sentencias

Una Statement es una interfaz que representa una sentencia SQL. Usted ejecuta objetos Statement, y ellos generan objetos ResultSet, los cuales son una tabla de datos que representan un conjunto de resultados de la base de datos. Necesita un objeto Connection para crear un objeto Statement.

Por ejemplo, CoffeesTables.viewTable crea un objeto Statement con el código siguiente: 

stmt = con.createStatement();

Hay tres clases diferentes de sentencias:

  • Statement: Usada para implementar sentencias SQL sin parámetros.
  • PreparedStatement: (Extiende Statement). Usada para las sentencias SQL precompiladas que podrían contener parámetros de entrada. Vea Usando Prepared Statements para más información.
  • CallableStatement: (Extiende PreparedStatement). Usado para ejecutar procedimientos almacenados que pueden contener tanto parámetros de entrada como de salida. Vea Procedimientos Almacenados para más información.

Ejecutar Consultas

Para ejecutar una consulta, llame al método execute desde Statement de las siguientes formas:

  • execute: Devuelve true si el primer objeto que la consulta devuelve es un objeto ResultSet. Use este método si la consulta pudiera devolver uno o más objetos ResultSet. Recupere los objetos ResultSet devueltos desde la consulta llamando repetidamente a Statement.getResultSet.
  • executeQuery: Devuelve un objeto ResultSet.
  • executeUpdate: Devuelve un entero que representa el número de filas afectados por la sentencia SQL. Use este método si está usando sentencias SQL INSERT, DELETE, o UPDATE.

Por ejemplo, CoffeesTables.viewTable ejecutó un objeto Statement con el código siguiente: 

ResultSet rs = stmt.executeQuery(query);

Vea Recuperando y Modificando Valores de los Result Sets para más información.

Procesar Objetos ResultSet

Puede acceder los datos en un objeto ResultSet a través de un cursor. Note que este cursor no es un cursor de base de datos. Este cursor es un puntero que apunta a una fila de datos en el objeto ResultSet. Inicialmente, el cursor se posiciona antes de la primera fila. Puede llamar a varios métodos definidos en el objeto ResultSet para mover el cursor.

Por ejemplo, CoffeesTables.viewTable repetidamente llama al método ResultSet.next para mover el cursor hacia adelante por una fila. Cada vez que llama a next, el método saca los datos en la fila donde el cursor está actualmente posicionado: 

try {
    stmt = con.createStatement();
    ResultSet rs = stmt.executeQuery(query);
    while (rs.next()) {
        String coffeeName = rs.getString("COF_NAME");
        int supplierID = rs.getInt("SUP_ID");
        float price = rs.getFloat("PRICE");
        int sales = rs.getInt("SALES");
        int total = rs.getInt("TOTAL");
        System.out.println(coffeeName + "\t" + supplierID +
                           "\t" + price + "\t" + sales +
                           "\t" + total);
    }
}
// ...

Vea Recuperando y Modificando Valores de los Result Sets para más información.

Cerrar las Conexiones

Cuando ha terminado de usar una Statement, llame al método Statement.close para liberar inmediatamente los recursos que está usando. Cuando llama a este método, sus objetos ResultSet se cierran.

Por ejemplo, el método CoffeesTables.viewTable se asegura de que el objeto Statement se cierra al final del método, independientemente de los objetos SQLException lanzados, envolviéndolos en un bloque finally: 

} finally {
    if (stmt != null) { stmt.close(); }
}

JDBC lanza una SQLException cuando encuentra un error durante una interacción con una fuente de datos. Vea Manejar Excepciones SQL para más información.

En JDBC 4.1, el cual está disponible en la versión 7 de Java y posteriores, puede usar una sentencia try-with-resources para cerrar automáticamente los objetos Connection, Statement, y ResultSet, independientemente de si una SQLException haya sido lanzada. Una instrucción de recurso automática consta de una instrucción try y uno o más recursos declarados. Por ejemplo, puede modificar CoffeesTables.viewTable de forma que su objeto Statement se cierra automáticamente, como sigue: 

public static void viewTable(Connection con) throws SQLException {

    String query = "select COF_NAME, SUP_ID, PRICE, " +
                   "SALES, TOTAL " +
                   "from COFFEES";

    try (Statement stmt = con.createStatement()) {

        ResultSet rs = stmt.executeQuery(query);

        while (rs.next()) {
            String coffeeName = rs.getString("COF_NAME");
            int supplierID = rs.getInt("SUP_ID");
            float price = rs.getFloat("PRICE");
            int sales = rs.getInt("SALES");
            int total = rs.getInt("TOTAL");
            System.out.println(coffeeName + ", " + supplierID +
                               ", " + price + ", " + sales +
                               ", " + total);
        }
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    }
}

La sentencia siguiente es una sentencia try-with-resources, el cual declara un recurso, stmt, que será automáticamente cerrado cuando termina el bloque try: 

try (Statement stmt = con.createStatement()) {
    // ...
}
