Usando Objetos RowSet

Un objeto RowSet de JDBC contiene datos tabulares de una forma que los hace más flexibles y más fáciles de usar que un conjunto de resultados.

Oracle ha definido cinco interfaces de RowSet para algunos de los usos más populares de un RowSet, y la referencia estándar está disponible para estas interfaces RowSet. En este tutorial aprenderá cómo usar estas implementaciones de referencia.

Estas versiones de la interfaz de RowSet y sus implementaciones han sido proporcionadas como una conveniencia para los programadores. Los programadores son libres de escribir sus propias versiones de la interfaz javax.sql.RowSet, para extender las implementaciones de las cinco interfaces de RowSet, o escribir sus propias implementaciones. Sin embargo, muchos programadores probablemente encontrarán que las implementaciones estándar de referencia se ajustan ya a sus necesidades y las usarán tal cual.

Esta sección le introduce a la interfaz RowSet y las interfaces siguientes que extienden esta interfaz:

  • JdbcRowSet
  • CachedRowSet
  • WebRowSet
  • JoinRowSet
  • FilteredRowSet

Se cubren los siguientes temas:

  • ¿Qué Pueden Hacer los Objetos RowSet?
  • Clases de Objetos RowSet

¿Qué Pueden Hacer los Objetos RowSet?

Todos los objetos RowSet se derivan de la interfaz ResultSet y por lo tanto comparten sus capacidades. Lo que hace especial a los objetos RowSet de JDBC es que agregan estas nuevas capacidades:

  • Funcionar como un Componente JavaBeans
  • Agregar Capacidad de Desplazamiento o Actualización

Funcionar como un Componente JavaBeans

Todos los objetos RowSet son componentes JavaBeans. Esto significa que tienen lo siguiente:

  • Propiedades
  • Mecanismo de Notificación de JavaBeans

Propiedades

Todos los objetos RowSet tienen propiedades. Una propiedad es un campo que tiene sus correspondientes métodos obtenedores (getter, en inglés) y establecedores (setter, en inglés). Las propiedades se exponen a las herramientas de construcción (tal como esos que vienen con los EIDs JDeveloper y Eclipse) que le habilitan a manipular visualmente los beans. Para más información, vea la lección Propiedades en el recorrido JavaBeans.

Mecanismo de Notificación de JavaBeans

Los objetos RowSet usan el modelo de eventos de JavaBeans, en el cual los componentes registrados son notificados cuando ocurren ciertos eventos. Para todos los objetos RowSet, tres eventos disparan notificaciones:

  • Un movimiento del cursor
  • La actualización, inserción, o borrado de una fila
  • Un cambio al contenido entero del RowSet

La notificación de un evento va a todos los listeners, los componentes que han implementado la interfaz RowSetListener y se han agregado a la lista de componentes del objeto RowSet para ser notificados cuando ocurra cualquiera de los tres eventos.

Un listener podría ser un componente del IGU como un gráfico de barras. Si el gráfico de barras está monitorizando los datos en un objeto RowSet, el listener querría conocer los nuevos valores cada vez que los datos cambien. El listener implementaría por lo tanto los métodos RowSetListener para definir qué hará cuando ocurra un evento en particular. Después el listener también debe ser agregado a la lista de listeners del objeto RowSet. La línea siguiente de código registra el componente de gráfico de barras bg con el objeto rs de tipo RowSet. 

rs.addListener(bg);

Ahora bg será notificado cada vez que el cursor se mueva, se cambie una fila, o todo el rs obtenga datos nuevos.

Agregar Capacidad de Desplazamiento o Actualización

Algunos DBMSs no admiten conjuntos de resultados que puedan ser recorridos (recorribles), y algunos no admiten conjuntos de resultados que puedan actualizarse (actualizables). Si un controlador para ese DBMS no agrega la habilidad de desplazarse o de actualización del conjunto de resultados, puede usar un objeto RowSet para hacerlo. Un objeto RowSet es recorrible y actualizable de forma predeterminada así que al rellenar un objeto RowSet con los contenidos de un conjunto de resultados, puede efectivamente hacer el conjunto de resultados recorrible y actualizable.

Clases de Objetos RowSet

Un objeto RowSet se considera conectado o desconectado. Un objeto RowSet conectado usa un controlador JDBC para hacer una conexión a una base de datos relacional y mantiene esa conexión a lo largo de su vida útil. Un objeto RowSet desconectado hace una conexión a la fuente de datos sólo para leer los datos del objeto ResultSet o para escribir de vuelta los datos a la fuente de datos. Después de leer los datos de o escribir los datos a su fuente de datos, el objeto RowSet se desconecta de ella, convirtiéndose así en "desconectado". Durante la mayoría de su vida útil, un objeto RowSet desconectado no tiene conexión a su fuente de datos y opera de forma independiente. Las dos próximas secciones le cuentan lo que significa estar conectado o desconectado en términos de lo que un objeto RowSet puede hacer.

Objetos RowSet Conectados

Sólo una de las implementaciones estándar de RowSet es un objeto RowSet conectado: JdbcRowSet. Estando siempre conectado a una base de datos, un objeto JdbcRowSet es lo más similar a un objeto ResultSet y es a menudo usado como envoltorio para hacer que un objeto ResultSet de sólo lectura y no desplazable sea desplazable y actualizable.

Como un componente JavaBeans, un objeto JdbcRowSet puede ser usado, por ejemplo, en una herramienta de IGU para seleccionar un controlador JDBC. Un objeto JdbcRowSet puede ser usado de esta forma porque es efectivamente un envoltorio para el controlador que obtuvo su conexión a la base de datos.

Objetos RowSet Desconectados

Las otras cuatro implementaciones son implementaciones de RowSet desconectadas. Los objetos RowSet desconectados tienen todas las capacidades de los objetos RowSet conectados y además tienen las capacidades adicionales disponibles sólo en los objetos RowSet desconectados. Por ejemplo, no tener que mantener una conexión a una fuente de datos hace a los objetos RowSet desconectados mucho más ligeros que un objeto JdbcRowSet o un objeto ResultSet. Los objetos RowSet desconectados son también serializables, y la combinación de ser tanto serializables como ligeros los hacen ideales para enviar datos a través de la red. Pueden incluso ser usados para enviar datos a clientes ligeros como PDAs y teléfonos móviles.

La interfaz CachedRowSet define las capacidades básicas disponibles en todos los objetos RowSet desconectados. Los otros tres son extensiones de la interfaz CachedRowSet, los cuales proporcionan capacidades más especializadas. La información siguiente muestra cómo están relacionados:

Un objeto CachedRowSet tiene todas las capacidades de un objeto JdbcRowSet y además puede hacer también lo siguiente:

  • Obtener una conexión a la fuente de datos y ejecutar una consulta
  • Leer los datos del objeto ResultSet resultante y rellenarse a sí mismo con esos datos
  • Manipular los datos y hacer cambios a los datos mientras está desconectado
  • Reconectar a la fuente de datos y escribir los cambios de vuelta en él
  • Comprobar los conflictos con la fuente de datos y resolver esos conflictos

Un objeto WebRowSet tiene todas las capacidades de un objeto CachedRowSet y puede también hacer lo siguiente:

  • Escribirse a sí mismo como un documento XML
  • Leer un documento XML que describe un objeto WebRowSet

Un objeto JoinRowSet tiene todas las capacidades de un objeto WebRowSet (y por lo tanto todas las de un objeto CachedRowSet) y además puede hacer también lo siguiente:

  • Formar el equivalente de un SQL JOIN sin tener que conectar a una fuente de datos

Un objeto FilteredRowSet del mismo modo tiene todas las capacidades de un objeto WebRowSet (y por lo tanto también de un objeto CachedRowSet) además de que también puede hacer lo siguiente:

  • Aplicar un criterio de filtrado de forma que solo los datos seleccionados estén visibles. Esto es equivalente a ejecutar una consulta en un objeto RowSet sin tener que usar un lenguaje de consulta o conectar a una fuente de datos.
