Usando Prepared Statements

Esta página cubre los temas siguientes:

Descripción General de las Prepared Statements

Algunas veces es más conveniente usar un objeto PreparedStatement para enviar sentencias SQL a la base de datos. Este tipo especial de sentencia se dervia de la clase más general, Statement, que usted ya conoce.

Si quiere ejecutar un objeto Statement varias veces, normalmente se reduce el tiempo de ejecución el usar un objeto PreparedStatement en su lugar.

La característica principal de un objeto PreparedStatement es que, a diferencia de un objeto Statement,se le da una sentencia SQL cuando se crea. La ventaja de esto es que en la mayoría de los casos, esta sentencia SQL se envía a la DBMS tal cual, donde es compilada. Como resultado, el objeto PreparedStatement contiene no simplemente una sentencia SQL, sino una sentencia SQL que ha sido precompilada. Esto significa que cuando se ejecuta la PreparedStatement, el DBMS puede simplemente ejecutar la sentencia SQL en PreparedStatement sin tener que compilarla primero.

Aunque los objetos PreparedStatement pueden usarse para sentencias SQL sin parámetros, probablemente las use más a menudo para sentencias SQL que toman parámetros. La ventaja de usar sentencias SQL que toman parámetros que que puede usar la misma sentencia y suministrarla con valores diferentes cada vez que la ejecute. Ejemplos de esto están en las secciones siguientes.

El método siguiente, CoffeesTable.updateCoffeeSales, almacena el número de libras de café vendido en la semana actual en la columna SALES para cada tipo de café, y actualiza el número total de libras de café vendidas en la columna TOTAL para cada tipo de café: 

public void updateCoffeeSales(HashMap<String, Integer> salesForWeek)
    throws SQLException {

    PreparedStatement updateSales = null;
    PreparedStatement updateTotal = null;

    String updateString =
        "update " + dbName + ".COFFEES " +
        "set SALES = ? where COF_NAME = ?";

    String updateStatement =
        "update " + dbName + ".COFFEES " +
        "set TOTAL = TOTAL + ? " +
        "where COF_NAME = ?";

    try {
        con.setAutoCommit(false);
        updateSales = con.prepareStatement(updateString);
        updateTotal = con.prepareStatement(updateStatement);

        for (Map.Entry<String, Integer> e : salesForWeek.entrySet()) {
            updateSales.setInt(1, e.getValue().intValue());
            updateSales.setString(2, e.getKey());
            updateSales.executeUpdate();
            updateTotal.setInt(1, e.getValue().intValue());
            updateTotal.setString(2, e.getKey());
            updateTotal.executeUpdate();
            con.commit();
        }
    } catch (SQLException e ) {
        JDBCTutorialUtilities.printSQLException(e);
        if (con != null) {
            try {
                System.err.print("Transaction is being rolled back");
                con.rollback();
            } catch(SQLException excep) {
                JDBCTutorialUtilities.printSQLException(excep);
            }
        }
    } finally {
        if (updateSales != null) {
            updateSales.close();
        }
        if (updateTotal != null) {
            updateTotal.close();
        }
        con.setAutoCommit(true);
    }
}

Crear un Objeto PreparedStatement

Lo siguiente crea un objeto PreparedStatement que toma dos parámetros de entrada: 

String updateString =
    "update " + dbName + ".COFFEES " +
    "set SALES = ? where COF_NAME = ?";
updateSales = con.prepareStatement(updateString);

Suplir Valores para los Parámetros de una PreparedStatement

Debe suministrar valores en lugar de los marcadores de posición de signo de interrogación (si hay alguno) antes de que pueda ejecutar un objeto PreparedStatement. Haga esto llamando a uno de los métodos establecedores (setter, en inglés) definidos en la clase PreparedStatement. Las sentencias siguientes proporcionan los dos marcadores de posición de signo de interrogación en el PreparedStatement llamado updateSales: 

updateSales.setInt(1, e.getValue().intValue());
updateSales.setString(2, e.getKey());

El primer argumento para cada uno de esto métodos establecedores especifica el marcador de posición de signo de interrogación. En el este ejemplo, setInt especifica el primer marcador de posición y setString especifica el segundo marcador de posición.

Después de que un parámetro ha sido establecido con un valor, retiene ese valor hasta que se restablece a otro valor, o se llama al método clearParameters. Usando el objeto updateSales de tipo PreparedStatement, el fragmento de código siguiente ilustra la reutilización de una sentencia preparada (prepared statement, en inglés) después de restablecer el valor de uno de sus parámetros y dejar el otro igual: 

// changes SALES column of French Roast
//row to 100

updateSales.setInt(1, 100);
updateSales.setString(2, "French_Roast");
updateSales.executeUpdate();

// changes SALES column of Espresso row to 100
// (the first parameter stayed 100, and the second
// parameter was reset to "Espresso")

updateSales.setString(2, "Espresso");
updateSales.executeUpdate();

Usar Bucles para Establecer Valores

A menudo puede hacer el código más fácil usando un bucle for o un bucle while para establecer los valores de los parámetros de entrada.

El método CoffeesTable.updateCoffeeSales usa un bucle for-each para establecer valores repetidamente en los objetos PreparedStatement updateSales y updateTotal: 

for (Map.Entry<String, Integer> e : salesForWeek.entrySet()) {

    updateSales.setInt(1, e.getValue().intValue());
    updateSales.setString(2, e.getKey());

    // ...
}

El método CoffeesTable.updateCoffeeSales toma un argumento, HashMap. Cada elemento en el argumento HashMap contiene el nombre de un tipo de café y el número de libras de ese tipo de café vendido durante la semana actual. El bucle for-each itera a través de cada elemento del argumento HashMap y establece los marcadores de posición de signo de interrogación apropiados en updateSales y updateTotal.

Ejecutar Objetos PreparedStatement

Como con los objetos Statement, para ejecutar un objeto PreparedStatement, llama a una sentencia de ejecución: executeQuery si la consulta devuelve sólo un ResultSet (tal como una sentencia SQL SELECT), executeUpdate si la consulta no devuelve un ResultSet (tal como una sentencia SQL UPDATE), o execute si la consulta puede devolver más de un objeto ResultSet. Ambos objetos PreparedStatement en CoffeesTable.updateCoffeeSales contiene sentencias SQL UPDATE, así que ambos son ejecutados llamando a executeUpdate: 

updateSales.setInt(1, e.getValue().intValue());
updateSales.setString(2, e.getKey());
updateSales.executeUpdate();

updateTotal.setInt(1, e.getValue().intValue());
updateTotal.setString(2, e.getKey());
updateTotal.executeUpdate();
con.commit();

No se suministran argumentoa a executeUpdate cuando se usa para ejecutar updateSales y updateTotals; Ambos objetos PreparedStatement ya contienen la sentencia SQL a ejecutar

Nota: Al principio de CoffeesTable.updateCoffeeSales, el modo de auto-confirmación se establece a false: 

con.setAutoCommit(false);

Como consecuencia, ninguna sentencia SQL es confirmada hasta que se llame al método commit. Para más información sobre el modo de auto-confirmación, vea Transacciones.

Devolver Valores para el Método executeUpdate

Mientras que executeQuery devuelve un objeto ResultSet conteniendo los resultados de la consulta enviada al DBMS, el valor devuelto para executeUpdate es un valor int que indica cuantas filas de la tabla se actualizaron. Por ejemplo, el código siguiente muestra el valor devuelto de executeUpdate siendo asignado a la variable n: 

updateSales.setInt(1, 50);
updateSales.setString(2, "Espresso");
int n = updateSales.executeUpdate();
// n = 1 because one row had a change in it

La tabla COFFEES se actualiza; el valor 50 reemplaza al valor en la columna SALES en la fila para Espresso. Esa actualización afecta a una fila en la tabla, por lo que n es igual a is equal to 1.

Cuando se usa el método executeUpdate para ejecutar una sentencia DDL (lenguaje de definición de datos), tal como en la creación de una tabla, devuelve el valor int de 0. Como consecuencia, en el siguiente fragmento de código, que ejecuta una sentencia DDL usada para crear la tabla COFFEES, se le asigna a n un valor de 0: 

// n = 0
int n = executeUpdate(createTableCoffees);

Dese cuenta de que cuando el valor devuelto para executeUpdate es 0, esto puede significar una de dos cosas:

  • La sentencia ejecutada era una sentencia de actualización que afectó a cero filas.
  • La sentencia ejecuta era una sentencia DDL.
