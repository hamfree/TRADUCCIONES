Utilizando Objetos JdbcRowSet

Un objeto JdbcRowSet es un objeto ResultSet mejorado. Mantiene una conexión a su fuente de datos, igual que hace un objeto ResultSet. La gran diferencia es que tiene un conjunto de propiedades y un mecanismo de notificación de listener que le hace un componente JavaBean.

Uno de los usos principales de un objeto JdbcRowSet es hacer un objeto ResultSet desplazable y actualizable cuando de otra manera no tiene esas capacidades.

Esta sección cubre los temas siguientes:

  • Crear Objetos JdbcRowSet
  • Objetos JdbcRowSet Predeterminados
  • Establecer Propiedades
  • Utilizar Objetos JdbcRowSet
  • Ejemplo de Código

Crear Objetos JdbcRowSet

Puede crear un objeto JdbcRowSet de varias formas:

  • Usando el constructor de la implementación de referencia que toma un objeto ResultSet
  • Usando el constructor de la implementación de referencia que toma un objeto Connection
  • Usando el constructor por defecto de la implementación de referencia
  • Usando una instancia de RowSetFactory, la cual se crea desde la clase RowSetProvider

Nota: Alternativamente, puede usar el constructor desde la implementación de JdbcRowSet de su controlador JDBC. Sin embargo, las implementaciones de la interfaz RowSet diferirán de la implementación de referencia. Estas implementaciones tienen diferentes nombres y constructores. Por ejemplo, la implementación de la interfaz de JdbcRowSet del controlador JDBC de Oracle se llama oracle.jdbc.rowset.OracleJDBCRowSet.

Pasar Objetos ResultSet

La forma más simple de crear un objeto JdbcRowSet es producir un objeto ResultSet y pasarlo al constructor de JdbcRowSetImpl. Hacer esto no sólo crea un objeto JdbcRowSet sino que también lo rellena con los datos en el objeto ResultSet.

Nota: El objeto ResultSet que se pasa al constructor JdbcRowSetImpl debe ser desplazable.

Como un ejemplo, el fragmento de código siguiente usa el objeto Connection con para crear un objeto Statement, stmt, el cual después ejecuta una consulta. La consulta produce el objeto ResultSet rs, el cual se pasa al constructor para crear un nuevo objeto JdbcRowSet inicializado con los datos en rs: 

stmt = con.createStatement(
           ResultSet.TYPE_SCROLL_SENSITIVE,
           ResultSet.CONCUR_UPDATABLE);
rs = stmt.executeQuery("select * from COFFEES");
jdbcRs = new JdbcRowSetImpl(rs);

Un objeto JdbcRowSet creado con un objeto ResultSet sirve como un envoltorio para el objeto ResultSet. Ya que el objeto RowSet rs es desplazable y actualizable, jdbcRs es también desplazable y actualizable. Si ha ejecutado el método createStatement sin ningún argumento, rs no será desplazable o actualizable, y tampoco lo será jdbcRs.

Pasar Objetos Connection

La primera sentencia en el trozo de código siguietne de JdbcRowSetSample crea un objeto JdbcRowSet que conecta a la base de datos con el objeto Connection con: 

jdbcRs = new JdbcRowSetImpl(con);
jdbcRs.setCommand("select * from COFFEES");
jdbcRs.execute();

El objeto jdbcRs no contiene datos hasta que especifique una sentencia SQL con el método setCommand, después ejecute el método execute.

El objeto jdbcRs es desplazable y actualizable; por defecto, JdbcRowSet y todos los otros objetos RowSet son desplazables y actualizables a menos que se especifique lo contrario. Vea Objetos JdbcRowSet Predeterminados para más información sobre las propiedades de JdbcRowSet que puede especificar.

Usar el Constructor por Defecto

La primera sentencia en el siguiente trozo de código cra un objeto JdbcRowSet vacío. 

public void createJdbcRowSet(String username, String password) {

    jdbcRs = new JdbcRowSetImpl();
    jdbcRs.setCommand("select * from COFFEES");
    jdbcRs.setUrl("jdbc:myDriver:myAttribute");
    jdbcRs.setUsername(username);
    jdbcRs.setPassword(password);
    jdbcRs.execute();
    // ...
}

El objeto jdbcRs no contiene datos hasta que especifique una sentencia SQL con el método setCommand, especifique cómo se conecta el objeto JdbcResultSet a la base de datos, y entonces ejecute el método execute.

Todos los constructores de la implementación de referencia asignan los valores predeterminados para las propiedades listadas en la sección Objetos JdbcRowSet Predetermiandos.

Usar la Interfaz RowSetFactory

Con RowSet 1.1, el cual es parte de Java SE 7 y posteriores, puede usar una instancia de RowSetFactory para crear un objeto JdbcRowSet. Por ejemplo, el siguiente trozo de código usa una instancia de la interfaz RowSetFactory para crear el objeto JdbcRowSet, jdbcRs: 

public void createJdbcRowSetWithRowSetFactory(
    String username, String password)
    throws SQLException {

    RowSetFactory myRowSetFactory = null;
    JdbcRowSet jdbcRs = null;
    ResultSet rs = null;
    Statement stmt = null;

    try {
        myRowSetFactory = RowSetProvider.newFactory();
        jdbcRs = myRowSetFactory.createJdbcRowSet();

        jdbcRs.setUrl("jdbc:myDriver:myAttribute");
        jdbcRs.setUsername(username);
        jdbcRs.setPassword(password);

        jdbcRs.setCommand("select * from COFFEES");
        jdbcRs.execute();

        // ...
    }
}

La sentencia siguiente crea el objeto RowSetProvider myRowSetFactory con la implementación por defecto de RowSetFactory, com.sun.rowset.RowSetFactoryImpl: 

myRowSetFactory = RowSetProvider.newFactory();

Alternativamente, si su controlador JDBC tiene su propia implementación de RowSetFactory, puede especificarlo com oun argumento del método newFactory.

Las siguentes sentencias crean el objeto JdbcRowSet jdbcRs y configuran sus propiedades de conexión a la base de datos: 

jdbcRs = myRowSetFactory.createJdbcRowSet();
jdbcRs.setUrl("jdbc:myDriver:myAttribute");
jdbcRs.setUsername(username);
jdbcRs.setPassword(password);

La interfaz RowSetFactory contiene métodos para crear los diferentes tipos de implementaciones de RowSet disponibles en RowSet 1.1 y posteriores:

  • createCachedRowSet
  • createFilteredRowSet
  • createJdbcRowSet
  • createJoinRowSet
  • createWebRowSet

Objetos JdbcRowSet Predeterminados

Cuando crea un objeto JdbcRowSet con el constructor predeterminado, el nuevo objeto JdbcRowSet tendrá las siguientes propiedades:

  • type: ResultSet.TYPE_SCROLL_INSENSITIVE (Tiene un cursor desplazable)
  • concurrency: ResultSet.CONCUR_UPDATABLE (puede ser actualizado)
  • escapeProcessing: true (el controlador hará escapado de proceso; cuando el escapado de proceso está habilitado, El controlador buscará cualquier sintaxis de escape y la traducirá a un código que comprenda la base de datos en particular).
  • maxRows: 0 (sin límite en el número de filas)
  • maxFieldSize: 0 (no hay límite en el número de bytes para un valor de columna; se aplica sólo a columnas que almacenan valores BINARY, VARBINARY, LONGVARBINARY, CHAR, VARCHAR, and LONGVARCHAR)
  • queryTimeout: 0 (no tiene límite de tiempo para el tiempo que se tarda en ejecutar una consulta)
  • showDeleted: false (las filas borradas no son visibles)
  • transactionIsolation: Connection.TRANSACTION_READ_COMMITTED (lee sólo datos que han sido confirmados)
  • typeMap: null (el tipo de mapa asociado con el objeto Connection usado por este objeto RowSet es null)

Lo principal que debe recordar de esta lista es que un JdbcRowSet y todos los demás objetos RowSet son desplazables y actualizables a menos que establezca valores diferentes para esas propiedades.

Establecer Propiedades

La sección Objetos JdbcRowSet Predeterminados lista las propiedades que se establecen por defecto cuando se crea un nuevo objeto JdbcRowSet. Si usa el constructor predeterminado, debe establecer algunas propiedades adicionales antes de que pueda rellenar su objeto JdbcRowSet con datos.

Para obtener sus datos, un objeto JdbcRowSet primero necesita conectarse a una base de datos. Las cuatro propiedades siguientes contienen la información usada en obtener una conexión a una base de datos.

  • username: el nombre que un usuario proporciona a una base de datos como parte del acceso
  • password: la contraseña del usuario de la base de datos
  • url: la URL JDBC para la base de datos a la cual el usuario quiere conectar
  • datasourceName: el nombre usado para recuperar un objeto DataSource que ha sido registrado con un servicio de nombres JNDI

Cuál de estas propiedades establezca dependerá de cómo vaya a realizar la conexión. La forma preferida es usar un objeto DataSource, pero puede que no le resulte práctico registrar un objeto DataSource con un servicio de nombres JNDI, que generalmente lo realiza un administrador del sistema. Por lo tanto, el código de todos los ejemplos usa el mecanismo de DriverManager para obtener una conexión, para lo cual usa la propiedad url y no la propiedad datasourceName.

Otra propiedad que debe establecer es la propiedad command. Esta propiedad es la consulta que determina qué datos contendrá el objeto JdbcRowSet. Por ejemplo, la siguiente línea de código establece la propiedad command con una consulta que produce un objeto ResultSet conteniendo todos los datos en la tabla COFFEES: 

jdbcRs.setCommand("select * from COFFEES");

Después de que ha establecido la propiedad command y las propiedades necesarias para hacer una conexión, está listo para rellenar el objeto jdbcRs con los datos llamando al método execute. 

jdbcRs.execute();

El método execute hace muchas cosas por usted en segundo plano:

  • Realiza una conexión a la base de datos usando los valores que asignó a las propiedades url, username, y password.
  • Ejecuta la consulta que estableció en la propiedad command.
  • Lee los datos del objeto ResultSet resultante y se los pasa al objeto jdbcRs.

Utilizar Objetos JdbcRowSet

Puede actualizar, insertar, y borrar una fila en un objeto JdbcRowSet de la misma forma que actualiza, inserta, y borra una fila en un objeto ResultSet actualizable. Del mismo modo, navega un objeto JdbcRowSet de la misma forma que navega un objeto ResultSet desplazable.

La cadena de cafeterías de The Coffe Break adquirió otra cadena de cafeterías y ahora tiene una base de datos heredada que no soporta el desplazamiento o la actualización de un conjunto de resultados. En otras palabras, cualquier objeto ResultSet producido por esta base de datos heredada no tiene un cursor desplazable, y los datos en éste no pueden ser modificados. Sin embargo, creando un objeto JdbcRowSet rellenado con los datos de un objeto ResultSet, usted puede, en efecto, hacer al objeto ResultSet desplazable y actualizable.

Como se mencionó previamente, un objeto JdbcRowSet es por defecto desplazable y actualizable. Ya que sus contenidos son idénticos a aquellos en el objeto ResultSet, operar en el objeto JdbcRowSet es equivalente a operar con el mismo objeto ResultSet. Y ya que un objeto JdbcRowSet tiene una conexión continua a la base de datos, los cambios que realiza en sus propios datos también se realizan en los datos en la base de datos.

Esta sección cubre los temas siguientes:

  • Navegar los Objetos JdbcRowSet
  • Actualizar Valores de Columna
  • Insertar Filas
  • Borrar Filas

Navegar los Objetos JdbcRowSet

Un objeto ResultSet que no es desplazable puede usar sólo el método next para mover su cursor hacia delante, y puede mover el cursor sólo hacia delantes desde la primera fila a la última fila. Un objeto JdbcRowSet predeterminado, sin embargo, puede usar todos los métodos de movimiento del cursor definidos en la interfaz ResultSet.

Un objeto JdbcRowSet puede llamar al método next, y puede también llamar a cualquiera de los otros métodos de movimiento del cursor de ResultSet. Por ejemplo, las líneas siguientes de código mueven el cursor a la cuarta fila en el objeto jdbcRs y entonces retrocede a la tercera fila: 

jdbcRs.absolute(4);
jdbcRs.previous();

El método previous es análogo al método next en que puede ser usado en un bucle while para recorrer todas las filas en orden. La diferencia es que debe mover el cursor a una posición después de la última fila, y previous mueve el cursor hacia el principio.

Actualizar Valores de Columna

Puede actualizar datos en un objeto JdbcRowSet de la misma forma que puede actualizar datos en un objeto ResultSet.

Asuma que el propietario de Coffee Break quiere subir el precio por una libra de café Espresso. Si el propietario sabe que Espresso está en la tercera fila del objeto jdbcRs, el código para hacer esto podría parecerse a lo siguiente: 

jdbcRs.absolute(3);
jdbcRs.updateFloat("PRICE", 10.99f);
jdbcRs.updateRow();

El código mueve el cursor a la tercera fila y cambia el valor de la columna PRICE a 10.99, y después actualiza la base de datos con el nuevo precio.

Llamar al método updateRow actualiza la base de datos porque jdbcRs ha mantenido su conexión a la base de datos. Para objetos RowSet desconectados, la situación es diferente.

Insertar Filas

Si el propietario de la cadena Coffee Break quiere agregar uno o más cafés a los que ofrece, el propietario necesitará agregar una fila a la tabla COFFEES por cada nuevo café, como se hace en el siguiente fragmento de código de JdbcRowSetSample. Tenga en cuenta que debido a que el objeto jdbcRs siempre está conectado a la base de datos, insertar una fila en un objeto JdbcRowSet es lo mismo que insertar una fila en un objeto ResultSet : usted mueve el cursor a la fila de inserción, usa el método actualizador apropiado para establecer un valor para cada columna, y llama al método insertRow: 

jdbcRs.moveToInsertRow();
jdbcRs.updateString("COF_NAME", "HouseBlend");
jdbcRs.updateInt("SUP_ID", 49);
jdbcRs.updateFloat("PRICE", 7.99f);
jdbcRs.updateInt("SALES", 0);
jdbcRs.updateInt("TOTAL", 0);
jdbcRs.insertRow();

jdbcRs.moveToInsertRow();
jdbcRs.updateString("COF_NAME", "HouseDecaf");
jdbcRs.updateInt("SUP_ID", 49);
jdbcRs.updateFloat("PRICE", 8.99f);
jdbcRs.updateInt("SALES", 0);
jdbcRs.updateInt("TOTAL", 0);
jdbcRs.insertRow();

Cuando llama al método insertRow, la nueva fila se inserta en el objeto jdbcRs y es también insertada en la base de datos. El fragmento de código precedente pasa por este proceso dos veces, por lo que se insertan dos filas nuevas en el objeto jdbcRs y en la base de datos.

Borrar Filas

Como ocurre con la actualización de datos y la inserción de una nueva fila, eliminar una fila es igual para un objeto JdbcRowSet que para un objeto ResultSet. El propietario quiere dejar de vender café descafeinado French Roast, que es la última fila del objeto jdbcRs. En las siguientes líneas de código, la primera línea mueve el cursor a la última fila, y la segunda línea borra la última fila del objeto jdbcRs y de la base de datos: 

jdbcRs.last();
jdbcRs.deleteRow();

Ejemplo de Código

El ejemplo JdbcRowSetSample hace lo siguiente:

  • Crea un nuevo objeto JdbcRowSet inicializado con el objeto ResultSet que fué producido por la ejecución de una consulta que recupera todas las filas en la tabla COFFEES
  • Mueve el cursor a la tercera fila de la tabla COFFEES y actualiza la columna PRICE en esa fila
  • Inserta dos nuevas filas, una para HouseBlend y otra para HouseDecaf
  • Mueve el cursor a la última fila y la borra
