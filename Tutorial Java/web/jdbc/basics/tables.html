Tabla COFFEES

La tabla COFFEES almacena información sobre los cafés disponibles para vender en The Coffee Break:

Tabla COFFEES
COF_NAME SUP_ID PRICE SALES TOTAL
Colombian 101 7.99 0 0
French_Roast 49 8.99 0 0
Espresso 150 9.99 0 0
Colombian_Decaf 101 8.99 0 0
French_Roast_Decaf 49 9.99 0 0

A continuación se describe cada una de las columnas en la tabla COFFEES:

  • COF_NAME: Almacena el nombre del café. Contiene valores con un tipo SQL de VARCHAR con una longitud máxima de 32 caracteres. Ya que los nombres son diferentes para cada tipo de café vendido, el nombre identifica de manera única un café en particular y sirve como la clave primaria.
  • SUP_ID: Almacena un número que identifica el proveedor del café. Contiene valores con un tipo SQL de INTEGER. Es definido como clave foránea que hace referencia a la columna SUP_ID en la tabla SUPPLIERS. Como consecuencia, el DBMS forzará que cada valor en esta columna coincida con uno de los valores en la columna correspondiente en la tabla SUPPLIERS.
  • PRICE: Almacena el coste del café por libra. Contiene valores con el tipo SQL de FLOAT porque neceista contener valores con dígitos decimales. (Tenga en cuenta que los valores monetarios normalmente se almacenarían en un tipo SQL DECIMAL o NUMERIC, pero debido a las diferencias entre DBMSs y para evitar incompatibilidades con versiones más tempranas de JDBC, el tutorial usa el tipo más estándar FLOAT).
  • SALES: Almacen el número de libras de café vendidas durante la semana actual. Contiene valores con un tipo SQL de INTEGER.
  • TOTAL: Almacena el número de libras de café vendidos hasta la fecha. Contiene valores con un tipo SQL de INTEGER.

Tabla SUPPLIERS

49 Superior Coffee 1 Party Place Mendocino CA 95460

A continuación se describe cada una de las columnas en la tabla SUPPLIERS:

  • SUP_ID: Almacena un número que identifica el proveedor de café. Contiene valores con un tipo SQL de INTEGER. Es la clave primaria en esta tabla.
  • SUP_NAME: Almacena el nombre del proveedor de café.

Tabla DATA_REPOSITORY

La tabla DATA_REPOSITORY almacena las URLs que hacen referencia a los documentos y otros datos de interés para The Coffe Break. El script populate_tables.sql no agrega ningún dato a esta tabla. A continuación se describe cada una de las columnas en esta tabla:

  • DOCUMENT_NAME: Almacena una cadena que identifica la URL.
  • URL: Almacena una URL.

Crear Tablas

Puede crear tablas con Apache Ant o la IPA de JDBC.

Crear Tablas con Apache Ant

Para crear las tablas usadas con el código de ejemplo del tutorial, ejecute el comando siguiente en el diretorio <directorio tutorail JDBC>: 

ant setup

Este comando ejecuta varios objetivos de Ant, incluyendo el siguiente, build-tables (desde el fichero build.xml): 

<target name="build-tables"
  description="Create database tables">
  <sql
    driver="${DB.DRIVER}"
    url="${DB.URL}"
    userid="${DB.USER}"
    password="${DB.PASSWORD}"
    classpathref="CLASSPATH"
    delimiter="${DB.DELIMITER}"
    autocommit="false" onerror="abort">
    <transaction src=
  "./sql/${DB.VENDOR}/create-tables.sql"/>
  </sql>
</target>

El ejemplo especifica valores para los siguentes parámetros sql de la tarea de Ant:

Parámetros de Ant
Parámetro Descripción
driver Nombre de clase completamente cualificado de su controlador JDBC. Este ejemplo usa org.apache.derby.jdbc.EmbeddedDriver para Java DB y com.mysql.jdbc.Driver para MySQL Connector/J.
url La URL de conexión de base de datos que su controlador JDBC usa para conectar a la base de datos.
userid Nombre de un usuario válido en su DBMS.
password Contraseña del usuario especificado en userid
classpathref Nombre de ruta completo del fichero JAR que contiene la clase especifica en driver
delimiter Cadena o carácter que separa las sentencias SQL. Este ejemplo usa el punto y coma (;).
autocommit Valor booleano; si se establece a false, todas las sentencias SQL se ejecutan como una transacción.
onerror Acción a realizar cuando una sentencia falla; los valores posibles son continue, stop, y abort. El valor abort especifica que si ocurre un error, se aborte la transacción.

El ejemplo almacena los valores de estos parámetros en un fichero separado. El fichero de construcción build.xml recupera estos valores con la tarea import: 

<import file="${ANTPROPERTIES}"/>

El elemento transaction especifica un ficheor que contiene sentencias SQL para ejecutar. El fichero create-tables.sql contiene sentencias SQL que crean todas las tablas descritas en esta página. Por ejemplo, el siguiente extracto de este fichero crea las tablas SUPPLIERS y COFFEES: 

create table SUPPLIERS
    (SUP_ID integer NOT NULL,
    SUP_NAME varchar(40) NOT NULL,
    STREET varchar(40) NOT NULL,
    CITY varchar(20) NOT NULL,
    STATE char(2) NOT NULL,
    ZIP char(5),
    PRIMARY KEY (SUP_ID));

create table COFFEES
    (COF_NAME varchar(32) NOT NULL,
    SUP_ID int NOT NULL,
    PRICE numeric(10,2) NOT NULL,
    SALES integer NOT NULL,
    TOTAL integer NOT NULL,
    PRIMARY KEY (COF_NAME),
    FOREIGN KEY (SUP_ID)
        REFERENCES SUPPLIERS (SUP_ID));

Nota: El fichero build.xml contiene otro objetivo llamado drop-tables que borra las tablas usadas por el tutorial. El objetivo setup ejecuta drop-tables antes de ejecutar el objetivo build-tables.

Crear Tablas con la IPA de JDBC

El método siguiente, SuppliersTable.createTable, crea la tabla SUPPLIERS: 

public void createTable() throws SQLException {
    String createString =
        "create table " + dbName +
        ".SUPPLIERS " +
        "(SUP_ID integer NOT NULL, " +
        "SUP_NAME varchar(40) NOT NULL, " +
        "STREET varchar(40) NOT NULL, " +
        "CITY varchar(20) NOT NULL, " +
        "STATE char(2) NOT NULL, " +
        "ZIP char(5), " +
        "PRIMARY KEY (SUP_ID))";

    Statement stmt = null;
    try {
        stmt = con.createStatement();
        stmt.executeUpdate(createString);
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}

El método siguiente, CoffeesTable.createTable, crea la tabla COFFEES: 

  public void createTable() throws SQLException {
    String createString =
        "create table " + dbName +
        ".COFFEES " +
        "(COF_NAME varchar(32) NOT NULL, " +
        "SUP_ID int NOT NULL, " +
        "PRICE float NOT NULL, " +
        "SALES integer NOT NULL, " +
        "TOTAL integer NOT NULL, " +
        "PRIMARY KEY (COF_NAME), " +
        "FOREIGN KEY (SUP_ID) REFERENCES " +
        dbName + ".SUPPLIERS (SUP_ID))";

    Statement stmt = null;
    try {
        stmt = con.createStatement();
        stmt.executeUpdate(createString);
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}

En ambos métodos, con es un objeto Connection y dbName es el nombre de la base de datos en la cual está creando la tabla.

Para ejecutar la consulta SQL, tal como esa especificad por la String createString, use un objeto Statement. Para crear un objeto Statement, llame al método Connection.createStatement desde un objeto Connection existente. Para ejecutar una consulta SQL, llame al método Statement.executeUpdate.

Todos los objetos Statement se cierran cuando la conexión que los creó se cierra. Sin embargo, es una buena práctica de codificación cerrar explícitamente los objetos Statement tan pronto como usted haya terminado con ellos. Esto permite que cualesquiera recursos externos que la sentencia está usando se liberen inmediatamente. Cierre una sentencia llamando al método Statement.close. Coloque esta sentencia en un finally para asegurar que se cierra incluso si el flujo normal del programa se interrumpe debido a que se lanze una excepción (tal como SQLException).

Nota: Debe crear la tabla SUPPLIERS antes de COFFEES porque COFFEES contiene una clave foránea, SUP_ID que hace referencia a SUPPLIERS.

Rellenar Tablas con la IPA de JDBC

El método siguiente, SuppliersTable.populateTable, inserta los datos en una tabla: 

public void populateTable() throws SQLException {

    Statement stmt = null;
    try {
        stmt = con.createStatement();
        stmt.executeUpdate(
            "insert into " + dbName +
            ".SUPPLIERS " +
            "values(49, 'Superior Coffee', " +
            "'1 Party Place', " +
            "'Mendocino', 'CA', '95460')");

        stmt.executeUpdate(
            "insert into " + dbName +
            ".SUPPLIERS " +
            "values(101, 'Acme, Inc.', " +
            "'99 Market Street', " +
            "'Groundsville', 'CA', '95199')");

        stmt.executeUpdate(
            "insert into " + dbName +
            ".SUPPLIERS " +
            "values(150, " +
            "'The High Ground', " +
            "'100 Coffee Lane', " +
            "'Meadows', 'CA', '93966')");
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}
