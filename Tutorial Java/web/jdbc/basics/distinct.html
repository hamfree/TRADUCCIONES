Usando el Tipo de Datos DISTINCT

Nota: MySQL y Java DB actualmente no soportan el tipo de dato SQL DISTINCT. Por consiguiente, no hay un ejemplo disponible en el tutorial de JDBC para demostrar las características descritas en esta sección.

El tipo de datos DISTINCT se comporta diferentemente de los otros tipos de datos avanzados SQL . Siendo un tipo definido por el usuario que está basado en uno de los tipos integrados ya existentes, no tiene interfaz como su mapeo en el lenguaje de programación Java. En su lugar, la asignación estándar para un tipo de datos DISTINCT es el tipo Java al cual su tipo de datos SQL subyacente se asigna.

Para ilustrarlo, cree un tipo de dato DISTINCT y luego vea cómo recuperarlo, establecerlo, o actualizarlo. Suponga que siempre usa una abreviatura de dos letras para un estado y quiere crear un tipo de dato DISTINCT que se use para estas abreviaturas. Podría definir su nuevo tipo de dato DISTINCT con la siguiente sentencia SQL: 

CREATE TYPE STATE AS CHAR(2);

Algunas bases de datos usan una sintaxis alternativa para crear un tipo de dato DISTINCT, el cual se muestra en la siguiente línea de código: Some databases use an alternate syntax for creating a DISTINCT data type, which is shown in the following line of code: 

CREATE DISTINCT TYPE STATE AS CHAR(2);

Si una sintaxis no funciona, puede intentar la otra. Alternativamente, puede comprobar la documentación para su controlador para ver la sintaxis exacta que se espera.

Estas sentencias crean un tipo de datos nuevo, STATE, el cual puede usarse como un valor de columna o como el valor para un atributo de un tipo estructurado de SQL. Ya que un valor del tipo STATE es en realidad un valor que es dos tipos CHAR, use el mismo método para recuperarlo que usaría para recuperar un valor CHAR, es decir, getString. Por ejemplo, asumiendo que la cuarta columna de ResultSet rs almacena valores del tipo STATE, la línea siguiente de código recupera su valor: 

String state = rs.getString(4);

Del mismo modo, usaría el método setString para almacenar un valor STATE en la base de datos y el método updateString para modificar su valor.
