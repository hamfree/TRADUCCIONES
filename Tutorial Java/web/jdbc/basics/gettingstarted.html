Empezando

El código de ejemplo que viene con este tutorial crea una base de datos que usa un propietario de una pequeña casa de café llamada The Coffee Break, donde los granos de café se venden por libras y el café elaborado se vende por tazas.

Los pasos siguientes configuran un entorno de desarrollo JDBC con el cual puede compilary ejecutar los ejemplos del tutorial:

  1. Instale la última versión del KDJ de Java en su computadora
  2. Instale su sistema gestor de base de datos (DBMS) si es necesario
  3. Instale un controlador JDBC del proveedor de su base de datos
  4. Instale Apache Ant
  5. Instale Apache Xalan
  6. Descargue el código de ejemplo
  7. Modifique el fichero build.xml
  8. Modifique el fichero de propiedades del tutorial
  9. Compile y empaquete los ejemplos
  10. Cree las bases de datos, tablas, y rellene las tablas
  11. Ejecute los ejemplos

Instale la última versión del KDJ de Java en su computadora

Instale la última versión del KDJ de Java en su computadora.

Asegúrese que la ruta completa del directorio del directorio bin del KDJ de Java está en su variable de entorno PATH de forma que pueda ejecutar el compilador de Java y el lanzador de aplicaciones Java desde cualquier directorio.

Instale su sistema gestor de base de datos (DBMS) si es necesario

Puede usar Java DB, la cual viene con la última versión del KDJ de Java. Este tutorial ha sido comprobado para los siguientes DBMS:

  • Java DB
  • MySQL

Tenga en cuenta que si está utilizando otro DBMS, es posible que deba modificar el código de los ejemplos del tutorial.

Instale un controlador JDBC del proveedor de su base de datos

Si está usando Java DB, este ya vien con un controlador JDBC. Si está usando MySQL, instale la última versión del Connector/J.

Contacte con el proveedor de su base de datos para obtener un controlador JDBC para su DBMS.

Hay muchas implementaciones posibles de los controladores JDBC. Estas implementaciones se categorizan como sigue:

  • Tipo 1: Los controladores que implementan la IPA de JDBC como un mapeo a otra IPA de acceso a datos, tal como ODBC (Conectividad de Base de Datos Abierta, Open Database Connectivity, en inglés). Los controladores de este tipo son dependientes generalmente de una librería nativa, lo cual limita su portabilidad. EL Puente JDBC-ODBC es un ejemplo de controlador de Tipo 1.

    Nota: El Puente JDBC-ODBC debería ser considerado una solución transitoria. No está admitido por Oracle. Considere usarlo sólo si su DBMS no ofrece un controlador JDBC hecho en Java.

  • Tipo 2: Los controladores que están escritos parcialmente en el lenguaje de programación Java y parcialmente en código nativo. Estos controladores usan una librería de cliente nativa específica para la fuente de datos a la cual se conectan. De nuevo, debido al código nativo, su portabilidad es limitada. El controlador del lado del cliente OCI de Oracle (Oracle Call Interface) es un ejemplo de controlador de Tipo 2.

  • Tipo 3: Los controladores que usan un cliente puro de Java y se comunican con un servidor middleware usando un protocolo independiente de la base de datos. El servidor middleware entonces comunica las peticiones del cliente a la fuente de datos.

  • Tipo 4: Los controladores que son puros Java e implementan el protocolo de red para una fuente de datos específica. El cliente se conecta directamente a la fuente de datos.

Comprueba que tipos de controladores vienen con su DBMS. Java DB viene con controladores de Tipo 4, un controlador Embebido y un controlador Cliente de Red. El Connector/J para MySQL es un controlador de Tipo 4.

Instalar un controlador JDBC generalmente consiste en copiar el controlador a su computadoras, después agregar la ubicación de éste a su ruta de clases. Además, muchos controladores JDBC distintos de los controladores de Tpo 4 requieren que instale una IPA del lado cliente. No se necesita usualmente ninguna otra configuración especial.

Instale Apache Ant

Estos pasos usan Apache Ant, una herramienta basada en Java, para construir, compilar, y ejecutar los ejemplos del tutorial de JDBC. Vaya al siguiente enlace para descargar Apache Ant:

http://ant.apache.org/

Asegúrese que el fichero ejecutable de Apache Ant está en su variable de entorno PATH de forma que puede ejecutarlo desde cualquier directorio.

Instale Apache Xalan

El ejemplo RSSFeedsTable.java, el cual se describe en Utilizando Objetos SQLXML, requiere Apacha Xalan si su DBMS es Java DB. El ejemplo usa Apacha Xalan-Java. Vaya al siguiente enlace para descargarlo:

http://xml.apache.org/xalan-j/

Modifique el fichero build.xml

El fichero build.xml es el fichero de construcción que Apache Ant usa para compilar y ejecutar los ejemplos de JDBC. Los ficheros properties/javadb-build-properties.xml y properties/mysql-build-properties.xml contienen propiedades adicionales de Apache Ant requeridas para Java DB y MySQL, respectivamente. Los ficheros properties/javadb-sample-properties.xml y properties/mysql-sample-properties.xml contienen propiedades requeridas por el ejemplo.

Modifique estos ficheros XML como sigue:

Modifique build.xml

En el fichero build.xml, modifique la propiedad ANTPROPERTIES que se refiera a properties/javadb-build-properties.xml o properties/mysql-build-properties.xml, dependiendo de su DBMS. Por ejemplo, si está usando Java DB, su fichero build.xml contendría esto: 

<property
  name="ANTPROPERTIES"
  value="properties/javadb-build-properties.xml"/>

  <import file="${ANTPROPERTIES}"/>

De forma similar, si está usando MySQL, su fichero build.xml contendría esto: 

<property
  name="ANTPROPERTIES"
  value="properties/mysql-build-properties.xml"/>

  <import file="${ANTPROPERTIES}"/>

Modifique los ficheros de propiedades específicos de base de datos

En el fichero properties/javadb-build-properties.xml o el fichero properties/mysql-build-properties.xml (dependiendo de su DBMS), modifique las propiedades siguientes, como se describen en la tabla siguiente:

propiedades de construcción específicas del DBMS
Propiedad Descripción
JAVAC La ruta completa de su compilador de Java, javac
JAVA La ruta completa de su ejecutable en tiempo de ejecución Java, java
PROPERTIESFILE El nombre del fichero de propiedades, o properties/javadb-sample-properties.xml o properties/mysql-sample-properties.xml
MYSQLDRIVER La ruta completa de su controlador MySQL. Para Connector/J, esta es típicamente <directorio instalación Connector/J>/mysql-connector-java-núm-version.jar .
JAVADBDRIVER La ruta completa de su controlador Java DB. Esta es típicamente <directorio instalación Java DB>/lib/derby.jar.
XALANDIRECTORY La ruta completa del directorio que contiene Apache Xalan.
CLASSPATH La ruta de clases que el tutorial JDBC usa. No necesita cambiar este valor.
XALAN La ruta completa del fichero xalan.jar.
DB.VENDOR Un valor que es derby o mysql dependiendo de si está usando Java DB o MySQL, respectivamente. El tutorial usa este valor para construir la URL requerida para conectar al DBMS e identificar código específico del DBMS y sentencias SQL.
DB.DRIVER El nombre de clase completamente cualificado del controlador JDBC. Para Java DB, este es org.apache.derby.jdbc.EmbeddedDriver. Para MySQL, este es com.mysql.jdbc.Driver.
DB.HOST El nombre de anfitrión de la computadora que aloja su DBMS.
DB.PORT El número de puerto de la computadora que aloja su DBMS.
DB.SID El nombre de la base de datos que el tutorial crea y usa.
DB.URL.NEWDATABASE La URL de conexión usada para conectar a su DBMS cuando crea una base de datos nueva. No necesita cambiar este valor.
DB.URL La URL de conexión usada para conectar a su DBMS. No necesita cambiar este valor.
DB.USER El nombre del usuario que tiene acceso para crear bases de datos en el DBMS.
DB.PASSWORD La contraseña del ususario especificado en DB.USER.
DB.DELIMITER El carácter usado para separar las sentencias SQL. No cambie este valor. Debería ser el carácter de punto y coma (;).

Modifique el fichero de propiedades del tutorial

Los ejemplos del tutorial usan los valores en el fichero properties/javadb-sample-properties.xml o en el fichero properties/mysql-sample-properties.xml (dependiendo de su DBMS) para conectar al DBMS e inicializar las bases de datos y tablas, como se describe en la tabla siguiente:

propiedades específicas de DBMS del ejemplo
Propiedad Descripción
dbms Un valor que es derby o mysql dependiendo de si usted está usando Java DB o MySQL, respectivamente. El tutorial usa este valor para construir la URL requerida para conectar al DBMS e identificar código específico del DBMS y sentencias SQL.
jar_file La ruta completa del fichero JAR que contiene todos los ficheros de clases de este tutorial.
driver El nombre de clase completamente cualificado del controlador JDBC. Para Java DB, este es org.apache.derby.jdbc.EmbeddedDriver. Para MySQL, este es com.mysql.jdbc.Driver.
database_name El nombre de la base de datos que el tutorial crea y usa.
user_name El nombre del usuario que tiene acceso a crear bases de datos en el DBMS.
password La contraseña del usuario especificado en user_name.
server_name El nombre de host de la computadora que aloja su DBMS.
port_number El número de puerto de la computadora que aloja su DBMS.

Nota: Por simplicidad en demostrar la IPA de JDBC, el código de ejemplo del tutorial JDBC no realiza las técnicas de gestión de contraseñas que un sistema normalmente desplegado usa. En un entorno de producción, puede seguir las pautas de administración de contraseñas de la Base de Datos de Oracle y deshabilitar cualquier cuenta de ejemplo. Vea la sección Protección de Contraseñas en el Diseño de Aplicaciones en Gestión de la Seguridad para Desarrolladores de Aplicaciones en la Guía de Seguridad de la Base de Datos Oracle para conocer las pautas de administración de contraseñas y otras recomendaciones de seguridad.

Compile y empaquete los ejemplos

En el prompt de comandos, cambie el directorio actual al <Directorio del tutorial JDBC>. Desde este directorio, ejecute el comando siguiente para compilar los ejemplos y empaquetarlos en un fichero jar: 

ant jar

Cree las bases de datos, tablas, y rellene las tablas

Si está usando MySQL, entonces ejecute el comando siguiente para crear una base de datos: 

ant create-mysql-database

Nota: No hay objetivo Ant correspondiente que exista en el fichero build.xml que cree un base de datos para Java DB. La URL de base de datos para Java DB, la cual se usa para establecer una conexión de base de datos, incluye la opción de crear la base de datos (si no existe aún). Vea Estableciendo una Conexión para más información.

Si usted está usando Java DB o MySQL, entonces desde el mismo directorio, ejecute el comando siguiente para borrar las tablas de ejemplo existentes de la base de datos, recrear las tablas, y rellenarlas. Para Java DB, este comando también crea la base de datos si aún no existe: 

ant setup

Nota: Debe ejecutar el comando ant setup cada vez antes de ejecutar una de las clases Java en el ejemplo. Muchos de estos ejemplos esperan datos específicos en los contenidos de las tablas de la base de datos de ejemplo.

Ejecute los ejemplos

Cada objetivo en el fichero build.xml se corresponde a una clase Java o script SQL en los ejemplos de JDBC. La tabla siguiente lista los objetos en el fichero build.xml, cuya clase o script cada objetivo ejecuta, y otras clases o ficheros que cada objetivo requiere:

Objetivos en build.xml
Objetivo Ant Clase o Script SQL Otras Clases o Ficheros Requeridos
javadb-create-procedure javadb/create-procedures.sql; vea el fichero build.xml para ver otras sentencias SQL que son ejecutadas No hay otros archivos necesarios
mysql-create-procedure mysql/create-procedures.sql. No hay otros archivos necesarios
run JDBCTutorialUtilities No hay otras clases obligatorias
runct CoffeesTable JDBCTutorialUtilities
runst SuppliersTable JDBCTutorialUtilities
runjrs JdbcRowSetSample JDBCTutorialUtilities
runcrs CachedRowSetSample, ExampleRowSetListener JDBCTutorialUtilities
runjoin JoinSample JDBCTutorialUtilities
runfrs FilteredRowSetSample JDBCTutorialUtilities, CityFilter, StateFilter
runwrs WebRowSetSample JDBCTutorialUtilities
runclob ClobSample JDBCTutorialUtilities, txt/colombian-description.txt

rundl DatalinkSample JDBCTutorialUtilities
runspjavadb StoredProcedureJavaDBSample JDBCTutorialUtilities, SuppliersTable, CoffeesTable

ant runct
