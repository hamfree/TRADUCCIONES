Utilizando Objetos RowId

Nota: MySQL y Java DB actualmente no soporta la interfaz JDBC RowId. Como consecuencia, no hay ejemplo disponible en el tutorial JDBC para demostrar las características en esta sección.

Un objeto RowId representa una dirección a una fila en una tabla de la base de datos. Dese cuenta, sin embargo, que el tipo ROWID no es un tipo SQL estándar. Los valores ROWID pueden ser útiles porque son típicamente la forma más rápida de acceder a una única fila e identifica de forma única las filas en una tabla. Sin embargo, no debe usar un valor ROWID como la clave primaria de una tabla. Por ejemplo, si borra una fila particular de una tabla, una base de datos podría reasignar su valor ROWID a una fila insertada después.

Se cubren los temas siguientes:

  • Recuperar Objetos RowId
  • Utilizar Objetos RowId
  • Duración de la Validez de RowId

Recuperar Objetos RowId

Recupera un objeto java.sql.RowId llamando a los métodos obtenedores definidos en los interfaces ResultSet y CallableStatement. El objeto RowId que se devuelve es un objeto inmutable que puede usar para referencias posteriores como identificador único para una fila. Lo siguiente es un ejemplo de llamada al método ResultSet.getRowId: 

java.sql.RowId rowId_1 = rs.getRowId(1);

Utilizar Objetos RowId

Puede establecer un objeto RowId como un parámetro en un objeto parametrizado PreparedStatement: 

Connection conn = ds.getConnection(username, password);
PreparedStatement ps = conn.prepareStatement(
    "INSERT INTO BOOKLIST" +
    "(ID, AUTHOR, TITLE, ISBN) " +
    "VALUES (?, ?, ?, ?)");
ps.setRowId(1, rowId_1);

Puede también actualizar una columna con un objeto RowId específico en un objeto ResultSet actualizable: 

ResultSet rs = ...
rs.next();
rs.updateRowId(1, rowId_1);

Un objeto RowId es típicamente no portable entre fuentes de datos y debería ser considerado como específico a la fuente de datos cuando se usa el método set o update en un los objetos PreparedStatement y ResultSet, respectivamente. Por lo tanto, no es aconsejable obtener un objeto RowId desde un objeto ResultSet con una conexión a una fuente de datos y después intentar usar el mismo objeto RowId en un objeto ResultSet no relacionado con una conexión a una fuente de datos diferente.

Duración de la Validez de RowId

Un objeto RowId es válido siempre que la fila identificada no se elimine y la vida útil del objeto RowId esté dentro de los límites de la vida útil especificada por la fuente de datos para el objeto RowId.

Para determinar la vida útil de los objetos RowId de su base de datos o fuente de datos, llame al método DatabaseMetaData.getRowIdLifetime. Devuelve un valor de un tipo enumerado RowIdLifetime. El siguiente método JDBCTutorialUtilities.rowIdLifeTime devuelve la vida útil de los objetos RowId: 

public static void rowIdLifetime(Connection conn)
    throws SQLException {

    DatabaseMetaData dbMetaData = conn.getMetaData();
    RowIdLifetime lifetime = dbMetaData.getRowIdLifetime();

    switch (lifetime) {
        case ROWID_UNSUPPORTED:
            System.out.println("ROWID type not supported");
            break;

        case ROWID_VALID_FOREVER:
            System.out.println("ROWID has unlimited lifetime");
            break;

        case ROWID_VALID_OTHER:
            System.out.println("ROWID has indeterminate lifetime");
            break;

        case ROWID_VALID_SESSION:
            System.out.println(
                "ROWID type has lifetime that " +
                "is valid for at least the " +
                "containing session");
            break;

        case ROWID_VALID_TRANSACTION:
            System.out.println(
                "ROWID type has lifetime that " +
                "is valid for at least the " +
                "containing transaction");
            break;
    }
}
