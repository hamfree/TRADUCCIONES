Utilizando Objetos Datalink

Un valor DATALINK referencia un recurso que está fuera de la fuente de datos subyacente a través de una URL, Una URL, localizador uniforme de recursos, es un puntero a un recurso en la Red Mundial. Un recurso puede ser algo tan simple como un fichero o un directorio, o puede ser una referencia a un objeto más complicado, tal como una consulta a una base de datos a un motor de búsqueda.

Se cubren los temas siguientes:

  • Almacenar Referencias a Datos Externos
  • Recuperar Referencias a Datos Externos

Almacenar Referencias a Datos Externos

Use el método PreparedStatement.setURL para especificar un objeto java.net.URL a una sentencia preparada. En casos donde el tipo de URL a ser establecida no es soportada por la plataforma Java, almacene la URL con el método setString.

Por ejemplo, suponga que al propietario de The Coffee Break le gustaría almacenar una lista de URLs importantes en una tabla de base de datos. El ejemplo siguiente, DatalinkSample.addURLRow agrega una fila de datos a la tabla DATA_REPOSITORY. La fila consiste de una cadena identificando la URL, DOCUMENT_NAME y la URL en sí mismo, URL: 

public void addURLRow(String description, String url)
    throws SQLException {

    PreparedStatement pstmt = null;

    try {
        pstmt = this.con.prepareStatement(
            "INSERT INTO data_repository" +
            "(document_name,url) VALUES (?,?)");

        pstmt.setString(1, description);
        pstmt.setURL(2,new URL(url));
        pstmt.execute();
    } catch (SQLException sqlex) {
        JDBCTutorialUtilities.printSQLException(sqlex);
    } catch (Exception ex) {
        System.out.println("Unexpected exception");
        ex.printStackTrace();
    } finally {
        if (pstmt != null) {
            pstmt.close();
        }
    }
}

Recuperar Referencias a Datos Externos

Use el método ResultSet.getURL para recuperar una referencia a datos externos como un objeto java.net.URL. En casos donde el tipo de URL devuelto por los métodos getObject o getURL no está admitido por la plataforma Java, recuperar la URL como un objeto String llamando al método getString.

El ejemplo siguiente, DatalinkSample.viewTable, visualiza los contenidos de todas las URLs almacenadas en la tabla DATA_REPOSITORY: 

public static void viewTable(Connection con, Proxy proxy)
    throws SQLException, IOException {

    Statement stmt = null;
    String query =
      "SELECT document_name, url " +
      "FROM data_repository";

    try {
        stmt = con.createStatement();
        ResultSet rs = stmt.executeQuery(query);

        if ( rs.next() )  {
            String documentName = null;
            java.net.URL url = null;

            documentName = rs.getString(1);

            // Retrieve the value as a URL object.
            url = rs.getURL(2);

            if (url != null) {

                // Retrieve the contents
                // from the URL
          
                URLConnection myURLConnection =
                    url.openConnection(proxy);
                BufferedReader bReader =
                    new BufferedReader(
                        new InputStreamReader(
                            myURLConnection.
                                getInputStream()));

                System.out.println("Document name: " + documentName);

                String pageContent = null;

                while ((pageContent = bReader.readLine()) != null ) {
                    // Print the URL contents
                    System.out.println(pageContent);
                }
            } else {
                System.out.println("URL is null");
            }
        }
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    } catch(IOException ioEx) {
        System.out.println("IOException caught: " + ioEx.toString());
    } catch (Exception ex) {
        System.out.println("Unexpected exception");
        ex.printStackTrace();
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}

El ejemplo DatalinkSample almacena la URL de Oracle, http://www.oracle.com en la tabla DATA_REPOSITORY. Después, visualiza los contenidos de todos los documentos referidos por las URLs almacenadas en DATA_REPOSITORY, el cual incluye la página de inicio de Oracle, http://www.oracle.com.

El ejemplo recupera la URL desde el conjunto de resultados como un objeto java.net.URL con la sentencia siguiente: 

url = rs.getURL(2);

El ejemplo accede a los datos referidos por el objeto URL con las sentencias siguientes: 

URLConnection myURLConnection = url.openConnection(proxy);
BufferedReader bReader = new BufferedReader(
    new InputStreamReader(
        myURLConnection.getInputStream()));

System.out.println("Document name: " + documentName);

String pageContent = null;

while ((pageContent = bReader.readLine()) != null ) {
    // Print the URL contents
    System.out.println(pageContent);
}

El método URLConnection.openConnection puede no tomar argumentos, el cual significa que la URLConnection representa una conexión directa a Internet. Si requiere un servidor proxy para conectar a Internet, el método openConnection acepta un objeto java.net.Proxy como un argumento. Las sentencias siguientes demuestran cómo crear un proxy HTTP con el nombre de servidor www-proxy.example.com y el número de puerto 80: 

Proxy myProxy;
InetSocketAddress myProxyServer;
myProxyServer = new InetSocketAddress("www-proxy.example.com", 80);
myProxy = new Proxy(Proxy.Type.HTTP, myProxyServer);
