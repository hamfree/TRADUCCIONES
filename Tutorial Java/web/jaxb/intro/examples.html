Ejemplos de JAXB

Las siguientes secciones describen cómo usar las aplicaciones de ejemplo que están incluídas en el paquete JAXB RI. El paquete JAXB RI está disponible en http://jaxb.java.net. Descargue e instale el paquete JAXB RI. Los ejemplos se localizan en el directorio jaxb-ri-install/samples/. Estos ejemplos demuestran y se basan en características y conceptos clave de JAXB. Siga estos procedimientos en el orden presentado.

Después de leer esta sección, debería sentirse lo suficientemente cómodo con JAXB como para poder:

  • Generar clases Java de JAXB desde un esquema XML
  • Usar las clases JAXB derivadas de esquemas para separar y agrupar contenido XML en una aplicación Java
  • Crear un árbol de contenido de JAva usando las clases JAXB derivadas del esquema
  • Validar el contenido XML durante la separación y en tiempo de ejecución
  • Personalizar los vínculos esquema a Java de JAXB

Este documento describe tres conjuntos de ejemplos:

  • Los ejemplos básicos (Modificar Agrupar, Desagrupar Validar) que demuestran los conceptos básicos de JAXB tales como el desagrupamiento, agrupamiento, y la validación del contenido XML usando configuraciones y vínculos predeterminados.
  • Los ejemplos de personalización (Personalizar En Línea, Convertidor de Tipos de Datos, Personalización Externa) que demuestran varias formas de personalizar la vinculación predeterminada de esquemas XML a objetos Java.
  • Los ejemplo java a esquema muestran cómo usar anotaciones para asignar clases de Java a un esquema XML.
Nota: Los ejemplos básicos y de personalización se basan en un escenario Orden de Compra. Cada uno usa un documento XML, po.xml, escrito contra un esquema XML, po.xsd. Estos documentos se derivan de W3C XML Schema Part 0: Primer, editado por David C. Fallside.

Los directorios de los ejemplos básicos y personalizados contienen varios ficheros base:

  • po.xsd es el esquema XML que se usa como entrada al compilador de vinculación JAXB, y desde el cual se generarán las clases Java de JAXB derivadas del esquema. Para los ejemplos Personalizar en Línea y Convertidor de Tipos de Datos, este fichero contiene personalizaciones de vinculación en línea.
  • po.xml es el fichero XML Orden de Compra que contiene el contenido XML de ejemplo, y es el fichero que se desagrupa en un árbol de contenido de Java en cada ejemplo. Este fichero es casi el mismo en cada ejemplo; hay diferencias de contenido memores para resaltar diferentes conceptos de JAXB.
  • Main.java es la clase Java principal para cada ejemplo.
  • build.xml es un proyecto Ant proporcionado para su conveniencia. Usa la herramienta Ant para generar, compilar, y ejecutar las clases JAXB derivadas del esquema automáticamente. El fichero build.xml varía según los ejemplos.
  • MyDatatypeConverter.java en el ejemplo inline-customize es una clase Java usada para proporcionar conversiones personalizadas de tipos de datos.
  • binding.xjb en el ejemplo Personalización Externa es un fichero externo de declaraciones de vínculos que se pasa al compilador de vinculación JAXB para personalizar las vinculaciones JAXB predeterminadas.

Opciones del Compilador JAXB

El compilador de vinculación de esquema JAXB XJC transforma, o vincula, un esquema XML de origen a un conjunto de clases de contenido JAXB en el lenguaje de programación Java. La clase del compilador, xjc, se proporciona como: xjc.sh en Solaris/Linux y xjc.bat en Windows en el paquete JAXB RI. La clase xjc se incluye en la librería de clases del KDJ (en tools.jar).

Tanto xjc.sh como xjc.bat tienen las mismas opciones de línea de comando. Puede visualizar rápidamente las instruccoines de uso infocando los scripts sin ninguna opción, o con la opción -help. La sintaxis es como sigue: 

xjc [-options ...] <schema file/URL/dir/jar>... [-b >bindinfo<] ...

Si se especifica dir, todos los ficheros de esquema en el directorio serán compilados. Si se especifica jar, el fichero de vinculación /META-INF/sun-jaxb.episode será compilado.

Las opciones de línea de comando de xjc son las siguientes:

-nv
No realiza validación estricta del esquema o esquemas de entrada. Por defecto, xjc realiza validación estricta del esquema fuente antes del procesado. Tenga en cuenta que esto no significa que el compilador no realizará ninguna validación; realizará simplemente una validación menos estricta.
-extension
Por defecto, el compilador de vinculación XJC aplica estrictamente las reglas descritas en el capítulo de compatibilidad de la especificación JAXB. En el modo predeterminado (estricto), está también limitado para usar sólo las personalizaciones de vinculación definidas en la especificación. Si usa la opción -extension, se le permite usar las Extensiones del Proveedor JAXB.
-b fichero
Especifica uno o más ficheros de vinculación externis a procesar. (Cada fichero de vinculación debe tener su propia opción -b). La sintaxis de los ficheros de vinculación externos es flexible. Puede tener un único fichero de vinculación que contenga personalizaciones para múltiples esquemas o puede dividir las personalizaciones en múltiples ficheros de vinculación. Además, el orden de los archivos de esquema y los archivos de vinculación en la línea de comandos no importa.
-d dir
De forma predeterminada, el compilador XJC de vinculación genera clases de contenido Java en el directorio actual. Use esta opción para especificar un directorio de salida alternativo. El directorio debe existir ya; el compilador XJC de vinculación no creará uno por usted.
-p paquete
Especifica un directorio de salida alternativo. Por defecto, el compilador XJC de vinculación generará las clases de contenido de Java en el directorio actual. el directorio de salida debe existir ya; el compilador XJC de vinculación no creará uno por usted.
-httpproxy proxy
Especifica el proxy HTTP/HTTPS. El formato es [usuario[:contraseña]@] proxyHost[:proxyPort]. Las antiguas opciones -host y -port son aún admitidas por la Implementación de Referencia por compatibilidad ocn versiones anteriores, pero han quedado obsoletas.
-httpproxyfile f
Funciona como la opción -httpproxy pero toma el argumento en un fichero para proteger la contraseña.
-classpath arg
Especifica dónde encontrar los ficheros de clases de la aplicación cliente usados por las personalizaciones <jxb:javaType> y <xjc:superClass>.
-catalog fichero
Especifica los ficheros de catálogo para resolver referencias de entidades externas. Admite TR9401, XCatalog y el formato del Catálogo OASIX XML. Para obtener más información, vea el documento de Entidades XML y Resolutores UR o examine la aplicación de ejemplo catalog-resolver.
-readOnly
Fuera al compilador XJC de vinculación a marcar las fuentes de Java generadas como de sólo lectura. De forma predeterminada, el compilador XJC de vinculación no protege contra escritura los ficheros fuente de Java que genera.
-npa
Suprime la generación de anotaciones a nivel de paquete dentro de **/package-info.java. Usar esta opción causa que el código generado internalize esas anotaciones dentro de las otras clases generadas.
-no-header
Suprime la generación de un encabezado de fichero con marca de tiempo.
-target (2.0|2.1)
Comportarse como XJC 2.0 o 2.1 y generar código que no use ninguna característica de XJC 2.2.
-enableIntrospection
Habilita la generación correcta de getters/setters booleanos para habilitar las ipas de Bean Instrospection.
-contentForWildcard
Genera propiedad de contenido para tipos con múltiples elementos derivados de xs:any.
-xmlschema
Trata los esquemas de entrada como Esquemas XML del W3C (predeterminado). Si no especifica esta opción, sus esquemas de entrada son tratados como Esqmeua XML del W3C.
-verbose
Es más detalladada la salida del compilador.
-quiet
Suprime la salida del compilador, tales como la información y aviso del progreso.
-help
Muestra un sumario breve de las opciones del compilador.
-version
Muestra la información de la versión del compilador.
-fullversion
Muestra la información de la versión completa del compilador.
-Xinject-code
Inyecta los fragmenteo de código de Java especificados en el código generado.
-Xlocator
Habilita la compatibilidad con la ubicación de origen para el código generado.
-Xsync-methods
Genera métodos de acceso con la palabra clave synchronized.
-mark-generated
Marque el código generado con la anotación -@javax.annotation. generada.
-episode FICHERO
Genera el fichero de episodio para la compilación separada

Opción de Generador de Esquema JAXB

El Generador de Esquema de JAXB, schemagen, crea un fichero de esquema para cada espacio de nombres referenciado en sus clases Java. El generador de esquema puede ser iniciado usando el script de comando schemagen apropiado en el directorio bin de su plataforma. El generador de esquema procesa sólo ficheros fuente de Java. Si sus fuentes Java referencian otras clases, esas fuentes deben estar accesibles desde su variable de entorno de sistema CLASSPATH; de otro modo ocurrirán errores cuando se genere el esquema. No hay forma de controlar el nombre de los ficheros de esquema generados.

Puede visualizar las intrucciones de uso rápido invocando los scripts sin ninguna opción o usando la opción -help. La sintaxis es la siguiente: 

schemagen [-d ruta] 
    [ficheros-fuente-java]

La opción -d ruta especifica la ubicación de los fihceros de clases generados por javac y por el procesador.

Clases JAXB Derivadas de Esquemas

Las secciones siguientes explican brevemente las funciones de las siguiente clases individuales generadas por el compilador de vinculación JAX para los ejemplos:

  • Clase Items
  • Clase ObjectFactory
  • Clase PurchaseOrderType
  • Clase USAddress

Clase Items

En Items.java:

  • La clase Items es parte del paquete primer.po.
  • La clase proporciona interfaces públicas para Items y ItemType.
  • El contenido en las instanciaciones de esta clase se vinculan a los ComplexTypes de XML Items y su elemento hijo ItemType.
  • Item proporciona el método getItem().
  • ItemType proporciona métodos para:
    • getPartNum();
    • setPartNum(String value);
    • getComment();
    • setComment(java.lang.String value);
    • getUSPrice();
    • setUSPrice(java.math.BigDecimal value);
    • getProductName();
    • setProductName(String value);
    • getShipDate();
    • setShipDate(java.util.Calendar value);
    • getQuantity();
    • setQuantity(java.math.BigInteger value);

Clase ObjectFactory

En ObjectFactory.java:

  • La clase ObjectFactory es parte del paquete primer.po.
  • ObjectFactory proporciona métodos de factoría para instanciar las interfaces Java que representa el contenido XML en el árbol de contenido de Java.
  • Los nombres de método se generan concatenando:
    • La cadena constante create.
    • Todos los nombres de clases de Java externas, si la interfaz de contenido de Java está anidada dentro de otra interfaz.
    • El nombre de la interfaz de contenido de Java.

Como un ejemplo, en este caso, para la interfaz Java primer.po.Items.ItemType, el ObjectFactory crea el método createItemsItemType().

Clase PurchaseOrderType

En PurchaseOrderType.java:

  • La clase PurchaseOrderType es parte del paquete primer.po.
  • El contenido en las instanciaciones de esta clase se vincula al elemento hijo del esquema XML llamado PurchaseOrderType.
  • PurchaseOrderType es una interfaz pública que proporciona los métodos siguientes:
    • getItems();
    • setItems(primer.po.Items value);
    • getOrderDate();
    • setOrderDate(java.util.Calendar value);
    • getComment();
    • setComment(java.lang.String value);
    • getBillTo();
    • setBillTo(primer.po.USAddress value);
    • getShipTo();
    • setShipTo(primer.po.USAddress value);

Clase USAddress

En USAddress.java:

  • La clase USAddress es parte del paquete primer.po.
  • El contenido en las instanciaciones de esta clase se vincula al elemento del esquema XML llamado USAddress.
  • USAddress es una interfaz pública que proporciona los métodos siguientes:
    • getState();
    • setState(String value);
    • getZip();
    • setZip(java.math.BigDecimal value);
    • getCountry();
    • setCountry(String value);
    • getCity();
    • setCity(String value);
    • getStreet();
    • setStreet(String value);
    • getName();
    • setName(String value);
