¿Porqué usar la Tecnología JMX?

La tecnología JMX proporciona a los desarrolladores un medio flexible para instrumentar aplicaciones basadas en tecnología Java (aplicaciones Java), crear agentes inteligentes, Implementar gestores y middleware de administración distribuida, e integrar sin problemas estas soluciones en los sistemas de administración y monitoreo existentes.

  • La tecnología JMX permite gestionar las aplicaciones Java sin grandes inversiones.
    Un agente basado en tecnología JMX (agente JMX) se puede ejecutar en la mayoría de los dispositivos habilitados para tecnología Java. Como consecuencia, las aplicaciones Java pueden convertirse en manejables con poco impacto en su diseño. Una aplicación Java necesita sólo embeber un objeto servidor gestionado y hacer alguna de sus funcionalidades disponibles como uno o varios beans gestionados (MBeans) registrados en el objeto servidor. Eso es todo lo que necesita para beneficiarse de la infraestructura de gestión.
  • La tecnologìa JMX proporciona una forma estándar para gestionar aplicaciones Java, sistemas, y redes.
    Por ejemplo, el Servidor de Aplicación de la Plataforma Java, Edición Empresarial (Java EE) se ajusta a la arquitectura JMX y como consecuencia puede ser gestionado usando tecnología JMX.
  • La tecnología JMX se puede utilizar para la gestión inmediata de la máquina virtual Java..
    La Máquina Virtual de Java (MV de Java) está altamente instrumentada utilizando la tecnología JMX. Puede iniciar un agente JMX para acceder a la instrumentación integranda en la MV de Java, y por lo tanto supervisar y gestionar un MV de Java remotamente.
  • La tecnología JMX proporciona una arquitectura de gestión escalable y dinámcia.
    Cada servicio de agente JMX es un módulo independiente que puede se conectar al agente de gestión, según los requisitos. Esta aproximación basada en componentes significa que las soluciones JMX pueden escalar desde dispositivos de tamaño reducido hasta grandes conmutadores de telecomunicaciones y más. La especificación JMX proporciona un conjunto de servicios de agente principales. Los servicios adicionales pueden ser desarrollados y cargados, descargados, o actualizados dinámicamente en la infraestructura de gestión.
  • La tecnología JMX aprovecha las tecnologías Java estándar existentes.
    Siempre que sea necesario, la especificación JMX hace referencia a especificaciones Java existentes, por ejemplo, la IPA de la Interfaz de Nombrado y Directorio de Java (J.N.D.I.).
  • Las aplicaciones basadas en tecnología JMX (aplicaciones JMX) se pueden crear a partir de un módulo EID de NetBeans.
    Puede obtener un módulo desde el Centro de Actualización de NetBeans (seleccionar Herramientas -> Centro de Actualización en la interfaz de NetBeans) que le habilita para crear aplicaciones JMX usando el EID de NetBeans. Esto reduce los costes de desarrollar aplicaciones JMX.
  • La tecnología JMX se integra con soluciones existentes de gestión y tecnologías emergentes.
    Las IPAs de JMX son interfaces abiertas que cualquier proveedor de sistemas de gestión puede implementar. Las soluciones JMX pueden usar servicios de búsqueda y descubrimiento y protoclos como la tecnología de red Jini y el Protocolo de Ubicación de Servicios (SLP).
