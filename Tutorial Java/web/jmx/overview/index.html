Lección: Descripción de la Tecnología JMX

La Tecnología de Extensiones de Gestión de Java (JMX) es una parte estándar de la Plataforma de Java, Edición Estandar (plataforma de Java SE). La tecnología JMX fue agregada a la plataforma en la versión 5.0 de la Plataforma Java 2, Edición Estándar (J2SE).

La tecnología JMX proporciona una forma simple y estándar de gestionar recursos tales como aplicaciones, dispositivos, y servicios. Debido a que la tecnología JMX es dinámica, puede usarla para supervisar y gestionar recursos mientras son creados, instalados e implementados. Puede también usar la tecnología JMX para supervisar y gestionar la Máquina Virtual de Java (MV de Java).

La especificación JMX define la arquitectura, patrones de diseño, IPAs, y servicios en el lenguaje de programación Java para gestión y supervisión de aplicaciones y redes.

Usando la tecnología JMX, un recurso dado es instrumentado por uno o más objetos Java conocidos como Beans Gestionados o MBeans. Estos MBeans se registran en un servidor de objetos gestionado por el núcleo, conocido como un servidor MBean. El servidor MBean actúa como un agente de gestión y puede ejecutarse en la mayoría de los dispositivos que se han habilitado para el lenguaje de programación Java.

Las especificaciones definen a agentes JMX que usa para gestionar cualquier recurso que ha sido correctamente configurado para la gestión. Un agente JMX consiste de un servidor MBean, en el cual se registran los MBeans, y un conjunto de servicios para gestionar los MBeans. De esta forma, los agentes JMX controlan directamente los recursos y los ponen a disposición de las aplicaciones de gestión remota.

La forma en que se instrumentan los recursos es completamente independiente de la infraestructura de gestión. Por lo tanto, los recursos pueden volverse manejables independientemente de cómo se implementen sus aplicaciones de administración.

La tecnología JMX define conectores estándar (conocidos como conectores JMX) que le habilitan a acceder a los agentes JMX desde aplicaciones de gestión remota. Los conectores JMX usan protocolos diferentes que proporcionan la misma interfaz de gestión. Como consecuencia, una aplicación de gestión puede gestionar recursos transparentemente, independientemente del protocolo de comunicación usado. Los agentes JMX puede también ser usados por sistemas o aplicaciones que no cumplen con la especificación JMX, siempre que esos sistemas o aplicaciones admitan agentes JMX.
