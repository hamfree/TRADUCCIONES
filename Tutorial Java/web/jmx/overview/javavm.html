Monitorización y Gestión de una Máquina Virtual de Java

La tecnología JMX puede también usarse para supervisar y gestionar la máquina virtual de Java (MV de Java).

La MV de Java tiene instrumentación integrada que le habilita a supervisar y gestionarla usando la tecnología JMX. Esta utilidades de gestión integradas son a menudos referenciadas como herramientas de gestión listas-para-usar para la MV de Java. Para supervisar y gestionar diferentes aspectos de la MV de Java, la MV de Java incluye un servidor MBean de plataforma y MXBeans especiales para usar por aplicaciones de gestión que se ajustan a la especificación JMX.

MXBeans de Plataforma y el Servidor MBean de Plataforma

Los MXBeans de plataforma son un conjunto de MXBeans que se proporcionan con la plataforma de Java SE para supervisar y gestionar la MV de Java y otros componentes del Entorno de Ejecución de Java (EEJ). Cada plataforma MXBean encapsula una parte de la funcionalidad de la MV de Java, tales como el sistema de carga de clases, el sistema de compilación Just-In-Time (JIT), recolector de basura, etcetera. Estos MXBeans pueden visualizarse e interactuar con ellos mediante el uso de una herramienta de supervisión y gestión que cumpla con la especificación JMX, para permitirle supervisar y gestionar estas diferentes funcionalidades de la MV. Una de estas herramientas de supervisión y gestión es la interfaz gráfica de usuario (IGU) JConsole de la plataforma Java SE.

La plataforma Java SE proporciona un servidor MBean de plataforma estándar en el cual estos MXBeans de plataforma se registran. El servidor de plataforma MBean puede también registrar cualquier otro MBeans que usted desee crear.

JConsole

La plataforma Java SE incluye la herramienta de supervisión y gestión JConsole, la cual cumple con la especificación JMX. JConsole usa la instrumentación extensvia de la MV de Java (la plataforma MXBeans) para proporcionar información sobre el rendimiento y consumo de recursos de aplicaciones que están ejecutándose en la plataforma Java.

Gestión Inmediata en Acción

Debido a que las utilidades de gestión y supervisión que implementa la tecnología JMX están integradas en la plataforma Java SE, puede ver la tecnología JMX inmediatamente en acción sin tener que escribir una sola línea de código de IPA de JMX. Puede hacerlo lanzando un aplicación Java y después monitorizarla usando JConsole.

Monitorizar una Aplicación Usando JConsole

Este procedimiento muestra cómo supervisar la aplicación de Java Notepad. Bajo las versiones de la plataforma de Java SE anteriores a la versión 6, las aplicaciones que quiere supervisar con JConsole necesitan ser iniciadas con la opción siguiente. 

-Dcom.sun.management.jmxremote

Sin embargo, la versión de JConsole proporcionada con la plataforma de Java SE 6 puede adjuntar a cualquier aplicación local que admita la IPA Attach. En otras palabras, cualquier aplicación que se inicia en la MV HotSpot de Java SE 6 se detecta automáticamente por JConsole, y no necesita ser iniciada usando la opción de línea de comandos de arriba.

  1. Inicie la aplicación Java Notepad, usando el comando siguiente en una ventana terminal: 
    java -jar 
    jdk_home/demo/jfc/Notepad/Notepad.jar

    Donde jdk_home es el directorio en el cual se instaló el Kit de Desarrollo de Java (KDJ). Si no está ejecutando la versión 6 de la plataforma de Java SE, necesitará usar el comando siguiente: 

    java -Dcom.sun.management.jmxremote -jar 
      jdk_home/demo/jfc/Notepad/Notepad.jar
  2. Una vez se ha abierto Notepad, en una ventana terminal difeente, inicie JConsole usando el comando siguiente: 
    jconsole

    Se visualiza un cuadro de diálogo Nueva Conexión

  3. En el cuadro de diálogo Nueva Conexión, seleccione Notepad.jar de la lista de Procesos Locales, y pulse en el botón Conectar.

    JConsole abre y se conecta a si misma al proceso Notepad.jar. Cuando se abre JConsole, se le presenta con un resumen de la información de supervisión y gestión relacionada con Notepad. Por ejemplo, puede ver la cantidad de memoria del montón que la aplicación está consumiendo, el número de hilos que se están ejecutando actualmente en la aplicación, y cuánta capacidad de la unidad de proceso central (UCP) es consumida por la aplicación.

  4. Pulse en las diferentes pestañas de JConsole.

    Cada pestaña presenta información más detallada sobre las diferentes áreas de funcionalidad de la MV de Java en la cual Notepad se está ejecutando. Toda la información presentada se obtiene de los diversos MXBeans de tecnología JMX mencionados en este recorrido. Todda la plataforma de MXBeans puede visualizarse en la pestaña MBeans. La pestaña MBeans se examina en la siguiente sección de este recorrido.

  5. Para cerrar JConsole, seleccione Conexión -> Salir.
