Arquitectura de la Tecnología JMX

La tecnología JMX puede dividirse en tres niveles, como sigue:

  • Instrumentación
  • Agente JMX
  • Gestión Remota

Instrumentación

Para gestionar recursos usando la tecnología JMX, debe primero instrumentar los recursos en el lenguaje de programación Java. Puede usar objetos Java conocidos como MBeans para implementar el acceso a la instrumentación de recursos. Los MBeans deben seguir los patrones de diseño e interfaces definidos en la especificación JMX. Hacerlo asegura que todos los MBeans proporcionan la instrumentación de un recurso gestionado de una forma estandarizada. Además de los MBeans estándares, la especificación JMX también define un tipo especial de MBean llamado MXBean. Un MXBean es un MBean que referencia sólo un conjunto predefinido de tipos de datos. Existen otros tipos de MBeans, pero este recorrido se concentrará en los MBeans estándar y los MXBeans.

Una vez un recurso ha sido instrumentado por MBeasn, puede ser gestionado a través de un agente JMX. Los MBeans no requieren conocimiento del agente JMX con el que operarán.

Los MBeans se diseñan para ser flexibles, simples, y fáciles de implementar. Los desarrolladores de aplicaciones, sistemas y redes pueden hacer sus productos gestionables de una forma estándar sin tener que entender o invertir en sistemas de gestión complejos. Los recursos existentes se pueden gestionar con un esfuerzo mínimo.

Además, el nivel de instrumentación de la especificación JMX proporciona un mecanismo de notificación. Este mecanismo habilita a los MBeans generar y propagar eventos de notificación a los componentes de otros niveles.

Agente JMX

Un agente basado en tecnología JMX (agente JMX) es un agente de gestión estándar que directamente controla los recursos y los hace disponibles para aplicaciones de gestión remota. Los agentes JMX se ubican habitualmente en la misma máquina que los recursos que controlan, pero esto arreglo no es un requisito.

El componente principal de un agente JMX es el servidor MBean, un servidor de objetos gestionados en el que se registran MBeans. Un agente JMX también incluye un conjunto de servicios para gestionar MBeans, y al menos un adaptador de comunicaciones o conector para permitir el acceso de una aplicación de gestión.

Cuando implementa un agente JMX, no necesita saber la semántica o funciones de los recursos que gestionará. En realidad, un agente JMX ni siquiera necesita saber qué recursos servirá porque cualquier recurso instrumentado de acuerdo con la especificación JMX puede usar cualquier agente JMX que ofrezca los servicios que el recurso requiere. Del mismo modo, el agente JMX no necesita saber las funciones de las aplicaciones de gestión que accederán a él.

Gestión Remota

Se puede acceder a la instrumentación de la tecnología JMX de muchas formas diferentes, ya sea a través de protocolos de administración existentes como el Protocolo Simple de Gestión de Red (SNMP) o a través de protocolos propietarios. El servidor MBean se basa en adaptadores de protocolo y conectores para hacer que un agente JMX sea accesible desde aplicaciones de gestión fuera de la Máquina Virtual de Java (MV de Java) del agente.

Cada adaptador proporciona una vista a través de un protocolo específico de todos los MBeans que se registraron en el servidor MBean. Por ejemplo, un adaptador HTML podría visualizar un MBean en un navegador.

Los conectores proporcionan una interfaz del lado del gestor que maneja la comunicación entre el gestor y el agente JMX. Cada conector proporciona la misma interfaz de gestión remota a través de un protocolo diferente. Cuando una aplicación de gestión remota usa esta interfaz, puede conectar a un agente JMX transparentemente a través de la red, independientemente del protocolo. La tecnología JMX proporciona una solución estándar para exportar la instrumentación de la tecnología JMX a aplicaciones remotas basadas en la Invocación de Métodos Remotes de Java (Java RMI).
