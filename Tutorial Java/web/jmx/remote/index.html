Lección: Gestión Remota

La IPA de JMX le habilita para realizar gestión remota de sus recursos usando conectores basados en tecnología JMX (conectores JMX). Un conector JMX hace a un servidor MBean accesible a los clientes remotos basados en tecnologías JMX. El extremo cliente de un conector exporta esencialmente la misma interfaz que el servidor MBean.

Un conector JMX consiste de un conector cliente y un conector servidor. Un conector servidor se enlaza a un servidor MBean y escucha las peticiones de conexión de los clientes. Un conector cliente es responsable de establecer una conexión con el conector servidor. Un conector cliente está usualmente en una Máquina Virtual de Java (MV de Java) diferente de la del conector servidor y está a menudo ejecutándose en una máquina diferente. La IPA de JMX define un protocolo de conexión estándar basado en la Invocación de Métodos Remotos (RMI). Este protocolo le habilita a conectar un cliente JMX a un MBean en un servidor MBean desde una ubicación remotea y realizar operaciones sobre el MBean, exactamente como si las operaciones estuvieran siendo realizadas localmente.

La plataforma Java SE proporciona un medio listo para usar para monitorear aplicaciones de forma remota mediante el uso del conector RMI estándar de la IPA de JMX. El conector RMI listo para usar expone automáticamente las aplicaciones para la gestión remota, sin necesidad de que usted mismo cree un conector servidor remoto dedicado. El agente de gestión remota listo para usar se activa al iniciar su aplicación Java con las propiedades correctas. Las aplicaciones de supervisión y gestión que son compatibles con la tecnología JMX pueden conectar a estas aplicaciones y supervisarlas remotamente.
