Exponiendo un Recurso de Gestión Remota por JConsole

Exponer sus aplicaciones Java para gestión remota usando la IPA de JMX puede ser extremadamente simple, si usa el agente de gestión remota listo para usar y una herramienta de supervisión y gestión tal como JConsole.

Para exponer su aplicación para la gestión remota, necesita iniciarla con las propiedades correctas. Este ejemplo muestra como exponer el agente JMX de Main para gestión remota.

Consideración de Seguridad: 

Por simplicidad, los mecanismos de seguridad de autenticación y cifrado están deshabilitados en este ejemplo. Sin embargo, debe implementar estos mecanismos de seguridad cuando implemente la gestión remota en entornos del mundo real. ¿Qué sigue? proporciona referencias a otra documentación de tecnología JMX que muestra cómo activar la seguridad.

Este ejemplo requiere la versión 6 de la plataforma Java SE. Para supervisar el agente JMX Main remotamente, siga estos pasos:

  1. Si no lo ha hecho ya, salve jmx_examples.zip dentro de su directorio work_dir.
  2. Descomprima el paquete de clases de ejemplo usando el comando siguiente en una ventana terminal. 
    unzip jmx_examples.zip
  3. Compile las clases Java de ejemplo desde el directorio work_dir. 
    javac com/example/*.java
  4. Inicie la aplicación Main, especificando las propiedades que exponen a Main para la gestión remota. (Para Windows, use el acento circunflejo (^) en vez de la barra invertida (\) para dividr un comando largo en múltiples líneas): 
    java -Dcom.sun.management.jmxremote.port=9999 \
     -Dcom.sun.management.jmxremote.authenticate=false \
     -Dcom.sun.management.jmxremote.ssl=false \
     com.example.Main

    Se genera una confirmación de que Main está esperando a que ocurra algo.

  5. Inicie JConsole en una ventana terminal diferente en una máquina diferente: 
    jconsole

    Se visualiza el cuadro de diálogo Nueva Conexión, presentando una lista de los agentes JMX en ejecución a los que puede conectar localmente.

  6. Seleccione Proceso Remoto, y escriba lo siguiente en el campo Proceso Remoto: 
    nombrehost:9999

    En esta dirección, nombrehost es el nombre de la máquina remota en la cual la aplicación Main se está ejecutando y 9999 es el número del puerto en el cual el conector JMX listo para usar estará conectado.

  7. Pulse Conectar.

    Se visualiza un resumen de la actividad actual de la Máquina Virtual Java (MV de Java) en la cual Main se está ejecutando.

  8. Pulse la pestaña MBeans.

    Este panel muestra todos los MBeans que están actualmente registrados en el servidor MBean remoto.

  9. En el marco a la izquierda, expanda el nodo com.example en el árbol MBean.

    Verá el ejemplo del MBean Hello que fue creado y registrado por Main. Si pulsa Hello, verá sus nodos asociados de Atributos y Operacoines en el árbol MBean, aunque se esté ejecutando en una máquina diferente.

  10. Para cerrar JConsole, seleccione Conexión -> Salir.
