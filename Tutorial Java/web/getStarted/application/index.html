Lección Una Mirada Más Cercana a la Aplicación "¡Hola Mundo!"

Ahora que usted ha visto la aplicación "¡Hola Mundo!" (y quizás incluso compilado y ejecutado), usted podría estarse preguntando cómo funciona. Aquí está de nuevo su código: 

class HolaMundoApl {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!"); // Muestra la cadena.
    }
}

La aplicación "¡Hola Mundo!" consiste en tres componentes primarios: comentarios del código fuente,la definición de clase HolaMundoApl, y el método main. La explicación siguiente le ofrecerá un entendimiento básico del código, pero las implicaciones más profundas se volverán claras después de que usted haya terminado de leer el resto del tutorial.

Comentarios del Código Fuente

El siguiente texto en negrita define los comentarios de la aplicación "¡Hola Mundo!": 

/**
 * La clase HolaMundoApl implementa una aplicación que
 * simplemente imprime "¡Hola Mundo!" a la salida estandar.
 */
class HolaMundoApl {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!"); // Muestra la cadena.
    }
}

Los comentarios son ignorados por el compilador pero son útiles a otros programadores. El lenguaje de programación Java soporta tres clases de comentarios:

/* texto */
El compilador ignora todo desde /* a */.
/** documentación */
Este indica un comentario de documentación (comentario doc , para abreviar). El compilador ignora esta clase de comentario, como ignora los comentarios que usan /* y */. La herramienta javadoc usa comentarios doc cuando prepara automáticamente la documentación generada. Para más información sobre javadoc, vea la documentación de la herramienta Javadoc™.
// texto
El compilador ignora todo desde // hasta el final de la línea.

La Definición de la Clase HolaMundoApl

El siguente texto en negrita empieza la definición del bloque de clase para la aplicación "¡Hola Mundo!": 

/**
 * La clase HolaMundoApl implementa una aplicación que
 * simplemente imprime "¡Hola Mundo!" a la salida estandar.
 */
class HolaMundoApl {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!"); // Muestra la cadena.
    }
}

Como se mostró arriba, la forma más básica de definición de clase es: 

class Nombre {
    . . .
}

La palabra clave class empieza la definición de clase para una clase nombrada Nombre, y el código para cada clase aparece entre las llaves de apertura y cierre marcadas en negrita arriba. El capítulo 2 ofrece una introducción de las clases en general, y el Capítulo 4 analiza las clases en detalle. Por ahora es bastante saber que cada aplicación empieza con una definición de clase.

El Método main

El siguiente texto en negrita empieza la definición del método main: 

/**
 * La clase HolaMundoApl implementa una aplicación que
 * simplemente imprime "¡Hola Mundo!" a la salida estandar.
 */
class HolaMundoApl {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!"); // Muestra la cadena.
    }
}

En el lenguaje de programación Java, cada aplicación debe contener un método main cuya firma es: 

public static void main(String[] args)

Los modificadores public y static pueden ser escritos en cualquier orden (public static o static public), pero la convención es usar public static como es mostrado arriba. Usted puede nombrar el argumento como quiera, pero la mayoría de los programadores seleccionan "args" o "argv".

El método main es similar a la función main en C y C++; es el punto de entrada para su aplicación y posteriormente invocará todos los otros métodos necesitados por su programa.

El método main acepta un único argumento: un vector de elementos del tipo String. 

public static void main(String[] args)

Este vector es el mecanismo a través del cual el sistema en ejecución pasa información a su aplicación. Por ejemplo: 

java MyApp arg1 arg2

Cada cadena en el vector es llamado un argumento de línea de comandos. Los argumentos de línea de comandos permiten a los usuarios modificar la operación de la aplicación sin recompilarla. Por ejemplo, un programa de ordenación puede permitir al usuario el especificar que los datos sean ordenados en orden con este argumento de línea de comandos: 

-descendiente

La aplicación "¡Hola Mundo!" ignora sus argumentos de línea de comandos, pero usted estará enterado del hecho de que tales argumentos existen.

Finalmente, la línea: 

System.out.println("¡Hola Mundo!");

usa la clase System del núcleo de la librería para imprimir el mensaje "¡Hola Mundo!" a la salida estándar. Porciones de ésta librería (también conocida como el "Interfaz de Programación de la Aplicació", o "IPA") será analizado a traves del resto del tutorial.
