Eventos

Una clase bean puede disparar cualquier tipo de evento, incluyendo eventos personalizados. Como con las propiedades, los eventos se identifican por un patrón específico de nombres de método. 

public void add<Event>Listener(<Event>Listener a)
public void remove<Event>Listener(<Event>Listener a)

El tipo del listener debe ser un descendiente de java.util.EventListener.

Por ejemplo, un JButton de Swing es un bean que dispara eventos action cuando el usuario pulsa sobre él. JButton incluye los métodos siguientes (en realidad heredados desde AbstractButton), los cuales son el patrón de bean para un evento: 

public void addActionListener(ActionListener l);
public void removeActionListener(ActionListener l);

Los eventos del bean se reconocen por las herramientas constructoras y se pueden usar para conectar componentes juntos. Por ejemplo, puede enlazar el evento action de un botón para hacer que algo ocurra, como invocar otro método del bean.
