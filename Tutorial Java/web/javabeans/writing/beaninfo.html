Usando un BeanInfo

Los beans, especialmente los componentes gráficos, puede tener una cantidad vertiginosa de propiedades. Si su clase hereda desde Component, o JComponent, u otras clases Swing, tendrá ya más de cien propiedades. Aunque una herramienta constructora como NetBeans hace fácil editar propiedades de los bean, puede ser difícil de encontrar las propiedades correctas para editar, especialmente para programadores inexpertos.

Descripción General de BeanInfo

Un BeanInfo es una clase que cambia como su bean aparece en una herramienta constructora. Una herramienta constructora puede consultar BeanInfo para encontrar qué propiedades debe visualizar primero y cuales debe ocultar.

La clase BeanInfo para su bean debe tener el mismo nombre que la clase bean, con BeanInfo agregado. Por ejemplo, la clase FaceBean tiene una clase FaceBeanBeanInfo que lo describe.

Aunque es posible implementar una clase BeanInfo "a mano" encontrará que es más fácil usar una herramienta como NetBeans para editar el BeanInfo.

Crear un BeanInfo en NetBeans

En el panel Projectos, pulse Control-click sobre el nombre de us clase bean y elija Editor BeanInfo... desde el menú emergente.

NetBeans se da cuenta de que no tiene un BeanInfo y pregunta si usted quiere crear uno. Pulse Sí.

NetBeans crea una clase nueva y le deja en el editor de código fuente. Pulse sobre Diseñador para cambiar al editor visual.


Pulse para una imagen completa

Seleccione propiedades desde la lista en el lado izquierdo del editor visual, entonces edite sus atributos en la parte derecha. Si no quiere que una propiedad particular aparezca a un desarrollador usando una herramienta constructora, pulse Oculto. Para indicar que una propiedad debe mostrarse antes que otras, haga click en Prefererida. Puede también indicar si una propiedad esta vinculada o restringida.

Puede proporcionar información similar para las fuentes de los eventos del bean y los métodos.

Cuando una herramienta constructora carga su clase bean para agregarlo a su paleta, encontrará automáticamente el BeanInfo correspondiente y lo usa para decidir cómo presentar su bean al desarrollador.
