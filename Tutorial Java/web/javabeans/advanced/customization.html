Personalización del Bean

La Personalización proporciona un significado para modificar la apariencia y comportamiento de un bean dentro de un constructor de aplicacón para que satisfaga sus necesidades específicas. Hay varios niveles de personalización disponibles para un desarrollador de beans para permitir a otros desarrolladores para obtener el máximo beneficio de la funcionalidad potencial de un bean.

Los enlaces siguientes son útiles para aprender sobre los editores de propiedades y personalizadores:

  • interfaz PropertyEditor
  • clase PropertyEditorSupport
  • clase PropertyEditorManager
  • interfaz Customizer
  • interfaz BeanInfo

La apariencia y comportamiento de un bean se pueden personalizar en tiempo de diseño dentro de las herramientas de construcción compatibles con beans. Hay dos formas de personalizar un bean:

  • Usando un editor de propiedades. Cada propiedad de bean tiene su propio editor de propiedad. El Constructor IGU de NetBeans usualmente visualiza los editores de propiedades del bean en la ventana Propiedades. El editor de propiedad que está asociado con un tipo de propiedad en particular edita ese tipo de propiedad.
  • Usando personalizadores. Los personalizadores le dan un control completo del IGU sobre la personalización del bean. Los personalizadores se usan cuando los editores de propiedades no son prácticos o aplicacbles. A diferencia de los editors de propiedades, que están asociados con una propiedad, un personalizador está asociado con un bean.

Editores de Propiedades

Un editor de propiedad es una herramienta para personalizar un tipo de propiedad en particular. Los editores de propiedades se activan en la ventana Propiedades. Este ventana determina el tipo de propiedad, busca un editor de propiedad relevante, y muestra el valor actual de la propiedad de una forma relevante.

Los editores de propiedades deben implementar la interfaz PropertyEditor, la cual proporciona métodos para especificar cómo una propiedad debe ser mostrado en una hoja de propiedades. La figura siguiente representa la ventana de Propiedades conteniendo las propiedades de myBean1:

Por ejemplo, para configurar el foreground o el background usa las cajas de selección con elecciones de colores, o presion el botón "..." para trabajar con una ventana estándar de Editor de Color. Pulsar en la propiedad toolTipText abre una ventana Editor de Cadena.

La clase de soporte PropertyEditorSupport proporciona una implementación predeterminada de la interfaz PropertyEditor. Al crear una subclase de su editor de propiedades desde PropertyEditorSupport, simplemente puede sobreescribir los métodos que necesita.

Para mostrar el valor de la propiedad actual "sample" dentro de la ventana Propiedades, necesita sobreescribir isPaintable para devolver true. Después debe sobreescribir paintValue para pintar el valor de la propiedad actual en un rectángulo de la hoja de propiedades. Aquí está cómo ColorEditor implementa paintValue: 

public void paintValue(java.awt.Graphics gfx, java.awt.Rectangle box) {
    Color oldColor = gfx.getColor();
    gfx.setColor(Color.black);
    gfx.drawRect(box.x, box.y, box.width-3, box.height-3);
    gfx.setColor(color);
    gfx.fillRect(box.x+1, box.y+1, box.width-4, box.height-4);
    gfx.setColor(oldColor);
}

Para admitir el editor de propiedades personalizado, sobreescriba dos métodos más. Sobreescriba supportsCustomEditor para devolver true, y después sobreescriba getCustomEditor para devolver una instancia personalizada del editor. ColorEditor.getCustomEditor devuelve this.

Además, la clase PropertyEditorSupport mantiene una lista de PropertyChangeListener, y dispara notificaciones de eventos de cambios de propiedad a esos listener cuando se cambia una propiedad enlazada.

Cómo Se Asocian los Editores de Propiedades con las Propiedades

Los editores de propiedades se descubren y asocian a una propiedad determinada de la siguiente forma:

  • Asociación explícita en forma de un objeto BeanInfo. El editor de una propiedad de título se establece con la línea de código siguiente: 
    pd.setPropertyEditorClass(TitleEditor.class);
  • Registro explícito mediante el método java.beans.PropertyEditorManager.registerEditor. Este método toma dos argumentos: el tipo de clase bean, y la clase editor que se asocia con ese tipo.
  • Búsqueda de nombre. Si una clase no tiene asociado explícitamente un editor de propiedad, entonces el PropertyEditorManager busca el editor de propiedades de esa clase de las formas siguientes:
    • Agrega "Editor" al nombre completamente cualificado de la clase. Por ejemplo, para la clase my.package.ComplexNumber, el gestor del editor de propiedad buscará la clase my.package.ComplexNumberEditor.
    • Agrega "Editor" al nombre de clase y buscando en una ruta de clases.

Personalizadores

Ha aprendido que las herramientas de construcción le proporcionan soporte para crear sus propios editores de propiedades. ¿Qué otras necesidades deben satisfacer los constructores visuales para los beans complejos de carácter empresarial? A menudo no es deseable tener todas las propiedades de un bean reveladas en una única (algunas veces enorme) hoja de propiedad. ¿Qué pasa si una sola elección de raíz sobre el tipo de bean hace que la mitad de las propiedades sean irrelevantes? La especificación JavaBeans proporciona personalizadores definidos por el usuario, a través de los cuales puede definir un nivel más alto de personalización para las propiedades de los beans que el que está disponible con los editores de propiedades.

Cuando use un bean Customizer, tiene el control completo sobre cómo configurar o editar un bean. Un Personalizador es una aplicación que se dirige específicamente a la personalización de un bean. Algunas veces las propiedades son insuficientes para representar los atributos configurables de un bean. Los personalizadores se usan cuando son necesarias instrucciones sofisticadas para cambiar un bean, y donde los editores de propiedades son demasiado primitivos para conseguir la personalización del bean.

Todos los personalizadores deben:

  • Extender java.awt.Component o alguna de sus subclases.
  • Implementar la interfaz java.beans.Customizer. Esto significa implementar los métodos para registrar objetos PropertyChangeListener, y disparar los eventos de cambios de propiedad a esos listeners cuando ocurre un cambio en el bean objetivo.
  • Implementar un constructor predeterminado.
  • Asociar el personalizador con su clase objetivo vía BeanInfo.getBeanDescriptor.
