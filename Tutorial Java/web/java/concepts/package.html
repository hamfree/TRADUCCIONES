¿Qué Es un Paquete?

Un paquete es un espacio de nombres que organiza un conjunto de clases e interfaces relacionadas. Conceptualmente puede pensar de los paquetes como si fueran similares a carpetas diferentes en su ordenador. Usted podría mantener las páginas HTML en una carpeta, las imágenes en otra, y los scripts o aplicaciones en otra aún. Debido a que el software escrito en el lenguaje de programación Java puede estar compuesto de cientos o miles de clases individuales, tiene sentido el mantener las cosas organizadas colocando las clases e interfaces relacionadas en paquetes.

La plataforma Java ofrece una enorme librería de clases (un conjunto de paquetes) adecuado para utilizar en sus propias aplicaciones. Esta librería es conocida como el "Interfaz de Programación de Aplicaciones", o "IPA" (API en inglés) para las aplicaciones. Sus paquetes representan las tareas más comunmente asociadas con la programación de propósito general. Por ejemplo, un objeto String contiene estado y comportamiento para cadenas de caracteres; un objeto File permite a un programador fácilmente crear, borrar, inspeccionar, comparar, o modificar un fichero en el sistema de ficheros; un objeto Socket permite la creación y uso de sockets de red; varios objetos de control de IGU (Interfaz Gráfica del Usuario, GUI en inglés) como botones y casillas de verificación y cualquier otro relacionado con las interfaces gráficas del usuario. Hay literalmente miles de clases para elegir. Esto le permite a usted, el programador, enfocarse en el diseño de su aplicación particular, más que en la infraestructura requerida para hacerla funcionar.

La Especificación IPA de la Plataforma Java contiene el listado completo de todos los paquetes, interfaces, clases, campos, y métodos suministrador por la plataforma Java SE. Cargue la página en su navegador y añádala a sus favoritos. Como programador, se convertirá en su pieza más importante de documentación de referencia.
