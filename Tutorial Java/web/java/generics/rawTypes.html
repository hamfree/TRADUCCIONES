Tipos Crudos

Un tipo crudo es el nombre de una clase genérica o interfaz sin ningún argumento de tipos. Por ejemplo, dada la clase genérica Box: 

public class Box<T> {
    public void set(T t) { /* ... */ }
    // ...
}

Para crear un tipo parametrizado de Box<T>, suple un argumento tipo actual para el parámetro tipo formal T: 

Box<Integer> intBox = new Box<>();

Si el argumento tipo actual es omitido, crea un tipo crudo de Box<T>: 

Box rawBox = new Box();

Por lo tanto, Box es el tipo crudo del tipo genérico Box<T>. Sin embargo, una clase no genérica o tipo de interfaz no es un tipo crudo.

Los tipos crudos aparecen en el código heredado porque un montón de clases IPA (tales como las clases de Collections) fueron no generícas antes del KDJ 5.0. Cuando se usan tipos crudos, esencialmente obtiene un comportamiento pre-genéricos — Un Box le devuelve a usted Objects. Por compatibilidad con versiones anteriores, asignar un tipo parametrizado a su tipo crudo está permitido: 

Box<String> stringBox = new Box<>();
Box rawBox = stringBox;               // OK

Pero si asigna un tipo crudo a un tipo parametrizado, obtiene una advertencia: 

Box rawBox = new Box();           // rawBox esun tipo crudo de Box<T>
Box<Integer> intBox = rawBox;     // advertencia: conversión no comprobada

También obtendrá un aviso si usa un tipo crudo para invocar métodos genéricos definidos en el tipo genérico correspondiente: 

Box<String> stringBox = new Box<>();
Box rawBox = stringBox;
rawBox.set(8);  // advertencia: invocación no comprobada a set(T)

Las advertencias muestran que los tipos crudos evitan las comprobaciones de los tipos genéricos, difiriendo la captrua de código no seguro al tiempo de ejecución. Por lo tanto, debería evitar el uso de tipos crudos.

La sección Borrado de Tipos tiene más información sobre cómo el compilador de Java usa los tipos crudos.

Mensajes de Error sin Marcar

Como se mencionó previamente, cuando se mezcla código heredado con código genérico, puede encotnrar mensajes de advertencia similares a los sguientes: 

Nota: Ejemplo.java usa operaciones no comprobadas o no seguras.
Nota: Recompile con -Xlint:unchecked para detalles.

Esto puede ocurrir cuando usa una vieja IPA que opera sobre tipos crudos, como es mostrado en el siguiente ejemplo: 

public class WarningDemo {
    public static void main(String[] args){
        Box<Integer> bi;
        bi = createBox();
    }

    static Box createBox(){
        return new Box();
    }
}

El término "no comprobado" significa que el compilador no tiene suficiente información del tipo para realizar todas las comprobaciones necesarias del tipo necesarias para asegurar la seguridad de tipo. La advertencia de "no comprobado" está deshabilitada, por defecto, aunque el compilador da un consejo. Para ver todas las advertencias "no comprobado", recompile con -Xlint:unchecked.

Recompilar el ejemplo previo con -Xlint:unchecked revela la siguiente información adicional: 

WarningDemo.java:4: warning: [unchecked] unchecked conversion
found   : Box
required: Box<java.lang.Integer>
        bi = createBox();
                      ^
1 warning

Para deshabilitar completamente las advertencias de no comprobado, use la bandera -Xlint:-unchecked. La anotación @SuppressWarnings("unchecked") suprime las advertencias de no comprobado. Si no está familiarizado con la sintaxis de@SuppressWarnings, vea Anotaciones .
