Respuestas a Preguntas y Ejercicios: Genéricos

  1. Escriba un método genérico que cuente el número de elementos en una colección que tiene una propiedad específica (por ejemplo, enteros impares, números primos, palíndromos).

    Respuesta: 
    public final class Algorithm {
    public static <T> int countIf(Collection<T> c, UnaryPredicate<T> p) {

        int count = 0;
        for (T elem : c)
            if (p.test(elem))
                ++count;
        return count;
    }
}
    Donde el interfaz genérico UnaryPredicate es definido como sigue: 
    public interface UnaryPredicate<T> {
    public boolean test(T obj);
}
    Por ejemplo, el siguiente programa cuenta el número de enteros impares en una lista de enteros: 
    import java.util.*;

class OddPredicate implements UnaryPredicate<Integer> {
    public boolean test(Integer i) { return i % 2 != 0; }
}

public class Test {
    public static void main(String[] args) {
        Collection<Integer> ci = Arrays.asList(1, 2, 3, 4);
        int count = Algorithm.countIf(ci, new OddPredicate());
        System.out.println("Número de enteros impares = " + count);
    }
}
    El programa imprime: 
    Número de enteros impares = 2
  2. ¿Compilará la siguiente clase? Si no, ¿porqué? 
    public final class Algorithm {
    public static <T> T max(T x, T y) {
        return x > y ? x : y;
    }
}
    Respuesta: No. El operador mayor que (>) se aplica sólo a tipos primitivos numéricos.

  3. Escriba un método genérico que intercambie las posiciones de dos elementos diferentes en una matriz.

    Respuesta: 
    public final class Algorithm {
    public static <T> void swap(T[] a, int i, int j) {
        T temp = a[i];
        a[i] = a[j];
        a[j] = temp;
    }
}
  4. Si el compilador borra todos los parámetros tipo en tiempo de compilación, ¿porqué utilizar los genéricos?

    Respuesta: Usará genéricos porque:
    • El compilador de Java hace cumplir estrictos controles de tipo en el código genérico en tiempo de compilación.
    • Los genéricos soportan la programación de tipos como parámetros.
    • Los genéricos le habilitan para implementar algorítmos genéricos.


  5. ¿En qué se convirtió la siguiente clase después del borrado de tipos? 
    public class Pair<K, V> {

    }

    public K getKey(); { return key; }

  7. ¿Compilará el siguiente método? Si no, ¿porqué? 
    public static void print(List<? extends Number> list) {
    for (Number n : list)
        System.out.print(n + " ");
    System.out.println();
}
    Respuesta: Sí.

  8. Escriba un método genérico para encontrar el elemento máximo en el rango [inicio, final) de una lista.

    Respuesta: 
    import java.util.*;

public final class Algorithm {
    public static <T extends Object & Comparable<? super T>>
        T max(List<? extends T> list, int begin, int end) {

        T maxElem = list.get(begin);

        for (++begin; begin < end; ++begin)
            if (maxElem.compareTo(list.get(begin)) < 0)
                maxElem = list.get(begin);
        return maxElem;
    }
}
  9. ¿Compilará la siguiente clase? Si no, ¿porqué? 
    public class Singleton<T> {

    public static T getInstance() {
        if (instance == null)
            instance = new Singleton<T>();

        return instance;
    }

    private static T instance = null;
}
    Respuesta: No. No puede crear un campo estático del tipo parámetro T.

  12. ¿Cómo invoca el siguiente método para encontrar el primer entero en una lista que es relativamente prima a una lista de enteros especificados? 
    public static <T>
    int findFirst(List<T> list, int begin, int end, UnaryPredicate<T> p)
    Note que los dos enteros a y b son primos relativamente si gcd(a, b) = 1, donde gcd es la abreviatura para máximo común divisor.

    Respuesta: 
    import java.util.*;

public final class Algorithm {

    public static <T>
        int findFirst(List<T> list, int begin, int end, UnaryPredicate<T> p) {

        for (; begin < end; ++begin)
            if (p.test(list.get(begin)))
                return begin;
        return -1;
    }

    // x > 0 and y > 0
    public static int gcd(int x, int y) {
        for (int r; (r = x % y) != 0; x = y, y = r) { }
            return y;
    }
}
    El interfaz genérico UnaryPredicate es definido como sigue: 
    public interface UnaryPredicate<T> {
    public boolean test(T obj);
}
    El siguiente programa comprueba el método findFirst: 
    import java.util.*;

class RelativelyPrimePredicate implements UnaryPredicate<Integer> {
    public RelativelyPrimePredicate(Collection<Integer> c) {
        this.c = c;
    }

    public boolean test(Integer x) {
        for (Integer i : c)
            if (Algorithm.gcd(x, i) != 1)
                return false;

        return c.size() > 0;
    }

    private Collection<Integer> c;
}

public class Test {
    public static void main(String[] args) throws Exception {

        List<Integer> li = Arrays.asList(3, 4, 6, 8, 11, 15, 28, 32);
        Collection<Integer> c = Arrays.asList(7, 18, 19, 25);
        UnaryPredicate<Integer> p = new RelativelyPrimePredicate(c);

        int i = ALgorithm.findFirst(li, 0, li.size(), p);

        if (i != -1) {
            System.out.print(li.get(i) + " es primo relativo a ");
            for (Integer k : c)
                System.out.print(k + " ");
            System.out.println();
        }
    }
}
    El programa imprime: 
    11 es primo relativo a to 7 18 19 25
