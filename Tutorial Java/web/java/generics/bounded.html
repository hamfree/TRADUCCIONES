Parámetros de Tipos Acotados

Puede haber veces que quiera restringir los tipos que pueden ser usados como argumentos tipo en un tipo parametrizado. Por ejemplo, un método que opera sobre números podría solo querer aceptar instancias de Number o sus subclases. Esto es para lo que están los parámetros de tipos acotados.

Para declarar un parámetro tipo acotado, listamos el nombre del parámetro tipo, seguido de la palabra clave extends, seguida de su límite superior, el cual en este ejemplo es Number. Note que, en este contexto, extends es usado en un sentido general que significa o "extender" (como en clases) o "implementar" (como en interfaces). 

public class Box<T> {

    private T t;          

    public void set(T t) {
        this.t = t;
    }

    public T get() {
        return t;
    }

    public <U extends Number> void inspect(U u){
        System.out.println("T: " + t.getClass().getName());
        System.out.println("U: " + u.getClass().getName());
    }

    public static void main(String[] args) {
        Box<Integer> integerBox = new Box<Integer>();
        integerBox.set(new Integer(10));
        integerBox.inspect("algo de texto"); // ¡error: esto es aún una cadena!
    }
}

Al modificar nuestro método genérico para incluir este parámetro tipo acotado, la compilación fallará, ya que nuestra invocación de inspect aún incluye una String: 

Box.java:21: <U>inspect(U) in Box<java.lang.Integer> cannot
  be applied to (java.lang.String)
                        integerBox.inspect("10");
                                  ^
1 error

Además de limitar los tipos que puede usar para instanciar un tipo genérico, los parámetros tipo acotados le permiten invocar métodos definidos en los límites: 

public class NumeroNatural<T extends Integer> {

    private T n;

    public NumeroNatural(T n)  { this.n = n; }

    public boolean esPar() {
        return n.intValue() % 2 == 0;
    }

    // ...
}

El método esPar invoca el método intValue definido en la clase Integer a través de n.

Límites Múltiples

El ejemplo precedente ilustra el uso de un parámetro tipo con un único límite, pero un parámetro tipo puede tener límites múltiples: 

<T extends B1 & B2 & B3>

Una variable tipo con múltiples límites es un subtipo de todos los tipos listados en el límite. Si uno de los límites es una clase, debe ser especificada primero. Por ejemplo: 

class A { /* ... */ }
interface B { /* ... */ }
interface C { /* ... */ }

class D <T extends A & B & C> { /* ... */ }

Si el límite A no es especificado primero, obtendrá un error en tiempo de compilación: 

class D <T extends B & A & C> { /* ... */ }  // error en tiempo de compilación
