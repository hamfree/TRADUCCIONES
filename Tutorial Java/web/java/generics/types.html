Tipos Genéricos

Un tipo genérico es una clase genérica o interfaz que es parametrizado sobre tipos. La siguiente clase Box será modificada para demostrar el concepto.

Una Clase Simple Box

Empecemos examinando una clase no genérica Box que opera sobre objetos de cualquier tipo. Sólo necesita establecer dos métodos: set, el cual añade un objeto a la caja, y get, el cual lo recupera: 

public class Box {
    private Object object;

    public void set(Object object) { this.object = object; }
    public Object get() { return object; }
}

Ya que sus métodos aceptan o devuelven un Object, usted es libre de pasarles lo que sea que quiera, siempre que no sean de uno de los tipos primitivos. No hay forma de verificar, en tiempo de compilación, cómo se usa la clase. Una parte del código puede colocar un Integer en la caja y esperar obtener Integers fuera de ella, mientras que la otra parte del código puede equivocadamente pasar una String, resultando en un error en tiempo de ejecución.

Una Versión Genérica de la Clase Box

Una clase genérica es definida con el siguiente formato: 

class nombre<T1, T2, ..., Tn> { /* ... */ }

La sección del parámetro de tipo, delimitado por paréntesis angular (<>), siguen al nombre de la clase. Especifica los parámetros de tipo (también llamados variables tipo) T1, T2,..., y Tn.

Para actualizar la clase Box para usar genéricos, cree una declaración de tipo genérico cambiando el código "public class Box" a "public class Box<T>". Esto introduce la variable tipo, T, que puede ser usada en cualquier parte dentro de la clase.

Con este cambio, la clase Box se convierte en : 

/**
 * Versión genérica de la clase Box.
 * @param <T> el tipo del valor a ser "encajado"
 */
public class Box<T> {
    // T significa "Tipo"
    private T t;

    public void set(T t) { this.t = t; }
    public T get() { return t; }
}

Como puede ver, todas las ocurrencias de Object son reemplazadas por T. Una variable tipo puede ser cualquier tipo no-primitivo que usted especifique: cualquier tipo de clase, cualquier tipo de interfaz, un tipo matriz, o incluso otra variable tipo.

Esta misma técnica puede ser aplicada para crear interfaces genéricas.

Convenciones de Nombrado de los Parámetros de Tipo

Por convención, los nombres de los parámetros tipo son letras únicas, en mayúsculas. Esto está en marcado contraste con la convención de nombrado de variables que ya conoce, y por una buena razón: sin ésta convención, sería difícil decir las diferencias entre una variable tipo y el nombre de una clase o interface ordinario.

Los nombres de parámetro tipo más comunmente usaos son:

  • E - Elemento (usado extensivamente por el Marco de Trabajo de Colecciones de Java)
  • K - Clave (Key en inglés)
  • N - Número
  • T - Tipo
  • V - Valor
  • S,U,V etc. - 2º, 3º, 4º tipos

Verá esos nombres a través del IPA de Java SE y el resto de esta lección.

Invocando e Instanciando un Tipo Genérico

Para hacer referencia a la clase genérica Box desde dentro de su código, debe realizar una invoación de tipo genérica, la cual reemplaza T con algún valor concreto, tal como un Integer: 

Box<Integer> integerBox;

Puede pensar de una invocación de tipo genérico como siendo similar a una invocación ordinaria de método, pero en vez de pasar un argumento a un método, usted está pasando un argumento tipo — Integer en este caso— a la clase Box en sí misma.

La Terminología Parámetro Tipo y ArguemntoTipo: Muchos desarrolladores usan el término "parámetro tipo" y "argumento tipo" de forma intercambiable, pero estos términos no son los mismos. Cuando se codifica, uno establece los argumentos tipo en orden a crear un tipo parametrizado. Así, la T en Foo<T> es un parámetro tipo y la String en Foo<String> f es un argumento de tipo. Esta lección observa esta definición cuando se usan estos términos.

Como cualquier otra declaración de variable, este código no crea en realidad un nuevo objeto Box. Simplemente declara que integerBox contendrá una referencia a "Box de Integer", el cual es cómo Box<Integer> es leído.

Una invocación de un tipo genérico es generalmente conocida como un tipo parametrizado.

Para instanciar esta clase, use la palabra clave new , como es habitual, pero coloque <Integer> entre el nombre de la clase y el paréntesis: 

Box<Integer> integerBox = new Box<Integer>();

El Diamante

En Java SE 7 y posteriores, puede reemplazar los argumentos de tipo requeridos para invocar el constructor de una clase genérica con un conjunto vacío de argumentos tipo (<>) siempre que el compilador pueda determinar, o inferir, los argumentos tipo del contexto.Este par de paréntesis angulares, <>, es informalmente llamado el diamante. Por ejemplo, puede crear una instancia de Box<Integer> con la siguiente sentencia: 

Box<Integer> integerBox = new Box<>();

Para más información sobre la notación diamante y la inferencia de tipos, vea Inferencia de Tipos.

Parámetros de Tipo Múltiples

Como se mencionó previamente, una clase genérica puede tener parámetros de tipo múltiples. Por ejemplo, la clase genérica ParOrdenado, que implementa el interfaz genérico Par interface: 

public interface Par<K, V> {
    public K getClave();
    public V getValor();
}

public class ParOrdenado<K, V> implements Par<K, V> {

    private K clave;
    private V valor;

    public ParOrdenado(K clave, V valor) {
	this.key = clave;
	this.value = valor;
    }

    public K getClave()	{ return clave; }
    public V getValor() { return valor; }
}

Las siguientes sentencias crean dos instancias de la clase ParOrdenado: 

Pair<String, Integer> p1 = new ParOrdenado<String, Integer>("Even", 8);
Pair<String, String>  p2 = new ParOrdenado<String, String>("hello", "world");

El código, new ParOrdenado<String, Integer>, instancia K como una String y V como un Integer. Por lo tanto, los tipos parámetro del constructor de ParOrdenado son String y Integer, respectivamente. Debido al autoboxing , es válido pasar una String y un int a la clase.

Como se mencionó en El Diamante, debido a que el compilador de Java puede inferir los tipos K y V desde la declaración ParOrdenado<String, Integer>, estas sentencias pueden ser acortadas usando la notación diamante: 

OrderedPair<String, Integer> p1 = new ParOrdenado<>("Even", 8);
OrderedPair<String, String>  p2 = new ParOrdenado<>("hello", "world");

Para crear un interfaz genérico, siga las mismas convenciones que para crear una clase genérica.

Tipos Parametrizados

Puede también substituir un parámetro de tipo (p.e., K o V) con un tipo parametrizado (p.e., List<String>). Por ejemplo, usando el ejemplo ParOrdenado<K, V>: 

ParOrdenado<String, Box<Integer>> p = new ParOrdenado<>("primes", new Box<Integer>(...));
