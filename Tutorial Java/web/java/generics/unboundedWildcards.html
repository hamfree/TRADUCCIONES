Comodines no Acotados

El tipo de comodín no acotado se especifica usando el carácter comodín (?), por ejemplo, List<?>. Esto es llamado una lista de tipo desconocido. Hay dos escenarios donde un comodín sin acotar es un enfoque útil:

  • Si está escribiendo un método que puede ser implementado usando funcionalidad establecida en la clase Object.
  • Cuando el código está usando métodos en la clase genérica que no dependen del parámetro tipo. Por ejemplo, List.size o List.clear. De hecho, Class<?> se usa tan a menudo porque la mayoría de los métodos en Class<T> no dependen de T.

Considere el siguiente método, imprimeLista: 

public static void imprimeLista(List<Object> list) {
    for (Object elem : list)
        System.out.println(elem + " ");
    System.out.println();
}

El objetivo de imprimeLista es imprimir una lista de cualquier tipo, pero falla en alcanzar el objetivo — sólo impriime una lista de instancias Object ; no puede imprimir List<Integer>, List<String>, List<Double>, y así, porque no son subtipos de List<Object>. Para escribir un método genérico imprimeLista , use List<?>: 

public static void imprimeLista(List<?> list) {
    for (Object elem: list)
        System.out.print(elem + " ");
    System.out.println();
}

Debido a que para cualquier tipo concreto A, List<A> es un subtipo de List<?>, puede usar imprimeLista para imprimir una lista de cualquier tipo: 

List<Integer> li = Arrays.asList(1, 2, 3);
List<String>  ls = Arrays.asList("uno", "dos", "tres");
imprimeLista(li);
imprimeLista(ls);
Nota: El método Arrays.asList es usando en los ejemplos a lo largo de ésta lección. Este método estático factoría convierte la matriz especificada y devuelve una lista de tamaño fijo.

Es importante tener en cuenta que List<Object> y List<?> no son lo mismo. Puede insertar un Object, o cualquier subtipo de Object, en una List<Object>. Pero solo puede insertar null en una List<?>. La sección Directrices para el Uso de Comodines tiene más información de cómo determinar qué clase de comodín, en su caso, debería ser usado en unadeterminada situación.
