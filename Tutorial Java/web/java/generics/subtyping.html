Comodines y Subtipado

Como se describió en Genéricos, Herencia, y Subtipos, las clases genéricas o interfaces no están relacionadas simplemente porque no hay una relación entre sus tipos. Sin embargo, puede usar comodines para crear una relación entre clases genéricas o interfaces.

Dadas las siguientes dos clases regulares (no genéricas): 

class A { /* ... */ }
class B extends A { /* ... */ }

Sería razonable escribir el siguiente código: 

B b = new B();
A a = b;

Este ejemplo muestra que la herencia de clases regulares sigue esta regla de derivación: class B es un subtipo de la clase A si B extiende A. Esta regla no se aplica a los tipos genéricos: 

List<B> lb = new ArrayList<>();
List<A> la = lb;   // error en tiempo de compilación

Dado que Integer es un subtipo de Number, ¿Cuál es la relación entre List<Integer> y List<Number>?

El padre común es List<?>.

Aunque Integer es un subtipo de Number, List<Integer> no es un subtipo de List<Number> y, en realidad, estos dos tipos no están relacionados. El padre común de List<Number> y List<Integer> es List<?>.

Con el fin de crear una relación entre estas clases de forma que el código puede acceder a los métodos de Number a través de los elementos de List<Integer>, use un comodín limitado superiormente: 

List<? extends Integer> intList = new ArrayList<>();
List<? extends Number>  numList = intList;  // OK. List<? extiende Integer> es un subtipo de List<? extends Number>

Ya que Integer es un subtipo de Number, y numList es una lista de objetos Number, una relación existe ahora entre intList (una lista de objetos Integer) y numList. El siguiente diagrama muestra las relaciones entre varias clases List declaradas con comodines limitados superior e inferiormente.
