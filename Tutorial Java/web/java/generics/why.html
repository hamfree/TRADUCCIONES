¿Porqué usar Genéricos?

En una palabra, los genéricos habilitan que los tipos (clases e interfaces) sean parámetros cuando se definen clases, interfaces y métodos. Al igual que los paráetros formales usados en las declaraciones de métodos, los parámetros de tipo establecen una forma por la que puede reutilizar el mismo código con diferentes entradas. La diferencia es que la entrada de los parámetros formales son valores, mientras que las entradas de los parámetros de tipos son tipos.

El código que usa genéricos tiene muchos beneficios sobre el código sin genéricos:

  • Comprobación de tipos más fuerte en tiempo de compilación.
    Un compilador de Java aplica una comprobación de tipos fuerte al código genérico y emite errores si el código viola la seguridad de los tipos. Arreglar errores en tiempo de compilación es más facil que arreglar errores en tiempo de ejecución, los cuales pueden ser difíciles de encontrar.

  • Eliminación de moldeos (conversiones de tipo mediante casting).
    El siguient trozo de código sin genéricos requiere moldeo: 
    List lista = new ArrayList();
lista.add("hola");
String s = (String) list.get(0);
    Cuando es reescrito para usar genéricos, el código no requiere moldeo: 
    List<String> lista = new ArrayList<String>();
list.add("hola");
String s = list.get(0);   // sin moldeo (cast)
  • Habilita a los programadores a implementar algoritmos genéricos.
    Usando genéricos, los programadores pueden implementar algoritmos genéricos que funcionan sobre colecciones de diferentes tipos, pueden ser personalizados, y son seguros para el tipado y fáciles de leer.
