Borrado de Métodos Genéricos

El compilador de Java también borra los parámetros de tipo en los argumentos de los métodos genéricos. Considere el siguiente método genérico: 

// Cuenta el número de ocurrencias de elem en anArray.
//
public static <T> int count(T[] anArray, T elem) {
    int cnt = 0;
    for (T e : anArray)
        if (e.equals(elem))
            ++cnt;
        return cnt;
}

Ya que T no es acotado, el compilador de Java lo reemplaza con Object: 

public static int count(Object[] anArray, Object elem) {
    int cnt = 0;
    for (Object e : anArray)
        if (e.equals(elem))
            ++cnt;
        return cnt;
}

Suponga que las siguientes clases están definidas: 

class Shape { /* ... */ }
class Circle extends Shape { /* ... */ }
class Rectangle extends Shape { /* ... */ }

Puede escribir un método genérico para dibujar diferentes formas: 

public static <T extends Shape> void draw(T shape) { /* ... */ }

El compilador de Java reemplaza T con Shape: 

public static void draw(Shape shape) { /* ... */ }
