Genéricos, Herencia, y Subtipos

Como ya sabe, es posible asignar un objeto de un tipo a otro objeto de otro tipo siempre que los tipos sean compatibles. Por ejemplo, puede asignar un Integer a un Object, ya que Object es uno de los supertipos de Integer: 

Object algunObjeto = new Object();
Integer algunEntero= new Integer(10);
algunObjeto = algunEntero;   // OK

En terminología orientada a objetos, esto es llamado una relación "es un". Desde que un Integer es una clase de Object, la asignación está permitida. Pero Integer es también una clase de Number, así que el siguiente código es válido también: 

public void algunMetodo(Number n) { /* ... */ }

algunMetodo(new Integer(10));   // OK
algunMetodo(new Double(10.1));   // OK

Lo mismo es también verdad con los genéricos. Puede realizar una invocación a un tipo genérico, pasando Number como su argumento tipo, y cualquier invocación subsecuente de add será permitida si el argumento es compatible con Number: 

Box<Number> box = new Box<Number>();
box.add(new Integer(10));   // OK
box.add(new Double(10.1));  // OK

Ahora considere el siguiente método: 

public void boxTest(Box<Number> n) { /* ... */ }

¿Qué tipo de argumento acepta? Mirando en su firma, puede ver que acepta un argumento simple cuyo tipo es Box<Number>. ¿Pero qué significa esto? ¿Se le permite pasar un Box<Integer> o Box<Double>, como podría esperar? La respuesta es "no", porque Box<Integer> y Box<Double> no son subtipos de Box<Number>.

Esto es un error común cuando se trata de la programación con genéricos, pero es un concepto importante a aprender.

Box<Integer> no es un subtipo de Box<Number> incluso aunque Integer es un subtipo de Number.

Nota: Dados los dos tipos concretos A y B (por ejemplo, Number e Integer), MyClass<A> no tiene relación con MyClass<B>, independientemente de si o no A y B están relacionados. El padre común de MyClass<A> y MyClass<B> es Object.

Para obtener información de cómo crear una relación a-lo-subtipo entre dos clases genéricas cuando los parámetros tipo están relacionados, vea Comodines y Subtipado .

Clases Genéricas y Subtipado

Puede derivar una clase genérica o interfaz extendiéndola o implementándola. La relación entre los tipos parámetro de una clase o interface y de los tipos parámetro de otra están determinados por las cláusulas extends y implements.

Usando las clases de Collections como un ejemplo, ArrayList<E> implementa List<E>, y List<E> extiende Collection<E>. Así que ArrayList<String> es un subtipo de List<String>, el cual es un subtipo de Collection<String>. Siempre y cuando no se varía el argumento de tipo, la relación de subtipos se conserva entre los tipos.

Un ejemplo de jerarquía de Collections

Ahora imagine que queremos definir nuestro propio interfaz de lista, PayloadList, que asocia un valor opcional del tipo genérico P con cada elemento. Su declaración podría ser como: 

interface PayloadList<E,P> extends List<E> {
  void setPayload(int index, P val);
  ...
}

Las siguientes parametrizaciones de PayloadList son subtipos de List<String>:

  • PayloadList<String,String>
  • PayloadList<String,Integer>
  • PayloadList<String,Exception>
