Comodines Acotados Inferiormente

La sección Comodines Acotados Superiormente muestra que un comodín acotado superiormente restringe el tipo desconocido a ser un tipo específico o un subtipo de ese tipo y es representado usando la palabra clave extends. De una forma similar, un comodín acotado inferiormente restringe el tipo desconocido a ser un tipo específico o un super tipo de ese tipo.

Un comodín acotado inferiormente es representado usando el carácter comodín ('?'), seguido por la palabra clave super, segudo por su límite inferior: <? super A>.

Nota: Puede especificar un límite superior para un comodín, o puede especificar un límite inferior, pero no puede especificar ambos.

Digamos que quiere escribir un método que coloque objetos Integer en una lista. Para maximizar la flexibilidad, le gustaría que el método trabajase sobre List<Integer>, List<Number>, y List<Object> — cualquier cosa que pueda contener valores Integer.

Para escribir el método que trabaja sobre listas de Integer y los supertipos de Integer, tales como Integer, Number, y Object, especificará List<? super Integer>. El término List<Integer> es mśs restrictivo que List<? super Integer> porque el primero coincide con una lista de tipo Integer solo, mientras que la segunda coincide una lista de cualquier tipo que sea un supertipo de Integer.

El siguiente código añade del número 1 al 10 al final de una lista: 

public static void añadeNumeros(List<? super Integer> list) {
    for (int i = 1; i <= 10; i++) {
        list.add(i);
    }
}

La sección Directrices para el Uso de Comodines porporciona orientación sobre cuándo utilizar comodines acotados superiormente y cuando usar comodines acotados inferiormente.
