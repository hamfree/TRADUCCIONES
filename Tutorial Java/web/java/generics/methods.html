Métodos Genéricos

Los métodos genéricos son métodos que introducen sus propios parámetros tipo. Esto es similar a declarar un tipo genérico, pero el alcance del parámetro de tipo está limitado al metodo donde es declarado. Los métodos genéricos estáticos y no estáticos están permitidos, como también los constructores genéricos de clase.

La sintaxis para un método genérico incluye un parámetro tipo, dentro de paréntesis angulares, y aparece antes del tipo de retorno del método. Para los métodos genéricos estáticos, la sección del parámetro tipo debe aparecer antes que el tipo de retorno del método.

La clase Util incluye una método genérico, compara, el cual compara dos objetos Par: 

public class Util {
    public static <K, V> boolean compara(Par<K, V> p1, Par<K, V> p2) {
        return p1.getClave().equals(p2.getClave()) &&
               p1.getValor().equals(p2.getValor());
    }
}

public class Par<K, V> {

    private K clave;
    private V valor;

    public Par(K clave, V valor) {
        this.clave = clave;
        this.valor = valor;
    }

    public void setClave(K clave) { this.clave = clave; }
    public void setValoe(V valor) { this.valor = valor; }
    public K getClave()   { return clave; }
    public V getValor() { return valor; }
}

La sintaxis completa para invocar este método sería: 

Par<Integer, String> p1 = new Par<>(1, "manzana");
Par<Integer, String> p2 = new Par<>(2, "pera");
boolean igual = Util.<Integer, String>compara(p1, p2);

El tipo ha sido establecido explícitamente, como es mostrado en negrita. Generalmente, esto puede obviarse y el compilador inferirá el tipo que es necesitado: 

Par<Integer, String> p1 = new Par<>(1, "manzana");
Par<Integer, String> p2 = new Par<>(2, "pera");
boolean igual = Util.compara(p1, p2);

Esta característica, conocida como inferencia de tipo, le permite invocar un método genérico como si fuera un método ordinario, sin especificar un tipo entre los paréntesis angulares. Este tópico es descrito en la siguiente sección, Inferencia de Tipos .
