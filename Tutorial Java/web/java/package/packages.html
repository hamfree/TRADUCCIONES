Creando y Usando Paquetes

Para hacer los tipos más fáciles de encontrar y usar, para evitar conflictos de nombrado, y controlar el acceso, los programadores juntan grupos de tipos relacionados en paquetes.

Definición: Un paquete es una agrupación de tipos relacionados ofreciendo protección de acceso y gestión del espacio de nombres. Note que los tipos se refieren a las clases, interfaces, enumeraciones y los tipos de anotación. Las enumeraciones y los tipos anotación son unas clases especiales de clases e interfaces, respectivamente, así que los tipos son a menudo referidos en esta lección simplemente como clases e interfaces.

Los tipos que son parte de la plataforma Java son miembros de varios paquetes que agrupan las clases por función: las clases fundamentales está en java.lang, las clasese para leer y escrbir (entrada y salida) están en java.io, y así. Puede poner sus tipos en paquetes también.

Suponga que escribe un grupo de clases que representan objetos gráficos, tales como círculos, rectángulos, líneas y puntos. También escribe una interfaz, Arrastrable, que las clases implementan si ellas pueden ser arrastradas con el ratón. 

//en el fichero Arrastrable.java
public interface Arrastrable {
    ...
}

//en el fichero Graphic.java
public abstract class Graphic {
    ...
}

//en el fichero Circle.java
public class Circle extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Rectangle.java
public class Rectangle extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Point.java
public class Point extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Line.java
public class Line extends Graphic
    implements Arrastrable {
    . . .
}

Usted agrupará estas clases y la interfaz en un paquete por varias razones, incluyendo las siguientes:

  • Usted y otros programadores pueden fácilmente determinar que estos tipos están relacionados.
  • Usted y otros programadores saben dónde encontrar los tipos que pueden ofrecer funciones relacionadas con los gráficos.
  • Los nombres de sus tipos no entrarán en conflicto con los nombres de tipo en otros paquetes porque el paquete crea un nuevo espacio de nombres.
  • Puede permitir a los tipos dentro del paquete tener acceso sin restricciones unos a otros aún que se restrinja el acceso para los tipos fuera del paquete.
