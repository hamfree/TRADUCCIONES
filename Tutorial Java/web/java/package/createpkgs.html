Creando un Paquete

Para crear un paquete, elije un nombre para el paquete (las convenciones de nombrado son descritas en la siguiente sección) y coloca una sentencia package con el nombre en la parte de arriba de cada fichero fuente que contiene los tipos (clases, interfaces, enumeraciones, y anotaciones) que quiere incluir en el paquete.

La sentencia package (por ejemplo, package graphics;) debe ser la primera línea en el fichero fuente, Sólo puede haber una sentencia package por cada fichero fuente, y se aplica a todos los tipos que hay en el fichero.

Nota: Si coloca múltiples tipos en un único fichero fuente, sólo uno puede ser public, y debe tener el mismo nombre que el fichero fuente. Por ejemplo, puede definir public class Circle en el fichero Circle.java, definir public interface Arrastrable en el fichero Arrastrable.java, definir public enum Day en el fichero Day.java, Etcétera.

Puede incluir tipos no públicos en el mismo fichero que un tipo público (esto está fuertemente desaconsejado, a menos que los tipos no públicos sean pequeños y estrechamente relacionados con el tipo público), pero sólo el tipo público será accesible desde fuera del paquete. Todos los tipos no públicos del nivel superior, serán privados del paquete.

Si coloca el interfaz de los gráficos y las clases listadas en la sección precedente en un paquete llamado graphics, necesitará seis ficheros fuente, como estos: 

//en el fichero Arrastrable.java
package graphics;
public interface Arrastrable {
    . . .
}

//en el fichero Graphic.java
package graphics;
public abstract class Graphic {
    . . .
}

//en el fichero Circle.java
package graphics;
public class Circle extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Rectangle.java
package graphics;
public class Rectangle extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Point.java
package graphics;
public class Point extends Graphic
    implements Arrastrable {
    . . .
}

//en el fichero Line.java
package graphics;
public class Line extends Graphic
    implements Arrastrable {
    . . .
}

Si no usa una sentencia package, su tipo acabará en un paquete sin nombre. Generalmente hablando, un paquete sin nombre es sólo para aplicaciones pequeñas o temporales o cuando está justo empezando el proceso de desarrollo. De otra forma, las clases e interfaces pertenecen a paquetes nombrados.
