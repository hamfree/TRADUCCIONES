Gestionando Ficheros Fuentes y de Clases

Muchas implementaciones de la plataforma Java confían en los sistemas de ficheros jerárquicos para gestionar los ficheros fuente y de clases, aunque La Especificación del Lenguaje Java no requiere esto. La estrategia es como sigue.

Coloque el código fuente para una clase, interfaz, enumeración, o tipo de anotación en un fichero de texto cuyo nombre es el nombre simple del tipo y cuya extensión es .java. Por ejemplo: 

//en el fichero Rectangle.java
package graphics;
public class Rectangle {
   ... 
}

Entonces, coloque el fichero fuente en un directorio cuyo nombre refleje el nombre del paquete al cual el tipo pertenece: 

.....\graphics\Rectangle.java

El nombre cualificado del miembro del paquete y el nombre de la ruta al fichero son paralelos, asumiento que el separador de nombres de fichero de Microsoft Windows es la barra invertida (para UNIX, se usa la barra inclinada).

  • Nombre de clase – graphics.Rectangle
  • Camino al fichero – graphics\Rectangle.java

Como usted debe recordar, por convención una compañía usa su nombre de dominio de Internet invertido para sus nombres de paquetes. La compañía Ejemplo, cuyo dominio de Internet es ejemplo.com, precederá todos sus nombres de paquetes con com.ejemplo. Cada componenete del nombre del paquete corresponde a un subdirectorio. Así, si la compañía Ejemplo tiene un paquete com.ejemplo.graphics que contiene un fichero fuente Rectangle.java, estará contenido en una serie de subdirectorios como este: 

....\com\ejemplo\graphics\Rectangle.java

Cuando compila un fichero fuente, el compilador crea un fichero de salida diferente para cada tipo definido en él. El nombre base del fichero de salida es el nombre del tipo, y su extensión es .class. Por ejemplo, si el fichero fuente es como este 

//en el fichero Rectangle.java
package com.ejemplo.graphics;
public class Rectangle {
      . . . 
}

class Helper{
      . . . 
}

entonces los ficheros compilados serán colocados en: 

<ruta al directorio padre de los ficheros de salida>\com\ejemplo\graphics\Rectangle.class
<ruta al directorio padre de los ficheros de salida>\com\ejemplo\graphics\Helper.class

Como los ficheros fuente .java, los ficheros compilados .class estarán en una serie de directorios que reflejarán el nombre del paquete. Sin embargo, la ruta a los ficheros .class no tiene que ser la misma ruta de los ficheros fuente .java. Puede organizar sus directorios fuente y de clases separadamente, como: 

<ruta_uno>\sources\com\ejemplo\graphics\Rectangle.java

<ruta_dos>\classes\com\ejemplo\graphics\Rectangle.class

Haciendo esto, puede dar el directorio de classes a otros programadores sin revelar sus fuentes. También necesita gestionar los ficheros fuente y class de esta manera para que el compilador y la Máquina Virtual de Java (MVJ, JVM en inglés) pueda encontrar todos los tipos que su programa usa.

La ruta completa al directorio de classes , <ruta_dos>\classes, es llamada class path, y es establecida con la variable del sistema CLASSPATH. Tanto el compilador como la MVJ construyen la ruta a sus ficheros .class añadiendo el nombre del paquete al class path. Por ejemplo, si 

<ruta_dos>\classes

es su class path, y el nombre del paquete es 

com.ejemplo.graphics,

entonces el compilador y la MVJ buscarán >los ficheros .class en 

<ruta_dos>\classes\com\ejemplo\graphics.

Un class path puede incluir varias rutas, separadas por punto y coma (Windows) o dos puntos (UNIX). Por defecto, el compilador y la MVJ buscan el directorio actual y el fichero JAR conteniendo las clases de la plataforma Java así que estos directorios están automáticamente en su class path.

Estableciendo la Variable del Sistema CLASSPATH

Para mostrar la variable actual CLASSPATH, use estos comandos en Windows y UNIX (shell Bourne): 

En Windows:   C:\> set CLASSPATH
En UNIX:      % echo $CLASSPATH

Para borrar los contenidos actuales de la variable CLASSPATH, use estos comandos: 

En Windows:   C:\> set CLASSPATH=
En UNIX:      % unset CLASSPATH; export CLASSPATH

Para establecer la variable CLASSPATH, use estos comandos (por ejemplo): 

En Windows:   C:\> set CLASSPATH=C:\users\jorge\java\clases
En UNIX:      % CLASSPATH=/home/jorge/java/clases; export CLASSPATH
