Usando Miembros del Paquete

Los tipos que componen un paquete son conocidos como los miembros del paquete.

Para usar un miembro public del paquete desde fuera de su paquete, debe realizar una acción de las siguientes:

  • Referirse al miembro por su nombre completamente cualificado
  • Importar el miembro del paquete
  • Importar el paquete entero del miembro

Cada una es apropiada para situaciones diferentes, como se explica en las secciones que siguen.

Referirse a un Miembro del Paquete por Su Nombre Cualficado

Hasta ahora, la mayoría de los ejemplos en este tutorial se han referido a tipos por sus nombres simples, tales como Rectangle y StackOfInts. Puede usar el nombre simple de un miembro de paquete si el código que está escribiendo está en el mismo paquete que ese miembro o si ese miembro ha sido importado.

Sin embargo, si está intentando usar un miembro de un paquete diferente y ese paquete no ha sido importado, debe usar el nombre completamente cualificado del miembro, que incluye el nombre del paquete. Aquí tiene el nombre cualificado completo de la clase Rectangle declarado en el paquete graphics en el ejemplo previo. 

graphics.Rectangle

Podría usar este nombre cualificado para crear una instancia de graphics.Rectangle: 

graphics.Rectangle miRect = new graphics.Rectangle();

Los nombres cualificados están bien para un uso poco frecuente. Cuando un nombre es usado repetitivamente, sin embargo, escribir el nombre repetidamente se vuelve tedioso y el código se vuelve difícil de leer. Como una alternativa, puede importar el miembro o su paquete y entonces usar su nombre simple.

Importar un Miembro del Paquete

Para importar un miembro específico en el fichero actual, coloque una sentencia import al principio del fichero antes de cualquier definición de tipos pero después de la sentencia package, si hay una. Aquí tiene cómo importaría la clase Rectangle del paquete graphics creado en la sección previa. 

import graphics.Rectangle;

Ahora puede referirse a la clase Rectangle por su nombre simple. 

Rectangle miRectangulo = new Rectangle();

Este enfoque funciona bien si usted usa tan sólo unos pocos miembros del paquete graphics. Pero si usted usa muchos tipos de un paquete, se debe importar el paquete entero.

Importar un Paquete Entero

Para importar todos los tipos contenidos en un paquete particular, use la sentencia import con el caracter comodín asterisco (*). 

import graphics.*;

Ahora puede referirse a cualquier clase o interfaz en el paquete graphics por su nombre simple. 

Circle miCirculo = new Circle();
Rectangle miRectangulo = new Rectangle();

El asterisco en la sentencia import puede ser usado sólo para especificar todas las clases dentro de un paquete, como se mostró aquí. No puede ser usada para que coincida con un subconjunto de clases en el paquete. Por ejemplo, lo siguiente no coincide con todas las clases en el paquete graphics que empiezan por A. 

// no funciona
import graphics.A*;

En vez de eso, genera un error del compilador. Con la sentencia import, generalmente importa sólo un miembro único del paquete o un paquete entero.

Nota: Otra, forma menos común de import le permite importar las clases públicas anidadas de una clase envolvente. Por ejemplo, si la clase graphics.Rectangle contiene clases anidades útiles, tales como Rectangle.DoubleWide y Rectangle.Square, podría importar Rectangle y sus clases anidadas usando las siguientes dos sentencias. 
import graphics.Rectangle;
import graphics.Rectangle.*;
Tenga en cuenta que la segunda sentencia import no importa Rectangle.

Otra forma menos común de import, la sentencia static import, será descrita al final de esta sección.

Por conveniencia, el compilador de Java automáticamente importa dos paquetes enteros para cada fichero fuente: (1) el paquete java.lang y (2) el paquete actual (el paquete para el fichero actual).

Jerarquías Aparentes de los Paquetes

De primeras, los paquetes parecen ser jerárquicos, pero no lo son. Por ejemplo, la IPA de Java incluye un paquete java.awt, un paquete java.awt.color, un paquete java.awt.font, y muchos otros que empiezan con java.awt. Sin embargo, el paquete java.awt.color, el paquete java.awt.font, y otros paquetes java.awt.xxxx no están incluídos en el paquete java.awt. El prefijo java.awt (Conjunto de Herramientas Abstractas de Ventanas de Java) es usado por un número de paquetes relacionados para hacer la relación evidente, pero no muestran inclusión.

Importar java.awt.* importa todos los tipos en el paquete java.awt, pero no importan java.awt.color, java.awt.font, o cualquier otro paquete java.awt.xxxx. Si planea usar las clases y otros tipos en java.awt.color también como aquellos en java.awt, debe importar ambos paquetes con todos sus ficheros: 

import java.awt.*;
import java.awt.color.*;

Ambigüedades de Nombre

Si un miembro en un paquete comparte su nombre con un miembro en otro paquete y ambos paquetes son importados, debe referirse a cada miembro por su nombre cualificado. Por ejemplo, el paquete graphics definió una clase llamada Rectangle. El paquete java.awt también contiene una clase Rectangle. Si ambos paquetes graphics y java.awt han sido importados, lo siguiente es ambiguo. 

Rectangle rect;

En tal situación, tiene que usar el nombre completamente cualificado del miembro para indicar exactamente qué clase Rectangle quiere. Por ejemplo, 

graphics.Rectangle rect;

La Sentencia Static Import

Hay situaciones donde necesita acceso frecuente a campos finales estáticos (constantes) y métodos estáticos de una o dos clases. Usar el prefijo del nombre de éstas clases una y otra vez puede resultar en código desordenado. La sentencia static import le da una forma de importar las constantes y métodos estáticos que quiere usar de forma que no necesita usar el prefjo del nombre de sus clases.

La clase java.lang.Math define la constante PI y muchos métodos estáticos, incluyendo métodos para calcular senos, cosenos, tangentes, raíces cuadradas, máximos, mínimos, exponentes, y mucho más. Por ejemplo, 

public static final double PI 
    = 3.141592653589793;
public static double cos(double a)
{
    ...
}

Ordinariamentte, al usar estos objetos desde otras clases, usted usa el prefijo del nombre de la clase, como sigue. 

double r = Math.cos(Math.PI * theta);

Puede usar la sentencia static import para importar los miembros estáticos de java.lang.Math de forma que no necesita usar el prefijo del nombre de la clase, Math. Los miembros estáticos de Math pueden ser importados ya sea individualmente: 

import static java.lang.Math.PI;

o como un grupo: 

import static java.lang.Math.*;

Una vez han sido importados, los miembros estáticos pueden ser usados sin cualificación. Por ejemplo, el trozo de código previo se convertirá en: 

double r = cos(PI * theta);

Obviamente, puede escribir sus propias clases que contengan constantes y métodos estáticos que use frecuentemente, y entonces usar la sentencia static import. Por ejemplo, 

import static mipaquete.MisConstantes.*;
Nota: Use la importación estática muy escasamente. El uso excesivo de la importación estática puede resultar en código que es difícil de leer y mantener, porque los lectores del código no sabrán qué clases definen un objeto estático particular. Usada apropiadamente, la importación estática hace el código más legible mediante la eliminación de las repeticiones de nombres de clases.
