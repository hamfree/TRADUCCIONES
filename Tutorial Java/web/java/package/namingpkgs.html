Nombrando un Paquete

Con los programadores del mundo entero escribiendo clases e interfaces usando el lenguaje de programación Java, es muy probable que muchos programadores usarán el mismo nombre para los diferentes tipos. En realidad, el ejemplo previo hace justo eso: define una clase Rectangle cuando hay ya una clase Rectangle en el paquete java.awt. Sin embargo, el compilador permite ambas clases a tener el mismo nombre si ellas están en paquetes diferentes. El nombre completamente cualificado de cada clase Rectangle incluye el nombre de paquete. Esto es, el nombre completamente cualificado de la clase Rectangle en el paquete graphics es graphics.Rectangle, y el nombre completamente cualificado de la clase Rectangle en el paquete java.awt es java.awt.Rectangle.

Esto funciona bien a menos que dos programadores independientes usen el mismo nombre para sus paquetes. ¿Qué es lo que previene este problema?. La convención (normas arbitrarias que siguen todos los programadores).

Convenciones de Nombres

Los nombres de paquetes están escritos en letras minúsculas para evitar conflictos con los nombres de las clases o interfaces.

Las compañías usan su nombre de dominio de Internet invertido para empezar sus nombres de paquete — por ejemplo com.ejemplo.mipaquete para un paquete llamado mipaquete creado por un programador en ejemplo.com.

Las colisiones de nombres que ocurren dentro de una única compañía necesitan ser manejadas por convención dentro de esa compañía, quizás incluyendo la región del nombre del proyecto después del nombre de la compañía (por ejemplo, com.ejemplo.region.mipaquete).

Los paquetes del lenguaje Java en sí mismos empiezan con java. o javax.

En algunos casos, el nombre de dominio de internet puede no ser un nombre válido de paqute. Estop puede ocurrir si el nombre de dominio contiene un guión u otro carácter especial, si el paquete empieza con un dígito u otro caracteres que es ilegal usar como principio de un nombre Java, o si el nombre del paquete contiene una palabra clave reservada de Java, tal como "int". En este evento, la convención sugerida es añadir un subrayado. Por ejemplo:

Legalizar Nombres de Paquete
Nombre de Dominio Prefijo de Nombre de Paquete
hyphenated-name.example.org org.example.hyphenated_name
example.int int_.example
123name.example.com com.example._123name
