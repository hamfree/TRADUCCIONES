Operadores de Igualdad, Relacionales y Condicionales

Los Operadores Relacionales y de Igualdad

Los operadores relacionales y de igualdad determinan si un operando es mayor que, menor que, igual a, o no es igual a otro operando. La mayoría de estos operadores probablemente le resultará familiar a usted también. Tenga en mente que debe usar "==", no "=", cuando comprueba si dos valores primitivos son iguales. 

==      igual a
!=      no igual a
>       mayor que
>=      mayor que o igual que
<       menor que
<=      menor que o igual que

El siguiente programa, ComparisonDemo , prueba los operadores de comparación: 


class ComparisonDemo {

    public static void main(String[] args){
        int valor1 = 1;
        int valor2 = 2;
        if(valor1 == valor2)
            System.out.println("valor1 == valor2");
        if(valor1 != valor2)
            System.out.println("valor1 != valor2");
        if(valor1 > valor2)
            System.out.println("valor1 > valor2");
        if(valor1 < valor2)
            System.out.println("valor1 < valor2");
        if(valor1 <= valor2)
            System.out.println("valor1 <= valor2");
    }
}

Output: 

valor1 != valor2
valor1 <  valor2
valor1 <= valor2

Los Operadores Condicionales

Los operadores && y || ejecutan operaciones Condicional-AND y Condicionales-OR sobre dos expresiones booleanas. Estos operadores exhiben un comportamiento de "circuito-corto" , lo cual significa que el segundo operando es evaluado sólo si se necesita. 

&& Condicional-AND
|| Condicional-OR

El siguiente programa, ConditionalDemo1 , prueba estos operadores: 


class ConditionalDemo1 {

    public static void main(String[] args) {
        int valor1 = 1;
        int valor2 = 2;
        if ((valor1 == 1) && (valor2 == 2)) {
            System.out.println("valor1 es 1 Y (AND) valor2 es 2");
        }
        if ((valor1 == 1) || (valor2 == 1)) {
            System.out.println("valor1 es 1 O (OR) valor2 es 1");
        }
    }
}

Otro operador condicional es ?:, el cual puede pensarse como un atajo para la sentencia if-then-else (descrita en la sección Sentencias de Control de Flujo de ésta lección). Éste operador es también conocido como el operador ternario porque usa tres operandos. En el siguiente ejemplo, este operador será leido como: "Si algunaCondición es verdad, asigna el valor de valor1 a resultado. De otra forma, asigna el valor de valor2 a resultado."

El siguiente programa, ConditionalDemo2 , comprueba el operador?: : 


class ConditionalDemo2 {

    public static void main(String[] args){
        int valor1 = 1;
        int valor2 = 2;
        int resultado;
        boolean algunaCondicion = true;
        resultado = algunaCondicion ? valor1 : valor2;

        System.out.println(resultado);
    }
}

Debido a que algunaCondicion es verdad, este programa imprime "1" en pantalla. Use el operador ?: en vez de una sentencia if-then-else si hace que su código sea más legible; por ejemplo, cuando las expresiones son compactas y sin efectos-laterales (como las asignaciones).

El Operador instanceof para Comparación de Tipos

El operador instanceof compara si un objeto es de un tipo especificado. Pude usarlo para comprobar si un objeto es una instancia de una clase, una instancia de una subclase, o una instancia de una clase que implementa un interfaz concreto.

El siguiente programa, InstanceofDemo , define una clase padre (nombrada Padre), un interfaz simple (nombrado MiInterfaz), y una clase hija (nombrada Hijo) que hereda del padre e implementa la interfaz. 


class InstanceofDemo {
    public static void main(String[] args) {

        Padre obj1 = new Padre();
        Padre obj2 = new Hijo();

        System.out.println("obj1 instanceof Padre: "
            + (obj1 instanceof Padre));
        System.out.println("obj1 instanceof Hijo: "
            + (obj1 instanceof Hijo));
        System.out.println("obj1 instanceof MiInterfaz: "
            + (obj1 instanceof MiInterfaz));
        System.out.println("obj2 instanceof Padre: "
            + (obj2 instanceof Padre));
        System.out.println("obj2 instanceof Hijo: "
            + (obj2 instanceof Hijo));
        System.out.println("obj2 instanceof MiInterfaz: "
            + (obj2 instanceof MiInterfaz));
    }
}

class Padre {}
class Hijo extends Padre implements MiInterfaz {}
interface MiInterfaz {}

Salida: 

obj1 instanceof Padre: true
obj1 instanceof Hijo: false
obj1 instanceof MiInterfaz: false
obj2 instanceof Padre: true
obj2 instanceof Hijo: true
obj2 instanceof MiInterfaz: true

Cuando use el operador instanceof, tenga en mente que null no es una instancia de nada.
