Sentencias de Bifurcación

La Sentencia break

La sentencia break tiene dos formas: con etiqueta y sin etiqueta. Usted vio la forma sin etiqueta en la presentación previa de la sentencia switch. Puede usar también un break sin etiqueta para terminar un bucle for, while, o do-while, como se muestra en el siguiente programa: BreakDemo 

class BreakDemo {
    public static void main(String[] args) {

        int[] matrizDeEnteros = { 32, 87, 3, 589, 12, 1076,
                              2000, 8, 622, 127 };
        int numeroBuscado = 12;

        int i;
        boolean encontrado = false;

        for (i = 0; i < matrizDeEnteros.length; i++) {
            if (matrizDeEnteros[i] == numeroBuscado) {
                encontrado = true;
                break;
            }
        }

        if (encontrado) {
            System.out.println("Encontrado " + numeroBuscado
                               + " en el índice " + i);
        } else {
            System.out.println(numeroBuscado
                               + " no está en la matriz.");
        }
    }
}

Este programa busca el número 12 en una matriz. La sentencia break, mostrada en negrita, termina el terminates the for loop when that value is found. Control flow then transfers to the statement after the for loop. This program's output is: 

Encontrado 12 en el índice 4

Una sentencia break sin etiqueta termina la sentencia switch, for, while, o do-while más interior, pero una sentencia break con etiqueta termina una sentencia más externa. El siguiente programa, BreakWithLabelDemo , es similar al programa previo, pero usa bucles anidados for para buscar un valor en una matriz bidimensional. Cuando el valor es encontrado, una sentencia break con etiqueta termina el bucle for más externo (etiquetado "busqueda"): 


class BreakWithLabelDemo {
    public static void main(String[] args) {

        int[][] matrizDeEnteros = {
            { 32, 87, 3, 589 },
            { 12, 1076, 2000, 8 },
            { 622, 127, 77, 955 }
        };
        int numeroBuscado = 12;

        int i;
        int j = 0;
        boolean encontrado = false;

    busqueda:
        for (i = 0; i < matrizDeEnteros.length; i++) {
            for (j = 0; j < matrizDeEnteros[i].length;
                 j++) {
                if (matrizDeEnteros[i][j] == numeroBuscado) {
                    encontrado = true;
                    break busqueda;
                }
            }
        }

        if (encontrado) {
            System.out.println("Encontrado " + numeroBuscado + " en " + i + ", " + j);
        } else {
            System.out.println(numeroBuscado + " no est\u00E1 en la matriz");
        }
    }
}

Esta es la salida del programa. 

Encontrado 12 en 1, 0

La sentencia break termina la sentencia con etiqueta; no transfiere el flujo de control a la etiqueta. El flujo de control es transferido a la sentencia inmediatamente siguiente a la sentencia etiquetada (terminada).

La sentencia continue

La sentencia continue salta la iteración actual de un bucle for, while , o do-while. La forma sin etiqueta salta al final del cuerpo más interior y evalúa la expresión booleana que controla el bucle. El siguiente programa, ContinueDemo , recorre una String, contando las ocurrencias de la letra "p". Si el caracter actual no es una p, la sentencia continue se salta el resto del bucle y procede al siguiente caracter. Si este es una "p", el programa incrementa el contador de la letra. 


class ContinueDemo {
    public static void main(String[] args) {

        String buscaMe = "porque puedo, puedes. porque puedes, puedo. pero si no puedes, yo tampoco puedo.";
        int maximo = buscaMe.length();
        int numeroDePes = 0;

        for (int i = 0; i < maximo; i++) {
            // interesados solo en las pes
            if (buscaMe.charAt(i) != 'p')
                continue;

            // procesa las pes
            numeroDePes++;
        }
        System.out.println("Encontradas " + numeroDePes + " pes en la cadena.");
    }
}

Aquí está la salida de este programa: 

Encontradas 10 pes en la cadena.

Para ver este efecto más claramente, intente eliminar la sentencia continue y recompile. Cuando ejecute el programa de nuevo, el contador estará equivocado, indicando que ha encontrado 70 pes en vez de 10.

Una setencia continue con etiqueta se salta la iteración actual de un bucle más externo marcado con la etiqueta dada. El siguiente programa de ejemplo, ContinueWithLabelDemo, usa bucles anidados para buscar una subcadena en otra cadena. Los dos bucles anidados son obligatorios: uno para iterar sobre la subcadena y otro para iterar sobre la cadena en la que se busca. El siguiente programa, ContinueWithLabelDemo , usa la forma con etiqueta de continue para saltar una iteración en el bucle más exterior. 


class ContinueWithLabelDemo {
    public static void main(String[] args) {

        String buscaMe = "Buscar una subcadena en m\u00ED";
        String subCadena = "sub";
        boolean encontrada = false;

        int max = buscaMe.length() -
                  subCadena.length();

    test:
        for (int i = 0; i <= max; i++) {
            int n = subCadena.length();
            int j = i;
            int k = 0;
            while (n-- != 0) {
                if (buscaMe.charAt(j++) != subCadena.charAt(k++)) {
                    continue test;
                }
            }
            encontrada = true;
                break test;
        }
        System.out.println(encontrada ? "Encontrada" : "No encontrada");
    }
}

Aquí está la salida de este programa. 

Encontrada

La Sentencia return

La última de las sentencias de bifurcación es la sentencia return. La sentencia return se sale del método actual, y el flujo de control vuelve a donde el método fue llamado. La sentencia return tien dos formas: una que devuelve un valor, y otra que no lo hace. Para devolver un valor, simplemente ponga el valor (o una expresión que calcule el valor) después de la palabra clave return. 

return ++contador;

El tipo de dato del valor retornado tiene que coincidir con el tipo declarado de valor de retorno del método. Cuando un método es declarado void, use la forma de return que no devuelve valor. 

return;

La lección Clases y Objetos cubrírá todo lo que necesita saber sobre la escritura de métodos.
