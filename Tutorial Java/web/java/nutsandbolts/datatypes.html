Tipos de Datos Primitivos

El lenguaje de programación Java es de tipado estático, lo que significa que todas las variables tienes que ser primero declaradas antes de que puedan ser utilizadas. Esto implica indicar el tipo de la varible y nombre, como ya ha visto: 

int marcha = 1;

Si lo hace, le dice a su programa que un campo llamado "marcha" existe, contiene datos numéricos, y tiene un valor inicial de "1". Un tipo de dato de una variable determina los valores que puede contener, más las operaciones que pueden ser realizadas con ella. Además de int, el lenguaje de programación Java soporta otros siete tipos de datos primitivos. Un tipo primitivo está predefinido por el lenguaje y es nombrado por una palabra clave reservada. Los valores primitivos no comparten el estado con otros valores primitivos. Los ocho tipos de datos primitivos soportados por el lenguaje de programación Java son:

  • byte: El tipo de dato byte es un entero de 8 bits con signo con complemento a dos. Tiene un valor mínimo de -128 y un valor máximo de 127 (inclusive). El tipo de dato byte puede ser útil para ahorrar memoria en grandes vectores , donde los ahorros de memoria realmente importan. Pueden también ser utilizados en vez de int donde sus límites ayudan a clarificar su código; el hecho de que el rango de una variable esté limitado puede servir como una forma de documentación.

  • short: El tipo de dato short es un entero de 16 bits con signo y complemento a dos. Tiene un valor mínimo de -32.768 y un valor máximo de 32.767 (inclusive). Como con byte, las mismaqs reglas se aplican: puede usar un short para ahorrar memoria en vectores grandes, en situaciones donde los ahorros de memoria importan realmente.

  • int: Por defecto, el tipo de dato int es un entero de 32 bits con signo y complemento a dos, el cual tiene un valor mínimo de -231 y un valor máximo de 231-1. En Java SE 8 y posteriores, puede usar el tipo de dato int para representar un entero de 32 bits sin signo, el cual tiene un valor mínimo de 0 y un valor máximo de 232-1. Utilize la clase Integer para utilizar el tipo de dato int como un entero sin signo. Vea la sección Las Clases de los Números para más información. Métodos estáticos como compareUnsigned, divideUnsigned etc. han sido añadidos a la clase Integer para soportar las operaciones arítmeticas en enteros sin signo.

  • long: El tipo de dato long es un entero de 64 bits con complemento a dos. El long con signo tiene un valor mínimo de -263 y un valor máximo de 263-1. En Java SE 8 y más tarde, puede usar el tipo de dato long para representar un long de 64 bits sin signo, el cual tiene un valor mínimo de 0 y un valor máximo de 264-1. Utilize este tipo de datos cuando necesite necesite un rango de valores más amplio que el que ofrece el tipo int. La clase Long también contiene métodos como compareUnsigned, divideUnsigned etc para soportar operaciones aritméticas para los tipos long sin signo.

  • float: El tipo de dato float es un IEEE 754 de punto flotante de 32 bits y precisión simple. Su rango de valores está más alla del alcance de ésta exposición, pero está especificado en la sección Tipos de Punto Flotante, Formatos, y Valores de la Especificación de Lenguaje Java. Como con las recomendaciones para byte y short, utilize un tipo float (en vez de double) si usted necesita ahorrar memoria en grandes vectores de números en punto flotante. Este tipo de dato nunca debería ser usado para valores precisos, como los monetarios. Para eso, usted necesitará utilizar la clase java.math.BigDecimal en su lugar. Números y Cadenas cubre BigDecimal y otras clases útiles ofrecidas por la plataforma Java.

  • double: El tipo de dato double es un IEEE 754 de punto flotante de 64 bits y precisión doble. Su rango de valores está más alla del alcance de ésta exposición, pero está especificado en la sección Tipos de Punto Flotante, Formatos, y Valores de la Especificación del Lenguaje Java. Para valores decimales, este tipo de dato es generalmente la elección por defecto. Como se mencionó arriba, este tipo de dato nunca debería ser utilizado para valores precisos, como los monetarios.

  • boolean: El tipo de dato boolean tiene sólo dos posibles valores: true y false. Utilize este tipo de datos para banderas simples que monitorizen condiciones verdadero/falso. Este tipo de datos representa un bit de información, pero su "tamaño" no es algo que está precisamente definido.

  • char: El tipo de dato char es un carácter Unicode de 16 bit simple. Tiene un valor mínimo de '\u0000' (o 0) y un valor máximo de '\uffff' (o 65.535 inclusive).

Además de los ocho tipos de datos primitivos listados arriba, el lenguaje de programación Java también ofrece un soporte especial a las cadenas de caracteres vía la clase java.lang.String . Encerrar su cadena de caracteres entre dobles comillas automáticamente creará un nuevo objeto String; por ejemplo, String s = "esto es una cadena";. Los objetos String son inmutables, lo cual significa que una vez creado, sus valores no pueden ser cambiados. La clase String no es técnicamente un tipo de dato primitivo, pero dado el soporte especial que se le da por el lenguaje, usted probablemente tenderá a pensar de ello como tal. Usted aprenderá másw sobre la clase String en Números y Clases

Valores por Defecto

No es siempre necesario asignar un valor cuando un campo es declarado. Los campos que son declarados pero no inicializados serán establecidos a un valor por defecto razonable por el compilador. Generalmente hablando, éste valor por defecto será 0 o null, dependiendo de los tipos de datos. Basarse en estos tipos de dato por defecto, sin embargo, es generalmente considerado un mal estilo de programación.

El siguiente cuadro resume los valores por defecto para los tipos de datos anteriores.

Valores por defecto para tipos primitivos
Tipo de Dato Valor por Defecto (para campos)
byte 0
short 0
int 0
long 0L
float 0.0f
double 0.0d
char '\u0000'
String (o cualquier otro objeto)   null
boolean false

Las variables locales son ligeramente diferente; el compilador nunca asigna un valor por defecto a una variable local no inicializado. Si usted no puede inicializar su variable local donde fue declarada, asegúrese de asignar un valor antes de que intente usarla. Acceder una variable local no inicializada resultará en un error en tiempo de compilación.

Literales

Puede haber notado que la palabra clave new no es usada cuando inicializamos una variable de un tipo primitivo. Los tipos primitivos son tipos de datos especiales contruidos dentro del lenguaje; no son objetos creados desde una clase. Un literal es la representación en el código fuente de un valor fijo; los literales están representados directamente en su código sin requerir computación. Como se muestra abajo es posible asignar un literal a una variable de un tipo primitivo: 

boolean result = true;
char capitalC = 'C';
byte b = 100;
short s = 10000;
int i = 100000;

Literales de Enteros

Un literal de entero es del tipo long si termina con la letra L o l; de otra forma es del tipo int. Se recomienda que utilize la letra en mayúsculas L porque la letra en minúsculas l es difícil de distinguir del dígito 1.

Loa valores de los tipos integrales byte, short, int, y long pueden ser creados desde literales int. Los valores de tipo long que exceden el rango de los int pueden ser creados desde literales long. Los literales de enteros pueden ser expresados por éstos sistemas numéricos:

  • Decimal: Base 10, cuyos dígitos consisten de los números 0 al 9; éste es el sistema numérico que usted utiliza cada día
  • Hexadecimal: Base 16, cuyos dígitos consisten de los números 0 al 9 y de las letras A a la F
  • Binario: Base 2, cuyos dígitos consisten de los números 0 y 1 (usted puede crear literales binarios en Java SE 7 y posteriores)

Para la programación de propósito general, el sistema decimal sea probablemente el único sistema numérico que siempre usará. Sin embargo, si necesita utilizar otro sistema numérico, el ejemplo siguiente le muestra la sintaxis correcta. El prefijo 0x indica hexadecimal y 0b indica binario: 

// El número 26, en decimal
int decVal = 26;
//  El número 26, en hexadecimal
int hexVal = 0x1a;
// El número 26, en binario
int binVal = 0b11010;

Literales de Punto-flotante

Un literal de punto-flotante es del tipo float si termina con la letra F o f; de otra forma es de tipo double y puede opcionalmente terminar con la letra D o d.

Los tipos de punto flotante (float y double) pueden también ser expresados utilizando E o e (para notación científica), F o f (literales de float de 32 bit) y D o d (literales de double de 64 bit; este es el tomado por defecto y por convención es omitido). 

double d1 = 123.4;
// mismo valor que d1, pero en notación científica
double d2 = 1.234e2;
float f1  = 123.4f;

Literales de Carácter y Cadena

Los literales de los tipos char y String pueden contener cualquier carácter Unicode (UTF-16). Si su editor y sistema de ficheros lo admiten, usted puede usar tales caracteres directamente en su código. Si no, puede usar un "escape Unicode" tal como '\u0108' (C mayúscula con circunflejo), o "S\u00ED Se\u00F1or" (Sí Señor en Español). Siempre use 'comillas simples' para literales char y "dobles comillas" para literales String. Las secuencias de escape Unicode pueden ser utlizadas en cualquier otra parte en un programa (tales como en nombres de campos, por ejemplo), no sólo en literales char o String.

El lenguaje de programación Java también sporta unas pocas secuencias de escape especiales para literales char y String: \b (retroceso), \t (tabulador), \n (salto de línea), \f (salto de página), \r (retorno de carro), \" (doble comilla), \' (comilla simple), y \\ (barra invertida).

Hay también un literal especial null que puede ser utilizado como un valor para cualquier tipo de referencia. null puede ser asignado a cualquier variable, excepto a variables de tipos primitivos. Hay poco que pueda hacer con un valor null más allá de comprobar su presencia. Por lo tanto, null es a menudo utilizado en programaas como un marcador para indicar que algún objeto no está disponible.

Finalmente, hay también una clase especial de literal llamado literal de clase, formado al tomar un nombre de tipo y añadir ".class"; por ejemplo, String.class. Esto refiere al objeto (de tipo Class) que representa al tipo en sí mismo.

Utilizando Caracteres de Subrayado en Literales Numéricos

En Java SE 7 y posteriores, cualquier número de caracteres de subrayado (_) pueden aparecer en cualquier parte entre dígitos en un literal numérico. Esta capacidad le permite, por ejemplo, a separar grupos de dígitos en literales numéricos, lo cual puede mejorar la legibilidad de su código.

Por ejemplo, si su código contiene números con muchos dígitos, puede usar un carácter subrayado para separar los dígitos en grupos de tres, similar a como usted usaría una marca de puntuación como una coma, o un espacio, como un separador.

el siguiente ejemplo muestra otras formas en que usted puede usar el subrayado en literales numéricos: 

long creditCardNumber = 1234_5678_9012_3456L;
long socialSecurityNumber = 999_99_9999L;
float pi =  3.14_15F;
long hexBytes = 0xFF_EC_DE_5E;
long hexWords = 0xCAFE_BABE;
long maxLong = 0x7fff_ffff_ffff_ffffL;
byte nybbles = 0b0010_0101;
long bytes = 0b11010010_01101001_10010100_10010010;

Usted puede colocar subrayados sólo entre dígitos; no puede colocar subrayados en los siguientes sitios:

  • Al principio o al final de un número
  • Al lado de un punto decimal en un literal de punto flotante
  • Antes de un sufijo F o L
  • En posiciones donde se espera una cadena de dígitos

Los siguientes ejemplos demuestran colocaciones válidas y no válidas del subrayado (las cuales están destacadas) en los literales numéricos: 

// No válido: no se pueden colocar subrayados
// al lado de un punto decimal
float pi1 = 3_.1415F;
// No válido: no se pueden colocar subrayados
// al lado de un punto decimal
float pi2 = 3._1415F;
// No válido: no se pueden colocar subrayados
// antes de un sufijo L
long socialSecurityNumber1 = 999_99_9999_L;

// OK (literal decimal)
int x1 = 5_2;
// No válido: no se pueden colocar subrayados
// Al final de un literal
int x2 = 52_;
// OK (literal decimal)
int x3 = 5_______2;

// No válido: no se pueden colocar subrayados
// dentro de un prefijo 0x
int x4 = 0_x52;
// No válido: no se pueden colocar subrayados
// al principio de un número
int x5 = 0x_52;
// OK (literal hexadecimal)
int x6 = 0x5_2;
// No válido: no se pueden colocar subrayados
// al final de un número
int x7 = 0x52_;
