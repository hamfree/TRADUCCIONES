Variables

Como ha aprendide en la lección anterior, un objeto almacena su estado en campos. 

int cadencia = 0;
int velocidad = 0;
int marcha = 1;

La lección ¿Qué Es un Objeto? le presentó los campos, pero usted probablemente tenga aún unas pocas cuestiones, tales como: ¿Cuáles son las normas y las convenciones para la denominación de un campo? Además del int, ¿qué otros tipos de datos hay? ¿Se les asigna un valor por defecto a los campos is ellos no son explícitamente inicializados? Exploraremos las respuestas a tales cuestiones en esta lección, pero antes de que lo hagamos, hay unas pocas distinciones técnicas de las que usted debe tomar concienca. En el lenguaje de programación Java, los términos "campo" y "variable" son ambos usados; ésta es una fuente común de confusión entre los nuevos desarrolladores, ya que ambos a menudo se refieren a la misma cosa.

El lenguaje de programación Java define las siguientes clases de variables:

  • Variables de Instancia (Campos No Estáticos) Hablando técnicamente, los objetos almacenan sus estados individuales en "campos no estáticos", esto es, campos declarados sin la palabra clave static. Los campos no estáticos son también conocidos como variables de instancia porque sus valores son únicos a cada instancia de una clase (a cada objeto, en otras palabras); la velocidadActual de una bicicleta es independiente de la velocidadActual de otra.
  • Variables de Clase (Campos EStáticos) Una variable de clase es cualquier campo declarado con el modificador static; esto le indica al compilador que hay exactamente una copia de ésta variable en existencia, independientemente de cuantas veces la clase haya sido instanciada. Un campo definiendo el número de ruedas para una clase particular de bicicleta podría ser marcada como static ya que conceptualmente el mismo número de ruedas se aplicará a todas las instancias. El código static int numeroRuedas = 6; creará tal campo estático. Adicionalmente, la palabra clave final podría ser añadida para indicar que el número de ruedas nunca cambiará.
  • Variables Locales De forma similar a cómo un objeto almacena su estado en campos, un método almacenará a menudo su estado temporal en variables locales. La sintaxis para declarar una variable local es similar a declarar un campo ( por ejemplo, int contador = 0;). No hay una palabra clave especial que designe una variable como local; esa determinación viene enteramente del lugar en el cual la variable es declarada— lo cual es entre las llaves de apertura y cierre de un método. Como tal, las variables locales son sólo visibles a los métodos en los cuales han sido declaradas; no son accesibles desde el resto de la clase.
  • Parámetros Uste ya ha visto ejemplos de parámetros, ambos en la clase Bicicleta y en el método main de la aplicación "¡Hola Mundo!". Recuerde que la firma para el método main es public static void main(String[] args). Aquí, la variable args es el parámetro para este método. Lo importante a recordar es que los parámetros están siempre clasificados como "variables" no "campos". Esto aplica a otras construcciones que aceptan parámetros también ( como los constructores y los manejadores de excepciones) que usted aprenderá más tarde en el tutorial.

Una vez dicho esto, el resto de este tutorial utiliza las siguientes reglas generales cuando se habla de campos y variables. Si nosotros estamos hablando sobre "campos en general" (excluyendo variables locales y parámetros), nosotros podremos decir simplemente "campos". Si la presentación se aplica a "todo lo de arriba", nosotros podremos simplemente decir "variables". Si el contexto requiere una distinción, usaremos términos específicos (campo estático, variables locales, etc.) como sea apropiado. Usted pude ocasionalmente también ver el término "miembro" usado así. A los campos de tipo, métodos y tipos anidados se les denominan colectivamente como miembros.

Nombrado

Cada lenguaje de programación tiene su propio conjunto de reglas y convenciones para las clases de nombres que usted está permitido a utilizar, y el lenguaje de programación Java no es diferente. Las reglas y convenciones para el nombrado de sus variables puede ser resumido como sigue:

  • Los nombres de variable son sensitivos al caso. Un mobre de variable puede ser cualquier identificador legal — una secuencia de longitud ilimintad de letras y dígitos Unicode, empezando con una letra, el signo del dólar "$", o el caracter de subrayado "_". La convención, sin embargo, es que siempre empiece sus nombres de variable con una letra, no con "$" o "_". Adicionalmente, el carácter del signo del dólar, por convención, nunca se utiliza en absoluto. Puede encontrar algunas situaciones cuando nombres autogenerados contendrán el signo del dólar, pero sus nombre de variables deberán siempre evitar usarlo. Una convención similar existe para el carácter de subrayado; mientras es técnicamente legal empezar su nombre de variable con "_", esta práctica no se recomienda. No se permite el espacio en blanco.
  • Los caracteres siguientes pueden ser letras, dígitos, signos del dolar, o caracteres de subrayado. Las Convenciones ( y el sentido común) se aplican a ésta regla también. Cuando elija un nombre para sus variables, utilize palabras completas en vez de abreviaciones crípticas. Haciéndolo así hará su código más fácil de leer y entender. En muchos casos hará que su código se autodocumente; campos nombrados como cadencia, velocidad, y marcha, por ejemplo, son mucho más intuitivos que sus versiones abreviadas, tales como s,c, y g. También tenga en mente que el nombre que elija no puede ser una palabra clave o palabra reservada .
  • Si el nombre que uste elije consiste de una sola palabra, esriba la palabra con todas las letras en minúsculas. Si consiste en más de una palabra, capitalize la primera letra de cada palabra. Los nombres ratioMarcha y marchaActual son los principales ejemplos de esta convención. Si sus variables almacenan un valor constante, tal como static final int NUM_GEARS = 6, la convención cambia ligeramente, se capitaliza cada letra y se separan las distintas palabras con el carácter de subrayado. Por convención, el carácter de subrayado nunca se utiliza en otros lugares.
