Expresiones, Sentencias, y Bloques

Ahora que entiende las variables y los operadores, es tiempo de aprender acerca de las expresiones, sentencias, y bloques. Los operadores pueden ser utilizados en la construcción de expresiones, las cuales calculan valores; las expresiones son el componente nuclear de las sentencias; Las sentencias pueden ser agrupadas en bloques.

Expresiones

Una expresión es una construcción compuesta de variables, operadores, e invocaciones a métodos, los cuales son construidos de acuerdo a la sintaxis del lenguaje, que evalúa a un valor único. Usted ya ha visto ejemplos de expresiones, ilustrados en negrita abajo: 

int cadencia = 0;
unaMatriz[0] = 100;
System.out.println("El elemento 1 en el índice 0: " + unaMatriz[0]);

int resultado = 1 + 2; // resultado es ahora 3
if (valor1 == valor2)
    System.out.println("valor1 == valor2");

El tipo de dato del valor retornado por una expresión depende de los elementos usados en la expresión. La expresión cadencia = 0 devuelve un int porque el operador de asignación devuelve un valor del mismo tipo de datos que su operando a su izquierda; en este caso cadencia es un int. Como puede ver de las otras expresiones, una expresión puede devolver otros tipos de valores también, como boolean o String.

El lenguaje de programación Java le permite construir expresiones compuestas de varias expresiones más pequeñas en tanto el tipo de dato requerido por una parte de la expresión coincida con el tipo de dato de la otra. Aquí tiene un ejemplo de una expresión compuesta: 

1 * 2 * 3

En éste ejemplo particular, el orden en el cual la expresión es evaluada no es importante porque el resultado de la multiplicación es independiente del orden; la salida es siempre la misma, no importa en que orden apliquemos las multiplicaciones. Sin embargo, esto no es verdad en todas las expresiones. Por ejemplo, la siguiente expresión da diferentes resultados, dependiendo de si ejecuta la operación de adición o división antes: 

x + y / 100    // ambigüo

Puede especificar exactamente cómo una expresión será evaluada usando paréntesis balanceados: ( y ). Por ejemplo, para hace la expresión previa no ambigüa, podría escribir lo siguiente: 

(x + y) / 100  // no ambigüo, recomendado

Si no indica explícitamente el orden de las operaciones a ser realizadas, el orden está determinado por la precedencia asignada a los operadores en uso dentro de la expresión. Los operadores que tienen una mayor precedencia son evaluados primero. Por ejemplo, el operador de división tienen una precedencia más alta que el operador de suma. Así, las siguientes dos sentencias son equivalentes: 

x + y / 100 


x + (y / 100) // no ambigüo, recomendado

Cuando escribe expresiones compuestas, sea explícito e indique con paréntesis qué operadores deberían ser evaluados primero. Esta práctica hace el código más fácil de leer y mantener.

Sentencias

Las declaraciones son más o menos equivalentes a las oraciones en lenguas naturales. Una sentencia forma una unidad completa de ejecución. Los siguientes tipos de expresiones pueden convertirse en una sentencia terminando la expresión con un punto y coma (;).

  • Expresiones de asignación
  • Cualquier uso de ++ o --
  • Invoaciones a métodos
  • Expresiones de creación de objetos

Tales sentencias son llamadas declaraciones de expresión. Aquí tiene algunos ejemplos de declaraciones de expresión. 

// declaración de asignación
unValor = 8933.234;
// declaración de incremento
unValor++;
// declaración de invocación a método
System.out.println("¡Hola mundo!");
// declaración de creación de un objeto
Bicicleta miBicicleta = new Bicicleta();

Además de las declaración de expresión, hay otras dos clases de declaraciones: sentencias de declaración y sentencias de control de flujo.. Una sentencia de declaración declara una variable. Usted ha visto muchos ejemplos de sentencias de declaración ya: 

// sentencia de declaración
double unValor = 8933.234;

Finalmente las sentencias de control de flujo regulan el orden en el cual las sentencias son ejecutadas. Aprenderá sobre las sentencias de control de flujo en la siguiente sección, Sentencias de Control de Flujo

Bloques

Un bloque es un grupo de cero o más sentencias entre llaves pareadas y puede ser usados en cualquier parte donde una declaración única esté permitida. El siguiente ejemplo, BlockDemo , ilustra el uso de los bloques: 

class BlockDemo {
     public static void main(String[] args) {
          boolean condicion = true;
          if (condicion) { // empieza bloque 1
               System.out.println("Condición es verdadera.");
          } // finaliza bloque one
          else { // empieza bloque 2
               System.out.println("Condición es falsa.");
          } // finaliza bloque 2
     }
}
