Lo Básico de las Anotaciones

El Formato de una Anotación

En su forma más simple, una anotación tiene el siguiente aspecto: 

@Entity

El signo arroba (@) indica al compilador que lo que sigue es una anotación. En el siguiente ejemplo , el nombre de la anotación es Override: 

@Override
void miSuperMetodo() { ... }

La anotación puede incluir elementos, los cuales pueden ser con nombre o sin nombre, y hay valores para estos elementos: 

@Author(
   name = "Benjamin Franklin",
   date = "3/27/2003"
)
class MiClase() { ... }

o 

@SuppressWarnings(value = "unchecked")
void miMetodo() { ... }

Si hay tan solo un elemento de nombre value, entonces el nombre puede ser omitido, como en: 

@SuppressWarnings("unchecked")
void miMetodo() { ... }

Si la anotación no tiene elementos, entonces los paréntesis pueden ser omitidos, como se mostró en el ejemplo @Override previo.

Es también posible usar múltiples anotaciones en la misma declaración: 

@Author(name = "Juana Exposito")
@EBook
class MiClase { ... }

Si las anotaciones tienen el mismo tipo, entonces esto es llamado una anotación de repetición: 

@Author(name = "Juana Exposito")
@Author(name = "Jose Fernandez")
class MiClase { ... }

Las anotaciones de repetición están soportadas en la versión 8 de Java SE. Para más información, vea Anotaciones de Repetición .

El tipo de anotación puede ser uno de los tipos que están definidos en los paquetes java.lang o java.lang.annotation del IPA de Java SE. En los ejemplos previos, Override y SuppressWarnings son anotaciones predefinidas de Java . Es también posible definir su propio tipo de anotación. Las anotaciones Author y Ebook en el ejemplo previo son tipos de anotación personalizados.

Donde se Pueden Usar Anotaciones

Las anotaciones pueden ser aplicadas a declaraciones: declaraciones de clases, campos, métodos, y otros elementos del programa. Cuando es usada en una declaración, cada anotación aparece a menudo, por convención, en su propia línea.

A partir de la versión 8 de Java SE, las anotaciones pueden ser también aplicadas al uso de tipos. Aquí hay algunos ejemplos:

  • Expresión de creación de instancia de la clase: 
        new @Interned MiObjeto();
  • Moldeo de tipos: 
        miCadena = (@NonNull String) cad;
  • Cláusula implements : 
        class ListaNoModificable<T> implements
        @Readonly List<@Readonly T> { ... }
  • Declaración de lanzamiento de excepción: 
        void monitorTemperatura() throws
        @Critical TemperaturaException { ... }

Esta forma de anotación es llamada una anotación de tipo. Para más información, vea Tipos de Anotaciones y los Sistemas de Tipos Enchufables .
