Tipos de Anotaciones y los Sistemas de Tipos Enchufables

Antes de la publicación de Java SE 8, las anotaciones sólo podían ser aplicadas a las declaraciones. A partir de la publicación de Java SE 8, las anotaciones también pueden ser aplicadas a cualquier uso de tipo. Esto significa que las anotaciones pueden ser usadas en cualquier lugar donde use un tipo. Unos pocos ejemplos de dónde los tipos son usados son en las expresiones de creación de instancias de clase (new), moldeos, cláusulas implements, y cláusulas throws. Esta forma de anotación es llamada una anotación de tipo y varios ejemplos están descritos en Lo Básico de las Anotaciones.

Las anotaciones de tipo fueron creadas para soportar el análisis mejorado de los programas Java de forma que se asegurase una comprobación de tipos más fuerte. La versión de Java SE 8 no ofrece un marco de trabajo de comprobación de tipos, pero le permite escribir (o descargar) un marco de trabajo de comprobación de tipos que esté implementado como uno o más módulos enchufables que son usados en conjunción con el compilador de Java.

Por ejemplo, quiere asegurarse que una variable particular en sus programa nunca tenga asignado null; usted quiere evitar el disparo de una NullPointerException. Puede escribir un complemento personalizado para comprobar esto. Modificaría entonces su código para anotar la variable particular, indicando que nunca se le asigne null. La declaración de la variable se vería así: 

@NonNull String cad;

Cuando compile el código, incluyendo el mòdulo NonNull en la línea de comandos, el compilador imprimirá un aviso si detecta un problema potencial, permitiéndole modificar el código para evitar el error. Después de corregir el código para eliminar todos los avisos, este error en particular no ocurrirá cuando el programa se ejecute.

Puede usar múltiples módulos de comprobación-de-tipos donde cada módulo copmprueba una clase diferente de error. De esta forma, puede construir encima del sistema de tipos de Java, añadiendo comprobaciones específicas cuando y donde usted las quiera.

Con el uso juicioso de las anotaciones de tipo y la presencia de comprobadores de tipo enchufables, puede escribir código que es más sólido y menos propenso a errores.

En muchos casos, no tiene que escribir sus propios módulos de comprobación de tipos. Hay terceras partes que han hecho el trabajo por usted. Por ejemplo, usted querría tomar ventaja del Marco de Trabajo Corrector creado por la Universidad de Washington. Este marco de trabajo incluye un módulo NonNull, como también un módulo de expresiones regulares, y un módulo de bloque de exclusión mútua. Para más información, vea el Marco Corrector.
