Tipos de Anotaciones Predefinidos

Un conjunto de tipos de anotación están predefinidos en la IPA de Java SE. Algunos tipos de anotación son usados por el compilador de Java, y algunos se aplican a otras anotaciones.

Tipos de anotación usados por el lenguaje Java

Los tipos de anotación predefinidos definidos en java.lang son @Deprecated, @Override, y @SuppressWarnings.

@Deprecated - La anotación @Deprecated indica que el elemento marcado está obsoleto y no debería usarse más. El compilador genera un aviso en cualquier lugar donde un programa usa un método, clase, o campo con la anotación @Deprecated. Cuando un elemento es obsoleto, también debería ser documentado usando la etiqueta de Javadoc @deprecated, como es mostrado en los siguientes ejemplos. El uso del signo at (@) en los comentarios Javadoc y en las anotaciones no es casual: están relacionados conceptualmente. También, note que la etiqueta Javadoc empieza con una d minúscula y la anotación empieza con una D mayúsucula. 

   // Sigue un comentario de Javadoc
    /**
     * @deprecated
     * explicación de porqué es obsoleto
     */
    @Deprecated
    static void metodoObsoleto() { }
}

@Override - La anotación @Override informa al compilador que el elemento está destinado a sobreescribir o reemplazar un elemento declarado en una superclase. Los métodos de sobreescritura serán descritos en Interfaces y Herencia . 

   // marca el método como un método de la superclase
   // que ha sido sobreescrito
   @Override 
   int metodoSobreescrito() { }

Aunque no es obligatorio usar esta anotación cuando se sobreescribe un método, ayuda a prevenir errores. Si un método marcado con @Override falla en sobreescribir correctamente un método en una de sus superclases, el compilador generará un error.

@SuppressWarnings - La anotación @SuppressWarnings le dice al compilador que suprima los avisos específicos que de otra forma generaría. En el siguiente ejemplo, un método obsoleto es usado, y el compilador usualmente genera un aviso. En este caso, sin embargo, la anotación causa que el aviso sea suprimido. 

   // uso de un método obsoleto y se le dice al
   // compilador que no genere un aviso
   @SuppressWarnings("deprecation")
    void usoMetodoObsoleto() {
        // aviso de obsolescencia
        // - suprimido
        objectUno.usoMetodoObsoleto();
    }

Cada aviso del compilador pertenece a una categoría. La Especificación del Lenguaje Java lista dos categorias: obsolescencia (deprecation, en inglés) y no comprobado (unchecked, en inglés). El aviso de no comprobado puede ocurrir cuando se interactúa con código heredado escrito antes de la llegada de los genéricos. Para suprimir múltiples categorías de avisos, use la siguiente sintaxis: 

@SuppressWarnings({"unchecked", "deprecation"})

@SafeVarargs - La anotación @SafeVarargs , cuando es aplicada a un método o constructor, asegura que el código no ejecuta operaciones potencialmente no seguras en su parámetro varargs. Cuando este tipo de anotación es utilizado, los avisos de no comprobación relacionados con el uso de varargs son suprimidos.

@FunctionalInterface - La anotación @FunctionalInterface , introducida en Java SE 8, indica que la declaración del tipo quiere utilizarse como una interfaz funcional, como está definido en la Especificación del Lenguaje Java.

Anotaciones Que Se Aplican a Otras Anotaciones

Las anotaciones que se aplican a otras anotaciones son llamadas meta-anotaciones. Hay varios tipos de meta-anotaciones definidos en java.lang.annotation.

@Retention - La anotación @Retention especifica como la anotación marcada es almacenada:

  • RetentionPolicy.SOURCE – La anotación marcada es retenida solo en el nivel fuente y es ignorada por el compilador.
  • RetentionPolicy.CLASS – La anotación marcada es retenida por el compilador en tiempo de compilación, pero es ignorada por la Máquina Virtual de Java (JVM, en inglés).
  • RetentionPolicy.RUNTIME – La anotación marcada es retenida por la MVJ (JVM, en inglés) así que puede ser usada por el entorno de ejecución.

@Documented - La anotación @Documented indica que en cualquier lugar donde la anotación especificada es usada esos elementos serán documentados usando la herramienta Javadoc. (Por defecto, las anotaciones no son incluidas en Javadoc). Para más información, vea la pagina de herramientas Javadoc.

@Target - La anotación @Target marca otra anotación para restringir a qué clase de elementos Java la anotación puede ser aplicada. Una anotación target especifica uno de los siguientes tipos de elemento como su valor:

  • ElementType.ANNOTATION_TYPE puede ser aplicado a un tipo de anotación.
  • ElementType.CONSTRUCTOR puede ser aplicado a un constructor.
  • ElementType.FIELD puede ser aplicado a un campo o propiedad.
  • ElementType.LOCAL_VARIABLE puede ser aplicado a una variable local.
  • ElementType.METHOD puede ser aplicado a una anotación a nivel de método.
  • ElementType.PACKAGE puede ser aplicado a una declaración de paquete.
  • ElementType.PARAMETER puede ser aplicado a los parámetros de un método.
  • ElementType.TYPE puede ser aplicado a cualquier elemento de una clase.

@Inherited - La anotación @Inherited indica que el tipo de anotación puede ser heredado de la super clase. (Esto no es verdad por defecto). Cuando el usuario consulta el tipo de anotación y la clase no tiene anotación para este tipo, la superclase de la clase es consultada por su tipo de anotación. Esta anotación aplica sólo a las declaraciones de clase.

@Repeatable - La anotación @Repeatable , introducida en Java SE 8, indica que la anotación marcada puede ser aplicada más de una vez a la misma declaración o uso de tipo. Para más información, vea Anotaciones de Repetición .
