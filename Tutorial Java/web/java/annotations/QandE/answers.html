Respuestas a Preguntas y Ejercicios: Anotaciones

Preguntas

  1. Pregunta: ¿Qué está mal en el siguiente interfaz?: 

    public interface Casa {
    @Deprecated
    void abrir();
    void abrirPuertaPrincipal();
    void abrirPuertaTrasera();
}

    Respuesta La documentación reflejará porqué abrir es obsoleto y qué usar a cambio. Por ejemplo: 

    public interface Casa {
    /**
     * @deprecated el uso de abrir
     * no se recomienda, use en su lugar
     * abrirPuertaPrincipal o
     * abrirPuertaTrasera.
     */
    @Deprecated
    public void open(); 
    public void abrirPuertaPrincipal();
    public void abrirPuertaTrasera();
}

  2. Pregunta: Considere esta implementación del interfaz Casa, mostrado en la Pregunta 1. 

    public class MiCasa implements Casa {
    public void abrir() {}
    public void abrirPuertaPrincipal() {}
    public void abrirPuertaTrasera() {}
}

    Si compila este programa, el compilador produce un aviso porque abrir estaba obsoleto (en el interfaz). ¿Qué puede hacer para deshacerse de esa advertencia?

    Respuesta: Puede hacer obsoleta la implementación de abrir: 

    public class MiCasa implements Casa {
    // La documentación es
    // heredada del interfaz.
    @Deprecated
    public void abrir() {}
    public void abrirPuertaPrincipal() {}
    public void abrirPuertaTrasera() {}
}

    Alternativamente, puede suprimir la advertencia: 

    public class MiCasa implements Casa {
    @SuppressWarnings("deprecation")
    public void abrir() {}
    public void abrirPuertaPrincipal() {}
    public void abrirPuertaTrasera() {}
}

  3. ¿Compilará el siguiente código sin error? ¿Porqué o porqué no? 

    public @interface Comida { ... }

@Comida("desayuno", platoPrincipal="cereal")
@Comida("almuerzo", platoPrincipal="pizza")
@Comida("cena", platoPrincipal="ensalada")
public void evaluaDieta() { ... }

    Respuesta: El código falla al compilar. Antes del KDJ 8, las anotaciones de repetición no estaban soportadas. A partir del KDJ 8, el código falla al compilar porque el tipo de anotación Meal no fue definido como repetible. Puede ser arreglado añadiendo la meta-anotación @Repeatable y definiendo un tipo de anotación contenedor: 

    @java.lang.annotation.Repeatable(ComidaContenedor.class)
public @interface Comida { ... }

public @interface ComidaContenedor {
    Comida[] value();
}

Ejercicios

  1. Ejercicio: Defina un tipo de anotación para una petición mejorada con elementos id, sinopsis,ingeniero, y fecha. Especifique el valor por defecto como sin asignar para ingeniero y desconocida para fecha.

    Respuesta: 

    /**
 * Describe la anotación de tipo Petición-para-Mejora (PPM)
 */
public @interface PeticionParaMejora {
    int id();
    String sinopsis();
    String ingeniero() default "[sin asignar]";
    String fecha() default "[desconocida]";
}
