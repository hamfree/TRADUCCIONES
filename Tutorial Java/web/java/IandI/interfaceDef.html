Definiendo una Interfaz

Una declaración de interfaz consiste de modificadores, la palabra clave interface, el nombre de la interfaz, una lista separada por comadas de interfaces padre (si los hay), y el cuerpo de la interfaz. Por ejemplo: 

public interface InterfazAgrupado extends Interfaz1, Interfaz2, Interfaze3 {

    // declaración de constantes
    
    // base de los logaritmos naturales
    double E = 2.718282;
 
    // firmas de métodos
    void hazAlgo (int i, double x);
    int hazAlgoMas(String s);
}

El especificador de acceso public indica que el interfaz puede ser usado por cualquier clase en cualquier paquete. Si no especifica que la interfaz es pública, entonces su interfaz estará accesible sólo para las clases definidas en el mismo paquete que la interfaz.

Una interfaz puede extender otras interfaces, igual que una clase subclase o extender otra clase. Sin embargo, An interface can extend other interfaces, just as a class subclass or extend another class. Sin embargo, mientras que una clase sólo puede ampliar otra clase, una interfaz puede extender cualquier número de interfaces. La declaración del interfaz incluye una lista separada por comas de toodas las interfaces que extiende.

El Cuerpo de la Intefaz

El cuerpo de la interfaz puede contener métodos abstractos, métodos por defecto, y métodos estáticos. Un método abstracto dentro de una interfaz lleva seguido un punto y coma, pero no llaves (un método abstracto no contiene ninguna implementación). Los métodos por defecto son definidios por el modificador default , y los métodos estáticos con la palabra clave static.Todos los métodos abstractos, por defecto, y estáticos en una interfaz son public implícitamente public, así puede omitir el modificador public.

Además, una interfaz puede contener declaraciones de constantes. Todos los valores de las constantes definidas en una interfaz son implícitamente public, static, y final. Una vez, de nuevo, puede omitir estos modificadores.>
