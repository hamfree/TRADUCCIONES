Interfaces

Hay un número de situaciones en la ingeniería del software cuando es importante para grupos dispares de programadores acordar un "contrato" que explica en detalle cómo interactúa su software. Cada grupo será capaz de escribir su código sin ningún conocimiento de cómo están escritos los códigos de otros grupos. En términos generales, las interfaces son como contratos.

Por ejemplo, imagine una sociedad futurista donde los coches robóticos controlados por ordenador transportan pasajeros a través de las calles de la ciudad sin operadores humanos. Los manufacturadores de automóviles escriben sofware (en Java, por supuesto) que operan el automóvil — parar, arrancar, acelerar, girar a la izquierda, etcétera. Otro grupo industrial, los manufacturadores de instrumentos de guía electrónicos, hacen sistemas de computadora que reciben los datos de la posición GPS (Sistema de Posicionamiento Global, GPS en inglés) y transmisión sin cables de las condiciones de tráfico y usan esa información para conducir el coche.

Los manufacturadores de automóviles deben publicar un interfaz estándar de la industria que explica en detalle qué métodos pueden ser invocados para hacer que el coche se mueva (cualquier coche, de cualquier manufacturador). Los manufacturadores de guía pueden entonces escribir softwre que invoque los métodos descritos en la interfaz para dirigir el vehículo. Ninguno de los grupos industriales necesita saber cómo el software de los otros grupos es implementado. De hecho, cada grupo considera su software altamente propietario y se reserva el derecho a modificarlo en cualquier momento, simepre que continúe adherido al interfaz publicado.

Interfaces en Java

En el lenguaje de programación de Java, una interfaz es un tipo por referencia, similar a una clase, que puede contener sólo constantes, firmas de métodos, métodos por defecto, métods estáticos, y tipos anidados. Los cuerpos de los métodos existen sólo para los métodos por defecto y los métodos estáticos. Las interfaces no pueden ser instanciadas — pueden ser sólo implementadas por clases o extendidas por otras interfaces. La extensión es descrita más tarde en esta lección.

Definir una interfaz es similar a crear una nueva clase:

public interface OperaCoche {

   // declaraciones de constantes, si las hay

   // firmas de métodos
   
   // Una enumeración con los valores RIGHT, LEFT
   int gira(Direccion direccion,
            double radio,
            double velocidadInicial,
            double velocidadFinal);
   int cambiaDeCarril(Direccion direccion,
                   double velocidadInicial,
                   double velocidadFinal);
   int señalizaGiro(Direccion direccion,
                  boolean señalEncendida);
   int getRadarFrontal(double distanciaAlCoche,
                     double velocidadDelCoche);
   int getRadarTrasero(double distanciaAlCoche,
                    double velocidadDelCoche);
         ......
   // mas firmas de métodos
}

Note que las firmas de los métodos no tienen llaves y están finalizados por un punto y coma.

Para usar una interfaz, escribe una clase que implementa la interfaz. Cuando una clase instanciable implementa una intrefaz, establece un cuerpo de método para cada uno de los métodos declarados en la interfaz. Por ejemplo, 

public class OperaBMW760i implements OperaCoche {

    // las firmas de los métodos en OperaCoche method signatures, con implementación --
    // por ejemplo:
    int señalizaGiro(Direccion direccion, boolean señalEncendida) {
       // código para encender el indicador de intermitencia IZQUIERDA del BMW
       // código para apagar el indicador de intermitencia IZQUIERDA del BMW
       // código para encender el indicador de intermitencia DERECHO del BMW
       // código para apagar el indicador de intermitencia DERECHO del BMW
    }

    // otros miembros, como se necesiten -- por ejemplo, clases ayudantes no
    // visibles a los clientes de interfaz
}

En el ejemplo de coche robótico de arriba, son los manufacturadores del automóvil los que implementarán la interfaz. La implementación de Chevrolet será substancialmente diferente de la de Toyota, por supuesto, pero ambos manufacturadores se adhirirá al mismo interfaz. Los manufacturadores de guía, que son clientes de la interfaz, construirán sistemas que usan los datos del GPS sobre la localización del coche, mapas digitales de calles, y los datos del tráfico para conducir el coche. De este modo, los sistemas de guía invocarán los métodos de interfaz: a su vez, cambiar de carril, frenar, acelerar, y así sucesivamente.

Interfaces como IPA

El ejemplo del coche robótico muestra una interfaz siendo usada como un estándar industrial Interfaz de Programación de Aplicaciones (API, en inglés). Las IPA son también comunes en los productos de software comercial. Típicamente, una compañía vende un paquete de software que contiene métodos complejos que otra compañía quiere usar en el software propio de su producto. Un ejemplo sería una paquete de métodos de procesado de imagen digital que son vendidos a compañías que hacen programas de gráficos para el usuario final. La compañia de procesado de imágenes escribe sus clases para implementar una interfaz, la cual se hace pública para sus clientes. La compañía de gráficos entonces invoca los métodos de procesado de imagen usando las firmas y tipos de retorno definidos en la interfaz. Miemtras la compañía del procesado de la imagen hace su IPA pública (a sus clientes), su implementación del IPA es mantenida como un secreto guardado celosamente — en realidad, puede revisar la implementación una fecha más tarde mientras que continúe implementando el interfaz orginal en los que sus clientes han confiado.
