Respuestas a Preguntas y Ejercicios: Interfaces

Preguntas

Pregunta 1: ¿Qué métodos tendría una clase que implementa la interfaz java.lang.CharSequence que implementar?
Respuesta 1: charAt, length, subSequence, y toString.

Pregunta 2: ¿Qué está mal en el siguiente interfaz? 

public interface AlgoEstaMal {
    void unMetodo(int unValor){
        System.out.println("Hola mamá");
    }
}

Respuesta 2: Tiene una implementación de un método dentro. Sólo los métodos por defecto y los estáticos tienen implementaciones.

Pregunta 3: Arregle el interfaz de la pregunta 2.
Respuesta 3: 

public interface AlgoEstaMal {
    void unMetodo(int unValor);
}

Alternativamente, puede definir unMetodo como un método por defecto: 

public interface AlgoEstaMal {
    default void unMetodo(int unValor) {
        System.out.println("Hola Mamá");
    }
}

Pregunta 4: ¿Es el siguiente interfaz válido? 

public interface Marker {
}

Respuesta 4: Sí. Los métodos no son obligatorios. Los interfaces vacíos pueden ser usados como tipos y para marcar clases sin requerir ninguna implementación particular de métodos. Un ejemplo de interfaz vacío útil, vea java.io.Serializable.

Respuesta 1: Vea DemoSecuenciaCaracter.java .

Ejercicio 2: Suponga que ha escrito un servidor de tiempo que periódicamente notifica a sus clientes la fecha y hora actual. Escriba un interfaz que el servidor podría usar para hacer cumplir un protocolo particular a sus clientes.
Respuesta 2: Vea ClienteTiempo.java .
