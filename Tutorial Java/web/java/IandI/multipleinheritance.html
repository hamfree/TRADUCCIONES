Herencia Múltiple de Estado, Implementación, y Tipo

Una diferencia significativa entre las clases y los interfaces es que las clases pueden tener campos mientras que las interfaces no pueden. Además, puede instanciar una clase para crear un objeto, lo cual no puede hacer con los interfaces. Como se explicón en la sección ¿Qué es un objeto? , un objeto almacena su estado en campos, los cuales están definidos en las clases. Una razón por la que el lenguaje de programación Java no le permite extender más de una clase es para evitar los problemas de la heremcia múltiple de estado, la cual es la habilidad de heredar campos de múltiples clases. Por ejemplo, suponga que es capaz de definir una nueva clase que extiende múltiples clases. Cuando crea un objeto instanciando esa clase, ese objeto heredará los campos de todas las superclases de la clase. ¿Qué ocurre si los métodos o constructores de las diferentes superclases instancian el mismo campo? ¿Qué método o constructor tomará predencia? Porque los interfaces no contienen campos, no tiene que preocuparse acerca de los problemas que resultan de la herencia múltiple de estado.

La herencia múltiple de implementación es la habilidad de heredar definiciones de métodos de múltiples clases. Los problemas surgen con este tipo de herencia múltiple, tales como conflictos de nombres y ambiguedad. Cuando los compiladores de los lenguajes de programación que soportan este tipo de herencia múltiple encuentran superclases que contienen métodos con el mismo nombre, a veces no pueden determinar cual miembro o método acceder o invocar. Además, un programador puede sin darse cuenta introducir un conflicto de nombres añadiendo un nuevo método a una superclase. Los métodos por defecto introducen una forma de herencia múltiple de implementación. Una clase puede implementar más de una interfaz, el cual puede contener métodos por defecto que tengan el mismo nombre. El compilador de Java establece algunas reglas para determinar qué método por defecto una clase particular usa.

El lenguaje de programación Java soporta herencia múltiple de tipo, que es la habilidad de una clase para implementar más de una interfaz. Un objeto puede tener múltiplos tipos: el tipo de su propia clase y los tipos de todos los interfaces que la clase implementa. Esto significa que si una variables es declarada a ser el tipo de una interfaz, entonces su valor puede referenciar cualquier objeto que está instanciado desde cualquier clase que implemente el interfaz. Esto es descrito en la sección Usando una Interfaz como un Tipo .

Como con la herencia múltiple de implementación, una clase puede heredar diferentes implementaciones de un método definido (como por defecto o estático) en las interfaces que extiende. En este caso, el compilador o el usuario debe decidir cuál usar.
