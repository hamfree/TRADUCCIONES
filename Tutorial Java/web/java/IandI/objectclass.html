Object como una Superclase

La clase Object , en el paquete java.lang, se asienta en lo alto del arbol de jerarquía de clases. Cada clase es una descendiente, directa o indirectamente, de la clase Object. Cada clase que use o escriba hereda los métodos de instnacia de Object. No necesita usar cualquiera de estos métodos, pero, si elije hacerlo, necesita sobreescribirlos con código que es específico a su clase. Los métodos heredados de Object que son descritos en esta sección son:

  • protected Object clone() throws CloneNotSupportedException
          Crea y devuelve una copia de este objeto.
  • public boolean equals(Object obj)
          Indica si algún otro objeto es "igual que " este.
  • protected void finalize() throws Throwable
          Llamado por el recolector de basura sobre un objeto cuando la
          recolección de basura determina que no hay más referencias a este objeto.
  • public final Class getClass()
          Devuelve la clase en tiempo de ejecución de un objeto.
  • public int hashCode()
          Devuelve un valor de código resumen para el objeto.
  • public String toString()
          Devuelve una cadena con la representación del objeto.

Los métodos notify, notifyAll, y wait de Object juegan todos una parte en la sincronización de las actividades de hilos ejecutándose independientemente en un programa, lo cual es descrito en una lección más tarde y no será cubierto aquí. Hay cinco de estos métodos:

  • public final void notify()
  • public final void notifyAll()
  • public final void wait()
  • public final void wait(long timeout)
  • public final void wait(long timeout, int nanos)
Nota: Hay algunos aspectos sutiles en un número de estos métodos, especialmente el método clone.

El Método clone()

Si una clase, o una de sus superclases, implementa el interfaz Cloneable , puede usar el método clone() para crear una copia de un objeto existente. Para crear un clon, escribe: 

unObjetoCloneable.clone();

La implementación de Objectde este método comprueba si el objeto en el cual clone() fue invocado implementa el interfaz Cloneable. Si el objeto no lo hace, el método lanzará una excepción CloneNotSupportedException. El manejo de excepciones será cubierto en una lección más tarde. Por el momento, necesita saber que clone() debe ser declarado como 

protected Object clone() throws CloneNotSupportedException

o: 

public Object clone() throws CloneNotSupportedException

si usted va a escribir un método clone() para sobreescribir el que está en Object.

Si el objeto en el que clone() fue invocado implementa el interfaz Cloneable, la implementación de Object del método clone() crea un objeto de la misma clase que el objeto original e inicializa las variables miembro del nuevo objeto con los mismos valores que tienen las correspondientes variables miembro del objeto original.

La forma más simple de hacer su clase cloneable es añadir implements Cloneable a su declaración de clase. Así sus objetos pueden invocar el método clone().

Para algunas clases, el comportamiento por defecto del método clone() de Object trabaja correctamente. Si, sin embargo, un objeto contiene una referencia a un objeto externo, digamos ObjExterno, puede necesitar sobreescribir clone() para tener el comportamiento correcto. De otra forma, un cambio en ObjExterno hecho por un objeto será visible en su clon también. Esto significa que el objeto original y su clone no son independientes — para desacoplarlos, debe sobreescribir clone() de forma que se clone el objeto y ObjExterno. Entonces las referencias al objeto original ObjExterno y las referencias del clone referencian un clone de ObjExterno, de forma que el objeto y su clone son realmente independientes.

El Método equals()

El método equals() compara dos objetos por su igualdad y devuelve true si ellos son iguales. El método equals() establecido en la clase Object usa el operador de identidad (==) para determinar si dos objetos son iguales. Para tipos de datos primitivos, esto da el resultado correcto. Para objetos, sin embargo, no lo hace. El método equals() establecido en Object comprueba si las referencias al objeto son iguales; esto es, si los objetos comparados son el mismo objeto exacto.

Para comprobar si dos objetos son iguales en el sentido de equivalencia (Contienen la misma información), debe sobreescribir el método equals(). Aquí tiene un ejemplo de la clase Libro que sobreescribe equals(): 

public class Libro {
    ...
    public boolean equals(Object obj) {
        if (obj instanceof Libro)
            return ISBN.equals((Libro)obj.getISBN());
        else
            return false;
    }
}

Considere este codigo que comprueba dos instancias de la clase Libro por su igualdad: 

// Tutorial de Swing, 2nd edicion
Libro primerLibro  = new Libro("0201914670");
Libro segundoLibro = new Libro("0201914670");
if (primerLibro.equals(segundoLibro)) {
    System.out.println("Los objetos son iguales");
} else {
    System.out.println("Los objetos no son iguales");
}

Este programa visualiza Los objetos son iguales incluso aunque primerLibro y segundoLibro referencian a dos objetos distintos. Ellos son considerados iguales porque los objetos comparados contienen el mismo número ISBN.

Simpre sobreescribirá el método equals() si el operador de identidad no es apropiado para su clase.

Nota: Si usted sobreescribe equals(), debe sobreescribir hashCode() también.

El Método finalize()

La clase Object establece un método callback, finalize(), que puede ser invocado en un objeto cuando se convierte en basura. La implementación de finalize() de Object no hace nada — usted puede sobreescribir finalize() para hacer limpieza, tal como liberar recursos.

El método finalize() puede ser llamado automáticamente por el sistema, pero cuándo es llamado, o incluso si es llamado, es incierto. Por ello, no debería confiar en este método para que haga la limpieza por usted. Por ejemplo, si no cierra los descriptores de ficheros en su código después de ejecutar E/S y espera que finalize() los cierre por usted, puede quedarse sin descriptores de ficheros.

El Método getClass()

No puede sobreescribir getClass.

El método getClass() devuelve un objeto Class, el cual tiene métodos que puede usar para obtener información sobre la clase, como suu nombre (getSimpleName()), su superclase (getSuperclass()), y los interfaces que implementa (getInterfaces()). Por ejemplo, el siguiente método obtiene y visualiza el nombre de la clase de un objeto: 

void imprimeNombreClase(Object obj) {
    System.out.println("La clase del objeto " + " es " +
        obj.getClass().getSimpleName());
}

La clase Class , en el paquete java.lang, tiene un gran número de métodos (más de 50). Por ejemplo, puede comprobar para ver si la clase es una anotación (isAnnotation()), un interfaz (isInterface()), o una enumeración (isEnum()). Puede ver cuales son los campos del objeto (getFields()) o que métodos tiene (getMethods()), etcétera.

El Método hashCode()

El valor devuelto por hashCode() es el código hash del objeto, el cual es la dirección del objeto en memoria en hexadecimal.

Por definición, si dos objetos son iguales, sus códigos hash tienen que ser también iguales. Si sobreescribe el método equals(), uste cambia la forma en que dos objetos son igualados y la implementación de hashCode() de Object ya no es válida. Por ello, si sobreescribe el método equals(), debe también que sobreescribir el método hashCode() también.

El Método toString()

Siempre se debe tener en cuenta sobreescribir el método toString() en sus clases.

El método toString() de Object devuelve una String con la representación de su objeto, lo cual es muy útil para depuración. La representación en String para un objeto depende enteramente del objeto, por ello es porque necesita sobreescribir toString() en sus clases.

Puede usar toString() junto con System.out.println() para mostrar una representación textual de un objeto, tal como una instancia de Libro: 

System.out.println(primerLibro.toString());

Lo cual, sería, para un método propiamente sobreescrito toString(), imprimir algo útil, como esto: 

ISBN: 0201914670; El Tutorial de Swing; Una Guía para Construir IGUs, 2ª Edición
