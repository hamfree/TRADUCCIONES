Usando la Palabra Clave super

Acceder a Miembros de la Superclase

Si su método sobreescribe uno de los métodos de sus superclases, puede invocar el método sobreescrito a través del uso de la palabra clave super. Puede también usar super para referirse a un campo oculto (aunque ocultar campos no está recomendado). Considere esta clase, Superclase: 

public class Superclase {

    public void metodoImprime() {
        System.out.println("Impreso en la Superclase.");
    }
}

Aquí tiene una sublcase, llamada Subclase, que sobreescribe metodoImprime(): 

public class Subclase extends Superclase {

    // sobreescribe metodoImprime en Superclase
    public void metodoImprime() {
        super.metodoImprime();
        System.out.println("Impreso en la Subclase.");
    }
    public static void main(String[] args) {
        Subclase s = new Subclase();
        s.metodoImprime();
    }
}

Dentro de Subclase, el nombre simple metodoImprime() se refiere al que está declarado en la Subclase, el cual sobreescribe el que está en Superclase. Así, para referirse a metodoImprime() heredado de Superclase, Subclase tiene que usar un nombre cualificado, usando super como se mostró. Compilar y ejecutar Subclase imprime lo siguiente: 

Impreso en la Superclase.
Impreso en la Subclase.

Constructores de la Subclase

El siguiente ejemplo ilustra cómo usar la palabra clave super para invocar un constructor de la superclase. Recuerde del ejemplo Bicicleta que BicicletaMontaña es una sublcase de Bicicleta. Aquí está el constructor BicicletaMontaña (de la subclase) que llama al constructor de la superclase y entonces añade código de inicialización propio: 

public BicicletaMontaña(int anchoInicial,
                    int cadenciaInicial,
                    int velocidadInicial,
                    int marchaInicial) {
    super(cadenciaInicial, velocidadInicial, marchaInicial);
    anchoSillin = anchoInicial;
}

La invocación de un constructor de la superclase debe ser la primera línea en el constructor de la subclcase.

La sintaxis para llamar a un constructor de la superclase es 

super();
o: 
super(lista de parámetros);

Con super(), el constructor sin-argumentos de la superclase es llamado. Con super(lista de parámetros), el constructor de la superclase con una lista de parámetros que coincide es llamada.

Nota: Si un constructor no invoca explícitamente al constructor de la subclase, el compilador de Java automáticamente inserta una llamada al constructor sin argumentos de la superclase. Si la superclase no tiene un constructor sin argumento, usted verá un error en tiempo de compilación. Object tiene un constructor, así que si Object es la única superclase, no hay problema.

Si un constructor de subclase invoca un constructor de su superclase, ya sea explícitamente o implícitamente, usted podría pensar que habrá una cadena completa de constructores a llamar, todo el camino hacia abajo hasta el constructor de Object. En realidad, éste es el caso. Esto se llama Encadenamiento de constructores, y usted necesita estar enterado de ello cuando haya una línea larga de descenso de clases.
