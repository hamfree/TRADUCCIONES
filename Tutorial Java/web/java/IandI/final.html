Escribiendo Clases y Métodos Finales

Puede declarar alguno o todos los métodos de una clase finales. Usted usa la palabra clave final en una declaración de método para indicar que el método no puede ser sobreescrito por las subclases. La clase Object hace esto — un número de sus métodos son finales.

Podría desear hacer un método final si tiene una implementación que no debería ser cambiaada y es crítica para el estado consistente del objeto. Por ejemplo, podría querer hacer el método getprimerJugador en esta clase AlgoritmoAjedrez final: 

class AlgoritmoAjedrez {
    enum JugadoresAjedrez { BLANCAS, NEGRAS }
    ...
    final JugadoresAjedrez getPrimerJugador() {
        return JugadoresAjedrez.BLANCAS;
    }
    ...
}

Los métodos llamados desde los constructores generalmente serían declarados finales. Si un constructor llama a un método no final, una subclase puede redefinir ese método con sorprendentes o indeseables resultados.

Note que puede también declarar una clase entera final. Una clase que es declarada final no puede ser derivada. Esto es particularmente útil, por ejemplo, cuando se crea una clase inmutable como la clase String.
