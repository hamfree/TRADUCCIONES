Implementando una Interfaz

Para declarar una clase que implementa una interfaz, incluya una cláusula implements en la declaración de la clase. Su clase puede implementar más de un interfaz, así que la palabra clave implements va seguida de una lista separada por comas de interfaces implementados por la clase. Por convención, la cláusula implements sigue a la cláusula extends, si hay una.

Un Interfaz de Muestra, Comparable

Considere una interfaz que define como comparar el tamaño de los objetos. 

public interface Relatable {
        
    // esta (objeto llamanda a esMayorQue)
    // y otras deben ser instancias de
    // la misma clase devolviendo 1, 0, -1
    // si esta es más grande que,
    // igual a, o menor que la otra
    public int esMayorQue(Relatable otro);
}

Si quiere ser capaz de comparar el tamaño de objetos similares, no importa cuales sean, la clase que las instancie implementará Comparable.

Cualquier clase puede implementar Comparable si hay alguna forma de comparar el "tamaño" relativo de los objetos instanciados desde la clase. Para las cadenas, podría ser el número de caracteres; para libros, podría ser el número de página; para estudiantes, podría ser el peso; etcétera. Para objetos geométricos planos, el área sería una buena elección (vea la clase RectanglePlus que sigue), mientras que volumen trabajaría para objetos geométricos tridimensionales. Todas estas clases pueden implementar el método esMayorQue().

Si conoce que una clase implementa Comparable, entonces sabe que puede comparar el tamaño de los objetos instanciados desde esa clase.

Implementando el Interfaz Relatable

Aquí está la clase Rectángulo que fue presentada en la sección Creando Objetos , reescrita para implementar Comparable. 

public class RectanguloPlus
    implements Relatable {
    public int ancho = 0;
    public int alto = 0;
    public Punto origen;

    // cuatro constructores
    public RectanguloPlus() {
        origen = new Punto(0, 0);
    }
    public RectanguloPlus(Punto p) {
        origen = p;
    }
    public RectanguloPlus(int w, int h) {
        origen = new Punto(0, 0);
        ancho = w;
        alto = h;
    }
    public RectanguloPlus(Punto p, int w, int h) {
        origen = p;
        ancho = w;
        alto = h;
    }

    // un método para mover el rectángulo
    public void mueve(int x, int y) {
        origen.x = x;
        origen.y = y;
    }

    // un método para calcular
    // el área de un rectángulo
    public int getArea() {
        return ancho * alto;
    }
    
    // un método requerido para implementar
    // el Interfaz Relatable
    public int esMayorQue(Relatable otro) {
        RectanguloPlus otroRectangulo = (RectanguloPlus) otro;
        if (this.getArea() < otroRectangulo.getArea())
            return -1;
        else if (this.getArea() > otroRectangulo.getArea())
            return 1;
        else
            return 0;               
    }
}

Ya que RectanguloPlus implementa Comparable, el tamaño de dos objetos RectanguloPlus cualquiera pueden ser comparados.

Nota: El método esMayorQue , como está definido en el interfaz Comparable, toma un objeto del tipo Comparable. La línea de codigo, mostrada en negrita en el ejemplo previo, moldea otro a una instancia RectanguloPlus. El moldeo de tipo indica al compilador qué objeto es realmente. Invocar getArea directamente en la instancia otro (other.getArea()) fallará al compilar porque el compilador no entiende que otro es en realidad una instancia de RectanguloPlus.
