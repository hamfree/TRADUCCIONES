Evolucionando interfaces

Considere una interfaz que ha desarrollado llamada Hazlo: 

public interface Hazlo {
   void hazAlgo(int i, double x);
   int hazAlgoMas(String s);
}

Suponga que, un tiempo más tarde, quiere añadir un tercer método a Hazlo, así que el interfaz ahora se convierte: 

public interface Hazlo {

   void hazAlgo(int i, double x);
   int hazAlgoMas(String s);
   boolean funciono(int i, double x, String s);
   
}

Si usted hace este cambio, entonces todas las clases que implementan el viejo interfaz Hazlo se romperán porque ellas ya no implementan el viejo interfaz. Los programadores que confían en este interfaz protestarán en voz alta.

Intente anticipar todos los usos de su interfaz y especifíquelo completamente desde el principio. Si quiere añadir métodos adicionales a un interfaz, tiene varias opciones. Podría crer un interfaz HazloPlus que extienda Hazlo: 

public interface HazloPlus extends Hazlo {

   boolean funciono(int i, double x, String s);
   
}

Ahora los usuarios de su código pueden elegir continuar usando el viejo interfaz o actualzar al nuevo interfaz.

Alternativamente, puede definir sus nuevos métodos como métodos por defecto. El siguiente ejemplo define un método por defecto llamado funciono: 

public interface Hazlo {

   void hazAlgo(int i, double x);
   int hazAlgoMas(String s);
   default boolean funciono(int i, double x, String s) {
       // Cuerpo del método
   }
   
}

Note que debe establecer una implementación para los métodos por defecto. Podría también definir nuevos métodos estáticos a los interfaces existentes. Los usuarios que tienen clases que implementan los interfaces mejorados con el nuevo método por defecto o los métodos estáticos no tienen que modificar o recompilarlo para acomodar los métodos adicionales.
