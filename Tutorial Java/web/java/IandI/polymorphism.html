Polimorfismo

La definición del diccionario de polimorfismo se refiere a un principio en biología en el cual un organismo o especie puede tener muchas diferentes formas o etapas. Este principio puede ser también aplicado a la programación orientada a objetos y a lenguajes como el lenguaje Java. Las subclases de una clase pueden definir sus propios comportamientos y aún compartir alguna de la misma funcionalidad de la clase padre.

El polimorfismo puede ser demostrado con una modificación menor a la clase Bicicleta. Por ejemplo, un método imprimeDescripcion puede ser añadido a la clase que muestra todos los datos almacenados actualmente en una instancia. 

public void imprimeDescripcion(){
    System.out.println("\nBici esta " + "en la marcha " + this.gear
        + " con una cadencia de " + this.cadence +
        " y desplazándose a una velocidad de " + this.speed + ". ");
}

Para demostrar las capacidades polimórficas en el lenguaje Java, extendemos la clase Bicicleta con las clases BicicletaMontaña y BicicletaCarretera. Para BicicletaMontaña, añadimos un campo para suspension, que es un valor String que indica si la bici tiene una suspensión delantera, frontal. O, la bici tiene suspensiones delante y detrás, Dual.

Aquí está la clase actualizada: 

public class BicicletaMontaña extends Bicicleta {
    private String suspension;

    public BicicletaMontaña(
               int cadenciaInicio,
               int velocidadInicio,
               int marchaInicial,
               String tipoSuspension){
        super(cadenciaInicio,
              velocidadInicio,
              marchaInicial);
        this.setSuspension(tipoSuspension);
    }

    public String getSuspension(){
      return this.suspension;
    }

    public void setSuspension(String tipoSuspension) {
        this.suspension = tipoSuspension;
    }

    public void imprimeDescripcion() {
        super.imprimeDescripcion();
        System.out.println("The " + "BicicletaMontaña tiene una suspensión " +
            getSuspension() + ".");
    }
}

Fíjese en el método sobreescrito imprimeDescripcion. Además de la información establecida antes, se incluye información adicional sobre la suspensión en la salida.

A continuación, creamos la clase BicicletaCarretera. Debido a que las bicis de carreras o carretera tienen ruedas más flacas, añadimos un atributo para monitorizar el ancho de la rueda. Aquí está la clase BicicletaCarretera: 

public class BicicletaCarretera extends Bicicleta {
    // En milímetros (mm)
    private int anchoRueda;

    public BicicletaCarretera(int cadenciaInicio,
                    int velocidadInicio,
                    int marchaInicial,
                    int nuevoAnchoRueda){
        super(cadenciaInicio,
              velocidadInicio,
              marchaInicial);
        this.setAnchoRueda(nuevoAnchoRueda);
    }

    public int getAnchoRueda(){
      return this.anchoRueda;
    }

    public void setAnchoRueda(int nuevoAnchoRueda){
        this.anchoRueda = nuevoAnchoRueda;
    }

    public void imprimeDescripcion(){
        super.imprimeDescripcion();
        System.out.println("La BicicletaCarretera" + " tiene ruedas de " + getAnchoRueda() +
            " mm.");
    }
}

Note que una vez de nuevo, el método imprimeDescripcion ha sido sobreescrito. Esta vez, se visualiza la información sobre el ancho de la rueda.

Para resumir, hay tres clases: Bicicleta, BicicletaMontaña, y BicicletaCarretera. Las dos subclases sobreescriben el método imprimeDescripcion e imprimen información única.

Aquí tiene un programa de prueba que crea tres variables Bicicleta. Cada variable es asignada a una de las tres clases de bicicleta. Cada variable es entonces impresa. 

public class PruebaBicis {
  public static void main(String[] args){
    Bicicleta bici01, bici02, bici03;

    bici01 = new Bicicleta(20, 10, 1);
    bici02 = new BicicletaMontaña(20, 10, 5, "Dual");
    bici03 = new BicicletaCarretera(40, 20, 8, 23);

    bici01.imprimeDescripcion();
    bici02.imprimeDescripcion();
    bici03.imprimeDescripcion();
  }
}

Lo siguiente es la salida del programa de prueba: 

Bici está en marcha 1 con una cadencia de 20 y desplazándose a una velocidad de 10.

Bici está en marcha 5 con una cadencia de 20 y desplazándose a una velocidad de 10.
La BicicletaMontaña tiene una suspensión Dual.

Bici está en marcha 8 con una cadencia de 40 y desplazándose a una velocidad de 20.
La BicicletaCarretera tiene ruedas de 23 mm.

La máquina virtual de java (JVM en inglés) llama al método apropiado para el objeto que es referido en cada variable. No llama al método que está definido por el tipo de la variable. Este comportamiento se refiere como una invocación virtual de método y demuestra un aspecto de las funcionalidades importantes polimórficas en el lenguaje de Java.
