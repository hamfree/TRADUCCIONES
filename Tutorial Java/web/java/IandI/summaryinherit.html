Sumario de Herencia

Excepto para la clase Object, una clase tiene exactamente una superclase directa. Una clase hereda los campos y métodos de todas sus superclases. ya sea directa o indirectamente. Una subclase puede sobreescribir métodos que ha heredados, o puede ocultar campos o métodos que ha heredado. (Note que ocultar campos es generalmente una mala práctica de programación).

La tabla en la sección Sobreescribiendo y Ocultando Métodos muestra el efecto de declarar un método con la misma firma que un método en la superclase.

La clase Object es el tope de la jerarquía de clases. Todas las clases son descendentes de esta clase y heredan los métodos de ella. Métodos útiles heredados de Object incluyen toString(), equals(), clone(), y getClass().

Puede eviar que una clase sea derivada utilizando la palabra clave final en la declaración de la clase. De forma similar, puede evitar que un método sea sobreescrito por las subclases declarándlolo como un método final.

Una clase abstracta puede sólo ser derivada; no puede ser instanciada. Una clase abstracta puede contener métodos abstractos — métodos que son declarados pero no implementados. Las subclases entonces establecen las implementaciones de los métodos abstractos.
