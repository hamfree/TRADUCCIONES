Usando una Interfaz Como un Tipo

Cuando define una interfaz nueva, está definiendo una nueva referencia al tipo de dato. Puede usar nombres de interfaz en cualquier lugar donde puede usar cualquier otro nombre de tipo para datos. Si define una variable referencia cuyo tipo es una interfaz, cualquier objeto que usted lo asigne a él, deberá ser una instancia de una clase que implementa el interfaz.

Como un ejemplo, aquí está el método para encontrar el objeto más grande en un par de objetos, para cualquier objeto que sea instanciados desde una clase que implementa Relatable: 

public Object buscaElMasGrande(Object object1, Object object2) {
   Relatable obj1 = (Relatable) object1;
   Relatable obj2 = (Relatable) object2;
   if ((obj1).esMasGrandeQue(obj2) > 0)
      return object1;
   else 
      return object2;
}

Al moldear object1 a un tipo Relatable, puede invocar el método esMayorQue.

Si hace un punto de implementación Relatable en una amplia variad de clases, los objetos instanciados desde cualquiera de esas clases puede ser comparado con el método buscaElMasGrande() — siempre que ambos objetos son de la misma clase. Similarmente, pueden ser comparados con los siguientes meétodos: 

public Object buscaElMasPequeño(Object object1, Object object2) {
   Relatable obj1 = (Relatable)object1;
   Relatable obj2 = (Relatable)object2;
   if ((obj1).esMasGrandeQue(obj2) < 0)
      return object1;
   else 
      return object2;
}

public boolean esIgual(Object object1, Object object2) {
   Relatable obj1 = (Relatable)object1;
   Relatable obj2 = (Relatable)object2;
   if ( (obj1).esMasGrandeQue(obj2) == 0)
      return true;
   else 
      return false;
}

Estos métodos funciona para cualesquiera objetos "relatables", no importa cual sea la herencia de la clase. Cuando se implementa Relatable, pueden ser ambos su propia tipo de clase (o superclase) y tipo Relatable. Esto les da algunas de las ventajas de la herencia múltiple, donde pueden tener comportamiento tanto de una superclase como de una interfaz.
