Autoboxing y Unboxing (Conversiones Automáticas)

Autoboxing es la conversión automática que el compilador de Java hace entre los tipos primitivos y sus correspondientes objetos de clases de envoltorio. Por ejemplo, convertir un int en un Integer, un double en un Double, y así. Si la conversión va en el otro camino, se le llamad unboxing.

Aquí tiene el ejemplo más simple de autoboxing: 

Character ch = 'a';

El resto de los ejemplos en esta sección usa genéricos. Si todavía no está familiarizado con la sintaxis de los genéricos, vea la lección Genéricos (Actualizado) .

Considere el siguiente código: 

List<Integer> li = new ArrayList<>();
for (int i = 1; i < 50; i += 2)
    li.add(i);

Aunque añada los valores int como tipos primitivos, en vez de objetos Integer, a li, el código compila. Porque li es una lista de objetos Integer, no una lista de valores int, usted puede preguntarse porqué el compilador de Java no emite un error en tiempo de compilación. El compilador no genera un error porque crea un objeto Integer desde i y añade el objeto a li. Por ello, el compilador convierte el código previo al siguiente en tiempo de ejecución: 

List<Integer> li = new ArrayList<>();
for (int i = 1; i < 50; i += 2)
    li.add(Integer.valueOf(i));

Convertir un valor primitivo (un int, por ejemplo) en un objeto de la correspondiente clase envoltorio (Integer) se le llama autoboxing. El compilador de Java aplica el autoboxin cuando un valor primitivo es:

  • Pasado como un parámetro a un método que espera un objeto de la clase envoltorio correspondiente.
  • Asignado a una variable de la clase envoltorio correspondiente.

Considere el siguiente método: 

public static int sumEven(List<Integer> li) {
    int sum = 0;
    for (Integer i: li)
        if (i % 2 == 0)
            sum += i;
        return sum;
}

Por que los operadores resto (%) y la suma unaria (+=) no se aplican a los objetos Integer, usted puede preguntarse porqué el compilador de java compila el método sin emitir ningún error. El compilador no genera un error porque invoca el método intValue para convertir un Integer en un int en tiempo de ejecución: 

public static int sumEven(List<Integer> li) {
    int sum = 0;
    for (Integer i : li)
        if (i.intValue() % 2 == 0)
            sum += i.intValue();
        return sum;
}

Convertir un objeto de un tipo envoltorio (Integer) a su correspondiente valor primitivo (int) se le llama unboxing. El compilador de Java aplica el unboxing cuando un objeto de una clase envoltorio es:

  • Pasado como parámetro a un método que espera un valor del tipo primitivo correspondiente.
  • Asignado a una variable del tipo primitivo correspondiente.

El ejemplo Unboxing muestra como funciona esto: 

import java.util.ArrayList;
import java.util.List;

public class Unboxing {

    public static void main(String[] args) {

        Integer i = new Integer(-8);

        // 1. Unboxing a través de la invocación del método.
        int absVal = absoluteValue(i);
        System.out.println("valor absoluto de " + i + " = " + absVal);

        List<Double> ld = new ArrayList<>();
        ld.add(3.1416);    // PI es autoboxed a través de la invocación del método.

        // 2. Unboxing a través de la asignación.
        double phi = ld.get(0);
        System.out.println("phi = " + phi);
    }

    public static int absoluteValue(int i) {
        return (i < 0) ? -i : i;
    }
}

El programa imprime lo siguiente: 

valor absoluto de -8 = 8
phi = 3.1416
