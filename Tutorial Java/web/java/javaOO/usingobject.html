Usando Objetos

Una vez ha creado un objeto, probablemente querrá usarlo para algo. Puede necesitar usar el valor de uno de sus campos, cambiar uno de sus campos, o llamar uno de esus métodod para realizar una acción.

Hacer Referencia los Campos de un Objeto

Los campos de un objeto son accedidos por su nombre. Debe usar un nombre que no sea ambiguo.

Puede usar un nombre simple para un campo dentro de su propia clase. Por ejemplo, podemos añadir una sentencia dentro de la clase Rectangle que imprima la anchura y altura: 

System.out.println("La anchura y altura son: " + anchura + ", " + altura);

En este caso, anchura y altura son nombres simples.

El código que está fuera de la clase del objeto debe usar una referencia al objeto o expresión, seguida por el operador punto (.), seguido por el nombre simple del campo, como en: 

referenciaObjeto.nombreCampo

Por ejemplo, el código en la clase CreateObjectDemo está fuera del código para la clase Rectangle. Así para referirse a los campos origen, anchura, y altura dentro del objeto Rectangle nombrado rectUno, la clase CreateObjectDemo debe usar los nombres rectUno.origen, rectUno.anchura, y rectUno.altura, respectivamente. El programa usa dos de esto nombre para visualizar la anchura y la altura de rectUno: 

System.out.println("Anchura de rectUno: "  + rectUno.anchura);
System.out.println("Altura de rectUno: " + rectUno.altura);

Intentar usar los nombres simples anchura y altura desde el código en la clase CreateObjectDemo no tiene sentido — esos campos sólo existen dentro de un objeto — y resulta en un error del compilador.

Más tarde, el programa usa código similar para visualizar información sobre rectDos. Los objetos del mismo tipo tienen su propia copia de los mismos campos de instancia. Así, cada objeto Rectangle tiene campos llamados origen, anchura, y altura. Cuando acced a un campo de instancia a través de una referencia a objeto, usted referencia ese campo particular del objeto. Los dos objetos rectUno y rectDos en el programa CreateObjectDemo tienen diferentes campos origen, anchura, y altura.

Para acceder a un campo, puede usar una referencia nombrada a un objeto, como en los ejemplos previos, o puede usar cualquier expresión que devuelva una referencia al objeto. Recuerde que el operador new devuelve una referencia a un objeto. Así podría usar el valor devuelto de new para acceder a los nuevos campos del objeto: 

int altura = new Rectangle().altura;

Esta sentencia crea un nuevo objeto Rectangle e inmediatamente obtiene su altura. En esencia, la sentencia calcula la altura por defecto de un Rectangle. Note que después de que esta sentencia ha sido ejecutada, el programa no mantiene el tener una referencia al Rectangle creado, porque el programa nunca almacena la referencia en ninguna parte. El objeto está desreferenciado, y sus recursos son libres de ser reciclados por la Máquina Virtual de Java.

Llamar a Métodos de un Objeto

Usted también usa una referencia al objeto para invocar un método del objeto. Añade el nombre siple del método a la referencia del objeto, con un operador punto interviniente (.). También, establece, encerrándolos entre paréntesis, varios argumentos al método. Si el método no requiere ningún argumento, usa los paréntesis vacíos. 

referenciaObjeto.nombreMetodo(listaArgumentos);

o: 

referenciaObjeto.nombreMetodo();

La clase Rectangle tiene dos métodos: getArea() para calcular el área del rectángulo y move() para cambiar el origen del rectángulo. Aqu'está el código de CreateObjectDemo que invoca los dos métodos: 

System.out.println("Area de rectUno: " + rectUno.getArea());
...
rectDos.move(40, 72);

La primera sentencia invoca el método getArea() de rectOne y visualiza los resultados. La segunda línea mueve rectDos porque el método move() asigna nuevos valores a los campos origen.x and origen.y del objeto.

Como con los campos de instancia, referenciaObjeto debe ser una referencia a un objeto. Puede usar un nombre de variable, pero también puede usar cualquier expresión que devuelva una referencia al objeto. El operador new devuelve una referencia al objeto, de forma que pueda usar el valor devuelto desde new para invocar los métodos del nuevo objeto: 

new Rectangle(100, 50).getArea()

La expresión new Rectangle(100, 50) devuelve una referencia al objeto que se refiera a un objeto Rectangle. Como se ha mostrado, puede usar la dotación punto para invocar el nuevo método getArea() de Rectangle's para calcular el área del nuevo rectangulo.

Algunos métodos, tales como getArea(), devuelven un valor. Para métodos que devuelven un valor, puede usar la invocación de métodos en expresiones. Puede asignar el valor de retorno a una variable, usarla para tomar decisiones, o controlar un bucle. Este código asigna el valor devuelto por getArea() a la variable areaOfRectangle: 

int areaDeRectangulo = new Rectangle(100, 50).getArea();

Recuerde, invocar un método en un objeto particular es lo mismo que enviar un mensaje al objeto. En este caso, el objeto cuyo método getArea() es invocado es el rectángulo devuelto por el constructor.

El Recolector de Basura

Algunos lenguajes orientado-a-objetos requieren que mantengas la pista de todos los objetos que crea y que explícitamente ha destrozado cuando ya no son necesarios por más tiempo. Gestionar la memoria explícitamente es tedioso y propenso-a-errores. La plataforma Java nos permite crear tantos objetos como queramos (limitado, por supuesto, a lo que su sistema puede manejar), y no tiene que preocuparse sobre destruirlos. El entorno de ejecución de Java borrar los objetos cuando determina que no van a ser más usados. Este proceso es llamdo recolectado de basura.

Un objeto es elegible para la recolección de basura cuando no hay más referencia a ese objeto. Referencias que están mantenidas en una variable son usualmente retiradas cuando la varible se sale de su alcance. O, puede retirar una referencia a objeto estableciendo la variable al valor especial . References that are held in a variable are usually dropped when the variable goes out of scope. Or, you can explicitly drop an retirar una referencia a objeto estableciendo la variable al valor especial . References that are held in a variable are usually dropped when the variable goes out of scope. Or, you can explicitly drop an retirar una referencia a objeto estableciendo la variable al valor especial null. Recuerde que un programa puede tener múltiples referencias al mismo objeto; todas las referencias a un objeto deben ser retiradas antes de que el objeto sea elegible para la recolección de basura.

El entorno de ejecución Java tiene un recolector de basura que periódicametne libera la memoria usada por los objetos que ya no son referenciados por más tiempo. El recolector de basura hace su trabajo automáticamente cuando determina que es el tiempo correcto.
