Ejemplo de Clase Interior

Para ver una clase interna en uso, primero consideremos una matriz. En el siguiente ejemplo, crea una matriz, lo llena con valores enteros, y entoces saca sólo los vaores de los índices pares de la matriz en orden ascendente.

El ejemplo EstructuraDatos.java que sigue consiste de:

  • La clase externa EstructuraDatos, la cual incluye un constructor para crear una instancia de EstructuraDatos conteniendo una matriz llena con valores enteros consecutivos (0, 1, 2, 3, y así) y un método que imprime los elementos de la matriz que tengan un valor de índice par.
  • La clase interna ParIterador, la cual implementa el interfaz EstructuraDatosIterador, el cual extiende el intefaz Iterator < Integer >. Los iteradores son usados para pasar a través de una estructura de datos y típicamente tienen métodos para comprobar si es el último elemento, recuperar el elemento actual, y moverse al siguiente elemento.
  • Un método main que instancia un objeto EstructuraDatos (ds), entonces invoca el método imprimePares para imprimir los elementos de la matriz matrizDeEnteros que tienen un valor de índice par. 
 
public class EstructuraDatos {

    // Crea una matriz
    private final static int TAMAÑO = 15;
    private int[] matrizDeEnteros = new int[TAMAÑO];

    public EstructuraDatos() {
        // llena la matriz con valores enteros ascendentes
        for (int i = 0; i < TAMAÑO; i++) {
            matrizDeEnteros[i] = i;
        }
    }

    public void imprimePares() {

        // Imprime los valores de los índices pares de la matriz
        EstructuraDatosIterador iterador = this.new ParIterador();
        while (iterador.hasNext()) {
            System.out.print(iterador.next() + " ");
        }
        System.out.println();
    }

    interface EstructuraDatosIterador extends java.util.Iterator <Integer> { }

    // clase interna que implementa el interfaz EstructuraDatosIterador,
    // la cual extiende el interfaz del Iterator<Integer>

    private class ParIterador implements EstructuraDatosIterador {

        // Inicia el recorrido a través de la matriz desde el principio
        private int siguienteIndice = 0;

        public boolean hasNext() {

            // Comprueba si el elemento actual es el último en la matriz
            return (siguienteIndice <= TAMAÑO - 1);
        }

        public Integer next() {

            // Graba un valor de un índice para de la matriz
            Integer valRetorno = Integer.valueOf(matrizDeEnteros[siguienteIndice]);

            // Obtiene el siguiente elemento par
            siguienteIndice += 2;
            return valRetorno;
        }
    }

    public static void main(String s[]) {

        // Llena la matriz con valores enteros e imprime solo
        // los valores de los índices pares
        EstructuraDatos ds = new EstructuraDatos();
        ds.imprimePares();
    }
}

La salida es: 

0 2 4 6 8 10 12 14

Note que la clase ParIterador hace referencia directamente a la variable de instancia matrizDeEnteros del objeto EstructuraDatos.

Puede usar clases internas para implementar clases ayudantes tales como la mostrada en este ejemplo. Para manejar eventos del interfaz de usuario, usted debe conocoer como usar las clases internas, porque el mecanismo de manejo-de-eventos hace un uso extensivo de ellas.

Clases Locales y Anónimas

Hay dos tipos adicionales de clases internas. Puede declarar una clase interna dentro del cuerpo de un método. Estas clases son conocidas como clases locales . También puede declarar una clase interna dentro del cuerpo de un método sin nombrar la clase. Estas clases son conocidas como clases anónimas .

Modificadores

Puede usar los mismos modificadores para las clases internas que usted usa para los otros miembros de la clase más externa. Por ejemplo, puede usar los especificadores de acceso private, public, y protected para restringir el acceso a las clases internas, de la misma forma que los usa para restringir el acceso a otros miembros de la clase.
