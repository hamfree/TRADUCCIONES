Inicializando Campos

Como ha visto, puede a menudo establecer un valor inicial para un campo en su declaración: 

public class CamaYDesayuno {

    // inicializa a 10
    public static int capacidad = 10;

    // inicializa a faslso
    private boolean lleno = false;
}

Esto funciona bien cuando el valor de inicialización está disponible y la inicialización puede ser colocada en una línea. Sin embargo, esta forma de inicialización tiene limitaciones debido a su simplicidad. Si la inicialización requiere alguna lógica (por ejemplo, manejo de errores o un bucle for para llenar una matriz compleja), la asignación simple es inadecuada. Las variables de instancia pueden ser inicializadas en constructores, donde el manejo de errores u otra lógica puede ser usada. Para establecer la misma capacidad para las variables de clase, el lenguaje de programación Java incluye los bloques estáticos de inicialización.

Nota: No es necesario declarar los campos al principio de la definición de clae, aunque ésta es la práctica más común. Solo es necesario que sean declarados e inicializados antes de ser usados.

Bloques Estáticos de Inicialización

Un bloque estático de inicialización es un bloque normal de código encerrado entre llaves, { }, y precedido de la palabra clave static. Aquí tiene un ejemplo: 

static {
    // cualquier código que se necesita para la inicialización va aquí
}

Una clase puede tener cualquier número de bloques estáticos de inicialización, y pueden aparecer en cualquier parte en el cuerpo de la clase. El sistema de ejecución garantiza que los bloques estáticos de inicialización son llamados en el orden en el que aparecen en el código fuente.

Hay una alternativa a los bloques estáticos — puede escribir un método estático privado: 

class LoQueSea {
    public static tipoVar miVar = inicializaVariableDeClase();
        
    private static tipoVar inicializaVariableDeClase() {

        // código de inicialización va aquí
    }
}

La ventaja de los métodos estáticos privados es que pueden ser reutilizados más tarde si necesita reinicializar la variable de clase.

Inicializar Miembros de Instancia

Normalmente, usted pondría el código para inicializar una variable de instancia en un constructor. Hay dos opciones a usar un constructor para inicializar variables de instancia: bloques inicializadores y métodos finales.

Bloques inicializadores para variables de instancia se parecen a los bloques estáticos inicializadores, pero sin la palabra clave static: 

{
    // cualquier código que se necesita para la inicialización va aquí
}

El compilador Java copia los bloques inicializadores dentro de cada constructor. Así, éste enfoque puede ser utilizado para compartir un bloque de código entre múltiples constructores.

Un metodo final no puede ser sobreescrito en una subclase. Esto se describió en la lección sobre interfaces y herencia. Aquí tiene un ejemplo usando un método final para inicializar una variable de instancia: 

class LoQueSea {
    private tipoVar miVar = inicializaVariableDeInstancia();
        
    protected final tipoVar inicializaVariableDeInstancia() {

        // el código de inicialización va aquí
    }
}

Esto es especialmente útil si las subclases quisiesen reutilizar el método de inicialización. El método es final porque la llamada a métodos no-finales durante la inicialización de la instancia puede causar problemas.
