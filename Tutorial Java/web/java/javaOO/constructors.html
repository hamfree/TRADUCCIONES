Proporcionando Constructores Para Sus Clases

Una clase contiene constructores que son invocados para crear objetos de la clase plantilla. Las declaraciones de constructores se parecen a las declaraciones de métodos — excepto que usan el nombre de la clase y no retornan nada. Por ejemplo, Bicicleta tiene un constructor: 

public Bicicleta(int cadenciaInicio, int velocidadInicio, int marchaInicio) {
    marcha = marchaInicio;
    cadencia = cadenciaInicio;
    velocidad = velocidadInicio;
}

Para crear un nuevo objeto Bicicleta llamado miBici, se llama a un constructor con el operador new : 

Bicicleta miBici = new Bicicleta(30, 0, 8);

new Bicicleta(30, 0, 8) crea espacio en memoria para el objeto e inicializa sus campos.

Aunque Bicicleta sólo tiene un constructor, podría tener otros, incluyendo un constructor sin argumentos: 

public Bicicleta() {
    marcha = 1;
    cadencia = 10;
    velocidad = 0;
}

Bicicleta tuBici = new Bicicleta(); invoca al constructor sin argumentos para crear un nuevo objeto Bicicleta llamado tuBici.

Ambos constructores podrían haber sido declarados en Bicicleta porque tienen diferentes listas de argumentos. Como con los métodos, la plataforma Java diferencia los constructores sobre la base del número de argumentos en la lista y sus tipos. No puede escribir dos constructores que tengan el mismo número y tipo de argumentos para la misma clase, porque la plataforma no sería capaz de distinguirlos. Hacerlo así causa un error en tiempo de compilación.

No tiene que establecer ningún constructor para su clase, pero sea cuidadoso cuando lo haga. El compilador automáticamente establece un constructor por defecto, sin argumentos para cualquier clase sin constructores. Este constructor por defecto llamará al constructor sin argumentos de la superclase. En esta situación, el compilador se quejará si la superclase no tiene un constructor sin argumentos así que usted deberá verificar que lo tiene. Si su clase no tiene una superclase explícita, entonces tiene una superclase implícita que es Object, la cual tiene un constructor sin argumentos.

Puede usar un constructor de superclase por sí mismo, La clase BicicletaMontaña al principio de esta lección hace justo eso. Esto será descrito más tarde, en la lección de interfaces y herencia.

Puede usar modificadores de acceso en una declaración de constructor para controlar cuáles de las otras clases pueden llamar al constructor.

Nota: Si otra clase no puede llamar a un constructor de MiClase, no puede crear directamente objetos MiClase.
