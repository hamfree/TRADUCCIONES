Clases Anidadas

El lenguaje de programación Java le permite definir una clase dentro de otra clase. Esta clase es llamada una clase anidada y está ilustrada aquí: 

class ClaseMasExterna {
    ...
    class ClaseAnidada {
        ...
    }
}
Terminologia: Las clases anidadas están divididas en dos categorias: estáticas y no estáticas. Las clases anidadas que son declaradas static son llamadas clases anidadas estáticas. Las clase anidadas no estáticas son llamadas clases internas. 
class ClaseMasExterna {
    ...
    static class ClaseAnidadaEstatica {
        ...
    }
    class ClaseInterna {
        ...
    }
}

Una clase anidada es un miembro de la clase que la encierra. Las clases anidadas no estáticas (clases internas) tienen acceso a otros miembros de la clase que la encierra, incluso si es declarada privada. Las clases anidadas estáticas no tienen acceso a otros miembros de la clase que la encierra. Como un miembro de la ClaseMasExterna, una clase anidada puede ser declarada privada, pública, protegida, o privada para el paquete. (Recuerde que las clases más externas pueden ser sólo declaradas públicas o privadas para el paquete).

¿Porqué Usar Clases Anidadas?

Razones de peso para el uso de clases anidadas incluyen las siguientes:

  • Es una forma de agrupar lógicamente clases que son sólo usadas en un lugar: Si una clase es útil sólo a otra clase, entonces es lógico embeberla en esa clase y mantener las dos juntas. Anidar tal "clase ayudante" hace su empaquetado más compacto.

  • Incrementa la encapsulación: Considere dos clases del nivel más alto, A y B, donde B necesita acceso a miembros de A que de otro modo deberían ser declarados privados. Ocultando la clase B dentro de la clase A, los miembros de la clase A pueden ser declarados privados y B puede acceder a ellos. Además, B en sí misma puede ser ocultada del mundoe exterior.

  • Puede conducir a un código más legible y mantenible: Anidar pequeñas clases dentro de de clases del más alto nivel coloca el código más cerca de donde es usado.

Clases Anidadas Estáticas

Como con los métodos y variables de clase, una clase anidada estática se asocia con su clase más externa. Y como los métodos de clase estáticos, un clase anidada estática no puede referirse directamente a las variables de instancia o métodos definidos en la clase que la encierra: puede usarlos sólo a través de referencias a objeto.

Noae: Un clase anidada estática interactúa con los miembros de instancia de su clase más externa (y otras clases) justo como cualquier otra clase del más alto nivel. En efecto, una claes estática anidada es desde el punto de vista del comportamiento una clase del más alto nivel que ha sido anidada en otra clase del más alto nivel por convenencia de empaquetado.

Las clases anidadas estáticas son accedidas usando el nombre de la clase que la encierra: 

ClaseMasExterna.ClaseAnidadaEstatica

Por ejemplo, para crear un objeto para la clase anidada estática, usaremos esta sintaxis: 

ClaseMasExterna.ClaseAnidadaEstatica objetoAnidado= new ClaseMasExterna.ClaseAnidadaEstatica();

Clases Internas

Como con los métodos de instancia y variables, una clase interna está asociada con una instancia de la clase que la encierra y tiene acceso directo a esos objetos, métodos y campos. También, porque una clase interna está asociada con una instancia, no puede defnir ningún miembro estático dentro de sí misma.

Los objetos que son instancias de una clase intena existen dentro de una instancia de la clase más esxterna. Considere las siguietnes clases: 

class ClaseMasExterna{
    ...
    class ClaseInterna {
        ...
    }
}

Una instancia de ClaseInterna puede existir sólo dentro de una instancia de la ClaseMasExterna y tiene acceso directo a los método sy campos de la instancia que la encierra.

Para instanciar una clase interna, primero instancia la clase más externa. Entonces, cree el objeto más interno dentro del objeto más externo con esta sintáxis: 

ClaseMasExterna.ClaseInterna objetoInterno= ClaseMasExterna.new ClaseInterna();

Hay dos clases especiales de clases internas: clases locales y clases anónimas.

Ocultación

Si una declaración de un tipo (como una variable miembro o un nombre de parámetro) en un alcance particular (tal como una clase interna o una definición de método) tiene el mismo nombre que otra declaración en el alcance de cierre, entonces la declaración oculta la declaración del alcance de cierre. No puede referirse a la declaración ocultada por su nombre sólo. El siguiente ejemplo, PruebaSombra , demuestra esto: 

 
public class PruebaSombra {

    public int x = 0;

    class PrimerNivel {

        public int x = 1;

        void metodoEnPrimerNivel(int x) {
            System.out.println("x = " + x);
            System.out.println("this.x = " + this.x);
            System.out.println("PruebaSombra.this.x = " + PruebaSombra.this.x);
        }
    }

    public static void main(String... args) {
        PruebaSombra st = new PruebaSombra();
        PruebaSombra.PrimerNivel fl = st.new PrimerNivel();
        fl.metodoEnPrimerNivel(23);
    }
}

Lo siguiente es la salida de este ejemplo: 

x = 23
this.x = 1
PruebaSombra.this.x = 0

Este ejemplo define tres variables nombradas x: la variable miembro de la clasess PruebaSombra, la variable miembro de la clase interna PrimerNivel, y el parámetro en el método metodoEnPrimerNivel. La variable x definida como un parámetro del método metodoEnPrimerNivel oculta la variable de la clase interna PrimerNivel. Consecuentemente, cuando usa la variable x en el método metodoEnPrimerNivel, se refiere al parámetro del método. Para referirse a la variable miembro de la clase interna PrimerNivel, use la palabra clave this para representar el alcance de cierre: 

System.out.println("this.x = " + this.x);

Se refieren a variables miembro que encierran alcances más grandes por el nombre de la clase a la cual pertenecen. Por ejemplo, la siguiente sentencia accede a la variable miembro de la clase PruebaSombra desde el método metodoEnPrimerNivel: 

System.out.println("PruebaSombra.this.x = " + PruebaSombra.this.x);

Serialización

La serialización de las clases internas, incluyendo clases locales y anónimas , no es recomendado. Cuando el compilador de Java compila ciertas construcciones, tales como una clase interna, crea construcciones sintéticas; estas son clases, métodos, campos, y otras construcciones que no tienen una construcción correspondiente en el código fuente. Las construcciones sintéticas habilitan a los compiladores de Java a implementar nuevas funcionalidades del lenguaje Java sin cambiar la MVJ (JVM en inglés). Sin embargo, las construcciones sintéticas pueden variar entre diferentes implementaciones de compiladores de Java, lo cual significa que los ficheros .class pueden variar entre diferentes implementaciones también. Por consiguiente, puede tener problemas de compatibilidad si serializa una clase interna y entonces la deserializa con una implementación diferente del EEJ (JRE en inglés). Vea la sección Parámetrros Implícitos y Sintéticos en la sección Obtener Nombres y Parámetros de Método para más información sobre las construcciones sintéticas generadas cuando una clase interna es compilada.
