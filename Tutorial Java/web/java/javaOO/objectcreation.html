Creando Objetos

Como sabe, una clase ofrece la plantilla para los objetos; usted crea un objeto desde una clase. Cada una de las siguientes declaraciones tomadas del programa CreateObjectDemo crea un objeto y lo asigna a una variable: 

Point origenUno = new Point(23, 94);
Rectangle rectUno = new Rectangle(origenUno, 100, 200);
Rectangle rectDos = new Rectangle(50, 100);

La primera línea crea un objeto de la clase Point , y la segunda y tercera líneas cada una crean un objeto de la clase Rectangle .

Cada una de estas sentencias tiene tres partes (descritas en detalle abajo):

  1. Declaración: El código fijado en negrita son todas las declaraciones de variables que asocian un nombre de variable con un tipo de objeto.
  2. Instanciación: La palabra clave new es un operador Java que crea el objeto.
  3. Inicialización: El operador new es seguido por una llamada a un constructor, el cual inicializa el objeto.

Declarar una Variable para Referirse a un Objeto

Previamente, aprendió a declarar una variable, usted escribe: 

tipo nombre;

Esto notifica al compilador que usará nombre para referirse al dato cuyo tipo es tipo. Con una variable primitiva, esta declaración también reserva la cantidad apropiada de memoria para la variable.

Puede también declarar una variable de referencia en su propia línea. Por ejemplo: 

Point origenUno;

Si declara origenUno como esto, su valor estará indeterminado hasta que un objeto esté realmente creado y asignado a el. Simplemente declarando una variable de referencia no crea un objeto. Para eso, necesita usar el operador new , como está descrito en la siguiente sección. Debe asignar un objeto a origenUno antes de usarlo en su código. De otra forma, obtendrá un error del compilador.

Instanciando una Clase

El operador new instancia una clase asignando memoria para un nuevo objeto y retorna una referencia a esa memoria. El operador new también invoca el constructor del objeto.

Nota: La frase "instancia una clase" significa la misma cosa que "crea un objeto." Cuando crea un objeto, está creando una "instancia" de una clase, por lo tanto "instancia" una clase.

El operador new requiere un argumento único postfijo: una llamada a un constructor. El nombre del constructor ofrece el nombre de la clase a instanciar.

El operador new devuelve una referencia al objeto que creó. Esta referencia es usualmente asignada a una variable del tipo apropiado, como: 

Point origenOUno = new Point(23, 94);

La referencia devuelta por el operador new no tiene que ser asignada a una variable. Puede también ser usada directamente en una expresión. Por ejemplo: 

int altura = new Rectangle().altura;

Esta sentencia será descrita en la siguiente sección.

Inicializando un Objeto

Aquí tiene el código para la clase Point : 

public class Point {
    public int x = 0;
    public int y = 0;

    // constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}

Esta clase contiene un único constructor. Puede reconocer un constructor porque su declaración usa el mismo nombre que el de la clase y no retorna ningún tipo. El constructor en la clase Point toma dos argumentos enteros, como fueron declarados por el código (int a, int b). La siguiente sentencia establece 23 y 94 como valores para esos argumentos: 

Point origenUno = new Point(23, 94);

Aquí está el código para la clase Rectangle class, la cual contiene cuatro constructores: 

public class Rectangle {

    public int ancho;
    public int altura;
    public Point origen;

    // cuatro constructores
    public Rectangle() {
        origen = new Point(0, 0);
    }

    public Rectangle(Point p) {
        origen = p;
    }

    public Rectangle(int anch, int alt) {
        this(new Point(0, 0), anch, alt);
    }

    public Rectangle(Point p, int anch, int alt) {
        origen = p;
        ancho = anch;
        altura = alt;
    }

    // un método para mover un rectángulo
    public void move(int x, int y) {
        origen.x = x;
        origen.y = y;
    }

    // un método para calcular el área de un rectángulo
    public int area() {
        return ancho * altura;
    }
}

Cada constructor le permite establecer valores iniciales para el origen, anchura, y altura del rectángulo, usando tipos primitivos y referencias. Si una clase tiene múltiples constructores, debe tener diferentes signaturas. El compilador de Java diferencia los constructores basándose en el número y tipo de los argumentos. Cuando el compilador de Java encuentra el siguiente código, sabe como llamar al constructor en la clase Rectangle que requiere un argumentoPoint seguido de dos argumentos enteros: 

Rectangle rectUno = new Rectangle(origenUno, 100, 200);

Este llama uno de los constructores de Rectangle que inicializa origin a origenUno. También, el constructor establece el ancho a 100 y la altura a 200. Ahora hay dos referencias al mismo objeto Point — un objeto puede tener múltiples referencias a sí mismo, como es mostrado en la siguiente figura:

La siguiente línea de código llama el constructor de Rectangle que requiere dos argumentos enteros, los cuales establecen los valores iniciales para anchura y altura. Si inspecciona el código dentro del constructor, verá que crea un nuevo objeto Point cuyos valores x y y son inicializados a 0: 

Rectangle rectDos = new Rectangle(50, 100);

El constructor de Rectangle usado en la siguiente sentencia no toma ningún argumento, así que se le llama un constructor sin argumentos: 

Rectangle rect = new Rectangle();

Todas las clases tienen al menos un constructor. Si una clase no declara explícitamente uno, el compilador de Java automáticamente asigna un constructor sin argumentos, llamado el constructor por defecto. Este constructor por defecto llama al constructor sin argumentos de la clase padre, o al constructor de Object si la clase no tiene otro padre. Si el padre no tiene constructor (Object tiene uno), el compilador rechazará el programa.
