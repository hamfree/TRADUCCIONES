Pasando Información a un método o a un constructor

La declaración de un método o un constructor declara el número y el tipo de los argumentos para ese método o constructor. Por eejemplo, lo siguiente es un método que computa los pagos mensuales para una hipoteca de una casa, basado en la cantidad de la hipoteca, la tasa de interés, la duración de la hipoteca (el número de períodos), y el valor futuro de la hipoteca: 

public double calculaPago(
                  double cantHipoteca,
                  double tasa,
                  double valorFuturo,
                  int numPlazos) {
    double interes = tasa / 100.0;
    double parcial1 = Math.pow((1 + interes),
                    - numPlazos);
    double denominador = (1 - parcial1) / interes;
    double respuesta = (-cantHipoteca / denominador)
                    - ((valorFuturo * parcial1) / denominador);
    return respuesta;
}

Este método tiene cuatro parámetros: la cantidad de la hipoteca, la tasa de interés , el valor futuro y el número de plazos. Los primeros tres son número de coma flotante de doble precisión, y el cuarto es un entero. Los parámetros se utilizan en el cuerpo del método y en tiempo de ejecución se llevará en los valores de los argumentos que se pasan.

Nota: Parámetros se refire a la lista de variables en una declaración de método. Argumentos son los valores actuales que son pasados dentro cuando el método es invocado. Cuando invoca un metodo, los argumentos usados deben coincidir con los parámetros de la declaración en tipo y orden.

Tipos de Parámetro

Puede usar cualquier tipo de dato para un parámetro de un método o un constructor. Esto incluye tipos de datos primitivos, como doubles, floats, e integers, domo vió en el método calculaPago, y los tipos de datos por referencia, como objetos y matrices.

Aquí tiene un ejemplo de un método que acepta una matriz como un argumento. En este ejemplo, el método crea un nuevo objeto Poligono y lo inicializa con los objetos de la matriz de Puntos (asumimos que Punto es una clase que representa una coordenada x, y): 

public Poligono poligonoDe(Punto[] esquinas) {
    // el cuerpo del método va aquí
}
Nota: Si quiere pasar un método dentro de un método, entonces use una If you want to pass a method into a method, then use a expresión lambda o un referencia a método.

Número Arbitrario de Argumentos

Puede usar una construcción llamada varargs para pasar un número arbitrario de valores a un método. Use varargs cuando no sepa cuantos valores de un tipo particular de argumento serán pasados al método. Es un atajo para crear una matriz manualmente (el método previo podría haber usado varargs en vez de una matriz).

Para usar varargs, debe seguir el tipo del último parámetro por puntos suspensivos (tres puntos, ...), y después un espacio, y el nombre del parámetro. El método puede entonces ser llamado con cualquier número de valores de ese parámetro, incluyendo ninguno. 

public Poligono poligonoDesde(Punto... esquinas) {
    int numeroDeCaras = esquinas.length;
    double cuadradoDeLado1, longitudDeLado1;
    cuadradoDeLado1 = (esquinas[1].x - esquinas[0].x)
                     * (esquinas[1].x - esquinas[0].x)
                     + (esquinas[1].y - esquinas[0].y)
                     * (esquinas[1].y - esquinas[0].y);
    longitudDeLado1 = Math.sqrt(cuadradoDeLado1);

    // sigue más código en el cuerpo del método que crea y devuelve un
    // polígono conectando los Puntos
}

Puede ver que, dentro del método, esquinas es tratado como una matriz. El método puede ser llamado con una matriz o con una secuencia de argumentos. El código en el cuerpo del método tratará el parámetro como una matriz en cualquier caso.

Verá más habitualmente varargs con los métodos de impresión; por ejemplo, éste método printf : 

public PrintStream printf(String format, Object... args)

le permite imprimir un número arbitrario de objetos. Puede ser llamado así: 

System.out.printf("%s: %d, %s%n", nombre, dni, direccion);

o como esto 

System.out.printf("%s: %d, %s, %s, %s%n", nombre, dni, direccion, telefono, correoElectronico);

O con aún un número diferente de argumentos.

Nombres de los Parámetros

Cuando declara un parámetro en un método o en un constructor, establece un nombre para el parámetro. Este nombre es usado dentro del cuerpo del método para referirse al argumento que se pasa dentro.

El nombre de un parámetro debe ser único en su alcance. No puede ser el mismo que el nombre de otro parámetro para el mismo método o constructor, y no puede ser el nombre de una variable local dentro del método o constructor.

Un parámetro puede tener el mismo nombre que uno de los campos de la clase. Si este es el caso, se dice que el parámetro oculta el campo. La ocultación de campos puede hacer su código difícil de leer y es convencionalmente usado sólo dentro de constructores y métodos que establecen un campo particular. Por ejemplo, considere la siguiente clase Circulo y su método setOrigen : 

public class Circulo {
    private int x, y, radio;
    public void setOrigen(int x, int y) {
        ...
    }
}

La clase Circulo tiene tres campos: x, y, y radio. El método setOrigen tiene dos parámetros, cada uno de los cuales tiene el mismo nombre que uno de los campos. Cada parámetros del método oculta el campo que comparte su nombre. Asñi usar los nombre simples x o y dentro del cuerpo del método hace referencia al parámetro, no al campo. Para acceder al campo, debe usar un nombre cualificado. Esto será descrito más tarde en esta lección en la sección titulada "Usando la palabra clave this ."

Pasando Argumentos de Tipos de Datos Primitivos

Los argumentos primitivos, tales como un int o un double, son pasados dentro de los métodos por valor. Esto significa que cualquier cambio a los valores de los parámetros existe sólo dentro del alcance del método. Cuando el método retorna, los parámetros se han ído y cualquier cambio en ellos se pierde. Aquí tiene un ejemplo: 

public class pasarPrimitivosPorValor {

    public static void main(String[] args) {
           
        int x = 3;
           
        // invocamos pasarMetodo() con
        // x como argumento
        pasarMetodo(x);
           
        // imprimimo x para ver si su
        // valor ha cambiado
        System.out.println("Despues de invocar pasarMetodo, x = " + x);
           
    }
        
    // cambia parámetro en pasarMetodo()
    public static void pasarMetodo(int p) {
        p = 10;
    }
}

Cuando ejecuta este programa, la salida es: 

Despues de invocar pasarMetodo, x = 3

Pasando Argumentos de Tipos de Datos por Referencia

Los parámetros de tipos de datos por referencia, como objetos, son también pasado a los métodos por valor. Esto significa que cuando el método retorna, la referencia pasada dentro aun referencia al mismo objeto como antes. Sin embargo , los valores de los campos del objeto pueden ser cambiados en el méotod, si tienen el nivel de acceso apropiado.

Por ejemplo, considere un método en una clase arbitraria que mueve objetos Circulo : 

public void mueveCirculo(Circulo circulo, int deltaX, int deltaY) {
    // código para mover el origen del circulo a x+deltaX, y+deltaY
    circulo.setX(circulo.getX() + deltaX);
    circulo.setY(circulo.getY() + deltaY);
        
    // código para asignar una nueva referencia a circulo
    circulo = new Circulo(0, 0);
}

Deje que el método sea invocado con estos argumentos: 

mueveCirculo(miCirculo, 23, 56)

Dentro del método, circulo inicialmente se refiere a miCirculo. El método cambia las coordenadas x e y del objeto al que circulo referencia (i.e., miCirculo) por 23 y 56, respectivamente. Estos cambios persistirán cuando el método retorne. Entonces circulo es asignado a una referencia a un nuevo objeto Circulo con x = y = 0. Esta reasignación no tiene permanencia, sin embargo, porque la referencia fue pasada por valor y no se puede cambiar. Dentro del método, el objeto apuntado a circulo ha cambiado, pero, cuando el método retorna, miCirculo aún referencia al mismo objeto Circulo que antes de que el método fuera llamado.
