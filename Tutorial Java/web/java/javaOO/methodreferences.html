Referencias a Métodos

Usted usa las You use expresiones lambda para crear métodos anónimos. Algunas veces, sin embargo, una expresión lambda no hace nada más que llamar a un método existente. En esos casos, es a menudo más claro referirse al método existente por el nombre. Las referencias a método le habilitan a hacer esto; son compactas, expresiones lambda fáciles-de-leer para métodos que ya tienen un nombre.

Considere de nuevo la clase Persona descrita en la sección Expresiones Lambda : 

public class Persona {
  
    public enum Sexo {
        MASCULINO, FEMENINO
    }
  
    String nombre; 
    LocalDate cumpleaños;
    Sexo genero;
    String correoElectronico;
  
    Persona(String nameArg, LocalDate birthdayArg,
        Sexo genderArg, String emailArg) {
        nombre = nameArg;
        cumpleaños = birthdayArg;
        genero = genderArg;
        correoElectronico = emailArg;
    }  

    public int getEdad() {
            //...
    }

    //...
    
    public static int comparaPorEdad(Persona a, Persona b) {
        return a.cumpleaños.compareTo(b.cumpleaños);
    }
}

Suponga que los miembros de su aplicación de red social están contenidos en una matriz, y usted quiere ordenar la matriz por edad. Podría usar el siguiente código (busque los extractos de código descritos en esta sección en el ejemplo PruebaReferenciaMetodo ): 

Persona[] agendaComoMatriz = 
            agenda.toArray(new Persona[agenda.size()]);
        
        class PersonaEdadComparator
            implements Comparator<Persona> {
            public int compare(Persona a, Persona b) {
                return a.getCumpleaños().compareTo(b.getCumpleaños());
            }
        }
        
Arrays.sort(agendaComoMatriz, new PersonaEdadComparator());

La signatura del método de esta invocación de sort es la siguiente: 

static <T> void sort(T[] a, Comparator<? super T> c)

Note que el interfaz Comparator es una interfaz funcional. De esta forma, podría usar una expresión lambda en vez de definir y entonces crear una nueva instancia de una clase que inmplementa Comparator: 

Arrays.sort(agendaComoMatriz,
    (Persona a, Persona b) -> {
        return a.getCumpleaños().compareTo(b.getCumpleaños());
    }
);

Sin embargo, este método para comparar las fechas de nacimiento de dos instancias Persona ya existe como Persona.comparePorEdad. Puede invocar este método en cambio en el cuerpo de la expresión lambda: 

Arrays.sort(agendaComoMatriz,
    (a, b) -> Person.comparePorEdad(a, b)
);

Porque esta expresión lambda invoca un método existente, puede usar una referencia a método en vez de una expresión lambda: 

Arrays.sort(agendaComoMatriz, Persona::comparePorEdad);

La referencia al método Person::compareByAge es semánticamente lo mismo que la expresión lambda (a, b) -> Persona.comparePorEdad(a, b). Cada una tiene las siguientes características:

  • Su lista de parámetros formales es copiada de Comparator<Persona>.compare, la cual es (Persona, Persona).
  • Su cuerpo llama el método Persona.comparePorEdad.

Clases de Referencias a Métodos

Hay cuatro clases de referencias a métodos:

Clases de referencias a métodos
Clase Ejemplo
Referencia a un método estático ClaseContenedora::estaticoNombreMetodo
Referencia a un método de instanca de un objeto particular objetoContenedor::instanciaNombreMetodo
Referencia a una instancia de método de un objeto arbitrario de un tipo particular TipoContenedor::nombreMetodo
Referencia a un constructor NombreClase::new

Referencia a un Método Estático

La referencia a método Persona::comparePorEdad es una referencia a un método estático.

Referencia a una Instancia de Método de un Objeto Particular

El siguiente es un ejemplo de una referencia a una instancia de método de un objeto particular: 

class ProveedorComparacion {
    public int comparaPorNombre(Persona a, Persona b) {
        return a.getNombre().compareTo(b.getNombre());
    }
        
    public int comparaPorEdad(Persona a, Persona b) {
        return a.getCumpleaños().compareTo(b.getCumpleaños());
    }
}

ProveedorComparacion miProveedorComparacion = new ProveedorComparacion();
Arrays.sort(agendaComoMatriz, miProveedorComparacion::comparaPorNombre);

La referencia al método miProveedorComparacion::comparaPorNombre invoca el método compareByName que es parte del objeto miProveedorComparacion. El EEJ (JRE en inglés) infiere los tipos de los argumentos del método, los cuales en este caso son (Persona, Persona).

Referencia a una Instancia de Método de un Objeto Arbitrario de un Tipo Particular

El siguiente es un ejemplo de una referencia a una instancia de método de un objeto arbitrario de un tipo particular: 

String[] matrizString = { "Barbara", "James", "Mary", "John",
    "Patricia", "Robert", "Michael", "Linda" };
Arrays.sort(matrizString, String::compareToIgnoreCase);

La expresión lambda equivalente para la referencia a método String::compareToIgnoreCase tendría la lista de parámetros formales (String a, String b), donde a y b son nombres arbitrarios usado para describir mejor este ejemplo. La referencia a método invocaría el método a.compareToIgnoreCase(b).

Referencia a un Constructor

Usted puede referenciar un constructor en la misma forma que un método estático usando el nombre new. El siguiente método copia elementos de una colección a otra: 

public static <T, ORIGEN extends Collection<T>, DEST extends Collection<T>>
        DEST pasaElementos(
            ORIGEN coleccionOrigen,
            Supplier<DEST> factoriaColecciones) {
        
            DEST resultado = factoriaColecciones.get();
            for (T t : coleccionOrigen) {
                resultado.add(t);
            }
            return resultado;
    }

El interfaz funcional Supplier contiene un método get que no toma argumentos y devuelve un objeto. Por consiguiente, usted puede invocar el método pasaElementos con una expresión lambda como sigue: 

Set<Persona> agendaSetLambda =
    pasaElementos(agenda, () -> { return new HashSet<>(); });

Puede usar una referencia a constructor en lugar de la expresión lambda como sigue: 

Set<Person> agendaSet = pasaElementos(agenda, HashSet::new);

El compilador de Java infiere que usted quiere crear una colección HashSet que contiene elementos del tipo Persona. Alternativamente, puede especificar esto como sigue: 

Set<Persona> agendaSet = pasaElementos(agenda, HashSet<Persona>::new);
