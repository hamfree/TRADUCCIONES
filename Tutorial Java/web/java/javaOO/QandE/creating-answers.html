Respuestas a Preguntas y Ejercicios: Clases

Preguntas

  1. Considere la siguiente clase: 

    public class IdentificaMisPartes {
    public static int x = 7;
    public int y = 3;
}

    1. Pregunta: ¿Cuáles son las variables de clase?

      Respuesta: x

    2. Pregunta: ¿Cuáles son las variables de instancia?

      Respuesta: y

    3. Pregunta: Cuál es la salida del siguiente código: 

      IdentificaMisPartes a = new IdentificaMisPartes();
IdentificaMisPartes b = new IdentificaMisPartes();
a.y = 5;
b.y = 6;
a.x = 1;
b.x = 2;
System.out.println("a.y = " + a.y);
System.out.println("b.y = " + b.y);
System.out.println("a.x = " + a.x);
System.out.println("b.x = " + b.x);
System.out.println("IdentificaMisPartes.x = " + IdentificaMisPartes.x);

      Respuesta: Aquí esta la salida: 

       a.y = 5 
 b.y = 6 
 a.x = 2 
 b.x = 2
 IdentificaMisPartes.x = 2

      porque x está definida como una public static int en la clase IdentifyMyParts, cada referencia a x tendrá el valor que tenía últimamente asignado porque x es una variable estática (y por lo tanto una variable de clase) compartida entre todas las instancias de la clase. Esto es, hay sólo una x: cuando el valor de x cambia en cualquier instancia ello afecta al valor de x para todas las instancias deIdentificaMisPartes.

      Esto se cubre en la sección Variables de Clase de Entendiendo los Miembros de una Clase.

    Respuesta: Carta.java .

  2. Ejercicio: Escriba una clase cuyas instancias representen una baraja completa de cartas. Deberá guardar también esta solución.

    Respuesta: Vea Baraja.java .

  3. Ejercicio: Escriba un pequeño programa para comprobar sus clases de baraja y cartas. El programa puede ser tan simple como crear una baraja de cartas y visualizar sus cartas.

    Respuesta: Vea VisualizaBaraja.java .
