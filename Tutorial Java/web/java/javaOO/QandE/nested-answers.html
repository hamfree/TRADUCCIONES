Respuestas a Preguntas y Ejercicios: Clases Anidadas

Preguntas

  1. Pregunta: El programa Problema.java no compila. ¿Qué necesita hacer para hacerlo compilar? ¿Porqué?

    Respuesta: Borre static en frente de la declaración de la clase Interna. Una clase interna estática no tiene acceso a los campos de la instancia de la clase exterior. Vea ProblemaResuelto.java .

  2. Use la documentación del IPA de Java para la clase Box (en el paquete javax.swing) para ayudarle a responder a las siguientes preguntas.

    1. Pregunta: ¿Qué clase anidada estática define Box?

      Respuesta: Box.Filler

    2. Pregunta: ¿Qué clase interior define Box?

      Respuesta: Box.AccessibleBox

    3. Pregunta: ¿Cuál es la superclase de la clase interior de Box?

      Respuesta: [java.awt.]Container.AccessibleAWTContainer

    4. Pregunta: ¿Cuál de las clases anidadas de Box puede usted usar desde cualquier clase?

      Respuesta: Box.Filler

    5. Pregunta: ¿Cómo crea una instancia de la clase Filler de Box?

      Respuesta: new Box.Filler(minDimension, prefDimension, maxDimension)

Ejercicios

  1. Ejercicio: Obtenga el fichero Clase1.java . Compile y ejecute Clase1. ¿Cuál es la salida?

    Respuesta: InteriorClase1: dameCadena invocado.
    InteriorClase1: dameOtraCadena invocado.

  2. Ejercicio: Los siguientes ejercicios implican modificar la clase EstructuraDatos.java , la cual la sección Ejemplo de Clase Interior describe.

    1. Defina un método nombrado print(EstructuraDatosIterador iterator). Invoque este método con una instancia de la clase ParIterador de forma que ejecute la misma función que el método imprimePares.

      Consejo: Estas sentencias no compilan si usted las especifica en el método main : 

      EstructuraDatos ds = new EstructuraDatos();
ds.print(new ParIterador());

      El compilador genera el mensaje de error, "la variable no-estática no puede ser referenciada desde un contexto estático" cuando encuentra la expresión new ParIterador(). La clase ParIterador es una clase interna, no estática. Esto significa que puede crear una instancia de ParIterador sólo dentro de una instancia de la clase envolvente, EstructuraDatos.

      Puede definir un método en EstructuraDatos que crea y devuelve una nueva instancia de ParIterador.

    2. Invoque el método print(EstructuraDatosIterador iterador) de forma que imprima elementos que tienen un valor de índice impar. Use una clase anónima como el argumento del método en vez de una instancia del interfaz EstructuraDatosIterador.

      Consejo: Usted no puede acceder a los miembros privados TAMAÑO y matrizDeEnteros fuera de la clase EstructuraDatos, lo cual significa que usted no puede acceder estos miembros privados desde una clase anónima definida fuera de EstructuraDatos.

      Puede definir métodos que accedan a los miembros privados TAMAÑO y matrizDeEnteros y entonces usarlos en su clase anónima.

    3. Defina un método nombrado print(java.util.Function<Integer, Boolean> iterator) que ejecute la misma función que print(EstructuraDatosIterador iterador). Invoque este método con una expresión lambda para imprimir elementos que tengan un valor de índice impar. Invoque este método de nuevo con una expresión lambda para imprimir elementos que tengan un valor de índice par.

      Consejo: En este método print, puede recorrer los elementos contenidos en la matriz matrizDeEnteros con una expresión for. Para cada valor de índice, invoque el método function.apply. Si este método devuelve un valor verdadero para un valor de índice particular, imprima el elemento contenido en ese valor de índice.

      Para invocar este método print para imprimir elementos que tienen un valor de índice par, puede especificar una expresión lambda que implemente el método Boolean Function.apply(Integer t). Esta expresión lambda toma un argumento Integer (el índice) y devuelve un valor Boolean (Boolean.TRUE si el valor de índice es par, Boolean.FALSE en caso contrario).

    4. Defina dos métodos de forma que estas sentencias impriman elementos que tengan un valor de índice par y entonces elementos que tengan un valor de índice impar: 

      EstructuraDatos ds = new EstructuraDatos()
// ...
ds.print(EstructuraDatos::esIndicePar);
ds.print(EstructuraDatos::esIndiceImpar);

      Consejo: Cree dos métodos en la clase EstructuraDatos de nombres esIndicePar y esIndiceImpar que tienen la misma lista de parámetros y tipo de retorno que el método abstracto Boolean Function<Integer, Boolean>.apply(Integer t). Esto significa que los métodos toman un argumento Integer (el índice) y devuelven un valor Boolean.

    Respuesta: Vea el fichero EstructuraDatos.java .
