Definiendo Métodos

Aquí tiene un ejemplo de una declaración de método típica: 

public double calcularRespuesta(double envergadura, int numeroDeMotores,
                              double longitud, double toneladasBrutas) {
    //hace los cálculos aquí
}

Los únicos elementos requeridos de una declaración de método son el tipo de retorno del método, el nombre, un par de paréntesis, (), y un cuerpo entre llaves, {}.

Más habitualmente, las declaraciones de métoodo tienen seis componentes, en orden:

  1. Modificadores — tales como public, private, y otros que aprenderá más tarde.
  2. El tipo de retorno — el tipo de dato del valor retornado por el método, o void si el método no devuelve ningún valor.
  3. El nombre del método — las reglas para los nombres de los campos se aplican a los nombres de métodos también, pero la convención es un poco diferente.
  4. La lista de parámetros entre paréntesis — una lista de parámetros de entrada delimitada por comas, precedidos por sus tipos de datos, encerrados entre paréntesis (). Si no hay parámetros, usará los paréntesis vacíos.
  5. Una lista de excepciones, que será descrita más tarde.
  6. El cuerpo del método, encerrado entre llaves — el cṕdigo del método, incluyendo la declaración de variables locales, va aquí.

Los modificadores, los tipos de retorno, y los parámetros serán descritos más tarde en esta lección. Las excepciones serán descritas en un laección posterior.

Definición: Dos de los componentes de una declaración de método comprenden la signatura del método — el nombre del método y los tipos de parámetros.

La signatura del método declarado arriba es: 

calcularRespuesta(double, int, double, double)

Nombrando un Método

Aunque un nombre de método puede ser cualquier identificador legal, las convencioes de código restringen los nombres de método. Por convención, los nombres de método serán un verbo en minúsculas o un nombre multi-palabra que empieza con un verbo en minúsculas, segudio de adjetivos, nombres, etc. En los nombres mult-palabra, la primera letra de la segunda y siguients palabras debe ser capitalizado. Aquí tiene unos ejemplos: 

corre
correRapido
defineFondo
obtenerDatosFinales
compararA
setX
estaVacio

Típicamente, un método tiene un nombre único dentro de su clase. Sin embargo, un método podría tener el mismo nombre que otros métodos debido a la sobrecarga de métodos.

Métodos Sobrecargados

El lenguaje de programación Java soporta sobrecarga de métodos, y Java puede distinguir entre métodos con diferentes signaturas de método. Esto significa que los métodos dentro de una clae pueden tener el mismo nombre si ellos tienen diferentes listas de parámetros (hay algunas requisitos a esto que serán debatidos en la lección titulada "Interfaces y Herencia").

Suponga que tiene una clase que puede usar caligrafía para dibujar varios tipos de datos (cadenas, enteros, y así) y que contiene un método para dibujar cada tipo. Es complicado de usar un nuevo nombre para cada método — por ejemplo, dibujaCadena, dibujaEntero, dibujaDecimal, y así. En la lista de argumentos de cada método, usted puede usar el mismo nombre para todos los métodos de dibujo pero pasando una lista de argumentos diferente a cada método. Así, la clase de dibujo podría declarar cuatro métodos nombrados draw, cada uno de los cuales tiene una lista de parámetros diferente. 

public class ArtistaDeDatos {
    ...
    public void dibujar(String s) {
        ...
    }
    public void dibujar(int i) {
        ...
    }
    public void dibujar(double f) {
        ...
    }
    public void dibujar(int i, double f) {
        ...
    }
}

Los métodos sobrecargados son diferenciados por el numero y el tipo de los argumentos pasados al métoodo. En el ejemplo de código, dibujar(String s) y dibujar(int i) son distintos u únicos métodos porque ellos requieren diferentes tipos de argumentos.

No puede declarar más que un método con el misno nombre y el mismo número y tipo de argumentos, porque el compilador no puede distinguirlos.

El compilador no considera los tipos de retorno cuando diferencia los métodos, así que no puede declarar dos métodos con la misma signatura incluso si tienen un tipo de retorno diferente.

Nota: Los métodos sobrecargados deberían ser usados escasamente, ya que pueden hacer el código mucho menos leíble.
