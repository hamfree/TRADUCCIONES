Usando la Palabra Clave this

Dentro de un método de instancia o un constructor, this es una referencia al objeto actual — el objeto cuyo método o constructor está siendo llamado. Puede referirse a cualquier miembro del objeto actual desde dentro de un método de instancia o un constructor usando this.

Usando this con un Campo

La razón más común para usar la palablra clave this es porque un campo es ocultado por un parámetro de método o constructor.

Por ejemplo, la clase Point fue escrita como esto: 

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}

pero podría haber sido escrita como esto: 

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

Cada argumento al constructor oculta uno de los campos del objeto — dentro del constructor x es una copia local del primer argumento del constructor. Para referirse al campo x de Point, el constructor debe usar this.x.

Usando this con un Constructor

Desde dentro de un constructor, puede también usar la palabra clave this para llamar a otro constructor en la misma clase. Hacer esto se llama una invocación explícita al constructor. Aquí tiene otra clase Rectangle , con una implementación diferente de la que hay en la sección Objetos . 

public class Rectangle {
    private int x, y;
    private int anchura, altura;
        
    public Rectangle() {
        this(0, 0, 1, 1);
    }
    public Rectangle(int anchura, int altura) {
        this(0, 0, anchura, altura);
    }
    public Rectangle(int x, int y, int anchura, int altura) {
        this.x = x;
        this.y = y;
        this.anchura = anchura;
        this.altura = altura;
    }
    ...
}

Esta clase contiene un conjunto de constructores. Cada constructor inicializa alguno o todas las variables miembor del rectángulo. Los constructores establecen un valor por defecto para cualquier variable miembro cuyo valor inicial no fue establecido por un argumento. Por ejemplo, el constructor sin-argumentos crea un Rectangle 1 x 1 en las coordenadas 0,0. El constructor de dos-argumentos llama al constructor de cuatro argumentos, pasándoles la anchura y altura pero siempre usando las coordenadas 0,0 . Como antes, el compilador determina qué constructor llamar, basado en el número y tipo de los argumentos.

Si está presente, la invocación de otro constructor debe ser la primera línea en el constructor.
