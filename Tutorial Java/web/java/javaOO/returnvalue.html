Devolviendo un Valor de un Método

Un método vuelve al código que lo invocó cuando

  • completa todas las sentencias en el método,
  • alcanza una sentencia return, o
  • lanza una excepción (cubierto más tarde),

lo que ocurra primero.

Usted declara un tipo de retorno del método en su declaración de método. Dentro del cuerpo del método, usa la sentencia return para devolver el valor.

Cualquier método declarado void no retorna ningún valor. No necesita contener una sentencia return, pero puede hacerlo. En ese caso, una sentencia return puede ser usada para bifurcar fuera del bloque el control de flujo y salir del método y es simplemente usado como esto: 

return;

Si intenta devolver un valor desde un método que está declarado void, obtendrá un error del compilador.

Cualquier método que no es declarado void debe contener una sentencia return con su correspondiente valor de retorno, como este: 

return valorDeRetorno;

El tipo de dato del valor de retorno debe coincidir con el tipo de retorno declarado en el método; no puede devolver un valor entero de un método que declaró devolver un booleano.

El método getArea() en la clase Rectangle Rectangle que fue descrita en las secciones sobre objetos devuelve un entero: 

    // un método para calcular el área de un rectángulo
    public int getArea() {
        return anchura * altura;
    }

Este método devuelve el entero que la expresión anchura * altura evalúa.

El método getArea devuelve un tipo primitivo. Un método puede también devolver un tipo de referencia. Por ejemplo, en un programa que manipula objetos Bicicleta, tendríamos un método como éste: 

public Bicicleta verCualEsMasRapida(Bicicleta miBici, Bicicleta tuBici,
                              Environment env) {
    Bicicleta laMasRapida;
    // código que calcual qué bici es
    // más rápida, dadas las marchas
    // y cadencias de cada bicicleta, y dado el
    // entorno (tierra y viento)
    return laMasRapida;
}

Devolución de una Clase o Interfaz

Si ésta sección le confunde, sáltesela y vuelva a ella después de que haya terminado la lección sobre interfaces y herencia.

Cuando un método usa un nombre de clase como su tipo de retorno, tal como verCualEsMasRapida hace, la clase del tipo del objeto devuelto debe ser o una subclase de, o la clase exacta de, el tipo de retorno. Suponga que tiene una jerarquía de clases en la cual NumeroImaginario es una subclase de java.lang.Number, la cual a su vez es una subclase de Object, como se ilustró en la siguiente figura.

La jerarquía de clases para NumeroImaginario

Ahora suponga que tiene un método declarado que devuelve un Number: 

public Number devuelveUnNumero() {
    ...
}

El método returnANumber puede devolver un NumeroImaginario pero no un Object. NumeroImaginario es un Number porque es una subclase de Number. Sin embargo, un Object no es necesariamente un Number — podría ser una String u otro tipo.

Usted puede sobreescribir un método y definirlo para que devuelva una subclase del método original, como esto: 

public NumeroImaginario devuelveUnNumero() {
    ...
}

Esta técnica, llamada tipo de retorno covariante, significa que el tipo de retorno está permitido a variar en la misma dirección que su subclase.

Nota: También puede usar nombres de interfaz como tipos de retorno. En este caso, el objeto devuelto debe implementar el interfaz especificado.
