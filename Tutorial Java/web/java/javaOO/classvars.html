Entendiendo los Miembros de una Clase

En esta sección, describiremos el uso de la palabra clave static para crear campos y métodos que pertenecen a la clase, en vez de una instancia de la clase.

Variables de Clase

Cuando un número de objetos son creados de la misma clase plantilla, cada uno de ellos tiene sus propias copias distintas de variables de instancia. En el caso de la clase Bicicleta, las variables de instancia son cadencia, marcha, y velocidad. Cada objeto Bicicleta tiene sus propios valores para estas variables, almacenadas en diferentes lugares de la memoria.

Algunas veces, usted quiere tener variables que sean comunes a todos los objetos. Esto se consigue con el modificador static. Los campos que tienen el modificador static en su declaración son llamados campos estáticos o variables de clase. Están asociados a la clase, en vez de con cualquier objeto. Cada instancia de la clase comparte una variable de clase, la cual está en un sitio fijo de la memoria. Cualquier objeto puede cambiar el valor de la variable clase, pero las variables de clase pueden también ser manipuladas sin crear una instancia de la clase.

Por ejemplo, suponga que quiere crear un número de objetos Bicicleta y asignar a cada uno un número de serie, empezando con el 1 para el primer objeto. Este número ID es único para cada objeto y es así una variable de instancia. Al mismo tiempo, necesita un campo para mantener la pista de cuanto objetos Bicicleta han sido creados de forma que usted sepa que ID asignar al siguiente. Tal campo no está relacionado con ningún objeto individual, sino a la clase como un todo. Para esto necesita una variable de clase, numeroDeBicicletas, como sigue: 

public class Bicicleta {
        
    private int cadencia;
    private int marcha;
    private int velocidad;
        
    // añadir una variable de instancia para el ID del objeto
    private int id;
    
    // añadir una variable de clase para el 
    // número de objetos Bicicleta instanciados
    private static int numeroDeBicicletas = 0;
        ...
}

Las variables de clase son referenciadas por el nombre de la clase en sí mismos, como en 

Bicicleta.numeroDeBicicletas

Esto aclara que son variables de clase.

Nota: Puede también referirse a campos estáticos con una referencia al bojeto como 
miBicicleta.numeroDeBicicletas
pero esto no es aconsejado porque no hace claro ver que sean variables de clase.

Puede usar el constructor Bicicleta pra establecer la variable de instancia id e incrementar la variable de clase numeroDeBicicletas: 

public class Bicicleta {
        
    private int cadencia;
    private int marcha;
    private int velocidad;
    private int id;
    private static int numeroDeBicicletas = 0;
        
    public Bicicleta(int cadenciaInicio, int velocidadInicio, int marchaInicio){
        marcha = marchaInicio;
        cadencia = cadenciaInicio;
        velocidad = velocidadInicio;

        // incrementa el numero de Bicicletas
        // y asigna un número ID
        id = ++numeroDeBicicletas;
    }

    // nuevo método que devuelve la variable de instancia ID
    public int getID() {
        return id;
    }
        ...
}

Métodos de Clase

El lenguaje de programación Java soporta tanto métodos estáticos como también variables estáticas. Los métodos estáticos, los cuales tienen el modificador static en sus declaraciones , serían invocados con el nombre de la clase, sin la neecesidad de crear una instancia de la clase, como en for creating an instance of the class, as in 

NombreClase.nombreMetodo(args)
Nota: Puede también referirse a los métodos estáticos con referencias a objeto como 
nombreInstancia.nombreMetodo(args)
pero esto se desaconseja porque no hace ver claro que son métodos de clase.

Un uso común para los métodos estáticos es acceder a los campos estáticos. Por ejemplo, podríamos añadir un método estático a la clase Bicicleta para acceder al campo estático numeroDeBicicletas: 

public static int getNumeroDeBicicletas() {
    return numeroDeBicicletas;
}

No todas las combinaciones de instancias y variables de clases y métodos son permitidas:

  • Los métodos de instancia puede acceder a variables de instancia y métodos de instancia directamente.
  • Los métodos de instancia pueden acceder variables de clase y métodos de clase directamente.
  • Los métodos de clase pueden acceder a variables de clase y métodos de clase directamente.
  • Los métodos de clase no pueden acceder a variables de instancia o métodos de instancia directamente — Tienen que usar una referencia a objeto. También, los métodos de clase no pueden usar la palabra clave this ya que no hay instancia para this a la que referirse.

Constantes

El modificador static, en combinación con el modificador final, es tambien usado para definir constantes. El modificador final indica que el valor de este campo no puede cambiar.

Por ejemplo, la siguiente declaración de variable define una constante nombrada PI, cuyo valor es una aproximación de pi (el radio de la circunferencia de un círculo a su diámetro): 

static final double PI = 3.141592653589793;

Las constantes definidas de esta forma no pueden ser reasignadas, y es un error en tiempo-de-compilación si su programa intenta hacerlo. Por convención, los nobres de los valores constantes son escritos en letras mayúsculas. Si el nombre está copuesto de más de una palabra, las palabras son separadas por un signo de subrayado (_).

Nota: Si un tipo primitivo o una cadena es definida como una constante y el valor es conocido en tiempo de compliación, el compilador reemplazará el nombre de la constante en cualquier parte en el código con su valor. Esto es llamado una constante de tiempo-de-compilación. Si el valor de la constante en el mundo exterior cambia (por ejemplo, si se legista que pi en realiadad sería 3.975, necesitará recompilar cualquier clase que use esta constante para obtener el valor actual.

La Clase Bicicleta

Después de todas las modificaciones hechas en esta sección la clase Bicicleta es ahora: 

public class Bicicleta {
        
    private int cadencia;
    private int marcha;
    private int velocidad;
        
    private int id;
    
    private static int numeroDeBicicletas = 0;

        
    public Bicicleta(int inicioCadencia,
                   int inicioVelocidad,
                   int inicioGear) {
        marcha = inicioMarcha;
        cadencia = inicioCadencia;
        velocidad = inicioVelocidad;

        id = ++numeroDeBicicletas;
    }

    public int getID() {
        return id;
    }

    public static int numeroDeBicicletas() {
        return numeroDeBicicletas;
    }

    public int getCadencia() {
        return cadencia;
    }
        
    public void setCadencia(int newValue) {
        cadencia = newValue;
    }
        
    public int getMarcha(){
        return marcha;
    }
        
    public void setMarcha(int newValue) {
        marcha = newValue;
    }
        
    public int getVelocidad() {
        return velocidad;
    }
        
    public void frena(int decremento) {
        velocidad -= decremento;
    }
        
    public void acelera(int incremento) {
        velocidad += incremento;
    }
}
