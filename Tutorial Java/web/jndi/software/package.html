Configuración de la aplicación Java

Para usar JNDI en su programa, necesita configurar su entorno de compilación y ejecución.

Importar las Clases de JNDI

A continuación se muestran los paquetes JNDI:

  • javax.naming
  • javax.naming.directory
  • javax.naming.event
  • javax.naming.ldap
  • javax.naming.spi

Los ejemplos en este recorrido usa clases e interfaces de los dos primeros paquetes. Necesita importar estos dos paquetes en su programa o importar las clases e interfaces individuales que use. La dos líneas siguientes importan todas las clases e interfaces de los dos paquetes javax.naming y javax.naming.directory. 

import javax.naming.*;
import javax.naming.directory.*;

Entorno de Compilación

Para compilar un programa que usa JNDI, necesita acceso a las clases JNDI. Java SE 6 también incluye las clases JNDI, de forma que si lo está usando no necesita realizar más acciones.

Entorno de Ejecución

Para ejecutar un programa que usa JNDI, necesita acceder a las clases JNDI y las clases de cualquier proveedor de servicio que el programa usa. El Entorno de Ejecución de Java (EEJ) 6 ya incluye las clases JNDI y los proveedores de servicio para LDPA, nombrado COS, el registro RMI y el DNS.

Si están usando algún otro proveedor de servicio, entonces necesita descargar e instalar sus ficheros comprimidos en el directorio JAVA_HOME/jre/lib/ext, donde JAVA_HOME es el directorio que contiene el EEJ. La página JNDI lista algunos proveedores de servicio. Puede descargar estos proveedores o usar proveedores de otras fabricantes.
