Lección: Configuración del Software

Software Requerido

Lo siguiente es una lista del software/sistemas que necesita:

  • Software de la plataforma Java
  • Software del Proveedor de Servicio
  • Software del servidor de nombres y directorio

Software Plataforma de Java

JNDI se incluye en la Plataforma Java SE.

Para ejecutar los applets, puede usar cualquier navegador web compatible con Java, como Firefox, o Internet Explorer v5 o posterior. Para asegurarse de que sus applets aprovechan al máximo las funciones más recientes del software de la plataforma Java, puede utilizar el complemento Java con su navegador web.

Software del Proveedor de Servicio

La IPA de JNDI es una IPA genérica para acceder a cualquier serivcio de nombres y directorio. El acceso real a un servicio de nombres y directorio se habilita conectando un proveedor de servicio bajo JNDI. Una presentación de la arquitectura JNDI y el rol de los proveedores de servicio se da en la lección Visión General de JNDI.

Un proveedor de servicio es el software que asocia la IPA de JNDI con las llamadas reales al servidor de nombres y directorio. Típicamente, los roles del proveedor de servicio y el servidor de nombres/directorio difieren. En la terminología del sofware cliente/servidor, el JNDI y el proveedor de servicio son el cliente (llamado el cliente JNDI) y el servidor de nombres/directorio es el servidor.

Los clientes y servidores pueden interactuar de muchas formas. De una manera común, utilizan un protocolo de red para que el cliente y el servidor puedan existir de forma autónoma en un entorno de red. El servidor típicamente adminte muchos clientes diferentes, no sólo clientes JNDI, siempre que los clientes se ajusten al protocolo especificado. JNDI no dicta ningún estilo particular de interacción entre los clientes JNDI y los servidores. Por ejemplo, en un extremo, el cliente y el servidor podrían ser la misma entidad.

Necesita obtener las clases para los proveedores de servicio que estará usando. Por ejemplo, si su plan es usar JNDI para acceder a un servidor de directorio LDAP, entonces necesitaría software para un proveedor de servicio LDAP.

El KDJ viene con proveedores de servicio para:

  • Protocolo de Directorio de Peso Ligero (LDAP)
  • CORBA Common Object Services naming (COS naming)
  • Registro RMI
  • Servicio de Nombres de Dominio (DNS)

Si está interesado en otros proveedores por favor compruebe la página de JNDI para descargar información.

Este tutorial usa sólo el proveedor del Servicio LDAP. Cuando use el proveedor de servicio LDAP, debe configurar su propio servidor o tener acceso a un servidor existente, como se explica a continuación.

Software del Servidor de Nombres y Directorio

Una vez ha obtenido el software del proveedor de servicio, necesita configurar o tener acceso a un servidor de nombres/directorio correspondiente. Configurar un servidor de nombres/directorio es típicamente el trabajo de un administrador de sistemas de red. Diferentes fabricantes tienen distintos procedimientos de instalación para sus servidores de nombres/directorio. Algunos requieren privilegios especiales de máquina antes de que se pueda instalar el servidor. Debe consultar las instrucciones de instalación del software del servidor de nombres/directorio.

Para los ejemplos de directorio en este tutorial, necesita acceder a un servidor LDAP. Si desea realizar un recorrido rápido de lo que es LDAP, consúltelo aquí. Puede usar cualquier servidor de su elección que sea compatible con LDAP. El Servidor de Directorio de Oracle, el cuaj se ejecuta en muchas plataformas, incluyendo Windows, está disponible para evaluación en: Servidor de Directorio de Oracle.

Puede también descargar los servidores LDAP gratuitos a continuación:

  • OpenDS
  • OpenLDAP
  • Servidor de Directorio de Fedora
  • Servidor de Directorio de Apache

Un servidor accesible públicamente se encuentra disponible en:

Dirección servidor: ldap://ldap.openldap.org
Contexto de nombrado: dc=OpenLDAP,dc=org
