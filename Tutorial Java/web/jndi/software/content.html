Configuración LDAP

A continuación, se muestran los pasos necesarios para crear una aplicación Java que acceda a un servidor de directorio LDAP.

  1. Instalar el Software de la Plataforma Java.
  2. Obtenga el software del Servidor de Directorio que se describió más temprano.
  3. Configure el Servidor de Directorio con el esquema deseado. Para usar los ejemplos en este tutorial se necesita que un esquema especial sea configurado en el servidor.
  4. Llene el servidor de directorio con el contenido deseado. Para usar los ejemplos en este tutorial es necesario completar un contenido especial en el servidor.
  5. Escribir una aplicación JNDI para acceder al Directorio, compilarla y ejecutarla contra el Servidor de Directorio para obtener sus resultados deseados. Los ejemplos JNDI se cubren en la siguiente lección.

Los primeros dos pasos se cubrieron en la sección anterior. El resto de esta lección describe el paso tres y parte del paso cuatro. El paso cinco que implica escribir una aplicación JNDI se cubre en la siguiente lección que muestra cómo escribir aplicaciones JNDI para realizar varias operaciones en el directorio.

Una vez ha configurado el directorio, o ha dirigido su programa para que se comunique con un directorio existente, ¿qué tipo de información puede esperar encontrar allí?

El directorio puede ser visto como compuesto por vinculaciones de nombre a objeto. Esto es, cada objeto en el directorio tiene un nombre correspondiente. Puede recuperar un objeto en el directorio buscándolo por su nombre.

También se almacenan en el directorio los atributos. Un objeto en el directorio, además de tener un nombre, también tiene un conjunto opcional de atributos. Puede pedir al directorio los atributos de un objeto, como también pedirle la búsqueda por un objeto que tenga ciertos atributos.

Paso 3: Esquema Directorio

Un esquema especifica los tipos de objetos que un directorio puede contener. Este tutorial llena el directorio con entradas, algunas de las cuales requieren definiciones de esquema especiales. Para acomodar estas entradas, debe primero deshabilitar la comprobación-de-esquema en el servidor o agregar los ficheros de esquema que acompañan a este tutorial al servidor. Ambas tareas son típicamente realizadas por el administrador del servidor de directorio.

Este tutorial viene con dos ficheros de esquemas que deben ser instalados:

  • Esquema para objetos Java
  • Esquema para objetos CORBA

El formato de estos ficheros es una descripción formal que posiblemente no puede ser directamente copiada y pegada en los ficheros de configuración del servidor. Específicamente, la sintaxis de los atributos se describen en términos de la RFC 2252.

Diferentes servidores de directorios tienen diferentes formas de configurar sus esquemas. Este tutorial incluye algunas herramientas para instalar los esquemas Java y CORBA en los servidores de directorio que permiten que sus esquemas sean modificados a través de LDAP. Lo siguiente es una lista de las tareas que las herramientas pueden realizar.

  1. Crear Esquema de Java
  2. Crear Esquema de CORBA

Siga las instrucciones del archivo README adjunto para ejecutar estos programas.

Nota: Directorio Activo de Windows. El Directorio Activo gestiona su esquema usando un formato interno. Para actualizar el esquema, puede usar el complemento de la Consola de Administración del Directorio Activo, ADSIEdit, o la utilidad CreateJavaSchema, siguiendo las instrucciones para el Directorio Activo.

Paso 4: Proporcionar Contenido de Directorio para Este Tutorial

En los ejemplos de este recorrido, los resultados mostrados reflejan como el directorio LDAP ha sido configurado usando el fichero de configuración ( tutorial.ldif) que acompaña este tutorial. Si está usando un servidor existente, o un servidor con una configuración diferente, entonce podría ver resultados diferentes. Antes de que pueda cargar el fichero de configuración ( tutorial.ldif) dentro del servidor de directorio, debe seguir las instrucciones para actualizar el esquema del servidor o puede usar los comandos ldapadd o ldapmodify si están disponibles en su sistema UNIX.

Por ejemplo, usando ldapmodify podría hacer (conectando los valores apropiados para el nombre del host, admnistrador DN (opción -D), y la clave de acceso): 

ldapmodify -a -c -v -h hostname -p 389\
        -D "cn=Administrator, cn=users, dc=xxx, dc=xxx"\
        -w passwd -f tutorial.ldif

Nota de Instalación: Control de Acceso. Diferentes servidores de directorio gestionan el control de acceso diferentemente. Algunos ejemplos en este tutorial realiza actualizaciones al directorio. También, la parte del espacio de nombres donde ha instalado el tutorial podría tener restricciones de acceso de lectura. Por lo tanto, debe realizar acciones específica en el servidor para que el directorio sea legible y/o actualizable para que esos ejemplos funcionen. Para el Servidor Directorio de Oracle agregue la entrada aci sugerida en el fichero sunds.aci.ldif a la entrada dn: o=JNDITutorial para hacer el directorio entero legible y actualizable. Alternativamente, puede cambiar los ejemplos de forma que puedan autenticarse al directorio. Los detalles de cómo hacer esto están descritos en la lección Seguridad.

Nota de Instalación: Configuración del Espacio de Nombres. Las entradas en el fichero tutorial.ldif usa el nombre distinguido (DN) "o=JNDITutorial" para el contexto de nombre raíz. Si no ha configurado su servidor de directorio para tener "o=JNDITutorial" como un contexto de nombre raíz, entonces su intento de importar tutorial.ldif fallará. La forma más fácil de evitar este problema es agregrar el DN de un contexto de nombre raíz para cada línea "dn:" en el fichero tutorial.ldif. Por ejemplo, si su servidor ya tiene el contexto de nombre ráiz "dc=imc,dc=org", entonces debe cambiar la línea 

dn: o=JNDITutorial

a 

dn: o=JNDITutorial, dc=imc, dc=org

Haga este cambio por cada línea que empieza con "dn:" en el fichero. Después, en todos los ejemplos en este tutorial, siempre que utilice "o=JNDITutorial", utilice "o=JNDITutorial, dc=imc, dc=org" en su lugar.

Nota de Instalación: Formato de Fichero. Dependiendo de la plataforma del sistema operativo que está usando, podría necesitar editar tutorial.ldif de forma que contenga los caracteres de nueva línea correctos para esa plataforma. Por ejemplo, si encuentra que tutorial.ldif contiene caracteres de nueva línea de estilo Windows (CRLF) y está importando este fichero en un servidor de directorio que se está ejecutando en una plataforma UNIX, entonces necesita editar el fichero y reemplazar los CRLF con LF. Un síntoma de este problema es que el servidor de directorio rechaze todas las entradas en tutorial.ldif.

Nota de Instalación: Directorio Activo de Windows.

  1. El contexto de nombre raíz no a va a ser "o=jnditutorial". Será de la forma "dc=x,dc=y,dc=z". Necesita seguir la nota previa de Configuración del Espacio de Nombres.
  2. Agregue las clases de objeto y atributos relacionados para las clases de objeto "inetOrgPerson" y "groupOfUniqueNames" al esquema de Directorio Activo usando el complemento de la Consola de Gestión del Directorio Activo, ADSIEdit. Se define "groupOfUniqueNames" en el RFC 2256 , "inetOrgPerson" en el RFC 2798.
  3. Alguna de las relaciones jerárquicas usadas por este tutorial no están permitidas de forma predeterminada en el Directorio Activo. Para habilitar estas relaciones, agréguelas uando el complemento de Consola de Gestión del Directorio Activo, ADSIEdit. 
    objectclass: organizationalUnit
possible superiors: domainDNS
                    inetOrgPerson
                    organizaton
                    organizationalPerson
                    organizationalUnit
                    person
                    top

objectclass: groupOfUniqueNames
possible superiors: top

objectclass: inetOrgPerson
possible superiors: container
                    organizationalPerson
                    person
                    top
  4. Borre una de los dos atributos "sn" desde la entrada Mark Twain en tutorial.ldif. El Directorio Activo define "sn" como un atributo de un único valor, al contrario de lo que indica el RFC 2256.
  5. Use la utilidad de línea de comandos ldifde para cargar el fichero modificado tutorial.ldif. 
    # ldifde -i -v -k -f tutorial.ldif
  6. La mayoría de los ejemplos asumen que el directorio ha sido configurado para permitir el acceso para leer y actualizar no autenticado. Su configuración de Directorio Activo podría no permitirle hacer eso. Vea la nota de instalación Control de Acceso.
  7. Leer una entrada algunas veces produce más atributos de los que son mostrados en el tutorial porque el Directorio Activo a menudo devuelve algunos atributos internos.
  8. La creación de entradas podría requerir la especificación de atributos adicionales específicos del Directorio Activo o el uso de otras clases de objeto.
