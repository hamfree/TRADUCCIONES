Gestión de Conexiones

JNDI proporciona una interfaz de alto nivel para acceder a los servicios de nombrado y directorio. La correspondencia entre una instancia JNDI Context y una conexión de red subyacente podría no ser uno-a-uno. El proveedor de servicio es libre de compartir y reusar las conexiones siempre que las semánticas de la interfaz se preserven. El desarrollado de la aplicación usualmente no necesita saber los detalles de cómo las instancias Context crean y usan conexiones. Estos detalles son útiles cuando el desarrollador necesita ajustar su programa.

Esta lección describe cómo el proveedor de servicio LDAP usa las conexiones. Describe cuando se crean las conexiones y cómo especificar parámetros especiales de conexión, tales como servidores múltiples y tiempos de espera de conexión. Esta lección también muestra cómo descubrir y usar dinámicamente servidores LDAP en entornos de red que lo soportan.

Una conexión que se crea debe finalmente cerrarse. Esta lección contiene una sección que describe los cierres de conexión por el cliente y el servidor.

Finalmente, esta lección le muestra cómo usar una agrupación de conexión para hacer que las aplicaciones que usan muchas conexiones de corta duración sean más eficientes.

Nota: La información presentada en esta lección es específica del proveedor de servicio LDAP en el KDJ. Los proveedores de servicio de LDAP de otros fabricantes podrían no usar las mismas políticas para gestionar las conexiones.
