Seguridad

Un servicio LDAP proporciona un servicio de directorio genérico. Puede usarse par almacenar información de todas las clases. Todos los servidores LDAP tienen algún sistema para controlar quién puede leer y actualizar la información en el directorio.

Para acceder al servicio LDAP, el cliente LDAP primero debe autenticarse a sí mismo al servicio. Esto es, debe decirle al servidor LDAP quién va a acceder a los datos de forma que el servidor pueda decidir que se le permite ver y hacer al cliente. Si el cliente se autentica con éxito en el servidor LDAP, después cuando el servidor posteriormente recibe una petición del cliente, comprobará si se le permite al cliente realizar la petición. Este proceso se llama control de acceso.

El estándar LDAP ha propuesto las formas en las cuales los clientes LDAP pueden autenticarse con servidores LDAP ( RFC 2251 y RFC 2829). Estos se discuten en general en la sección de Autenticación LDAP y en la sección de Mecanismos de Autenticación. Esta lección también contiene descripciones de cómo usar los mecanismos de autenticación anónimo, simple y SASL.

El control de acceso se admite de diferentes formas por diferentes implementaciones de servidores LDAP. No se describen en esta lección.

Otro aspecto de seguridad del servicio LDAP es el soporte del uso de canales seguros para comunicarse con los clientes, por ejemplo para enviar y recibir atributos que contienen secretos, tales como contraseñas y claves. Los servidores LDAP usan SSL para este propósito. Esta lección también muestra cómo usar SSL con el proveedor de servicio LDAP.
