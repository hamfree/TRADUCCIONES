Cómo las Operaciones LDAP se Mapean a las IPAs de JNDI

LDAP define un conjunto de operaciones o peticiones (vea RFC 2251). En JNDI, estas se corresponden con operaciones sobre las interfaces DirContext y LdapContext (las cuales son subinterfaces de Context). Por ejemplo, cuando un llamante invoca a un método DirContext, el proveedor de servicio LDAP implementa el método enviando peticiones LDAP al servidor LDAP.

La tabla siguiente muestra cómo se corresponden las operaciones en LDAP con los métodos de JNDI.

Métodos de JNDI que se corresponden con operaciones LDAP
Operación LDAP Métodos JNDI Correspondientes
vincular La forma correspondiente de crear una conexión inicial al servidor LDAP en JNDI es la creación de un InitialDirContext. Cuando la aplicación crea un contexto inicial, proporciona información de autenticación del cliente a través de las propiedades de entorno. Para cambiar esa información de autenticación para un contexto existente, use Context.addToEnvironment() y Context.removeFromEnvironment().
desvincular Context.close() se usa para liberar los recursos utilizados por un contexto. Difiere de la operación "desvincular" de LDAP en que dada la implementación de un proveedor de servicio, los recursos pueden ser compartidos entre contextos, de forma que cerrar un contexto no liberará todos los recursos si esos recursos están siendo compartidos con otro contexto. Asegúrese de cerrar todos los contextos si intenta liberar todos los recursos.
buscar El método correspondiente en JNDI es la sobrecarga de DirContext.search() que acepta un filtro de búsqueda ( RFC 2254). Vea el ejemplo del filtro.
modificar El método correspondiente en JNDI es la sobrecarga de DirContext.modifyAttributes() que acepta una matriz de DirContext.ModificationItems. Vea la sección Modificar Atributos para un ejemplo.
agregar Los métodos correspondientes en JNDI son DirContext.bind() y DirContext.createSubcontext(). Puede usar uno de ellos para agregar una entrada LDAP nueva. Usando bind(), puede especificar no sólo un conjunto de atributos para la nueva entrada sino también un objeto Java que se agregará junto con los atributos. Vea la sección Añadir y Reemplazar Vínculos con Atributos para un ejemplo.
eliminar Los métodos correspondientes en JNDI son Context.unbind() y Context.destroySubcontext(). Puede usar cualquiera de ellos para eliminar una entrada LDAP.
modificar DN/RDN El método correspondiente en JNDI es Context.rename(). Vea la sección Renombrar Objetos para más detalles.
comparar La operación corresponidente en JNDI es una restricción adecuada de DirContext.search(). Vea la sección Comparación LDAP para un ejemplo.
abandonar Cuando cierra un contexto, todas sus solicitudes pendientes se abandonan. De forma similar, cuando cierra una NamingEnumeration, la petición correspondiente de LDAP "buscar" se abandona.
operación extendida El método correspondiente en JNDI es LdapContext.extendedOperation(). Vea el Tutorial JNDI para más detalles.
