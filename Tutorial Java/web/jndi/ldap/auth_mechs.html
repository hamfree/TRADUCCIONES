Mecanismos de Autenticación

Diferentes versiones de LDAP soportan diferentes tipos de autenticación. LDAP v2 define tres tipos de autenticación: anónima, simple (contraseña en texto claro), y Kerberos v4.

LDAP v3 soporta anónima, simple, y autenticación SASL. SASL es Capa de Seguridad y Autenticación Simple ( RFC 2222). Especifica un protoco de desafío-respuesta en el cual los datos se intercambian entre el cliente y el servidor para propósitos de autenticación y el establecimiento de una capa de seguridad sobre la cual realizar la comunicación posterior. Al usar SASL, LDAP puede soportar cualquier tipo de autenticación acordado por el cliente y el servidor LDAP.

Esta lección contiene descripciones de cómo autenticar usando autenticación Anónimo, Simple, y SASL.

Especificar el Mecanismo de Autenticación

El mecanismo de autenticación se especifica usando la propiedad de entorno Context.SECURITY_AUTHENTICATION. La propiedad puede tener uno de los siguientes valores.

Posibles valores de la propiedad de entorno Context.SECURITY_AUTHENTICATION
Nombre de la Propiedad Valor de la Propiedad
sasl_mech Una lista separada por espacios de nombres de mecanismos SASL. Use uno de los mecanismos SASL listados (p.e., "CRAM-MD5" significa que usa el mecanismo SASL CRAM-MD5 descrito en RFC 2195).
none No usa autenticación (anónimo)
simple Usa autenticación débil (contraseña en texto claro)

El Mecanismo Predeterminado

Si el cliente no especifica ninguna propiedad de entorno de autenticación, entonces el mecanismos de autenticación predeterminado es "none". El cliente entonces será tratado como un cliente anónimo.

Si el cliente especifica información de autenticación sin especificar explícitamente la propiedad Context.SECURITY_AUTHENTICATION, entonces el mecanismo de autenticación predeterminado es "simple".
