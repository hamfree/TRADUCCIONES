Resultados de la Búsqueda

Cuando usa los métodos de búsqueda en la interfaz DirContext, obtiene de vuelta una NamingEnumeration. Cada elemento en NamingEnumeration es una SearchResult, la cual contiene la información siguiente:

  • Nombre
  • Objeto
  • Nombre de Clase
  • Atributos

Nombre

Cada SearchResult contiene el nombre de la entrada LDAP que cumplió con el filtro de búsqueda. Obtenga el nombre de la entrada usando getName(). Este método devuelve el nombre compuesto de la entrada LDAP relativa al contexto objetivo. El contexto objetivo es el contexto al cual el parámetro nombre resuelve. En lenguaje LDAP, el contexto objetivo es el objeto base para la búsqueda. Aquí tiene un ejemplo. 

NamingEnumeration answer = ctx.search("ou=NewHires", 
    "(&(mySpecialKey={0}) (cn=*{1}))",  // Filter expression
    new Object[]{key, name},                // Filter arguments
    null);                                  // Default search controls

El contexto objetivo en este ejemplo es ese nombrado por "ou=NewHires". Los nombres en SearchResults en la respuesta es relativa a "ou=NewHires". Por ejemplo, si getName() devuelve "cn=J. Duke", entonces su nombre relativo a ctx será "cn=J. Duke, ou=NewHires".

Si realizó la búsqueda usando SearchControls.SUBTREE_SCOPE o SearchControls.OBJECT_SCOPE y el contexto objetivo en sí mismo cumplió el filtro de búsqueda, entonces el nombre devuelto será "" (el nombre vacío) porque ese el nombre relativo al contexto objetivo.

Esta no es la historia completa. Si la búsqueda implica referencias (vea el Tutorial JNDI) o aliases deferenciados (vea el Tutorial JNDI), entonces los SearchResults correspondientes tendrán nombres que no son relativos al contexto objetivo. En vez de eso, serán URLs que se refieren directamente a la entrada. Para determinar si el nombre devuelto por getName() es relativo o absoluto, use isRelative(). Si este método devuelve true, entonces el nombre es relativo al contexto objetivo; si este devuelve false, el nombre es una URL.

Si el nombre es una URL y necesita usar esa URL, entonces puede pasársela al contexto inicial, que entiende las URLs (vea el Tutorial JNDI).

Si necesita obtener el DN completo de la entrada, puede usar NameClassPair.getNameInNamespace().

Objeto

Si la búsqueda se realizó solicitando que se devuelva el objeto de la entrada (SearchControls.setReturningObjFlag() fue invocado con true), entonces SearchResult contendrá un objeto que representa la entrada. Para recuperar este objeto, invoque getObject(). Si un objeto java.io.Serializable, Referenceable, o Reference fue previamente vinculado a ese nombre LDAP, entonces se usan los atributos de la entrada para reconstruir ese objeto (vea el ejemplo en el Tutorial JNDI). De lo contrario, los atributos de la entrada son usados para crear una instancia DirContext que representa la entrada LDAP. En cualquier caso, el proveedor LDAP invoca a DirectoryManager.getObjectInstance() sobre el objeto y devuelve los resultados.

Nombre de Clase

Si la búsqueda se realizó solicitando que se devolviera el objeto de la entrada, entonces el nombre de la clase se deriva del objeto devuelto. Si la búsqueda solicitó atributos que incluían la recuperación del atributo "javaClassName" de la entrada LDAP, entonces el nombre de la clase es el valor de ese atributo. De lo contrario, el nombre de la clase es "javax.naming.directory.DirContext". El nombre de la clase se obtiene de getClassName().

Atributos

Cuando realiza una búsqueda, puede seleccionar los atributos devueltos suministrando un parámetro a uno de los método search() o estableciendo los controles de búsqueda usando SearchControls.setReturningAttributes() . Si no se han especificado explícitamente atributos, entonces se devuelven todos los atributos de la entrada LDAP. Para especificar que no se devuelvan atributos, debe pasar una matriz vacía (new String[0]).

Para recuperar los atributos de la entrada LDAP, invoque getAttributes() sobre SearchResult.

Controles de Respuesta

Vea la lección Controles y Extensiones del Tutorial JNDI para obtener detalles sobre cómo recuperar los controles de respuesta de un resultado de búsqueda.
