Lección: Temas Avanzados para los Usuarios LDAP

Las lecciones en el recorrido LDAP proporcionan detalles sobre la asignación entre LDAP y JNDI. También brindan sugerencias y consejos para acceder al servicio LDAP a través del JNDI.

LDAP

X.500, es un estándar del CCITT para servicios de directorios que es parte de la suite de servicios OSI. El estándar X.500 define un protocolo (entre otros) para que una aplicación cliente acceda a un directorio X.500 llamado el Protoco de Acceso a Directorio (DAP). Está superpuesto a la pila de protocolos de interconexión de sistemas abiertos (OSI).

La comunidad de Internet, reconociendo la necesidad de un servicio parecido a X.500 pero enfrentada con una infraestructura de red subyacente diferente (TCP/IP en vez de OSI), diseñó un nuevo protocolo basado en el protocolo DAP de X.500, llamado DAP The Internet community, recognizing the need for an X.500-like service but faced with different underlying network infrastructure (TCP/IP instead of OSI), designed a new protocol based on the X.500 DAP protocol, called Ligero DAP, o LDAP. RFC 2251 define lo que ahora se llama version 3 de LDAP (o LDAP v3), una versión mejorada de su predecesor LDAP v2 especificado en RFC 1777.

El objetivo de LDAP era un protocolo que pudiera ser fácilmente implementado, con un foco especial en ser capaz de construir clientes pequeños y simples. Una forma en que intentó lograr la simplificación fue utilizar muchas cadenas y minimizar, siempre que fuera posible, el uso de estructuras. Los DN, por ejemplo, se representan en el protocolo como cadenas, al igual que los nombres de tipo de atributo y muchos valores de atributo.

El protocolo consiste del cliente enviando peticiones al servidor, al cual el servidor responde, aunque no necesariamente en el mismo orden en el que las peticiones fueron enviadas. Cada petición es etiquetada con un ID de forma que las peticiones y respuestas puedan coincidir. El protocolo trabaja sobre TCP o UDP, aunque la versión TCP es la más comunmente usada.

Debido al foco en los clientes, la comunidad LDAP también definió estándares para la representación de cadenas de los DNs (RFC 2553), filtros de búsqueda (RFC 1960), y sintaxis de atributos (RFC 1778), para una IPA basada en el lenguaje C (RFC 1823), y para el formato de las URLs para acceder a los servicios LDAP (RFC 1959).

LDAP v3 soporta la internacionalización, varios mecanismos de autenticación, referencias, y un mecanismo genérico de despliegue. Permite agregar nuevas funciones al protocolo sin requerir también cambios en el protocolo mediante el uso de extensiones y controles.
