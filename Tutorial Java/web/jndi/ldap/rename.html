Más Operaciones LDAP

El resto de la lección LDAP cubre cómo JNDI proporciona la habilidad de realizar ciertas operaciones interesantes de LDAP.

Renombrar Objetos

Puede usar Context.rename() para renombrar un objeto en el directorio. En LDAP v2, esto corresponde a la operación "modify RDN" que renombra una entrada dentro del mismo contexto (esto es, renombra un hermano). En LDAP v3, esto corresponde a la operación "modify DN" , que es como "modify RDN," excepto que las entradas viejas y nuevas no necesitan estar en el mismo contexto. Puede usar Context.rename() para renombrar una entrada de hoja o un nodo interior. El ejemplo mostrado en la lección Operaciones de Nombrado y Directorio renombra una entrada de hoja. El siguiente código renombra un nodo interior de "ou=NewHires" a "ou=OldHires": 

ctx.rename("ou=NewHires", "ou=OldHires");
Nota: El Servidor de Directorio de Oracle v5.2 no soporta el renombrado de nodos interiores. Si ejecuta este ejemplo, obtendrá una ContextNotEmptyException.

Renombrar a una Parte Diferente del DIT

Con LDAP v3, puede renombrar una entrada a una parte diferente del DIT. Para hacer esto usando Context.rename(), debe usar un contexto que sea el ancestro común de las nuevas y viejas entradas. Por ejemplo, para renombrar "cn=C. User, ou=NewHires, o=JNDITutorial" a "cn=C. User, ou=People, o=JNDITutorial", debe usar el contexto nombrado por "o=JNDITutorial". Lo siguiente es un ejemplo que demuestra esto. Si intenta ejecutar este ejemplo contra un servidor LDAP v2, obtendrá una InvalidNameException porque la versión 2 no soporta esta característica. 

ctx.rename("cn=C. User, ou=NewHires", "cn=C. User, ou=People");
Nota: El Servidor de Directorio de Oracle v5.2 no soporta el renombrado con nodos padre diferentes. Si ejecuta este ejemplo usando este servidor, obtendrá una OperationNotSupportedException (indicando un "error de protocolo").

Mantener los Nombres de Atributos Viejos

En LDAP, cuando renombra una entrada, tiene la opción de mantener las viejas entradas RDN como un atributo de la entrada actualizada. Por ejemplo, si renombra la entrada "cn=C. User" a "cn=Claude User", puede especificar si quiere que se mantenga el viejo RDN "cn=C. User" como un atributo.

Para especificar si quiere mantener el viejo nombre del atributo cuando usa Context.rename(), use la propiedad de entorno "java.naming.ldap.deleteRDN". Si el valor de esta propiedad es "true" (valor predeterminado), el viejo RDN se elimina. Si su valor es "false", entonces el viejo RDN se mantiene como un atributo de la entradda actualizada. El ejemplo completo está aquí. 

// Set the property to keep RDN
env.put("java.naming.ldap.deleteRDN", "false");

// Create the initial context
DirContext ctx = new InitialDirContext(env);

// Perform the rename
ctx.rename("cn=C. User, ou=NewHires", "cn=Claude User,ou=NewHires");
