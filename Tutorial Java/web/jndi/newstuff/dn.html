Recuperando el Nombre Distinguido

En las versiones del KDJ anteriores a la 5.0, no hay forma directa de obtener el Nombre Distinguido (DN) de los resultados de la búsqueda. El método SearchResults.getName() siempre devuelve el nombre que es relativo al contexto en el cual se realizó la consulta. Para obtener el absoluto, o nombre completo de la entrada de búsqueda se requiere cierta cantidad de contabilidad para rastrear los contextos ancestrales. Se agregan dos nuevas IPAS a continuación en el KDJ 5.0 para recuperar el nombre absoluto del NameClassPair, siempre que se realize una búsqueda, listado, o listado de vínculos en un contexto:

  • NameClassPair.getNameInNameSpace(Name name)
  • NameClassPair.getNameInNameSpace(String name)

Aquí tiene un ejemplo que recupera DNs desde una búsqueda LDAP: 

     public static void printSearchEnumeration(NamingEnumeration retEnum) {
         try {
             while (retEnum.hasMore()) {
                 SearchResult sr = (SearchResult) retEnum.next();
                 System.out.println(">>" + sr.getNameInNamespace());
             }
         } catch (NamingException e) {
             e.printStackTrace();
         }
     }
