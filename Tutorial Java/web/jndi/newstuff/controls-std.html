Controles estándar de LDAP

En LDAP v3, un Control es un mensaje que mejora la operación existente de LDAP asociándola con más información útil para el servidor o el cliente. Un control puede ser un control de petición o un control de respuesta. Un control de petición se envía del cliente al servidor junto con una petición LDAP. Un control de respuesta se envía del servidor al cliente junto con una respuesta LDAP. Cualquiera de ellos está representado por la interfaz javax.naming.ldap.Control.

Vea la lección de controles en el Tutorial JNDI si no ha programado usando controles antes.

En esta lección describiremos los controles estándar de LDAP que fueron agregados al KDJ 5.0. Los controles LDAP necesarios ya están soportados en el paquete de extensión LDAP Booster Pack disponible para el proveedor de servicios JNDI/LDAP en el paquete com.sun.jndi.ldap.ctl. Los controles LDAP que están estandarizados por IETF están ahora disponibles en el paquete javax.naming.ldap del KDJ a través de las siguientes clases.

  • ManageReferralControl ( RFC 3296)
  • PagedResultsControl ( RFC 2696)
  • PagedResultsResponseControl
  • SortControl ( RFC 2891)
  • SortKey
  • SortResponseControl
  • BasicControl
