Manipulando LdapName (Nombre Distinguido)

El Nombre Distinguido (DN) se usa por LDAP en su representación de cadena. El formato de cadena usado para representar un DN se especifica en la especificación RFC 2253. El DN se compone de componentes denominados Nombres Distinguidos Relativos (RDN). A continuación se muestra un ejemplo de un DN:

"CN=John Smith, O=Isode Limited, C=GB"

Consiste en los siguientes RDNs:

  • CN=John Smith
  • O=Isode Limited
  • C=GB

Las clases de abajo representan el DN y el RDN respectivamente.

  • javax.naming.ldap.LdapName
  • javax.naming.ldap.Rdn

La clase LdapName implementa la interfaz javax.naming.Name similar a las clases javax.naming.CompoundName y javax.naming.CompositeName.

Las clases LdapName y Rdn permiten la manipulación fácil de los DNs y RDNs. Usando estas IPAs es fácil construir un RDN emparejando nombres y valores. Un DN puede ser construído con una lsita de RDNs. De manera similar, los componentes individuales de DN y RDN se pueden recuperar de su representación en cadena.

LdapName

Un LdapName se puede crear con su representación en cadena como se definió en la especificación RFC 2253 o con una lista de RDNs. Cuando se usa la primera forma, la cadena especificada es analizada según las reglas definidas en RFC 2253. Se lanza una InvalidNameException si la cadena no es una DN válida. Aquí tiene un ejemplo que usa el constructor para analizar un nombre LDAP e imprime sus componentes. 

String name = "cn=Mango,ou=Fruits,o=Food";
try {
    LdapName dn = new LdapName(name);
    System.out.println(dn + " has " + dn.size() + " RDNs: ");
    for (int i = 0; i < dn.size(); i++) {
        System.out.println(dn.get(i));
    }
} catch (InvalidNameException e) {
    System.out.println("Cannot parse name: " + name);
}

Ejecutar este ejemplo con una entrada de "cn=Mango,ou=Fruits,o=Food" produce los resultados siguientes: 

cn=Mango,ou=Fruits,o=Food has 3 RDNs: 
o=Food
ou=Fruits
cn=Mango

La clase LdapName contiene métodos para acceder a sus componentes como RDNs y cadenas, para modificar un LdapName, para comparar dos LdapNames por igualdad, y para obtener una representación en cadena del nombre.

Acceder a los componentes de nombre de un nombre LDAP:

Aquí están los métodos que puede usar para acceder a los componentes de nombre como RDNs y cadenas:

getRdn(int posn)
get(int posn)
getRdns()
getAll()
getPrefix(int posn)
getSuffix(int posn)
clone()

Para recuperar el componente en una posición particular dentro de un LdapName, use getRdn() o get(). El ejemplo previo del constructor muesra un ejemplo de su uso. getRdns() devuelva una lista de todos los RDNs y getAll() devuelve todos los componentes de un LdapName como una enumeración.

EL RDN más a la derecha está en el índice 0, y el RDN más a la izquierda está en el índice n-1. Por ejemplo, el nombre distinguido: "cn=Mango, ou=Fruits, o=Food" se numera en la siguiente secuencia desde 0 a 2: {o=Food, ou=Fruits, cn=Mango}

Puede obtener un sufijo o prefijo de LdapNames como una instancia LdapName. Aquí tiene un ejemplo que obtiene el sufijo y prefijo de un nombre LDAP. 

LdapName dn = new LdapName("cn=Mango, ou=Fruits, o=Food");
Name suffix = dn.getSuffix(1);  // 1 <= index < cn.size()
Name prefix = dn.getPrefix(1);  // 0 <= index < 1

Cuando ejecuta este programa, genera la salida siguiente: 

cn=Mango ou=Fruits
o=Food

Para hacer una copia de un LdapName, use clone().

Modificar un nombre LDAP

A continuación se muestran los métodos que puede utilizar para modificar un nombre LDAP:

add(Rdn rdn)
add(String comp)
add(int posn, String comp)
addAll(List suffixRdns)
addAll(Name suffix)
addAll(int posn, List suffixRdns)
addAll(int posn, Name suffix)
remove(int posn)

Después de crear una instancia LdapName, puede agregar y eliminar componentes de él. Aquí tiene un ejemplo que agrega un LdapName a un LdapName existente, agrega componentes al inicio y al final, y elimna el segundo componente. 

     LdapName dn = new LdapName("ou=Fruits,o=Food");
     LdapName dn2 = new LdapName("ou=Summer");
     System.out.println(dn.addAll(dn2)); // ou=Summer,ou=Fruits,o=Food
     System.out.println(dn.add(0, "o=Resources")); 
// ou=Summer,ou=Fruits,o=Food,o=Resources
     System.out.println(dn.add("cn=WaterMelon")); 
// cn=WaterMelon,ou=Summer,ou=Fruits,o=Food,o=Resources
     System.out.println(dn.remove(1));  // o=Food
     System.out.println(dn);  
// cn=WaterMelon,ou=Summer,ou=Fruits,o=Resources

Comparar un nombre LDAP

A continuación se muestran los métodos que puede utilizar para comparar dos nombres LDAP:

compareTo(Object name)
equals(Object name)
endsWith(List)
endWith(Name name)
startsWith(List rdns)
startsWith(Name name)
isEmpty()

Puede usar compareTo() para ordenar una lista de instancias LdapName. El método equals() le permite determina si dos LdapNames son sintácticamente iguales. Dos LdapNames son iguales si ambos tienen los mismos componentes (coincidencia exacta de mayúsculas y minúsculas) en el mismo orden.

Con startsWith() y endsWith(), puede aprender si un LdapName inicia o termina con otro LdapName; esto es, si un LdapName es un sufijo o prefijo de otro LdapName.

El método de conveniencia isEmpty() le habilita a determinar si un LdapName tiene cero componentes. Puede también usar la expresión size() == 0 para realizar la misma comprobación.

Aquí tiene un ejemplo, CompareLdapNames, que usa alguna de estos métodos de comparación. 

LdapName one = new LdapName("cn=Vincent Ryan, ou=People, o=JNDITutorial");
LdapName two = new LdapName("cn=Vincent Ryan");
LdapName three = new LdapName("o=JNDITutorial");
LdapName four = new LdapName("");

System.out.println(one.equals(two));        // false
System.out.println(one.startsWith(three));  // true
System.out.println(one.endsWith(two));      // true
System.out.println(one.startsWith(four));   // true
System.out.println(one.endsWith(four));     // true
System.out.println(one.endsWith(three));    // false
System.out.println(one.isEmpty());          // false
System.out.println(four.isEmpty());         // true
System.out.println(four.size() == 0);       // true

Obtener la Representación de Cadena

El método de abajo obtiene la cadena de representación de un nombe LDAP formateado de acuerdo a la sintaxis especificada en la especificación RFC 2253:

toString()

Cuando usa el constructor LdapName, proporciona la representación de cadena de un nombre LDAP y obtiene de vuelta una instancia LdapName. Para hacer lo nverso, esto es, para obtener la representación de cadena de una instancia LdapName, use toString(). El resultado de toString() se puede retroalimentar al constructor para producir una instancia de LdapName que sea igual a la instancia de LdapName original. Aquí tiene un ejemplo, LdapNametoString: 

LdapName dn = new LdapName(name);
String str = dn.toString();
System.out.println(str);
LdapName dn2 = new LdapName(str);
System.out.println(dn.equals(dn2));  // true

LdapName como un Argumento a Métodos de Contexto

La clase Context requiere que se pase un nombre compuesto o compuesto como argumento a sus métodos. Por lo tanto, un LdapName se puede pasar directamente a un método de contexto como se muestra en LookupLdapName: 

// Create the initial context
Context ctx = new InitialContext(env);

// An LDAP name
LdapName dn = new LdapName("ou=People,o=JNDITutorial");

// Perform the lookup using the dn
Object obj = ctx.lookup(dn);

De forma similar, cuando los métodos de Context devuelven resultados desde las operaciones list(), listBindings(), o search(), el DN puede ser recuperado llamando a getNameInNamespace(). El LdapName puede ser construído directamente desde el DN como se muestra en el ejemplo, RetrievingLdapName: 

while (answer.hasMore()) {
    SearchResult sr = (SearchResult) answer.next();
    String name = sr.getNameInNamespace();
    System.out.println(name);
    LdapName dn = new LdapName(name);
