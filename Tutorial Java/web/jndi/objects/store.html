Almacenando y Leyendo Objetos

Las aplicaciones y servicios pueden usar el directorio en formas diferentes para almacenar y localizar objetos:

  • Almacenar (una copia) del objeto en sí.
  • Almacenar una referencia a un objeto.
  • Almacenar atributos que describen el objeto.

En términos generales, una forma serializada de un objeto de Java contiene el estado de objeto y una referencia del objeto en una representación compacta de información de direccionamiento que se puede utilizar para contactar con el objeto. Algunos ejemplos se dan en la lección Buscar un Objeto. Los atributos de un objeto son propiedades que se usan para describir el objeto; los atributos podrían incluir el direccionamiento y/o información del estado.

Cuál de estas tres formas de usar depende de la aplicación / sistema que se está construyendo y cómo necesita interoperar con otras aplicaciones y sistemas que compartirán los objetos almacenados en el directorio. Otro factor es el soporte proporcionado por el proveedor de servicio y el servicio de directorio subyacente.

Programáticamente, todas las aplicaciones utilizan uno de los siguientes métodos al almacenar objetos en el directorio:

  • Context.bind()
  • DirContext.bind()
  • Context.rebind()
  • DirContext.rebind()

La aplicación pasa el objeto que quiere almacenar a uno de estos métodos. Luego, dependiendo de los tipos de objetos que el proveedor de servicios admite, el objeto será transformado en una representación aceptable al servicio de directorio subyacente.

Esta lección muestra cómo almacenar los objetos serializables en el directorio una vez el objeto se almacena, puede simplemente usar lookup() para obtener una copia del objeto de vuelta desde el directorio, independientemente de qué tipo de información fue en realidad almacenada.

Puede obtener el objeto de vuelta no sólo usando lookup(), pero también cuando usted lista un contexto y cuando busca un contexto o su subárbol. En todos estos casos, pueden estar involucradas las factorías de objetos. Las factorías de objetos se describen en detalle en el Tutorial JNDI.

Para almacenar los siguientes tipos de objetos, consulte el Tutorial de JNDI:

  • Objetos referenciables y Referencias JNDI
    Los ejemplos bind() en la lección Añadir, reemplazar o eliminar un vínculo (Binding) usan objetos Referenciables.
  • Objetos con atributos (DirContext)
  • Objetos RMI (Invocación de Metodos Remotos de Java) (incluyendo esos que usan IIOP)
  • Objetos CORBA

Antes de que continúe: Para ejecutar estos ejemplos con éxito, debe deshabilitar la comprobación de esquemas en el servidor o agregar el esquema Java que acompaña este tutorial en el servidor. Esta tarea es típicamente realizada por el administrador del servidor de directorio. Vea la lección Configuración de Software para más información.

Directorio Activo de Windows: Context.rebind() y DirContext.rebind() no funcionan contra el Directorio Activo porque estos métodos funcionan leyendo los atributos de la entrada a ser actualizados, eliminando la entrada, y después agregar una entrada nueva que contiene los atributos modificados. El Directorio Activo dvuelve algunos atributos que no pueden ser establecidos por el usuario, causando que el paso de agregación final falle. La solución para este problema es utilizar DirContext.getAttributes() para obtener y salvar los atributos que quiere mantener. Después, elimine la entrada y agréguela de vuelta con los atributos salvados (y otros cualquier que quiera agregar) usando DirContext.bind().
