Objetos Serializables

Serializar un objeto significa convertir su estado a un flujo de bytes de forma que el flujo de bytes puede ser revertido de vuelta en una copia del objeto. Un objeto de Java es serializable si su clase o cualquiera de sus superclases implementa la interfaz java.io.Serializable o su subinterfaz, java.io.Externalizable. La Deserialización es el proceso de convertir la forma serializada de un objeto de vuelta a una copia del objeto.

Por ejemplo, la clase java.awt.Button implementa la interfaz Serializable, de forma que puede serializar un objeto java.awt.Button y almacenar ese estado serializado en un fichero. Posteriormente, puede leer de vuelta el estado serializado y deserializarlo en un objeto java.awt.Button.

La plataforma Java especifica un forma predeterminada por la cual los objetos serializables son serializados. Una clase (Java) puede sobreescribir esta serialización predeterminada y definir su propia forma de serializar objetos de esa clase. La Especificación de Serialización de Objetos describe la serialización de objetos en detalle.

Cuando un objeto se serializa, la información que identifica su clase se graba en el flujo serializado. Sin embargo, la definición de clase ("el fichero class") en sí misma no se graba. Es responsabilidad del sistema que está deserializando el objeto determinar cómo ubicar y cargar los archivos de clase necesarios. Por ejemplo, una aplicación Java podría incluir en su ruta de clases un fichero JAR que contiene los ficheros de clases de los objetos serializados o cargar las definiciones de clase usando información almacenada en el directorio, como se explicó más tarde en esta lección.

Vincular un Objeto Serializable

Puede almacenar un objeto serializable en el directorio si el proveedor de servicio subyacente soporta esa acción, como hace el proveedor de servicio LDAP de Oracle.

El ejemplo siguiente invoca Context.bind para vincular un botón AWT al nombre "cn=Button". Para asociar los atributos con el nuevo vínculo, puede usar DirContext.bind . Para sobreescribir un vínculo existente, use Context.rebind y DirContext.rebind. 

// Create the object to be bound
Button b = new Button("Push me");

// Perform the bind
ctx.bind("cn=Button", b);

Puede entonces leer el objeto de vuelta usando Context.lookup, como sigue. 

// Check that it is bound
Button b2 = (Button)ctx.lookup("cn=Button");
System.out.println(b2);

Ejecutar este ejemplo produce la siguiente salida. 

# java SerObj
java.awt.Button[button0,0,0,0x0,invalid,label=Push me]

Especificar una Base de Código

Nota: Los procedimientos descritos aquí son para vincular un objeto serializable en un servicio de directorio que sigue el esquema definido en RFC 2713. Estos procedimientos podrían no ser generalmente aplicables a otros servicios de directorio y nombrado que soporta vincular un objeto serializable con una Base de Código especificada.

Cuando un objeto serializado se vincula en el directorio como se muestra en el ejemplo previo, las aplicaciones que leen el objeto serializado desde el directorio deben tener acceso a las definiciones de clase necesarios para deserializar el objeto.

Alternativamente, puede grabar una base de código con el objeto serializado en el directorio, o cuando usted vincula el objeto o posteriormente agregando un atributo usando DirContext.modifyAttributes . Puede usar cualquier atributo para registrar esta base de código y hacer que su aplicación lea ese atributo del directorio y lo use de manera apropiada. O puede usar el atributo "javaCodebase" especificado dentro. En el último caso, el proveedor de servicio de LDAP de Oracle automáticamente usará el atributo para cargar las definiciones de clases según sea necesario. "javaCodebase" debe contener la URL de un directorio base de código de un fichero JAR. Si la base de código contiene más de una URL, entonces cada URL debe estar separada por un carácter espacio.

El ejemplo siguiente se parece al de vincular un java.awt.Button. Se diferencia en que utiliza una clase Serializable definida por el usuario, Flower , y proporciona un atributo "javaCodebase" que contiene la ubicación de la definición de clase Flower. Aquí está el código que hace la vinculación. 

String codebase = ...;

// Create the object to be bound
Flower f = new Flower("rose", "pink");

// Perform the bind and specify the codebase
ctx.bind("cn=Flower", f, new BasicAttributes("javaCodebase", codebase));

Cuando ejecuta este ejemplo, debe proporcionar la URL de la ubicación en la cual se instaló el fichero de clase Flower.class. Por ejemplo, si Flower.class fue instalada en el servidor web web1, en el directorio example/classes, entonces ejecutaría este ejemplo de la siguiente manera. 

# java SerObjWithCodebase http://web1/example/classes/
pink rose

Después, puede eliminar Flower.class desde su ruta de clase y ejecutar cualquier programa que busca o lista este objeto sin directamente referenciar la clase Flower. Si su programa referencia Flower directamente, luego debe hacer que su archivo de clase esté disponible para su compilación y ejecución.
