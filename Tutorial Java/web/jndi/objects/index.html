Lección: Objetos de Java en el Directorio

Tradicionalmente, los directorios han sido usados para almacenar datos. Los usuarios y programadores piensan del directorio como una jerarquía de entradas de directorio, cada una conteniendo un conjunto de atributos. Usted busca una entrada en el directorio y extrae los atributos de interés.

Para las aplicaciones escritas en el lenguaje de programación Java, los objetos Java pueden algunas veces ser compartidos entre aplicaciones. Para tales aplicaciones, tiene sentido ser capaz de usar el directorio como un repositorio para objetos Java. El directorio proporciona un servicio administrado centralmente, y posiblemente replicado, para usar por aplicaciones Java distribuidas a través de la red. Por ejemplo, un servidor de aplicación podría usar el directorio para registrar objetos que representan los servicios que gestiona de forma que un cliente pueda más tarde a buscar el directorio para localizar esos servicios según sea necesario. Un ejemplo de JNDI usado como directorio de servicios es Apache DS. Más información sobre esto puede encontrarse en Directorio Apache.

JNDI proporciona una vista orientada a objetos del directorio, por lo tanto permite que los objetos de Java sean agregados y recuperados desde el directorio sin requerir que el cliente gestione problemas de representación de datos. Esta lección describe el uso del directorio para almacenar y recuperar los objetos de Java en un nivel básico. JNDI proporicona lo que se conoce como factorías de objeto y estado para crear y almacenar los objetos accedidos desde el directorio.

Factoría de Objetos

Una factoría de objetos es un productor de objetos. Acepta alguna información sobre cómo crear un objeto, como una referencia, y entonces devuelve una instancia de ese objeto. Para obtener detalles sobre las Factoría de Objetos y el formato en el que los objetos se almacenan en el directorio por favor refíérase al Tutorial JNDI.

Factoría de Estado

Una factoría de estado transforma un objeto en otro objeto. La entrada es el objeto y atributos opcionales, proporcionados a Context.bind() y la salida es otro objeto y atributos opcionales, para ser almacenados en el directorio o servicio de nombres subyacentes. Para obtener detalles sobre las Factorías de Estado y cómo escribir su propia factoría de estado refiérase al Tutorial JNDI.

La siguiente parte de la lección describe cómo accecder a Objetos en el Directorio. Describe cómo los objetos serializables pueden ser almacenados y leídos en el directorio. Para otros tipos de objetos por favor compruebe el Tutorial JNDI.
