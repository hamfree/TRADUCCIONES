Paquetes Directory y LDAP

Paquete Directorio

El paquete javax.naming.directory extiende el paquete javax.naming para proporcionar funcionalidad para acceder a los servicios de directorio además de los servicios de nombres. Este paquete permite que las aplicaciones recuperen los objetos asociados con los almacenados en el directorio y busquen objetos usando atributos específicos.

El Contexto de Directorio

La interfaz DirContext representa un contexto de directorio. DirContext también se comporta como un contexto de nombres extendiendo la interfaz Context. Esto significa que cualquier objeto directorio puede también proporcionar un contexto de nombres. Define métodos para examinar y actualizar atributos asociados con una entrada de directorio.

Atributos
Use el método getAttributes() para recuperar los atributos asociados con una entrada de directorio (para la cual suple el nombre). Los atributos se modifican usando el método modifyAttributes(). Puede agregar, reemplazar, o eliminar atributos y/o valores de atributos usando esta operación.
Búsquedas
DirContext contiene métodos para realizar búsquedas basadas en el contenido del directorio. En la forma de uso más simple y común, la aplicación especifica un conjunto de atributos posiblemente con valores específicos para coincidir y envía este conjunto de atributos al método search(). Otras formas sobreeescritas de search() admiten filtros de búsqueda más sofisticados.

Paquete LDAP

El paquete javax.naming.ldap contiene clases e interfaces para usar características que son específicas de LDAP v3 que no están ya cubiertas por el paquete javax.naming.directory más genérico. En realidad, la mayoría de las aplicaciones JNDI que usan LDAP encontrarán el paquete javax.naming.directory suficiente y no necesitarán usar el paquete javax.naming.ldap en absoluto. Este paquete es primariamente para esas aplicaciones que necesitan usar operaciones "extendidas", controles, o notificaciones no solicitadas.

Operación "Extendida"
Además de especificar operaciones bien definidas como buscar y modificar, LDAP v3 (RFC 2251) especifica una forma de transmitir operaciones aún por definir entre el cliente LDAP y el servidor. Estas operaciones se llaman operaciones "extendidas". Una operación "extendida" puede ser definida por unos estándares de organización tales como el Grupo de Trabajo de Ingeniería de Internet (Nota del Traductor: IETF o Internet Engineering Task Force, en inglés) o por un fabricante.
Controles
El protocolo LDAP v3 permite que cualquier petición o respuesta sea aumentada por modificadores aún por definir, llamados controles. Un control enviado con una petición es una petición de control y un control enviado con una respuesta es una respuesta de control. Un control puede ser definido por unos estándares de organización tales como el IETF o por un vendedor. Las peticiones de controles y las respuestas de contrles no están necesariamente emparejadas, esto es, no es necesario que haya un control de respuesta para cada control de solicitud enviado, y viceversa.
Notificaciones No Solicitadas
Además del estilo normal de petición/respuesta de interacción entre el cliente y el servidor, el protocolo LDAP v3 también especifica notificaciones no solicitadas — mensajes que son enviados desde el servidor al cliente asincrónicamente y no en respuesta a una petición del cliente.

El Contexto LDAP

La interfaz LdapContext representa un contexto para realizar operaciones "extendidas", enviar peticiones de contrles, y recibir respuestas de controles. Ejemplos de cómo usar estas características se describen en la lección del Tutorial de JNDI Controles y Extensiones.
