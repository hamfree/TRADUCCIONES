Paquete de Nombrado

El paquete javax.naming contiene clases e interfaces para acceder a los servicios de nombres.

Contexto

El paquete javax.naming define una interfaz de Context, que es la interfaz principal para buscar, vincular/desvincular, renombrar objetos y crear y destruir subcontextos.

Lookup
La operación más habitualmente usada es lookup(). Proporciona a lookup() el nombre del objeto que desea buscar y devuelve el objeto vinculado a ese nombre.
Bindings
listBindings() devuelve una enumeración de vínculos nombre-a-objeto. Un vínculo es una tupla que contiene el nombre del objeto vinculado, el nombre de la clase del objeto, y el objeto en sí.
List
list() es similar a listBindings(), excepto que devuelve una enumeración de nombres conteniendo un nombre de objeto y el nombre de la clase del objeto. list() es útil para aplicaciones como los navegadoers que quieren descubrir información sobre los objetos vinculados dentro de un contexto pero que no necesitan todos los objetos reales. Aunque listBindings() proporciona toda la misma información, es potencialmente una operación mucho más costosa.
Name
Name es una interfaz que representa un nombre genérico — un secuencia ordenad de cero o más componentes. Los Sistemas de Nombre usan esta interfaz para definir los nombres que siguen sus convenciones como se describió en la lección Conceptos de Nombrado y Directorio.
References
Los objetos se almacenan en los servicios de nombres y directorio de formas diferentes. Una referencia podría ser una representación muy compacta de un objeto.

JNDI define una clase Reference para representar una referencia. Una referencia contiene información de cómo construir una copia del objeto. JNDI intentará convertir las referencias buscadas desde el directorio en los objetos Java que representan de forma que los clientes JNDI tengan la ilusión de que lo que está almacenado en el directorio son objetos Java.

El Contexto Inicial

EN JNDI, todas las operaciones de nombres y directorio se realizan en relación con un contexto. No hay raíces absolutas. Por lo tanto JNDI define un InitialContext, el cual proporciona un punto de partida para las operaciones de nombres y directorios. Una vez que tiene un contexto inicial, puede usarlo para buscar otros contextos y objetos.

Excepciones

JNDI define una jerarquía de clases para las excepciones que se pueden lanzar en el curso de la realización de operaciones de nombres y directorio. La raíz de esta jerarquía de clases es NamingException. Los programas interesados en tratar con una excepción particular pueden capturar la subclase correspondiente de la excepción. De lo contrario, deberían capturar NamingException.
