Paquetes de Eventos y Proveedor de Servicios

Paquete Evento

El paquete javax.naming.event contiene clases e interfaces que admiten notificaciones de eventos en los servicios de nombres y directorios. La notificación de eventos se describe en detalle en el recorrido Notificación de Eventos.

Eventos
Un NamingEvent representa un evento que se genera por un servicio de nombres/directorio. El evento contiene un tipo que identifica el tipo de evento. Por ejemplo, los tipos de evento están categorizados en aquellos que afectan el espacio de nombres, tales como "objetos agregados" y aquellos que no, tales como "objeto cambiado".
Listeners
Un NamingListener es un objeto que monitoriza los NamingEvents. Cada categoría de tipo de evento tiene su correspondiente tipo de NamingListener. Por ejemplo, un NamespaceChangeListener representa una listener interesado en los eventos de cambio en el espacio de nombres y un ObjectChangeListener representa un listener interesado en eventos de cambio de objetos.

Para recibir notificaciones de eventos, se debe registrar un listener con un EventContext o un EventDirContext. Una vez registrado, el listener recibirá notificaciones de enveto cuando los cambios correspondiente ocurren en el servicio de nombre/directorio. Los detalles sobre Notificación de Evento pueden encontrarse en el Tutorial JNDI.

Paquete del Proveedor de Servicio

El paquete javax.naming.spi proporciona la forma por la cual los desarrolladores de diferentes proveedores de nombre/directorio pueden desarrollar y conectar sus implementaciones para que los servicios correspondientes sean accesibles desde aplicaciones que usan JNDI.

Arquitectura Plug-In
El paquete javax.naming.spi permite diferentes implementaciones a ser conectados dinámicamente. Estas implementaciones incluyen aquellas para el contexto inicial y para los contextos que puedan ser alcanzados desde el contexto inicial.
Soporte de Objetos Java
El paquete javax.naming.spi admite implementadores de lookup y métodos relacionados para devolver objetos Java que son naturales e intuitivos para el programador Java. Por ejemplo, si busca un nombre de impresora desde el directorio, es probable que espere obtener un objeto de impresora en el que operar. Este soporte se proporciona en la forma de factorías de objetos.

Este paquete también proporciona soporte para hacer lo contrario. Esto es, implementadores de Context.bind() y métodos relacionados pueden aceptar objetos Java y almacena los objetos en un formato aceptable para el servicio de nombre/directorio subyacente. Este soporte se proporciona en la forma de factorías de estado.

Múltiples Sistemas de Nombres (Federación)
Las operaciones JNDI permiten a las aplicaciones proporcionen nombres que abarcan varios sistemas de nombres. En el proceso de completar una operación, un proveedor de servicio podría necesitar interactuar con otro proveedor de servicio, por ejemplo, para pasar la operación a continuar en el siguiente sistema de nombres. Este paquete proporciona el soporte para que diferentes proveedores cooperen para completar operaciones JNDI.

Los detalles sobre el mecanismo del Proveedor de Servicio pueden encontrarse en el Tutorial JNDI.
