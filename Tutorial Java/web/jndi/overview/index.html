Lección: Visión General de JNDI

Nombrado en Java e Interfaz de Directorio™ (JNDI) es una Interfaz de Programación de Aplicaciones (IPA) que proporciona funcionalidad de nombrado y directorio a las aplicaciones escritas usando el lenguaje de programación Java™. Se define como independiente de cualquier implementación específica de servicio de directorios. Por lo tanto, se puede acceder de manera común a una variedad de directorios — nuevos, emergentes, y ya implementados.

Arquitectura

La arquitectura JNDI consiste de una IPA y una interfaz de proveedor de servicio (SPI). Las aplicaciones Java usan la IPA de JNDI para acceder a una variedad de servicios de nombre y directorio. La SPI permite conectar de forma transparente una variedad de servicios de nombres y directorios, lo que permite que la aplicación Java que utiliza la IPA de JNDI acceda a sus servicios. Empaquetado

JNDI está incluído en la Plataforma Java SE. Para usar JNDI, debe tener las clases JNDI y uno o más proveedores de servicio. El KDJ incluye proveedores de servicio para los siguientes servicios de nombres/directorio:

  • Protocolo Ligero de Acceso a Directorios (LDAP)
  • Common Object Request Broker Architecture (CORBA) Common Object Services (COS) servicio de nombres
  • Registro de Invocación de Métodos Remotos de Java (RMI)
  • Servicio de Nombres de Dominio (DNS)

Otros proveedores de servicio pueden descargarse desde la página JNDI o obtenerse de otros fabricantes.

JNDI se divide en cinco paquetes:

  • javax.naming
  • javax.naming.directory
  • javax.naming.ldap
  • javax.naming.event
  • javax.naming.spi
