Añadir y Reemplazar Vínculos con Atributos

Los ejemplos de nombrado describieron cómo puede usar bind(), rebind(). La interfaz DirContext contiene versiones sobrecargadas de estos métodos que aceptan atributos. Puede usar estos métodos de DirContext para asociar atributos con el objeto a la vez que la vinculación o el subcontexto es agregado al espacio de nombres. Por ejemplo, podría crear un objeto Person y vincularlo al espacio de nombres y a la vez asociar atributos sobre el objeto Person.

Agregar un Vínculo Que Tiene Atributos

Se usa DirContext.bind() para agregar un vínculo que tiene atributos a un contexto. Acepta como argumentos el nombre del objeto, el objeto a vincularse, y un conjunto de atributos. 

// Create the object to be bound
Fruit fruit = new Fruit("orange");

// Create attributes to be associated with the object
Attributes attrs = new BasicAttributes(true); // case-ignore
Attribute objclass = new BasicAttribute("objectclass");
objclass.add("top");
objclass.add("organizationalUnit");
attrs.put(objclass);

// Perform bind
ctx.bind("ou=favorite, ou=Fruits", fruit, attrs);

Este ejemplo crea un objeto de clase Fruit y lo vincula al nombre "ou=favorite" dentro del contexto llamado "ou=Fruits", relativo a ctx. Esta vinculación tiene el atributo "objectclass". Si posteriormente busca el nombre "ou=favorite, ou=Fruits" en ctx, luego obtendría el objeto fruit. Si luego obtiene los atributos de "ou=favorite, ou=Fruits", obtendrá esos atributos con los que se creó el objeto. A continuación se muestra la salida de este ejemplo. 

# java Bind
orange
attribute: objectclass
value: top
value: organizationalUnit
value: javaObject
value: javaNamingReference
attribute: javaclassname
value: Fruit
attribute: javafactory
value: FruitFactory
attribute: javareferenceaddress
value: #0#fruit#orange
attribute: ou
value: favorite

Los atributos extra y los valores de atributo mostrados se usan para almacenar información sobre el objeto (fruit). Estos atributos extra se describen con más detalle en el recorrido.

Si fuera a ejecutar este ejemplo dos veces, entonces el segundo intento fallaría con una NameAlreadyBoundException. Esto se debe a que el nombre "ou=favorite" ya está vinculado en el contexto "ou=Fruits". Para que el segundo intento tenga éxito, tendría que usar rebind().

Reemplazar un Vínculo Que Tiene Atributos

Se usa DirContext.rebind() para agregar o reemplazar un vínculo y sus atributos. Acepta los mismos argumentos que bind(). Sin embargo, la semántica de rebind() requiere que si el nombre ya está vinculado, se desvinculará y se vincularán el objeto y los atributos recién proporcionados. 

// Create the object to be bound
Fruit fruit = new Fruit("lemon");

// Create attributes to be associated with the object
Attributes attrs = new BasicAttributes(true); // case-ignore
Attribute objclass = new BasicAttribute("objectclass");
objclass.add("top");
objclass.add("organizationalUnit");
attrs.put(objclass);

// Perform bind
ctx.rebind("ou=favorite, ou=Fruits", fruit, attrs);

Cuando ejecute este ejemplo, reemplazará el vínculo que creó el ejemplo bind(). 

# java Rebind
lemon
attribute: objectclass
value: top
value: organizationalUnit
value: javaObject
value: javaNamingReference
attribute: javaclassname
value: Fruit
attribute: javafactory
value: FruitFactory
attribute: javareferenceaddress
value: #0#fruit#lemon
attribute: ou
value: favorite
