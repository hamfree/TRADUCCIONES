Nombres de Atributos

Un atributo consiste de un identificador de atributo y un conjunto de valores de atributo. El identificador de atributo, también llamado nombre de atributo, es una cadena que identifica un atributo. Un valor de atributo es el contenido del atributo y su tipo no está restringido a una cadena. Puede usar un nombre de atributo cuando quiera especificar un atributo particular para recuperar, buscar, o modificar. Los nombres son también devueltos por las operaciones que devuelven atributos (como cuando realiza lecturas o búsquedas en el directorio).

Cuando usa nombres de atributo, necesita estar al tanto de ciertas características del servidor de directorio de forma que no se sorprenda por el resultado. Estas características se describen en las siguientes subsecciones.

Tipo de Atributo

En directorios como LDAP, el nombre de atributo identifica el tipo de atributo y es a menudo llamado atributo de nombre y tipo. Por ejemplo, el nombre de atributo "cn" es además llamado el atributo de nombre y tipo. Una definición de tipo de atributo especifica la sintaxis que el valor del atributo debe tener, si puede tener valores múltiples, y las reglas de igualda y ordenación a usar cuando se realizen operaciones de comparación y clasificación sobre los valores del atributo.

Subclases de Atributos

Algunas implementaciones de directorio soportan subclasificación de atributos, en el que el servidor permite que los tipos de atributos se definan en términos de otros tipos de atributos. Por ejemplo, un atributo "name" podría ser la superclase de todos los atributos relacionados con el nombre: "commonName" podría ser una subclase de "name". Para las implementaciones de directorio que admiten esto, solicitar el atributo "name" podría devolver el atributo "commonName".

Cuando accede a directorios que admiten la subclasificación de atributos, tiene que estar al tanto de que el servidor podría devolver atributos que tienen nombres diferentes de aquellos que pidió. Para minimizar la posibilidad de que esto suceda, use la subclase más derivada.

Sinónimos de Nombres de Atributo

Algunas implementaciones de directorio soportan sinónimos para los nombres de atributo. Por ejemplo, "cn" podría ser un sinónimo para "commonName". Así una petición por el atributo "cn" podría devolver el atributo "commonName".

Cuando se accede a directorios que admiten sinónimos para los nombres de atributo, debe tener en cuenta que el servidor podría devolver atributos con nombres diferentes a los solicitados. Para ayudar a evitar que esto suceda, use el nombre del atributo canónico en lugar de uno de sus sinónimos. El nombre de atributo canónico es el nombre usado en la defición del atributo; un sinónimo es el nombre que hace referencia al nombre de atributo canónico en su definición.

Preferencias de Idioma

Una extensión a LDAP v3 ( RFC 2596) le permite especificar un código de idioma junto con un nombre de atributo. Esto se asemeja a la subclasificación de atributos en que un nombre de atributo puede representar varios atributos diferentes. Un ejemplo es un atributo "description" que tiene dos variaciones de idioma: 

description: software
description;lang-en: software products
description;lang-de: Softwareprodukte

Una petición del atributo "description" devolvería los tres atributos.

Cuando acceda a directorios que admiten esta característica, debe tener en cuenta de que el servidor podría devolver atributos con nombres diferentes a los solicitados.
