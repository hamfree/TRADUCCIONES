Listar el Contexto

En vez de obtener un objeto único a la vez, como con Context.lookup(), puede listar un contexto entero usando una única operación. Hay dos métodos para listar un contexto: uno que devuelve los vínculos y uno que devuelve sólo los pares de nombre de clase de nombre a objeto.

El Método Context.List()

Context.list() devuelve una enumeración de NameClassPair. Cada NameClassPair consiste del nombre del objeto y su nombre de clase. El siguiente fragmento de código lista los contenidos del directorio "ou=People" (p.e., los ficheros y directorios encontrados en el directorio "ou=People"). 

NamingEnumeration list = ctx.list("ou=People");

while (list.hasMore()) {
    NameClassPair nc = (NameClassPair)list.next();
    System.out.println(nc);
}

El Método Context.listBindings()

Context.listBindings() devuelve una enumeración de Binding. Binding es una subclase de NameClassPair. Una vinculación (binding) no sólo contiene el nombre del objeto y el nombre de clase, sino también el objeto. El código siguiente enumera el contexto "ou=People", mostrando el nombre y objeto de cada vinculación. 

NamingEnumeration bindings = ctx.listBindings("ou=People");

while (bindings.hasMore()) {
    Binding bd = (Binding)bindings.next();
    System.out.println(bd.getName() + ": " + bd.getObject());
}

Terminar una NamingEnumeration

Se puede terminar una NamingEnumeration de una de tres formas: naturalmente, explícitamente, o inesperadamente.

  • Cuando NamingEnumeration.hasMore() devuelve false, se ha completado la enumeración y está efectivamente terminada.
  • Puede terminar una enumeración explícitamente antes de que se complete invocando NamingEnumeration.close(). Hacer esto proporciona una pista a la implementación subyacente para que libere cualquier recurso asociado con al enumeración.
  • Si hasMore() o next() lanzan una NamingException, entonces la enumeración se termina efectivamente.

Independientemente de cómo se ha terminado una enumeración, una vez terminada, ya no se puede utilizar. Invocar un método sobre una enumeración terminada produce un resultado no definido.

¿Porqué Hay Dos Métodos List Diferentes?

list() está diseñado para aplicaciones de estilo navegador que sólo desean mostrar los nombres de los objetos en un contexto. Por ejemplo, un navegador listaría los nombres en un contexto y esperaría a que el usuario seleccione uno o unos pocos de los nombres mostrados para realizar operaciones posteriores. Tales aplicaciones típicamente no necesitan acceder a todos los objetos en un contexto.

listBindings() está diseñado paora aplicaciones que necesitan realizar operaciones sobre los objetos en un contexo en masa. Por ejemplo, una aplicación de respaldo podría necesitar realizar operaciones de "estadísticas de archivo" en un directorio de ficheros. O un programa de administración de impresora podría querer reiniciar todas las impresoras en un edificio. Para realizar tales operaciones, estas aplicaciones necesitan obtener todos los objetos vinculados en un contexto. Por lo tanto, es más conveniente que los objetos se devuelvan como parte de la enumeración.

La aplicación puede usar list() o el potencialmente más caro listBindings(), dependiendo del tipo de información que necesite.
