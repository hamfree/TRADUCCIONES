Añadir, reemplazar o eliminar un vínculo (Binding)

La interfaz Context contiene métodos para agregar, reemplazar, y eliminar un vínculo en un contexto.

Agregar un Vínculo (Binding)

Se usa Context.bind() para agregar un vínculo a un contexto. Acepta como argumentos el nombre del objeto y el objeto a ser vinculado.

Antes de continuar: Los ejemplos en esta lección requieren que haga adicciones al esquema. Debe desactivar la comprobación de esquemas en el servidor LDAP o agregar el esquema que acompaña a este tutorial al servidor. Ambas tareas son típicamente realizadas por el administrador del servidor de directorio. Vea la lección Configuración LDAP. 

// Create the object to be bound
Fruit fruit = new Fruit("orange");

// Perform the bind
ctx.bind("cn=Favorite Fruit", fruit);

Este ejemplo crea un objeto de clase Fruit y lo enlaza a el nombre "cn=Favorite Fruit" en el contexto ctx. Si posteriormente busca el nombre "cn=Favorite Fruit" en ctx, entonces obtendría el objeto fruit. Dese cuenta que para compilar la clase Fruit, necesita la clase FruitFactory.

Si fuera a ejecutar este ejemplo dos veces, entonces el segundo intento fallaría con una NameAlreadyBoundException. Esto es porque el nombre "cn=Favorite Fruit" ya está vinculado. Para que el segundo intento tenga éxito, tendría que usar rebind().

Agregar o Reemplazar un Vínculo

Se usa rebind() para agregar o reemplazar un vínculo. Acepta los mismos argumentos que bind(), pero la semántica es tal que si el nombre ya está vinculado, entonces se desvinculará y se vinculará el objeto recién dado. 

// Create the object to be bound
Fruit fruit = new Fruit("lemon");

// Perform the bind
ctx.rebind("cn=Favorite Fruit", fruit);

Cuando ejecuta este ejemplo, reemplazará el vínculo creado por el ejemplo bind().

Eliminar un Vínculo

Para eliminar un vínculo, use unbind(). 

// Remove the binding
ctx.unbind("cn=Favorite Fruit");

Este ejemplo, cuando se ejecuta, elimina el vínculo que fue creado por el ejemplo de bind() o rebind().
