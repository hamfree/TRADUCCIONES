Excepciones de Nombrado

Muchos métodos en los paquetes JNDI lanzan una NamingException cuando necesitan indicar que la operación requerida no ha podido ser realizada. Comúnmente, verá un envoltorio try/catch alrededor de los métodos que pueden lanzar una NamingException: 

try {
    Context ctx = new InitialContext();
    Object obj = ctx.lookup("somename");
} catch (NamingException e) {
    // Handle the error
    System.err.println(e);
}

Jerarquía de Clases de Excepción

JNDI tiene una rica jerarquía de excepciones derivada de la clase NamingException. Los nombres de las clases de las excepciones se explican por sí mismas y se enumeran aquí.

Para gestionar una subclase particular de NamingException especialmente, puede capturar la subclase separadamente. Por ejemplo, el código siguiente trata especialmente la AuthenticationException y sus subclases. 

try {
    Context ctx = new InitialContext();
    Object obj = ctx.lookup("somename");
} catch (AuthenticationException e) {
    // attempt to reacquire the authentication information
    ...
} catch (NamingException e) {
    // Handle the error
    System.err.println(e);
}

Enumeraciones

Operaciones tales como Context.list() y DirContext.search() devuelven una NamingEnumeration. En estos casos, si ocurre un erro y no se devuelven resultados, entonces se lanzará NamingException o una de sus subclases apropiadas en el momento en que se invoca el método. Si un error ocurre pero hay algunos resultados a devolver, entonces se devuelve una NamingEnumeration de forma que pueda obtener esos resultados. Cuando se agotan todos los resultados, invocar NamingEnumeration.hasMore() causará que sea lanzada una NamingException (o una de sus subclases) para indicar el error. En este punto, la enumeración deja de ser válida y no se deben invocar más metodos en ella.

Por ejemplo, si realiza una search() y especifica un límite de recuento (n) de cuantas respuestas devolver, entonces search() devolverá una enumeración que consiste de un máximo de n resultados.Si el número de resultados excede n, entonces cuando se invoque NamingEnumeration.hasMore() para la vez n+1, se lanzará una SizeLimitExceededException. Vea el Recuento de Resultados de esta lección por un ejemplo de código.
