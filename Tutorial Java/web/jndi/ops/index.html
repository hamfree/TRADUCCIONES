Lección: Operaciones de Nombrado y Directorio

Puede usar JNDI para realizar operaciones de nombrado, incluyendo operaciones de lectura y operaciones para actualizar el espacio de nombres. Las operaciones siguientes se describen en esta lección:

  • Buscar un objeto
  • Listar los contenidos de un contexto
  • Agregar, sobreescribir, y eliminar un vínculo
  • Renombrar un objeto
  • Crear y destruir subcontextos

Configuración

Antes de realizar cualquier operación sobre un servicio de directorio o nombrado, necesita adquirir un contexto inicial — el punto de inicio en el nombre de espacio. Esto es porque todos los métodos en los servicios de directorio y nombrado se realizan relativos a algún contexto. Para obtener un contexto inicial, debe seguir estos pasos.

  1. Seleccione el proveedor de servicio del servicio correspondiente al que quiere acceder.
  2. Especifique cualquier configuración que necesite el contexto inicial.
  3. Llame al constructor InitialContext.

Paso 1: Seleccione el Proveedor de Servicio para el Contexto Inicial

Puede especificar el proveedor de servicio a utilizar para el contexto inicial creando un conjunto de propiedades de entorno (una Hashtable) y agregar el nombre de la clase de proveedor de servicio a ella. Las propiedades de entorno se describen en detalle en el Tutorial JNDI.

Si está usando el proveedor de servicio LDAP incluído en el KDJ, entonces su código se parecería al siguiente. 

Hashtable<String, Object> env = new Hashtable<String, Object>();
env.put(Context.INITIAL_CONTEXT_FACTORY, 
        "com.sun.jndi.ldap.LdapCtxFactory");

Para especificar el proveedor de servicio del sistema de ficheros en el KDJ, escribiría un código similar a este. 

Hashtable<String, Object> env = new Hashtable>String, Object>();
env.put(Context.INITIAL_CONTEXT_FACTORY, 
        "com.sun.jndi.fscontext.RefFSContextFactory");

Puede usar tambíén propiedades del sistema para especificar el proveedor de servicio a usar. Compruebe el Tutorial JNDI para los detalles.

Paso 2: Suministrar la Información que Necesita el Contexto Inicial

Los clientes de diferentes directorios podrían necesitar información varia para contactar con el directorio. Por ejemplo, podría necesitar especificar en qué máquina se está ejecutando el servidor y qué información se necesita para identificar al usuario del directorio. Tal información se pasa al proveedor de servicio a través de propiedades de entorno. JNDI especifica algunas propiedades de entorno genéricas que los proveedores de servicio pueden usar. La documentación de su proveedor de servicio le dará detalles de la información requerida para estas propiedades.

El proveedor LDAP requiere que el programa especifque la ubicación del servidor LDAP, como también información de la identidad de usuario. Para proporcionar esta información, escribiría código que se parece al siguiente. 

env.put(Context.PROVIDER_URL, "ldap://ldap.wiz.com:389");
env.put(Context.SECURITY_PRINCIPAL, "joeuser");
env.put(Context.SECURITY_CREDENTIALS, "joepassword");

Este tutorial usa el proveedor de servicio LDAP en el KDJ. Los ejemplos asumen que ha sido configurado un servidor en la máquina local en el puerto 389 con el nombre distinguido raíz de "o=JNDITutorial" y que no se requiere autenticación para actualizar el directorio. Esto incluye el código siguiente para configurar el entorno. 

env.put(Context.PROVIDER_URL, "ldap://localhost:389/o=JNDITutorial");

Si está usando un directorio que está configurado de forma diferente, entonces necesitará configurar estas propiedades de entorno en consecuencia. Necesitará reemplazar "localhost" con el nombre de esa máquina. Puede ejecutar estos ejemplos contra cualquier servidor de directorio público o su propio servidor ejecutándose en una máquina diferente. Necesitará reemplazar "localhost" con el nombre de esa máquina y reemplazar o=JNDITutorial con el contexto de nombrados que esté disponible.

Paso 3: Crear el Contexto Inicial

Ahora está listo para crear el contexto inicial. Para hacer eso, pasa al constructor de InitialContext las propiedades de entorno que creó anteriormente: 

Context ctx = new InitialContext(env);

Ahora que tiene una referencia a un objeto Context, puede empezar a acceder al servicio de nombrado.

Para realizar las operaciones de directorio, necesita usar un InitialDirContext. Para hacer eso, use uno de estos constructores: 

DirContext ctx = new InitialDirContext(env);

Esta sentencia devuelve una referencia a un objeto DirContext para realizar las operaciones de directorio.
