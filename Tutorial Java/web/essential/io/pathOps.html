Operaciones con Path

La clase Path incluye varios métodos que pueden ser usados para obtener información sobre la ruta de acceso, los elementos de acceso de la ruta de acceso, convertir la ruta de acceso a otras formas, o extraer porciones de una ruta de acceso. Hay también métodos para hacer coincidir la cadena de la ruta de acceso y métodos para quitar redundancias en una ruta de acceso. Esta lección trata estos métodos de Path, algunas veces llamados operaciones sintácticas, porque operan sobre la ruta de acceso en sí misma y no acceden al sistema de ficheros.

Esta sección cubre lo siguiente:

  • Creando un Path
  • Recuperando Información sobre un Path
  • Eliminando Redundancias de un Path
  • Convirtiendo un Path
  • Uniendo Dos Paths
  • Creando un Path Entre Dos Paths
  • Comparando Dos Paths

Creando un Path

Una instancia de Path contiene la información usada para especificar la localización de un fichero o directorio. En el momento en que se define, un Path se proporciona con una serie de uno o más nombres. Un elemento raíz o un nombre de fichero podría ser incluído, pero ninguno de ellos son requeridos. Un Path podría consistir de sólo un único directorio o nombre de fichero.

Puede fácilmente crear un objeto Path usando uno de los siguientes métodos get desde la clase auxiliar Paths (note el plural): 

Path p1 = Paths.get("/tmp/foo");
Path p2 = Paths.get(args[0]);
Path p3 = Paths.get(URI.create("file:///Users/joe/FileTest.java"));

El método Paths.get es un atajo par el siguiente código: 

Path p4 = FileSystems.getDefault().getPath("/users/sally");

El siguiente ejemplo crea /u/joe/logs/foo.log asumiendo que su directorio de usuario es /u/joe, o C:\joe\logs\foo.log si usted están en Windows. 

Path p5 = Paths.get(System.getProperty("user.home"),"logs", "foo.log");

Recuperando Información sobre un Path

Puede pensar de un Path como el almacenamiento de estos elementos de nombre como una secuencia. El elemento más alto en la estructura de directorios estaría localizado en el índice 0. El elemento más bajo en la estructura de directorios estaría localizado en el índice [n-1], donde n es el número de elementos de nombres en el Path. Hay métodos disponibles para recuperar elementos individuales o una subsecuencia del Path usando estos índices.

Los ejemplos en esta lección usan la siguiente estructura de directorios.

Ejemplo de Estructura de Directorios

El siguiente trozo de código define una instancia Path y entonces invoca varios métodos para obtener información sobre la ruta de acceso: 

// Ninguno de estos métodos requieren que el fichero
// en el Path exista.
// Sintaxis Microsoft Windows
Path path = Paths.get("C:\\home\\joe\\foo");

// Sintaxis Solaris
Path path = Paths.get("/home/joe/foo");

System.out.format("toString: %s%n", path.toString());
System.out.format("getFileName: %s%n", path.getFileName());
System.out.format("getName(0): %s%n", path.getName(0));
System.out.format("getNameCount: %d%n", path.getNameCount());
System.out.format("subpath(0,2): %s%n", path.subpath(0,2));
System.out.format("getParent: %s%n", path.getParent());
System.out.format("getRoot: %s%n", path.getRoot());

Eliminando Redundancias de un Path

Muchos sistemas de ficheros usan la notación "." para indicar el directorio actual y ".." para indicar el directorio padre. Usted podría tener una situación donde un Path contiene información redudante de directorios. Quizás un servidor está configurado para salvar sus ficheros de trazas en el directorio "/dir/logs/.", y usted quiere eliminar la notación final "/." del path.

Los siguientes ejemplos incluyen redundancias: 

/home/./joe/foo
/home/sally/../joe/foo

El método normalize elimina cualquier elemento redundante, lo cual incluye cualquier ocurrencia de "." o "directorio/..". Cualquier de los ejemplos precedentes se normalizan a /home/joe/foo.

Es importante notar que normalize no comprueba en el sistema de ficheros cuando limpia un path. Es una operación puramente sintáctica. En el segundo ejemplo, si sally fuera un enlace simbólico, eliminar sally/.. podría resultar en un Path que ya no localiza el fichero deseado.

Para limpiar un path mientras nos aseguramos que el resultado localiza el fichero correcto, puede usar el método toRealPath. Este método es descrito en la siguiente sección, Convirtiendo un Path.

Convirtiendo un Path

Puede usar tres métodos para convertir el Path. Si necesita convertir el path en una cadena que pueda ser abierta desde un navegador, puede usar toUri. Por ejemplo: 

Path p1 = Paths.get("/home/logfile");
// El resultado es file:///home/logfile
System.out.format("%s%n", p1.toUri());

El método toAbsolutePath convierte un path en un path absoluto. Si el path pasado es ya absoluto, devuelve el mismo objeto Path. El método toAbsolutePath puede ser de mucha ayuda cuando se procesan nombres de ficheros introducidos por el usuario. Por ejemplo: 

public class FileTest {
    public static void main(String[] args) {

        if (args.length < 1) {
            System.out.println("uso: FileTest fichero");
            System.exit(-1);
        }

        // Convierte la cadena de entrada en un objeto Path.
        Path inputPath = Paths.get(args[0]);

        // Convierte el Path entrante
        // en un path absoluto.
        // Generalmente, esto significa prepender
        // el directorio de trabajo
        // actual.  Si este ejemplo
        // fuera llamado así:
        //     java FileTest foo
        // los métodos getRoot y getParent
        // devolvería null
        // en la instancia "inputPath"
        // original. Invocar getRoot y
        // getParent en la instancia
        // "fullPath" devuelve los valores esperados.
        Path fullPath = inputPath.toAbsolutePath();
    }
}

El método toAbsolutePath convierte la entrada de usuario y devuelve un Path que devuelve valores útiles cuando se le pregunta. El fichero no necesita existir para que este método funcione.

El método toRealPath devuelve la ruta de acceso real de un fichero existente. Este método realiza varias operacioens en una:

  • Si se le pasa true a este método y el sistema de fichero soporta enlaces simbólicos, este método resolverá cualquier enlace simbólico en la ruta de acceso.
  • Si el Path es relativo, lo retorna como una ruta de acceso absoluta.
  • Si el Path contiene cualquier elemento redundante, devuelve un path con esos elementos eliminados.

Este método lanza una excepción si el fichero no existe o no puede ser accedido. Puede capturar la excepción cuando quiera manejar alguno de estos casos. Por ejemplo: 

try {
    Path fp = path.toRealPath();
} catch (NoSuchFileException x) {
    System.err.format("%s: el fichero o directorio" + " no existe%n", path);
    // Lógica para el caso de que el fichero no exista.
} catch (IOException x) {
    System.err.format("%s%n", x);
    // Lógica para cualquier otra clase de error.
}

Uniendo Dos Paths

Puede combinar paths usando el método resolve. Usted le pasa un path parcial , el cual es un path que no incluye un elemento raíz, y el path parcial es añadido al path original.

Por ejemplo, considere el siguiente trozo de código: 

// Solaris
Path p1 = Paths.get("/home/joe/foo");
// El resultado es /home/joe/foo/bar
System.out.format("%s%n", p1.resolve("bar"));

o

// Microsoft Windows
Path p1 = Paths.get("C:\\home\\joe\\foo");
// El resultado es C:\home\joe\foo\bar
System.out.format("%s%n", p1.resolve("bar"));

Pasar un path absoluto al método resolve devuelve el path pasado: 

// El resultado es /home/joe
Paths.get("foo").resolve("/home/joe");

Creando un Path Entre Dos Paths

Un requerimiento común cuando está escribiendo código de E/S de fichero es la capacidad para construir una ruta de acceso de una localización en el sistema de ficheros a otra localización. Puede satisfacer esto usando el método relativize. Este método construye una ruta de acceso que se origina en la ruta de acceso original y termina en la localización especificada por el path que se le pasa. El nuevo path es relativo a la ruta de acceso original.

Por ejemplo, considere dos paths relativos definidos como joe y sally: 

Path p1 = Paths.get("joe");
Path p2 = Paths.get("sally");

En ausencia de cualquier otra información, se asumió que joe y sally son hermanos, lo que significa nodos que residen en el mismo nivel dentro de la estructura de árbol. Para navegar desde joe a sally, Usted esperaría navegar primero un nivel hasta el nodo padre y luego abajo a sally: 

// El resultado es ../sally
Path p1_to_p2 = p1.relativize(p2);
// El resultado es ../joe
Path p2_to_p1 = p2.relativize(p1);

Considere un ejemplo un poco más complicado: 

Path p1 = Paths.get("home");
Path p3 = Paths.get("home/sally/bar");
// El resultado es sally/bar
Path p1_to_p3 = p1.relativize(p3);
// El resultado es ../..
Path p3_to_p1 = p3.relativize(p1);

En este ejemplo, las dos rutas de acceso comparten el mismo nodo, home. Para navegar desde home a bar, primero navega un nivel abajo a sally y entonces un nivel más abajo a bar. Navegar desde bar a home requiere moverse hacia arriba dos niveles.

Un path relativo no puede ser construído si sólo uno de los paths incluye un elemento raíz. Si ambos paths incluyen un elemento raíz, la capacidad de construir un path relativo dependerá del sistema.

El ejemplo recursivo Copia usa los métodos relativize y resolve.

Comparando Dos Paths

La clase Path soporta equals, habilitándole a comprobar la igualdada de dos paths. Los métodos startsWith y endsWith le habilitan a comprobar si un path empieza o termina con una cadena particular. Estos métodos son fáciles de usar. Por ejemplo: 

Path path = ...;
Path otroPath = ...;
Path inicio = Paths.get("/home");
Path final = Paths.get("foo");

if (path.equals(otroPath)) {
    // lógica de igualdad aquí
} else if (path.startsWith(inicio)) {
    // el path empieza con "/home"
} else if (path.endsWith(final)) {
    // path finaliza con "foo"
}

La clase Path implementa el interfaz Iterable. El método iterator devuelve un objeto que le habilita a iterar sobre los elementos de nombres en la ruta de acceso. El primer elemento devuelto es el más cercano a la raíz en el árbol de directorios. El siguiente trozo de código itera sobre un path, imprimiendo cada elemento de nombre: 

Path path = ...;
for (Path nombre: path) {
    System.out.println(nombre);
}

La clase Path también implementa el interfaz Comparable. Puede comparar objetos Path usando compareTo lo que es útil para la ordenación.

Puede también colocar objetos Path dentro de una Colección. Vea el recorrido Colecciones para obtener más información sobre esta poderosa característica.

Cuando quiera verificar si dos objetos Path localizan el mismo fichero, puede usar el método isSameFile, como se describió en Comprobar Si Dos Paths Localizan el Mismo Fichero.
