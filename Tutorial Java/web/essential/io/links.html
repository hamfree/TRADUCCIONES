Enlaces, Simbólicos o Inversos

Como se mención previamente, el paquete java.nio.file , y la clase Path en particular, es "consciente de los enlaces". Cada método Path o detecta que hacer cuando se encuentra un enlace simbólico, o establece una opción habilitándole a configurar el comportamiento cuando se encuentra el enlace simbólico.

La descripción hasta ahora ha sido sobre enlaces simbólicos o blandos, pero algunos sistemas de ficheros también soportan enlaces duros. Los enlaces duros son más restrictivos que los enlaces simbólicos, de la siguiente forma:

  • El objetivo del enlace debe existir.
  • Los enlaces duros generalmente no están permitidos sobre directorios.
  • Los enlaces duros no pueden cruzar particiones o volúmenes. Por ello, no pueden existir a través de sistemas de ficheros.
  • Un enlace duro parece, y se comporta, como un fichero regular, así que puede ser difícil de encontrar.
  • Un enlace duro es, para todos los efectos, la misma entidad que el fichero original. Ellos tienen los mismos permisos de ficheros, marcas de tiempo, etcétera. Todos los atributos son idénticos.

Debido a estas restricciones, los enlaces duros no son usados tan a menudo como los enlaces simbólicos, pero los métodos Path funcionan perfectamente con los enlaces duros.

Varios métodos tratan específicamente con enlaces y son cubiertos en las siguientes secciones:

  • Crear un Enlace Simbólico
  • Crear un Enlace Duro
  • Detectar un Enlace Simbólico
  • Encontrar el Objetivo de un Enlace

Crear un Enlace Simbólico

Si su sistema de ficheros lo soporta, puede crear un enlace simbólico usando el método createSymbolicLink(Path, Path, FileAttribute<?>) . El segundo argumento Path representa el fichero o directorio objetivo y podría o no podría existir. El siguiente trozo de código crea un enlace simbólico con permisos por defecto: 

Path newLink = ...;
Path target = ...;
try {
    Files.createSymbolicLink(newLink, target);
} catch (IOException x) {
    System.err.println(x);
} catch (UnsupportedOperationException x) {
    // Algunos sistemas de ficheros no soportan enlaces simbólicos.
    System.err.println(x);
}

La vararg FileAttributes le habilita a especificar atributos de fichero iniciales que son establecidos atómicamente cuando el enlace es creado. Sin embargo, este argumento está destinado para futuro uso y no está actualmente implementado.

Crear un Enlace Duro

Puede crear un enlace duro (o regular) a un fichero existente usando el método createLink(Path, Path) . El segundo argumento Path localiza el fichero existente, y debe existir o se lanza una NoSuchFileException. El siguiente trozo de código muestra cómo crear un enlace: 

Path newLink = ...;
Path existingFile = ...;
try {
    Files.createLink(newLink, existingFile);
} catch (IOException x) {
    System.err.println(x);
} catch (UnsupportedOperationException x) {
    // Algunos sistemas de ficheros no
    // soportan añadir un
    // fichero existente a un directorio.
    System.err.println(x);
}

Detectar un Enlace Simbólico

Para determinar si una instancia Path es un enlace simbólico, puede usar el método isSymbolicLink(Path) . El trozo de código siguiente muestra cómo: 

Path file = ...;
boolean isSymbolicLink =
    Files.isSymbolicLink(file);

Para obtener más información, vea Gestionando Metadatos.

Encontrar el Objetivo de un Enlace

Puede obtener el destino de un enlaces simbólico usando el método readSymbolicLink(Path) , como sigue: 

Path link = ...;
try {
    System.out.format("El destino del enlace" +
        " '%s' es '%s'%n", link,
        Files.readSymbolicLink(link));
} catch (IOException x) {
    System.err.println(x);
}

Si el Path no es un enlace simbólico, este método lanzará una NotLinkException.
