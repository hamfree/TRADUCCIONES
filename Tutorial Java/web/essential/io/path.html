¿Qué es un Path? (Y Otros Hechos del Sistema del Ficheros)

Un sistema de ficheros almacena y organiza ficheros en algún tipo de soporte, generalmente uno o más discos duros, de tal manera que se puedan recuperar fácilmente. La mayoría de los sistemas de ficheros en uso hoy en día almacenan los ficheros en una estructura de árbol (o jerárquica). En lo alto del árbol hay uno (o más) nodos raíz. Bajo el nodo raíz, hay ficheros y directorios (carpetas en Microsoft Windows). Cada directorio puede contener ficheros y subdirectorios, los cuales a su vez pueden contener ficheros y subdirectorios, etcétera, potencialmente hasta una profundidad casi ilimitada.

Esta sección cubre lo siguiente:

  • ¿Qué Es un Path?
  • ¿Relativo o Absoluto?
  • Enlaces Simbólicos

¿Qué Es un Path?

La siguiente figura muestra un ejemplo de árbol de directorios conteniendo un único nodo raíz. Microsoft Windows soporta nodos raíz múltiples. Cada nodo raíz se asigna a un volumen, tales como C:\ o D:\. El SO Solaris soporta un único nodo raíz, el cual es denotado por el caraćter barra, /.

Ejemplo de Estructura de Directorio

Un fichero es identificado por su ruta de acceso a través del sistema de fichero, empezando desde el nodo raíz. Por ejemplo, el fichero statusReport en la figura previa está descrito por la siguiente notación en el SO Solaris: 

/home/sally/statusReport

En Microsoft Windows, statusReport está descrito por la siguiente notación: 

C:\home\sally\statusReport

El carácter usado para separar los nombres de directorio (también llamado el delimitador) es específico al sistema de ficheros: El SO Solaris usa la barra inclinada (/), y Microsoft Windows usa la barra diagonal (\).

¿Relativo o Absoluto?

Una ruta de acceso es o relativa o absoluta. Una ruta de acceso absoluta siempre contiene el elementoi raíz y la lista completa de directorios requeridos para localizar el fichero. Por ejemplo, /home/sally/statusReport es una ruta de acceso absoluta. Toda la información necesitada para localizar el fichero está contenida en la cadena de la ruta de acceso.

Una ruta de acceso relativo necesita ser combinada con otra ruta de acceso para poder acceder a un fichero. Por ejemplo, joe/foo es una ruta de acceso relativa. Sin más información, un programa no puede localizar de forma fiable el directorio joe/foo en el sistema de ficheros.

Enlaces Simbólicos

Los objetos del sistema de ficheros son en su mayoría típicamente directorios o ficheros. Todo el mundo está familiarizado con estos objetos. Pero algunos sistemas de ficheros también soportan la noción de enlaces simbólicos. Un enlace simbólico también referido como un symlink o un enlace blando.

Un enlace simbólico es un fichero especial que sirve como referencia a otro fichero. En la mayor parte, los enlace simbólicos son transparentes para las aplicaciones, y las operaciones sobre los enlaces simbólicos son automáticamente redirigidas al objetivo del enlace. (El fichero o directorio que está siendo apuntado es llamado objetivo del enlace.) Las excepciones se producen cuando un enlace simbólico es borrado, o renombrado en cuyo caso el enlace en sí mismo es borrado, o renombrado y no el objetivo del enlace.

En la siguiente figura, logFile aparece como un fichero regular para el usuario, pero es en realidad un enlace simbólico a dir/logs/HomeLogFile. HomeLogFile es el objetivo del enlace.

Ejemplo de Enlace Simbólico.

Un enlace simbólico es usualmente transparente al usuario. Leer o escribir a un enlace simbólico es lo mismo que leer o escribir a cualquier otro fichero o directorio.

La frase resolver un enlace significa substituir la localización actual en el sistema de ficheros para el enlace simbólico. En el ejemplo, resolver logFile produce dir/logs/HomeLogFile.

En los escenarios del mundo real, la mayoría de los sistemas de ficheros hacen un uso liberal de los enlaces simbólicos. Ocasionalmente, un enlace simbólico creado sin cuidado puede causar una referencia circular. Una referencia circular ocurre cuando el objetivo de un enlace apunta de nuevo al enlace original. La referencia circular podría ser indirecta: directorio a apunta a directorio b, el cual apunta al directorio c, el cual contiene un subdirectorio que apunta de vuelta al directorio a. Las referencias circulares pueden causar estragos cuando un programa está recorriendo recursivamente una estructura de directorios. Sin embargo, este escenario ha sido tenido en cuenta y no causará que su programa se quede en el bucle infinitamente.
