Gestionando Metadatos (Fichero y Almacenamiento de Atributos del Fichero)

La definición de metados es "datos sobre otros datos". En un sistema de ficheros, los datos están contenidos en sus ficheros y directorios, y la información de pistas de metadatos sobre cada uno de estos objetos: ¿es un fichero regular, un directorio, o un enlace? ¿Cuál es su tamaño, fecha de creación, última fecha de modificación, propietario del fichero, grupo propietario, y permisos de acceso?

Los metadatos de un sistema de ficheros suelen denominarse como sus atributos de ficheros. La clase Files incluye métodos que pueden ser usados para obtener un atributo único de un fichero, o establecer un atributo.

Esta tabla muestra algunos métodos para la clase Files
Métodos Comentario
size(Path) Devuelve el tamaño del fichero especificado en bytes.
isDirectory(Path, LinkOption) Devuelve verdadero si el Path especificado localiza un fichero que es un directorio.
isRegularFile(Path, LinkOption...) Devuelve verdadero si el Path especificado localiza un fichero que es un fichero regular.
isSymbolicLink(Path) Devuelve verdadero si el Path especificado localiza un fichero que es un enlace simbólico.
isHidden(Path) Devuelve verdadero si el Path especificado localiza un fichero que está considerado oculto por el sistema de ficheros.
getLastModifiedTime(Path, LinkOption...)
setLastModifiedTime(Path, FileTime) 		Devuelve o establece la última hora de modificación del fichero.
getOwner(Path, LinkOption...)
setOwner(Path, UserPrincipal) 		Devuelve o establece el propietario del fichero.
getPosixFilePermissions(Path, LinkOption...)
setPosixFilePermissions(Path, Set<PosixFilePermission>) 		Devuelve o establece los permisos POSIX de un fichero.
getAttribute(Path, String, LinkOption...)
setAttribute(Path, String, Object, LinkOption...) 		Devuelve o establece el valor de un atributo de fichero.

Si un programa necesita varios atributos de fichero al mismo tiempo, puede ser ineficiente usar métodos que recuperan un atributo único. El acceder repetidamente el sistema de ficheros para recuperar un atributo único puede afectar negativamente al rendimiento. Por esta razón, la clase Files ofrece dos mñetodos readAttributes para coger los atributos de un fichero en una operación en bloque.

Esta tabla muestra algunos métodos readAttributes para la clase Files
Método Comentario
readAttributes(Path, String, LinkOption...) Lee los atributos de un fichero como una operación en bloque. El parámetro String identifica los atributos a leer.
readAttributes(Path, Class<A>, LinkOption...) Lee los atributos de un fichero como una operación en bloque. El parámetro Class<A> es el tipo de atributos pedidos y el método devuelve un objeto de esa clase.

Antes de mostrar los ejemplos de los métodos readAttributes, hay que mencionar que diferentes sistemas de ficheros tienen diferentes nociones sobre qué atributos serán rastreados. Por esta razón, los atributos relacionados con el fichero son agrupados juntos en vistas. Una vista asigna a una implementación de un sistema de ficheros particular, tal como POSIX o DOS, o a su funcionalidad común, tal como la propiedad de un archivo.

Las vistas soportadas son las siguientes:

  • BasicFileAttributeView – Ofrece una vista de los atributos básicos que son requeridos a estar soportados por todas las implementaciones de sistemas de ficheros.
  • DosFileAttributeView – Extiende la vista de atributos básicos con los cuatro bits estándar soportados en sistemas de ficheros que soportan los atributos DOS.
  • PosixFileAttributeView – Extiende la vista de atributos básicos con atributos soportados en sistemas de ficheros que soportan la familia POSIX de estándares, tal como UNIX. Estos atributos incluyen el propietario del fichero, el grupo propietario, y los nueve permisos de acceso relacionados.
  • FileOwnerAttributeView – Soportada por cualquier implementación de sistema de ficheros que soportan el concepto de propietario de fichero.
  • AclFileAttributeView – Soporta la lectura y actualización de Listas de Control de Acceso (ACL) de un fichero. El modelo ACL de NFSv4 es soportado. Cualquier modelo ACL, tal como el modelo ACL de Windows, que tenga un mapeo bien-definido al modelo NFSv4 podría también ser soportado.
  • UserDefinedFileAttributeView – Habilita el soporte de metadatos que son definidos por el usuario. Esta vista puede ser mapeada a cualquier mecanismo de extensión que un sistema soporte. En el SO Solaris, por ejemplo, puede usar esta vista para almacenar el tipo MIME de un fichero.

Una implementación específica de un sistema de ficheros podría soportar sólo la vista básica de atributos de ficheros, o podría soportar varias de estas vistas de atributos de ficheros. Una implementación de sistema de ficheros podría soportar otras vistas de atributos no incluídas en esta IPA.

En la mayoría de los casos, no tendrá que tratar directamente con ninguna de las interfaces FileAttributeView. (Si necesita trabajar directamente con FileAttributeView, puede accederlas a través del método getFileAttributeView(Path, Class<V>, LinkOption...)).

Los métodos readAttributes usan genéricos y pueden ser usados para leer los atributos de cualquiera de las vistas de atributos del fichero. Los ejemplos en el resto de esta página usan los métodos readAttributes.

Atributos Básicos del Fichero

Como se mencionó previamente, para leer los atributos básicos de un fichero, puede usar uno de los métodos Files.readAttributes, que leen todos los atributos básicos en una operación en bloque. Esto es de lejos más eficiente que acceder al sistema de ficheros separadamente para leer cada atributo individual. Los argumentos varargs actualmente soportan la enumeración LinkOption , NOFOLLOW_LINKS. Use ésta opción cuando no quiera que se sigan los enlaces simbólicos.

Una palabra sobre las marcas de tiempo: El conjunto de atributos básicos incluye tres marcas de tiempo: creationTime, lastModifiedTime, y lastAccessTime. Cualquiera de estas marcas de tiempo podrían no estar soportadas en una implementación particular, en cuyo caso el correspondiente método de acceso devuelve un valor específico-de-la-implementación. Cuando se soporte, la marca de tiempo es devuelta como un objeto FileTime.

El siguiente trozo de código lee e imprime los atributos básicos de fichero para un fichero dado y usa los métodos en la clase BasicFileAttributes. 

Path file = ...;
BasicFileAttributes attr = Files.readAttributes(file, BasicFileAttributes.class);

System.out.println("Hora de creación: " + attr.creationTime());
System.out.println("Hora de último acceso: " + attr.lastAccessTime());
System.out.println("Hora de última modificación: " + attr.lastModifiedTime());

System.out.println("es un directorio: " + attr.isDirectory());
System.out.println("es otro: " + attr.isOther());
System.out.println("es un fichero regular: " + attr.isRegularFile());
System.out.println("es un enlace simbólico: " + attr.isSymbolicLink());
System.out.println("tamaño: " + attr.size());

Además de los metodos de acceso mostrados en este ejemplo, hay un método fileKey que devuelve o un objeto que identifica de manera única el fichero o null si la clave del fichero no está disponible.

Establecimiento de Marcas de Tiempo

El siguiente trozo de código establece la hora de última modificación en milisegundos: 

Path file = ...;
BasicFileAttributes attr =
    Files.readAttributes(file, BasicFileAttributes.class);
long currentTime = System.currentTimeMillis();
FileTime ft = FileTime.fromMillis(currentTime);
Files.setLastModifiedTime(file, ft);
}

Atributos DOS de Fichero

Los atributos de fichero DOS están también soportados en sistemas de ficheros distintos de DOS, tales como Samba. El siguiente trozo de código usa los métodos de la clase DosFileAttributes. 

Path file = ...;
try {
    DosFileAttributes attr =
        Files.readAttributes(file, DosFileAttributes.class);
    System.out.println("es de Solo Lectura es  " + attr.isReadOnly());
    System.out.println("está Oculto es " + attr.isHidden());
    System.out.println("tiene la marca de Archivo es " + attr.isArchive());
    System.out.println("es del Sistema es " + attr.isSystem());
} catch (UnsupportedOperationException x) {
    System.err.println("Atributos DOS " +
        " de fichero no soportados:" + x);
}

Sin embargo, puede establecer un atributo DOS usando el método setAttribute(Path, String, Object, LinkOption...), como sigue: 

Path file = ...;
Files.setAttribute(file, "dos:hidden", true);

Permisos de Fichero POSIX

POSIX es un acrónimo para Interfaz de Sistema Operativo Portable para UNIX y es un conjunto de estándares de IEEE e ISO diseñados para asegurar la interoperabilidad entre diferentes sabores de UNIX. Si un programa respeta estos estándares POSIX, es fácilmente portado a otros sistemas operativos que cumplen con POSIX.

Además de propietario de fichero y grupo propietario, POSIX soporta nueve permisos de fichero: los permisos leer, escribir, y ejecutar para el propietario del fichero, los miembros del mismo grupo, y el "resto del mundo."

El siguiente trozo de código lee los atributos POSIX de fichero para un fichero dado y los imprime en la salida estándar. El código usa los método de la clase PosixFileAttributes. 

Path file = ...;
PosixFileAttributes attr =
    Files.readAttributes(file, PosixFileAttributes.class);
System.out.format("%s %s %s%n",
    attr.owner().getName(),
    attr.group().getName(),
    PosixFilePermissions.toString(attr.permissions()));

La clase auxiliar PosixFilePermissions ofrece varios métodos útiles, como los que siguen:

  • El método toString , usado en el trozo de código previo, convierte los permisos de fichero en una cadena (por ejemplo, rw-r--r--).
  • El método fromString acepta una cadena representando los permisos de fichero y construye un Set de permisos de fichero.
  • El método asFileAttribute acepta un Set de permisos de fichero y construye un atributo de fichero que puede ser pasado al método Path.createFile o Path.createDirectory.

El siguiente trozo de ćodigo lee los atributos de un fichero y crea un nuevo fichero, asignándole los atributos del fichero original al nuevo fichero: 

Path sourceFile = ...;
Path newFile = ...;
PosixFileAttributes attrs =
    Files.readAttributes(sourceFile, PosixFileAttributes.class);
FileAttribute<Set<PosixFilePermission>> attr =
    PosixFilePermissions.asFileAttribute(attrs.permissions());
Files.createFile(file, attr);

El método asFileAttribute envuelve los permisos como un FileAttribute. El código entonces intenta crear un fichero nuevo con esos permisos. Note que la umask también se aplica, así que el nuevo fichero podría ser más seguro que los permisos que fueron requeridos.

Para establecer los permisos de un fichero a valores representados por una cadena codificada a mano, puede usar el siguiente código: 

Path file = ...;
Set<PosixFilePermission> perms =
    PosixFilePermissions.fromString("rw-------");
FileAttribute<Set<PosixFilePermission>> attr =
    PosixFilePermissions.asFileAttribute(perms);
Files.setPosixFilePermissions(file, perms);

El ejemplo Chmod recursivamente cambia los permisos de los ficheros en una manera similar a la utilidad chmod.

Establecer un Propietario o Grupo de un Fichero

Para traducir un nombre a un objeto que puede almacenar como propietario de un archivo o como propietario de un grupo, puede usar el servicio UserPrincipalLookupService . Este servicio busca un nombre o un nombre de grupo como una cadena y devuelve un objeto UserPrincipal representando esa cadena. Puede obtener el servicio de búsqueda de usuarios para el sistema de ficheros por defecto usando el método FileSystem.getUserPrincipalLookupService.

El siguiente trozo de código muestra cómo establecer el propietario del fichero usando el método setOwner: 

Path file = ...;
UserPrincipal owner = file.GetFileSystem().getUserPrincipalLookupService()
        .lookupPrincipalByName("sally");
Files.setOwner(file, owner);

No hay un método especial en la clase Files para establecer un propietario de grupo. Sin embargo, una forma segura de hacerlo es directamente a través de la vista de atributos de fichero POSIX, como sigue: 

Path file = ...;
GroupPrincipal group =
    file.getFileSystem().getUserPrincipalLookupService()
        .lookupPrincipalByGroupName("green");
Files.getFileAttributeView(file, PosixFileAttributeView.class)
     .setGroup(group);

Atributos Definidos-Por-Usuario del Fichero

Si los atributos de fichero soportados por su implementación de sistema de ficheros no son suficientes para sus necesidades, puede usar la UserDefinedAttributeView para crear y trazar sus propios atributos de fichero.

Algunas implementaciones asignan este concepto a características como los Flujos de Datos Alternativos de NTFS y los atributos extendidos en sistemas de ficheros como ext3 y ZFS. La mayoría de las implementaciones imponen restricciones en el tamaño del valor, por ejemplo, ext3 limita el tamaño a 4 kilobytes.

Un tipo MIME del fichero puede ser almacenado como un atributo definido-por-el-usuario usando este trozo de código: 

Path file = ...;
UserDefinedFileAttributeView view = Files
    .getFileAttributeView(file, UserDefinedFileAttributeView.class);
view.write("user.mimetype",
           Charset.defaultCharset().encode("text/html");

Para leer el atributo de tipo MIME, usará este trozo de código: 

Path file = ...;
UserDefinedFileAttributeView view = Files
.getFileAttributeView(file,UserDefinedFileAttributeView.class);
String name = "user.mimetype";
ByteBuffer buf = ByteBuffer.allocate(view.size(name));
view.read(name, buf);
buf.flip();
String value = Charset.defaultCharset().decode(buf).toString();

El ejemplo Xdd muestra como obtener, establecer, y borrar un atributo definido-por-el-usuario.

Nota: En Linux, podría tener que habilitar los atributos extendidos para que los atributos definidos-por-el-usuario funcionen. Si recibe una UnsupportedOperationException cuando intenta acceder a la vista de atributos definida-por-el-usuario, necesita remontar el sistema de fichero. El siguiente comando remonta la partición raíz con atributos extendidos para el sistema de ficheros ext3. Si este comando no funciona para su sabor de Linux, consulte la documentación. 
$ sudo mount -o remount,user_xattr /

Si quiere hacer el cambio permanente, añada una entrada en /etc/fstab.

Atributos de Almacenamiento de Fichero

Puede usar la clase FileStore para aprender información sobre el almacenamiento de ficheros, tal como cuánto espacio hay disponible. El método getFileStore(Path) obtiene el almacenamiento de fichero para el fichero especificado.

El siguiente trozo de código imprime el uso del espacio para el almacenamiento de fichero donde un fichero particular reside: 

Path file = ...;
FileStore store = Files.getFileStore(file);

long total = store.getTotalSpace() / 1024;
long used = (store.getTotalSpace() -
             store.getUnallocatedSpace()) / 1024;
long avail = store.getUsableSpace() / 1024;

El ejemplo UsoDelDisco usa esta IPA para imprimir información sobre el espacio en disco para todos los almacenamientos en el sistema de ficheros por defecto. Este ejemplo usa el método getFileStores en la clase FileSystem para obtener todos los almacenes de ficheros para el sistema de ficheros.
