Creando y Leyendo Directorios

Algunos de los métodos previamente descritos, tales como delete, funcionan sobre ficheros, enlaces y directorios. Pero ¿cómo enumeras todos los directorios en la parte superior de un sistema de ficheros? ¿Cómo listar el contenido de un directorio o crear un directorio?

Esta sección cubre la siguiente funcionalidad específica a directorios:

  • Listado de los directorios raíz de un sistema de archivos
  • Crear un Directorio
  • Crear un Directorio Temporal
  • Listando los Contenidos de un Directorio
  • Filtrar una lista de directorios mediante Globbing
  • Escribir su propio filtro de directorio

Listado de los directorios raíz de un sistema de archivos

Puede listar todos los directorios raíz de un sistema de ficheros usando el método FileSystem.getRootDirectories. Este método devuelve un Iterable, el cual le habilita a usar la sentencia mejorada for para iterar sobre todos los directorios raíz.

El trozo de código siguiente imprime los directorios raíz del sistema de ficheros por defecto: 

Iterable<Path> dirs = FileSystems.getDefault().getRootDirectories();
for (Path name: dirs) {
    System.err.println(name);
}

Crear un Directorio

Puede crear un directorio nuevo usando el método createDirectory(Path, FileAttribute<?>) . Si no quiere especificar ningún FileAttributes, el directorio nuevo tendrá los atributos por defecto. Por ejemplo: 

Path dir = ...;
Files.createDirectory(path);

El trozo de código siguiente crea un nuevo directorio sobre un sistema de ficheros PIS que tiene permisos específicos: 

Set<PosixFilePermission> perms =
    PosixFilePermissions.fromString("rwxr-x---");
FileAttribute<Set<PosixFilePermission>> attr =
    PosixFilePermissions.asFileAttribute(perms);
Files.createDirectory(file, attr);

Para crear un directorio en varios niveles más profundos cuando uno o más de los directorios padres podrían no existir aún, puede usar el método de convenencia, createDirectories(Path, FileAttribute<?>). Como con el método createDirectory(Path, FileAttribute<?>) , puede especificar un conjunto opcional de atributos de fichero iniciales. El trozo de código siguiente usa los atributos por defecto: 

Files.createDirectories(Paths.get("foo/bar/test"));

Los directorios son creados, como se necesiten, de arriba abajo. En el ejemplo foo/bar/test , si el directorio foo no existe, es creado. A continuación, el directorio bar es creadi, si se necesita, y, finalmente, el directorio test es creado.

Es posible para este método fallar después de crear algunos, pero no todos, de los directorios padre.

Crear un Directorio Temporal

Puede crear un directorio temporal usando un de los métodos createTempDirectory :

  • createTempDirectory(Path, String, FileAttribute<?>...)
  • createTempDirectory(String, FileAttribute<?>...)

El primer método permite al código especificar un lugar para el directorio temporal y el segundo método crea un nuevo directorio en el directorio para ficheros temporales por defecto.

Listando los Contenidos de un Directorio

Puede lista todos los contenidos de un directorio usando el método newDirectoryStream(Path) . Este método devuelve un objeto que implementa el interfaz DirectoryStream . La clase que implementa el interfaz DirectoryStream también implementa Iterable, así que puede iterar a traves del flujo del directorio, leyendo todos los objetos. Este enfoque escala bien con directorios muy grandes.

Recuerde: El DirectoryStream devuelto es un flujo. Si no está usando una sentencia try-with-resources, no olvide cerrar el flujo en el bloque finally . La sentencia try-with-resources se ocupa de esto por usted.

El trozo de código siguiente muestra cómo imprimir el contenido de un directorio: 

Path dir = ...;
try (DirectoryStream<Path> stream = Files.newDirectoryStream(dir)) {
    for (Path file: stream) {
        System.out.println(file.getFileName());
    }
} catch (IOException | DirectoryIteratorException x) {
    // La IOException puede no ser nuncan lazanda por la iteración.
    // En este trozo, puede ser sólo lanzada por newDirectoryStream.
    System.err.println(x);
}

Loso objetos Path devueltos por el iterador son los nombres de las entradas resueltas contra el directorio. Así, si usted está listando el contenido del directorio /tmp , las entradas son devueltas con la forma /tmp/a, /tmp/b, etcétera.

Este método devuelve el contenido entero de un directorio: ficheros, enlaces, subdirectorios, y ficheros ocultos. Si quiere ser más selectivo sobre el contenido que es recuperado, puede usar uno de los otros métodos newDirectoryStream , como se describe más tarde en esta página.

Note que si hay una excepción durante la iteración del directorio entonces se lanza una DirectoryIteratorException con una IOException como la causa. Los métodos del Iterator no pueden lanzar una excepción de excepciones.

Filtrar una lista de directorios mediante Globbing

Si sólo quiere traer únicamente ficheros y subdirectorios donde cada nombre coincide con un patrón particular, puede hacerlo usando el método newDirectoryStream(Path, String) , el cual establece un filtro glob interno. Si no está familiarizado con la sintaxis de glob, vea ¿Qué Es un Glob?

Por ejemplo, el siguiente trozo de código lista los ficheros relacionados con Java: los ficheros .class, .java, y .jar .: 

Path dir = ...;
try (DirectoryStream<Path> stream =
     Files.newDirectoryStream(dir, "*.{java,class,jar}")) {
    for (Path entry: stream) {
        System.out.println(entry.getFileName());
    }
} catch (IOException x) {

    // IOException no puede ser lanzado nunca por la iteración
    // En este trozo, solo puede
    // ser lanzado por newDirectoryStream.
    System.err.println(x);
}

Escribir su propio filtro de directorio

Quizás quiera filtrar el contenido de un directorio basado en alguna condición distinta que la coincidencia de patrones. Puede crear su propio filtro implementando el interfaz DirectoryStream.Filter<T> . Este interfaz consiste de un método, accept, que determina si un fichero cumple los requerimientos de la búsqueda.

Por ejemplo, el trozo de código siguiente implementa un filtro que recuperar sólo directorios: 

DirectoryStream.Filter<Path> filter =
    newDirectoryStream.Filter<Path>() {
    public boolean accept(Path file) throws IOException {
        try {
            return (Files.isDirectory(path));
        } catch (IOException x) {
            // Falló en determinar si esto es un directorio.
            System.err.println(x);
            return false;
        }
    }
};

Una vez el filtro ha sido creado, puede ser invocaco usando el método newDirectoryStream(Path, DirectoryStream.Filter<? super Path>) . El siguiente trozo de código usa el filtro isDirectory para imprimir sólo los subdirectorios del directorio a la salida estándar: 

Path dir = ...;
try (DirectoryStream<Path>
                       stream = Files.newDirectoryStream(dir, filter)) {
    for (Path entry: stream) {
        System.out.println(entry.getFileName());
    }
} catch (IOException x) {
    System.err.println(x);
}

Este método es usado para filtrar un directorio único sólo. Sin embargo, si quiere encontrar todos los subdirectorios en un árbol de ficheros, usará el mecanismo de Recorriendo el Árbol de Ficheros.
