Borrando un Fichero o Directorio

Puede borrar ficheros, directorios o enlaces. Con los enlaces simbólicos, el enlace es borrado y no el objetivo del enlace. Con los directorios, el directorio debe estar vacío, o el borrado fallará.

La clase Files ofrece dos métodos de borrado.

El método delete(Path) borra el fichero o lanza una excepción si el borrado falla. Por ejemplo, si el fichero no existe se lanza una NoSuchFileException. Puede capturar la excepción para determinar porqué el borrado falló como sigue: 

try {
    Files.delete(path);
} catch (NoSuchFileException x) {
    System.err.format("%s: El fichero o directorio" + " no existe%n", path);
} catch (DirectoryNotEmptyException x) {
    System.err.format("%s no está vacío%n", path);
} catch (IOException x) {
    // Los problemas de permisos de fichero son capturados aquí.
    System.err.println(x);
}

El método deleteIfExists(Path) también borra el fichero, pero si el fichero no existe, no se lanza la excepción. El fallar silenciosamente es útil cuando tiene múltiples hilos borrando ficheros y no quiere lanzar una excepción sólo porque un hilo lo hizo así primero.
