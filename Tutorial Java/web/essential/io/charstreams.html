Flujos de Caracteres

La plataforma Java almacena los valores de caracteres usando convenciones Unicode. Los flujos de E/S de caracteres automáticamente transladan este formato interno a y desde el conjunto local de caracteres. En regiones Occidentales, el carácter local establecido es usualmente un superconjunto de 8 bits de ASCII.

Para la mayoría de las aplicaciones, la E/S con flujos de caracteres no es más complicada que la E/S con flujos de bytes. La entrada y salida hecha con clases de flujo automáticamente traduce a y desde el conjunto local de caracteres. Un programa que usa flujos de caracteres en vez de flujos de bytes automáticamente adapta el conjunto local de caracteres y está listo para la internacionalización — todo sin esfuerzo extra por parte del programador.

Si la internacionalización no es un prioridad, puede simplemente usar las clases de flujo de caracteres sin prestar mucha atención a los problemas del conjunto de caracteres. Más tarde, si la internacionalización se convierte en una prioridad, su programa puede ser adaptado sin una recodificación extensiva. Vea el recorrido Internacionalización para obtener más información.

Usar Flujos de Caracteres

Todas las clases de flujo de caracteres son descendientes de Reader y Writer. Como con los flujos de bytes, hay clases de flujos de caracteres que se especializan en la E/S de ficheros: FileReader y FileWriter. El ejemplo CopiaCaracteres ilustra estas clases. 

import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;


// fuente original: CopyCharacters.java
public class CopiaCaracteres {
    public static void main(String[] args) throws IOException {

        FileReader flujoEntrada = null;
        FileWriter flujoSalida = null;

        try {
            flujoEntrada = new FileReader("xanadu.txt");
            flujoSalida = new FileWriter("salidacaracteres.txt");

            int c;
            while ((c = flujoEntrada.read()) != -1) {
                flujoSalida.write(c);
            }
        } finally {
            if (flujoEntrada != null) {
                flujoEntrada.close();
            }
            if (flujoSalida != null) {
                flujoSalida.close();
            }
        }
    }
}

CopiaCaracteres es muy similar a CopiaBytes. La mayor diferencia importante es que CopiaCaracteres usa FileReader y FileWriter para la entrada y la salida en lugar de FileInputStream y FileOutputStream. Note que ambos CopiaBytes y CopiaCaracteres usan una variable int para leer en y escribir a. Sin embargo, en CopiaCaracteres, la variable int contiene un valor carácter en sus últimos 16 bits; en CopiaBytes, la variable int contiene un valor byte en sus últimos 8 bits.

Flujos de Caracteres que Usan Flujos de Bytes

Los flujos de caracteres son a menudo "envoltorios" para flujos de bytes. El flujo de caracteres usa el flujo de bytes para ejecutar la E/S física, mientras el flujo de caracteres maneja la traducción entre caracteres y bytes. FileReader, por ejemplo, usa FileInputStream, mientras FileWriter usa FileOutputStream.

Hay dos flujos "puente" de byte-a-carácter-de-propósito-general: InputStreamReader y OutputStreamWriter. Úselos para crear flujos de caracteres cuando no haya clases de flujos de caracteres preempaquetadas que coincidan con sus necesidades. La lección de sockets en el recorrido de red muestra cómo crear flujos de caracteres desde los flujos de bytes ofrecidos por las clases de sockets.

E/S Orientada-a-Líneas

La E/S de caracteres usualmente ocurre en unidades más grandes que caracteres individuales. Una unidad común es la línea: una cadena de caracteres con un terminador de línea al final. Un terminador de línea puede ser una secuencia de caracteres retorno-de-carro/salto-de-línea ("\r\n"), un retorno-de-carro individual ("\r"), o un salto-de-línea individual ("\n"). El soportar todos los posibles terminadores de línea permite a los programas leer ficheros de texto creados por cualquiera de los ampliamente usados sistemas operativos.

Modifiquemos el ejemplo de CopiaCaracteres para usar E/S orientada-a-líneas. Para hacer esto, tenemos que usar dos clases que no hemos visto antes, BufferedReader y PrintWriter . Exploraremos estas clases en mayor profundidad en E/S Cacheada y Formateando. Ahora mismo, Solo estamos interesados en su apoyo a la E/S orientada a líneas.

El ejemplo CopiaLineas invoca BufferedReader.readLine y PrintWriter.println para hacer la entrada y la salida una línea a la vez. 

import java.io.FileReader;
import java.io.FileWriter;
import java.io.BufferedReader;
import java.io.PrintWriter;
import java.io.IOException;


// fuente original : CopyLines.java
public class CopiaLineas {
    public static void main(String[] args) throws IOException {

        BufferedReader flujoEntrada = null;
        PrintWriter flujoSalida = null;

        try {
            flujoEntrada = new BufferedReader(new FileReader("xanadu.txt"));
            flujoSalida = new PrintWriter(new FileWriter("salidacaracteres.txt"));

            String l;
            while ((l = flujoEntrada.readLine()) != null) {
                flujoSalida.println(l);
            }
        } finally {
            if (flujoEntrada != null) {
                flujoEntrada.close();
            }
            if (flujoSalida != null) {
                flujoSalida.close();
            }
        }
    }
}

Invocar readLine devuelve una línea de texto con la línea. CopiaLineas escribe cada línea usando println, la cual añade el terminador de línea para el sistema operativo actual. Éste podría no ser el mismo terminador de línea que el que fue usado en el fichero de entrada.

Hay muchas formas de estructurar la entrada de texto y salida más allá de caracteres y líneas. Para obtener más información, vean Escaneando y Formateando.
