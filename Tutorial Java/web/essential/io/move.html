Moviendo un Fichero o Directorio

Puede mover un fichero o directorio usando el método move(Path, Path, CopyOption...). El mover falla si el fichero destino ya existe, a menos que la opción REPLACE_EXISTING esté especificada.

Los directorios vacíos pueden ser movidos. Si el directorio no está vacío, el moverlo estará permitido cuando el directorio puede ser movido sin mover los contenidos de ese directorio. En sistemas UNIX, mover un directorio dentro de la misma partición generalmente consiste en renombrar el directorio. En esa situación, este método funciona incluso cuando el directorio contiene ficheros.

Este método toma un argumento varargs – las siguientes enumeraciones StandardCopyOption están soportadas:

  • REPLACE_EXISTING – Realiza el movimiento incluso cuando el fichero de destino aún existe. Si el destino es un enlace simbólico, el enlace simbólico es reemplazado pero a lo que apunta no es afectado.
  • ATOMIC_MOVE – Realiza el movimiento como una operación atómica de ficheros. Si el sistema de ficheros no soporta un movimiento atómico, se lanza una excepción. Con un ATOMIC_MOVE puede mover un fichero en un directorio y estar seguro de que cualquier proceso monitorizando el directorio accede al fichero completo.

Lo siguiente muestra cómo usar el método move: 

import static java.nio.file.StandardCopyOption.*;
...
Files.move(source, target, REPLACE_EXISTING);

Aunque usted puede implementar el método move sobre un directorio único como se mostró, el método es usado más a menudo con el mecanismo de recursión del arbol de ficheros. Para más información, vea Recorriendo el Árbol de Fichero.
