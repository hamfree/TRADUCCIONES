Copiando un Fichero o Directorio

Puede copiar un fichero o directorio usando el método copy(Path, Path, CopyOption...). La copia fallará si el fichero destino ya existe, a menos que la opción REPLACE_EXISTING sea especificada.

Los directorios pueden ser copiados. Sin embargo, los ficheros dentro del directorio no son copiados, así que el nuevo directorio está vacío incluso cuando el directorio original contiene ficheros.

Cuando copia un enlace simbólico, el destino del enlace es copiado. Si quiere copiar el enlace en sí mismo, y no el contenido del enlace, especifique la opción NOFOLLOW_LINKS o REPLACE_EXISTING.

Este método toma un argumento varargs. Las enumeraciones siguientes StandardCopyOption y LinkOption están soportadas:

  • REPLACE_EXISTING – Realiza la copia incluso cuando el fichero de destino aún existe. Si el destino es un enlace simbólico, el enlace en sí mosmo es copiado (y no el objetivo del enlace). Si el destino es un directorio no vacío, la copia falla con la excepción FileAlreadyExistsException.
  • COPY_ATTRIBUTES – Copia los atributos de fichero asociados con el fichero al fichero destino. Los atributos de ficheros exactos soportados son dependientes del sistema de ficheros y la plataforma, pero last-modified-time es soportado a través de todas las plataformas y es copiado al fichero destino.
  • NOFOLLOW_LINKS – Indica que los enlaces simbólicos no serán seguidos. Si el fichero a ser copiado es un enlace simbólico, el enlace es copiado (y no el destino del enlace).

Si no está familiarizado con las enumeraciones, vea Tipos Enum.

Lo siguiente muestra cómo usar el método copy: 

import static java.nio.file.StandardCopyOption.*;
...
Files.copy(source, target, REPLACE_EXISTING);

Además de la copia de ficheros, la clase Files también define métodos que pueden ser usados para copiar entre un fichero y un flujo. El método copy(InputStream, Path, CopyOptions...) puede ser usado para copiar todos los bytes de un flujo de entrada a un fichero. El método copy(Path, OutputStream) puede ser usado para copiar todos los byts de un fichero a un flujo de salida.

El ejemplo Copia usa los métodos copy y Files.walkFileTree para soportar una copia recursiva. Vea Recorriendo el Árbol de Ficheros para más información.
