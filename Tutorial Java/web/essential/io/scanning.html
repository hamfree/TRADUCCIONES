Escaneando

Los objetos de tipo Scanner son útiles para descomponer la entrada formateada en fichas (tokens, en inglés) y traducir estas fichas individuales de acuerdo con su tipo de datos.

Troceando la Entrada en Fichas

Por defecto, un scanner usa el espacio en blanco para separar las fichas. (Los caracteres de espacio en blanco incluyen los blancos, los tabuladores, y los terminadores de línea. Para ver una lista completa, refiérase a la documentación para Character.isWhitespace.) Para ver como el escaneado funciona, echemos un vistazo a EscaneaXanadu, un programa que lee las palabras individuales en xanadu.txt y las imprime, una por línea. 

import java.io.*;
import java.util.Scanner;


// fuente original: ScanXan.java
public class EscaneaXanadu {
    public static void main(String[] args) throws IOException {

        Scanner s = null;

        try {
            s = new Scanner(new BufferedReader(new FileReader("xanadu.txt")));

            while (s.hasNext()) {
                System.out.println(s.next());
            }
        } finally {
            if (s != null) {
                s.close();
            }
        }
    }
}

Note que EscaneaXanadu invoca el método close de Scanner cuando ha terminado con el objeto scanner. Aunque un scanner no es un flujo, necesita cerrarlo para indicar que ha terminado con su flujo subyacente.

La salida de EscaneaXanadu se ve como esto: 

En
Xanadu
hizo
Kubla
Khan
Un
decreto
majestuoso:
...

Para usar un separador de fichas distinto, invoque useDelimiter(), especificando una expresión regular. Por ejemplo, suponga que quería que el separador de fichas fuera una coma, opcionalmente seguida por un espacio en blanco. Usted invocará, 

s.useDelimiter(",\\s*");

Traducir Fichas Individuales

El ejemplo EscaneaXanadu trata a todas las fichas de entrada como simples valores String. Scanner también soporta fichas para todos los tipos primitivos del lenguaje Java (excepto para los char), como también para BigInteger y BigDecimal. Además, los valores numéricos pueden usar los separadores de miles. Así, en una configuración regional US , Scanner leerá correctamente la cadena "32,767" como representando un valor entero.

Tenemos que mencionar la localización, porque los separadores de miles y los símbolos de decimales son específicos de la localización regional. Así, el siguiente ejemplo no funcionará correctamente en todas las localizaciones regionales si no especificamos que scanner debería usar la localización regional US. Esto no es algo en lo que usualmente tenga que preocuparse, porque sus datos de entrada usualmente vendrán de fuentes que usan la misma localización regional que usted usa. Pero este ejemplo es parte del Tutorial de Java y es distribuido en todo el mundo.

El ejemplo EscaneaSuma lee una lista de valores double y los suma. Aquí tiene el fuente: 

import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;
import java.util.Scanner;
import java.util.Locale;

// fuente original: ScanSum.java
public class EscaneaSuma {
    public static void main(String[] args) throws IOException {

        Scanner s = null;
        double suma = 0;

        try {
            s = new Scanner(new BufferedReader(new FileReader("usnumbers.txt")));
            s.useLocale(Locale.US);

            while (s.hasNext()) {
                if (s.hasNextDouble()) {
                    suma += s.nextDouble();
                } else {
                    s.next();
                }
            }
        } finally {
            s.close();
        }

        System.out.println(suma);
    }
}

Y aquí está el el fichero de entrada de ejemplo, usnumbers.txt 

8.5
32,767
3.14159
1,000,000.1

La cadena de salida es "1032778.74159". El punto será un carácter diferente en algunas localizaciones regionales, porque System.out es un objeto PrintStream, y esa clase está establecida de forma que sobreescriba la localización regional por defecto. Podríamos sobreescribir la localización regional para el programa entero — o podríamos sólo usar el formateo, como es descrito en el siguiente tema, Formateando.
