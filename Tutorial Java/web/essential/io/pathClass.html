La Clase Path

La clase Path, presentada en la publicación de Java SE 7, es uno de los puntos de entrada primarios del paquete java.nio.file. Si su aplicación usa E/S de ficheros, querrá aprender sobre las potentes características de ésta clase.

Nota de la Versión: Si tiene código pre-JDK7 que usa java.io.File, aún puede tomar ventajas de la funcionalidad de la clase Path usando el método File.toPath . Vea Código de E/S de Ficheros Heredado para obtener más información.

Como su nombre indica, la clase Path es una representación programática de una ruta de acceso en un sistema de ficheros. Un objeto Path contiene el nombre del fichero y la lista de directorios usada para construir la ruta de acceso, y es usada para examinar, localizar, y manipular ficheros.

Una instancia Path refleja la plataforma subyacente. En el SO Solaris, un Path usa la sintaxis de Solaris (/home/joe/foo) y en Microsoft Windows, un Path usa la sintaxis de Windows (C:\home\joe\foo). Un Path no es indpendiente del sistema. No puede comparar un Path de un sistema de ficheros Solaris y esperar que coincida con un Path de un sistema de ficheros de Windows, incluso si la estructura de directorios es idéntica y ambas instancias localizan el mismo fichero relativo.

El fichero o directorio correspondiente al Path podría no existir. Puede crear una instancia de Path y manipularla de varias formas: puede añadir a ella, extraer piezas de ella, compararla con otra ruta de acceso. En el tiempo apropiado, puede usar los métodos en la clase Files para comprobar la existencia del fichero correspondiente a Path, crear el fichero, abrirlo, borrarlo, cambiar sus permisos, etcétera.
