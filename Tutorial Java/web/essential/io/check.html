Comprobando un Fichero o Directorio

Usted tiene una instancia Path que representa un fichero o directorio, pero ¿existe ese fichero en el sistema de ficheros? ¿Se puede leer? ¿Se puede escribir? ¿Es ejecutable?

Verificar la Existencia de un Fichero o Directorio

Los métodos en la clase Path son sintácticos, significa que operan sobre la instancia Path. Pero eventualmente usted tiene que acceder al sistema de ficheros para verificar que un Path particular existe, o no existe. Puede hacerlo con los métodos exists(Path, LinkOption...) y notExists(Path, LinkOption...). Note que !Files.exists(path) no es equivalente a Files.notExists(path). Cuando está comprobando la existencia de un fichero, tres resultados son posibles:

  • Se comprueba que el archivo existe.
  • Se comprueba que el archivo no existe.
  • El estado del fichero es desconocido. Este resultado puede ocurrir cuando el programa no tiene acceso al fichero.

Si tanto exists como notExists devuelven false, la existencia del fichero no puede ser verificada.

Comprobar la Accesibilidad de un Fichero

Para verificar que un programa puede acceder un fichero como lo necesita, puede usar los métodos isReadable(Path), isWritable(Path), y isExecutable(Path).

El siguiente trozo de código verifica que un fichero particular existe y que el programa tiene la capacidad de ejecutar el fichero. 

Path file = ...;
boolean isRegularExecutableFile = Files.isRegularFile(file) &
         Files.isReadable(file) & Files.isExecutable(file);
Nota: Una vez cualquiera de estos métodos terminan, no hay garantía de que el fichero pueda ser accedido. Un fallo común de seguridad en muchas aplicaciones es realizar una comprobación y entonces acceder el fichero. Para más información, use su motor de búsqueda favorito para buscar TOCTTOU (pronunciado TOCK-too).

Comprobar Si Dos Paths Encuentran el Mismo Fichero

Cuando tiene un sistema de ficheros que usa enlaces simbólicos, es posible tener dos paths diferentes que localizan el mismo fichero. El método isSameFile(Path, Path) compara dos paths para determinar si localizan el mismo fichero en el sistema de ficheros. Por ejemplo: 

Path p1 = ...;
Path p2 = ...;

if (Files.isSameFile(p1, p2)) {
    // Lógica cuando los paths localizan el mismo fichero
}
