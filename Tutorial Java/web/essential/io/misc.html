Otros Métodos Útiles

Unos pocos métodos útiles no encajan en ninguna parte en esta lección y son cubiertos aquí. Esta sección cubre los siguientes:

  • Determinar el Tipo MIME
  • Sistema de Archivos Predeterminado
  • Separador de Cadenas de Rutas de Acceso
  • Volúmenes de Fichero del Sistema de Archivos

Determinar el Tipo MIME

Para determinar el tipo MIME de un fichero, podría encontrar el método probeContentType(Path) útil. Por ejemplo: 

try {
    String tipo = Files.probeContentType(nombrefichero);
    if (tipo == null) {
        System.err.format("'%s' tiene un" + " tipo de fichero desconocido.%n", nombrefichero);
    } else if (!tipo.equals("text/plain") {
        System.err.format("'%s' no es" + " un fichero de texto plano.%n", nombrefichero);
        continue;
    }
} catch (IOException x) {
    System.err.println(x);
}

Note que probeContentType devuelve null si el tipo de contenido no pudo ser determinado.

La implementación de este método es altamente específico de la plataforma y no es infalible. El tipo de contenido es determinado por el detector por defecto de tipos de fichero de la plataforma. Por ejemplo, si el detector determina que el tipo de contenido de un fichero es application/x-java basado en la extensión .class, podría ser engañado.

Puede establecer un FileTypeDetector personalizado si el por defecto no es suficiente para sus necesidades.

El ejemplo Email usa el método probeContentType.

Sistema de Archivos Predeterminado

Para recuperar el sistema de ficheros por defecto, use el método getDefault . Típicamente, este método FileSystems (no te el plural) está encadenado a uno de los métodos FileSystem (note el singular), como sigue: 

PathMatcher matcher =
    FileSystems.getDefault().getPathMatcher("glob:*.*");

Separador de Cadenas de Rutas de Acceso

El separador de rutas de acceso para los sistemas de ficheros POSIX es la barra inclinada, /, y para Microsoft Windows es la barra invertida, \. Otros sistemas de ficheros podrían usar otros delimitadores. Para recuperar el separador de Path para el sistema de ficheros por defecto, puede usar uno de los siguientes enfoques: 

String separador = File.separator;
String separador = FileSystems.getDefault().getSeparator();

El método getSeparator es también usado para recuperar el separador de rutas de acceso para cualquier sistema de ficheros.

Volúmenes de Fichero del Sistema de Archivos

Un sistema de ficheros tiene uno o más almacenes de fichero para mantener sus ficheros y directorios. El almacén de fichero representa el dispositivo de almacenamiento subyacente. En los sistemas operativos UNIX, cada sistema de ficheros montado es representado por un almacén de fichero. En Microsoft Windows, cada volumen es representado por un almacén de fichero: C:, D:, etcétera.

Para recuperar una lista de todos los almacenes de ficheros para el sistema de ficheros, puede usar el método getFileStores . Este método devuelve un Iterable, que le permite usar la sentencia for mejorada para iterar sobre todos los directorios raíz. 

for (FileStore store: FileSystems.getDefault().getFileStores()) {
   ...
}

Si quiere recuperar el almacén de ficheros donde un fichero particular está localizado, use el método getFileStore en la clase Files, como sigue: 

Path file = ...;
FileStore store= Files.getFileStore(file);

El ejemplo UsoDelDisco usa el método getFileStores.
