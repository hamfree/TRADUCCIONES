E/S de Ficheros (caracterizando NIO.2)

Nota: Este tutorial refleja el mecanismo de E/S de ficheros introducido en la publicación del KDJ 7. La versión de Java SE 6 del tutorial de Ficheros E/S era breve, pero puede descargar la versión Java SE Tutorial 2008-03-14 del tutorial el cual contiene el contenido de E/S de Ficheros anterior.

El paquete java.nio.file y su paquete relacionado, java.nio.file.attribute, ofrece soporte comprensivo para la E/S de ficheros y para acceder al sistema de ficheros por defecto. Aunque la IPA tiene muchas clases, usted necesita enfocarse en sólo unos pocos puntos de entrada. Usted verá que este IPA es muy intuitivo y fácil de usar.

El tutorial empieza preguntando ¿Qué es un path? Después, la Clase Path, el punto de entrada primario en el paquete, es presentada. Los métodos en la clase Path relacionados con operaciones sintácticas son explicados. El tutorial entonces se mueve a la otra clase primaria en el paquete, la clase Files, la cual contiene métodos que tratan con operaciones de ficheros. Primero, algunos conceptos comunes a muchas operaciones de ficheros son presentados. El tutorial entonces cubre los métodos para comprobar, borrar, copiar, y mover ficheros.

El tutorial muestra cómo los metadatos son gestionados, antes de moverse a E/S del fichero y E/S del directorio. El Acceso aleatorio a Ficheros es explicado y los problemas específicos en los enlaces simbólicos y duros son examinados.

A continuación, algunos de los temas muy poderosos, pero más avanzados, son cubiertos. Primero, la capacidad de recorrer recursivamente el árbol de ficheros es demostrada, seguida por la información sobre cómo buscar ficheros usando comodines. Lo siguiente, cómo monitorizar un directorio por cambios es explicado y demostrado. Entonces, los métodos que no encajan en ninguna parte son a los que se da alguna atención.

Finalmente, si usted tiene código de E/S de ficheros escrito anterior a la publicación de Java SE 8, hay un mapeo de la IPA vieja a la nueva IPA, como también información importante sobre el método File.toPath para desarrolladores a los que le gustaría aprovechar la nueva IPA sin volver a escribir el código existente.
