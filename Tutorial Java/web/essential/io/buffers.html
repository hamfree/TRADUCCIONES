Flujos Cacheados

La mayoría de los ejemplos que hemos visto hasta ahora usan E/s sin cachear. Esto signfica que cada petición de lectura o escritura es manejada directamente por el SO subyacente. Esto puede hacer a un programa mucho menos eficiente, ya que cada petición a menudo dispara el acceso a disco, actividad de red, o alguna otra operación que es relativamente costosa.

Para reducir esta clase de sobrecarga, la plataforma Java implementa los flujos de E/S cacheados. Los flujos de entrada cacheados leen datos desde una área de memoria conocida como buffer; La IPA de la entrada nativa es llamada sólo cuando el buffer está vacío. Similarmente, los flujos de salida cacheados escriben los datos a un buffer, y la IPA de salida nativo es llamado sólo cuando el buffer está lleno.

Un programa puede convertir un flujo sin cachear en un flujo cacheado usando el idioma de envolvimiento que hemos usado varias veces ahora, donde el objeto del flujo sin cachear es pasado al constructor de una clase de flujo cacheado. Aquí tiene cómo podría modificar las invocaciones al constructor en el ejemplo CopiaCaracteres para usar E/S cacheada: 

flujoEntrada = new BufferedReader(new FileReader("xanadu.txt"));
flujoSalida = new BufferedWriter(new FileWriter("caracteresdesalida.txt"));

Hay cuatro clases de flujo cacheado usadas para envolver flujos sin cachear: BufferedInputStream y BufferedOutputStream crean flujos de bytes cacheados, mientras que BufferedReader y BufferedWriter crean flujos de caracteres cacheados.

Vaciar Flujos Cacheados

A menudo tiene sentido escribir un buffer en puntos críticos, sin esperar a que se llene. Esto es conocido como vaciar (flushing en inglés) el buffer.

Algunas clases cacheadas de salida soportan el autovaciado, especificado por un argumento opcional en el constructor. Cuando se habilita el autovaciado, ciertos eventos clave causan que el buffer sea vaciado. Por ejemplo, un objeto PrintWriter con autovaciado vacia el buffer en cada invocación de println o format. Vea Formateando para saber mas de estos métodos.

Para vaciar un flujo manualmente, invoque su método flush. El método flush es váĺido en cualquier flujo de salida, pero no tiene efecto a menos que el flujo esté cacheado.
