Métodos de la Clase PatternSyntaxException

Una PatternSyntaxException es una excepción no comprobada que indica un error de sintaxis en un patrón de expresión regular. La clase PatternSyntaxException describe los siguientes métodos para ayudarles a determinar que fue mal:

  • public String getDescription(): Recupera la descripción del error.
  • public int getIndex(): Recupera el índice de error.
  • public String getPattern(): Recupera el patrón con la expresión regular errónea.
  • public String getMessage(): Devuelve una cadena multi-línea conteniendo la descripción del error de sintaxis y su índice, el patrón con la expresión regular errónea, y una indicación visual del índice de error dentro del patrón.

El siguiente código fuente, PruebasExpReg2.java, actualiza nuestra prueba para comprobar si hay expresiones regulares mal formadas: 


import java.io.Console;
import java.util.regex.Pattern;
import java.util.regex.Matcher;
import java.util.regex.PatternSyntaxException;

public class PruebasExpReg2 {

    public static void main(String[] args){
        Pattern patron = null;
        Matcher analizador = null;

        Console consola = System.console();
        if (consola == null) {
            System.err.println("Sin consola.");
            System.exit(1);
        }
        while (true) {
            try{
                patron =
                Pattern.compile(consola.readLine("%nIntroduza su expresión regular: "));

                analizador =
                patron.matcher(consola.readLine("Introduzca la cadena de entrada en la que buscar: "));
            }
            catch(PatternSyntaxException pse){
                consola.format("¡Hay un problema" +
                               " con la expresión regular!%n");
                consola.format("El patrón en cuestión es: %s%n",
                               pse.getPattern());
                consola.format("La descripción es : %s%n",
                               pse.getDescription());
                consola.format("El mensaje es: %s%n",
                               pse.getMessage());
                consola.format("El índice es: %s%n",
                               pse.getIndex());
                System.exit(0);
            }
            boolean found = false;
            while (analizador.find()) {
                consola.format("He encontrado el texto" +
                    " \"%s\" empezando en el índice  " +
                    "%d y terminando en el índice %d.%n",
                    analizador.group(),
                    analizador.start(),
                    analizador.end());
                found = true;
            }
            if(!found){
                consola.format("No se encontraron coincidencias.%n");
            }
        }
    }
}

Para ejecutar esta prueba, introduzca ?i)foo como la expresión regular. Este error es un escenario común en el cual el programador ha olvidado los paréntesis de apertura en la expresión de bandera embebida (?i). Hacer esto producirá lo siguiente: 

Introduza su expresión regular: ?i)
¡Hay un problema con la expresión regular!
El patrón en cuestión es: ?i)
La descripción es : Dangling meta character '?'
El mensaje es: Dangling meta character '?' near index 0
?i)
^
El índice es: 0

Desde esta salida, podemos ver que el error de sintaxis es un metacarácter colgante (el signo de interrogación) en el índice 0. Un parénteis de apertura perdido es el culpable.
