Introducción

¿Qué Son las Expresiones Regulares?

Las Expresiones Regulares son una forma de describir un conjunto de cadenas basadas en características comunes compartidas por cada cadena en el conjunto. Pueden ser usadas para buscar, editar, o manipular texto y datos. Debe aprender una sintaxis especial para crear expresiones regulares — una que va más allá de la sintaxis normal del lenguaje de programación Java. Las expresiones regulares varían en complejidad, pero una vez entienda lo básico de cómo son construídas, será capaz de descifrar (o crear) cualquier expresión regular.

Este recorrido le enseña la sintaxis de expresión regular soportada por la IPA java.util.regex y presenta varios ejemplos funcionales para ilustrar cómo interactáun los diversos objetos. En el mundo de las expresiones regulares, hay muchas diferentes formas de hacer para elegir, tales como grep, Perl, Tcl, Python, y awk. La sintaxis de expresión regular en la IPA de java.util.regex es más similar a la que se encuentra en Perl.

¿Cómo Se Representan las Expresiones Regulares en Este Paquete?

El paquete java.util.regex consiste primariamente de tres clases: Pattern, Matcher, y PatternSyntaxException.

  • Un objeto Pattern es una representación compilada de una expresión regular. La clase Pattern no ofrece constructores públicos. Para crear un patrón, debe primero invocar a uno de sus métodos public static compile , los cuales entonces retornarán un objeto Pattern. Estos métodos aceptan una expresión regular como el primer argumento; las primeras lecciones en este recorrido le enseñarán la sintaxis requerida.
  • Un objeto Matcher es el motor que interpreta el patrón y realiza las operaciones de coincidencia contra una cadena de entrada. Como la clase Pattern , Matcher no define constructores públicos. Obtiene un objeto Matcher invocando el método matcher en un objeto Pattern.
  • Un objeto PatternSyntaxException es una excepción no comprobada que indica un error de sintaxis en un patrón de expresión regular.
