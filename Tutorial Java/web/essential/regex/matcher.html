Métodos de la Clase Matcher

Esta sección describe algunos métodos útiles adicionales de la clase Matcher. Por convenencia, los métodos listados abajo están agrupados de acuerdo a la funcionalidad.

Métodos de Índice

Los Métodos de índice ofrecen valores útiles de índice que muestran precisamente dónde fue encontrada la coincidencia en la cadena de entrada:

  • public int start(): Devuelve el índice de inicio de la coincidencia anterior.
  • public int start(int group): Devuelve el índice de inicio de la subsecuencia capturada por el grupo dado durante la operación previa de correspondencia.
  • public int end(): Devuelve el desplazamiento después del último carácter coincidente.
  • public int end(int group): Devuelve el desplazamiento después del último carácter de la subsecuencia capturada por el grupo dado durante la operación previa de correspondencia.

Métodos de Estudio

Los Métodos de estudio revisan la cadena de entrada y devuelven un booleano indicando si se encontró o no el patrón.

  • public boolean lookingAt(): Intenta hacer coincidir la secuencia de entrada, empezando al principio de la región, contra el patrón.
  • public boolean find(): Intenta encontrar la siguiente subsecuencia de la secuencia de entrada que coincida con el patrón.
  • public boolean find(int start): Restablece este matcher y luego intenta encontrar la subsecuencia siguiente de la secuencia de entrada que coincide con el patrón, comenzando en el índice especificado.
  • public boolean matches(): Intenta hacer coincidir la región entera contra el patrón.

Métodos de reemplazo

Los métodos de reemplazo son métodos útiles para reemplazar texto en una cadena de entrada.

  • public Matcher appendReplacement(StringBuffer sb, String replacement): Implementa un paso añadir-y-reemplazar no-terminal.
  • public StringBuffer appendTail(StringBuffer sb): Implementa un paso añadir-y-reemplazar terminal.
  • public String replaceAll(String replacement): Reemplaza cualquier subsecuencia de la secuencia de entrada que coincice con el patrón con la cadena de reemplazo dada.
  • public String replaceFirst(String replacement): Reemplaza la primera subsecuencia de la secuencia de entrada que coincide con el patrón con la cadena de reemplazo dada.
  • public static String quoteReplacement(String s): Devuelve un literal de reemplazo String para la String especificada. Este método produce una String que funcionará como un reemplazo literal s en el método appendReplacement de la clase Matcher. La String producida coincidirá con la secuencia de caracteres en s tratada como una secuencia literal. Las barras ('\') y los signos de dólar ('$') no tendrán un significado especial.

Usar los Métodos start y end

Aquí hay un ejemplo, DemoMatcher.java, que cuenta el numero de veces que la palabra "dog" aparece en la cadena de entrada. 


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class DemoMatcher {

    private static final String EXPREG =
        "\\bdog\\b";
    private static final String ENTRADA =
        "dog dog dog doggie dogg";

    public static void main(String[] args) {
       Pattern p = Pattern.compile(EXPREG);
       //  obtenemos un objeto matcher
       Matcher m = p.matcher(ENTRADA);
       int count = 0;
       while(m.find()) {
           count++;
           System.out.println("Coincide "
                              + count + " veces.");
           System.out.println("start(): "
                              + m.start());
           System.out.println("end(): "
                              + m.end());
      }
   }
}

                    
SALIDA:

Coincide 1 veces.
start(): 0
end(): 3
Coincide 2 veces.
start(): 4
end(): 7
Coincide 3 veces.
start(): 8
end(): 11

Puede ver que este ejemplo usa límites de palabra para asegurar que las letras "d" "o" "g" no son meramente una subcadena en una palabra más larga. También da algo de información útil sobre dónde la coincidencia ha ocurrido en la cadena de entrada. El método start devuelve el índice de inicio de la subsecuencia capturada por el grupo dado durante la operación de coincidencia previa, y end devuelve el índice del último carácter que coincidió, más uno.

Usar los Métodos matches y lookingAt

Los métodos matches y lookingAt intentan coincidir una secuencia de entrada contra un patrón. La diferencia, sin embargo, es que matches requiere que la secuencia de entrada entera coincida, mientras que lookingAt no. Ambos métodos siempre empiezan al principio de la cadena de entrada. Aquí tiene el código completo, BuscandoCoincidencias.java: 


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class BuscandoCoincidencias {

    private static final String EXPREG = "foo";
    private static final String ENTRADA =
        "fooooooooooooooooo";
    private static Pattern patron;
    private static Matcher analizador;

    public static void main(String[] args) {

        // Inicializamos
        patron = Pattern.compile(EXPREG);
        analizador = patron.matcher(ENTRADA);

        System.out.println("EXPREG actual es : "
                           + EXPREG);
        System.out.println("ENTRADA actual es: "
                           + ENTRADA);

        System.out.println("lookingAt(): "
            + analizador.lookingAt());
        System.out.println("matches(): "
            + analizador.matches());
    }
}

                    
EXPREG actual es : foo
ENTRADA actual es: fooooooooooooooooo
lookingAt(): true
matches(): false

Usar replaceFirst(String) y replaceAll(String)

Los métodos replaceFirst y replaceAll reemplazan el texto que coincide con una expresión regular dada. Como sus nombres indican, replaceFirst reemplaza la primera ocurrencia, y replaceAll reemplaza todas las ocurrencias. Aquí está el código de DemoReemplazo.java: 


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class DemoReemplazo {

    private static String EXPREG = "perro";
    private static String ENTRADA =
        "El perro dice miau. Todos los perros dicen miau.";
    private static String REPLACE = "gato";

    public static void main(String[] args) {
        Pattern p = Pattern.compile(EXPREG);
        // obtenemos un objeto matcher
        Matcher m = p.matcher(ENTRADA);
        ENTRADA = m.replaceAll(REPLACE);
        System.out.println(ENTRADA);
    }
}

                    
SALIDA: El gato dice miau. Todos los gatos dicen miau.

En esta primera versión, todas las ocurrencias de perro son reemplazadas con gato. Pero ¿porqué parar aquí? En vez de reemplazar un literal simple como perro, puede reemplazar texto que coincida con cualquier expresión regular. La IPA para este método indica que "dada la expresión regular a*b, la entrada aabfooaabfooabfoob, y la cadena de reemplazo -, una invocación de este método en un matcher para esa expresión generaría la cadena -foo-foo-foo-."

Aquí está el código de Here's the DemoReemplazo2.java: 


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class DemoReemplazo2 {

    private static String EXPREG = "a*b";
    private static String ENTRADA =
        "aabfooaabfooabfoob";
    private static String REEMPLAZO = "-";

    public static void main(String[] args) {
        Pattern p = Pattern.compile(EXPREG);
        // obtenemos un objeto Matcher.
        Matcher m = p.matcher(ENTRADA);
        ENTRADA = m.replaceAll(REEMPLAZO);
        System.out.println(ENTRADA);
    }
}

                    
SALIDA: -foo-foo-foo-

Para reemplazar sólo la primera ocurrencia del patrón, simplemente llame a replaceFirst en vez de a replaceAll. Acepta el mismo parámetro.

Usar appendReplacement(StringBuffer,String) y appendTail(StringBuffer)

La clase Matcher también ofrece los métodos appendReplacement y appendTail para el reemplazo de texto. El siguiente ejemplo, DemoRegex.java, usa estos dos métodos para conseguir el mismo efecto que replaceAll. 


import java.util.regex.Pattern;
import java.util.regex.Matcher;

// fuente original: RegexDemo.java
public class DemoRegex {

    private static String EXPREG = "a*b";
    private static String ENTRADA = "aabfooaabfooabfoob";
    private static String REEMPLAZO = "-";

    public static void main(String[] args) {
        Pattern p = Pattern.compile(EXPREG);
        Matcher m = p.matcher(ENTRADA); // obtenemos un objeto matcher
        StringBuffer sb = new StringBuffer();
        while(m.find()){
            m.appendReplacement(sb,REEMPLAZO);
        }
        m.appendTail(sb);
        System.out.println(sb.toString());
    }
}


                    
SALIDA: -foo-foo-foo-

Métodos de Matcher Equivalentes en java.lang.String

Por conveniencia, la clase String imita un par de métodos de Matcher también:

  • public String replaceFirst(String regex, String replacement): Reemplaza la primera subcadena de esta cadena que coincide con la expresión regular dada con el reemplazo dado. Una invocación de este método de la forma str.replaceFirst(regex, repl) produce exactamente el mismo resultado que la expresión Pattern.compile(regex).matcher(str).replaceFirst(repl)
  • public String replaceAll(String regex, String replacement): Reemplaza cada subcadena de esta cadena que coincida con la expresión regular dada con el reemplazo dado. Una invocación de este método de la forma str.replaceAll(regex, repl) produce exactamente el mismo resultado que la expresión Pattern.compile(regex).matcher(str).replaceAll(repl)
