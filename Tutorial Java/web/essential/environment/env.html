Variables de Entorno

Muchos sistemas operativos usan variables de entorno para pasar información de configuración a las aplicaciones. Como las propiedades en la plataforma Java, las variables de entorno son pares clave/valor, donde tanto la clave como el valor son cadenas. Las convenciones para establecer y usar variables de entorno varían entre sismteas operativos, y también entre intérpretes de línea de comandos. Para aprender a cómo pasar variables de entorno a aplicaciones en su sistema, refiérase a su documentación del sistema.

Preguntar por las Variables de Entorno

En la plataforma Java, una aplicación usa System.getenv para recuerpar valores de variables de entorno. Sin un argumento, getenv devuelve una instancia de sólo-lectura de java.util.Map, donde las claves del mapa son los nombres de las variables de entorno, y los valores del mapa son los valores de las variables de entorno. Esto está demostrado en el ejemplo EnvMap: 


import java.util.Map;

public class EnvMap {
    public static void main (String[] args) {
        Map<String, String> env = System.getenv();
        for (String envName : env.keySet()) {
            System.out.format("%s=%s%n",
                              envName,
                              env.get(envName));
        }
    }
}

Con un argumento String, getenv devuelve el valor de la variable especificada. Si la variable no está definida, getenv devuelve null. El ejemplo Env usa getenv de esta forma para preguntar por variables de entorno específicas, especificadas en la línea de comandos: 


public class Env {
    public static void main (String[] args) {
        for (String env: args) {
            String value = System.getenv(env);
            if (value != null) {
                System.out.format("%s=%s%n",
                                  env, value);
            } else {
                System.out.format("%s is"
                    + " not assigned.%n", env);
            }
        }
    }
}

Pasar Variables de Entorno a Nuevos Procesos

Cuando una aplicación Java usa un objeto ProcessBuilder para crear un proceso nuevo, el conjunto de variables de entorno por defecto pasado al proceso nuevo es el mismo conjunto ofrecido al proceso de la aplicación de la máquina virtual. La aplicación puede cambiar este conjunto usando ProcessBuilder.environment.

Problemas de dependencia de la plataforma

Hay muchas sutiles diferencias entre la forma en que las variables de entorno son implementadas en sistemas diferentes. Por ejemplo, Windows ignora el caso en los nombres de variables de entorno, mientras que UNIX no lo hace. La forma que las variables de entorno son usadas también varían. Por ejemplo, Windows ofrece el nombre del usuario en una variable de entorno llamada USERNAME, mientas que las implementaciones de UNIX puede proporcionar el nombre de usuario en USER, LOGNAME, o ambos.

Para maximizar la portabilidad, nuca se refiera a una variable de entorno cuando el mismo valor está disponible en un propiedad del sistema. Por ejemplo, si el sistema operaivo ofrece un nombre de usuario, siempre estar´ disponible en la propiedad del sistema user.name.
