Propiedades del Sistema

En Propiedades, examinamos la forma en que una aplicación puede usar los objetos Properties para mantener su configuración. La plataforma Java por sí misma usa un objeto Properties para mantener su propia configuración. La clase System mantiene un objeto Properties que describe la configuración del entorno de trabajo actual. Las propiedades del sistema incluye información sobre el usuario actual, la versión actual del sistema en tiempo de ejecución Java, y el carácter usado para separar los componentes de un nombre de ruta de archivo.

La siguiente tabla describe algunas de las más importantes propiedades del sistema

Propiedades del sistema importantes
Clave Significado
"file.separator" Carácter que separa los componentes de una ruta de ficheros. Esta es "/" en UNIX y "\" en Windows.
"java.class.path" La ruta de ficheros usada para encontrar directorios y archivos JAR conteniendo ficheros de clases. Los elementos de la ruta de clases están separados por un carácter específico-de-la-plataforma especificada en la propiedad path.separator.
"java.home" Directorio de instalación para el Entorno de Ejecución Java (JRE, en inglés)
"java.vendor" Nombre del proveedor del EEJ
"java.vendor.url" URL del proveedor del EEJ
"java.version" Número de versión del EEJ
"line.separator" Secuencia usada por el sistema operativo para separar las líneas en los ficheros de texto
"os.arch" Arquitectura del sistema operativo
"os.name" Nombre del sistema operativo
"os.version" Versión del sistema operativo
"path.separator" Caracter separador del Path usado en java.class.path
"user.dir" Directorio de trabajo del usuario
"user.home" Directorio inicial del usuario
"user.name" Nombre de la cuenta del usuario
Consideración de Seguridad:  El acceso a las propiedades del sistema puede ser restringido por el Gestor de Seguridad. Esto es más a menudo un problema en los applets, los cuales no pueden leer algunas propiedades del sistema, y de escribir cualquier propiedad del sistema. Para obtener más información sobre el acceso a las propiedades del sistema en los apllets, refíerase a Propiedades del Sistema en la lección Haciendo Más con Aplicaciones Java Ricas de Internet.

Leer Propiedades del Sistema

La clase System tiene dos métodos usados para leer las propiedades del sistema: getProperty y getProperties.

La clase System tiene dos versiones diferentes de getProperty. Ambos recuperan el valor de la propiedad nombrada en la lista de argumentos. La más simple de dos métodos getProperty toma un único argumento, una clave de propiedad. Por ejemplo, para obtener el valor de path.separator, usa la siguiente sentencia: 

System.getProperty("path.separator");

El método getProperty devuelve una cadena conteniendo el valor de la propiedad. si la propiedad no existe, esta versión de getProperty devuelve null.

La otra versión de getProperty requiere dos argumentos String: el primer argumento es la clave a buscar y el segundo argumento es un valor por defecto para devolver si la clave no puede ser encontrada o si no tiene valor. Por ejemplo, la siguiente invocación de getProperty busca la propiedad de System llamada mensaje.subliminal. Ésta no es una propiedad del sistema válida, así que en vez de devolver null, este método devuelve el valor por defecto ofrecido como segundo argumento: "¡Compre Marshmallows de StayPuft!" 

System.getProperty("mensaje.subliminal", "¡Compre Marshmallows de StayPuft!");

El último método ofrecido por la clase System para acceder a los valores de la propiedad es el método getProperties, el cual devuelve un objeto Properties . Este objeto contiene un conjunto completo de definiciones de propiedades del sistema.

Escribir Propiedades del Sistema

Para modificar el conjunto existente de propiedades del sistema, use System.setProperties. Este método toma un objeto Properties que ha sido inicializado para contener las propiedades a ser establecidas. Este método reemplaza el conjunto entero de propiedades del sistema con el nuevo conjunto representado por el objeto Properties.

Advertencia: Cambiar las propiedades del sistema es potencialmente peligroso y debería ser hecho con discrección. Muchas propiedades del sistema no son releídas después del inicio y están ahí con propósitos informativos. Cambiar algunas propiedades puede tener efectos-laterales no esperados.

El siguiente ejemplo, TestPropiedades, crea un objeto Properties y lo inicializa desde misPropiedades.txt . 

mensaje.subliminal=¡Compre Marshmallows de StayPuft!

TestPropiedades entonces usa System.setProperties para instalar los nuevos objetos Properties como el conjunto actual de propiedades del sistema. 


import java.io.FileInputStream;
import java.util.Properties;

public class TestPropiedades {
    public static void main(String[] args)
        throws Exception {

        // configura un nuevo objeto properties
        // desde el fichero "misPropiedades.txt"
        FileInputStream ficheroProp =
            new FileInputStream( "misPropiedades.txt");
        Properties p =
            new Properties(System.getProperties());
        p.load(ficheroProp);

        // establece las propiedades del sistema
        System.setProperties(p);
        // muestra las nuevas propiedades
        System.getProperties().list(System.out);
    }
}

Note cómo TestPropiedades crea el objeto Properties, p, el cual es usado como el argumento para setProperties: 

Properties p = new Properties(System.getProperties());

Esta sentencia inicializa el nuevo objeto properties, p, con el conjunto actual de propiedades del sistema, el cual en el caso de esta pequeña aplicación, es el cojunto de propiedades inicializadas por el sistema en tiempo de ejecución. Entonces la aplicación carga propiedades adicionales en p desde el fichero misPropiedades.txt y estable las propiedades del sistema a p. Esto tiene el efecto de añadir las propiedades listada en misPropiedades.txt al conjunto de propiedades creado por el sistema en tiempo de ejecución en el arranque. Note que una aplicación puede crear p sin ningún objeto por defecto Properties, como esto: 

Properties p = new Properties();

!Note también que el valor de las propiedades del sistema pueden ser sobreescritos! Por ejemplo, si misPropiedades.txt contiene la siguiente línea, la propiedad del sistena java.vendor será sobreescrita: 

java.vendor=Compañía de Software Acme

En general, sea cuidadoso para no sobreeescribir propiedades del sistema.

El método setProperties cambia el conjunto de propiedades del sistema para la aplicación que se está ejecutando actualmente. Estos cambios no son persistentes. Estos es, cambiar las propiedades del sistema en un aplicación no afectará a las invocaciones futuras del interprete de Java para ésta u otra aplicación. El sistema en tiempo de ejecución re-inicializa las propiedades del sismtea cada vez que se inicia. Si los cambios a las propiedades de sistema tienen que ser persistentes, entonces las aplicación debe escribir los valores a algún fichero antes de salir y leerlos de nuevo en su arranque.
