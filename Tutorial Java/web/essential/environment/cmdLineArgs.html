Argumentos de la Línea de Comandos

Una aplicación Java puede aceptar cualquier número de argumentos desde la línea de comandos. Esto permite al usuario especificar información de configuración cuando la aplicación es ejecutada.

El usuario introduce argumentos de línea de comandos cuando invoca la aplicación y los especifica después del nombre de la clase a ser ejecutada. Por ejemplo, suponga una aplicación Java llamada Sort que ordena líneas en un fichero. Para ordenar los datos en un fichero llamado amigos.txt, un usuario introduciría: 

java Sort amigos.txt

Cuando una aplicación es lanzada, el sistema en tiempo de ejecución pasa los argumentos de la línea de comandos al método main de la aplicación vía una matriz de Strings. En el ejemplo previo, los argumentos de línea de comandos pasados a la aplicación Sort en una matriz que contiene una única String: "amigos.txt".

Mostrar Argumentos de la Línea de Comandos

El ejemplo Echo visualiza cada uno de los argumentos de la línea de comandos en una línaa por sí mismos: 


public class Echo {
    public static void main (String[] args) {
        for (String s: args) {
            System.out.println(s);
        }
    }
}

El siguiente ejemplo muestra como un usuario podría ejecutar Echo. La entrada del usuario está en itálics. 

java Echo Bebe Java Caliente
Bebe
Java
Caliente

Note que la aplicación visualiza cada palabra — Bebe, Java, y Caliente — en una línea por sí misma. Esto es porque el carácter de espacio separa los argumentos de línea de comandos. Para que Bebe, Caliente, y Java sean interpretados como un argumento único, el usuario deberá unirlos encerrándolos entre comillas. 

java Echo "Bebe Java Caliente"
Bebe Java Caliente

Analizando Argumentos Numéricos de la Línea de Comandos

Si una aplicación necesita soportar un argumento de línea de comandos numérico, debe convertir un argumento String que representa un número, tal como "34", a un valor numérico. Aquí hay un trozo de código que convierte un argumento de línea de comandos a un int: 

int primerArg;
if (args.length > 0) {
    try {
        primerArg = Integer.parseInt(args[0]);
    } catch (NumberFormatException e) {
        System.err.println("El argumento" + args[0] + " debe ser un entero.");
        System.exit(1);
    }
}

parseInt lanza una NumberFormatException si el formato de args[0] no es válido. Todas las clases Number — Integer, Float, Double, y demás — tienen métodos parseXXX que convierten una String representando un número a un objeto de su tipo.
