Otras Utilidades de Configuración

Aquí está el sumario de algunas otras utilidades de configuración.

El IPA de Preferencias permite aplicaciones para almacenar y recuperar datos de configuración en un almacen de respaldo dependeiente-de-la-implementación. Las actualizaciones asíncronas están soportadas, y el mismo conjunto de preferencias puede ser actualizado de forma segura por hilos múltiples e incluso múltiples aplicaciones. Para obtener más información, refíerase a la Guía del IPA de Preferencias.

Una aplicación desplegada en un archivo JAR usa un manifiesto para describir el contenido del archivo. Para obtener más información, refíerase a la lección Empaquetando Programas en Ficheros JAR .

La configuración de una aplicación Java Web Start está contenida en un fichero JNLP. Para obtener más información, refíerase a la lección Java Web Start.

La configuración de un applet Java Plug-in es parcialmente determinada por las etiquetas HTML usadas para embeber el applet en la página web. Dependiendo del applet y el navegador, estas etiquetas pueden incluir <applet>, <object>, <embed>, y <param>. Para obtener más información, refíerase a la lección Applets de Java.

La clase java.util.ServiceLoader ofrece una facilidad proveedor de servicio simple. Un proveedor de servicio es una implementación de un servicio — un bien conocido conjunto de interfaces y clases (usualmente abstractas). Las clases en un proveedor de servicios típicamente implementa los interfaces y subclases que las clases definieron en el servicio. Los proveedores de servicio pueden ser instalados como extensiones (vea El Mecanismo de Extensión). Los proveedores pueden también ser hechos disponibles añadiéndolos a las rutas de clases o por algun otro medio específico de la plataforma.
