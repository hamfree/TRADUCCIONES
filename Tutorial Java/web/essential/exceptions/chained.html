Excepciones Encadenadas

Una aplicación a menudo responde a una excepción lanzando otra excepción. En efecto, la primera excepción causa la segunda excepción. Puede ser muy útil saber cuando una excepción causa otra. Las Excepciones Encadenadas ayudan al programador a hacer esto.

Lo siguiente son los métodos y constructores en Throwable que soportan las excepciones encadenadas. 

Throwable getCause()
Throwable initCause(Throwable)
Throwable(String, Throwable)
Throwable(Throwable)

El argumento Throwable en initCause y los constructores Throwable son la excepción que causó la excepción actual. getCause devuelve la excepción que causó la excepción actual, e initCause estable la causa de la excepción actual.

El siguiente ejemplo muestra cómo usar una excepción encadenada. 

try {

} catch (IOException e) {
    throw new SampleException("Otra IOException", e);
}

En este ejemplo, cuando una IOException es capturada, una nueva excepción SampleException es creada con la causa original enlazada y la cadena de excepciones es lanzada hacia arriba al siguiente manejador de excepción del nivel más alto.

Acceder a la Información de Seguimiento de Pila

Ahora supongamos que el manejador de excepción del nivel más alto quiere volcar el seguimiento de pila en su propio formato.

Definición: Un seguimiento de pila ofrece información de la historia de ejecución del hilo actual y lista los nombres de las clases y métodos que fueron llamados hasta el punto en el que la excepción ocurrió. Un seguimiento de la pila es una herramienta útil de depuración de la que normalmente nos aprovecharemos cuando una excepción ha sido lanzada.

El siguiente código muestra cómo llamar al método getStackTrace del objeto excepción. 

catch (Exception cause) {
    StackTraceElement elements[] = cause.getStackTrace();
    for (int i = 0, n = elements.length; i < n; i++) {       
        System.err.println(elements[i].getFileName()
            + ":" + elements[i].getLineNumber() 
            + ">> "
            + elements[i].getMethodName() + "()");
    }
}

IPA de Registro

El siguiente trozo de código registra dónde una excepción ocurrió desde dentro del bloque catch. Sin embargo, en vez de analizar manualmente el seguimiento de la pila y enviar la salida a System.err(), envía la salida a un fichero usando la facilidad de registro en el paquete java.util.logging . 

try {
    Handler handler = new FileHandler("OutFile.log");
    Logger.getLogger("").addHandler(handler);
    
} catch (IOException e) {
    Logger logger = Logger.getLogger("package.name"); 
    StackTraceElement elements[] = e.getStackTrace();
    for (int i = 0, n = elements.length; i < n; i++) {
        logger.log(Level.WARNING, elements[i].getMethodName());
    }
}
