El Bloque finally

El bloque finally siempre se ejecuta cuando el bloque try termina. Esto asegura que el bloque finally es ejecutado incluso si una excepción inesperada ocurre. Pero finally es útil para más que justo el manejo de la excepción — esto permite al programador evitar tener código de limpieza accidentalmente anulados por un return, continue, o break. Colocar código de limpieza en el bloque finally es siempre una buena práctica, incluso cuando no se anticipan excepciones.

Nota: Si la MVJ termina mientras el código de try o catch está siendo ejecutado, entonces el bloque finally puede no ejecutarse. Igualmente, si el hilo que ejecuta el código try o catch es interrumpido o terminado, el bloque finally puede no ejecutarse incluso aunque la aplicación como un todo continúa.

El bloque try del método escribeLista con el que usted ha estado trabajado aquí abre un PrintWriter. El programa cerrará ese flujo antes de salir del método escribirLista. Esto plantea un problema algo complicado porque el bloque try de escribeLista puede terminar de una de tres formas.

  1. La sentencia new FileWriter falla y lanza una IOException.
  2. La sentencia list.get(i) falla y lanza una IndexOutOfBoundsException.
  3. Todo funciona bien y el bloque try termina normalmente.

El sistema en tiempo de ejecución siempre ejecuta las sentencias dentro de un bloque finally independientemente de lo que ocurra con el bloque try. Así que es el lugar perfecto para limpiar.

El siguiente bloque finally para el método escribeLista limpia y entonces cierra el PrintWriter. 

finally {
    if (salida != null) { 
        System.out.println("Cerrando PrintWriter");
        salida.close(); 
    } else { 
        System.out.println("PrintWriter no abierto");
    } 
}
Importante: El bloque finally es una herramienta clave para prevenir fugas de recursos. Cuando cierra un fichero o de otra forma recuperar recursos, coloque el código en un bloque finally para asegurar que el recurso es siempre recuperado.

Considere usar la sentencia try-with-resources en estas situaciones, la cual automáticamente libera recursos del sitema cuando ya no se necesitan más. La sección La Sentencia try-with-resources tiene más información.
