La Sentencia try-with-resources

La sentencia try-with-resources es una sentecia try que declara uno o más recursos. Un recurso es un objeto que debe ser cerrado después de que el programa haya terminado con él. La sentencia try-with-resources asegura que cada recurso es cerrado al final de la sentencia. Cualquier objeto que implementa java.lang.AutoCloseable, lo cual incluye todos los objetos que implementan java.io.Closeable, pueden ser usados como un recurso.

El siguiente ejemplo lee la primera línea de un fichero. Usa una instancia de BufferedReader para leer datos desde el fichero. BufferedReader es un recurso que debe ser cerrado después de que el programa ha terminado con él: 

static String readFirstLineFromFile(String path) throws IOException {
    try (BufferedReader br =
                   new BufferedReader(new FileReader(path))) {
        return br.readLine();
    }
}

En este ejemplo, el recurso declarado en la sentencia try-with-resources es un BufferedReader. La declaración de la sentencia aparece entre paréntesis inmediatamente después de la palabra clave try. La clase BufferedReader, en Java SE 7 y posteriores, implementan la interfaz java.lang.AutoCloseable. Ya que la instancia BufferedReader es declarada en una sentencia try-with-resource, será cerrado independientemente de si la sentencia try se completa normalmente o abruptamente (como resultado del método BufferedReader.readLine lanzando una IOException).

Anteriormente a Java SE 7, podía usar un bloque finally para asegurar que un recurso está cerrado independientemente de si la sentencia try se completa normalmente o abruptamente. El siguiente ejemplo usa un bloque finally en vez de una sentencia try-with-resources: 

static String readFirstLineFromFileWithFinallyBlock(String path)
                                                     throws IOException {
    BufferedReader br = new BufferedReader(new FileReader(path));
    try {
        return br.readLine();
    } finally {
        if (br != null) br.close();
    }
}

Sin embargo, en este ejemplo, si ambos métodos readLine y close lanzan excepciones, entonces el método readFirstLineFromFileWithFinallyBlock lanza la excepción lanzada desde el bloque finally; la excepción lanzada desde el bloque try se suprime. En contraste, en el ejemplo readFirstLineFromFile, si las excepciones son lanzadas desde ambos el bloque try y la sentencia try-with-resources, entonces el método readFirstLineFromFile lanza la excepción lanzada desde el bloque try; la excepción lanzada desde el bloque try-with-resources es suprimida. En Java SE 7 y posteriores, puede recuperar excepciones suprimidas; vea la sección Excepciones Suprimidas para obtener más información.

Puede declarar uno o más recursos en una sentencia try-with-resources. El siguiente ejemplo recupera los nombres de los ficheros empaquetados en el fichero zip zipFileName y crea un fichero de texto que contiene los nombres de estos ficheros: 

public static void writeToFileZipFileContents(String zipFileName,
                                           String outputFileName)
                                           throws java.io.IOException {

    java.nio.charset.Charset charset =
         java.nio.charset.StandardCharsets.US_ASCII;
    java.nio.file.Path outputFilePath =
         java.nio.file.Paths.get(outputFileName);

    // Abre el fichero zip y crea el fichero de salida con
    // la sentencia try-with-resources

    try (
        java.util.zip.ZipFile zf =
             new java.util.zip.ZipFile(zipFileName);
        java.io.BufferedWriter writer = 
            java.nio.file.Files.newBufferedWriter(outputFilePath, charset)
    ) {
        // Enumeramos cada entrada
        for (java.util.Enumeration entries =
                                zf.entries(); entries.hasMoreElements();) {
            // Obtenemos el nombre de la entrada y lo escribimos en el fichero de salida
            String newLine = System.getProperty("line.separator");
            String zipEntryName =
                 ((java.util.zip.ZipEntry)entries.nextElement()).getName() +
                 newLine;
            writer.write(zipEntryName, 0, zipEntryName.length());
        }
    }
}

En este ejemplo, la sentencia try-with-resources contiene dos declaraciones que están separadas por un punto y coma: ZipFile y BufferedWriter. Cuando el bloque de código que directamente lo sigue termina, o normalmente o debido a una excepción, los métodos close de los objetos BufferedWriter y ZipFile son automáticamente llamados en este orden. Note que los métodos close de los recursos son llamados en el orden opuesto de su creación.

El siguiente ejemplo usa una sentencia try-with-resources para automáticamente cerrar un objeto java.sql.Statement: 

public static void viewTable(Connection con) throws SQLException {

    String query = "select COF_NAME, SUP_ID, PRICE, SALES, TOTAL from COFFEES";

    try (Statement stmt = con.createStatement()) {
        ResultSet rs = stmt.executeQuery(query);

        while (rs.next()) {
            String coffeeName = rs.getString("COF_NAME");
            int supplierID = rs.getInt("SUP_ID");
            float price = rs.getFloat("PRICE");
            int sales = rs.getInt("SALES");
            int total = rs.getInt("TOTAL");

            System.out.println(coffeeName + ", " + supplierID + ", " + 
                               price + ", " + sales + ", " + total);
        }
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    }
}

El recurso java.sql.Statement usado en este ejemplo es parte de JDBC 4.1 y IPA posteriores.

Nota: Una sentencia try-with-resources puede tener bloques catch y finally justo como una ordinaria sentencia try. En una sentencia try-with-resources, cualquier bloque catch o finally es ejecutado después de que los recursos declarados han sido cerrados.

Excepciones Suprimidas

Una excepción puede ser lanzada desde el bloque de código asociado con la sentencia try-with-resources. En el ejemplo writeToFileZipFileContents, una excepción puede ser lanzada desde el bloque try, y hasta dos excepciones pueden ser lanzadas desde la sentencia try-with-resources cuando intenta cerrar los objetos ZipFile y BufferedWriter. Si una excepción es lanzada desde el bloque try y una o más excepciones son lanzadas desde la sentencia try-with-resources, entonces esas excepciones lanzadas desde la sentencia try-with-resources son suprimidas, y la excepción lanzada por el bloque es la que es lanzada por el método writeToFileZipFileContents. Puede recuperar esas excepciones suprimidas llamando al método Throwable.getSuppressed desde la excepción lanzada por el bloque try.

Clases Que Implementan la Interfaz AutoCloseable o Closeable

Vea el Javadoc de las interfaces AutoCloseable y Closeable para una lista de clases que implementan una u otra de estas interfaces. El interfaz Closeable extiende el interfaz AutoCloseable. El método close de la interfaz Closeable lanza excepciones del tipo IOException mientras el método close de la interfaz AutoCloseable lanza excepciones del tipo Exception. Consecuentemente, las subclases de la interfaz AutoCloseable pueden sobreescribir este comportamiento del método close par lanzar excepciones especializadas, como una IOException, o ninguna excepción en absoluto.
