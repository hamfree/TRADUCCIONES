Especificando las Excepciones Lanzadas por un Método

La sección previa mostró como escribir un manejador de excepción para el método escribeLista en la clase ListaDeNumeros. Algunas veces, es apropiado para el código capturar las excepciones que pueden ocurrir en él. En otros casos, sin embargo, es mejor dejar a un método más arriba de la pila de llamadas que maneje la excepción. Por ejemplo, si usted estuviera proporcionando la clase ListaDeNumeros como parte de un paquete de clases, probablemente no podría anticipar la necesidad de todos los usuarios de su paquete. En este caso, es mejor no capturar la excepción y permitir que un método más arriba en la pila de llamadas la maneje.

Si el método escribeLista no captura las excepciones comprobadas que puedan ocurrir dentro de él, el método escribeLista deberá especiificar que puede lanzar esas excepciones. Modifiquemos el método original escribeLista para especificar las excepciones que puede lanzar en vez de capturarlas. Para recordarle, aquí está la versión original del método escribeLista que no compilará. 

public void escribeLista() {
    PrintWriter salida = new PrintWriter(new FileWriter("FicheroSalida.txt"));
    for (int i = 0; i < TAMAÑO; i++) {
        salida.println("Valor en: " + i + " = " + lista.get(i));
    }
    salida.close();
}

Para especificar que escribeLista puede lanzar dos excepciones, añada un cláusula throws a la declaración del método escribeLista. La cláusula throws se compone de la palabra clave throws seguida por una lista separada por comas de todas las excepciones lanzadas por ese método. La cláusula va después del nombre del método y la lista de argumentos y antes de la llave que define el alcance del método; aquí tiene un ejemplo. 

public void escribeLista() throws IOException, IndexOutOfBoundsException {

Recuerde que IndexOutOfBoundsException es una excepción no comprobada; incluirla en la cláusula throws no es obligatorio. Sólo podría escribir lo siguiente: 

public void escribeLista() throws IOException {
