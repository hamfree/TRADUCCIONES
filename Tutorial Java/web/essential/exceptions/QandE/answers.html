Respuestas a Preguntas y Ejercicios

Preguntas

  1. Pregunta: ¿Es el siguiente código legal? 
    try {
    
} finally {
   
}
    Respuesta: Sí, es legal — y muy útil. Una sentencia try no tiene que tener un bloque catch si tiene un bloque finally. Si el código en la sentencia try tiene múltiples puntos de salida y no tiene cláusulas catch asociadas, el código en el bloque finally es ejecutado no importando como termine el bloque try. Así tine sentido establecer un bloque finally cuando hay código que siempre debe ser ejecutado. Esto incluye código de recuperación de recursos, tal como el código para cerrar flujos de E/S.
  2. Pregunta: ¿Qué tipos de excepción pueden ser capturados por el siguiente manejador? 
    catch (Exception e) {
     
}
    ¿Qué está mal al usar este tipo de manejador de excepción?

    Respuesta: Este manejador captura excepciones del tipo Exception; por lo tanto, captura cualquier excepción. Esto puede ser una implementación pobre porque está perdiendo información valiosa sobre el tipo de excepción que está siendo lanzado y haciendo su código menos eficiente. Como resultado, su programa puede ser forzado a determinar el tipo de la excepción antes de que pueda decidir sobre la mejor estrategia de recuperación.

  3. Pregunta: ¿Hay algo equivocado con el siguiente manejador de excepción tal como está escrito? ¿Este codigo compilará? 
    try {

} catch (Exception e) {
   
} catch (ArithmeticException a) {
    
}
    Respuesta: Este primer manejador captura excepciones de tipo Exception; por consiguiente , captura cualquier excepción, incluyendo ArithmeticException. El segundo manejador nunca podría ser alcanzado. Este código no compilará.
  4. Pregunta: Haga coincidir cada situación de la primera lista con un elemento de la segunda lista.
    1. int[] A;
      A[0] = 0;
    2. La MVJ inicia ejecutando su programa, pero la MVJ no puede encontrar las clases de la plataforma Java. (Las clases de la plataforma Java residen en classes.zip o rt.jar.)
    3. Un programa está leyendo un flujo y alcance el marcador de fin del flujo.
    4. Antes de cerrar el flujo y después de alcanzar el marcador de fin del flujo, un programa intenta leer el flujo de nuevo.
    1. __error
    2. __excepción comprobada
    3. __error de compilación
    4. __sin excepción

    Respuesta:

    1. 3 (error de compilación). La matriz no está inicializada y no compilará.
    2. 1 (error).
    3. 4 (sin exception). Cuando lee un flujo, espera que haya un marcador de final de fluno. Usará excepciones para capturar comportamiento no esperado en su programa.
    4. 2 (excepción comprobada).
