El Bloque try

El primer paso en la construcción de un manejador de excepción es encerrar el codigo que podría lanzar una excepción dentro de un bloque try. En general, un bloque try se parece a lo siguiente: 

try {
    código
}
bloques catch y finally  . . .

El segmento en el ejemplo etiquetado código contiene una o más líneas legales de código que podrían lanza una excepción. (Los bloques catch y finally son explicados en las siguientes subsecciones).

Para construir un manejaador de excepciones para el método escribeLista de la clase ListaDeNumeros, encierre las sentencias que-lanzan-excepciones del método escribeLista dentro de un bloque try. Hay más de una manera de hacer esto. Puede colocar cada línea de codigo que podría lanzar una excepción en su propio bloque try y establecer manejadores de excepción separados para cada uno. O, puede colocar todo el código de escribeLista dentro de un bloque unico try y asociar múltiples manejadores con él. El siguiente listado usa un bloque try para el método entero porque el código en cuestión es muy corto. 

private List<Integer> lista;
private static final int TAMAÑO = 10;

public void escribeLista() {
    PrintWriter salida = null;
    try {
        System.out.println("Entrados en sentencia try");
        salida = new PrintWriter(new FileWriter("ficheroSalida.txt"));
        for (int i = 0; i < TAMAÑO; i++) {
            salida.println("Valor en " + i + " = " + lista.get(i));
        }
    }
    bloques catch y finally . . .
}

Si una excepción ocurre dentro del bloque try , esa excepción es manejada por un manejador de excepciones asociado con ella. Para asociar un manejador de excepciones con un bloque try, debe colocar un bloque catch después de él; la siguiente sección, Los Bloques catch, le muestran cómo.
