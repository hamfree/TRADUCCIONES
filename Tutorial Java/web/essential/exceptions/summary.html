Sumario

Un programa puede usar excepciones para indicar que un error ocurrió. Para lanzar una excepción, use la sentencia throw y proporciónele un objeto excepción — un descendiente de Throwable — para ofrecer información sobre el error específico que ocurrió. Un método que lanza un excepción no comprobada no capturada debe incluir una cláusula throws en su declaración.

Un programa puede capturar excepciones usando una combinación de los bloques try, catch, y finally.

  • El bloque try identifica un bloque de código en el cual una excepción puede ocurrir.
  • El bloque catch identifica un bloque de código, conocido como un manejador de excepción, que puede manejar un tipo particular de excepción.
  • El bloque finally identifica un bloque de código que está garantizado que se ejecutará, y es el lugar correcto para cerrar ficheros, recuperar recursos, y de otra manera limpiar después del código encerrado en el bloque try.

La sentencia try contendrá al menos un bloque catch o un bloque finally y puede tener múltiples bloques catch.

La clase del objeto excepción indica el tipo de excepción lanzada. El objeto excepción puede contener más información sobre el error, incluyendo un mensaje de error. Con el encadenamiento de excepciones, una excepción puede apuntar a la excepción que la cauśo, la cual puede a su vez apuntar la excepción que la causó, y así.
