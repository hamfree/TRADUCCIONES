Los Bloques catch

Usted asocia manejadores de excepción con un bloque try estableciendo uno o más bloques catch directamente después del bloque try. No puede haber código entre el final del bloque try y el principio del primer bloque catch. 

try {

} catch (ExceptionTipo nombre) {

} catch (ExceptionTipo nombre) {

}

Cada bloque catch es un manejador de excepciones que maneja el tipo de excepción indicado por su argumento. El tipo del argumento, ExceptionTipo, declara el tipo de la excepción que el manejador puede manejar y debe ser el nombre de una clase que herede de la clase Throwable. El manejador puede referirse a la excepción con nombre.

El bloque catch contiene código que es ejecutado si y cuando el manejador de excepción es invocado. El sistema en tiempo de ejecución invoca el manejador de excepción cuando el manejador es el primero en la pila de llamadas cuyo TipoDeExcepción coincide con el tipo de la excepción lanzada. El sistema lo considera una coincidencia si el objeto lanzado puede legalmente ser asignado al argumento del manejador de excepcion.

Los siguientes son dos manejadores de excepción para el método escribeLista: 

try {

} catch (IndexOutOfBoundsException e) {
    System.err.println("IndexOutOfBoundsException: " + e.getMessage());
} catch (IOException e) {
    System.err.println("Cogida IOException: " + e.getMessage());
}

Los manejadores de excepción pueden hacer más que sólo imprimir mensajes de error o parar el programa. Pueden hacer una recuperación del error, preguntar al usuario para que tome una decisión, o propagar el error hacia arriba a un manejador de más-alto-nivel usando la cadena de excepciones, como se describe en la sección Excepciones Encadenadas .

Capturando Más de Un Tipo de Excepción con Un Manejador de Excepción

En Java SE 7 y posteriores, un único bloque catch puede manejar más de un tipo de excepción. Esta característica puede reducir la duplicación de código y disminuir la tentación de capturar una excepción excesivamente amplia.

En la cláusula catch, especifique el tipo de excepciones que el bloque puede manejar, y separe cada tipo de excepción con una barra vertical (|): 

catch (IOException|SQLException ex) {
    logger.log(ex);
    throw ex;
}

Nota: Si un bloque catch maneja más de un tipo de excepción, entonces el parámetro catch es implícitamente final. En este ejemplo, el parámetro catch ex es final y por lo tanto no puede asignar ningún valor a él dentro del bloque catch.
