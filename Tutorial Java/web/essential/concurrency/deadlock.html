Bloqueo Mutuo

El bloqueo Mutuo describe una situación donde dos o más hilos están bloqueados para siempre, esperando unos por los otros. Aquí tiene un ejemplo.

Alfonso y Gastón son amigos, y grandes creyentes en la cortesía. Una regla estricta de cortesía es que cuando usted se inclina ante un amigo, debe permanecer inclinado hasta que su amigo tenga la oportunidad de devolver la inclinación. Desafortunadamente, esta regla no tiene en cuenta la posibilidad de que los dos amigos podrían inclinarse cada uno al mismo tiempo. Esta aplicación de ejemplo, BloqueMutuo, modela ésta posibilidad: 


public class BloqueMutuo {
    static class Amigo {
        private final String nombre;
        public Amigo(String nombre) {
            this.nombre = nombre;
        }
        public String getName() {
            return this.nombre;
        }
        public synchronized void inclinarse(Amigo inclinador) {
            System.out.format("%s: ¡%s"
                + "  se ha inclinado ante mí!%n",
                this.nombre, inclinador.getName());
            inclinador.devolverLaInclinacion(this);
        }
        public synchronized void devolverLaInclinacion(Amigo inclinador) {
            System.out.format("%s: ¡%s"
                + " me ha devuelto la inclinación!%n",
                this.nombre, inclinador.getName());
        }
    }

    public static void main(String[] args) {
        final Amigo alfonso =
            new Amigo("Alfonso");
        final Amigo gaston =
            new Amigo("Gastón");
        new Thread(new Runnable() {
            public void run() { alfonso.inclinarse(gaston); }
        }).start();
        new Thread(new Runnable() {
            public void run() { gaston.inclinarse(alfonso); }
        }).start();
    }
}

Cuando BloqueoMutuo se ejecuta, es extremadamente probable que ambos hilos se bloqueen cuando intenten invocar devolverLaInclinacion. Ningún bloque terminará nunca, porque cada hilo espera a que el otro salga de la inclinación.
