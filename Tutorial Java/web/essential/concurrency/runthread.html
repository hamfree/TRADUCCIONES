Definiendo e Iniciando un Hilo

Una aplicación que crea una instancia de Thread debe establecer el código que se ejecutará en ese hilo. Hay dos formas de hacer esto:

  • Ofrecer un objeto Runnable. La interfaz Runnable define un método único, run, destinado a contener el código ejecuado en el hilo. El objeto Runnable es pasado al constructor de Thread como en el ejemplo HolaRunnable : 
    
public class HolaRunnable implements Runnable {

    public void run() {
        System.out.println("¡Hola desde un hilo!");
    }

    public static void main(String args[]) {
        (new Thread(new HolaRunnable())).start();
    }

}
  • Derivar de Thread. La clase Thread en sí misma implementa Runnable, aunque su método run no hace nada. Una aplicación puede derivar Thread, estableciendo su propia implementación de run, como en el ejemplo HolaThread : 
    
public class HolaThread extends Thread {

    public void run() {
        System.out.println("¡Hola desde un hilo!");
    }

    public static void main(String args[]) {
        (new HolaThread()).start();
    }

}

Note que ambos ejemplos invocan Thread.start en orden a iniciar el nuevo hilo.

¿Cuáles de estos idiomas debería usar? El primer idioma, el cual emplea el objeto Runnable , es más general, porque el objeto Runnable puede derivar una clase distinta de Thread. El segundi idioma es más fácil de usar en aplicaciones simples, pero está limitado por el hecho de que su clase tarea debe ser una descendiente de Thread. Esta lección se enfoca se en la primera aproximación, la cual separa la tarea Runnable del objeto Thread que ejecuta la tarea. No sólo es esta aproximación más flexible, sino que es aplicable a las IPA de gestión de hilos de alto-nivel cubiertos más tarde.

La clase Thread define una variedad de métodos útiles para la gestión de hilos. Estos incluyen métodos estáticos, los cuales ofrecen información sobre, o afectan el estatus de, del hilo que invoca el método.
