Métodos Sincronizados

El lenguaje de programación Java establece dos idiomas básicos de sincronización: métodos sincronizados y sentencias sincronizadas. La más compleja de las dos, sentencias sincronizadas, son descritas en la siguiente sección. Esta sección es sobre métodos sincronizados.

Para hacer un método sincronizado, simplemente añada la palabra clave synchronized a su declaración: 

public class ContadorSincronizado {
    private int c = 0;

    public synchronized void incrementa() {
        c++;
    }

    public synchronized void decrementa() {
        c--;
    }

    public synchronized int value() {
        return c;
    }
}

Si cuenta es una instancia de ContadorSincronizado, entonces hacer estos métodos sincronizados tiene dos efectos:

  • Primero, no es posible para dos invocaciones de métodos sincronizados sobre el mismo objeto el solaparse. Cuando un hilo está ejecutando un método sincronizado en un objeto, todos los demás hilos que invoquen métodos sincronizados para el mismo objeto se bloquean (suspenden su ejecución) hasta que el primer hilo haya acabado con el objeto.
  • Segundo, cuando termina un método sincronizado, automáticamente establece una relación ocurre-antes con cualquier invoación subsecuente de un método sincronizado para el mismo objeto. Esto garantiza que los cambios al estado del objeto son visibles a todos los hilos.

Note que los constructores no pueden ser sincronizados — usar la palabra clave synchronized con un constructor es un error de sintaxis. Sincronizar constructores no tiene sentido, porque sólo el hilo que crea un objeto tendría acceso a él mientras está siendo construído.

Advertencia: Cuando construye un objeto que será compartido entre hilos, tenga mucho cuidado de que una referencia al objeto no "escape" prematuramente. Por ejemplo, suponga que quiere mantener una List llamada instancias conteniendo cada instancia de clase. Podría estar tentando a añadir la siguiente línea a su constructor: 
instancias.add(this);
Pero entonces otros hilos pueden usar instancias para acceder al objeto antes de que la construcción del objeto esté completa.

Los métodos sincronizados habilitan una estrategia simple para prevenir las interferencias entre hilos y los errores de consistencia en memoria: si un objeto es visible a más de un hilo, todas las lecturas o escrituras a las variables de ese objeto son hechas a través de métodos sincronizados methods. (Una excepción importante: campos finales, los cuales no pueden ser modificados después del que el objeto esté construído, pueden ser leídos de forma segura a través de métodos no-sincronizados, una vez el objeto es construído) Esta estrategia es efectiva, pero puede presentar problemas con el tiempo de vida, como veremos más tarde en esta lección.
