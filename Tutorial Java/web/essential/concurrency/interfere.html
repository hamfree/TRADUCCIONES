Interferencias entre Hilos

Considere una clase simple llamada Contador 


class Contador {
    private int c = 0;

    public void incrementa() {
        c++;
    }

    public void decrementa() {
        c--;
    }

    public int valor() {
        return c;
    }

}

Contador está diseñada de forma de que cada invocación de incrementa añadirá 1 a c, y cada invocación de decrementa substrerá 1 de c. Sin embargo, si un objeto Contador es referenciado por múltiples hilos, la interferencia entre hilos puede prevenir esto de ocurrir como se esperaba.

La interferencia ocurre cuando dos operaciones, ejecutándose en hilos diferentes, pero actuando sobre los mismos datos, se intercalan. Esto significa que las dos operaciones consisten de múltiples pasos, y las secuencias de los pasos se superponen.

Podría no parecer posible para operaciones en instancias de Contador que se intercalen, ya que ambas operacioens en c c son sentencias únicas, simples. Sin embargo, incluso las sentencias simples pueden traducirse en múltiples pasos por la máquina virtual. No examinaremos los pasos específicos que la máquina virtual toma — es bastante con saber que la expresión única c++ puede ser descompuesta en tres pasos:

  1. Recupera el valor actual de c.
  2. Incrementa el valor recuperado en 1.
  3. Almacena el valor incrementado de vuelta en c.

La expresión c-- puede ser descompuesta de la misma forma, excepto que el segundo paso decrementa en vez de incrementar.

Suponga que el Hilo A invoca incrementa más o menos al mismo tiemepo que el Hilo B invoca decrementa. Si el valor inicial de c es 0, sus acciones superpuestas podrían seguir esta secuencia:

  1. Hilo A: Recupera c.
  2. Hilo B: Recupera c.
  3. Hilo A: Incrementa el valor recuperado; el resultado es 1.
  4. Hilo B: Decrementa el valor recuperado; el resultado es -1.
  5. Hilo A: Almacena el resultado en c; c es ahora 1.
  6. Hilo B: Almacena el resultado en c; c es ahora -1.

El resultado del Hilo A se pierde, sobreescrito por el Hilo B. Esta superposición particular es sólo una posibilidad. Bajo diferentes circunstancias podría ser que el se perdiera el resultado del Hilo B, o podría ser que no se dieran errores en absoluto. Debido a que son impredecibles, los errores de interferencia entre hilos pueden ser difíciles de detectar y corregir.
