Interfaces del Ejecutor

El paquete java.util.concurrent define tres interfaces de ejecutor: three executor interfaces:

  • Executor, un interfaz simple que soporta lanzar nuevas tareas.
  • ExecutorService, una subinterfaz de Executor, la cual añade características que ayudan a gestionar el ciclo de vida, tanto de las tareas individuales como del ejecutor en sí mismo.
  • ScheduledExecutorService, una subinterfaz de ExecutorService, que soporta la ejecución futura y/o periódica de tareas.

Típicamente, las variables que se refieren a objetos ejecutor son declaradas como una de esos tres tipos de interfaz, no con un tipo de clase ejecutor.

El Intefaz Executor

El interfaz Executor establece un método único, execute, diseñado para ser un reemplazo para un lenguaje común de creación de subprocesos. Si r es un objeto Runnable , y e es un objeto Executor puede reemplazar 

(new Thread(r)).start();

con 

e.execute(r);

Sin embargo, la definición de execute es menos específica. El idioma de bajo-nivel crea un nuevo hilo y lo lanza inmediatamente. Dependiendo de la implementación del Executor , execute puede hacer lo mismo, pero es más probable que utilice un hilo de trabajo existente para ejecutarr, o colocar r en una cola para esperar que un hilo de trabajo se vuelva disponible. (Describiremos los hilos de trabajo in la sección Agrupaciones de Hilos.)

Las implementaciones de executor en java.util.concurrent están diseñadas para hacer un uso completo de los más avanzados interfaces ExecutorService y ScheduledExecutorService , aunque también funcionarán con el interfaz base Executor .

El Interfaz ExecutorService

El intefaz ExecutorService suplementa execute con un similar, pero más versátil método submit. Como execute, submit acepta objetos Runnable , pero también acepta objetos Callable , los cuales permiten a la tarea devolver un valor.El método submit devuelve un objeto Future , el cual es usado para recuperar el valor devuelto por Callable y gestionar el estado de ambas tareas Callable y Runnable .

ExecutorService también ofrece métodos para submitir grandes colecciones de objetos Callable . Finalmente, ExecutorService ofrece un número de métodos para gestionar el apagado de un ejecutor. Para soportar el apagado inmediato, las tareas deberían manejar las interrupciones correctamente.

El Interfaz ScheduledExecutorService

El interfaz ScheduledExecutorService suplmenta los métodos de su padre ExecutorService con schedule, el cual ejecuta una tarea Runnable o Callable después de un retardo especificado. Además, el interfaz define scheduleAtFixedRate y scheduleWithFixedDelay, los cuales ejecutan las tareas especificadas repetidamente, en intervalos definidos.
