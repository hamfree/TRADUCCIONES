Un Ejemplo de Clase Sincronizada

La clase, RGBSincronizado, define objetos que representan colores. Cada objeto representa el color como tres enteros que representan los valores de los colores primarios y una cadena que da el nombre del color. 


public class RGBSincronizado {

    // Los valores deben estar entre 0 y 255.
    private int rojo;
    private int verde;
    private int azul;
    private String nombre;

    private void comprueba(int rojo,
                       int verde,
                       int azul) {
        if (rojo < 0 || rojo > 255
            || verde < 0 || verde > 255
            || azul < 0 || azul > 255) {
            throw new IllegalArgumentException();
        }
    }

    public RGBSincronizado(int rojo,
                           int verde,
                           int azul,
                           String nombre) {
        comprueba(rojo, verde, azul);
        this.rojo = rojo;
        this.verde = verde;
        this.azul = azul;
        this.nombre = nombre;
    }

    public void fija(int rojo,
                    int verde,
                    int azul,
                    String nombre) {
        comprueba(rojo, verde, azul);
        synchronized (this) {
            this.rojo = rojo;
            this.verde = verde;
            this.azul = azul;
            this.nombre = nombre;
        }
    }

    public synchronized int dameRGB() {
        return ((rojo << 16) | (verde << 8) | azul);
    }

    public synchronized String dameNombre() {
        return nombre;
    }

    public synchronized void invierte() {
        rojo = 255 - rojo;
        verde = 255 - verde;
        azul = 255 - azul;
        nombre = "Inverso de " + nombre;
    }
}

RGBSincronizado debe ser usado cuidadosamente para evitar ser visto en un estado inconsistente. Suponga, por ejemplo, que un hilo ejecuta el siguiente código: 

RGBSincronizado color =
    new RGBSincronizado(0, 0, 0, "Negro Brea");
...
int miNumColor = color.dameRGB();      //Sentencia 1
String miNombreColor = color.dameNombre(); //Sentencia 2

Si otro hilo invoca color.fija despueés de la sentencia 1 pero antes de la sentencia 2, el valor de miNumColor no coincidirá con el valor de miNombreColor. Para evitar este resultado, las dos sentencias deben estar unidas: 

synchronized (color) {
    int miNumColor = color.dameRGB();
    String miNombreColor = color.dameNombre();
}

Esta clase de inconsistencia es sólo posible en objetos mutables — no será un problema para la versión inmutable de RGBSincronizado.
