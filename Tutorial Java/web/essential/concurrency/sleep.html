Pausando la Ejecución con Sleep

Thread.sleep causa que el hilo actual suspenda la ejecución por un periodo especificado. Esta es una manera eficiente de hacer el tiempo del procesador disponible a otros hilos de una aplicación u otras aplicaciones que podrían estar ejecutándose en un sistema de computadora. El método sleep puede ser usado para estimular, como es mostrado en el ejemplo que sigue, y esperar por otro hilo con deberes que se entienden deben tener requerimientos de tiempo, como en el ejemplo SimpleThreads en una sección posterior.

Dos versiones sobrecargadas de sleep son ofrecidas: una que especifica el tiempo de suspensión al milisegundo y una que especifica el tiempo de suspensión al nanosegundo. Sin embargo, no se garantiza que estos tiempos de suspensión sean precisos, porque están limitados por las facilidades ofrecidas por el SO subyacente. Además, el periodo de suspensión puede ser terminado por interrupciones, como veremos en una sección más tarde. En cualquier caso, no puede asumir que invocar sleep suspenderá el hilo por el período de tiempo preciso especificado.

El ejemplo SleepMensajes usa sleep para imprimir mensajes en intervalos de cuatro segundos: 


public class SleepMensajes {
    public static void main(String args[])
        throws InterruptedException {
        String infoImportante[] = {
            "Las yeguas comen avena",
            "Los conejos comen avena",
            "Los pequeños corderos comen hiedra",
            "Un niño también comerá hiedra"
        };

        for (int i = 0;
             i < infoImportante.length;
             i++) {
            //Pausa por 4 segundos
            Thread.sleep(4000);
            //Imprime un mensaje
            System.out.println(infoImportante[i]);
        }
    }
}

Note que main declara que throws InterruptedException. Esta es una excepción que sleep lanza cuando otro hilo interrumpe el hilo actual mientras sleep está activo. Ya que esta aplicación no tiene definido otro hilo que cause la interrupción, no se molesta en capturar InterruptedException.
