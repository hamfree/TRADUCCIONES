Interrupciones

Una interrupción es una indicación a un hilo de que debería parar lo que esté haciendo y hacer otra cosa. Depende del programador decidir exactamente cómo un hilo responde a una interrupción, pero es muy común para el hilo el terminar. Éste es el uso enfatizado en esta lección.

Un hilo envía una interrupción invocando interrupt sobre el objeto Thread para el hilo a ser interrumpido. Para que el mecanismo de interrupción funcione correctamente, el hilo interrumpido debe soportar su propia interrupción.

Soportar la Interrupción

¿Cómo un hilo soporta su propia interrupción? Esto depende de lo que esté haciendo actualmente. Si el hilo esta frecuentemente invocando métodos que lanzan InterruptedException, es simple volver desde el método run después de que capture la excepción. Por ejemplo, suponga que el bucle central de mensajes en el ejemplo SleepMensajes fuera en el método run de un hilo de un objeto Runnable. Entonces podría ser modificado como sigue para soportar interrupciones: 

for (int i = 0; i < infoImportante.length; i++) {
    // Pausa por 4 segundos
    try {
        Thread.sleep(4000);
    } catch (InterruptedException e) {
        // Hemos sido interrumpidos: no más mensajes.
        return;
    }
    // Imprime un mensaje
    System.out.println(infoImportante[i]);
}

Muchos métodos que lanzan InterruptedException, tales como sleep, están diseñados para cancelar su operación actual y volver inmediatamente cuando se recibe una interrupción.

¿Qué ocurre si un hilo va a tardar mucho tiempo sin invocar un método que lance InterruptedException? Entonces deberá periódicamente invocar Thread.interrupted, el cual devuelve true si una interrupción ha sido recibida. Por ejemplo: 

for (int i = 0; i < inputs.length; i++) {
    heavyCrunch(inputs[i]);
    if (Thread.interrupted()) {
        // Hemos sido interrumpidos: no más
        // We've been interrupted: no más procesado pesado.
        return;
    }
}

En este ejemplo simple, el código simplemente comprueba si hay interrupción y se sale del hilo si una ha sido recibida. En aplicaciones más complejas, podría tener más sentido lanzar una InterruptedException: 

if (Thread.interrupted()) {
    throw new InterruptedException();
}

Esto permite al código que maneja la interrupción a estar centralizado en una cláusula catch.

La Bandera de Estado de la Interrupción

El mecanismo de interrupción está implementado usando una bandera interna conocida como el estado de interrupción. Invocar Thread.interrupt establece esta bandera. Cuando un hilo comprueba si hay una interrupción invocando el método estático Thread.interrupted, el estado de la interrupción es borrado. El método no-estático isInterrupted, el cual es usado por un hilo para preguntar por el estado de interrupción de otro, no cambia la bandera de estado de interrupción.

Por convención, cualquier método que termina lanzando una InterruptedException borra el estado de la interrupción cuando lo hace. Sin embargo, siempre es posible que el estado de la interrupción sea establecido inmediatamente de nuevo, por otro hilo invocando una interrupt.
