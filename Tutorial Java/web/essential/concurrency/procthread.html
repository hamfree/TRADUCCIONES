Procesos e Hilos

En programación concurrente, hay dos unidades básicas de ejecución: procesos e hilos. En el lenguaje de programación Java, la programación concurrente está mayormente implicada con los hilos. Sin embargo, los procesos son también importantes.

Un sistema de computadora normalmente tiene muchos procesos activos e hilos. Esto es verdad incluso en sistemas que sólo tienen un núcleo de ejecución única, y así sólo tiene un hilo en realidad ejecutándose en un momento dado. El tiempo de proceso para un núcleo único es compartido entre procesos e hilos a través de una característica del SO llamada tiempo compartido.

Cada vez es más común que los sistemas informáticos tengan múltiples procesadores o procesadores con múltiples núcleos de ejecución. Esto mejora en gran medida la capacidad de un sistema para la ejecución concurrente de procesos e hilos — pero la concurrencia es posible incluso en sistemas simples, sin procesadores múltiples o núcleos de ejecución.

Procesos

Un proceso tiene un entorno de ejecución auto-contenido. Un proceso generalmente tiene un conjunto completo, privado de recursos básicos de ejecución; en particular, cada proceso tiene su propio espacio de memoria.

Los procesos son a menudos vistos como sinónimos con programas o aplicaciones. Sin embargo, que el usuario vea como una aplicación puede de hecho ser un conjunto de procesos cooperativos. Para facilitar la comunicación entre procesosm la mayoría de los sistemas operativos soportan los recursos de Comunicación Entre Procesos (IPC), tales como tuberías y sockets. IPC no es usado sólo para la comunicación entre procesos en el mismo sistema, sino también en los procesos en sistemas diferentes.

La mayoría de las implementaciones de la máquina virtual Java se ejecutan como un proceso único. Una aplicación Java puede crear procesos adicionales usando un objeto ProcessBuilder . Las aplicaciones multiproceso están más allá del alcancde de esta lección.

Hilos

Los hilos son a veces llamados procesos ligeros. Ambos, procesos e hilos, establecen un entorno de ejecución, pero crear un nuevo hilo requiere menos recursos que crear un proceso nuevo.

Los hilos existen dentro de un proceso — cada procesos tiene al menos uno. Los hilos comparten los recursos del proceso, incluyendo la memoria y los ficheros abiertos. Esto hace que la comunicación sea eficiente, pero potencialmente problemática.

La ejecución multihilo es una característica esencial de la plataforma Java. Cada aplicación tiene al menos un hilo — o varios, si cuenta los hilos del "sistema" que hacen cosas como la gestión de memoria y el manejo de señales. Pero desde el punto de vista del programador de aplicaciones, usted empieza justo con un hilo, llamado el hilo principal. Este hilo tiene la habilidad de crear hilos adicionales, como bien se demostrará en la siguiente sección.
