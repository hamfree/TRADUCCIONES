Lección: IPA Simple para XML

Esta lección se enfoca en la IPA Simple para XML (SAX), un mecanismo de acceso en serie controlado por eventos para acceder a documentos XML. Este protocolo se usa frecuentemente por servlets y programas orientados a red que necesitan transmitir y reciben documentos XML, porque este es el mecanismo más rapido y menos intensivo en memoria que está disponible actualmente para tratar con documentos XML, aparte de la IPA de Transmisión para XML (StAX).

Nota — En pocas palabras, SAX está orientado hacia el procesamiento independiente del estado, donde el manejo de un elemento no depende de los elementos que vieneron antes. StAX, por otro lado , está orientado hacia el procesado dependiente del estado. Para una comparación más detallada, vea Cuando usar SAX.

Configurar un programa para usar SAX requiere un poco más de trabajo que configurarlo para usar el Modelo de Objeto de Documento (DOM). SAX es un modelo conducido por eventos (usted proporciona los métodos de retrollamada, y el analizador los invoca mientras lee los datos XML), y eso lo hace más duro de visualizar. Por último, no puede "realizar una copia de seguridad " a una parte anterior del documento, o reorganizarlo, al igual que no puede hacer una copia de seguridad de un flujo de datos en serie o reorganizar los caracteres que ha leído de ese flujo.

Por esas razones, los desarrolladores que están escribiendo una aplicación orientada al usuario que visualiza un documento XML y posiblemente lo modifica querrán usar el mecanismo DOM descrito en Modelo de Objeto de Documento.

Sin embargo, incluso si planea construir aplicaciones con DOM exclusivamente, hay varias razones importantes para que se familiarize con el modelo SAX:

  • Misma Gestión del Error: La misma clase de excepciones se generan por las IPAs de SAX y DOM, así que el código de la gestión de errores es virtualmente idéntico.

  • Gestión de Errores de Validación: Por defecto, las especificaciones requieren que los errores de validación sean ignorados. Si quiere lanzar una excepción en el caso de un error de validación (y probablemente lo haga), entonces necesita entender cómo funciona en SAX la gestión de errores.

  • Conversión de Datos Existentes: Como verá en Modelo de Objeto de Documento, hay un mecanismo que puede usar para convertir un conjunto de datos existentes a XML. Sin embargo, aprovechar ese mecanismo requiere una compresión del modelo SAX.
