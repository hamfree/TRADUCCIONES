Cómo trabaja XPath

La especificación XPath es la base para una variedad de especificaciones, incluye XSLT y las especificaciones de enlace/direccionamiento tal como XPointer. Así un entendimiento de XPath es fundamental para una gran cantidad de uso avanzado de XML. Esta sección proporciona una introducción a XPath en el contexto de XSLT.

Expresiones XPath

En general, una expresión XPath especifica un patrón que selecciona un conjunto de nodos XML. Las plantillas XSLT después usan esos patrones cuando aplica las transformaciones. (XPointer, por otro lado, agrega mecanismos para definir un punto o un rango de forma que las expresiones XPath pueden usarse para el direccionamiento).

Los nodos en una expresión XPath se refieren a algo más que elementos. También se refieren a texto y atributos, entre otras cosas. En realidad, la especificación XPath define un modelo de documento abstracto que define siete clases de nodos:

  • Raiz

  • Elemento

  • Texto

  • Atributo

  • Comentario

  • Instrucción de procesamiento

  • Espacio de nombres

El elemento raíz de los datos XML es modelado por un nodo elemento. El nodo raíz de XPath contiene el elemento raíz del documento, así como otra información relacionada con el documento.

Modelo de Datos XSLT/XPath

Como el Modelo de Objetos del Documento (DOM), el modelo de datos XSLT/XPath consiste de un árbol que contiene una variedad de nodos. Bajo cualquier nodo elemento dado, hay nodos de texto, nodos de atributo, nodos elemento, nodos comentario, y nodos de instrucciones de procesamiento.

En este modelo abstracto, las distinciones sintácticas desaparecen, y se queda con una vista normalizada de los datos. En un nodo de texto, por ejemplo, no importa si el texto se definió en una sección CDATA o si incluía referencias a entidades. El nodo de texto consistirá de datos normalizados, ya que existe después de que se completa todo el análisis. Así el texto contendrá un carácter <, independientemente de que se haya utilizado o no una referencia de entidad como < o una sección CDATA para incluirlo. (Del mismo modo, el texto contendrá un carácter &, ya sea que se entregó usando & o si estaba en una sección CDATA).

En esta sección, trataremos mayormente con nodos elemento y nodos texto. Para los otros mecanismos de direccionamiento, vea la especificación de XPath.

Plantillas y Contextos

Uns plantilla XSLT es un conjunto de instrucciones de formateo que se aplican a los nodos seleccionados por una expresión XPath. En una hoja de estilos, una plantila XSLT se parecería a algo como esto: 

<xsl:template match="//LIST">
    ...
</xsl:template>

La expresión //LIST selecciona el conjunto de los nodos LIST del flujo de entrada. Instrucciones adicionales dentro de la plantilla le indican al sistema qué hacer con ellos.

El conjunto de nodos seleccionados por dicha expresión define el contexto en el que se evalúan otras expresiones en la plantilla. Ese contexto puede ser considerado como el conjunto completo — por ejemplo, cuando se determina el número de nodos que contiene.

El contexto puede ser también considerado como un sólo miembro del conjunto, ya que cada miembro se procesa uno a uno. Por ejemplo, dentro de la plantilla que procesa LIST, la expresión @type se refiere al tipo atributo del nodo actual LIST. (Del mismo modo, la expresión @* se refiere a todos los atributos para el elemento LIST actual).

Direccionamiento Básico de XPath

Un documento XML es una colección estructurada en árbol (jerárquica) de nodos. Como con una estructura jerárquica de directorio, es útil especificar una ruta que apunte a un nodo particular en la jerarquía ( de ahí el nombre de la especificación: XPath). De hecho, gran parte de la notación de rutas de directorio se mantiene intacta:

  • La barra inclinada (/) se usa como separador de ruta.

  • Una ruta absoluta desde la raíz del documento empieza con un /.

  • Una ruta relativa desde una ubicación determinada empieza con cualquier otra cosa.

  • Un punto doble (..) indica el padre del nodo actual.

  • Un punto sólo (.) indica el nodo actual.

Por ejemplo, en un documento Extensible HTML (XHTML) (un documento XML que parace como HTML pero está bien formado de acuerdo a las reglas XML), la ruta /h1/h2/ indicaría un elemento h2 bajo un h1. (Recuerde que en XML, los nombres de elemento son sensibles a mayúsculas y minúsculas, así que esta clase de especificación funciona mucho mejor en XHTML que lo haría en HTML plano, porque HTML no es sensible a mayúsculas y minúsculas).

En una especificación de coincidencia de patrones como XPath, la especificación /h1/h2 selecciona todos los elementos h2 que se encuentran bajo un elemento h1. Para seleccionar un elemento h2 específico, use corchetes [] para indexar (como aquellos usados para las matrices). La ruta /h1[4]/h2[5] por lo tanto seleccionaría el quinto elemento h2 bajo el cuarto elemento h1.

Nota — En XHTML, todos los nombres de elementos están en minúsculas. Esa es una convención bastante común para los documentos XML. Sin embargo, los nombres en mayúsculas son más fáciles de leer en un tutorial como este. Entonces, durante el resto de la lección XSLT, todos los nombres de elementos XML estarán en mayúsculas. (Los nombres de los atributos, por otro lado, permanecerán en minúsculas).

Un nombre especificado en una expresión XPath se refiere a un elemento. Por ejemplo, h1 en /h1/h2 se refiere a un elemento h1. Para referirse a un atributo, prefije el nombre de atributo con un signo @. Por ejemplo, @type se refierte al atributo type de un elemento. Asumiendo que tiene un documento XL con elementos LIST, por ejemplo, la expresión LIST/@type selecciona el atributo type del elemento LIST.

Nota — Ya que la expresión no empieza con /, la referencia especifica un nodo de lista relativo al contexto actual, cualquier posición en el documento en el que se encuentre.

Expresiones XPath Básicas

El rango completo de expresiones XPath se aprovecha de los comodines, operadores, y funciones que XPath define. Aprenderá más sobre esos pronto. Aquí, vemos un par de las expresiones XPath más comunes simplemente para presentarlas.

La expresión @type="unordered" especifica un atributo llamado type cuyo valor es unordered. Una expresión como LIST/@type especifica el atributo type de un elemento LIST.

Puede combinar estas dos notaciones para obtener algo interesante. En XPath, la notación de corchete ([]) normalmente asociada con la indexación se amplía para especificar los criterios de selección. Así la expresión LIST[@type="unordered"] selecciona todos los elementos LIST cuyo valor en el atributo type es unordered.

Expresiones similares existen para los elementos. Cada elemento tiene una cadena-valor asociada, que se forma concatenado todos los segmentos de texto que se encuentran debajo del elemento. (Una explicación más detallada de cómo funciona ese proceso se presenta en Cadena-Valor de un Elemento).

Suponga que modela lo que está sucediendo en su organización utilizando una estructura XML que consta de elementos PROJECT y elementos ACTIVITY que tienen una cadena de texto con el nombre del proyecto, varios elementos PERSON para listar las personas involucradas y, opcionalmente, un elemento STATUS que registra el estado del proyecto. Aquí tiene otros ejemplos que usan la notación extendida de corchetes:

  • /PROJECT[.="MyProject"]: Selecciona un PROJECT llamado "MyProject".

  • /PROJECT[STATUS]: Selecciona todos los proyectos que tienen un elemento hijo STATUS.

  • /PROJECT[STATUS="Critical"]: Selecciona todos los proyectos que tiene un hijo elemento STATUS con la cadena-valor Critical.

Combinar Direcciones de Índice

La especificación XPath define bastantes mecanismos de direccionamiento y se pueden combinar de muchas formas diferentes. Como resultado, XPath ofrece mucho poder expresivo para una especificación relativamente simple. Esta sección ilustra otras combinaciones interesantes:

  • LIST[@type="ordered"][3]: Selecciona todos los elementos LIST con el atributo type con valor ordered, y devuelve el tercero.

  • LIST[3][@type="ordered"]: Selecciona el tercer elemento LIST, pero sólo si tiene el atributo type con el valor ordered.

Nota — Muchas más combinaciones de operadores de dirección se listan en la sección 2.5 de la Especificación XPath. Esta es posiblemente la sección más útil de la especificación para definir una transformación XSLT.

Comodines

Por definición, un expresión de XPath no cualificada selecciona un conjunto de nodos XML que coinciden con el patrón especificado. Por ejemplo, /HEAD coincide con todas las entradas HEAD en el nivel más alto, mientras que /HEAD[1] sólo coincide el primero. La Tabla 4-1 lista los comodines que pueden usarse en las expresiones XPath para ampliar el alcance de la coincidencia de patrones.

Tabla 4-1 Comodines de XPath

Comodín

Significado

*

Coincide con cualquier nodo de elemento (ni atributos ni texto).

node()

Coincide con cualquier nodo de cualquier tipo: nodo de elemento, nodo de texto, nodo de atributo, nodo de instrucción de procesamiento, nodo de espacio de nombres o nodo de comentario.

@*

Coincide con cualquier nodo de atributo.

En el ejemplo de base de datos del proyecto, /*/PERSON[.="Fred"] coincide con cualquier elemento PROJECT o ACTIVITY que nombre a Fred.

Direccionamiento de Ruta Extendida

Hasta ahora, todos los patrones que ha visto especificaban un número exacto de niveles en la jerarquía. Por ejemplo, /HEAD especifica cualquier elemento HEAD en el primer nivel en la jerarquía, mientras que /*/* especifica cualquier elemento en el segundo nivel en la jerarquía. Para especificar un nivel indeterminado en la jerarquía, use una barra inclinada doble (//). Por ejemplo, la expresión XPath //PARA selecciona todos los elementos párrafo en un documento, dondequiera que se encuentren.

El patrón // puede también ser usado dentro de una ruta. Así la expresión /HEAD/LIST//PARA indica todos los elementos párrafos en un subárbol que empieza desde /HEAD/LIST.

Tipos de Datos y Operadores de XPath

Las expresiones XPath producen un conjunto de nodos, una cadena, un Booleano (un valor verdadero/falso), o un número. La Tabla 4-2 lista los operadores que pueden usarse en una expresión XPath:

Tabla 4-2 Operadores XPath

Operador

Significado

|

Alternativo. Por ejemplo, PARA|LIST selecciona todos los elementos PARA y LIST.

or, and

Devuelve el or/and de dos valores Booleanos.

=, !=

Igual o no igual, para Booleanos, cadenas, y números.

<, >, <=, >=

Menos que, mayor que, menor que o igual a, mayor que o igual a, para números.

+, -, *, div, mod

Operaciones de agregar, substraer, multiplicar, dividir en punto flotante, y módulo (resto) (e.g., 6 mod 4 = 2).

Las expresiones pueden ser agrupadas en paréntesis, así que no tiene que preocuparse por la precedencia de los operadores.

Nota — La precedencia de operadores es un término que responde a la cuestión, "Si usted especifica a + b * c, ¿Eso significa (a+b) * c o a + (b*c)?" (La precedencia de operadores es aproximadamente la misma que la que se muestra en la tabla).

Cadena-Valor de un Elemento

La cadena-valor de un elemento es la concatenación de todos los nodos descendientes, sin importar la profundidad. Considere estos datos XML de contenido mezclado: 

<PARA>This paragraph contains a <b>bold</b> word</PARA>

La cadena-valor del elemento <PARA> es This paragraph contains a bold word. En particular, dese cuenta de que <B> es un hijo de <PARA> y que el texto bold es un hijo de <B>.

El punto es que todo el texto en todos los hijos de un nodo se une en la concatenación para formar la cadena-valor.

Además, vale la pena comprender que el texto en el modelo de datos abstracto definido por XPath está completamente normalizado. Así si la estructura XML contiene la referencia de entidad &lt; o < en una sección CDATA, los elementos de la cadena-valor contendrán el carácter <. Por lo tanto, cuando se genera HTML o XML con una hoja de estilos XSLT, debe convertir las ocurrencias de < a &lt; o encerrarlas en una sección CDATA. Del mismo modo, las ocurrencias de & deben ser convertidas a &amp;.

Funciones XPath

Esta sección termina con una introducción de las funciones XPath. Puede usar las funciones XPath para seleccionar una colección de nodos de la misma forma que usaría una especificación de elemento como aquellas que ya ha visto. Otras funciones devuelven una cadena, un número, o un valor Booleano. Por ejemplo, la expresión /PROJECT/text() obtiene la cadena-valor de los nodos PROJECT.

Muchas funciones dependen del contexto actual. En el ejemplo precedente, el contexto para cada invocación de la función text() es el nodo PROJECT que está actualmente seleccionado.

Hay muchas funciones XPath — demasiadas para describirlas en detalle aquí. Esta sección proporciona un breve listado que muestra las funciones disponibles de XPath, junto con un sumario de lo que hacen. Para más información sobre las funciones, vea la sección 4 de la Especificación XPath.

Funciones de Conjuntos de Nodos

Muchas expresiones XPath seleccionan un conjunto de nodos. En esencia, devuelven un conjunto de nodos, Una función hace eso, también. La función id(...) devuelve el nodo con el ID especificado. (Los elementos tienen un ID sólo cuando el documento tiene un DTD, que especifica qué atributo tiene el tipo ID).

Funciones Posicionales

Estas funciones devuelven valores numéricos basados en la posición.

  • last(): Devuelve el índice del último elemento. Por ejemplo, /HEAD[last()] selecciona el último elemento HEAD.

  • position(): Devuelve la posición del índice. Por ejemplo, /HEAD[position() <= 5] selecciona los primeros cinco elementos HEAD.

  • count(...): Devuelve el recuento de elementos. Por ejemplo, /HEAD[count(HEAD)=0] selecciona todos los elementos HEAD que no tengan subtítulos.

Funciones de Cadena

Estas funciones operan con o devuelven cadenas.

  • concat(string, string, ...): Concatenca los valores de cadena.

  • starts-with(string1, string2): Devuelve verdadero si string1 empieza con string2.

  • contains(string1, string2): Devuelve verdadero si string1 contiene a string2.

  • substring-before(string1, string2): Devuelve el inicio de string1 antes de que aparezca string2.

  • substring-after(string1, string2): Devuelve el resto de string1 después de que string2 aparezca en él.

  • substring(string, idx): Devuelve la subcadena desde la posición del índice al final, donde el índice del primer char = 1.

  • substring(string, idx, len): Devuelve la subcadena de la longitud especificad desde la posición del índice.

  • string-length(): Devuelve el tamaño de la cadena-valor del nodo de contexto; el nodo de contexto es el nodo seleccionado actualmente — el nodo que fue seleccionado por una expresión XPath en la cual se aplicó una función tal como string-length().

  • string-length(string): Devuelve el tamaño de la cadena especificada.

  • normalize-space(): Devuelve la cadena-valor normalizada del nodo actual (sin espacios en blanco iniciales o finales, y secuencias de caracteres de espacios en blanco convertidos en un solo espacio).

  • normalize-space(string): Devuelve la cadena-valor normalizada de la cadena especificada.

  • translate(string1, string2, string3): Convierte string1, reemplazando las ocurrencia de los caracteres en string2 con el carácter correspondiente de string3.

Nota — XPath define tres formas de obtener el texto de un elemento: text(), string(object), y la cadena-valor implicada por un nombre de elemento en una expresión como esta: /PROJECT[PERSON="Fred"].

Funciones Booleanas

Esta funciones operan sobre o devuelven valores Booleanos.

  • not(...): Niega el valor Booleano especificado.

  • true(): Devuelve verdadero.

  • false(): Devuelve falso.

  • lang(string): Devuelve verdadero si el lenguaje del nodo de contexto (especificado por los atributos xml:Lang) es el mismo (o un sub-lenguaje de) que el lenguaje especificado; por ejemplo, Lang("en") es verdadero para <PARA_xml:Lang="en">...</PARA>.

Funciones Numéricas

Estas funciones operan con o devuelven valores numéricos.

  • sum(...): Devuevle la suma de los valores numéricos de cada nodo en el conjunto de nodo especifcado.

  • floor(N): Devuelve el entero más grande que no es mayor que N.

  • ceiling(N): Devuelve el entero más pequeño qu no es menor que N.

  • round(N): Devuelve el entero que está más cerca a N.

Funciones de Conversión

Estas funciones convierten un tipo de datos a otro.

  • string(...): Devuelve el valor de cadena de un número, un Booleano, o un conjunto de nodos.

  • boolean(...): Devuelve un valor Booleano para un nombre, cadena, o conjunto de nodos (un número no cero, un conjunto de nodos no vacío, y una cadena no vacía son todas verdaderas).

  • number(...): Devuelve el valor numérico de un Boolean, una cadena o un conjunto de nodos (verdadero si 1, falso es 0, una cadena que contiene un número se convierte en ese número, la cadena-valor de un conjunto de nodos se convierte en un número).

Funciones de Espacio de Nombres

Estas funciones le permiten determinar las características de un nodo.

  • local-name(): Devuelve el nombre del nodo actual, menos el prefijo del espacio de nombres.

  • local-name(...): Devuelve el nombre del primer nodo del conjunto de nodos especificado, menos el prefijo del espacio de nombres.

  • namespace-uri(): Devuelve el URI del espacio de nombres del nodo actual.

  • namespace-uri(...): Devuelve el URI del espacio de nombres del primer nodo del conjunto de nodos especificado.

  • name(): Devuelve el nombre expandido (URI más nombre local) del nodo actual.

  • name(...): Devuelve el nombre expandido (URI más nombre local) del primer nodo en el conjunto de nodos especificado.

Sumario

Los operadores, funciones, comodines, y mecanismos de direccionamiento de nodos de XPath pueden ser combinados en una amplia variedad de formas. La introducción que ha tenido hasta ahora debería darle una buena ventaja al especificar el patrón que necesita para cualquier propósito en particular.
