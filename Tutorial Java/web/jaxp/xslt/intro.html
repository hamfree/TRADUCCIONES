Introduciendo XSL, XSLT, y XPath

El Lenguaje Extensible de Hojas de Estilo (XSL) tiene tres subcomponentes principales:

XSL-FO

El estándar de Formateado de Objetos. De lejos el subcomponente más grande, este estándar da mecanismos para describir tamaños de fuentes, diseños de páginas, y otros aspectos de la representación de objetos. Este subcomponente no está cubierto por JAXP, ni está incluído en este tutorial.

XSLT

Este es el lenguaje de transformación, que le permite definir una transformación desde XML a algún otro formato. Por ejemplo, podría usar XSLT para producir HTML o una estructura XML diferente. Podría incluso usarlo para producir texto plano o para colocar la información en algún otro formato de documento. (y como verá en in Generando XML desde una Estructura de Datos Arbitraria, una aplicación inteligente puede ponerla en servicio para manipular también datos que no sean XML).

XPath

Al final, XSLT es un leguaje que le permite especificar que clases de cosas hacer cuando se encuentra un elemento en particular. Pero para escribir un programa para diferentes partes de una estructura XML de datos, necesita especificar la parte de la estructura de la que está hablando en un momento dado. XPATH es ese lenguaje de especificación. Es un mecanismo de direccionamiento que le permite especificar una ruta a un elemento de forma que, por ejemplo, <article><title> puede ser distinguido de <person><title>. De esa forma, puede describir diferentes tipos de traducciones para los diferentes elementos <title>.

El resto de esta sección describe los paquetes que componen las IPAs de Transformación de JAXP.

Paquetes de Transformación de JAXP

Aquí tiene una descripción de los paquets que componen las IPAs de Transformación de JAXP:

javax.xml.transform

Esta paquete define la clase factoría que usa para obtener un objeto Transformer. Luego configura el transformador con objetos de entrada (fuente) y salida (resultado), e invoca su método transform() para que la transformación suceda. Los objetos fuente y resultado se crean utilizando clases de uno de los tres paquetes.

javax.xml.transform.dom

Define las clases DOMSource y DOMResult, que le permiten usar un DOM como entrada a o salida de una transformación.

javax.xml.transform.sax

Define las clases SAXSource y SAXResult, que le permiten usar un generador de eventos SAX como entrada a una transformación, o entregar eventos SAX como salida a un procesador SAX de eventos.

javax.xml.transform.stream

Define las clases StreamSource y StreamResult, que le permiten usar un flujo de E/S como una entrada a o una salida de una transformación.
