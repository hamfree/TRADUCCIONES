Descripción de los Paquetes

Las IPAs de SAX y DOM están definidas por el grupo XML-DEV y por el W3C, respectivamente. Las librerías que definen estas IPAs son las siguientes:

  • javax.xml.parsers: Las IPAs de JAXP, que proporcionan una interfaz común para analizadores de SAX y DOM de diferentes fabricantes.

  • org.w3c.dom: Define la clase Document (un DOM) así como las clases para todos los componentes de un DOM.

  • org.xml.sax: Define las IPAs básicas de SAX.

  • javax.xml.transform: Define las IPAs XSLT que le permiten transformar XML en otras formas.

  • javax.xml.stream: Proporciona las IPAs de transformación específicas de StAX.

La IPA Simple para XML (SAX) es el mecanismo de acceso en serie impulsado por eventos que realiza el procesamiento elemento por elemento. La IPA para este nivel lee y escribe XML a un repositorio de datos o a la web. Para aplicaciones del lado del servidor y de alto rendimiento, querrá entender completamente este nivel. Pero para muchas aplicaciones, un entendimiento mínimo será suficiente.

La IPA de DOM es generalmente una IPA más fácil de usar. Proporciona una familiar estructura en árbol de objetos. Puede usar la IPA de DOM para manipular la jerarquía de objetos de aplicación que encapsula. La IPA de DOM es ideal para aplicaciones interactivas porque el modelo de objetos entero está presente en memoria, donde puede ser accedido y manipulado por el usuario.

Por otro lado, construir el DOM requiere leer la estructura XML entera y mantener el árbol de objetos en memoria, así que es mucho más intensivo en UCP y memoria. Por esta razón, la IPA de SAX tiende a ser preferida para aplicaciones del lado del servidor y filtros de datos que no requieren una representanción en memoria de los datos.

Las IPAs XSLT definidas en javax.xml.transform le permiten escribir datos XML a un fichero o convertirlos a otras formas. Como se ve en la sección XSLT de este tutorial, puede incluso usarla en conjunción con las IPAs de SAX para convertir datos heredados en XML.

Finalmente, las IPAs StAX definidas en javax.xml.stream proporcionan una IPA de análisis y extracción, impulsada por eventos, y basada en tecnología de flujos Java para leer y escribir documentos XML.
