Definiciones de Límites de Procesamiento

La siguiente lista describe los Límites de Procesamiento de XML de JAXP admitidos en el KDJ. Estos límites se pueden especificar a través de las IPAs de Factoría, las Propiedades del Sistema, y/o el fichero jaxp.properties.

entityExpansionLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/entityExpansionLimit
Definición: Limita el número de expansiones de la entidad.
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 64,000. Este es el límite actual.
Propiedad del sistema: jdk.xml.entityExpansionLimit. La entityExpansionLimit se admite también por compatibilidad con versiones anteriores. Se recomienda usar jdk.xml.entityExpansionLimit.

elementAttributeLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/elementAttributeLimit
Definición: Limita el número de atributos que puede tener un elemento.
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 10,000
Propiedad del sistema: jdk.xml.elementAttributeLimit. Se admite también el elementAttributeLimit por compatibilidad con versiones anteriores. Se recomienda usar jdk.xml.elementAttributeLimit.

maxOccurLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/maxOccurLimit
Definición: Limita el número de nodos del modelo de contenido que pueden crearse cuando se construye una gramática para un Esquema XML del W3C que contiene maxOccurs atributos con valores distintos de "unbounded".
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 5000
Propiedad del sistema: jdk.xml.maxOccurLimit. Se admite tambien el maxOccurLimit por compatibilidad con versiones anteriores. Se recomienda usar jdk.xml.maxOccurLimit.

totalEntitySizeLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/totalEntitySizeLimit
Definición: Limita el tamaño total de todas las entidades que incluyen entidades generales y de parámetro. El tamaño se calcula como una agregación de todas las entidades.
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 5x10^7
Propiedad del sistema: jdk.xml.totalEntitySizeLimit

maxGeneralEntitySizeLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/maxGeneralEntitySizeLimit
Definición: Limita el tamaño máximo de cualquier entidad general. Se recomienda que los usuarios establezcan el límite al número más pequeño posible de forma que los ficheros xml con formato incorrecto puedan ser capturados rápidamente.
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 0, sin límite.
Propiedad del sistema: jdk.xml.maxGeneralEntitySizeLimit

maxParameterEntitySizeLimit

Nombre: http://www.oracle.com/xml/jaxp/properties/maxParameterEntitySizeLimit
Definición: Limita el tamaño máximo de cualquier entidad de parámetro, incluyendo el resultado de anidar múltiples entidades de parámetros. Se recomienda que los usuarios establezcan el límite al número más pequeño posible de forma que los ficheros xml con formato incorrecto puedan ser capturados rápidamente.
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 1,000,000.
Propiedad del sistema: jdk.xml.maxParameterEntitySizeLimit

maxElementDepth

Nombre: http://java.sun.com/xml/jaxp/properties/maxElementDepth
Definición: Limita la profundidad máxima del elemento
Valor: Un entero positivo. 0 se trata como sin límite. Los números negativos se tratan como el 0.
Valor por defecto: 0
Propiedad del sistema: jdk.xml.maxElementDepth

{java.home}/lib/jaxp.properties

Las propiedades del sistema se pueden especificar en el fichero jaxp.properties para definir el comportamiento de todas las invocaciones del KDJ o EEJ. El formato es system-property-name=value. Por ejemplo: 

jdk.xml.maxGeneralEntitySizeLimit=1024
