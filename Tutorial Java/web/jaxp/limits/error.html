Gestión de Errores

Se recomienda que las aplicaciones capturen la excepción org.xml.sax.SAXNotRecognizedException cuando se establecen alguna de las nuevas propiedades de forma que las aplicaciones puedan trabajar apropiadamente en versiones más antiguas. Por ejemplo, el código de ejemplo descargable contienen el método siguiente, isNewPropertySupported, que detecta si el ejemplo se ejecuta con una versión del KDJ que admite la propiedad JDK_GENEAL_ENTITY_SIZE_LIMIT: 

public boolean isNewPropertySupported() {
    try {
        SAXParser parser = getSAXParser(false, false, false);
        parser.setProperty(JDK_GENEAL_ENTITY_SIZE_LIMIT, "10000");
    } catch (ParserConfigurationException ex) {
        fail(ex.getMessage());
    } catch (SAXException ex) {
        String err = ex.getMessage();
        if (err.indexOf("Property '" + JDK_GENEAL_ENTITY_SIZE_LIMIT +
                                       "' is not recognized.") > -1) {
            //expected before this patch
            debugPrint("New limit properties not supported. Samples not run.");
            return false;
        }
    }
    return true;
}

Cuando los ficheros de entrada contienen construcciones que causan una excepción por-encima-del-límte, las aplicaciones pueden comprobar el código de error para determinar la naturaleza del falor. Los códigos de error siguientes se definen para los límites:

  • EntityExpansionLimitExceeded: JAXP00010001
  • ElementAttributeLimit: JAXP00010002
  • MaxEntitySizeLimit: JAXP00010003
  • TotalEntitySizeLimit: JAXP00010004
  • maxElementDepth: JAXP00010006

El código de error tiene el formato siguiente: 

"JAXP" + componentes (dos dígitos) + categoría de error (dos dígitos) + número de secuencia

El código JAXP00010001, por lo tanto, representa el límite de seguridad base JAXP del analizador EntityExpansionLimit.
