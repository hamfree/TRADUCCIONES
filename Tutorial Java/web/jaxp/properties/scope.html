Alcance y Orden

(FSP) javax.xml.XMLConstants#FEATURE_SECURE_PROCESSING es una característica requerida para los procesadors XML incluyendo DOM, SAX, Validación de Esquema, XSLT y XPath. Cuando se establece a true, se recomienda que las implementaciones habiliten las restricciones de acceso como se definen por las nuevas propiedades anteriormente. Por compatibilidad, JAXP 1.5 no habilita las nuevas restricciones, aunque FSP es verdadera de forma predeterminada para DOM, SAX y Validación de Esquema.

Para el KDJ 8, se propone que las nuevas propiedades accessExternal* se establezcan con la cadena vacía cuando FSP se establece explícitamente. Este es solo el caso cuando FSP se establece a través de la IPA, p.e. factory.setFeature(FSP, true). Aunque FSP es verdadero de forma predeterminada para DOM, SAX y Validación de ESquema no se trata como si se estableciera "explícitamente", por lo tanto, el KDJ 8 no establece restricciones de forma predeterminada.

Las propiedades especificadas en el fichero jaxp.properties afectan a todas las invoaciones del KDJ o del EEJ, y sobreescribirán sus valores predeterminados, o aquellas que hayan sido establecidas por FEATURE_SECURE_PROCESSING.

Las propiedades del sistema, cuando se establecen, afectan sólo a una invocación, y sobreescribirán las configuracones predeterminadas o aquellas establecidas en jaxp.properties, o aquellas que pueden haber sido fijadas por FEATURE_SECURE_PROCESSING.

Las propiedades JAXP especificadas a través de las factorías JAXP o SAXParser tienen preferencia sobre las propiedades del sistema, el fichero jaxp.properties, como también javax.xml.XMLConstants#FEATURE_SECURE_PROCESSING.

Las propiedades nuevas de JAXP no tienen ningún efecto sobre las construcciones relevantes que intentan restringir en las siguientes situaciones:

  • Cuando hay un resolver y la fuente devuelta por el resolver no es nula. Esto aplica a resolvers de entidades que pueden ser establecidos en analizadores SAX y DOM, resolvers XML en analizadores StAX, LSResourceResolver en SchemaFactory, un Validator o ValidatorHandler, o URIResolver en un transformador.
  • Cuando se crea un esquema explícitamente llamando al método newSchema de SchemaFactory.
  • Cuando no se requieren recursos externos. Por ejemplo, las siguientes características/propiedades son admitidas por la implementación de referencia y pueden usarse para indicar al procesador que no cargue la DTD externa o resuelva entidades externas. 
    http://apache.org/xml/features/disallow-doctype-decl true
http://apache.org/xml/features/nonvalidating/load-external-dtd false
http://xml.org/sax/features/external-general-entities false
http://xml.org/sax/features/external-parameter-entities false
