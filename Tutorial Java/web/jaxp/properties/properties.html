Nuevas Propiedades

JAXP 1.5 define tres nuevas propiedades que se pueden usar para regular si los procesadores XML resuelven o no los recursos externos como se enumeraron anteriormente. Las propiedades son:

  • javax.xml.XMLConstants.ACCESS_EXTERNAL_DTD
  • javax.xml.XMLConstants.ACCESS_EXTERNAL_SCHEMA
  • javax.xml.XMLConstants.ACCESS_EXTERNAL_STYLESHEET

Estas propiedades de la IPA tienen sus propiedades de sistema correspondientes y jaxp.properties.

ACCESS_EXTERNAL_DTD

Nombre: http://javax.xml.XMLConstants/property/accessExternalDTD
Definición: restringe el acceso a DTDs externas y Referencias de Entidades externas a los protocolos especificados.
Valor: vea Valores de las Propiedades
Valor predeterminado: all, conexión permitida para todos los protocolos.
Propiedad del sistema: javax.xml.accessExternalDTD

ACCESS_EXTERNAL_SCHEMA

Nombre: http://javax.xml.XMLConstants/property/accessExternalSchema
Definición: restringe el acceso a los protocolos especificados al conjunto de referencias externas establecidas por el atributo schemaLocation y los elementos Import e Include.
Valor: vea Valores de las Propiedades
Valor predeterminado: all, conexión permitida para todos los protocolos.
Propiedad del sistema: javax.xml.accessExternalSchema

ACCESS_EXTERNAL_STYLESHEET

Nombre: http://javax.xml.XMLConstants/property/accessExternalStylesheet
Definición: restringe el acceso a los protocolos especificados al conjunto de referencias externas establecido por la instrucción de procesado de la hoja de destilo, función del documento y elementos Import e Include.
Valor: vea Valores de las Propiedades
Valor predeterminado: all, conexión permitida para todos los protocolos.
Propiedad del sistema: javax.xml.accessExternalStylesheet

${java.home}/lib/jaxp.properties

Estas propiedades se pueden especificar en jaxp.properties para definir el comportamiento de todas las aplicaciones que usan Java en tiempo de ejecución. El formato es nombre-propiedad=[valor][,valor]*. Por ejemplo: 

javax.xml.accessExternalDTD=file,http

Los nombres de propiedad son los mismo que los de las propiedades del sistema: javax.xml.accessExternalDTD, javax.xml.accessExternalSchema, y javax.xml.accessExternalStylesheet.

Valores de las Propiedades

Todas las propiedades tienen valores en el mismo formato.

Valor: una lista de protocolos separados con coma. Un protocolo es la porción del esquema de una URI, o en el caso del protocolo JAR, "jar" mas la porción de esquema separada por dos puntos. Un esquema se define como:

scheme = alpha *( alpha | digit | "+" | "-" | "." )
donde alpha = a-z and A-Z.

Y el protocolo JAR:
jar[:scheme]

Los protocolos no son sensibles a mayúsculas y minúsculas. Cualquier espacio en blanco como se define en Character.isSpaceChar en el valor será ignorado. Ejemplos de protocolos son file, http, jar:file.

Valor predeterminado: el valor predeterminado es específico de la implementación. En JAXP 1.5 RI, Java SE 7u40 y Java SE 8, el valor predeterminado es all, otorgando permisos a todos los protocolos.

Otorgar todo el acceso: la palabra clave all otroga permiso a todos los protocolos. Por ejemplo establecer javax.xml.accessExternalDTD=all en jaxp.properties permitiría a un sistema trabajar como antes sin restricciones para acceder a DTDs externos y Referencias de Entidades.

Negar cualquier acceso: una cadena vacía, es decir, "", significa que no se otorga ningún permiso a ningún protocolo. Por ejemplo, establecer javax.xml.accessExternalDTD="" en jaxp.properties indicaría a los protocolos JAXP que rechazen cualquier conexión externa.
