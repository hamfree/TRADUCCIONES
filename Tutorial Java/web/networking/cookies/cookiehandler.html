Mecanismo de LLamada de Vuelta del CookieHandler

La gestión de estado de HTTP está implementada en Java SE a través de la clase java.net.CookieHandler. Un objeto CookieHandler proporciona un mecanismo de retrollamda que soporta una implementación de la política de gestión del estado de HTTP en el manejador de protocolo de HTTP. Es decir, las URLs que usan HTTP como protocolo, new URL("http://example.com") por ejemplo, usarán el manejador de protocolo HTTP. Este manejador de protocolo llama de vuelta al objeto CookieHandler, si está establecido, para manejar la gestión de estado.

La clase CookieHandler es una clase abstracta que tiene dos pares de métodos relacionados. El primer par, getDefault() y setDefault(cookieHandler), son métodos estáticos que le habilitan para descubrir el manejador actual que está instalado e instalar su propio manejador.

No se instala ningún controlador predeterminado y la instalación de un controlador se realiza en todo el sistema. Para aplicaciones ejecutándose dentro de un entorno seguro, es decir, que tienen un gestor de seguridad instalado, debe tener un permiso especial para obtener y establecer el manejador. Para más información, vea java.net.CookieHandler.getDefault.

El segundo par de métodos relacionados, put(uri, responseHeaders) y get(uri, requestHeaders), le habilitan a establecer y obtener todas las cookies aplicables en y desde una cache de cookies para la URI especificada en las cabeceras de respuesta/petición, respectivamente. Estos métodos son abstractos, y una implementación concreta de un CookieHandler debe proporcionar la implementación.

Java Web Start y Java Plug-in tienen un CookieHandler por defecto instalado. Sin embargo, si está ejecutando una aplicación independiente y desea habilitar la administración de estado HTTP, debe configurar un controlador para todo el sistema.
