Gestión del Estado HTTP Con Cookies

El mecanismo de gestión de estado de HTTP especifica una forma de crear una sesión con estado con solicitudes y respuestas HTTP.

Generalmente, los pares solicitud/respuesta de HTTP son independientes unos de otros. Sin embargo, el mecanismo de gestión de estado habilita a los clientes y servidores que pueden intercambiar información de estado colocar estos pares en un contexto más grande, el cual se llama una sesión. La información de estado usada para crear y mantener la sesión se llama una cookie.

Una cookie es una pieza de datos que puede ser almacenada en una caché del navegador. Si visita un sitio web y entonces lo revisita, los datos de la cookie puede usarse para identificarle como un visitante de vuelta. Las cookies permiten que se recuerde información de estado, como un carrito de compras en línea. Una cookie puede ser de corta duración y contener datos para una sola sesión web, es decir, hasta que cierre el navegador, o una cookie puede durar más tiempo y almacenar datos durante una semana o un año.

Par más información sobre la gestión del estado HTTP, vea RFC 2965: HTTP State Management Mechanism.
