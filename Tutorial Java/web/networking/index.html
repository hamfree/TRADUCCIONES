Recorrido: Personalizando la Red

La plataforma Java es muy apreciada en parte debido a su idoneidad para escribir programas que utilizan e interactúan con los recursos de Internet y la World Wide Web. De hecho, los navegadores compatibles con Java utilizan esta capacidad de la plataforma Java al extremo para transportar y ejecutar applets a través de Internet.

Este recorrido le guía a través de las complejidades de la escritura de aplicaciones y applets de Java que pueden utilizarse en Internet.

 Visión General de las Redes tiene dos secciones. La primera describe las capacidades de red de la plataforma Java que usted ha podido estar usando sin darse cuenta de que está usando la red. La segunda ofrece una breve introducción de la red para que se familiarize con los términos y conceptos que debe entender antes de leer cómo usar las URLs, los sockets, y los datagramas.

  Trabajando con URLs describe como sus programas Java pueden usar URLs para acceder a la información en Internet. Una URL (Localizador Uniforme de Recursos) es la dirección de un recurso en Internet. Sus programas Java pueden usar URLs para conectar y recuperar información a través de una red. Esta lección ofrece una definición más completa de una URL y muestra cómo crear y analizar una URL, cómo abrir una conexión a una URL, y cómo leer y escribir con esa conexión.

 Todo Sobre Sockets explica cómo usar los sockets de forma que sus programas puedan comunicarse con otros programas a través de la red. Un socket es un punto final de un enlace de comunicación bidireccional entre dos programas que se ejecutan en la red. Esta lección le muestra cómo un cliente puede conectar a un servidor estándar, el servidor Echo, y comunicar con él a través de un socket. Después le guía a través de los detalles de un ejemplo completo de cliente/servidor, el cual le muestra como implementar tanto el lado cliente como el lado servidor de un par cliente/servidor.

 Todo Sobre los Datagramas le lleva paso a paso a través de un ejemplo simple de cliente/ servidor que usa datagramas para comunicarse. Luego, lo desafía a reescribir el ejemplo utilizando un socket de multidifusión.

 Acceso Prográmatico a los Parámetros de Red explicar porqué podría querer acceder a los parámetros del interfaz de red y cómo hacer eso. Le da ejemplos de cómo listar todas las direcciones IP asignadas a la máquina como también otra información útil tal como si el interfaz está funcionando.

 Trabajando con Cookies describe como las cookies se usan para crear una sesión entre un cliente y un servidoor, y cómo puede aprovecharse de las cookies en sus conexiones HTTP con URL.

Consideraciones de seguridad: 

Dese cuenta de que las comunicaciones a través de la red están sujetas a la aprobación del gestor de seguridad actual. El Gestor de Seguridad describe qué es un gestor de seguridad y cómo impacta en sus aplicaciones. Para información general sobre las características de seguridad ofrecidas por el KDJ, diríjase a Características de Seguridad en Java SE.

Los programas de ejemplo en las lecciones siguientes que cubren URLs, sockets y datagramas son aplicaciones independientes, que por defecto, no tienen gestor de seguridad. Si convierte estas aplicaciones en applets, pueden ser incapaces de comunicar a través de la red, dependiendo del navegador o visor en el cual se ejecuten. Vea Qué Pueden Hacer los Applets y Qué No para información sobre las restricciones de seguridad impuestas a los applets.
