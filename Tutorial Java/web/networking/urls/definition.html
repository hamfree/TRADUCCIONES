¿Qué es una URL?

Si ha estado navegando en la Web, indudablemente ha escuchado el término URL y ha usado URLs para acceder a páginas HTML desde la Web.

Es a menudo más fácil, aunque no del todo exacto, pensar de una URL como el nombre de un fichero en la World Wide Web ya que la mayoría de las URLs se refieren a un fichero en alguna máquina en la red. Sin embargo, recuerde que las URLs también pueden apuntar a otros recursos en la red, tales como consultas de bases de datos y salida de comandos.

Definición: 

URL es un acrónimo para Localizador Uniforme de Recurso y es una referencia (una dirección) a un recurso en Internet.

Una URL tiene dos componentes principales:

  • Identificador de protocolo: Para la URL http://example.com, el identificador de protocolo es http.
  • Nombre de recurso: Para la URL http://example.com, el nombre de recurso es example.com.

Dese cuenta de que el identificador de protocolo y el nombre de recurso están separados por dos puntos y dos barras diagonales. El identificador de protocolo indica el nombre del protocolo a usarse para obtener el recurso. El ejemplo usa el Protocolo de Transferencia de Hipertexto (HTTP), el cual se usa habitualmente para servir documentos de hipertexto. HTTP es simplemente uno de los muchos protocolos diferentes usados para acceder diferentes tipos de recursos en la red. Otros protocolos incluyen el Protocolo de Transferencia de Ficheros (FTP), Gopher, Ficheros, y Noticias.

El nombre del recurso es la dirección completa del recurso. El formato del nombre del recurso depende enteramente del protocolo utilizado, pero para muchos protocolos, incluído HTTP, el nombre del recurso contiene uno o más de los componentes siguientes:

Nombre del Anfitrión
El nombre de la máquina donde reside el recurso.
Nombre del fichero
La ruta al fichero en la máquina.
Número de Puerto
El número de puerto al cual hay que conectar (habitualmente opcional).
Referencia
Una referencia a un marcador con nombre dentro del recurso que usualmente identifica una ubicación específica dentro de un fichero (habitualmente opcional).

Para muchso protocolos, el nombre del anfitrión y el nombre del fichero son obligatorios, mientras que el número de puerto y la referencia son opcionales. Por ejemplo, el nombre de recurso para una URL de HTTP debe especificar un servidor en la red (Nombre del Anfitrión) y la ruta al documento en esa máquina (Nombre de Fichero); también puede especificar un número de puerto y una referencia.
