Analizando una URL

La clase URL proporciona varios métodos que le permiten preguntar a los objetos URL. Puede obtener el protocolo, la autoridad, el nombre de la máquina, el número de puerto, la ruta, la cadena de consulta, el nombre del fichero, y la referencia desde una URL usando estos métodos de acceso:

getProtocol
Devuelve el componente del identificador de protocolo de la URL.
getAuthority
Devuelve el componente de la autoridad de la URL.
getHost
Devuelve el componente del nombre de la máquina de la URL.
getPort
Devuelve el componente del número de puerto de la URL. El método getPort devuelve un entero que es el número de puerto. Si no está establecido el puerto, getPort devuelve -1.
getPath
Devuelve el componente de la ruta de esta URL.
getQuery
Devuelve el componente de cadena de consulta de esta URL.
getFile
Devuelve el componente del nombre de fichero de la URL. El método getFile devuelve lo mismo que getPath, mas la concatenación del valor de getQuery, si existe.
getRef
Devuelve el componente referencia de la URL.
Nota: 

Recuerde que no todas las direcciones URL contienen estos componentes. La clase URL proporciona estos métodos porque las URLs HTTP contienen estos componentes y son quizás las URLs más usadas habitualmente. La clase URL está algo centrada en HTTP.

Puede usar estos métodos getXXX para obtener información sobre la URL independientemente del constructor que usó para crear el objeto URL.

La clase URL, junto con estos métodos de acceso, ¡le liberan para siempre de tener que analizar URLs de nuevo! Dada cualquier cadena con una especificación de una URL, simplemente cree un nuevo objeto URL y llame a cualquiera de los métodos de acceso para la información que necesite. Este pequeño programa de ejemplo crea una URL desde una cadena de especificacón y entonces usa los métodos de acceso del objeto URL para analizar la URL: 

import java.net.*;
import java.io.*;

public class ParseURL {
    public static void main(String[] args) throws Exception {

        URL aURL = new URL("http://example.com:80/docs/books/tutorial"
                           + "/index.html?name=networking#DOWNLOADING");

        System.out.println("protocol = " + aURL.getProtocol());
        System.out.println("authority = " + aURL.getAuthority());
        System.out.println("host = " + aURL.getHost());
        System.out.println("port = " + aURL.getPort());
        System.out.println("path = " + aURL.getPath());
        System.out.println("query = " + aURL.getQuery());
        System.out.println("filename = " + aURL.getFile());
        System.out.println("ref = " + aURL.getRef());
    }
}

Aquí está la salida visualizada por el programa: 

protocol = http
authority = example.com:80
host = example.com
port = 80
path = /docs/books/tutorial/index.html
query = name=networking
filename = /docs/books/tutorial/index.html?name=networking
ref = DOWNLOADING
