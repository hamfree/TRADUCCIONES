Creando una URL

La forma más fácil de crear un objeto URL es desde un String que representa la forma legible por humanos de la dirección URL. Esta es habitualmente la forma que cualquier persona usará para una URL. En su programa Java, puede usar un String que contenga este texto para crear un objeto URL: 

URL myURL = new URL("http://example.com/");

El objeto URL creado arriba representa una URL absoluta. Una URL absoluta contiene toda la información necesaria para alcanzar el recurso en cuestión. Puede también crear objetos URL desde una dirección URL relativa.

Creando una URL Relativa a Otra

Una URL relativa sólo contiene información suficiente para alcanzar el recurso en relación con (o en el contexto de) otra URL.

Las especificaciones de URL relativas se usan a menudos dentro de los ficheros HTML. Por ejemplo, suponga que escribe un fichero HTML llamado JoesHomePage.html. Dentro de esta página, hay enlaces a otras páginas, PicturesOfMe.html y MyKids.html, que están en la misma máquina y en el mismo directorio que JoesHomePage.html. Los enlaces a PicturesOfMe.html y MyKids.html desde JoesHomePage.html se podrían especificar simplemente como nombres de fichero, tal que así: 

<a href="PicturesOfMe.html">Pictures of Me</a>
<a href="MyKids.html">Pictures of My Kids</a>

Estas direcciones URL son URLs relativas. Esto es, las URLs se especifican relativas al fichero en el cual están contenidas — JoesHomePage.html.

En sus programas Java, puede crear un objeto URL desde una especificación de URL relativa. Por ejemplo, suponga que conoce dos URLs en el sitio example.com: 

http://example.com/pages/page1.html
http://example.com/pages/page2.html

Puede crear objetos URL para estas páginas relativas a su URL base común: http://example.com/pages/ tal que así: 

URL myURL = new URL("http://example.com/pages/");
URL page1URL = new URL(myURL, "page1.html");
URL page2URL = new URL(myURL, "page2.html");

Este trozo de código usa el constructor URL que le permite crear un objeto URL desde otro objeto URL (la base) y una especificacón de URL relativa. La forma general de este constructor es: 

URL(URL baseURL, String relativeURL)

El primer argumento es un objeto URL que especifica la base de la nueva URL. El segundo argumento es un String que especifica el resto del nombre del recurso relativo a la base. Si baseURL es nulo, entonces este constructor trata relativeURL como una especificación de URL absoluta. A la inversa, si relativeURL es una especificación de URL absoluta, entonces el constructor ignora baseURL.

Este constructor es útil también para crear objetos URL para marcadores con nombre (también llamados referencias) dentro de un fichero. Por ejemplo, suponga que el fichero page1.html tiene un marcador con nombre llamado BOTTOM en el final del fichero. Puede usar el constructor de URL relativa para crear un objeto URL para él tal que así: 

URL page1BottomURL = new URL(page1URL,"#BOTTOM");

Otros Constructores URL

La clase URL proporciona dos constructores adicionales para crear un objeto URL. Estos constructores son útiles cuando está trabajando con URLs, tales como las URLs HTTP, que tienen nombre de anfitrión, nombre de fichero, número de puerto, y componentes de referencia en la porción del nombre de recurso de la URL. Estos dos constructores son útiles cuando no tiene un String conteniendo la especificación completa de la URL, pero conoce varios componentes de la URL.

Por ejemplo, suponga que diseña un panel de navegación de red similar a un panel de navegación de ficheros que permite a los usuarios a elegir el protocolo, el nombre del anfitrión, el número de puerto, y el nombre del fichero. Puede construir un URL desde los componentes del panel. El primer constructor crea un objeto URL desde un protocolo, un nombre de anfitrión, y un nombre de fichero. El trozo de código siguiente crea una URL al fichero page1.html en el sitio example.com: 

new URL("http", "example.com", "/pages/page1.html");

Esto es equivalente a 

new URL("http://example.com/pages/page1.html");

El primer argumento es el protocolo, el segundo es el nombre del anfitrión o máquina, y el último es la ruta del fichero. Note que el nombre del fichero contiene una barra diagonal en el comienzo. Esto indica que el nombre del fichero se especifica desde la raíz de la máquina o anfitrión.

El constructor URL final agrega el número de puerto a la lista de argumentos usados en el constructor previo: 

URL gamelan = new URL("http", "example.com", 80, "pages/page1.html");

Esto crea un objeto URL para la URL siguiente: 

http://example.com:80/pages/page1.html

Si construye un objeto URL usando uno de estos constructores, puede obtener un String conteniendo la dirección completa de la URL usando el método toString del objeto URL o el método equivalente toExternalForm.

Direcciones de URL con Caracteres Especiales

Algunas direcciones de URL contienen caracteres especiales, por ejemplo el carácter espacio. Como este: 

http://example.com/hello world/

Para hacer estos caracteres legales necesitan ser codificados antes de pasarlos al constructor de URL. 

URL url = new URL("http://example.com/hello%20world");

Codificar el carácter especial en este ejemplo es fácil ya que sólo hay un carácter que necesita codificación, pero para direcciones URL que tiene varios de estos caracteres o si no está seguro cuando escribe su código de qué direcciones URL necesitará acceder, puede usar los constructores multi-argumento de la clase java.net.URI para que se ocupe automaticámente de la codificación por usted. 

URI uri = new URI("http", "example.com", "/hello world/", "");

Y después convierta el objeto URI en un objeto URL. 

URL url = uri.toURL();

MalformedURLException

Cada uno de los cuatro constructores URL lanzan una MalformedURLException si los argumentos del constructor se refieren a null o un protocolo desconocido. Habitualmente, quiere capturar y manejar esta excepción embebiendo sus sentencias del constructor URL en un par try/catch, como esto: 

try {
    URL myURL = new URL(...);
} 
catch (MalformedURLException e) {
    // exception handler code here
    // ...
}

Vea Excepciones para información sobre el manejo de excepciones.

Nota: 

Los objetos URLs son de "escritura única". Una vez ha creado un objeto URL, no puede cambiar ninguno de sus atributos (protocolo, nombre de máquina, nombre de fichero, o número de puerto).
