Lección: Trabajando con URLs

URL es el acrónimo para Localizador Uniforme de Recurso. Es una una referencia (una dirección) a un recurso en Internet. Usted proporciona URLs a su navegador web favorito de forma que pueda localizar ficheros en Internet de la misma forma que usted proporciona direcciones en las cartas de forma que la oficina de correos puede localizar sus destinatarios.

Los programas Java que interactúan con Internet también pueden usar URLs para encontrar los recursos en Internet a los que desean acceder. Los programas de Java pueden usar un clase llamada URL en el paquete java.net para representar una dirección URL.

Nota de Terminología: 

El término URL puede ser ambiguo. Puede referirse a una dirección de Internet o a un objeto URL en un programa Java. Cuando el significado de URL necesita ser específico, este texto utiliza "dirección URL" para hacer referencia a una dirección de Internet y "URL objeto" para hacer referencia a una instancia de la clase URL en un programa.

¿Qué es una URL?

Una URL toma la forma de una cadena que describe cómo encontrar un recurso en Internet. Las URLs tienen dos componentes principales: el protocolo necesitado para acceder al recurso y la ubicación del recurso.

Creando una URL

Dentro de sus programas Java, puede crear un objeto URL que representa una dirección URL. El objeto URL siempre se refiere a una URL absoluta pero puede ser construído desde una URL absoluta, una URL relativa, o desde componentes de la URL.

Analizando una URL

Atrás quedaron los días de analizar una URL para averiguar el nombre del host, el nombre del archivo y otra información. ¡Con un objeto URL válido puede llamar a cualquiera de sus métodos de acceso para obtener toda esa información desde la URL sin hacer ningún análisis de la cadena!

Conectando a una URL

Si quiere hacer más que simplemente leer desde una URL, puede conectarse a ella llamando a openConnection() en la URL. El método openConnection() devuelve un objeto URLConnection que puede usar para comunicaciones más generales con la URL, tales como leer de ella, escribir en ella, o preguntarla por el contenido y otra información.

Leyendo de y Escribiendo a una URLConnection

Algunas URLs, tales como muchas que están conectadas a scripts cgi-bin, le permiten (o incluso le requieren) escribir información a la URL. Por ejemplo, un script de búsqueda puede requerir que se escriban datos detallados de consulta en la URL antes de que la búsqueda pueda ser realizada.
