Lección: Todo Sobre Sockets

URLs y URLConnections proporcionan un mecanismo relativamente de alto nivel para acceder a los recursos en Internet. Algunas veces sus programas requieren comunicación de red de bajo nivel, por ejemplo, cuando quiera escribir una aplicación cliente-servidor.

En aplicaciones cliente-servidor, el servidor proporciona algún servicio, tales como procesar consultas de bases de datos o enviando los precios actuales de las acciones. El cliente usa el servicio proporcionado por el servidor, ya sea visualizando los resultados de la consulta de la base de datos al usuario o haciendo recomendaciones de compras de acciones a un inversor. La comunicación que ocurre entre el cliente y el servidor debe ser confiable. Es decir, no se pueden eliminar datos y deben llegar al lado del cliente en el mismo orden en que los envió el servidor.

TCP proporciona un canal de comunicaciones confiable y punto a punto que las aplicaciones cliente-servidor en Internet usan para comunicarse entre sí. Para comunicar sobre TCP, un programa cliente y un programa servidor establecen una conexión entre sí. Cada programa une un socket a su extremo de la conexión. Para comunicar, el cliente y el servidor leen y escriben en el socket vinculado a la conexión.

¿Qué es un Socket?

Un socket es un punto final de un enlace de comunicación bidireccional entre dos programas que se ejecutan en red. Las clases de Socket se usan para representar la conexión entre un programa cliente y un programa servidor. El paquete java.net proporciona dos clases -- Socket y ServerSocket -- que implementan el lado cliente de la conexión y el lado servidor de la conexión, respectivamente.
