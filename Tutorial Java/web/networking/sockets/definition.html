¿Qué es un Socket?

Normalmente, un servidor se ejecuta en una computadora específica y tiene un socket que está vinculado a un número de puerto específico. El servidor simplemente espera, escuchando en el socket por un cliente que haga una petición de conexión.

En el lado del cliente: el cliente conoce el nombre de máquina de la máquina en la cual se está ejecutando el servidor y el número de puerto en el cual el servidor está escuchando. Para hacer una petición de conexión, el cliente intenta reunirse con el servidor en la máquina y el puerto del servidor. El cliente también necesita identificarse a sí mismo al servidor así que se vincula a un número de puerto local que usará durante esta conexión. Este es usualmente asignado por el sistema.

Si todo va bien, el servidor acepta la conexión. Tras la aceptación, el servidor obtiene un nuevo socket vinculado al mismo puerto local y también tiene su punto final remoto configurado en la dirección y el puerto del cliente. Necesita un nuevo socket de forma que puede continuar escuchando al socket original para las peticiones de conexión mientras atiende a las necesidades del cliente conectado.

En el lado cliente, si la conexión es aceptada, se crea correctamente un socket y el cliente puede utilizar el socket para comunicarse con el servidor.

El cliente y el servidor pueden ahora comunicarse escribiendo o leyendo desde sus sockets.

Definición: 

Un socket es un punto final de un enlace de comunicación bidireccional entre dos programas que se ejecutan en red. Un socket está vinculado a un número de puerto de forma que la capa TCP pueda identificar la aplicación a la que están destinados los datos.

Un punto final es una combinación de una dirección IP y un número de puerto. Cada conexión TCP puede ser identificada de forma única por sus dos punto finales. De esa forma pueden tener múltiples conexiones entre su máquina y el servidor.

El paquete java.net en la plataforma Java proporciona una clase, Socket, que implementa un lado de una conexión de dos vías entre su programa Java y otro programa en la red. La clase Socket se asienta sobre una implementación dependiente de la plataforma, ocultando los detalles de cualquier sistema en particular de su programa Java. Al utilizar la clase java.net.Socket en lugar de depender del código nativo, sus programas Java pueden comunicarse a través de la red de forma independiente de la plataforma.

Adicionalmente, java.net incluye la clase ServerSocket, la cual implementa un socket que los servidores pueden usar para escuchar y aceptar conexiones de los clientes. Esta lección muestra cómo usar las clases Socket y ServerSocket.

Si está intentando conectar la la Web, la clase URL y clases relacionadas (URLConnection, URLEncoder) son probablemente más apropiadas que las clases de socket. En realidad, las URLs son una conexión de alto nivel relativa a la Web y usa sockets como parte de la implementación subyacente. Vea Trabajando con URLs para información sobre conectar a la Web vía URLs.
