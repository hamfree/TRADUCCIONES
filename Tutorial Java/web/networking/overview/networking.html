Fundamentos de Redes

Las computadoras que funcionan en Internet se comunican entre sí mediante el Protocolo de Control de Transmisión (Nota del Traductor: Transmission Control Protocol o TCP en inglés, se usará la abreviatura en inglés) o el Protocolo de Datagramas de Usuario (Nota del Traductor: User Datagram Protocol o UDP en inglés, se usará la abreviatura en inglés) , como ilustra este diagrama:

Cuando escribe programas de Java que se comunican a través de la red, usted está programando en la capa de aplicación. Habitualmente, no necesita preocuparse con las capas de TCP y UDP. En vez de eso, puede usar las clases en el paquete java.net. Estas clases proporcional una comunicación de red independiente del sistema. Sin embargi, para decidir qué clases de Java deben usar sus programas, necesita entender cómo difieren TCP y UDP.

TCP

Cuando dos aplicaciones quieren comunicarse entre sí de manera confiable, establecen una conexión y envían datos de un lado a otro a través de esa conexión. Esto es análogo a hacer una llamada telefónica. Si quiere hablar a la tía Beatrice en Kentucky, se establece una conexión cuando marca su número de teléfono y ella responde. Usted envía datos de una lado a otro a través de la conexión hablando entre sí a través de las líneas telefónicas. Al igual que la compañía telefónica, TCP garantiza que los datos enviados desde un extremo de la conexión lleguen al otro extremo y en el mismo orden en el que se enviaron. De lo contrario, se informa de un error.

TCP proporciona un canal punto-a-punto para aplicaciones que requieren comunicaciones confiables. El Protoco de Transferencia de Hipertexto (HTTP), el Protocolo de Transferencia de Ficheros (FTP), y Telnet son todos ejemplos de aplicaciones que requieren un canal de comunicaciones confiable. El orden en el que los datos son enviados y recibidos a través de la red es crítico para el éxito de estas aplicaciones. Cuando se usa HTTP para leer desde una URL, los datos deben ser recibidos en el orden en el que fueron enviados. De lo contrario, terminará con un archivo HTML desordenado, un archivo zip dañado, o alguna otra información no válida.

Definición: 

TCP (Protocolo de Control de Transmisión) es un protocolo basado en la conexión que proporciona un flujo confiable de datos entre dos computadoras.

UDP

El protocolo UDP proporciona comunicación que no está garantizada entre dos aplicaciones a través de la red. UDP no está basada en conexiones como TCP. Más bien, envía paquetes independientes de datos, llamados datagramas, de una aplicación a otra. Enviar datagramas es muy parecido a mandar una carta a través del servicio postal: El orden de entrega no es importante y no está garantizado, y cada mensaje es independiente de los demás.

Definición: 

UDP (Protocolo de Datagramas de Usuario) es un protocolo que envía paquetes independientes de datos, llamados datagramas, de una computadora a otra sin garantías sobre su llegada. UDP no está basado en la conexión como TCP.

Para muchas aplicaciones, la garantía de confiabilidad es crítica para el éxito de la transferencia de información de un extremo de la conexión al otro. Sin embargo, otras formas de comunicación no requieren tales estándares estrictos. De hecho, pueden ralentizarse debido a la sobrecarga adicional o la conexión confiable puede invalidar el servicio por completo.

Considere, por ejemplo, un servidor de tiempo que envía la hora actual a su cliente cuando se le requiere a hacerlo. Si el cliente pierde un paquete, realmente no tiene sentido reenviarlo porque la hora será incorrecta cuando el cliente lo reciba en el segundo intento. Si el cliente hace dos peticiones y recibe paquetes del servidor desordenados, realmente no importa porque el cliente puede darse cuenta de que los paquetes están desordenados y hacer otra petición. La confiabilidad de TCP no es necesaria en esta instancia porque causa una degradación del rendimiento y puede obstaculizar la utilidad del servicio.

Otro ejemplo de un servicio que no necesita la garantía de una canal confiable es el comando ping. El propósito del comando ping es comprobar la comunicación entre dos programas a través de la red. De hecho, ping necesita saber sobre paquetes caídos o desordenados para determinar lo buena o mal que es la conexión. Un canal confiable invalidaría este servicio por completo.

El protocolo UDP proporciona una comunicación que no está garantizada entre dos aplicaciones a través de la red. UDP no está basado en la conexión como TCP. Más bien, envía paquetes de datos independientes de una aplicación a otra. Enviar datagramas es muy parecido a enviar una carta a través del servicio postal: el orden de entrega no es importante y no está garantizado, y cada mensaje es independiente de los demás.

Nota: 

Muchos cortafueegos y enrutadores han sido configurados para no permitir paquetes UDP. Si usted está teniendo problemas conectando a un servicio fuera de su cortafuegos, o si los clientes tienen problemas conectando a su servicio, pregunte a su administrador de sistemas si se permite UDP.

Entendiendo los Puertos

En términos generales, una computadora tiene una única conexión física a la red. Todos los datos destinados a una computadora en particular llegan a través de esa conexión. Sin embargo, los datos pueden estar destinados a diferentes aplicaciones que se ejecuten en la computadora. Entonces, ¿cómo sabe la computadora a qué aplicación enviar los datos? A través del uso de puertos.

Los datos transmitidos a través de Internet se acompañan con información de direccionamiento que identifica el ordenador y el puerto al que están destinados. La computadora se identifica por su dirección IP de 32 bits, que IP utiliza para enviar los datos a la computadora correcta en la red. Los puertos se identifican por un número de 16 bits, que TCP y UDP utilizan para entregar los datos a la aplicación correcta.

En comunicaciones basadas en la conexión tales como TCP, una aplicación servidor enlaza un socket a un número de puerto específico. Esto tiene el efecto de registrar el servidor con el sistema para recibir todos los datos destinados para ese puerto. Definición: 

Los protocolos TCP y UDP usan puertos para asignar los datos entrantes a un proceso particular que se ejecuta en un ordenador.

Los números de los puertos tienen un rango de 0 a 65.535 porque los puertos se representan por números de 16 bits. Los números de puertos en el rango de 0 a 1023 están restringidos; están reservandos para servicios bien conocidos como HTTP y FTP y otros servicios de sistema. Estos puertos se llaman puertos bien conocidos. Sus aplicaciones no deberían intentar vincularse a ellos.

Clases de Red en el KDJ

A través de las clases en java.net, los programas de Java pueden usar TCP o UDP para comunicarse a través de Internet. Todas las clases URL, URLConnection, Socket, y ServerSocket usan TCP para comunicar a través de la red. Las clases DatagramPacket, DatagramSocket, y MulticastSocket son para usar con UDP.
