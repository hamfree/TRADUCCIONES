Lección: Todo Sobre los Datagramas

Algunas aplicaciones que escribe para comunicar a través de la red no requieren el canal confiable punto a punto provisto por TCP. Más bien, sus aplicaciones pueden beneficiarse de un modo de comunicación que ofrece paquetes de información independientes cuya llegada y orden de llegada no están garantizados.

El protocolo UDP proporciona un modo de comunicación de red donde las aplicaciones se envían paquetes de datos, llamados datagramas, entre sí. Un datagrama es un mensaje independiente auto-contenido, enviado a través de la red cuya llegada, tiempo de llegada, y contenido no están garantizados. Las clases DatagramPacket y DatagramSocket en el paquete java.net implementan comunicación con datagramas independiente del sistema usando UDP.

¿Qué es un Datagrama?

Escribiendo un Cliente y Servidor de Datagramas

Esta sección lo guía a través de un ejemplo que contiene dos programas Java que usan datagramas para comunicarse. El lado del servidor es un servidor de cotizaciones que escucha a su DatagramSocket y envía una cotización a un cliente cada vez que el cliente se lo solicita. El lado del cliente es un programa simple que simplemente realiza una petición al servidor.

Difusion a Varios Destinatarios

Esta sección modifica el servidor de cotizaciones de modo que en lugar de enviar una cotización a un solo cliente que lo solicite, el servidor de cotizaciones transmite una cotización cada minuto a todos los clientes que estén escuchando. El programa cliente debe ser modificado en consecuencia.

Nota: 

Muchos cortafuegos y enrutadores están configurados para no permitir paquetes UDP. Si tiene problemas conectando a un servidor fuera de su cortafuegos, o si los clientes tienen problemas conectando a su servicio, pregunte a su administrador de sistemas si se permite UDP.
