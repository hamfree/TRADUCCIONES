Lección: Entendiendo el Protocolo de Sockets Directo

Para entornos informáticos de alto rendimiento, la capacidad de mover datos a través de una red de forma rápida y eficiente es un requisito. Normalmente, estas redes se describen como que requieren un alto rendimiento y baja latencia. Alto rendimiento se refiere a un entorno que puede ofrecer una gran cantidad de capacidad de procesamiento durante un largo período de tiempo. Baja latencia se refiere al retardo mínimo entre el procesamiento de la entrada y el suministro de salida, como es de esperar en una aplicación en tiempo real.

En estos entornos, las redes convencionales que utilizan flujos de sockets pueden crear cuellos de botella cuando se trata de mover datos. Introducida en 1999 por InfiniBand Trade Association, InfiniBand (IB) se creó para abordar la necesidad de computación de alto rendimiento. Una de las características más importantes de IB es el acceso directo a memoria remota (RDMA). RDMA permite mover datos directamente desde la memoria de una computadora a otra, sin pasar por el sistema operativo de ambas computadoras y dando como resultado ganancias significativas en el rendimiento.

El Protocolo directo de sockets (SDP) es un protocolo de red desarrollado para admitir conexiones de flujo a través de la estructura InfiniBand. La compatibilidad con SDP se introdujo en la versión KDJ 7 de Java Platform, Standard Edition ("Java SE Platform") para aplicaciones implementadas en el sistema operativo Solaris ("Solaris OS") y en los sistemas operativos Linux. El sistema operativo Solaris es compatible con SDP e InfiniBand desde Solaris 10 5/08. En Linux, el paquete InfiniBand se llama OFED (OpenFabrics Enterprise Distribution). La versión KDJ 7 es compatible con las versiones 1.4.2 y 1.5 de OFED.
