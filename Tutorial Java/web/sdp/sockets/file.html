Creando un Fichero de Configuración SDP

Un archivo de configuración SDP es un archivo de texto y usted decide en qué parte del sistema de archivos residirá este archivo. Cada línea del archivo de configuración es un comentario o una regla. Un comentario se indica con el carácter de almohadilla (#) al principio de la línea, y todo lo que sigue al carácter de almohadilla se ignorará.

Hay dos tipos de reglas, como sigue:

  • Una regla de "vinculación" indica que el transporte del protocolo SDP debe usarse cuando un socket TCP se vincula a una dirección y un puerto que coinciden con la regla.
  • Una regla de "conexión" indica que el transporte del protocolo SDP debe usarse cuando un socket TCP no vinculado intenta conectarse a una dirección y un puerto que coinciden con la regla.

Una regla tiene la siguiente forma: 

("bind"|"connect")1*LWSP-char(hostname|ipaddress)["/"prefix])1*LWSP-char("*"|port)["-"("*"|port)]
Decodificando la notación: 

1*LWSP-char significa que cualquier número de espacios en blanco lineales (tabulaciones o espacios) puede separar los tokens. Los corchetes indican texto opcional. La notación (xxx | yyy) indica que el token incluirá xxx o yyy, pero no ambos. Los caracteres entre comillas indican texto literal.

La primera palabra clave indica si la regla es una regla de vinculación o de conexión. El siguiente token especifica un nombre de host o una dirección IP literal. Cuando especifica una dirección IP literal, también puede especificar un prefijo, que indica un rango de direcciones IP. El tercer y último token es un número de puerto o un rango de números de puerto.

Considere la siguiente notación en este archivo de configuración de muestra: 

# Utilice SDP al vincularse a 192.0.2.1
bind 192.0.2.1 *

# Utilice SDP cuando se conecte a todos los servicios de aplicaciones en 192.0.2. *
connect 192.0.2.0/24     1024-*

# Utilice SDP cuando se conecte al servidor http o una base de datos en examplecluster
connect examplecluster.example.com   80
connect examplecluster.example.com   3306

La primera regla en el archivo de muestra especifica que SDP se usa para cualquier puerto (*) en la dirección IP local 192.0.2.1. Agregaría una regla de vinculación para cada dirección local asignada a un adaptador InfiniBand. (Un adaptador InfiniBand es el equivalente a una tarjeta de interfaz de red (NIC) para InfiniBand). Si tuviera varios adaptadores IB, usaría una regla de vinculación para cada dirección asignada a esos adaptadores.

La segunda regla del archivo de muestra especifica que siempre que se conecte a 192.0.2. * y el puerto de destino sea 1024 o superior, se utilizará SDP. El prefijo de la dirección IP /24 indica que los primeros 24 bits de la dirección IP de 32 bits deben coincidir con la dirección especificada. Cada parte de la dirección IP utiliza 8 bits, por lo que 24 bits indica que la dirección IP debe coincidir con 192.0.2 y el byte final puede ser cualquier valor. La notación - * en el token de puerto especifica "y superior". Un rango de puertos, como 1024 — 2056, también sería válido e incluiría los puntos finales del rango especificado.

Las reglas finales en el archivo de muestra especifican un nombre de host (examplecluster), primero con el puerto asignado a un servidor http (80) y luego con el puerto asignado a una base de datos (3306). A diferencia de una dirección IP literal, un nombre de host puede traducirse en varias direcciones. Cuando especifica un nombre de host, coincide con todas las direcciones en las que está registrado el nombre de host en el servicio de nombres.
