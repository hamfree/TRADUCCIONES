Problemas Técnicos con SDP

  • Incompatibilidad de IPv4 e IPv6

    La versión 4 del Protocolo de Internet (IPv4) ha sido durante mucho tiempo la versión estándar de la industria del Protocolo de Internet (IP) para la entrega de datos a través de Internet. El Protocolo de Internet versión 6 (IPv6) es el protocolo de capa de Internet de próxima generación. Actualmente se utilizan ambas versiones de IP.

    Las direcciones IPv4 tienen una longitud de 32 bits, están escritas en formato decimal y están separadas por puntos. Las direcciones IPv6 tienen una longitud de 128 bits, están escritas en formato hexadecimal y están separadas por dos puntos. Las direcciones IPv4 no se pueden usar tal cual en IPv6, pero IPv6 admite una clase especial de direcciones: la dirección asignada a IPv4. En una dirección asignada a IPv4, los primeros 80 bits se establecen en cero, los siguientes 16 bits se establecen en 1 y los últimos 32 bits representan la dirección IPv4.

    Por ejemplo, aquí está la misma dirección IP expresada en ambos formatos: 

    dirección IPv4              Dirección asignada a IPv4 (para usar en IPv6)
192.0.2.1                   ::ffff:192.0.2.1

    De forma predeterminada, si IPv6 está habilitado en cualquiera de los adaptadores IB, la plataforma Java utiliza IPv6. Sin embargo, las direcciones asignadas a IPv4 no están disponibles actualmente en el sistema operativo Solaris o en Linux. Por esta razón, si desea utilizar el formato de dirección IPv4 en KDJ 7, debe especificar la propiedad java.net.preferIPv4Stack, como se muestra en este ejemplo: 

    % java -Dcom.sun.sdp.conf=sdp.conf -Djava.net.preferIPv4Stack=true  MiAplicacion
