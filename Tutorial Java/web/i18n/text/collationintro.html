Comparando Cadenas

Las aplicaciones que clasifican el texto realizan comparaciones frecuentes de cadenas. Por ejemplo, un generador de informes realiza comparaciones de cadena cuando ordena una lista de cadenas en orden alfabético.

Si la audiencia de su aplicación se limita a gente que habla inglés, puede realizar probablemente comparaciones de cadena con el método String.compareTo. El método String.compareTo realiza una comparación binaria de los caracteres Unicode dentro de las dos cadenas. Para la mayoría de los idiomas, sin embargo, no se puede confiar en esta comparación binaria para ordenar cadenas, porque los valores Unicode no se corresponden al orden relativo de los caracteres.

Afortunadamente la clase Collator le permite a su aplicación realizar comparaciones de cadena para diferentes idiomas. En esta sección, aprenderá cómo usar la clase Collator cuando se ordena el texto.

Realizando Comparaciones Independientes de la Localización

La reglas de intercalación definen la secuencia de clasificación de las cadenas. Estas reglas varía con la configuración regional, porque varios lenguajes naturales clasifican las palabras de manera diferente. Usando las reglas de intercalación predefinidas proporcionadas por la clase Collator, puede ordenar cadenas de una manera independiente de la configuración regional.

Personalización de Reglas de intercalación

En algunos casos, las reglas de intercalación predefinidas proporcionadas por la clase Collator pueden no funcionar para usted. Por ejemplo, quiere ordenar cadenas en un idioma cuya configuración regional no está admitida por Collator. En esta situación, puede definir sus propias reglas de intercalación, y asignarlas a un objeto RuleBasedCollator.

Mejorando el rendimiento de la intercalación

Con la clase CollationKey, puede incrementar la eficiencia de las comparaciones de cadena. Esta clase convierte objetos String para ordenar claves que sigan las reglas de un Collator dado.
