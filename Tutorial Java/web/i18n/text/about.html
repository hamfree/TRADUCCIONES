Sobre la Clase BreakIterator

La clase BreakIterator es sensible a la configuración regional, porque los límites de texto varían con el idioma. Por ejemplo, las reglas de sintaxis para los saltos de línea no son los mismos para todos los idiomas. Para determinar que configuraciones regionales soporta la clase BreakIterator, invoque el método getAvailableLocales, como sigue: 

Locale[] locales = BreakIterator.getAvailableLocales();

Puede analizar cuatro clases de límites con la clase BreakIterator class: de carácter, de palabra, de oración, y saltos de línea potenciales. Cuando se instancia un BreakIterator, invoque el método de factoria apropiado:

  • getCharacterInstance
  • getWordInstance
  • getSentenceInstance
  • getLineInstance

Cada instancia de BreakIterator puede detectar simplemente un tipo de límite. Si quiere ubicar tanto límites de palabra como carácter, por ejemplo, puede crear dos instancias separadas.

Un BreakIterator tiene un cursor imaginario que apunta al límite actual en una cadena de texto. Puede mover este cursor dentro del texto con los métodos previous y next. Por ejemplo, si usted ha creado un BreakIterator con getWordInstance, el cursor se mueve al siguiente límite de palabra en el texto cada vez que invoca el método next. Los métodos de movimiento del cursor devuelven un entero indicando la posición del límite. Esta posición es el índice del carácter en la cadena de texto que seguiría el límite. Como los índices de cadena, los límites están basados en cero. El primer límite está en 0, y el último límite es la longitud de la cadena.
