Normalizando Texto

La normalización es el proceso por el cual puede realizar ciertas transformaciones de texto para hacerlo reconciliable de una manera que puede que no haya sido antes. Digamos, que le gustaría buscar u ordenar texto, en este caso debe normalizar ese texto para tener en cuenta los puntos de código que deben representarse como el mismo texto.

¿Qué puede ser normalizado? La normalización es aplicable cuando necesita convertir caracteres con marcas diacríticas, cambiar todas las letras entre mayúsculas y minúsculas, descomponer ligaduras o convertir caracteres katakana de ancho medio a caracteres de ancho completo, etc.

De acuerdo con el Anexo nº 15 del Estándar Unicode la IPA de Normalizer admite todas las formas siguientes de normalización de texto Unicode que se definen en java.text.Normalizer.Form:

  • Forma de Normalización D (NFD): Descomposición Canónica
  • Forma de Normalización C (NFC): Descomposición Canónica, seguida por Composición Canónica
  • Forma de Normalización KD (NFKD): Descomposición de Compatibilidad
  • Forma de Normalización KC (NFKC): Descomposición de Compatibilidad, seguida por Composición Canónica

Examinemos como la letra minúscula latina "o" con diéresis puede ser normalizada usando estas formas de normalización:

cómo normalizar una palabra que contiene la letra ö
Palabra original NFC NFD NFKC NFKD
"schön" "schön" "scho\u0308n" "schön" "scho\u0308n"

Puede notar que una palabra original se deja sin cambiar en NFC y NFKC. Esto es porque con NFD y NFKD, los caracteres de composición se asignan a sus descomposiciones canónicas. Pero con NFC y NFKC, las secuencias de combinación de caracteres se asignan a las composiciones, si es posible. No hay composiciones para la diéresis, así que se deja descompuesta en NFC y NFKC.

En el código del ejemplo, NormSample.java, el cual está representado más tarde, puede también notar otra característica de la normalización. Los caracteres katakana de mitad de ancho y ancho completo tienen la misma compatibilidad de descomposición y son así equivalentes de compatibilidad. Sin embargo, no hay equivalentes canónicos.

Para estar seguro de que realmente necesita normalizar el texto puede usar el método isNormalized para determinar si la secuencia dada de valores char está normalizada. Si este método devuelve falso, significa que tiene que normalizar esta secuencia y debe usar el método normalize el cual normaliza unos valores char de acuerdo a la forma de normalización especificada. Por ejemplo, para transformar texto en la forma canónica descompuesta tendrá que usar el siguiente método normalize: 

normalized_string = Normalizer.normalize(target_chars, Normalizer.Form.NFD);

Además, el método de normalización reorganiza los acentos en el orden canónico adecuado, de modo que no tenga que preocuparse por la reordenación del acento por su cuenta.
