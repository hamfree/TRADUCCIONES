Mejorando el rendimiento de la intercalación

La clasificación de largas listas de cadenas suele llevar mucho tiempo. Si su algoritmo de clasificación compara cadenas repetidamente, puede acelerar el proceso usando la clase CollationKey.

Un objeto CollationKey representa una clave de clasificación para una String y Collator dados. Comparar dos objetos CollationKey implica una comparación bit a bit de claves de ordenación y es más rápido que comparar objetos String con el método Collator.compare. Sin embargo, generar los objetos CollationKey requiere tiempo. Por lo tanto si una String va a ser comparada simplemente una vez, Collator.compare ofrece mejor rendimiento.

El ejemplo que sigue usa un objeto CollationKey para clasificar un vector de palabras. El código fuente para este ejemplo está en KeysDemo.java.

El progrmaa KeysDemo crea un vector de objetos CollationKey en el método main. Para crear un CollationKey, invoque el método getCollationKey en un objeto Collator. No puede comparar dos objetos CollationKey a menos que se originen del mismo Collator. El método main es como sigue: 

static public void main(String[] args) {
    Collator enUSCollator = Collator.getInstance(new Locale("en","US"));
    String [] words = {
        "peach",
        "apricot",
        "grape",
        "lemon"
    };

    CollationKey[] keys = new CollationKey[words.length];

    for (int k = 0; k < keys.length; k ++) {
        keys[k] = enUSCollator. getCollationKey(words[k]);
    }

    sortArray(keys);
    printArray(keys);
}

El método sortArray invoca el método CollationKey.compareTo. El método compareTo devuelve un entero menor que, igual a, o mayor que cero si el objeto keys[i] es menor que, igual a, o mayor que el objeto keys[j]. Dese cuenta que el programa compara los objetos CollationKey, no los objetos String del vector original de palabras. Aquí está el código para el método sortArray: 

public static void sortArray(CollationKey[] keys) {
    CollationKey tmp;

    for (int i = 0; i < keys.length; i++) {
        for (int j = i + 1; j < keys.length; j++) {
            if (keys[i].compareTo(keys[j]) > 0) {
                tmp = keys[i];
                keys[i] = keys[j];
                keys[j] = tmp; 
            }
        }
    }
}

El programa KeysDemo clasifica un vector de objetos CollationKey, pero el objetivo original era clasificar un vector de objetos String. Para recuperar la representación String de cada CollationKey, el programa invoca a getSourceString en el método displayWords, como sigue: 

static void displayWords(CollationKey[] keys) {
    for (int i = 0; i < keys.length; i++) {
        System.out.println(keys[i].getSourceString());
    }
}

El método displayWords imprime las líneas siguientes: 

apricot
grape
lemon
peach
