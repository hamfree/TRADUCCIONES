IPAs de Character y String

La clase Character encapsula el tipo de datos char. Para la versión 5 de J2SE, muchos métodos fueron agregados a la clase Character para admitir los caracteres suplementarios. Esta IPA cae en dos categorías: métodos que convierten entre char y valores de puntos de código y métodos que verifican la validez de puntos de código de mapa o de ellos.

Esta sección describe un subcojunto de los métodos disponibles en la clase Character. Para la lista completa de IPAs disponibles, vea la especificación de la clase Character.

Métodos de Conversión y la Clase Carácter

La tabla siguiente incluye los métodos de conversión más útiles, o métodos que facilitan la conversión, en la clase Character. Los métodos codePointAt y codePointBefore se incluyen en esta lista porque el texto se encuentra generalmente en una secuencia, tal como una String, y estos métodos pueden utilizarse para extraer la subcadena deseada.

Verificación y Métodos de Asignación en la Clase Character

Alguno de los métodos anteriores que usaban el tipo de datos primitivo char, tal como isLowerCase(char) y isDigit(char), fueron sustituidos por métodos que soportan los caracteres suplementarios, tales como isLowerCase(int) y isDigit(int). Los métodos anteriores se admiten pero no funcionan con los caracteres suplementarios. Para crear una aplicación global y asegurarse que su código funciona sin problemas con cualquier idioma, se recomienda que use las formas más nuevas de estos métodos.

Tenga en cuenta que, por motivos de rendimiento, la mayoría de los métodos que aceptan un punto de código no verifican la validez del parámetro del punto de código. Puede usar el método isValidCodePoint para ese propósito.

La tabla siguiente lista algunos de los métodos de verificación y asignación en la clase Character.

isHighSurrogate(char) Devuelve verdadero si el char especificado está dentro del rango de altos sustitutos de \uD800 a \uDBFF, inclusive.
