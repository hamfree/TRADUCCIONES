Consideraciones de Diseño

Para escribir código que funciona sin problemas para cualquier idioma usando cualquier silabario, hay unas pocas cosas a tener en cuenta.

Consideraciones de diseño para escribir código fáacilmente localizable
Consideración Razón
Evite métodos que usen el tipo de datos char. Evite usar el tipo de dato primitivo char o métodos que usen el tipo de dato char, porque el código que usa ese tipo de dato no funciona para los caracteres suplementarios. Para métodos que toman un tipo de parámetro char, use el método int correspondiente, donde esté disponible. Por ejemplo, use el método Character.isDigit(int) en vez del método Character.isDigit(char).
Use el método isValidCodePoint para verificar los valores de punto de código. Un punto de código se define como un tipo de datos int, lo cual permite valores fuera del rango válido de valores de puntos de código de 0x0000 a 0x10FFFF. Por razones de rendimiento, los métodos que toman un valor de punto de código como parámetro no comprueban la validez del parámetro, pero puede usar el método isValidCodePoint para comprobar el valor.
Use el método codePointCount para contar caracteres. El método String.length() devuelve el número de unidades de código, o valores char de 16 bits, en la cadena. Si la cadena contiene caracteres suplementarios, el recuento puede ser engañoso porque no reflejará el número real de puntos de código. Para obtener un recuento preciso del número de caracteres (incluyendo caracteres suplementarios), use el método codePointCount.
Use los métodos String.toUpperCase(int codePoint) y String.toLowerCase(int codePoint) en vez de los métodos Character.toUpperCase(int codePoint) o Character.toLowerCase(int codePoint). Mientras los métodos Character.toUpperCase(int) y Character.toLowerCase(int) trabajan con valores de punto de código, hay algunos caracteres que no pueden ser convertidos uno a uno. El carácter minúscula alemán ß, por ejemplo, se convierte en dos caracteres, SS, cuando se convierte a mayúsculas. Igualmente, el carácter pequeño de Sigma griego es diferente según la posición en la cadena. Los métodos Character.toUpperCase(int) y Character.toLowerCase(int) no pueden manejar estos tipos de casos; sin embargo, los métodos String.toUpperCase y String.toLowerCase manejan estos casos correctamente.
Sea cuidadoso cuando borre caracteres. Cuando invoca los métodos StringBuilder.deleteCharAt(int index) o StringBuffer.deleteCharAt(int index) donde el índice apunta a un carácter suplementario, sólo la primera mitad del carácter (el primer valor char) se elimina. Primero, invoque el método Character.charCount en el carácter para determinar si se deben eliminar uno o dos valores char.
Tenga cuidado al invertir caracteres en una secuencia. Cuando invoca los métodos StringBuffer.reverse() o StringBuilder.reverse() en texto que contiene caracteres suplementarios, los pares sustitutos alto y bajo se invierten, lo que da como resultado pares sustitutos incorrectos y posiblemente inválidos.
