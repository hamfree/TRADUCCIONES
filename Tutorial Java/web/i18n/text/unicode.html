Unicode

Unicode es un estándar de la industria informática diseñado para codificar de manera uniforme y única los caracteres utilizados en los lenguajes escritos en todo el mundo. El estándar Unicode usa la notación hexadecimal para expresar un carácter. Por ejemplo, el valor 0x0041 representa el carácter latino A. El estándar Unicode fue inicialmente diseñado usando 16 bits para codificar los caracteres porque las máquinas principales eran Computadoras Personales (Nota del traductor: Personal Computer o PC, en inglés) de 16 bits.

Cuando se creó la especificación para el lenguaje Java, el estándar Unicode fue aceptado y el tipo primitivo char fue definido como un tipo de datos de 16 bits, con los caracteres en el rango hexadecimal desde 0x0000 hasta 0xFFFF.

Ya que la codificación en 16 bits soporta 216 (65.536) caracteres, lo cual es insuficiente para definir todos los caracteres en uso en todo el mundo, el estándar Unicode fue extendido hasta 0x10FFFF, lo cual soporta sobre un millón de caracteres. La definición de un carácter en el lenguaje de programación de Java no se puede cambiar de 16 bits a 32 bits sin causar que millones de aplicaciones Java dejen de funcionar correctamente. Para corregir la definición, se desarrolló un esquema para manejar los caracteres que no podían codificarse en 16 bits.

Los caracteres con valores que están fuera del rango de 16 bits, y dentro del rango de 0x10000 a 0x10FFFF, se llaman caracteres suplementarios y se definen como un par de valores char.
