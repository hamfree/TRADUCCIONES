Codificación de Byte y Strings

Si un vector de bytes contiene texto no Unicode, puede convertir el texto a Unicode con uno de los métodos de constructor de String. A la inversa, puede convertir un objeto String en un vector de bytes de caracteres no Unicode con el método String.getBytes. Cuando invoca cualquiera de estos métodos, especifica el identificador de codificación como uno de los parámetros.

A continuación, el ejemplo convierte caracteres entre UTF-8 y Unicode. UTF-8 es un formato de transmisión para Unicode que es seguro para los sistemas de ficheros UNIX. El código fuente completo para el ejemplo está en el fichero StringConverter.java.

El programa StringConverter empieza creando un objeto String que contiene caracteres Unicode: 

String original = new String("A" + "\u00ea" + "\u00f1" + "\u00fc" + "C");

Cuando se imprime, el objeto String llamado original aparece como: 

AêñüC

Para convertir el objeto String a UTF-8, invoque el método getBytes y especifique el identificador de codificación apropiado como un parámetro. El método getBytes devuelve un vector de bytes en formato UTF-8. Para crear un objeto String desde un vector de bytes no Unicode, invoque el constructor de String con el parámetro de codificación. El código que hace estas llamadas está encerrado en un bloque try, en caso de que no se admita la codificación especificada: 

try {
    byte[] utf8Bytes = original.getBytes("UTF8");
    byte[] defaultBytes = original.getBytes();

    String roundTrip = new String(utf8Bytes, "UTF8");
    System.out.println("roundTrip = " + roundTrip);
    System.out.println();
    printBytes(utf8Bytes, "utf8Bytes");
    System.out.println();
    printBytes(defaultBytes, "defaultBytes");
} 
catch (UnsupportedEncodingException e) {
    e.printStackTrace();
}

El programa StringConverter imprime los valores en los vectores utf8Bytes y defaultBytes para demostrar un punto importante: la longitud de texto convertido no tiene que ser la misma que la longitud del texto origen. Algunos caracteres Unicode se traducen en bytes únicos, otros en pares o tríos de bytes.

El método printBytes visualiza los vectores de bytes invocando el método byteToHex, el cual se define en el fichero fuente, UnicodeFormatter.java. Aquí está el método printBytes: 

public static void printBytes(byte[] array, String name) {
    for (int k = 0; k < array.length; k++) {
        System.out.println(name + "[" + k + "] = " + "0x" +
            UnicodeFormatter.byteToHex(array[k]));
    }
}

A continuación, se muestra la salida del método printBytes. Tenga en cuenta que solo el primer y último bytes, los caracteres A y C, son iguales en ambas matrices: 

utf8Bytes[0] = 0x41
utf8Bytes[1] = 0xc3
utf8Bytes[2] = 0xaa
utf8Bytes[3] = 0xc3
utf8Bytes[4] = 0xb1
utf8Bytes[5] = 0xc3
utf8Bytes[6] = 0xbc
utf8Bytes[7] = 0x43
defaultBytes[0] = 0x41
defaultBytes[1] = 0xea
defaultBytes[2] = 0xf1
defaultBytes[3] = 0xfc
defaultBytes[4] = 0x43
