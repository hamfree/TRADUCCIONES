Terminología

Un carácter es la unidad mínima de texto que tiene valor semántico.

Un conjunto de caracteres es una colección de caracteres que podrían usarse por múltiples idiomas. Por ejemplo, el conjunto de caracteres Latin es usado por el inglés y la mayoría de los idiomas europeos, aunque el el conjunto de caracteres griego es usado sólo por el idioma griego.

Un conjunto de caracteres codificado es un conjunto de caracteres donde a cada carácter se le asigna un número único.

Un punto de código es un valor que puede ser utilizado en un conjunto de caracteres codificado. Un punto de código es un tipo de datos int de 32 bits, donde los 21 bits más bajos representan un valor de punto de código válido y los 11 bits más altos son 0.

Una unidad de código Unicode es un valor char de 16 bits. Por ejemplo, imagine un String que contiene las letras "abc" seguidas por el Deseret LONG I, el cual se representa con dos valores char. Esa cadena contiene cuatro caracteres, cuatro puntos de código, pero cinco unidades de código.

Para expresar un carácter en Unicode, el valor hexadecimal es prefijado con la cadena U+. El rango de puntos de códigos válidos para el estándar Unicode es de U+0000 a U+10FFFF, inclusive. El valor del punto de código para el carácter Latin A es U+0041. El carácter € el cual representa la moneda Euro, tiene el valor de punto de código U+20AC. La primera letra en el alfabeto Deseerte, la LONG I, tiene el valor de punto de código U+10400.

La tabla siguiente muestra valores de punto de código para varios caracteres:

Como se describió previamente, los caracteres que están en el rango U+10000 a U+10FFFF se llaman caracteres suplementarios. El conjunto de caracteres desde U+0000 a U+FFFF a veces se denomina Plano multilingüe básico (BMP).

Puede encontrar más terminología en el Glosario de Términos Unicode, listada en la página Más Información.
