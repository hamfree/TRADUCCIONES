Personalización de Reglas de intercalación

La sección previa describía cómo usar las reglas predeterminadas de una configuración regional para comparar las cadenas. Estas reglas de intercalación determinan el orden de clasificación de las cadenas. Si las reglas de intercalación predeterminadas no se ajusta a sus necesidates, puede diseñar sus propias reglas y asignarlas a un objeto RuleBasedCollator.

Las reglas de intercalación personalizadas están contenidas en un objeto String que se pasan al constructor de RuleBasedCollator. Aquí tiene un ejemplo simple: 

String simpleRule = "< a < b < c < d";
RuleBasedCollator simpleCollator =  new RuleBasedCollator(simpleRule);

Para el objeto simpleCollator en el ejemplo anterior, a es menor que b, el cual es menor que c, etcétera. El método simpleCollator.compare hace referencia a estas reglas al comparar cadenas. La sintaxis completa utilizad para construir una regla de intercalación es más flexislbe y compleja que este simple ejemplo. Para una descripción completa de la sintaxis, refiérase a la documentación de la IPA para la clase RuleBasedCollator.

El ejemplo que sigue ordena una lista de palabras en español con dos clasificadores. El código fuente completo para este ejemplo es in RulesDemo.java.

El programa RulesDemo empieza definiendo reglas de intercalación para el inglés y el español. El programa ordenar´las palabras en español de la manera tradicional. Cuando ordena por las reglas tradicionales, las letras ch y ll y sus equivalentes en mayúsculas tienen sus propias posiciones en el orden de clasificación. Estos pares de caracteres se comparan como si fueran un carácter. Por ejemplo, ch se clasifica como una letra única, siguiendo a cz en el orden de clasificación. Tenga en cuenta cómo difieren las reglas para los dos clasificadores: 

String englishRules = (
    "< a,A < b,B < c,C < d,D < e,E < f,F " +
    "< g,G < h,H < i,I < j,J < k,K < l,L " +
    "< m,M < n,N < o,O < p,P < q,Q < r,R " +
    "< s,S < t,T < u,U < v,V < w,W < x,X " +
    "< y,Y < z,Z");

String smallnTilde = new String("\u00F1");    // ñ
String capitalNTilde = new String("\u00D1");  // Ñ

String traditionalSpanishRules = (
    "< a,A < b,B < c,C " +
    "< ch, cH, Ch, CH " +
    "< d,D < e,E < f,F " +
    "< g,G < h,H < i,I < j,J < k,K < l,L " +
    "< ll, lL, Ll, LL " +
    "< m,M < n,N " +
    "< " + smallnTilde + "," + capitalNTilde + " " +
    "< o,O < p,P < q,Q < r,R " +
    "< s,S < t,T < u,U < v,V < w,W < x,X " +
    "< y,Y < z,Z");

Las siguientes líneas de código crean los clasificadores e invoca la rutina de ordenación: 

try {
    RuleBasedCollator enCollator = new RuleBasedCollator(englishRules);
    RuleBasedCollator spCollator =
        new RuleBasedCollator(traditionalSpanishRules);

    sortStrings(enCollator, words);
    printStrings(words);
    System.out.println();

    sortStrings(spCollator, words);
    printStrings(words);
} catch (ParseException pe) {
    System.out.println("Parse exception for rules");
}

La rutina de ordenación, llamada sortStrings, es genérica. Ordenará cualquier vector de palabras de acuerdo a las reglas de cualquier objeto Collator: 

public static void sortStrings(Collator collator, String[] words) {
    String tmp;
    for (int i = 0; i < words.length; i++) {
        for (int j = i + 1; j < words.length; j++) {
            if (collator.compare(words[i], words[j]) > 0) {
                tmp = words[i];
                words[i] = words[j];
                words[j] = tmp;
            }
        }
    }
}

Cuando se ordena con las reglas de clasificación en inglés, el vector de palabras es el siguiente: 

chalina
curioso
llama
luz

Compare la lista precedente con la siguiente, la cual está ordenada de acuerdo a las reglas de intercalación tradicional del español: 

curioso
chalina
luz
llama
