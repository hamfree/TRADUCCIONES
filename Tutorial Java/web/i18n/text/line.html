Límites de Línea

Las aplicaciones que dan formato al texto o que realizan ajustes de línea deben localizar posibles saltos de línea. Puede encontrar estos saltos de linea, o límites, con un BreakIterator que ha sido creado con el método getLineInstance: 

BreakIterator lineIterator =
    BreakIterator.getLineInstance(currentLocale);

Este BreakIterator determina las posiciones en una cadena donde el texto puede saltar para continuar en la línea siguiente. Las posiciones detectadas por BreakIterator son posibles saltos de línea. Los saltos de línea reales que se muestran en la pantalla pueden no ser los mismos.

Los dos ejemplos que siguen usan el método markBoundaries de BreakIteratorDemo.java para mostrar los límites de línea detectados por un BreakIterator. El método markBoundaries indica los límites de línea imprimiendo signos de intercalación (^) bajo la cadena objetivo.

De acuerdo a un BreakIterator, un límite de línea sucede después de la terminación de una secuencia de caracteres espaciadores (espacio, tabulador, nueva línea). En el ejemplo siguiente, observe que puede romper la línea en cualquiera de los límites detectados: 

She stopped.  She said, "Hello there," and then went on.
^   ^         ^   ^     ^      ^     ^ ^   ^    ^    ^  ^

Los saltos de línea potenciales también ocurren inmediatamente después de un guión: 

There are twenty-four hours in a day.
^     ^   ^      ^    ^     ^  ^ ^   ^

El ejemplo siguiente rompe una cadena larga de texto en líneas de longitud fija con un método llamado formatLines. Este método usa un BreakIterator para localizar los saltos de línea posibles. El método formatLines es corto, simple, y, gracias a BreakIterator, independiente de la configuración regional. Aquí está el código fuente: 

static void formatLines(
    String target, int maxLength,
    Locale currentLocale) {

    BreakIterator boundary = BreakIterator.
        getLineInstance(currentLocale);
    boundary.setText(target);
    int start = boundary.first();
    int end = boundary.next();
    int lineLength = 0;

    while (end != BreakIterator.DONE) {
        String word = target.substring(start,end);
        lineLength = lineLength + word.length();
        if (lineLength >= maxLength) {
            System.out.println();
            lineLength = word.length();
        }
        System.out.print(word);
        start = end;
        end = boundary.next();
    }
}

El programa BreakIteratorDemo invoca el método formatLines como sigue: 

String moreText =
    "She said, \"Hello there,\" and then " +
    "went on down the street. When she stopped " +
    "to look at the fur coats in a shop + "
    "window, her dog growled. \"Sorry Jake,\" " +
    "she said. \"I didn't know you would take " +
    "it personally.\"";

formatLines(moreText, 30, currentLocale);

La salida de esta llamada a formatLines es: 

She said, "Hello there," and
then went on down the
street. When she stopped to
look at the fur coats in a
shop window, her dog
growled. "Sorry Jake," she
said. "I didn't know you
would take it personally."
