Convirtiendo Texto No Unicode

En el lenguaje de programación Java los valores char representan caracteres Unicode. Unicode es una codificación de caracteres de 16 bits que admite los principales idiomas del mundo. Puede aprender más sobre el estándar Unicode en el Sitio Web del Consorcio Unicode .

Pocos editores de etxto soportan actualmente entradas de texto Unicode. El editor de texto que usamos para escribir los ejemplos de código de esta sección soportan sólo caracteres ASCII, que están limitados a 7 bits. Para indicar caracteres Unicode que no pueden ser representados en ASCII, tales como ö, usamos la secuencia de escape \uXXXX. Cada X en la secuencia de escape es un digito hexadecimal. El ejemplo siguiente muestra cómo indicar el carácter ö con una secuencia de escape: 

String str = "\u00F6";
char c = '\u00F6';
Character letter = new Character('\u00F6');

Una variedad de codificaciones de carácteres se usan por los sistemas en todo el mundo. Actualmente pocas de estas codificaciones se ajustan a Unicode. Ya que su programa espera caracteres en Unicode, los datos de texto que obtiene del sismtea deben ser convertidos a Unicode, y viceversa. Los datos en los ficheros de texto son convertidos automáticamente a Unicode cuando su codificación coincide con la codificación predeterminada de fichero de la Máquina Virtual de Java. Puede identificar la codificación predeterminada de fichero creando un OutputStreamWriter y usándolo para preguntarle por su nombre canónico: 

OutputStreamWriter out = new OutputStreamWriter(new ByteArrayOutputStream());
System.out.println(out.getEncoding());

Si la codificación predeterminada de fichero difiere de la codificación de los datos de texto que quiere procesar, entonces debe realizar la conversión por usted mosmo. Podría necesitar hacer esto cuando procesa texto de otro país o plataforma de computación.

Esta sección describe las IPAs que usa para traduci texto que no es Unicode a Unicode. Antes de usar estas IPAs, debe verificar que la codificación de caracteres que desea convertir a Unicode está admitida. La lista de codificaciones de carácter admitidas no es parte de la especificación del lenguaje de programación Java. Por lo tanto las codificaciones de caracteres admitidas por las IPAs pueden variar con la plataforma. Para ver las codificaciones que admite el Kit de Desarrollo de Java, vea el documento Codificaciones admitidas.

El material qeu sigue describe dos técnicas para convertir texto no Unicode a Unicode. Puede convertir vectores de bytes no Unicode en objetos String, y viceversa. O puede traducir entre flujos de caracteres Unicode y flujos de bytes de texto no Unicode.
