Lección: Estableciendo la Localización

Un programa internacionalizado puede visualizar información de manera diferente en todo el mundo. Por ejemplo, el programa visualizará mensajes diferentes en París, Tokio, y Nueva York. Si el proceso de localización ha sido ajustado, el programa visualizará diferentes mensajes en Nueva York y Londres para tener en cuenta las diferencias entre el inglés americano y el británico. ¿Cómo un programa internacionalizado identifica el idioma apropiado y región de sus usuarios finales? Fácil. Hace referencia a un objeto Locale.

Un objeto Locale es un identificador para un combinación particular de idioma y región. Si una clase varía su comportamiento de acuerdo a Locale, se dice que es sensible a la configuración regional. Por ejemplo, la clase NumberFormat es sensible a la configuración regional; el formato del número que devuelve depende de Locale. Así NumberFormat puede devolver un número como 902 300 (Francia), o 902.300 (Alemania), o 902,300 (Estados Unidos de América). Los objetos Locale son sólo identificadores. El trabajo real, tal como dar formato y detectar límites de palabras, se realizan por los métodos de clases sensibles a la configuración regional.

Las secciones siguientes explican como trabajan con objetos Locale:

Creando una Localización

Cuando crea un objeto Locale, habitualmente especifica un código de idioma y un código de país. Un tercer parámetro, la variante, es opcional.

Identificando Localizaciones Disponibles

Las clases sensibles a la configuración regional soportan sólo ciertas definiciones de Locale. Esta sección le muestra cómo determina qué definiciones de Locale se soportan.

Filtrado de Etiquetas de Lenguaje y Búsqueda

Esta sección describe el soporte de internacionalización para etiquetas de idioma, etiquetas de filtrado de idiomas, y etiquetas de búsqueda de idiomas.

El Alcance de una Localización

En la plataforma Java no especifica un Locale global estableciendo una variable de entorno antes de ejecutar la aplicación. En su lugar, confía en la configuración regional predeterminada o asigna un Locale a cada objeto sensible a la configuración regional.
