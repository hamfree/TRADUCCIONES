Extensiones BCP 47

La versión 7 de Java SE se ajusta al estándar IETF BCP 46, el cual soporta agregar extensiones a un Locale. Cualquier carácter único puede usarse para denotar una extensión, pero hay dos códigos de extensión predefinidos: 'u' especifica una extensión Unicode de configuración regional, y 'x' especifica una extensión de uso privado.

Las extensiones Unicode de configuración regional están definidas por el proyecto Common Locale Data Repository (CLDR) de Unicode. Se usan para especificar información que no es específica del idioma tales como calendarios o moneda. Una extensión de uso privado pude usarse para especificar cualquier otra información, tales como la plataforma (por ejemplo, Windows, UNIX, o Linux), o información de versión (por ejemplo, 6u23 o KDJ 7).

Una extensión se especifica como un par clave/valor, donde la claves es un carácter único (habitualmente 'u' o 'x'). Un valor bien-formado tiene el formato siguiente: 

SUBTAG ('-' SUBTAG)*

En este formato: 

SUBTAG = [0-9a-zA-Z]{2,8}    Para la clave='u'
SUBTAG = [0-9a-zA-Z]{1,8}    Para la clave='x'

Tenga en cuenta que se permite un valor de un solo carácter para la extensión de uso privado. Sin embargo, hay un mínimo de dos caracteres para los valores en la extensión Unicode de configuración regional.

Las cadenas de extensión no distinguen entre mayúsculas y minúsculas, pero la clase Locale asocia todas las claves y valores en minúsculas.

El método getExtensionKeys() devuelve el conjunto de claves de extensión, si las hay, para un Locale. El método getExtension(key) devuelve la cadena con el valor para la clave especificada, si lo hay.

Extensiones Unicode de Configuración Regional

Como se mencionó previamente, una extensión Unicode de configuración regional se especifica por el código clave 'u' o la constante UNICODE_LOCALE_EXTENSION. El valor en sí mismo es también especificado por un par clave/valor. Los valores legales están definidos en la tabla Claves/Definiciones de Tipos en el sitio web Unicode. Un código de clave se especifica por dos caracteres alfabéticos. La tabla siguiente lista las claves de extensión Unicode de configuración regional:

Una tabla mostrando las claves de extensión Unicode de la configuración regional
Código Clave Descripción
ca algoritmo de calendario
co tipo de colación
ka parámetros de colación
cu tipo de moneda
nu tipo de número
va tipo de variante común
Nota: 

Especificar una extensión Unicode de configuración regional, tal como un formato de número, no garantiza que los servicios de configuración regional para la plataforma subyacente cumplan esa solicitud.

La tabla siguiente muestra algunos ejemplos de pares clave/valor para una extensión Unicode de configuración regional.

Algunos ejemplos de pares clave/valor para una extensión Unicode de configuración regional
Par Clave/Valor Descripción
ca-buddhist Calendario Thai Buddhist
co-pinyin Ordenamiento Pinyin para Latin
cu-usd Dólares de EE.UU
nu-jpanfin Números financieros Japonés
tz-aldav Europa/Andorra

La cadena siguiente representa la configuración regional del idioma Alemán para el país de Alemania usando un estilo de agenda telefónica de ordenamiento para la plataforma Linux. Este ejemplo también contiene un atributo llamado "email". 

de-DE-u-email-co-phonebk-x-linux

Los siguientes métodos de Locale pueden usarse para acceder a la información sobre las extensiones Unicode de configuración regional. Estos métodos están descritos usando el ejemplo de configuración regional alemán anterior.

  • getUnicodeLocaleKeys() – Devuelve los códigos clave Unicode de la configuración regional o un conjunto vacio si la configuración regional no tiene ninguna. Para el ejemplo del alemán, este devolvería un conjunto conteniendo la cadena única "co".
  • getUnicodeLocaleType(String) – Devuelve el tipo asociado Unicode de la configuración regional asociado con el código clave Unicode de la configuración regional. Invocar getUnicodeLocaleType("co") para el ejemplo alemán devolvería la cadena "phonebk".
  • getUnicodeLocaleAttributes() – Devuelve el conjunto de atributos Unicode de la configuración regional asociados con esta configuración regional, si los hay. En el ejemplo del alemán, esto devolvería un conjunto conteniendo la cadena única "email".

Extensiones de Uso Privado

La extensión de uso privado, especificada por la clave de código 'x' o la constante PRIVATE_USE_EXTENSION, puede ser cualquier cosa, siempre que el valor esté bien formado.

Lo siguiente son ejemplos de posibles extensiones de uso privado: 

x-jdk-1-7
x-linux
