Creando una Localización

Hay varias formas de crear un objeto Locale. Independientemente de la técnica utilizada, la creación puede ser tan simple como especificar el código del idioma. Sin embargo, puede distinguir aún más la configuración regional estableciendo la región (también denominada "país") y los códigos de variante. Si está usando la versión KDJ 7 o posterior, puede también especificar el código de script y extensiones Unicode de locale.

Las cuatro formas de crear un objeto Locale son:

  • La Clase Locale.Builder
  • Constructores Locale
  • El Método de Factoría Locale.forLanguageTag
  • Constantes Locale
Nota de Versión: La clase Locale.Builder y el método forLanguageTag fueron agregados en la versión Java SE 7.

La Clase Locale.Builder

La clase de utilidad Locale.Builder puede ser usada para construir un objeto Locale que se ajuste a la sintaxis IETF BCP 47. Por ejemplo, para especificar el idioma francés y el país de Canadá, podría invocar el constructor Locale.Builder y entonces encadenar los métodos setter como sigue: 

Locale aLocale = new Locale.Builder().setLanguage("fr").setRegion("CA").build();

El ejemplo siguiente crea objetos Locale para el idioma inglés en los Estados Unidos de América y Gran Bretaña: 

Locale bLocale = new Locale.Builder().setLanguage("en").setRegion("US").build();
Locale cLocale = new Locale.Builder().setLanguage("en").setRegion("GB").build();

El ejemplo final crea un objeto Locale para el idioma ruso: 

Locale dLocale = new Locale.Builder().setLanguage("ru").setScript("Cyrl").build();

Constructores Locale

Hya tres constructores disponibles en la clase Locale para crear un objeto Locale:

  • Locale(String language)
  • Locale(String language, String country)
  • Locale(String language, String country, String variant)

Los ejemplos siguientes crean objetos Locale para el idioma francés en Canadá, el idioma inglés en EE.UU y Gran Bretaña, y el idioma ruso. 

aLocale = new Locale("fr", "CA");
bLocale = new Locale("en", "US");
cLocale = new Locale("en", "GB");
dLocale = new Locale("ru");

No es posible establecer un código de silabario en un objeto Locale en una versión anterior al KDJ 7.

El Método de Factoría Locale.forLanguageTag

Si tiene una cadena de etiqueta de idioma que se ajusta al estándar IETF BCP 47, puede usar el método de factoría forLanguageTag(String) , el cual fue introducido en la versión 7 de Java SE. Por ejemplo: 

Locale aLocale = Locale.forLanguageTag("en-US");
Locale bLocale = Locale.forLanguageTag("ja-JP-u-ca-japanese");

Constantes Locale

Para su convenencia la clase Locale proporciona constantes para algunos idiomas y paises. Por ejemplo: 

cLocale = Locale.JAPAN;
dLocale = Locale.CANADA_FRENCH;

Cuando especifica una constante de idioma, la porción de la región de Locale no está definida. Las tres sentencias siguientes crea objetos Locale equivalentes: 

j1Locale = Locale.JAPANESE;
j2Locale = new Locale.Builder().setLanguage("ja").build();
j3Locale = new Locale("ja");

Los objetos Locale creados para las tres sentencias siguientes son también equivalente: 

j4Locale = Locale.JAPAN;
j5Locale = new Locale.Builder().setLanguage("ja").setRegion("JP").build();
j6Locale = new Locale("ja", "JP");

Códigos

Las secciones siguientes describen el código de idioma y el silabario opcional, la región, y los códigos de variantes.

Código de Idioma

El código de idioma tiene dos o tres letras minúsculas que se ajustan al estándar ISO 639. Puede encontrar una lista completa de los códigos ISO 639 en http://www.loc.gov/standards/iso639-2/php/code_list.php.

La tabla siguiente lista unos pocos códigos de idioma.

Código de Silabario

El código de silabario comienza con una letra mayúscula seguida por tres letras en minúsculas y de acuerdo al estándar ISO 15924. Puede encontrar una lista completa de los códigos ISO 15924 en http://unicode.org/iso15924/iso15924-codes.html.

La tabla siguiente lista unos pocos códigos de silabario.

Códigos de Silabario de Ejemplo
Códigos de Silabario Descripción
Arab Arábico
Cyrl Cirílico
Kana Katakana
Latn Latín

Hay tres métodos para recuperar la información de silabario de un Locale:

  • getScript() – devuelve el código de 4 letras del silabario de un objeto Locale. Si no hay definido silabario para el locale, se devuelve una cadena vacía.
  • getDisplayScript() – devuelve un nombre para el silabario de locale que es apropiado para visualizar al usuario. Si es posible, el nombre será localizado para el locale por defecto. Así, por ejemplo, si el código de silabario es "Latn," el nombre de visualización del silabario sería la cadena "Latin" para un locale del idioma inglés.
  • getDisplayScript(Locale) – devuelve el nombre de visualización del Locale especificado localizado, si es posible, del locale.

Código de Región

El código de región (país) consiste en dos o tres letras en mayúsculas que se ajustan al estándar ISO 3166, o tres números que se ajustan al estándar UN M.49. Una copia de estos códigos pueden encontrarse en http://www.chemie.fu-berlin.de/diverse/doc/ISO_3166.html.

Código de Variante

El código de variante opcional puede ser usado para distinguir aún más su Locale. Por ejemplo, el código de variante puede usarse para indicar diferencias dialécticas que no se cubren con el código regional.

Nota de Versión:  Anteriormente a la versión 7 de Java SE, el código de variante se usó algunas veces para identificar diferencias que no eran específicas al idioma o la región. Por ejemplo, podría haber sido usado para identificar diferencias entre plataformas de computación, tales como Windows o UNIX. Bajo el estándar IETF BCP 47, se desaconseja este uso.

Para definir variaciones no específicas de idioma relevantes para su entorno, use los mecanismos de extensiones, como se explica en Extensiones BCP 47.

A partir de la versión Java SE 7, que cumple con el estándar IETF BCP 47, el código de variante se usa específicamente para indicar variaciones adicionales que definen un idioma o sus dialectos. El estándar IETF BCP 47 impone restricciones sintácticas en la subetiqueta de variante. Puede ver una lista de códigos de variante (busque por variant en http://www.iana.org/assignments/language-subtag-registry.

Por ejemplo, Java SE usa el código de variante para soportar el idioma Thai. Por convención, un objeto NumberFormat para los locales th y th_TH usarán las figuras de dígito arábicas, o numerales arábicos, para dar formato a los números Thai. Sin embargo, un NumberFormat para el locale th_TH_TH usa las figuras de dígito Thai. El extracto de ThaiDigits.java demuestra esto: 

String outputString = new String();
Locale[] thaiLocale = {
             new Locale("th"),
             new Locale("th", "TH"),
             new Locale("th", "TH", "TH")
         };

for (Locale locale : thaiLocale) {
    NumberFormat nf = NumberFormat.getNumberInstance(locale);
    outputString = outputString + locale.toString() + ": ";
    outputString = outputString + nf.format(573.34) + "\n";
