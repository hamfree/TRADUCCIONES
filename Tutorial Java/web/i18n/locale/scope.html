El Alcance de una Localización

La plataforma Java no le requiere que use el mismo Locale en todo su programa. Si usted lo desea, puede asignar un Locale diferente a cada objeto sensible a la configuración reigional en su programa. Esta flexibilidad le permite desarrollar aplicaciones multilingües, las cuales pueden visualizar información en múltiples idiomas.

Sin embarig, la mayoría de las aplicaciones no son multilingües y sus objetos sensibles a la configuración regional se basan en el valor predeterminado de Locale. Establecido por la Máquina Virtual de Java cuando se arranca, el valor predeterminado de Locale se corresponde con la configuración regional de la plataforma anfitriona. Para determinar el valor predeterminado de Locale de su Máquina Virtual de Java, invoque el método Locale.getDefault.

Nota: 

Es posible establecer independientemente la configuración regional por defecto para dos tipos de usos: la configuración de formato se usa para dar formato a los recursos, y la configuración de visualización se usa en menús y diálogos. Introducidos en la versión 7 de Java SE, el método Locale.getDefault(Locale.Category) toma un parámetro Locale.Category. Pasando la enumeración FORMAT al método getDefault(Locale.Category) devuelve la configuración regional predeterminada para el formato de recursos. De forma similar, pasando la enumeración DISPLAY devuelve la configuración regional predeterminada usada por el IU. El método correspondiente setDefault(Locale.Category, Locale) permite establecer la configuración regional para la categoría deseada. El método sin argumento getDefault devuelve el valor por defecto para DISPLAY.

En la plataforma Windows, estos valores predeterminados son inicializados de acuerdo a las configuraciones "Estándares y Formatos" y "Idioma de Pantalla" en el panel de control de Windows.

No debería establecer el valor predeterminado para Locale mediante programación porque es compartido por todas las clases sensibles a la configuración regional.

La computación distribuida plantea algunos problemas interesantes. Por ejemplo, suponga que está diseñando una aplicación de servidor que recibirá peticiones desde clientes en varios países. Si el Locale para cada cliente es diferente, ¿Cuál debería ser el Locale del servidor? Quizás el servidor sea multiproceso, con cada hilo establecido al Locale del cliente al que sirve. O quizás todos los datos transmitidos entre el servidor y los clientes deberían ser independientes de la configuración regional.

¿Qué enfoque de diseño debería adoptar? Si es posible, los datos transmitidos entre el servidor y los clientes deberían ser independientes de la configuración regional. Esto simplifica el diseño del servidor haciendo a los clientes responsables de visualizar los datos en una manera sensible a la configuración regional. Sin embargo, este enfoque no funcionará si el servidor debe almacenar los datos en una forma específica de configuración regional. Por ejemplo, el servidor podría almacenar versiones en español, inglés y francés de los mismos datos en diferentes columnas de la base de datos. En este caso, el servidor pordría querer preguntar al cliente por su Locale, ya que el Locale podría haber cambiado desde la última petición.
