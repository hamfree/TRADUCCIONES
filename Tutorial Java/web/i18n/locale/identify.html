Identificando Localizaciones Disponibles

Puede crear un Locale con cualquier combianción de idioma válido y códigos de país, pero eso no significa que pueda usarlos. Recuerde, un objeto Locale es sólo un identificador. Usted pasa el objeto Locale a otros objetos, los cuales después hacen el trabajo real. Estos otros objetos, los cuales llamamos sensibles a la configuración regional, no saben cómo tratar con todas las definicios posibles de Locale.

Para encontrar qué tipos de definiciones Locale reconoce una clase sensible a la configuración regional, invoque el método getAvailableLocales. Por ejemplo, para encontrar qué definiciones de Locale están soportadas por la clase DateFormat, podría escribir una rutina tal como la siguiente: 

import java.util.*;
import java.text.*;

public class Available {
    static public void main(String[] args) {
        Locale list[] = DateFormat.getAvailableLocales();
        for (Locale aLocale : list) {
            System.out.println(aLocale.toString());
        }
    }
}

Note que el objeto String devuelto por toString contiene el idioma y los códigos de país, separados por un carácter de subrayado: 

ar_EG
be_BY
bg_BG
ca_ES
cs_CZ
da_DK
de_DE
...

Si quiere visualizar una lista de nombres de Locale a los usuarios finales, debería mostrarles algo más fácil de entender que los códigos de idioma y país devueltos por toString. En su lugar puede invocar el método Locale.getDisplayName, el cual recupera un objeto String localizado de un objeto Locale. Por ejemplo, cuando se reemplaza toString con getDisplayName en el código precedente, el programa imprime las líneas siguientes: 

Arabic (Egypt)
Belarussian (Belarus)
Bulgarian (Bulgaria)
Catalan (Spain)
Czech (Czech Republic)
Danish (Denmark)
German (Germany)
...

Puede ver diferentes listas de configuración regional dependiendo de las implementaciones de la Plataforma Java.
