Servicios SPI sensibles a la Localización

Esta característica habilita el complemento de datos y servicios dependientes de la configuración regional. De esta forma, terceras partes son capaces de proporcionar implementaciones de las clases más sensibles a la configuración regional en los paquetes java.text y java.util.

La implementación de SPIs (Interfaz de Proveedor de Servicio) está basada en clases abstractas e interfaces de Java que son implementados por el proveedor de servicio. En tiempo de ejecución el mecanismo de carga de clases de Java se usa para ubicar y cargar dinámicamente las clases que implementan la SPI.

Puede usar los servicios sensibles a la configuración regional del SPI para proporcionar las siguientes implementaciones sensibles a la configuración regional:

  • Objetos BreakIterator
  • Objetos Collator
  • Código de Idioma, código de País, y nombre de Variante para la clase Locale
  • Nombres de Zonas Horarias
  • Símbolos de Monedas
  • Objetos DateFormat
  • Objetos DateFormatSymbol
  • Objetos NumberFormat
  • Objetos DecimalFormatSymbols

Las SPIs correspondientes están contenidas en los paquetes java.text.spi y en java.util.spi:

java.util.spi java.text.spi
  • CurrencyNameProvider
  • LocaleServiceProvider
  • TimeZoneNameProvider
  • CalendarDataProvider
  • BreakIteratorProvider
  • CollatorProvider
  • DateFormatProvider
  • DateFormatSymbolsProvider
  • DecimalFormatSymbolsProvider
  • NumberFormatProvider

Por ejemplo, si desea proporcionar un objeto NumberFormat para una nueva configuración regional, tiene que implementar la clase java.text.spi.NumberFormatProvider. Necesita extender esta clase e implementar sus métodos:

  • getCurrencyInstance(Locale locale)
  • getIntegerInstance(Locale locale)
  • getNumberInstance(Locale locale)
  • getPercentInstance(Locale locale)
Locale loc = new Locale("da", "DK");
NumberFormat nf = NumberFormatProvider.getNumberInstance(loc);

Estos métodos primero comprueban si el entorno de ejecución de Java soporta la configuración regional requerida; si es así, usan ese soporte. De lo contrario, los métodos llaman a los métodos getAvailableLocales() de los proveedores instalados para que la interfaz adecuada encuentre un proveedor que admita la configuración regional solicitada.
