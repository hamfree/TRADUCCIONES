Lista de Verificación

Muchos programas no se internacionalizaron cuando se escribieron por primera vez. Estos programas puedieron empezarse como prototipos, o quizás no estaban destinados a la distribución internacional. Si debe internacionalizar un programa existente, tome los pasos siguientes:

Identificar los Datos Culturalmente Dependientes

Los mensajes de texto son la forma de datos más obiva que varía con la cultura. Sin embargo, otros tipos de datos pueden variar con la región o el idioma. La lista siguiente contiene ejemplos de datos culturalmente dependientes:

  • Mensajes
  • Etiquetas en componentes del IGU
  • Ayuda en línea
  • Sonidos
  • Colores
  • Gráficos
  • Iconos
  • Fechas
  • Horas
  • Números
  • Monedas
  • Mediciones
  • Números de teléfono
  • Honoríficos y títulos personales
  • Direcciones postales
  • Diseños de página

Aislar Texto Traducible en Paquetes de Recursos

La traducción es cara. Puede ayudar a reducir los costes aislando el texto que debe ser traducido en objetos ResourceBundle. El texto traducible incluye mensajes de estado, mensajes de error, entradas del fichero de bitácora, y componentes de etiquetas en el IGU. Este texto está codificado dentro de los programas que no han sido internacionalizados. Necesita localizar todas las ocurrencias del texto codificado que se visualizan a los usuarios finales. Por ejemplo, debería limpiar un código como este: 

String buttonLabel = "OK";
// ...
JButton okButton = new JButton(buttonLabel);

Vea la sección Aislando Datos de una Localización Específica para los detalles.

Tratar con Mensajes Compuestos

Los mensajes compuestos contienen datos variables. En el mensaje "El disco contiene 1100 ficheros" el entero 1100 puede variar. Este mensaje es difícil de traducir porque la posición del entero en la sentencia no es la misma en todos los idiomas. El mensaje siguiente no es traducible, porque el orden de los elementos de la sentencia están codificados por concatenación: 

Integer fileCount;
// ...
String diskStatus = "The disk contains " + fileCount.toString() + " files";

Siempre que sea posible, debería evitar construir mensajes compuestos, porque son muy difíciles de traducir. Sin embargo, si su aplicación requiere mensajes compuestos, puede manejarlos con las técnicas descritas en la sección Mensajes.

Dar Formato a Números y Monedas

Si su aplicación muestra números y monedas, debe darles formato de una manera independiente a la regionalización. El código siguiente no está aún internacionalizado, porque no visualizará el número correctamente en todos los países: 

Double amount;
TextField amountField;
// ...
String displayAmount = amount.toString();
amountField.setText(displayAmount);

Debería reemplazar el código precedente con una rutina que da forma al número correctamente. El lenguaje de programación Java proporciona varias clases que dan formato a los números y las monedas. Estas clases están descritas en la sección Números y Monedas.

Dar Formato a Fechas y Horas

Los formatos de fecha y hora difieren con la región y el idioma. Si su código contiene sentencias como la siguiente, necesita cambiarla: 

Date currentDate = new Date();
TextField dateField;
// ...
String dateString = currentDate.toString();
dateField.setText(dateString);

Si usa las clases de formato de fecha, su aplicación puede visualizar fechas y horas correctamente en todo el mundo. Para ejemplos e instrucciones, vea la sección Fechas y Horas.

Usar Propiedades de Caracteres Unicode

El código siguiente intenta verificar que un carácter es una letra: 

char ch;
// This code is incorrect
if ((ch >= 'a' && ch <= 'z') || (ch >= 'A' && ch <= 'Z'))

Tenga cuidado con el código como este, porque no funcionará con otros idiomas además del inglés. Por ejemplo, la instrucción if pierde el carácter ü en la palabra alemana Gr ü n.

Los métodos de comparación de Character usan el estándar Unicode para identificar las propiedades de los caracteres. Así que debería reemplazar el código previo con lo siguiente: 

char ch;
// ...
if (Character.isLetter(ch))

Para más información sobre los métodos de comparación de Character, vea la sección Comprobando las Propiedades de los Caracteres.

Comparar las Cadenas Correctamente

Cuando ordena texto a menudo compara cadenas. Si el texto se visualiza, no debería usar los métodos de comparación de la clase String. Un programa que no ha sido internacionalizado podría comparar cadenas como sigue: 

String target;
String candidate;
// ...
if (target.equals(candidate)) {
// ...
if (target.compareTo(candidate) < 0) {
// ...

Los métodos String.equals y String.compareTo realizan comparaciones binarias, las cuales son inefectivas cuando se ordena en la mayoría de lenguajes. En su lugar, debe utilizar la clase Collator, que se describe en la sección Comparando Cadenas.

Convertir texto no Unicode

Los caracteres en el lenguaje de programación Java están codificados en Unicode. Si su aplicación gestiona texto no Unicode, podría necesitar traducirlo a Unicode. Para más información, use la sección Convirtiendo Texto No Unicode.
