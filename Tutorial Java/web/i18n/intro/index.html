Lección: Introducción

Internacionalización es el proceso de diseñar una aplicación de forma que pueda ser adaptada a varios idiomas y regiones sin cambios de ingeniería. Algunas veces el término internacionalización se abrevia como i18n, porque hay 18 letras entre la primera "i" y la última "n".

Un programa internacionalizado tiene las características siguientes:

  • Con la adicción de datos localizados, el mismo ejecutable puede ejecutarse en todo el mundo.
  • Los elementos textuales, tales como los mensajes de estado y los componentes de etiquetas del IGU, no son escritas a mano en el programa. En vez de eso son almacenadas aparte del código fuente y recuperadas dinámicamente.
  • El soporte para nuevos idiomas no requiere recompilación.
  • Los datos culturamente dependientes, tales como las fechas y las monedas, aparecen en formatos que se corresponden con la región y el idioma del usuario final.
  • Puede localizarse rápidamente.

La localización es el proceso de adaptar el software para un idioma y región específica agregando componentes específicos de regionalización y texto traducido. El término localización es a menudo abreviado como l10n, porque hay 10 letras entre la "l" y la "n".

La tarea primaria de localización es traducir los elementos del interfaz de usuario y la documentación. La localización no sólo involucra cambiar el idioma de interacción, sino también otros cambios relevantes como la visualización de números, fechas, monedas, etcétera. Otros tipos de datos, como los sonidos y las imágenes, pueden requerir localización si son sensibles culturalmente. Cuanto mejor está internacionalizada una aplicación, más fácil es localizarla para un idioma y un esquema de codificacón de caracteres particular.

La internacionalización puede parecer un poco desalentadora al principio. Leer las secciones siguientes le ayudará a familiarizarse con el tema.
