Internacionalizando el Programa de Ejemplo

Si usted mira el código fuente internacionalizado, notará que los mensajes codificados en inglés han sido eliminados. Ya que los mensajes ya no están codificados y porque se especifica el código de idioma en tiempo de ejecución, el mismo ejecutable puede ser distribuído en todo el mundo. No se requiere recompilación para la localización. El programa ha sido internacionalizado.

Puede estarse preguntando qué le ha ocurrido al texto de los mensajes o qué significan los códigos de idioma y país. No se preocupe. Aprenderá estos conceptos a medida que avanza en el proceso de internacionalización del programa de muestra.

1. Crear el Fichero de Propiedades

Un fichero de propiedades almacena información sobre las características de un programa o entorno. Un fichero de propiedades está en formato de texto plano. Puede simplemente crear el fichero con cualquier editor de texto.

En el ejemplo los ficheros de propiedades almacenan el texto traducible de los mensajes a visualizarse. Antes de que fuese internacionalizado el programa, la versión inglesa de este texto estaba codificada en las sentencias System.out.println. El fichero de propiedades por defecto, que se llama MessagesBundle.properties, contiene las líneas siguientes: 

greetings = Hello
farewell = Goodbye
inquiry = How are you?

Ahora que los mensajes están en un fichero de propiedades, pueden ser traducidos en varios idiomas. No se requieren cambios al código fuente. El traductor de francés ha creado un fichero de propiedades llamado MessagesBundle_fr_FR.properties, el cual contiene estas líneas: 

greetings = Bonjour.
farewell = Au revoir.
inquiry = Comment allez-vous?

Note que los valores al lado derecho del signo igual han sido traducidos pero que las claves en el lado izquierdo no han sido cambiadas. Estas claves no deben cambiar, porque serán referenciadas cuando su programa recupera el texto traducido.

El nombre del fichero de propiedades es importante. Por ejemplo, el nombre del fichero MessagesBundle_fr_FR.properties contiene el código de idioma fr y el código de país FR. Estos código son también usados cuando crea un objeto Locale.

2. Definir Locale

El objeto Locale identifica un idioma y país particular. La sentencia siguiente define un Locale en el cual el idioma es inglés y el país son los Estados Unidos de América: 

aLocale = new Locale("en","US");

El ejemplo siguiente crea objetos Locale para el idioma francés en Canadá y en Francia: 

caLocale = new Locale("fr","CA");
frLocale = new Locale("fr","FR");

El programa es flexible. En vez de usar códigos de idioma y país codificados, el programa los obtiene desde la línea de comandos en tiempo de ejecución: 

String language = new String(args[0]);
String country = new String(args[1]);
currentLocale = new Locale(language, country);

Los objetos Locale son solo identificadores. Después de definir un Locale, se lo pasa a otros objetos que realizan tareas útiles, tales como dar formato a las fechas y los números. Estos objetos son sensibles a la regionalización porque su comportamiento varía de acuerdo a Locale. Un ResourceBundle es un ejemplo de objeto sensible a la regionalización.

3. Crear un ResourceBundle

Los objetos ResourceBundle contienen objetos específicos de regionalización. Uste usa ResourceBundle para aislar datos sensibles a la configuración regional, como el texto traducible. En el programa de ejemplo ResourceBundle se respalda con los ficheros de propiedades que contienen el mensaje de texto que queremos visualizar.

El ResourceBundle se crea como sigue: 

messages = ResourceBundle.getBundle("MessagesBundle", currentLocale);

Los argumentos pasados al método getBundle identifican qué fichero de propiedades será accedido. El primer argumento, MessagesBundle, se refiere a esta familia de ficheros de propiedades: 

MessagesBundle_en_US.properties
MessagesBundle_fr_FR.properties
MessagesBundle_de_DE.properties

El Locale, el cual es el segundo argumento de getBundle, especifica cual de los ficheros MessagesBundle se elije. Cuando se creó el Locale, el código de idioma y el código de país fueron pasados a su constructor. Dese cuenta de que los códigos de idioma y de país siguen a MessagesBundle en los nombre de los ficheros de propiedades.

Ahora todo lo que tiene que hacer es obtener los mensajes traducidos desde ResourceBundle.

4. Obtener el Texto desde el ResourceBundle

Los ficheros de propiedades contienen pares clave-valor. Los valores consisten en el texto traducido que el programa mostrará. Especifique las claves cuando obtenga los mensajes traducidos desde el ResourceBundle con el método getString. Por ejemplo, para recuperar el mensaje identificado con la clave greetings, invoque getString como sigue: 

String msg1 = messages.getString("greetings");

El programa de ejemplo usa la clave greetings porque refleja el contenido del mensaje, pero podría haber usando otra String, tal como s1 o msg1. Simplemente recuerde que la clave está codificada en el programa y debe estar presente en los ficheros de propiedades. Si sus traductores accidentalmente modifican las claves en los ficheros de propiedades, getString no será capaz de encontrar los mensajes.

Conclusión

Eso es todo. Como puede ver, internacionalizar un programa no es demasiado difícil. Requiere algo de planificación y un poco de codificación extra, pero los beneficios son enormes. Para proporcionarle una introducción del proceso de internacionalización, el programa de ejemplo en esta lección se mantuvo intencionalmente simple. A medida que lea las lecciones que siguen, aprenderá sobre las características más avanzadas de la internacionalización del lenguaje de programación Java.
