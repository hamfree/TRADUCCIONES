Nombre de Dominio Internacionalizado

Históricamente, un nombre de dominio de Internet sólo contenía símbolos ASCII. Mientras Internet ganó popularidad y fue adoptada alrededor del mundo, se hizo necesario que soportara la internacionalización de los nombres de dominio, específicamente para soportar nombres de dominio que incluyeran los caracteres Unicode.

El mecanismo de Internacionalización de los Nombres de Dominio en Aplicaciones (IDNA) se adoptó como el estándar para convertir caracteres Unicode al estándar ASCII de nombres de dominio y así preservar la estabilidad del sistema de nombres de dominio. Este sistema realiza un servicio de búsqueda para traducir nombres amistosos para el usuario a direcciones de red.

Ejemplos de nombres de dominio internacionalizados:

  • http://清华大学.cn
  • http://www.транспорт.com

Si usted sigue estos enlaces verá que el nombre de dominio Unicode representado en la barra de direcciones es substituido con una cadena ASCII.

Para implementar una funcionalidad similar en su aplicación, la clase java.net.IDN proporciona métodos para convertir nombres de dominio entres formatos ASCII y no ASCII.

Métodos de la clase java.net.IDN
Método Propósito
toASCII(String)
toASCII(String, flag) 		Se usa antes de enviar un IDN al sistema de resolución de nombres de dominio o escribir un IDN en un archivo donde se esperan caracteres ASCII, como un archivo maestro DNS. Si la cadena de entrada no se ajusta al RFC 3490, estos métodos lanzan una IllegalArgumentException.
toUnicode(String)
toUnicode(String, flag) 		Usado cuando se muestran nombres a los usuarios, por ejemplo nombres obtenidos desde una zona DNS. Este método traduce una cadena de la Codificación Compatible con ASCII (ACE) a puntos de código Unicode. Este método nunca falla; en caso de un error la cadena de entrada permanece igual y es devuelta sin modificar.

El parámetro opcional flag especifica el comportamiento del proceso de conversión. El flag ALLOW_UNASSIGNED permite incluir puntos de código que no están asignados en Unicode 3.2. El flag USE_STD3_ASCII_RULES se asegura de que se cumplan las reglas ASCII STD-3. Puede usar estos flags separadamente o combinados lógicamente mediante un operador OR juntos. Si no se desea ninguno de los flags, use la versión de un parámetro del método.
