Números y Monedas

Los programa almacenan y operan con números de una forma independiente de la configuración regional. Antes de visualizar o imprimir un número, un programa debe convertirlo a un objeto String que está en un formato sensible a la configuración regional. Por ejemplo, en Francia el número 123456.78 debe ser formateado como 123 456,78, y en Alemania debe aparecer como 123.456,78. En esta sección, aprenderá cómo hacer sus programas independientes de las convenciones de configuración regional para los puntos decimales, los separadores de miles, y otras propiedades de formato.

Usando Formatos Predefinidos

Usando los métodos de factoría proporcionados por la clase NumberFormat, puede obtener formatos específicos de configuración regional para números, monedas, y porcentajes.

Dar Formato con Patrones

Con la clase DecimalFormat puede especificar un formato de número con un patrón String. La clase DecimalFormatSymbols le permite modificar los símbolos de formato tales como los separadores de decimales y los signos menos.
