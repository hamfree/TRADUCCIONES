Usando Formatos Predefinidos

Al invocar los métodos proporcionados por la clase NumberFormat , puede dar formato a números, monedas, y porcentajes de acuerdo a Locale. El material que sigue demuestra las técnicas de formato con un programa de ejemplo llamado NumberFormatDemo.java.

Números

Puede usar los métodos de NumberFormat para dar formato a números de tipo primitivo, tales como double, y sus correspondientes objetos de envoltura, tales como Double.

El siguiente código de ejemplo da formato a un Double de acuerdo a Locale. Invocar el método getNumberInstance devuelve una instancia específica de configuración regional de NumberFormat. El método format acepta un Double como un argumento y devuelve el número formateado en un objeto String. 

static public void displayNumber(Locale currentLocale) {

    Integer quantity = new Integer(123456);
    Double amount = new Double(345987.246);
    NumberFormat numberFormatter;
    String quantityOut;
    String amountOut;

    numberFormatter = NumberFormat.getNumberInstance(currentLocale);
    quantityOut = numberFormatter.format(quantity);
    amountOut = numberFormatter.format(amount);
    System.out.println(quantityOut + "   " + currentLocale.toString());
    System.out.println(amountOut + "   " + currentLocale.toString());
}

Este ejemplo imprime lo siguiente; muestra cómo el formato del mismo número varía con Locale: 

123 456   fr_FR
345 987,246   fr_FR
123.456   de_DE
345.987,246   de_DE
123,456   en_US
345,987.246   en_US

Uso de Formas de Dígitos que no Sean Números Arábigos

Por defecto, cuando el texto contiene valores numéricos, estos valores se visualizan usando dígitos arábigos. Cuando se prefieren otras formas de dígitos Unicode, use la clase java.awt.font.NumericShaper. La IPA de NumericShaper le permite visualizar un valor numérico representado internamente como un valor ASCII en cualquier forma de dígito Unicode. Vea Convirtiendo Dígitos Latinos a Otros Dígitos Unicode para más información.

Además, algunas configuraciones regionales tienen códigos de variante que especifican que se usen esas formas de dígito Unicode en vez de los dígitos arábigos, tal como la configuración regional para el idioma Thai. Vea la sección Código de Variante en Creando una Localización para más información.

Monedas

Si está escribiendo aplicaciones de negocios, probablemente necesitará dar formato y visualizar monedas. Usted da formato a las monedas de las misma manera que a los números, excepto que llama a getCurrencyInstance para crear un formateador. Cuando invoca el método format, devuelve un objeto String que incluye el número formateado y el signo de moneda apropiado.

Este código de ejemplo muestra cómo dar formato a la moneda de una manera específica de la configuración regional: 

static public void displayCurrency( Locale currentLocale) {

    Double currencyAmount = new Double(9876543.21);
    Currency currentCurrency = Currency.getInstance(currentLocale);
    NumberFormat currencyFormatter = 
        NumberFormat.getCurrencyInstance(currentLocale);

    System.out.println(
        currentLocale.getDisplayName() + ", " +
        currentCurrency.getDisplayName() + ": " +
        currencyFormatter.format(currencyAmount));
}

La salida generada por las líneas de código anteriores es el siguiente: 

French (France), Euro: 9 876 543,21 €
German (Germany), Euro: 9.876.543,21 €
English (United States), US Dollar: $9,876,543.21

A primera vista, esta salida podría parecerle incorrecta porque los valores numéricos son todos los mismos. Por supuesto, 9 876 543,21 € no es equivalente a $9,876,543.21. Sin embargo, tenga en cuenta que la clase NumberFormat desconoce los tipos de cambio. Los métodos que pertenecen a la clase NumberFormat dan formato a las monedas pero no las convierten.

Dese cuenta de que la clase Currency está diseñada de forma que no hay nunca más de una instancia de Currency para cualquier moneda dada. Por lo tanto, no hay constructor público. Como se demostró en el ejemplo de código anterior, obtiene una instancia de Currency usando los métodos getInstance.

La clase Currency contiene otros métodos para recuperar información relacionada con la moneda:

  • getAvailableCurrencies : Devuelve todas las monedas disponibles en el KDJ

  • getCurrencyCode : Devuelve el código numérico ISO 4217 para una instancia de Currency

  • getSymbol : Devuelve el símbolo para una instancia de Currency. Puede opcionalmente especificar como un argumento un objeto Locale. Considere el siguiente extracto: 

    Locale enGBLocale = 
    new Locale.Builder().setLanguage("en").setRegion("GB").build();

Locale enUSLocale =
    new Locale.Builder().setLanguage("en").setRegion("US").build();

Currency currencyInstance = Currency.getInstance(enUSLocale);

System.out.println(
    "Symbol for US Dollar, en-US locale: " +
    currencyInstance.getSymbol(enUSLocale));

System.out.println(
    "Symbol for US Dollar, en-UK locale: " +
    currencyInstance.getSymbol(enGBLocale));

    El extracto imprime lo siguiente: 

    Symbol for US Dollar, en-US locale: $
Symbol for US Dollar, en-UK locale: USD

    Este extracto demuestra que el símbolo de una moneda puede variar dependiendo de la configuración regional.

  • getDisplayName : Devuelve el nombre de visualización para una instancia de Currency. Como el método getSymbol, puede opcionalmente especificar un objeto Locale.

Soporte Extensible para Códigos de Moneda ISO 4217

El ISO 4217 es un estándar publicado por la Organización Internacional de Estándares. Especifica códigos de tres letras (y los códigos equivalentes de tres dígitos numéricos) para representar monedas y fondos. Este estándar es mantenido por una agencia externa y se publica de forma independiente a la plataforma Java SE.

Suponga que un país adopta una moneda diferente y la agencia de mantenimiento ISO 4217 publica una actualización de moneda. Para implementar esta actualización y así reemplazar la moneda predeterminada en tiempo de ejecución, cree un archivo de propiedades llamado <JAVA_HOME>/lib/currency.properties . Este fichero contiene los pares clave/valor del código de país ISO 3166, y los datos de moneda ISO 4217. La parte del valor consiste en tres valores de moneda ISO 4217 separados por comas: un código alfabético, un código numérico, y una unidad menor. Cualquier línea que comienze con el carácter almohadilla (#), se trata como línea de comentarios. Por ejemplo: 

# Sample currency property for Canada
CA=CAD,124,2

CAD significa dólar canadiense; 124 es el código numérico para el dólar canadiense; y 2 es la unidad menor, la cual es el número de lugares decimales que la moneda requiere para representar fracciones de la moneda. Por ejemplo, el siguiente fichero de propiedades reemplazará la moneda canadiense predeterminada por un dólar canadiense que no tiene unidades más pequeñas que el dólar: 

CA=CAD,124,0

Porcentajes

Pude también usar los métodos de la clase NumberFormat para dar formato a los porcentajes. Para obtener un formateador específico de la configuración regional, invoque el método getPercentInstance. Con este formateador, una fracción decimal tal como 0.75 se visualiza como 75%.

El código de ejemplo siguiente muestra como dar formato a un porcentaje. 

static public void displayPercent(Locale currentLocale) {

    Double percent = new Double(0.75);
    NumberFormat percentFormatter;
    String percentOut;

    percentFormatter = NumberFormat.getPercentInstance(currentLocale);
    percentOut = percentFormatter.format(percent);
    System.out.println(percentOut + "   " + currentLocale.toString());
}

Este ejemplo imprime lo siguiente: 

75 %   fr_FR
75%   de_DE
75%   en_US
