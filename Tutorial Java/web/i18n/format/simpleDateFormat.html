Personalizando Formatos

Nota de Versión: Esta sección de Fecha y Tiempo usa las IPAs de fecha y tiempo en el paquete java.util. Las IPAs de java.time, disponibles en la versión 8 del KDJ, proporcionan un modelo de fecha y tiempo completo que ofrece mejoras significativas sobre las clases de java.util. Las IPAs de java.time se describen en el recorrido Fecha y Tiempo. La página Código Date-Time Heredado puede ser de especial interés.

La sección anterior, Usando Formatos Predefinidos, describía los estilos de formato proporcionados por la clase DateFormat. En la mayoría de los casos estos formatos predefinidos son adecuados. Sin embargo, si quiere crear sus propios formatos personalizados, puede usar la clase SimpleDateFormat.

Los ejemplos de código que siguen demuestran los métodos de la clase SimpleDateFormat. Puede encontrar el código fuente completo para los ejemplos en el fichero llamado SimpleDateFormatDemo.

Sobre los Patrones

Cuando crea un objeto SimpleDateFormat, especifica un objeto String de patrón. Los contenidos del objeto String de patrón determinan el formato de la fecha y la hora. Para una descripción completa de la sintaxis de patrones, vea las tablas en Sintaxis del Patrón de Formato de Fecha.

El código siguiente da formato a una fecha y hora de acuerdo al objeto String de patrón pasado al constructor de SimpleDateFormat. El objeto String devuelto por el método format contiene la fecha y hora formateada que va a ser visualizada. 

Date today;
String output;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat(pattern, currentLocale);
today = new Date();
output = formatter.format(today);
System.out.println(pattern + " " + output);

Patrones y Configuración Regional

La clase SimpleDateFormat es sensible a la configuración regional. Si instancia SimpleDateFormat sin un parámetro Locale, dará formato a la fecha y hora de acuerdo al Locale predeterminado. Tanto el patrón como el Locale determinan el formato. Para el mismo patrón, SimpleDateFormat pueda dar formato a una fecha y hora de manera diferente si el Locale varía.

En el código de ejemplo que sigue, el patrón está codificado en la sentencia que crea el objeto SimpleDateFormat: 

Date today;
String result;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat("EEE d MMM yy", currentLocale);
today = new Date();
result = formatter.format(today);
System.out.println("Locale: " + currentLocale.toString());
System.out.println("Result: " + result);

Cuando se establece currentLocale a valores diferentes, el código de ejemplo anterior genera esta salida: 

Locale: fr_FR
Result: mar. 30 juin 09
Locale: de_DE
Result: Di 30 Jun 09
Locale: en_US
Result: Tue 30 Jun 09

Los caracteres que no son letras son tratados como texto entrecomillado. Esto es, aparacerá en el texto formateado incluso si no está encerrado dentro de comillas simples.

El número de letras de símbolo que especifique también determina el formato. Por ejemplo, si el patrón "zz" resulta en "PDT," entonces el patrón "zzzz" genera "Pacific Daylight Time". La tabla siguiente resume estas reglas:
