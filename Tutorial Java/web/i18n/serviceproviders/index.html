Lección: Proveedores de Servicio para Internacionalización

Los proveedores de servicio para la internacionalización habilitan el complemento de servicios y datos dependientes de la configuración regional. Ya que los servicios y datos dependientes de la configuración regional se pueden conectar, los terceros pueden proporcionar implementaciones de la mayoría de las clases sensibles a la configuración regional en los paquetes java.text y java.util.

Un servicio es un conjunto de interfaces y clases de programación que proporcionan acceso a una funcionalidad o característica específica de la aplicación. Un interfaz de proveedor de servicios (SPI) es el conjunto de interfaces públicas y clases abstractas que un servicio define. Un proveedor de servicios implementa el SPI. Los proveedores de servicio le capacitan para crear aplicaciones extensibles, los cuales puede extender sin modificar el código base original. Puede mejorar su funcionalidad con nuevos complementos o módulos. Para más infromación sobre los proveedores de servicios y aplicaciones extensibles, vea Creando Aplicaciones Extensibles.

Puede usar proveedores de servicio para internacionalización para proporcionar implementaciones personalizadas de las clases sensibles a la configuración regional siguientes:

  • Objetos BreakIterator .
  • Objetos Collator.
  • Código de idioma, código de país, y nombre de variante para la clase Locale.
  • Nombres de zonas horarias.
  • Símbolos de moneda
  • Objetos DateFormat.
  • Objetos DateFormatSymbols.
  • Objetos NumberFormat .
  • Objetos DecimalFormatSymbols.

Los SPIs correspondientes están contenidos tanto en el paquete java.text.spi como en el paquete java.util.spi:

java.util.spi java.text.spi
  • CurrencyNameProvider
  • LocaleServiceProvider
  • TimeZoneNameProvider
  • BreakIteratorProvider
  • CollatorProvider
  • DateFormatProvider
  • DateFormatSymbolsProvider
  • DecimalFormatSymbolsProvider
  • NumberFormatProvider

Por ejemplo, si quiere proporcionar un objeto NumberFormat para una nueva configuración regional, implemente la clase java.text.spi.NumberFormatProvider e implemente estos métodos:

  • getCurrencyInstance(Locale locale)
  • getIntegerInstance(Locale locale)
  • getNumberInstance(Locale locale)
  • getPercentInstance(Locale locale) 
Locale loc = new Locale("da", "DK");
NumberFormat nf = NumberFormatProvider.getNumberInstance(loc);

Estos métodos primero comprueban si el entorno de ejecución de Java soporta la configuración regional requerida. Si es así, los métodos usan ese soporte. De lo contrario, los métodos invocan los métodos getAvailableLocales de los proveedores instalados para el interfaz apropiado para encontrar un proveedor que admita la configuración regional requerida.

Para un ejemplo en profundidad de cómo usar los proveedores de servicio para internacionalización, vea Instalando un Resource Bundle Personalizado Como una Extensión. Esta sección le muestra cómo implementar el interfaz ResourceBundleControlProvider, el cual le habilita a usar cualesquiera clases ResourceBundle.Control sin ningún cambio adicional al código fuente de su aplicación.
