Sobre la Clase ResourceBundle

Cómo se Relaciona un ResourceBundle con un Locale

Conceptualmente cada ResourceBundle es un conjunto de subclases relacionadas que comparten el mismo nombre de base. La lista que sigue muestra un conjunto de subclases relacionadas. ButtonLabel es el nombre base. Los caracteres que siguen al nombre base indican el código de idioma, el código de país, y la variante de un Locale. ButtonLabel_en_GB, por ejemplo, coincide con el Locale especificado por el código de idioma para el inglés (en) y el código de país para Gran Bretaña (GB). 

ButtonLabel
ButtonLabel_de
ButtonLabel_en_GB
ButtonLabel_fr_CA_UNIX

Para seleccionar el ResourceBundle apropiado, invoque el método ResourceBundle.getBundle. El ejemplo siguiente seleciona el ResourceBundle ButtonLabel para el Locale que coincide con el idioma francés, el país de Canadá, y la plataforma UNIX. 

Locale currentLocale = new Locale("fr", "CA", "UNIX");
ResourceBundle introLabels = ResourceBundle.getBundle(
                                 "ButtonLabel", currentLocale);

Si un clase ResourceBundle para el Locale especificado no existe, getBundle intenta encontrar la coincidencia más cercana. Por ejemplo, si ButtonLabel_fr_CA_UNIX es la clase deseada y el Locale predeterminado es en_US, getBundle buscará clases en el orden siguiente: 

ButtonLabel_fr_CA_UNIX
ButtonLabel_fr_CA
ButtonLabel_fr
ButtonLabel_en_US
ButtonLabel_en
ButtonLabel

Note que getBundle busca clases basados en el Locale predeterminado antes de que seleccione la clase base (ButtonLabel). Si getBundle falla en encontrar una coincidencia en la lista precedente de clases, lanza una MissingResourceException. Para evitar lanzar esta excepción, siempre debe proporcionar una clase base sin sufijos.

Las Subclases ListResourceBundle y PropertyResourceBundle

La clase abstracta ResourceBundle tiene dos subclases: PropertyResourceBundle y ListResourceBundle.

Un PropertyResourceBundle está respaldado por un fichero de propiedades. Un fichero de propiedades es un fichero de texto plano que contiene texto traducible. Los ficheros de propiedades no son parte del código fuente de Java, y sólo pueden contener valores para objetos String. Si necesita almacenar otros tipos de objetos, use en su lugar un ListResourceBundle. La sección Respaldando un ResourceBundle con Ficheros de Propiedades le muestra cómo usar un PropertyResourceBundle.

La clase ListResourceBundle gestiona los recursos con una lista apropiada. Cada ListResourceBundle está respaldada por un fichero class. Puede almacenar cualquier objeto específico de configuración regional en una ListResourceBundle. Para agregar soporte para un Locale adicional, cree otro fichero fuente y compílelo en un fichero class. La sección Usando un ListResourceBundle tiene un ejemplo de código que puede encontrar útil.

La clase ResourceBundle es flexible. Si coloca primero sus objetos String específicos de configuración regional en un PropertyResourceBundle y después más tarde decide usar ListResourceBundle en su lugar, no hay impacto en su código. Por ejemplo, la siguiente llamada a getBundle recuperará un ResourceBundle para el Locale apropiado, ya sea que ButtonLabel esté respaldado por una clase o por un fichero de propiedades: 

ResourceBundle introLabels = ResourceBundle.getBundle(
                                 "ButtonLabel", currentLocale);

Pares Clave-Valor

Los objetos ResourceBundle contienen una matriz de pares clave-valor. Usted especifica la clave, la cual debe ser un String, cuando quiere recuperar el valor desde el ResourceBundle. El valor es el objeto específico de configuración regional. Las claves en el ejemplo siguiente son las cadenas OkKey y CancelKey: 

class ButtonLabel_en extends ListResourceBundle {
    // English version
    public Object[][] getContents() {
        return contents;
    }
    static final Object[][] contents = {
        {"OkKey", "OK"},
        {"CancelKey", "Cancel"},
    };
}

Para recuperar la String OK desde el ResourceBundle, debe especificar la clave apropiada cuando invoca getString: 

String okLabel = ButtonLabel.getString("OkKey");

Un fichero de propiedades contiene pares clave-valor. La clave está al lado izquierdo del signo igual, y el valor está a la derecha. Cada par está en una línea separada. Los valores pueden sólo representar objetos String. El ejemplo siguiente muestra los contenidos de un fichero de propiedades llamado ButtonLabel.properties: 

OkKey = OK
CancelKey = Cancel
