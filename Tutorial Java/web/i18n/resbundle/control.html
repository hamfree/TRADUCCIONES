Personalizando la Carga de un Resource Bundle

Más temprano en esta lección ha aprendido cómo crear y acceder objetos de la clase ResourceBundle. Esta sección amplía sus conocimientos y explica cómo aprovechar las ventajas de las capacidades de la clase ResourceBundle.Control.

La clase ResourceBundle.Control fue creada para especificar cómo ubicar e instanciar paquetes de recursos. Define un conjunto de métodos de retrollamada que son invocados por los métodos de factoría de ResourceBundle.getBundle durante el proceso de carga del paquete.

A diferencia del método ResourceBundle.getBundle descrito anteriormente, este método ResourceBundle.getBundle define un paquete de recursos usando el nombre base especificado, la configuración regional predeterminada y el control especificado. 

public static final ResourceBundle getBundle(
    String baseName,
    ResourceBundle.Control cont
    // ...

El control especificado proporciona información para el proceso de carga del paquete de recursos.

El programa de ejemplo siguiente llamado RBControl.java ilustra cómo definir sus propias rutas de búsqueda para las configuraciones regionales del chino.

1. Crear los Ficheros de properties.

Como se describió antes puede cargar sus recursos desde clases o desde ficheros de properties. Estos ficheros contienen descripciones para las configuraciones regionales siguientes:

  • RBControl.properties – Global
  • RBControl_zh.properties — Sólo idioma: Chino Simplificado
  • RBControl_zh_cn.properties – Sólo región: China
  • RBControl_zh_hk.properties – Sólo región: Hong Kong
  • RBControl_zh_tw.properties – Taiwán

En este ejemplo una aplicación crea una nueva configuración regional para la región de Hong Kong.

2. Crear una instancia de ResourceBundle.

Como en el ejemplo de la sección previa, esta aplicación crea una instancia de ResourceBundle invocando el método getBundle: 

private static void test(Locale locale) {
    ResourceBundle rb = ResourceBundle.getBundle(
                            "RBControl",
                            locale,
                            new ResourceBundle.Control() {
                                    // ...
                            }
                        );

El método getBundle busca ficheros de properties con el prefijo RBControl. Sin embargo, este método contiene un parámetro Control, el cual dirige el proceso de búsqueda de las configuraciones regionales del idioma chino.

3. Invocar el método getCandidateLocales

El método getCandidateLocales devuelve una lista de los objetos Locales como configuraciones regionales candidatas para el nombre base y configuración regional. 

new ResourceBundle.Control() {
    @Override
    public List<Locale> getCandidateLocales(
                            String baseName,
                            Locale locale) {
                // ...                                        
    }
}

La implementación predeterminada devuelve una lista de objetos Locale como sigue: Locale(language, country).

Sin embargo, este método es sobreescrito para implementar el siguiente comportamiento específico: 

if (baseName == null)
    throw new NullPointerException();

if (locale.equals(new Locale("zh", "HK"))) {
    return Arrays.asList(
               locale,
               Locale.TAIWAN,
               // no Locale.CHINESE here
               Locale.ROOT);
} else if (locale.equals(Locale.TAIWAN)) {
    return Arrays.asList(
               locale,
               // no Locale.CHINESE here
               Locale.ROOT);
}

Tenga en cuenta que el último elemento de la secuencia de entornos locales candidatos debe ser un entorno local raíz.

4. Llamar la clase test

Llame a la clase test para las siguientes cuatro configuraciones regionales diferentes: 

public static void main(String[] args) {
    test(Locale.CHINA);
    test(new Locale("zh", "HK"));
    test(Locale.TAIWAN);
    test(Locale.CANADA);
}

5. Ejecutar el Programa de Ejemplo

Verá la salida del programa como sigue: 

locale: zh_CN
        region: China
        language: Simplified Chinese
locale: zh_HK
        region: Hong Kong
        language: Traditional Chinese
locale: zh_TW
        region: Taiwan
        language: Traditional Chinese
locale: en_CA
        region: global
        language: English

Tenga en cuenta que al recién creado se le asignó la región de Hong Kong, porque se especificó en un archivo de properties apropiado. El chino tradicional se asignó como el idioma para la configuración regional de Taiwán.

Otros dos métodos interesantes de la clase ResourceBundle.Control no se usaron en el ejemplo RBControl, pero merecen ser mencionados. El método getTimeToLive se usa para determinar cuánto tiempo puede existir el paquete de recursos en la caché. Si el tiempo límite para un paquete de recursos en la caché ha expirado, el método needsReload se invoca para determinar si el paquete de recursos necesita ser recargado.
