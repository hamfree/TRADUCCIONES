Lección: Aislando Datos de una Localización Específica

Los datos específicos de la configuración regional deben adaptarse de acuerdo con las convenciones del idioma y la región del usuario final. El texto visualizado por un interfaz de usuario es el ejemplo más obvio de datos específicos de la configuración regional. Por ejemplo, una aplicación con un botón Cancel en los EE.UU tendrá un botón Abbrechen en Alemania. En otros países este botón tendrá otras etiquetas. Obviamente no quiere escribir a mano esta etiqueta de botón. ¿No sería bueno si pudiera obtener automáticamente la etiqueta correcta para un Locale dado? Afortunadamente puede, siempre que aísle los objetos específicos de la configuración regional en un ResourceBundle.

En esta lección aprenderá cómo crear y acceder a objetos ResourceBundle. Si tiene prisa examine algunos ejemplos de codificación, continúe y consulte las dos últimas secciones en esta lección. Después puede volver atrás a las dos primeras secciones para obtener algo de información conceptual sobre los objetos ResourceBundle.

Sobre la Clase ResourceBundle

Los objetos ResourceBundle contiene objetos específicos a una configuración regional. Cuando necesita un objeto específico de una configuración regional, lo obtiene desde un ResourceBundle, el cual devuelve el objeto que coincide con el Locale del usuario final. Esta sección le explica cómo un ResourceBundle está relacionado con un Locale, y describe las subclases de ResourceBundle.

Preparándose para Utilizar un ResourceBundle

Antes de crear sus objetos ResourceBundle, debería hacer un poco de planificación. Primero, identifique los objetos específicos de configuración regional en su programa. Después organízelos en categorías y almacénelos en diferentes objetos ResourceBundle según corresponda.

Respaldando un ResourceBundle con Ficheros de Propiedades

Si su aplicación contiene objetos String que necesitan ser traducidos en varios idiomas, puede almacenar estos objetos String en un PropertyResourceBundle, el cual está respaldado por un conjunto de ficheros de propiedades. Ya que los ficheros de propiedades son simples ficheros de texto, pueden ser creados y mantenidos por sus traductores. No tiene que cambiar el código fuente. En esta sección aprenderá cómo establecer los ficheros de propiedades que respaldan un PropertyResourceBundle.

Usando un ListResourceBundle

La clase ListResourceBundle, la cual es una subclase de ResourceBundle, gestiona objetos específicos de configuración regional con una lista. Una ListResourceBundle está respaldada por un fichero de clase, lo que significa que debe codificar y compilar un fichero fuente nuevo cada vez que se necesite soporte para un Locale adicional. Sin embargo, los objetos ListResourceBundle son útiles porque a diferencia de los ficheros de propiedades, pueden almacenar cualquier tipo de objeto específico de la configuración regional. Al recorrer un progrmaa de muestra, esta sección demuestra como usar una ListResourceBundle.

Personalizando la Carga de un Resource Bundle

Esta sección representa las nuevas capacidades para mejorar la flexibilidad de la factoría ResourceBundle.getBundle. La clase ResourceBundle.Control colabora con los métodos de factoría para cargar paquetes de recursos. Esto permite considerar cada paso sustancial del proceso de carga del paquete de recursos y su control de caché como un método separado.
