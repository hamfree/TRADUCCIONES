Usando un ListResourceBundle

Esta sección ilustra el uso de un objeto ListResourceBundle con un programa de ejemplo llamado ListDemo. El texto que sigue explica cada paso involucrado en la creación del programa ListDemo, junto con las subclases ListResourceBundle que lo soportan.

1. Crear las Subclases ListResourceBundle

Una ListResourceBundle está respaldada por un fichero class. Por lo tanto el primer paso es crear un fichero classs para cada Locale admitido. En el programa ListDemo el nombre base de ListResourceBundle es is StatsBundle. Ya que ListDemo adminte tres objetos Locale, requiere los tres ficheros class siguientes: 

StatsBundle_en_CA.class
StatsBundle_fr_FR.class
StatsBundle_ja_JP.class

La clase StatsBundle para Japón se define en el código fuente que sigue. Dese cuenta de que el nombre de la clase se construye agragando el código de idioma y país al nombre base de la ListResourceBundle. Dentro de la clase la matriz bidimensional contents es inicializada con los pares clave-valor. Las claves son el primer elemento en cada par: GDP, Population, y Literacy. La claves deben ser objetos String y deben ser los mismos en cada clase del conjunto StatsBundle. Los valores pueden ser cualquier tipo de objeto. En este ejemplo los valores son dos objetos Integer y un objeto Double. 

import java.util.*;
public class StatsBundle_ja_JP extends ListResourceBundle {
    public Object[][] getContents() {
        return contents;
    }

    private Object[][] contents = {
        { "GDP", new Integer(21300) },
        { "Population", new Integer(125449703) },
        { "Literacy", new Double(0.99) },
    };
}

2. Especificar el Locale

El programa ListDemo define los objetos Locale como sigue: 

Locale[] supportedLocales = {
    new Locale("en", "CA"),
    new Locale("ja", "JP"),
    new Locale("fr", "FR")
};

Cada objeto Locale se corresponde con una de las clases StatsBundle. Por ejemplo, el Locale japonés, el cual está definido con los códigos ja y JP, coincide con StatsBundle_ja_JP.class.

3. Crear el ResourceBundle

Para crear la ListResourceBundle, invoque el método getBundle. La línea siguietne de código especifica el nombre base de la clase (StatsBundle) y el Locale: 

ResourceBundle stats = ResourceBundle.getBundle("StatsBundle", currentLocale);

El método getBundle busca una clase cuyo nombre empieza con StatsBundle y va seguida de los códigos de idioma y país del Locale especificado. Si currentLocale se crea con los códigos ja y JP, getBundle devuelve una ListResourceBundle correspondiente con la clase StatsBundle_ja_JP, por ejemplo.

4. Obtener los Objetos Localizados

Ahora que el programa tiene una ListResourceBundle para el Locale adecuado, puede obtener los objetos traducidos por sus claves. La siguiente línea de código recupera la tasa de alfabetización invocando a getObject con el parámetro clave Literacy. Ya que getObject devuelve un objeto, lo moldeamos a un Double: 

Double lit = (Double)stats.getObject("Literacy");

5. Ejecutar el Programa Demostración

El programa ListDemo imprime los datos que recuperó con el método getBundle: 

Locale = en_CA
GDP = 24400
Population = 28802671
Literacy = 0.97

Locale = ja_JP
GDP = 21300
Population = 125449703
Literacy = 0.99

Locale = fr_FR
GDP = 20200
Population = 58317450
Literacy = 0.99
