Java Web Start y la Seguridad

Esta sección describe lo fundamental de seguridad para las aplicaciones desplegadas a través de Java Web Start e incluye:

Las aplicaciones lanzadas con Java Web Start son, por defecto, ejecutadas en un entorno restringido, conocido como una caja de arena (sandbox, en inglés). En esta caja de arena, Java Web Start:

  • Protege a los usuarios contra código malicioso que podría afectar a ficheros locales
  • Protege a las empresas contra código que podría intentar acceder o destruir datos en redes

Las aplicaciones de caja de arena que son lanzadas por Java Web Start permanecen en esta caja de arena, lo que significa que no pueden acceder a los ficheros locales o a la red. Vea La Seguridad en Aplicaciones Ricas de Internet por información.

Descarga Dinámica de Certificados HTTPS

Java Web Start importa dinámicamente los certificados como suelen hacer los navegadores. Para hacer esto, Java Web Start establece su propio manejador https, usando la propiedad del sistema java.protocol.handler.pkgs, para inicializar valores por defecto para SSLSocketFactory y HostnameVerifier. Establece los valores por defecto con los métodos HttpsURLConnection.setDefaultSSLSocketFactory y HttpsURLConnection.setDefaultHostnameVerifier.

Si su aplicación usa estos dos métodos, asegúrese de que son invocados después de que Java Web Start inicialize el manejador https, de otra forma su manejador personalizado será reemplazado con el manejador por defecto de Java Web Start.

Puede asegurarse de que sus propios y personalizados SSLSocketFactory y HostnameVerifiter son usados realizando una de las siguientes acciones:

  • Instalar su propio manejador https, para reemplazar el manejador https de Java Web Start.
  • En su aplicación, invoque HttpsURLConnection.setDefaultSSLSocketFactory o HttpsURLConnection.setDefaultHostnameVerifier sólo después de que el primer objeto https URL sea creado, el cual ejecuta el código de inicialización del controlador Java Web Start https primero.
