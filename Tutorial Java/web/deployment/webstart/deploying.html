Desplegando una Aplicación Java Web Start

Para desplegar su aplicación de Java Web Start, compile primero el código fuente, empaquételo como un fichero JAR, y firme el fichero JAR.

Las aplicaciones Java Web Start son lanzadas usando el Protocolo de Lanzamiento de Red de Java (JNLP). Por lo tanto, debe crear un fichero JNLP para desplegar su aplicación.

El script de las Herramientas de Despliegue contiene funciones de JavaScript útiles que pueden ser usadas para desplegar aplicaciones Java Web Start en una página web.

Si no está familiarizado con estas tecnologías de despliegue, revise la lección Despliegue En Profundidad antes de proceder.

Aquí tiene algunas instrucciones paso a paso para empaquetar y desplegar su aplicación. La aplicación Demo de Arbol Dinámico es usada para ilustrar el despliegue de las aplicaciones Java Web Start. Podría querer el configurar scripts para ejecutar alguno de los siguienes pasos.

Pulse en el botón Lanzar siguiente para lanzar la aplicación Demo del Árbol Dinámico.

Nota:  Si no ve el ejemplo ejecutándose, podría necesitar el habilitar el intérprete de JavaScript en su navegador de forma que el script de Herramientas de Despliegue pueda funcionar adecuadamente.
  1. Compile el código Java de su aplicación y asegúrese de que todos los ficheros de clases y recursos tales como imágenes están en un directorio separado.

    En la aplicación Demo del Árbol Dinámico, las clases compiladas son colocadas en el directorio build/classes/webstartComponentArch.

  2. Cree un fichero de texto que contenga los atributos de manifiesto del archivo JAR que su applet necesite.

    Para el applet Demo del Árbol Dinámico, cree un fichero llamado mymanifest.txt en el directorio build/classes, y agregue los atributos Permissions, Codebase, y Application-Name. El applet no requiere acceso a los recursos del sistema del usuario, así que use sandbox para los permisos. Use el dominio desde el cual usted cargará el ejemplo para el código base, por ejemplo, myserver.com. Agregue los atributos siguientes al fichero mymanifest.txt. 

    Permissions: sandbox
Codebase: myserver.com
Application-Name: Dynamic Tree Demo

    Otros atributos del manifiesto están disponibles para restringir a un applet a que use sólo código confiable, y a ofrecer seguridad para applets que necesitan hacer llamadas entre código de Java Privilegiado y código de Java de sandbox, o tiene código de JavaScript que llama el applet. Vea la lección Mejorando la Seguridad con los Atributos del Manifiesto para aprender más sobre los atributos de manifiesto que están disponibles.

  3. Cree un fichero JAR que contenga sus ficheros de clase y recursos de la aplicación. Incluya los atributos de manifiesto en el fichero mymanifest.txt que creó en el paso previo.

    Por ejemplo, el comando siguiente crea un fichero JAR con los ficheros class en el directorio build/classes/webstartComponentArch y el fichero de manifiesto en el directorio build/classes. 

    % cd build/classes
% jar cvfm  DynamicTreeDemo.jar  mymanifest.txt webstartComponentArch

    Vea la lección Empaquetando Programas en Ficheros JAR para aprender más sobre la creación y uso de los ficheros JAR.

  4. Firme el fichero JAR y haga una marca de tiempo de la firma. Utilice un certificado de firma de código válido y actual emitido por una autoridad de certificación confiable para brindar a los usuarios la seguridad de que es seguro ejecutar el applet.

    Vea la lección Firmando Ficheros JAR para más información.

    Si quiere usar un fichero JNLP firmado por seguridad, cree el fichero JNLP como está descrito en el siguiente paso e incluyalo en el fichero JAR antes de que el fichero JAR sea firmado. Vea Ficheros JNLP Firmados en la Plataforma Java, Guía de Despliegue de la Edición Estándar para más información.

  5. Cree un fichero JNLP que describa cómo debe ser lanzada su aplicación.

    Aqui está el fichero JNLP que se usa para lanzar la aplicación Demo del Árbol Dinámico. Los permisos no son pedidos para esta aplicación ya que se ejecuta en la caja de arena de seguridad. El código para dynamictree_webstart.jnlp se muestra a continuación: 

    <?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase=
"https://docs.oracle.com/javase/tutorialJWS/samples/deployment/webstart_ComponentArch_DynamicTreeDemo" 
    href="dynamictree_webstart.jnlp">
    <information>
        <title>Dynamic Tree Demo</title>
        <vendor>Dynamic Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.7+"
              href="http://java.sun.com/products/autodl/j2se"/>
        <jar href="DynamicTreeDemo.jar"
            main="true" />

    </resources>
    <application-desc
         name="Dynamic Tree Demo Application"
         main-class=
           "webstartComponentArch.DynamicTreeApplication"
         width="300"
         height="300">
     </application-desc>
     <update check="background"/>
</jnlp>
    Estructura del Fichero JNLP describe la sintaxis y opciones del fichero JNLP.
    Nota: Los atributos codebase y href son opcionales cuando se implementan aplicaciones Java Web Start que se ejecutarán en al menos la versión Java SE 6 actualización 18 o posterior. Debe especificar los atributos codebase y href cuando despliegue aplicaciones Java Web Start que se ejecutarán en versiones anteriores del software del Entorno de Ejecución de Java.
  6. Cree la página HTML desde la cual su aplicación será lanzada. Invoque las funciones de las Herramientas de Desplieuge para desplegar la aplicación Java Web Start.

    En el ejemplo, la aplicación Demo del Árbol Dinámico es desplegada en JavaWebStartAppPage.html. 

    <body>
    <!-- ... -->
    <script src=
      "https://www.java.com/js/deployJava.js"></script>
    <script>
        // using JavaScript to get location of JNLP
        // file relative to HTML page
        var dir = location.href.substring(0,
            location.href.lastIndexOf('/')+1);
        var url = dir + "dynamictree_webstart.jnlp";
        deployJava.createWebStartLaunchButton(url, '1.7.0');
    </script>
    <!-- ... -->
</body>

    Si no está seguro de si sus usuarios finales tendrán el intérprte de JavaScript habilitado en sus navegadores, puede desplegar la aplicación Java Web Start directamente creando un enlace al fichero JNLP como sigue: 

    <a href="/ruta absoluta al fichero JNLP/dynamictree_webstart.jnlp">Lanza Aplicación Bloc de Notas</a>

    Si despliega la aplicación Java Web Start con un enlace directo, no puede aprovecharse de las comprobaciones adicionales que las funciones de las Herramientas de Despliegue ofrecen. Vea Desplegando una Aplicación Java Web Start en la lección Despliegue en Profundidad por los detalles.

  7. Coloque el fichero JAR de la aplicación, el fichero JNLP, y la página HTML en las carpetas apropiadas.

    Para este ejemplo, coloque DynamicTreeDemo.jar, dynamictree_webstart.jnlp, y JavaWebStartAppPage.html en el mismo directorio en la máquina local o en un servidor web. Se prefiere un servidor web. Para ejecutar desde la máquina local, debe agregar su aplicación a la lista de excepciones de sitios, que es gestionada desde la pestaña de Seguridad del Panel de Control de Java.

  8. Abra la página HTML de la aplicación en un navegador para ver la aplicación. Acepte ejecutar la aplicación cuando se le solicite. Compruebe el log de la Consola Java por mensajes de error y depuración.

Descargar código fuente para el ejemplo Demo del Árbol Dinámico para experimentar más.
