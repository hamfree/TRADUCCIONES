Desarrollando una Aplicación Java Web Start

El software diseñado usando una arquitectura basada en componentes puede ser fácilmente desarrollado y desplegado como una aplicación Java Web Start. Considere el ejemplo de una aplicación Java Web Start con un interfaz gráfico de usuario (IGU) basado en Swing. Con el diseño basado en componentes, el IGU puede ser construído con bloques de construcción más pequeños o componentes. Los pasos generales siguientes se usan para crear un IGU de aplicación:

  • Cree una clase MyTopJPanel que sea una subclase de JPanel. Coloque sus componentes del IGU de la aplicación en el constructor de la clase MyTopJPanel.
  • Cree una clase llamada MyApplication que sea una subclase de la clase JFrame.
  • En el método main de la clase MyApplication, cree una instancia de la clase MyTopJPanel y establézcala como el panel de contenido de JFrame.

Las secciones siguientes exploran estos pasos en mayor detalle usando la aplicación Demo del Árbol Dinámico. Si no está familiarizado con Swing, vea Creando una IGU (Interfaz Gráfica de Usuario) con JFC/Swing para aprender más sobre el uso de componentes IGU de Swing.

Pulse el botón Lanzar siguiente para lanzar la aplicación Demo de Árbol Dinámico.

Nota:  Si no ve el ejemplo ejecutándose, es posible que necesite habilitar el intérprete de JavaScript en su navegador para que el script de las Herramientas de Despliegue funcione adecuadamente.

Crear la Clase Superior JPanel

Cree una clase que sea una subclase de JPanel. Esta JPanel superior actúa como un contenedor de todos sus otros componentes de IU (Interfaz de Usuario). En el ejemplo siguiente, la clase DynamicTreePanel es la JPanel superior. El constructor de la clase DynamicTreePanel invoca otros métodos para crear y colocar los controles del IU apropiadamente. 

public class DynamicTreePanel extends JPanel implements ActionListener {
    private int newNodeSuffix = 1;
    private static String ADD_COMMAND = "add";
    private static String REMOVE_COMMAND = "remove";
    private static String CLEAR_COMMAND = "clear";
    
    private DynamicTree treePanel;

    public DynamicTreePanel() {
        super(new BorderLayout());
        
        //Crea los componentes.
        treePanel = new DynamicTree();
        populateTree(treePanel);

        JButton addButton = new JButton("Add");
        addButton.setActionCommand(ADD_COMMAND);
        addButton.addActionListener(this);
        
        JButton removeButton = new JButton("Remove");
        ....
        
        JButton clearButton = new JButton("Clear");
        ...
        
        //Coloca todo.
        treePanel.setPreferredSize(
            new Dimension(300, 150));
        add(treePanel, BorderLayout.CENTER);

        JPanel panel = new JPanel(new GridLayout(0,3));
        panel.add(addButton);
        panel.add(removeButton); 
        panel.add(clearButton);
        add(panel, BorderLayout.SOUTH);
    }
    // ...
}

Cree la Aplicación

Para una aplicación que tiene un IGU basado en Swing, cree una clase que sea una subclase de javax.swing.JFrame.

Haga una instancia de su clase JPanel superior y configúrela como el panel contenedor de la clase JFrame en el método main de la aplicación. El método main de la clase DynamicTreeApplication invoca el método createGUI en el hilo del Despachador de Eventos de AWT. 

package webstartComponentArch;

import javax.swing.JFrame;

public class DynamicTreeApplication extends JFrame {
    public static void main(String [] args) {
        DynamicTreeApplication app = new DynamicTreeApplication();
        app.createGUI();
    }

    private void createGUI() {
        //Crea y configura el panel de contenido.
        DynamicTreePanel newContentPane = new DynamicTreePanel();
        newContentPane.setOpaque(true); 
        setContentPane(newContentPane);
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        pack();
        setVisible(true);
    }    
}

Beneficios de Separar la Funcionalidad Básica del Mecanismo de Despliegue Final

Otra forma de crear una aplicación es simplemente eliminar la capa de abstracción (separar la clase superior JPanel) y disponer todos los controles en el propio método main de la aplicación. La desventaja de crear la GUI directamente en el método main de la aplicación es que será más difícil implementar su funcionalidad como un applet, si elige hacerlo más adelante.

En el ejemplo Demo del Árbol Dinámico, la funcionalidad básica está separada en la clase DynamicTreePanel. Es ahora trivial dejar caer la clase DynamicTreePanel dentro de una clase JApplet y desplegarla como un applet.

Por lo tanto, para conservar la portabilidad y mantener abiertas las opciones de implementación, siga el diseño basado en componentes como se describe en este tema.
