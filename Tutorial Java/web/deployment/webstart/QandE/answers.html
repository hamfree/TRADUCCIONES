Respuestas a Preguntas y Ejercicios: Java Web Start

Preguntas

  1. Pregunta: En un enlace que está para ejecutar una aplicación de Java Web Start, ¿Qué fichero es especificado como atributo href de la etiqueta a?

    Respuesta: Usa el nombre de fichero JNLP de la aplicación como el valor del atributo href. Cuando un usuario hace click sobre el enlace al fichero JNLP, Java Web Start carga la aplicación especificada por ese fichero JNLP.

  2. Pregunta: ¿Qué tipo MIME debe reconocer un servidor Web para que pueda alojar aplicaciones Java Web Start?

    Respuesta: Debe configurar el servidor web de forma que los ficheros con extensión .jnlp se asocien con el tipo MIME application/x-java-jnlp-file.

  3. Pregunta: En el fichero JNLP de una aplicación, ¿qué dos elementos deben ser especificados dentro del elemento resources?

    Respuesta: El elemento resources debe contener:

    • El elemento j2se, el cual especifica la plataforma Java en la cual se ejecuta la aplicación.
    • El elemento jar, el cual especifica el fichero JAR para la aplicación.

  4. Pregunta: ¿Qué interfaz ofrece la habilidad de controlar cómo son cacheados los recursos de la aplicación Java Web Start?

    1. BasicService
    2. DownloadService
    3. PersistenceService
    4. ExtendedService

    Respuesta: B. El interfaz DownloadService suministra la habilidad a una aplicación de controlar cómo se cachean sus propios recursos.

  5. Pregunta: Verdadero o Falso: las aplicaciones Java Web Start se ejecutan en cajas de arena seguras por defecto.

    Respuesta: Verdadero.

  6. Pregunta: Verdadero o Falso: Si una aplicación Java Web Start se está ejecutando en una caja de arena segura, los ficheros JAR para la aplicación pueden residir en servidores diferentes.

    Respuesta: Falso. Todos los ficheros JAR para la aplicación deben residir en el mismo servidor.

  7. Pregunta: Para que una aplicación Java Web Start soporte operaciones fuera de la caja de arena segura, ¿qué tenemos que hacer?

    Respuesta: Debe incluir el elemento security en su fichero .jnlp y especificar all-permissions para habilitar a su aplicación a que trabaje fuera de la caja de arena.

Ejercicios

  1. Ejercicio: Escriba el código XML que agregaría a un fichero JNLP para solicitar que la aplicación tenga acceso completo al sistema cliente.

    Respuesta: 

    <security>
   <all-permissions/>
</security>

  2. Ejercicio: Para una aplicación Java Web Start, tiene usted dos iconos, one.gif y two.gif, en el directorio images en un fichero JAR. Escriba el código de la aplicación que usaría para acceder a estas imágenes.

    Respuesta: 

    // Obtengo el classloader actual
ClassLoader cl = this.getClass().getClassLoader();
// Creo los iconos
Icon firstIcon  = new ImageIcon(cl.getResource("images/one.gif"));
Icon secondIcon   = new ImageIcon(cl.getResource("images/two.gif"));
