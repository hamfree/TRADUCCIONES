Problemas Comunes en Java Web Start

Esta sección cubre algunos problemas comunes que podría encontrar cuando desarrolla y despliega aplicaciones Java Web Start. Después de cada problema hay una lista de posibles razones y soluciones.

Problema: Mi navegador muestra el fichero de Protocolo de Lanzamiento de Java en Red (JNLP) para mi aplicación como texto plano.

Lo más probable, es que su servidor web no conozca el tipo MIME adecuado para los ficheros JNLP. Vea la sección Configurando un Servidor Web para más información.

Además, si está usando un servidor proxy, asegúrese de que se devuelvan las versiones actualizadas de los archivos, actualizando la marca de tiempo de los recursos en el servidor web de modo que los servidores proxy actualicen sus cachés.

Problema: Cuando intento lanzar mi fichero JNLP, obtengo el error siguiente: 

MissingFieldException[ The following required field is missing from the launch
  file: (<application-desc>|<applet-desc>|<installer-desc>|<component-desc>)]
        at com.sun.javaws.jnl.XMLFormat.parse(Unknown Source)
        at com.sun.javaws.jnl.LaunchDescFactory.buildDescriptor(Unknown Source)
        at com.sun.javaws.jnl.LaunchDescFactory.buildDescriptor(Unknown Source)
        at com.sun.javaws.jnl.LaunchDescFactory.buildDescriptor(Unknown Source)
        at com.sun.javaws.Main.launchApp(Unknown Source)
        at com.sun.javaws.Main.continueInSecureThread(Unknown Source)
        at com.sun.javaws.Main.run(Unknown Source)
        at java.lang.Thread.run(Unknown Source)

A menudo este error ocurre cuando su XML está mal formado. Puede mirar el código hasta que lo descubra, pero es más fácil ejecutar un verificador de sintaxis XML sobre el archivo. (El EID NetBeans y JEdit tienen ambos comprobadores de sintaxis de XML).

Sin embargo, este error puede ocurrir en otras situaciones y lo anterior fue causado por la siguiente línea en un archivo XML bien formado: 

<description kind="short">Demonstrates choosing the drop location in the target <code>TransferHandler</code></description>

El error fue causado por las etiquetas ilegales embebidas code.
