Evitando innecesarias comprobaciones de actualización

Las aplicaciones ricas de Internet (ARIs) son cacheadas localmente para mejorar el tiempo de arranque. Sin embargo, antes de lanzar una ARI, el software de lanzamiento comprueba para asegurarse que cada fichero JAR referenciado en el fichero de Protocolo de Lanzamiento en Rede de Java (JNLP) de la ARI esté actualizado. En otras palabras, el software de lanzamiento se adegura de que usted está ejecutando la última versión de la ARI y no una copia cacheada más antigua. Esta comprobación de actualización puede tomar unos pocos cientos de milisegundos dependiendo del número de ficheros JAR y la velocidad de la red. Use las técnicas descritas en este tema para evitar comprobaciones de actualización innecesarias y para mejorar el tiempo de inicio de su ARI.

Nota: 

El término "software de lanzamiento" se usa aquí para referirse colectivamente al software del Plug-in de Java y el software de Java Web Start. El software del Plug-in de Java lanza los applets mientras que el software de Java Web Start lanza las aplicaciones Java Web Start.

Aprovechando el protocolo de descarga de versión

Usted puede aprovechar el protocolo de descarga de versión para eliminar comprobaciones de versión no necesarias. Vea los siguientes pasos para habilitar este protocolo.

  1. Renombre los ficheros JAR para que incluyan un sufijo de número de versión con la siguiente convención de nombrado: 
        <nombre del fichero JAR>__V<número de versión>.jar
    Por ejemplo, renombre DynamicTreeDemo.jar a DynamicTreeDemo__V1.0.jar.
  2. En el fichero JNLP, especifique una versión para cada fichero JAR, y establezca la propiedad jnlp.versionEnabled a true. 
    <resources>
    <!-- Application Resources -->
    <j2se version="1.6+"
        href="http://java.sun.com/products/autodl/j2se"
            max-heap-size="128m" />
    <jar href="DynamicTreeDemo.jar"
        main="true" version="1.0"/>   
    <jar href="SomeOther.jar" version="2.0"/>
    <property name="jnlp.versionEnabled"
        value="true"/>
    <!-- ... -->
</resources>

    Cuando se habilita la propiedad jnlp.versionEnabled, el software de lanzamiento realiza sólo una comprobación de actualización para asegurarse de que el fichero JNLP está actualizado. El software compara los números de versión que se especifican en el fichero JNLP con las versiones correspondientes de los ficheros JAR (de acuerdo a la convención de nombrado mencionada en el paso 1) y actualiza sólo los ficheros JAR desactualizados. Este enfoque es eficiente porque solo la verificación de actualización para el archivo JNLP ocurre a través de la red. Todas las otras comprobaciones de versión ocurren localmente.

    Si un fichero con el número de versión correcto no se encuentra, el sofware de lanzamiento intenta cargar el fichero JAR por defecto (por ejemplo, DynamicTreeDemo.jar).

Realizar comprobaciones de actualización en segundo plano

Si no es crítico para el usuario ejecutar inmediatamente la última versión de su RIA, usted puede especificar que todas las comprobaciones de actualización ocurran en segundo plano. En este caso, el software de lanzamiento lanza la copia cacheada localmente para el uso inmediato y descarga una nueva versión de la ARI en segundo plano. La versión más nueva de la ARI será lanzada la próxima vez que el usuario intente usar su ARI. Para habilitar las comprobaciones de actualización en segundo plano, agregue la línea siguiente a su fichero JNLP: 

<update check='background'/>

El trozo de código siguiente muestra un fichero JNLP de ejemplo con la comprobación de actualizaciones en segundo plano habilitada: 

<?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="" href="">
    <information>
        <title>Applet Takes Params</title>
        <vendor>Dynamic Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.6+" href=
            "http://java.sun.com/products/autodl/j2se"/>
        <jar href="applet_AppletWithParameters.jar"
            main="true" />
    </resources>
    <applet-desc 
         name="Applet Takes Params"
         main-class="AppletTakesParams"
         width="800"
         height="50">
             <param name="paramStr" value="someString"/>
             <param name="paramInt" value="22"/>
     </applet-desc>
     <update check="background"/>
</jnlp>
