Las Mejoras Prácticas de Despliegue

Usted puede mejorar la experiencia de usuarios de su aplicación rica de Internet (ARI) usando las mejores prácticas descritas en este tema.

  • Firme la ARI usando un certificado de una autoridad de certificados reconocida. Asegúrese de que todos los artefactos estén firmados y que el certificado no ha expirado. Vea Firmando y Verificando los Ficheros JAR para obtener información sobre la firma.
  • Solicitar el nivel mínimo de permisos que se necesitan. Si la ARI no requiere acceso no restringido al sistema de usuario, especifique el nivel de permiso para que sea sandbox. Vea La Seguridad en Aplicaciones Ricas de Internet para más pautas de seguridad.
  • Optimize el tamaño de los ficheros JAR y recursos relacionados de forma que la ARI se pueda cargar rápidamente. Vea Reduciendo el Tiempo de Descarga para técnicas de optimización.
  • Habilite el protocolo de descarga de versión y use comprobaciones de actualización en segundo plano para permitir que la ARI se inicie rápidamente. Vea Evitando innecesarias comprobaciones de actualización para aprender más sobre el protocolo de descarga de versión y comprobaciones de actualización.
  • Asegúrese que el cliente tiene la versión requerida del software del Entorno de Ejecución de Java. Vea Asegurándose la Presencia del Software del EEJ para los detalles de cómo el script de Herramientas de Despliegue puede ser usado para este propósito.
  • Embeba los contenidos de su fichero JNLP del applet en la etiqueta <applet> para evitar cargar el fichero JNLP desde la red. Esta característica fue introducida en la versión 7 de Java. Vea Embebiendo un Fichero JNLP en una Etiqueta Applet para aprender como embeber los contenidos del fichero JNLP del applet en la página web.
  • Precarge su aplicación Java Web Start, si es posible. Si planea desplegar su ARI con una aplicación Java Web Start en una empresa donde usted tiene algún control administrativo, puede precargar su aplicación para varios clientes de forma que esta esté cacheada y lista para usar. Use el siguiente comando para precargar su aplicación Java Web Start: 
    javaws -import -silent <jnlp url>
