Reduciendo el Tiempo de Descarga

Las aplicaciones ricas de Internet (ARIs) son descargadas de un sitio web cuando el usuario intenta acceder a ellas. (Las ARIs pueden ser cacheadas después de la descarga inicial para mejorar el rendimiento). El tiempo tomado para descargar una ARI depende del tamaño del fichero JAR de la ARI. Los archivos JAR más grandes tardan más en descargarse.

Puede reducir el tiempo de descargar de su IPA aplicando las técnicas siguientes:

  • Comprima el fichero JAR de su ARI usando la herramienta pack200.
  • Elimine los espacios en blanco innecesarios del fichero de Protocolo de Lanzamiento de Java en Red (JNLP) y de los ficheros de JavaScript.
  • Optimize las imágenes y las animaciones.

Los pasos siguientes describen cómo crear y desplegar un fichero JAR comprimido para una ARI firmada.

  1. Normalize el fichero JAR usando la opción --repack.

    Este paso asegura que el certificado de seguridad y el fichero JAR pasarán las comprobaciones de verificación cuando se inicie la ARI. 

    pack200 --repack DynamicTreeDemo.jar
  2. Firme el fichero JAR normalizado. 
    jarsigner -keystore myKeyStore DynamicTreeDemo.jar me
    donde myKeyStore es el nombre del almacén de claves y me es el alias dentro del almacén de claves.
  3. Empaquete el fichero JAR firmado 
    pack200 DynamicTreeDemo.jar.pack.gz DynamicTreeDemo.jar
  4. Establezca la propiedad jnlp.packEnabled a true en el fichero JNLP de la ARI. 
    <resources>    
    <j2se version="1.6+"
        href="http://java.sun.com/products/autodl/j2se"
              max-heap-size="128m" />
    <jar href="DynamicTreeDemo.jar"
        main="true"/>
    <property name="jnlp.packEnabled"
        value="true"/>
    <!-- ... -->
</resources>

Cuando la propiedad jnlp.packEnabled es establecida en el fichero JNLP, el software del Plug-in de Java busca el fichero JAR comprimido con la extensión .pack.gz (por ejemplo, DynamicTreeDemo.jar.pack.gz). Si lo encuentra, el software del Plug-in de Java automáticamente desempaqueta y carga el fichero JAR. Si no se encuentra un fichero con la extensión .pack.gz, entonces el software del Plug-in de Java intenta cargar el fichero JAR normal (por ejemplo, DynamicTreeDemo.jar).

Nota: Necesita desplegar su ARI en un servidor web para comprobar la propiedad jnlp.packEnabled.
