Asegurándose la Presencia del Software del EEJ

La aplicaciones ricas de Internet (ARIs) necesitan usualmente una versión mínima del software del Entorno de Ejecución de Java (EEJ) presente en la máquina cliente. Cuando despliega una ARI, usted necesita asegurarse que las máquinas cliente tengan la versión requerida del software EEJ de forma que su ARI pueda funcionar bien. Con el script de Herramientas de Despliegue, usted tiene al menos dos formas de manejar este requisito.

  • Puede comprobar la versión del software EEJ del cliente tan pronto como los usuarios acceden a su sitio web e instalar la última versión si es necesario.
  • Usted puede permitir a los usuarios navegar el sitio web, y comprobar e instalar el último EEJ sólo cuando ellos intentan usar su ARI.

Comprobando e Instalando el Último Software del EEJ Cuando el Usuario Accede a su Web

El ejemplo siguiente comprueba si un usuario tiene al menos la versión 1.6.0_13 del software EEJ instalado. Si no, el código instala el último software EEJ. Vea los comentarios dentro del código. 

<script src="https://www.java.com/js/deployJava.js"></script>
<script>
    
    // check if current EEJ version is greater than 1.6.0 
    alert("versioncheck " + deployJava.versionCheck('1.6.0_10+'));
    if (deployJava.versionCheck('1.6.0_10+') == false) {                   
        userInput = confirm(
            "You need the latest Java(TM) Runtime Environment. " +
            "Would you like to update now?");        
        if (userInput == true) {  
    
            // Set deployJava.returnPage to make sure user comes back to 
            // your web site after installing the EEJ
            deployJava.returnPage = location.href;
            
            // Install latest EEJ or redirect user to another page to get EEJ
            deployJava.installLatestJRE(); 
        }
    }
</script>

Instalando el Software EEJ Correcto Sólo Cuando el Usuario Intenta Usar su ARI

Cuando usted especifica la versión mínima del software EEJ en la función runApplet o createWebStartLaunchButton, el script de Herramientas de Despliegue se asegura de que la versión requerida del software EEJ exista en el cliente antes de ejecutar su ARI.

Use la función runApplet para desplegar un applet, como se muestra en el ejemplo siguiente. El último parámetro de la funcion runApplet es la versión mínima que se requiere para ejecutar su applet (versión 1.6). 

<script src="https://www.java.com/js/deployJava.js"></script>
<script>
    var attributes = { code:'components.DynamicTreeApplet',
        width:300, height:300};
    var parameters = {jnlp_href: 'dynamictree_applet.jnlp'};
    deployJava.runApplet(attributes, parameters, '1.6');
</script>

Para desplegar una aplicación Java Web Start, use la función createWebStartLaunchButton con el parámetro de la versión mínima correcta (versión 1.6). 

<script src="https://www.java.com/js/deployJava.js"></script>
<script>
    var url = "dynamictree_applet.jnlp";
    deployJava.createWebStartLaunchButton(url, '1.6.0');
</script>

Las funciones runApplet y createWebStartLaunchButton comprueban la versión del software EEJ del cliente. Si la versión mínima no está instalada, las funciones instalan la última versión del software EEJ.
