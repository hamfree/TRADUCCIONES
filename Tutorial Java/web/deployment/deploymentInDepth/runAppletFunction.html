Desplegando un Applet

Puede desplegar applets usando la función runApplet del script de Herramientas de Despliegue. La función runApplet se asegura de que exista la versión mínima requerida del Entorno de Ejecución Java (EEJ) en el cliente y entonces ejecuta el applet. La función runApplet genera una etiqueta HTML <applet> con la información proporcionada.

Nota: Dependiendo dle tipo de navegador, podría ser incapaz de ver el HTML generado por el script de herramientas de despliegue cuando intente ver el código fuente de la página web. Para ver el HTML generardo, intente salver la página HTML después de que haya sido cargada, o use una herramienta como Firebug (una extensión de Mozilla Firefox).

Puede desplegar applets especificando las opciones de desplieuge como atributos y parámetros de la etiqueta <applet>. También puede especificar las opciones de despliegue en un fichero del Protocolo de Lanzamiento de Java en Red (JNLP) para aprovechar las funciones avanzadas. Vea el tema Protocolo de Lanzamiento en Red de Java (JNLP) para más información sobre este protocolo.

Nota: 

Si el cliente no tiene la versión mínima requerida del software del EEJ, el script de Herramientas de Despliegue redirige el navegador a http://www.java.com para permitir a los usuarios descargar el último software. Los parámetros para la función runApplet varían dependiendo de si está usando JNLP. Los applets desplegados usando JNLP pueden soólo ejecutarse si existe el software Java Plug-in de nueva generación en la máquina cliente (el software de próxima generación de Java Plug-in fue presentado en la versión 6 actualización 10 de la Edición Estándar de la Plataforma Java).

La próxima sección muestra cómo usar la función runApplet en la página HTML que visualizará el applet. Se describen los siguientes escenarios de usuario:

  • Especificación de opciones de implementación como pares de nombre y valor de atributo y parámetro
  • Usar el parámetro jnlp_href para especificar opciones de despliegue en un fichero JNLP
  • Especifiar atributos y parámetros como pares nombre-valor tanto como un fichero JNLP (permite que el applet se ejecute en el software Java Plug-in antiguo y de próxima generación).

Firma de la Función: runApplet: function(attributes, parameters, minimumVersion)

Parámetros:

  • attributes – Los nombres y valores de los atributos de la etiqueta generada <applet>
  • parameters – Los nombres y valores de las etiquetas <param> en la etiqueta generada <applet>
  • minimumVersion – La versión mínima del software del EEJ que se requiere para ejecutar este applet

Uso:

  • Especificación de opciones de implementación como pares de nombre y valor de atributo y parámetro

    Los atributos y parámetros pasados como pares nombre-valor están escritos como atributos anidados en las etiquetas <param> en la etiqueta generada <applet>. Los applets desplegados de esta manera pueden ser ejecutados por el software viejo de Java Plug-in. 

    // lanza el applet Java 2D en la versión 1.6.0 del EEJ
// o mayor con un parámetro (fontSize)
<script src=
    "https://www.java.com/js/deployJava.js"></script>
<script>
    var attributes = {code:'java2d.Java2DemoApplet.class',
        archive:'Java2Demo.jar', width:710, height:540};
    var parameters = { fontSize:16, permissions:'sandbox' };
    var version = '1.6';
    deployJava.runApplet(attributes, parameters, version);
</script>

    Abra DeployUsingNameValuePairs.html en un navegador para ver el applet Java2D.

    Nota:  Si no ve el applet ejecutándose, necesita instalar al menos la versión 7 del Kit de Desarrollo de Java SE (KDJ).
  • Usar el parámetro jnlp_href para especificar opciones de despliegue en un fichero JNLP

    Los atributos y parámetros (jnlp_href en este caso) pasados como pares nombre-valor son escritos como atributos y anidadas en las etiquetas <param> en la etiqueta <applet>. Los applets desplegados de esta manera pueden ser sólo ejecutados por el software de próxima generación de Java Plug-in. Es mejor especificar el ancho y alto del applet como atributos como siguen: 

    <script src="https://www.java.com/js/deployJava.js"></script>
<script> 
    var attributes = { code:'java2d.Java2DemoApplet', width:710, height:540 }; 
    var parameters = { jnlp_href: 'java2d.jnlp' }; 
    deployJava.runApplet(attributes, parameters, '1.6'); 
</script>

    Abra DeployUsingJNLP.html en un navegador para ver el applet Java2D.

    Nota:  Si no el applet ejecutarse, necesita instalar al menos la versión 6 actualizacion 10 del Kit de Desarrollo de Java SE (KDJ).
  • Especificar pares nombre-valor de atributo y parámetros como también un fichero JNLP

    Los applets desplegados usando JNLP se ejecutarán sólo si los usuarios finales tienen el sofware Java Plug-in de próxima generación ejecutándose en sus navegadores. Si desea que su applet también se ejecute en el antiguo software Java Plug-in, especifique las opciones de implementación utilizando pares de nombre y valor de atributo y parámetro así como un archivo JNLP. 

    <script src="https://www.java.com/js/deployJava.js"></script>
<script>  
    var attributes = {code:'java2d.Java2DemoApplet.class', 
            archive:'Java2Demo.jar', width:710, height:540}; 
    var parameters = { fontSize:16, jnlp_href:'java2d.jnlp' }; 
    var version = '1.6' ; 
    deployJava.runApplet(attributes, parameters, version);      
</script>

Las siguientes pautas son útiles si algunas opciones de implementación tienen valores diferentes en los pares nombre-valor del atributo y en el archivo JNLP:

  • Especificar width y height como atributos en pares nombre-valor (no en el fichero JNLP).
  • Especificar parámetros tales como image y boxbgcolor como parámetros en pares nombre-valor (no en el fichero JNLP). Estos parámetros son necesarios al principio del proceso de inicio del applet.
  • En el fichero JNLP, dejar el atributo codebase vacío o especificar una URL absoluta. Cuando el atributo codebase se deja vacío, el valor predeterminado es el directorio que contiene el archivo JNLP.
  • Si el applet es lanzado usando un fichero JNLP, los valores para los atributos code, codebase, y archive están tomados desde el fichero JNLP. Si estos atributos son también especificados separadamente como pares nombre-valor de atributo, los pares nombre-valor de atributo son ignorados.

Abra DeployUsingNameValuePairsAndJNLP.html en un navegador para ver el applet Java2D.

Nota:  Si no ve el applet ejecutarse, necesita instalar al menos la versión 7 del Kit de Desarrollo de Java SE (KDJ).
