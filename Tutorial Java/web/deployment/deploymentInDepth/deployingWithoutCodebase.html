Desplegando Sin Codebase

A partir de la versión 7 de Java SE, no tiene que especificar una ruta absoluta para el atributo codebase en el fichero de Protocolo de Lanzamiento de Red de Java (JNLP) de una aplicación Java Web Start. Puede desarrollar y comprobar sus aplicaciones en diferentes entornos sin tener que modificar la ruta en el atributo codebase. Si no se especifica codebase, el software de Java Web Start asume que codebase es relativo a la página web desde la cual la aplicación Java Web Start es lanzada.

Las siguientes funciones en el script de Herramientas de Despliegue pueden ser utilizadas para desplegar aplicaciones Java Web Start en una página web cuando el fichero JNLP no contiene el atributo codebase:

  • launchWebStartApplication – Use esta función en un enlace HTML para desplegar su aplicación Java Web Start.
  • createWebStartLaunchButtonEx – Use esta función para crear un botón 'Launc' para su aplicación Java Web Start..
Nota: Para ejecutarse, las aplicaciones Java Web Start desplegadas usando las funciones especificadas previamente requieren al menos la versión 7 de Java SE. Si el cliente no tiene al menos la versión 7 de Java SE, las funciones dirigirán al usuario a instalar el software del Entorno de Ejecución de Java (EEJ) requerido antes de lanzar la aplicación Java Web Start.

Firma de la función: launchWebStartApplication: function(jnlp)

Parámetro:

jnlp – La ruta al fichero JNLP que contiene la información de desplieuge para la aplicación Java Web Start. Esta ruta puede ser relativa a la página web en la cual la aplicación Java Web Start es desplegada.

Uso:

En el ejemplo siguiente, la función launchWebStartApplication es invocada en el atributo href de una etiqueta ancla (a).

El fichero JNLP dynamictree_webstart_no_codebase.jnlp es usado para desplegar la aplicación Demo de Árbol Dinámico. 

<script src="https://www.java.com/js/deployJava.js"></script>
<a href="javascript:deployJava.launchWebStartApplication('dynamictree_webstart_no_codebase.jnlp');">Launch</a>

La aplicación Java Web Start es lanzada cuando el usuario pulsa el enlace HTML resultante.


Firma de la Función: createWebStartLaunchButtonEx: function(jnlp)

Parámetro:

jnlp – La ruta al fichero JNLP que contiene la información de despliegue para la aplicación Java Web Start. Esta ruta puede ser relativa a la página web en la cual la aplicación Java Web Start es desplegada.

Uso:

El ejemplo siguiente muestra el uso de la función createWebStartLaunchButtonEx.

El fichero JNLP dynamictree_webstart_no_codebase.jnlp es usado para desplegar la aplicación Demo de Árbol Dinámico. 

<script src="https://www.java.com/js/deployJava.js"></script>
<script>        
    var jnlpFile = "dynamictree_webstart_no_codebase.jnlp";
    deployJava.createWebStartLaunchButtonEx(jnlpFile);
</script>

La aplicación Java Web Start es lanzada cuando el usuario pulsa el botón 'Launch' resultante.

Abra JavaWebStartAppPage_No_Codebase.html en un navegador para ver la aplicación Demo de Árbol Dinámico que está desplegada usando las funciones descritas en este tema.

Nota: 

Usted puede también lanzar la aplicación Java web Start en la línea de comandos del sistema invocando el comando javaws con la url completa del fichero JNLP como se muestra en el trozo de código siguiente. 

javaws http://example.com/dynamictree_webstart_no_codebase.jnlp

Descargue el código fuente del ejemplo Demo del Árbol Dinámico para experimentar más.
