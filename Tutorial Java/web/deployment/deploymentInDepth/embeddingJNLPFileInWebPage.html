Embebiendo un Fichero JNLP en una Etiqueta Applet

Cuando los applets son descargados usando el Protocolo de Lanzamiento en Red de Java (JNLP), el software del Complemento de Java lanza el applet después de descargar el fichero JNLP desde la red. Empezando con la versión 7 de Java SE, puede reducir el tiempo que se toman los applets en lanzarse, embebiendo el fichero JNLP en la misma página web de forma que una petición adicional a la red puede ser evitada la primera vez que se carga el applet. Esto dará como resultado que los applets se inicien rápidamente en el navegador web.

Un fichero JNLP codificado en Base64 puede ser embebido en el parámetro jnlp_embedded cuando se despliega un applet en una página web. Los atributos del elemento <jnlp> deben cumplir con las siguientes restricciones:

  • El atributo href debería contener una ruta relativa.
  • El atributo codebase no debe especificarse. Esto implica que la base de código se derivará de la URL de la página web en la que se carga el applet.

Los pasos siguientes describen como embeber un fichero JNLP en una página web para desplegar un applet.

  1. Cree un fichero JNLP para su applet. Un fichero de ejemplo se muestra a continuación. 
    <?xml version="1.0" encoding="UTF-8"?>
<!-- href attribute contains relative path;
     codebase attribute not specified -->
<jnlp href="dynamictree_applet.jnlp">
    <information>
        <title>Dynamic Tree Demo</title>
        <vendor>Dynamic Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.7+" />
        <jar href=
            "dist/applet_ComponentArch_DynamicTreeDemo/DynamicTreeDemo.jar" 
             main="true" />
    </resources>
    <applet-desc 
         name="Dynamic Tree Demo Applet"
         main-class="appletComponentArch.DynamicTreeApplet"
         width="300"
         height="300">
     </applet-desc>
     <update check="background"/>
</jnlp>
  2. Codifique el contenido del archivo JNLP utilizando el esquema Base64. Puede usar cualquier herramienta de codificación Base64 para codificar el fichero JNLP. Compruebe el uso de la herramienta para crear una cadena con codificación Base64. Algunos ejemplos de herramientas y sitios web que pueden usarse son los siguientes:
    • Comandos de UNIX – base64, uuencode
    • Sitios web – Codificación y Decodificación Base64, Codificador Base64
  3. Cuando se despliega el applet en una página web, especifique el parámetro jnlp_embedded con su valor establecido en una cadena JNLP codificada en Base64. Asegúrese de incluir solo los bytes Base64 reales sin ningún encabezado o pie de página específico de la herramienta de codificación. 
    <script src="https://www.java.com/js/deployJava.js"></script>
<script>
    var attributes = {} ;
    <!-- Base64 encoded string truncated below for readability -->
    var parameters = {jnlp_href: 'dynamictree_applet.jnlp',
        jnlp_embedded: 'PCEtLSANCi8qDQogKiBDb ... bmxwPg=='
    } ;
    deployJava.runApplet(attributes, parameters, '1.6');
</script>

    Algunas herramientas de codificación pueden envolver la cadena codificada en varias líneas de 76 columnas. Para usar este valor multilínea como atributo en código JavaScript, especifique el valor del atributo como un conjunto de cadenas concatenadas. Puede incluir el valor multilínea del atributo como si el apllet es desplegado directamente con la etiqueta HTML <applet>.

Abra AppletPage.html en un navegador para ver el applet Demo Árbol Dinámico que es lanzado usando el fichero JNLP embebido en la página web.

Nota:  Si no ve el applet ejecutarse, necesita instalar al menos la versión 7 del Kit de Desarrollo de Java SE (KDJ).
Nota:  Si no ve el ejemplo ejecutarse, podría necesitar habilitar el intérprete de JavaScript en su navegador de forma que el script de la herramienta de despliegue pueda funcionar adecuadamente.

Descargue el código fuente para el ejemplo JNLP Embebido para experimentar más.
