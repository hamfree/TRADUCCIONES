Comprobar la Versión del Software del EEJ del Cliente

Hay muchas razones para comprobar si una versión particular del software del Entorno de Ejecución de Java (EEJ) está disponible en la máquina de un cliente. Por ejemplo, es posible que desee lanzar una versión diferente de su aplicación Rica de Internet (ARI) o redirigir al usuario a una página diferente dependiendo de la versión del software del EEJ del cliente.

Use la función del script de las Herramientas de Despliegue versionCheck para comprobar si una versión particular o un rango de versiones de EEJ está instalado en el cliente.

Firma de la Función: versionCheck: function(versionPattern)

Parámetros:

  • versionPattern – Cadena para especificar la versión o rango de versiones a comprobar, tales como "1.4", "1.5.0*" (familia 1.5.x), y "1.6.0_02+" (cualquier versión mayor o igual a 1.6.0_02).

Uso: Crear una experiencia de usuario diferente dependiendo de la versión del software del EEJ del cliente

En este ejemplo, se crea un botón 'Launch' para la aplicación Notepad sólo si la versión del software del EEJ en el cliente es mayor o igual a 1.6. Si no, el navegador es redirigido a oracle.com. 

<script src="https://www.java.com/js/deployJava.js"></script>
<script>
    if (deployJava.versionCheck('1.6+')) {            
        var url = "https://docs.oracle.com/javase/tutorialJWS/deployment/webstart/examples/Notepad.jnlp";
        
        <!-- tambien puede invocar deployJava.runApplet aqui -->
        deployJava.createWebStartLaunchButton(url, '1.6.0'); 
    } else {
        document.location.href="http://oracle.com";
    }
</script>
Nota: Dependiendo del sistema operativo del cliente y la versión de la plataforma Java, es posible que que pueda verificar la información de la versión del software del EEJ en el nivel de la versión mayor (por ejemplo, 1.6) o en un nivel de actualización más fino (por ejemplo, 1.6.0_10).
