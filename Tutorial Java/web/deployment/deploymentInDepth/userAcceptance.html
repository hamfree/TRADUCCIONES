Aceptación del Usuario de las RIAs

Por seguridad, se solicita a los usuarios permiso para ejecutar una ARI antes de iniciarla por primera vez, incluso si la aplicación está firmada o no requiere acceso fuera del entorno limitado de seguridad. La solicitud incluye la siguiente información, según la ARI que se esté ejecutando:

  • Nombre de la ARI, o notificación de que la aplicación no está firmada.
  • Información sobre el editor, si la aplicación está firmada con un certificado de una autoridad de confianza. Si el certificado ha expirado, se incluirá una advertencia. Si la aplicación está auto-firmada, el editor se muestra como DESCONOCIDO.
  • Advertencias si el certificado ha caducado, revocado o el estado de revocación no se puede verificar.
  • Ubicación desde la que se accede a la aplicación.
  • Nivel de acceso requerido por la aplicación. El acceso limitado restringe la aplicación al entorno limitado de seguridad, el acceso sin restricciones proporciona a la aplicación acceso a los recursos del sistema del usuario.
  • Advertencias sobre la falta de atributos de manifiesto en el fichero JAR si los atributos recomendados no están presentes.
  • Para las aplicaciones autofirmadas o sin firmar, una casilla de verificación que el usuario debe seleccionar para aceptar la aplicación.
  • En algunos casos, la opción de no volver a mostrar el mensaje.

Para una descripción de las solicitudes, vea ¿Qué debo hacer cuando veo un mensaje de seguridad de Java?

Los usuarios también son avisados si están ejecutando un EEJ desactualizado y se les da la oportunidad de actualizar a la última versión antes de ejecutar la aplicación. Los usuarios pueden también elegir ejecutarla con el EEJ de su sistema, o impedir a la aplicación ejecutarse.

La configuración del nivel de seguridad en el Panel de Control de Java determina si los usuarios tienen la oportunidad de ejecutar ARIs. La configuración predeterminada Alta solicita a los usuarios permiso para ejecutar aplicaciones firmadas con un certificado válido e incluye el atributo Permisos en el manifiesto del archivo JAR principal. Si no se puede verificar el estado de revocación de una aplicación, la aplicación también puede ejecutarse con la autorización del usuario.

Firmar su ARI proporciona al usuario un nivel de confianza. Tenga en cuenta lo siguiente al preparar su aplicación para la implementación:

  • La mejor experiencia de usuario la proporciona una aplicación firmada con un certificado emitido por una autoridad de certificación reconocida.

  • Las aplicaciones autofirmadas y sin firmar no se pueden ejecutar a menos que se haya creado una lista de sitios de excepción o un conjunto de reglas de implementación que permitan explícitamente que la aplicación se ejecute.

  • Las aplicaciones firmadas pueden ser o aplicaciones privilegiadas o aplicaciones de entorno limitado. Las aplicaciones privilegiadas reciben acceso sin restricciones a los recursos del sistema del usuario. Las aplicaciones de entorno limitado están restringidas al entorno limitado de seguridad de Java. Las aplicaciones sin firmar están restringidas al entorno limitado.
