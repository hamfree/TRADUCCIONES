Protocolo de Lanzamiento en Red de Java (JNLP)

El Protocolo de Lanzamiento en Red de Java (JNLP) permite a una aplicación a ser lanzada en el escritorio de un cliente usando recursos que están almacenados en un servidor web remoto. El software del Plug-in de Java y el software de Java Web Start son considerados clientes JNLP porque pueden lanzar applets y aplicaciones almacenados remotamente en un escritorio cliente. Vea Java Network Launching Protocol and IPA Specification Change Log para más detalles.

Las mejoras recientes en las tecnologías de despliegue nos permiten lanzar aplicaciones ricas de Internet (ARIs) usando JNLP. Tanto los applets como las aplicaciones Java Web Start pueden iniciarse usando este protocolo. Las ARIs que son lanzadas usando JNLP también tienen acceso a las IPAs de JNLP. Estas IPAs de JNLP permiten a las ARIs a acceder al escritorio del cliente con el permiso del usuario.

El JNLP es habilitado por un fichero JNLP de la ARI. El fichero JNLP describe la ARI. El fichero JNLP especifica el nombre del fichero JAR principal, la versión del software del Entorno de Ejecución de Java que se requiere para ejecutar la ARI, el nombre y la información visual, los paquetes opcionales, los parámetros de ejecución, las propiedades del sistema, etc.

Puede encontrar más información sobre el despliegue de ARIs usando JNLP en los siguientes temas:

  • Implantando un Applet
  • Desplegando una Aplicación Java Web Start
  • IPA de JNLP
  • Estructura del Fichero JNLP
