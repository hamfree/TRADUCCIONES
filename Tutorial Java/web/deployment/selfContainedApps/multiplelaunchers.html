Usando Múltiples Puntos de Entrada

Las aplicacoines autocontenidas son útiles cuando tienes un conjunto de aplicaciones relacionadas que quieren los usuarios implantar. Una aplicación autocontenida ofrece un paquete único, instalable que instala todas las aplicaciones y el EEJ necesitado para ejecutarlas.

La Demostración de Múltiples Lanzadores incluye la Demostración de Arbol Dinámico descrita en Convirtiendo una Aplicación Existente y la Demostración de Asociación de Fichero descrita en  Usando Asociaciones de Fichero. El directorio /src odel proyecto contiene el fichero fuente para ambas aplicaciones.

Usted puede descargar los ficherso fuentes para la Demostración de Lanzadores Múltiples desde Ejemplos de Aplicaciones Autocontenidas.

El punto de entrada primaria para una aplicación autocontenida es identificado por el atributo mainClass del elemento <fx:application>. En la Demostración Lanzadores Múltiples, el punto de entrada primario es la Demostración de Asociación de Fichero. La clase main es sample.fa.ScriptRunnerAppliation para Linux y Windows, o sample.fa.ScriptRunnerApplicationMac para OS X. Vea Usando un Fichero de Construcción Común para Todas las Plataformas para información de cómo la clase a usar es determinada cuando un fichero único de construcción es utilizada entre plataformas.

Cada punto de entrada secundaria es identificado por una instancia del elemento <fx:secondaryLauncher>. Vea <fx:secondaryLauncher> para más información sobre este elemento.

En la Demostración de Múltiples Lanzadores, el punto de entrada secundario es la Demostración de Arbol Dinámico. El código siguietne en el fihcero build.xml muestra como es definido el segundo punto de entrada: 

<fx:secondaryLauncher name="Dynamic Tree Demo"
    mainClass="webstartComponentArch.DynamicTreeApplication"
    version="1.0"
    title="Dynamic Tree Demo"
    vendor="My Company"
    description="A Demo of Multiple Launchers for JavaPackager"
    copyright="(c) 2014 My Company"
     menu="true"
     shortcut="false"
     >
</fx:secondaryLauncher>

Vea build.xml para el código de construcción completo.

Para generar los paquetes instalables para la Demostración de Lanzadores Múltiples, vea la sección "Generando los Paquetes" en Convirtiendo una Aplicación Existente.

Cuando una aplicación autocontenida es instalada, la Demostración de Asociación de Fichero es instalada con el punto de entrada de Lanzadores Múltiples y la Demostración de Arbol Dinámico es instalada con su propio punto de entrada. Por ejemplo, en Windows, la carpeta Java Demos en el menu Inicio contiene dos entradas: Demostración de Arbol Dinámico y Demostración de Lanzadores Múltiples. Note que las asociaciones de fichero están configuradas para el punto de entrada de Lanzadores Múltiples, de forma que abrir un fichero de JavaScript o Groovy inicia Lanzadores Múltiples.
