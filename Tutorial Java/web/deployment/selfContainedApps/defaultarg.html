Proporcionando un Argumento por Defecto

Los argumentos son pasados a las aplicaciones Java cuando una aplicación es iniciada. Las aplicaciones autocontenidas pueden ser configuradas con un argumento por defecto que es utilizado cuando no se han especificado ningún argumento. El elemento <fx:argument> es usado para definir un argumento. Más de un argumento puede ser pasado añadiendo un elemento <fx:argument> para cada argumento. Vea <fx:argument> para tener más información sobre este elemento.

La Demostración de Asociación de Fichero descrita en Usando Asociaciones de Fichero está configurada usando el nombre de uno de los ficheros de ejemplo que es empaquetado con la aplicación como el argumento por defecto.

El siguiente código en la tarea <fx:deploy> en el fichero build.xml mostrado como argumento por defecto es definido: 

<fx:application id="fileassociationdemo"
                name="Demostración de Asociación de Fichero"
                mainClass="${main.class}"
                version="1.0">
    <fx:argument>sample.js</fx:argument>
</fx:application>

Vea build.xml para el código de construccción completo.

Usted puede descargar los ficheros fuente de la Demostración Asociación de Fichero desde Ejemplos de Aplicaciones Autocontenidas.

Recursos Adicionales

Para más información sobre argumentos por defecto, ver Pasando Argumentos a Aplicaciones Autocontenidas.

Para más información sobre los argumentos de Ant JavaFX, vea Referencia de Tareas de Ant para JavaFX.
