Usando Asociaciones de Fichero

Una de las ventajas de ofrecer una aplicación autocontenida a sus usuarios es la habilidad para configurar asociaciones de fichero. Tipos de ficheros específicos pueden ser asociados con su aplicación basados en el tipo MIME de la extensión del fichero de forma que su aplicación sea utilizada para abrir un fichero asociado. Por ejemplo, si su aplicación edita ficheros de texto, usted puede configurar una asociación de fichero que ejecuta su aplicación cuando un usuario hace doble click sobre un fichero con la extensión .txt.

La Demostración de Asociación de Fichero lee código JavaScript y Groovy. Usando ambos tipos MIME y extensiones deficheros, la aplicación es asociada con los ficheros JavaScript y Groovy.

Usted puede descargar los ficheros fuente para la Demostración Asociación de Fichero desde Ejemplos de Aplicación Autocontenida.

Configurando las Asociaciones de Fichero

Las tareas de Ant para generar los paquetes de la aplicación autocntenida están en el fichero build.xml para la Demostración de Asociación de Fichero. El elemento de Ant <fx:association> es utilizado par asociar las extensiones de fichero o los tipos MIME con su aplicación. Los empaquetadores Linux requieren el tipo MIME, los empaquetadores de Windows requieren la extensión del fichero, y los empaquetadores de OS x requieren al menos una de las propiedades. Es una buena práctica usar ambas propiedades con un mapeo uno-a-uno entre el tipo MIME y la extensión del fichero, lo cual le posibilita el uso del mismo fichero de construcción para múltiples plataformas. Vea <fx:association> para más información sobre este elemento.

El código siguiente muestra lo que necesita ser incluido en el elemento fx:deploy para asociar la aplicación con las extensiones .js y .groovy y los tipos MIME text/javascript y text/x-groovy. 

<fx:info title="Demostración de Asociación de Fichero"
         vendor="MySamples"
         description="Una Demostración de Asociación de Fichero para el Empaquetador de Java"
         category="Demostraciones"
         license="3 Clause BSD">
    <fx:association extension="js" mime description="Fuente JavaScript"/>
    <fx:association extension="groovy" mimetype="text/x-groovy" description="Fuente Groovy"/>
</fx:info>

Si un empaquetador no soporta las asociaciones de ficero, las asociaciones son ignoradas. A partir de la versión 8u40 del KDJ, los empaquetadores de Windows EXE y MSI, empaquetadores de Linux DEB y RPM, y el empaquetador de Mac .app soportan asociaciones de fichero. Los empaquetadores de OS X PKG y DMG soportan asociaciones de fichero a través del uso del empaquetador de Mac .app.

Vea build.xml para el código de construcción completo.

Para generar los paquetes instalables para la Demostración de Asociación de Fichero, vea la sección "Generando los Paquetes" en Convirtiendo una Aplicación Existente.

Lanzando desde Ficheros Asociados

Las asociaciones de fichero son configuradas por el instalador cuando el paquete de la aplicación autocontenida es instalada en el sistema del usuario. Después de que la aplicación está instalada, abrir un fichero que está asociado con tu aplicación causa a su aplicación el que se inicie. Las acciones tomadas para lanzar su aplicación dependen de la plataforma sobre la cual se ejecuta.

Lanzando en Linux y Windows

En Linux y Windows, cuando una aplicación es lanzada basados en un asociación de ficheros, los ficheros que están siendo abiertos son pasadas como argumentos a una clase principal, la cual sobreescribe el argumento por defecto para la clase. Para la Demostración de Asociaciones de Fichero, los argumentos son pasados al método loadscript después de que una instancia de la aplicación está iniciada. Una instancia diferente de la aplicación es iniciad para cada fichero abierto.

Vea ScriptRunnerApplication.java para la versión del código para Linux y Windows.

Lanzando en OS X

En OS X, sólo una única instancia de una aplicación se ejecuta. Cuando un fichero asociado es abierto, un evento es enviado a la aplicación. La aplicación debe tener un escuchador del evento registrado para manejar el evento.

La Demostración de Asociación de Fichero para OS X tiene una subclase con un método main diferente que la versión para Linux y Windows. Este método main maneja el argumento por defecto en la misma forma que el método main para la versión de Linux y Windows, y entonces registra un escuchador con OS X para FileOpenHandler. El método del evento para este escuchador sea llamado cuando un fichero asociado sea abierto, y el nombre del fichero es extraído desde el método getFiles del objeto OpenFilesEvent.

Vea ScriptRunnerApplicationMac.java para la versión OS X del código.

Construir la versión OS X de la Demostración de Asociación de Fichero requiere el acceso a clases específicas de OS Xque vienen con el KDJ de Oracle. La mayoría de las clases com.apple.eawt no son incluidas en el fichero de símbolos que el compilador javac usa. Para decirle al compilador que ignore el fichero de símbolos, el argumento -XDignore.symbol.file=true es pasado al compilador javac en la tarea de Ant -pre-init en el fichero de construcción. Vea build.xml.

Más Sobre la Demostración de Asociación de Fichero

El proyecto para la Demostración de Asociación de Fichero contiene los ficheros fuentes de Java para la aplicación en el directorio /src/sample/fa. Los iconos personalizados son ofrecidos en el directorio /src/package/platform. Los ficheros de ejemplo a ser empaquetados con la aplicación están en el directorio /src.

Para manejar el código Groovy, la Demostración de Asociación de Fichero requiere la librería Groovy. El proceso de construcción descarga la libreria Groovy al directorio /lib. Vea Añadiendo una Librería Externa para más información.

Después de que el fichero JAR es generado, el proceso de construcción copia los directorios /src y /lib al directorio /dist. El directorio /dist entonces contiene todos los ficheros para su aplicación.

La Demostración de Asociaciones de Fichero toma un nombre de fichero como argumento. Si la aplicación es iniciada abriendo un fichero asociado, el nombre del fichero asociado es introducido. Si la aplicación es ejecutada directamente, el fichero de ejemplo sample.js, el cual es empaquetado con la aplicación, es instroducido. Vea Proporcionando un Argumento Predeterminado para más información.

Los privilegios de Administrador son requeridos para configurar asociaciones de fichero. Por defecto el instalador de EXE's par Windows no requiere privilegios administrativos. Para forzar una petición de privilegios de administración para la Demostración de Asociación de Fichero, el argumento del paquete win.exe.systemWide está fijando en true. Esta configuración indica que se ejecute una instalación para todo el sistema, lo cual requiere privilegios administrativos.

La Demostración de Asociaciones de Fichero se ejecuta en Linux, OS X y Windows. La demostración está configurada para usar un fihcero único de construcción que contiene la información para todas las plataformas. Vea Usando un Fichero Común de Construcción para Todas las Plataformas para más información.
