Lección : Implantando Aplicaciones Autocontenidas

Una aplicación autocontenida consiste de un único, empaquetado instalable que contiene tu aplicación y una copia del EEJ necesario para ejecutar la aplicación. Cuando la aplicación está instalada, se comporta de la misma forma que una aplicación nativa. Proporcionando a los usuarios una aplicación autocontenida evita los problemas de seguridad relacionado con ejecutar una aplicación en el navegador.

Usted puede personalizar una aplicación autocontenida proporcionando sus propios iconos e indicador de progreso. Las asociaciones de fichero pueden ser configuradas cuando un usuario abre un fichero que tu aplicación puede manejar, tu aplicación será iniciada automáticamente. Se soportan múltiples puntos de entrada de fora que usted pueda ofrecer un paquete de aplicaciones en una única agrupación de aplicación autocontenida.

Las aplicaciones autocontenidas pueden ser empaquetadas usando las herramientas de Empaquetado de Java. El comando javavpackager crea el empaquetado para aplicaciones autocontenidas desde la línea de comandos. Netbeans puede también ser utilizado para crear empaquetados de aplicaciones autocontenidas. Esta lección describe como usar tareas de Ant para crear los empaquetados.
