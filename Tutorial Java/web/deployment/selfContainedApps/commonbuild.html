Usando un Fichero Común de Construcción para Todas las Plataformas

Para generar un empaquetado de aplicación autocontenida para cada plataforma en la cual tu aplicación se ejecuta, debe ejecutar la herramienta de empaquetado en cada platforma. Tiene la opción de usar ficheros de construcción específicos de la plataforma o configurar un fichero de construcción que pueda ejecutarse en todas las plataformas. Los ficheros específicos de la plataforma pueden ser más simpels de configurar, pero entonces debe mantener múltiples ficheros.

La Demostración Asociación de Ficheros descrita en Usando Asociaciones de Fichero utiliza un único fichero de construcción que funciona en todas las plataformas.

Los siguientes elementos del fichero de construcción soportan su uso para todas las plataformas:

  • La clase principal para la aplicación es ScriptRunnerApplication.java para Linux y Windows y ScriptRunnerApplicationMac.java para OS X. El siguiente códigoe en la tarea -pre-init es utilizada para determinar qué clase usar: 

    <condition property="main.class"
           value="sample.fa.ScriptRunnerApplication"
           else="sample.fa.ScriptRunnerApplicationMac">
    <not><os family="mac"/></not>
</condition>

  • El siguiente código en la tara -pre-init es usado para prevenir que la clase main para OS X sea compilada cuando se ejecuta en Linux o Windows: 

    <condition property="excludes" value="**/*Mac.java">
    <not><os family="mac"/></not>
</condition>

  • Los elementos <fx:bundleArgument> son usados para pasar argumentos a los diferentes empaquetadores disponibles. Los argumentos que no son utiliadsos por el empaquetador son ignorados, de forma que el fichero de construcción pueda contener los argumentos necesitados por todas las plataformas. El siguiente código define argumentos para Linux, OS X y Windows: 

    <fx:bundleArgument arg="classpath" value="FileAssociationsDemo.jar lib/groovy-all-2.3.8.jar"/>

<fx:bundleArgument arg="win.exe.systemWide" value="true"/>

<fx:bundleArgument arg="linux.bundleName" value="file-association-demo"/>
<fx:bundleArgument arg="email" value="maintainer@example.com"/>
<fx:bundleArgument arg="mac.CFBundleName" value="File Assoc Demo"/>
<fx:bundleArgument arg="win.menuGroup" value="Java Demos"/>

Vea build.xml para el código completo de construcción.

Puede descargar los ficheros fuentes para el Demo de Asociación de Fichero desde los Ejemplos de Aplicación Autocontenida.
