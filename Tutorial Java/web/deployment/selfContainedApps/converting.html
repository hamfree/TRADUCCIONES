Convirtiendo una Aplicación Existente

Cualquier aplicación Java independiente o aplicación Java Web Start puede ser empaquetada como una aplicación autocontenida. Si usted tien un applet Java, vea Reescribiendo un Applet de Java como una Aplicación Java Web Start para información sobre convertir el applet a una aplicación Java Web Start, la cual puede ser empaquetada como una aplicación autocontenida.

Antes de convertir una aplicación, asegúrese de que tiene los prerequisitos requeridos instalados para su plataforma. Vea Prerequisitos para Empaquetar Aplicaciones Autocontenidas para más información.

Esta sección convierte la Demostración Árbol Dinámico desde Desplegando una Aplicación Java Web Start a una aplicación autocontenida. Usted puede descargar los ficheros fuente de esta demostración desde Ejemplos de Aplicación Autocontenida.

Configurando los Directorios

Identifique y organize los ficheros que son necesitados por su aplicación. Una simple aplicación podría requerir solo un fichero JAR. Una aplicación más compleja podría requerir librerías adicionales o recursos. Los recursos personalizados tales como los iconos o ficheros de configuración pueden ser usados por las aplicaciones autocontenidas.

La Demostración Arbol Dinámico requieres solo el fichero DynamicTreeDemo.jar, el cual está en el directorio /dist del proyecto. Los ficheros HTML y JNLP que fueron necesitados para la versión Java Web Start de la aplicación no son necesitados y son ignorados por los empaquetadores para las aplicaciones autocontenidas.

Para proporcionar un icono personalizado para la Demostración Árbol Dinámico, el cual representa la aplicación cuando es instalada en el escritorio de un usuario, un icono es ofrecido para cada plataforma soportada. Estos iconos son colocados en los directorios /src/package/platform. El icono es ofrecido en un formato diferente para cada plataforma soportada: formato .ico para Windows, formato .png para Linux, y formato .icns para OS X.

El siguiente ejemplo muestra la estructura del directorio para el proyecto de Demostración Árbol Dinámico antes de que los paquetes autocontenidos son creados: 

/packager_DynamicTreeDemo     <--- Proyecto de la aplicación
   /dist
      DynamicTreeDemo.jar
      ...
   /src
      /package                <--- Recursos personalizados
         /linux
         /macosx
         /windows
      /webstartComponentArch  <--- Ficheros fuente de la aplicación
      ...

Configurando el Fichero de Construcción

Configure las tareas de Ant para las tareas de empaquetado que son necesitadas. Estas tareas pueden ser añadidas al fichero build.xml en el proyecto, o colocadas en un fichero separado que es importado por el fichero build.xml.

Para la Demostración Árbol Dinámico, el fichero packager.xml en el directorio raíz del proyecto contiene las tareas de Ant para generar los paquetes de la aplicación autocontenida. El fuente para el fichero packager.xml es mostrado en el siguiente ejemplo: 

<project name="DynamicTreePackaging" default="default" basedir="." xmlns:fx="javafx:com.sun.javafx.tools.ant">
    <echo>${java.home}/../lib/ant-javafx.jar</echo>
    <target name="package" depends="jar">
        <taskdef resource="com/sun/javafx/tools/ant/antlib.xml"
                 uri="javafx:com.sun.javafx.tools.ant"
                 classpath="${java.home}/../lib/ant-javafx.jar;src"/>

        <fx:deploy outdir="${basedir}/build/packager"
                   outfile="DynamicTreeDemo"
                   nativeBundles="all"
                   verbose="false">

            <fx:application name="Dynamic Tree Demo"
                        mainClass="webstartComponentArch.DynamicTreeApplication"
                        version="1.0"
            />

            <fx:resources>
                <fx:fileset dir="dist" includes="DynamicTreeDemo.jar"/>
            </fx:resources>

            <fx:info title="Dynamic Tree Demo"
                     vendor="My Company"
                     description="A Demo of a Dynamic Swing Tree"
                     category="Demos"
                     copyright="(c) 2014 My Company"
                     license="3 Clause BSD"
            />

            <fx:bundleArgument arg="linux.bundleName" value="dynamic-tree-demo"/>
            <fx:bundleArgument arg="email" value="maintainer@example.com"/>
            <fx:bundleArgument arg="mac.CFBundleName" value="Java Tree Demo"/>
            <fx:bundleArgument arg="win.menuGroup" value="Java Demos"/>

        </fx:deploy>
    </target>
</project>

Use la siguiente información para configurar las tareas de Ant:

  • Use xmlns:fx="javafx:com.sun.javafx.tools.ant como espacio de nombres.

  • La tarea taskdef tiene que ser ejecutada antes que la tarea fx:deploy. El atributo classpath contiene la localización del fichero ant-javafx.jar del KDJ y el directorio que contiene los recursos personalizados. Para la Demostración Árbol Dinámico, el atributo classpath incluye el directorio /src, el cual contiene los iconos personalizados.

  • Coloque la tarea fx:deploy dentro del objetivo deseado. Especifique el directorio de salida donde los binarios nativos serán colocados, y especifique los binarios nativos que usted quiera producir.

    Si all es especificado para los binarios nativos, todos los posibles binarios para la plataforna sobre la cual usted ejecuta este fichero de tareas serán generados, incluyendo la imagen de disco. Valores válidos para todos las plataformas son all; image, las cuales generan el fichero directorio en Windows y Linux y el fichero .app en OSX; y installer, el cual genera sólo paquetes instalables para la plataforma, pero no la imagen de disco. Valores válidos para los binarios específicos por plataforma son exe y msi para Windows; deb y rpm para Linux; deb, pkg, y mac.appStore para OS X. Usted debe tener las herramientas requeridas instaladas para construir los binarios de su elección.

    Para la Demostración Árbol Dinámico, el atributo outdir está fijado en ${basedir}/build/packager. basedir está definido en el elemento project, en este caso éste es fijado al directorio actual. El atributo nativeBundles es fijado a all de forma que todos los formatos para la plataforma en la cual la tarea de empaquetado es ejecutada son construidos.

  • El atributo verbose es opcional. Use este atributo para ofrecer información de diágnostico.

  • Ofrezca información sobre la aplicación. Fije el nombre de la aplicación en el atributo name del elemento fx:application y el atributo title del elemento fx:info. Fije la versión de la aplicación en el atributo version del elemento fx:application. Use el elemento fx:info para ofrecer una descripción de la aplicación, el nombre del vendedor, información de licencia, y otros metadatos.

  • La información sobre el fichero JAR y otros recursos están fijados en el elemento fx:resources.

  • La información de inicio está fijada en el atributo mainclass del elemento fx:application.

    Para la Demostración Árbol Dinámico, un lanzador único y simple es utilizado, webstartComponentArch.DynamicTreeApplication, el cual es la clase principal de la aplicación.

  • Otras personalizaciones específicas de cada plataforma son ofrecidas en los elementos fx:bundleArgument. Los argumentos que no son reconocidos por un empaquetador son ignorados, así que un fichero de construcción puede contener la información de empaquetado para todas las plataformas.

    Para la Demostración Árbol Dinámico, las siguientes personalizaciones son aplicadas:

    • El nombre del paquete para Linux es fijado como dynamic-tree-demo.
    • Una dirección de correo es ofrecida.
    • El nombre que aparece en la barra de menú para OS X es fijado como Java Tree Demo.
    • El nombre del grupo de menú en el cual la aplicación es almacenada en Windows está fijada como Java Demos.

Generando los Paquetes

Ejecute las tareas de empaquetado que usted creó sobre la plataforma para la cual usted quiere construir el paquete para su aplicación autocontenida.

Para la Demostración Árbol Dinámico, ejecute el siguiente comando desde la carpeta raíz del proyecto: 

     ant package

Cuando la tarea de empaquetado se completa, el directorio build/packager/bundles en el proyecto de la aplicación contendrá los binarios nativos que fueron producidos.

El siguiente ejemplo muestra la estructura de directorios para el proyecto de Demostración de Árbol Dinámico después que los paquetes autocontenidos son generados para Windows: 

/packager_DynamicTreeDemo     <--- Proyecto de la aplicación
   /build
      /packager
         /bundles
            Dynamic Tree Demo         <---Carpeta de imágenes
            Dynamic Tree Demo-1.0.exe <---Instalador EXE
            Dynamic Tree Demo-1.0.msi <---Instalador MSI
      ...
   /dist
      DynamicTreeDemo.jar
      ...
   /src
      /package                <--- recursos personalizados
         /linux
         /macosx
         /windows
      /webstartComponentArch  <--- ficheros fuente de la aplicación
      ...

Note que en adición de los paquetes autocontenidos, la herramienta de empaquetado siempre genera los ficheros JAR, JNLP y HTML para la aplicación. Estos ficheros nos ofrecen otras opciones para distribuir su aplicación.
