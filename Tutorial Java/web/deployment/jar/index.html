Lección: Empaquetando Programas en Ficheros JAR

El formato de fichero Java™ Archive (JAR) le permite agrupar múltiples ficheros en un único archivo comprimido. Normalmente un fichero JAR contiene los ficheros class y recursos auxiliares asociados con los applets y aplicaciones.

El formato de fichero JAR ofrece muchos beneficios:

  • Seguridad: Puede firmar digitalmente los contenidos de un fichero JAR. Los usuarios que reconozcan su firma pueden entonces otorgar opcionalmente a su software privilegios de seguridad que de otro modo no tendría.
  • Disminución del tiempo de descarga: Si su applet está incluído en un archivo JAR, los ficheros class del applet y los recursos asociados pueden ser descargados a un navegador en una única transacción HTTP sin la necesidad de abrir una nueva conexión por cada fichero.
  • Compresión: El formato JAR le permite comprimir sus ficheros para un almacenamiento eficiente.
  • Embalaje para extensiones: El marco de extensiones proporciona un medio por el cual puede agregar funcionalidad a la plataforma central de Java, y el formato de fichero JAR define el empaquetado para las extensiones. Al usar el formato de archivo JAR, también puede convertir su software en extensiones.
  • Sellado de Paquetes: Los paquetes almacenados en ficheros JAR pueden ser opcionalmente sellados de forma que el paquete pueda imponer la coherencia de la versión. Sellar un paquete dentro de un fichero JAR significa que todas las clases definidas en ese paquete deben encontrarse en el mismo fichero JAR.
  • Versiones de Paquetes: Un fichero JAR puede contener datos sobre los archivos que contiene, como la información del proveedor y la versión.
  • Portabilidad: El mecanismo para manejar archivos JAR es una parte estándar de la IPA del núcleo de la plataforma Java.
