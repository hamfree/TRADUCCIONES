Mejorando la Seguridad con los Atributos del Manifiesto

Los siguientes atributos del manifiesto del fichero JAR están disponibles para ayudar a garantizar la seguridad de su applet o aplicación Java Web Start. Sólo se requiere el atributo Permissions.

  • El atributo Permissions se usa para garantizar que la aplicación solicite sólo el nivel de permisos que se especifica en la etiqueta applet o fichero JNLP usados para invocar la aplicación. Use este atributo para ayudar a prevenir que alguien vuelva a redesplegar una aplicación que está firmada con su certificado y ejecutarla en un nivel de privilegio diferente.

    Este atributo se requiere en el manifiesto para el fichero JAR principal. Vea Atributo Permissions en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Codebase se usa para asegurar que el código base del fichero JAR se restringe a dominios específicos. Use este atributo para prevenir que alguien vuelva a redesplegar su aplicación en otro sitio web para propósitos maliciosos. Vea Atributo Codebase en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Application-Name se usa para proporcionar el título que se muestra en las indicaciones de seguridad para las aplicaciones firmadas. Vea Atributo Application-Name en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Application-Library-Allowable-Codebase se usa para identificar las ubicaciones donde se espera que su aplicación va a ser encontrada. Use este atributo para reducir el número de ubicaciones mostradas en las indicaciones de seguridad cuando el fichero JAR está en una ubicación diferente que la del fichero JNLP o la página HTML. Vea Atributo Application-Library-Allowable-Codebase en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Caller-Allowable-Codebase se usa para identificar los dominios desde los cuales el código JavaScript puede hacer llamadas a su aplicación. Use este atributo para prevenir que código JavaScript desconocido acceda a su aplicación. Vea Atributo Caller-Allowable-Codebase en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Entry-Point se usa para identificar las clases que están permitidas para ser usadas como puntos de entrada en su ARI. Use este atributo para prevenir que código no autorizado sea ejecutado desde otros puntos de entrada disponibles en el fichero JAR. Vea Atributo Entry-Point en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Trusted-Only se usa para prevenir que se carguen componentes no confiables. Vea Atributo Trusted-Only en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

  • El atributo Trusted-Library se usa para permitir las llamadas entre código de Java privilegiado y código Java del entorno limitado de seguridad sin solicitar permiso al usuario. Vea Atributo Trusted-Library en la Guía de Despliegue de la Plataforma Java, Edición Estándar para más información.

Vea Modificando un Fichero de Manifiesto para más información de como agregar estos atributos al fichero de manifiesto.
