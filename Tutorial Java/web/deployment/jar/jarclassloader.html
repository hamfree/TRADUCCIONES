La Clase JarClassLoader

La clase JarClassLoader extiende java.net.URLClassLoader. Como su nombre implica, URLClassLoader está diseñada para ser usada en la carga de clases y recursos que son accedidos al buscar un conjunto de LURs (Localizador Uniforme de Recursos, URL en inglés). Los LURs pueden referirse a directorios o a archivos JAR.

Además de extender URLClassLoader, JarClassLoader también hace uso de características de otras dos nuevas IPAs relacionadas con JAR, el paquete java.util.jar y la clase java.net.JarURLConnection. En esta sección, veremos en detalle el constructor y los dos métodos de JarClassLoader.

El Constructor de JarClassLoader

El constructor toma una instancia de java.net.URL como argumento. EL LUR pasado a este contructor será usado en otras partes en JarClassLoader para encontrar el archivo JAR desde el que se cargarán las clases. 

public JarClassLoader(URL url) {
    super(new URL[] { url });
    this.url = url;
}

El objeto URL se pasa al constructor de la superclase, URLClassLoader, el cual toma una matriz URL[], en vez de una instancia única de URL, como argumento.

El Método getMainClassName

Una vez se construye un objeto JarClassLoader con el LUR de una aplicación empaquetada en un JAR, se va a necesitar una forma de determinar qué clase en el archivo JAR es el punto de entrada de la aplicación. Ese es el trabajo del método getMainClassName: 

public String getMainClassName() throws IOException {
    URL u = new URL("jar", "", url + "!/");
    JarURLConnection uc = (JarURLConnection)u.openConnection();
    Attributes attr = uc.getMainAttributes();
    return attr != null
                   ? attr.getValue(Attributes.Name.MAIN_CLASS)
                   : null;
}

Puede recordar de una lección previa que el punto de entrada de una aplicación empaquetada en un JAR se especifica en la cabecera Main-Class del manifiesto del archivo JAR. Para entender cómo getMainClassName accede al valor de la cabecera Main-Class, veamos el método en detalle, prestando atención especial a las nuevas características de manejo del JAR que usa:

La clase JarURLConnection y los LURs de JAR

El método getMainClassName usa el formato LUR de JAR especificado por la clase java.net.JarURLConnection. La sintaxis para el LUR de un archivo JAR es como en este ejemplo: 

jar:http://www.example.com/jarfile.jar!/

El separador !/ de terminación indica que el LUR se refiere a un archivo JAR entero. Cualquier cosa que siga al separador se refiere a contenidos específicos del archivo JAR, como en este ejemplo: 

jar:http://www.example.com/jarfile.jar!/mypackage/myclass.class

La primera línea en el método getMainClassName es: 

URL u = new URL("jar", "", url + "!/");

Esta sentencia construye un objeto URL nuevo representando un LUR de JAR, agregando el separador !/ al LUR que fue usado en la creación de la instancia de JarClassLoader.

La clase java.net.JarURLConnection

Esta clase representa un enlace de comunicaciones entre una aplicación y un archivo JAR. Tiene métodos para acceder al manifiesto del archivo JAR. La segunda línea de getMainClassName es: 

JarURLConnection uc = (JarURLConnection)u.openConnection();

En esta sentencia, la instancia URL creada en la primera línea abre una URLConnection. La instancia URLConnection se moldea a una JarURLConnection para que pueda aprovechar las funciones de manejo de JAR de JarURLConnection.

Obtener Atributos del Manifiesto: java.util.jar.Attributes

Con una JarURLConnection abierta a un archivo JAR, puede acceder a la información de cabecera del manifiesto del archivo JAR usando el método getMainAttributes de JarURLConnection. Este método devuelve una instancia de java.util.jar.Attributes, una clase que asigna nombres de cabecera de los manifiestos de los archivos JAR con sus valores de cadena asociados. La tercera línea en getMainClassName crea un objeto Attributes: 

Attributes attr = uc.getMainAttributes();

Para obtener el valor de la cabecera Main-Class del manifiesto, la línea cuarta de getMainClassName invoca el método Attributes.getValue: 

return attr != null
               ? attr.getValue(Attributes.Name.MAIN_CLASS)
               : null;

El argumento del método, Attributes.Name.MAIN_CLASS, especifica que es el valor de la cabecera Main-Class que usted quiere. (La clase Attributes.Name también ofrece campos estáticos tales como MANIFEST_VERSION, CLASS_PATH, y SEALED para especificar cabeceras estándar del manifiesto).

El Método invokeClass

Hemos visto cómo JarURLClassLoader pueden identificar la clase principal en una aplicación empaquetada en un JAR. El último método a considerar, JarURLClassLoader.invokeClass, permite que la clase principal sea invocada para lanzar la aplicación empaquetada en el JAR: 

public void invokeClass(String name, String[] args)
    throws ClassNotFoundException,
           NoSuchMethodException,
           InvocationTargetException
{
    Class c = loadClass(name);
    Method m = c.getMethod("main", new Class[] { args.getClass() });
    m.setAccessible(true);
    int mods = m.getModifiers();
    if (m.getReturnType() != void.class || !Modifier.isStatic(mods) ||
        !Modifier.isPublic(mods)) {
        throw new NoSuchMethodException("main");
    }
    try {
        m.invoke(null, new Object[] { args });
    } catch (IllegalAccessException e) {
        // Esto no debería ocurrir, ya que hemos deshabilitado las comprobaciones de acceso
    }
}

El método invokeClass toma dos argumentos: el nombre de la clase que es el punto de entrada de la aplicación y una matriz de argumentos de cadena a pasar al método main de la clase punto de entrada. Primero, se carga la clase principal: 

Class c = loadClass(name);

El método loadClass se hereda desde java.lang.ClassLoader.

Una vez la clase principal se ha cargado, la IPA de reflexión del paquete java.lang.reflect se usa para pasar los argumentos a la clase y lanzarla. Puede referirse al tutorial en La IPA de Reflexión para una revisión de la reflexión.
