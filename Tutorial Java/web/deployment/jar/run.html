Ejecutando Software Empaquetado en Ficheros JAR

Ahora que ha aprendido cómo crear ficheros JAR, ¿Cómo ejecuta realmente el código que empaquetó? Considere estos escenarios:

  • Su fichero JAR contiene un applet que va a ser ejecutado dentro de un navegador.
  • Su fichero JAR contiene una aplicación que va a ser iniciad desde la línea de comandos.
  • Su fichero JAR contiene código que quiere usar como una extensión.

Esta sección cubrirá las primeras dos situaciones. Un recorrido separado en el tutorial sobre el mecanismo de extensión cubre el uso de los ficheros JAR como extensiones.

Applets Empaquetados en Ficheros JAR

Para iniciar cualquier applet desde un fichero HTML para ejecutar dentro de un navegador, usted usa la etiqueta applet. Para más información, vea la lección Applets de Java . Si el applet se incluye como un fichero JAR, la única cosa que necesita hacer diferente es usar el parámetro archive para especificar la ruta relativa al fichero JAR.

Como ejemplo, use la demo del applet TicTacToe. La etiqueta applet en el fichero HTML que visualiza el applet puede marcarse así: 

<applet code=TicTacToe.class 
        width="120" height="120">
</applet>

Si la demo TicTacToe fue empaquetada en un fichero JAR de nombre TicTacToe.jar, puede modificar la etiqueta applet con la adicción de un parámetro archive: 

<applet code=TicTacToe.class 
        archive="TicTacToe.jar"
        width="120" height="120">
</applet>

El parámetro archive especifica la ruta relativa al fichero JAR que contiene TicTacToe.class. Para este ejemplo se asume que el fichero JAR y el fichero HTML están en el mismo directorio. Si no lo estuvieran, debe incluir las rutas relativas a los ficheros JAR en el valor del parámetro archive. Por ejemplo, si el fichero JAR estuviera en un directorio por debajo de un fichero HTML en un directorio llamado applets, la etiqueta applet se vería así: 

<applet code=TicTacToe.class 
        archive="applets/TicTacToe.jar"
        width="120" height="120">
</applet>

Ficheros JAR como Aplicaciones

Puede ejecutar aplicaciones empaquetadas en JAR con el lanzador Java (comando java). El comando básico es: 

java -jar jar-file

El parámetro -jar le indica al lanzador que la aplicación está empaquetada con el formato de fichero JAR. Usted sólo puede especificar un fichero JAR, el cual debe contener todo el código específico de la aplicación.

Antes de que ejecute este comando, asegúrese de que el entorno de ejecución tiene información sobre qué clase dentro del fichero JAR es el punto de entrada de la aplicación.

Para indicar qué clase es el punto de entrada de la aplicación, debe agregar una cabecera Main-Class en el manifiesto del fichero JAR. La cabecera toma la forma: 

Main-Class: nombreClase

El valor de la cabecera, nombreClase, es el nombre de la clase que es el punto de entrada de la aplicación.

Para más información vea la sección Configurando el Punto de Entrada de la Aplicación.

Cuando se establece Main-Class en el fichero de manifiesto, puede ejecutar la aplicación desde la línea de comandos: 

java -jar app.jar

Para ejecutar la aplicación desde un fichero JAR que está en otro directorio, debe especificar la ruta de ese directorio: java -jar path/app.jar
