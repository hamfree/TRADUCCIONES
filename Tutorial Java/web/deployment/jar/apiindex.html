Usando IPAs relacionadas con el formato JAR

La plataforma Java contiene varias clases para usar con archivos JAR. Algunas de estas IPAs son:

  • El paquete java.util.jar
  • La clase java.net.JarURLConnection
  • La clase java.net.URLClassLoader

Para darle una idea de las posibilidades que se abren con estas nuevas IPAs, esta lección le guía a través del funcionamiento interno de una aplicación de muestra llamada JarRunner.

Un Ejemplo - La Aplicación JarRunner

JarRunner le permite ejecutar una aplicación que se incluye en un archivo JAR especificando la URL del archivo JAR en la línea de comando. Por ejemplo, si una aplicación llamada TargetApp fue empaquetada en un archivo JAR en http://www.example.com/TargetApp.jar, podría ejecutar la aplicación usando este comando: 

java JarRunner http://www.example.com/TargetApp.jar

Para que JarRunner funcione, debe ser capaz de realizar las siguientes tareas, todas las cuales se realizan mediante el uso de las nuevas IPAs:

  • Acceder al fichero JAR remoto y establecer un enlace de comunicaciones con él.
  • Inspeccionar el manifiesto del archivo JAR para ver cuál de las clases en el archivo es la clase principal.
  • Cargar las clases en el archivo JAR.

La aplicación JarRunner consiste en dos clases, JarRunner y JarClassLoader. JarRunner delega la mayoría de las tareas de manejo del JAR a la clase JarClassLoader. JarClassLoader extiende la clase java.net.URLClassLoader.  Puede navegar el código fuente de las clases JarRunner y JarClassLoader antes de proceder con la lección:

  • JarRunner.java
  • JarClassLoader.java
