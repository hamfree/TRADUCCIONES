Respuestas a Preguntas y Ejercicios: JAR

Preguntas

  1. Pregunta: ¿Cómo invocar un applet que está empaquetado como un archivo JAR?

    Respuesta: Para invocar un applet empaquetado como un archivo JAR, abra una página que contenga el applet: 

    <applet code=AppletClassName.class
        archive="JarFileName.jar"
        width=320 height=240>
</applet>

  2. Pregunta: ¿Cuál es el objetivo de la opción -e en un comando jar?

    Respuesta: Esta opción está disponible desde Java SE 6. Establece el punto de entrada como el punto de entrada de la aplicación para aplicaciones autónomas empaquetadas dentro de archivos jar ejecutables. El uso de esta opción crea o sobreescribe el valor del atributo Main-Class en el archivo de manifiesto. Esta opción puede ser utilizada durante la creación de un archivo jar o mientras se actualiza un archivo jar. Esta opción especifica el punto de entrada de la aplicación sin editar o crear el archivo de manifiesto. Por ejemplo, este comando crea Main.jar donde el valor del atributo Main-Class en el manifiesto es establecido a Main: 

    jar cfe Main.jar Main Main.class

  3. Pregunta: ¿Cuál es la significancia del manifiesto en un archivo JAR?

    Respuesta: El manifiesto de un archivo JAR proporciona metainformación sobre los otros contenidos del archivo JAR. El manifiesto en sí reside en META-INF/MANIFEST.mf. La metainformación puede incluir

    • Dependencias en otros archivos jar
    • El nombre de la clase a ejecutar cuando se invoca "java -jar file.jar"
    • Información de versiones
    • Información de seguridad

  4. Pregunta: ¿Cómo se modifica el archivo de manifiesto del JAR?

    Respuesta: Por lo general, la modificación del manifiesto predeterminado implica agregar encabezados de propósito especial al manifiesto que permiten que el archivo JAR realice una función particular deseada.

    Para modificar el manifiesto, primero debe preparar un archivo de texto con un archivo de manifiesto completo y válido. Luego utiliza la opción m de la herramienta JAR para agregar la información de su archivo al manifiesto.

    El archivo de manifiesto que prepare debe terminar con una nueva línea o retorno de carro. La última línea no se analizará correctamente si no termina con una nueva línea o retorno de carro.
