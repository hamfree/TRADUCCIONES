Sellando Paquetes dentro de un Fichero JAR

Los paquetes dentro de los ficheros JAR pueden ser sellados opcionalmente, lo que significa que todas las clases definidas en ese paquete deben ser archivadas en el mismo fichero JAR. Podría querer sellar un paquete, por ejemplo, para garantizar la coherencia de la versión entre las clases de su software.

Usted sella un paquete en un fichero JAR agregando el encabezado Sealed en el manifiesto, el cual tiene la forma general: 

Name: myCompany/myPackage/
Sealed: true

El valor myCompany/myPackage/ es el nombre del paquete a sellar.

Note que el nombre del paquete debe terminar con una "/".

Un Ejemplo

Queremos sellar dos paquetes firstPackage y secondPackage en el fichero JAR MyJar.jar.

Primero creamos un fichero de texto llamado Manifest.txt con los contenidos siguientes: 

Name: myCompany/firstPackage/
Sealed: true

Name: myCompany/secondPackage/
Sealed: true
Advertencia: El fichero de texto debe terminar con una nueva línea o retorno de carro. La última línea no será analizada apropiadamente si no termina con una nueva línea o retorno de carro.

Entonces creamos un fichero JAR llamado MyJar.jar introduciendo el comando sigiuente: 

jar cfm MyJar.jar Manifest.txt MyPackage/*.class

Esto crea el fichero JAR con un manifiesto con los contenidos siguientes: 

Manifest-Version: 1.0
Created-By: 1.7.0_06 (Oracle Corporation)
Name: myCompany/firstPackage/
Sealed: true
Name: myCompany/secondPackage/
Sealed: true

Sellar Ficheros JAR

Si quiere garantizar que todas las clases en un paquete vengan de la misma fuente de código, use sellado de JAR. Un JAR sellado especifica que todos los paquetes definidos por el JAR están sellados a menos que se anulen por paquete.

Para sellar un fichero JAR, use el encabezado Sealed del manifiesto con el valor true. Por ejemplo, 

Sealed: true

Especifica que todos los paquetes en este archivo están sellados a menos que se anulen explícitamente para paquetes particulares con el atributo Sealed en una entrada de manifiesto.
