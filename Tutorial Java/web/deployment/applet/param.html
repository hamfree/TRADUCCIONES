Definiendo y Usando Parámetros del Applet

Los parámetros son a los applets de Java como los argumentos de línea de comando son a las aplicaciones. Habilitan al usuario a personalizar la operación del applet. Al definir los parámetros, puede aumentar la flexibilidad de su applet, haciendo que su applet trabaje en múltiples situaciones sin recodificarlo y recompilarlo.

Especificar los Párametros de Entrada de un Applet

Puede especificar los parámetros de entrada de un applet en el fichero Java Network Laun Protocol (JNLP) del applet o en el elemento <parameter> de la etiqueta <applet>. Es usualmente mejor especificar los parámetros en el fichero JNLP del applet de forma que los parámetros se puedan suministrar de manera consistente, incluso si el applet se implementa en varias páginas web. Si los parámetros del applet variaran por página web, entonces deberá especificar los parámetros en el elemento <parameter> de la etiqueta <applet>.

Si no está familiarizado con JNLP, vea el tema Protocolo de Lanzamiento en Red de Java para más información.

Considere un applet que toma tres parámetros. Los parámetros paramStr y paramInt están especificados en el fichero JNLP del applet, applettakesparams.jnlp. 

<?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="" href="">
    <!-- ... -->
    <applet-desc
         name="Applet Takes Params"
         main-class="AppletTakesParams"
         width="800"
         height="50">
             <param name="paramStr"
                 value="someString"/>
             <param name="paramInt" value="22"/>
     </applet-desc>
     <!-- ... -->
</jnlp>

El parámetro paramOutsideJNLPFile es especificado en la variable parameters pasada a al script de la Herramienta de Despliegues en la función runApplet en AppletPage.html. 

<html>
  <head>
    <title>Applet Takes Params</title>
    <meta http-equiv="Content-Type" content="text/html;
        charset=windows-1252">
  </head>
  <body>
    <h1>Applet Takes Params</h1>

    <script
      src="https://www.java.com/js/deployJava.js"></script>
    <script>
        var attributes = { code:'AppletTakesParams.class',
            archive:'applet_AppletWithParameters.jar',
            width:800, height:50 };
        var parameters = {jnlp_href: 'applettakesparams.jnlp',
            paramOutsideJNLPFile: 'fooOutsideJNLP' };
        deployJava.runApplet(attributes, parameters, '1.7');
    </script>

  </body>
</html>

Vea Desplegando un Applet para más información sobre la función runApplet.

Recuperar los Parámetros de Entrada del Applet.

Puede recuperar los parámetros de entrada del applet usando el método getParameter de la clase Applet. El applet AppletTakesParams.java recupera y visualiza todos sus parámetros de entrada (paramStr, paramInt, y paramOutsideJNLPFile). 


import javax.swing.JApplet;
import javax.swing.SwingUtilities;
import javax.swing.JLabel;

public class AppletTakesParams extends JApplet {
    public void init() {
        final String  inputStr = getParameter("paramStr");        
        final int inputInt = Integer.parseInt(getParameter("paramInt"));
        final String inputOutsideJNLPFile = getParameter("paramOutsideJNLPFile");

        try {
            SwingUtilities.invokeAndWait(new Runnable() {
                public void run() {
                    createGUI(inputStr, inputInt, inputOutsideJNLPFile);
                }
            });
        } catch (Exception e) {
            System.err.println("createGUI didn't successfully complete");
        }
    }
    private void createGUI(String inputStr, int inputInt, String inputOutsideJNLPFile) {
        String text = "Applet's parameters are -- inputStr: " + inputStr +
                ",   inputInt: " + inputInt +
                ",   paramOutsideJNLPFile: " + inputOutsideJNLPFile;
        JLabel lbl = new JLabel(text);
        add(lbl);
    }
}

el applet AppletTakesParams es mostrado a continuación.

Nota:  Si usted no ve el applet ejecutándose, necesita instalar al menos la versión Java SE Development Kit (KDJ) 6 update 10.
Nota:  Si usted no ve el ejemplo ejeucutándose, podría necesitar habilitar el intérprete de JavaScript en su navegador de forma que el script de la Herramienta de Desplieuge pueda funcionar adecuadamnte.
