Definiendo una Subclase de un Applet

Cada applet de Java debe definir una subclase de la clase Applet o JApplet. En el applet Hello World, esta subclaes es llamada HelloWorld. Lo siguiente es el código para la clase HelloWorld . 


import javax.swing.JApplet;
import javax.swing.SwingUtilities;
import javax.swing.JLabel;

public class HelloWorld extends JApplet {
    //Llamado cuando el applets es cargado en el navegador.
    public void init() {
        //Ejecuta un trabajo en el hilo de envío de eventos; creando el IGU del applet.
        try {
            SwingUtilities.invokeAndWait(new Runnable() {
                public void run() {
                    JLabel lbl = new JLabel("Hello World");
                    add(lbl);
                }
            });
        } catch (Exception e) {
            System.err.println("createGUI didn't complete successfully");
        }
    }
}

Los applets de Java heredan funcionalidad significativa desde la clase Applet o JApplet, incluyendo las capacidades para comunicarse con el navegador y presentar un interfaz de usuario gráfico (GUI) al usuario.

Un applet que esté usando componentes GUI de Swing (Kit de herramientas para el IGU de Java) extenderá la clase base javax.swing.JApplet , la cual ofrece la mejor integración con las facilidades IGU de Swing.

JApplet ofrece un panel raíz, el cual es el mismo componente en lo más alto de la estructura que los componentes de Swing JFrame y JDialog, mientas que Applet ofrece sólo un panel básico. Vea Cómo Usar los Paneles Raíz para más detalles de cómo usar esta característica.

Un applet puede extender la clase java.applet.Applet cuando no use componentes del IGU de Swing.
