Implantando Con la Etiqueta Applet

Si usted no está seguro si los navegadores de sus usuarios finales tendrán el intérprete de JavaScript habilitado, puede desplegar su applet de Java codificando manualmente la etiqueta HTML <applet>, en vez de usar las ufnciones del Kit de Herramientas de Despliegue. Dependiendo de los navegadores usted necesita soportar, puede necesitar desplegar su applet de Java usando la etiqueta HTML <object> o <embed>. Compruebe la Especificación HTML del W3C para los detalles del uso de estas etiquetas.

Puede lanzar su applet usando el Protocolo de Lanzamiento en Red de Java (JNLP) o especicar los atributos de lanzamiento directamente en la etiqueta <applet>.

Preparándose para el Despliegue

Siga los pasos descritos en el tema Implantando un Applet para compilar su código fuente, crear y firmar el fichero JAR, y crear el fichero JNLP si es necesario. Los pasos generales para el despliegue siguen siendo relevantes. Sólo los contenidos de su página HTML conteniendo el applet cambiarán.

Codificar Manualmente el Tag Applet, Lanzar Usando JNLP

La página AppletPage_WithAppletTag.html despliega el applet de la Demo del Árbol Dinámico con una etiqueta <applet> que ha sido manualmente codificada (lo que significa que el applet no se implementa utilizando el Kit de Herramientas de Despliegue que genera automáticamente el HTML requerido). El applet es todavía lanzado usando JNLP. El fichero JNLP es especificado en el atributo jnlp_href. 

<applet code = 'appletComponentArch.DynamicTreeApplet' 
        jnlp_href = 'dynamictree_applet.jnlp'
        width = 300
        height = 300 />

Codificación Manual de la Etiqeuta Applet, Lanzamiento Sin JNLP

Usar JNLP es la forma preferida de desplegar un applet, sin embargo, puede también desplegar su applet sin un fichero JNLP.

La página AppletPage_WithAppletTagNoJNLP.html despliega el applet de la Demo del Árbol Dinámico como se muestra en el siguiente trozo de código. 

<applet code = 'appletComponentArch.DynamicTreeApplet' 
    archive = 'DynamicTreeDemo.jar'
    width = 300
    height = 300>
    <param name="permissions" value="sandbox" />
</applet>

donde

  • code es el nombre de la clase del applet.
  • archive es el nombre del fichero jar que contiene el applet y sus recursos.
  • width es el ancho del applet.
  • height es la altura del applet.
  • permissions indican si el applet se ejecuta en el recinto de seguridad (sandbox). Especifique "sandbox" para el valor de ejecutar en el recinto de seguridad. Especifique "all-permissions" para ejecutar fuera del recinto de seguridad. Si el parámetro permissions no está presente, los applets firmados por defecto tienen asignados el permiso "all-permissions" y los applets sin firmar por defecto tienen asignados el permiso "sandbox".
