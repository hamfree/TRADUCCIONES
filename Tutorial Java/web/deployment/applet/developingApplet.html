Desarrollando un Applet

Una aplicación diseñada usando una arquitectura basada en compoentes puede ser desarrollada dentro de un applet de Java. Considere el ejemplo de un applet de Java con un interfaz gráfico de usuario basado en Swing (GUI en inglés, IGU en castellano). Con el diseño basado en componentes, el IGU puede ser construído con bloques de construcción más pequeños o componentes. Los pasos generales siguientes son usados para crear un IGU de applet:

  • Cree una clase MyTopJPanel que sea una subclase de javax.swing.JPanel. Coloque los componentes IGU de su applet en el constructor de la clase MyTopJPanel.
  • Cree una clase llamada MyApplet que sea una subclase de javax.swing.JApplet.
  • En el método init de MyApplet, instancie MyTopJPanel y establézcalo como el panel de contenido del applet.

Las secciones siguientes exploran estos pasos con mayor detalle mediante el uso del applet de la Demo del Árbol Dinámico. Si no está familiarizado con Swing, vea Recorrido: Creando una IGU (Interfaz Gráfica de Usuario) con JFC/Swing para aprender más sobre el uso de componentes Swing para el IGU.

Nota:  Si no ve el ejemplo ejecutándose, podría necesitar habilitar el intérprete de JavaScript en su navegador de forma que el script del Kit de Herramientas de Despliegue pueda funcionar aecuadamente.

Crear la Clase Superior JPanel

Cree una clase que sea una subclase de JPanel. Este JPanel superior actúa como un contenedor para todos sus otros componentes de IU (Interfaz de usuario). En el ejemplo siguiente, la clase DynamicTreePanel es el JPanel más alto. El constructor de la clase DynamicTreePanel invoca otros métodos para crear y organizar los controles del IU adecuadamente. 

public class DynamicTreePanel extends JPanel implements ActionListener {
    private int newNodeSuffix = 1;
    private static String ADD_COMMAND = "add";
    private static String REMOVE_COMMAND = "remove";
    private static String CLEAR_COMMAND = "clear";
    
    private DynamicTree treePanel;

    public DynamicTreePanel() {
        super(new BorderLayout());
        
        //Creamos los componentes.
        treePanel = new DynamicTree();
        populateTree(treePanel);

        JButton addButton = new JButton("Add");
        addButton.setActionCommand(ADD_COMMAND);
        addButton.addActionListener(this);
        
        JButton removeButton = new JButton("Remove");
        
        // ...
        
        JButton clearButton = new JButton("Clear");
        
        // ...
        
        //Organizamos todo.
        treePanel.setPreferredSize(
            new Dimension(300, 150));
        add(treePanel, BorderLayout.CENTER);

        JPanel panel = new JPanel(new GridLayout(0,3));
        panel.add(addButton);
        panel.add(removeButton); 
        panel.add(clearButton);
        add(panel, BorderLayout.SOUTH);
    }
    // ...
}

Crear el Applet

Para un applet de Java que tiene un IGU basado en Swing, cree una clase que sea una subclase de javax.swing.JApplet. Un applet que no contenga un IGU basado en Swing puede extender la clase java.applet.Applet.

Sobreescriba el método init del applet para instanciar su clase inicial en la jerarquía JPanel y cree el IGU dl applet. El método init de la clase DynamicTreeApplet invoca al método createGUI en el hilo del Despachador de Eventos de AWT. 

package appletComponentArch;

import javax.swing.JApplet;
import javax.swing.SwingUtilities;

public class DynamicTreeApplet extends JApplet {
    //Llamado cuando este applet es cargado en el navegador.
    public void init() {
        //Ejecuta un trabajo en el hilo despachador de eventos, creando este IGU del applet.
        try {
            SwingUtilities.invokeAndWait(new Runnable() {
                public void run() {
                    createGUI();
                }
            });
        } catch (Exception e) { 
            System.err.println("createGUI no se completó con éxito");
        }
    }
    
    private void createGUI() {
        //Crea y configura el panel de contenidos.
        DynamicTreePanel newContentPane = new DynamicTreePanel();
        newContentPane.setOpaque(true); 
        setContentPane(newContentPane);        
    }        
}

Beneficios de separar la funcionalidad básica del mecanismo de implementación final

Otra forma de crear un applet es simplemente eliminar la capa de abstracción (separar JPanel) y organizar todos los controles en el método init del applet por sí mismo. La desventaja de crear la IGU directamente en el applet es que ahora será más dificil implementar su funcionalidad como una aplicación Java Web Start, si elige hacerlo más adelante.

En el ejemplo Demo Árbol Dinámico, la funcionalidad principal reside en la clase DynamicTreePanel. Es ahora trivial soltar la clase DynamicTreePanel dentro de un JFrame y desplegarla como una aplicación Java Web Start.

Por lo tanto, para preservar la portabilidad y mantener abiertas las opciones de implementación, siga el diseño basado en componentes como se describe en esta página.
