Métodos para las Etapas

La clase Applet suministra un marco de trabajo para la ejecución de applets, definiendo métodos que el sistema llama cuando se producen hitos. Los hitos son eventos importantes en el ciclo de vida de un applet. La mayoría de los applets sobreescriben alguno o todos los métodos para responder apropiadamente a estos hitos.

Método init

El método init es útil para la inicialización de una vez que no toma mucho tiempo. El método init contiene habitualmente el código que usted pondría normalmente dentro de un constructor. La razón de que los applets no tengan usualmente constructores es que no se les garantiza un entorno completo hasta que se llama a su método init. Mantenga el método init corto de forma que su applet pueda cargar rápidamente.

Método start

Cada applet que ejecuta tareas después de la inicialización (excepto en respuesta directa a las acciones del usuario) debe sobreescribir el método start. El método start inicia la ejecución del applet. Es una buena práctica salir rápidamente del método start. Si necesita ejecutar operaciones computacionalmente intensivas podría ser mejor iniciar un nuevo hilo para este propósito.

Método stop

La mayoría de los applets que sobreescriben start también deberían sobreeescribir el método stop. El método stop debe suspender la ejecución del applet, de forma que no ocupe los recursos del sistema cuando el usuario no esté viendo la página del applet. Por ejemplo, un applet que visualiza una animación debe parar de dibujar la animación cuando el usuario no la está viendo.

Método destroy

Muchos applets no necesitan sobreescribir el método destroy porque su método stop (el cual es llamado antes de destroy) realizará todas las tareas necesarias para terminar la ejecución del applet. Sin embargo, el método destroy está disponible para los applets que necesitan liberar recursos adicionales.

Nota: Mantenga las implementaciones del método destroy tan cortas como le sea posible, porque no hay garantiía de que este método sea completamente ejecutado. La Máquina Virtual de Java podría terminar antes de que un método destroy largo se haya completado.
