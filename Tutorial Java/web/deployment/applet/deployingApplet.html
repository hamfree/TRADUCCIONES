Implantando un Applet

Para desplegar su applet de Java, compile primero el código fuente, empaquételo como un fichero JAR, y firme el fichero JAR.

Los applet de Java Pueden ser lanzados de dos formas.

  • Puede lanzar su applet usando el Protocolo de Lanzamiento en Red de Java (JNLP). Los applets lanzados usando JNLP tienen acceso a las IPAS poderosas de JNLP y las extensiones.
  • Alternativamente, puede lanzar un applet especificando las propiedades de lanzamiento del applet directamente en la etiqueta applet. Sin embargo, este vieja forma de desplegar applets impone severas restricciones al applet.

El script del Kit de Herramientas de Despliegue contiene útiles funciones de JavaScript que pueden ser utilizadas para desplegar applets en una página web.

Si no está familizarizado con estas tecnologías de despliegue, revise la lección Despliegue En Profundidad antes de continuar.

Aquí hay algunas instrucciones paso a paso para empaquetar y desplegar su applet. El applet de la Demo del Árbol Dinámico se usa para ilustrar el despliegue del applet. Usted podría querer configurar scripts de ejecución para ejecutar algunos de los pasos siguientes.

Nota:  Si no ve el ejemplo ejecutándose, podría necesitar habilitar el interpretador de JavaScript en su navegador de forma que el script del Kit de Herramientas de Despliegue pueda funcionar adecuadamente.
  1. Compile el código Java de su applet y asegúrese de que todos los ficheros de clas y recursos tales como imágenes están en un directorio separado.

    En el caso del applet Demo DynamicTree, las clases compiladas están colocadas en el directorio build/classes/appletComponentArch.

  2. Cree un fichero de texto que contenga los atributos de manifiesto del archivo JAR que su applet necesita.

    Para el applet de la Demo DynamicTree, cree un fichero de nombre mymanifest.txt en el directorio build/classes, y agrege los atributos Permissions, Codebase, y Application-Name. El applet no requiere acceso a los recursos del sistema del usuario, así que use sandbox para los permisos. Use el dominio desde el cual usted cargará el ejemplo para el código base, por ejemplo, myserver.com. Agregue los atributos siguientes al fichero mymanifest.txt. 

    Permissions: sandbox
Codebase: myserver.com
Application-Name: Dynamic Tree Demo

    Otros atributos del manifiesto están disponibles para restringir un applet para usar sólo código de confianza, y para suministrar seguridad para applets que necesitan hacer llamadas entre código privilegiado de Java y código de Java del recinto de seguridad (sandbox), o tener código de Javascript que llame al applet. Vea la lección Mejorando la Seguridad con los Atributos del Manifiesto para aprender más sobre los atributos de manifiesto que están disponibles.

  3. Cree un fichero JAR que contenga sus ficheros de clase del applet y recursos. Incluya los atributos de manifiesto en el fichero mymanifest.txt que usted creó en el paso previo.

    Por ejemplo, el comando siguiente crea un fichero JAR con los ficheros class en el directorio build/classes/appletComponentArch y el fichero de manifiesto en el directorio build/classes. 

    % cd build/classes
% jar cvfm DynamicTreeDemo.jar mymanifest.txt appletComponentArch

    Vea la lección Lección: Empaquetando Programas en Ficheros JAR para aprender más sobre la creación y el uso de ficheros JAR.

  4. Firme el fichero JAR para su applet y marque la firma. Utilice un certificado de firma de código válido y actual emitido por una autoridad de certificación de confianza para proporcionar a sus usuarios la seguridad de que es seguro ejecutar el applet.

    Vea la lección Firmando Ficheros JAR para más información.

    Si quiere usar un fichero JNLP firmado por seguridad, cree el fichero JNLP como es descrito en el paso siguiente e inclúyalo en el fichero JAR antes de que el fichero JAR sea firmado. Vea Ficheros JNLP Firmados en la Guía de implementación de la la plataforma Java, edición estándar para obtener más información.

  5. Cree un fichero JNLP que describa como debe ser lanzado su applet.

    Aquí está el fichero JNLP usado para lanzar el applet de la Demo del Árbol Dinámico.

    El fuente para dynamictree_applet.jnlp a continuación: 
    <?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="" href="">
    <information>
        <title>Dynamic Tree Demo</title>
        <vendor>Dynamic Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.7+"
            href="http://java.sun.com/products/autodl/j2se" />
        <jar href="DynamicTreeDemo.jar" main="true" />

    </resources>
    <applet-desc 
         name="Dynamic Tree Demo Applet"
         main-class="components.DynamicTreeApplet"
         width="300"
         height="300">
     </applet-desc>
     <update check="background"/>
</jnlp>

    Note que el elemento de seguridad para solicitar permisos adicionales no está presente en el fichero JNLP, por lo tanto el applet se ejecuta sólo en el recinto de seguridad.

    El tema, Estructura del Fichero JNLP, describe la sintaxis de un fichero JNLP y las opciones.

  6. Cree la página HTML que mostrará el applet. Invoque las funciones del Kit de Herramientas de Despliegue para desplegar el applet.

    En nuestro ejemplo, el appleet de la Demo del Árbol Dinámico es desplegado en AppletPage.html. 

    <body>
    <!-- ... -->
    <script src="https://www.java.com/js/deployJava.js"></script>
    <script> 
        var attributes = {
            code:'components.DynamicTreeApplet',  width:300, height:300} ; 
        var parameters = {jnlp_href: 'dynamictree_applet.jnlp'} ; 
        deployJava.runApplet(attributes, parameters, '1.7'); 
    </script>
    <!-- ... -->
</body>
  7. Coloque el fichero JAR del applet, el fichero JNLP y la página HTML en las carpetas apropiadas.

    Para este ejemplo, coloque DynamicTreeDemo.jar, dynamictree_applet.jnlp, y AppletPage.html en el mismo directorio en la máquina local o e un servidor web. Se prefiere un servidor web. Para ejecutar desde la máquina loca, debe agregar a su aplicación en la lista de excepciones de sitios, que es gestionada desde la pestaña de Seguridad del Panel de Control de Java.

  8. Abra la página HTML del applet en un navegador para ver el applet. De su conformidad para ejecutar el applet cuando se le pregunte. Compruebe el log de la Consola de Java por errores y mensajes de depuración.

Descargar código fuente para el ejemplo Dynamic Tree Demo Applet para experimentar más.
