Escribiendo Diágnosticos a los Flujos de la Salida Estándar y Error

Un applet de Java puede escribir mensanes a los flujos de salida y error estándar. Escribir diagnósticos a la salida estándar puede ser una herramienta invaluable cuando está depurando un applet de Java.

El trozo de código siguiente escribe mensajes al flujo de salida estádar y al fluho de errores estándar. 

// Donde las variables de instancia son declaradas:
boolean DEBUG = true;
// ...
// Más tarde, cuando queremos imprimir algún estado:
if (DEBUG) {
    try {
        // ...
        //algún código que lanza una exceocion
        System.out.
            println("Llamado algunMetodo(" + x + "," + y + ")");
    } catch (Exception e) {
        e.printStackTrace()
    }
}

Compruebe el registro de la Consola de Java por los mensajes escritos al flujo de salida estándar o al flujo de errores estándar. Para almacenar mensajes en un fichero de registro, habilite el registro en el Panel de Control de Java. Los mensajes serán escritos en un fichero de registro en el directorio inicial del usuario (por ejemplo, en Windows, el fichero de registro podría estar en C:\Documents and Settings\someuser\Application Data\Sun\Java\Deployment\log).

Nota: Asegúrese de deshabilitar toda la salida de depuración antes de publicar su aplet.
