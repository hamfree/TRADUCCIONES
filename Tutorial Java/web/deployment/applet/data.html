Buscando y Cargando Ficheros de Datos

Cada vez que un applet de Java necesita cargar datos de un archivo que se especifica con una URL relativa (una URL que no especifica completamente la ubicación del fichero), el applet usualmente o el código base o el documento base para formar la URL completa.

El código base, devuelto por el método getCodeBase de JApplet, es una URL que especifica el directorio desde el cual serán cargadas ls clases de los applets. Para applets desplegados localmente, el método getCodeBase devuelve null.

El documento base, devuelto por el método getDocumentBase de JApplet, especifica el directorio de la página HTML que contiene el applet. Para applets desplegados localmente, el método getDocumentBase devuelve null.

A menos que la etiqueta <applet> especifique un código base, tanto el código base como el documento base se refieren al mismo directorio en el mismo servidor.

Los datos que el applet pudiera necesitar, o necesita confiar como copia de seguridad, son usualmente especificados relativo al código base. Los datos que el desarrollador de applet especifica, a menudo usando parámetros, es usualmente especificado relativo al documento base.

Nota: Por razones de seguridad, los navegadores limitan las URLs desde las cuales los applets no confiables pueden leer. Por ejemplo, la mayoría de los navegadores no permiten a los applets no confiables usar ".." para acceder a directorios por encima del código base o documento base. Además, ya que los applets no confiables no pueden leer fichero excepto aquellos ficheros que estén en el host originario del applet, el documento base no es generalmente útil si el documento y el applet no confiable residen en servidores diferentes.

La clase JApplet define formas convenientes de métodos de carga de imágenes y carga de sonidos que le habilitan a especificar imágenes y sonidos relativos a la URL base.
