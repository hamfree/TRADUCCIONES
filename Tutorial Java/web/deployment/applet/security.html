Qué Pueden Hacer los Applets y Qué No

Los applets de Java son cargados en un cliente cuando el usuario visita una página conteniendo un applet. El modelo de seguridad detrás de los applets de Java ha sido diseñado con el objetivo de proteger al usuario de los applets maliciosos.

Los applets son o applets de sandbox o applets privilegiados. Los applets de sandbox se ejecutan en un sandbox de seguridad que permite sólo un conjunto de operaciones seguras. Los applets privilegiados pueden ejecutarse fuera de la sandbox de seguridad y tienen amplias capacidades para acceder al cliente.

Los applets que no están firmados están restringidos a la sandbox de seguridad, y se ejecuta sólo si el usuario acepta el applet. Los applets que están firmados por un certificado desde una autoridad de certificados reconocida pueden ejecutarse en la sandbox, o pueden pedir permiso para ejecutarse fuera del sandbox. En cualquier caso, el usuario debe aceptar el certificado de seguridad del applet, de otra forma el applet será bloqueado.

Se recomienda que inicie su applet utilizando Java Network Launch Protocol (JNLP) para aprovechar las capacidades ampliadas y mejorar la experiencia del usuario. Vea Implantando un Applet para obtener las instrucciones paso a paso sobre el despliegue de Applet.

Se recomienda que despliegue sus applets en un servidor web, incluso para las pruebaas. Para ejecutar applets localmente, agrege los applets a la lista de excepciones de sitios, la cual es gestionada desde la pestaña de Seguridad del Panel de Control de Java.

En este tema describiremos las restricciones de seguridad y capacidades de los applets.

Applets de Sandbox

Los applets de Sandbox están restringidos a la sandbox de seguridad y pueden realizar las siguientes operaciones:

  • Pueden realizar conexiones de red a la máquina y puerto donde ellos estén. Los protocolos deben coincidir, y si se usa un nombre de dominio para cargar el applet, el nombre de dominio debe ser usado para conectar de vuelta a la máquina, no la dirección IP.
  • Pueden visualizar facilmente documentos HTML usando el método showDocument de la clase java.applet.AppletContext.
  • Pueden invocar métodos públicos de otros applets en la misma página.
  • Los applets que son cargados desde el sistema de ficheros local (desde un directorio en el CLASSPATH) no tienen ninguna de las restricciones que tienen los applets cargados de la red.
  • Pueden leer propiedades del sistema seguras. Vea Propiedades del Sistema para una lista de propiedades del sistema seguras.
  • Cuando son lanzados usando JNLP, los applets de sandbox pueden tambien realizar las operaciones siguientes:
    • Pueden abrir, leer, y salvar ficheros en el cliente.
    • Pueden acceder al portapapeles compartido de todo el sistema.
    • Pueden acceder a funciones de impresión.
    • Pueden almacenar datos en el cliente, decidir cómo los applets deberían ser descargados y cacheados, y mucho más. Vea JNLP IPA para más información sobre el desarrollo de applets usando JNLP IPA.

Los applets de Sandbox no pueden realizar las siguientes operaciones:

  • No pueden acceder a recursos del cliente tales como el sistema local de ficheros, ficheros ejecutables, el portapapeles del sistema, y las impresoras.
  • No pueden conectar o recuperar recursos de ningún servidor de terceras partes (cualquier servidor que no sea el servidor desde el cual se originó).
  • No pueden cargar librerías nativas.
  • No pueden cambiar el SecurityManager.
  • No pueden crear un ClassLoader.
  • No pueden leer ciertas propiedades del sistema. Vea Propiedades del Sistema para una lita de propiedades del sistema prohibidas.

Applets Privilegiados

Los applets privilegiados no tienen las restricciones de seguridad que son impuestas a los applets de sandbox y pueden ejecutarse fuera de la sandbox de seguridad.

Nota:  El código de JavaScript es tratado como código sin firmar. Cunado un applet privilegiado es accedido desde el código JavaScript en unan página HTML, el applet es ejecutado dentro de la sandbox de seguridad. Esto implica que el applet privilegiado se comporta esencialmente como un applet de sandbox.

Vea La Seguridad en Aplicaciones Ricas de Internet por información de cómo trabajar con applets.
