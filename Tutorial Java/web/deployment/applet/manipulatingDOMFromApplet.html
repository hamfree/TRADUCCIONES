Manipulando el DOM de la Página Web del Applet

Cada página web está compuesta de una serie de objetos anidados. Estos objetos componen el Modelo de Objetos del Documento (MOD, DOM, Document Object Model en inglés). Un applet de Java puede atravesar y modificar objetos de su página web padre usando la IPA Común del MOD.

Considere un ejemplo de un applet Java que vuelca los contenidos de su página web padre.

Para recorrer y manipular el árbol MOD, primero debe obtener una referencia al objeto Document para la página web. Puede hacerlo usando el método getDocument en la clase com.sun.java.browser.plugin2.DOM. Aquí tiene un trozo de código que recupera una referencia a un objeto Document en el método DOMDump start del applet. Vea los comentarios en línea en el código. 

public void start() {
    try {
        // use reflexión para obtener el documento
        Class c =
          Class.forName("com.sun.java.browser.plugin2.DOM");
        Method m = c.getMethod("getDocument",
          new Class[] { java.applet.Applet.class });
        
        // moldee o convierte el objeto devuelto como HTMLDocument;
        // entonces atraviese o modifique el MOD
        HTMLDocument doc = (HTMLDocument) m.invoke(null,
            new Object[] { this });
        HTMLBodyElement body =
            (HTMLBodyElement) doc.getBody();
        dump(body, INDENT);
    } catch (Exception e) {
        System.out.println("El nuevo Complemento de Java no está disponible");
        // En este caso, podría recurrir al viejo
        // mecanismo de arranque disponible en el
        // paquete com.sun.java.browser.plugin.dom
    }
}

Ahora que tiene una referencia al objeto Document, puede atravesar y modificar el árbol MOD usando la IPA Común del MOD. El applet DOMDump atraviesa el árbol MOD y escribe sus contenidos al registro de la Consola Java. 

private void dump(Node root, String prefix) {
    if (root instanceof Element) {
        System.out.println(prefix +
            ((Element) root).getTagName() + 
            " / " + root.getClass().getName());
    } else if (root instanceof CharacterData) {
        String data =
            ((CharacterData) root).getData().trim();
        if (!data.equals("")) {
            System.out.println(prefix +
                "CharacterData: " + data);
        }
    } else {
        System.out.println(prefix +
            root.getClass().getName());
    }
    NamedNodeMap attrs = root.getAttributes();
    if (attrs != null) {
        int len = attrs.getLength();
        for (int i = 0; i < len; i++) {
            Node attr = attrs.item(i);
            System.out.print(prefix + HALF_INDENT +
                "atributo " + i + ": " +
                attr.getNodeName());
            if (attr instanceof Attr) {
                System.out.print(" = " +
                    ((Attr) attr).getValue());
            }
            System.out.println();
        }
    }

    if (root.hasChildNodes()) {
        NodeList children = root.getChildNodes();
        if (children != null) {
            int len = children.getLength();
            for (int i = 0; i < len; i++) {
                dump(children.item(i), prefix +
                    INDENT);
            }
        }
    }
}

Abra AppletPage.html en un navegador para ver ejecutándose el applet DOMDump. Compruebe el registro de la Consola Java por el volcado del árbol MOD de la página web.

Nota:  Si no ve el applet ejecutarse, necesita instalar al menos la versión Java SE Development Kit (KDJ) 6 update 10.
Nota:  Si no ve el ejemplo ejecutándose, podría necesitar habilitar el intérprete de JavaScript en su navegador de forma que el script de las Herramientas de Despliegue pueda funcionar adecuadamente.
