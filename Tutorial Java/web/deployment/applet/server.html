Trabajando Con Aplicación del Lado del Servidor

Los applets de Java, como otros programas de Java, pueden usar la IPA definida en el paquete java.net para comunicarse a través de la red. Un applet de Java puede comunicarse con aplicaciones de servidor que se ejecutan en el mismo anfitrión que el applet. Esta comunicación no requiere ninguna configuración especial en el servidor.

Nota: Dependiendo del entorno de red en el que un applet esté cargado, y dependendiendo del navegador que ejecuta el applet, un applet podría ser incapaz de comunicar con su anfitrión origen. Por ejemplo, navegadores que se ejecutan en hosts dentro de cortafuegos a menudo no pueden obtener mucha información sobre el mundo fuera del cortafuegos. Como resultado, es posible que algunos navegadores no permitan la comunicación del applet a hosts fuera del firewall.

Cuando el applet es desplegado en un servidor web, use el método getCodeBase de Applet y el método getHost de java.net.URL para determinar de qué anfitrión vino el applet, como sigue: 

String host = getCodeBase().getHost();

Si el applet es desplegado localmente, el método getCodeBase devuelve null. El uso de un servidor web es recomendado.

Despues de que tenga el nombre correcto del anfitrión, usted puede usar todo el código de red que está documentado en el Recorrido Redes Personalizadas.

Nota: No todos los navegadores soportan todo el código de red impecablemente. Por ejemplo, un navegador ampliamente utilizado compatible con la tecnología Java no admite la publicación en una URL.

Para un ejemplo de implentar un applet que es un cliente de red, vea Ejemplo de Applet de Cliente de Red.
