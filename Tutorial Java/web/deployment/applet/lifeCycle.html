Ciclo de Vida de un Applet

Un applet puede reaccionar a los eventos principales de las siguientes maneras:

  • Puede inicializarse a sí mismo.
  • Puede empezar la ejecución.
  • Puede parar la ejecución.
  • Puede realizar una limpieza final, en preparación para ser descargado.

Esta sección introduce un nuevo applet, Simple, que usa todos estos métodos. A diferencia de las aplicaciones de Java, los applets no necesitan implementar un método main.

Aquí está el applet Simple.

Nota:  Si no ve la ejecución del ejemplo, podría necesitar habilitar el intérprete de JavaScript en su navegador de forma que el script del Kit de Herramientas de Despliegue pueda funcionar apropiadamente.

Lo siguiente es el código fuente para el applet Simple. Este applet visualiza una cadena descriptiva cuando encuentra un hito significativo en su vida, tal como cuando el usuario visita por primera vez la página donde se encuentra el applet. 



import java.applet.Applet;
import java.awt.Graphics;

//No hay necesidad de extender de JApplet, ya que no agregaremos ningún componente;
//tan solo pintamos.
public class Simple extends Applet {

    StringBuffer buffer;

    public void init() {
        buffer = new StringBuffer();
        addItem("inicializando... ");
    }

    public void start() {
        addItem("empezando... ");
    }

    public void stop() {
        addItem("parando... ");
    }

    public void destroy() {
        addItem("preparando para descargar...");
    }

    private void addItem(String newWord) {
        System.out.println(newWord);
        buffer.append(newWord);
        repaint();
    }

    public void paint(Graphics g) {
	//Dibuja un Rectángulo alrededor del area de visualización del applet.
        g.drawRect(0, 0, 
		   getWidth() - 1,
		   getHeight() - 1);

	//Dibuja la cadena actual dentro del rectángulo.
        g.drawString(buffer.toString(), 5, 15);
    }
}
Nota: En este ejemplo, la clase Applet es extendida, no la clase de Swing JApplet, ya que los componentes de Swing no necesitan ser agregados a este applet.

Cargando el Applet

Como resultado de la carga del applet, debería ver el texto "inicializando... empezando...". Cuando un applet es cargado, esto es lo que ocurre:

  • Una instancia de la clase controladora del applet (una subclase de Applet) se crea.
  • El applet se inicializa a sí mismo.
  • El applet empieza a ejecutarse.

Saliendo y Volviendo a la Página del Applet

Cuando el usuario sale de la página, por ejemplo, para ir a otra página, el navegado para y destruye el applet. El estado del applet no se preserva. Cuando el usuario vuelve a la página, el navegador inicializa y arranca una instancia nueva del applet.

Recargando el Applet

Cuando usted refresca o recarga una página en el navegador, la instancia actual del applet es parada y destruída y se crea una nueva instancia.

Saliendo del navegador

Cuando el usuario finaliza el navegador, el applet tiene la oportunidad de pararse por sí mismo y realizar una limpieza final antes del que navegador termine.

Descarge el código fuente para el ejemplo Simple Applet para experimentar más.
