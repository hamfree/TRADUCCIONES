Entorno de Ejecución de un Applet

Un applet de Java se ejecuta en el contexto de un navegador. El software del complemento de Java en el navegador controla el lanzamiento y la ejecución de los applets de Java. El navegador también tiene un intérprete de JavaScript, el cual ejecuta el código JavaScript de una página web. Este tema describe el comportamiento del software del Complemento de Java lanzando en la Plataforma Java, Edición Estándar 6 actualización 10.

Complemento de Java

El software del Complemento de Java crea un hilo de trabajo para cada applet de Java. Lanza un applet en una instancia del software del Entorno de Ejecución de Java (EEJ en inglés, EEJ en castellaño). Normalmente, todos los applets se ejecutan en la misma instancia del EEJ. El software del Complemento de Java inicia una nueva instancia del EEJ en los casos siguientes:

  • Cuando un applet solicita ser ejecutado en una versión específica del EEJ.
  • Cuando un applet especifica sus propios parámetros de inicio del EEJ, por ejemplo, el tamaño del montón. Un nuevo applet usa un EEJ existente si sus requerimientos son un subconjunto de una EEJ existente, de otras formas, una nueva instancia se inicia.

Un applet se ejecutará en una EEJ si las condiciones siguientes se cumplen:

  • La versión requerida del EEJ por el applet coincide con la de la EEJ existente.
  • Los parámetros de inicio de la EEJ satisfacen los requerimientos del applet.

El Complemento de Java y su Interacción con el Intérprete de JavaScript

Los applets de Java pueden invocar funciones de JavaScript presentes en la página web. Las funciones de JavaScript también pueden invocar métodos de un applet embebidos en la misma página web. El software del Complemento de Java y el intérprete de JavaScript organizan las llamadas desde código de Java a código de JavaScript y las llamadas desde código de JavaScript a código de Java.

El software del Complemento de Java es multi-hilo, mientras que el intérprete de JavaScript se ejecuta en un único hilo. Por lo tanto, para evitar problemas relacionados con los subprocesos, especialmente cuando se ejecutan varios applets simultáneamente, mantenga cortas las llamadas entre el código Java y el código JavaScript, y evite viajes de ida y vuelta, si es posible.
