Guía de Decisión para Aplicaciones Java Ricas de Internet

Tanto los applets como las aplicaciones Java Web Start son consideradas Aplicaciones Ricas de Internet (RIAs). Las RIAs preguntan a los usuarios por permiso para ejecutarse la primera vez que son lanzadas y estarán firmadas con un certificado de una autoridad de certificación de confianza. Evalúe las siguientes características de los applets y aplicaciones Java Web Star para decidir como implantar su RIA.

Applets

  • Los Applets se ejecutan en el contexto de un navegador.
  • Los Applets tienen acceso a las cookies de sesión y a las cookies persistentes.
  • Los Applets pueden interactuar con la página web donde están embebidos. los Applets pueden atravesar y manipular el Modelo de Objeto del Documento de la página web e interactuar con el JavaScript que exista en la página web. El código JavaScript puede acceder a los métodos públicos y variables de un applet.
  • Los Applets pueden ser lanzados utlizando el Protocolo de Lanzamiento en Red de Java (JNLP). Cuando son lanzados usando JNLP, a los applets de la caja de arena se les permite acceder al almacenamiento persistente, controlar la descarga, la E/S de ficheros, y más. Los Applets lanzados usando JNLP tienen capacidades que son comparables a aquellas de una aplicación Java Web Star.
    Los applets pueden también ser lanzados sin JNLP, y pueden ser restringidos a la caja de arean de seguridad o pedir todos los permisos.
  • Los applets pueden solicitar una versión particular del Software del Entorno de Ejecución de Java para su ejecución.

Aplicaciones Java Web Start

  • Las aplicaciones Java Web Start son lanzads desde una página web la primera vez. Java Web Start applications are launched from a web page the first time. Posteriormente, pueden ser re-lanzadas desde la página web o desde un acceso directo del escritorio.
  • Las aplicaciones Java Web Start no se ejecutan en el contexto de un navegador. Las aplicaciones no pueden interactuar con el HTML y el JavaScript de la página web y tiene sólo acceso a las cookies persistentes.
  • Se les permite a las aplicaciones Java Web Start el acceso al almacenamiento persistente, la E/S de ficheros, y otros servicios relacionados con el cliente.
  • Las aplicaciones Java Web Start pueden solicitar una versión particular del software del Entorno de Ejecución de Java para su ejecución.
