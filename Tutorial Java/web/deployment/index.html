Recorrido: Implantación

Las aplicaciones de Java ricas de internet (RIA) son aplicaciones que tienen rasgos similares a las aplicaciones de escritorio, pero son desplegadas vía internet. Las RIAs de Java pueden ser desarrolladas y desplegadas como applets de Java o aplicaciones de Java Web Start (Inicio Web de Java, JWS).

  • Applets - Los applets Java se ejecutan en el contexto de un navegador. El software Plug-in de Java controla la ejecución y el ciclo de vida de los applets de Java.
  • Aplicaciones de Java Web Start - Las aplicacoines de Java Web Star son lanzadas a través de un navegador la primera vez. Ellas pueden ser lanzadas posteriormente desde un enlace del escritorio. Una vez una aplicación Java Web Star es descargada y su certificado de seguridad ha sido aceptado por el usuario, se comporta casi como una aplicación independiente.

Arquitectura Basada en Componentes para RIAs

En el pasado, la decisión de si implantar un aplicación Java rica de internet dentro de un navegador como un applet, o fuera del navegasdor como una aplicación Java Web Star, podría impactar significativamente el diseño de la aplicación. Con el último Plug-in de Java, esta decisión ha sido gratamente simplificada.

Tradicionalmente, las aplicaciones construyen sus interfaces de usuario, incluyendo el Frame del nivel de arriba, en el método main. Este estilo de programación previene el facil re-despliegue de la aplicación en el navegador, porque asume que la aplicación crea su propio Frame. Cuando se ejecuta en el navegador como un applet, el applet es el contenedor de más alto nivel que mantendría el interfaz de usuario para la aplicación. No se necesita un Frame en el nivel de arriba.

Use la arquitectura basada en componentes cuando diseñe su aplicación Java rica de internet. Intente organizar su funcionalidad dentro de uno o más componentes que puedan ser compuestos juntos. En este contexto, el término "componente" se refiere a un elemenot del IGU que es una subclase de la clase Componente, la clase JComponent de Swing, u otra subclase. Por ejemplo, podría tener en el nivel de arriba la clase JPanel la cual contiene otros componentes de IU en él (como una combinación de más JPanels anidados y campos de texto, cajas combo, etc.). Con tal diseño, resulta relativamente sencillo implantar la funcionalidad principal como un applet o una aplicación Java Web Star.

Para desplegar un applet Java, usted solo necesita envolver la funcionalidad principal en un Applet o JApplet y añadir la funcionalidad especifica del navegador, si es necesario. Para implantar una aplicación Java Web Start, envuelva la funcionalidad en un JFrame.

Eligiendo Entre Apples de Java y Aplicaciones Java Web Start

La Guía de Decisión de Aplicaciones Ricas de Internet contiene información detallada para ayudarle a decidir si implantar su código como un applet de Java o una aplicación de Java Web Start.

La Alternativa de la Aplicación Auto-Contenida

Las aplicaciones Auto-contenidas proveen una opción de implantación que no requiere un navegador. Los usuarios instalan su aplicación localmente y las ejecutan de forma similar a las aplicaciones nativas. Las aplicaciones auto-contenidas incluyen el EEJ necesitado para ejecutar la aplicación, así los usuarios siempre tienen el EEJ correcto.

Este recorrido analiza el desarrollo y despliegue de RIAs y aplicacoines auto-contenidas. Vea This trail discusses the development and deployment of RIAs and self-contained applications.
