Fijando Argumentos Confiables y Propiedades Seguras

Puede establecer ciertos argumentos y propiedades seguras en la Máquina Virtual de Java para su aplicación rica de Internet (RIA) en el fichero Protocolo de Lanzamiento en Red de Java (JNLP) de la RIA. Para los applets, puede también establecer argumentos en el parámetro java_arguments de la etiqueta <applet>. Aunque hay un conjunto predefinido de propiedades seguras, usted puede también definir nuevas propiedades seguras prefijando el nombre de la propiedad con "jnlp." o "javaws.". Las propiedades pueden ser recuperadas en su RIA usando el método System.getProperty.

Considere las Propiedades y Argumentos del applet Demo. Los siguientes argumentos y propiedades de la Máquina Virtual de Java son establecidos en el fichero JNLP del applet appletpropsargs.jnlp.

  • -Xmx – Un argumento seguro que se establece igual a "256M"
  • sun.java2d.noddraw – Una propiedad segura predefinida establecida a true "true"
  • jnlp.myProperty – Una propiedad segura definida por el usuario que ese establece igual a "a user-defined property" 
<?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="" href="">
    <information>
        <title>Properties and Arguments Demo Applet</title>
        <vendor>Dynamic Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.6+"
              href="http://java.sun.com/products/autodl/j2se"
              <!-- secure java vm argument -->
              java-vm-args="-Xmx256M"/>
        <jar href="applet_PropertiesAndVMArgs.jar"
            main="true" />
            <!-- secure properties -->
        <property name="sun.java2d.noddraw"
            value="true"/>
        <property name="jnlp.myProperty"
            value="a user-defined property"/>
    </resources>
    <applet-desc 
         name="Properties and Arguments Demo Applet"
         main-class="PropertiesArgsDemoApplet"
         width="800"
         height="50">             
     </applet-desc>
     <update check="background"/>
</jnlp>

La clase PropertiesArgsDemoApplet usa el método System.getProperty para recuperar la propiedad java.version y otras propiedades que son establecidas en el fichero JNLP. La clase PropertiesArgsDemoApplet también muestra las propiedades. 


import javax.swing.JApplet;
import javax.swing.SwingUtilities;
import javax.swing.JLabel;

public class PropertiesArgsDemoApplet extends JApplet {
    public void init() {
        final String javaVersion = System.getProperty("java.version");
        final String  swing2dNoDrawProperty = System.getProperty("sun.java2d.noddraw");
        final String  jnlpMyProperty = System.getProperty("jnlp.myProperty");        

        try {
            SwingUtilities.invokeAndWait(new Runnable() {
                public void run() {
                    createGUI(javaVersion, swing2dNoDrawProperty, jnlpMyProperty);
                }
            });
        } catch (Exception e) {
            System.err.println("createGUI didn't successfully complete");
        }
    }
    private void createGUI(String javaVersion, String swing2dNoDrawProperty, String jnlpMyProperty) {
        String text = "Properties: java.version = " + javaVersion + 
                ",  sun.java2d.noddraw = " + swing2dNoDrawProperty +
                ",   jnlp.myProperty = " + jnlpMyProperty;
        JLabel lbl = new JLabel(text);
        add(lbl);
    }
}

El applet de la Demo de Propiedades y Argumentos es mostrado a continuación. Puede también ver el applet ejecutándose en AppletPage.html.

Nota:  Si no ve el applet ejecutándose, necesita instalar al menos la versión Java SE Development Kit (KDJ) 6 update 10.
Nota:  Si no ve el ejemplo ejecutándose, es posible que necesite habilitar el intérprete de JavaScript en su navegador para que el script de las Herramientas de Despliegue pueda funcionar adecuadamente.
