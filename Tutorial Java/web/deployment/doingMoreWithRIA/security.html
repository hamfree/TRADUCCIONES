La Seguridad en Aplicaciones Ricas de Internet

El modelo de seguridad detras de las aplicaciones ricas de Internet (ARIs) trabaja para protejer el usuario de aplicaciones maliciosas de Internet. Este tem describe los aspectos de seguridad que son comunes a applets y aplicaciones de Java Web Start. Vea los temas siguientes para más información:

  • Qué Pueden Hacer los Applets y Qué No
  • Java Web Start y la Seguridad

Las ARIs pueden ser restringidas en el entorno limitado de seguridad de Java o solicitar permiso para acceder a recursos fuera del entorno limitado. La primera vez que una ARI es lanzada, el usuario es preguntado pidiéndole permiso para ejecutarse. El diálogo mostrado ofrece información sobre el firmante del certificado e indica si la ARI pide permiso para ejecutarse fuera de la caja de arena. El usuario puede entonces tomar una decisión informada sobre la ejecución de la aplicación.

Aplique las siguientes pautas para ayudar a asegurar sus ARIs.

  • Firme el fichero JAR de la ARI con un certificado de una autoridad de certificados reconocida. Para más información, vea el tema Firmando y Verificando los Ficheros JAR.
  • Si la RIA requiere acceso fuera del entorno limitado de seguridad, especifique el elemento all-permissions en el fichero JNLP para la RIA. De lo contrario, deje que el RIA se ejecute de manera predeterminada en el entorno limitado de seguridad. El trozo de código siguiente muestra el elemento all-permissions en el fichero JNLP de la RIA. 
    <security>
   <all-permissions/>
</security>
    Si se usa la etiqueta applet, vea Implantando Con la Etiqueta Applet para obtener información sobre cómo establecer el nivel de permisos.
  • Un fichero JNLP puede sólo incluir ficheros JAR firmados por el mismo certificado. Si usted tiene ficheros JAR que están firmados usando certificados diferentes, específiquelos en ficheros JNLP separados. En el fichero JNLP principal de la ARI, especifique el elemento component-desc para incluir los otros ficheros JNLP como extensiones de componente. Vea Estructura del Fichero JNLP para obtener información.
  • El modelo de seguridad para ARIs no permite al código de JavaScript de una página web invocar código sensible a la seguridad en un fichero JAR firmado a menos que lo habilite explícitamente. En el fichero JAR firmado, ajuste la sección de código que usted quiera que el código de JavaScript sea capaz de invocar en un bloque AccessController.doPrivileged. Esto permite al código de JavaScript a ejecutarse con permisos elevados cuando ejecuta el código en un bloque de código doPrivileged.
  • Evite la mezcla de componentes privilegiados y limitados en una ARI, si es posible, ya que pueden disparar avisos de seguridad sobre código mezclado. Vea Mixing Privileged Code and Sandbox Code para más información.
  • Incluya los atributos Permissions y Codebase en el manifiesto del fichero JAR para asegurarse que su ARI pida sólo los permisos que usted especifica, y que la ARI es accedida desde la ubicación correcta. Vea JAR File Manifest Attributes for Security para más información.
  • Los atributos del manifiesto del fichero JAR le habilitan a restringir el acceso a su ARI y ayuda a asegurar que su código no sea alterado. Vea Mejorando la Seguridad con los Atributos del Manifiesto para obtener información de todos los atributos del manifiesto de fichero JAR que están disponibles.
