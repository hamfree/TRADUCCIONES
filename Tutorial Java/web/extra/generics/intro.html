Introducción

El KDJ 5.0 presenta varias extensiones nuevas al lenguaje de programación Java. Una de ellas es la introducción de los genéricos.

Este recorrido es una introducción a los genéricos. Es posible que esté familiarizado con construcciones similares de otros lenguajes, especialmente las plantillas de C ++. Si es así, verá que hay similaridades y diferencias importantes. Si no está familiarizado con las construcciones similares de otros lenguajes, mucho mejor; puede comenzar de nuevo, sin tener que desaprender conceptos erróneos.

Los genéricos le permiten abstraerse de los tipos. Los ejemplos más comunes son los tipos contenedor, como los de la jerarquía de Colecciones (Collections, en inglés).

Aquí hay un uso típico de ese tipo: 

List myIntList = new LinkedList(); // 1
myIntList.add(new Integer(0)); // 2
Integer x = (Integer) myIntList.iterator().next(); // 3

El moldeo (o conversión de tipo) en la línea 3 es un poco molesto. Normalmente, el programador sabe qué tipo de datos se han colocado en una lista particular. Sin embargo, el moldeo es esencial. El compilador puede sólo garantizar que un Object será devuelto por el iterador. Para asegurar que la asignación a una variable de tipo Integer sea de tipo seguro, se requiere la conversión de tipo (o moldeo).

Por supuesto, el moldeo (o conversión de tipo) no solo introduce desorden. También introduce la posibilidad de un error en tiempo de ejecución, ya que el programador puede estar equivocado.

¿Qué pasaría si los programadores pudieran expresar su intención y marcar una lista como restringida para contener un tipo de datos en particular? Esta es la idea central detrás de los genéricos. Aquí tiene una versión del fragmento de programa que se proporcionaba arriba usando genéricos: 

List<Integer> 
    myIntList = new LinkedList<Integer>(); // 1'
myIntList.add(new Integer(0)); // 2'
Integer x = myIntList.iterator().next(); // 3'

Observe la declaración de tipo para la variable myIntList. Esto especifica que esta no es sólo una List arbitraria, sino una List de Integer,escrito como List<Integer>. Decimos que esa List es un interfaz genérico que toma un parámetro de tipo--en este caso, Integer. También especificamos un parámetro de tipo al crear el objeto de lista.

Tenga en cuenta también que el moldeo en la línea 3 ya no está.

Ahora, puede pensar que todo lo que hemos logrado es mover el desorden. En vez de un moldeo a Integer en la línea 4, tenemos Integer como un parámetro de tipo en la línea 1. Sin embargo, hay una diferencia muy grande aquí. El compilador puede ahora comprobar la corrección de los tipos del programa en tiempo de compilación. Cuando decimos que myIntList es declarada con tipo List<Integer>, esto nos dice algo sobre la variable myIntList, lo cual es válido donde y cuando se use, y el compilador lo garantizará. En contraste, el moldeo nos indica algo que el programador piensa que es cierto en un sólo punto del código.

El efecto neto, especialmente en programas grandes, es la legibilidad y robustez mejoradas.
