Modo Exclusivo de Pantalla-Completa

Los programadores que usan la IPA DirectX de Microsoft pueden estar ya familiarizados con el modo exclusivo de pantalla completa. Otros programadores pueden ser algo nuevos en el concepto. En cualquier caso, el modo exclusivo de pantalla completa es una característica poderosa de la versión 1.4 de J2SE™ que permite al programador suspender el sistema de ventanas para que el dibujo se pueda hacer directamente en la pantalla.

Este es un ligero cambio de paradigma del tipo habitual de programa de IGU en muchos sentidos. En los programas tradicionales de IGU de Java, el AWT es responsable de propagar eventos de pintar desde el sistema operativo, a través del hilo de despacho de eventos, y llamando al método Component.paint de AWT cuando sea apropiado. En las aplicaciones exclusivas de pantalla completa, el pintado se realiza activamente por el propio programa. Adicionalmente, una aplicación IGU tradicional está limitada a la profundidad de bit y el tamaño de la pantalla elegida por el usuario. En una aplicación de pantalla completa exclusiva, el programa puede controlar la profundidad de bit y el tamaño (modo de pantalla) de la pantalla. Finalmente, muchas más técnicas avanzadas, tales como intercambio de páginas (descrita abajo) y el búfer estéreo (utilizando sistemas que usan un conjunto separado de marcos para cada ojo) requieren, en algunas plataformas, que una aplicación primero se encuentre el módo exclusivo de pantalla completa.

Conceptos básicos de imagen acelerada por hardware

Para entender la IPA del modo exclusivo de pantalla completa, necesita entender algunos principios básicos de las imágenes acelerados por el hardware. El interfaz VolatileImage encapsula una superficie la cual puede o no aprovechar las ventajas de la aceleración por hardware. Tales superficies pueden perder su aceleración hardware o su memoria a instancias del sistema operativo (de ahí el nombre 'volátil'). Vea el Tutorial VolatileImage (próximamente) para más información sobre las imágenes volátiles.

El modo exclusivo de pantalla completa se maneja a través de un objeto java.awt.GraphicsDevice. Para obtener una lista de todos los dispositivos gráficos de pantalla disponibles (en sistemas de uno o múltiples monitores), puede llamar al método getScreenDevices del java.awt.GraphicsEnvironment local; para la pantalla por defecto (primaria) (la única pantalla en un sistema de único monitor), puede llamar al método getDefaultScreenDevice.

Una vez que tiene el dispositivo gráfico, puede llamar a uno de los siguientes métodos:

  • public boolean isFullScreenSupported()
    Este método devuelve si está disponible o no el modo exclusivo de pantalla completa. En sistemas donde el modo exclusivo de pantalla completa no está disponible, es probablemente mejor ejecutar una aplicación en modo de ventana con un tamaño fijo que establecer una ventana de pantalla completa.
  • public void setFullScreenWindow(Window w)
    Dada una ventana, este método habilita el modo exclusivo de pantalla completa usando esa ventana. Si el modo exclusivo de pantalla completa no está disponible, la ventana se posiciona en (0,0) y se redimensiona para ocupar toda la pantalla. Use este método con un parámetro null para salir del modo exclusivo de pantalla completa.

Consejos de programación

Aquí hay algunos consejos sobre la programación usando el modo exclusivo de pantalla completa:

  • Verifique isFullScreenSupported antes de ingresar en el modo exclusivo de pantalla completa. Si no es compatible, el rendimiento puede verse degradado.
  • Entrar y salir del modo de pantalla completa es más robusto cuando se usa una cláusula try...finally. Esto es no sólo una buena práctica de codificación, sino que también evita que su programa permanezca en el modo exclusivo de pantalla completa más tiempo del que debería: 
    GraphicsDevice myDevice;
Window myWindow;

try {
    myDevice.setFullScreenWindow(myWindow);
    ...
} finally {
    myDevice.setFullScreenWindow(null);
}
  • La mayoría de las aplicaciones de pantalla completa exclusiva son más adecuadas para usar ventanas no decoradas. Desactive las decoraciones en el marco o diálogo usando el método setUndecorated.
  • Las aplicaciones de pantalla completa exclusiva no deberían ser redimensionable, ya que redimensionar una aplicación de pantalla completa puede causar un comportamiento impredecible (o posiblemente peligrosa).
  • Por razones de seguridad, el usuario debe otorgar el permiso fullScreenExclusive al usar el modo exclusivo de pantalla completa en un applet.
