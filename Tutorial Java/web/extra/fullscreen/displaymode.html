Modo de Pantalla

Una vez que una aplicación está en modo exclusivo de pantalla completa, puede aprovechar la configuración activa del modo de pantalla. Un modo de pantalla (java.awt.DisplayMode) está compuesto del tamaño ( ancho y alto del monitor, en pixels), profundidad de bit (número de bits por pixel), y la frecuencia de refresco (con qué frecuencia el monitor se actualiza a sí mismo). Algunos sistemas operativos le permiten usar varias profundidades de bit al mismo tiempo, en cuyo caso el valor especial BIT_DEPTH_MULTI se usa para el valor de la profundidad de bit. También, algunos sistemas operativos puede que no tengan ningún control sobre la frecuencia de refresco (o puede que no le importe la configuración de la frecuencia de actualización). En este caso, el valor especial REFRESH_RATE_UNKNOWN se usa para el valor de la frecuencia de actualización.

Cómo Establecer el Modo de Pantalla

Para obtener el modo de pantalla actual, llame simplemente al método getDisplayMode en su dispositivo gráfico. Para obtener una lista de todos los modos de pantalla posibles, llame al método getDisplayModes. Tanto getDisplayMode como getDisplayModes pueden llamarse en cualquier momento, independientemente de si se encuentra o no en modo exclusivo de pantalla completa.

Antes de intentar cambiar el modo de pantalla, primero debería llamar al método isDisplayChangeSupported. Si este método devuelve false, el sistema operativo no soporta cambiar a ese modo de pantalla.

Sólo se puede cambiar el modo de pantalla en el modo exclusivo de pantalla completa. Para cambiar el modo de pantalla, llame al método setDisplayMode con el modo de pantalla deseado. Se lanzará una excepción en tiempo de ejecución si el modo de pantalla no está disponible, si los cambios de modo de pantalla no se soportan, o si usted no está ejecutando la aplicación en el modo exclusivo de pantalla completo.

Razones para Cambiar el Modo de Pantalla

La razón principal para establecer el modo de pantalla es el rendimiento. Una aplicación puede ejecutarse mucho más rápidamente si las imágenes que elige mostrar comparten la misma profundidad de bits que la pantalla. Además, si puede contar con que la pantalla de un tamaño particular, hace que dibujar en esa pantalla sea mucho más simple, ya que no tiene que reducir o aumentar las cosas dependiendo de cómo el usuario haya configurado la pantalla.

Consejos de Programación

Aquí tiene algunos consejos para elegir y configurar el modo de pantalla:

  • Comprobar el valor devuelto por el método isDisplayChangeSupported antes de intentar cambiar el modo de pantalla en un dispositivo gráfico.
  • Asegúrese de que se encuentra en el modo exclusivo de pantalla completa antes de intentar cambiar el modo de pantalla.
  • Como con el uso del modo de pantalla completa, configurar el modo de pantalla es más robusto cuando se encuentre dentro de una cláusula try...finally: 
    GraphicsDevice myDevice;
Window myWindow;
DisplayMode newDisplayMode;
DisplayMode oldDisplayMode 
    = myDevice.getDisplayMode();

try {
    myDevice.setFullScreenWindow(myWindow);
    myDevice.setDisplayMode(newDisplayMode);
    ...
} finally {
    myDevice.setDisplayMode(oldDisplayMode);
    myDevice.setFullScreenWindow(null);
}
  • Cuando elige un modo de pantalla para su aplicación, es posible que quiera mantener una lista de modos de pantalla preferidos, y después elegir el mejor de la lista de modos de pantalla disponibles.
  • Como alternativa, si el modo de pantalla que desea no está disponible, puede simplemente ejecutarlo en el modo de ventanas con un tamaño fijo.
