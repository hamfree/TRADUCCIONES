Clock

La mayoría de los objetos basados en el tiempo suministran un método now() sin argumentos que ofrecen la fecha y hora actual usando el reloj del sistema y la zona horaria por defecto. Estos objetos basados en el tiempo también proveen un método now(Clock) con un argumento que le permite pasar un Clock alternativo.

La hora y fecha actual depende de la zona horaria y, para las aplicaciones globalizadas, un Clock es necesario para asegurar que la fecha/hora se crea con la zona horaria correcto. Así que, aunque el uso de la clase Clock es opcional, esta característica le permite comprobar su código para otras zonas horarias, o usar un reloj fijo, donde el tiempo no cambia.

La clase Clock es abstracta, así que no puede crear una instancia de ella. Los siguientes métodos factoría pueden ser útiles para probar.

  • Clock.offset(Clock, Duration) devuelve un clock que está compensado por Duration.
  • Clock.systemUTC() devuelve un reloj que representa la zona horaria de Greenwich / UTC.
  • Clock.fixed(Instant, ZoneId) siempre devuelve el mismo Instant. Para este reloj, el tiempo se detiene.
