Parseado y Formateado

Las clases basadas-en-el-tiempo en la IPA Fecha-Tiempo suministran métodos parse para analizar una cadena que contiene información sobre fecha y tiempo. Estas clases también suministran métodos format para dar formato para visualizar los objetos basados-en-el-tiempo. En ambos casos, el proceso es similar: usted ofrece una patrón a DateTimeFormatter para crear un objeto formatedor. Este formateador es entonces pasado al método parse o format.

La clase DateTimeFormatter ofrece numerosos formateadores predefinidos, o puede definir el suyo propio.

Los métodos parse y format lanzan una excepción si ocurre un problema durante el proceso de conversión. Por lo tanto, su código de análisis debería capturar el error DateTimeParseException y su código de formateo debería capturar el error DateTimeException. Para más información sobre el manejo de excepciones, vea Capturando y Manejando Excepciones.

La clase DateTimeFormatter es a la vez inmutable y segura para programación multihilo; puede (y debe) asignarse a una constante estática cuando sea apropiado.

Nota de Versión: Los objetos de fecha-tiempo java.time pueden ser usados directamente con java.util.Formatter y String.format usando el formato familiar basado en patrones que era usado con las clases heredadas java.util.Date y java.util.Calendar.

Analizar

El método de un argumento parse(CharSequence) en la clase LocalDate usa el formateador ISO_LOCAL_DATE. Para especificar un formateador diferente, puede usar el método de dos argumentos parse(CharSequence, DateTimeFormatter). El ejemplo siguiente usa el formateador predefinido BASIC_ISO_DATE, el cual sua el formato 19590709 para July 9, 1959. 

String in = ...;
LocalDate date = LocalDate.parse(in, DateTimeFormatter.BASIC_ISO_DATE);

Puede también definir un formateador usando su propio patrón. El código siguiente, del ejemplo Parse, crea un formateador que aplica un formato de "MMM d yyyy". Esto formato especifica tres caracteres para representar el mes, un dígito para representar el dia del mes, y cuatro dígitos para representar el año. Un formateador creado usando este patrón reconocería cadenas tales como "Jan 3 2003" o "Mar 23 1994". Sin embargo, para especificar el formato como "MMM dd yyyy", con dos caracteres para el día del mes, entonces tendría que usar siempre dos caracteres, rellenando con un cero para las fechas de-un-sólo-dígito: "Jun 03 2003". 

String input = ...;
try {
    DateTimeFormatter formatter =
                      DateTimeFormatter.ofPattern("MMM d yyyy");
    LocalDate date = LocalDate.parse(input, formatter);
    System.out.printf("%s%n", date);
}
catch (DateTimeParseException exc) {
    System.out.printf("%s no es analizable!%n", input);
    throw exc;      // Relanza la excepción.
}
// 'date' ha sido analizada exitosamente

La documentación para la clase DateTimeFormatter especifica la lista completa de símbolos que puede usar para especificar un patrón para formatear o analizar.

El ejemplo StringConverter en la página Conversión de Fechas No-ISO suministra otros ejemplos de un formateador de fechas.

Formatear

El método format(DateTimeFormatter) convierte un objeto basado-en-el-tiempo en una representación de cadena usando el formato especificado. El código siguiente, del ejemplo Flight, convierte una instancia de ZonedDateTime usando el formato "MMM d yyy hh:mm a". La fecha es definida en la misma forma que fue usada para el ejemplo previo de análisis, pero este patrón también incluye la hora, minutos, y componentes a.m. y p.m. 

ZoneId leavingZone = ...;
ZonedDateTime departure = ...;

try {
    DateTimeFormatter format = DateTimeFormatter.ofPattern("MMM d yyyy  hh:mm a");
    String out = departure.format(format);
    System.out.printf("SALIDA:  %s (%s)%n", out, leavingZone);
}
catch (DateTimeException exc) {
    System.out.printf("%s no puedo ser formateado!%n", departure);
    throw exc;
}

La salida para este ejemplo, que imprime tanto la hora de llegada como la de salida, es como sigue: 

SALIDA:  Jul 20 2013  07:30 PM (America/Los_Angeles)
LLEGADA: Jul 21 2013  10:20 PM (Asia/Tokyo)
