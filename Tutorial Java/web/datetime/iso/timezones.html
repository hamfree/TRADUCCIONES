Zonas horarias y Clases de Compensación

Una zona horaria es una región de la Tierra donde se usa la misma hora estándar. Cada zona horaria es descrita por un identificador y usualmente tiene el formato region/ciudad (Asia/Tokyo) y un desplazamiento de la hora de Greenwich/UTC. Por ejemplo, el desplazamiento para Tokio es +09:00.

ZoneId y ZoneOffset

La IPA de Fecha-Tiempo suministra dos clases para especificar una zona horaria o un desplazamiento:

  • ZoneId especifica un identificador de zona horaria y suministra reglas para convertir entre un Instant y un LocalDateTime.
  • ZoneOffset especifica un desplazamiento de zona horaria desde la hora de Greenwich/UTC.

Los desplazamientos de la hora Greenwich/UTC son habitualmente definidos por horas completas, pero hay excepciones. El código siguiente, del ejemplo TimeZoneId, imprime una lista de todas las zonas horarias que usan desplazamientos desde Greenwich/UTC que no están definidos en horas completas. 

Set<String> allZones = ZoneId.getAvailableZoneIds();
LocalDateTime dt = LocalDateTime.now();

// Crea una Lista usando el conjunto de zonas y las ordena.
List<String> zoneList = new ArrayList<String>(allZones);
Collections.sort(zoneList);

...

for (String s : zoneList) {
    ZoneId zone = ZoneId.of(s);
    ZonedDateTime zdt = dt.atZone(zone);
    ZoneOffset offset = zdt.getOffset();
    int secondsOfHour = offset.getTotalSeconds() % (60 * 60);
    String out = String.format("%35s %10s%n", zone, offset);

    // Escribe a la salida estándar sólo las zonas horarias que tienen un desplazamiento de hora completa.
    if (secondsOfHour != 0) {
        System.out.printf(out);
    }
    ...
}

Las Clases Date-Time

La IPA de Fecha-Tiempo suministra tres clases basadas-en-el-tiempo que funcionan con zonas horarias:

  • ZonedDateTime maneja una fecha y hora con su correspondiente zona horaria con el desplazamiento de la zona horaria desde Greenwich/UTC.
  • OffsetDateTime maneja una fecha y hora con su correspondiente desplazamiento de zona horaria desde Greenwich/UTC, sin un ID de zona horaria.
  • OffsetTime maneja la hora con su correspondiente desplazamiento de zona horaria desde Greenwich/UTC, sin un ID de zona horaria.

¿Cuando usaría OffsetDateTime en vez de ZonedDateTime? Si está escribiendo un software complejo que modela sus propias reglas para los cálculos de fecha y tiempo basados en ubicaciones geográficas, o si está almacenando sellos de tiempo en una base de datos que rastrea sólo los desplazamientos absolutos de la hora Greenwich/UTC, entonces podría querer utilizar OffsetDateTime. Además, el XML y otros formatos de red definen la transferencia de fecha-tiempo como OffsetDateTime o OffsetTime.

Aunque las tres clases mantienen un desplazamiento de la hora Greenwich/UTC. sólo ZonedDateTime usa la parte ZoneRules, del paquete java.time.zone, para determinar cómo un desplazamiento varia en una zona horaria en particular. Por ejemplo, la mayoría de las zonas horarias experimentan un desfase (típicamente de 1 hora) al adelantar el reloj al horario de verano, y una superposición de tiempo al mover el reloj a la hora estándar y la última hora antes de que se repita la transición. La clase ZonedDateTime se acomoda a este escenario, mientras que las clases OffsetDateTime y OffsetTime, que no tienen acceso a las ZoneRules, no lo hacen.

ZonedDateTime

La clase ZonedDateTime, en efecto, combina la clase LocalDateTime con la clase ZoneId. Es usada para representar una fecha completa (año, mes, día) y una hora (hora, minuto, segundo, nanosegundo) con una zona horaria (región/ciudad, tal como Europe/Paris).

El código siguiente, del ejemplo Flight, define la hora de salida para un vuelo desde San Francisco a Tokyo como un ZonedDateTime en la zona horaria de America/Los Angeles. Los métodos withZoneSameInstant y plusMinutes son usados para crear una instancia de ZonedDateTime que representa el tiempo proyectado de llegada en Tokyo, después de un vuelo de 650 minutos. El método ZoneRules.isDaylightSavings determina si es horario de verano cuando el vuelo llega a Tokyo.

Un objeto DateTimeFormatter se usa para formatear las instancias ZonedDateTime para imprimir: 

DateTimeFormatter format = DateTimeFormatter.ofPattern("MMM d yyyy  hh:mm a");

// Saliendo de San Francisco el 20 de julio, 2013, a las 7:30 de la tarde
LocalDateTime leaving = LocalDateTime.of(2013, Month.JULY, 20, 19, 30);
ZoneId leavingZone = ZoneId.of("America/Los_Angeles"); 
ZonedDateTime departure = ZonedDateTime.of(leaving, leavingZone);

try {
    String out1 = departure.format(format);
    System.out.printf("SALIDA:  %s (%s)%n", out1, leavingZone);
} catch (DateTimeException exc) {
    System.out.printf("¡%s no pudo ser formateado!%n", departure);
    throw exc;
}

// El vuelo es de 10 horas y 50 minutos, o 650 minutos
ZoneId arrivingZone = ZoneId.of("Asia/Tokyo"); 
ZonedDateTime arrival = departure.withZoneSameInstant(arrivingZone)
                                 .plusMinutes(650);

try {
    String out2 = arrival.format(format);
    System.out.printf("LLEGADA: %s (%s)%n", out2, arrivingZone);
} catch (DateTimeException exc) {
    System.out.printf("¡%s no pudo ser formateado!%n", arrival);
    throw exc;
}

if (arrivingZone.getRules().isDaylightSavings(arrival.toInstant())) 
    System.out.printf("  (%s el horario de verano estará vigente).%n",
                      arrivingZone);
else
    System.out.printf("  (%s el horario estándar estará vigente).%n",
                      arrivingZone);

Esto produce la salida siguiente: 

SALIDA:  Jul 20 2013  07:30 PM (America/Los_Angeles)
LLEGADA: Jul 21 2013  10:20 PM (Asia/Tokyo)
  (Asia/Tokyo el horario estándar estará vigente).

OffsetDateTime

LA clase OffsetDateTime, en efecto, combina la clase LocalDateTime con la clase ZoneOffset. Es usada para representar una fecha completa (año, mes, día) y una hora (hora, minuto, segundo, nanosegundo) con un desplazamiento de la hora Greenwich/UTC (+/-horas:minutos, tales como +06:00 o -08:00).

El ejemplo siguiente usa OffsetDateTime con el método TemporalAdjuster.lastDay para encontrar el último jueves de julio de 2013. 

// Encontrar el último jueves de julio de 2013.
LocalDateTime localDate = LocalDateTime.of(2013, Month.JULY, 20, 19, 30);
ZoneOffset offset = ZoneOffset.of("-08:00");

OffsetDateTime offsetDate = OffsetDateTime.of(localDate, offset);
OffsetDateTime lastThursday =
        offsetDate.with(TemporalAdjusters.lastInMonth(DayOfWeek.THURSDAY));
System.out.printf("El último jueves de Julio de 2013 es el %sth.%n",
                   lastThursday.getDayOfMonth());

La salida de ejecutar este código es: 

El último jueves de Julio de 2013 es el 25th.

OffsetTime

La clase OffsetTime, en efecto, combina la clase LocalTime con la clase ZoneOffset. Es usada para representar la hora (hora, minuto, segundo, nanosegundo) con un desplazamiento de la hora de Greenwich/UTC (+/-hours:minutes, such as +06:00 or -08:00). It is used to represent time (hour, minute, second, nanosecond) with an offset from Greenwich/UTC time (+/-horas:minutos, tales como +06:00 o -08:00).

La clase OffsetTime es usada en las mismas situaciones que la clase OffsetDateTime, pero cuando el seguimiento de la fecha no es necesario.
