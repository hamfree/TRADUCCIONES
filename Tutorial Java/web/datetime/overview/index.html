Lección: Introducción de Fecha-Hora

El tiempo parece ser un tema simple; incluso un reloj económico puede proporcionar una fecha y hora razonablemente precisas. Sin embargo, con un examen más detallado, se da cuenta de las complejidades sutiles y los muchos factores que afectan a su comprensión del tiempo. Por ejemplo, el resultado de agregar un mes al 31 de Enero es diferente para un año bisiesto que para el resto de años. Las zonas de tiempo también agregan complejidad. Por ejemplo, un país puede entrar y salir del horario de verano a corto plazo, o más de una vez en un año o puede omitir el horario de verano por completo durante un año determinado.

La IPA de Fecha y Hora usa el sistema de calendario definido en ISO-8601 como el calendario por defecto. Este calendario está basado en el sistema del calendario Gregoriano y es usado globalmente como el estándar defacto para representar la fecha y la hora. Las clases principales en la IPA de Fecha y Hora tienen nombres tales como LocalDateTime, ZonedDateTime, y OffsetDateTime. Todos estos usan el sistemea de calendario ISO. Si quiere usar un sistema de calendario alternativo, tal como el Hijrah o el Thai Buddhist, el paquete java.time.chrono le permite usar uno de los sistemas de calendario predefinidos. O puede crear el suyo propio.

La IPA de Fecha y Hora usa el Repositorio de datos de configuración regional común de Unicode (CLDR es su acrónimo en inglés) . Este repositorio admite los idiomas del mundo y contiene la colección de datos de configuración regional más grande del mundo disponible. La información en este repositorio ha sido traducida a cientos de idiomas. La IPA de Fecha y Tiempo también usa la Base de datos de Zonas Horarias (TZDB es su acrónimo en inglés) . Esta base de datos proporciona información sobre cada cambio de zona horaria a nivel mundial desde 1970, con un historial para zonas horarias primarias desde que se introdujo el concepto.
