Principios de Diseño de Fecha-Hora

La IPA de Fecha y Tiempo fue desarrollada usando varios principios de diseño.

Claro

Los métodos en la IPA están bien defnidos y su comportamiento es claro y esperado. Por ejmplo, invocar un método de Fecha y Hora con un parámetro null típicamente dispara una NullPointerException.

Fluído

La IPA de Fecha y Hora suministra un interfaz fluído, haciendo el código más fácil de leer. Ya que la mayoría de los métodos no permiten parámetros con entrada null y no devuelven un valor null, las llamadas a los métodos pueden ser encadenadas juntas y el código resultante puede ser entendido rápidamente. Por ejemplo: 

LocalDate today = LocalDate.now();
LocalDate payday = today.with(TemporalAdjusters.lastDayOfMonth()).minusDays(2);

Inmutable

La mayoría de las clases en la IPA de Fecha y Tiempo crean objetos que son inmutables, significa que, después de que el objeto es creado, ya no puede ser modificado. Para alterar el valor de un objeto inmutable, se debe construir un objeto nuevo como una copia modificada del original. Esto también significa que la IPA de Fecha y Hora es, por definición, segura para la programación con multihilos. Esto afecta a la IPA en que la mayoría de métodos usados para crear objetos de fecha y hora tienen el prefijo of, from, o with, en lugar de constructores, y no hay métodos set. Por ejemplo: 

LocalDate dateOfBirth = LocalDate.of(2012, Month.MAY, 14);
LocalDate firstBirthday = dateOfBirth.plusYears(1);

Extensible

La IPA de Fecha y Tiempo es extensible siempre que sea posible. Por ejemplo, puede definir sus propios ajustadores de tiempo y consultas, o construir su propio sistema de calendario.
