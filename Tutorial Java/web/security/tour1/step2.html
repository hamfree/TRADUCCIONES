Estableciendo un Fichero de Políticas para Conceder los Permisos Requeridos

Un archivo de políticas es un archivo de texto ASCII y puede ser compuesto a través de un editor de texto o con la utilidad gráfica Herramienta de Políticas demostrada en esta sección. La Herramienta de Políticas le ahorra tecleo y elimina la necesidad de que sepa la sintaxis requeridad de los archivos de políticas, reduciendo así los errores.

Esta lección usa la Herramienta de Políticas para crear un archivo de políticas llamado examplepolicy, en el que usted agregará una entrada de política que otorga al código del directorio examples permiso para escribir.

Siga estos pasos para crear y modificar su nuevo archivo de políticas:

  1. Inicie la Herramienta de Políticas
  2. Conceder el Permiso Requrido
  3. Salvar el fichero de Politícas
Note para Usuarios de UNIX: Los pasos ilustran la creación del fichero de políticas para un sistema Windows. Los pasos son exactamente los mismos si usted está trabajando en un sistema UNIX. Donde el texto dice almacenar el fichero de políticas en el directorio C:\Test, puede almacenarlo en otro directorio. Los ejemplos en la lección Visita Rápida para Controlar Aplicaciones asumen que usted lo almacenó en el directorio ~/test.
