Conceder el Permiso Requrido

Para crear una nueva entrada, pulse el botón Agregar Entrada de Política en la ventana principal de la Herramienta de Políticas. Esto visualiza el cuadro de diálogo de Entrada de Política como se muestra en la siguiente figura.

Una entrada de política especifica uno o más permisos para el código de un código fuente particular - código de una ubicación particular (URL), código firmado por una entidad particular, o ambos.

Las cajas de texto CodeBase y SignedBy especifican a qué código quiere otorgar el(los) permiso(s) que serán agregados en el archivo.

  • Un valor CodeBase indica la ubicación del código fuente; usted otorga el(los) permiso(s) al código desde esa ubicación. Una entrada CodeBase vacía significa "cualquier código" -- no importa dónde se origine el código.
  • Un valor SignedBy indica el alias para un certificado almacenado en un almacén de claves. La clave pública dentro de ese certificado se usa para verificar la firma digital en el código. Usted otorga el permiso al código firmado por la clave privada correspondiente a la clave pública en el almacén de claves especificado por el alias. La entrada SignedBy es opcional; omitirla significa "cualquier firmante" -- no importa si el código se firma, o por quién.

Si usted tiene una entrada CodeBase y una entrada SignedBy, el(los) permiso(s) se otorgan solo al código que está en la ubicación especificadad y firmado por el alias nombrado.

Puede otorgar permiso a todo el código de la ubicación (URL) donde se almacenan los ejemplos.

Teclee la siguietne URL dentro de la caja de texto CodeBase del cuadro de diálogo Entrada de Política: 

https://docs.oracle.com/javase/tutorial/security/tour1/examples/

Nota: Esto es una URL. Por lo tanto, siempre debe usar barras inclinadas como separadores, no barras invertidas.

Deje la caja de texto SignedBy en blanco, ya que no se le requiere que el código esté firmado.

Nota: Para otorgar permiso a cualquier código (archivo .class) no solo del directorio especificado anteriormente sino del directorio security y sus subdirectorios, teclee la siguiente URL dentro de la caja CodeBase box: 
https://docs.oracle.com/javase/tutorial/security/

Ha especificado de dónde proviene el código (el CodeBase), y que el código no tiene que estar firmado (ya que no hay ningún valor SignedBy).

Ya ha especificado esta entrada de política, así que haga click en el botón Hecho en el diálogo Entrada de Política. La ventana de Herramienta de Políticas ahora contiene una línea representando la entrada de política, mostrando el valor de CodeBase.
