Añadir una Entrada de Política con un Alias SignedBy

Para otorgar al código firmado por susan permiso para leer cualquier fichero en el directorio C:\TestData, necesita crear una entrada de política otorgando este permiso. Tenga en cuenta que "Código firmado por susan" es una forma abreviada de decir "Código en un archivo class contenido en un archivo JAR, donde el archivo JAR fue firmado usando la clave privada correspondiente a la clave pública que aparece en un certificado de un almacén de claves en una entrada con el alias susan."

Elija el botón Add Policy Entry en la ventana principal de la Herramienta de Políticas. Esto mostrará el cuadro de diálogo de Entrada de Política:

Usando este cuadro de diálogo, teclee el siguiente alias dentro de la caja de texto SignedBy: 

susan

Deje la caja de texto de CodeBase en blanco, para otorgar a todo el código firmado por susan el permiso, sin importar de dónde venga.

Nota: Si quería restringir el permiso a solo el código firmado por susan que está en el directorio C:\Test\, debería teclear la URL siguiente en la caja de texto CodeBase: 
file:/C:/Test/*

Para agregar el permiso, elija el botón Add Permission. Esto muestra el cuadro de diálogo Permisos.

Haga lo siguiente.

  1. Elija File Permission de la lista desplegable de Permiso. El nombre del tipo de permiso completo (java.io.FilePermission) aparece ahora en la caja de texto a la derecha de la lista desplegable.
  2. Teclee lo siguiente en la caja de texto a la derecha de la lista etiquetada Target Name para especificar todos los ficheros en el directorio C:\TestData\: 
    C:\TestData\*
  3. Especifique acceso de lectura eligiendo la opción read de la lista desplegable de Acciones.

Ahora el cuadro de diálogo de Permisos se parece a lo siguiente.

Elija el botón OK. El nuevo permiso aparece en una línea en el diálogo de Entrada de Política, a continuación.

Nota: Cada barra invertida en la ruta de ficheros que tecleó ha sido reemplazada con dos barras invertidas, para su conveniencia. Las cadenas en un archivo de políticas son procesadas por un tokenizador que permite que se use \ como un carácter de escape (por ejemplo, \n para indicar una nueva línea), así que el archivo de políticas requiere dos barras invertidas para indicar una única barra invertida. Si usa barras invertidas únicas como sus separadores de directorio, la Herramienta de Políticas automáticamente las conviernte en dobles barras invertidas por usted.

Ahora que ha terminado de especificar esta entrada de política, elija el botón Done en el diálogo de Entrada de Política. La ventana de Herramienta de Políticas ahora contiene una línea representando la entrada de política, mostrando el valor SignedBy.
