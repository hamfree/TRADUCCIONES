Lección: Firmando Código y Dándole Permisos

Esta lección le muestra cómo usar keytool, jarsigner, Policy Tool y jar para colocar archivos dentro de archivos JAR (Nota del Traductor: Java ARchive, en inglés) para su posterior firma por la herramienta jarsigner.

Esta lección tiene dos partes. En la primera, creará y desplegará una aplicación. En la segunda; actuará como el remitente de una aplicación firmada.

Aquí tiene los pasos para crear y desplegar una aplicación:

Nota:  Por conveniencia, pretenderá ser una usuaria/desarrolladora llamada Susan Jones. Necesita definir a Susan Jones cuando genere las claves.

  • Coloque los archivos class de Java que componen su aplicación en un archivo JAR
  • Firme el archivo JAR
  • Exporte el certificado de clave pública correspondiente a la clave privada usada para firmar el archivo JAR

Aquí tiene los pasos para otorgar permisos a una aplicación

Nota:  Por conveniencia, usted pretenderá ser un usuario llamado Ray.

  • Verá cómo la aplicación firmada normalmente no puede leer un archivo cuando se ejecuta bajo un administrador de seguridad.
  • Use keytool para importar un certificado dentro del almacén de claves de Ray en una entrada con el alias de susan
  • Use la Herramienta de Políticas para crear una entrada en el archivo de políticas de Ray para permitir al código firmado por susan leer el fichero especificado.
  • Finalmente, verá como la aplicación que se ejecuta bajo un gestor de seguridad puede ahora leer el fichero, ya que se le ha otorgado permiso para hacerlo.

Para más información sobre las firmas digitales, certificados, almacenes de claves, y las herramientas, vea la lección IPA y Herramientas usadas para Código Seguro e Intercambio de Ficheros.

Importante: Necesita realizar las tareas en esta lección mientras trabaja en el directorio en el que almacena la aplicación de ejemplo, pero debe almacenar el fichero de datos necesitado por la aplicación en un directorio diferente. Todos los ejemplos en este recorrido asumen que usted está trabajando en el directorio C:\Test, y que el fichero de datos están en el directorio C:\TestData.

Si está trabajando en un sistema UNIX, substituya sus propios nombres de directorio.
