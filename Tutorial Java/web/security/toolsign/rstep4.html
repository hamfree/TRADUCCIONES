Ver los Efectos del Fichero de Políticas

En los pasos anteriores creó una entrada en el archivo de políticas exampleraypolicy otorgando al código firmado por susan permiso para leer fichero del directorio C:\TestData\ (o del directorio testdata en su directorio de inicio si está trabajando en UNIX). Ahora debe ser capaz de ejecutar con éxito el programa Count para leer y contar los caracteres de un fichero del directorio especificado, incluso cuando ejecute la aplicación con un gestor de seguridad.

Como se describió al final de la lección Crear un Archivo de Políticas, hay dos posibles formas en que puede hacer que el archivo exampleraypolicy sea considerado como parte de la política general, además de los archivos de políticas especificados en el archivo de propiedades de seguridad. La primera aproximación es especificar el archivo de políticas adicional en una propiedad pasada al sistema de tiempo de ejecución. La segunda aproximación es agregar una línea en el archivo de propiedades de seguridad especificando el archivo de políticas adicional.

Aproximación 1

Puede usar el argumento de línea de comandos -Djava.security.policy para especificar un archivo de políticas que debe ser utilizado además de o en vez de los especificados en el archivo de propiedades de seguridad.

Para ejecutar la aplicación Count y tener el archivo de políticas exampleraypolicy incluído, teclee lo siguiente mientras está en el directorio que contiene los archivos sCount.jar y exampleraypolicy: 

java -Djava.security.manager
    -Djava.security.policy=exampleraypolicy
    -cp sCount.jar Count C:\TestData\data

Nota: Teclee el comando en una única línea, con un espacio antes de -D y -cp.

El programa debe informar del número de caracteres en el archivo especificado.

Si aún informa de un error, algo está equivocado en el archivo de políticas. Use la Herramienta de Políticas para comprobar el permiso que acaba de crear en el paso previo, y cambie cualquier error tipográfico u otros errores.

Aproximación 2

Puede especificar una variedad de URLs -- incluyendo aquellas de la forma "http://" -- en las propiedades policy.url.n en el archivo de propiedades de seguridad, y todos los archivos de politicas designados serán cargados.

Así una forma de que sean consideradas por el intérprete sus entradas de política de su archivo exampleraypolicy es agregar una entrada indicando ese fichero en el archivo de propiedades de seguridad.

Importante: Si está ejecutando su propia copia del KDJ, puede fácilmente editar su archivo de propiedades de seguridad. Si está ejecutando una versión compartida con otros, puede ser solo capaz de modificar el archivo de propiedades de seguridad de todo el sistema si tiene acceso de escritura a este o si pide a su administrador de sistemas que modifique el archivo cuando sea apropiado. Sin embargo, probablemente no es apropiado para usted hacer modificaciones a un archivo de políticas de todo el sistema para esta prueba del tutorial; sugerimos que simplemente lea lo siguiente para ver cómo se hace o que instale su propia versión privada del KDJ para usarla para las lecciones del tutorial.

El archivo de propiedades de seguridad está ubicado en

  • Windows: java.home\lib\security\java.security
  • UNIX: java.home/lib/security/java.security

La porción java.home indica el directorio en el que fue instalado el EEJ.

Para modificar el archivo de propiedades de seguridad, ábralo con un editor capaz de editar un fichero de texto ASCII. Después agregue la línea siguiente despúes de la línea que empieza con policy.url.2:

  • Windows: policy.url.3=file:/C:/Test/exampleraypolicy
  • UNIX: policy.url.3=file:${user.home}/test/exampleraypolicy

En un sistema UNIX puede alternativamente especificar explícitamente su directorio de inicio, como en 

policy.url.3=file:/home/susanj/test/exampleraypolicy

A continuación, en su ventana de comandos, vaya al directorio que contiene el archivo sCount.jar, es decir, C:\Test o el directorio ~/test. Teclee el comando siguiente en una línea: 

java -Djava.security.manager
        -cp sCount.jar Count C:\TestData\data

Como con la aproximación 1, si el programa aún informa de un error, algo está equivocado en el archivo de políticas. Use la Herramienta de Políticas para comprobar el permiso que acaba de crear en el paso previo, y cambie cualquier error tipográfico u otros errores.

Importante: Antes de continuar, puede querer eliminar la línea que acaba de agregar en el archivo de propiedades de seguridad (o comentarla), ya que probablemente no quiera que el archivo exampleraypolicy se incluya cuando no está ejecutando las lecciones del tutorial.
