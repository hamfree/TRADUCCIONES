Observar la Aplicación Restringida

La última parte de la lección Visita Rápida para Controlar Aplicaciones le muestra cómo una aplicación puede ser ejecutada bajo un gestor de seguridad invocando al intérprete con el nuevo argumento de línea de comandos -Djava.security.manager. Pero ¿qué pasa si la aplicación que se invoca reside dentro de un archivo JAR?

Una de las opciones del intérprete es la opción -cp (por class path), que le permite especificar una ruta de búsqueda para las clases y recursos de la aplicación. Por lo tanto, para ejecutar la aplicación Count dentro de un archivo JAR sCount.jar, especificando el fichero C:\TestData\data como su argumento, puede teclear el comando siguiente mientras está en el directorio que contiene sCount.jar: 

java -cp sCount.jar Count C:\TestData\data

Para ejecutar la aplicación con un gestor de seguridad, agregue -Djava.security.manager, como se muestra debajo: 

java -Djava.security.manager -cp sCount.jar Count C:\TestData\data

Importante:  Cuando ejecuta este comando, su intérprete Java lanzará una excepción que se muestra debajo: 

Exception in thread "main" java.security.AccessControlException:
access denied (java.io.FilePermission C:\TestData\data read)
    at java.security.AccessControlContext.checkPermission(Compiled Code)
    at java.security.AccessController.checkPermission(Compiled Code)
    at java.lang.SecurityManager.checkPermission(Compiled Code)
    at java.lang.SecurityManager.checkRead(Compiled Code)
    at java.io.FileInputStream.<init>(Compiled Code)
    at Count.main(Compiled Code)

En este ejemplo, la AccessControlException informó que la aplicación count no tiene permiso para leer el fichero C:\TestData\data. Su intérprete lanzó esta excepción porque no permitirá que cualquier aplicación ejecutándose bajo un gestor de seguridad lea un fichero o acceda a otros recursos a menos que tenga el permiso explícito para hacer eso - usualmente especificado en una sentencia grant contenido en un archivo de políticas.
