Lección: Generando y Verificando Firmas

Esta lección le lleva a través de los pasos necesarios para usar la IPA de Seguridad del KDJ para generar una firma digital para datos y para verificar que una firma es auténtica. Esta lección está pensada para desarrolladores que desan incorporar funcionalidad de seguridad en sus programas, incluyendo servicios criptográficos.

Esta lección demuestra el uso de la IPA de Seguridad del KDJ con respecto al firmado de documentos. La lección muestra que un programa, ejecutado por la persona que tiene el documento original, debería generar claves, generar una firma digital para el documento usando la clave privada, y exportar la clave pública y la firma a los archivos.

Después muestra un ejemplo de otro programa, ejecutado por el receptor del documento, firma, y clave pública. Muestra cómo el programa podría importar la clave pública y verificar la autenticidad de la firma. La lección también describe y demuestra posibles aproximaciones alternativas y métodos de proporcionar e importar claves, incluso en certificados.

Para información posterior sobre los conceptos y terminología (firmas digitales, certificados, almacenes de claves), vea la lección IPA y Herramientas usadas para Código Seguro e Intercambio de Ficheros.

En esta lección crea dos aplicaciones básicas, una para la generación de firma digital y la otra para la verificación. A esto le sigue una discusión y demostración de posibles mejoras.La lección contiene tres secciones.

  • Generando una Firma Digital muestra cómo usar la IPA para generar claves y una firma digital para datos usando la clave privada y para exportar la clave pública y la firma a archivos. La aplicación obtiene el nombre del archivo de datos de la línea de comandos.

  • Verificando una Signatura Digital muestra como usar la IPA para importar una clave pública y una firma que supuestamente es la firma de un archivo de datos específico y para verificar la autoridad de la firma. Los datos, la clave pública, y los nombres del archivo de firma se especifican en la línea de comandos.

  • Debilidades y Alternativas analiza las posibles debilidades del enfoque utilizado por los programas básicos. Luego presenta y demuestra posibles enfoques y métodos alternativos para suministrar e importar claves, incluido el uso de archivos que contienen bytes de clave codificados y el uso de certificados que contienen claves públicas.
