Lección: Implementando su Propio Permiso

Esta lección demuestra cómo escribir una clase que define su propio permiso especial. Los componentes básicos en esta lección incluyen:

  1. Un juego de ejemplo llamado ExampleGame.
  2. Una clase llamada HighScore, que es utilizada por ExampleGame para almacenar la última puntuación más alta de un usuario.
  3. Un clase llamada HighScorePermission, que se usa para proteger el acceso al valor de la puntuación más alta almacenada del usuario.
  4. Un archivo de políticas de seguridad del usuario, que otorga permiso a ExampleGame para actualizar la puntuación más alta del usuario/a.

El escenario básico es como sigue:

  1. Un usuario juega con ExampleGame.
  2. Si el usuario alcanza una nueva puntuación alta, ExampleGame usa la clase HighScore para salvar este nuevo valor.
  3. La clase HighScore busca en la política de seguridad del usuario para comprobar si ExampleGame tiene permiso para actualizar el valor de puntuación alta del usuario.
  4. Si ExampleGame tiene permiso para actualizar la puntuación alta, entonces la clase HighScore actualiza ese valor.
