Pasos para el Desarrollador de ExampleGame (Terry)

Los pasos que Terry debería tomar, después de crear un juego (ExampleGame ) que llama a los métodos getHighScore y setHighScore de HighScore para obtener y establecer, respectivamente, las puntuaciones más altas del usuario, son:

Compilar la Clase del Juego 

javac ExampleGame.java -classpath hs.jar -d .

Colocar su archivo class en un Archivo JAR 

jar cvf terry.jar com/gamedev/games/ExampleGame.class

Crear un Almacén de Claves y Claves para Firmar 

keytool -genkey -keystore terry.keystore -alias signTJars

Especifique lo que desee para las contraseñas y la información del nombre distinguido.

Firmar el Fichero JAR 

jarsigner -keystore terry.keystore terry.jar signTJars

Exportar el Certificado de Clave Pública 

keytool -export -keystore terry.keystore
    -alias signTJars -file Terry.cer

Proporcionar Archivos e Información Necesitada por los Usuarios

Es decir, proporcionarles

  • el archivo JAR firmado terry.jar,
  • el archivo del certificado de clave pública Terry.cer, e
  • información sobre los permisos que necesita la clase ExampleGame. Para esto, Terry podría proporcionar la entrada de otorgamiento exacta necesaria.

Los usuarios del juego también necesitan archivos e información de Chris. Para su convenencia, Terry puede reenviarles esta información:

  • el archivo JAR firmado hs.jar,
  • el archivo del certificado de clave pública Chris.cer, e
  • información como los permisos que las clases HighScore y HighScorePermission deben tener en un archivo de políticas para que funcionen. Esto podría ser la entrada exacta de otorgamiento necesitada.
