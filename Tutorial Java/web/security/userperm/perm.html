La Clase HighScorePermission

La clase HighScorePermission define el permiso que ExampleGame necesita para actualizar la puntuación más alta del usuario.

Todas las clases de permiso deben derivar de java.security.Permission o java.security.BasicPermission. La diferencia básica entre las dos es que java.security.Permission define permisos más complejos que requieren nombres y acciones. Por ejemplo, un java.io.FilePermission extiende de java.security.Permission, y requiere de un nombre (el nombre de un archvo), y las acciones permitidas para ese fichero (leer/escribir/eliminar).

En contraste, java.security.BasicPermission define permisos más simples que solo requieren un nombre, Por ejemplo, java.lang.RuntimePermission extiende de java.security.BasicPermission y simplemente necesita un nombre (como "exitVM"), que permite a los programas terminar la Máquina Virtual de Java.

Nuestro HighScorePermission es un permiso simple, y por lo tanto puede extenderse de java.security.BasicPermission.

A menudo, las implementaciones del método en la clase BasicPermission en sí misma no necesita ser sobreescrita por sus subclases. Este es el caso con nuestro HighScorePermission, así que todo lo que necesitamos implementar son los constructores, que simplemente invocan a los constructores de la superclase, como se muestra a continuación: 


package com.scoredev.scores;

import java.security.*;

public final class HighScorePermission extends BasicPermission {

    public HighScorePermission(String name)
    {
	super(name);
    }

    // dese cuenta de que se ignoran laas acciones y no se usan,
    // pero se necesita aún este constructor
    public HighScorePermission(String name, String actions) 
    {
	super(name, actions);
    }
}
