La Clase HighScore

La clase HighScore almacena y protege el acceso a la puntuación más alta del usuario para ExampleGame (y cualquier otro juego que lo llame). Por simplicidad, esta clase salva el valor de puntuación más alta en un archivo, llamado .highscore, en el directorio de inicio del usuario. Sin embargo, antes de permitir que ExampleGame recupere o actualize el valor de puntuación más alta del usuario, esta clase comprueba para asegurarse de que el usuario ha otorgado a ExampleGame permiso para acceder a la puntuación más alta en su archivo de políticas de seguridad.

Comprobando que ExampleGame tiene el HighScorePermission

Para comprobar si ExampleGame tiene o no permiso para acceder a la puntuación más alta del usuario, la clase HighScore debe:

  1. Llamar a System.getSecurityManager() para obtener el gestor de seguridad actualmente instalado.
  2. Si el resultado no es nulo (p.e., hay un gestor de seguridad, en lugar de que el llamador sea una aplicación sin restricciones), entonces
    1. Construye un objeto HighScorePermission, y
    2. Llama al método checkPermission del gestor de seguridad, y le pasa el objeto nuevo construído HighScorePermission.

Aquí está el código: 

SecurityManager sm = System.getSecurityManager();
if (sm != null) {
    sm.checkPermission(
        new HighScorePermission(gameName));
}

El método checkPermission esencialmente pregunta al gestor de seguridad si ExampleGame tiene el HighScorePermission especificado. En otras palabras, pregunta al gestor de seguridad si ExampleGame tiene permiso para actualizar el valor de puntuación más alta del usuario para el juego especificado (ExampleGame). El marco de trabajo de seguridad subyacente consultará la política de seguridad del usuario para ver si ExampleGame tiene este permiso.
