Pasos para un Usuario Ejecutando ExampleGame (Kim)

Los pasos como usuario, tal como Kim, debería dar, son:

Importar los Certificados como Certificados de Confianza 

keytool -import -alias chris -file Chris.cer -keystore kim.keystore
keytool -import -alias terry -file Terry.cer -keystore kim.keystore

Configurar un Archivo de Políticas con los Permisos Requeridos

Aquí está el archivo de políticas kim.policy, como se describió en Un Ejemplo del Fichero de Políticas.

Ejecutar ExampleGame

Para establecer la puntuación más alta: 

java -Djava.security.manager 
    -Djava.security.policy=kim.policy
    -classpath hs.jar;terry.jar
    com.gamedev.games.ExampleGame set 456

Para obtener la puntuación más alta: 

java -Djava.security.manager
    -Djava.security.policy=kim.policy
    -classpath hs.jar;terry.jar
    com.gamedev.games.ExampleGame get

Notas:

  • Si no especifica -Djava.security.manager, la aplicación se ejecutará de forma no restringida (los archivos de política y los permisos no serán comprobados).
  • La opción -Djava.security.policy=kim.policy le indica dónde está el archivo de políticas. Por ejemplo, puede agregar una entrada el archivo de políticas de segurdad que especifique la inclusión de kim.policy, como se describe al final de la lección Ver los Efectos del Fichero de Politica.
  • -classpath hs.jar;terry.jar especifica los archivos JAR que contienen los archivos class necesitados. Para Windows, use un punto y coma (";") para separar los archivos JAR; para Unix, use los dos puntos (":").
  • El archivo de políticas kim.policy especifica el almacén de claves kim.keystore. Ya que no proporciona una ubicación de URL absoluta para el almacén de claves, se asume que el almacén de claves esta en el mismo directorio que el archivo de políticas.
