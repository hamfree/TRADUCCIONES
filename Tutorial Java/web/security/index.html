Recorrido: Características de Seguridad en Java SE

En este recorrido aprenderá cómo las características de seguridad integradas en Java™ le protegen de los programas maliciosos. Verá cómo usar las herramientas para controlar el acceso a recursos, para generar y comprobar firmas digitales, y para crear y gestionar las claves necesitadas para la generación de firmas y su comprobación. Verá también cómo incorporar servicios criptográficos, tales como la generación de firma digital y comprobación, en sus programas.

Las características de seguridad proporcionadas por el Kit de Desarrollo Java (KDJ™) están destinadas a una variedad de audiencias:

  • Usuarios que ejecutan programas:

    La funcionalidad de seguridad integrada le protege de programas maliciosos (incluyendo viruses), mantiene la privacidad de sus archivos y la información sobre usted, y autentica la identidad de cada proveedor de código. Puede someter aplicaciones y sus applets a controles de seguridad cuando lo necesite.

  • Desarrolladores:

    Puede usar métodos de la IPA para incorporar funcionalidad de seguridad en sus programas, incluyendo servicios de cifrado y comprobaciones de seguridad. El marco de trabajo de la IPA le habilita para definir e integrar sus propios permisos (que controlan el acceso a recursos específicos), implementaciones de servicios de cifrado, implementaciones del gestor de seguridad, e implementaciones de políticas. Además, las clases se proporcionan para la gestión de sus pares de claves pública/privada y certificados de clave pública de personas en las que confía.

  • Administradores de sistemas, desarrolladores, y usuario:

    Las herramientas del KDJ gestionan su almacén de claves (base de datos de claves y certificados); generan firmas digitales para archivos JAR, y verifican la autenticidad de tales firmas y la integridad de los contenidos firmados; y crean y modifican los ficheros de políticas que definen la política de seguridad de su instalación.

Nota: Para los desarrolladores que quieren crear applets y aplicaciones de Java Web Start, vea Applets de Java para información sobre seguridad.

Lecciones del Recorrido

  Creando un Fichero de Políticas muestra cómo el acceso a los recursos se puede controlar con un archivo de políticas. Para la última información sbore los ficheros de configuración de políticas, vea la página Creación y gestión de archivos de políticas.

  Visita Rápida para Controlar Aplicaciones se basa en la lección anterior y muestra cómo el acceso a recursos, como leer o escribir un archivo, no está permitido para aplicaciones que se ejecutan bajo un administrador de seguridad, a menos que lo permita explícitamente un permiso en un archivo de política.

  IPA y Herramientas usadas para Código Seguro e Intercambio de Ficheros define las firmas digitales, los certificados, y los almacenes de claves y describe porqué se necesitan. También revisa la información aplicable a las tres lecciones siguientes en relación a los pasos que se necesitan habitualmente para usar las herramientas o la IPA para generar firmas, importación y exportación de certificados, etcetera.

  Firmando Código y Dándole Permisos ilustra el uso de todas las herramientas relacionadas con la seguridad. Muestra los pasos que un desarrollador debería tomar para firmar y distribuir código para que otros lo ejecuten. La lección también muestra a alguien que ejecutará el código (o un administrador de sistemas) podría agregar una entrada en un archivo de políticas para otorgar al código el permiso para el acceso al recurso que necesita.

  Intercambiando Ficheros muestra el uso de las herramientas por una persona para firmar un documento importante, como un contrato, y para exportar el certificado de clave pública para la clave pública correspondiente a la clave privada usada para firmar el contrato. Después la lección muestra cómo otra persona, que recibe el contrato, la firma, y el certificado de clave pública, puede importar el certificado y verificar la firma.

  Generando y Verificando Firmas le lleva paso a paso a través de un ejemplo de escribir un programa de Java usando la IPA de Seguridad del KDJ para generar claves, para generar una firma digital para datos usando la clave privada, y para exportar la clave pública y la firma al ordenador, que importa la clave pública y verifica la autenticidad de la firma. Finalmente, el ejemplo describe las debilidades potenciales del enfoque usado por los programas básicos y demuestra posibles enfoques y métodos alternativos de suministro e importación de claves, incluso en los certificados.

  Implementando su Propio Permiso demuestra cómo escribir una clase que define su propio permiso especial.
