Ver como Restringir Aplicaciones

Como vió en el paso previo, el entorno de ejecución de Java no instala automáticamente un Gestor de Seguridad cuando ejecuta una aplicación. Para aplicar la misma política de seguridad a una aplicación encontrada en el sistema de ficheros local o a applets de caja de arena descargados, puede invocar al intérprete con el nuevo argumento en línea de comandos -Djava.security.manager.

Para ejecutar la aplicación GetProps con el gestor de seguridad predeterminado, teclee lo siguiente: 

java -Djava.security.manager GetProps

Aquí tiene la salida del programa: 

C:\TEST>java -Djava.security.manager GetProps
    About to get os.name property value
      The name of your operating system is: SunOS
    About to get java.version property value
      The version of the JVM you are running is: 1.7.0
    About to get user.home property value
    Caught exception java.security.AccessControlException:
        access denied ("java.util.PropertyPermission"
        "user.home" "read")

El proceso se muestra en la figura siguiente.


Propiedades Sensibles a la Seguridad

El entorno de ejecución Java carga un archivo de políticas predeterminado y otorga todos los permisos al código para acceder a algunas propiedades útiles habitualmente tales como "os.name" y "java.version". Estas propiedades no son sensibles para la seguridad, así que otorgar estos permisos normalmente no representan un riesgo para la seguridad.

Las otras propiedades que GetProps intenta acceder, "user.home" y "java.home", no están entre las propiedades para las cuales el archivo de políticas del sistem otorga el permiso de lectura. Por lo tanto, tan pronto como GetProps intenta acceder a la primera de estas propiedades ("user.home"), el gestor de seguridad impide el acceso e informa de una AccessControlException. Esta excepción indica que la política actualmente en activo, que consiste de entradas en uno o más archivos de políticas, no otorga el permiso de lectura a la propiedad "user.home".

Nota: El código siempre puede leer un fichero del mismo directorio donde se encuentra (o un subdirectorio de ese directorio); no necesita explícitamente permiso para hacer eso. El código puede también obtener la ruta del directorio desde el que se ejecuta, y esta ruta puede contener información sensible. Por ejemplo, si el código se ejecuta desde un directorio de inicio (o un subdirectorio del directorio de inicio), la ruta puede revelar el nombre del usuario actual.

El Archivo de Políticas Predeterminado

El archivo de políticas predeterminado, java.policy está (de forma predeterminada) ubicado en :

  • Windows: java.home\lib\security\java.policy
  • UNIX: java.home/lib/security/java.policy

Tenga en cuenta que java.home representa el valor de la propiedad "java.home", que es una propiedad del sistema que especifica el directorio en el cual se instaló el EEJ. Así si el EEJ fue instalado en el directorio llamado C:\jdk\jre en Windows y /jdk/jre en UNIX, el archivo de políticas del sistema está ubicado en :

  • Windows: C:\jdk\jre\lib\security\java.policy
  • UNIX: /jdk/jre/lib/security/java.policy
