Ver los Efectos del Fichero de Politica

Ahora que ha agregado la entrada de política requerida al archivo de políticas examplepolicy, debería ser capaz de leer las propiedades especificadas cuando ejecute la aplicación GetProps con un gestor de seguridad, como se muestra en la figura siguiente.

Siempre que ejecute un applet, o una aplicación con un gestor de seguridad, los archivos de política que se cargan y usan de forma predeterminada son los que se especificaron en el "archivo de propiedades de seguridad", que se ubicó en uno de los directorios siguientes:

  • Windows: java.home\lib\security\java.security
  • UNIX: java.home/lib/security/java.security

Nota:   La variable de entorno java.home nombra el directorio dentro del cual fue instalado el EEJ.

Las ubicaciones del archivo de políticas se especifican como los valores de las propiedades cuyos nombres toman la forma: 

policy.url.n

Donde la variable n indica un número. Especifica cada valor de propiedad en una línea que toma la forma siguiente: 

policy.url.n=URL

Donde URL es una especificación URL. Por ejemplo, los archivos predeterminados de políticas, algunas veces referidos como los archivos de políticas de usuario y sistema, respectivamente, se definen en el archivo de propiedades de seguridad como 

policy.url.1=file:${java.home}/lib/security/java.policy
policy.url.2=file:${user.home}/.java.policy
Nota: El uso de la notación ${propName} en el archivo de propiedades de seguridad es una forma de especificar el valor de una propipad. Así ${java.home} será reemplazada en tiempo de ejecución por el valor real de la propiedad "java.home", que indica el directorio dentro del cual se instaló el EEJ, y ${user.home} será reemplazado por el valor de la propiedad "user.home", por ejemplo, C:\Windows.

Hay dos posibles formas de hacer que sea considerado el archivo examplepolicy como parte de la política general, además de los archivos de política especificados en el archivo de propiedades de seguridad. Puede especificar el archivo de políticas adicional en una propiedad que pasa al sistema en tiempo de ejecución, como se describe en Aproximación 1, o agregar una línea en el archivo de propiedades de seguridad especificando el archivo de políticas adicional, como se describe en la Aproximación 2.

Aproximación 1

Puede usar un argumento de línea de comando -Djava.security.policy del intérprete para especificar un archivo de política que debería ser usado además de los especificados en el archivo de propiedades de seguridad.

Asegúrese de que está en el directorio que contiene GetProps.class y examplepolicy. Después puede ejecutar la aplicación GetProps y pasar el archivo de políticas examplepolicy al intérprete tecleando el comando siguiente en una línea: 

java -Djava.security.manager -Djava.security.policy=examplepolicy GetProps

Nota:   Recuerde que -Djava.security.manager se requiere para ejecutar una aplicación con un gestor de seguridad, como se muestran en el paso Ver como Restringir Aplicaciones.

El programa informa de los valores de las propiedades "user.home" y "java.home".

Si la aplicación aún informa de un error, algo está mal en el archivo de políticas. Use la Herramienta de Políticas para comprobar la entrada de política que acabó de crear en el paso Establecer el Fichero de Políticas para Conceder los Permisos Requeridos.

Aproximación 2

Puede especificar un número de URLs en la propiedades policy.url.n en el archivo de propiedades de seguridad, y todos los archivos de políticas designado serán cargados. Así una forma de que las entradas de su archivo de políticas examplepolicy sean consideradas por el intérprete java es agregar una entrada especificando el archivo de políticas en el archivo de propiedades de seguridad.

Importante: Si está ejecutando su propia copia del KDJ, puede fácilmente editar su archivo de propiedades de seguridad. Si está ejecutando una versión compartida con otros usuarios, puede ser solo capaz de modificar el archivo de propiedades de seguridad del sistema si tiene acceso de escritura a él o si se lo pide a su administrador de sistemas para que modifique el archivo cuando sea apropiado. Sin embargo, probablemente no sea apropiado para usted hacer modificaciones a un archivo de políticas de todo el sistema para esta prueba de tutorial. Sugerimos que simplemente lea lo siguiente para ver cómo se hizo o que instale su propia versión privada del KDJ para utilizar en las lecciones del tutorial.

Para modificar el archivo de propiedades de seguridad, ábralo con un editor capaz de editar un fichero de texto ASCII. Después agregue la línea siguiente después de la línea que contiene policy.url.2: si está en un sistema Windows, agregue 

policy.url.3=file:/C:/Test/examplepolicy

Si está en un sistema UNIX, agrega 

policy.url.3=file:${user.home}/test/examplepolicy

En un sistema UNIX puede alternativamente especificar explícitamente su directorio de inicio, como en 

policy.url.3=file:/home/jones/test/examplepolicy

Ejecute la Aplicación

Ahora debe ser capaz de ejecutar de forma exitosa lo siguiente. 

java -Djava.security.manager GetProps

Como con la aproximación 1, si usted aún obtiene una excepción de seguridad, algo está mal en el archivo de políticas. Use la Herramienta de Políticas para comprobar la entrada de política que acabó de crea en el paso Establecer el Fichero de Políticas para Conceder los Permisos Requeridos. Después corrija cualquier error tipográfico u otros errores.

Importante: No necesita incluir el archivo examplepolicy a menos que usted esté ejecutando esta lección del Tutorial. Para excluir este fichero, abra el archivo de propiedades de seguridad y borre la línea que acaba de agregar.

Antes de continuar, puede querer eliminar la línea que acaba de agregar en el archivo de propiedades de seguridad (o comentarla), ya que probablemente no quiere que el archivo examplepolicy se incluya cuando no está ejecutando las lecciones del tutorial.
