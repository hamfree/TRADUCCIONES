Observar la Libertad de una Aplicación

Un gestor de seguridad no se instala automáticamente cuando se ejecutan una aplicación. En el siguiente paso, verá cómo aplicar la misma política de seguridad a una aplicación encontrada el sistema de ficheros local como a applets de caja de arena descargados. Pero primero, demostremos que un gestor de seguridad no está instalado para una aplicación de forma predeterminada, y que por lo tanto la aplicación tiene acceso completo a los recursos.

Cree un archivo llamado GetProps.java en su computadores ya sea copiando o descargando el código fuente de GetProps.java.

Los ejemplos en esta lección asumen que usted coloca GetProps.java en el directorio C:\Test si usted está usando un sistema Windows o en el directorio ~/test en UNIX.

Como puede ver si examina el fichero fuente, este programa intenta obtener (leer) los valores de propiedad, cuyos nombres son "os.name" , "java.version", "user.home", y "java.home".

Ahora compile y ejecute GetProps.java. Debe ver una salida como la siguiente: 

C:\TEST>java GetProps
    About to get os.name property value
      The name of your operating system is:
      Windows XP
    About to get java.version property value
      The version of the JVM you are running is:
      1.6.0
    About to get user.home property value
      Your user home directory is: C:\WINDOWS
    About to get java.home property value
      Your EEJ installation directory is:
      C:\JDK7.0.0\JRE

Esto muestra que se le permitió a la aplicación accede a todos los valores de propiedad, como se muestra en la siguiente figura.
