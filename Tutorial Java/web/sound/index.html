Recorrido: Sonido

La IPA de Sonido de Java es una IPA de bajo nivel para efectuar y controlar la entrada y salida de medios de sonido, incluídos los datos de audio y de Interfaz Digital de Instrumentos Digitales (MIDI). La IPA de Sonido de Java proporciona control explícito sobre las capacidades normalmente requeridas para la entrada y salida del sonido, en un marco de trabajo que promueve extensibilidad y flexibilidad.

La IPA de Sonido de Java satisface las necesidades de una amplia gama de desarrolladores de aplicaciones. Las áreas de aplicaciones potenciales incluyen:

  • Marcos de trabajo de comunicación, tales como conferencias y telefonía
  • Sistemas de entrega de contenido para el usuario final, como reproductores multimedia y música que utilizan contenido transmitido
  • Programas de aplicaciones interactivas, tales como los juegos y sitios Web que usan contenido dinámico
  • Creación y edición de contenidos
  • Herramientas, kits de herramientas y utilidades

La IPA de Sonido de Java proporciona el nivel más bajo de soporte de sonido de la plataforma Java. Proporciona a los programas de aplicación una gran cantidad de control sobre las operaciones de sonido y extensible. Por ejemplo, la IPA de Sonido Java proporciona mecanismos para instalar, acceder y manipular recursos del sistema, como mezcladores de audio, sintetizadores MIDI, otros dispositivos audio o MIDI, lectores y escritores de archivos, y convertidores de formatos de sonido. La IPA de Sonido y Java no incluya editores de sonido sofisticados o herramientas gráficas, pero proporciona capacidades sobre las cuales se pueden construir dichos programas. Enfatiza el control de bajo nivel más allá de lo que comúnmente espera el usuario final.

La IPA de Sonido de Java incluye soporte tanto para audio digital como datos MIDI. Estos dos módulos principales de funcionalidad se proporcionan en paquetes separados:

  • javax.sound.sampled – Este paquete especifica interfaces para capturar, mezclar, y reproducción digital de audio (muestreada).
  • javax.sound.midi – Este paquete proporciona interfaces para síntesis MIDI, secuenciación, y transporte de eventos.

Los otros dos paquetes permiten proveedores de servicio (a diferencia de los desarrolladores de aplicaciones) para crear componentes de software personalizado que extiende las capacidades de una implementación de la IPA de Sonido de Java:

  • javax.sound.sampled.spi
  • javax.sound.midi.spi

Esta página introduce el sistema de audio muestreado, el sistema MIDI, y los paquetes SPI. Cada paquete es después descrito en mayor detalle más tarde en el tutorial.

Nota: 

Hay otra IPA de la plataforma Java que también tiene elementos relacionados con el sonido. La IPA del Marco de Trabajo de Media de Java (JMF) es una de más alto nivel que está actualmente disponible como una Extensión Estándar a la plataforma Java. JMF especifica una arquitectura unificada, protocolo de mensajes, e interfaz de programación para capturar y reproducir medios basados en el tiempo. JMF proporciona una solución más simple para programas básicos de aplicación de reproducción de medios, y habilita la sincronización entre diferentes tipos de medios, tales como audio y vídeo. Por el otro lado, los programas que se enfocan en el sonido pueden beneficiarse de la IPA de Sonido de Java, especialmente si requieren características más avanzadas, tales como la habilidad de controlar cuidadosamente la reproducción de audio en búfer o manipular directamente un sintetizador MIDI. Otra IPA de Java con aspectos de sonido incluyen Java 3D e IPA para telefonía y voz. Una implementación de cualquiera de estas IPAS podrían usar una implementación de la IPA de Sonido de Java internamente, pero no es necesario que lo haga.

¿Qué es el Audio Muestreado?

El paquete javax.sound.sampled gestiona datos de audio digital, a los que la IPA de Sonido de Java se referie como audio muestreado. Las Muestras (Samples, en inglés) son instantáneas sucesivas de una señal. En el caso del audio, la señal es una onda de sonido. Un micrófono cinvierte la señal acústica en su correspondiente señal eléctrica analógica, y un convertidor analógico-a-digital transforma esa señal analógica en una forma digital muestreada. La figura siguiente muestra un breve momento en una grabación de sonido.

Una Onda de Sonido Muestreada

Esta gráfica traza la presión de sonido (amplitud) en el eje vertical, y el tiempo en el eje horizontal. La amplitud de una onda analógica de sonido se mide periódicamente a una cierta velocidad, lo que da como resultado las muestras discretas (los puntos de datos rojos en la figura) que componen la señal de audio digital. La línea central horizontal indica la amplitud cero; los puntos bore la línea son muestras con valor positivo, y los puntos debajo son negativos. La precisión de la aproximación digital de la señal analógica depende de su resolución en el teimpo (la tasa de muestreo) y su cuantificación, o resolución en amplitud (el número de bits usados para representar cada muestreo). Como un punto de referencia, el audio grabado para almacenar en los discos compactos está muestreado a 44,100 veces por segundo y está representado con 16 bits por muestra.

El término "muestra de audio" se usa aquí ligeramente. Se puede muestrear una onda de sonido a intervalos discretos mientras se deja en forma analógica. Para los propósitos de la IPA de Sonido de Java, sin embargo, la "muestra de audio" es equivalente al "audio digital"

Habitualmente, el audio muestrado en una computadores viene desde una grabación de sonido, pero el sonido podría ser en cambio generado sintéticamente (por ejemplo, para crear los sonidos de teléfono de tonos). El término "muestra de audio" se refiere al tipo de datos, no a su origen.

La IPA de Sonido de Java no asume una configuración de hardware de audio específica; está diseñada para permitir que diferentes clases de componentes de audio se instalen en un sistema y se accedan por la IPA. La IPA de Sonido de Java admite la funcionalidad común tal como la entrada y salida desde una tarjeta de sonido (por ejemplo, para grabación y reproducción de ficheros de sonido) como también la mezcla de varios flujos de audio. ¿Qué es MIDI?

El paquete javax.sound.midi contiene una IPA para transportar y secuenciar eventos MIDI, y para sintetizar sonidos desde estos eventos.

Considerando que el audio muestreado es una representación directa de un sonido en sí, se puede pensar de los datos MIDI como de una receta para crear un sonido, especialmente un sonido musical. Los datos de MIDI, a diferencia de los datos de audio, no describen sonido directamente. En su lugar, describe eventos que afecta los sonidos (o acciones) realizados por un dispositivo habilitado para MIDI o instrumento, como un sintetizador. Los datos MIDI son análogos a los eventos de teclado y ratón de la interfaz gráfica de usuario. En el caso de MIDI, se puede pensar de los eventos como acciones en un teclado musical, junto con acciones en varios pedales, controles deslizantes, interruptores y perillas de ese instrumento musical. Estos eventos no necesitan en realidad originarse con un instrumento musical físico; puede ser simulado en software, y puede ser almacenado en archivos MIDI. Un programa que puede crear, editar, y realizar estos archivos se llama un secuenciador. Muchas tarjetas de sonido de computadora incluyen chips de música sintetizada controlables por MIDI a los cuales los secuenciadores pueden enviar sus eventos MIDI. Los sintetizadores pueden también ser implementados completamente en software. El sintetizador interpreta los eventos MIDI que recibe y produce la salida de audio. Habitualmente el sonido sintetizado de los datos MIDI es sonido musical (a diferencia del habla, por ejemplo). Los sintetizadores MIDI también son capaces de generar varias clases de efectos de sonido.

Algunas tarjetas de sonido incluyen puertos de entrada y salida MIDI a los cuales dispositivos externos de hardware MIDI (tales como teclados sintetizadores u otros instrumentos) pueden ser conectados. Desde un puerto de entrada MIDI, un programa de aplicación puede recibir eventos generados por un instrumentos musical externo equipado como MIDI. El programa puede reproducir la interpretación musical usando el sintetizador interno de la computadorea, salvarlo a disco como un archivo MIDI, o presentarlo en notación musical. Un programa podría usar un puerto de salida MIDI para reproducirlo en un instrumento externo, o controlar otros dispositivos externos tales como un equipo de grabación.

Una Posible Configuración MIDI

En este ejemplo, el programa de aplicación prepara una interpretación musical cargando una partitura que se almacenó como un archivo MIDI estándar en un disco (lado izquierdo del diagrama). Los archivos MIDI estándar contienen pistas, en cada una de las cuales hay una lista de eventos MIDI etiquetados en el tiempo. La mayoría de los eventos representan notas musicales (tonos y ritmos). Este archivo MIDI se lee y después es "ejecutado" por un secuenciador software. Un secuenciador ejecuta su música enviando mensajes MIDI a algún otro dispositivo, tal como un sintetizador interno o externo. El propio sintetizador puede leer un archivo con un banco de sonidos que contiene instrucciones para emular los sonidos de ciertos instrumentos musicales. Si no, el sintetizador tocará las notas almacenadas en el archivo MID usando cualquier sonido de instrumento que ya tenga cargado.

Como se ilustra, los eventos MIDI deben traducirse a MIDI sin formato (sin etiqueta de tiempo) antes de enviarse a través de un puerto de salida MIDI a un instrumento MIDI externo. De forma similar, los datos MIDI en crudo que entran en el computador desde una fuente MIDI externa (un instrumento de teclado, en el diagrama) se traducen en mensajes MIDI etiquetados en el tiempo que pueden controlar un sintetizador, o que un secuenciador puede almacenar para uso posterior.

Interfaces de Proveedores de Servicio

Los paquetes javax.sound.sampled.spi y javax.sound.midi.spi contienen una IPA que permite a los desarrolladores de software crear nuevos recursos de audio o MIDI que pueden ser proporcionados separadamente al usuario y "conectarse" a una implementación existente de la IPA de Sonido de Java. Aquí tiene algunos ejemplos de servicios (recursos) que pueden ser agregados de esta forma:

  • Un mezclador de audio
  • Un sintetizador MIDI
  • Un analizador de archivos que puede leer o escribir un nuevo tipo archivo de audio o MIDI
  • Un convertidor que traduce entre diferentes formatos de datos de audio

En algunos casos, los servicios son interfaces software a las capacidades de los dispositivos hardware, tales como tarjetas de sonido, y el proveedor de servicio podría ser el mismo que el del fabricante del hardware. En otros casos, los servicios existen puramente en software. Por ejemplo, un sintetizador o un mezclador podría ser una interfaz a un chip en una tarjeta de sonido, o podría ser implementado sin ningún soporte de hardware.

Una implementación de la IPA de Sonido de Java contiene un cnjunto básico de servicios, pero los paquetes de la interfaz del proveedor de servicios (SPI) permiten a terceras partes crear nuevos servicios. Estos servicios de terceras partes se integran dentro del sistema de la misma forma que los servicios integrados. La clase AudioSystem y la clase MidiSystem actúan como coordinadors que permiten a los programas de aplicación acceder a los servicios explícitamente o implícitamente. A menudo la existencia de un servicio es completamente transparente a una aplicación de progrmaa que lo usa. El mecanismo de proveedor de servicio beneficia a los usuarios de los programas de aplicación basados en la IPA de Sonido de Java, porque las nuevas características de sonido pueden ser agregadas a un programa sin requerir una nueva versión del KDJ o entorno de ejecución, y, en muchos casos, sin incluso requerir una nueva versión del mismo programa de aplicación.
