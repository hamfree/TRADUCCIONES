Accediendo a los Recursos del Sistema de Audio

La IPA de Sonido de Java toma un enfoque flexible a la configuración del sistema. Diferentes clases de dispositivos (mezcladores) pueden ser instalados en una computadora. La IPA hace pocas suposiciones sobre qué dispositivos se han instalado y cuáles son sus capacidades. En su lugar, proporciona formas para que el sistema informe sobre los componentes de audio disponibles y formas para que su programa acceda a ellos.

Las secciones siguientes muestran cómo su programa puede aprender qué recursos de audio muestreado han sido instalados en el computador, y cómo puede obtener acceso a los recursos disponibles. Entre otras cosas, los recursos incluyen los mezcladores y los varios tipos de líneas propiedad de los mezcladores.

La Clase AudioSystem

La clase AudioSystem actúa como una cámara de compensación para los componentes de audio, incluyendo servicios integrados y separadamente servicios instalados desde proveedores de terceras partes. AudioSystem sirve como un punto de entrada al programa de aplicación para acceder a esos recursos de audio muestreado instalados. Puede preguntar a AudioSystem para saber qué clases de recursos han sido instalados, y después puede obtener el acceso a ellos. Por ejemplo, un programa de aplicación puede comenzar preguntando a la clase AudioSystem si hay un mezclador que tenga una determinada configuración, tal como una de las configuraciones de entrada y salida ilustradas anteriormente en la descripción de líneas. Desde el mezclador, el programa obtendría después las líneas de datos, y así.

Aquí tiene alguno de los recursos que un programa de aplicación puede obtener de AudioSystem:

  • Mezcladores — Un sistema habitualmente tiene una variedad de mezcladores instalados. Hay usualmente al menos uno para entrada de audio y uno para salida de audio. También puede haber mezcladores que no tengan puertos de E/S, sino que acepten audio de un programa de aplicación y devuelvan el audio mezclado al programa. La clase AudioSystem proporciona una lista de todos los mezcladores instalados.
  • Líneas — Aunque cada línea está asociada con un mezclador, un programa de aplicación puede obtener una línea directamente del AudioSystem, sin tratar explícitamente con los mezcladores.
  • Conversiones de formato — Un programa de aplicación puede usar las conversiones de formato para traducir datos de audio de un formato a otro.
  • Archivos y flujos — La clase AudioSystem proporciona métodos para traducir entre archivos de audio y flujos de audio. Puede también informar del formato de archivo de un archivo de sonido y puede escribir archivos en diferentes formatos.

Objetos de Información

Varias clases en la IPA de Sonido de Java proporciona información útil sobre las interfaces asociadas. Por ejemplo, Mixer.Info proporciona detalles sobre un mezclador instalado, tal como el fabricante, nombre, descripción y versión del mezclador. Line.Info obtiene la clase de una línea específica. Las subclases de Line.Info incluyen Port.Info y DataLine.Info , que obtienen detalles relevantes para un puerto y línea de datos específicamente, respectivamente. Cada una de estas clases se describe más adelante en la sección apropiada abajo. Es importante no confundir el objeto Info con el objeto del mezclador o línea que describe.

Obtener un Mixer (Mezclador)

Habitualmente, una de las primeras cosas que un programa que usa la IPA de Sonido de Java necesita hacer es obtener un mezclador, o al menos una línea del mezclador, de forma que pueda obtener sonido dentro y fuera de la computadora. Su programa podría necesitar una clase de mezclador específico, o podría querer visualizar una lista de todos los mezcladores disponibles de forma que el usuario puede seleccionar uno. En otro caso, necesita aprender qué clases de mezcladores están instalados. AudioSystem proporciona el siguiente método: 

static Mixer.Info[] getMixerInfo()

Cada objeto Mixer.Info devuelto por este método identifica un tipo de mezclador que está instalado. (Usualmente un sistema tiene al menos un mezclador de un tipo dado. Si resulta que hay más de uno de un tipo determinado, la matriz devuelta todavía solo tiene uno Mixer.info para ese tipo). Un programa de aplicación puede iterar sobre los objetos Mixer.Info para encontrar uno apropiado, de acuerdo a sus necesidades. El Mixer.Info incluye las cadenas siguientes para identificar la clase de mezclador:

  • Nombre
  • Version
  • Vendedor/Fabricante
  • Descripción

Estas son cadenas arbitrarias, de forma que un programa de aplicación que necesita un mezclador específico sabe que´esperar y con qué comparar las cadenas. La compañía que proporciona el mezclador debe incluir esta información en su documentación. Alternativamente, y quizás más típicamente, el programa de aplicación mostrará todas las cadenas de los objetos Mixer.Info al usuario y permite al usuario elegir el mezclador correspondiente.

Una vez se encuentra un mezclador apropiado, el programa de aplicación invoca el siguiente método de AudioSystem para obtener el mezclado deseado: 

static Mixer getMixer(Mixer.Info info)

¿Qué pasa si su programa necesita un mezclador que tenga ciertas capacidades, pero no necesita un mezclador específico fabricado por un proveedor específico? ¿Y si no puede depender de que el usuario sepa qué mezclador debe elegir? En ese caso, la información en los objetos Mixer.Info no será de mucha utilidad. En su lugar, puede iterar sobre todos los objetos Mixer.Info devueltos por getMixerInfo, obtener un mezclador para cada uno invocando a getMixer, y preguntar a cada mezclador por sus capacidades. Por ejemplo, podría necesitar un mezclador que puede escribir sus datos de audio mezclados a cierto número de líneas de datos destino simultáneamente. En ese caso, preguntaria a cada mezclador usando este método de Mixer: 

int getMaxLines(Line.Info info)

Aquí, Line.Info especificaría una TargetDataLine. La clase Line.Info se describe en la sección siguiente.

Obtener una Line (Línea) del Tipo Deseado

Hay dos formas de obtener una línea:

  • Directamente desde el objeto AudioSystem
  • Desde un mezclador que ya ha obtenido desde el objeto AudioSystem

Obtener una Line (Línea) Directamente desde el AudioSystem

Asumamos que no ha obtenido un mezclador, y que su programa es uno simple que realmente solo necesita una cierta clase de línea; los detalles del mezclador no le importan. Puede usar el método de AudioSystem: 

static Line getLine(Line.Info info)

el cual es análogo al método getMixer descrito previamente. A diferencia de Mixer.Info , la Line.Info usada como un arguemnto no almacena información textual para especificar la línea deseada. En su lugar, almacena información sobre la clase de línea deseada.

Line.Info es una clase abstracta, así que use una de sus subclases ( Port.Info o DataLine.Info ) para obtener una línea. El siguiente extracto de código usa la subclase DataLine.Info para obtener y abrir una línea de datos destino: 

TargetDataLine line;
DataLine.Info info = new DataLine.Info(TargetDataLine.class, 
    format); // format es un objeto AudioFormat
if (!AudioSystem.isLineSupported(info)) {
    // Maneja el error.
    }
    // Obtiene y abre la línea.
try {
    line = (TargetDataLine) AudioSystem.getLine(info);
    line.open(format);
} catch (LineUnavailableException ex) {
        // Maneja el error.
    //... 
}

Este código obtiene un objeto TargetDataLine sin especificar ningún atributo mas que su clase y su formato de audio. Puede usar código análogo para obtener otros tipos de líneas. Para una SourceDataLine o un Clip, simplemente substituya esa clase para TargetDataLine como la clase de la variable de la línea, y tambíen en el primer argumento del constructor DataLine.Info.

Para un Port, puede usar instancias estáticas de Port.Info, en código como el siguiente: 

if (AudioSystem.isLineSupported(Port.Info.MICROPHONE)) {
    try {
        line = (Port) AudioSystem.getLine(
            Port.Info.MICROPHONE);
    }
}

Tenga en cuenta que el uso del método isLineSupported para ver si el mezclador incluso tiene una línea del tipo deseado.

Recuerde que una línea de origen es una entrada a un mezclador, es decir, un objeto Port si el mezclador representa un dispositivo de entrada de audio, y un SourceDataLine o objeto Clip si el mezclaor representa un dispositivo de audio de salida. De forma similar, una línea de destino es una salida del mezclador: un objeto Port para un mezclador de salida de audio, y un objeto TargetDataLine para un mezclador de entrada de audio. ¿Qué pasa si un mezclador no se conecta a ningún dispositivo de hardware externo? Por ejemplo, considere un mezclador interno o de solo software que obtiene el audio desde un programa de aplicación y envía su audio mezclado de vuelta al programa. Esta clase de objetos TargetDataLine para sus líneas de salida.

Puede también usar los siguientes métodos de AudioSystem para aprender más sobre las líneas de origen y destino de un tipo específico que son admitidas por un mezclador instalado: 

static Line.Info[] getSourceLineInfo(Line.Info info)
static Line.Info[] getTargetLineInfo(Line.Info info)

Tenga en cuenta que la matriz devuelta por cada uno de estos métodos indican tipos únicos de líneas, no necesariamente todas las líneas. Por ejemplo, si dos líneas de un mezclador, o dos líneas de diferentes mezcladores, tienen objetos Line.Info idénticos, las dos líneas serán representadas por solo una Line.Info en la matriz devuelta.

Obtener una Line (Línea) desde un Mixer (Mezclador)

La interfaz Mixer incluye variaciones de los métodos de acceso de AudioSystem para líneas de origen y destino, descritas arriba. Estos métodos de Mixer incluyen los que toman argumentos Line.Info, simplemente como los métodos de AudioSystem hacen. Sin embargo, Mixer también incluye estas variantes, que no toman argumentos: 

Line.Info[] getSourceLineInfo()
Line.Info[] getTargetLineInfo()

Estos métodos devuelven matrices de todos los objetos Line.Info para la mezcladora particular. Una vez que ha obtenido las matrices, puede iterar sobre ellas, llamando al método getLine de Mixer para obtener cada línea, seguido por el método open de Line para reservar el uso de cada línea para su programa.

Seleccionar Puertos de Entrada y Salida

La sección previa, sobre cómo obtener una línea del tipo deseado, se aplica tanto a los puertos como a otros tipos de líneas. Puede obtener todos los puertos de origen (p.e., entrada) y destino (p.e., salida) pasando un objeto Port.Info a los métodos de AudioSystem (o Mixer) getSourceLineInfo y getTargetLineInfo que toman un argumento Line.Info. Después itera sobre la matriz de objetos devuelta e invoca al método del Mixer getLine para obtener cada puerto.

Puede después abrir cada Port invocando el método open de Line. Abrir un puerto significa que lo activa — es decir, permitir al sonido entrar o salir por el puerto. De forma similar, puede cerrar puertos por los que no desea que pase el sonido, porque algunos puertos podrían estar aún abiertos antes de que incluso usted los obtenga. Algunas plataformas dejan todos los puertos activados de forma predeterminada; o un usuario administrador podría haber seleccionado que ciertos puertos estén activos o no, usando otra programa de aplicación o software del sistema operativo.

Advertencia: Si quiere seleccionar un cierto puerto y asegurarse de que el sonido en realidad entre o salga por el puerto, puede abrir el puerto como se ha descrito. Sin embargo, ¡esto puede ser considerado un comportamiento hostil al usuario! Por ejemplo, un usuario podría tener el puerto del altavoz apagado de forma que no perturbe a sus compañeros. Ella estaría bastante molesta si su programa de repente anulara sus deseos y comenzara a sonar música a todo volumen. Como otro ejemplo, un usuario querría asegurarse de que el micrófono de su computadora no está nunca activado sin su conocimiento, para evitar las escuchas a escondidas. En general, no se recomienda abrir o cerrar puertos a menos que su programa esté respondiendo a las intenciones del usuario, como expresa a través de la interfaz del usuario. En su lugar, respeta las configuraciones que el usuario o el sistema operativo ya ha seleccionado.

No es necesario abrir o cerrar un puerto antes de que el mezclador al que está conectado funcione correctamente. Por ejemplo, puede iniciar la reproducción de sonido en un mezclador de salida de audio, incluso aunque todos sus puertos de salida estén cerrados. Los datos aún fluyen dentro del mezclador; la reproducción no se bloquea. El usuario simplemnte no quiere oir nada. Tan pronto como el usuario abre un puerto de salida, el sonido será audible a través de ese puerto, comenzando en cualquier punto del medio al que haya llegado la reproducción.

También, no necesita acceder a los puertos para aprender si el mezclador tiene ciertos puertos. Para aprender si un mezclador es realmente un mezclador de salida de audio, por ejemplo, puede invocar getTargetLineInfo paa ver si este tiene puertos de salida. No hay razón para acceder a los puertos por si mismos a menos que quiera cambiar sus configuraciones (tales como su estado abierto-o-cerrado, o las configuraciones de cualquier control que pudieran tener).

Permiso para Usar Recursos de Audio

La IPA de Sonido de Java incluye una clase AudioPermission que indica qué clases de acceso un applet (o una aplicación ejecutándose con un gestor de seguridad) pueden tener al sistema de audio muestreado. El permiso para grabar sonido se controla separadamente. Este permiso debe ser otorgado con cuidado, para ayudar a prevenir riesgos de seguridad tales como las escuchas a escondidas no autorizadas. De forma predeterminada, los applets y aplicaciones tienen los permisos otorgados como sigue:

  • Un applet ejecutándose con el gestor de seguridad del applet puede reproducir, pero no grabar, audio.
  • Una aplicación ejecutándose sin gestor de seguridad puede tanto reproducir como grabar audio.
  • Una aplicación ejecutándosse con el gestor de seguridad predeterminado puede reproducir, pero no grabar, audio.

En general, los applets se ejecutan bajo el escrutinio de un gestor de seguridad y no se les permite grabar sonido. Las aplicaciones, por otro lado, no instalan automáticamente un gestor de seguridad, y son capaces de grabar sonido. (Sin embargo, si se invoca el gestor de seguridad predeterminado de forma explícita por una aplicación, no se le permite a la aplicación grabar sonido).

Tanto los applets como las aplicaciones pueden grabar sonido incluso cuando se ejecutan con un gestor de seguridad si se les ha otorgado explícitamente permiso para hacerlo.

Si su programa no tiene permisos para grabar (o reproducir) sonido, se lanzará una excepción cuando intenta abrir una línea. No hay nada que puede hacer sobre esto en su programa, mas que capturar la excepción e informar del problema al usuario, porque los permisos no se pueden cambiar a través de la IPA. (Si pudieran, serían inútiles, ¡porque nada estaría seguro!) Generalmente, los permisos se establecen en uno o más ficheros de políticas de configuración, los cuales un usuario o administrador del sistema pueden editar usando un editor de texto o el programa de Herramienta de Políticas.
