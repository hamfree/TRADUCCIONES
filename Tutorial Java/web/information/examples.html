Ejecutando los Ejemplos del Tutorial en el EID NetBeans

Los ejemplos en el Tutorial de Swing y en el Tutorial de Implantación también han sido ofrecidos como proyectos de Netbeans que están listos para ser abiertos y ejecutados dentro del EID Netbeans, un entorno de desarrollo integrado gratuito que puede usar para simplicar el desarrollo de aplicaciones.

Esta página ofrece información sobre las siguientes tareas:

  • Descargando, Instalando y Ejecutando el EID Netbeans
  • Abriendo un Proyecto para un Ejemplo
  • Compilando y Ejecutando un Proyecto

Descargando, Instalando y Ejecutando el EID Netbeans

La forma más facil de obtener el EID Netbeans es descargar el instalador del EID Netbeans para su sistema operativo desde página de descargas de Netbeans.

Puede ejecutar el instalador y completar una secuencia corta de pasos en el asistente de instalación para configurar el EID. Puede encontrar más detalles del proceso de instalación de las instrucciones de instalación que están enlazadas desde la página de descargas.

Puede ejecutar el EID en una de las siguientes formas:

  • En sistemas Microsoft Windows, puede usar el item del EID NetBeans en el menú de Inicio.
  • En los sistemas Solaris OS y Linux, ejecuta el script del lanzador del EID navegando al directorio bin del EID y escribiendo ./netbeans.
  • En sistemas Mac OS X, pulse en el icono de la aplicación EID Netbeans.

Abriendo un Proyecto para un Ejemplo

Abrir un proyecto en el EID Netbeans consiste de los siguientes pasos:

  • Descargue un ejemplo del tutorial o el conjunto completo de ejemplos. Los ejemplos individuales están disponibles en el Ejemplo como una columna de Proyecto de EID Netbeans en las diversas páginas índice de ejemplo ( como ésta página de índice ): Usando Componentes de Swing: Ejemplos. Puede descargar todos los ejemplos como parte del conjunto del Tutorial de Java. Vea el enlace "Descargar el último Tutorial de Java SE" en la página inicial del tutorial.
  • Si ha descargado un proyecto individual, descomprima el proyecto en su sistema. Si ha descargado un fichero comprimido del tutorial completo o un fichero comprimido de todos los ejemplos, descomprima ese fichero.
  • En el EID NetBeans , elija File > Open Project. En el seleccionador de ficheros, seleccione el proyecto y pulse Open Project.

Una vez abierto, el proyecto aparece en la ventana de Proyectos del EID. Expanda los nodos del proyecto para mostrar la estructura del proyecto. El proyecto incluye los siguientes subnodos principales:

  • Source Packages. Contiene nodos representando paquetes de Java para sus ficheros fuente. Expanda estos nodos para acceder a los ficheros fuente .java del proyecto.
  • Libraries. Contiene nodos representando las librerías de las cuales la aplicación depende. Para los ejemplos en el Tutorial Swing, sólo el KDJ está listado aquí.

Para abrir un fichero, haga doble-click sobre el nodo del fichero en la ventana Projects. Cuando esté abierto, un fichero es mostrado en el Source Editor, a la derecha de la ventana Proyects.

La ventana de Proyectos representa la estructura "lógica" del proyecto, la cual no corresponde con la estructura de los ficheros en su sistema. Esta estructura lógica destaca las partes del proyecto que ustede necesita para acceder a menudo. La ventana de Proyectos no muestra ficheros compilados .class. Si quiere ver como el proyecto entero está representado físicamente en el disco, haga click en la pestaña Files.

Compilando y Ejecutando un Proyecto

Para aplicaciones que son accedidas a traves de un navegador, añada los atributos Codebase y Permissions en el manifiesto del fichero JAR antes de compilar el proyecto. En la pestaña Files, expanda la carpeta del proyecto para ver las subcarpetas y ficheros. Haga doble click sobre manifest.mf para abrir el fichero de manifiesto, y fije el atributo Codebase al lugar desde el cual la aplicación es cargada. Fije el atributo Permissions al nivel de permisos que la aplicación requiera. Vea Atributos para la Seguridad en el Fichero de Manifiesto JAR en la Plataforma Java, Edición Estándar de la Guía de Implantación para más información.

Para compilar su proyecto entero, haga click derecho en el nodo del proyecto en la ventana Proyects y elija Build Project.

La ventana Output se abre y muestra la salida del progreso de la compilación.

Si salida de la construcción concluye con la sentencia BUILD FAILED, probablemente tenga un error de sintaxis en su código. Los errores son informados en la ventana Output como texto hiperenlazado. Haga doble-click sobre uno de los hiperenlaces para navegar a la fuente del error. Entonces podrá arreglar el error y una vez de nuevo elija Build Project.

Cuando uste construye el proyecto ficheros bytecode .class son generados. Puede ver dónde los nuevos ficheros son generados abriendo la ventana Files y expandiendo la carpeta para el proyecto.

Para ejecutar su proyecto, haga click derecho sobre el nodo del proyecto en la ventana Projects y elija Run Project.

Nota: Puede también usar los comandos Build Main Project y Run Main Project en el menú principal. Sin embargo, estos comandos sólo funcionan sobre el proyecto que está actualmente designado como el proyecto "principal". Usted puede hacer que un proyecto sea el proyecto principal haciendo click derecho sobre su nodo en la ventana Projects y eligiendo Set Main Project.

Para obtener más consejos del uso del EID de Netbeans con el Tutorial de Java, vea "¡Hola, Mundo!" para el EID NetBeans.
