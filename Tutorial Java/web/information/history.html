¿Qué es y Qué es Viejo? La Historia del Tutorial

3 Marzo 2015 -

Esta versión del tutorial corresponde a la versión 8u40 del KDJ.

Esta versión incluye una nueva lección en el recorrido Desplieuge que describe cómo usar las herramientas de empaquetado de Java para generar aplicaciones auto-contenidas. Las aplicaciones auto-contenidas son aplicaciones Java que son empaquetadas con el Entorno de Ejecución de Java (JRE, en inglés) que necesitan para ejecutarse. Estas aplicaciones son instaladas en el disco local del usuario y lanzadas de la misma forma que las aplicaciones nativas. Vea Desplegando Aplicaciones Auto-Contenidas para más información.

19 Agosto 2014 -

Esta versión del tutorial corresponde a la versión 8u20 del KDJ. Esta versión incluye los siguientes cambios:

  • El nivel de seguridad Medio ya no está disponible en el Panel de Control de Java. Para ejecutar applets localmente, añadiéndolas a la lista de excepciones de sitios en el Panel de Control de Java. Vea Lista de Excepciones de Sitios para más información.

  • Una nueva propiedad de límite de procesamiento, maxElementDepth, ha sido añadida a JAXP. Vea Límites de Procesamiento para más información.

  • El atributo Entry-Point es un nuevo atributo para el fichero de manifiesto JAR. Vea Mejorando la Seguridad con Atributos del Manifiesto para más información.

18 Marzo 2014 -

Esta versión del tutorial se corresponde con la versión de KDJ 8. Vea Qué es Nuevo en el KDJ 8 para más información sobre las nuevas funcionalidades en esta versión, la cual incluye las siguientes:

  • Expresiones Lambda le habilita a tratar la funcionalidad como un argumento de método, o código como datos. Las expresiones Lambda le permiten expresar instancias de interfaces de método-único (referidos como interfaces funcionales) más compactadas.

  • Métodos por defecto son métodos en una interfaz que tienen una implementación. Habilitan que la nueva funcionalidad sea añadida a los interfaces de las librerias y asegura la compatibilidad binaria con código escrito para versiones más viejas de esos interfaces.

  • Operaciones de agregados le habilita a ejecutar operaciones de estilo-funcional sobre flujos de elementos, en particular, operaciones masivas sobre colecciones, tales como las transformaciones secuenciales o paralelas.

  • Fecha-Hora Estas IPA le habilitan a representar fechas y tiempos y manipular valores de fecha y tiempo. Soportan tanto el sistema de calendario de la Organización Internacional para la Estandarización (ISO) como otros calendarios globales comunmente usados.

  • Anotaciones de tipos pueden ser usadas en conjunción con los sistemas de tipos enchufables para la comprobación mejorada de tipos, y las anotaciones repetidas habilitan aplicar la misma anotación a una declaración o uso de tipo.

Aparte de arreglar errores tipográficos y errores, esta actualización también incluye lo siguiente:

  • El recorrido de Seguridad ha sido reestructurado; busque información de seguridad sobre applets y aplicaciones Java Web Start en la lección Applets de Java.

  • El atributo Permissions del fichero de manifiesto JAR es requerido independientemente de la configuración del nivel de seguridad. Vea Mejorando la Seguridad con Atributos de Manifiesto.

  • La lección Lo Básico de JDBC ha sido actualizada de forma que es compatible con Java DB 10.10, la última versión que KDJ 8 actualmente soporta. Esto incluye añadir que DERBY es una opción valida para el elemento del procedimiento PARAMETER STYLE para pasar parámetros a un procedimiento almacenado. Vea Creando Procedimientos Almacenados en Java DB con Scripts de SQL o la IPA de JDBC.

Esta versión del tutorial corresponde a la versión 7u51 del KDJ. Esta versión incluye lo siguiente:

  • Cuando firma un fichero JAR, se recomienda el sellado de tiempo de la firma. Una advertencia es mostrada si la signatura no tiene un sellado de tiempo. Vea Firmando Ficheros JAR para más información.

  • Puede obtener los nombres de los parámetros formales de cualquier método o constructor en KDJ 8 con el metodo java.lang.reflect.Executable.getParameters. Vea la sección Obteniendo Nombres de los Parámetros del Método.

  • El recorrido Interfaces y Herencia ha sido mejorado con información sobre los Métodos por Defecto en KDJ 8, los cuales son métodos en una interfaz que tienen una implementación. Además, puede definir métodos estáticos en las interfaces.

  • El recorrido Colecciones ha sido actualizado con nuevas Preguntas y Ejercicios para Interfaces y Operaciones de Agregación en KDJ 8. Estas actualizaciones le ayudan a enfocarse en algunos puntos importantes cuando trabaja con expresiones lambda, flujos, y operaciones de agregación.

10 Septiembre 2013 -

Esta versión del tutorial corresponde a la versión 7u40 del KDJ. Esta versión incluye lo siguiente:

  • Nos complace anunciar el lanzamiento del nuevo recorrido Fecha-Hora como parte de las actualizaciones del tutorial para el acceso temprano al KDJ 8. Las APIS Fecha-Hora, que vienen en KDJ 8, son el resultado del trabajo del JSR 310 y ofrece un modelo comprehensivo para la fecha y hora. El nuevo recorrido cubre las clases basadas en el sistema de calendario estándar de la ISO e incluye información sobre la conversión de valores de fecha y hora a otros sistemas de calendario.Hay también una página Legado de Fecha-Hora de como migrar de las clases java.util de fecha a las nuevas IPA de java.time. Usted puede descargar una versión de acceso temprano de KDJ 8 de java.net.
  • La lección Operaciones de Agregación de Datos ha sido añadida al recorrido Colecciones. Las operaciones de Agregación, una nueva funcionalidad viene en KDJ 8, iteran sobre colecciones en su nombre, lo cual le permite escribir código más conciso y eficiente que procesa elementos almacenados en las colecciones.
  • Ejecutar aplicaciones ricas de Internet (RIAs) a través de un navegador puede ser arriesgado para los usuarios debido a los intentos maliciosos de otros buscando vulnerabilidades a través del cual acceder al sistema del usuario. Directrices para la Seguridad de las Aplicaciones Ricas de Internet, en el recorrido de Despliegue, ofrece sugerencias para hacer sus RIAs menos vulnerables a los ataques.
  • Una nueva página, Filtrado de Etiquetas de Lenguaje y Búsqueda, ha sido añadido a la lección Configurando Locale en el recorrido Internacionalización. Esta página discute etiquetas del lenguaje, rangos de lenguajes, y listas de prioridad de lenguaje. Esto también incluye secciones sobre el filtrado de etiquetas del lenguaje y búsqueda de etiquetas del lenguaje, con multitud de ejemplos que puede compilar y ejecutar.
  • El recorrido JAXP ha sido actualizado con una nueva lección JAXP 1.5 y Nuevas Propiedades.

El empaquetado de descarga del tutorial, y los libros electrónicos, disponibles de la Página de descarga del Tutorial de Java SE, han sido actualizados para esta version.

18 Junio 2013 -

El empaquetado descargable del tutorial ha sido actualizado para esta versión.

  • Esta versión del tutorial corresponde a la versión 7u25 del KDJ. La versión 7u25 del KDJ introduce los nuevos atributos de manifiesto del fichero JAR que son usados para defender las Aplicaciones Ricas de Internet (RIAs) contra la reutilización de código no autorizado. Los atributos Permissions y Codebase son descritos en Mejorando la Seguridad del Fichero JAR. Las secciones Desplegando un Applet y Desplegando una Aplicación Java Web Start describen como incluir estos atributos en el manifiesto.
  • Consejos adicionales para desplegar RIAs han sido añadidos a Mejores Prácticas en Despliegue y Seguridad en las Aplicaciones Ricas de Internet.
  • Información sobre el sellado de tiempo de la firma de un fichero JAR ha sido añadido a Firmando Ficheros JAR. Dependiendo del nivel de seguridad fijado en el Panel de Control de Java, las RIAs que tiene un certificado expirado podrían no ser permitidas a ejecutarse. Sin embargo, si la firma tiene una sello de tiempo y el sello de tiempo muestra que el certificado era válido cuando se firmó la RIA, se permitirá la ejecución de la RIA.
