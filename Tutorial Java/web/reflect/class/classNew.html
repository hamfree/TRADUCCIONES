Recuperando Objetos de la Clase

El punto de entrada para todas las operaciones de reflexión es java.lang.Class. Con la excepción de java.lang.reflect.ReflectPermission, ninguna de las clases en java.lang.reflect tiene constructores públicos. Para obtener estas clases, es necesario invocar los métodos apropiados en Class. Hay varias formas de obtener una Class dependiendo de si el código tiene acceso a un objeto, el nombre de clase, un tipo, o una Class existente.

Object.getClass()

Si una instancia de un objeto está disponible, entonces la forma más simplete de obtener su Class es invocar a Object.getClass(). Por supuesto, esto solo funciona para tipos referencia porque todos heredan de Object. A continuación se muestran algunos ejemplos. 

Class c = "foo".getClass();

Devuelve la Class para String 

Class c = System.console().getClass();

Hay una única consola asociada con la máquina virtual que es devuelta por el método static System.console(). El valor devuelto por getClass() es la Class correspondiente a java.io.Console. 

enum E { A, B }
Class c = A.getClass();

A es una instancia de la enumeración E; por lo tanto getClass() devuelve la Class correspondiente al tipo de enumeración E. 

byte[] bytes = new byte[1024];
Class c = bytes.getClass();

Ya que las matrices son Objects, es también posible invocar getClass() en una instancia de una matriz. La Class devuelta corresponde a una matriz con el tipo de componente byte. 

import java.util.HashSet;
import java.util.Set;

Set<String> s = new HashSet<String>();
Class c = s.getClass();

En este caso, java.util.Set es una interfaz a un objeto del tipo java.util.HashSet. El valore devuelto por getClass() es la clase correspondiente a java.util.HashSet.

La Sintaxis de .class

Si el tipo está disponible pero no hay instancia entonces es posible obtener una Class agregando ".class" al nombre del tipo. Esta es también la forma más fácil de obtener la Class para un tipo primitivo. 

boolean b;
Class c = b.getClass();   // error en tiempo de compilación

Class c = boolean.class;  // correcto

Dese cuenta que la sentencia boolean.getClass() produciría un error en tiempo de compilación porque un boolean es un tipo primitivo y no se puede desreferenciar. La sintaxis .class devuelve la Class correspondiente al tipo boolean. 

Class c = java.io.PrintStream.class;

La variable c será la Class correspondiente al tipo java.io.PrintStream. 

Class c = int[][][].class;

La sintaxis .class se puede usar para recuperar una Class correspondiente a una matriz multi-dimensional de un tipo dado.

Class.forName()

Si el nombre completamente cualificado de una clase está disponble, es posible obtener la correspondiente Class usando el método estático Class.forName(). Este no puede utilizarse para los tipos primitivos. La sintaxis para los nombres de las clases de matriz se describen en Class.getName(). Esta sintaxis es aplicable para referencias y tipos primitivos. 

Class c = Class.forName("com.duke.MyLocaleServiceProvider");

Esta sentencia creará una clase a partir del nombre completamente cualificado proporcionado. 

Class cDoubleArray = Class.forName("[D");

Class cStringArray = Class.forName("[[Ljava.lang.String;");

La variable cDoubleArray contendrá la Class correspondiente a una matriz del tipo primitivo double (p.e. la misma como double[].class). La variable cStringArray contendrá la Class correspondiente a una matriz bidimensional de String (p.e. idéntica a String[][].class).

Campo TYPE para Envolturas de Tipo Primitivo

La sintaxis .class es una forma preferida y más conveniente para obtener la Class para un tipo primitivo; sin embargo hay otra forma de adquirir la Class. Cada uno de los tipos primitivos y void tiene una clase envoltorio en java.lang que se usa para la conversión de tipos primitivos a tipos referencia. Cada clase envoltorio contiene un campo llamado TYPE que es igual a la Class para el tipo primitivo que está siendo encapsulado. 

Class c = Double.TYPE;

Hay una clase java.lang.Double que se usa para envolver el tipo primitivo double siempre que se requiera un Object. El valor de Double.TYPE es idéntico al de double.class. 

Class c = Void.TYPE;

Void.TYPE es idéntico a void.class.

Métodos que Devuelven Clases

Hay varias IPA de Reflexión que devuelven clases pero estas sólo pueden ser accedidas si una Class ha sido ya obtenida de forma directa o indirecta.

Class.getSuperclass()
Devuelve la clase superior a la clase dada. 
Class c = javax.swing.JButton.class.getSuperclass();
La super clase de javax.swing.JButton es javax.swing.AbstractButton.
Class.getClasses()
Devuelve todas las clases públicas, interfaces, y enumeraciones qeu son miembros de la clase incluyendo miembros heredados. 
Class<?>[] c = Character.class.getClasses();
Character contiene dos clases miembro Character.Subset y Character.UnicodeBlock.
Class.getDeclaredClasses()
Devuelve todas las interfaces de clases, y enumeraciones que están explícitamente declaradas en esta clase. 
Class<?>[] c = Character.class.getDeclaredClasses();
Character contiene dos miembros públicos que son clases Character.Subset y Character.UnicodeBlock y una clase privada Character.CharacterCache.
Class.getDeclaringClass()
java.lang.reflect.Field.getDeclaringClass()
java.lang.reflect.Method.getDeclaringClass()
java.lang.reflect.Constructor.getDeclaringClass()
Devuelven la Class en la cual esos miembros fueron declarados. Las Declaraciones de Clases Anónimas no tendrán una clase declarante pero tendrán una clase que las encierra o envuelve. 
import java.lang.reflect.Field;

Field f = System.class.getField("out");
Class c = f.getDeclaringClass();
El campo out está declarado en System. 
public class MyClass {
    static Object o = new Object() {
        public void m() {} 
    };
    static Class<c> = o.getClass().getEnclosingClass();
}
La clase declarante de la clase anónima definida por o es null.
Class.getEnclosingClass()
Devuelve la clase de la clase que la encierra inmediatamente. 
Class c = Thread.State.class().getEnclosingClass();
La clase que encierra a la enumeración Thread.State es Thread. 
public class MyClass {
    static Object o = new Object() { 
        public void m() {} 
    };
    static Class<c> = o.getClass().getEnclosingClass();
}
La clase anónima definida por o está encerrada por MyClass.
