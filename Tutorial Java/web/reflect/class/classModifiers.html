Examinando Modificadores de Clases y Tipos

Una clase se puede declarar con uno o más modificadores que afectan a su comportamiento en tiempo de ejecución:

  • Modificadores de acceso: public, protected, y private
  • Modificador que requiere anulación (sobreescritura): abstract
  • Modificador que restringe a una instancia: static
  • Modificador que prohibe la modificación del valor: final
  • Modificador que fuerza un comportamiento estricto de punto flotante:strictfp
  • Anotaciones

No todos los modificadores están permitidos en todas las clases, por ejemplo una interfaz no puede ser final y una enumeración no puede ser abstract. java.lang.reflect.Modifier contiene declaraciones para todos los modificadores posibles. También contiene métodos que se pueden usar para decodificar el conjunto de modificadores devueltos por Class.getModifiers().

El ejemplo ClassDeclarationSpy muestra cómo obtener los componentes de la declaración de una clase incluyendo los modificadores, parámetros de tipos genéricos, interfaces implementadas, y la ruta de herencia. Desde que Class implementa la interfaz java.lang.reflect.AnnotatedElement es también posible consultar las anotaciones de tiempo de ejecución. 


import java.lang.annotation.Annotation;
import java.lang.reflect.Modifier;
import java.lang.reflect.Type;
import java.lang.reflect.TypeVariable;
import java.util.Arrays;
import java.util.ArrayList;
import java.util.List;
import static java.lang.System.out;

public class ClassDeclarationSpy {
    public static void main(String... args) {
	try {
	    Class<?> c = Class.forName(args[0]);
	    out.format("Class:%n  %s%n%n", c.getCanonicalName());
	    out.format("Modifiers:%n  %s%n%n",
		       Modifier.toString(c.getModifiers()));

	    out.format("Type Parameters:%n");
	    TypeVariable[] tv = c.getTypeParameters();
	    if (tv.length != 0) {
		out.format("  ");
		for (TypeVariable t : tv)
		    out.format("%s ", t.getName());
		out.format("%n%n");
	    } else {
		out.format("  -- No Type Parameters --%n%n");
	    }

	    out.format("Implemented Interfaces:%n");
	    Type[] intfs = c.getGenericInterfaces();
	    if (intfs.length != 0) {
		for (Type intf : intfs)
		    out.format("  %s%n", intf.toString());
		out.format("%n");
	    } else {
		out.format("  -- No Implemented Interfaces --%n%n");
	    }

	    out.format("Inheritance Path:%n");
	    List<Class> l = new ArrayList<Class>();
	    printAncestor(c, l);
	    if (l.size() != 0) {
		for (Class<?> cl : l)
		    out.format("  %s%n", cl.getCanonicalName());
		out.format("%n");
	    } else {
		out.format("  -- No Super Classes --%n%n");
	    }

	    out.format("Annotations:%n");
	    Annotation[] ann = c.getAnnotations();
	    if (ann.length != 0) {
		for (Annotation a : ann)
		    out.format("  %s%n", a.toString());
		out.format("%n");
	    } else {
		out.format("  -- No Annotations --%n%n");
	    }

        // production code should handle this exception more gracefully
	} catch (ClassNotFoundException x) {
	    x.printStackTrace();
	}
    }

    private static void printAncestor(Class<?> c, List<Class> l) {
	Class<?> ancestor = c.getSuperclass();
 	if (ancestor != null) {
	    l.add(ancestor);
	    printAncestor(ancestor, l);
 	}
    }
}

A continuación se muestran algunos ejemplos del resultado. La entrada del usuario está en cursiva. 

$ java ClassDeclarationSpy java.util.concurrent.ConcurrentNavigableMap
Class:
  java.util.concurrent.ConcurrentNavigableMap

Modifiers:
  public abstract interface

Type Parameters:
  K V

Implemented Interfaces:
  java.util.concurrent.ConcurrentMap<K, V>
  java.util.NavigableMap<K, V>

Inheritance Path:
  -- No Super Classes --

Annotations:
  -- No Annotations --

Esta es la declaración real para java.util.concurrent.ConcurrentNavigableMap en el código fuente: 

public interface ConcurrentNavigableMap<K,V>
    extends ConcurrentMap<K,V>, NavigableMap<K,V>
Tenga en cuenta que, dado que se trata de una interfaz, es implícitamente abstract.. El compilador agrega este modificador para cada interfaz. Además, esta declaración contiene dos parámetros de tipos genéricos, K y V. El código del ejemplo simplemente imprime los nombres de estos parámetros, pero es posible recuperar información adicional sobre ellos usando métodos de java.lang.reflect.TypeVariable. Las interfaces pueden también implementar otras interfaces como se muestra arriba. 
$ java ClassDeclarationSpy "[Ljava.lang.String;"
Class:
  java.lang.String[]

Modifiers:
  public abstract final

Type Parameters:
  -- No Type Parameters --

Implemented Interfaces:
  interface java.lang.Cloneable
  interface java.io.Serializable

Inheritance Path:
  java.lang.Object

Annotations:
  -- No Annotations --

Dado que las matrices son objetos en tiempo de ejecución, toda la información del tipo la define la máquina virtual de Java.En particular, las matrices implementan Cloneable y java.io.Serializable y su superclase directa es siempre Object. 

$ java ClassDeclarationSpy java.io.InterruptedIOException
Class:
  java.io.InterruptedIOException

Modifiers:
  public

Type Parameters:
  -- No Type Parameters --

Implemented Interfaces:
  -- No Implemented Interfaces --

Inheritance Path:
  java.io.IOException
  java.lang.Exception
  java.lang.Throwable
  java.lang.Object

Annotations:
  -- No Annotations --

Del camino de herencia se puede deducir que java.io.InterruptedIOException es una excepción comprobada porque RuntimeException no está presente. 

$ java ClassDeclarationSpy java.security.Identity
Class:
  java.security.Identity

Modifiers:
  public abstract

Type Parameters:
  -- No Type Parameters --

Implemented Interfaces:
  interface java.security.Principal
  interface java.io.Serializable

Inheritance Path:
  java.lang.Object

Annotations:
  @java.lang.Deprecated()

Esta salida muestra que java.security.Identity, una IPA obsoleta, posee la anotación java.lang.Deprecated . Esto puede ser utilizado por el código reflectante para detectar IPA obsoleto.

Nota: No todas las anotaciones están disponibles mediante reflexión. Solo aquellas que tienen una java.lang.annotation.RetentionPolicy de RUNTIME son accesibles. De las tres anotaciones predefinidas en el lenguaje @Deprecated, @Override, y @SuppressWarnings solo @Deprecated está disponible en tiempo de ejecución.
