Encontrando Constructores

Una declaración de constructor incluye el nombre, los modificadores, los parámetros, y una lista de excepciones que se pueden lanzar. La clase java.lang.reflect.Constructor proporciona una forma de obtener esta información.

El ejemplo ConstructorSift ilustra cómo buscar en los constructores declarados de una clase uno que tenga un parámetro de un tipo determinado. 


import java.lang.reflect.Constructor;
import java.lang.reflect.Type;
import static java.lang.System.out;

public class ConstructorSift {
    public static void main(String... args) {
        try {
            Class<?> cArg = Class.forName(args[1]);

            Class<?> c = Class.forName(args[0]);
            Constructor[] allConstructors = c.getDeclaredConstructors();
            for (Constructor ctor : allConstructors) {
                Class<?>[] pType  = ctor.getParameterTypes();
                for (int i = 0; i < pType.length; i++) {
                    if (pType[i].equals(cArg)) {
                    out.format("%s%n", ctor.toGenericString());

                    Type[] gpType = ctor.getGenericParameterTypes();
                    for (int j = 0; j < gpType.length; j++) {
                        char ch = (pType[j].equals(cArg) ? '*' : ' ');
                        out.format("%7c%s[%d]: %s%n", ch,
                               "GenericParameterType", j, gpType[j]);
                    }
                    break;
                    }
                }
            }

          // production code should handle this exception more gracefully
        } catch (ClassNotFoundException x) {
            x.printStackTrace();
        }
    }
}

Method.getGenericParameterTypes() consultará el Atributo de Firma en el fichero class si este está presente. Si el atributo no está disponible, recurre a Method.getParameterType() que no fue cambiado por la introducción de los genéricos. Los otros métodos con nombre getGenericFoo() para algún valor de Foo en reflexión se implementan de forma similar. La sintaxis para los valores devueltos de Method.get*Types() se describen en Class.getName() .

La sintaxis aceptable para expresar matrices de referencia y tipos primitivos para Class.forName() se describe en Class.getName(). Observe que el primer constructor listado es package-private, no public. Es devuelto porque el código de ejemplo usa Class.getDeclaredConstructors() en vez de Class.getConstructors(), que devuelve sólo constructores public.

Este ejemplo muestra que la búsqueda de argumentos de aridad variable (que tienen un número variable de parámetros) requiere el uso de la sintaxis de matrices: 

$ java ConstructorSift java.lang.ProcessBuilder "[Ljava.lang.String;"
public java.lang.ProcessBuilder(java.lang.String[])
      *GenericParameterType[0]: class [Ljava.lang.String;

Esta es la declaración real del constructor ProcessBuilder en el código fuente: 

public ProcessBuilder(String... command)

El parámetro se representa con una matriz de una única dimensión del tipo java.lang.String. Esto puede distinguirse de un parámetro que es explícitamente una matriz de java.lang.String invocando a Constructor.isVarArgs().

El ejemplo final informa la salida de un constructor que se ha declarado con un parámetro de tipo genérico: 

$ java ConstructorSift java.util.HashMap java.util.Map
public java.uti.HashMap(java.util.Map<? extends K, ? extends V>)
      *GenericParameterType[0]: java.util.Map<? extends K, ? extends V>

Los tipos de excepción se pueden recuperar de los constructores de forma similar a los métodos. Vea el ejemplo MethodSpy descrito en la sección Obteniendo Información del Tipo del Método para obtener más detalles.
