Lección: Miembros

La Reflexión define una interfaz java.lang.reflect.Member que se implementa por java.lang.reflect.Field, java.lang.reflect.Method, y java.lang.reflect.Constructor. Estos objetos serán descritos en esta lección. Para cada miembro, la lección describirá la IPA asociada para recuperar la información de la declaración y el tipo, cualquier operación única al miembro (por ejemplo, establecer el valor de un campo o invocar un método), y errores encontrados habitualmente. Cada concepto será ilustrado con ejemplos de código y resultados relacionados que se aproximan a algunos usos de reflexión esperados.

Nota: De acuerdo a la Especificación del Lenguaje Java, Java SE 7 Edición, los miembros de una clase son los componentes heredados del cuerpo de la clase incluyendo campos, métodos, clases anidadas, interfaces, y tipos enumerados. Ya que los constructores no se heredan, estos no son miembros. Esto difiere de las clases de implementación de java.lang.reflect.Member.

Campos

Los campos tienen un tipo y un valor. La clase java.lang.reflect.Field proporciona métodos para acceder a la información de tipo y establece y obtiene valores de un campo en un objeto dado.

  • Obteniendo los Tipos de los Campos describe cómo obtener los tipos genéricos y declarados de un campo
  • Recuperando y Analizando Modificadores de los Campos muestra cómo obtener porciones de la declaración del campo tales como public o transient
  • Obteniendo y Estableciendo los Valores de los Campos ilustra cómo acceder al valor del campo
  • Solución de Problemas describe algunos errores de codificación comunes que pueden causar confusión

Métodos

Los métodos tienen valores de retorno, parámetros, y pueden lanzar excepciones. La clase java.lang.reflect.Method proporciona métodos para obtener la información de tipo para los parámetros y el valor de retorno. Puede también usarse para invocar métodos en un objeto dado.

  • Obteniendo Información del Tipo del Método muestra cómo enumerar los métodos declarados en una clase y obtener información de tipos
  • Obteniendo los Nombre de los Parámetros del Método muestra cómo recuperar nombres y otra información de los parámetros de un método o constructor
  • Recuperando y Analizando Modificadores del Método describe cómo acceder y decodificar los modificadores y otra información asociada con el método
  • Invocando Métodos ilustra cómo ejecutar un método y obtener su valor de retorno
  • Solución de Problemas cubre los errores comunes encontrados cuando se buscan o invocan métodos

Constructores

La IPA de Reflexión para los constructores se definen en java.lang.reflect.Constructor y son similares a aquellos de los métodos, con dos mayores excepciones; primero, los constructores no tienen valores de retorno; segundo, la invocación de un constructor crea una nueva instancia de un objeto para una clase dada.

  • Encontrando Constructores ilustra cómo recuperar constructores con parámetros específicos
  • Recuperando y Analizando Modificadores del Constructor muestra cómo obtener los modificadores de una declaración de constructor y otra información sobre el constructor
  • Creando Nuevas Instancias de Clase muestra cómo instanciar una instancia de un objeto invocando su constructor
  • Solución de Problemas describe errores comunes que podrían encontrarse mientras se buscan o invocan constructores
