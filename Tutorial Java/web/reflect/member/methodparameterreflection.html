Obteniendo los Nombre de los Parámetros del Método

Puede obtener los nombres de los parámetros formales de cualquier método o constructor con el método java.lang.reflect.Executable.getParameters. (Las clases Method y Constructor extienden la clase Executable y por lo tanto heredan el método Executable.getParameters). Sin embargo, los ficheros .class no almacenan los nombres de los parámetros formales de forma predeterminada. Esto es porque muchas herramientas que producen y consumen ficheros class pueden no esperar la huella más larga estática y dinámica de ficheros .class que contienen nombres de parámetros. En particular, estas herramientas tendrían que manejar ficheros .class más grandes, y la Máquina Virtual de Java (MVJ) usaría más memoria. Además, algunos nombres de parámetros, tales como secret o password, podrían exponer información sobre métodos sensibles a la seguridad.

Para almacenar nombres de parámetros formales en un fichero .class particular, y así habilitar a la IPA de Reflexión a recuperar los nombres de los parámetros formales, compile el fichero fuente con la opción -parameters en el compilador javac.

El ejemplo MethodParameterSpy ilustra cómo recuperar los nombres de los parámetros formales de todos los constructores y métodos de una clase dada. El ejemplo también imprime otra información sobre cada parámetro.

El comando siguiente imprime los nombres de los parámetros formales de los constructores y métodos de la clase ExampleMethods. Nota: Recuerde compilar el ejemplo ExampleMethods con la opción del compilador -parameters:

El ejemplo MethodParameterSpy usa los métodos siguientes de la clase Parameter:

  • getType: Devuelve un objeto Class que identifica el tipo declarado para el parámetro.

  • getName: Devuelve el nombre del parámetro. Si el nombre del parámetro está presente, entonces este método devuelve el nombre proporcionado por el fichero .class. De otra forma, este método sintetiza un nombre de la forma argN, donde N es el índice del parámetros en el descriptor del método que declara el parámetro.

    Por ejemplo, suponga que compiló la clase ExampleMethods sin especificar la opción del compilador -parameters. El ejemplo MethodParameterSpy imprimiría lo siguiente para el método ExampleMethods.simpleMethod:

  • getModifiers : Devuelve un entero que representa varias características que los parámetros formales poseen. Este valor es la suma de los siguientes valores, si son aplicables al parámetro formal:

    Valor (en decimal) Valor (en hexadecimal Descripción
    16 0x0010 El parámetro forma está declarado como final
    4096 0x1000 El parámetro formal es sintético. Alternativamente, puede invocar al método isSynthetic.
    32768 0x8000 El parámetros está declarado implícitamente en el código fuente. Alternativamente, puede invocar el método isImplicit

  • isImplicit: Devuelve true si este parámetros está implícitamente declarado en el código fuente. Vea la sección Parámetros Implícitos y Sintéticos para obtener más información

  • isNamePresent: Devuelve true si el parámetro tiene un nombre de acuerdo al fichero .class.

  • isSynthetic: Devuelve true si el parámetro no es implicita ni explicítamente declarado en el código fuente. Vea la sección Parámetros Implícitos y Sintéticos para obtener más información.

Parámetros Implícitos y Sintéticos

Ciertos constructores están implícitamente declarados en el código fuente si no han sido escritos explícitamente. Por ejemplo, el ejemplo ExampleMethods no contiene un constructor. Un constructor predeterminado está implicitamente declarado para eso. El ejemplo MethodParameterSpy imprime información sobre el constructor declarado implícitamente de ExampleMethods: 

Number of declared constructors: 1
public ExampleMethods()

Considere el trozo siguiente desde MethodParameterExamples: 

public class MethodParameterExamples {
    public class InnerClass { }
}

La clase InnerClass es un constructor no estático clase anidada o clase interior. Un constructor para las clases internas está también implícitamente declarado. Sin embargi, este constructor contendrá un parámetro. Cuando el compilador de Java compila InnerClass, crea un fichero .class que representa un código similar al siguiente: 

public class MethodParameterExamples {
    public class InnerClass {
        final MethodParameterExamples parent;
        InnerClass(final MethodParameterExamples this$0) {
            parent = this$0; 
        }
    }
}

El constructor de InnerClass contiene un parámetros cuyo tipo es la clase que encierra InnerClass, que es MethodParameterExamples. Consecuentemente, el ejemplo MethodParameterExamples imprime lo siguiente: 

public MethodParameterExamples$InnerClass(MethodParameterExamples)
         Parameter class: class MethodParameterExamples
          Parameter name: this$0
               Modifiers: 32784
            Is implicit?: true
        Is name present?: true
           Is synthetic?: false

Ya que el constructor de la clase InnerClass está declarado implícitamente, su parámetro es implícito también.

Nota:

  • El compilador de Java crea una parámetro formal para el constructor de una clase anidada para habilitar al compilador a pasar una referencia (que representa la instancia inmediatamente adjunta) desde la expresión de craeción al constructor miembro de la clase.
  • El valor 32784 significa que el parámetro del constructor InnerClasses tanto final )(16) como implícito(32768).
  • El lenguaje de programación Java permite los nombres de variable con signos de dólar ($); sin embargo, por convención, los signos de dolar no se usan en nombres de variables.

Los constructores emitidos por un compilador de Java se marcan como synthetic si no corresponden a una construcción declarada explícitamete o implícitamente en el código fuente, a menos que sean métodos de inicialización de la clase. Los constructores sintéticos son artefactos generados por los compiladores que varían entre diferentes implementaciones. Considere el siguiente esbozo de MethodParameterExamples: 

public class MethodParameterExamples {
    enum Colors {
        RED, WHITE;
    }
}

Cuando el compilador Java encuentra un constructo enum, crea varios métodos que son compatibles con la estructura del fichero .class y proporcionan la funcionalidad esperada del constructor enum Colors que representa código similar al siguiente: 

final class Colors extends java.lang.Enum<Colors> {
    public final static Colors RED = new Colors("RED", 0);
    public final static Colors BLUE = new Colors("WHITE", 1);
 
    private final static values = new Colors[]{ RED, BLUE };
 
    private Colors(String name, int ordinal) {
        super(name, ordinal);
    }
 
    public static Colors[] values(){
        return values;
    }
 
    public static Colors valueOf(String name){
        return (Colors)java.lang.Enum.valueOf(Colors.class, name);
    }
}

El compilador de Java crea tres constructores y métodos para este constructo enum: Colors(String name, int ordinal), Colors[] values(), y Colors valueOf(String name). Los métodos values y valueOf están declarados implícitamente. En consecuencia, sus nombres de parámetros formales están implícitamente declarados también.

El constructor de enum Colors(String name, int ordinal) es un constructor por defecto y está implícitamente declarado. Sin embargo, los parámetros formales de este constructor (name y ordinal) no están implícitamente declarados. Ya que estos parámetros formales no están declarados ni explicíta ni implícitamente, son sintéticos. (Los parámetros formales para el constructor por defecto de una construcción enum no están implícitamente declarados porque compiladores diferentes no tienen que estar de acuerdo sobre la forma de este constructor; otro compilador de Java podría especificar diferentes parámetros formales para este. Cuando los compiladores compilan expresiones que usan constantes enum, sólo confían en los campos públicos estáticos de la construcción enum, que están implícitamente declarados, y no en sus constructores o como se inicializaron estas constantes).
