Obteniendo los Tipos de los Campos

Un campo puede ser de tipo referencia o primitivo. Hay ochos tipos primitivos: boolean, byte, short, int, long, char, float, y double. Un tipo de referencia es cualquier cosa que es directa o indirectamente una subclase de java.lang.Object incluyendo interfaces, matrices, y tipos enumerados.

El ejemplo FieldSpy imprime el tipo del campo y el tipo genérico dado el nombre completamente cualificado de una clase binaria y un nombre de campo. 


import java.lang.reflect.Field;
import java.util.List;

public class FieldSpy<T> {
    public boolean[][] b = { { false, false }, { true, true } };
    public String name  = "Alice";
    public List<Integer> list;
    public T val;

    public static void main(String... args) {
        try {
            Class<?> c = Class.forName(args[0]);
            Field f = c.getField(args[1]);
            System.out.format("Type: %s%n", f.getType());
            System.out.format("GenericType: %s%n", f.getGenericType());

            // production code should handle these exceptions more gracefully
        } catch (ClassNotFoundException x) {
            x.printStackTrace();
        } catch (NoSuchFieldException x) {
            x.printStackTrace();
        }
    }
}

La salida del ejemplo recuperar el tipo de tres campos públicos en esta clase (b, name, y el tipo parametrizado list), a continuación. La entrada del usuario está en cursivas. 

$ java FieldSpy FieldSpy b
Type: class [[Z
GenericType: class [[Z
$ java FieldSpy FieldSpy name
Type: class java.lang.String
GenericType: class java.lang.String
$ java FieldSpy FieldSpy list
Type: interface java.util.List
GenericType: java.util.List<java.lang.Integer>
$ java FieldSpy FieldSpy val
Type: class java.lang.Object
GenericType: T

El tipo para el campo b es una matriz de dos dimensiones de booleanos. La sintaxis para el nombre del tipo se describe en Class.getName().

El tipo para el campo val se informa como java.lang.Object porque los genéricos se implementan vía borrado del tipo que elimina toda la información relacionada con los tipos genéricos durante la compilación. Así T se reemplaza por el límite superior de la variable de tipo, en este caso, java.lang.Object.

Field.getGenericType() consultará el Atributo de Firma en el fichero de clase si está presente. Si el atributo no está disponible, recurre a Field.getType() que no se modificó con la introducción de genéricos. Los otros métodos en reflexión con el nombre getGenericFoo para algún valor de Foo se implelmenta de forma similar.
