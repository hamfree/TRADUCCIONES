Examinando las Enumeraciones

Reflexión proporciona tres IPA específicas de las enumeraciones:

Class.isEnum()
Indica si esta clase representa un tipo de enumeración
Class.getEnumConstants()
Recupera la lista de constantes de enumeración definidas por la enumeración en el orden que fueron declaradas.
java.lang.reflect.Field.isEnumConstant()
Indica si este campo representa un elemento de un tipo enumerado

Algunas veces es necesario recuperar dinámicamente la lista de constantes de enumeración; en código no reflectivo esto se consigue invocando el método estático immplícitamente declarado values() sobre la enumeración. Si una instancia de un tipo enumerado no está disponible la única forma de obtener una lista de posibles valores es invocar a Class.getEnumConstants() ya que es imposible instanciar un tipo enumerado.

Dado un nombre completamente cualificado, el ejemplo EnumConstants muestra como recuperar una lista ordenada de constantes en una enumeración usando Class.getEnumConstants(). 


import java.util.Arrays;
import static java.lang.System.out;

enum Eon { HADEAN, ARCHAEAN, PROTEROZOIC, PHANEROZOIC }

public class EnumConstants {
    public static void main(String... args) {
	try {
	    Class<?> c = (args.length == 0 ? Eon.class : Class.forName(args[0]));
	    out.format("Enum name:  %s%nEnum constants:  %s%n",
		       c.getName(), Arrays.asList(c.getEnumConstants()));
	    if (c == Eon.class)
		out.format("  Eon.values():  %s%n",
			   Arrays.asList(Eon.values()));

        // el código de producción debería manejar esta excepción con más gracia
	} catch (ClassNotFoundException x) {
	    x.printStackTrace();
	}
    }
}

Ejemplos de la salida a continuación. La entrada del usuario está en cursiva. 

$ java EnumConstants java.lang.annotation.RetentionPolicy
Enum name:  java.lang.annotation.RetentionPolicy
Enum constants:  [SOURCE, CLASS, RUNTIME]

Ya que las enumeraciones son clases, se puede obtener otra información usando la misma IPA de Reflexión descrita en las secciones de Campos, Métodos, y Constructores de este recorrido. El código EnumSpy ilustra cómo usar esta IPA para obtener información adicional sobre la declaración de enumeración. El ejemplo usa Class.isEnum() para restringir el conjunto de clases examinadas. También usa Field.isEnumConstant() para distinguir constantes de enumeración de otros campos en la declaración de enumeración (no todos los campos son constantes de enumeración). 


import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;
import java.lang.reflect.Member;
import java.util.List;
import java.util.ArrayList;
import static java.lang.System.out;

public class EnumSpy {
    private static final String fmt = "  %11s:  %s %s%n";

    public static void main(String... args) {
        try {
            Class<?> c = Class.forName(args[0]);
            if (!c.isEnum()) {
                out.format("%s is not an enum type%n", c);
                return;
            }
            out.format("Class:  %s%n", c);

            Field[] flds = c.getDeclaredFields();
            List<Field> cst = new ArrayList<Field>();  // constantes de enumeración
            List<Field> mbr = new ArrayList<Field>();  // campos miembro
            for (Field f : flds) {
            if (f.isEnumConstant())
                cst.add(f);
            else
                mbr.add(f);
            }
            if (!cst.isEmpty())
                print(cst, "Constant");
            if (!mbr.isEmpty())
                print(mbr, "Field");

            Constructor[] ctors = c.getDeclaredConstructors();
            for (Constructor ctor : ctors) {
                out.format(fmt, "Constructor", ctor.toGenericString(),
                       synthetic(ctor));
            }

            Method[] mths = c.getDeclaredMethods();
            for (Method m : mths) {
                out.format(fmt, "Method", m.toGenericString(),
                       synthetic(m));
            }

            // el código de producción debería manejar esta excepción con más gracia
        } catch (ClassNotFoundException x) {
            x.printStackTrace();
        }
    }

    private static void print(List<Field> lst, String s) {
        for (Field f : lst) {
            out.format(fmt, s, f.toGenericString(), synthetic(f));
        }
    }

    private static String synthetic(Member m) {
        return (m.isSynthetic() ? "[ synthetic ]" : "");
    }
}

La salida muestra que la declaración de java.lang.annotation.RetentionPolicy solo contiene las tres constantes de enumeración. Las constantes de enumeración se exponen como campos public static final. El campo, el constructor y los métodos se generan por el compilador. El campo $VALUES se relaciona con la implementación del método values().

Nota: Por varias razones, incluyendo el soporte para la evolución del tipo enumerado, el orden de declaración de las constantes de enumeración es importante. Class.getFields() y Class.getDeclaredFields() no garantice que el orden de los valores devueltos coincida con el orden en el código fuente declarante. Si se requiere el ordenamiento por una aplicación, use Class.getEnumConstants().
