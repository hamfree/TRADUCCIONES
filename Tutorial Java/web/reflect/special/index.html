Lección: Matrices y Tipos Enumerados

Desde la perspectiva de la máquina virtual de java, las matrices y los tipos enumerados (o enumeraciones) son simplemente clases. Muchos de los métodos en Class se pueden usar sobre estos. Reflexión proporciona alguna IPA específica para matrices y enumeraciones. Esta lección usa una serie de ejemplos de código para describir como distinguir cada uno de estos objetos desde otras clases y operar con ellos. Se examinan también varios errores.

Matrices

Las matrices tienen un tipo componente y una longitud (que no es parte del tipo). Las matrices pueden ser manipuladas en su totalidad o componente por componente. Reflexión proporciona la clase java.lang.reflect.Array para el último propósito.

  • Identificando los Tipos del Matriz describe cómo determinar si un miembro de clase es un campo de tipo matriz
  • Creando Nuevas Matrices ilustra cómo crear nuevas instancias de matrices con tipos de componente simples y complejos
  • Obteniendo y estableciendo Matrices y sus Componentes muestra cómo acceder a los campos de tipo matriz y acceder individualmente a los elementos de la matriz
  • Solución de Problemas cubre los errores comunes y conceptos erróneos de programación

Tipos Enumerados

Las enumeraciones se tratan muy parecido a las clases ordinarias en el código de reflexión. Class.isEnum() indica si una Class representa una enumeración. Class.getEnumConstants() recupera las constantes de enumeración definidas en una enumeración. java.lang.reflect.Field.isEnumConstant() indica si un campo es de un tipo enumerado.

  • Examinando las Enumeraciones ilustra cómo recuperar unas constantes de enumeración y otros campos, constructores, y métodos
  • Obteniendo y Estableciendo Campos en los Tipos Enumerados muestra cómo establecer y obtener campos con un valor constante de enumeración
  • Solución de Problemas describe los errores comunes asociados con las enumeraciones
