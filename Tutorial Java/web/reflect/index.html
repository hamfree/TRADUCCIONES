Recorrido: La IPA de Reflexión

Usos de la Reflexión

La reflexión es usada habitualmente por programas que requieren la capacidad de examinar o modificar el comportamiento en tiempo de ejecución de aplicaciones que se ejecutan en la máquina virtual de Java. Esta es una característica relativamente avanzada y solo debe ser utilizada por desarrolladores que tengan un conocimiento sólido de los fundamenteos del lenguaje. Con esa advertencia en mente, la reflexión es una técnica poderosa y puede permitir que las aplicaciones realicen operaciones que de otra manera serían imposibles.

Funciones de Extensibilidad
Una aplicación puede hacer uso de clases externas definidas por el usuario creando instancias de objetos de extensibilidad utilizando sus nombres completos.
Navegadores de Clase y Entornos de Desarrolo Visual
Un navegador de clases debe poder enumerar los miembros de una clase.Los entornos de desarrollo visual pueden beneficiarse de haceer uso del tipo de información disponible en reflexión para ayudar al desarrollador a escribir código correcto. code.
Depuradores y Herramientas de Prueba
Los depuradores necesitan ser capaces de examinar los miembros privados en las clases. Las unidades de prueba pueden hacer uso de la reflexión para llamar sistemáticamente a un IPA de conjunto detectable definido en una clase, para asegurar un alto nivel de cobertura de código en un conjunto de pruebas.

Inconvenientes de la Reflexión

La reflexión es poderosa, pero no debe ser usada indiscriminadamente. Si es posible realizar una operación sin usar reflexión, entonces es preferible evitar usarla. Se deben tener en cuenta las siguientes preocupaciones al acceder al código mediante la reflexión.

Sobrecarga de Rendimiento
Debido a que la reflexión involucra tipos que se resuelven dinámicamente, no se pueden realizar ciertas optimizaciones de máquinas virtuales Java. En consecuencia, las operaciones reflectantes tienen un rendimiento más lento que sus contrapartes no reflectantes y deben evitarse en secciones de código que se llaman con frecuencia en aplicaciones sensibles al rendimiento.
Restricciones de Seguridad
La reflexión requiere un permiso de tiempo de ejecución que puede no estar presente cuando se ejecuta bajo un gestor de seguridad. Esta es una importante consideración para el código que tiene que ejecutarse en un contexto de seguridad restringido, como un applet.
Exposición de Partes Internas
Dado que la reflexión permite que el código realice operaciones que serían ilegales en código no reflectante, como acceder a campos y métodos privados, el uso de la reflexión puede producir efectos secundarios inesperados, que pueden hacer que el código sea disfuncional y puede destruir la portabilidad. El código reflectante rompe las abstracciones y, por lo tanto, puede cambiar el comportamiento con las actualizaciones de la plataforma.

Lecciones del Recorrido

Este recorrido cubre los usos comunes de la reflexión para acceder y manipular clases, campos, metodos, y constructores. Cada lección contiene ejemplos de código, consejos, e información sobre solución de problemas.

Clases
Esta lección muestra las diversas formas de obtener un objeto Class y usarlo para examinar las propiedades de una clase, incluyendo su declaración y contenidos.
Miembros
Esta lección describe cómo usar la IPA de Reflexión para encontrar los campos, métodos, y constructores de una clase. Se proporcionan ejemplos para establecer y obtener valores de campos, invocar métodos, y crear nuevas instancias de objetos usando constructores específicos.
  Matrices y Tipos Enumerados
Esta lección introduce dos tipos especiales de clases: matrices, que se generan en tiempo de ejecución, y tipos enum, que definen instancias únicas de objetos con nombre. El código de ejemplo muestra cómo recuperar el tipo componente para una matriz y cómo establecer y obtener campos con matrices o tipos enum.
Nota: 

Los ejemplos en esta lección están diseñados para experimentar con la IPA de Reflexión. El manejo de las excepciones por lo tanto no es el mismo que sería usado en código de produción. En particular, en código de producción no se recomienda volcar trazas de pila que sean visibles por el usuario.
