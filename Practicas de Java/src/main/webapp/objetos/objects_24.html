Nunca confíe en finalize

No puede confiar en finalize para reclamar siempre los recursos. En cambio, si un objeto tiene recursos que necesitan ser recuperados, debe definir un método para ese propósito - por ejemplo un método close, o un método dispose. Luego, el llamante debe llamar explícitamente a ese método cuando termine con el objeto, para garantizar que se recuperen los recursos relacionados.

Con JDK 7+, la característica try-with-resources puede ser usada para reclamar los recursos de cualquier objeto que implemente AutoCloseable .

Para una discusión extensa, por favor vea el tema "Evite los Finalizadores" en Java Efectivo.

El método System.runFinalizersOnExit(boolean) es obsoleto.
