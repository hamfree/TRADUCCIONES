Forzar la habilitación de las aserciones

Es algunas veces útil requerir que las aserciones esten siempre habilitadas para una clase particular en tiempo de ejecución.

Ejemplo

Si NuclearReactor se carga dentro de un entorno donde se han deshabilitado las aserciones, entoces su inicializador estático lanzará una RuntimeException.

Dos ejecuciones del ejemplo de NuclearReactor dan:
>java -cp . -enableassertions NuclearReactor
Proceder normalmente
>java -cp . NuclearReactor
Exception in thread "main" java.lang.ExceptionInInitializerError
Caused by: java.lang.RuntimeException: Las aserciones deben estar habilitadas para el reactor nuclear.
        at NuclearReactor.<clinit>(NuclearReactor.java:18) 

public final class NuclearReactor {

  public static void main(String... arguments){
    NuclearReactor reactor = new NuclearReactor("Fermi");
    System.out.println("Proceder normalmente...");
  }

  /**
  * Evite que esta clase se ejecute con aserciones deshabilitadas.
  *
  * Si las aserciones están deshabilitadas, entonces se lanzará 
  * una RuntimeException al cargar la clase.
  */
  static {
    boolean hasAssertEnabled = false;
    assert hasAssertEnabled = true; // raro - ¡un efecto secundario intencional!
    if(!hasAssertEnabled){
      throw new RuntimeException("Las aserciones deben estar habilitadas para el reactor nuclear.");
    }
  }

  /**
  * @param aName es no nulo.
  */
  public NuclearReactor(String aName){
    //NO use assert para verificar los parámetros pasados a 
    //métodos no privados
    if(aName == null){
      throw new IllegalArgumentException("Name no puede ser nulo.");
    }
    fName = aName;
  }

  public String getName(){
    return fName;
  }

  // PRIVADO
  private final String fName;
}
