Determinar si las Strings son iguales

Para determinar si dos objetos String coinciden exactamente, debe casi siempre usar el método equals, y no el operador ==.

El método equals compara el contenido actual de las Strings, usando la representación Unicode subyacente, mientras que == compara solo las identidades de los objetos, usando sus direcciones en memoria (que generalmente no se desea).

Ejemplo

import java.util.Objects;

/**
* Ilustra la incorrección de usar == para las comparaciones
* habituales de String.
*/
public final class EqualStrings {

  /**
  * Pase la cadena "blah" en la línea de comando 
  * para ejercitar el código.
  */
  public static void main(String... arguments) {
    String text = (arguments.length == 1 ? arguments[0] : null);
    testUsingEquals(text);
    testUsingOperator(text);
  }

  // PRIVADO
  private static final String BLAH = "blah";

  private static void testUsingEquals(String aString) {
    if (BLAH.equals(aString)) {
      log("Equals: SI, el usuario introdujo \"blah\".");
    }
    else {
      log("Equals: NO, el usuario no introdujo \"blah\".");
    }
  }

  private static void testUsingOperator(String aString) {
    if (aString == BLAH) {
      log("operador == : SI, el usuario introdujo \"blah\".");
    }
    else {
      log("operador == : NO, el usuario no introdujo \"blah\".");
    }
  }
  
  private static void log(String aMessage){
    System.out.println(Objects.toString(aMessage));
  }
}

Un ejemplo de ejecución de esta clase:

>java -cp . EqualStrings blah
Equals: SI, el usuario introdujo "blah".
operador ==: NO, el usuario no introdujo "blah".

Tenga en cuenta que x.equals(y) generará una NullPointerException si x es null, pero no generará dicha excepción si solo y es null. En el ejemplo anterior, por ejemplo, BLAH.equals(aString) se usa en vez de aString.equals(BLAH), ya que aString puede ser null.

Es verdad que el método intern puede usarse para permitir comparaciones significativas usando ==. (Vea El Lenguaje de Programación Java por Arnold, Gosling, y Holmes para más información.) La técnica intern está pensada como una optimización del rendimiento y, por lo general, debe evitarse ya que es más compleja.
