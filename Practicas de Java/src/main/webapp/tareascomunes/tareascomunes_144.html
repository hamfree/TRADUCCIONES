Enviar un correo

La IPA de JavaMail se usa para enviar correos electrónicos:
  • la últma versión requiere el KDK 1.7+. Versiones más antiguas están disponibles para KDJ más antiguos.
  • no es parte del núcleo de Java SE, sino una extensión opcional.
  • en la Edición Empresarial de Java (JEE), JavaMail es proporcionado por el contenedor.

Los paquetes de JavaMail se pueden acceder de dos formas:

  • si está usando un contenedor JEE, entonces JavaMail está automáticamente disponible en el entorno de ejecución.
  • al colocar javax.mail.jar (anteriormente mail.jar) en la ruta de clases (classpath). Si está usando una versión más antigua de JavaMail con el KDJ 1.4 o 1.5, entonces necesita agregar activation.jar a la ruta de clases (classpath). Proviene del Marco de Trabajo de Activación de JavaBeans.
La IPA de JavaMail usa un archivo de propiedades para leer nombres de servidor y configuración relacionada. Estas configuraciones sobreescribirán cualquier configuración predeterminada del sistema. Alternativamente, la configuración puede ser establecida directamente en el código, usando la IPA de JavaMail.

Ejemplo

Un archivo de configuración de correo electrónico (para ejecutar el ejemplo, substituya por valores válidos los elementos descomentados):
 

# Archivo de configuración para javax.mail
# Si un valor para un elemento no es proporcionado, entonces
# se usarán los predeterminados del sistema. Estos elementos pueden
# también ser establecidos en código.

# Equipo cuyos servicios de correo serán usados
# (Valor predeterminado : localhost)
mail.host=mail.blah.com

# Dirección de respuesta que aparece en los correos electrónicos
# (Valor predeterminado : username@host)
mail.from=webmaster@blah.net

# Otros posibles elementos incluyen:
# mail.user=
# mail.store.protocol=
# mail.transport.protocol=
# mail.smtp.host=
# mail.smtp.user=
# mail.debug=

Una clase que usa este archivo para enviar un correo electrónico: 

import java.util.*;
import java.io.*;
import java.nio.file.*;
import javax.mail.*;
import javax.mail.internet.*;

/**
* Demostración simple de usar la IPA de javax.mail.
*
* Ejecutar desde la lónea de comandos. Por favor edite la implementación
* para usar las direcciones de correo correctas y el nombre del equipo.
*/
public final class Emailer {

  public static void main( String... aArguments ){
    Emailer emailer = new Emailer();
    //los dominios de estas direcciones de correo deben ser válidos,
    //o el ejemplo fallará:
    emailer.sendEmail(
      "de_bla@bla.com", "a_bla@bla.com",
       "Probando 1-2-3", "bla bla bla"
    );
   }

  /**
  * Envía un único correo electrónico.
  */
  public void sendEmail(
    String aFromEmailAddr, String aToEmailAddr,
    String aSubject, String aBody
  ){
    //Aquí, no se usa el argumento Authenticator argument (es nulo).
    //Los Authenticators se usan para preguntar el usuario para el nombre de user
    //usuario y contraseña.
    Session session = Session.getDefaultInstance(fMailServerConfig, null);
    MimeMessage message = new MimeMessage(session);
    try {
      //la dirección "from" puede establecerse en el código, o establecerse en el 
      //archivo de configuración bajo "mail.from" ; aquí, se usa el último estilo 
      //message.setFrom(new InternetAddress(aFromEmailAddr));
      message.addRecipient(
        Message.RecipientType.TO, new InternetAddress(aToEmailAddr)
      );
      message.setSubject(aSubject);
      message.setText(aBody);
      Transport.send(message);
    }
    catch (MessagingException ex){
      System.err.println("No puedo enviar el correo electrónico. " + ex);
    }
  }

  /**
  * Permite que la configuración se actualice en tiempo de ejecución, en lugar de
  * requerir un reinicio.
  */
  public static void refreshConfig() {
    fMailServerConfig.clear();
    fetchConfig();
  }

  // PRIVADO

  private static Properties fMailServerConfig = new Properties();

  static {
    fetchConfig();
  }

  /**
  * Abrir un archivo de texto específico que contiene parámetros del 
  * servidor de correo, y rellenar un objeto de Properties correspondiente.
  */
  private static void fetchConfig() {
    //Este archivo contiene las propiedaes de configuración de javax.mail mencionadas antes.
    Path path = Paths.get("C:\\Temp\\MyMailServer.txt");
    try (InputStream input = Files.newInputStream(path)) {
      fMailServerConfig.load(input);
    }
    catch (IOException ex){
      System.err.println("No puedo abrir y cargar el archivo de propiedades del servidor de correo.");
    }
  }
}
