Mide el rendimiento de la aplicación

Mide el tamaño de tu aplicación en la memoria y su velocidad de ejecución.

Medir el rendimiento durante la construcción puede ayudar a identificar los problemas de rendimiento principales lo más rápido posible, lo que casi siempre es beneficioso. Si se hace evidente un problema importante, es probable que sea necesario un rediseño inmediato. Si no hay ningún problema importante, entonces, si es necesario, se pueden realizar optimizaciones menores más tarde, cuando la construcción esté casi completa. (Muchos dirían que no es aconsejable preocuparse por optimizaciones menores durante la construcción, ya que las optimizaciones a veces pueden hacer que el código sea más oscuro).

Estas herramientas de monitorizacion vienen con el KDJ:

  • Java Mission Control
  • Visual VM
  • JConsole

Si las herramientas anteriores no están disponibles, entonces está atascado con medios más primitivos para medir el tamaño de una aplicación en la memoria. Dependen de tu plataforma. Por ejemplo:

  • NT: Gestor de Tareas, busque java.exe o javaw.exe
  • Unix: obtenga el id del proceso "ps -e | grep java", páselo a pmap, como en "pmap -x 6598", y busque la entrada de KB totales en la columna Private

Además, el propio KDJ viene con herramientas básicas de creación de perfiles. Por ejemplo:

  • Runtime.freeMemory
  • Runtime.totalMemory
  • java -Xprof MyClass
  • java -Xrunhprof:[opciones] MyClass
Otras herramientas incluyen vmstat para máquinas Unix, y perfmon para máquinas Windows.
