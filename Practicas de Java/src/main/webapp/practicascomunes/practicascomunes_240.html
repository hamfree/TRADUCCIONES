Etiquete o marque los interfaces

Una interfaz de etiqueta (también llamada una interfaz de marcador) es simplemente una interfaz sin métodos. Algunos ejemplos de interfaces de etiqueta del KDJ:
  • Serializable
  • EventListener

¿Porqué definir una interfaz sin métodos? Ya que no hay métodos, una interfaz de etiqueta nunca puede definir comportamiento, al menos no en el sentido habitual. Sin embargo, incluso aunque no tiene métodos, una interfaz de etiqueta siempre lleva información de tipo. En algunos casos, la información de tipo en sí misma puede ser usada para resolver un problema.

Por ejemplo, el mecanismo de serialización de Java requiere que un objeto implemente Serializable antes de que lo serialize. Como se indicó en su javadoc:

La interfaz de serialización no tiene métodos o campos y sirve solo para identificar las semánticas de ser serializable.

Las herramientas a menudo usarán instanceof o reflexión para inspeccionar objetos, para ver si implmementan una interfaz de etiqueta dada. Pero esta no es la única forma de que una etiqueta interfaz pueda ser útil. Hay algunos casos comunes para los que podría considerar definir su propia interfaz de etiqueta, incluso sin el uso correspondiente de instanceof o reflexión:

  • objetos inmutables (o clases inmutables, si usted prefiere)
  • clases singleton
Las ventajas de hacerlo son:
  • enfatiza los aspectos de alto nivel, importantes, de una clase, no expresado de otra manera en el código.
  • tanto para lectores humanos como para herramientas, permite la identificación rápida de clases que tienen propiedades específicas.
  • el javadoc de la interfaz etiqueta es un hogar natural para documentar todas las características de elementos que implementan la interfaz dada.
