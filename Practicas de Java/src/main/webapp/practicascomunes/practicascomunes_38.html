No rompa la portabilidad

La portabilidad entre diferentes sistemas operativos es una de las ventajas principales de usar Java.

Aqui tiene algunas formas de asegurar que su aplicación permanezca portable:

  • casi nunca codifique caracteres de nueva línea como \r o \n. En su lugar, use el método System. getProperty(String) para referise a los elementos que dependan del sistema, tal como los terminadores de línea y los separadores de ruta.
  • no use nombres de archivo y rutas codificadas - use la clase Path, y permita que las ubicaciones de los archivos se configuren durante el despliegue, o en tiempo de ejecución.
  • recuerde que algunos sistemas son sensibles a mayúsculas y minúsculas en los nombres de los archivos (Unix), mientras otros no lo son (Windows).
  • Sea muy cauteloso con Runtime.exec y Method.invoke .
  • no confíe en la programación de subprocesos y prioridades de hilo para definir la lógica del programa.
  • no use métodos native.
  • no codifique tamaños y posiciones de los elementos del IGU - use un LayoutManager en su lugar.
  • no confíe en una resolución de pantalla específica - use java.awt.Toolkit.getScreenResolution en su lugar. Para las aplicaciones web, CSS media queries puede ayudarle a adaptarse a un amplio rango de tamaños de pantalla.
  • no codifique colores usando valores numéricos - use las constantes simbólicas en java.awt.Color y java.awt.SystemColor en su lugar.
  • no codifique tamaños del texto.

Los elementos comunes System.getProperty(String) pueden colocarse en una clase de constantes de propósito general: 

/**
* Constantes recopiladas de utilidad muy general.
*
* Todas las constantes deben ser inmutables.
* No se pueden construir instancias de esta clase.
*/
public final class Consts {

  /**
  * Se previene la construcción del objeto fuera de esta clase.
  */
  private Consts(){
    //vacío
  }

  /**
  * Solo se refiere a primitivas y objetos inmutables.
  *
  * Las matrices presentan un problema ya que las matrices son siempre mutables.
  * NO USE campos matriz públicos, estáticos y finales.
  * Un estilo es en su lugar usar una List inmodificable, construida en 
  * un bloque de inicialización estático.
  *
  * Otro estilo es usar una matriz privada y envolverla de esta forma:
  * <pre>
  *  private static final Vehicle[] PRIVATE_VEHICLES = {...};
  *  public static final List VEHICLES =
  *         Collections.unmodifiableList(Arrays.asList(PRIVATE_VEHICLES));
  * </pre>
  */

  //caracteres
  public static final String NUEVA_LINEA = System.getProperty("line.separator");
  public static final String SEPARADOR_FICHERO = System.getProperty("file.separator");
  public static final String SEPARADOR_RUTA = System.getProperty("path.separator");

  public static final String CADENA_VACIA = "";
  public static final String ESPACIO = " ";
  public static final String PUNTO = ".";
  public static final String TABULADOR = "\t";

  //signos algebraicos
  public static final int POSITIVO = 1;
  public static final int NEGATIVO = -1;
  public static final String SIGNO_MAS = "+";
  public static final String SIGNO_MENOS = "-";
}
