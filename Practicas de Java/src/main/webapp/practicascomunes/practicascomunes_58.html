Convenciones de nombrado

Es común usar una convención de nombrado de variables para distinguir entre campos, argumentos, y variables locales.

Dentro del cuerpo de un método, todos estos tipos de variables pueden aparecer. Muchos encuentran que las convenciones de nombrado son una ayuda útil para comprender rápidamente la implementación de un método. Estas convenciones solo tienen un propósito: para verter la comprensión en el cerebro del lector lo más rápido posible.

Para entender un método, necesita entender sus datos. Una parte importante de entender los datos es entender dónde se definieron - como un campo, argumento, o variable local. Las convenciones de nombrado que distinguen estos casos usualmente permiten al lector entender una implementación más rápidamente, ya que no necesitan escanear la clase para determinar dónde se definen los elementos.

Algunos estilos diferentes son:

  • campo: fBlah, _blah, blah_, this.blah, m_blah, myBlah
  • argumento: aBlahBlah, pBlah
  • variable local: blahBlah
  • constante: BLAH_BLAH
También, algunos usan una convención de nombres para nombres de tipos, para distinguir entre interfaces y clases regulares:
  • interfaz: IBlahBlah
  • clase: BlahBlah, CBlahBlah
Tenga en cuenta que esta convención de 'I' e 'C' es fundamentalmente diferente de las otras convenciones mencionadas arriba. Esta convención es visible al llamador, mientras las otras convenciones de nombres, siendo confinadas a la implementación, no lo son.

El código de ejemplo en este sitio usualmente sigue estas convenciones:

  • campo: fBlahBlah
  • argumento: aBlahBlah
  • variable local: blahBlah
  • constante: BLAH_BLAH
  • clase: BlahBlah
  • interfaz: BlahBlah (igual que la clase)
El valor de vuelta de un método a veces también se le da un nombre convencional.
