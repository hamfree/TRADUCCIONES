Evite los tipos crudos

Los tipos crudos se refieren a usar un tipo genérico sin especificar un parámetro de tipo. Por ejemplo, List es un tipo crudo, mientras que List<String> es un tipo parametrizado.

Cuando se introdujeron los genéricos en el KDJ 1.5, los tipos crudos se mantuvieron solo para mantener la compatibilidad hacia atrás con versiones más antiguas de Java. Aunque usar tipos crudos es aún posible, deben ser evitados:

  • usualmente requieren conversiones (moldeos)
  • no son tipos seguros, y algunas clases importantes de errores solo aparecerán en tiempo de ejecución
  • son menos expresivos, y no son autodocumentados de la misma forma que los tipos parametrizados

Ejemplo

import java.util.*;

public final class AvoidRawTypes {

  void withRawType(){
    //La List cruda no se autodocumenta, 
    //no indica explícitamente lo que puede contener
    List stars = Arrays.asList("Arcturus", "Vega", "Altair");
    Iterator iter = stars.iterator();
    while(iter.hasNext()) {
      String star = (String) iter.next(); //se necesita conversión (moldeo)
      log(star);
    }
  }
  
  void withParameterizedType(){
    List<String> stars = Arrays.asList("Spica", "Regulus", "Antares");
    for(String star : stars){
      log(star);
    }
  }
  
  private void log(Object aMessage) {
    System.out.println(Objects.toString(aMessage));
  }
}
