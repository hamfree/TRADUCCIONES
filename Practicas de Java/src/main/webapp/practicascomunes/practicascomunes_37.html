Parámetros de salida

Un método puede ocasionalmente necesitar el usar un parámetro para su valor de retorno - lo que podría llamarse libremente un "parámetro de salida" o un "parámetro de resultado". El llamador crea un objeto de parámetro de salida, y después se lo pasa a un método que cambia el estado del objeto (sus datos). Cuando el método vuelve, el llamador después examina este nuevo estado.

Parámetros de salida:

  • deben ser visibles tanto en el método como en su llamador
  • no puede ser null, ya que el objeto necesita ser creado por el llamador
  • no pueden ser objetos inmutables - una String, por ejemplo, no se puede usar como un parámetro de salida, ya que no puede cambiar su estado
  • probablemente debería usarse solo ocasionalmente, y después de una cuidadosa consideración
Ejemplo

Aquí, hay un parámetro de salida (aCookieValue) y un valor de retorno booleano: si el parámetro de salida está efectivamente poblado, entonces el valor de retorno se establece en true. 

import javax.servlet.http.*;

public final class CookieValue {

   /**
   * Determina si una petición de usuario contiene una cookie particular, y si si es así 
   * entonces coloca su valor en un parámetro "out".
   * 
   * Ejemplo
   * <pre>
   * if ( CookieValue.getCookieValue("ID", aRequest, aIdValue) ) {
   *    //usa aIdValue
   * } 
   * </pre>
   * 
   * @param aCookieName el nombre no nulo adjunto a la Cookie.
   * @param aRequest es la petición no nula reenviada desde el navegador del usuario que 
   * va a ser examinada por la presencia de la cookie dada.
   * @param aCookieValue es el parámetro "out" no nulo en el cual el 
   * valor de la cookie se coloca - si se encuentra - para examinación posterior por el 
   * llamador; debe tener una longitud de cero.
   * @throws IllegalArgumentException si aCookieValue.length() > 0.
   */
   static boolean getCookieValue(
     String aCookieName, HttpServletRequest aRequest, final StringBuilder aCookieValue
   ){
      if (aCookieValue.length()>0) throw new IllegalArgumentException();
      boolean userHasCookie = false;
      Cookie[] cookies = aRequest.getCookies();
      if (cookies != null){
        for(Cookie cookie : cookies){
          if (aCookieName.equals(cookie.getName())){          
            userHasCookie = true;
            //cambia el estado del parámetro de salida (aCookieValue)
            aCookieValue.append(cookie.getValue());
          }
        }
      }
      return userHasCookie;
   }
}

Un diseño alternativo es devolver un valor null si la cookie no está en realidad presente. Esto reemplazaría el valor devuelto booleano actual con un valor de retorno String que debe ser comprobado por null. Sin embargo, ese diseño alternativo parece más débil, ya que parece prudente evitar el uso de null en los valores de retorno.
