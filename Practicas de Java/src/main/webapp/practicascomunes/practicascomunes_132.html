Considere generadores de código

Algunos tipos de clases comparten la misma estructura básica, y difieren solo en detalles. Enumeraciones Seguras y los Modelos de Objetos centrados en los datos son dos ejemplos. En tales casos, las ayudas para la generación rápida de código se convierten en muy útiles. Estos pueden tomar varias formas:
  • una plantilla simple de texto destinada a copiar y pegar
  • un generador de código pasivo - genera ficheros fuentes que pueden necesitar ediciones posteriores. Si las ediciones posteriores se realizan, entonces se perderán si el generador de código se ejecuta una segunda vez.
  • un generador de código activo - genera ficheros fuente que no necesitarán ediciones posteriores (la idea aquí es usar el generador de código como parte de la compilación real, en lugar de depender de él para una operación única)
  • un generador de código de ida y vuelta - genera ficheros fuentes que pueden necesitar ediciones posteriores. Si las ediciones posteriores se realizan, entonces no se perderán si el generador de código se ejecuta una segunda vez, ya que detectará tales ediciones, y las usa para actualizar su modelo interno.
Los generadores de código a menudo usan descripciones textuales simples y ad hoc como entrada. El sitio web codegeneration.net es una referencia útil para la generación de código.

El uso extensivo de generación de código es a menudo llamado Desarrollo Basado en Modelos (DBM), o Arquitectura Basada en Modelos (ABM).
