Incluya 'desde', excluya 'hasta'

Hay una convención en el JDK que involucra rangos de valores desde..hasta . La convención es que se incluya desde, mientra se excluye hasta

Algunos ejemplos del KDJ:

  • String.substring(from,to)
  • List.sublist(from,to)
  • Arrays.copyOfRange(T[],from, to)

Rangos de Fecha
Un problema relacionado es específico con valores de fechas. Es común tener un rango de fechas especificado usando sólo fechas (si la porción horaria), mientras que los datos subyacentes en realidad incluyen tanto una fecha como una hora. Esto puede facilmente a errores de por uno.

Por ejemplo, tome la expresión siguiente: 

2010-01-01 <= x <= 2010-01-31
entonces lo siguiente es verdad:

Fecha x ¿En rango?
2010-01-01 00:00:00 Sí
2010-01-01 01:00:00 Sí
2010-01-30 00:00:00 Sí
2010-01-30 23:59:59 Sí
2010-01-31 00:00:00 Sí (por lo general)
2010-01-31 00:00:01 No
2010-01-31 02:00:00 No

(La advertencia para 2010-01-31 00:00:00 se hace ya que dependerá de los detalles de cómo se hace la comparación por la clase que modela la fecha/hora; por lo general, será dentro del rango).

La tabla de arriba muestra que las horas después de 2010-01-31 00:00:00 son excluídas del rango. Esto es habitualmente no deseable. En este caso, necesita alterar la comparación ligeramente a: 

2010-01-01 <= x < (2010-01-31 + 1 día)

De nuevo, esto usa el estilo incluya-desde, excluya-hasta mencionado arriba.
