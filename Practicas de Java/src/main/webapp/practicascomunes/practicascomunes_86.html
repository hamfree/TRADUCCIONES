Use System.exit con cuidado

System.exit debe ser utilizado con cuidado. El método normal de terminar un programa es terminar todos los subprocesos del usuario.

Casos en los cuales es apropiado System.exit:

  • scripts de utilidad
  • Aplicaciones de escritorio, en los cuales el subproceso de usuario de distribución de eventos se crea en segundo plano. Aquí, el programa puede ser terminado llamando a System.exit, o, por ejemplo, estableciendo: 
		
          JFrame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE)
Del Libro de Cocina de Portabilidad de Oracle:

"El método System.exit fuerza la terminación de todos los subprocesos en la máquina virtual de Java. Esto es drástico... System.exit debe reservarse para una salida de error catastrófica, o para casos donde un programa está destinado a ser utilizado como una utilidad de script que puede depender del código de salida del programa."
