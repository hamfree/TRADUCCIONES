Use final generosamente

Use la palabra clave final liberalmente para comunicar sus intenciones.

La palabra clave final tiene más de un significado:

  • una clase final no puede ser extendida
  • un método final no puede ser sobreescrito
  • los campos, parámetros, y variables locales final no pueden cambiar su valor una vez establecido
En el último caso, el "valor" para los primitivos se entiende en el sentido usual, mientras que "valor" para objetos significa la identidad del objeto, no su estado. Una vez la identidad de una referencia a un objeto final se establece, puede aún cambiar su estado, pero no su identidad (es decir, no puede volver a apuntar la referencia de un objeto a algún otro objeto).

Declarar campos primitivos como final automáticamente asegura la seguridad para multiproceso para ese campo.

Algunos habitualmente declaran los parámetros como final, ya que este es casi siempre el comportamiento deseado. Otros encuentran esto detallado, y de poco beneficio real.

Usar consistentemente final con las variables locales (cuando es apropiado) también puede ser útil. Llama la atención sobre las variables locales no final, que normalmente tienen más lógica asociada (por ejemplo, variables de resultado, acumuladores, variables de bucle). Muchos encuentran esto farragoso. Un enfoque razonable es usar ocasionalmente final para variables locales, pero solo si existe alguna condición inusual, por lo que hacer explícito final puede llamar la atención sobre al menos una variable local no final en el método; esto sirve para distinguir rápidamente las variables locales no-final de las otras.

Usando final:

  • comunica claramente su intención
  • permite al compilador y a la máquina virtual realizar optimizaciones menores
  • marca claramente los elementos que tienen un comportamiento más simple - final dice, "si está buscando complejidad, no la va a encontrar aquí".
Ejemplo 
import java.util.*;
import java.lang.reflect.Field;

/** Esta clase no puede ser extendida, ya que es final. */
public final class Boat {

  public Boat(final String aName, final int aLength, final Date aDateManufactured){
    fName = aName;
    fLength = aLength;
    //hace una copia defensiva de la fecha
    fDateManufactured = new Date(aDateManufactured.getTime());

    //no compila ya que los elementos son final:
    //aDateManufactured = null;
    //aLength = 0;
  }

  /** No puede ser sobreescrita, ya que la clase en sí misma es final. */
  public void setDate(final Date aNewDate){
    //incluso aunque el campo es final, su estado puede cambiar:
    fDateManufactured.setTime(aNewDate.getTime());

    //no compila, ya que el campo es final:
    //fDateManufactured = aNewDate;
  }

  /** Devuelve la puntuación de carrera más alta. */
  public Integer bestRaceScore(){
    //la referencia de resultado no puede ser final, ya que puede ser
    //redirigida a diferentes objetos
    Integer result = Integer.valueOf(0); 
    //final Integer result = Integer.valueOf(0); //no compila
    
    //este ejemplo es artificial, ya que fRaceScores podría ser 
    //referenciado directamente aquí...
    final List<Integer> scores = fRaceScores;
    for(Integer score : scores){
      if (score > result){
        result = score; //vuelve a apuntar al valor máximo
      }
    }
    return result;
  }
  
  //..omitido

  // PRIVADO
  private final String fName;
  private final int fLength;
  private List<Integer> fRaceScores = new ArrayList<>();
  private final Date fDateManufactured;
}
