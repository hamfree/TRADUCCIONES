Use enumeraciones para restringir los argumentos

Las enumeraciones seguras son útiles para restringir los argumentos que deben tener un pequeño conjunto de valores, tales como un boolean o un conjunto restringido de Strings o ints.

Como un ejemplo del IPA de servlets, recuerde este método para la creación de una HttpSession:

aRequest.getSession(true);

Cualquier lector que no está familiarizado con la especificación de este método se verá forzado a buscar el significado del literal boolean antes de que puedan entenfer la llamada de método. Un estilo alternativo es definir una variable bien nombrada que explique el intento:

boolean CREATE_IF_ABSENT = true;
aRequest.getSession(CREATE_IF_ABSENT);

Sin embargo, el llamador no está obligado a usar ese estilo. Si se usa una enumeración de tipo seguro para el argumento, entonces el llamador se ve obligado a adoportar un estilo que tiene mayor claridad.

aRequest.getSession(HttpServletRequest.CreateIfAbsent.YES);

Aquí, CreateIfAbsent sería una enumeración segura estática anidada con dos elementos, YES y NO. Este estilo tiene estas ventajas:

  • se fuerza al método a ser más inteligible en el punto de llamada
  • la necesidad de buscar la especificación se reduce en gran medida
  • sirve como recordatorio para aquellos que ya están familiarizados con la especificación
  • tiene seguridad en tiempo de compilación (no un problema con un argumento boolean, pero importantes si el argumento es una String o un int)
Este estilo puede ser un poco detallado. Podría ser una de esas raras ocasioens donde un static import de la clase enum es apropiada, para evitar usar los nombres completamente cualificados.

Ejemplo 1

Aquí tiene una clase esbozo que ilustra una forma alternativa de createSession:

import javax.servlet.http.HttpSession;

/** Clarifica la llamada al métod con un argumento enum.  */
public final class HttpRequestWithEnum {

  /**
  * Ejemplo de llamada a método con un argumento enum.
  */
  public static void main(String... aArgs){
     HttpRequestWithEnum request = new HttpRequestWithEnum();
     HttpSession session = request.createSession(CreateIfAbsent.YES);
  }
  
  public HttpSession createSession(CreateIfAbsent aCreateIfNeeded){
    return null; //aplastado
  }
  
  /** La enumeración en sí misma.  */
  public enum CreateIfAbsent {YES, NO};
}
Ejemplo 2

Aquí tiene un segundo ejemplo que usa una enumeración en vez de una literal booleano.

import java.io.*;

public final class LawnMower {

  enum Power{ON, OFF};

  void switchPower(Power aPower){
     if (Power.ON == aPower){ 
       //omitido..
     }
     else if (Power.OFF == aPower){
       //omitido..
     }
  }

  //..omitido

  public static void main(String... args){
    LawnMower mower = new LawnMower();
    //esto es más claro que pasar un literal booleano:
    mower.switchPower(Power.ON);
  }

}
