Use los imports estáticos raramente

Los Static imports permiten que los elementos estáticos de una clase sean referenciados en otra sin cualificación. Usados indiscriminadamente, estos probablemente harán el código más difícil de entender, no más fácil de entender.

Ejemplo

import java.util.*;
import static java.util.Collections.*;

public final class StaticImporter {

  public static void main(String... aArgs){
    List<String> things = new ArrayList<>();
    things.add("blah");

    //Esto parece una simple llamada de un método que pertenece a esta clase:
    List<String> syncThings = synchronizedList(things);

    //Sin embargo, en realidad resuelve en :
    //List<String> syncThings = Collections.synchronizedList(things);
  }
}

Un ejemplo en el cual un import estático es probablemente aceptable es una clase de constantes. Por ejemplo, una aplicación científica o de ingeniería podría hacer un uso amplio de Math.PI. Un import static de java.lang.Math.* permitiría a una clase reemplazar Math.PI con PI.

Más generalmente, una aplicación de negocios podría definir una clase de constantes, e importarla estáticamente. Esto permitiría que Consts.NEW_LINE sea referenciada como NEW_LINE, por ejemplo.
