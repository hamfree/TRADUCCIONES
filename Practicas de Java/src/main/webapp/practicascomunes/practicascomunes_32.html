Interfaz para constantes

Es posible colocar constantes ampliamente utilizadas en una interfaz. Si una clase implementa tal interfaz, entonces la clase puede referirse a estas constantes sin un nombre de clase cualificado. Esto es solo una ventaja menor.

La funcionalidad de importaciones estáticas debe siempre ser considerada un reemplazo para esta práctica.

Colocar constantes en una interfaz era una técnica popular en los primeros días de Java, pero ahora muchos lo consideran un uso desagradable de las interfaces, ya que las interfaces deben tratar con los servicios proporcionados por un objeto, no sus datos. Además, las constantes utilizadas por una clase suelen ser un detalle de implementación, pero colocarlas en una interfaz las promueve a la IPA pública de la clase.

Ejemplo 

interface OlympicMedal {
  static final String ORO = "Oro";
  static final String PLATA = "Plata";
  static final String BRONCE = "Bronce";
}
Aquí tiene un ejemplo del uso de esta interfaz para referenciar sus constantes: 
public final class OlympicAthlete implements OlympicMedal {

  public OlympicAthlete(int aId){
    //..omitido
  }

  //..omitido

  public void winEvent(){
    //el atleta obtiene una medalla de oro
    //tenga en cuenta que la referencia NO está cualificada, como en 
    //OlympicMedal.ORO
    fMedal = ORO;
  }

  // PRIVADO
  private String fMedal; //posiblemente nulo
}
