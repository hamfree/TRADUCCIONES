Las estructuras son ocasionalmente útiles

Algunos lenguajes de programación (tales como C y Algol 68) definen formalmente la idea de una struct, como una forma simple de agrupar junto datos relacionados. Aunque la Especificación del Lenguaje Java no define formalmente las estructuras (Nota del Traductor: structs en inglés), el equivalente de una estructura ciertamente se puede crear en Java. Aquí tiene un ejemplo de una estructura que recopila datos relacionados con los libros:
import java.math.BigDecimal;

/** Estructura simple que contiene información de libros. */
final class Book {
  String TITLE;
  String ISBN;
  Integer NUM_PAGES;
  BigDecimal PRICE;
}
A muchos les disgusta el uso de tales estructuras porque están expuestas al abuso. Cualquier miembro público mutable es peligroso, ya que no hay encapsulación. De hecho, esta es una objeción válida a su uso generalizado. Sin embargo, las structs pueden aún ser ocasionalmente útiles, y uno no debería tener miedo de usarlas cuando sea apropiado. En particular, si los datos no son públicos ni mutables, entonces las objeciones usuales a las structs pierden mucha de su fuerza.

Por ejemplo, si la clase Book anterior era en realidad una clave anidada privada, y era simplemente un pequeño portador de datos tonto usado solo dentro de una sola clase, entonces habría poca objeción a usarlo:

  • como todos los datos son privados, la struct sería invisible fuera de la clase
  • ya que los miembros de la struct son todos inmutables, entonces no hay forma de cambiar el estado de esos objetos miembro
