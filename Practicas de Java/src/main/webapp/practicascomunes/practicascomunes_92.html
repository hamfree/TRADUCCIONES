Campos autoencapsulados

Dentro de la implementación de una clase, es común referirse directamente a los campos privados. Hay otro estilo, en el cual todas las referencias internas a los campos son indirectas, y proceden a través de los métodos apropiados get y set. A esto se le llama autoencapsulación.

Autoencapsulación:

  • se necesita para la inicialización perezosa
  • es un poco más difícil de leer que el acceso directo al campo
  • puede establecer una política de sincronización para el campo
  • permite a las subclases sobreescribir los métodos get y set , y proporcionar su propia definición de cómo se comportan los campos
Ejemplo 
public final class Infant {

  public Infant (final String aName, final int aWeight, final String aMood){
    setName(aName);
    setWeight(aWeight);
    setMood(aMood);
  }

  public void becomeHungry(){
    //en lugar fMood = "Cranky", usar:
    setMood("Cranky");
  }

  public String toString(){
    //usar métodos get en vez del acceso directo
    final StringBuilder result = new StringBuilder();
    result.append("Infante - Nombre: + ");
    result.append(getName());
    result.append(" Peso: ");
    result.append(getWeight());
    result.append(" Estado anímico: ");
    result.append(getMood());
    return result.toString();
  }

  public String getName(){
    return fName;
  }
  public void setName(String aName){
    fName = aName;
  }

  public int getWeight(){
    return fWeight;
  }
  public void setWeight(final int aWeight){
    fWeight = aWeight;
  }

  public String getMood(){
    return fMood;
  }
  public void setMood(final String aMood){
    fMood = aMood;
  }

  //..otros métodos omitidos

  // PRIVADO
  private String fName;
  private int fWeight;
  private String fMood;
}
