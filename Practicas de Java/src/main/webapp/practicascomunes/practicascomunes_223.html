Use @Override generosamente

La anotación @Override funciona más o menos como un nuevo modificador de método. Su presencia indica al compilador que el método anotado debe sobreescribir un método de supertipo existente, ya sea de una interfaz, o una clase base abstracta.

(Cuando fue por primera vez introducida en el KDJ 1.5, @Override solo era aplicable a métodos definidos en una superclase, y no en un supertipo. Es decir, era originalmente aplicado a métodos sobreescritos en una clase base, pero no a métodos de implementación definidos en una interfaz. Esto se cambió en el KDJ 7, y ahora @Override se puede aplicar a ambos).

El uso más común para @Override es con los métodos de Object: 

@Override public String toString(){...}
@Override public boolean equals(Object){...}
@Override public int hashCode(){...}
La razón principal de que se creara @Override fue tratar con los simples (pero desagradables) errores tipográficos. Por ejemplo, un métodos declarado erróneamente como 
public int hashcode(){...}
es de hecho no una anulación - el nombre del método tiene todas las letras en minúsculas, así que no coincide exactamente con el nombre del método hashCode(). Sin embargo compilará perfectamente. Tal error es fácil de cometer, y difícil de capturar, lo que es una combinación peligrosa. Usar la anotación @Override le evita de cometer tales errores.

Debe tener el hábito de usar @Override siempre que sobreescriba un método de superclase, o implemente un método de interfaz.
