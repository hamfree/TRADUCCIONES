Multiples sentencias return

Algunos programadores encuentran que, ocasionalmente, algunos métodos se vuelven más claros si se usan múltiples sentencias return, en vez del usual punto único de salida.

Se puede abusar facilmente de esta técnica, y debe ser usada con cuidado. Se usa principalmente para reemplazar las estructuras anidadas de if.

Las sentencias múltiples return parecen funcionar bien para "código de protección" en el principio de un método, en el cual el cuerpo principal del método se ejecuta solo si se satisfacen ciertas condiciones.

Ejemplo

Aquí, equals tiene múltiples sentencias return, ya que una prueba exitosa implica que la siguiente comparación es redundante. La alternativa es usar múltiples sentencias if, las cuales algunos encontrarán menos legible.

import java.util.*;

public final class Auto {

  @Override public boolean equals(Object aThat) {
    if (this == aThat) return true;
    if (!(aThat instanceof Auto)) return false;

    Auto that = (Auto)aThat;
    return
      EqualsUtil.areEqual(this.fName, that.fName) &&
      EqualsUtil.areEqual(this.fNumDoors, that.fNumDoors) &&
      EqualsUtil.areEqual(this.fOptions, that.fOptions) &&
      EqualsUtil.areEqual(this.fGasMileage, that.fGasMileage) &&
      EqualsUtil.areEqual(this.fColor, that.fColor) &&
      Arrays.equals(this.fMaintenanceChecks, that.fMaintenanceChecks)
    ;
  }

  //..omitido

  // PRIVADO

  private String fName;
  private int fNumDoors;
  private List<String> fOptions;
  private double fGasMileage;
  private String fColor;
  private Date[] fMaintenanceChecks;
}
