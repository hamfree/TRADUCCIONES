No use caracteres tabulador

La mayoría de los editores de código le permiten usar caracteres TAB en su código fuente. Tales caracteres TAB aparecerán al final de cada línea de su código, controlando la indentación. El problema es que los caracteres TAB son ambiguos inherentemente, y diferentes herramientas pueden representar los caracteres TAB de diferentes formas. Lo que puede parecer razonable al autor, en el momento de escribir en su entorno particular, puede ser representado en una forma significativamente diferente en alguna otra herramienta.

Por ejmeplo, digamos que el código siguiente aparece en un editor como:

import java.util.*;

public final class TabsOne {

  void decideWhatToDo(){
    if(! bored){
      continueWhatYourAreDoing();
    }
    else {
      if (! hasCarpalTunnelIssues()) {
        code();
      }
      else {
        eatSomething();
      }
    }
  }

}
Si está usando caracteres TAB, entonces esto podría facilmente aparecer en otra herramienta como: 
import java.util.*;

public final class TabsTwo {

    void decideWhatToDo(){
            if(!bored){
                    continueWhatYourAreDoing();
            }
            else {
                    if (! hasCarpalTunnelIssues()) {
                            code();
                    }
                    else {
                            eatSomething();
                    }
            }
    }
}

Incluso peor, algunas herramientas pueden incluso representar al texto con TABs con indentación negativa:

import java.util.*;

public final class TabsThree {

    void decideWhatToDo(){
      if(!bored){
    continueWhatYourAreDoing();
      }
      else {
        if (! hasCarpalTunnelIssues()) {
      code();
        }
        else {
      eatSomething();
        }
      }
    }
}
La mayoría de la gente consideraría tal código como mucho más difícil de leer, y molesto. Así, usar caracteres TAB en código fuente a menudo se considera altamente indeseable.

Debe usar de 2 a 4 espacios para indentar su código, no caracteres TAB. Los EID modernos le permiten vincular las pulsaciones TAB a un número dado de espacios en su lugar.

Aquí hay una cita relacionada de Code Complete, por Steve McConnell:

"El estudio concluyó que la sangría de dos a cuatro espacios era óptima. Curiosamente, muchos sujetos en el experimento sintieron que la sangría de seis espacios era más fácil de usar que las sangrías más pequeñas, a pesar de que sus puntuaciones eran más bajas. Eso es probablemente porque seis los espacios se ven agradables. Pero independientemente de lo bonito que se vea, la sangría de seis espacios resulta ser menos legible. Este es un ejemplo de una colisión entre el atractivo estético y la legibilidad ".
