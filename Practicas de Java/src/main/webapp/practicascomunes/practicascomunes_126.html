No declare variables locales antes de usarlas

La mayoría de las variables locales se declaran e inicializan en la misma línea, en el punto en el método donde tanto su valor inicial está disponible y la variable en sí misma es inmediatamente útil.

Declarar variables locales sin usarlas inmediatamente puede incrementar innecesariamente su alcance. Esto decrementa la legibilidad, e incremente la probabilidad de error.

Hay dos casos comunes donde a una variable local se le asigna algún valor inicial (típicamente null, 0, false, o una String vacía):

  • las variables que necesitan ser visibles fuera de un bloque try, y son así declaradas e inicializadas justo antes del bloque try (en código moderno usando try-with-resources, esto es ahora relativamente raro)
  • algunas variables de bucle, que son inicializadas a algún valor predeterminado justo antes del bucle
Ejemplo

Aquí, input y output son ejemplos de variables locales siendo inicializadas a null, ya que necesitan ser visibles tanto en los bloques try y finally.

Tambíen, line es un ejemplo de variable de bucle declarada e inicializada fuera del bucle.

Tenga en cuenta que este ejemplo usa KDJ 6, simplemente para ilustrar el punto En KDJ 7, try-with-resources sería usada para automáticamente cerrar los flujos, y el problema de las referencias de flujo siendo inicializadas a null no ocurrirían.

import java.io.*;

/** KDJ 6 o anterior. */
public class ReadWriteTextFile {

  /**
  * Obtiene los contenidos enteros de un fichero de texto, y lo devuelve en un String.
  * Este estilo de implementación no lanza Excepciones al llamador.
  *
  * @param aFile es un fichero que ya existe y puede ser leído.
  */
  static public String getContents(File aFile) {
    //...comprobaciones sobre aFile son omitidas
    StringBuilder contents = new StringBuilder();
    
    try {
      //usa almacenamiento en búfer, leyendo una línea a la vez
      //¡FileReader siempre asume que la codificación predeterminada es correcta!
      BufferedReader input =  new BufferedReader(new FileReader(aFile));
      try {
        String line = null; //no declarada dentro del bucle while
        /*
        * readLine es un poco peculiar :
        * devuelve el contenido de una línea MENOS el retorno de carro.
        * devuelve null solo para el FIN del flujo.
        * devuelve una String vacía si aparecen dos retornos de carro en una fila.
        */
        while (( line = input.readLine()) != null){
          contents.append(line);
          contents.append(System.getProperty("line.separator"));
        }
      }
      finally {
        input.close();
      }
    }
    catch (IOException ex){
      ex.printStackTrace();
    }
    
    return contents.toString();
  }

  /**
  * Cambia el contenido del fichero de texto por completo, sobreescribiendo cualquier
  * texto existente.
  *
  * Este estilo de implementación lanza todas las excepciones al llamador.
  *
  * @param aFile es un fichero existente que puede ser escrito.
  * @throws IllegalArgumentException si el parámetro no cumple.
  * @throws FileNotFoundException si el fichero no existe.
  * @throws IOException si se encuentra un problema durante la escritura.
  */
  static public void setContents(File aFile, String aContents)
                                 throws FileNotFoundException, IOException {
    if (aFile == null) {
      throw new IllegalArgumentException("El archivo no debe ser nulo.");
    }
    if (!aFile.exists()) {
      throw new FileNotFoundException ("El archivo no existe: " + aFile);
    }
    if (!aFile.isFile()) {
      throw new IllegalArgumentException("No debería ser un directorio: " + aFile);
    }
    if (!aFile.canWrite()) {
      throw new IllegalArgumentException("El archivo no se puede escribir: " + aFile);
    }

    //usa almacenamiento en búfer
    Writer output = new BufferedWriter(new FileWriter(aFile));
    try {
      //¡FileReader siempre asume que la codificación predeterminada es correcta!
      output.write( aContents );
    }
    finally {
      output.close();
    }
  }

  /** Prueba simple.   */
  public static void main (String... aArguments) throws IOException {
    File testFile = new File("C:\\Temp\\blah.txt");
    System.out.println("Contenido del archivo original: " + getContents(testFile));
    setContents(testFile, "El contenido de este archivo se ha sobrescrito ...");
    System.out.println("Contenido de archivo nuevo: " + getContents(testFile));
  }
}
