Oculte los .html y los .jsp

En una aplicación web de Java, tiene tres elecciones para implementar una página:
  • una página HTML (.html)
  • una Página de Servidor Java (Java Server Page, en inglés) (.jsp)
  • un servlet (.do, o similar)
Debe considerar siempre el uso de un servlet, independientemente de la implementación subyacente. Solo un servlet puede manejar los tres casos.

Es simple escribir un servlet para reenviar a un archivo HTML o JSP. En este caso, una convención de nombrado es a menudo útil. Por ejemplo, una URL que termina en: 

some/path/blah.do
podría ser mapeada internamente por un servlet a un archivo: 
..some/path/blah.html
a alguna ubicación en el arbol de fuentes de la aplicación.

Si siempre usa un servlet (o controlador), entonces la implementación subyacente está oculta del navegador. Esa es una buena cosa, ya que le deja completamente libre para cambiar de opinión más adelante, sin necesitar hacer trabajo extra para las URLs marcadas existentes.

Si elije no usar un servlet, y sus URLs terminan en .html o .jsp, entonces, si más tarde cambia la implementación, necesitará decidir que hacer con los enlaces obsoletos:

  • o hacer el trabajo de redirigir los enlaces antiguos al nuevo mundo (un poco tedioso)
  • o dejar que se rompan los enlaces antiguos (lo cual es malo)
