La sobrecarga puede ser complicada

Los metodos sobrecargados deben ser hechos con cuidado. El compilador decide qué versión de un método sobrecargado será llamado basado en el tipo declarado en tiempo de compilación, no el tipo en tiempo de ejecución. Para el caso en el que los métodos sobrecargados tienen el mismo número de argumentos, las reglas relativas a esta decisión pueden ser un poco complicadas.

Si puede haber confusión, puede simplificar el diseño:

  • use diferentes nombres de método, y evite la sobrecarga por completo
  • retenga la sobrecarga, pero asegúrese de que cada método tenga un número distinto de argumentos
Además, se recomienda que no se use varargs cuando un método se sobrecargue, ya que esto hace más difícil determinar qué sobrecarga se está llamando.

Recordatorio:

La sobrecarga requiere métodos con distintas firmas. La firma de un método incluye su nombre y la lista ordenada de sus tipos de argumento. Todos los otros elementos que aparecen en un encabezado de método, tales como las excepciones, el tipo de retorno, final, y synchronized, no contribuyen a la firma del método.
