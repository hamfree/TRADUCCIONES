Clase para constantes

Crear una clase cuyo único trabajo es definir constantes ampliamente usadas es simple.

Ejemplo

package hirondelle.web4j.util;

/**
 Constantes recopiladas de utilidad general.

<P>Todos los miembros de esta clase son inmutables. 

 <P>(Este es un ejemplo de 
 <a href='http://www.javapractices.com/Topic2.cjp'>clase para constantes</a>.)
*/
public final class Consts  {

  /** Opuesto de {@link #FAILS}.  */
  public static final boolean PASSES = true;
  /** Opuesto de {@link #PASSES}.  */
  public static final boolean FAILS = false;
  
  /** Opuesto de {@link #FAILURE}.  */
  public static final boolean SUCCESS = true;
  /** Opuesto de  {@link #SUCCESS}.  */
  public static final boolean FAILURE = false;

  /** 
   Útil para {@link String} operaciones, que devuelve un índice de  <tt>-1</tt> cuando 
   un elemento no se encuentra.
  */
  public static final int NOT_FOUND = -1;
  
  /** Propiedad del sistema - <tt>line.separator</tt>*/
  public static final String NEW_LINE = System.getProperty("line.separator");
  /** Propiedad del sistema - <tt>file.separator</tt>*/
  public static final String FILE_SEPARATOR = System.getProperty("file.separator");
  /** Propiedad del sistema - <tt>path.separator</tt>*/
  public static final String PATH_SEPARATOR = System.getProperty("path.separator");
  
  public static final String EMPTY_STRING = "";
  public static final String SPACE = " ";
  public static final String TAB = "\t";
  public static final String SINGLE_QUOTE = "'";
  public static final String PERIOD = ".";
  public static final String DOUBLE_QUOTE = "\"";

  // PRIVADO //

  /**
   El llamador referencia las constantes usando <tt>Consts.EMPTY_STRING</tt>, 
   etc.. Por lo tanto, el llamador debe estar prevenido de construir objetos de 
   esta clase, declarando este constructor privado. 
  */
  private Consts(){
    //esto evita que incluso la clase nativa de 
    //llamar a este constructor también:
    throw new AssertionError();
  }
}
El llamador se refiere a estas constantes utilizando referencias estáticas, tales como Consts.EMPTY_STRING, Consts.SUCCESS, etcetera. Ya que el llamador nunca necesitará crear un objeto Consts, se usa un constructor privado para hacer cumplir esa política. Los miembros de tal clase de constantes son candidatos excelentes para una importación estática.

Es también posible usar una interfaz para llevar las constantes al alcance. Muchos argumentan firmemente que tal práctica es un uso extraño de las interfaces, que están destinadas a indicar los servicios proporcionados por una clase.
