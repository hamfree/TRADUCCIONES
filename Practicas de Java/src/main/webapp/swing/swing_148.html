Pautas para la Apariencia

Las Pautas de diseño de Apariencia de Java publicadas por Oracle tienen muchas recomendaciones útiles respecto a las interfaces de usuario.

Algunos puntos importantes:

  • cambiar la apariencia está pensado como una herramienta de diseño, no como una herramienta de tiempo de ejecución; es decir, Oracle recomienda que las aplicaciones no permitan que el usuario final cambie la apariencia en tiempo de ejecución.
  • las etiquetas siempre deben tener un mnemónico asociado, para permitir la navegación a través del teclado (Alt+F, por ejemplo) en vez del ratón. Dos excepciones comunes son el botón predeterminado (activado al pulsar Intro o Retorno) y el botón Cancelar (activado al pulsar Escape) - estos no deben tener un mnemónico asociado.
  • elegir los mnemónicos de acuerdo a los estilos de menú habituales, si es posible; de otra forma, seleccionar la primera letra, una consonante prominente, o una vocal prominente, en ese orden de preferencia.
  • use el método JLabel.setLabelFor para asociar un mnemónico con un elemento que de otro modo no tendría uno, por ejemplo, un JTextField.
  • las teclas de método abreviado comunes (Ctrl-N para Archivo-> Nuevo, por ejemplo) habitualmente tienen una tecla Ctrl como modificador.
  • el espaciado de los elementos deben ser múltiplos de 6 pixels (6N); si uno de los elementos tiene un borde blanco, entonces el espaciado debe ser un pixel menos (6N - 1).
  • Las barras de título del cuadro de diálogo deben contener el nombre de la aplicación, por ejemplo, "MiApp: Preferencias" y "MiApp: Acerca de".
  • los mensajes de JOptionPane deben empezar con un encabezado corto y en negrita en una sola línea (esto se puede implementar mediante las etiquetas HTML <b> y <br>).
  • Oracle publica un útil repositorio de gráficos
Menús
  • el estilo típico de menú sigue el orden Archivo, Objeto, Editar, Formato, Vista, y Ayuda. Cualquier otro menú se coloca inmediatamente antes de la Ayuda.
  • los títulos de menú deben ser una única palabra, mientras que los elementos de menú pueden tener más de una palabra
  • las elipses (...) se utilizan solo cuando se requiere una entrada adicional del usuario
Capitalización
  • Casi todos los elementos de texto breves (menús, botones, etiquetas, por ejemplo) deben tener el título en mayúsculas, donde todas las palabras están en mayúscula, excepto las palabras comunes con hasta tres letras (un, una, el, la, los, las, o, así, aún, etc.), y no aparecen como primera o última palabra.
  • si el texto no es breve, entonces se debe utilizar en su lugar el uso de mayúsculas en la prosa ordinaria (capitalación de la oración).
