Prefiera las Colecciones a clases más viejas

La mayoría de los programadores prefieren el Marco de Colecciones de Java a las matrices, Vector, y Hashtable, por las razones siguientes:
  • la intención del Marco de Colecciones es reemplazar las herramientas más antiguas y menos completas
  • la sincronización puede ser fácilmente controlada (mientras Hashtable y Vector están siempre sincronizadas)
  • la inmutabilidad puede ser fácilmente controlada
  • los algoritmos de búsqueda y clasificación están listos
  • el tipo de una referencia a una Collection puede ser genérico, mientras que la implementación subyacente puede ser fácilmente cambiada
  • las matrices son de tamaño fijo, y son más propensas a errores - los errores fuera-por-uno con los índices de las matrices han sido siempre una fuente fructífera de error.
  • Arrays.asList puede ser usada para pasar una matriz a un método que espera una Collection o una List
  • Collection.toArray puede ser usada para convertir una Collection a una Object[] o una T[]
  • Collections.enumeration y Collections.list pueden ser usadas para convertir entre enumeraciones y colecciones
Desde JDK 1.5, se puede argumentar que las Colecciones deberían preferirse casi siempre a las matrices.

Antes de JDK 1.5, la principal desventaja de usar colecciones es que siempre tenían un Object, y la recuperación de sus contenidos requería moldeo. El moldeo no se puede verificar en tiempo de compilación. Las matrices, por el otro lado, tienen la ventaja de estar siempre "parametrizadas" para contener objetos de un tipo específico (en todas las versiones del JDK), y esto es verificado por el compilador. Por lo tanto, en versiones anteriores de JDK, cambiar de una colección a una matriz obliga a que aparezca un tipo particular de error en tiempo de compilación en lugar de tiempo de ejecución.

Una ventaja menor de las matrices es que son ligeramente más fáciles de inicializar. Sin embargo, usando Arrays.asList hace la diferencia trivial: 

import java.util.*;

public final class ArrayInit {

  /** Initialize arrays and Lists.  */
  public static void main(String... args){
    //los inicializadores de matrices son compactos
    String[] paintings = {"oil", "watercolour"};

    //Construye una List usando Arrays.asList(T...)
    //Esto funciona para cualquier tipo, no solo para String
    List<String> languages = Arrays.asList(
      "urdu", "hindi", "pali", "sanskrit"
    );
    
    //Crea una List de manera más detallada 
    List<String> nicePlaces = new ArrayList<>();
    nicePlaces.add("luxembourg gardens");
    nicePlaces.add("sea side");
    nicePlaces.add("large magellanic cloud");
  }
}
