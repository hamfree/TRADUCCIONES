Utilice for each generosamente

El bucle for-each es usado con colecciones y matrices. Su objetivo es simplificar la forma más común de iteración, donde el iterador o índice es utilizado solamente para la iteración, y no para cualquier otra clase de operación, tal como eliminar o editar un elemento en la colección o matriz. Cuando hay una elección, el bucle for-each debe ser preferido sobre el bucle for, ya que incrementa la legibilidad.

Ejemplo

Aquí tiene algunos ejemplos del bucle for-each.

import java.util.*;
import java.math.BigDecimal;

public final class ForEachExamples {

  public static void main(String... aArgs){
    List<Number> numbers = new ArrayList<>();
    numbers.add(new Integer(42));
    numbers.add(new Integer(-30));
    numbers.add(new BigDecimal("654.2"));

    //bucle for-each típico
    //procesa cada elemento, sin cambiar la colección o matriz.
    for (Number number : numbers){
      log(number);
    }

    //uso con una matriz
    String[] names = {"Ethan Hawke", "Julie Delpy"};
    for(String name : names){
      log("Nombre : " + name);
    }

    //la eliminación de elementos requiere un iterador explícito,
    //así que no puede usar un bucle for-each en este caso
    Collection<String> words = new ArrayList<>();
    words.add("Il ne lui faut que deux choses: ");
    words.add("le");
    words.add("pain");
    words.add("et");
    words.add("le");
    words.add("temps.");
    words.add("- Alfred de Vigny.");
    for(Iterator<String> iter = words.iterator(); iter.hasNext();){
      if (iter.next().length() == 4){
        iter.remove();
      }
    }
    log("Palabras editadas: " + words.toString());

    //si es usado con un tipo no parametrizado (no recomendado), 
    //entonces se debe usar Object, junto con un moldeado
    Collection stuff = new ArrayList();
    stuff.add("blah");
    for (Object thing : stuff){
      String item = (String)thing;
      log("Cosa : " + item);
    }
  }

  // PRIVADO
  private static void log(Object aThing){
    System.out.println(aThing);
  }
}
