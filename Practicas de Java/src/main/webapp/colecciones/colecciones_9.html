Utilizar Colecciones estándares

En el Marco de Colecciones, hay muchas clases de implementación para elegir. La mayoría de las veces, una de estas "implementaciones principales" es la opción adecuada
  • ArrayList, para una implementación de la interfaz List
  • LinkedHashMap, para una implementación de la interfaz Map
  • LinkedHashSet, para una implementación de la interfaz Set
  • ArrayDeque, para una implementación de las interfaces Deque y Queue

Antes del JDK 1.5, HashMap y HashSet eran las implementaciones prefereridas de Map y Set. Sin embargo, los iteradores devueltos por estas clases tiene la propiedad algo extraña de tener un orden indefinido. Esto es, iterar sobre un HashMap o HashSet puede devolver elementos en un orden diferente en tiempos diferentes. Su orden de iteración no está garantizado a que sea repetible (y a menudo no lo es). Aunque HashMap y HashSet tiene un rendimiento general ligeramente mejor, para la mayoría de las aplicaciones de negocios es mejor evitar este comportamiento indefinido.

Para retener la ordenación de los elementos que están siendo manipulados en un interfaz gráfico del usuario, TreeSet y TreeMap son útiles. Forzan a una colección a mantener un orden de clasificación cuando el usuario agrega y elimina elementos.
