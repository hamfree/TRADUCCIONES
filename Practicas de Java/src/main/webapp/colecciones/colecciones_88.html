Iterar sin un índice

Muchos programadores tienen una preferencia fuerte por usar un bucle for-each o un Iterator en vez de un bucle for.

El bucle for tradicional siempre usa un índice de bucle explícito - un entero que identifica cada iteración. El problema es que los índices de bucle han sido siempre una fuente fructífera de error. Por ejemplo, errores por uno son muy comunes en programación, y a menudo están relacionados con estos índices de bucle. Además, la tradición informática (algo poco natural) de iniciar índices de bucle desde 0 en lugar de 1 a menudo contribuye a este tipo de errores.

Ya que son más propensos a errores, los bucles for con índices deben ser generalmente evitados, en favor de los más robustos, compactos, y elegantes bucles for-each.

Ejemplo

import java.util.*;

/** Diferentes estilos de iteración. */
public class IterateNoIndex {
   
  /** 
  * Iterar sin un índice es más compacto y menos  
  * propenso a errores. 
  */
   public void withoutIndex(){

     //el bucle for-each loop se prefiere generalmente
     List<String> trees = Arrays.asList("Maple", "Birch", "Poplar");
     for(String tree: trees){
       log(tree);
     }
     
     //Los Iteradores no son tan compactos como un bucle for-each 
     //pero algunas veces los necesita: eliminar/reemplazar elementos,
     //e iteraciones 'paralelas' a través de dos estructuras de datos
     Iterator<String> iter = trees.iterator();
     while (iter.hasNext()) {
       log(iter.next());
     }
   }
   
   /** Iterar con un índice es más propenso a errores. */
   public void withIndex(){
     //bucle for tradicional
     for(int idx=0; idx < 10; ++idx){
       log("Iteración..." + idx);
     }
   }
   
   // PRIVADO
   private void log(String aMessage){
     System.out.println(aMessage);
   }
}
