Elegir la Colección adecuada

Aquí hay una guía para seleccionar la implementación apropiada de un Set,  List, or  Map. Fue compilada para Java 1.4. Muchas adiciones han sido hechas al Marco de Colecciones desde entonces (notablemente las interfaces Queue y Deque, y varios elementos en java.util.concurrent). Estas adiciones posteriores se han omitido aquí, ya que este resumen más breve debería ser suficiente para la mayoría de los casos.

Las interfaces de mejor propósito general o implementaciones 'primarias' son probablemente ArrayList, LinkedHashMap, y LinkedHashSet. Su rendimiento general es mejor, y debe usarlas a menos que necesite una característica especial proporcionada por otra implementación. Esa característica especial suele ser la ordenación y la clasificación.

Por conveniencia, "ordenar" hará referencia aquí al orden de los elementos devueltos por un Iterator, y "clasificar" hará aquí referencia a la ordenación de los elementos de acuerdo a las interfaces Comparable o Comparator.
 

Interfaz ¿Tiene Duplicados? Implementaciones Histórico
Set no HashSet ... LinkedHashSet ... TreeSet ...
List sí ... ArrayList ... LinkedList ... Vector, Stack
Map sin claves duplicadas  HashMap ... LinkedHashMap ... TreeMap Hashtable, Properties

Características principales de las implementaciones no primarias:

  • HashMap tiene un rendimiento ligeramente mejor que LinkedHashMap, pero el orden de su iteración está indefinido
  • HashSet tiene un rendimiento ligeramente mejor que LinkedHashSet, pero el orden de su iteración está indefinido
  • TreeSet está ordenado y clasificado, pero es más lento
  • TreeMap está ordenado y clasificado, pero es más lento
  • LinkedList se agrega rápidamente al comienzo de la lista y se elimina rápidamente del interior mediante iteración
Orden de iteración para implementaciones anteriores:
  • HashSet - indefinida
  • HashMap - indefinida
  • LinkedHashSet - orden de inserción
  • LinkedHashMap - orden de inserción de las claves (por defecto), o 'orden de acceso'
  • ArrayList - orden de inserción
  • LinkedList - orden de inserción
  • TreeSet - orden ascendente, de acuerdo a Comparable / Comparator
  • TreeMap - orden ascendente de claves, de acuerdo a Comparable / Comparator
Para LinkedHashSet y LinkedHashMap, la re-inserción de un elemento no afecta al orden de inserción.

Para LinkedHashMap, el 'orden de acceso' es desde el acceso menos reciente al acceso más reciente. En este contexto, sólo las llamadas a get, put, y putAll constituyen un acceso, y sólo las llamadas a estos métodos afectan al orden de acceso.

Mientras se usan en un Map o Set, estos elementos no deben cambiar de estado (por lo tanto, se recomienda que estos elementos sean objetos inmutables):

  • claves de un Map
  • elementos en un Set
La clasificación requiere que:
  • los elementos almacenados implementan Comparable
  • Se defina un Comparator para los objetos almacenados
Para retener el orden de un ResultSet como es especificado en una clausa ORDER BY, inserte los registros en una List o una LinkedHashMap.
