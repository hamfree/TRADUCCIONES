Elija el estilo de validación del formulario con cuidado

En aplicaciones web, la validación de la entrada del usuario se puede hacer con dos herramientas diferentes:
  • el código de scripting ejecutado por le navegador (javascript, por ejemplo)
  • el código de java regular ejecutado por el servidor, como parte de la aplicación web
Validación del servidor:
  • no puede ser deshabilitada por un cambio de las preferencias del navegador
  • se ejecuta solo después de que se presione el botón SUBMIT
  • valida todos los campos del formulario cmo una unidad
  • si se detecta un error, se visualiza una página web nueva, con un mensaje de error apropiado
Validación del navegador:
  • puede ser deshabilitado por un cambio en las preferencias del navegador
  • puede ser iniciada en dos formas: después que se presione el botón SUBMIT, o después de que se ha cambiado un campo de entrada
  • si se ejecuta después de que presione SUBMIT, los datos del formulario son POST-ed (publicados) al servidor solo si la validación tiene éxito
  • si se presenta un error, se presenta un mensaje al usuario
  • habitualmente, no se envía una segunda petición HTTP al servidor; javascript simplemente trabaja con la página existente
Otra importante distinción entre los dos estilos trata no con su comportamiento, sino con su testabilidad. El código de validación en el servidor puede siempre ser comprobado económicamente usando herramientas tales como JUnit. Durante la vida entera de una aplicación, tales pruebas pueden ser rápidamente repetidas usando medio automatizados, ayudando a identificar errores posibles causados por cambios de código. Aunque herramientas tales como HttpUnit pueden comprobar el comportamiento de javascript en aplicaciones web, tales pruebas no parecen ser tan simples de implementar como las pruebas de JUnit en un Modelo de Objetos.

La validación en el navegador parece más útil para:

  • proporcionar una experiencia de usuario específica
  • entornos con anchos de banda limitados
  • formularios con un gran número de campos, cada uno validado mientras se introducen los datos
Si usa scripting, ¿Debería realizarse la validación después de presionar ENVIAR o cuando se cambia cada campo? Si no hay dependencias entre los campos, no hay mucha diferencia entre los dos estilos. Sin embargo, si hay dependencias entre campos, lo que es bastante común, entonces el estilo validar-despues-de-ENVIAR parece más simple de implementar.

Muchos prefieren tratar la validación del servidor como obligatoria y la validación del navegador como opcional, como una optimización del rendimiento o como la experiencia de usuario preferida. Si se implementan ambas, esto habitualmente representará una repetición significativa de código.
