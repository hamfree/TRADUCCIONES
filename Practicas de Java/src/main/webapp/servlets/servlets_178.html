Prevenga el enlazarse a sí mismo

A menudo ocurre que una página web se vincula a sí misma. Esto puede ser frustante para el usuario, especialmente si la página se toma mucho tiempo en cargar.

En vez de autoenlazarse, una página puede suprimir los enlaces a sí misma, y al mismo tiempo resaltar el texto del enlace de alguna manera, para indicar claramente la página actual. El usuario puede responder sin dolor a la pregunta común "¿Dónde estoy en este sitio?", al mismo tiempo que se le impide seguir un enlace que generalmente es inútil. Tal estilo tiene claramente más compasión por la experiencia del usuario.

Hay dos formas de evitar el autoenlazado en un conjunto de enlaces de navegación:

  • copiar y pegar el conjunto de enlaces de navegación a través de todas las páginas, y manualmente alterar cada ocurrencia posible de autoenlazado.
  • definir el conjunto de enlaces de navegación en un lugar, y crear algunos medios para suprimir dinámicamente los autoenlaces al generar cada página. Este es el estilo preferido, ya que define el menú en un solo lugar.

Para el estilo dinámico, no hay una forma estándar de definir como se suprimen tales enlaces. Una implementación podría confíar en una convención de nombrado, por ejemplo, para identificar los elementos de autoenlazado que deben ser alterados de alguna forma. Además, el estilo dinámico podría ser implementado en el servidor (Java), o en el cliente (Javascript).

Ejemplo

Este ejemplo usa la etiqueta personalizada de WEB4J HighlightCurrentPage para evitar dinámicamente el autoenlazado. En este caso, la implementación está en el servidor.

El siguiente trozo de JSP aparece en una plantilla, y visualizar una pequeña barra de navegación. En tiempo de ejecución, un enlace en la barra de navegación se suprimer, y muestra solo texto plano en vez del enlace completo. Además, resalta el texto (usando una clase CSS) para presentarla de forma más obvia como la página actual. Esto se implementa todo con <w:highlightCurrentPage>. La etiqueta usa una de las dos políticas para identificar enlaces que van a ser alterados:

  • hacer coincidir el URI actual con el final del destino HREF del enlace (este es el estilo predeterminado)
  • hacer coincidir el cuerpo textual del enlace con una subcadena del parámetro de solicitud TTitle utilizado por el mecanismo de plantillas WEB4J 
<div class="menu-bar">
 <w:highlightCurrentPage styleClass='highlight'>
  <c:url value="/main/home/HomePageAction.do" var="homeURL"/> 
  <a href='${homeURL}'>Home</a>
  <c:url value="/main/rsvp/RsvpShow.do" var="showRsvpURL"/> 
  <a href='${showRsvpURL}'>Rsvp</a>
  <c:url value="/main/visit/VisitAction.do?Operation=List" var="visitEditURL"/> 
  <a HREF='${visitEditURL}'>Visits</a>
  <c:url value="/main/rating/RatingAction.do?Operation=List" var="ratingEditURL"/> 
  <a HREF='${ratingEditURL}'>Ratings</a>
  <c:url value="/all/logoff/LogoffAction.do" var="logoffURL"/> 
  <a href='${logoffURL}'>Log Off</a>
 </w:highlightCurrentPage>
</div>
