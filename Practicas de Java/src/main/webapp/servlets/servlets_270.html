Sesiones compartidas entre pestañas

Es importante entender que varias pestañas/instancias del mismo navegador compartirán la misma sesión del lado del servidor al interactuar con el mismo dominio.

Las sesiones del lado del servidor pueden ser implementadas en dos formas:

  • cookies del navegador
  • reescritura de URL
Las aplicaciones deben usar habitualmente la siguiente lógica para implementar sesiones:
  • si el navegador tiene las cookies habilitadas(generalmente es el caso), entonces usar las cookies para implementar las sesiones.
  • de lo contrario, usar reescritura de URL para implementar las sesiones, como respaldo. (Use <c:url> consistentemente para conseguir esto).
Cada cookie se adjunta a un dominio y nada más. Por lo tanto, cada vez que un navegador interactúa con ese dominio, se envía la cookie al servidor de ese dominio. Para un navegador dado, la misma cookie se envía por cada pestaña/instancia que interactúa con ese dominio.

Como consecuencia, con un navegador y cliente determinados, no hay forma de tener varias sesiones múltiples al mismo tiempo

Si el usuario tiene varias cuentas, e inicia sesión por segunda vez con una cuenta diferente, en una pestaña diferente, entonces está sobreescribiendo el primer inicio de sesión.
