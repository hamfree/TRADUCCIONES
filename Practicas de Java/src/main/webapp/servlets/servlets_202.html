Use Hojas de Estilo en Cascada

Las Hojas de Estilo en Cascada (CSS) centralizan las decisiones de cómo las páginas web son estilizadas- fuentes, colores, diseños, y así.

Algunas ventajas de usar CSS:

  • como es habitual, las políticas centralizadas reducen en gran medida el esfuerzo de mantenimiento
  • muchos efectos visuales ofrecidos por CSS no pueden ser producidos por HTML simple
  • los tiempos de descarga de archivos mejoran ligeramente, ya que se eliminan las marcas repetidas
  • los diseños agradables son fáciles de producir
La compatibilidad con CSS varía entre los navegadores. Se debe tener cuidado de que las páginas se muestren adecuadamente en las más populares.

Algunas recomendaciones de usar CSS:

  • nombrar elementos con un alto nivel de abstracción, sin exponer los detalles de implementación. Por ejemplo, un buen nombre para una clase de estilo es destacado, y un mal nombre es bold-and-yellow
  • para maximizar el reuso, prefiera las hojas de estilos externas a los estilos definidos en una página única de HTML
  • nombre las hojas de estilo con una extensión .css, para asegurar que funcionan con herramientas que pueden requerir incorrectamente esa extensión
  • si usa pseudoclases para enlaces, use a:link y a:visited como un par
  • para los tamaños de las fuente, prefiera usualmente em o % (relativo), sobre ex y px (absoluto)
  • los estilos font-size de xx-small, x-small, y así, no son consistentes entre navegadores, y probablemente debería evitarse, como una especie de medida absoluta.
  • cuando se refiere a fuentes específicas, siempre incluya una familia de fuente genérica a usarse como una alternativa: serif, sans-serif, monospace
  • algunos recomiendan que !important debería ser evitado
  • favorecer técnicas que minimicen la cantidad de marcado
  • use la Especificación CSS como referencia, y use el Validador CSS para verificar su trabajo
Recordatorios:
  • . denote una clase
  • # denota un id
  • : denota una pseudo-clase, como en a:visited
  • [] denota selectores de atributos
  • de acuerdo a la especificación, los nombres de las clases y los id son sensibles a mayúsculas y minúsculas
  • cuadro de elemento = área de contenido + relleno + borde + margen (en orden desde el medio hacia afuera)
  • la anchura de un elemento se refiere solo al área de contenido, y no al cuadro de elemento entero
  • la etiqueta LINK se coloca en la etiqueta HEAD
  • un LINK típico:
<link rel="stylesheet" type="text/css" href="styles.css" media="all">

Tenga cuidado con los cachés del navegador

Los navegadores almacenan en caché las hojas de estilo. Si una actualización a un sitio web incluye cambios a una hoja de estilo, entonces el desarrollador a menudo necesitará refrescar manualmente la caché del navegador, para ver las actualizaciones (los detalles aparentemente dependen del navegador). Sin embargo, cuando se despliega tal actualización, no es aceptable experar que el usuario final haga lo mismo.

Para asegurar que todos los usuarios ven la versión actualizada, hay una solución sencilla; simplemente agregue un parámetro de versión en todos los enlaces a la hoja de estilo.
