Analizar parámetros en objetos de dominio

Todos los nombres y valores de los parámetros de las peticiones de HTTP son enteramente textuales, y la Especificación Servlet los trata como simples Strings. El problema siempre surge en una aplicación web de cómo traducir o analizar tales Strings en objetos con más significado en el dominio del problema. Habitualmente, esto significa realizar dos clases de operaciones:
  1. traducir Strings a Integer,BigDecimal, y así
  2. agrupar dichos elementos traducidos en Modelos de Objetos completos

Estas operaciones variarán dependiendo del marco de trabajo que esté utilizando.

Ejemplo

Este ejemplo está tomado de una aplicación construída con el marco de trabajo WEB4J.

/** Construye un Modelo de Objetos a partir de los parámetros de la solicitud. */
public final class RestoAction extends ActionTemplateListAndEdit {

  public static final RequestParameter RESTO_ID = RequestParameter.withLengthCheck("Id");
  public static final RequestParameter NAME = RequestParameter.withLengthCheck("Name");
  public static final RequestParameter LOCATION = RequestParameter.withLengthCheck("Location");
  public static final RequestParameter PRICE = RequestParameter.withLengthCheck("Price");
  public static final RequestParameter COMMENT = RequestParameter.withLengthCheck("Comment");

  protected void validateUserInput() {
    try {
      ModelFromRequest builder = new ModelFromRequest(getRequestParser());
      fResto = builder.build(Resto.class, RESTO_ID, NAME, LOCATION, PRICE, COMMENT);
    }
    catch (ModelCtorException ex){
      //muestra mensaje de error al usuario final
      addError(ex);
    }    
  }
  
  //..muchos elementos omitidos

  //Modelo de Objetos destino, construído a partir de la entrada del usuario
  private Resto fResto;
}
En este caso, el análisis de los parámetros en crudo de la petición HTTP son compartidos entre estas clases:

  • RequestParameter representa los parámetros en crudo subyacentes aceptados por la Action
  • RequestParser define políticas predeterminadas para traducir parámetros de petición individuales en objetos String, Date, Boolean, Integer, y BigDecimal, y Collections de los mismos.
  • ModelFromRequest traduce los conjuntos de parámetros de petición en Modelos de Objetos. Mientras un RequestParser devuelve objetos únicos de "bloques de construcción" (Integer, Date, etc.) ModelFromRequest va un paso más allá, y devuelve un Modelo de Objetos entero, que encapsula más o menos una serie de objetos de "bloques de construcción".
