Use el marco de trabajo Modelo Vista Controlador

La mayoría de las aplicaciones web usan un marco de trabao Modelo-Vista-Controlador. En particular, habitualmente usan lo que se llama el estilo "Modelo 2", que tiene estos componentes:
  • Controller (Nota del Traductor: Controlador, en español) - extiende a HttpServlet, actúa como punto de entrada en la aplicación, y delega a varias clases de trabajo para completar una petición. En particular, el Controller es un usuario de los objetos de Modelo y Vista.
  • Model (Nota del Traductor: Modelo, en español) - clases centradas en los datos que encapsulan los objetos del dominio del problema. Cada clase corresponde aproximadamente a las filas de una tabla de la base de datos. Los objetos del modelo pueden ser construídos desde un ResultSet de una consulta a una base de datos, o desde los parámetros de una petición HTTP.
  • View (Nota del Traductor: Vista, en español) - implementada como Páginas de Servidor de Java (Java Server Pages o JSP, en inglés) o una herramienta similar, principalmente relacionadas con la presentación y el formato de los objetos del modelo que han sido incluidos en el alcance por el controlador (o su delegado).
