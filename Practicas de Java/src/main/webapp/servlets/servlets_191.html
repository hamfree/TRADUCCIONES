Administrar las sesiones de cerca

La IPA de Servlet es, desafortunadamente, bastante liberal en la creación de sesiones. Varias herramientas tienen comportamientos predeterminados que pueden implícitamente crear sesiones en segundo plano. Es muy fácil para una aplicación "accidentalmente" crear una sesión, incluso cuando no fue pedida explíctamente una.

Como un ejemplo, los JSPs a menudo crearán una sesión si una no existe ya. Esto permite a los JSPs usar la variable implícita session. Como segundo ejemplo, el método request.getSession() también creará automáticamente una sesión si aún no existe una.

Sin embargo, las sesiones deben ser gestionadas con algo de cuidado, por dos razones principales:

  • las sesiones tienen varios riesgos de seguridad asociados con ellas.
  • las sesiones consumen recursos del servidor.

Por lo tanto, es probable que la aplicación controle estrictamente la creación y destrucción de sesiones.

Aquí tiene un ejemplo de un conjunto razonable de políticas con respecto a las sesiones:

  • use una directiva <%@ page session="false" %> al principio de cada JSP que no use una sesión
  • considere deshabilitar la reescritura de URL totalmente
  • cree una nueva sesión solo cuando el usuario se conecta (y no antes)
  • cuando el usuario se desconecta, invalide la sesión y elimine cualquier cookie relacionada
  • en web.xml, asegúrese que el tiempo de expiración de la sesión está fijado a un valor que no sea innecesariamente largo
  • defenderse de los ataques de falsificación de solicitudes entre sitios (que secuestran sesiones existentes)
