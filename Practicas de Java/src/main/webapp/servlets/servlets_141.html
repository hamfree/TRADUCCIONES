Pérdida de datos durante el reinicio

Si un servidor se reinicia, entonces los datos almacenados en el alcance de sesión pueden perderse. El comportamiento de un contenedor web en este caso no está definido por la especificación servlet, y varía entre implementaciones. Algunos contenedores web proporcionan "conmutación por error", en el que los datos de la sesión no se pierden durante un reinicio, mientras que otros no.

En aquellos que implementan la conmutación por error, un requisito común es que solo se puedan restaurar los objetos Serializable.
