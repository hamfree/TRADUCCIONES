Tenga cuidado de los ampersands doblemente escapados

No es raro ver páginas web con texto mostrando incorrectamente &quot;blah&quot; en vez de "blah". (Una búsqueda simple por el texto '&quot;' devolverá numerosos ejemplos de tales errores).

Esto es causado por un escapado excesivo de caracteres especiales. El carácter ampersand '&', en particular, es doblemente especial: es a la vez un carácter especial, y forma parte del propio mecanismo de escape, como en '&gt;' y '&quot;'.

Si el texto que contiene cualquier carácter especial es escapado dos veces, entonces ocurren los problemas mencionados anteriormente. Un ejemploi simple muestra porqué:

  • forma original: "blah" (incluyendo las comillas)
  • escapar una vez a la forma: &quot;blah&quot; (aún se presenta como "blah")
  • escapar una segunda vez a la forma: &amp;quot;blah&amp;quot; (se presenta cmo &quot;blah&quot;). Tenga en cuenta el ampersand doblemente escapado.
La causa más común de de tal escapado doble ocurre por:
  • primer escape al almacenar la entrada del usuario original en la base de datos
  • escapar una segunda vez cuando se presenta el mismo elemento en la capa de presentación, usando una herramienta como <c:out> en JSTL
Ya que el escapado está relacionado con HTML - es decir, con la capa de presentación - se recomienda que la base de datos no almacene datos en su forma escapada.
