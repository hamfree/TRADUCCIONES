Forward versus redirect

Un Controller (en este contexto, una implementación de HttpServlet) puede realizar una operación de reenvío o redirección al final del procesado de una petición. Es importante entender la diferencia entre estos dos casos, en particular con respecto a las recargas de páginas web de los navegadores.

Forward (Reenvío)

  • un reenvío lo realiza internamente el servlet
  • el navegador desconoce por completo que ha tenido lugar, por lo que su URL original permanece intacta
  • cualquier recarga del navegador de la página resultante simplemente repetirá la solicitud original, con la URL original
Redirect (Redirección)
  • una redirección es un proceso de dos pasos, donde la aplicación web indica al navegador que busque una segunda URL, que difiere de la original
  • una recarga del navegador de la segunda URL no repetirá la solicitud original, sino que buscará la segunda URL
  • el redireccionamiento es ligeramente más lento que un reenvío, ya que requiere dos solicitudes del navegador, no una.
  • los objetos colocados en el alcance de petición original no están disponibles para la segunda petición
En general, un reenvío se debe usar si la operación se puede repetir de manera segura al volver a cargar el navegador de la página web resultante; de lo contrario, se debe usar la redirección. Normalmente, si la operación realiza una edición en el almacén de datos, se requiere un redireccionamiento, no un reenvío. Esto es simplemente para evitar la posibilidad de duplicar inadvertidamente una edición en la base de datos.

Más explícitamente (en términos de operaciones comunes SQL):

  • para operaciones SELECT, use un reenvío
  • para operaciones INSERT, UPDATE, o DELETE, use una redirección
En HTML, una etiqueta <FORM> puede GET o POST sus datos. En este contexto, un GET corresponde a una SELECT-después-reenviar, y un POST corresponde a una edición-despues-redirigir.

Se recomienda encarecidamente que los formularios para la introducción de criterios de búsqueda utilizen GET, mientras que los formularios par editar registros de la base de datos usen POST.

El mayor síntoma común de no usar el reenvío/redirección correctamente es un mensaje de advertencia en el navegador, preguntando al usuario si realmente desea hacer POST (publicar) sus datos del formulario por segunda vez.

Ejemplo

Este ejemplo sigue el estilo de la clase Controller de WEB4J. Los métodos importantes de la IPA Servlet son:

  • ServletRequest.getRequestDispatcher(String)
  • RequestDispatcher.forward(request, response)
  • HttpServletResponse.sendRedirect(String) 

import java.io.IOException;
import javax.servlet.*;
import javax.servlet.http.*;
import hirondelle.web4j.action.Action;
import hirondelle.web4j.action.ResponsePage;
import hirondelle.web4j.request.RequestParser;
import hirondelle.web4j.model.BadRequestException;

public class RedirectForward extends HttpServlet {
  
  //..muchos elementos omitidos

  @Override public final void doGet(
      HttpServletRequest aRequest, HttpServletResponse aResponse
   ) throws ServletException, IOException {
    processRequest(aRequest, aResponse);
  }

  @Override public final void doPost(
      HttpServletRequest aRequest, HttpServletResponse aResponse
   ) throws ServletException, IOException {
    processRequest(aRequest, aResponse);
  }

  /**
  * Maneja todas las peticiones HTTP <tt>GET</tt> y <tt>POST</tt>.
  */
  protected void processRequest(
    HttpServletRequest aRequest, HttpServletResponse aResponse
  ) throws ServletException, IOException {
    RequestParser requestParser = RequestParser.getInstance(aRequest, aResponse);
    try {
      Action action = requestParser.getWebAction();
      ResponsePage responsePage = action.execute();
      if ( responsePage.getIsRedirect() ) {
        redirect(responsePage, aResponse);
      }
      else {
        forward(responsePage, aRequest, aResponse);
      }
    }
    catch (BadRequestException ex){
      //..omitido
      //usa Response.sendError()
    }
    catch (Throwable ex) {
      //..omitido
      //usa Response.sendError()
    }
  }
  
  // PRIVADO //
  
  private void redirect(
    ResponsePage aDestinationPage, HttpServletResponse aResponse
  ) throws IOException {
    String urlWithSessionID = aResponse.encodeRedirectURL(aDestinationPage.toString());
    aResponse.sendRedirect( urlWithSessionID );
  }

  private void forward(
    ResponsePage aResponsePage, HttpServletRequest aRequest, HttpServletResponse aResponse
  ) throws ServletException, IOException {
    RequestDispatcher dispatcher = aRequest.getRequestDispatcher(aResponsePage.toString());
    dispatcher.forward(aRequest, aResponse);
  }
}
