Leer y escribir ficheros de texto

En el KDJ 7, las clases más importantes para los ficheros de texto son:
  • Paths y Path - ubicaciones/nombres de fichero, pero no su contenido.
  • Files - operaciones sobre el contenido de ficheros.
  • StandardCharsets and Charset (una clase más vieja), para las codificaciones de los ficheros de texto.
  • el método File.toPath, que permite al código más viejo interactuar fácilmente con la IPA java.nio más nueva.
Además, las clases siguientes son también usadas comunmente con los ficheros de texto, tanto en el KDJ 7 como en versiones anteriores:
  • Scanner - permite la lectura de ficheros de una forma compacta
  • BufferedReader - readLine
  • BufferedWriter - write + newLine

Cuando lee y escribe ficheros de texto:

  • es a menudo una buena idea usar el almacenamiento en búfer (el tamaño predeterminado es de 8K)
  • hay siempre una necesidad de prestar atención a las excepciones (en particular, IOException y FileNotFoundException)

Codificación de Caracteres

Para leer y escribir correctamente archivos de texto, debe comprender que esas operaciones de lectura/escritura usan una codificación de caracteres implícita para traducir los bytes sin procesar - los 1s y 0s - en texto. Cuando se salva un fichero de texto, la herramienta que lo salva debe usar siempre una codificación de caracteres (se recomienda UTF-8). Hay un problema, sin embargo. La codificación de caracteres no es, en general, explícita: no se salva como parte del fichero en sí mismo. Por lo tanto, un programa que consume un fichero de texto debe saber de antemano cuál es su codificación Si no lo hace entonces lo mejor que puede hacer es hacer una suposición. Los problemas con la codificación usualmente se muestran como caracteres extraños en una herramienta que ha leído el fichero.

Las clases FileReader y FileWriter son un poco difíciles, ya que utilizan implícitamente la codificación de caracteres predeterminada del sistema. Si este valor predeterminado no es apropiado, las alternativas recomendadas son, por ejemplo:

FileInputStream fis = new FileInputStream("test.txt");
InputStreamReader in = new InputStreamReader(fis, "UTF-8");

FileOutputStream fos = new FileOutputStream("test.txt");
OutputStreamWriter out = new OutputStreamWriter(fos, "UTF-8");

Scanner scanner = new Scanner(file, "UTF-8");

Ejemplo 1 - KDJ 7+

import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.charset.Charset;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.util.Arrays;
import java.util.List;
import java.util.Scanner;

public class ReadWriteTextFileJDK7 {
  
  public static void main(String... aArgs) throws IOException{
    ReadWriteTextFileJDK7 text = new ReadWriteTextFileJDK7();
    
    //tratado como un archivo pequeño
    List<String> lines = text.readSmallTextFile(FILE_NAME);
    log(lines);
    lines.add("Esta es una línea agregada en código.");
    text.writeSmallTextFile(lines, FILE_NAME);
    
    //tratado como un archivo grande: usa un poco de almacenamiento de búfer
    text.readLargerTextFile(FILE_NAME);
    lines = Arrays.asList("Hasta la línea de flotación", "Agua de Amor");
    text.writeLargerTextFile(OUTPUT_FILE_NAME, lines);   
  }

  final static String FILE_NAME = "C:\\Temp\\input.txt";
  final static String OUTPUT_FILE_NAME = "C:\\Temp\\output.txt";
  final static Charset ENCODING = StandardCharsets.UTF_8;
  
  //Para ficheros más pequeños

  /**
   Nota: el javadoc de Files.readAllLines indica que está destinado para 
   archivos pequeños. Pero su implementación usa almacenamiento en búfer, por lo que es probable 
   que sea bueno incluso para archivos bastante grandes.
  */  
  List<String> readSmallTextFile(String aFileName) throws IOException {
    Path path = Paths.get(aFileName);
    return Files.readAllLines(path, ENCODING);
  }
  
  void writeSmallTextFile(List<String> aLines, String aFileName) throws IOException {
    Path path = Paths.get(aFileName);
    Files.write(path, aLines, ENCODING);
  }

  //Para ficheros más grandes
  
  void readLargerTextFile(String aFileName) throws IOException {
    Path path = Paths.get(aFileName);
    try (Scanner scanner =  new Scanner(path, ENCODING.name())){
      while (scanner.hasNextLine()){
        //procesa cada línea de alguna manera
        log(scanner.nextLine());
      }      
    }
  }
  
  void readLargerTextFileAlternate(String aFileName) throws IOException {
    Path path = Paths.get(aFileName);
    try (BufferedReader reader = Files.newBufferedReader(path, ENCODING)){
      String line = null;
      while ((line = reader.readLine()) != null) {
        /procesa cada línea de alguna manera
        log(line);
      }      
    }
  }
  
  void writeLargerTextFile(String aFileName, List<String> aLines) throws IOException {
    Path path = Paths.get(aFileName);
    try (BufferedWriter writer = Files.newBufferedWriter(path, ENCODING)){
      for(String line : aLines){
        writer.write(line);
        writer.newLine();
      }
    }
  }

  private static void log(Object aMsg){
    System.out.println(String.valueOf(aMsg));
  }
  
}

Ejemplo 2 - KDJ 7+

Este ejemplo demuestra el uso de Scanner para leer un fichero que contiene línea de datos estructurados. Un Scanner se usa para leer en cada línea, y un segundo Scanner se usa para analizar cada línea en un para simple de nombre-valor. La clase Scanner se usa solo para leer, no para escribir.

import java.io.IOException;
import java.nio.charset.Charset;
import java.nio.charset.StandardCharsets;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.util.Objects;
import java.util.Scanner;

/** Asume la codificación UTF-8. KDJ 7+. */
public class ReadWithScanner {

  public static void main(String... aArgs) throws IOException {
    ReadWithScanner parser = new ReadWithScanner("C:\\Temp\\test.txt");
    parser.processLineByLine();
    log("Hecho.");
  }
  
  /**
   Constructor.
   @param aFileName nombre completo de un fichero existente, y legible.
  */
  public ReadWithScanner(String aFileName){
    fFilePath = Paths.get(aFileName);
  }
  
  
  /** Método plantilla que llama {@link #processLine(String)}.  */
  public final void processLineByLine() throws IOException {
    try (Scanner scanner =  new Scanner(fFilePath, ENCODING.name())){
      while (scanner.hasNextLine()){
        processLine(scanner.nextLine());
      }      
    }
  }
  
  /** 
   Método sobreescribible para procesar las líneas de diferentes formas.
    
   <P>Esta implementación simple predeterminada espera pares simples de nombre-valor, separados por un signo 
   '='. Ejemplos de entradas válidas: 
   <tt>altura = 167cm</tt>
   <tt>masa =  65kg</tt>
   <tt>disposición =  "gruñón"</tt>
   <tt>este es el nombre = este es el valor</tt>
  */
  protected void processLine(String aLine){
    //usa un segundo Scanner para analizar el contenido de cada línea
    try(Scanner scanner = new Scanner(aLine)){
      scanner.useDelimiter("=");
      if (scanner.hasNext()){
        //asume que la línea tiene una cierta estructura
        String name = scanner.next();
        String value = scanner.next();
        log("El Nombre es : " + quote(name.trim()) + ", y el Valor es  : " + quote(value.trim()));
      }
      else {
        log("Línea vacía o no válida. Incapaz de procesar.");
      }
    }
  }
  
  // PRIVADO
  private final Path fFilePath;
  private final static Charset ENCODING = StandardCharsets.UTF_8;  
  
  private static void log(Object aObject){
    System.out.println(Objects.toString(aObject));
  }
  
  private String quote(String aText){
    String QUOTE = "'";
    return QUOTE + aText + QUOTE;
  }
}

Ejecución de ejemplo de esta clase: 

Nombre es : 'altura', y el Valor es : '167cm'
Nombre es : 'masa', y el Valor es : '65kg'
Nombre es : 'disposición', y el Valor es : '"gruñón"'
Nombre es : 'este es el nombre', y el Valor es : 'este es el valor'
Done.

Ejemplo 3 - KDJ < 7

La característica try-with-resources no está disponible anteriormente al KDJ 7. En este caso, debe tener cuidado con el método close:

  • siempre hay que llamarlo, de lo contrario se filtrarán recursos
  • Vaciará automáticamente el flujo, si es necesario
  • Llamar a close en un flujo "envoltorio" llamará automáticamente a close en su flujo subyacente
  • cerrar un flujo por segunda vez no tiene consecuencias
  • cuando se llama a un Scanner, la operación close solo funciona si el elemento pasado a su constructor implementa Closeable. Advertencia: Si usted pasa un File a un Scanner, ¡no será capaz de cerrarlo! Intente usar en su lugar un FileReader.
Aquí hay un ejemplo bastante compacto (para JDK 1.5) de lectura y escritura de un archivo de texto, usando una codificación explícita. Si elimina todas las referencias a la codificación de esta clase, aún funcionará - la codificación predeterminada del sistema será usada en su lugar. 
import java.io.*;
import java.util.Scanner;

/** 
 Lee y escribe un fichero usando codificación explícita.
 JDK 1.5.
 Eliminar la codificación de este código simplemente causará que 
 se use en su lugar la codificación predeterminada del sistema.
*/
public final class ReadWriteTextFileWithEncoding {

  /** Requiere dos argumentos - el nombre del fichero, y la codificación a usar.  */
  public static void main(String... aArgs) throws IOException {
    String fileName = aArgs[0];
    String encoding = aArgs[1];
    ReadWriteTextFileWithEncoding test = new ReadWriteTextFileWithEncoding(
      fileName, encoding
    );
    test.write();
    test.read();
  }
  
  /** Constructor. */
  ReadWriteTextFileWithEncoding(String aFileName, String aEncoding){
    fEncoding = aEncoding;
    fFileName = aFileName;
  }
  
  /** Escribe contenido fijo en el archivo dado.. */
  void write() throws IOException  {
    log("Escribiendo a un fichero llamado " + fFileName + ". Codificación: " + fEncoding);
    Writer out = new OutputStreamWriter(new FileOutputStream(fFileName), fEncoding);
    try {
      out.write(FIXED_TEXT);
    }
    finally {
      out.close();
    }
  }
  
  /** Lee el contenido del fichero dado. */
  void read() throws IOException {
    log("Leyendo del fichero.");
    StringBuilder text = new StringBuilder();
    String NL = System.getProperty("line.separator");
    Scanner scanner = new Scanner(new FileInputStream(fFileName), fEncoding);
    try {
      while (scanner.hasNextLine()){
        text.append(scanner.nextLine() + NL);
      }
    }
    finally{
      scanner.close();
    }
    log("Texto leído: " + text);
  }
  
  // PRIVADO
  private final String fFileName;
  private final String fEncoding;
  private final String FIXED_TEXT = "¡Pero suave! ¿Qué código de ese programa se rompe?";
  
  private void log(String aMessage){
    System.out.println(aMessage);
  }
}

Ejemplo 4 - KDJ < 7

Este ejemplo usa FileReader y FileWriter, que implícitamente use la codificación predeterminada del sistema. También usa almacenamiento en búfer. Para hacer este ejemplo compatible con el KDJ 1.4, simplemente cambie StringBuilder a StringBuffer: 

import java.io.*;

/** KDJ 6 o anterior. */
public class ReadWriteTextFile {

  /**
  * Obtenga todo el contenido de un archivo de texto y devuélvalo en una cadena.
  * Este estilo de implementación no lanza Excepciones al llamador.
  *
  * @param aFile es un fichero que ya existe y puede ser leído.
  */
  static public String getContents(File aFile) {
    //...comprobaciones sobre aFile eludidas
    StringBuilder contents = new StringBuilder();
    
    try {
      //usa almacenamiento en búfer, leyendo una línea a la vez
      //¡FileReader siempre asume que la codificación predeterminada es correcta!
      BufferedReader input =  new BufferedReader(new FileReader(aFile));
      try {
        String line = null; //no declarada dentro del bucle while
        /*
        * readLine es un poco peculiar :
        * devuelve el contenido de una línea MENOS la nueva línea.
        * devuelve nulo solo para el FINAL del flujo.
        * devuelve una cadena vacía si aparecen dos líneas nuevas en una fila.
        */
        while (( line = input.readLine()) != null){
          contents.append(line);
          contents.append(System.getProperty("line.separator"));
        }
      }
      finally {
        input.close();
      }
    }
    catch (IOException ex){
      ex.printStackTrace();
    }
    
    return contents.toString();
  }

  /**
  * Cambia los contenidos del fichero de texto en su totalidad, sobreescribiendo cualquier
  * texto existente.
  *
  * Este estilo de implementación lanza todas las excepciones al método que llama.
  *
  * @param aFile es un fichero existente en el cual se puede escribir.
  * @throws IllegalArgumentException si el parámetro no cumple.
  * @throws FileNotFoundException si el fichero no existe.
  * @throws IOException si se encuentra un problema durante la escritura.
  */
  static public void setContents(File aFile, String aContents)
                                 throws FileNotFoundException, IOException {
    if (aFile == null) {
      throw new IllegalArgumentException("El archivo no debe ser nulo.");
    }
    if (!aFile.exists()) {
      throw new FileNotFoundException ("El archivo no existe: " + aFile);
    }
    if (!aFile.isFile()) {
      throw new IllegalArgumentException("No debería ser un directorio: " + aFile);
    }
    if (!aFile.canWrite()) {
      throw new IllegalArgumentException("El archivo no se puede escribir: " + aFile);
    }

    //usa almacenamiento en búfer
    Writer output = new BufferedWriter(new FileWriter(aFile));
    try {
      //¡FileWriter siempre asume que la codificación predeterminada es correcta!
      output.write( aContents );
    }
    finally {
      output.close();
    }
  }

  /** Test simple.   */
  public static void main (String... aArguments) throws IOException {
    File testFile = new File("C:\\Temp\\blah.txt");
    System.out.println("Contenido del fichero original: " + getContents(testFile));
    setContents(testFile, "El contenido de este fichero ha sido sobreescrito...");
    System.out.println("Contenido del archivo nuevo: " + getContents(testFile));
  }
}
