Borrar un árbol de directorio

Cuando borra un árbol de directorios:
  • está en problemas si lo hace incorrectamente. Es una operación peligrosa.
  • no puede borrar un directorio que aún tenga ficheros en el.
  • no puede borrar elementos si no tiene permisos suficientes (sistema de ficheros) para hacerlo.
  • debe saber cómo se tratan los enlaces simbólicos.
Si la operación de borrado sigue los enlaces simbólicos, entonces el fichero al cual apunta el enlace también es borrado, además del enlace en sí mismo. La mayoría de las veces, es probable que desee evitarlo.

Ejemplo

import java.io.File;
import java.io.IOException;
import java.nio.file.FileVisitResult;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.nio.file.SimpleFileVisitor;
import java.nio.file.attribute.BasicFileAttributes;
import java.util.Comparator;
import java.util.Objects;
import java.util.stream.Stream;

/** 
 Borra un árbol de ficheros.
 
 En Windows, use 'mklink' para crear un enlace simbólico, si desea 
 comprobar el comportamiento.
*/
public final class RemoveDirTree {
  
  /** Prueba simple */
  public static void main(String... args) throws IOException {
    RemoveDirTree test = new RemoveDirTree();
    String dirToDelete = "C:\\temp\\testme\\"; //¡Con cuidado!
    //test.rmDirWithStreams(Paths.get(root));
    test.rmDirWithFileVisitor(Paths.get(dirToDelete));
    log("Hecho");
  }
  
  /**
   Aquí, 'reverseOrder' se usa para reemplazar el orden 
   de clasificación natural de los objetos File. Esto evita la posibilidad de intentar 
   borrar directorios no vacios.
   @since Java 8 
  */
  void rmDirWithStreams(Path dir) throws IOException {
    //Stream<Path> le habla al sistema de ficheros: necesita estar cerrado, así que 
    //usamos un bloque try-with-resources
    //Files.walk(dir, FileVisitOption.FOLLOW_LINKS) // ¡peligroso!
    if (shouldDelete(dir)){
      try(Stream<Path> paths = Files.walk(dir)){
        paths
          .map(Path::toFile) // cambia a Stream<File> 
          .sorted(Comparator.reverseOrder()) //coloca los directorios al final
          .forEach(File::delete) // lo borra
        ;
      }
    }
  }
  
  /** 
   Este estilo permite más variaciones, porque el mecanismo de recorrido de archivos 
   es más detallado.
   
   Si quiere seguir los enlaces simbólicos (más peligroso), use este método:
   Files.walkFileTree(
     Path start, Set<FileVisitOption> opts, int depth, FileVisitor visitor
   );
   junto con:
   EnumSet<FileVisitOption> opts = EnumSet.of(FOLLOW_LINKS);

   Ref: https://docs.oracle.com/javase/tutorial/essential/io/walk.html
   
   @since Java 7 
  */
  void rmDirWithFileVisitor(Path dir) throws IOException {
    if (shouldDelete(dir)){
      Files.walkFileTree(dir, new SimpleFileVisitor<Path>() {
        /** Borra los ficheros en un directorio (pero no el directorio en sí mismo). */ 
        @Override public FileVisitResult visitFile(
            Path file, BasicFileAttributes attrs
         ) throws IOException {
            Files.delete(file);
            return FileVisitResult.CONTINUE;
        }
        /** 
         Después de eliminar todos los ficheros en el directorio, entonces elimina el directorio en sí.
        */
        @Override public FileVisitResult postVisitDirectory(
          Path dir, IOException exc
        ) throws IOException {
            Files.delete(dir);
            return FileVisitResult.CONTINUE;
        }
      });
    }
  }
  
  private boolean shouldDelete(Path dir){
    return dir.toFile().exists() && dir.toFile().isDirectory();
  }
  
  private static void log(Object thing) {
    System.out.println(Objects.toString(thing));
  }
}
