Leer y escribir objetos serializables

Leer y escribiro objetos Serializable es muy similar a leer y escribir texto:
  • es casi siempre una buena idea usar almacenamiento en búfer (el tamaño predeterminado es 8K)
  • es a menudo posible usar referencias a clases base abstractas, en vez de referencias a a clases concretas
  • hay siempre una necesidad de prestar atención a las excepciones; en particular, IOException y ClassNotFoundException

Ejemplo

Aquí, una List de objetos String se serializa y después se deserializa.

import java.io.*;
import java.util.*;
import java.util.logging.*;

/**
 Usa almacenamiento de búfer, y clases base abstractas.
 KDJ 7+. 
*/
public class ExerciseSerializableNew {

  public static void main(String... aArguments) {
    //crea una Lista serializable
    List<String> quarks = Arrays.asList(
      "up", "down", "strange", "charm", "top", "bottom"
    );

    //serializa la Lista
    try (
      OutputStream file = new FileOutputStream("quarks.ser");
      OutputStream buffer = new BufferedOutputStream(file);
      ObjectOutput output = new ObjectOutputStream(buffer);
    ){
      output.writeObject(quarks);
    }  
    catch(IOException ex){
      fLogger.log(Level.SEVERE, "Cannot perform output.", ex);
    }

    //deserializa el fichero quarks.ser
    try(
      InputStream file = new FileInputStream("quarks.ser");
      InputStream buffer = new BufferedInputStream(file);
      ObjectInput input = new ObjectInputStream (buffer);
    ){
      //deserializa la Lista
      List<String> recoveredQuarks = (List<String>)input.readObject();
      //visualiza sus datos
      for(String quark: recoveredQuarks){
        System.out.println("Quark Recuperado: " + quark);
      }
    }
    catch(ClassNotFoundException ex){
      fLogger.log(Level.SEVERE, "No se puede realizar la entrada. Clase no encontrada", ex);
    }
    catch(IOException ex){
      fLogger.log(Level.SEVERE, "No se puede realizar la entrada.", ex);
    }
  }

  // PRIVADO

  private static final Logger fLogger =
    Logger.getLogger(ExerciseSerializableNew.class.getPackage().getName())
  ;
}

Ejemplo - KDJ 6-

Si no está disponible try-with-resources (KDJ 6-), entonces debe ser cuidadoso con el método close:

  • siempre hay que llamarlo, de lo contrario se filtrarán recursos
  • vaciará automáticamente el flujo, si es necesario
  • llamar a close en un flujo "envoltorio" llamará automáticamente a close en su flujo subyacente
  • cerrar un flujo una segunda vez no tiene consecuencias

Ejemplo

Aquí está el mismo ejemplo que arriba, pero usando KDJ 6-.

import java.io.*;
import java.util.*;
import java.util.logging.*;

/** KDJJ antes de la versión 7. */
public class ExerciseSerializable {

  public static void main(String... aArguments) {
    //crea una Lista serializable
    List<String> quarks = Arrays.asList(
      "up", "down", "strange", "charm", "top", "bottom"
    );

    //serializa la Lista
    //tenga en cuenta el uso de referencias abstractas de clases base

    try{
      //usa almacenamiento en búfer
      OutputStream file = new FileOutputStream("quarks.ser");
      OutputStream buffer = new BufferedOutputStream(file);
      ObjectOutput output = new ObjectOutputStream(buffer);
      try{
        output.writeObject(quarks);
      }
      finally{
        output.close();
      }
    }  
    catch(IOException ex){
      fLogger.log(Level.SEVERE, "No puedo realizar la entrada.", ex);
    }

    //deserializa el fichero quarks.ser
    //tenga en cuenta el uso de referencias abstractas de clases base
    
    try{
      //usa almacenamiento en búfer
      InputStream file = new FileInputStream("quarks.ser");
      InputStream buffer = new BufferedInputStream(file);
      ObjectInput input = new ObjectInputStream (buffer);
      try{
        //deserializa la Lista
        List<String> recoveredQuarks = (List<String>)input.readObject();
        //visualiza sus datos
        for(String quark: recoveredQuarks){
          System.out.println("Quark recuperado: " + quark);
        }
      }
      finally{
        input.close();
      }
    }
    catch(ClassNotFoundException ex){
      fLogger.log(Level.SEVERE, "No puedo realizar la entrada. Clase no encontrada.", ex);
    }
    catch(IOException ex){
      fLogger.log(Level.SEVERE, "No puedo realizar la entrada.", ex);
    }
  }

  // PRIVADO

  //Usa las facilidades de registro de Java para grabar las excepciones
  //El comportamiento del logger se puede configurar a través de un 
  //fichero de texto, o mediante programación a través de la IPA de logging.
  private static final Logger fLogger =
    Logger.getLogger(ExerciseSerializable.class.getPackage().getName())
  ;
}
