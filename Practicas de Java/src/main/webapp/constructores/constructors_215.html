Los constructores no deben llamar a los sobreescribibles

Los constructores no llamarán a un método sobreescribible - esto es, deberían llamar sólo a los métodos que son private , static , or final .

La razón por la cual se ilustra mejor con un poco de código: 

import java.util.Objects;

public final class ConstructorOverridable {

  public static void main(String... aArgs){
    log("CONSTRUIR UN ANIMAL :");
    Animal animal = new Animal();
    log(" ");
    log("CONSTRUIR UN PESCADO :");
    Animal fish = new Fish();
  }
  
  // PRIVADO
  private static void log(Object aObject){
    System.out.println(Objects.toString(aObject));
  }
  
  private static class Animal{
    Animal(){
      log("Animal ctor");
      //llamar a un método sobreescribible desde ctor - peligros
      perform();
    }
    void perform(){
      log("Animal perform.");
    }
  }
  
  private static class Fish extends Animal {
    Fish(){
      //cada constructor tiene una llamada implícita a super(); en la primera línea 
      super();
      log("Fish ctor. Esta es la forma de cola más antigua que se puede inicializar.");
    }
    void perform(){
      //en general, este método puede depender de los datos de Fish
      log("Fish perform. Puede fallar desde que la Forma de Cola es : " + fTailShape);
    }
    private String fTailShape;
  }
}
Un ejemplo de ejecución de esta clase demuestra el problema: 

CONSTRUIR UN ANIMAL :
Animal ctor
Animal perform.
 
CONSTRUIR UN PESCADO :
Animal ctor
Fish perform. Puede fallar desde que la Forma de Cola es : null
Fish ctor. Esta es la forma de cola más antigua que se puede inicializar.
