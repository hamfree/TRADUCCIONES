No pase 'this' fuera de un constructor

Detro de una clase, la palabra clave de Java 'this' se refiere al objeto nativo, la instancia actual de la clase. Dentro de un constructor, puede usar la palabra clave this de tres formas diferentes:
  • en la primera línea, puede llamar a otro constructor, usando this(...);
  • como un cualificador cuando referencia campos, como en this.fName
  • como una referencia pasada a un método de algún otro objeto, como en blah.operation(this);

Puede tener problemas con la última forma. El problema es que, dentro de un constructor, un objeto no está aún completamente construido. Un objeto sólo se construye por completo después de que su constructor haya terminado completamente, y no antes. Pero cuando lo pase como un parámetro a un método de otro objeto, la referencia this siempre debe referirse a un objeto completamente formado.

Este problema ocurre principalmente con los oyentes (listeners, en inglés). Aquí tiene un ejemplo que ilustra este punto:

import java.util.Observable;
import java.util.Observer;

/**
 @author javapractices.com
 @author Andrew Sackett
*/
public final class EscapingThisReference {

  /** Una Estación de Radio es observada por la gente que la escucha. */
  static final class RadioStation extends Observable {
    //elided
  }
  
  /** 
   Un oyente el cual espera hasta que este objeto esté completamente 
   formado antes de permitirle ser referenciado por el mundo exterior.
   Usa un constructor privado para construir primero el objeto, y 
   entonces configura el objeto completamente formado como un oyente. 
  */
  static final class GoodListener implements Observer {
    /** método Factoría. */
    static GoodListener buildListener(String aPersonsName, RadioStation aStation){
      //primero llamada al constructor privado
      GoodListener result = new GoodListener(aPersonsName);
      //el objeto 'result' está ahora construido completamente, y puede ahora ser 
      //pasado de forma segura al mundo exterior
      aStation.addObserver(result);
      return result;
    }
    @Override public void update(Observable aStation, Object aData) {
      //..omitido
    }
    private String fPersonsName;
    /** Constructor privado. */
    private GoodListener(String aPersonsName){
      this.fPersonsName = aPersonsName; //ok
    }
  }
  
  /** 
   Un oyente que pasa incorrectamente una referencia 'this' al mundo exterior
   antes de que la construcción se haya completado.
  */
  static final class BadListenerExplicit implements Observer {
    /** Constructor ordinario. */
    BadListenerExplicit(String aPersonsName, RadioStation aStation){
      this.fPersonsName = aPersonsName; //OK
      //PELIGROSO - la referencia 'this' no debe pasarse al oyente,
      //ya que el constructor no ha sido completado; ¡no importa si  
      //esta es la última sentencia en el constructor!
      aStation.addObserver(this);
    }
    @Override public void update(Observable aStation, Object aData) {
      //..omitido
    }
    private String fPersonsName;
  }
  
  /** 
   Otro oyente que pasa una referencia 'this' al mundo exterior antes 
   de que la construcción se complete; aquí, la referencia 'this' está 
   implícita, a través de la clase anidada anónima.
  */
  static final class BadListenerImplicit {
    /** Constructor ordinario. */
    BadListenerImplicit(String aPersonsName, RadioStation aStation){
      this.fPersonsName = aPersonsName; //OK
      //PELIGROSO
      aStation.addObserver(
        new Observer(){
          @Override public void update(Observable aObservable, Object aData) {
            doSomethingUseful();
          }
        }
      );
    }
    private void doSomethingUseful() {
      //..omitido
    }
    private String fPersonsName;
  }
}
