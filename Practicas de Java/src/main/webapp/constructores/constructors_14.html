Inicializar campos a 0 falso o nulo es redundante

Uno de los aspectos más fundamentales de un lenguaje de programación es cómo inicializa los datos. Para Java, esto se define explícitamente en la especificación. Para campos y componentes de matriz, cuando los elementos son creados, son establecidos automáticamente los siguientes valores por defecto del sistema:
  • números: 0 or 0.0
  • booleanos: false
  • referencias a objeto: null

Esto significa que establecer explícitamente los campos a 0, falso, o nulo (como podría ser el caso) es innecesario y redundante. Ya que esta característica del lenguaje fue incluída en orden, en parte, reducir el código repetitivo, es una buena idea aprovecharlo al máximo. Insistir en que los campos deben inicializarse explícitamente en 0, falso o nulo es una expresión que probablemente sea inapropiada para el lenguaje de programación Java.

Además, establecer un campo explícitamente en 0, falso o nulo puede incluso causar que la misma operación se realice dos veces (dependiendo de su compilador).

Ejemplo 

public final class Quark {

  public Quark(String aName, double aMass){
    fName = aName;
    fMass = aMass;
  }

  //PRIVADO

  //SIN inicialización redundante a valores por defecto
  //private String fName;
  //private double fMass;

  //CON inicialización redundante a valores por defecto
  private String fName = null;
  private double fMass = 0.0d;
}

Si el bytecode de la clase Quark es examinado, las operaciones duplicadas se vuelven claras (aquí, el compilador javac de Oracle fue usado):
