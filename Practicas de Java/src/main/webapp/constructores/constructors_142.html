Cuidado con las redeclaraciones de campo equivocadas

Tenga en cuidado de este simple error, lo cual puede ser dificil de rastrear: si un campo es redeclarado erróneamente dentro del cuerpo de un método, entonces el campo no está siendo referenciado, sino más bien una variable local temporal con el mismo nombre.

Un síntoma común de este problema es que el rastro de pila indica que un campo es null, pero un examen superficial del código hace que parezca que el campo se ha inicializado correctamente.

Las Convenciones de nombrado pueden reducir la frecuencia de este error, pero no puede eliminarlo.

Ejemplo

public final class MistakenRedeclare {

  public static void main(String... aArguments ) {
    MistakenRedeclare mistake = new MistakenRedeclare("blah");
    //Lanza NullPointerException, ya que el valor devuelto de 
    //getName es nulo.
    System.out.println("Longitud del nombre: " + mistake.getName().length());
  }

  public MistakenRedeclare(String aName) {
    //¡Oops! ¡Estableciendo una variable local, no el campo!
    String fName = aName;
    //este estilo puede ayudar a eliminar tales errores:
    //this.fName = aName;
  }

  public String getName(){
    return fName;
  }

  //..omitido

  // PRIVADO
  private String fName;
}

Ejemplo de ejecución de esta clase:

>java -cp . MistakenRedeclare
Exception in thread "main" java.lang.NullPointerException
        at MistakenRedeclare.main(MistakenRedeclare.java:7)
