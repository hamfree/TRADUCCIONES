Pase todos los datos pertinentes

Cuando ocurre una excepción, es importante que todos los datos pertinentes se pasen al constructor de la excepción. Tales datos son a menudos críticos para entender y resolver el problema, y puede reducir en gran medida el tiempo necesario para encontrar una solución.

La referencia this es algunas veces útil para este propósito, ya que se llama implícitamente a toString. Además, si está definiendo clases de excepción usted mismo, puede incluso diseñar sus constructores para forzar al llamador a pasar los datos pertinentes.

Las trazas de pila no informativas son muy frustrantes para el mantenedor, y se sabe que inspiran maldiciones desagradables.

Ejemplo

public final class RangeChecker {

  /**
  * Devuelve <tt>true</tt> solo si <tt>aNumber</tt> está en el rango 
  * <tt>aLow..aHigh</tt> (inclusive).
  *
  * @param <tt>aLow</tt> menor o igual que <tt>aHigh</tt>.
  */
  static public boolean isInRange(int aNumber, int aLow, int aHigh){
    if (aLow > aHigh) {
      throw new IllegalArgumentException("Inferior:" + aLow + " es mayor que Superior:" + aHigh);
    }
    return (aLow <= aNumber && aNumber <= aHigh);
  }
}

Effective Java menciona a IndexOutOfBoundsException como contraejemplo. Esta excepción no proporciona información sobre los índices relevantes, que a menudo sería útil para resolver el problema:

  • el valor del índice ofensivo
  • los valores de índice mínimos y máximos permitidos
