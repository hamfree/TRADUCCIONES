Finally y catch

El bloque finally se usa par asegurar que los recursos son recuperados independientemente de cualesquiera problemas que peucdan ocurrir.

Hay diversas variaciones para usar el bloque finally. de acuerdo a cómo se manejan las excepciones. (Vea el excelente libro El Lenguaje de Programación Java de Arnold, Goslin, y Holmes por información relacionada).

Si está usando el KDJ 7+, entonces la mayoría de los usos del bloque finally se pueden eliminar, simplemente usando una sentencia try-with-resources. Si un recurso no implementa AutoCloseable , entonces aún sen necesitará un bloque finally, incluso en el KDJ 7+. Dos ejemplos podrían ser:

  • recursos relacionados con hilos
  • recursos relacionados con un objeto Graphics

KDJ < 7, y recursos que no son AutoCloseable
Los ejemplos siguientes son apropiados para las versiones más viejas del KDJ. También son apropaidos en el KDJ 7+, pero solo para recursos que no implementan AutoCloseable.

Estilo 1

Si un método lanza todas las excepciones, entonces se puede usar un finally sin catch: 

import java.io.*;

/** Antes del KDJ 7. */
public final class SimpleFinally {
  
  public static void main(String... aArgs) throws IOException {
    simpleFinally("C:\\Temp\\test.txt");
  }
  
  private static void simpleFinally(String aFileName) throws IOException {
    //Si esta línea lanza una excepción, entonces ni el bloque try 
    //ni el bloque finally se ejecutarán.
    //Eso es bueno, ya que el lector sería nulo.
    BufferedReader reader = new BufferedReader(new FileReader(aFileName));
    try {
      //Cualquier excepción en el bloque try causará que se ejecute el bloque finally
      String line = null;
      while ((line = reader.readLine()) != null) {
        //procesa la línea...
      }
    }
    finally {
      //El objeto reader nunca será nulo aquí.
      //A este finally solo se entra después de que se entre en el bloque try.
      //Pero, NO ES POSIBLE entrar en el bloque try 
      //con un objeto reader nulo.
      reader.close();
    }
  }
}
Estilo 2

Si un método maneja todas las excepciones comprobadas que pueden ser lanzadas por su implementación, entonces una variación interesante es anidar un try..finally dentro de un try..catch. Este estilo es particularmente útil cuando el bloque finally lanza las mismas excepciones que el resto del código (lo que es común con las operaciones java.io). Aunque este estilo podría parecer ligeramente complicado, parece ser superior a los estilos alternativos: 

import java.io.*;
import java.util.logging.*;

/** Antes KDJ 7. */
public final class NestedFinally {
  
  public static void main(String... aArgs) {
    nestedFinally("C:\\Temp\\test.txt");
  }
  
  private static void nestedFinally(String aFileName) {
    try {
      //Si el constructor lanza una excepción, el bloque finally NO se ejecutará
      BufferedReader reader = new BufferedReader(new FileReader(aFileName));
      try {
        String line = null;
        while ((line = reader.readLine()) != null) {
          //procesa la línea...
        }
      }
      finally {
        //no se necesita comprobar nulo 
        //en ninguna excepción lanzada aquí, será recogida por 
        //el bloque catch más externo
        reader.close();
      }
    }
    catch(IOException ex){
      fLogger.severe("Problem occured : " + ex.getMessage());
    }
  }
  
  private static final Logger fLogger =
    Logger.getLogger(NestedFinally.class.getPackage().getName())
  ;
}
Estilo 3

Un estilo más detallado coloca un catch dentro de finally. Este estilo es probablemente el menos deseable, ya que tiene mayor cantidad de bloques: 

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;
import java.util.logging.Logger;

/** Antes KJD 7. */
public final class CatchInsideFinally {
  
  public static void main(String... aArgs) {
    catchInsideFinally("C:\\Temp\\test.txt");
  }
  
  private static void catchInsideFinally(String aFileName) {
    //Declaradas aquí para ser visible desde el bloque finally
    BufferedReader reader = null;
    try {
      //si esta línea falla, se ejecutará finally, y el reader será nulo
      reader = new BufferedReader(new FileReader(aFileName));
      String line = null;
      while ( (line = reader.readLine()) != null ) {
        //procesa la línea...
      }
    }
    catch(IOException ex){
      fLogger.severe("Ocurrió un problema : " + ex.getMessage());
    }
    finally {
      try {
        //necesita comprobar el nulo
        if ( reader != null ) {
          reader.close();
        }
      }
      catch(IOException ex){
        fLogger.severe("Ocurrió un problema. No puedo cerrar el reader : " + ex.getMessage());
      }
    }
  }
  
  private static final Logger fLogger =
    Logger.getLogger(CatchInsideFinally.class.getPackage().getName())
  ;
}
