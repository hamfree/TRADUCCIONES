Las excepciones y el control de flujo

Cuando ocurre una excepción dentro de un método, el flujo de control definido por Java puede tomar varias formas.

Si ocurre una excepción fuera de un bloque try..catch, la excepción simplemente se propaga "hacia arriba" al llamador.

Si ocurre una excepción dentro de un bloque try..catch, entonces el flujo de control puede tomar formas diferentes. Aquí están los casos, junto con los indicadores para los cuales los bloques de código se incluyen en el control de flujo:
 

Caso try  catch  finally fondo
try no lanza excepción y - y y
try lanza una excepción gestionada y y y y
try lanza una excepción no gestionada y - y -

Aquí, "fondo" se refiere simplemente a cualquier código que siga al bloque finally, como se muestra aquí: 

final class Bottom {

  void doStuff() {
    try {
      //....obviado
    }
    catch( Exception ex ) {
      //....obviado
    }
    finally {
      //....obviado
    }
    //cualquier código que aparece aquí, después del bloque finally
    //es código "fondo"
  }
}

Existe una idea errónea, especialmente común entre los programadores de C que migran a Java, de que se pueden usar excepciones para definir el flujo de control ordinario. Este es un mal uso de la idea de excepciones, que están destinadas solo para defectos o para elementos fuera del control directo del programa..
