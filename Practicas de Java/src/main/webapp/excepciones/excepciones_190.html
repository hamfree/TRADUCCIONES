Use plantilla para try catch repetidos

Los bloques try-catch de Java son particularmente comunes cuando se usan IPAs lo que les da un rol importante a las excepciones comprobadas (tales como SQLException en JDBC). Cuando se usan tales IPA, muchos ejemplos publicados simplemente repiten la misma estructura de código try-catch donde sea necesaria. Sin embargo, es simple eliminar tal repetición de código usando el patrón del Método Template. La idea es definir la estrtuctura del bloque try-catch en un lugar, en un clase de base abstracta (CBA). Dicha CBA se amplía cada vez que se necesita ese bloque try-catch en particular. La implementación concreta de tal CBA a menudos tendrá un código simple, "en línea recta".

Un ejemplo extendido de esta técnica se da en el tema Use plantillas para las transacciones.
