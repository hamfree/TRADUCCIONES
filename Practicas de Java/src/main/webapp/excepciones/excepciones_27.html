Sea específico en la cláusula throws

En la cláusula throws de un encabezado de método, sea lo más específico posible. No agrupe juntas excepciones relacionadas en una clase de excepción genérica - eso representaría una pérdida de información posiblemente importante.

Una alternativa es la práctica Traducción de Excepciones, en la cual una excepción de bajo nivel se traduce primero a una excepción de más alto nivel antes de ser lanzada fuera del método.

Ejemplo

Aquí, tanto IOException como FileNotFoundException están agrupados incorrectamente como Exception.

import java.io.*;
import java.util.*;

public final class BadGenericThrow {
  
  //..obviado

  /**
   * MAL: este método arroja una excepción genérica, en vez de 
   * de FileNotFoundException y IOException.
   */
  public void makeFile() throws Exception {
    //crea una Lista Serializable
    List<String> quarks = new ArrayList<>();
    quarks.add("up");
    quarks.add("down");
    quarks.add("strange");
    quarks.add("charm");
    quarks.add("top");
    quarks.add("bottom");

    //serializa la Lista
    try (
      ObjectOutputStream output  = new ObjectOutputStream(
        new FileOutputStream("quarks.ser")
      )
    ){
      output.writeObject(quarks);
    }
  }
}
