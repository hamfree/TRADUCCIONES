Traducción de Excepciones

Ocasionalmente, es apropiado traducir un tipo de excepción a otra.

La capa de datos, por ejemplo, puede beneficiarse de esta técnica. Aquí, la capa de datos busca ocultar casi todos los detalles de implementación de las otras partes del programa. Incluso busca ocultar el mecanismo básico de persistencia - ya sea o no una base de datos o se use un esquema de ficheros ad hoc, por ejemplo.

Sin embargo, cada estilo de persistencia tiene excepciones específicas - SQLException para las bases de datos, y IOException para ficheros, por ejemplo. Si el resto del programa va a permanecer realmente ignorante del mecanismo de persistencia, entonces estas excepciones no pueden propagarse fuera de la capa de datos, y deben ser traducidas en una abstracción de nivel superior, por ejemplo, DataAccessException.

Vea el tema Encapsulado de la excepción de Datos para un ejemplo extendido.
