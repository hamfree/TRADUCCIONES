Serialización y generación de subclases

Las interfaces y clases diseñadas para la herencia raramente deberían implementar Serializable, ya que esto forzaría una tarea significativa (y a menudo no deseada) en sus implementadores y subclases.

Sin embargo, aunque la mayoría de las clases abstractas no implementan Serializable, es posible que deban permitir que sus subclases lo hagan, si así lo desean.

Hay dos casos. Si la clase abstracta no tiene estado, entonces puede simplemente proporcionar un constructor sin argumentos a sus subclases. Si la clase abstracta tiene estado, sin embargo, entonces hay más trabajo que hacer, como se demuestra en el siguiente ejemplo.

Ejemplo

public abstract class Government {

  public Government(String aWinningParty) {
    init(aWinningParty);
    //nunca invoque un método sobreescribible en un constructor
  }

  public final void raiseTaxes() {
    validateInit(); //llamado siempre por métodos públicos
    //..omitido
  }

  public final void lowerTaxes() {
    validateInit();
    //..omitido
  }

  /**
  * Un método de plantilla que llama a un método abstracto protegido, para 
  * ser sobreescrito por las subclases.
  */
  public boolean makeLaw(){
    validateInit();
    String bill = draftLegislation(); //método abstracto y protegido
    return vote(bill);
  }

  // PROTEGIDO

  /**
  * Cada gobierno redactará la legislación de una manera particular.
  * Una tiranía, por ejemplo, no tendría un amplio debate entre sus
  * ciudadanos.
  */
  protected abstract String draftLegislation();

  /**
  * Este constructor sin argumentos es necesitado por el mecanismo de Serialización de 
  * las subclases
  */
  protected Government() {
    //vacío
  }

  /**
  * Llamado tanto por el constructor público de esta clase como por el método 
  * readObject de una subclase. Tenga en cuenta que este método es definitivo.
  */
  protected final void init(String aWinningParty){
    if (fIsInitialized) {
      throw new IllegalStateException("No se puede llamar a init dos veces.");
    }
    fWinningParty = aWinningParty;
    fIsInitialized = true;
  }

  /**
  * La subclase necesitará acceso al estado interno de la superclase, 
  * ya sea a través de gets públicos o protegidos. Tenga en cuenta que get 
  * debe ser final
  */
  protected final String getWinningParty() {
    return fWinningParty;
  }

  // PRIVADO
  private String fWinningParty;
  private boolean fIsInitialized = false;

  /**
  * Debe ser llamado por métodos públicos.
  */
  private void validateInit() throws IllegalStateException {
    if (!fIsInitialized) {
      throw new IllegalStateException("Objeto no inicializado");
    }
  }

  private boolean vote(String aBill){
    return true; //implementación de juguete
  }
} 
    

import java.io.*;

/** Una subclase Serializable. */
public final class Tyranny extends Government implements Serializable {

  public Tyranny(String aWinningParty) {
    super(aWinningParty);
  }

  // PROTEGIDO

  protected String draftLegislation() {
    return "Prohibir todo uso de C ++.";
  }

  // PRIVADO

  /**
  * Se necesita deserialización personalizada.
  */
  private void readObject(
    ObjectInputStream aStream
  ) throws IOException, ClassNotFoundException {
    aStream.defaultReadObject();
    //deserializa manualmente e inicia la superclase
    String winningParty = (String)aStream.readObject();
    init(winningParty);
  }

  /**
  * Se necesita serialización personalizada.
  */
  private void writeObject(ObjectOutputStream aStream) throws IOException {
    aStream.defaultWriteObject();
    //serializa manualmente la superclase
    aStream.writeObject(getWinningParty());
  }
}
