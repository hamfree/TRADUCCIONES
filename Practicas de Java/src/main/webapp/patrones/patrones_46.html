Singleton

Las clases singleton representan objetos para los que solo debe existir una instancia.

La pregunta importante aquí es esta: ¿Se producirá algún daño real en el sistema si se crea más de un objeto? Si la respuesta es "no" (y por lo general lo es), entonces no es necesario usar un singleton. Si la respuesta es "sí", entonces necesitará considerarlo. El punto principal es que un singleton solo debe usarse si es realmente necesario.

Aquí tiene un artículo de Oracle describiéndolos. Este artículo, junto con muchos otros, informa de muchos problemas sutiles con los singletons. Es probable que deba tener cuidado al utilizar este patrón.

Ejemplo 1

Este es el estilo preferido de implementar singletons. Usa una enumeración simple. No tiene necesidades especiales de serialización, y es inmune a ataques inteligentes.

/** Estilo preferido para singletons. */
public enum SantaClaus {
  INSTANCE;
  
  /**Agrega algo de comportamiento al objeto. */
  public void distributePresents(){
    //omitido
  }
  
  /** Demuestra el uso de SantaClaus. */
  public static void main(String... aArgs){
    SantaClaus fatGuy = SantaClaus.INSTANCE;
    fatGuy.distributePresents();
    
    //no compila :
    //SantaClaus fatGuy = new SantaClaus();
  }
}
Ejemplo 2

Aquí tiene un estilo alternativo. Si decide que la clase ya no debería ser un singleton, simplemente puede cambiar la implementación de getInstance.

public final class Universe {

  public static Universe getInstance() {
     return fINSTANCE;
  }

  // PRIVADO

  /**
  * Instancia única creada al cargar la clase
  */
  private static final Universe fINSTANCE =  new Universe();

  /**
  * El constructor privado previene la construcción fuera de esta clase.
  */
  private Universe() {
    //..omitido
  }
}
Ejemplo 3

Si el estilo de arriba de un singleton va a ser Serializable también, entonces debe agregar un método readResolve.

import java.io.*;

public final class EasterBunny implements Serializable {

  public static EasterBunny getInstance() {
     return fINSTANCE;
  }

  // PRIVADO

  /**
  * La instancia única creada al cargar la clase.
  */
  private static final EasterBunny fINSTANCE =  new EasterBunny();

  /**
  * El constructor privado previene la construcción fuera de esta clase.
  */
  private EasterBunny() {
    //..omitido
  }

  /**
  * Si el singleton implementa Serializable, entonces se debe suministrar 
  * el método.
  */
  private Object readResolve() throws ObjectStreamException {
    return fINSTANCE;
  }
}

Podría también considerar definir su propia interfaz de etiqueta para las clases singleton.
