Métodos de la Factoría

Los métodos de la factoría son métodos estáticos que devuelven una instancia de la clase nativa. Ejemplos en el KDJ:
  • LogManager. getLogManager
  • Pattern. compile
  • Collections. unmodifiableCollection , Collections. synchronizeCollection , etcetera
  • LocalDateTime.now
Los métodos de factoría:
  • tienen nombres, a diferencia de los constructores, que pueden aclarar el código.
  • no es necesario crear un objeto nuevo en cada invocación - los objetos se pueden almacenar en caché y reutilizar, si es necesario.
  • pueden devolver un subtipo de su tipo de retorno, en particular, puede devolver un objeto cuya clase de implementación es desconocida para el llamante. Esta es una característica muy valiosa y ampliamente utilizada en muchos marcos que usan interfaces como el tipo de retorno de métodos de factoría estáticos.
Los nombres comunes para los métodos de factoría incluyen getInstance y valueOf. Estos nombre no son obligatorios - elija lo que tenga sentido para cada caso.

Ejemplo

public final class ComplexNumber {

  /**
  * El método estático de factoría devuelve un objeto de esta clase.
  */
  public static ComplexNumber valueOf(float aReal, float aImaginary) {
    return new ComplexNumber(aReal, aImaginary);
  }

  /**
  * La persona que llama no puede ver este constructor privado.
  *
  * La única forma de construir un ComplexNumber es llamando al método
  * estático de factoría.
  */
  private ComplexNumber(float aReal, float aImaginary) {
    fReal = aReal;
    fImaginary = aImaginary;
  }

  private float fReal;
  private float fImaginary;

  //..omitido
}
