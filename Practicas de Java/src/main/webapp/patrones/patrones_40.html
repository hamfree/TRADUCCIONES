Constructor privado

Los constructores privados previenen una clase de ser explícitamente instanciada por sus llamadores.

Hay algunos casos comunes donde un constructor privado puede ser útil:

  • clases que contienen solo métodos de utilidad estáticos
  • clases que contienen solo constantes
  • enumeraciones seguras
  • singletons
Estos ejemplos caen en dos categorías.

La construcción de objetos está totalmente prohíbida

Ningún objeto se puede construir, ya sea por el llamador o por la clase nativa. Esto solo es adecuado para clases que ofrecen solo miembros estáticos al llamador.

En estos casos, la falta de un constructor accesible le dice a la persona que llama: "No hay casos de uso para esta clase en los que necesite construir un objeto. Solo puede usar elementos estáticos. Estoy impidiendo que incluso intente para construir un objeto de esta clase". Vea Clase para constantes para una ilustración.

Si el programador no proporciona un constructor para una clase, entonces el sistema siempre proporcionará un constructor público sin argumentos predeterminado. Para deshabilitar este constructor predeterminado, simplemente agregue un constructor sin argumentos privado a la clase. Este constructor privado puede estar vacío. De manera algo paradójica, la creación de objetos por parte del llamador está de hecho deshabilitada por la adición de este constructor privado sin argumentos.

La construcción del objeto es solo privada

Los objetos pueden construirse, pero solo internamente. Por alguna razón, una clase necesita prevenir que el llamador cree objetos.

En el caso de un singleton, la política es que solo un objeto de esa clase se supone que debe existir. La creación de múltiples objetos de esa clase está prohíbido.

En el caso del KDJ 1.4 las enumeraciones seguras, más de un objeto puede ser creado, pero de nuevo, hay una limitación. Solo se permite un conjunto específico de objetos, uno por cada elemento de la enumeración. (Para las enumeraciones de KDJ 1.5, la creación de dichos objetos se hace implícitamente por el lenguaje Java, no explícitamente por el programador de la aplicación).
