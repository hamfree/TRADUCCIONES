Manejar la InterruptedException

En código relacionado con hilos, a menudo necesitará manejar una InterruptedException. Hay dos formas comunes de manejarla:
  • simplemente lance la excepción hacia arriba al llamador (tal vez después de hacer una limpieza)
  • llame al método interrupt en el hilo actual

En el segundo caso, el estilo estándar es: 

try {
  ...código relacionado-con-el-hilo...
}
catch(InterruptedException ex){
  ...quizás algún manejo del error aquí...
  //re-interrupir el hilo actual
  Thread.currentThread().interrupt();
}

Antecedentes

En código multihilo, los hilos pueden a menudo bloquearse; los hilos pausan la ejecución hasta que se cumpla alguna condición externa, tal como:
  • se libera un bloqueo
  • otro hilo completa una operación
  • se completa alguna operación de E/S
  • y así...

Los hilos pueden ser interrumpidos. Una interrupción pide a un hilo que pare lo que está haciendo de una forma ordenada. Sin embargo, la respuesta exacta a la interrupción depende del estado del hilo, y cómo se implementó el hilo: 

si el hilo está actualmente bloqueado 
  dejar de bloquear temprano
  lanzar InterruptedException
si no, el hilo está haciendo un trabajo real 
  el estado interrumpido del hilo se establece en verdadero
  si el hilo consulta isInterrupted() periódicamente
    (lo cual se prefiere)
    limpiar ordenadamente y detener la ejecución
    lanzar InterruptedException
  si no
    la ejecución regular continúa
La idea principal aquí es que una interrupción no debería apagar un hilo inmediatamente, en medio de un cálculo. Más bien, un hilo y su llamador se comunican entre sí para permitir un apagado ordenado.
