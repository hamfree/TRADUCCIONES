La integridad de los datos primero

El código multi-hilo puede ser complejo. Con la llegada de java.util.concurrent, el código multihilo puede ahora habitualmente ser escrito a un nivel superior al que era posible previamente. Sin embargo, el código multihilo puede ser aún complejo.

Es probable que se deba tratar la seguridad básica de subprocesos de una clase como fundamental. Deberían ser evitables las optimizaciones de rendmiento y código "con trucos" a menos que sean realmente necesarios.

Como es habitual, cuando se enfrentan a diversas decisiones de implementación, se deben sopesar cuidadosamente los pros y los contras, tales como:

  • ¿el mantenedor se sentirá cómodo manteniendo el código? ¿Sería suficiente una implementación más simple?
  • ¿realmente vale la pena cada optimización de rendimiento? ¿se ha medido su beneficio y es significativo?
Por ejemplo, usar un bloque de lectura-escritura con un mapa podría ser tentandor en una circunstancia dada, pero probablemente encontrará que un ConcurrentHashMap funciona bien.
