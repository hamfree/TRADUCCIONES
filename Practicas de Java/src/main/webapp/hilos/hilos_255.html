Volcar información del hilo

Hay una forma de pedir al Entorno de Ejecución de Java que genere un "volcado de subprocesos" para todos los subprocesos en ejecución (tanto subproceso del sistema como subprocesos de la aplicación). Por cada subproceso, el volcado del subproceso incluirá un seguimiento de pila, e información sobre cualquier bloque actualmente mantenido por el subproceso. Los volcados de subprocesos no causan que su aplicación se pare. Son como una captura: su aplicación pausa la ejecución, se le pregunta a los subprocesos en ejecución, la información es generada, y después su aplicación simplemente reanuda la ejecución.

Para causar un volcado de subprocesos desde la consola que lanzó la aplicación:

  • en Windows, presione Control + Break
  • en Unix, presione Control + Backslash
En Unix, puede también generar un volcado de subprocesos desde cualquier consola usando estos comandos:
  • encuentre el id del proceso de la aplicación, usando ps x | grep java
  • kill -3 <id-proceso-java>

Tenga en cuenta que esto en realidad no mata el proceso.

La sintaxis precisa y el contenido de un volcado de subprocesos es dependiente del Entorno de Ejecución de Java.

Las herramientas Java Mission Control, JConsole, y Visual VM también le permiten ver información sobre los subprocesos.
