Evite ThreadGroup

ThreadGroup fue originalmente destinado para la seguridad de los applets, y casi siempre debería ignorarse. Los servicios que proporciona son limitados, y a veces, incluso defectuosos.

El único método que podría ser útil es ThreadGroup. uncaughtException. Una sobreescritura de este método definirá el comportamiento personalizado para las excepciones no capturadas.

Vea Java Efectivo para una descripción detallada.
