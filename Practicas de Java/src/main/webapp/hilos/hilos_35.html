Recuerde los tipos de bloqueos intrínsecos

En Java, cada uso de la palabra clave synchronized implica un bloqueo intrínseco. En este caso, el bloqueo es realizado por Java en segundo plano. (Esto es distinto de que el programador utilice o defina un objeto de bloqueo explícito por sí mismo).

Cada uso de la palabra clave synchronized se asocia con uno de los dos tipos de bloqueos intrínsecos:

  • un "bloqueo de instancia", adjuntado a un solo objeto
  • un "bloqueo estático", adjuntado a una clase

Si se declara un método como synchronized, entonces adquirirá o el bloqueo de instancia o el bloqueo estático cuando se invoque, según sea un método de instancia o un método estático.

Los dos tipos de bloqueo tienen un comportamiento similar, pero son completamente independientes unos de otros.

Adquirir el bloqueo de instancia solo bloquea otros subprocesos de invoar un método de instancia synchronized; no bloquea a otros subprocesos de invocar un método no-sincronizado, ni los bloquea de invocar un método estático synchronized.

Del mismo modo, adquirir el bloqueo estático solo bloquea a otros subprocesos de invocar un método estático synchronized; no bloquea a otros subprocesos de invocar un método no-sincronizado, ni los bloquea de invocar un método de instancia synchronized.

Fuera del encabezado de un método, synchronized (this) adquiere el bloqueo de instancia.

Se puede adquirir el bloqueo estático fuera de un encabezado de método de dos formas:

  • synchronized(Blah.class), usando el literal class
  • synchronized(this.getClass()), si un objeto está disponible
