Sobre los WebSockets en JSF

Utiliza la etiqueta f:websocket en una vista para permitir que las comunicaciones del lado del servidor se envíen a todas las instancias de un socket que contenga el mismo nombre de canal. Cuando se recibe la comunicación, se puede configurar un controlador de eventos de JavaScript del lado del cliente onmessage que se llama cada vez que llega una notificación del servidor.

El lado del servidor de una comunicación WebSocket tiene la capacidad de enviar mensajes. Puede hacer esto usando javax.faces.push.PushContext, que es un contexto inyectable, lo que permite que un servidor envíe a un canal con nombre.
