Registrando un Renderizador Personalizado con un Kit de Renderizado

Cuando el desarrollador de la aplicación crea un renderizador personalizado, como se describe en Delegación de Renderizado a un Renderizador. debe registrarlo utilizando el kit de renderizado apropiado. Debido a que la aplicación de mapas de imágenes implementa un mapa de imágenes HTML, las clases AreaRenderer y MapRenderer en el estudio de caso de la librería Duke's Bookstore deben registrarse usando el kit de renderizado HTML.

El renderizador se registra mediante la anotación @FacesRenderer, como se describe en Creando la Clase del Renderizador, o utilizando el elemento render-kit del archivo de recursos de configuración de la aplicación. Aquí hay una configuración hipotética de AreaRenderer: 


<render-kit>
    <renderer>
        <component-family>Area</component-family>
        <renderer-type>DemoArea</renderer-type>
        <renderer-class>
            dukesbookstore.renderers.AreaRenderer
        </renderer-class>
        <attribute>
            <attribute-name>onmouseout</attribute-name>
            <attribute-class>java.lang.String</attribute-class>
        </attribute>
        <attribute>
            <attribute-name>onmouseover</attribute-name>
            <attribute-class>java.lang.String</attribute-class>
        </attribute>
        <attribute>
            <attribute-name>styleClass</attribute-name>
            <attribute-class>java.lang.String</attribute-class>
        </attribute>
    </renderer>
    ...

Los atributos especificados en una etiqueta renderer anulan cualquier configuración en la anotación @FacesRenderer.

El elemento render-kit representa una implementación javax.faces.render.RenderKit. Si no se especifica render-kit-id, se asume el kit de renderizado HTML predeterminado. El elemento renderer representa una implementación de javax.faces.render.Renderer. Al anidar el elemento renderer dentro del elemento render-kit, está registrando el renderizador con la implementación RenderKit asociada con el elemento render-kit.

render-class es el nombre de clase completamente calificado del Renderer.

Los elementos component-family y renderer-type son utilizados por un componente para encontrar renderizadores que puedan renderizarlo. El identificador component-family debe coincidir con el devuelto por el método getFamily de la clase de componente. La familia de componentes representa un componente o conjunto de componentes que puede representar un renderizador en particular. El renderer-type debe coincidir con el devuelto por el método getRendererType de la clase de controlador de etiquetas.

Al utilizar la familia de componentes y el tipo de renderizador para buscar renderizadores de componentes, la implementación de JavaServer Faces permite que un componente sea renderizado por múltiples renderizadores y permite que un renderizador renderice múltiples componentes.

Cada una de las etiquetas de attribute especifica un atributo dependiente de la representación y su tipo. El elemento attribute no afecta la ejecución en tiempo de ejecución de su aplicación. Más bien, proporciona información a las herramientas sobre los atributos que admite el Renderer.

El objeto responsable de renderizar un componente (ya sea el propio componente o un renderizador en el que el componente delega la renderización) puede usar facetas para ayudar en el proceso de renderizado. Estas facetas permiten al desarrollador de componentes personalizados controlar algunos aspectos de la representación del componente. Considere este ejemplo de etiqueta de componente personalizado: 


<d:dataScroller>
    <f:facet name="header">
        <h:panelGroup>
            <h:outputText value="Account Id"/>
            <h:outputText value="Customer Name"/>
            <h:outputText value="Total Sales"/>
        </h:panelGroup>
    </f:facet>
    <f:facet name="next">
        <h:panelGroup>
            <h:outputText value="Siguiente"/>
            <h:graphicImage url="/images/arrow-right.gif" />
        </h:panelGroup>
    </f:facet>
    ...
</d:dataScroller>

La etiqueta del componente dataScroller incluye un componente que representará el encabezado y un componente que representará el botón Siguiente. Si el renderizador asociado con este componente renderiza las facetas, puede incluir los siguientes elementos facet en el elemento render: 


<facet>
    <description>This facet renders as the header of the table. It should be
         a panelGroup with the same number of columns as the data.
    </description>
    <display-name>header</display-name>
    <facet-name>header</facet-name>
</facet>
<facet>
    <description>This facet renders as the content of the "next" button in
         the scroller. It should be a panelGroup that includes an outputText
         tag that has the text "Siguiente" and a right arrow icon.
    </description>
    <display-name>Siguiente</display-name>
    <facet-name>next</facet-name>
</facet>

Si un componente que admite facetas proporciona su propio renderizado y desea incluir elementos facet en el archivo de recursos de configuración de la aplicación, debe colocarlos en la configuración del componente en lugar de en la configuración del renderizador.
