Protección del Servidor GlassFish

Este tutorial describe la implementación en el servidor GlassFish, que proporciona computación de componentes distribuida, interoperable y altamente segura basada en el modelo de seguridad Java EE. El servidor GlassFish es compatible con el modelo de seguridad Java EE 8. Puede configurar el servidor GlassFish para los siguientes propósitos.

  • Agregar, eliminar o modificar usuarios autorizados. Para obtener más información sobre este tema, consulte Trabajar con Dominios, Usuarios, Grupos y Roles. Grupos y Roles.

  • Configuración de oyentes seguros de HTTP e Internet Inter-Orb Protocol (IIOP).

  • Configuración de conectores JMX (Java Management Extensions) seguros.

  • Agregar, eliminar o modificar dominios personalizados o existentes.

  • Definición de una interfaz para proveedores de autorización conectables mediante el contrato de autorización de Java para contenedores (JACC). JACC define los contratos de seguridad entre el servidor GlassFish y los módulos de políticas de autorización. Estos contratos especifican cómo se instalan, configuran y utilizan los proveedores de autorización en las decisiones de acceso.

  • Uso de módulos de auditoría enchufables.

  • Personalización de los mecanismos de autenticación. Todas las implementaciones de contenedores web compatibles con Java EE 7 deben admitir el perfil de servlet de JSR 196, que ofrece una vía para personalizar el mecanismo de autenticación aplicado por el contenedor web en nombre de una o más aplicaciones.

  • Configuración y cambio de permisos de política para una aplicación.
