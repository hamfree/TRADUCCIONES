Recibiendo una Respuesta Ajax

Después de que la aplicación envía una solicitud Ajax, se procesa en el lado del servidor y se envía una respuesta al cliente. Como se describió anteriormente, Ajax permite la actualización parcial de páginas web. Para habilitar dicha actualización parcial, la tecnología JavaServer Faces permite el procesamiento parcial de la vista. El manejo de la respuesta está definido por el atributo render de la etiqueta f:ajax.

Similar al atributo execute, el atributo render define qué secciones de la página se actualizarán. El valor de un atributo render puede ser uno o más valores de componente id, una de las palabras clave @this, @all, @none o @form, o una expresión EL. En el siguiente ejemplo, el atributo render identifica un componente de salida que se mostrará cuando se haga clic en el componente de botón (el evento predeterminado para un botón de comando): 


<h:commandButton id="submit" value="Submit">
    <f:ajax execute="userNo" render="result" />
</h:commandButton>
<h:outputText id="result" value="#{userNumberBean.response}" />

Nota:

Detrás de escena, una vez más, el método jsf.ajax.request() maneja la respuesta. Registra una devolución de llamada de manejo de respuesta cuando se crea la solicitud original. Cuando la respuesta se envía al cliente, se invoca la devolución de llamada. Esta devolución de llamada actualiza automáticamente el DOM del lado del cliente para reflejar la respuesta renderizada.
