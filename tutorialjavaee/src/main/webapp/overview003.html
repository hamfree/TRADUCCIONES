Modelo de aplicación Java EE

El Modelo de aplicación Java EE comienza con el lenguaje de programación Java y la máquina virtual Java. La portabilidad probada, la seguridad y la productividad del desarrollador que brindan forman la base del modelo de aplicación. Java EE está diseñado para admitir aplicaciones que implementan servicios empresariales para clientes, empleados, proveedores, socios y otras personas que realizan demandas o contribuciones a la empresa. Dichas aplicaciones son intrínsecamente complejas, y potencialmente acceden a datos de una variedad de fuentes y distribuyen aplicaciones a una variedad de clientes.

Para controlar y administrar mejor estas aplicaciones, las funciones comerciales para dar soporte a estos distintos usuarios se llevan a cabo en el nivel medio. El nivel medio representa un entorno que está estrechamente controlado por el departamento de tecnología de la información de una empresa. El nivel medio generalmente se ejecuta en hardware de servidor dedicado y tiene acceso a todos los servicios de la empresa.

El Modelo de aplicación Java EE define una arquitectura para implementar servicios como aplicaciones de varios niveles que brindan la escalabilidad, accesibilidad y capacidad de administración que necesitan las aplicaciones de nivel empresarial. Este modelo divide el trabajo necesario para implementar un servicio de varios niveles en las siguientes partes:

  • La lógica comercial y de presentación que debe implementar el desarrollador.

  • Los servicios del sistema estándar proporcionados por la plataforma Java EE.

El desarrollador puede confiar en la plataforma para proporcionar soluciones para los problemas difíciles a nivel de sistemas del desarrollo de un servicio de varios niveles.
