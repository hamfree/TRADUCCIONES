Configurando una Aplicación CDI

Cuando sus beans se anotan con un tipo de alcance, el servidor reconoce la aplicación como un archivo de beans y no se requiere ninguna configuración adicional. Los posibles tipos de ámbito para beans CDI se enumeran en Usando Ámbitos.

CDI usa un descriptor de implementación opcional llamado beans.xml. Al igual que otros descriptores de implementación de Java EE, los ajustes de configuración en beans.xml se usan además de los ajustes de anotación en las clases CDI. La configuración en beans.xml anula la configuración de anotación si hay un conflicto. Un archivo debe contener el descriptor de implementación beans.xml solo en ciertas situaciones limitadas, descritas en Capítulo 27, "Contextos e inyección de dependencia para Java EE : Temas avanzados" .

Para una aplicación web, el descriptor de implementación beans.xml, si está presente, debe estar en el directorio WEB-INF. Para módulos EJB o archivos JAR, el descriptor de implementación beans.xml, si está presente, debe estar en el directorio META-INF.
