Introducción a la Tecnología JavaServer Faces

La tecnología JavaServer Faces consta de lo siguiente:

  • Una IPA para representar componentes y administrar su estado; manejo de eventos, validación del lado del servidor y conversión de datos; definir la navegación de la página; apoyar la internacionalización y la accesibilidad; y proporcionar extensibilidad para todas estas funciones

  • Librerías de etiquetas para agregar componentes a páginas web y para conectar componentes a objetos del lado del servidor

La tecnología JavaServer Faces proporciona un modelo de programación bien definido y varias librerías de etiquetas. Las librerías de etiquetas contienen controladores de etiquetas que implementan las etiquetas de los componentes. Estas características alivian significativamente la carga de crear y mantener aplicaciones web con interfaces de usuario (IU) del lado del servidor. Con un mínimo esfuerzo, puede completar las siguientes tareas.

  • Crear una página web.

  • Colocar componentes en una página web agregando etiquetas de componentes.

  • Vincular componentes de una página a datos del lado del servidor.

  • Conectar los eventos generados por componentes al código de la aplicación del lado del servidor.

  • Guardar y restaurar el estado de la aplicación más allá de la vida útil de las solicitudes del servidor.

  • Reutilizar y ampliar los componentes mediante la personalización.

Este capítulo proporciona una descripción general de la tecnología JavaServer Faces. Después de explicar qué es una aplicación JavaServer Faces y revisar algunos de los beneficios principales de usar la tecnología JavaServer Faces, este capítulo describe el proceso de creación de una aplicación JavaServer Faces simple. Este capítulo también presenta el ciclo de vida de JavaServer Faces describiendo el ejemplo de la aplicación JavaServer Faces y su progresión a través de las etapas del ciclo de vida.
