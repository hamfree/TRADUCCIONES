Inyección de Dependencias

La inyección de dependencia le permite convertir clases Java normales en objetos gestionados e inyectarlos en cualquier otro objeto gestionado. Mediante la inyección de dependencias, su código puede declarar dependencias en cualquier objeto administrado. El contenedor proporciona automáticamente instancias de estas dependencias en los puntos de inyección en tiempo de ejecución y también administra el ciclo de vida de estas instancias por usted.

La inyección de dependencia en Java EE define ámbitos, que determinan el ciclo de vida de los objetos que el contenedor crea e inyecta. Por ejemplo, un objeto administrado que solo se necesita para responder a una solicitud de un solo cliente (como un convertidor de moneda) tiene un alcance diferente al de un objeto administrado que se necesita para procesar múltiples solicitudes de clientes dentro de una sesión (como un carrito de compras) .

Puede definir objetos administrados (también llamados beans administrados) que luego puede inyectar asignando un alcance a una clase regular: 


@javax.enterprise.context.RequestScoped
public class CurrencyConverter { ... }

Utilice la anotación javax.inject.Inject para inyectar beans gestionados; por ejemplo: 


public class MyServlet extends HttpServlet {
    @Inject CurrencyConverter cc;
    ...
}

A diferencia de la inyección de recursos, la inyección de dependencias es segura para los tipos porque se resuelve por tipo. Para desacoplar su código de la implementación del bean administrado, puede hacer referencia a las instancias inyectadas usando un tipo de interfaz y hacer que su bean administrado implemente esa interfaz.

Para obtener más información sobre la inyección de dependencia, consulte Capítulo 25, "Introducción a los contextos e inyección de dependencias para Java EE" y JSR 299 (Contextos e inyección de dependencia para la plataforma Java EE).
