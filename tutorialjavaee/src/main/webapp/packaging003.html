Empaquetando Archivos Web

En la arquitectura Java EE, un módulo web es la unidad de recursos web desplegable y utilizable más pequeña. Un módulo web contiene componentes web y archivos de contenido de Web estáticos, como imágenes, que se denominan recursos web. Un módulo web de Java EE corresponde a una aplicación web tal como se define en la especificación del servlet Java.

Además de los componentes web y los recursos web, un módulo web puede contener otros archivos:

  • Clases de utilidad del lado del servidor, como carritos de compras

  • Clases del lado del cliente, como las clases de utilidad.

Un módulo web tiene una estructura específica. El directorio de nivel superior de un módulo web es la raíz del documento de la aplicación. La raíz del documento es donde se almacenan las páginas XHTML, las clases y los archivos del lado del cliente, y los recursos web estáticos, como las imágenes.

La raíz del documento contiene un subdirectorio llamado WEB-INF, que puede contener los siguientes archivos y directorios:

  • classes, un directorio que contiene clases del lado del servidor: Servlets, archivos de clase de Bean Enterprise, clases de utilidad y componentes de Javabeans

  • lib, un directorio que contiene archivos JAR que contienen beans empresariales y archivos JAR o librerías llamadas por las clases del lado del servidor

  • Descriptores de implementación, como web.xml (el descriptor de implementación de la aplicación web) y ejb-jar.xml (un descriptor de implementación de EJB)

Un módulo web necesita un archivo web.xml si utiliza la tecnología de JavaServer Faces, si debe especificar ciertos tipos de información de seguridad, o si desea anular la información especificada por las anotaciones de componentes web.

También puede crear subdirectorios específicos de la aplicación (es decir, directorios de paquetes) en la raíz del documento o en el directorio WEB-INF/classes/.

Se puede implementar un módulo web como una estructura de archivos desempaquetada o se puede empaquetar en un archivo JAR conocido como un archivo de archivo web (WAR). Debido a que los contenidos y el uso de los archivos WAR difieren de los de los archivos JAR, los nombres de los archivos WAR utilizan una extensión .war. El módulo web que se acaba de describir es portátil; Puede implementarlo en cualquier contenedor web que cumpla con la especificación del servlet Java.

Puede proporcionar un descriptor de implementación de tiempo de ejecución (DD) cuando implemente un WAR en el servidor GlassFish, pero no se requiere en la mayoría de las circunstancias. El DD en tiempo de ejecución es un archivo XML que puede contener dicha información como la raíz de contexto de la aplicación web, el mapeo de los nombres portátiles de los recursos de una aplicación a los recursos del servidor de Glassfish y el mapeo de los roles de seguridad de una aplicación a los usuarios, grupos y principales definidos en el servidor de Glassfish. El DD en tiempo de de ejecución de la aplicación web en el servidor Glassfish, si se usa, se llama glassfish-web.xml y se encuentra en el directorio WEB-INF.
