Consultar Entidades

La IPA de Persistencia de Java proporciona los siguientes métodos para consultar entidades.

  • El Lenguaje de Consulta de Persistencia de Java (JPQL, en inglés) es un lenguaje simple basado en cadenas similar al SQL que se usa para consultar entidades y sus relaciones. Ver Capítulo 42, "El Lenguaje de Consulta de la Persistencia de Java" para más información.

  • La API de Criterios se usa para crear consultas con seguridad de tipos utilizando las IPA del lenguaje de programación Java para consultar entidades y sus relaciones. Ver Capítulo 43, "Usando la IPA Criteria para Crear Consultas" para más información.

Tanto JPQL como la IPA Criteria tienen ventajas y desventajas.

Con solo unas pocas líneas, las consultas JPQL suelen ser más concisas y más legibles que las consultas Criteria. A los desarrolladores familiarizados con SQL les resultará fácil aprender la sintaxis de JPQL. Las consultas con nombre JPQL se pueden definir en la clase de entidad mediante una anotación del lenguaje de programación Java o en el descriptor de implementación de la aplicación. Sin embargo, las consultas JPQL no tienen seguridad de tipos y requieren una conversión al recuperar el resultado de la consulta del administrador de entidades. Esto significa que es posible que los errores de conversión de tipos no se detecten en tiempo de compilación. Las consultas JPQL no admiten parámetros abiertos.

Las consultas con Criterios le permiten definir la consulta en el nivel de negocios de la aplicación. Aunque esto también es posible mediante consultas dinámicas JPQL, las consultas de criterios proporcionan un mejor rendimiento porque las consultas dinámicas JPQL deben analizarse cada vez que se llaman. Las consultas de criterios son de tipo seguro y, por lo tanto, no requieren conversión, como sí lo hacen las consultas JPQL. La IPA de Criterios es solo otra IPA del lenguaje de programación Java y no requiere que los desarrolladores aprendan la sintaxis de otro lenguaje de consulta. Las consultas de criterios suelen ser más detalladas que las consultas JPQL y requieren que el desarrollador cree varios objetos y realice operaciones en esos objetos antes de enviar la consulta al administrador de la entidad.
