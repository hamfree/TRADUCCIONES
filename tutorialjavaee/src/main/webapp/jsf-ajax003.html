Usando Ajax con Facelets

Como se mencionó en la sección anterior, la tecnología JavaServer Faces es compatible con Ajax mediante el uso de una biblioteca de recursos JavaScript incorporada que se proporciona como parte de las bibliotecas centrales de JavaServer Faces. Este recurso Ajax integrado se puede utilizar en aplicaciones web JavaServer Faces de una de las siguientes formas.

  • Usando la etiqueta f:ajax junto con otro componente estándar en una aplicación Facelets. Este método agrega la funcionalidad Ajax a cualquier componente de la interfaz de usuario sin codificación ni configuración adicionales.

  • Usando el método de la IPA de JavaScript jsf.ajax.request() directamente dentro de la aplicación Facelets. Este método proporciona acceso directo a los métodos Ajax y permite un control personalizado del comportamiento de los componentes.

  • Mediante el uso del componente <h:commandScript> para ejecutar métodos arbitrarios del lado del servidor desde una vista. El componente genera una función de JavaScript con un nombre dado que cuando se invoca, a su vez invoca, un método del lado del servidor dado a través de Ajax.

Usando la Etiqueta f:ajax

La etiqueta f:ajax es una etiqueta principal de JavaServer Faces que proporciona funcionalidad Ajax a cualquier componente de IU normal cuando se usa junto con ese componente. En el siguiente ejemplo, el comportamiento de Ajax se agrega a un componente de entrada al incluir la etiqueta principal f:ajax: 


<h:inputText value="#{bean.message}">
    <f:ajax />
</h:inputText>

En este ejemplo, aunque Ajax está habilitado, los otros atributos de la etiqueta f:ajax no están definidos. Si no se define un evento, se realiza la acción predeterminada para el componente. Para el componente inputText, cuando no se especifica ningún atributo event, el evento predeterminado es valueChange. La tabla 13-1 enumera los atributos de la etiqueta f:ajax y sus acciones predeterminadas.

Tabla 13-1 Atributos de la etiqueta f:ajax

Nombre

Tipo

Descripción

disabled

javax.el.ValueExpression que evalúa a un Boolean

Un valor Boolean que identifica el estado de la etiqueta. Un valor de true indica que el comportamiento de Ajax no se debe representar. Un valor de false indica que se debe representar el comportamiento de Ajax. El valor predeterminado es false.

event

javax.el.ValueExpression que evalúa a una String

Un objeto del tipo String que identifica el tipo de evento al que se aplicará la acción Ajax. Si se especifica, debe ser uno de los eventos admitidos por el componente. Si no se especifica, el evento predeterminado (el evento que desencadena la solicitud Ajax) se determina para el componente. El evento predeterminado es action para los componentes javax.faces.component.ActionSource y valueChange para los componentes javax.faces.component.EditableValueHolder.

execute

javax.el.ValueExpression que evalúa a un Object

A Collection that identifies a list of components to be executed on the server. If a literal is specified, it must be a space-delimited String of component identifiers and/or one of the keywords. If a ValueExpression is specified, it must refer to a property that returns a Collection of String objects. If not specified, the default value is @this.

immediate

javax.el.ValueExpression que evalúa a un Boolean

Un valor Boolean que indica si las entradas deben procesarse al principio del ciclo de vida. Si es true, los eventos de comportamiento generados a partir de este comportamiento se difunden durante la fase Aplicar Valores de Solicitud. De lo contrario, los eventos se transmitirán durante la fase de Solicitud de Invocación.

listener

javax.el.MethodExpression

El nombre del método de oyente que se llama cuando se ha transmitido un javax.faces.event.AjaxBehaviorEvent para el oyente.

onevent

javax.el.ValueExpression que evalúa a una String

El nombre de la función de JavaScript que maneja los eventos de la interfaz de usuario.

onerror

javax.el.ValueExpression que evalúa a una String

El nombre de la función de JavaScript que maneja los errores.

render

javax.el.ValueExpression que evalúa a un Object

Una Collection que identifica una lista de componentes que se mostrarán en el cliente. Si se especifica un literal, debe ser un objeto de tipo String delimitado por espacios de identificadores de componentes y/o una de las palabras clave. Si se especifica una ValueExpression, debe hacer referencia a una propiedad que devuelva una Collection de objetos String. Si no se especifica, el valor predeterminado es @none.

Las palabras clave enumeradas en la tabla 13-2 se pueden usar con los atributos execute y render de la etiqueta f:ajax.

Tabla 13-2 Palabras Clave de Ejecución y Procesamiento

Palabra Clave

Descripción

@all

Todos los identificadores de componentes

@form

La forma que encierra el componente

@none

Sin identificadores de componentes

@this

El elemento que desencadenó la solicitud.

Tenga en cuenta que cuando usa la etiqueta f:ajax en una página de Facelets, la biblioteca de recursos de JavaScript se carga implícitamente. Esta biblioteca de recursos también se puede cargar explícitamente como se describe en Cargando JavaScript como un Recurso.
