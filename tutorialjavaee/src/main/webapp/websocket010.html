Especificando una Clase Configurator Endpoint

El IPA de Java para WebSocket le permite configurar cómo el contenedor crea instancias de punto final del servidor. Puede proporcionar una lógica de configuración de punto final personalizada para:

  • Acceder a los detalles de la solicitud HTTP inicial para una conexión WebSocket

  • Realizar comprobaciones personalizadas en el encabezado HTTP Origin

  • Modificar la respuesta del protocolo de enlace WebSocket

  • Eligir un subprotocolo WebSocket de los solicitados por el cliente

  • Controlar la creación de instancias y la inicialización de instancias de punto final

Para proporcionar una lógica de configuración de punto final personalizada, extienda la clase ServerEndpointConfig.Configurator y anule algunos de sus métodos. En la clase de punto final, especifica la clase de configurador usando el parámetro configurator de la anotación ServerEndpoint.

Por ejemplo, la siguiente clase de configurador hace que el objeto de solicitud de protocolo de enlace esté disponible para instancias de punto final: 

              
public class CustomConfigurator extends ServerEndpointConfig.Configurator {
    @Override
    public void modifyHandshake(ServerEndpointConfig conf,
                                HandshakeRequest req,
                                HandshakeResponse resp) {
        conf.getUserProperties().put("handshakereq", req);
    }
}

La siguiente clase de punto final configura instancias de punto final con el configurador personalizado, lo que les permite acceder al objeto de solicitud de protocolo de enlace: 

              
@ServerEndpoint(
    value = "/myendpoint",
    configurator = CustomConfigurator.class
)
public class MyEndpoint {
    @OnOpen
    public void open(Session s, EndpointConfig conf) {
        HandshakeRequest req = (HandshakeRequest) conf.getUserProperties()
                                                      .get("handshakereq");
        Map<String,List<String>> headers = req.getHeaders();
        ...
    }
}

La clase de punto final puede usar el objeto de solicitud de protocolo de enlace para acceder a los detalles de la solicitud HTTP inicial, como sus encabezados o el objeto HttpSession.

Para obtener más información sobre la configuración de puntos finales, consulte la referencia de la IPA para la clase ServerEndpointConfig.Configurator.
