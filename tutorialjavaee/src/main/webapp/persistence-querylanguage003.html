Crear Consultas Usando el Lenguaje de Consultas de la Persistencia de Java

Los métodos EntityManager.createQuery y EntityManager.createNamedQuery se utilizan para consultar el almacén de datos mediante consultas del lenguaje de consulta de persistencia de Java.

El método createQuery se utiliza para crear consultas dinámicas, que son consultas definidas directamente dentro de la lógica de negocio de una aplicación: 

              
public List findWithName(String name) {
return em.createQuery(
    "SELECT c FROM Customer c WHERE c.name LIKE :custName")
    .setParameter("custName", name)
    .setMaxResults(10)
    .getResultList();
}

El método createNamedQuery se utiliza para crear consultas estáticas o consultas definidas en metadatos mediante la anotación javax.persistence.NamedQuery. El elemento name de @NamedQuery especifica el nombre de la consulta que se usará con el método createNamedQuery. El elemento query de @NamedQuery es la consulta: 

              
@NamedQuery(
    name="findAllCustomersWithName",
    query="SELECT c FROM Customer c WHERE c.name LIKE :custName"
)

Aquí hay un ejemplo de createNamedQuery, que usa @NamedQuery: 

              
@PersistenceContext
public EntityManager em;
...
customers = em.createNamedQuery("findAllCustomersWithName")
    .setParameter("custName", "Smith")
    .getResultList();

Parámetros Nombrados en Consultas

Los parámetros con nombre son parámetros de consulta que tienen el prefijo de dos puntos (:). Los parámetros con nombre en una consulta están vinculados a un argumento mediante el siguiente método: 

                
javax.persistence.Query.setParameter(String name, Object value)

En el siguiente ejemplo, el argumento name del método de negocio findWithName está vinculado al parámetro con nombre :custName en la consulta llamando a Query.setParameter: 

                
public List findWithName(String name) {
    return em.createQuery(
        "SELECT c FROM Customer c WHERE c.name LIKE :custName")
        .setParameter("custName", name)
        .getResultList();
}

Los parámetros con nombre distinguen entre mayúsculas y minúsculas y pueden ser utilizados tanto por consultas dinámicas como estáticas.

Parámetros Posicionales en Consultas

Puede usar parámetros posicionales en lugar de parámetros con nombre en las consultas. Los parámetros posicionales tienen como prefijo un signo de interrogación (?) seguido de la posición numérica del parámetro en la consulta. El método Query.setParameter(posición de número entero, valor del objeto) se utiliza para establecer los valores de los parámetros.

En el siguiente ejemplo, el método de negocio findWithName se reescribe para usar parámetros de entrada: 

                
public List findWithName(String name) {
    return em.createQuery(
        "SELECT c FROM Customer c WHERE c.name LIKE ?1")
        .setParameter(1, name)
        .getResultList();
}

Los parámetros de entrada se numeran a partir de 1. Los parámetros de entrada distinguen entre mayúsculas y minúsculas y se pueden usar tanto en consultas dinámicas como estáticas.
