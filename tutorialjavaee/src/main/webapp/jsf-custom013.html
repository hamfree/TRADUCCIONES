Enlazando Valores e Instancias de Componente a Propiedades del Bean Gestionado

Una etiqueta de componente puede conectar sus datos a un bean administrado mediante uno de los siguientes métodos:

  • Vinculando el valor de su componente a una propiedad del bean

  • Vinculando la instancia de su componente a una propiedad del bean

Para vincular el valor de un componente a una propiedad del bean gestionado, el atributo value de una etiqueta de componente usa una expresión de valor EL. Para vincular una instancia de componente a una propiedad del bean, el atributo binding de una etiqueta de componente usa una expresión de valor.

Cuando una instancia de componente está vinculada a una propiedad del bean gestionado, la propiedad contiene el valor local del componente. Por el contrario, cuando el valor de un componente está vinculado a una propiedad del bean gestionado, la propiedad contiene el valor almacenado en el bean gestionado. Este valor se actualiza con el valor local durante la fase Actualizar Valores del modelo del ciclo de vida. Ambos métodos tienen ventajas.

La vinculación de una instancia de componente a una propiedad de bean tiene las siguientes ventajas.

  • El bean gestionado puede modificar mediante programación los atributos de los componentes.

  • El bean gestionado puede crear instancias de componentes en lugar de permitir que el autor de la página lo haga.

Vincular el valor de un componente a una propiedad del bean tiene las siguientes ventajas.

  • El autor de la página tiene más control sobre los atributos del componente.

  • El bean gestionado no depende de la IPA de JavaServer Faces (como las clases de componentes), lo que permite una mayor separación de la capa de presentación de la capa del modelo.

  • La implementación de JavaServer Faces puede realizar conversiones en los datos según el tipo de propiedad del bean sin que el desarrollador necesite aplicar un convertidor.

En la mayoría de las situaciones, vinculará el valor de un componente en lugar de su instancia a una propiedad de bean. Deberá utilizar un enlace de componente solo cuando necesite cambiar uno de los atributos del componente de forma dinámica. Por ejemplo, si una aplicación renderiza un componente solo bajo ciertas condiciones, puede establecer la propiedad rendered del componente en consecuencia accediendo a la propiedad a la que está vinculado el componente.

Al hacer referencia a la propiedad utilizando el atributo value de la etiqueta de componente, debe utilizar la sintaxis adecuada. Por ejemplo, suponga que un bean gestionado llamado MyBean tiene esta propiedad int: 


protected int currentOption = null;
public int getCurrentOption(){...}
public void setCurrentOption(int option){...}

El atributo value que hace referencia a esta propiedad debe tener esta expresión de vinculación de valor: 


#{myBean.currentOption}

Además de vincular el valor de un componente a una propiedad de bean, el atributo value puede especificar un valor literal o puede asignar los datos del componente a cualquier tipo primitivo (como int), estructura (como una matriz) o colección (como una lista), independientemente de un componente JavaBeans. La tabla 15-3 enumera algunos ejemplos de expresiones de vinculación de valores que puede usar con el atributo value.

Las siguientes dos secciones explican cómo usar el atributo value para vincular el valor de un componente a una propiedad de bean u otros objetos de datos y cómo usar el atributo binding para vincular una instancia de componente a una propiedad de bean.

Enlazando un Valor de Componente a una Propiedad

Para vincular el valor de un componente a una propiedad de bean gestionado, especifique el nombre del bean y la propiedad usando el atributo value.

Esto significa que la primera parte de la expresión de valor EL debe coincidir con el nombre del bean gestionado hasta el primer punto (.) y la parte de la expresión de valor después del punto debe coincidir con la propiedad del bean gestionado.

Por ejemplo, en el caso de estudio de Duke's Bookstore, la etiqueta h:dataTable en bookcatalog.xhtml establece el valor del componente en el valor de la propiedad books del bean de respaldo BookstoreBean, cuyo nombre es store: 


<h:dataTable id="books"
             value="#{store.books}"
             var="book"
             headerClass="list-header"
             styleClass="list-background"
             rowClasses="list-row-even, list-row-odd"
             border="1"
             summary="#{bundle.BookCatalog}">

El valor se obtiene llamando al método getBooks del bean de respaldo, que a su vez llama al método getBooks del bean de sesión BookRequestBean.

Si usa el archivo de recursos de configuración de la aplicación para configurar beans gestionados en lugar de definirlos en clases de beans gestionados, el nombre del bean en la expresión value debe coincidir con el elemento nombre-bean-gestionado de la declaración del bean gestionado hasta el primer punto (.) en la expresión. De manera similar, la parte de la expresión de valor después del punto debe coincidir con el nombre especificado en el elemento nombre-propiedad correspondiente en el archivo de recursos de configuración de la aplicación.

Por ejemplo, considere esta configuración de bean gestionado, que configura el bean ImageArea correspondiente al libro superior izquierdo en el mapa de imágenes en la página index.html del caso de estudio de Duke's Bookstore: 


<managed-bean eager="true">
    ...
    <managed-bean-name>Book201</managed-bean-name>
    <managed-bean-class>dukesbookstore.model.ImageArea</managed-bean-class>
    <managed-bean-scope>application</managed-bean-scope>
    <managed-property>
        ...
        <property-name>shape</property-name>
        <value>rect</value>
    </managed-property>
    <managed-property>
        ...
        <property-name>alt</property-name>
        <value>Duke</value>
    </managed-property>
    ...

Este ejemplo configura un bean llamado Book201, que tiene varias propiedades, una de las cuales se llama shape.

Aunque las etiquetas bookstore:area en la página index.xhtml no se vinculan a una propiedad ImageArea (se vinculan al bean en sí), podría hacer referencia a la propiedad mediante una expresión de valor del atributo value de la etiqueta del componente: 


<h:outputText value="#{Book201.shape}" />

Vea Configurando Beans Gestionados para obtener información sobre cómo configurar beans en el archivo de recursos de configuración de la aplicación.

Enlazando un Valor de Componente a un Objeto Implícito

Una fuente de datos externa a la que puede hacer referencia un atributo value es un objeto implícito.

La página bookreceipt.xhtml del caso de estudio de Duke's Bookstore tiene una referencia a un objeto implícito: 


<h:outputFormat title="thanks"
                value="#{bundle.ThankYouParam}">
    <f:param value="#{sessionScope.name}"/>
</h:outputFormat>

Esta etiqueta obtiene el nombre del cliente del alcance de la sesión y lo inserta en el mensaje parametrizado en la clave ThankYouParam del paquete de recursos. Por ejemplo, si el nombre del cliente es Gwen Canigetit, esta etiqueta mostrará: 


Gracias, Gwen Canigetit, por comprarnos sus libros.

La recuperación de valores de otros objetos implícitos se realiza de forma similar al ejemplo que se muestra en esta sección. La tabla 15-4 enumera los objetos implícitos a los que puede hacer referencia un atributo de valor. Todos los objetos implícitos, excepto los objetos de alcance, son de solo lectura y, por lo tanto, no deben usarse como valores para un componente UIInput.

Enlazando una Instancia de Componente a una Propiedad de Bean

Una instancia de componente se puede vincular a una propiedad de bean usando una expresión de valor con el atributo binding de la etiqueta del componente. Por lo general, vincula una instancia de componente en lugar de su valor a una propiedad de bean si el bean debe cambiar dinámicamente los atributos del componente.

Aquí hay dos etiquetas de la página bookcashier.xhtml que vinculan componentes a propiedades de bean: 


<h:selectBooleanCheckbox id="fanClub"
                         rendered="false"
                         binding="#{cashierBean.specialOffer}" />
<h:outputLabel for="fanClub"
               rendered="false"
               binding="#{cashierBean.specialOfferText}"
               value="#{bundle.DukeFanClub}"/>
</h:outputLabel>

La etiqueta h:selectBooleanCheckbox muestra una casilla de verificación y vincula el componente fanClub UISelectBoolean a la propiedad specialOffer del bean cashier. La etiqueta h:outputLabel une el componente que representa la etiqueta de la casilla de verificación a la propiedad specialOfferText del bean cashier. Si la configuración regional de la aplicación es inglés, se representa la etiqueta h:outputLabel. 


I'd like to join the Duke Fan Club, free with my purchase of over $100

Los atributos rendered de ambas etiquetas se establecen en false para evitar que la casilla de verificación y su etiqueta se rendericen. Si el cliente hace un pedido grande y hace clic en el botón Enviar, el método submit de CashierBean establece las propiedades rendered de ambos componentes en true, lo que hace que se representen la casilla de verificación y su etiqueta.

Estas etiquetas utilizan enlaces de componentes en lugar de enlaces de valores porque el bean gestionado debe establecer dinámicamente los valores de las propiedades rendered de los componentes.

Si las etiquetas usaran enlaces de valor en lugar de enlaces de componentes, el bean gestionado no tendría acceso directo a los componentes y, por lo tanto, requeriría código adicional para acceder a los componentes desde la instancia FacesContext para cambiar las propiedades rendered de los componentes.
