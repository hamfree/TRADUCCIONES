Conjuntos de Caracteres y Codificaciones

Las siguientes secciones describen juegos de caracteres y codificaciones de caracteres.

Aquí se tratan los siguientes temas:

  • Conjuntos de Caracteres

  • Codificación de Caracteres

Conjuntos de Caracteres

Un conjunto de caracteres es un conjunto de símbolos textuales y gráficos, cada uno de los cuales se asigna a un conjunto de números enteros no negativos.

El primer conjunto de caracteres utilizado en informática fue US-ASCII. Está limitado en el sentido de que solo puede representar inglés americano. US-ASCII contiene alfabetos latinos en mayúsculas y minúsculas, números, puntuación, un conjunto de códigos de control y algunos símbolos diversos.

Unicode define un conjunto de caracteres universales y estandarizados que se pueden ampliar para adaptarse a las adiciones. Cuando la codificación del archivo fuente del programa Java no es compatible con Unicode, puede representar los caracteres Unicode como secuencias de escape utilizando la notación `\u`XXXX, donde XXXX es la representación hexadecimal de 16 bits del carácter. Por ejemplo, la versión en español de un archivo de mensaje podría usar Unicode para caracteres que no sean ASCII, de la siguiente manera: 

                
admin.nav.main=P\u00e1gina principal de administraci\u00f3n

Codificación de Caracteres

Una codificación de caracteres asigna un conjunto de caracteres a unidades de un ancho específico y define las reglas de ordenación y serialización de bytes. Muchos conjuntos de caracteres tienen más de una codificación. Por ejemplo, los programas Java pueden representar juegos de caracteres japoneses utilizando las codificaciones EUC-JP o Shift-JIS, entre otras. Cada codificación tiene reglas para representar y serializar un juego de caracteres.

La serie ISO 8859 define 13 codificaciones de caracteres que pueden representar textos en docenas de idiomas. Cada codificación de caracteres ISO 8859 puede tener hasta 256 caracteres. ISO-8859-1 (Latin-1) comprende el conjunto de caracteres ASCII, caracteres con signos diacríticos (acentos, diéresis, cedillas, circunflejos, etc.) y símbolos adicionales.

UTF-8 (Formato de Transformación Unicode, formato de 8 bits) es una codificación de caracteres de ancho variable que codifica caracteres Unicode de 16 bits como uno a cuatro bytes. Un byte en UTF-8 es equivalente a ASCII de 7 bits si su bit de orden superior es cero; de lo contrario, el carácter comprende un número variable de bytes.

UTF-8 es compatible con la mayoría del contenido web existente y brinda acceso al juego de caracteres Unicode. Las versiones actuales de navegadores y clientes de correo electrónico admiten UTF-8. Además, muchos estándares web especifican UTF-8 como su codificación de caracteres. Por ejemplo, UTF-8 es una de las dos codificaciones requeridas para documentos XML (la otra es UTF-16).

Los componentes web suelen utilizar PrintWriter para generar respuestas; PrintWriter codifica automáticamente usando ISO-8859-1. Los servlets también pueden generar datos binarios utilizando las clases OutputStream, que no realizan codificación. Una aplicación que utiliza un conjunto de caracteres que no puede utilizar la codificación predeterminada debe establecer explícitamente una codificación diferente.
