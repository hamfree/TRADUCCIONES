Sintaxis Simplificada del Lenguaje de Consultas

Esta sección describe brevemente la sintaxis del lenguaje de consulta para que pueda pasar rápidamente a Consultas de Ejemplo. Cuando esté listo para conocer la sintaxis con más detalle, consulte Sintaxis Completa del Lenguaje de Consulta.

Aquí se tratan los siguientes temas:

  • Sentencias Select

  • Sentencias Update y Delete

Sentencias Select

Una consulta de selección tiene seis cláusulas: SELECT, FROM, WHERE, GROUP BY, HAVING y ORDENAR POR. Las cláusulas SELECT y FROM son obligatorias, pero WHERE, GROUP BY, HAVING, y las cláusulas ORDER BY son opcionales. Esta es la sintaxis BNF de alto nivel de una consulta de selección de idioma de consulta: 

                
QL_statement ::= select_clause from_clause
  [where_clause][groupby_clause][having_clause][orderby_clause]

La sintaxis BNF define las siguientes cláusulas.

  • La cláusula SELECT define los tipos de objetos o valores devueltos por la consulta.

  • La cláusula FROM define el alcance de la consulta al declarar una o más variables de identificación, a las que se puede hacer referencia en las cláusulas SELECT y WHERE. Una variable de identificación representa uno de los siguientes elementos:

    • El nombre de esquema abstracto de una entidad.

    • Un elemento de una relación de colección.

    • Un elemento de una relación de un solo valor

    • Un miembro de una colección que es el lado múltiple de una relación de uno a muchos

  • La cláusula WHERE es una expresión condicional que restringe los objetos o valores recuperados por la consulta. Aunque la cláusula es opcional, la mayoría de las consultas tienen una cláusula WHERE.

  • La cláusula GROUP BY agrupa los resultados de la consulta según un conjunto de propiedades.

  • La cláusula HAVING se usa con la cláusula GROUP BY para restringir aún más los resultados de la consulta según una expresión condicional.

  • La cláusula ORDER BY ordena los objetos o valores devueltos por la consulta en un orden específico.

Sentencias Update y Delete

Las declaraciones de actualización y eliminación proporcionan operaciones masivas sobre conjuntos de entidades. Estas sentencias tienen la siguiente sintaxis: 

                
update_statement :: = update_clause [where_clause]
delete_statement :: = delete_clause [where_clause]

Las cláusulas de actualización y eliminación determinan el tipo de entidades que se actualizarán o eliminarán. La cláusula WHERE se puede utilizar para restringir el alcance de la operación de actualización o eliminación.
