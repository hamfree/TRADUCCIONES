Registrando un Validador Personalizado

Si el desarrollador de la aplicación proporciona una implementación de la interfaz javax.faces.validator.Validator para realizar la validación, debe registrar este validador personalizado utilizando la anotación @FacesValidator, como descrito en Implementando la Interfaz Validator, o utilizando el elemento XML validator en el archivo de recursos de configuración de la aplicación: 


<validator>
    ...
    <validator-id>FormatValidator</validator-id>
    <validator-class>
        myapplication.validators.FormatValidator
    </validator-class>
    <attribute>
        ...
        <attribute-name>formatPatterns</attribute-name>
        <attribute-class>java.lang.String</attribute-class>
    </attribute>
</validator>

Los atributos especificados en una etiqueta validator anulan cualquier configuración en la anotación @FacesValidator.

Los elementos validator-id y validator-class son subelementos obligatorios. El elemento validator-id representa el identificador bajo el cual se debe registrar la clase Validator. Este ID es utilizado por la clase de etiqueta correspondiente a la etiqueta personalizada validator.

El elemento validator-class representa el nombre de clase completamente calificado de la clase Validator.

El elemento attribute identifica un atributo asociado con la implementación de Validator. Tiene subelementos obligatorios attribute-name y attribute-class. El elemento attribute-name se refiere al nombre del atributo tal como aparece en la etiqueta validator. El elemento attribute-class identifica el tipo de Java del valor asociado con el atributo.
