Inyección de Recursos

La inyección de recursos le permite inyectar cualquier recurso disponible en el espacio de nombres JNDI en cualquier objeto administrado por contenedor, como un servlet, un bean empresarial o un bean administrado. Por ejemplo, puede utilizar la inyección de recursos para inyectar fuentes de datos, conectores o recursos personalizados disponibles en el espacio de nombres JNDI.

El tipo que usa para la referencia a la instancia inyectada suele ser una interfaz, que desacopla su código de la implementación del recurso.

Por ejemplo, el siguiente código inyecta un objeto de fuente de datos que proporciona conexiones a la base de datos Apache Derby predeterminada que se envía con GlassFish Server: 


public class MyServlet extends HttpServlet {
    @Resource(name="java:comp/DefaultDataSource")
    private javax.sql.DataSource dsc;
    ...
}

Además de la inyección basada en el campo como en el ejemplo anterior, puede inyectar recursos mediante la inyección basada en el método: 


public class MyServlet extends HttpServlet {
    private javax.sql.DataSource dsc;
    ...
    @Resource(name="java:comp/DefaultDataSource")
    public void setDsc(java.sql.DataSource ds) {
        dsc = ds;
    }
}

Para usar la inyección basada en métodos, el método setter debe seguir las convenciones de JavaBeans para nombres de propiedad: El nombre del método debe comenzar con set, tener un tipo de retorno void y solo un parámetro.

La anotación @Resource está en el paquete javax.annotation y está definida en JSR 250 (Anotaciones Comunes para la Plataforma Java). La inyección de recursos se resuelve por nombre, por lo que no es seguro para los tipos: el tipo del objeto de recurso no se conoce en el momento de la compilación, por lo que puede obtener errores en tiempo de ejecución si los tipos del objeto y su referencia no coinciden.
