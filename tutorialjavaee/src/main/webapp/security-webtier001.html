Descripción General de la Seguridad de Aplicaciones Web

Se accede a una aplicación web mediante un navegador web a través de una red, como Internet o la intranet de una empresa. Como se explica en Aplicaciones distribuidas de varios niveles, la plataforma Java EE utiliza un modelo de aplicación distribuida de varios niveles y las aplicaciones web se ejecutan en el nivel web.

Las aplicaciones web contienen recursos a los que muchos usuarios pueden acceder. Estos recursos a menudo atraviesan redes abiertas y desprotegidas, como Internet. En tal entorno, un número considerable de aplicaciones web requerirán algún tipo de seguridad.

La protección de las aplicaciones y sus clientes en el nivel empresarial y el nivel EIS se analiza en el Capítulo 52, "Introducción a la Protección de Aplicaciones Empresariales" .

En la plataforma Java EE, los componentes web proporcionan capacidades de extensión dinámica para un servidor web. Los componentes web pueden ser servlets Java o páginas JavaServer Faces.

Ciertos aspectos de la seguridad de la aplicación web se pueden configurar cuando la aplicación se instala o implementa en el contenedor web. Las anotaciones y/o los descriptores de implementación se utilizan para transmitir información al implementador sobre seguridad y otros aspectos de la aplicación. Especificar esta información en anotaciones o en el descriptor de implementación ayuda al implementador a configurar la política de seguridad adecuada para la aplicación web. Cualquier valor especificado explícitamente en el descriptor de implementación anula cualquier valor especificado en las anotaciones.

La seguridad de las aplicaciones web Java EE se puede implementar de las siguientes formas.

  • La seguridad declarativa se puede implementar utilizando anotaciones de metadatos o el descriptor de implementación de una aplicación. Consulte la Descripción General de la Seguridad de Java EE para obtener más información.

    La seguridad declarativa para aplicaciones web se describe en Protección de Aplicaciones Web.

  • La seguridad programática está integrada en una aplicación y se puede utilizar para tomar decisiones de seguridad cuando la seguridad declarativa por sí sola no es suficiente para expresar el modelo de seguridad de una aplicación. La seguridad declarativa por sí sola puede no ser suficiente cuando se requiere un inicio de sesión condicional en un flujo de trabajo particular, en lugar de para todos los casos, en medio de una aplicación. Consulte la Descripción General de la Seguridad de Java EE para obtener más información.

    Servlet 4.0 proporciona los métodos authenticate, login y logout de la interfaz HttpServletRequest. Con la adición de los métodos authenticate, login y logout a la especificación de Servlet, ya no se requiere un descriptor de implementación de aplicaciones para las aplicaciones web, pero todavía se puede utilizar para especificar más requisitos de seguridad más allá de los valores predeterminados básicos.

    La seguridad programática se trata en Uso de Seguridad Programática con Aplicaciones Web .

  • La seguridad de mensajes funciona con servicios web e incorpora funciones de seguridad, como firmas digitales y cifrado, en el encabezado de un mensaje SOAP, trabajando en la capa de aplicación, lo que garantiza la seguridad de un extremo a otro. La seguridad de mensajes no es un componente de Java EE y se menciona aquí solo con fines informativos.
