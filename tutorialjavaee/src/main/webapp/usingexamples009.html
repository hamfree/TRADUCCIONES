Depurando Aplicaciones de Java EE

Esta sección explica cómo determinar qué está causando un error en la implementación o ejecución de su aplicación.

Uso del registro del servidor

Una forma de depurar aplicaciones es mirar el registro del servidor en domain-dir`/logs/server.log`. El registro contiene la salida de GlassFish Server y sus aplicaciones. Puede registrar mensajes de cualquier clase de Java en su aplicación con System.out.println y las IPA de registro de Java (documentadas en http://docs.oracle.com/javase/8/docs/technotes/guides/logging/index.html) y de componentes web con el método ServletContext.log.

Si usa NetBeans IDE, la salida del registro aparece en la ventana Salida, así como en el registro del servidor.

Si inicia GlassFish Server con el indicador --verbose, todas las salidas de registro y depuración aparecerán en la ventana del terminal o en el símbolo del sistema y en el registro del servidor. Si inicia GlassFish Server en segundo plano, la información de depuración solo está disponible en el registro. Puede ver el registro del servidor con un editor de texto o con el visor de registros de la Consola de administración.

Para Usar el Visor del Registro de la Consola de Administración

  1. Seleccione el nodo GlassFish Server.

  2. Pulse Ver archivos de registro.

    El visor de registro se abre y visualiza las últimas 40 entradas.

  3. Para mostrar otras entradas, siga estos pasos:

    1. Pulse Modificar Búsqueda.

    2. Especifique las restricciones en las entradas que desea ver.

    3. Haga clic en Buscar en la parte superior del visor de registros.

Usando un Depurador

GlassFish Server es compatible con Plataforma Java Debugger Architecture (JPDA). Con JPDA, puede configurar GlassFish Server para comunicar información de depuración mediante un socket.

Para Depurar una Aplicación Usando un Depurador

  1. Siga estos pasos para habilitar la depuración en GlassFish Server usando la Consola de administración:

    1. Expanda el nodo Configuraciones, luego expanda el nodo server-config.

    2. Seleccione el nodo Configuración de JVM. Las opciones de depuración predeterminadas se establecen en: 

      
-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=9009

      Como puede ver, el puerto del socket del depurador predeterminado es 9009. Puede cambiarlo a un puerto que GlassFish Server u otro servicio no utilice.

    3. Seleccione la casilla de verificación Depuración habilitada.

    4. Pulse Salvar.

  2. Detenga GlassFish Server y luego reinícielo.
