Recursos Web

Los recursos web son cualquier artefacto de software que la aplicación web requiere para una representación adecuada, incluidas imágenes, archivos de script y cualquier biblioteca de componentes creada por el usuario. Los recursos deben recolectarse en una ubicación estándar, que puede ser una de las siguientes.

  • Un recurso empaquetado en la raíz de la aplicación web debe estar en un subdirectorio de un directorio resources en la raíz de la aplicación web: `resources/`resource-identifier.

  • Un recurso empaquetado en la ruta de clases de la aplicación web debe estar en un subdirectorio del directorio META-INF/resources dentro de una aplicación web: `META-INF/resources/`resource-identifier. Puede utilizar esta estructura de archivos para empaquetar recursos en un archivo JAR incluido en la aplicación web.

El tiempo de ejecución de JavaServer Faces buscará los recursos en las ubicaciones enumeradas anteriormente, en ese orden.

Los identificadores de recursos son cadenas únicas que se ajustan al siguiente formato (todo en una línea): 

              
[locale-prefix/][library-name/][library-version/]resource-name[/resource-version]

Los elementos del identificador de recurso entre paréntesis ([]) son opcionales, lo que indica que solo un nombre de recurso, que suele ser un nombre de archivo, es un elemento obligatorio. Por ejemplo, la forma más común de especificar una hoja de estilo, imagen o secuencia de comandos es usar los atributos library y name, como en la siguiente etiqueta del ejemplo guessnumber-jsf: 

              
<h:outputStylesheet library="css" name="default.css"/>

Esta etiqueta especifica que la hoja de estilo default.css está en el directorio web/resources/css.

También puede especificar la ubicación de una imagen usando la siguiente sintaxis, también del ejemplo guessnumber-jsf: 

              
<h:graphicImage value="#{resource['images:wave.med.gif']}"/>

Esta etiqueta especifica que la imagen llamada wave.med.gif está en el directorio web/resources/images.

Los recursos se pueden considerar como una ubicación de biblioteca. Cualquier artefacto, como un componente compuesto o una plantilla que se almacena en el directorio resources, se vuelve accesible para los otros componentes de la aplicación, que pueden usarlo para crear una instancia de recurso.
