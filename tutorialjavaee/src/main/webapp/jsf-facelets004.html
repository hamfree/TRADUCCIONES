Usando Plantillas de Facelets

La tecnología JavaServer Faces proporciona las herramientas para implementar interfaces de usuario que son fáciles de ampliar y reutilizar. La creación de plantillas es una característica útil de Facelets que le permite crear una página que actuará como base, o plantilla, para las otras páginas de una aplicación. Al usar plantillas, puede reutilizar el código y evitar volver a crear páginas construidas de manera similar. La creación de plantillas también ayuda a mantener una apariencia estándar en una aplicación con una gran cantidad de páginas.

La Tabla 8-2 enumera las etiquetas de Facelets que se utilizan para crear plantillas y sus respectivas funciones.

Tabla 8-2 Etiquetas de plantillas de facelets

Etiqueta

Función

ui:component

Define un componente que se crea y se agrega al árbol de componentes.

ui:composition

Define una composición de página que opcionalmente usa una plantilla. El contenido fuera de esta etiqueta se ignora.

ui:debug

Define un componente de depuración que se crea y se agrega al árbol de componentes.

ui:decorate

Similar a la etiqueta de composición, pero no ignora el contenido fuera de esta etiqueta.

ui:define

Define el contenido que se inserta en una página mediante una plantilla.

ui:fragment

Similar a la etiqueta de componente, pero no ignora el contenido fuera de esta etiqueta.

ui:include

Encapsula y reutiliza el contenido de varias páginas.

ui:insert

Inserta contenido en una plantilla.

ui:param

Se utiliza para pasar parámetros a un archivo incluido.

ui:repeat

Se utiliza como alternativa a las etiquetas de bucle, comoc:forEach o h:dataTable.

ui:remove

Elimina contenido de una página.

Para obtener más información sobre las etiquetas de plantillas de Facelets, consulte la documentación de la biblioteca de etiquetas de Facelets de JavaServer Faces.

La biblioteca de etiquetas Facelets incluye la etiqueta de plantilla principal ui:insert. Una página de plantilla que se crea con esta etiqueta le permite definir una estructura predeterminada para una página. Una página de plantilla se utiliza como plantilla para otras páginas, generalmente denominadas páginas de cliente.

A continuación, se muestra un ejemplo de una plantilla guardada como template.xhtml: 

              
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:ui="http://xmlns.jcp.org/jsf/facelets"
      xmlns:h="http://xmlns.jcp.org/jsf/html">

    <h:head>
        <meta http-equiv="Content-Type"
              content="text/html; charset=UTF-8" />
        <h:outputStylesheet library="css" name="default.css"/>
        <h:outputStylesheet library="css" name="cssLayout.css"/>
        <title>Facelets Template</title>
    </h:head>

    <h:body>
        <div id="top" class="top">
            <ui:insert name="top">Top Section</ui:insert>
        </div>
        <div>
        <div id="left">
             <ui:insert name="left">Left Section</ui:insert>
        </div>
        <div id="content" class="left_content">
             <ui:insert name="content">Main Content</ui:insert>
        </div>
        </div>
    </h:body>
</html>

La página de ejemplo define una página XHTML que se divide en tres secciones: una sección superior, una sección izquierda y una sección principal. Las secciones tienen hojas de estilo asociadas. La misma estructura se puede reutilizar para las otras páginas de la aplicación.

La página del cliente invoca la plantilla utilizando la etiqueta ui:composition. En el siguiente ejemplo, una página de cliente denominada templateclient.xhtml invoca la página de plantilla denominada template.xhtml del ejemplo anterior. Una página de cliente permite insertar contenido con la ayuda de la etiqueta ui:define. 

              
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:ui="http://xmlns.jcp.org/jsf/facelets"
      xmlns:h="http://xmlns.jcp.org/jsf/html">

    <h:body>
        <ui:composition template="./template.xhtml">
            <ui:define name="top">
                Welcome to Template Client Page
            </ui:define>

            <ui:define name="left">
                <h:outputLabel value="You are in the Left Section"/>
            </ui:define>

            <ui:define name="content">
                <h:graphicImage value="#{resource['images:wave.med.gif']}"/>
                <h:outputText value="You are in the Main Content Section"/>
            </ui:define>
        </ui:composition>
    </h:body>
</html>

Puede utilizar el EID NetBeans para crear plantillas de Facelets y páginas de clientes. Para obtener más información sobre cómo crear estas páginas, consulte https://netbeans.org/kb/docs/web/jsf20-intro.html.
