El Ejemplo billpayment: Usando Eventos e Interceptores

El ejemplo de billpayment muestra cómo usar eventos e interceptores.

Los archivos fuente se encuentran en el directorio tut-install/examples/cdi/pago de facturas/src/main/java/javaeetutorial/pago de facturas/ .

Aquí se tratan los siguientes temas:

  • Descripción General del Ejemplo billpayment

  • La Clase de Evento PaymentEvent

  • El Oyente de Evento PaymentHandler

  • Las Páginas Facelets y Bean Gestionado de billpayment

  • La Clase Interceptadora LoggedInterceptor

  • Ejecutando el Ejemplo billpayment

Descripción General del Ejemplo billpayment

El ejemplo simula pagar una cantidad usando una tarjeta de débito o tarjeta de crédito. Cuando el usuario elige un método de pago, el bean administrado crea un evento apropiado, proporciona su carga útil y lo activa. Un detector de eventos simple maneja el evento usando métodos de observador.

El ejemplo también define un interceptor que se establece en una clase y en dos métodos de otra clase.

La Clase de Evento PaymentEvent

La clase de evento, event.PaymentEvent, es una clase de bean simple que contiene un constructor sin argumentos. También tiene un método toString y métodos getter y setter para los componentes de carga útil: un String para el tipo de pago, un BigDecimal para el monto del pago y una Date para la marca de tiempo. 

                
public class PaymentEvent implements Serializable {
    ...
    public String paymentType;
    public BigDecimal value;
    public Date datetime;
    public PaymentEvent() {
    }
    @Override
    public String toString() {
        return this.paymentType
                + " = $" + this.value.toString()
                + " en " + this.datetime.toString();
    }
    ...

La clase de evento es un bean simple que el bean administrado crea una instancia usando new y luego lo llena. Por esta razón, el contenedor CDI no puede interceptar la creación del bean y, por lo tanto, no puede permitir la interceptación de sus métodos getter y setter.

El Oyente de Evento PaymentHandler

El detector de eventos, listener.PaymentHandler, contiene dos métodos de observación, uno para cada uno de los dos tipos de eventos: 

                
@Logged
@SessionScoped
public class PaymentHandler implements Serializable {
    ...
    public void creditPayment(@Observes @Credit PaymentEvent event) {
        logger.log(Level.INFO, "PaymentHandler - Credit Handler: {0}",
                event.toString());
        // llama a una clase específica de manejador de crédito...
    }
    public void debitPayment(@Observes @Debit PaymentEvent event) {
        logger.log(Level.INFO, "PaymentHandler - Debit Handler: {0}",
                event.toString());
        // llama a una clase de controlador de débito específica...
    }
}

Cada método de observador toma como argumento el evento, anotado con @Observes y con el calificador para el tipo de pago. En una aplicación real, los métodos del observador pasarían la información del evento a otro componente que realizaría la lógica empresarial en el pago.

Los calificadores se definen en el paquete payment, descrito en Las Páginas Facelets y Bean Gestionado de billpayment.

El bean PaymentHandler se anota como @Logged para que todos sus métodos puedan ser interceptados.

Las Páginas Facelets y Bean Gestionado de billpayment

El ejemplo de billpayment contiene dos páginas Facelets, index.xhtml y la muy simple response.xhtml. El cuerpo de index.xhtml se ve así: 

                
    <h:body>
        <h3>Opciones de pago de facturas</h3>
        <p>Ingrese una cantidad, seleccione Tarjeta de débito o Tarjeta de crédito,
            luego haga clic en Pagar.</p>
        <h:form>
            <p>
            <h:outputLabel value="Cantidad: $" for="amt"/>
            <h:inputText id="amt" value="#{paymentBean.value}"
                         required="true"
                         requiredMessage="Se requiere una cantidad."
                         maxlength="15" />
            </p>
            <h:outputLabel value="Opciones:" for="opt"/>
            <h:selectOneRadio id="opt" value="#{paymentBean.paymentOption}">
                <f:selectItem id="debit" itemLabel="Tarjeta de débito"
                              itemValue="1"/>
                <f:selectItem id="credit" itemLabel="Tarjeta de crédito"
                              itemValue="2" />
            </h:selectOneRadio>
            <p><h:commandButton id="submit" value="Pagar"
                                action="#{paymentBean.pay}" /></p>
            <p><h:commandButton value="Restablecer"
                                action="#{paymentBean.reset}" /></p>
        </h:form>
        ...
    </h:body>

El campo de entrada toma un monto de pago, pasado a paymentBean.value. Dos opciones le piden al usuario que seleccione un pago con Tarjeta de Débito o Tarjeta de Crédito, pasando el valor entero a paymentBean.paymentOption. Finalmente, la acción del botón de comando Pagar se establece en el método paymentBean.pay, y la acción del botón Restablecer se establece en el método paymentBean.reset.

El bean administrado payment.PaymentBean utiliza calificadores para diferenciar entre los dos tipos de eventos de pago: 

                
@Named
@SessionScoped
public class PaymentBean implements Serializable {
   ...
    @Inject
    @Credit
    Event<PaymentEvent> creditEvent;
    @Inject
    @Debit
    Event<PaymentEvent> debitEvent;

Los calificadores, @Credit y @Debit, se definen en el paquete payment junto con PaymentBean.

A continuación, el PaymentBean define las propiedades que obtiene de la página Facelets y pasará al evento: 

                
    public static final int DEBIT = 1;
    public static final int CREDIT = 2;
    private int paymentOption = DEBIT;
    @Digits(integer = 10, fraction = 2, message = "Valor no válido")
    private BigDecimal value;
    private Date datetime;

El valor de paymentOption es un número entero pasado desde el componente de opción; el valor predeterminado es DEBIT. El value es un BigDecimal con una restricción de Validación de Beans que impone un valor de moneda con un número máximo de dígitos. La marca de tiempo del evento, datetime, es un objeto Date que se inicializa cuando se llama al método pay.

El método pay del bean primero establece la marca de tiempo para este evento de pago. Luego crea y completa la carga útil del evento, usando el constructor para el PaymentEvent y llamando a los métodos setter del evento, usando las propiedades del bean como argumentos. Luego dispara el evento. 

                
    @Logged
    public String pay() {
        this.setDatetime(Calendar.getInstance().getTime());
        switch (paymentOption) {
            case DEBIT:
                PaymentEvent debitPayload = new PaymentEvent();
                debitPayload.setPaymentType("Debit");
                debitPayload.setValue(value);
                debitPayload.setDatetime(datetime);
                debitEvent.fire(debitPayload);
                break;
            case CREDIT:
                PaymentEvent creditPayload = new PaymentEvent();
                creditPayload.setPaymentType("Credit");
                creditPayload.setValue(value);
                creditPayload.setDatetime(datetime);
                creditEvent.fire(creditPayload);
                break;
            default:
                logger.severe("¡Opción de pago no válida!");
        }
        return "response";
    }

El método pay devuelve la página a la que se redirige la acción, response.xhtml.

La clase PaymentBean también contiene un método reset que vacía el campo de valor en la página index.xhtml y establece la opción de pago por defecto: 

                
    @Logged
    public void reset() {
        setPaymentOption(DEBIT);
        setValue(BigDecimal.ZERO);
    }

En este bean, solo se interceptan los métodos pay y reset.

La página response.xhtml muestra la cantidad pagada. Utiliza una expresión rendered para mostrar el método de pago: 

                
    <h:body>
        <h:form>
            <h2>Pago de Facturas: Resultado</h2>
            <h3>Monto pagado con
                <h:outputText id="debit" value="Tarjeta de débito: "
                              rendered="#{paymentBean.paymentOption eq 1}" />
                <h:outputText id="credit" value="Tarjeta de crédito: "
                              rendered="#{paymentBean.paymentOption eq 2}" />
                <h:outputText id="result" value="#{paymentBean.value}">
                    <f:convertNumber type="currency"/>
                </h:outputText>
            </h3>
            <p><h:commandButton id="back" value="Atrás" action="index" /></p>
        </h:form>
    </h:body>

La Clase Interceptadora LoggedInterceptor

La clase de interceptor, LoggedInterceptor, y su enlace de interceptor, Logged, están ambos definidos en el paquete interceptor. El enlace del interceptor Logged se define de la siguiente manera: 

                
@Inherited
@InterceptorBinding
@Retention(RUNTIME)
@Target({METHOD, TYPE})
public @interface Logged {
}

La clase LoggedInterceptor se ve así:

                
@Logged
@Interceptor
public class LoggedInterceptor implements Serializable {
    ...
    public LoggedInterceptor() {
    }
    @AroundInvoke
    public Object logMethodEntry(InvocationContext invocationContext)
            throws Exception {
        System.out.println("Método de entrada: "
                + invocationContext.getMethod().getName() + " en clase "
                + invocationContext.getMethod().getDeclaringClass().getName());
        return invocationContext.proceed();
    }
}

La clase se anota con las anotaciones @Logged y @Interceptor. El método @AroundInvoke, logMethodEntry, toma el argumento InvocationContext requerido y llama al método proceed requerido. Cuando se intercepta un método, logMethodEntry muestra el nombre del método que se invoca, así como su clase.

Para habilitar el interceptor, el archivo beans.xml lo define de la siguiente manera: 

                
<interceptors>
    <class>javaeetutorial.billpayment.interceptor.LoggedInterceptor</class>
</interceptors>

En esta aplicación, las clases PaymentEvent y PaymentHandler se anotan como @Logged, por lo que se interceptan todos sus métodos. En PaymentBean, solo los métodos pay y reset se anotan @Logged, por lo que solo se interceptan esos métodos.

Ejecutando el Ejemplo billpayment

Puede usar el EID NetBeans o Maven para compilar, empaquetar, implementar y ejecutar la aplicación billpayment.

Aquí se tratan los siguientes temas:

  • Para Construir, Empaquetar, y Desplegar el Ejemplo billpayment Usando el EID NetBeans

  • Para Construir, Empaquetar, y Desplegar el Ejemplo billpayment Usando Maven

  • Para Ejecutar el Ejemplo billpayment

Para Construir, Empaquetar, y Desplegar el Ejemplo billpayment Usando el EID NetBeans

  1. Asegúrese de que GlassFish Server se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, vaya a:

                            
tut-install/examples/cdi

  4. Seleccione la carpeta pago de facturas.

  5. Haga clic en Abrir proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto billpayment y seleccione Build.

    Este comando compila y empaqueta la aplicación en un archivo WAR, billpayment.war, ubicado en el directorio target, y luego lo implementa en el Servidor GlassFish.

Para Construir, Empaquetar, y Desplegar el Ejemplo billpayment Usando Maven

  1. Asegúrese de que GlassFish Server se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/cdi/billpayment/

  3. Ingrese el siguiente comando para implementar la aplicación:

                            
mvn install

    Este comando compila y empaqueta la aplicación en un archivo WAR, billpayment.war, ubicado en el directorio target, y luego lo implementa en el Servidor GlassFish.

Para Ejecutar el Ejemplo billpayment

  1. En un navegador web, ingrese la siguiente URL:

                            
http://localhost:8080/billpayment

  2. En la página Opciones de pago de facturas, ingrese un valor en el campo Cantidad.

    El importe puede contener hasta 10 dígitos e incluir hasta dos decimales. Por ejemplo: 

                            
9876.54

  3. Seleccione Tarjeta de débito o Tarjeta de crédito y haga clic en Pagar.

    Se abre la página Pago de facturas: Resultado, que muestra el monto pagado y el método de pago: 

                            
Importe pagado con tarjeta de crédito: $9,876.34

  4. Haga clic en Atrás para volver a la página Opciones de pago de facturas.

    También puede hacer clic en Restablecer para volver a los valores de la página inicial.

  5. Examine la salida del registro del servidor.

    En el EID NetBeans, la salida es visible en la pestaña Salida del servidor GlassFish. De lo contrario, vea domain-dir`/logs/server.log`.

    La salida de cada interceptor aparece en el registro, seguida de la salida del registrador adicional definida por el constructor y los métodos: 

                            
INFO: Método de entrada: pay en clase billpayment.payment.PaymentBean
INFO: PaymentHandler created.
INFO: Método de entrada: debitPayment en clase billpayment.listener.PaymentHandler
INFO: PaymentHandler - Debit Handler: Debit = $1234.56 en Tue Dec 14 14:50:28 EST 2010
