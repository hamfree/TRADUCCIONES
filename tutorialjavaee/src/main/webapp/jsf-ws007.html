Usando Ajax Con WebSockets

Si desea realizar actualizaciones complejas de la interfaz de usuario según el mensaje push recibido, puede anidar la etiqueta f:ajax dentro de la etiqueta f:websocket. Vea el siguiente ejemplo: 

              
<h:panelGroup id="foo">
... (some complex UI here) ...
</h:panelGroup>
<h:form>
<f:websocket channel="someChannel" scope="view">
<f:ajax event="someEvent"
listener="#{bean.pushed}" render=":foo" />
</f:websocket>
</h:form>

Aquí, el mensaje push simplemente representa el nombre del evento ajax. Puede utilizar cualquier nombre de evento personalizado.

someChannel.send("someEvent");

Una alternativa es combinar la etiqueta f:websocket con la etiqueta h:commandScript. El <f:websocket onmessage> hace referencia exactamente al <h:commandScript name>.

For example:

              
<h:panelGroup id="foo">
... (some complex UI here) ...
</h:panelGroup>
<f:websocket channel="someChannel" scope="view"
onmessage="pushed" />
<h:form>
<h:commandScript name="pushed"
action="#{bean.pushed}" render=":foo" />
</h:form>

Si pasa un Map<String,V> o un JavaBean como objeto de mensaje push, entonces todas las entradas o propiedades estarán disponibles de forma transparente como parámetros de solicitud en el método de script de comando #{bean.pushed}.
