Conceptos Básicos de Concurrencia

La concurrencia es el concepto de ejecutar dos o más tareas al mismo tiempo (en paralelo). Las tareas pueden incluir métodos (funciones), partes de un programa o incluso otros programas. Con las arquitecturas informáticas actuales, la compatibilidad con múltiples núcleos y múltiples procesadores en una sola CPU es muy común.

La Plataforma Java siempre ha ofrecido soporte para la programación concurrente, que fue la base para implementar muchos de los servicios que ofrecen los contenedores Java EE. En Java SE 5, el paquete java.util.concurrent proporcionó soporte adicional de IPA de alto nivel para la concurrencia.

Hilos y Procesos

Los dos conceptos principales de concurrencia son procesos e hilos.

Los procesos se asocian principalmente con aplicaciones que se ejecutan en el sistema operativo (SO). Un proceso tiene recursos de tiempo de ejecución específicos para interactuar con el sistema operativo subyacente y asignar otros recursos, como su propia memoria, tal como lo hace el proceso JVM. Una JVM es de hecho un proceso.

El lenguaje y la plataforma de programación Java se ocupan principalmente de los hilos.

Los subprocesos comparten algunas características con los procesos, ya que ambos consumen recursos del sistema operativo o del entorno de ejecución. Pero los subprocesos son más fáciles de crear y consumen muchos menos recursos que un proceso.

Debido a que los subprocesos son tan livianos, cualquier CPU moderna que tenga un par de núcleos y unos pocos gigabytes de RAM puede manejar miles de subprocesos en un solo proceso de JVM. El número exacto de subprocesos dependerá de la salida combinada de la CPU, el sistema operativo y la RAM disponibles, así como de la configuración (ajuste) correcta de la JVM.

Aunque la programación concurrente resuelve muchos problemas y puede mejorar el rendimiento de la mayoría de las aplicaciones, hay una serie de situaciones en las que múltiples líneas de ejecución (hilos o procesos) pueden causar problemas importantes. Estas situaciones incluyen lo siguiente:

  • Interbloqueos (Deadlocks, en inglés)

  • Inanición de hilos (Thread starvation, en inglés)

  • Acceso simultáneo de recursos compartidos

  • Situaciones en las que el programa genera datos incorrectos
