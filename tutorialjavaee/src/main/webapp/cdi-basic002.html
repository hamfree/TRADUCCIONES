Descripción General de CDI

CDI es un conjunto de servicios que, usados juntos, facilitan a los desarrolladores el uso de beans empresariales junto con la tecnología JavaServer Faces en aplicaciones web. Diseñado para su uso con objetos con estado, CDI también tiene muchos usos más amplios, lo que permite a los desarrolladores una gran flexibilidad para integrar varios tipos de componentes de forma poco acoplada pero segura.

CDI 2.0 está especificado por JSR 365. Las especificaciones relacionadas que usa CDI incluyen lo siguiente:

  • JSR 330, Inyección de Dependencias para Java

  • La especificación Beans Gestionados, una rama de la especificación de la plataforma Java EE (JSR 366)

Los servicios más fundamentales proporcionados por CDI son los siguientes.

  • Contextos: este servicio le permite vincular el ciclo de vida y las interacciones de los componentes con estado a contextos de ciclo de vida bien definidos pero extensibles.

  • Inyección de dependencia: este servicio le permite inyectar componentes en una aplicación de forma segura y elegir en el momento de la implementación qué implementación de una interfaz en particular inyectar.

Además, CDI brinda los siguientes servicios:

  • Integración con Expression Language (EL), que permite que cualquier componente se use directamente dentro de una página JavaServer Faces o una página JavaServer Pages

  • La capacidad de decorar componentes inyectados.

  • La capacidad de asociar interceptores con componentes utilizando enlaces de interceptor con seguridad de tipos

  • Un modelo de notificación de eventos

  • Un ámbito de conversación web además de los tres ámbitos estándar (solicitud, sesión y aplicación) definidos por la especificación Java Servlet

  • Una Interfaz de Proveedor de Servicios (SPI, en inglés) completa que permite que los marcos de terceros se integren limpiamente en el entorno Java EE

Un tema importante de CDI es el acoplamiento relajado. CDI hace lo siguiente:

  • Desacopla el servidor y el cliente mediante tipos y calificadores bien definidos, por lo que la implementación del servidor puede variar.

  • Desacopla los ciclos de vida de los componentes colaboradores al

    • Fabricar los componentes contextuales, con gestión automática del ciclo de vida

    • Permitir que los componentes con estado interactúen como servicios, simplemente mediante el paso de mensajes

  • Desvincula por completo a los productores de mensajes de los consumidores, por medio de eventos.

  • Desacopla los cometidos ortogonales por medio de interceptores Java EE

Junto con el acoplamiento relajado, CDI proporciona tipado fuerte al

  • Eliminar la búsqueda usando nombres basados en cadenas para el cableado y las correlaciones para que el compilador detecte errores de tipeo

  • Permitir el uso de anotaciones declarativas de Java para especificar todo, eliminando en gran medida la necesidad de descriptores de implementación XML y facilitando el suministro de herramientas que introspeccionen el código y comprendan la estructura de dependencia en el momento del desarrollo.
