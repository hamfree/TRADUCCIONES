Uso de Adaptadores de Recursos con Contextos e Inyección de Dependencias para Java EE (CDI)

Para obtener detalles sobre CDI, consulte Capítulo 25, "Introducción a los Contextos e Inyección de Dependencias para Java EE" y Capítulo 27, "Contextos e Inyección de Dependencias para Java EE: Temas Avanzados" .

No especifique las clases siguientes en el adaptador de recursos como beans gestionados por CDI (es decir, no los inyecte), porque el comportamiento de estas clases como beans gestionados por CDI no se ha definido de forma portable.

  • Beans de adaptador de recursos: estos beans son clases que se anotan con la anotación javax.resource.spi.Connector o se declaran como elementos correspondientes en el descriptor de implementación del adaptador de recursos, ra.xml.

  • Beans de fábrica de conexiones gestionadas: estos beans son clases que se anotan con la anotación javax.resource.spi.ConnectorDefinition o la anotación javax.resource.spi.ConnectorDefinitions o se declaran como elementos correspondientes en ra.xml.

  • Beans de especificación de activación: estos beans son clases que se anotan con la anotación javax.resource.spi.Activation o se declaran como elementos correspondientes en ra.xml.

  • Beans de objetos administrados: estos beans son clases que se anotan con la anotación javax.resource.spi.AdministeredObject o se declaran como elementos correspondientes en ra.xml.

Otros tipos de clases en el adaptador de recursos pueden ser beans administrados por CDI y se comportarán de manera portable.
