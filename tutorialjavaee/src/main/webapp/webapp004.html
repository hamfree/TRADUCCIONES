Un Módulo Web Que Usa Tecnología Java Servlet: El Ejemplo hello2

La aplicación hello2 es un módulo web que utiliza la tecnología Java Servlet para mostrar un saludo y una respuesta. Puede usar un editor de texto para ver los archivos de la aplicación, o puede usar NetBeans IDE.

El código fuente de esta aplicación se encuentra en el directorio _tut-install_/examples/web/servlet/hello2/.

Aquí se tratan los siguientes temas:

  • Asociando URLs a Componentes Web

  • Examinando el Módulo Web hello2

  • Ejecutando El Ejemplo hello2

Asociando URLs a Componentes Web

Cuando recibe una solicitud, el contenedor web debe determinar qué componente web debe manejar la solicitud. El contenedor web lo hace asignando la ruta URL contenida en la solicitud a una aplicación web y un componente web. Una ruta de URL contiene la raíz del contexto y, opcionalmente, un patrón de URL: 


http://host:port/context-root[/url-pattern]

Establece el patrón de URL para un servlet usando la anotación @WebServlet en el archivo fuente del servlet. Por ejemplo, el archivo GreetingServlet.java en la aplicación hello2 contiene la siguiente anotación, especificando el patrón de URL como /greeting: 


@WebServlet("/greeting")
public class GreetingServlet extends HttpServlet {
    ...

Esta anotación indica que el patrón de URL /greeting sigue la raíz del contexto. Por lo tanto, cuando el servlet se implementa localmente, se accede a él con la siguiente URL: 


http://localhost:8080/hello2/greeting

Para acceder al servlet utilizando solo la raíz de contexto, especifique "/" como patrón de URL.

Examinando el Módulo Web hello2

La aplicación hello2 se comporta de forma casi idéntica a la aplicación hello1, pero se implementa utilizando la tecnología Java Servlet en lugar de la tecnología JavaServer Faces. Puede usar un editor de texto para ver los archivos de la aplicación, o puede usar el EID de NetBeans.

Para Ver el Módulo Web hello2 Usando el EID NetBeans

Para ver el módulo web hello2 usando el EID de NetBeans:

  1. En el menú Archivo, elija Abrir proyecto.

  2. En el cuadro de diálogo Abrir proyecto, navegue hasta:

                            
tut-install/examples/web/servlet

  3. Seleccione la carpeta hello2 y haga clic en Abrir proyecto.

  4. Expanda el nodo Paquetes de origen, luego expanda el nodo javaeetutorial.hello2.

  5. Haga doble clic en el archivo GreetingServlet.java para verlo.

    Este servlet sobreescribe el método doGet, implementando el método GET de HTTP. El servlet muestra un formulario de saludo HTML simple cuyo botón Enviar, como el de hello1, especifica una página de respuesta para su acción. El siguiente extracto comienza con la anotación @WebServlet, que especifica el patrón de URL relativo a la raíz del contexto: 

                            
@WebServlet("/greeting")
public class GreetingServlet extends HttpServlet {
    @Override
    public void doGet(HttpServletRequest request,
            HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("text/html");
        response.setBufferSize(8192);
        try (PrintWriter out = response.getWriter()) {
            out.println("<html lang=\"en\">"
                    + "<head><title>Servlet Hello</title></head>");
            // luego escribe los datos de la respuesta
            out.println("<body  bgcolor=\"#ffffff\">"
                + "<img src=\"duke.waving.gif\" "
                + "alt=\"Duke waving his hand\">"
                + "<form method=\"get\">"
                + "<h2>Hello, my name is Duke. What's yours?</h2>"
                + "<input title=\"My name is: \"type=\"text\" "
                + "name=\"username\" size=\"25\">"
                + "<p></p>"
                + "<input type=\"submit\" value=\"Submit\">"
                + "<input type=\"reset\" value=\"Reset\">"
                + "</form>");
            String username = request.getParameter("username");
            if (username != null && username.length()> 0) {
                RequestDispatcher dispatcher =
                    getServletContext().getRequestDispatcher("/response");
                if (dispatcher != null) {
                    dispatcher.include(request, response);
                }
            }
            out.println("</body></html>");
        }
    }
    ...

  6. Haga doble clic en el archivo ResponseServlet.java para verlo.

    Este servlet también sobreescribe el método doGet , mostrando solo la respuesta. El siguiente extracto comienza con la anotación @WebServlet, que especifica el patrón de URL relativo a la raíz del contexto: 

                            
@WebServlet("/response")
public class ResponseServlet extends HttpServlet {
    @Override
    public void doGet(HttpServletRequest request,
            HttpServletResponse response)
            throws ServletException, IOException {
        try (PrintWriter out = response.getWriter()) {
            // luego escribe los datos de la respuesta
            String username = request.getParameter("username");
            if (username != null && username.length()> 0) {
                out.println("<h2>Hello, " + username + "!</h2>");
            }
        }
    }
    ...

Ejecutando El Ejemplo hello2

Puede usar NetBeans IDE o Maven para construir, empaquetar, implementar y ejecutar el ejemplo hello2.

Aquí se tratan los siguientes temas:

  • Para Ejecutar el Ejemplo hello2 Usando el EID Netbeans

  • Para Ejecutar el Ejemplo hello2 Usando Maven

Para Ejecutar el Ejemplo hello2 Usando el EID Netbeans

Para ejecutar el ejemplo hello2 usando el EID De NetBeans:

  1. Inicie el Servidor GlassFish como se describe en Para iniciar el servidor GlassFish con NetBeans IDE, si aún no lo ha hecho.

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, navegue hasta:

                            
tut-install/examples/web/servlet

  4. Seleccione la carpeta hello2.

  5. Haga clic en Abrir proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto hello2 y seleccione Compilar para empaquetar e implementar el proyecto.

  7. En un navegador web, abra la siguiente URL:

                            
http://localhost:8080/hello2/greeting

    La URL especifica la raíz del contexto, seguida del patrón de URL.

    La aplicación se parece mucho a la aplicación hello1. La principal diferencia es que después de hacer clic en Enviar, la respuesta aparece debajo del saludo, no en una página separada.

Para Ejecutar el Ejemplo hello2 Usando Maven

Para ejecutar el ejemplo hello2 usando Maven:

  1. Inicie el Servidor GlassFish como se describe en Para iniciar GlassFish Server mediante la línea de comandos , si aún no lo ha hecho.

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/web/servlet/hello2/

  3. Ingrese el siguiente comando:

                            
mvn install

    Este objetivo crea el archivo WAR, lo copia en el _tut-install_/examples/web/hello2/target/ y lo implementa.

  4. En un navegador web, abra la siguiente URL:

                            
http://localhost:8080/hello2/greeting

    La URL especifica la raíz del contexto, seguida del patrón de URL.

    La aplicación se parece mucho a la aplicación hello1. La principal diferencia es que después de hacer clic en Enviar, la respuesta aparece debajo del saludo, no en una página separada.
