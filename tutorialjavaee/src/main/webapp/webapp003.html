Un Módulo Web Que Usa la Tecnología JavaServer Faces: El Ejemplo hello1

La aplicación hello1 es un módulo web que utiliza la tecnología JavaServer Faces para mostrar un saludo y una respuesta. Puede usar un editor de texto para ver los archivos de la aplicación, o puede usar el EID NetBeans.

El código fuente de esta aplicación se encuentra en el directorio tut-install/examples/web/jsf/hello1/.

Aquí se tratan los siguientes temas:

  • Para Ver el Módulo Web hello1 Usando el EID NetBeans

  • Empaquetando y Desplegando el Módulo Web hello1

  • Viendo los Módulos Web Desplegados

  • Ejecutando el Módulo Web Desplegado hello1

  • Replegando el Módulo Web hello1

Para Ver el Módulo Web hello1 Usando el EID NetBeans

Para ver el módulo web hello1 usando el EID NetBeans:

  1. En el menú Archivo, elija Abrir proyecto.

  2. En el cuadro de diálogo Abrir proyecto, navegue hasta:

                          
tut-install/examples/web/jsf

  3. Seleccione la carpeta hello1 y haga clic en Abrir proyecto.

  4. Expanda el nodo Páginas web y haga doble clic en el archivo index.xhtml para verlo en el editor.

    El archivo index.xhtml es la página de destino predeterminada para una aplicación Facelets. En una aplicación típica de Facelets, las páginas web se crean en XHTML. Para esta aplicación, la página utiliza un marcado de etiqueta simple para mostrar un formulario con una imagen gráfica, un encabezado, un campo y dos botones de comando: 

                          
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html lang="es"
      xmlns="http://www.w3.org/1999/xhtml"
      xmlns:h="http://xmlns.jcp.org/jsf/html">
    <h:head>
        <title>Facelets Hello Greeting</title>
    </h:head>
    <h:body>
        <h:form>
            <h:graphicImage url="#{resource['images:duke.waving.gif']}"
                            alt="Duke waving his hand"/>
            <h2>Hello, my name is Duke. What's yours?</h2>
            <h:inputText id="username"
                         title="My name is: "
                         value="#{hello.name}"
                         required="true"
                         requiredMessage="Error: A name is required."
                         maxlength="25" />
            <p></p>
            <h:commandButton id="submit" value="Submit" action="response">
            </h:commandButton>
            <h:commandButton id="reset" value="Reset" type="reset">
            </h:commandButton>
        </h:form>
        ...
    </h:body>
</html>

    El elemento más complejo de la página es el campo inputText. El atributo maxlength especifica la longitud máxima del campo. El atributo required especifica que el campo debe completarse; el atributo requiredMessage proporciona el mensaje de error que se mostrará si el campo se deja vacío. El atributo title proporciona el texto que utilizarán los lectores de pantalla para las personas con discapacidad visual. Finalmente, el atributo value contiene una expresión que será proporcionada por el bean administrado Hello.

    La página web se conecta al bean administrado Hello a través de la expresión de valor de Expression Language (EL) #{hello.name}, que recupera el valor de la propiedad name del bean administrado. Tenga en cuenta el uso de hello para hacer referencia al bean administrado Hello. Si no se especifica ningún nombre en la anotación @Named del bean administrado, siempre se accede al bean administrado con la primera letra del nombre de la clase en minúsculas.

    El elemento Submit commandButton especifica la acción como response, lo que significa que cuando se hace clic en el botón, se muestra la página response.xhtml.

  5. Haga doble clic en el archivo response.xhtml para verlo.

    Aparece la página de respuesta. Incluso más simple que la página de saludo, la página de respuesta contiene una imagen gráfica, un encabezado que muestra la expresión proporcionada por el bean administrado y un solo botón cuyo elemento action lo transfiere de nuevo a la página index.xhtml: 

                          
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html lang="es"
      xmlns="http://www.w3.org/1999/xhtml"
      xmlns:h="http://xmlns.jcp.org/jsf/html">
    <h:head>
        <title>Facelets Hello Response</title>
    </h:head>
    <h:body>
        <h:form>
            <h:graphicImage url="#{resource['images:duke.waving.gif']}"
                            alt="Duke waving his hand"/>
            <h2>Hello, #{hello.name}!</h2>
            <p></p>
            <h:commandButton id="back" value="Back" action="index" />
        </h:form>
    </h:body>
</html>

  6. Expanda el nodo Paquetes Fuente y luego el nodo javaeetutorial.hello1.

  7. Haga doble clic en el archivo Hello.java para verlo.

    La clase Hello, denominada clase bean administrada, proporciona métodos getter y setter para la propiedad name utilizada en las expresiones de la página Facelets. Por defecto, la expresión idioma se refiere al nombre de la clase, con la primera letra en minúscula (hello.name). 

                          
package javaeetutorial.hello1;
import javax.enterprise.context.RequestScoped;
import javax.inject.Named;
@Named
@RequestScoped
public class Hello {
    private String name;
    public Hello() {
    }
    public String getName() {
        return name;
    }
    public void setName(String user_name) {
        this.name = user_name;
    }
}

    Si usa el nombre predeterminado para la clase de bean, puede especificar @Model como la anotación en lugar de tener que especificar tanto @Named y @RequestScoped. La anotación @Model se llama estereotipo, un término para una anotación que encapsula otras anotaciones. Se describe más adelante en Uso de estereotipos en aplicaciones CDI. Algunos ejemplos usarán @Model donde sea apropiado.

  8. En el nodo Páginas web, expanda el nodo WEB-INF y haga doble clic en el archivo web.xml para verlo.

    El archivo web.xml contiene varios elementos necesarios para una aplicación Facelets. Todo lo siguiente se crea automáticamente cuando usa NetBeans IDE para crear una aplicación.

    • Un parámetro de contexto que especifica la etapa del proyecto:

                                  
    <context-param>
        <param-name>javax.faces.PROJECT_STAGE</param-name>
        <param-value>Development</param-value>
    </context-param>

      Un parámetro de contexto proporciona la información de configuración que necesita una aplicación web. Una aplicación puede definir sus propios parámetros de contexto. Además, la tecnología JavaServer Faces y la tecnología Java Servlet definen los parámetros de contexto que puede utilizar una aplicación.

    • Un elemento servlet y su elemento servlet-mapping que especifica el FacesServlet. Todos los archivos con el sufijo .xhtml coincidirán: 

                                  
    <servlet>
        <servlet-name>Faces Servlet</servlet-name>
        <servlet-class>javax.faces.webapp.FacesServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>Faces Servlet</servlet-name>
        <url-pattern>*.xhtml</url-pattern>
    </servlet-mapping>

    • Un elemento welcome-file-list que especifica la ubicación de la página de destino: 

                                      
    <welcome-file-list>
        <welcome-file>index.xhtml</welcome-file>
    </welcome-file-list>

Introducción a los Ámbitos

En la clase Hello.java, las anotaciones javax.inject.Named y javax.enterprise.context.RequestScoped identifican la clase como un bean administrado utilizando el alcance de la solicitud. El alcance define cómo los datos de la aplicación persisten y se comparten.

Los ámbitos más utilizados en las aplicaciones JavaServer Faces son los siguientes:

  • Request (@RequestScoped): el alcance de la solicitud persiste durante una sola solicitud HTTP en una aplicación web. En una aplicación como hello1, en la que la aplicación consta de una única solicitud y respuesta, el bean usa el alcance de la solicitud.

  • Session ( @SessionScoped ): el alcance de la sesión persiste en varias solicitudes HTTP en una aplicación web. Cuando una aplicación consta de múltiples solicitudes y respuestas donde los datos deben mantenerse, los beans usan alcance de sesión.

  • Application (@ApplicationScoped): El alcance de la aplicación persiste en todas las interacciones de los usuarios con una aplicación web.

Para obtener más información sobre los ámbitos en la tecnología JavaServer Faces, consulte Uso de Ámbitos de Bean Administrados.

Empaquetando y Desplegando el Módulo Web hello1

Un módulo web debe estar empaquetado en un WAR en ciertos escenarios de implementación y siempre que desee distribuir el módulo web. Puede empaquetar un módulo web en un archivo WAR usando Maven o usando la herramienta EID de su elección. Este tutorial le muestra cómo usar el EID NetBeans o Maven para construir, empaquetar e implementar la aplicación de muestra hello1.

Puede implementar un archivo WAR en GlassFish Server mediante:

  • el uso del EID NetBeans

  • el uso del comando asadmin

  • el uso de la Consola de Administración

  • Copiar el archivo WAR dentro del directorio domain-dir/autodeploy/

A lo largo del tutorial, utilizará NetBeans IDE o Maven para empaquetar e implementar.

Para Construir y Empaquetar el Módulo Web hello1 Usando el EID NetBeans

Para construir y empaquetar el módulo web hello1 usando el EID NetBeans:

  1. Inicie GlassFish Server como se describe en Para iniciar el servidor GlassFish con el EID NetBeans, si aún no lo ha hecho.

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, navegue hasta:

                            
tut-install/examples/web/jsf

  4. Seleccione la carpeta hello1.

  5. Haga clic en Abrir proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto hello1 y seleccione Compilar. Este comando implementa el proyecto en el servidor.

Para Construir y Empaquetar el Módulo Web hello1 Usando Maven

Para construir y empaquetar el módulo web hello1 web usando Maven:

  1. Inicie GlassFish Server como se describe en Para Iniciar el Servidor GlassFish Server Usando la Línea de Comandos, si aún no lo ha hecho.

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/web/jsf/hello1/

  3. Ingrese el siguiente comando:

                            
mvn install

    Este comando genera las compilaciones necesarias y crea el archivo WAR en tut-install/examples/web/jsf/hello1/target/. Luego implementa el proyecto en el servidor.

Viendo los Módulos Web Desplegados

GlassFish Server proporciona dos formas de ver los módulos web implementados: la Consola de administración y el comando asadmin. También puede usar NetBeans IDE para ver los módulos implementados.

Para Ver los Módulos Web Usando la Consola de Administración

Para ver los módulos web implementados usando la Consola de administración:

  1. Abra la URL http://localhost:4848/ en un navegador.

  2. Seleccione el nodo Aplicaciones.

    Los módulos web implementados aparecen en la tabla Aplicaciones implementadas.

Para Ver los Módulos Web Usando el Comando asadmin

Ingrese el siguiente comando:

                  
asadmin list-applications

Para Ver los Módulos Web Usando el EID NetBeans

Para ver módulos web implementados usando el EID NetBeans:

  1. En la pestaña Servicios, expanda el nodo Servidores, luego expanda el nodo GlassFish Server.

  2. Expanda el nodo Aplicaciones para ver los módulos implementados.

Ejecutando el Módulo Web Desplegado hello1

Ahora que el módulo web está implementado, puede verlo abriendo la aplicación en un navegador web. De forma predeterminada, la aplicación se implementa para alojar localhost en el puerto 8080. La raíz de contexto de la aplicación web es hello1 .

Para ejecutar el m´dulo web desplegado hello1:

  1. Abra un navegador web.

  2. Ingrese la siguiente URL:

                          
http://localhost:8080/hello1/

  3. En el campo, ingrese su nombre y haga clic en Enviar.

    La página de respuesta muestra el nombre que envió. Haga clic en Atrás para volver a intentarlo.

Recarga Dinámica de los Módulos Desplegados

Si la recarga dinámica está habilitada, no es necesario que vuelva a implementar una aplicación o módulo cuando cambie su código o descriptores de implementación. Todo lo que tiene que hacer es copiar las páginas modificadas o los archivos de clase en el directorio de implementación de la aplicación o el módulo. El directorio de implementación para un módulo web llamado context-root es domain-dir/applications/context-root. El servidor comprueba los cambios periódicamente y vuelve a implementar la aplicación, automática y dinámicamente, con los cambios.

Esta capacidad es útil en un entorno de desarrollo porque permite que los cambios de código se prueben rápidamente. Sin embargo, la recarga dinámica no se recomienda para un entorno de producción porque puede degradar el rendimiento. Además, cada vez que se realiza una recarga, las sesiones en ese momento dejan de ser válidas y el cliente debe reiniciar la sesión.

En el Servidor GlassFish, la recarga dinámica está habilitada de forma predeterminada.

Replegando el Módulo Web hello1

Puede anular la implementación de módulos web y otros tipos de aplicaciones empresariales utilizando NetBeans IDE o Maven.

Para Replegar el Módulo Web hello1 Usando el EID NetBeans

Para anular la implementación del módulo web hello1 usando NetBeans IDE:

  1. En la pestaña Servicios, expanda el nodo Servidores, luego expanda el nodo Servidor GlassFish.

  2. Expanda el nodo Aplicaciones.

  3. Haga clic con el botón derecho en el módulo hello1 y seleccione Anular Implementación.

  4. Para eliminar los archivos de clase y otros artefactos de compilación, vuelva a la pestaña Proyectos, haga clic con el botón derecho en el proyecto y seleccione Limpiar.

Para Replegar el Módulo Web hello1 Usando Maven

Para anular la implementación del módulo web hello1 con Maven:

  1. En una ventana de terminal, vaya a:

                            
tut-install/examples/web/jsf/hello1/

  2. Ingrese el siguiente comando:

                            
mvn cargo:undeploy

  3. Para eliminar los archivos de clase y otros artefactos de compilación, ingrese el siguiente comando: 

                            
mvn clean
