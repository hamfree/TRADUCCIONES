Introducción a la Creación de Componentes Personalizados

La tecnología JavaServer Faces ofrece un conjunto básico de componentes de IU estándar y reutilizables que permiten la construcción rápida y sencilla de interfaces de usuario para aplicaciones web. En su mayoría, estos componentes se asignan uno a uno a los elementos en HTML 4. Pero a menudo una aplicación requiere un componente que tenga una funcionalidad adicional o requiera un componente completamente nuevo. La tecnología JavaServer Faces permite la extensión de componentes estándar para mejorar su funcionalidad o para crear componentes personalizados. Un rico ecosistema de bibliotecas de componentes de terceros se basa en esta capacidad de extensión, pero examinarlas está más allá del alcance de este tutorial. Una búsqueda en la web de "Bibliotecas de componentes JSF" es un buen punto de partida para aprender más sobre este importante aspecto del uso de la tecnología JavaServer Faces.

Además de ampliar la funcionalidad de los componentes estándar, un escritor de componentes puede querer darle al autor de una página la capacidad de cambiar la apariencia del componente en la página o alterar el comportamiento del oyente. Alternativamente, el escritor de componentes podría querer renderizar un componente a un tipo diferente de tipo de dispositivo cliente, como un teléfono inteligente o una tableta en lugar de una computadora de escritorio. Habilitado por la arquitectura flexible de JavaServer Faces, un escritor de componentes puede separar la definición del comportamiento del componente de su apariencia al delegar la representación del componente a un renderizador separado. De esta manera, un escritor de componentes puede definir el comportamiento de un componente personalizado una vez, pero crear múltiples renderizadores, cada uno de los cuales define una forma diferente de renderizar el componente en un tipo particular de dispositivo cliente.

Un javax.faces.component.UIComponent es una clase Java que es responsable de representar una parte autónoma de la interfaz de usuario durante el ciclo de vida de procesamiento de solicitudes. Tiene la intención de representar el significado del componente; la representación visual del componente es responsabilidad del javax.faces.render.Renderer. Puede haber varias instancias de la misma clase UIComponent en cualquier vista de JavaServer Faces, al igual que puede haber varias instancias de cualquier clase Java en cualquier programa Java dado.

La tecnología JavaServer Faces proporciona la capacidad de crear componentes personalizados ampliando la clase UIComponent, la clase base para todos los componentes estándar de la interfaz de usuario. Se puede usar un componente personalizado en cualquier lugar donde se pueda usar un componente ordinario, como dentro de un componente compuesto. Un UIComponent se identifica con dos nombres: component-family especifica el propósito general del componente (entrada o salida, por ejemplo) y component-type indica el propósito específico de un componente, como un campo de entrada de texto o un botón de comando.

Un Renderer es un ayudante del UIComponent que se ocupa de cómo debe aparecer esa clase específica de UIComponent en un tipo específico de dispositivo cliente. Los renderizadores se identifican por dos nombres: render-kit-id y render-type. Un kit de renderizado es solo un depósito en el que se coloca un grupo particular de renderizadores, y el render-kit-id identifica el grupo. La mayoría de las bibliotecas de componentes de JavaServer Faces proporcionan sus propios kits de renderizado.

Un objeto javax.faces.view.facelets.Tag es un ayudante para UIComponent y Renderer que permite al autor de la página incluir una instancia de un UIComponent en una vista JavaServer Faces. Una etiqueta representa una combinación específica de component-type y renderer-type.

Vea Combinaciones de Componente, Renderizador, y Etiqueta para obtener información sobre cómo interactúan los componentes, los representadores y las etiquetas.

Este capítulo utiliza el componente de mapa de imagen del ejemplo de estudio de caso de la librería Duke's Bookstore para explicar cómo puede crear componentes personalizados simples, renderizadores personalizados y etiquetas personalizadas asociadas, y ocuparse de todos los demás detalles asociados con el uso de componentes y renderizadores en una aplicación. Consulte el Capítulo 60, "Ejemplo de Estudio de Caso de la Librería de Duke" para obtener más información sobre este ejemplo.

El capítulo también describe cómo crear otros objetos personalizados: convertidores personalizados, detectores personalizados y validadores personalizados. También describe cómo vincular instancias y valores de componentes a objetos de datos y cómo vincular objetos personalizados a propiedades de beans administrados.
