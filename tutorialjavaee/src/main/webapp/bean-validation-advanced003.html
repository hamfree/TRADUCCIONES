Agrupando Restricciones

Las restricciones se pueden agregar a uno o más grupos. Los grupos de restricciones se utilizan para crear subconjuntos de restricciones para que solo se validen ciertas restricciones para un objeto en particular. De forma predeterminada, todas las restricciones se incluyen en el grupo de restricciones Default.

Los grupos de restricciones están representados por interfaces.

              
public interface Employee {}
public interface Contractor {}

Los grupos de restricciones pueden heredar de otros grupos.

              
public interface Manager extends Employee {}

Cuando se agrega una restricción a un elemento, la restricción declara los grupos a los que pertenece esa restricción especificando el nombre de la clase del nombre de la interfaz del grupo en el elemento groups de la restricción. 

              
@NotNull(groups=Employee.class)
Phone workPhone;

Se pueden declarar varios grupos encerrando los grupos con llaves ({ y }) y separando los nombres de clase de los grupos con comas. 

              
@NotNull(groups={ Employee.class, Contractor.class })
Phone workPhone;

Si un grupo hereda de otro grupo, la validación de ese grupo da como resultado la validación de todas las restricciones declaradas como parte del supergrupo. Por ejemplo, validar el grupo Manager da como resultado que se valide el campo workPhone, porque Employee es una superinterfaz de Manager.

Personalizando el Orden Del Grupo de Validación

De forma predeterminada, los grupos de restricciones se validan sin ningún orden en particular. Hay casos en los que unos grupos deben ser validados antes que otros. Por ejemplo, en una clase en particular, los datos básicos deben validarse antes que los datos más avanzados.

Para establecer el orden de validación de un grupo, agregue una anotación javax.validation.GroupSequence a la definición de la interfaz, enumerando el orden en que debe ocurrir la validación. 

                
@GroupSequence({Default.class, ExpensiveValidationGroup.class})
public interface FullValidationGroup {}

Al validar FullValidationGroup, primero se valida el grupo Default. Si todos los datos pasan la validación, se valida el grupo ExpensiveValidationGroup. Si una restricción es parte de los grupos Default y ExpensiveValidationGroup, la restricción se valida como parte del grupo Default y no se validará en el paso subsiguiente de ExpensiveValidationGroup.
