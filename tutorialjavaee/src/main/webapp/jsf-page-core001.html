Usando los Convertidores Estándar

La implementación de JavaServer Faces proporciona un conjunto de implementaciones de Converter que puede utilizar para convertir datos de componentes. El propósito de la conversión es tomar los datos basados en cadenas provenientes de la IPA de Servlet y convertirlos en objetos Java fuertemente tipados adecuados para el dominio empresarial. Para obtener más información sobre los detalles conceptuales del modelo de conversión, consulte Modelo de Conversión .

Las implementaciones estándar de Converter se encuentran en el paquete javax.faces.convert. Normalmente, los convertidores se asignan implícitamente en función del tipo de expresión EL apuntada por el valor del componente. Sin embargo, también se puede acceder a estos convertidores mediante un ID de convertidor. La Tabla 11-1 muestra las clases de convertidores y sus ID de convertidores asociados.

Se asocia un mensaje de error estándar con cada uno de estos convertidores. Si ha registrado uno de estos convertidores en un componente de su página y el convertidor no puede convertir el valor del componente, el mensaje de error del convertidor se mostrará en la página. Por ejemplo, aparece el siguiente mensaje de error si BigIntegerConverter no puede convertir un valor: 


{0} debe ser un número que consta de uno o más dígitos

En este caso, el parámetro de sustitución {0} se reemplazará con el nombre del componente de entrada en el que está registrado el convertidor.

Dos de los convertidores estándar (DateTimeConverter y NumberConverter) tienen sus propias etiquetas, que le permiten configurar el formato de los datos del componente utilizando los atributos de la etiqueta. Para obtener más información sobre el uso de DateTimeConverter, consulte Usando DateTimeConverter. Para obtener más información sobre el uso de NumberConverter, consulte Usando NumberConverter. La siguiente sección explica cómo convertir el valor de un componente, incluido cómo registrar otros convertidores estándar con un componente.

Convirtiendo el Valor de un Componente

Para utilizar un convertidor en particular para convertir el valor de un componente, debe registrar el convertidor en el componente. Puede registrar cualquiera de los convertidores estándar de una de las siguientes formas.

  • Anida una de las etiquetas de convertidor estándar dentro de la etiqueta del componente. Estas etiquetas son f:convertDateTime y f:convertNumber, que se describen en Usando DateTimeConverter y Usando NumberConverter, respectivamente.

  • Vincule el valor del componente a una propiedad de bean gestionado del mismo tipo que el convertidor. Ésta es la técnica más común.

  • Consulte el convertidor desde el atributo converter de la etiqueta de componente, especificando el ID de la clase de convertidor.

  • Inserte una etiqueta f:converter dentro de la etiqueta del componente y utilice el atributo converterId de la etiqueta f:converter o su atributo binding para hacer referencia al convertidor.

Como ejemplo de la segunda técnica, si desea que los datos de un componente se conviertan en un Integer, simplemente puede vincular el valor del componente a una propiedad de bean gestionado. Aquí hay un ejemplo: 


Integer age = 0;
public Integer getAge(){ return age;}
public void setAge(Integer age) {this.age = age;}

Los datos de la etiqueta h:inputText en este ejemplo se convertirán a un valor java.lang.Integer. El tipo Integer es un tipo admitido de NumberConverter. Si no necesita especificar ninguna instrucción de formato usando los atributos de etiqueta f:convertNumber, y si uno de los convertidores estándar es suficiente, simplemente puede hacer referencia a ese convertidor usando el atributo converter de la etiqueta del componente.

También puede anidar una etiqueta f:converter dentro de la etiqueta del componente y usar el atributo converterId de la etiqueta del convertidor o su atributo binding para hacer referencia al convertidor.

El atributo converterId debe hacer referencia al ID del convertidor. A continuación, se muestra un ejemplo que utiliza uno de los ID de convertidor que se enumeran en la Tabla 11-1: 


<h:inputText value="#{loginBean.age}">
    <f:converter converterId="javax.faces.Integer" />
</h:inputText>

En lugar de usar el atributo converterId, la etiqueta f:converter puede usar el atributo binding. El atributo binding debe resolverse en una propiedad de bean que acepte y devuelva una instancia de Converter adecuada.

También puede crear convertidores personalizados y registrarlos en componentes usando la etiqueta f:converter. Para obtener más información, consulte Creando y Usando un Convertidor Personalizado.

Usando DateTimeConverter

Puede convertir los datos de un componente en un java.util.Date anidando la etiqueta convertDateTime dentro de la etiqueta del componente. La etiqueta convertDateTime tiene varios atributos que le permiten especificar el formato y el tipo de datos. La Tabla 11-2 enumera los atributos.

Aquí hay un ejemplo simple de una etiqueta convertDateTime:


<h:outputText value="#{cashierBean.shipDate}">
    <f:convertDateTime type="date" dateStyle="full" />
</h:outputText>

Al vincular DateTimeConverter a un componente, asegúrese de que la propiedad del bean gestionado a la que está vinculado el componente sea del tipo java.util.Date. En el ejemplo anterior, cashierBean.shipDate debe ser del tipo java.util.Date.

La etiqueta de ejemplo puede mostrar la siguiente salida:

Saturday, September 21, 2013

También puede mostrar la misma fecha y hora utilizando la siguiente etiqueta en la que se especifica el formato de fecha: 


<h:outputText value="#{cashierBean.shipDate}">
    <f:convertDateTime pattern="EEEEEEEE, MMM dd, yyyy" />
</h:outputText>

Si desea mostrar la fecha de ejemplo en español, puede usar el atributo locale: 


<h:outputText value="#{cashierBean.shipDate}">
    <f:convertDateTime dateStyle="full"
                       locale="es"
                       timeStyle="long" type="both" />
</h:outputText>

Esta etiqueta mostraría el siguiente resultado:


jueves 24 de octubre de 2013 15:07:04 GMT

Consulte la lección "Personalización de formatos" del Tutorial de Java en http://docs.oracle.com/javase/tutorial/i18n/format/simpleDateFormat.html para obtener más información sobre cómo formatear la salida utilizando el atributo pattern de la etiqueta convertDateTime.

Usando NumberConverter

Puede convertir los datos de un componente en un java.lang.Number anidando la etiqueta convertNumber dentro de la etiqueta del componente. La etiqueta convertNumber tiene varios atributos que le permiten especificar el formato y el tipo de datos. La Tabla 11-4 enumera los atributos.

El siguiente ejemplo usa una etiqueta convertNumber para mostrar los precios totales del contenido de un carrito de compras: 


<h:outputText value="#{cart.total}">
    <f:convertNumber currencySymbol="$" type="currency"/>
</h:outputText>

Al vincular NumberConverter a un componente, asegúrese de que la propiedad del bean administrado a la que está vinculado el componente sea de un tipo primitivo o tenga un tipo de java.lang.Number. En el ejemplo anterior, cart.total es de tipo double.

A continuación, se muestra un ejemplo de un número que puede mostrar esta etiqueta:


$934

Este resultado también se puede mostrar utilizando la siguiente etiqueta en la que se especifica el patrón de moneda: 


<h:outputText id="cartTotal" value="#{cart.total}">
    <f:convertNumber pattern="$####" />
</h:outputText>

Consulte la lección "Personalización de formatos" del tutorial de Java en http: //docs.oracle.com/javase/tutorial/i18n/format/decimalFormat.html para obtener más información sobre cómo dar formato a la salida mediante el atributo pattern de la etiqueta convertNumber.
