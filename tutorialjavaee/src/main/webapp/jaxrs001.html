¿Qué Son los Servicios Web RESTful?

Los servicios web RESTful son servicios web livianos y poco acoplados que son particularmente adecuados para crear API para clientes repartidos por Internet. La transferencia de estado representacional (REST) es un estilo arquitectónico de aplicación cliente-servidor centrado en la transferencia de representaciones de recursos a través de solicitudes y respuestas. En el estilo arquitectónico REST, los datos y la funcionalidad se consideran recursos y se accede a ellos mediante identificadores uniformes de recursos (URI), normalmente enlaces en la web. Los recursos están representados por documentos y se actúa sobre ellos mediante un conjunto de operaciones simples y bien definidas.

Por ejemplo, un recurso REST podría ser las condiciones meteorológicas actuales de una ciudad. La representación de ese recurso puede ser un documento XML, un archivo de imagen o una página HTML. Un cliente puede recuperar una representación particular, modificar el recurso actualizando sus datos o eliminar el recurso por completo.

El estilo arquitectónico REST está diseñado para usar un protocolo de comunicación sin estado, normalmente HTTP. En el estilo de arquitectura REST, los clientes y servidores intercambian representaciones de recursos mediante el uso de una interfaz y un protocolo estandarizados.

Los siguientes principios fomentan que las aplicaciones RESTful sean simples, livianas y rápidas:

  • Identificación de recursos a través de URI: un servicio web RESTful expone un conjunto de recursos que identifican los objetivos de la interacción con sus clientes. Los recursos se identifican mediante URI, que proporcionan un espacio de direccionamiento global para el descubrimiento de recursos y servicios. Consulte La Anotación @Path y las Plantillas de Ruta URI para obtener más información.

  • Interfaz uniforme: los recursos se manipulan mediante un conjunto fijo de cuatro operaciones de creación, lectura, actualización y eliminación: PUT, GET, POST y DELETE. PUT crea un nuevo recurso, que luego se puede eliminar usando DELETE. GET recupera el estado actual de un recurso en alguna representación. POST transfiere un nuevo estado a un recurso. Consulte Respondiendo a los Métodos HTTP y Peticiones para obtener más información.

  • Mensajes autodescriptivos: los recursos se desacoplan de su representación para que se pueda acceder a su contenido en una variedad de formatos, como HTML, XML, texto sin formato, PDF, JPEG, JSON y otros formatos de documentos. Los metadatos sobre el recurso están disponibles y se utilizan, por ejemplo, para controlar el almacenamiento en caché, detectar errores de transmisión, negociar el formato de representación apropiado y realizar autenticación o control de acceso. Ver Respondiendo a los Métodos HTTP y Peticiones y Usando Proveedores de Entidad para Asociar Respuestas HTTP y Cuerpos de Entidades de Petición para más información.

  • Interacciones con estado a través de enlaces: cada interacción con un recurso no tiene estado; es decir, los mensajes de solicitud son autónomos. Las interacciones con estado se basan en el concepto de transferencia de estado explícita. Existen varias técnicas para intercambiar estados, como la reescritura de URI, las cookies y los campos de formulario ocultos. El estado se puede incrustar en los mensajes de respuesta para señalar estados futuros válidos de la interacción. Ver Usando Proveedores de Entidad para Asociar Respuestas HTTP y Cuerpos de Entidades de Petición y Extrayendo Parámetros de Petición en el documento de descripción general de JAX-RS para obtener más información.
