Operaciones Sobre Objetos de Colección

El EL admite operaciones sobre objetos de colección: conjuntos, listas y mapas. Permite la creación dinámica de objetos de colección, que luego se pueden operar mediante flujos y canalizaciones.

Note:

Al igual que las expresiones lambda, las operaciones en los objetos de colección son parte de Java SE 8.

Por ejemplo, puede construir un conjunto de la siguiente manera:

              
{1,2,3}

Puede construir una lista de la siguiente manera; una lista puede contener varios tipos de elementos: 

              
[1,2,3]
[1, "two", [three,four]]

Puede construir un mapa usando dos puntos para definir las entradas, de la siguiente manera:

              
{"one":1, "two":2, "three":3}

Usted opera en objetos de colección usando llamadas a métodos al flujo de elementos derivados de la colección. Algunas operaciones devuelven otro flujo, lo que permite operaciones adicionales. Por lo tanto, puede encadenar estas operaciones en una canalización.

Una canalización de flujo consta de lo siguiente:

  • Una fuente(el objeto Stream)

  • Cualquier cantidad de operaciones intermedias que devuelvan una secuencia (por ejemplo, filter y map)

  • Una operación de terminal que no devuelve una secuencia (por ejemplo, toList())

El método stream obtiene un Stream de un java.util.Collection o una matriz de Java. Las operaciones de flujo no modifican el objeto de colección original.

Por ejemplo, puede generar una lista de títulos de libros de historia de la siguiente manera:

              
books.stream().filter(b->b.category == 'history')
              .map(b->b.title)
              .toList()

El siguiente ejemplo más simple devuelve una versión ordenada de la lista original:

              
[1,3,5,2].stream().sorted().toList()

Las secuencias y las operaciones de secuencias están documentadas en la documentación de la API de Java SE 8, disponible en http://docs.oracle.com/javase/8/docs/api/ . El siguiente subconjunto de operaciones es compatible con EL:

allMatch
anyMatch
average
count
distinct
filter
findFirst
flatMap
forEach
iterator
limit
map
max
min
noneMatch
peek
reduce
sorted
substream
sum
toArray
toList

Consulte la especificación EL en http://www.jcp.org/en/jsr/detail?id=341 para obtener detalles sobre estas operaciones.
