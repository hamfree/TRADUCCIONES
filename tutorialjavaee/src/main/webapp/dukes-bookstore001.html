Diseño y Arquitectura de la Librería de Duke

La Librería de Duke es una aplicación web sencilla que utiliza muchas características de la tecnología JavaServer Faces, además de otras características de Java EE:

  • Tecnología JavaServer Faces, así como Contextos e Inyección de Dependencia para Java EE (CDI)

    • Un conjunto de páginas Facelets, junto con una plantilla, proporciona la interfaz de usuario de la aplicación.

    • Los beans administrados por CDI están asociados con cada una de las páginas de Facelets.

    • Un componente de mapa de imagen personalizado en la página principal le permite seleccionar un libro para ingresar a la tienda. Cada área del mapa está representada por un bean gestionado JavaServer Faces. También se proporcionan hipervínculos de texto para facilitar la accesibilidad.

    • Los detectores de acciones se registran en el mapa de imágenes y los enlaces de texto. Estos oyentes recuperan el valor de ID para el libro seleccionado y lo almacenan en el mapa de la sesión para que el bean administrado pueda recuperarlo para la página siguiente.

    • La etiqueta h:dataTable se usa para representar dinámicamente el catálogo de libros y el contenido del carrito de compras.

    • Se registra un convertidor personalizado en el campo de la tarjeta de crédito en la página de pago, bookcashier.xhtml, que también usa una etiqueta f:validateRegEx para garantizar que la entrada tenga el formato correcto.

    • Un listener de cambio de valor se registra en el campo de nombre en bookcashier.xhtml. Este listener guarda el nombre en un parámetro para que la página siguiente, bookreceipt.xhtml, pueda acceder a él.

  • Enterprise beans: bean de sesión sin estado local, sin vista de interfaz y bean singleton

  • Una entidad de la IPA de Persistencia de Java

Los paquetes de la aplicación Librería de Duke, ubicados en el directorio tut-install`/examples/case-studies/dukes-bookstore/src/main/java/javaeetutorial/dukesbookstore/`, son los siguientes:

  • components: incluye las clases de componentes de interfaz de usuario personalizados, MapComponent y AreaComponent

  • converters: incluye la clase de convertidor personalizado, CreditCardConverter

  • ejb: incluye dos enterprise beans:

    • Un bean singleton, ConfigBean, que inicializa los datos en la base de datos

    • Un bean de sesión sin estado, BookRequestBean, que contiene la lógica de negocio para administrar la entidad

  • entity: incluye la clase de entidad Book

  • exceptions: incluye tres clases de excepción

  • listeners: incluye las clases de controlador de eventos y detectores de eventos

  • model: incluye una clase modelo de JavaBeans

  • renderers: incluye los renderizadores personalizados para las clases de componentes de interfaz de usuario personalizados.

  • web.managedbeans: incluye los beans administrados para las páginas de Facelets

  • web.messages: incluye los archivos del paquete de recursos para mensajes localizados
