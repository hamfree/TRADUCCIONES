Creando Aplicaciones WebSocket en la Plataforma Java EE

La plataforma Java EE incluye el IPA de Java para WebSocket (JSR 356), que le permite crear, configurar e implementar puntos finales WebSocket en aplicaciones web. La IPA de cliente de WebSocket especificada en JSR 356 también le permite acceder a puntos finales de WebSocket remotos desde cualquier aplicación Java.

El IPA de Java para WebSocket consta de los siguientes paquetes.

  • El paquete javax.websocket.server contiene anotaciones, clases e interfaces para crear y configurar puntos finales de servidor.

  • El paquete javax.websocket contiene anotaciones, clases, interfaces y excepciones que son comunes a los extremos del cliente y del servidor.

Los extremos de WebSocket son instancias de la clase javax.websocket.Endpoint. El IPA de Java para WebSocket le permite crear dos tipos de puntos finales: puntos finales programáticos y puntos finales anotados. Para crear un punto final programático, extiende la clase Endpoint y anula sus métodos de ciclo de vida. Para crear un punto final anotado, decora una clase Java y algunos de sus métodos con las anotaciones proporcionadas por los paquetes mencionados anteriormente. Una vez que haya creado un punto final, lo implementa en un URI específico en la aplicación para que los clientes remotos puedan conectarse a él.

Noae:

En la mayoría de los casos, es más fácil crear e implementar un punto final anotado que un punto final programático. Este capítulo proporciona un ejemplo simple de un punto final programático, pero se centra en los puntos finales anotados.

Creando y Desplegando un WebSocket Endpoint

El proceso para crear e implementar un punto final de WebSocket:

  1. Cree una clase de punto final.

  2. Implementar los métodos de ciclo de vida del endpoint.

  3. Agregue su lógica de negocios al punto final.

  4. Implemente el punto final dentro de una aplicación web.

El proceso es ligeramente diferente para puntos finales programáticos y puntos finales anotados, y se trata en detalle en las siguientes secciones.

Nota:

A diferencia de los servlets, los puntos finales de WebSocket se instancian varias veces. El contenedor crea una instancia de un punto final por conexión a su URI de implementación. Cada instancia está asociada a una y sólo una conexión. Esto facilita el mantenimiento del estado del usuario para cada conexión y facilita el desarrollo, porque solo hay un subproceso que ejecuta el código de una instancia de punto final en un momento dado.
