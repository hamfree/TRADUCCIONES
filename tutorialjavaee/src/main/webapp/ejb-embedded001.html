Descripción del Contenedor de Enterprise Beans Integrado

El contenedor de enterprise bean integrado se utiliza para acceder a los componentes de enterprise bean desde el código del cliente ejecutado en un entorno Java SE. El contenedor y el código del cliente se ejecutan dentro de la misma máquina virtual. El contenedor de enterprise beans integrados se suele utilizar para probar enterprise beans sin tener que implementarlos en un servidor.

La mayoría de los servicios presentes en el contenedor de enterprise bean en un servidor Java EE están disponibles en el contenedor de enterprise bean integrado, incluida la inyección, las transacciones gestionadas por contenedor y la seguridad. Los componentes de Enterprise Bean se ejecutan de manera similar tanto en entornos integrados como Java EE y, por lo tanto, el mismo Enterprise Bean se puede reutilizar fácilmente en aplicaciones independientes y en red.
