Endpoints Programáticos

El siguiente ejemplo muestra cómo crear un punto final extendiendo el Endpoint clase: 


public class EchoEndpoint extends Endpoint {
   @Override
   public void onOpen(final Session session, EndpointConfig config) {
      session.addMessageHandler(new MessageHandler.Whole<String>() {
         @Override
         public void onMessage(String msg) {
            try {
               session.getBasicRemote().sendText(msg);
            } catch (IOException e) { ... }
         }
      });
   }
}

Este punto final repite cada mensaje recibido. La clase Endpoint define tres métodos de ciclo de vida: onOpen, onClose y onError. La clase EchoEndpoint implementa el método onOpen, que es el único método abstracto en la clase Endpoint.

El parámetro Session representa una conversación entre este extremo y el extremo remoto. El método addMessageHandler registra los controladores de mensajes y el método getBasicRemote devuelve un objeto que representa el extremo remoto. La interfaz Session se cubre en detalle en el resto de este capítulo.

El controlador de mensajes se implementa como una clase interna anónima. El método onMessage del controlador de mensajes se invoca cuando el punto final recibe un mensaje de texto.

Para implementar este punto final programático, use el siguiente código en su aplicación Java EE:


ServerEndpointConfig.Builder.create(EchoEndpoint.class, "/echo").build();

Cuando implementa su aplicación, el punto final está disponible en ws://<host>:<port>/<application>/echo; por ejemplo, ws://localhost:8080/echoapp/echo.
