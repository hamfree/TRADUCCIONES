Validando Cadenas Vacías y Nulo

El lenguaje de programación Java distingue entre cadenas nulas y vacías. Una cadena vacía es una instancia de cadena de longitud cero, mientras que una cadena nula no tiene ningún valor.

Una cadena vacía se representa como "". Es una secuencia de caracteres de cero caracteres. Una cadena nula está representada por null. Se puede describir como la ausencia de una instancia de cadena.

Los elementos de bean gestionados representados como un componente de texto de JavaServer Faces, como inputText, se inicializan con el valor de la cadena vacía mediante la implementación de JavaServer Faces. La validación de estas cadenas puede ser un problema cuando no se requiere la entrada del usuario para dichos campos. Considere el siguiente ejemplo, en el que la cadena testString es una variable de bean que se establecerá utilizando la entrada ingresada por el usuario. En este caso, no se requiere la entrada del usuario para el campo. 

              
if (testString==null) {
    doSomething();
} else {
    doAnotherThing();
}

De forma predeterminada, se llama al método doAnotherThing incluso cuando el usuario no ingresa datos, porque el elemento testString se ha inicializado con el valor de una cadena vacía.

Para que el modelo de Validación de Beans funcione según lo previsto, debe establecer el parámetro de contexto javax.faces.INTERPRET_EMPTY_STRING_SUBMITTED_VALUES_AS_NULL en true en el archivo descriptor de implementación web, web.xml: 

              
<context-param>
    <param-name>
        javax.faces.INTERPRET_EMPTY_STRING_SUBMITTED_VALUES_AS_NULL
    </param-name>
    <param-value>true</param-value>
</context-param>

Este parámetro permite que la implementación de JavaServer Faces trate las cadenas vacías como nulas.

Suponga, por otro lado, que tiene una restricción @NotNull en un elemento, lo que significa que se requiere una entrada. En este caso, una cadena vacía pasará esta restricción de validación. Sin embargo, si establece el parámetro de contexto javax.faces.INTERPRET_EMPTY_STRING_SUBMITTED_VALUES_AS_NULL en true, el valor del atributo de bean administrado se pasa al tiempo de ejecución de Bean Validation como un valor nulo, lo que provoca la restricción @NotNull falle.
