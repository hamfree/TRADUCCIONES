Marcado compatible con HTML5

Cuando desee producir funciones de interfaz de usuario para las que HTML no tiene sus propios elementos, puede crear un componente JavaServer Faces personalizado e insertarlo en su página de Facelets. Este mecanismo puede hacer que un elemento simple cree un código web complejo. Sin embargo, crear un componente de este tipo es una tarea importante (consulte Capítulo 15, "Creación de componentes de interfaz de usuario personalizados y otros objetos personalizados" ).

HTML5 ofrece nuevos elementos y atributos que pueden hacer innecesario escribir sus propios componentes. También proporciona muchas capacidades nuevas para los componentes existentes. La tecnología JavaServer Faces admite HTML5 no al introducir nuevos componentes de interfaz de usuario que imitan a los de HTML5, sino al permitirle utilizar el marcado HTML5 directamente. También le permite utilizar atributos de JavaServer Faces dentro de elementos HTML5. El soporte de la tecnología JavaServer Faces para HTML5 se divide en dos categorías:

  • Elementos de paso

  • Atributos de transferencia

El efecto de la función de marcado compatible con HTML5 es ofrecer al autor de la página Facelets un control casi completo sobre la salida de la página renderizada, en lugar de tener que pasar este control a los autores de los componentes. Puede mezclar y combinar JavaServer Faces y componentes y elementos HTML5 como mejor le parezca.

Uso de Elementos de Paso a Través

Los elementos de paso le permiten usar etiquetas y atributos HTML5, pero tratarlos como equivalentes a los componentes JavaServer Faces asociados con una instancia UIComponent del lado del servidor.

Para convertir un elemento que no es un elemento JavaServer Faces en un elemento de paso, especifique al menos uno de sus atributos utilizando el espacio de nombres http://xmlns.jcp.org/jsf. Por ejemplo, el siguiente código declara el espacio de nombres con el nombre corto jsf: 

                
<html ... xmlns:jsf="http://xmlns.jcp.org/jsf"
...
    <input type="email" jsf:id="email" name="email"
           value="#{reservationBean.email}" required="required"/>

Aquí, el prefijo jsf se coloca en el atributo id para que los atributos de la etiqueta de entrada HTML5 se traten como parte de la página Facelets. Esto significa que, por ejemplo, puede utilizar expresiones EL para recuperar propiedades de bean gestionadas.

La Tabla 8-4 muestra cómo los elementos de paso a través se representan como etiquetas Facelets. La implementación de JSF usa el nombre del elemento y el atributo de identificación para determinar la etiqueta Facelets correspondiente que se usará en el procesamiento del lado del servidor. Sin embargo, el navegador interpreta el marcado que ha escrito el autor de la página.

Uso de Atributos de Transferencia

Los atributos de paso son opuestos a los elementos de paso. Le permiten pasar atributos que no son atributos de JavaServer Faces al navegador sin interpretación. Si especifica un atributo de paso a través en un JavaServer Faces UIComponent, el nombre y el valor del atributo se pasan directamente al navegador sin ser interpretados por los componentes o representadores de JavaServer Faces. Hay varias formas de especificar atributos de paso a través.

  • Utilice el espacio de nombres JavaServer Faces para los atributos de paso a través para prefijar los nombres de los atributos dentro de un componente JavaServer Faces. Por ejemplo, el siguiente código declara el espacio de nombres con el nombre corto p, luego pasa los atributos type, min, max , required y title hasta el componente HTML5 input: 

                          
<html ... xmlns:p="http://xmlns.jcp.org/jsf/passthrough"
...

<h:form prependId="false">
<h:inputText id="nights" p:type="number" value="#{bean.nights}"
             p:min="1" p:max="30" p:required="required"
             p:title="Enter a number between 1 and 30 inclusive.">
        ...

    Esto hará que se represente el siguiente marcado (asumiendo que bean.nights tiene un valor predeterminado establecido en 1): 

                          
<input id="nights" type="number" value="1" min="1" max="30"
       required="required"
       title="Enter a number between 1 and 30 inclusive.">

  • Para pasar un solo atributo, anide la etiqueta f:passThroughAttribute dentro de una etiqueta de componente. Por ejemplo: 

                          
<h:inputText value="#{user.email}">
    <f:passThroughAttribute name="type" value="email" />
</h:inputText>

    Este código se representaría de manera similar a lo siguiente:

                          
<input value="me@me.com" type="email" />

  • Para pasar un grupo de atributos, anide la etiqueta f:passThroughAttributes dentro de una etiqueta de componente, especificando un valor EL que debe evaluarse como un Map<String,Object>. Por ejemplo: 

                          
<h:inputText value="#{bean.nights">
    <f:passThroughAttributes value="#{bean.nameValuePairs}" />
</h:inputText>

    Si el bean usó la siguiente declaración Map e inicializó el mapa en el constructor de la siguiente manera, el marcado sería similar a la salida del código que usa el espacio de nombres del atributo de paso a través: 

                          
private Map<String, Object> nameValuePairs;
...
public Bean() {
    this.nameValuePairs = new HashMap<>();
    this.nameValuePairs.put("type", "number");
    this.nameValuePairs.put("min", "1");
    this.nameValuePairs.put("max", "30");
    this.nameValuePairs.put("required", "required");
    this.nameValuePairs.put("title",
            "Enter a number between 1 and 4 inclusive.");
}

La Aplicación de Ejemplo reservation

La aplicación de ejemplo reservation proporciona un conjunto de elementos HTML5 input de varios tipos para simular la compra de entradas para un evento teatral. Consiste en dos páginas Facelets, booking.xhtml y confirm.xhtml, y un bean de respaldo, ReservationBean.java. Las páginas utilizan tanto atributos de paso a través como elementos de paso.

El código fuente de esta aplicación se encuentra en el directorio tut-install`/examples/web/jsf/reservations/`.

Las Páginas Facelets para la Aplicación reservation

La primera característica importante de las páginas Facelets para la aplicación reservation es el encabezado DOCTYPE. La mayoría de las páginas de Facelets en las aplicaciones JavaServer Faces se refieren a XHTML DTD. Las páginas de facelets para esta aplicación comienzan simplemente con el siguiente encabezado DOCTYPE, que indica una página HTML5: 

                  
<!DOCTYPE html>

Las declaraciones de espacio de nombres en el elemento html de la página reservation.xhtml especifican los espacios de nombres jsf y passthrough: 

                  
<html lang="es"
      xmlns="http://www.w3.org/1999/xhtml"
      xmlns:f="http://xmlns.jcp.org/jsf/core"
      xmlns:h="http://xmlns.jcp.org/jsf/html"
      xmlns:p="http://xmlns.jcp.org/jsf/passthrough"
      xmlns:jsf="http://xmlns.jcp.org/jsf">

A continuación, una etiqueta h:head vacía seguida de una etiqueta h:outputStylesheet dentro de la etiqueta h:body ilustra el uso de un recurso reubicable (como se describe en Recursos reubicables): 

                  
<h:head>
</h:head>
<h:body>
    <h:outputStylesheet name="css/stylesheet.css" target="head"/>

La página reservation.xhtml utiliza elementos de paso a través para la mayoría de los campos de formulario de la página. Esto le permite utilizar algunos tipos de elementos de input específicos de HTML5, como date y email. Por ejemplo, el siguiente elemento representa tanto un formato de fecha como un calendario entre los que puede elegir una fecha. El prefijo jsf en el atributo id hace que el elemento sea de paso directo: 

                  
    <input type="date" jsf:id="date" name="date"
           value="#{reservationBean.date}" required="required"
           title="Enter or choose a date."/>

Sin embargo, el campo para el número de tickets usa la etiqueta h:passThroughAttributes para pasar un Map definido en el bean administrado. También recalcula el total en respuesta a un cambio en el campo: 

                  
    <h:inputText id="tickets" value="#{reservationBean.tickets}">
        <f:passThroughAttributes value="#{reservationBean.ticketAttrs}"/>
        <f:ajax event="change" render="total"
                listener="#{reservationBean.calculateTotal}"/>
    </h:inputText>

El campo para el precio especifica el tipo number como un atributo de transferencia del elemento h:inputText, que ofrece un rango de cuatro precios de entradas. Aquí, el prefijo p en los atributos HTML5 los pasa al navegador sin que el componente de entrada JavaServer Faces los interprete: 

                      
    <h:inputText id="price" p:type="number"
                 value="#{reservationBean.price}"
                 p:min="80" p:max="120"
                 p:step="20" p:required="required"
                 p:title="Enter a price: 80, 100, 120, or 140.">
        <f:ajax event="change" render="total"
                listener="#{reservationBean.calculateTotal}"/>
    </h:inputText>

La salida del método calculateTotal que se especifica como el oyente para el evento Ajax se representa en el elemento de salida cuyo valor de id y name es total. Consulte el Capítulo 13, "Uso de Ajax con Tecnología JavaServer Faces", para obtener más información.

La segunda página de Facelets, confirmation.xhtml, utiliza un elemento output de paso a través para mostrar los valores ingresados por el usuario y proporciona una etiqueta h:commandButton de Facelets para permitir que el usuario regrese a la página reservation.xhtml.

El Bean Gestionado para la Aplicación reservation

El bean administrado de ámbito de sesión para la aplicación de reserva, ReservationBean.java, contiene propiedades para todos los elementos en las páginas de Facelets. También contiene dos métodos, calculateTotal y clear, que actúan como oyentes de eventos Ajax en la página reservation.xhtml.

Para Construir, Empaquetar, y Desplegar el Ejemplo reservation Usando el EID NetBeans

  1. Asegúrese de que el Servidor GlassFish se haya iniciado (consulte Iniciar y detener el Servidor GlassFish).

  2. En el menú Archivo, elija Abrir proyecto.

  3. En el cuadro de diálogo Abrir proyecto, navegue hasta:

                            
tut-install/examples/web/jsf

  4. Seleccione la carpeta reservation.

  5. Haga clic en Abrir proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto reservation y seleccione Compilar.

    Esta opción crea la aplicación de ejemplo y la implementa en su instancia del Servidor GlassFish.

Para Construir, Empaquetar, y Desplegar el Ejemplo reservation Usando Maven

  1. Asegúrese de que el Servidor GlassFish se haya iniciado (consulte Iniciar y detener el Servidor GlassFish).

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/web/jsf/reservation/

  3. Ingrese el siguiente comando:

                            
mvn install

    Este comando compila y empaqueta la aplicación en un archivo WAR, reservation.war, que se encuentra en el directorio target. Luego implementa el archivo WAR en su instancia del Servidor GlassFish.

Para Ejecutar el Ejemplo reservation

En el momento de la publicación de este tutorial, el navegador que más implementa HTML5 es Google Chrome, y se recomienda que lo use para ejecutar este ejemplo. Sin embargo, otros navegadores se están poniendo al día y es posible que funcionen igualmente bien cuando lea esto.

  1. Ingrese la siguiente URL en su navegador web:

                            
http://localhost:8080/reservation

  2. Introduzca información en los campos de la página reservation.xhtml.

    El campo Fecha de Rendimiento tiene un campo de fecha con flechas hacia arriba y hacia abajo que le permiten incrementar y disminuir el mes, el día y el año, así como una flecha hacia abajo más grande que muestra un editor de fechas en forma de calendario.

    Los campos Número de boletos y Precio del boleto también tienen flechas hacia arriba y hacia abajo que le permiten incrementar y disminuir los valores dentro del rango y los pasos permitidos. El total estimado cambia cuando cambia cualquiera de estos dos campos.

    Se comprueba el formato de las direcciones de correo electrónico y las fechas, pero no su validez (por ejemplo, puede hacer una reserva para una fecha pasada).

  3. Haga clic en Hacer reserva para completar la reserva o Borrar para restaurar los campos a sus valores predeterminados.

  4. Si hace clic en Hacer reserva, aparece la página confirm.xhtml, que muestra los valores enviados.

    Haga clic en Atrás para volver a la página reservation.xhtml.
