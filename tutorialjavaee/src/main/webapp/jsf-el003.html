Expresiones de Valor y Método

EL define dos tipos de expresiones: expresiones de valor y expresiones de método. Las expresiones de valor se pueden evaluar para producir un valor y las expresiones de método se utilizan para hacer referencia a un método.

Aquí se tratan los siguientes temas:

  • Expresiones de Valor

  • Expresiones de Método

  • Expresiones Lambda

Expresiones de Valor

Las expresiones de valor se pueden clasificar en expresiones rvalue y lvalue. Una expresión lvalue puede especificar un objetivo, como un objeto, una propiedad de bean o elementos de una colección, al que se le puede asignar un valor. Una expresión rvalue no puede especificar tal objetivo.

Todas las expresiones que se evalúan inmediatamente usan los delimitadores ${} y, aunque la expresión puede ser una expresión lvalue, nunca se realizarán asignaciones. Las expresiones cuya evaluación se puede aplazar utilizan los delimitadores #{} y pueden actuar como expresiones rvalue y lvalue; si la expresión es una expresión lvalue, se le puede asignar un nuevo valor. Considere las siguientes dos expresiones de valor: 

                
${customer.name}

#{customer.name}

El primero usa sintaxis de evaluación inmediata, mientras que el segundo usa sintaxis de evaluación diferida. La primera expresión accede a la propiedad name, obtiene su valor y pasa el valor al controlador de etiquetas. Con la segunda expresión, el controlador de etiquetas puede posponer la evaluación de la expresión a un momento posterior en el ciclo de vida de la página si la tecnología que utiliza esta etiqueta lo permite.

En el caso de la tecnología JavaServer Faces, la expresión de la última etiqueta se evalúa inmediatamente durante una solicitud inicial de la página. Durante una solicitud de devolución de datos, esta expresión se puede utilizar para establecer el valor de la propiedad name con la entrada del usuario.

Referenciando Objetos

Un identificador de nivel superior (como customer en la expresión customer.name) puede hacer referencia a los siguientes objetos:

  • Parámetros lambda

  • Variables EL

  • Beans gestionados

  • Objetos implícitos

  • Clases de campos estáticos y métodos

Para hacer referencia a estos objetos, escribe una expresión utilizando una variable que es el nombre del objeto. La siguiente expresión hace referencia a un bean administrado llamado customer: 

                  
${customer}

Puede utilizar un resolutor EL personalizado para modificar la forma en que se resuelven las variables. Por ejemplo, puede proporcionar un resolutor EL que intercepte objetos con el nombre customer, de modo que ${customer} devuelva un valor en el resolutor EL en su lugar. (La tecnología JavaServer Faces usa un resolutor EL para manejar beans gestionados).

Una constante enum es un caso especial de un campo estático, y puede hacer referencia a dicha constante directamente. Por ejemplo, considere esta clase enum: 

                  
public enum Suit {hearts, spades, diamonds, clubs}

En la siguiente expresión, en la que mySuit es una instancia de Suit, puede comparar suit.hearts con la instancia: 

                  
${mySuit == suit.hearts}

Referenciando Propiedades de Objeto o Elementos de la Colección

Para hacer referencia a las propiedades de un bean, campos estáticos o métodos de una clase, o elementos de una colección, utilice la notación . o []. La misma sintaxis se puede utilizar para los atributos de un objeto implícito, porque los atributos se colocan en un mapa.

Para hacer referencia a la propiedad name del bean customer, utilice la expresión ${customer.name} o la expresión ${customer["nombre"]}. Aquí, la parte dentro de los corchetes es un literal String que es el nombre de la propiedad a la que se hace referencia. La sintaxis de [] es más general que la de ., porque la parte dentro de los corchetes puede ser cualquier expresión de String, no solo literales.

Puede utilizar comillas simples o dobles para el literal String. También puede combinar las notaciones [] y ., como se muestra aquí: 

                  
${customer.address["street"]}

Puede hacer referencia a un campo o método estático utilizando la sintaxis classname.field, como en el siguiente ejemplo: 

                  
Boolean.FALSE

El nombre de clase es el nombre de la clase sin el nombre del paquete. De forma predeterminada, se importan todos los paquetes java.lang. Puede importar otros paquetes, clases y campos estáticos según sea necesario.

Si está accediendo a un elemento en una matriz o lista, debe usar la notación [] y especificar un índice en la matriz o lista. El índice es una expresión que se puede convertir a int. El siguiente ejemplo hace referencia al primero de los pedidos de los clientes, asumiendo que customer.orders es una List: 

                  
${customer.orders[1]}

Si está accediendo a un elemento en un Map, debe especificar la clave para el Map. Si la clave es un literal String, se puede utilizar la notación de punto (.). Suponiendo que customer.orders es un Map con una clave String, los siguientes ejemplos hacen referencia al elemento con la clave "socks": 

                  
${customer.orders["socks"]}

${customer.orders.socks}

Referenciando Literales

El EL define los siguientes literales:

  • Booleano: true y false

  • Entero: como en Java

  • Punto flotante: como en Java

  • String: Con comillas simples y dobles; " se escapa como \", ' se escapa como \' y \ se escapa como \\

  • Null: null

Aquí hay unos ejemplos:

  • ${"literal"}

  • ${true}

  • ${57}

Llamadas a Métodos Parametrizados

El EL ofrece soporte para llamadas a métodos parametrizados.

Los operadores . y [] pueden usarse para invocar llamadas a métodos con parámetros, como se muestra en la siguiente sintaxis de expresión:

  • expr-a`[expr-b](parameters)`

  • expr-a`.identifier-b(parameters)`

En la sintaxis de la primera expresión, expr-a se evalúa para representar un objeto bean. La expresión expr-b se evalúa y se convierte en una cadena que representa un método en el bean representado por expr-a. En la sintaxis de la segunda expresión, expr-a se evalúa para representar un objeto bean, e identifier-b es una cadena que representa un método en el objeto bean. Los parámetros entre paréntesis son los argumentos para la invocación del método. Los parámetros pueden ser cero o más valores de expresiones, separados por comas.

Los parámetros son compatibles tanto con expresiones de valor como con expresiones de método. En el siguiente ejemplo, que es una etiqueta modificada de la aplicación guessnumber, se proporciona un número aleatorio como argumento en lugar de la entrada del usuario a la llamada al método: 

                  
<h:inputText value="#{userNumberBean.userNumber('5')}">

El ejemplo anterior usa una expresión de valor.

Considere el siguiente ejemplo de una etiqueta de componente JavaServer Faces que usa una expresión de método: 

                  
<h:commandButton action="#{trader.buy}" value="buy"/>

La expresión EL trader.buy llama al método buy del bean trader. Puede modificar la etiqueta para transmitir un parámetro. Aquí está la etiqueta revisada en la que se pasa un parámetro: 

                  
<h:commandButton action="#{trader.buy('SOMESTOCK')}" value="buy"/>

En el ejemplo anterior, está pasando la cadena 'SOMESTOCK' (un símbolo bursátil) como parámetro del método buy.

Dónde Se Pueden Usar las Expresiones de Valor

Se pueden utilizar expresiones de valor que utilicen los delimitadores ${}

  • En texto estático

  • En cualquier atributo de etiqueta estándar o personalizado que pueda aceptar una expresión

El valor de una expresión en texto estático se calcula y se inserta en la salida actual. A continuación, se muestra un ejemplo de una expresión incrustada en texto estático: 

                  
<some:tag>
    some text ${expr} some text
</some:tag>

Un atributo de etiqueta se puede configurar de las siguientes formas.

  • Con una única construcción de expresión:

                             
<some:tag value="${expr}"/>

<another:tag value="#{expr}"/>

    Estas expresiones se evalúan y el resultado se convierte al tipo esperado del atributo.

  • Con una o más expresiones separadas o rodeadas por texto:

                            
<some:tag value="some${expr}${expr}text${expr}"/>

<another:tag value="some#{expr}#{expr}text#{expr}"/>

    Estos tipos de expresión, llamados expresiones compuestas, se evalúan de izquierda a derecha. Cada expresión incrustada en la expresión compuesta se convierte en un String y luego se concatena con cualquier texto intermedio. El String resultante se convierte luego al tipo esperado del atributo.

  • Solo con texto:

                            
<some:tag value="sometext"/>

    El valor String del atributo se convierte al tipo esperado del atributo.

Puede usar el operador de concatenación de cadenas + = para crear una expresión única a partir de lo que de otro modo sería una expresión compuesta. Por ejemplo, puede cambiar la expresión compuesta 

                  
<some:tag value="sometext ${expr} moretext"/>

a

                  
<some:tag value="${sometext += expr += moretext}"/>

Todas las expresiones utilizadas para establecer valores de atributo se evalúan en el contexto de un tipo esperado. Si el resultado de la evaluación de la expresión no coincide exactamente con el tipo esperado, se realizará una conversión de tipo. Por ejemplo, la expresión ${1.2E4} proporcionada como valor de un atributo de tipo float dará como resultado la siguiente conversión: 

                  
Float.valueOf("1.2E4").floatValue()

Expresiones de Método

Otra característica de EL es su soporte de expresiones de método diferido. Una expresión de método se utiliza para hacer referencia a un método público de un bean y tiene la misma sintaxis que una expresión lvalue.

En la tecnología JavaServer Faces, una etiqueta de componente representa un componente en una página. La etiqueta de componente usa expresiones de método para especificar métodos que se pueden invocar para realizar algún procesamiento para el componente. Estos métodos son necesarios para manejar los eventos que generan los componentes y para validar los datos de los componentes, como se muestra en este ejemplo: 

                
<h:form>
    <h:inputText id="name"
                 value="#{customer.name}"
                 validator="#{customer.validateName}"/>
    <h:commandButton id="submit"
                     action="#{customer.submit}" />
</h:form>

La etiqueta h:inputText se muestra como un campo. El atributo validator de esta etiqueta h:inputText hace referencia a un método, llamado validateName, en el bean, llamado customer.

Debido a que un método puede invocarse durante diferentes fases del ciclo de vida, las expresiones de método siempre deben usar la sintaxis de evaluación diferida.

Al igual que las expresiones lvalue, las expresiones de método pueden usar los operadores . y []. Por ejemplo, #{object.method} es equivalente a #{object["method"]}. El literal dentro del [] se convierte en String y se usa para encontrar el nombre del método que coincide con él.

Las expresiones de método solo se pueden usar en atributos de etiqueta y solo de las siguientes maneras:

  • Con una construcción de expresión única, donde bean se refiere a un componente JavaBeans y método se refiere a un método del componente JavaBeans: 

                          
<some:tag value="#{bean.method}"/>

    La expresión se evalúa como una expresión de método, que se pasa al controlador de etiquetas. El método representado por la expresión del método se puede invocar más tarde.

  • Solo con texto:

                          
<some:tag value="sometext"/>

    Las expresiones de método admiten literales principalmente para admitir atributos action en la tecnología JavaServer Faces. Cuando se invoca el método al que hace referencia esta expresión de método, el método devuelve el literal String, que luego se convierte al tipo de retorno esperado, como se define en el descriptor de la biblioteca de etiquetas de la etiqueta.

Expresiones Lambda

Una expresión lambda es una expresión de valor con parámetros. La sintaxis es similar a la de la expresión lambda en el lenguaje de programación Java, excepto que en EL, el cuerpo de la expresión lambda es una expresión EL.

Para obtener información básica sobre expresiones lambda, consulte http://docs.oracle.com/javase/tutorial/java/javaOO/lambdaexpressions.html .

Nota:

Las expresiones Lambda son parte de Java SE 8, pero puede usarlas en expresiones EL con Java SE 7, la versión de Java asociada con la plataforma Java EE 7.

Una expresión lambda usa el operador de token de flecha (→) . Los identificadores a la izquierda del operador se denominan parámetros lambda. El cuerpo, a la derecha del operador, debe ser una expresión EL. Los parámetros lambda están entre paréntesis; los paréntesis se pueden omitir si solo hay un parámetro. Aquí hay unos ejemplos: 

                
x -> x+1
(x, y) -> x + y
() -> 64

Una expresión lambda se comporta como una función. Se puede invocar de inmediato. Por ejemplo, la siguiente invocación se evalúa como 7: 

                
((x, y) -> x + y)(3, 4)

Puede utilizar una expresión lambda junto con los operadores de asignación y punto y coma. Por ejemplo, el siguiente código asigna la expresión lambda anterior a una variable y luego la invoca. El resultado es nuevamente 7: 

                
v = (x, y) -> x + y; v(3, 4)

Una expresión lambda también se puede pasar como argumento a un método y se puede invocar en el método. También se puede anidar en otra expresión lambda.
