¿Qué Es una Aplicación JavaServer Faces?

La funcionalidad proporcionada por una aplicación JavaServer Faces es similar a la de cualquier otra aplicación web Java. Una aplicación típica de JavaServer Faces incluye las siguientes partes.

  • Un conjunto de páginas web en las que se distribuyen los componentes.

  • Un conjunto de etiquetas para agregar componentes a la página web.

  • Un conjunto de beans administrados, que son objetos livianos administrados por contenedores (POJO). En una aplicación JavaServer Faces, los beans administrados sirven como beans de respaldo, que definen propiedades y funciones para los componentes de la interfaz de usuario en una página.

  • Un descriptor de implementación web (archivo web.xml).

  • Opcionalmente, uno o más archivos de recursos de configuración de la aplicación, como un archivo faces-config.xml, que se puede usar para definir reglas de navegación de página y configurar beans y otros objetos personalizados, como componentes personalizados.

  • Opcionalmente, un conjunto de objetos personalizados, que pueden incluir componentes personalizados, validadores, convertidores u oyentes, creados por el desarrollador de la aplicación.

  • Opcionalmente, un conjunto de etiquetas personalizadas para representar objetos personalizados en la página.

La Figura 7-1 muestra la interacción entre el cliente y el servidor en una aplicación típica de JavaServer Faces. En respuesta a la solicitud de un cliente, el contenedor web representa una página web que implementa la tecnología JavaServer Faces.

Figura 7-1 Respuesta a una solicitud de cliente para una página de JavaServer Faces

La página web, myfacelet.xhtml, se crea utilizando etiquetas de componentes JavaServer Faces. Las etiquetas de componente se utilizan para agregar componentes a la vista (representada por myView en el diagrama), que es la representación del lado del servidor de la página. Además de los componentes, la página web también puede hacer referencia a objetos, como los siguientes:

  • Cualquier detector de eventos, validadores y convertidores que estén registrados en los componentes.

  • Los componentes de JavaBeans que capturan los datos y procesan la funcionalidad específica de la aplicación de los componentes.

A petición del cliente, la vista se representa como respuesta. La renderización es el proceso mediante el cual, basándose en la vista del lado del servidor, el contenedor web genera resultados, como HTML o XHTML, que el cliente puede leer, como un navegador.
