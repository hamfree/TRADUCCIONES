La Aplicación de Ejemplo websocketbot

La aplicación de ejemplo websocketbot, que se encuentra en el directorio tut-install`/examples/web/websocket/websocketbot/`, demuestra cómo usar un extremo de WebSocket para implementar un chat. El ejemplo se asemeja a una sala de chat en la que muchos usuarios pueden unirse y tener una conversación. Los usuarios pueden hacer preguntas sencillas a un agente bot que siempre está disponible en la sala de chat.

Aquí se tratan los siguientes temas:

  • Arquitectura de la Aplicación de Ejemplo websocketbot

  • Ejecutando la Aplicación de Ejemplo websocketbot

Arquitectura de la Aplicación de Ejemplo websocketbot

La aplicación de ejemplo websocketbot consta de los siguientes elementos:

  • El Bean CDI – Un bean CDI (BotBean) que contiene la lógica para que el agente bot responda a los mensajes

  • El Endpoint del WebSocket – Un punto final de WebSocket (BotEndpoint) que implementa la sala de chat

  • Los Mensajes de la Aplicación – Un conjunto de clases (Message, ChatMessage, InfoMessage, JoinMessage y UsersMessage ) que representan mensajes de aplicación

  • Las Clases Codificadoras – Un conjunto de clases (ChatMessageEncoder, InfoMessageEncoder, JoinMessageEncoder y UsersMessageEncoder) que codifican mensajes de aplicaciones en mensajes de texto WebSocket como datos JSON

  • El Decodificador del Mensaje – Una clase (MessageDecoder) analiza los mensajes de texto WebSocket como datos JSON y los decodifica en objetos JoinMessage o ChatMessage

  • La Página HTML – Una página HTML (index.html) que usa código JavaScript para implementar el cliente para la sala de chat

El Bean CDI

El bean CDI (BotBean) es una clase de Java que contiene el método respond. Este método compara el mensaje de chat entrante con un conjunto de preguntas predefinidas y devuelve una respuesta de chat. 

                  
@Named
public class BotBean {
    public String respond(String msg) { ... }
}

El Endpoint del WebSocket

El punto final de WebSocket (BotEndpoint) es un punto final anotado que realiza las siguientes funciones:

  • Recibe mensajes de los clientes.

  • Reenvía mensajes a clientes

  • Mantiene una lista de clientes conectados

  • Invoca la funcionalidad del agente bot.

El punto final especifica su URI de implementación y los codificadores y decodificadores de mensajes mediante la anotación ServerEndpoint. El punto final obtiene una instancia de la clase BotBean y un recurso de servicio de ejecutor administrado a través de la inyección de dependencia: 

                  
@ServerEndpoint(
   value = "/websocketbot",
   decoders = { MessageDecoder.class },
   encoders = { JoinMessageEncoder.class, ChatMessageEncoder.class,
                InfoMessageEncoder.class, UsersMessageEncoder.class }
)
/* Hay una instancia de BotEndpoint por conexión */
public class BotEndpoint {
   private static final Logger logger = Logger.getLogger("BotEndpoint");
   /* Bean de funcionalidad de bot */
   @Inject private BotBean botbean;
   /* Servicio ejecutor para procesamiento asíncrono */
   @Resource(name="comp/DefaultManagedExecutorService")
   private ManagedExecutorService mes;
   @OnOpen
   public void openConnection(Session session) {
       logger.log(Level.INFO, "Conexión abierta.");
   }
   ...
}

El método message procesa los mensajes entrantes de los clientes. El decodificador convierte los mensajes de texto entrantes en objetos JoinMessage o ChatMessage, que heredan de la clase Message. El método message recibe un objeto Message como parámetro: 

                  
@OnMessage
public void message(Session session, Message msg) {
   logger.log(Level.INFO, "Received: {0}", msg.toString());
   if (msg instanceof JoinMessage) {
      /* Agregar el nuevo usuario y notificar a todos */
      JoinMessage jmsg = (JoinMessage) msg;
      session.getUserProperties().put("name", jmsg.getName());
      session.getUserProperties().put("active", true);
      logger.log(Level.INFO, "Received: {0}", jmsg.toString());
      sendAll(session, new InfoMessage(jmsg.getName() +
              " se ha unido a la charla"));
      sendAll(session, new ChatMessage("Duke", jmsg.getName(),
              "¡¡Hola!!"));
      sendAll(session, new UsersMessage(this.getUserList(session)));
   } else if (msg instanceof ChatMessage) {
      /* Reenviar el mensaje a todos */
      ChatMessage cmsg = (ChatMessage) msg;
      logger.log(Level.INFO, "Recibido: {0}", cmsg.toString());
      sendAll(session, cmsg);
      if (cmsg.getTarget().compareTo("Duke") == 0) {
         /* El bot responde al mensaje. */
         mes.submit(new Runnable() {
            @Override
            public void run() {
               String resp = botbean.respond(cmsg.getMessage());
               sendAll(session, new ChatMessage("Duke",
                       cmsg.getName(), resp));
            }
         });
      }
   }
}

Si el mensaje es un mensaje de unión, el extremo agrega el nuevo usuario a la lista y notifica a todos los clientes conectados. Si el mensaje es un mensaje de chat, el punto final lo reenvía a todos los clientes conectados.

Si un mensaje de chat es para el agente bot, el punto final obtiene una respuesta mediante la instancia de BotBean y la envía a todos los clientes conectados. El método sendAll es similar al ejemplo en Enviando Mensajes a Todos los Pares Conectados a un Endpoint .

Consideraciones de Simultaneidad y Procesamiento Asíncrono

El extremo de WebSocket llama al método BotBean.respond para obtener una respuesta del bot. En este ejemplo, esta es una operación de bloqueo; el usuario que envió el mensaje asociado no podrá enviar ni recibir otros mensajes de chat hasta que se complete la operación. Para evitar este problema, el punto final obtiene un servicio ejecutor del contenedor y ejecuta la operación de bloqueo en un subproceso diferente utilizando el método ManagedExecutorService.submit de Utilidades de Concurrencia para Java EE.

La especificación deñ IPA de Java para WebSocket requiere que las implementaciones de Java EE creen instancias de clases de punto final una vez por conexión. Esto facilita el desarrollo de puntos finales de WebSocket, porque tiene la garantía de que solo un subproceso está ejecutando el código en una clase de punto final de WebSocket en un momento dado. Cuando introduce un nuevo subproceso en un punto final, como en este ejemplo, debe asegurarse de que las variables y los métodos a los que accede más de un subproceso sean seguros para subprocesos. En este ejemplo, el código en BotBean es seguro para subprocesos y el método BotEndpoint.sendAll se ha declarado synchronized.

Consulte el Capítulo 59, "Utilidades de Concurrencia para Java EE" para obtener más información sobre el servicio ejecutor administrado y las Utilidades de Concurrencia para Java EE.

Los Mensajes de la Aplicación

Las clases que representan los mensajes de la aplicación (Message, ChatMessage, InfoMessage, JoinMessage y UsersMessage) contienen solo propiedades y métodos getter y setter. Por ejemplo, la clase ChatMessage se ve así: 

                  
public class ChatMessage extends Message {
    private String name;
    private String target;
    private String message;
    /* ... Constructor, getters, y setters ... */
}

Las Clases Codificadoras

Las clases de codificador convierten objetos de mensajes de aplicación en texto JSON utilizando la IPA de Java para procesar JSON. Por ejemplo, la clase ChatMessageEncoder se implementa de la siguiente manera: 

                  
/* Codifica un ChatMessage como JSON.
 * Por ejemplo, (new ChatMessage("Peter","Duke","¿Cómo estás?"))
 * se codifica de la siguiente manera:
 * {"type":"chat","target":"Duke","message":"¿Cómo estás?"}
 */
public class ChatMessageEncoder implements Encoder.Text<ChatMessage> {
   @Override
   public void init(EndpointConfig ec) { }
   @Override
   public void destroy() { }
   @Override
   public String encode(ChatMessage chatMessage) throws EncodeException {
      // Acceda a las propiedades en chatMessage y escriba texto JSON...
   }
}

El Decodificador del Mensaje

La clase del decodificador de mensajes (MessageDecoder) convierte los mensajes de texto de WebSocket en mensajes de aplicación mediante el análisis del texto JSON. Se implementa de la siguiente manera: 

                  
/* Descodifica un mensaje JSON en un JoinMessage o ChatMessage.
 * Por ejemplo, el mensaje entrante
 * {"type":"chat","name":"Peter","target":"Duke","message":"¿Cómo estás?"}
 * se decodifica como (new ChatMessage("Peter", "Duke", "¿Cómo estás?"))
 */
public class MessageDecoder implements Decoder.Text<Message> {
    /* Almacena los pares de nombre y valor de un mensaje JSON como un map */
    private Map<String,String> messageMap;
    @Override
    public void init(EndpointConfig ec) { }
    @Override
    public void destroy() { }
    /* Cree un nuevo objeto Message si el mensaje se puede decodificar */
    @Override
    public Message decode(String string) throws DecodeException {
       Message msg = null;
       if (willDecode(string)) {
          switch (messageMap.get("type")) {
             case "join":
                msg = new JoinMessage(messageMap.get("name"));
                break;
             case "chat":
                msg = new ChatMessage(messageMap.get("name"),
                                      messageMap.get("target"),
                                      messageMap.get("message"));
          }
       } else {
          throw new DecodeException(string, "[Message] no puedo decodificar.");
       }
       return msg;
   }
   /* Decodifica un mensaje JSON en un mapa y verifica si contiene
    * todos los campos obligatorios según su tipo. */
   @Override
   public boolean willDecode(String string) {
      // Convierta los datos JSON del mensaje en un mapa de nombre y valor...
      // Compruebe si el mensaje tiene todos los campos para su tipo de mensaje...
   }
}

La Página HTML

La página HTML (index.html) contiene un campo para el nombre de usuario. Después de que el usuario escriba un nombre y haga clic en Unirse, hay tres áreas de texto disponibles: una para escribir y enviar mensajes, otra para la sala de chat y otra con la lista de usuarios. La página también contiene una consola WebSocket que muestra los mensajes enviados y recibidos como texto JSON.

El código JavaScript de la página utiliza la API de WebSocket para conectarse al punto final, enviar mensajes y designar métodos de devolución de llamada. La API de WebSocket es compatible con la mayoría de los navegadores modernos y se usa ampliamente para el desarrollo de clientes web con HTML5.

Ejecutando la Aplicación de Ejemplo websocketbot

Esta sección describe cómo ejecutar la aplicación de ejemplo websocketbot utilizando el EID de NetBeans y desde la línea de comandos.

Aquí se tratan los siguientes temas:

  • Para Ejecutar la Aplicación de Ejemplo websocketbot Usando el EID NetBeans

  • Para Ejecutar la Aplicación de Ejemplo websocketbot Usando Maven

  • Para Comprobar la Aplicación de Ejemplo websocketbot Usando Maven

Para Ejecutar la Aplicación de Ejemplo websocketbot Usando el EID NetBeans

  1. Asegúrese de que GlassFish Server se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En el menú Archivo, elija Abrir Proyecto.

  3. En el cuadro de diálogo Abrir Proyecto, vaya a:

                            
tut-install/examples/web/websocket

  4. Seleccione la carpeta websocketbot.

  5. Haga clic en Abrir Proyecto.

  6. En la pestaña Proyectos, haga clic con el botón derecho en el proyecto websocketbot y seleccione Ejecutar.

    Este comando compila y empaqueta la aplicación en un archivo WAR, websocketbot.war, ubicado en el directorio target/; lo implementa en el servidor; y abre una ventana del navegador web con la siguiente URL: 

                            
http://localhost:8080/websocketbot/

    Vea Para Comprobar la Aplicación Ejemplo websocketbot Usando Maven para más información.

Para Ejecutar la Aplicación de Ejemplo websocketbot Usando Maven

  1. Asegúrese de que GlassFish Server se haya iniciado (consulte Arrancando y Parando el Servidor GlassFish).

  2. En una ventana de terminal, vaya a:

                            
tut-install/examples/web/websocket/websocketbot/

  3. Ingrese el siguiente comando para implementar la aplicación:

                            
mvn install

  4. Abra una ventana del navegador web y escriba la siguiente dirección:

                            
http://localhost:8080/websocketbot/

    Vea Para Comprobar la Aplicación de Ejemplo websocketbot Usando Maven para más información.

Para Comprobar la Aplicación de Ejemplo websocketbot Usando Maven

  1. En la página principal, escriba su nombre en el primer campo de texto y presione la tecla Intro.

    La lista de usuarios conectados aparece en el área de texto de la derecha. El área de texto de la izquierda es la sala de chat.

  2. Escriba un mensaje en el área de texto debajo del botón de inicio de sesión. Por ejemplo, escriba los mensajes en negrita y presione enter para obtener respuestas similares a las siguientes: 

                            
[--Peter se ha unido a la charla--]
Duke: @Peter ¡¡Hola!!
Peter: @Duke ¿cómo estás?
Duke: @Peter ¡Estoy muy bien, gracias!
Peter: @Duke ¿cuándo es tu cumpleaños?
Duke: @Peter Mi cumpleaños es el 23 de mayo. ¡Gracias por preguntar!

  3. Únase al chat desde otra ventana del navegador copiando y pegando el URI en la barra de direcciones y uniéndose con un nombre diferente.

    El nuevo nombre de usuario aparece en la lista de usuarios en ambas ventanas del navegador. Puede enviar mensajes desde cualquier ventana y ver cómo aparecen en la otra.

  4. Haga clic en Mostrar Consola WebSocket.

    La consola muestra los mensajes enviados y recibidos como texto JSON.
