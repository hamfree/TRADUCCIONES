Referenciando un Método de un Bean Gestionado

Una etiqueta de componente tiene un conjunto de atributos para hacer referencia a métodos de bean gestionados que pueden realizar ciertas funciones para el componente asociado con la etiqueta. Estos atributos se resumen en la Tabla 11-7.

Tabla 11-7 Atributos de etiqueta de componente que hacen referencia a métodos de bean gestionados

Atributo

Función

action

Se refiere a un método de bean gestionad que realiza el procesamiento de navegación para el componente y devuelve un resultado lógico String

actionListener

Se refiere a un método de bean gestionado que maneja eventos de acción.

validator

Se refiere a un método de bean gestionado que realiza la validación del valor del componente.

valueChangeListener

Se refiere a un método de bean gestionado que maneja eventos de cambio de valor.

Solo los componentes que implementan ActionSource pueden usar los atributos action y actionListener. Solo los componentes que implementan EditableValueHolder pueden usar los atributos validator o valueChangeListener.

La etiqueta de componente se refiere a un método de bean gestionado que utiliza una expresión de método como valor de uno de los atributos. El método al que hace referencia un atributo debe seguir una firma particular, que está definida por la definición del atributo de etiqueta en la documentación de la biblioteca de etiquetas de Facelets de JavaServer Faces. Por ejemplo, la definición del atributo validator de la etiqueta inputText es la siguiente: 


void validate(javax.faces.context.FacesContext,
              javax.faces.component.UIComponent, java.lang.Object)

Las siguientes secciones dan ejemplos de cómo utilizar los atributos.

Referenciando un Método Que Realiza la Navegación

Si su página incluye un componente, como un botón o un enlace, que hace que la aplicación navegue a otra página cuando el componente está activado, la etiqueta correspondiente a este componente debe incluir un atributo action. Este atributo realiza una de las siguientes acciones:

  • Especifica un resultado lógico String que le dice a la aplicación a qué página acceder a continuación

  • Hace referencia a un método de bean gestionado que realiza algún procesamiento y devuelve un resultado lógico String

El siguiente ejemplo muestra cómo hacer referencia a un método de navegación:


<h:commandButton value="#{bundle.Submit}"
                 action="#{cashierBean.submit}" />

Vea Escribiendo un Método para Manejar la Navegación para obtener información sobre cómo escribir dicho método.

Referenciando un Método Que Maneja un Evento Action

Si un componente en su página genera un evento de acción, y si ese evento es manejado por un método de bean gestionado, consulte el método usando el atributo actionListener del componente.

El siguiente ejemplo muestra cómo se puede hacer referencia a un método de este tipo:


<h:commandLink id="Duke" action="bookstore"
               actionListener="#{actionBean.chooseBookFromLink}">

El atributo actionListener de esta etiqueta de componente hace referencia al método chooseBookFromLink utilizando una expresión de método. El método chooseBookFromLink maneja el evento cuando el usuario hace clic en el enlace representado por este componente. Consulte Escribiendo un Método para Manejar un Evento Action para obtener información sobre cómo escribir dicho método.

Referenciando un Método Que Realiza la Validación

Si la entrada de uno de los componentes en su página es validada por un método de bean gestionado, consulte el método de la etiqueta del componente usando el atributo validator.

El siguiente ejemplo simplificado de El Ejemplo de CDI guessnumber-cdi muestra cómo hacer referencia a un método que realiza la validación en inputGuess, un componente de entrada: 


<h:inputText id="inputGuess"
    value="#{userNumberBean.userNumber}"
    required="true" size="3"
    disabled="#{userNumberBean.number eq userNumberBean.userNumber ...}"
    validator="#{userNumberBean.validateNumberRange}">
</h:inputText>

El método de bean gestionado validateNumberRange verifica que el valor de entrada esté dentro del rango válido, que cambia cada vez que se realiza otra conjetura. Consulte Escribiendo un Método para Realizar una Validación para obtener información sobre cómo escribir dicho método.

Referenciando un Método Que Maneja un Evento Value-Change

Si desea que un componente en su página genere un evento de cambio de valor y desea que ese evento sea manejado por un método de bean gestionado en lugar de una implementación de ValueChangeListener, consulte el método usando el atributo valueChangeListener del componente: 


<h:inputText id="name"
             size="30"
             value="#{cashierBean.name}"
             required="true"
             valueChangeListener="#{cashierBean.processValueChange}" />
</h:inputText>

El atributo valueChangeListener de esta etiqueta de componente hace referencia al método processValueChange de CashierBean utilizando una expresión de método. El método processValueChange maneja el evento de un usuario que ingresa un nombre en el campo de entrada representado por este componente.

Escribiendo un Método para Manejar un Evento Value-Change describe cómo implementar un método que maneja un ValueChangeEvent.
