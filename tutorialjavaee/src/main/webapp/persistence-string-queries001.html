Descripción General de las Consultas de la IPA Criteria basadas en cadenas

Las consultas API de criterios basadas en cadenas ("consultas basadas en cadenas") son consultas del lenguaje de programación Java que utilizan cadenas en lugar de objetos de metamodelo fuertemente tipados para especificar atributos de entidad al atravesar una jerarquía de datos. Las consultas basadas en cadenas se construyen de manera similar a las consultas de metamodelo, pueden ser estáticas o dinámicas y pueden expresar el mismo tipo de consultas y operaciones que las consultas de metamodelo fuertemente tipadas.

Las consultas de metamodelo fuertemente tipadas son el método preferido para construir consultas de la API de Criterios.

La principal ventaja de las consultas basadas en cadenas sobre las consultas de metamodelos es la capacidad de construir consultas de Criterios en el momento del desarrollo sin la necesidad de generar clases de metamodelos estáticos o acceder de otro modo a clases de metamodelos generadas dinámicamente.

La principal desventaja de las consultas basadas en cadenas es su falta de seguridad de tipo; este problema puede generar errores de tiempo de ejecución debido a discrepancias de tipos que se detectarían en el momento del desarrollo si utiliza consultas de metamodelo fuertemente tipadas.

Para obtener información sobre la creación de consultas de criterios, consulte Capítulo 43, "Usando la IPA Criteria para Crear Consultas".
