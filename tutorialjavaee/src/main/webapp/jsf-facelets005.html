Componentes Compuestos

La tecnología JavaServer Faces ofrece el concepto de componentes compuestos con Facelets. Un componente compuesto es un tipo especial de plantilla que actúa como componente.

Cualquier componente es esencialmente un fragmento de código reutilizable que se comporta de una manera particular. Por ejemplo, un componente de entrada acepta la entrada del usuario. Un componente también puede tener validadores, convertidores y escuchas adjuntos para realizar ciertas acciones definidas.

Un componente compuesto consta de una colección de etiquetas de marcado y otros componentes existentes. Este componente reutilizable creado por el usuario tiene una funcionalidad definida y personalizada y puede tener validadores, convertidores y oyentes adjuntos como cualquier otro componente.

Con Facelets, cualquier página XHTML que contenga etiquetas de marcado y otros componentes se puede convertir en un componente compuesto. Con la función de recursos, el componente compuesto se puede almacenar en una biblioteca que está disponible para la aplicación desde la ubicación de recursos definida.

La Tabla 8-3 enumera las etiquetas compuestas más utilizadas y sus funciones.

Tabla 8-3 Etiquetas de Componentes Compuestos

Etiqueta

Función

composite:interface

Declara el contrato de uso de un componente compuesto. El componente compuesto se puede utilizar como un solo componente cuyo conjunto de características es la unión de las características declaradas en el contrato de uso.

composite:implementation

Define la implementación del componente compuesto. Si aparece un elemento composite:interface, debe haber un composite:implementation correspondiente.

composite:attribute

Declara un atributo que se puede otorgar a una instancia del componente compuesto en el que se declara esta etiqueta.

composite:insertChildren

Cualquier componente secundario o texto de plantilla dentro de la etiqueta del componente compuesto en la página de uso se volverá a crear en el componente compuesto en el punto indicado por la ubicación de esta etiqueta dentro de la sección composite:deployment.

composite:valueHolder

Declara que el componente compuesto cuyo contrato es declarado por el composite:interface en el que este elemento está anidado expone una implementación de ValueHolder adecuada para su uso como destino de objetos adjuntos en el usando la página.

composite:editableValueHolder

Declara que el componente compuesto cuyo contrato es declarado por el composite:interface en el que este elemento está anidado expone una implementación de EditableValueHolder adecuada para su uso como destino de objetos adjuntos en el usando la página.

composite:actionSource

Declara que el componente compuesto cuyo contrato es declarado por el composite:interface en el que este elemento está anidado expone una implementación de ActionSource2 adecuada para su uso como destino de objetos adjuntos en el usando la página.

Para obtener más información y una lista completa de las etiquetas compuestas de Facelets, consulte la documentación de la biblioteca de etiquetas de Facelets de JavaServer Faces.

El siguiente ejemplo muestra un componente compuesto que acepta una dirección de correo electrónico como entrada: 

              
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml"
  xmlns:composite="http://xmlns.jcp.org/jsf/composite"
  xmlns:h="http://xmlns.jcp.org/jsf/html">

    <h:head>
        <title>This content will not be displayed</title>
    </h:head>
    <h:body>
        <composite:interface>
            <composite:attribute name="value" required="false"/>
        </composite:interface>

        <composite:implementation>
            <h:outputLabel value="Email id: "></h:outputLabel>
            <h:inputText value="#{cc.attrs.value}"></h:inputText>
        </composite:implementation>
    </h:body>
</html>

Tenga en cuenta el uso de cc.attrs.value al definir el valor del componente inputText. La palabra cc en JavaServer Faces es una palabra reservada para componentes compuestos. La expresión #{cc.attrs.`attribute-name}` se utiliza para acceder a los atributos definidos para la interfaz del componente compuesto, que en este caso resulta ser value.

El contenido del ejemplo anterior se almacena como un archivo llamado email.xhtml en una carpeta llamada resources/emcomp, en el directorio raíz web de la aplicación. Este directorio es considerado una biblioteca por JavaServer Faces, y se puede acceder a un componente desde dicha biblioteca. Para obtener más información sobre los recursos, consulte Recursos web.

La página web que utiliza este componente compuesto generalmente se denomina página de uso. La página de uso incluye una referencia al componente compuesto, en las declaraciones del espacio de nombres xml: 

              
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml"
  xmlns:h="http://xmlns.jcp.org/jsf/html"
  xmlns:em="http://xmlns.jcp.org/jsf/composite/emcomp">

    <h:head>
        <title>Using a sample composite component</title>
    </h:head>

    <body>
        <h:form>
            <em:email value="Enter your email id" />
        </h:form>
    </body>
</html>

La biblioteca de componentes compuestos local se define en el espacio de nombres xmlns con la declaración xmlns:em="http://xmlns.jcp.org/jsf/composite/emcomp". Se accede al componente en sí a través de la etiqueta em:email. El contenido del ejemplo anterior se puede almacenar como una página web denominada emuserpage.xhtml en el directorio raíz de la web. Cuando se compila y se implementa en un servidor, se puede acceder con la siguiente URL: 

              
http://localhost:8080/application-name/emuserpage.xhtml
