Estructura de Directorio de Ejemplo de Tutorial

Para facilitar el desarrollo iterativo y mantener los archivos fuente de la aplicación separados de los archivos compilados, los ejemplos del tutorial utilizan la estructura de directorios de la aplicación Maven.

Cada módulo de aplicación tiene la siguiente estructura:

  • pom.xml: Archivo de compilación de Maven

  • src/main/java: Archivos fuente de Java para el módulo

  • src/main/resources:archivos de configuración del módulo, a excepción de las aplicaciones web

  • src/main/webapp: páginas web, hojas de estilo, archivos de etiquetas e imágenes (solo aplicaciones web)

  • src/main/webapp/WEB-INF: archivos de configuración para aplicaciones web (solo aplicaciones web)

Cuando un ejemplo tiene varios módulos de aplicación empaquetados en un archivo EAR, sus directorios de submódulos utilizan las siguientes convenciones de nomenclatura:

  • nombre-del-ejemplo`-app-client`: clientes de la aplicación

  • nombre-del-ejemplo`-ejb`: archivos JAR de enterprise bean

  • nombre-del-ejemplo`-war`: aplicaciones web

  • nombre-del-ejemplo`-ear`: Aplicaciones empresariales

  • nombre-del-ejemplo`-common`: JAR de biblioteca que contiene componentes, clases y archivos utilizados por otros módulos

Los archivos de compilación de Maven(pom.xml) distribuidos con los ejemplos contienen objetivos para compilar y ensamblar la aplicación en el directorio target e implementar el archivo en GlassFish Server.
