Una Aplicación JavaServer Faces Simple

La tecnología JavaServer Faces proporciona un proceso fácil y amigable para el usuario para crear aplicaciones web. El desarrollo de una aplicación simple JavaServer Faces generalmente requiere las siguientes tareas, que ya se han descrito en Un módulo web que utiliza Tecnología JavaServer Faces: el ejemplo hello1 :

  • Creación de páginas web con etiquetas de componentes

  • Desarrollar beans administrados

  • Asignación de la instancia de FacesServlet

El ejemplo hello1 incluye un bean administrado y dos páginas web Facelets. Cuando accede un cliente, la primera página web le pide al usuario su nombre y la segunda página responde con un saludo.

Para obtener detalles sobre la tecnología Facelets, consulte Capítulo 8, "Introducción a Facelets". Para obtener detalles sobre el uso de expresiones EL, consulte Capítulo 9, "Lenguaje de expresión". Para obtener detalles sobre el modelo de programación JavaServer Faces y la creación de páginas web con la tecnología JavaServer Faces, consulte el Capítulo 10, "Uso de Tecnología JavaServer Faces en páginas web". .

Cada aplicación web tiene un ciclo de vida. Las tareas habituales, como la gestión de solicitudes entrantes, la decodificación de parámetros, la modificación y el almacenamiento del estado y la presentación de páginas web en el navegador, se realizan durante el ciclo de vida de una aplicación web. Algunos marcos de aplicaciones web le ocultan los detalles del ciclo de vida, mientras que otros requieren que los administre manualmente.

De forma predeterminada, JavaServer Faces gestiona automáticamente la mayoría de las acciones del ciclo de vida. Sin embargo, también expone las distintas etapas del ciclo de vida de la solicitud para que pueda modificar o realizar diferentes acciones si los requisitos de su aplicación lo justifican.

El ciclo de vida de una aplicación JavaServer Faces comienza y termina con la siguiente actividad: El cliente realiza una solicitud para la página web y el servidor responde con la página. El ciclo de vida consta de dos fases principales: Ejecutar y Renderizar.

Durante la fase de Ejecución, pueden tener lugar varias acciones.

  • La vista de la aplicación se crea o se restaura.

  • Se aplican los valores de los parámetros de solicitud.

  • Las conversiones y validaciones se realizan para los valores de los componentes.

  • Los beans gestionados se actualizan con los valores de los componentes.

  • Se invoca la lógica de la aplicación.

Para una primera solicitud (inicial), solo se crea la vista. Para solicitudes posteriores (devolución de datos), pueden tener lugar algunas o todas las demás acciones.

En la fase de Renderizado, la vista solicitada se renderiza como una respuesta al cliente. La renderización es típicamente el proceso de generar resultados, como HTML o XHTML, que el cliente, normalmente un navegador, puede leer.

La siguiente descripción breve del ejemplo de la aplicación JavaServer Faces que pasa por su ciclo de vida resume la actividad que tiene lugar detrás de escena.

La aplicación de ejemplo hello1 pasa por las siguientes etapas cuando se implementa en el Servidor GlassFish.

  1. Cuando la aplicación hello1 se crea e implementa en el Servidor GlassFish, la aplicación se encuentra en un estado no iniciado.

  2. Cuando un cliente realiza una solicitud inicial para la página web index.xhtml, se compila la aplicación Facelets hello1.

  3. Se ejecuta la aplicación Facelets compilada y se construye un nuevo árbol de componentes para la aplicación hello1 y se coloca en un FacesContext.

  4. El árbol de componentes se completa con el componente y la propiedad del bean administrado asociada a él, representada por la expresión EL hello.name.

  5. Se crea una nueva vista, basada en el árbol de componentes.

  6. La vista se muestra al cliente solicitante como respuesta.

  7. El árbol de componentes se destruye automáticamente.

  8. En solicitudes posteriores (devolución de datos), el árbol de componentes se reconstruye y se aplica el estado guardado.

Para obtener detalles completos sobre el ciclo de vida, consulte El ciclo de vida de una aplicación JavaServer Faces.
