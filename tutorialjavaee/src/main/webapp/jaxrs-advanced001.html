Anotaciones Para Campos y Propiedades de Bean de las Clases Recurso

Las anotaciones JAX-RS para clases de recursos le permiten extraer partes o valores específicos de un identificador uniforme de recursos (URI) o un encabezado de solicitud.

JAX-RS proporciona las anotaciones enumeradas en la Tabla 32-1.

Tabla 32-1 Anotaciones JAX-RS Avanzadas

Anotación

Descripción

@Context

Inyecta información en un campo de clase, propiedad de bean o parámetro de método

@CookieParam

Extrae información de las cookies declaradas en el encabezado de solicitud de cookies

@FormParam

Extrae información de una representación de solicitud cuyo tipo de contenido es application/x-www-form-urlencoded

@HeaderParam

Extrae el valor de un encabezado.

@MatrixParam

Extrae el valor de un parámetro de matriz URI

@PathParam

Extrae el valor de un parámetro de plantilla URI

@QueryParam

Extrae el valor de un parámetro de consulta URI

Extraer Parámetros de Ruta

Las plantillas de ruta de URI son URI con variables incrustadas dentro de la sintaxis de URI. La anotación @PathParam le permite usar fragmentos de ruta URI variables cuando llama a un método.

El siguiente fragmento de código muestra cómo extraer el apellido de un empleado cuando se proporciona la dirección de correo electrónico del empleado: 

                
@Path("/employees/{firstname}.{lastname}@{domain}.com")
public class EmpResource {
    @GET
    @Produces("text/xml")
    public String getEmployeelastname(@PathParam("lastname") String lastName) {
      ...
    }
}

En este ejemplo, la anotación @Path define las variables URI (o parámetros de ruta) {firstname}, {lastname} y {dominio}. El @PathParam en el parámetro de método del método de solicitud extrae el apellido de la dirección de correo electrónico.

Si su solicitud HTTP es GET /employees/john.doe@example.com, el valor “doe” se inyecta en {lastname}.

Puede especificar varios parámetros de ruta en un URI.

Puede declarar una expresión regular con una variable URI. Por ejemplo, si se requiere que el apellido conste solo de caracteres en minúsculas y mayúsculas, puede declarar la siguiente expresión regular: 

                
@Path("/employees/{firstname}.{lastname[a-zA-Z]*}@{domain}.com")

Si el apellido no coincide con la expresión regular, se devuelve una respuesta 404.

Extraer Parámetros de Consulta

Utilice la anotación @QueryParam para extraer parámetros de consulta del componente de consulta del URI de solicitud.

Por ejemplo, para consultar a todos los empleados que se han unido dentro de un rango específico de años, use una firma de método como la siguiente: 

                
@Path("/employees/")
@GET
public Response getEmployees(
        @DefaultValue("2003") @QueryParam("minyear") int minyear,
        @DefaultValue("2013") @QueryParam("maxyear") int maxyear)
    {...}

Este fragmento de código define dos parámetros de consulta, minyear y maxyear. La siguiente solicitud HTTP consultaría a todos los empleados que se unieron entre 2003 y 2013: 

                
GET /employees?maxyear=2013&minyear=2003

La anotación @DefaultValue define un valor predeterminado, que se utilizará si no se proporcionan valores para los parámetros de consulta. De forma predeterminada, JAX-RS asigna un valor nulo para los valores de Object y cero para los tipos de datos primitivos. Puede usar la anotación @DefaultValue para eliminar valores nulos o cero y definir sus propios valores predeterminados para un parámetro.

Extraer Datos del Formulario

Utilice la anotación @FormParam para extraer parámetros de formulario de formularios HTML. Por ejemplo, el siguiente formulario acepta el nombre, la dirección y el nombre del gerente de un empleado: 

                
<FORM action="http://example.com/employees/" method="post">
  <p>
    <fieldset>
      Employee name: <INPUT type="text" name="empname" tabindex="1">
      Employee address: <INPUT type="text" name="empaddress" tabindex="2">
      Manager name: <INPUT type="text" name="managername" tabindex="3">
    </fieldset>
  </p>
</FORM>

Utilice el siguiente fragmento de código para extraer el nombre del administrador de este formulario HTML: 

                
@POST
@Consumes("application/x-www-form-urlencoded")
public void post(@FormParam("managername") String managername) {
    // Almacenar el valor
    ...
}

Para obtener un mapa de nombres de parámetros de formulario a valores, use un fragmento de código como el siguiente: 

                
@POST
@Consumes("application/x-www-form-urlencoded")
public void post(MultivaluedMap<String, String> formParams) {
    // Almacenar el valor
}

Extraer el Tipo Java de una Petición o Respuesta

La anotación javax.ws.rs.core.Context recupera los tipos Java relacionados con una solicitud o respuesta.

La interfaz javax.ws.rs.core.UriInfo proporciona información sobre los componentes de un URI de solicitud. El siguiente fragmento de código muestra cómo obtener un mapa de nombres de parámetros de consulta y ruta a valores: 

                
@GET
public String getParams(@Context UriInfo ui) {
    MultivaluedMap<String, String> queryParams = ui.getQueryParameters();
    MultivaluedMap<String, String> pathParams = ui.getPathParameters();
}

La interfaz javax.ws.rs.core.HttpHeaders proporciona información sobre encabezados de solicitud y cookies. El siguiente fragmento de código muestra cómo obtener un mapa de nombres de parámetros de cookies y encabezados en valores: 

                
@GET
public String getHeaders(@Context HttpHeaders hh) {
    MultivaluedMap<String, String> headerParams = hh.getRequestHeaders();
    MultivaluedMap<String, Cookie> pathParams = hh.getCookies();
}
