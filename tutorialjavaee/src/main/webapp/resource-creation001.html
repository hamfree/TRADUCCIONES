Recursos y Nombrado JNDI

En una aplicación distribuida, los componentes necesitan acceder a otros componentes y recursos, como bases de datos. Por ejemplo, un servlet puede invocar métodos remotos en un bean empresarial que recupera información de una base de datos. En la plataforma Java EE, el servicio de nombres Java Naming and Directory Interface (JNDI) permite a los componentes localizar otros componentes y recursos.

Un recurso es un objeto de programa que proporciona conexiones a sistemas, como servidores de bases de datos y sistemas de mensajería. (Un recurso de conectividad de base de datos Java a veces se denomina fuente de datos). Cada objeto de recurso se identifica mediante un nombre exclusivo y fácil de usar, denominado nombre JNDI. Por ejemplo, el nombre JNDI del recurso JDBC preconfigurado para Apache Derby enviado con GlassFish Server es java:comp/DefaultDataSource .

Un administrador crea recursos en un espacio de nombres JNDI. En GlassFish Server, puede usar la Consola de administración o el comando asadmin para crear recursos. Las aplicaciones luego usan anotaciones para inyectar los recursos. Si una aplicación utiliza la inyección de recursos, GlassFish Server invoca la API JNDI y la aplicación no está obligada a hacerlo. Sin embargo, también es posible que una aplicación localice recursos realizando llamadas directas a la API JNDI.

Un objeto de recurso y su nombre JNDI están vinculados por el servicio de nomenclatura y directorio. Para crear un nuevo recurso, se ingresa un nuevo enlace de nombre/objeto en el espacio de nombres JNDI. Usted inyecta recursos usando la anotación @Resource en una aplicación.

Puede utilizar un descriptor de implementación para anular la asignación de recursos que especifique en una anotación. El uso de un descriptor de implementación le permite cambiar una aplicación volviéndola a empaquetar en lugar de volver a compilar los archivos fuente y volver a empaquetarla. Sin embargo, para la mayoría de las aplicaciones no es necesario un descriptor de implementación.
