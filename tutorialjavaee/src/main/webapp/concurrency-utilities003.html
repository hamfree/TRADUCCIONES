Concurrencia y Transacciones

Las operaciones más básicas para las transacciones son la confirmación y la reversión, pero, en un entorno distribuido con procesamiento simultáneo, puede ser difícil garantizar que las operaciones de confirmación o reversión se procesarán con éxito, y la transacción se puede distribuir entre diferentes subprocesos, núcleos de CPU, máquinas físicas y redes.

Asegurarse de que una operación de reversión se ejecute con éxito en tal escenario es crucial. Las utilidades de concurrencia se basan en la IPA de Transacciones de Java (JTA) para implementar y admitir transacciones en sus componentes a través de javax.transaction.UserTransaction, lo que permite a los desarrolladores de aplicaciones administrar explícitamente los límites de las transacciones. Hay más información disponible en la especificación JTA.

Opcionalmente, los objetos de contexto pueden iniciar, confirmar o revertir transacciones, pero estos objetos no pueden inscribirse en transacciones de componentes principales.

El siguiente fragmento de código ilustra una tarea Runnable que obtiene una UserTransaction y luego inicia y confirma una transacción mientras interactúa con otros componentes transaccionales, como un bean empresarial y una base de datos: 

              
public class MyTransactionalTask implements Runnable {
   UserTransaction ut = ... // obtained through JNDI or injection
   public void run() {
       // Start a transaction
       ut.begin();
       // Invoke a Service or an EJB
       myEJB.businessMethod();
       // Update a database entity using an XA JDBC driver
       myEJB.updateCustomer(customer);
       // Commit the transaction
       ut.commit();
   }
}
