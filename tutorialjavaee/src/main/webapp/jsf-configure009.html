Registrando un Convertidor Personalizado

Como es el caso con un validador personalizado, si el desarrollador de la aplicación crea un convertidor personalizado, debe registrarlo con la aplicación utilizando la anotación @FacesConverter, como se describe en Creando un Convertidor Personalizado, o utilizando el elemento XML converter en el archivo de recursos de configuración de la aplicación. A continuación, se muestra una configuración hipotética de converter para CreditCardConverter del caso de estudio de Duke's Bookstore: 


<converter>
    <description>
        Converter for credit card numbers that normalizes
        the input to a standard format
    </description>
    <converter-id>CreditCardConverter</converter-id>
    <converter-class>
        dukesbookstore.converters.CreditCardConverter
    </converter-class>
</converter>

Los atributos especificados en una etiqueta converter anulan cualquier configuración en la anotación @FacesConverter.

El elemento converter representa una implementación de javax.faces.convert.Converter y contiene los elementos requeridos converter-id y converter-class.

El elemento converter-id identifica un ID que utiliza el atributo converter de una etiqueta de componente de la interfaz de usuario para aplicar el convertidor a los datos del componente. Usando un Convertidor Personalizado incluye un ejemplo de referencia al convertidor personalizado desde una etiqueta de componente.

El elemento converter-class identifica la implementación de Converter.

Creando y Usando un Convertidor Personalizado explica cómo crear un convertidor personalizado.
