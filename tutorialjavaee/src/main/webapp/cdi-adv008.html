Usando Estereotipos en Aplicaciones CDI

Un estereotipo es un tipo de anotación, aplicada a un bean, que incorpora otras anotaciones. Los estereotipos pueden ser particularmente útiles en aplicaciones grandes en las que tiene varios beans que realizan funciones similares. Un estereotipo es un tipo de anotación que especifica lo siguiente:

  • Un ámbito predeterminado

  • Cero o más enlaces de interceptor

  • Opcionalmente, una anotación @Named, que garantiza la denominación EL predeterminada

  • Opcionalmente, una anotación @Alternative, especificando que todos los beans con este estereotipo son alternativos

Un bean anotado con un estereotipo particular siempre usará las anotaciones especificadas, por lo que no tiene que aplicar las mismas anotaciones a muchos beans.

Por ejemplo, puede crear un estereotipo llamado Action, utilizando la anotación javax.enterprise.inject.Stereotype: 

              
@RequestScoped
@Secure
@Transactional
@Named
@Stereotype
@Target(TYPE)
@Retention(RUNTIME)
public @interface Action {}

Todos los beans anotados como @Action tendrán un alcance de solicitud, usarán nombres EL predeterminados y tendrán los enlaces de interceptor @Transactional y @Secure.

También podría crear un estereotipo llamado Mock:

              
@Alternative
@Stereotype
@Target(TYPE)
@Retention(RUNTIME)
public @interface Mock {}

Todos los beans con esta anotación son alternativas.

Es posible aplicar múltiples estereotipos al mismo bean, por lo que puede anotar un bean de la siguiente manera: 

              
@Action
@Mock
public class MockLoginAction extends LoginAction { ... }

También es posible anular el alcance especificado por un estereotipo, simplemente especificando un alcance diferente para el bean. La siguiente declaración proporciona el alcance de la sesión del bean MockLoginAction en lugar del alcance de la solicitud: 

              
@SessionScoped
@Action
@Mock
public class MockLoginAction extends LoginAction { ... }

CDI pone a disposición un estereotipo incorporado llamado Model, que está diseñado para usarse con beans que definen la capa de modelo de una arquitectura de aplicación de modelo-vista-controlador. Este estereotipo especifica que un bean es tanto @Named como @RequestScoped: 

              
@Named
@RequestScoped
@Stereotype
@Target({TYPE, METHOD, FIELD})
@Retention(RUNTIME)
public @interface Model {}
