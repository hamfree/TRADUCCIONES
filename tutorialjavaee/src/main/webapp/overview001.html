Introducción a Java EE

Los desarrolladores de hoy reconocen cada vez más la necesidad de aplicaciones distribuidas, transaccionales y portátiles que aprovechen la velocidad, la seguridad y la confiabilidad de la tecnología del lado del servidor. Las aplicaciones empresariales proporcionan la lógica empresarial para una empresa. Se gestionan de forma centralizada y, a menudo, interactúan con otro software empresarial. En el mundo de la tecnología de la información, las aplicaciones empresariales deben diseñarse, construirse y producirse por menos dinero, con mayor velocidad y con menos recursos.

Con la Plataforma Java, Edición Empresarial (Java EE), el desarrollo de aplicaciones empresariales Java nunca ha sido más fácil ni más rápido. El objetivo de la plataforma Java EE es proporcionar a los desarrolladores un potente conjunto de API al tiempo que acorta el tiempo de desarrollo, reduce la complejidad de la aplicación y mejora el rendimiento de la aplicación.

La plataforma Java EE se desarrolla a través del Java Community Process JCP), que es responsable de todas las tecnologías Java. Grupos de expertos compuestos por partes interesadas han creado solicitudes de especificación de Java (JSR) para definir las diversas tecnologías de Java EE. El trabajo de la Comunidad Java bajo el programa JCP ayuda a asegurar los estándares de estabilidad y compatibilidad multiplataforma de la tecnología Java.

La plataforma Java EE utiliza un modelo de programación simplificado. Los descriptores de despliegue XML son opcionales. En su lugar, un desarrollador puede simplemente ingresar la información como una anotación directamente en un archivo fuente de Java, y el servidor Java EE configurará el componente durante la implementación y el tiempo de ejecución. Estas anotaciones se utilizan generalmente para incrustar en un programa datos que de otro modo se proporcionarían en un descriptor de implementación. Con las anotaciones, coloca la información de la especificación en su código junto al elemento del programa afectado.

En la plataforma Java EE, la inyección de dependencia se puede aplicar a todos los recursos que necesita un componente, ocultando efectivamente la creación y búsqueda de recursos del código de la aplicación. La inyección de dependencias se puede utilizar en contenedores Enterprise JavaBeans (EJB), contenedores web y clientes de aplicaciones. La inyección de dependencias permite que el contenedor Java EE inserte automáticamente referencias a otros componentes o recursos necesarios mediante anotaciones.

Este tutorial utiliza ejemplos para describir las funciones disponibles en la plataforma Java EE para desarrollar aplicaciones empresariales. Tanto si es un desarrollador empresarial nuevo como si tiene experiencia, los ejemplos y el texto que los acompañan deberían constituir una base de conocimientos valiosa y accesible para crear sus propias soluciones.
