Validar los Datos del Recurso con Validación de Bean

JAX-RS soporta la Validación de Beans para verificar clases de recursos JAX-RS. Este apoyo consiste en:

  • Adición de anotaciones de restricciones a los parámetros del método de recursos

  • Garantizar que los datos de la entidad sean válidos cuando la entidad se pasa como parámetro

Aquí se tratan los siguientes temas:

  • Usando Anotaciones de Restricción en los Métodos de Recurso

  • Validar los Datos de la Entidad

  • Manejo de la Excepción de Validación y Códigos de Respuesta

Usando Anotaciones de Restricción en los Métodos de Recurso

Las anotaciones de restricciones de Validación de Beans se pueden aplicar a los parámetros de un recurso. El servidor validará los parámetros y pasará o lanzará una javax.validation.ValidationException. 

                
@POST
@Path("/createUser")
@Consumes(MediaType.APPLICATION_FORM_URLENCODED)
public void createUser(@NotNull @FormParam("username") String username,
                       @NotNull @FormParam("firstName") String firstName,
                       @NotNull @FormParam("lastName") String lastName,
                       @Email @FormParam("email") String email) {
    ...
}

En el ejemplo anterior, la restricción integrada @NotNull se aplica a los campos de formulario username, firstName y lastName . Otra restricción incorporada @Email valida que la dirección de correo electrónico proporcionada por el campo de formulario email tenga el formato correcto.

Las restricciones también se pueden aplicar a campos dentro de una clase de recurso.

                
@Path("/createUser")
public class CreateUserResource {
  @NotNull
  @FormParam("username")
  private String username;
  @NotNull
  @FormParam("firstName")
  private String firstName;
  @NotNull
  @FormParam("lastName")
  private String lastName;
  @Email
  @FormParam("email")
  private String email;
  ...
}

En el ejemplo anterior, las mismas restricciones que se aplicaron a los parámetros del método en el ejemplo anterior se aplican a los campos de clase. El comportamiento es el mismo en ambos ejemplos.

Las restricciones también se pueden aplicar a las propiedades JavaBeans de una clase de recurso agregando las anotaciones de restricción al método getter. 

                
@Path("/createuser")
public class CreateUserResource {
  private String username;
  @FormParam("username")
  public void setUsername(String username) {
    this.username = username;
  }
  @NotNull
  public String getUsername() {
    return username;
  }
  ...
}

Las restricciones también se pueden aplicar a nivel de clase de recurso. En el siguiente ejemplo, @PhoneRequired es una restricción definida por el usuario que garantiza que un usuario ingrese al menos un número de teléfono. Es decir, homePhone o mobilePhone pueden ser nulos, pero no ambos. 

                
@Path("/createUser")
@PhoneRequired
public class CreateUserResource {
  @FormParam("homePhone")
  private Phone homePhone;
  @FormParam("mobilePhone")
  private Phone mobilePhone;
  ...
}

Validar los Datos de la Entidad

Las clases que contienen anotaciones de restricciones de validación se pueden usar en parámetros de método en una clase de recursos. Para validar estas clases de entidad, use la anotación @Valid en el parámetro del método. Por ejemplo, la siguiente clase es una clase definida por el usuario que contiene restricciones de validación estándar y definidas por el usuario. 

                
@PhoneRequired
public class User {
  @NotNull
  private String username;
  private Phone homePhone;
  private Phone mobilePhone;
  ...
}

Esta clase de entidad se utiliza como parámetro para un método de recursos.

                
@Path("/createUser")
public class CreateUserResource {
  ...
  @POST
  @Consumers(MediaType.APPLICATION_XML)
  public void createUser(@Valid User user) {
    ...
  }
  ...
}

La anotación @Valid garantiza que la clase de entidad se valide en tiempo de ejecución. Las restricciones adicionales definidas por el usuario también pueden desencadenar la validación de una entidad. 

                
@Path("/createUser")
public class CreateUserResource {
  ...
  @POST
  @Consumers(MediaType.APPLICATION_XML)
  public void createUser(@ActiveUser User user) {
    ...
  }
  ...
}

En el ejemplo anterior, la restricción @ActiveUser definida por el usuario se aplica a la clase User además de @PhoneRequired y @ Restricciones NotNull definidas dentro de la clase de entidad.

Si un método de recurso devuelve una clase de entidad, la validación puede desencadenarse aplicando @Valid o cualquier otra anotación de restricción definida por el usuario al método de recurso. 

                
@Path("/getUser")
public class GetUserResource {
  ...
  @GET
  @Path("{username}")
  @Produces(MediaType.APPLICATION_XML)
  @ActiveUser
  @Valid
  public User getUser(@PathParam("username") String username) {
    // encuentra el Usuario
    return user;
  }
  ...
}

Como en el ejemplo anterior, la restricción @ActiveUser se aplica a la clase de entidad devuelta, así como las restricciones @PhoneRequired y @NotNull definidas dentro la clase de entidad.

Manejo de la Excepción de Validación y Códigos de Respuesta

Si se lanza una javax.validation.ValidationException o cualquier subclase de ValidationException excepto ConstraintValidationException, el tiempo de ejecución de JAX-RS responderá a la solicitud del cliente con un código de estado HTTP 500 (Error interno del servidor).

Si se lanza una ConstraintValidationException, el tiempo de ejecución de JAX-RS responderá al cliente con uno de los siguientes códigos de estado HTTP:

  • 500 (Error interno del servidor) si se lanzó la excepción al validar un tipo de retorno de método

  • 400 (Solicitud incorrecta) en todos los demás casos
