Aplicaciones Web

Una aplicación web es una extensión dinámica de un servidor web o de aplicaciones. Las aplicaciones web son de los siguientes tipos:

  • Orientado a la presentación: una aplicación web orientada a la presentación genera páginas web interactivas que contienen varios tipos de contenido de marcas (HTML, XHTML, XML, etc.) y contenido dinámico en respuesta a las solicitudes. El desarrollo de aplicaciones web orientadas a la presentación está cubierta en Capítulo 7, "Tecnología JavaServer Faces", hasta el Capítulo 18, "Tecnología Servlet Java".

  • Orientado por el servicio: una aplicación web orientada al servicio implementa el punto final de un servicio web. Las aplicaciones orientadas a la presentación son a menudo clientes de aplicaciones web orientadas al servicio. El desarrollo de aplicaciones web orientadas al servicio está cubierta en el capítulo 31, "Construyendo servicios web con JAX-WS," y capítulo 32, "Construyendo los servicios web RESTful con JAX-RS" en la Parte III, "Servicios web".

En la plataforma Java EE, los componentes web proporcionan las capacidades de extensión dinámica para un servidor web. Los componentes web pueden ser servlets Java, páginas web implementadas con tecnología JavaServer Faces, puntos finales de servicios web o páginas JSP. La Figura 6-1 ilustra la interacción entre un cliente web y una aplicación web que utiliza un servlet. El cliente envía una solicitud HTTP al servidor web. Un servidor web que implementa la tecnología Java Servlet y JavaServer Pages convierte la solicitud en un objeto HTTPServletRequest. Este objeto se entrega a un componente web, que puede interactuar con componentes JavaBeans o una base de datos para generar contenido dinámico. El componente web puede generar un HTTPServletResponse o puede pasar la solicitud a otro componente web. Un componente web genera eventualmente un objeto HTTPServletResponse. El servidor web convierte este objeto en una respuesta HTTP y lo devuelve al cliente.

Los servlets son clases del lenguaje de programación Java que procesan dinámicamente solicitudes y construyen respuestas. Las tecnologías Java, como JavaServer Faces y Facelets, se utilizan para crear aplicaciones web interactivas. (Los marcos de trabajo también se pueden usar para este propósito). Aunque los servlets y las páginas JavaServer Faces y Facelets se pueden usar para lograr cosas similares, cada uno tiene sus propios puntos fuertes. Los servlets son los más adecuados para aplicaciones orientadas a servicios (los puntos finales de servicios web se pueden implementar como servlets) y las funciones de control de una aplicación orientada a presentaciones, como el envío de solicitudes y el manejo de datos no textuales. Las páginas JavaServer Faces y Facelets son más apropiadas para generar marcado basado en texto, como XHTML, y generalmente se usan para aplicaciones orientadas a presentaciones.

Los componentes web son compatibles con los servicios de una plataforma en tiempo de ejecución denominada contenedor web. Un contenedor web proporciona servicios como envío de solicitudes, seguridad, simultaneidad y gestión del ciclo de vida. Un contenedor web también brinda a los componentes web acceso a IPAs como nombres, transacciones y correo electrónico.

Ciertos aspectos del comportamiento de la aplicación web se pueden configurar cuando la aplicación se instala o implementa en el contenedor web. La información de configuración se puede especificar mediante anotaciones Java EE o se puede mantener en un archivo de texto en formato XML llamado descriptor de implementación de aplicaciones web (DD). Una aplicación web DD debe ajustarse al esquema descrito en la especificación de Java Servlet.

Este capítulo ofrece una breve descripción de las actividades involucradas en el desarrollo de aplicaciones web. Primero, resume el ciclo de vida de la aplicación web y explica cómo empaquetar e implementar aplicaciones web muy simples en GlassFish Server. A continuación, el capítulo pasa a la configuración de aplicaciones web y analiza cómo especificar el parámetro de configuración más utilizado.
