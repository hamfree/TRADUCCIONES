Ciclo de Vida de una Aplicación Web

Una aplicación web consta de componentes web; archivos de recursos estáticos, como imágenes y hojas de estilo en cascada (CSS); y librerías y clases auxiliares. El contenedor web proporciona muchos servicios de apoyo que mejoran las capacidades de los componentes web y los hacen más fáciles de desarrollar. Sin embargo, debido a que una aplicación web debe tener en cuenta estos servicios, el proceso para crear y ejecutar una aplicación web es diferente al de las clases Java independientes tradicionales.

El proceso para crear, implementar y ejecutar una aplicación web se puede resumir de la siguiente manera:

  1. Desarrolle el código del componente web.

  2. Desarrolle el descriptor de implementación de la aplicación web, si es necesario.

  3. Compile los componentes de la aplicación web y las clases auxiliares a las que hacen referencia los componentes.

  4. Opcionalmente, empaquete la aplicación en una unidad desplegable.

  5. Implemente la aplicación en un contenedor web.

  6. Acceda a una URL que haga referencia a la aplicación web.

El desarrollo de código de componentes web se trata en los capítulos posteriores. Los pasos 2 a 4 se amplían en las siguientes secciones y se ilustran con una aplicación orientada a presentaciones al estilo Hello, World. Esta aplicación permite al usuario ingresar un nombre en un formulario HTML y luego muestra un saludo después de que se envía el nombre.

La aplicación Hello contiene dos componentes web que generan el saludo y la respuesta. Este capítulo analiza las siguientes aplicaciones simples:

  • hello1, una aplicación basada en la tecnología JavaServer Faces que utiliza dos páginas XHTML y un bean gestionado

  • hello2, una aplicación web basada en servlets en la que los componentes son implementados por dos clases de servlets

Las aplicaciones se utilizan para ilustrar las tareas relacionadas con el empaquetado, la implementación, la configuración y la ejecución de una aplicación que contiene componentes web.
