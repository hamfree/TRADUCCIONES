Creando Recursos de Forma Administrativa

Antes de implementar o ejecutar muchas aplicaciones, es posible que deba crear recursos para ellas. Una aplicación puede incluir un archivo glassfish-resources.xml que se puede utilizar para definir recursos para esa aplicación y otras. Luego puede usar el comando asadmin, especificando como argumento un archivo llamado glassfish-resources.xml, para crear los recursos administrativamente, como se muestra. 


asadmin add-resources glassfish-resources.xml

El archivo glassfish-resources.xml se puede crear en cualquier proyecto usando EID de NetBeans o manualmente. Algunos de los ejemplos de JMS utilizan este enfoque para la creación de recursos. Se puede encontrar un archivo para crear los recursos necesarios para el ejemplo del productor simple de JMS en el directorio jms/simple/producer/src/main/setup.

También puede usar el comando asadmin create-jms-resource para crear los recursos para este ejemplo. Cuando haya terminado de usar los recursos, debe usar el comando asadmin list-jms-resources para mostrar sus nombres, y el comando asadmin delete-jms-resource para eliminarlos, independientemente de la forma en que creó los recursos.
