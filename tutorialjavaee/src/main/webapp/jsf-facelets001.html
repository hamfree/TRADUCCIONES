¿Qué Son los Facelets?

Facelets es un lenguaje de declaración de página poderoso pero liviano que se usa para construir vistas de JavaServer Faces usando plantillas de estilo HTML y para construir árboles de componentes. Las características de Facelets incluyen lo siguiente:

  • Uso de XHTML para crear páginas web

  • Soporte para bibliotecas de etiquetas Facelets además de JavaServer Faces y bibliotecas de etiquetas JSTL

  • Soporte para el Lenguaje de Expresión (EL)

  • Creación de plantillas para componentes y páginas

Las ventajas de Facelets para proyectos de desarrollo a gran escala incluyen las siguientes:

  • Soporte para la reutilización de código a través de plantillas y componentes compuestos.

  • Extensibilidad funcional de componentes y otros objetos del lado del servidor a través de personalización.

  • Tiempo de compilación más rápido

  • Validación de EL en tiempo de compilación

  • Renderizado de alto rendimiento

En resumen, el uso de Facelets reduce el tiempo y el esfuerzo que se debe dedicar al desarrollo y la implementación.

Las vistas de facelets generalmente se crean como páginas XHTML. Las implementaciones de JavaServer Faces admiten páginas XHTML creadas de conformidad con la Definición de tipo de documento de transición (DTD) XHTML, como se indica en http://www.w3.org/TR/xhtml1/#a_dtd_XHTML-1.0-Transitional . Por convención, las páginas web creadas con XHTML tienen una extensión .xhtml.

La tecnología JavaServer Faces admite varias bibliotecas de etiquetas para agregar componentes a una página web. Para admitir el mecanismo de la biblioteca de etiquetas JavaServer Faces, Facelets utiliza declaraciones de espacio de nombres XML. Facelets proporciona dos espacios de nombres para admitir el marcado compatible con HTML5. Para obtener más información, consulte Marcado compatible con HTML5.

Facelets admite etiquetas para componentes compuestos, para los cuales puede declarar prefijos personalizados. Para obtener más información sobre componentes compuestos, consulte Componentes compuestos.

Los prefijos de espacio de nombres que se muestran en la tabla son convencionales, no obligatorios. Como siempre ocurre cuando declaras un espacio de nombres XML, puedes especificar cualquier prefijo en tu página de Facelets. Por ejemplo, puede declarar el prefijo de la biblioteca de etiquetas de componentes compuestos como 

              
xmlns:composite="http://java.sun.com/jsf/composite"

en vez de 

              
xmlns:cc="http://java.sun.com/jsf/composite"

Basado en el soporte de JavaServer Faces para la sintaxis de Expression Language (EL), Facelets usa expresiones EL para hacer referencia a propiedades y métodos de beans administrados. Las expresiones EL se pueden utilizar para vincular objetos o valores de componentes a métodos o propiedades de beans gestionados que se utilizan como beans de respaldo. Para obtener más información sobre el uso de expresiones EL, consulte Uso de EL para hacer referencia a Beans Gestionados.
