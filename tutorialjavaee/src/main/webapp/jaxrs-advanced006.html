Negociación de Contenido en Tiempo de Ejecución

Las anotaciones @Produces y @Consumes manejan la negociación de contenido estático en JAX-RS. Estas anotaciones especifican las preferencias de contenido del servidor. Los encabezados HTTP como Accept, Content-Type y Accept-Language definen las preferencias de negociación de contenido del cliente.

Para obtener más detalles sobre los encabezados HTTP para la negociación de contenido, consulte HTTP /1.1 - Negociación de contenido (http://www.w3.org/Protocols/rfc2616/rfc2616-sec12.html).

El siguiente fragmento de código muestra las preferencias de contenido del servidor:

              
@Produces("text/plain")
@Path("/employee")
public class Employee {
    @GET
    public String getEmployeeAddressText(String address) {...}
    @Produces("text/xml")
    @GET
    public String getEmployeeAddressXml(Address address) {...}
}

Se llama al método getEmployeeAddressText para una solicitud HTTP similar a la siguiente: 

              
GET /employee
Accept: text/plain

Esto producirá la siguiente respuesta:

              
500 Oracle Parkway, Redwood Shores, CA

Se llama al método getEmployeeAddressXml para una solicitud HTTP similar a la siguiente: 

              
GET /employee
Accept: text/xml

Esto producirá la siguiente respuesta:

              
<address street="500 Oracle Parkway, Redwood Shores, CA" country="USA"/>

Con la negociación de contenido estático, también puede definir varios tipos de contenido y medios para el cliente y el servidor. 

              
@Produces("text/plain", "text/xml")

Además de admitir la negociación de contenido estático, JAX-RS también admite la negociación de contenido en tiempo de ejecución mediante la clase javax.ws.rs.core.Variant y los objetos Request. La clase Variant especifica la representación de recursos de la negociación de contenido. Cada instancia de la clase Variant puede contener un tipo de medio, un idioma y una codificación. El objeto Variant define la representación de recursos que admite el servidor. La clase Variant.VariantListBuilder se utiliza para crear una lista de variantes de representación.

El siguiente fragmento de código muestra cómo crear una lista de variantes de representación de recursos: 

              
List<Variant> vs = Variant.mediatypes("application/xml", "application/json")
        .languages("en", "fr").build();

Este fragmento de código llama al método build de la clase VariantListBuilder. La clase VariantListBuilder se invoca cuando llama a los métodos mediatypes, languages o encodings. El método build construye una serie de representaciones de recursos. La lista Variant creada por el método build tiene todas las combinaciones posibles de elementos especificados en mediatypes, languages y métodos de encodings.

En este ejemplo, el tamaño del objeto vs como se define en este fragmento de código es 4 y el contenido es el siguiente: 

              
[["application/xml","en"], ["application/json","en"],
    ["application/xml","fr"],["application/json","fr"]]

El método javax.ws.rs.core.Request.selectVariant acepta una lista de objetos Variant y elige el objeto Variant que coincide con la solicitud HTTP. Este método compara su lista de objetos Variant con los encabezados Accept, Accept-Encoding, Accept-Language y Accept-Charset de la solicitud HTTP.

El siguiente fragmento de código muestra cómo usar el método selectVariant para seleccionar la Variant más aceptable de los valores en la solicitud del cliente: 

              
@GET
public Response get(@Context Request r) {
    List<Variant> vs = ...;
    Variant v = r.selectVariant(vs);
    if (v == null) {
        return Response.notAcceptable(vs).build();
    } else {
        Object rep = selectRepresentation(v);
        return Response.ok(rep, v);
    }
}

El método selectVariant devuelve el objeto Variant que coincide con la solicitud o nulo si no se encuentran coincidencias. En este fragmento de código, si el método devuelve un valor nulo, se crea un objeto Response para una respuesta no aceptable. De lo contrario, se devuelve un objeto Response con un estado OK y que contiene una representación en forma de una entidad Object y una Variant.
