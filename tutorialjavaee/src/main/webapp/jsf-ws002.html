Configurando WebSockets

Para configurar WebSockets para su uso en aplicaciones web JSF, primero habilite el extremo de WebSocket usando el parámetro de contexto en web.xml: 

              
<context-param>
<param-name>javax.faces.ENABLE_WEBSOCKET_ENDPOINT</param-name>
<param-value>true</param-value>
</context-param>

Si su servidor está configurado para ejecutar un contenedor WebSocket en un puerto TCP diferente al del contenedor HTTP, puede usar el parámetro de contexto entero opcional javax.faces.WEBSOCKET_ENDPOINT_PORT para especificar explícitamente el puerto: 

              
<context-param>
<param-name>javax.faces.WEBSOCKET_ENDPOINT_PORT</param-name>
<param-value>8000</param-value>
</context-param>

Uso del WebSocket: Lado del Cliente

Declare la etiqueta f:websocket en la vista JSF con un nombre de canal y una función de escucha de JavaScript onmessage.

El siguiente ejemplo hace referencia a una función de escucha de JavaScript existente:

                
<f:websocket channel="someChannel"
onmessage="someWebsocketListener" />
function someWebsocketListener(message, channel, event) { console.log(message);
}
<f:websocket channel="someChannel"
onmessage="

Este ejemplo declara una función de escucha de JavaScript en línea:

                
<f:websocket channel="someChannel"
onmessage="function(m){console.log(m);}" />

La función de escucha JavaScript onmessage se invoca con tres argumentos:

  • message: El mensaje push como un objeto JSON

  • channel: El nombre del canal

  • event: La instancia MessageEvent

Cuando se conecta correctamente, el WebSocket está abierto de forma predeterminada mientras el documento esté abierto y se volverá a conectar automáticamente, a intervalos crecientes, cuando la conexión se cierre o se anule, como resultado de eventos como un error de red o reinicio del servidor. No se volverá a conectar automáticamente cuando falle el primer intento de conexión. El WebSocket se cerrará implícitamente después de que se descargue el documento.

Uso del WebSocket: Lado del Servidor

En el lado de la programación de Java, inyecte un PushContext usando la anotación @Push en el canal dado en cualquier CDI o artefacto administrado por contenedor, como @Named, o @WebServlet, donde desee enviar un mensaje push. Luego invoque PushContext.send(Object) con cualquier objeto Java que represente el mensaje push.

For example:

                
@Inject @Push
private PushContext someChannel;
public void sendMessage(Object message) {
someChannel.send(message);
}

Por defecto, el nombre del canal se toma del nombre de la variable en la que se realiza la inyección.

Opcionalmente, el nombre del canal se puede especificar usando el atributo channel. El siguiente ejemplo inyecta el contexto push para el nombre del canal foo en una variable llamada bar.

@Inject @Push(channel="foo") private PushContext bar;

El objeto del mensaje se codificará como JSON y se entregará como un argumento de mensaje de la función de escucha de JavaScript onmessage asociada con el nombre del canal. Puede ser una cadena, pero también puede ser una colección, un mapa o un JavaBean.
