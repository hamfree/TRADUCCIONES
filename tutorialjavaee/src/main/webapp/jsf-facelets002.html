El Ciclo de Vida de una Aplicación Facelets

La especificación JavaServer Faces define el ciclo de vida de una aplicación JavaServer Faces. Para obtener más información sobre este ciclo de vida, consulte El ciclo de vida de una aplicación JavaServer Faces. Los siguientes pasos describen ese proceso aplicado a una aplicación basada en Facelets.

  1. Cuando un cliente, como un navegador, realiza una nueva solicitud a una página que se crea con Facelets, se crea un nuevo árbol de componentes o javax.faces.component.UIViewRoot y se coloca en el FacesContext.

  2. El UIViewRoot se aplica a los Facelets y la vista se completa con componentes para renderizar.

  3. La vista recién construida se devuelve como respuesta al cliente.

  4. Al renderizar, el estado de esta vista se almacena para la siguiente solicitud. Se almacena el estado de los componentes de entrada y los datos del formulario.

  5. El cliente puede interactuar con la vista y solicitar otra vista o cambiar desde la aplicación JavaServer Faces. En este momento, la vista guardada se restaura desde el estado almacenado.

  6. La vista restaurada vuelve a pasar por el ciclo de vida de JavaServer Faces, que eventualmente generará una nueva vista o volverá a representar la vista actual si no hubo problemas de validación y no se activó ninguna acción.

  7. Si se solicita la misma vista, la vista almacenada se renderiza una vez más.

  8. Si se solicita una nueva vista, se continúa con el proceso descrito en el Paso 2.

  9. A continuación, la nueva vista se devuelve como respuesta al cliente.
